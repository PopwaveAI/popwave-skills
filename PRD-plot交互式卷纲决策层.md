# PRD：全管线交互式创作决策层

> 版本：v0.2 | 日期：2026-07-22
> 状态：方案讨论稿
> 作者：pop
> 变更：v0.1只覆盖plot，v0.2扩展到seed→world→character→plot全管线

## 一、问题诊断

### 1.1 现状

当前管线Phase 0有一个"深问四层"采集用户意图，然后Phase 1-4全部是agent自主决策：

```
Phase 0: 深问（交互）→ 用户意图.md
Phase 1: seed → 骨架.md + 创意.md + 主角设计.md + ch001.txt（全自主）
Phase 2: seed → 主角设计.md（全自主）
Phase 3: world → 全书设定/（全自主）
Phase 3.5: character → 角色库.md（全自主）
Phase 4: plot → 卷纲.md + 章锚点表.md（全自主）
Phase 5-6: write↔review循环（全自主）
```

用户在Phase 0说了"我想写DND武僧登神"，然后全程被排除在创作决策之外。直到Phase 4产出40章卷纲，用户才能提修改意见——但此时底层骨架/世界/角色都定了，调整空间很有限。

### 1.2 系统性问题

| Phase | Agent自主决策的内容 | 用户不可控的损失 |
|:--|:--|:--|
| **Phase 1 seed** | 力量体系类型（7阶武道？修仙境界？游戏系统？）+动力引擎范式（探索型？压迫型？）+金手指设计（三件套？单件？）+主角性格底色 | 力量体系定了就不能改，如果用户不喜欢"7阶武道"而想要"修仙境界"，已经晚了 |
| **Phase 2 seed** | 主角核心欲望+恐惧+底线+爽感矛盾公式的权重分配 | 主角性格定了40章就定了，用户可能想要"腹黑谋略型"但agent给了"热血正面型" |
| **Phase 3 world** | 地图区域划分+势力4层结构+危机三层面+神系设计 | 世界结构定了plot的空间就定了，用户可能想要"多城游击"但world给了"单城据点" |
| **Phase 3.5 character** | NPC roster（谁出场谁不出场）+反派设计+NPC关系网+引入章/退场卷 | 角色库定了plot的Boss和配角就定了，用户可能想要某个特定类型的反派但character没设计 |
| **Phase 4 plot** | 卷终点+Boss+爽感优先级+节奏参考+NPC登场顺序 | 已经在v0.1 PRD中分析 |

### 1.3 根因

**根因不是"某个skill缺交互"，而是"管线设计哲学是'agent全权决策'而非'agent+用户协作决策'"**。

Phase 0的深问采集的是"用户想写什么类型"（高维抽象），但Phase 1-4需要的是"用户想要什么具体设计"（低维具体）。中间没有交互层衔接——agent从高维意图直接跳到低维设计，命中率全靠猜。

### 1.4 对标

RPG游戏的设计流程不是"策划定一切"，而是：
- 核心系统设计师定方向 → 主策划审批 → 关卡设计师定关卡 → 主策划审批 → 细节填充
- 每一层都有"决策→确认"的交互

当前管线缺少的就是"决策→确认"环节——agent做了决策但没有向用户确认。

## 二、底牌盘点：每个Phase开始前有什么

### 2.1 全管线底牌流转图

```
Phase 0                    Phase 1                  Phase 2                  Phase 3                 Phase 3.5               Phase 4
深问(交互)                  seed                    seed                    world                   character               plot
    ↓                        ↓                       ↓                       ↓                       ↓                       ↓
用户意图.md ──────────→ 骨架.md ──────────────→ 主角设计.md ──────→ 全书设定/ ──────→ 角色库.md ──────→ 卷纲.md
                        创意.md                                          世界圣经.md              ↓                    章锚点表.md
                        ch001.txt                                                              ↓
                                                                                          ↓
                    ┌──────────────────────────────────────────────────────────────┐
                    │                    各Phase的底牌来源                            │
                    └──────────────────────────────────────────────────────────────┘
```

### 2.2 各Phase底牌清单

| Phase | 开始前的底牌 | 硬约束（不可变） | 素材池（可选消费） | 空白（需要交互补全） |
|:--|:--|:--|:--|:--|
| **Phase 1 seed** | 用户意图.md + research产出 | 赛道类型（DND西幻/修仙/末日等） | decon-lite 9表 + 写作燃料 + 赛道调研 | 力量体系选择 / 引擎范式 / 金手指设计方向 / 主角性格底色 |
| **Phase 2 seed** | 骨架.md + 创意.md | 骨架自洽10项结论 | decon-lite表1（力量体系）+表9（动力引擎） | 主角核心欲望 / 恐惧 / 底线 / 爽感权重 |
| **Phase 3 world** | 骨架.md + 主角设计.md + ch001.txt | 力量体系5层 + 引擎6组成 + 金手指限制 | decon-lite表6（势力）+ 全书设定参考 | 地图布局偏好 / 势力结构偏好 / 危机类型偏好 |
| **Phase 3.5 character** | 全书设定/ + 世界圣经.md | 地图区域 + 势力分层 + 坐标系阶位 | decon-lite表6（势力中的NPC） | NPC roster偏好 / 反派类型偏好 / 关系网设计 |
| **Phase 4 plot** | 骨架+主角+全书设定+角色库 | 阶位区间 + 引擎节点 + 角色清单 | 写作燃料 + decon-lite表2（节奏） | 卷终点 / Boss / 爽感优先级 / 节奏参考 / NPC登场顺序 |

### 2.3 关键洞察

**越靠前的Phase，交互的价值越大**——因为前Phase的决策约束后Phase的选项空间。

- Phase 1的"力量体系选择"如果交互确认，后续Phase 2-4都在确认的体系内生成，不需要猜
- Phase 3的"地图布局"如果交互确认，Phase 4的"分幕走向"就在确认的地图上设计
- 如果Phase 1-3.5都是agent自主决策，到Phase 4时交互的可调空间已经被压缩到最小

**因此：交互层应该从Phase 4向前延伸到Phase 1**，在最有杠杆的位置采集用户决策。

## 三、设计原则

### 3.1 何时交互、何时自动

| 决策类型 | 交互还是自动 | 原因 |
|:--|:--|:--|
| **方向性决策**（力量体系类型/引擎范式/金手指类型/主角性格底色） | **交互** | 用户有明确偏好，agent猜不中 |
| **结构化决策**（地图区域数/势力层数/NPC数量/分幕数） | **交互** | 影响后续Phase选项空间，用户可能有想法 |
| **填充型决策**（具体区域名/具体势力名/具体NPC性格细节） | **自动** | 用户通常没有偏好，agent基于设定生成即可 |
| **验证型决策**（骨架自洽检查/世界观一致性/角色库完整性） | **自动** | 有客观标准，不需要用户判断 |
| **质量型决策**（爽感公式权重/精彩度五问/节奏密度） | **交互** | 用户偏好直接影响质量，agent默认值可能不合用户口味 |

### 3.2 交互格式原则

1. **给选项不给空白**——每轮提供2-3个结构化选项+1个自定义入口，不是开放式提问
2. **选项基于底牌**——选项必须基于已有设定生成，不是凭空想象
3. **标注影响**——每个选项标注"选这个会影响后续什么"
4. **核心轮必选、细节轮可选**——用户可以跳过细节轮，agent按默认值生成

### 3.3 交互密度控制

| Phase | 交互轮次 | 核心/可选 | 预估耗时 |
|:--|:--|:--|:--|
| Phase 1 seed | 3轮 | 全核心 | 3-5分钟 |
| Phase 2 seed | 2轮 | 1核心+1可选 | 2-3分钟 |
| Phase 3 world | 2轮 | 1核心+1可选 | 2-3分钟 |
| Phase 3.5 character | 2轮 | 1核心+1可选 | 2-3分钟 |
| Phase 4 plot | 5轮 | 前3核心+后2可选 | 5-8分钟 |
| **合计** | **14轮** | **9核心+5可选** | **14-22分钟** |

用户如果只回答核心轮（9轮），总交互时间约10-14分钟。可选轮可以跳过。

## 四、各Phase交互设计

### 4.1 Phase 1 seed：3轮核心交互

#### S1 力量体系+引擎范式

**底牌消费**：用户意图.md（赛道类型）+ research-写作燃料.md（事件形状）+ decon-lite表1+表9

**Agent动作**：
- 基于赛道类型，提出2-3个力量体系方案
- 每个方案标注：主养成线（几阶？什么名称？）+ 子养成线（几条？）+ 引擎范式（探索/压迫/混合）

**交互示例**：
```
基于你的赛道（DND西幻·动荡之年），提供3个力量体系方案：

A) 7阶武道阶位（D&D面板流）
   → 主养成线：7阶（见习→云游→宗匠→传奇→半神→真神→神系之主）
   → 子养成线：5条（气功/武技/禅定/专长/炼体）
   → 引擎范式：混合型（前3卷探索，后4卷压迫）
   → 参考书：《深渊主宰》

B) 9境修武（修仙面板流）
   → 主养成线：9境（练气→筑基→金丹→元婴→化神→...）
   → 子养成线：3条（功法/法宝/神通）
   → 引擎范式：探索型
   → 参考书：《遮天》

C) 游戏系统（数据面板流）
   → 主养成线：等级制（LV1-100）
   → 子养成线：4条（技能树/装备/天赋/职业）
   → 引擎范式：混合型
   → 参考书：《全职高手》

请选择，或描述你想要的力量体系。
```

**影响标注**：力量体系定下后，后续所有Phase都在此体系内生成。改力量体系=重做Phase 1-4。

#### S2 金手指设计方向

**底牌消费**：S1选择的力量体系 + decon-lite表7（金手指）+ 主角设计模板

**Agent动作**：
- 基于力量体系，提出2-3个金手指方案
- 每个方案标注：类型/加速机制/限制/代价/与引擎的关系

**交互示例**：
```
基于7阶武道阶位，提供3个金手指方案：

A) 重生信息差+系统面板+高天赋（三件套）
   → 加速：经验可视化+最优路径+时间预知
   → 限制：不能替代修炼本身+蝴蝶效应+感知过载
   → 代价：暴露风险+戒律反噬
   → 钥匙非生成器：面板显示进度但不替代战斗

B) 纯系统面板（单件）
   → 加速：经验可视化+武技熟练度系统
   → 限制：面板只在战斗中激活+冷却时间
   → 代价：面板过载时头痛
   → 钥匙非生成器：同上

C) 血脉觉醒（天赋型）
   → 加速：血脉中封印的远古武技记忆
   → 限制：觉醒需要特定触发条件
   → 代价：每次觉醒消耗生命力
   → 钥匙非生成器：血脉给的是记忆不是力量

请选择，或描述你想要的金手指。
```

**影响标注**：金手指定下后，主角设计.md的爽感公式和write的面板弹出逻辑都基于此。

#### S3 主角性格底色

**底牌消费**：S2选择的金手指 + 用户意图.md（风格关键词）

**Agent动作**：
- 提出2-3个主角性格方案
- 每个方案标注：核心欲望/恐惧/底线/行动偏好/长篇穿透方式

**交互示例**：
```
基于"爽文+史诗"风格，提供3个主角性格方案：

A) 冷静务实型（守序中立偏邪恶）
   → 欲望：绝对自由意志——不再被任何力量摆布
   → 恐惧：再次被AO扫回原点
   → 底线：武道戒律（不杀无力反抗者/不违约/不欺弱小）
   → 穿透方式：武力碾压——打不过就发育，发育完再打回来
   → 参考人物：深渊主宰罗德（但走正面硬刚路线）

B) 热血正义型（混乱善良）
   → 欲望：保护重要的人，变强是为了不再失去
   → 恐惧：无力保护
   → 底线：不杀无辜
   → 穿透方式：越战越强——每次失败都是突破契机
   → 参考人物：路飞

C) 腹黑谋略型（守序邪恶）
   → 欲望：掌控一切，让所有人成为棋子
   → 恐惧：失去控制
   → 底线：不违约（但会钻空子）
   → 穿透方式：借刀杀人+布局——实力不够就用计谋补
   → 参考人物：夜神月

请选择，或描述你想要的主角性格。
```

**影响标注**：主角性格定下后，角色弧光/对话风格/决策模式全部基于此。

### 4.2 Phase 2 seed：2轮交互（1核心+1可选）

#### S4 主角核心欲望细化（核心）

**底牌消费**：S3选择的主角性格 + 骨架.md

**Agent动作**：
- 基于性格底色，提出2-3个核心欲望的具体表达方案
- 每个方案标注：9字段行为模式的差异

**交互示例**：
```
基于"冷静务实型"，核心欲望"绝对自由意志"可以有不同的表达：

A) 孤狼登顶型
   → 不依赖任何人，独行变强

C) 隐忍布局型
   → 表面低调，暗中布局
   → 第一卷变化：从"隐藏实力"→被迫暴露→学会在暴露中收益
```

## 五、research补强：超越用户预期

### 5.1 问题

交互决策解决了"命中用户诉求"，但没有解决"超越用户预期"——用户选了Boss和爽感类型后，agent生成的内容可能还是平庸的。

### 5.2 需要补强的research维度

| 维度 | 当前有 | 需要补强 | 消费者 |

### 7.3 交互方式

**推荐同步交互**——pipeline agent在每个Phase开始时，逐轮输出选项，等用户选择后再进入下一轮。核心轮必答，可选轮可跳过。

Pipeline需要支持"暂停等待用户输入"——Phase执行到交互轮时，输出选项后暂停，等用户回复后继续。

### 7.4 实施顺序

1. **先做plot**（P0，已有v0.1详细设计）——验证交互式决策的效果
2. **再做seed**（P0）——向上延伸到最有杠杆的位置
3. **然后character**（P0）——NPC roster是用户最关心的
4. **然后world**（P0）——世界结构影响plot空间
5. **最后research**（P1）——decon-plot增强选项质量

## 八、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|:--|:--|:--|:--|
| 用户觉得14轮太多 | 高 | 用户放弃交互 | 核心轮9轮必答，可选轮5轮可跳过。核心轮总时间10-14分钟 |
| pipeline不支持暂停等待 | 中 | 交互无法实现 | Phase执行到交互轮时，agent输出选项后结束本轮，用户回复后触发下一轮 |
| 用户自定义输入太模糊 | 中 | agent无法执行 | 选项中提供"自定义"入口，agent追问1轮澄清 |
| 交互决策与设定冲突 | 低 | 用户选了不可行的方案 | agent生成选项时已在硬约束内，不可行的不会出现在选项中 |
| 前Phase的决策限制后Phase的选项 | 中 | 后Phase选项空间被压缩 | 这正是设计目的——前Phase决策约束后Phase选项，保证全链路一致 |
