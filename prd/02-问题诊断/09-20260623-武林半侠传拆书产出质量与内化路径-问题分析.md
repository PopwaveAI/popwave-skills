## 一、Gap 1：剧情线拆解 — "线"的单位太薄，承载不了厚重的剧情单元

### 1.1 现状

当前剧情线产出分两层：

| 层 | 产出 | 数量 | 深度 |
|:---|:-----|:-----|:-----|
| 卷纲层 | `设计/卷/卷1.md` §四 | 4条线（武道/金手指/势力/古龙） | 每线5-7阶段，符合规范下限 |
| 独立深度层 | `剧情设计/剧情线/` | 7+1文件 | 每线12节点+咬合网络+债务管理，远超规范 |

独立深度层的质量很高——每条线有冲突引擎（三层）、读者追问链（4级）、剧情发展骨架（9-12节点）、线间咬合表、叙事债务管理。**但老板指出的问题不在产出质量，在于"线"这个单位本身的表达力不够。**

### 1.2 Gap 诊断："线"太薄

老板的洞察是对的。这本书有**主世界剧情 + 副本世界剧情**双轨结构，理论上有非常丰富的线可以拆：

**当前"线"的问题：**

| 问题 | 说明 |
|:-----|:-----|
| **线是"轨迹"不是"单元"** | 当前剧情线记录的是"主角从A到B的发展轨迹"（12个节点串成一条线），但读者感受到的"厚重剧情"是一个个**完整的剧情单元**——有起承转合、有情感投入、有主题表达的故事块 |
| **线是"连续"的，但剧情是"块状"的** | 线的模型假设剧情是连续推进的，但实际网文的节奏是"单元剧"式的——每个副本/每个事件弧是一个相对独立的故事块，块与块之间有过渡但内核独立 |
| **线缺少"厚度"维度** | 一条线只记录"发生了什么"，不记录"这个剧情单元为什么好"——它的情感设计、主题表达、节奏控制、读者预期管理 |
| **线不能表达"嵌套"关系** | 主世界线和副本世界线不是平行的，是**嵌套**的——主角在副本世界的经历反哺主世界的势力格局，这种嵌套关系用"线"无法表达 |

**老板说的"薄弱"本质是：** 剧情线拆的是"轨迹"（what happened），不是"单元"（why it's good）。最有价值的是后者。

### 1.3 缺失的产出：剧情单元卡

`pop-decon-volume` 的 Step 5 定义了"剧情单元提取"，产出剧情单元卡入库 `pop-trope-library/剧情库/`。但当前项目：

| 应产出 | 实际 |
|:-------|:-----|
| 剧情单元卡（事件链+结构解析+金句+复用要点） | ❌ 未产出 |
| 入库 `pop-trope-library/剧情库/` | ❌ 未入库 |

**剧情单元卡才是"厚重剧情"的正确表达单位。** 它应该回答：
- 这个剧情单元的**结构**是什么（起承转合、铺垫-爆发-余波）
- 它的**情感设计**是什么（读者在这个单元里经历了什么情绪弧线）
- 它的**主题表达**是什么（这个单元在讲什么）
- 它的**节奏控制**是什么（快慢交替、张弛有度怎么做的）
- 它的**复用要点**是什么（拆给别人写时怎么用）

### 1.4 其他问题

| 问题 | 严重度 | 说明 |
|:-----|:------:|:-----|
| 副本世界收割循环线文件缺失 | 中 | `主角行动-02-副本世界收割循环线.md` 被其他线引用但文件不存在 |
| 产出路径偏离规范 | 中 | 独立深度文件在 `剧情设计/剧情线/`，规范目录树无此路径，下游 skill 可能找不到 |
| 卷纲与独立文件粒度不一致 | 低 | 卷纲4条线 vs 独立文件7条线，"势力线"在卷纲是1条，独立文件拆成2条 |

### 1.5 修复方向

| 修复项 | 做法 | 优先级 |
|:-------|:-----|:------:|
| **新增剧情单元卡产出** | pop-decon-volume Step 5 真正执行——从设计包事件链提取剧情单元卡（结构+情感+主题+节奏+复用），入库 `pop-trope-library/剧情库/` | P0 |
| **升级"线"模型为"线+单元"双层** | 线保留"轨迹追踪"职责（宏观），新增单元卡承载"厚度分析"职责（微观） | P0 |
| **修复缺失文件** | 补产出 `副本世界收割循环线.md` | P1 |
| **规范产出路径** | 独立深度文件纳入规范目录树，或建立索引文件 | P2 |

## 二、Gap 2：设定拆解 — 精巧点未提炼为可复用的设计模式

### 2.1 现状

设定产出质量很高——9个文件齐全，证据链扎实，世界宪法20条规律提炼到位。但问题不在产出质量，在于**设定拆解止步于"记录"，没有走到"提炼可复用设计模式"**。

### 2.2 Gap 诊断：记录了"是什么"，没提炼"为什么好"和"怎么用"

老板关心的三个点：

| 老板关心 | 当前产出 | Gap |
|:---------|:---------|:----|
| **如何塑造危机** | 世界宪法第15条"绝境突破法则"、combat_capability.yaml 的 T0-T4 对手梯度 | ✅ 记录了"有绝境突破"和"有对手梯度"，❌ 但没提炼"如何用设定塑造危机"的设计模式 |
| **如何让主角合理变强** | L1-02力量体系（境界链+功法流派+金手指体系）、世界宪法第3-5条（升级法则） | ✅ 记录了"主角怎么变强的"，❌ 但没提炼"如何让变强合理"的设计模式 |
| **如何加强代入感** | L1-01世界蓝图（地理+位面+时间规则）、L1-05势力格局 | ✅ 记录了"世界长什么样"，❌ 但没提炼"如何用设定加强代入感"的设计模式 |

**本质问题：** 设定拆解产出的是"世界观百科"（what），不是"设计模式手册"（how）。前者是参考文档，后者才是能反哺写作的操作指南。

### 2.3 缺失的产出：设定设计模式卡

应从 L1 设定中提炼设计模式卡，入库 `pop-trope-library/设定库/`。例如：

| 设计模式 | 来源 | 可复用要点 |
|:---------|:-----|:---------|
| **有代价的金手指** | 演武令血祭机制 | 力量获取必须绑定牺牲机制，让每次使用都是叙事决策点 |
| **跨体系突破链** | 易筋经+太玄经+坐忘心经三经联动 | 不同来源的功法可以组成突破阶梯，每步需要不同条件 |
| **双世界时间红利** | 副本世界修炼→回归主世界兑现 | 用时间差制造"悄悄变强"的爽感，同时不破坏主世界节奏 |
| **绝境-突破-兑现三拍子** | 世界宪法第15条 | 危机→突破→收割的循环结构，每轮抬高一级对手 |
| **势力渗透式扩张** | 双身份架构（方明/岳鹏/神锋楼主） | 主角通过身份切换渗透多方势力，比直接打天下更有多线叙事空间 |

### 2.4 入库状态

| 应入库 | 实际 |
|:-------|:-----|
| `pop-trope-library/设定库/` 整本 L1 设定 | ❌ 未入库 |
| `pop-trope-library/设定库/` 设计模式卡 | ❌ 未产出 |

### 2.5 修复方向

| 修复项 | 做法 | 优先级 |
|:-------|:-----|:------:|
| **设定入库** | 将 L1-01~06 + 世界宪法 + PRD.md 整本入库 `pop-trope-library/设定库/` | P0 |
| **提炼设定设计模式卡** | 从 L1 设定中提炼"如何塑造危机/如何让变强合理/如何加强代入感"的设计模式卡 | P0 |

## 四、根因总结：内化路径全面断裂

三个 Gap 表面是不同维度的问题，根因是同一个：

```
拆书产出（✅ 质量不错）
    │
    ├──→ 剧情线：线太薄，缺剧情单元卡 ──→ 未入库 剧情库
    ├──→ 设定：止步于记录，缺设计模式 ──→ 未入库 设定库
    └──→ 采风：止步于分析，缺SOP转化 ──→ 未入库 立项库
                                          │
                                          └──→ 写作管线无法消费
```

**拆书的两个目的都没完全达成：**

| 目的 | 达成度 | Gap |
|:-----|:------:|:----|
| 1. 层层拆解看人家怎么写、为什么写的好 | 60% | 拆了"是什么"（设计包/设定/创意溯源），但"为什么好"的深度分析不足（剧情单元卡缺失、设计模式未提炼） |
| 2. 有价值的内化到 library | 10% | 300+套路卡 + 9个设定文件 + 7个创意溯源文件全部未入库 pop-trope-library，写作管线无法消费 |

## 五、修复方案

### 5.1 优先级排序

| 优先级 | 修复项 | 维度 | 预期效果 |
|:------:|:-------|:-----|:---------|
| **P0** | 执行入库流程 | 全部 | 将300+套路卡 + L1设定 + 创意溯源入库 pop-trope-library 对应模块 |
| **P0** | 产出剧情单元卡 | 剧情线 | 从设计包事件链提取剧情单元卡（结构+情感+主题+节奏+复用），入库剧情库 |
| **P0** | 执行 SOP 转化 | 采风 | 按4步流程产出融合模式汇总，入库立项库 |
| **P0** | 提炼设定设计模式卡 | 设定 | 从L1设定提炼"如何塑造危机/如何让变强合理/如何加强代入感"的模式卡 |
| **P1** | 升级"线"模型 | 剧情线 | 线保留轨迹追踪（宏观），单元卡承载厚度分析（微观），形成双层模型 |
| **P1** | 建仓 pop-decon-trace | 采风 | 将 Phase 4 标准化为独立 skill |
| **P2** | 修复缺失文件 | 剧情线 | 补产出副本世界收割循环线 |
| **P2** | 模板结构对齐 | 设定 | L1-02 和 combat_capability.yaml 按模板重排 |

### 5.2 "线+单元"双层模型设计

```
宏观层（线）：
  主角成长线 ── 主角行动线 ── 世界事件线 ── 支线
  职责：追踪"轨迹"——谁在哪里做了什么，从A到B的发展路径
  粒度：卷级/幕级
  产出：剧情线文件（当前已有，保留）

微观层（单元）：
  ┌──────────────────────────────────────┐
  │ 剧情单元卡                              │
  │ ├── 结构：起承转合（铺垫-爆发-余波）      │
  │ ├── 情感：读者情绪弧线（期待-紧张-释放）   │
  │ ├── 主题：这个单元在讲什么                │
  │ ├── 节奏：快慢交替、张弛有度怎么做的       │
  │ ├── 复用：拆给别人写时怎么用              │
  │ └── 关联：属于哪条线 / 与哪些单元咬合      │
  └──────────────────────────────────────┘
  职责：分析"厚度"——这个剧情单元为什么好
  粒度：章级/事件弧级
  产出：剧情单元卡（当前缺失，需新增）
```

> **根因一句话：** 拆书产出本身质量不错（设计包150/150、设定9文件、创意溯源7文件），但内化路径全面断裂——300+套路卡、9个设定文件、7个创意溯源文件全部未入库 pop-trope-library，写作管线无法消费。剧情线维度缺"剧情单元卡"（线太薄），设定维度缺"设计模式卡"（止步于记录），采风维度缺"SOP转化"（止步于分析）。三个 Gap 的共同根因是拆书管线只设计了"产出"环节，没有设计"内化入库"环节的执行闭环。
