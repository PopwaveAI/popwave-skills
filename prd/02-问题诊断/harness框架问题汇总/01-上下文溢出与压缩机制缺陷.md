# PRD-01: 上下文溢出与压缩机制缺陷

> **严重级别：** P0 / 严重
> **来源文档：** 01, 03, 04, 08, 10, 17
> **最后更新：** 2026-07-01

---

## 问题概述

OpenClaw 的上下文管理架构存在系统性缺陷——Truncation（截断）与 Compaction（压缩）触发条件互斥导致信息被永久丢弃而非压缩保留；toolCall payload 无生命周期管理导致链式累积膨胀；overflow 时硬拦截而非降级处理。这是被诊断次数最多、影响最严重的框架层问题。

---

## 根因分析

### 1. Truncation 替代 Compaction

Layer 1（Truncation 每轮预截断）把上下文压回安全线内，使 Layer 2（Compaction 溢出被动触发）的触发条件永远不满足。三层 compaction 触发条件均未满足：

- **overflow 触发**——truncation 阻止了 overflow
- **超时 + tokenUsedRatio > 0.65**——truncation 压制了使用率
- **后台 compaction**——`openclaw.json` 未配置 `turnMaintenanceMode: "background"`

### 2. toolCall payload 链式累积

write 类 toolCall 的 `input.content`（文件全部内容）被完整存入对话历史，每轮累积无清理。compaction 只压缩自然语言对话，不碰 toolCall payload，导致压缩空转。

### 3. overflow 硬拦截

overflow 时完全拒绝请求（非截断非降级），agent 只看到 668 字节 "prompt too large"，完全失忆。

## 证据链

### 文档10（10-1）

- 41 个 run 的 `events.jsonl` 显示 promptTokens 14.6%→56.6% 从未溢出 900K
- 42.9% 时 truncation 开始截断
- 全部 41 个 run 中 Compaction 列值全为 0
- openclaw 日志 61MB 搜索 `compact` 关键词返回 0 条实际执行记录
- 截断首次触发时间与设计包字数断崖（27,786→6,102）吻合

### 文档04（04-1 ~ 04-5）

- 6-12 项目a 15 次 run 中 5 次 overflow（33%）
- 6-14 项目a 24 次 run 中 19 次 overflow（79%）
- toolCall 累积路径：creative ~15KB → world ~63KB → plot ~111KB → chapter ~151KB → deconstructor >325KB 溢出
- 4 个 compaction 事件全部空摘要（"No prior history. None."），却占 37,379 字符（10% 预算）

### 文档03（03-A1 ~ A5）

- toolCall 膨胀占 72%（~563K chars）
- session 163320 回灌循环 6 次
- compaction 后上下文不减反增（19:38→61K, 19:43→103K, 净增 68%）

### 文档08（08-2）

- 14 轮对话历史注入，每轮含子 agent spawn 详情（~2K tokens）+ 结果 + 进度表格，接近 900K 上限

### 文档17（17-2）

- OpenClaw 有 6 项 context 管理机制（文件截断 2K行/50KB、bootstrap 60K、工具结果 16K、剪枝 50%+LLM 摘要、预压缩刷新、非破坏性剪枝），但 skill 文件注入不在覆盖范围内——压缩后不重注入 skill 文件，agent 拿到的是对话历史摘要而非 skill 原文

---

## 影响表现

- 设计包从 20K 字退化为 4-6K 字；正文从场景渲染退化为事件流水账
- PRD 定义的文风消失；角色行为一致性下降；伏笔追踪和跨章弧线断裂
- overflow 后所有创作进度归零，agent 无法继续
- 任何在 OpenClaw 上进行长会话操作（超过 ~10 轮工具调用）的 agent 都会遇到相同问题
