# PRD-04: Read工具截断

> **严重级别：** P1 / 高
> **来源文档：** 01, 03
> **最后更新：** 2026-07-01

---

## 问题概述

Paopao 平台 Read 工具有硬性字符截断上限（~2416 chars），远小于文档型文件实际体积，且为静默截断（不提示不完整）。agent 拿到残缺指令，关键输出指令（在文件后半段）丢失，导致执行不完整。

---

## 根因分析

平台 Read 工具返回最大字符数约 2416 chars，而 exec 工具 stdout 上限约 30000 chars。SKILL.md 文件通常 20-70K 字符，Read 只能读到约 11%。截断为静默截断——不提示内容不完整，agent 不知道自己拿到的是残缺数据。

---

## 证据链

### 文档01（01-1, 01-7）

- expert-writer v2.3.0 的 SKILL.md 为 21,298 字符，Read 只能读到 11.3%
- 6-9 测试项目全量 47 次 run 审计显示 Manual Read 截断率 15/17
- 截断率从 15/17 → 0/30（改用 Get-Content 后）
- ch011-014 五轮连续零次 Read 子 skill 文件，agent 切换到"全凭记忆写"模式

### 文档03（03-A10）

- 标注 P0，来源诊断报告 #3
- Hermes read_file 大文件模式解决

---

## 影响表现

- agent 拿到残缺指令，关键输出指令（在 step 子文件后半段）丢失，导致执行不完整
- 正文质量断崖
- 长会话中 agent 放弃 Read 改用记忆，形成惰性侵蚀
