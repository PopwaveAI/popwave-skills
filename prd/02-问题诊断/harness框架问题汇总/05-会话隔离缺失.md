# 会话隔离缺失 PRD：跨项目上下文泄露与子 agent 隔离失效

> **PRD · OpenClaw 阶段 · Framework**
> **P2 · 框架层缺陷** | session-isolation | context-leak | sub-agent
> Hermes 迁移后大部分已解决，本文作为框架层问题历史记录保留

---

## 1. 问题概述

OpenClaw 阶段会话隔离机制不完善——不同项目的上下文信息发生泄露，项目间上下文未隔离，子 agent 隔离失效导致子任务无法可靠分发执行。Hermes 迁移后大部分已解决，但作为框架层问题历史记录保留。

---

## 2. 根因分析

| # | 根因 | 说明 |
|:--|:-----|:-----|
| 1 | 跨项目信息泄露 | 平台会话隔离机制缺失，不同项目的上下文信息发生泄露，agent 引入其他项目设定污染当前产出。 |
| 2 | 会话隔离不彻底 | 会话间隔离不彻底导致上下文串扰。 |
| 3 | 子 agent 隔离失效 | OpenClaw 子 agent 调度 / 物化 / 隔离机制失效，6 次路由全部失败。 |

---

## 3. 证据链（按来源文档组织）

### 3.1 文档 01（01-5, 01-6）

- 不同项目上下文信息发生泄露；标注 **P2**。
- 修复路径：通过 expert-writer v3.0.0 删除所有项目名示例修复；Hermes Agent 迁移后彻底解决。

### 3.2 文档 03（03-A9）

- 6 次路由全部失败，子 agent 隔离失效；出自诊断报告 #4。
- 修复路径：Hermes 迁移后通过 `delegate_task` 解决，6 次路由全部成功。

---

## 4. 影响表现

- agent 引入其他项目设定，污染当前产出
- 子任务无法可靠分发执行
- 50 章自动化崩溃
