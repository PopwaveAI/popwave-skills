# 次要框架层问题 PRD：版本锁过时 / 令牌过期 / 异常会话

> **PRD · Framework · 杂项**
> **P3 · 框架层可靠性补充** | version-lock | token-expiry | anomalous-session
> 单个影响较小，作为框架层可靠性问题的补充记录

## 1. 问题概述

从诊断文档中提取的 3 个次要框架层问题——skill 版本锁过时、AI 访问令牌过期、极短异常会话。单个影响较小，但作为框架层可靠性问题的补充记录。

---

## 2. 问题 1：skill 版本锁与实际版本不一致

### 2.1 问题现象

`config.json` 的 `lockedVersions` 与实际安装版本不一致——pop-decon 锁 v13.1.0 但实际 v14.1.0，design-pack 锁 v1.3.0 但实际 v3.3.0。

### 2.2 根因分析

skill 加载保障机制（版本锁）未与实际安装版本同步，版本锁可能干扰远程 skill 加载流程。本地 skill 路径优先使得实际仍用最新版，但版本不一致存在风险。

### 2.3 证据链

- 文档 08 第四节次要问题：pop-decon 锁 v13.1.0，实际 v14.1.0；design-pack 锁 v1.3.0，实际 v3.3.0。

### 2.4 影响表现

- 严重度低。
- 当前因本地路径优先未造成实际故障，但版本锁可能在远程加载场景下导致 skill 版本回退或加载失败。

---

## 3. 问题 2：AI 访问令牌过期导致 Run 完全失败

### 3.1 问题现象

两次 run 因 401 AI 访问令牌无效或已过期而完全失败（OutKB=0），重试机制增加上下文轮次，间接加速 truncation 触发。

### 3.2 根因分析

API 认证令牌过期属平台层可靠性问题；框架的错误重试机制在非 overflow 错误时仍会增加上下文轮次。

### 3.3 证据链

- 文档 10 第 2.5 节：
  - Run `5c0a8449`（23:06）：401 令牌无效，OutKB=0。
  - Run `b58a91bb`（23:10）：401 令牌过期，OutKB=0。

### 3.4 影响表现

- 两次 Run 完全失败无产出。
- 重试消耗上下文预算，间接加速 truncation 触发。

---

## 4. 问题 3：极短异常会话（API 超时或 agent crash 后自动恢复）

### 4.1 问题现象

`conversations` 目录中存在一个仅 0.4KB 的极短会话文件（`49498a22.jsonl`），可能是 API 超时或 agent crash 后自动恢复产生的空会话残留。

### 4.2 根因分析

框架层在 API 超时或 agent crash 时触发了自动恢复，但恢复过程产生了无实质内容的极短会话残留，说明 completion 超时 / 崩溃恢复机制存在异常边界情况。

### 4.3 证据链

- 文档 12 第五节问题 4：`49498a22.jsonl` 仅 0.4KB（对比主会话 `77c8af26.jsonl` 1571KB）。

### 4.4 影响表现

- 无实质影响，但说明执行过程中有一次异常重启。
