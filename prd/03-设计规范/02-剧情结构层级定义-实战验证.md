**可迁移要点：**
- 结构模式：副本世界 = 困境→获得→修炼→突破→因果→回归，六拍子完整弧线
- 情感设计：绝望→狂喜→沉淀→震撼→满足，完整的情绪周期
- 节奏控制：快（穿越）→急转（获得）→慢（修炼）→爆发（突破）→收束（规划）
- 嵌套策略：副本内部有独立成长线+因果线+身份线，但都服务于主世界的力量跃迁

## 二、验证结论

### 2.1 四层定义可落地

| 层级 | 数据来源 | 提取难度 | 可落地 | 说明 |
|:-----|:---------|:---------|:------:|:-----|
| L1 事件 | 设计包事件链表格 | 零（已有） | ✅ | 设计包每行即一个事件 |
| L2 剧情单元 | 跨章设计包合成 | 中（需跨章模式识别） | ✅ | 需读多章设计包，识别边界信号（穿越入口/出口、地点切换、力量跃迁） |
| L3 剧情线 | 跨单元追踪 | 中高（需跨单元模式识别） | ✅ | 需从多个单元中追踪同一主题的演进 |
| L4 全书事件 | 跨卷线交汇 | 高（需足够数据窗口） | ⚠️ | 150章/27%可识别候选，但无法确认完整形态 |

### 2.2 设计包数据足够支撑四层提取

设计包提供了提取所需的全部关键信号：

| 需要的信号 | 设计包中的来源 | 示例 |
|:-----------|:-------------|:-----|
| 事件边界 | 事件链表格的「类型」列 | "穿越·起"/"穿越·选择"/"突破·先天" |
| 单元边界 | 章末钩子的「回收章」+ 事件类型突变 | ch098 穿越·起 → ch110 回归主世界 |
| 线索追踪 | 角色行为锚定表 + 价值点分流 | "实力提升"/"势力扩张"/"世界观深化" |
| 力量跃迁 | 事件链的武功类事件 + 角色状态 | ch100 突破先天 → ch142 领悟神元 |
| 因果链 | 章末钩子 + 价值点分流的「未来回收」 | ch098 易筋经伏笔 → ch100 易筋经大成 |
| 嵌套关系 | 场景切换 + POV 变化 | ch142 从枫林（副本余波）→万仞城（主世界） |

### 2.3 当前 skill 的提取路径问题

**当前路径（Step 3）：** 设计包事件 → 直接聚类为剧情线

**问题：** 跳过了 L2 单元层，从 150 章的原子事件直接聚类为线，中间没有"分组"步骤。这就是为什么线感觉"薄"——它试图追踪一条线跨 150 章的原始事件，没有中间的单元层来提供结构。

**正确路径应为：**

```
设计包事件链（L1）
    ↓ 跨章模式识别（识别边界信号）
L2 剧情单元（有起承转合的完整故事块）
    ↓ 跨单元主题追踪（追踪同一主题的演进）
L3 剧情线（跨单元的纵向追踪）
    ↓ 跨线交汇识别（识别多条线的交叉点）
L4 全书事件（跨卷的大事件）
```

每一层都需要上一层作为输入，不能跳过。

### 2.4 L2 单元是当前最关键的缺失层

| 层级 | 当前状态 | 缺失影响 |
|:-----|:---------|:---------|
| L1 | ✅ 设计包已有 | — |
| **L2** | **❌ 完全缺失** | **线失去结构支撑——没有"单元"作为中间层，线只能从原子事件直接聚类，导致线太薄** |
| L3 | ⚠️ 有但薄 | 因为跳过 L2，线的阶段推进缺乏单元级别的结构支撑 |
| L4 | ❌ 缺失 | 无法表达跨卷大事件 |

**L2 单元层的核心价值：**
1. 为 L3 线提供结构支撑——线追踪的是"单元之间的演进"，不是"事件之间的演进"
2. 承载"厚度分析"——每个单元有独立的情感弧线、主题表达、节奏控制
3. 表达嵌套关系——副本世界是一个单元，内部有多条子线
4. 提供入库粒度——可迁移单元卡是剧情库的正确入库对象

## 三、实战中暴露的问题

### 3.1 单元边界识别需要跨章阅读

单章设计包无法判断单元边界。ch098 单独看只是一次穿越，但结合 ch110 回归主世界才能确认 ch098-ch109 是一个完整的副本单元。

**对 skill 的影响：** L2 提取不能在逐章设计包产出时做，必须在 Phase 2（volume）跨章分析时做。当前 Step 3 反推剧情线已经需要跨章分析，L2 提取应该在 Step 3 之前或同时进行。

### 3.2 嵌套关系需要"子线"概念

副本世界（L2 单元）内部的成长线、因果线、身份线，从主世界看是"子线"——它们在单元内部有完整的起承转合，但单元结束后就结束了（或转化为主世界线的推进）。

**对定义的影响：** L3 线需要区分"主线"（跨单元追踪）和"子线"（单元内部追踪）。子线有边界，主线可以贯穿全书。

### 3.3 L4 全书事件在数据窗口不足时只能标注"候选"

150 章/27% 的数据窗口无法确认演武令终极秘密的完整形态。L4 识别需要标注置信度——"候选"vs"确认"。

**对 skill 的影响：** L4 提取应该在全书拆解时做，前 N 章只做"候选识别"。

### 3.4 设计包的「价值点分流」字段是 L2/L3 提取的金矿

设计包第六节「价值点分流」标注了每章的价值流向（实力提升/势力扩张/世界观深化/反派塑造/控制手段），这些标签直接对应 L3 线的分类。当前 skill 没有利用这个字段做线追踪——这是一个低成本高收益的改进点。

## 四、对概念设计的修订建议

基于实战验证，对 `01-剧情结构层级定义-概念设计.md` 的修订：

| 修订项 | 原设计 | 修订为 | 理由 |
|:-------|:-------|:-------|:-----|
| L2 单元提取时机 | 未明确 | Phase 2 Step 2.5（幕边界识别后、剧情线反推前） | 单元边界需要跨章识别，但不需要等到全书读完 |
| L3 线分类 | 自定义 | 利用设计包「价值点分流」字段做初始分类 | 低成本高收益，已有数据 |
| L3 线区分子线/主线 | 未区分 | 增加"子线"概念——单元内部的线有边界，主线贯穿全书 | 嵌套关系需要 |
| L4 置信度 | 未定义 | "候选"vs"确认"两级 | 前 N 章只能做候选识别 |
| 提取路径 | 未明确 | L1→L2→L3→L4 逐层提取，不可跳过 | 跳过 L2 是当前线太薄的根因 |

---

## 五、结论

**四层定义可落地。** 设计包数据足够支撑四层提取，关键缺失是 L2 单元层——当前 skill 从 L1 事件直接跳到 L3 线，跳过了 L2，导致线失去结构支撑。

**实战拆解证明了概念设计的核心价值：**
- L2 单元能表达副本世界的嵌套结构（单元 B 笑傲副本内部有 3 条子线）
- L3 线在 L2 支撑下变得有结构（线的阶段推进对应单元的演进，不是原子事件的串联）
- L4 全书事件能容纳跨卷大事件（演武令终极秘密不是一条线，是多条线的交汇）

**下一步：** 将四层定义落地到 pop-decon-volume skill 规范，新增 Step 2.5（L2 单元提取），改造 Step 3（L3 线基于 L2 追踪而非直接从事件聚类），新增 Step 7（L4 全书事件识别）。

> **实战一句话：** 从 6 章设计包中成功拆出 3 个 L2 单元、4 条 L3 线、1 个 L4 候选全书事件。四层定义可落地，关键缺失是 L2 单元层——补上这一层，线就有了结构支撑，不再"薄"。
