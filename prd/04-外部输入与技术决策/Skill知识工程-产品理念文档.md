## 1. 核心理念：什么样的知识才能变成好 Skill

### 1.1 一句话总结

> **源材料的「抽象层级」决定 Skill 的复用质量。教「方法」的能变好 Skill，教「故事」的只能变摘要。**

### 1.2 「换皮测试」

把源材料里所有项目名、公司名、具体数据、技术栈名字全部抹掉。然后看看剩下的部分，读者还能不能从中学到一套可以直接用的方法。

| 结果 | 结论 |
|:-----|:-----|
| 能 | 源材料的「骨架」是硬的，转出来的 Skill 会很好用 |
| 不能 | 源材料的价值和具体案例粘在一起，删掉案例就什么都不剩 |

### 1.3 三类知识 → 三类 Skill

| 源材料类型 | 转化成的 Skill 形态 | AI 体感 |
|:-----------|:-------------------|:--------|
| SOP 型（菜谱） | **工作流型** — AI 引导你一步步做 | 🟢 最惊艳 |
| 复盘型（项目心得） | **能力参考型** — AI 随叫随到给方案参考 | 🟡 有用但不够 |
| 分析型（横向对比） | **决策参考型** — AI 辅助信息梳理 | 🟠 高级摘要机器人 |

## 2. 方法论汇总：Skill 知识工程的五条原则

| # | 原则 | 文章出处 | pop-novel-plot v4.1 实践 |
|:-:|:-----|:---------|:-----------------------|
| 1 | **抽象层级决定复用性** | 「换皮测试」——抹掉案例后骨架能否独立 | SKILL.md 路由层只留骨架，steps/ 和 templates/ 是皮肉 |
| 2 | **教方法，不教故事** | 「好文章教的是红烧的方法，不是红烧这条鱼」 | templates/ 每份模板有字段说明 + 示例 + 错误表 |
| 3 | **渐进式披露，按需加载** | 元数据→指令→参考，三层不一次性全吃 | SKILL.md(L1)+steps/(L2)+templates/(L3) |
| 4 | **不创造能力，只传递能力** | Skill 不会让 AI 变聪明，但能让聪明的方向对 | Canvas 强制消费链路 |
| 5 | **文章质量 = 瓶颈，模板是补偿** | Skill 上限 = 源材料里藏了多少可复用的东西 | 模板末尾的 error list |

## 3. Skill 统一架构标准

```
{skill-name}/
├── SKILL.md              ← L1: 路由层
├── steps/                ← L2: 执行指令（按需加载）
├── templates/            ← L3: 参考文档 + 产出模板
├── skill.json
└── CHANGELOG.md
```

### 模板质量自检清单

- [ ] 是否有「字段说明」
- [ ] 是否有「使用场景说明」
- [ ] 是否有「上下游关系」
- [ ] 是否有「常见错误表」（≥5 条）
- [ ] 是否有「产出自检清单」
- [ ] 是否通过「换皮测试」
