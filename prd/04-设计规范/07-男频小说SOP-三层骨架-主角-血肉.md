## 一、SOP的第一性原理

男频小说的创作,本质是回答四个问题:

1. **这是什么坐标系?**(力量体系)——划出舞台,定义强弱刻度和位置空间
2. **众生为什么爬、怎么爬、付出什么?**(动力引擎)——让坐标系活起来的动力系统,设计的是世界里所有人(众生)的攀登逻辑,不是只设计主角
3. **主角是众生中的谁?凭什么爬得比别人快?**(主角+天赋)——主角是众生之一,他的特殊性在于天赋(助力),不在于攀登逻辑本身
4. **众生爬的时候遇到什么?**(地图/势力/人物/剧情)——攀登过程的具体表现,势力/人物从众生攀登方式的差异中生长

这四个问题有严格的依赖顺序:没有坐标系就没有攀登,没有引擎就没有动力,没有众生就没有主角(主角是众生之一),没有血肉就没有故事。SOP必须按这个顺序设计。

**核心视角修正**(老板指出):动力引擎设计的不是"主角如何爬",而是"众生如何爬"。主角是众生之一,他的攀登方式遵循众生通用的逻辑,只是多了一个天赋(助力)让他爬得快。这个视角修正的含义是——世界里有各种各样的人在爬,不同阶层/不同群体的人攀登方式不同,这种差异本身就是社会结构(势力)和人物(角色库)的来源。

## 二、SOP的三层结构

### 第一层:骨架层(力量体系 + 动力引擎)

**核心交付物**:坐标系的定义 + 攀登动力系统

这是整本书的地基。骨架没搭好,后面的主角和血肉都是空中楼阁。这一层要回答:

**力量体系(坐标系)**:
- 这是什么形态的阶层金字塔?(修仙位阶/社会地位/经济等级/术法品质/强度梯度)
- 几层?每层什么含义?每层多少人?
- 层间差距多大?(决定越级击杀的爽感强度)
- 跨越门槛需要什么?(修炼/金钱/知识/人脉)
- 力量体系与世界观的关系?(同构度——力量体系就是世界观的阶层表达)

**动力引擎(众生攀登系统)**:
- **驱动逻辑**(众生为什么爬):世界级驱动力是什么?(生存压力/资源稀缺/外部威胁/好奇探索)——必须是世界级,让所有人都在爬,不只是主角
- **运转机制**(众生怎么爬的闭环):输入什么→转化什么→输出什么→反馈什么?——这是世界级的攀登逻辑,不是主角专属
- **众生攀登方式分层**:不同阶层/不同群体的人攀登方式不同——
  - 有资源的人怎么爬?(高门士族靠血统认证、富家子弟靠金钱砸)
  - 没资源的人怎么爬?(寒门靠知识、穷人靠命换)
  - 爬到顶层的人怎么维持?(垄断资源/封锁门槛/打压后来者)
  - 爬不动/掉下来的人怎么办?(成为背景/成为反例/成为主角的同路人)
  - **众生攀登方式的差异 = 势力和人物的来源**(不是凭空设计势力,而是从攀登方式差异中生长)
- **代价结构**(众生付出什么):攀登的真实代价是什么?不同群体的代价是否不同?(穷人付出健康、富人付出金钱、权贵付出人脉)
- **范式归类**:飞轮型/探索型/混合型/压迫型/对抗型?——决定世界基调
- **演化节点**:长篇中引擎是否演化?何时演化?演化触发事件是什么?

**骨架层的关键红线**:
- 力量体系必须广义化——不限于修仙位阶,任何阶层金字塔都算
- 动力引擎三组成缺一不可,缺任一组世界观就转不起来
- 引擎是世界级的,设计的是**众生如何爬**,不是主角如何爬——主角是众生之一
- 众生攀登方式的差异本身就是势力/人物的来源——不要脱离引擎凭空设计势力

### 第二层:主角层(主角是众生之一 + 特殊天赋)

**核心交付物**:主角在众生中的定位 + 天赋(凭什么爬得比别人快) + 爽感矛盾

骨架(坐标系+众生攀登逻辑)搭好后,从众生中选一个作为主角。这一层要回答:

**主角定位(众生之一)**:
- 主角是众生中的**哪一类人**?(底层逆袭/天才陨落/穿越者/重生者)——他遵循众生通用的攀登逻辑,不是另有一套
- 主角在坐标系中的**起点位置**在哪?(决定读者代入的起点)
- 主角的**身份底色**是什么?(决定读者的情感投射——他与众生中其他人有什么相同和不同)
- 主角的**核心欲望**是什么?(与动力引擎的驱动逻辑对齐——主角的欲望是世界级驱动力的个人化表达,不是脱离众生的独立动机)
- 主角与众生中其他人的**关系**?(谁是对手/谁是盟友/谁是垫脚石——这些关系从众生攀登方式的差异中产生)

**特殊天赋/金手指(凭什么爬得快)**:
- 天赋是什么?(系统/穿越知识/重生信息差/特殊血脉/异宝)
- 天赋如何**合理化加速**主角跨越坐标系?(主角的攀登方式与众生相同,但天赋让他爬得快——降低跨越难度,提供越级可能)
- 天赋的**限制**是什么?(防止喧宾夺主——金手指不能取消坐标系本身的意义,否则众生攀登的意义也被取消)
- 天赋的**代价**是什么?(使用天赋要付出什么,制造爽感矛盾)
- 天赋与动力引擎的关系?(天赋属于引擎"运转机制"的加速器,不是引擎本身——主角的攀登逻辑仍遵循众生通用闭环,天赋只是加速)

**爽感矛盾的设计**:
- 爽感来自主角在坐标系中**位置跃迁的爆发**——他比众生爬得快,这个"快"的瞬间就是爽感
- 矛盾来自天赋加速 vs 代价约束的张力——主角想快但代价拖后腿,想越级但有限制
- 爽感矛盾公式:坐标系门槛(众生都面对的阻力) × 天赋加速(主角独有的助力) × 代价约束(张力) = 爽感强度
- 爽感的对比效应:主角爬得快,是因为众生爬得慢——没有众生作为参照,主角的"快"就没有爽感

**主角层的关键红线**:
- 主角是众生之一,不是脱离众生的独立存在——主角的攀登逻辑遵循众生通用闭环,天赋只是加速
- 天赋是助力,不能喧宾夺主——天赋降低跨越难度,但不能取消坐标系本身的意义(否则众生攀登的意义也被取消)
- 主角的欲望必须与动力引擎的驱动逻辑对齐——主角爬的原因就是世界级驱动力的个人化
- 主角必须有主动性——每章至少1决策+1行动,被动接收型主角不成立
- 爽感需要众生作为参照——没有众生攀登的"慢",主角的"快"就没有对比效应

### 第三层:血肉层(地图 + 势力 + 人物 + 剧情)

**核心交付物**:众生攀登过程的具体表现

骨架(坐标系+众生攀登逻辑)和主角(众生之一+天赋)就位后,填充血肉让世界活起来。这一层的核心原则是:**血肉从骨架生长,不是凭空设计**——势力从众生攀登方式的差异中生长,人物从不同攀登方式的代表者中生长,地图从坐标系的空间分布中生长,剧情从众生攀登的碰撞中生长。

**地图(空间)**:
- 坐标系在空间上怎么分布?(哪个区域对应哪个阶层)
- 主角的攀登路径在地图上怎么走?(起点→中转→终点)
- 地图如何制造空间法则?(力量禁区/阶位壁垒/不可逾越区)
- 地图与动力引擎的关系?(不同区域提供不同的攀登资源/阻力——众生在不同区域攀登方式不同)

**势力(从众生攀登方式差异中生长)**:
- 势力 = 众生中**同一种攀登方式的群体**(不是凭空设计的组织,而是攀登方式相同的人形成的利益集团)
- 占据坐标系高层的势力:靠什么维持地位?(垄断资源/封锁门槛/打压后来者——这就是众生攀登方式分层中的"爬到顶层的人怎么维持")
- 占据坐标系中层的势力:靠什么往上爬?(资源积累/联姻/投靠——这就是众生攀登方式分层中的"有资源的人怎么爬")
- 占据坐标系底层的势力:靠什么挣扎?(命换/投机/依附——这就是众生攀登方式分层中的"没资源的人怎么爬")
- 势力天花板=阶位天花板(每个势力的力量上限由坐标系决定)
- 主角与势力的关系?(依附/对抗/利用/推翻——主角作为众生之一,与不同攀登方式的群体产生关系)

**人物(从不同攀登方式的代表者中生长)**:
- 每个势力(攀登方式群体)的**代表角色**是谁?(具名角色 = 某种攀登方式的典型代表)
- 角色在坐标系中的位置?(等级坐标——决定角色与主角的等级关系)
- 角色的攀登方式与主角的对比?(同路竞争/异路参照/对立阻碍)
- 角色卡(性格/小传/口头禅/成长数据/出场记录)
- 角色与主角的等级关系如何制造冲突?(同阶较量/越级击杀/被高阶压制)

**剧情(众生攀登的碰撞)**:
- 主角在每一卷/每一幕的坐标系位置变化?(起点→终点)
- 主角与不同攀登方式的众生碰撞时遇到什么困难?(坐标系阻力/引擎阻力/天赋约束)
- 困难如何克服?(靠天赋/靠努力/靠盟友/靠智斗)
- 爽感节点在哪?(位置跃迁的爆发时刻——主角比众生爬得快的瞬间)
- 章末钩子如何驱动追读?(主角主动决策型钩子)

**血肉层的关键红线**:
- 势力从众生攀登方式的差异中生长,不是凭空设计——"爬到顶层的人怎么维持"就是高层势力,"没资源的人怎么爬"就是底层势力
- 人物是某种攀登方式的代表者,不是脱离引擎的角色——角色的行为逻辑由他的攀登方式决定
- 地图/势力/人物/剧情都必须映射到坐标系坐标——脱离坐标系的血肉是无效填充
- 地图的空间法则、势力的天花板、角色的等级位置,都由力量体系(坐标系)决定,不能自行发明
- 剧情的困难来自三个层面:坐标系门槛(阻力)+引擎代价(张力)+天赋限制(约束)

## 三、SOP的Phase映射

基于三层结构,男频小说创作的SOP应该按以下Phase推进:

### Phase 0:素材准备(已有,不变)
- 下载参考书 / 提取文风DNA / decon-lite拆书(含表9动力引擎) / 赛道调研
- 产出:文风锚定.md / decon-lite-{书名}.md / 赛道调研.md

### Phase 1:骨架层(力量体系 + 动力引擎)—— 核心立项

**这是SOP的重心,决定整本书的成败。**

**Step 1a:力量体系设计(坐标系)**
- 确定力量体系形态(修仙位阶/社会地位/经济等级/术法品质/强度梯度)
- 设计金字塔(层数/层间差距/每层人数/门槛难度)
- 明确力量体系与世界观的关系(同构度)
- 产出:坐标系定义.md

**Step 1b:动力引擎设计(攀登系统)**
- 设计驱动逻辑(世界级驱动力——为什么爬)
- 设计运转机制(闭环——怎么爬的通用逻辑)
- 设计代价结构(真实代价——付出什么)
- 确定范式归类(飞轮型/探索型/混合型/压迫型/对抗型)
- 设计演化节点(长篇中引擎何时演化)
- 产出:动力引擎设计.md

**Step 1c:骨架自洽检查**
- 力量体系是否广义化?(不能只有修仙位阶,要考虑赛道适配)
- 动力引擎三组成是否齐全?(缺一组转不起来)
- 引擎是世界级还是主角专属?(必须是世界级)
- 骨架是否独特?(引擎独特性决定辨识度)
- 产出:骨架.md(合并坐标系+引擎)

> 现有pipeline对应:Phase 1(seed)的创意部分 + Phase 2(world)的力量体系部分。
> 调整点:seed应先定骨架(力量体系+动力引擎),再定创意。world应消费骨架,而非自己发明力量体系。

### Phase 2:主角层(主角设计 + 特殊天赋)

**骨架搭好后,设计读者代入的对象。**

**Step 2a:主角设计**
- 确定主角在坐标系中的起点位置
- 设计主角的身份底色(底层逆袭/天才陨落/穿越者/重生者)
- 设计主角的性格特质(主动性/决断力/抗压性)
- 明确主角的核心欲望(与动力引擎的驱动逻辑对齐)
- 产出:主角设计.md

**Step 2b:特殊天赋/金手指设计**
- 确定天赋类型(系统/穿越知识/重生信息差/特殊血脉/异宝)
- 设计天赋的加速机制(如何合理化跨越坐标系)
- 设计天赋的限制(防止喧宾夺主——不能取消坐标系本身的意义)
- 设计天赋的代价(使用天赋要付出什么)
- 明确天赋与动力引擎的关系(天赋是引擎加速器,不是引擎本身)
- 产出:金手指设计.md

**Step 2c:爽感矛盾设计**
- 设计爽感矛盾公式:坐标系门槛 × 天赋加速 × 代价约束 = 爽感强度
- 设计爽感闭环:触发→爆发→后果(三段缺一=废章)
- 产出:爽感矛盾设计.md

> 现有pipeline对应:Phase 1(seed)的主角设计部分 + Phase 2(world)的金手指部分。
> 调整点:主角和天赋应在骨架之后设计,不能先有主角再定坐标系。爽感矛盾应显性设计,不能隐含在剧情里。

### Phase 3:血肉层(地图 + 势力 + 人物 + 剧情)

**骨架和主角就位后,填充血肉。**

**Step 3a:地图设计(空间)**
- 坐标系在空间上分布
- 主角攀登路径
- 空间法则(力量禁区/阶位壁垒)
- 产出:地图.md

**Step 3b:势力设计(社会结构)**
- 每层势力占据者
- 势力天花板=阶位天花板
- 势力博弈关系
- 产出:势力格局.md

**Step 3c:人物设计(角色库)**
- 每层具名角色
- 角色等级坐标
- 角色 卡(性格/小传/口头禅/成长数据)
- 产出:角色库.md

**Step 3d:剧情设计(攀登表现)**
- 卷纲(每卷坐标系位置变化+引擎状态+演化节点)
- 幕纲(每幕的困难/克服/爽感节点/章末钩子)
- 场景卡(爽感闭环:触发→爆发→后果)
- 产出:卷纲.md / 幕纲.md / 场景卡.md

**Step 3e:血肉自洽检查**
- 地图/势力/人物/剧情是否全部映射到坐标系坐标?
- 地图的空间法则是否由力量体系决定?
- 势力天花板是否=阶位天花板?
- 剧情困难是否来自三个层面(坐标系+引擎+天赋)?
- 产出:全书设定/(多文件)

> 现有pipeline对应:Phase 2(world)的地图/势力部分 + Phase 3(plot) + Phase 3.5(character)。
> 调整点:地图和势力应在骨架和主角之后设计(消费坐标系和引擎),不能平行设计。character应在plot之后(消费分幕设计的出场角色)。

### Phase 4:正文写作(已有,不变)
- 消费骨架+主角+血肉,逐章写作
- 爽感锚定"跨越坐标系的瞬间"
- 产出:正文/chXXX.txt

### Phase 5:审核(已有,需调整)
- 新增检查项:骨架一致性(力量体系/动力引擎是否被遵守)
- 新增检查项:天赋约束(金手指是否喧宾夺主)
- 新增检查项:剧情映射(血肉是否映射到坐标系)
- 产出:review-chXXX.md

## 四、SOP的关键变化(对比现有pipeline)

| 维度 | 现有pipeline | 新SOP |
|:--|:--|:--|
| 设计顺序 | 创意→world设定→plot→character→write | **骨架(力量体系+引擎)→主角(代入+天赋)→血肉(地图/势力/人物/剧情)→write** |
| 力量体系定位 | world的一个子模块 | **SOP的第一优先级,骨架层的核心** |
| 动力引擎定位 | 不存在(验证后新增概念) | **骨架层的核心,与力量体系并列** |
| 主角设计时机 | seed的1d阶段 | **骨架之后,血肉之前** |
| 金手指定位 | seed/world的子模块 | **主角层的核心,显性设计限制和代价** |
| 爽感矛盾 | 隐含在plot场景卡 | **主角层显性设计,公式化** |
| 地图/势力 | world的子模块 | **血肉层,消费骨架** |
| 剧情设计 | plot独立Phase | **血肉层的一部分,消费骨架+主角** |

## 五、SOP的依赖链(硬约束)

SOP的Phase之间有严格的依赖关系,不能跳过或并行:

```
Phase 1 骨架层(力量体系+动力引擎)
    ↓ [骨架就绪才能设计主角,否则主角悬空]
Phase 2 主角层(主角设计+特殊天赋)
    ↓ [主角就绪才能填充血肉,否则血肉无附着点]
Phase 3 血肉层(地图+势力+人物+剧情)
    ↓ [血肉就绪才能写作,否则正文无依据]
Phase 4 写作
    ↓ [写完才能审核]
Phase 5 审核
```

**关键依赖点**:
- Phase 2依赖Phase 1:主角的起点位置和核心欲望必须与坐标系和引擎对齐
- Phase 3依赖Phase 1+2:地图/势力/人物/剧情必须映射到坐标系,角色的等级关系必须与主角的坐标系位置匹配
- Phase 4依赖Phase 1+2+3:正文必须遵守骨架(力量体系/引擎)+主角(天赋限制)+血肉(剧情设计)

## 六、与验证发现的印证

| SOP设计点 | 验证发现 |
|:--|:--|

## 七、待确认问题

以下问题留待老板确认后才能进入skill改动:

1. **seed和world的职责重划**:骨架层(力量体系+动力引擎)应该放在seed还是world?还是新建一个"骨架"Phase?
   - pop思考:骨架是立项核心,应该放在seed(立项Phase)。seed的故事纲领应改为"骨架优先"——先定坐标系+引擎,再定创意。world消费骨架,负责血肉层(地图/势力)。

2. **主角设计的位置**:主角层应该放在seed还是world?
   - pop思考:主角设计应该在骨架之后、血肉之前。如果骨架在seed,主角也应该在seed(紧跟骨架)。world不负责主角设计。

3. **plot和character的合并**:血肉层的地图/势力/人物/剧情是否应该合并为一个Phase?
   - pop思考:地图和势力可以合到world(血肉层),人物和剧情保持plot+character。但依赖关系要明确:地图/势力(world)→剧情(plot)→人物(character)。

4. **爽感矛盾设计的位置**:应该在主角层还是剧情层?
   - pop思考:爽感矛盾公式(坐标系门槛×天赋加速×代价约束)应该在主角层设计(因为依赖天赋和代价),但爽感闭环(触发→爆发→后果)应该在剧情层设计(因为依附场景)。

5. **审核Phase的检查项调整**:新增的骨架一致性/天赋约束/剧情映射检查项,是否需要review skill升级?
   - pop思考:需要。review应新增"骨架一致性检查"(力量体系/动力引擎是否被遵守)、"天赋约束检查"(金手指是否喧宾夺主)、"剧情映射检查"(血肉是否映射到坐标系)。
