# 子agent 写正文 PE 模板

> **用途**：主 agent 调度子 agent 写单章正文时，复制此模板填入具体参数，作为 sessions_spawn 的 task 全文注入。
> 核心原则：原文给剧情骨架，文风DNA给笔触参照，子 agent 用自己的笔触套剧情。

---

## 注入物清单（主 agent 在 spawn 前必须准备齐全）

| 注入物 | 来源 | 大小 | 必须全文 | 说明 |
|:------|:-----|:-----|:--------|:-----|
| 原文（本章剧情骨架） | `原文/第N章 XXX.txt` | 2-5KB | ✅ | 子 agent 要套笔触的剧情骨架 |
| 文风DNA全文 | `文风库/XXX.md` | 5-25KB | ✅ | 笔触参照，禁止摘要/压缩 |
| 上一章末尾 | v1/v2 上一章 txt 最后20行 | ~500字 | ✅ | 衔接用，不注入全文只注入末尾 |
| 写作指令 | 本PE模板 | ~1KB | ✅ | 告诉子 agent 做什么、不做什么 |

**不注入**：
- ❌ 主 agent 对文风DNA的摘要/要点提炼（违反全文红线）
- ❌ 其他文风的DNA文件（避免笔触混淆）
- ❌ v1/v2 旧版本正文（避免子 agent 做对比/微调而非原创）
- ❌ seed/research/设定库/人物库/剧情线（当前项目这些文件几乎为空，且写单章不需要）

**Token 预算估算**：

```
原文 2-5KB     → ~1.5K-3K tokens
DNA  5-25KB    → ~3K-14K tokens
末尾  500字     → ~300 tokens
PE    1KB      → ~600 tokens
────────────────────────────
总计 ~5K-18K tokens（远在子 agent 上下文窗口内）
```

---

## PE 模板正文（复制以下内容，替换 {花括号} 内的参数）

```
你是一个小说写手。你的任务是：读取下方「原文」和「文风DNA」，用文风DNA的笔触重写原文的剧情。产出完整正文，写入文件。

## 任务

把「原文」的剧情骨架用「文风DNA」的笔触重写一遍。

**重写的定义**：保留原文的剧情事件、角色行为、信息揭露节奏、章末钩子；改变的是句式节奏、感官顺序、叙事距离、段落呼吸、对话引导方式——这些跟笔触相关的层面。

## 原文（剧情骨架，保留这个故事的"发生了什么"）

{粘贴原文全文}

## 文风DNA（笔触参照，学"作者怎么写"，不是学"作者写了什么"）

{粘贴文风DNA全文，必须完整粘贴，不得删减或摘要}

## 上一章末尾（衔接用）

{粘贴上一章正文最后20行，如果是第一章则填"无"}

## 写作规则

### 笔触 vs 内容（最重要）

文风DNA传递的是**笔触**，不是**内容**：

- ✅ 学笔触：句式怎么断、感官怎么排、叙事距离怎么站、段落怎么呼吸、对话怎么引导
- ❌ 不学内容：打谁、怎么打、打完想什么、战术怎么设计、角色怎么复盘

文风DNA里的原文摘录是让你**感受作者怎么写**，不是让你**抄作者写了什么**。
原文摘录里如果出现了战术思维、战斗复盘、击杀余韵、数据面板——那些是那本书的**剧情内容**，不是文风。
你要做的是：用文风DNA的笔触，写原文的剧情。

### 具体笔触参照（从文风DNA提取，写正文时套用）

读文风DNA时，关注以下笔触维度：
1. **句式节奏**：短句链？长句铺陈？独句段？象声词独立成段？
2. **感官顺序**：嗅觉优先？视觉优先？触觉优先？
3. **叙事距离**：贴角色肩膀？上帝视角？固定不进他人内心？
4. **段落呼吸**：段落平均几行？什么时候断段？
5. **对话引导**：有引导词？无引导词？动作前置？
6. **情绪外化**：直述"他感到"？用身体动作外化？用环境映射？

### 禁止

- 禁止从文风DNA原文摘录里提取剧情设计、战斗表现、角色思维作为创作要求
- 禁止改变原文的剧情事件、角色行为、信息揭露顺序
- 禁止合并下一章内容
- 禁止在正文里添加原文没有的新设定

### 产出

1. 正文写入文件：`{输出路径，如 v3/文风dna有效性测试-第N章-XXX.txt}`
2. 正文长度：与原文相当（原文2-5KB，正文也应在2-5KB范围内）
3. 正文格式：纯文本，# 开头标题，正文不分小节

## 开始

先读文风DNA，感受笔触。再读原文，理解剧情。然后写正文。
```

## 主 agent 使用示例

假设要写第5章，用遮天文风：

```
1. 主 agent 用 exec 读取遮天.md 全文（9765 chars）
2. 主 agent 用 exec 读取原文第5章全文（4350 chars）
3. 主 agent 用 exec 读取 v2 第4章末尾20行（衔接）
4. 主 agent 组装 PE：把三个全文粘贴进模板
5. 主 agent 调用 sessions_spawn，task = 组装好的 PE
6. 子 agent 收到 PE，按指令写正文，写入 v3/第5章.txt
```

组装后的 PE 总大小：9765 + 4350 + 500 + 1000 ≈ 15.6KB ≈ 8.7K tokens
子 agent 上下文窗口足够容纳，还能留出 30K+ tokens 给正文创作。

---

## 为什么这样设计

| 设计决策 | 理由 |
|:--------|:-----|
| 原文全文注入 | 子 agent 需要完整剧情骨架，不能只给摘要 |
| 文风DNA全文注入 | 红线要求：禁止摘要压缩传递，子 agent 需要看到原文摘录感受笔触 |
| 上一章只注入末尾20行 | 衔接只需要末尾，全文注入浪费上下文 |
| 不注入v1/v2旧版本 | 避免子 agent 做"对比修改"而非原创写作 |
| PE里明确"笔触vs内容" | 防止子 agent 把文风DNA当内容素材库 |
| 不注入seed/research/设定库 | 当前项目这些文件为空，写单章不需要；正式项目按需注入 |
