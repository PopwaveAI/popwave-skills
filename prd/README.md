## 目录结构

```
prd/
├── 00-规范/         ← SKILL.md / PRD / 知识工程 规范标准
├── 01-管线架构/     ← 全链路依赖图、三级拆解、改造方案
├── 02-问题诊断/     ← 问题复盘、根因分析、修复记录
├── 03-参考资料/     ← 外部方法论、排查手册、素材
├── 99-储备方案/     ← backlog（待实施的方案）
├── README.md
└── 整理-全量文档整合报告.md
```

## 分类导航

| 要做什么 | 看哪个目录 |
|:---------|:----------|
| 写/改 SKILL.md 规范 | `00-规范/01-skill写作规范.md` |
| 理解管线上下游关系 | `01-管线架构/01-全链路依赖图-PRD.md` |
| 查某个问题是否已分析过 | `02-问题诊断/` grep 关键词 |
| 排查 Agent 行为异常 | `03-参考资料/01-排查技巧手册.md` |
| 查 backlog 方案 | `99-储备方案/` |
| 全量文档概览 | `整理-全量文档整合报告.md` |

## 命名规范

- `01-` 前缀表示目录内排序，**不代表优先级**
- 文件名用中英文混排，保留原始 PRD 标题
- 重复文件已在整理时合并，保留版本标注在文件名中

## 新增文档规则

1. 新 PRD 放在对应分类目录
2. 首屏包含元数据块（版本/日期/状态/分类/关联）
3. 在 `整理-全量文档整合报告.md` 中追加索引
