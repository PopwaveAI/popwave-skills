# 拆书 deconstructor Lv1 快速拆解 — Layer 1 改造 PRD

> 版本：v1.0 | 2026-06-12
> 范围：**只改 deconstructor SKILL.md 自己**，不碰任何下游 skill
> 状态：可执行 | 关联：拆书v3.0-分级拆解PRD.md

---

## 零、一句话范围

> 在 deconstructor 现有的 Phase 0~4 管线前，新增一个 **Lv1 快速扫描 Phase**（代号 Phase S），产出 6 个文件，~25 分钟跑完。Phase 0~4 全部保留，不受影响。

---

## 一、改造背景

### 1.1 当前问题

| 问题 | 表现 | 影响 |
|:-----|:------|:------|
| 拆书时间成本高 | Phase 0→4 全流程 ≈ 数小时 | 用户在前 30 分钟得不到任何有用信息，容易流失 |
| 先读后写没有切入点 | 用户说"拆《吞噬星空》"→等数小时后→才看到完整的 T1~T7 分析报告 | 用户可能已经失去开书的热情 |
| 早期信息密度过高 | T1~T7 每个都是一份完整报告，用户需要通读才能找到有用信息 | 用户选择成本高 |

### 1.2 改造目标

```
当前：download → Phase 0→4 (数小时) → T1~T7 → 用户才开始消化
改造：download → Phase S (~25min) → 6个轻量文件 → "老板，可以开书了"
     └→ 用户可选的 Phase 0→4 (保留现有完整流程)
```

### 1.3 不动什么

```
❌ 不动 SKILL.md 中已有的 Phase 0~4
❌ 不动 T1~T7 模板文件
❌ 不动 scripts/ 目录
❌ 不动 fragment-pipeline/
❌ 不动 bookstrap/plot/character-schema 等下游 skill
❌ 不动 consumer-side 的分级协议（等 Layer 2）
```

## 二、新增 Phase S：快速拆解

### 2.1 Phase 定位

```
Phase S（Read: Scanning Layer）
  │  范围：前 10-20 章精读 + 目录级全卷概览（跳读）
  │  耗时：~25 分钟（Agent 全自动，无需人在场）
  │  模式：一次性精读，不做 validation 回环
  │  产出：6 个文件，覆盖 story-engine + 主角 + 快照 + 文风指纹
  │
  ├─ 触发条件：用户说"拆这本书"、"快速拆"、"先拆看看"
  │  (自适应：用户只说"拆书" → 默认 Phase S + Phase 0→4 全跑)
  │
  └─ 产出位置：与当前 Phase 0 同输出目录
     → _参考书/{书名}/
     → 与 T1~T7 同一目录，不冲突
```

### 2.2 Lv1 vs 现有 Phase 0 的分工

| 维度 | Phase 0 (现有) | Phase S (新增) |
|:-----|:--------------|:--------------|
| 读入策略 | 全文件格式检测 + 分页 + 采样策略规划 | 直接读前 10-20 章 + 目录跳读 |
| 输出 | 采样日志 + 诊断依据 | 6 个可消费轻量文件 |
| 耗时 | 长（采样日志一般数十 KB） | ~25 分钟 |
| 后续依赖 | 导向 Phase 1-4 完整拆解 | 独立存在，用户可在此止步 |
| 与 bookstrap 的关系 | 通过 Phase 0.6 消费 | 同样通过 Phase 0.6 消费 |

### 2.3 执行流程

```
Phase S：
  Step 1 - 确认赛道 (30秒)
    → 读目录/简介 → 确认赛道 → 选择角色卡示例对照赛道
  
  Step 2 - 精读前 10-20 章 (15分钟)
    → 逐章精读：主角首次登场 → 冲突升级 → 第一个小高潮 → 卷1结束
    → 记录：
      - 主角的初始状态（身份/驱力/能力）
      - 卷1结束时主角的状态
      - 核心驱动事件（触发主角行动的那个推手）
      - 5 条可观察的文风特征

  Step 3 - 目录级概览（~5分钟）
    → 扫读后续章节标题/开头 → 确认：
      - 全书有多少卷
      - 卷1在总章数中的占比
      - 卷1的核心矛盾是否在第 1-20 章就有明确暗示

  Step 4 - 产出 6 个文件（~5分钟）
    → 按以下格式产出
```

#### file 6：`快速文风指纹-top5.md`

```
# 快速文风指纹 Top 5 — 《吞噬星空》

📍 基于第 1-10 章精读统计

## 1. 句式：短句为主，少用长从句
原文示例："天空湛蓝，就仿佛一块巨大的蓝色翡翠。"
→ 短句 + 比喻收尾，信息密度适中

## 2. 对话：口语化，去引号式描写
原文示例："罗峰师兄！罗峰师兄！一道粗厚的声音响起。"
→ 对话线不使用传统引号，用逗号/句号接续

## 3. 出场介绍：先行为后外貌
原文示例：罗峰出场时先写"他拿着书籍走路"后写"一米七五精瘦"
→ 人物出场不堆砌外貌特征，通过动作让读者脑补

## 4. 心理：间接叙述为主，不直接写"他想"
原文示例：通过魏文之口说"那次在武馆的比武台上…"展示罗峰的战斗力
→ 角色的能力/特质尽量通过他人视角或事件展示

## 5. 节奏：日常→危机，快切场景
原文示例：放学闲聊→黑冠金雕袭击→玻璃碎裂→回家
→ 3-5 段日常后切入危机事件，切换干脆

> 注意：Lv1 指纹基于前 10 章统计，不保证覆盖全书。Lv3 完整 DNA 会有更全面的验证。
```

## 四、SKILL.md 改造方案

### 4.1 改动范围

只改 SKILL.md 的一个位置：**在 `Phase 0` 前插入 `Phase S`**。

```
当前结构：
  Phase 0 → Phase 1 → Phase 2 → Phase 3 → Phase 4

改造后结构：
  Phase S (新增，可选触发) → Phase 0 → Phase 1 → Phase 2 → Phase 3 → Phase 4
  │
  └─ 独立产出，不影响 Phase 0-4 任何环节
```

### 4.2 SKILL.md 各区块修改清单

| 区块 | 修改类型 | 具体操作 |
|:-----|:--------|:---------|
| Frontmatter + 定位 | ✅ 小幅修改 | description 末尾加 "v9.0 新增 Lv1 快速拆解模式" |
| 触发条件 | ✅ 新增 | "快速拆/先拆看看" → 只走 Phase S |
| 质量红线 | ✅ 新增 1-2 条 | "❌ Lv1 不编造主人公尚未显示的信息" |
| 核心流程 | ✅ 新增 Phase S 区块 | 在 Phase 0 前插入，标注 [NEW] |
| 产出物清单 | ✅ 追加 6 个文件 | Lv1 产出物列表 |
| Phase 0-4 | ❌ 不碰 | 完全保留现有内容 |
| T1~T7 模板 | ❌ 不碰 | 完全保留 |
| 异常边界条件 | ✅ 追加 Lv1 特有场景 | "前 10 章不足以提取 story-engine" |
| 版本 | ✅ 更新 | v8.0.0 → v9.0.0 |

### 4.3 建议新增的红线

```
❌5 | **Lv1 不编造** | 前 10 章未显示的信息（主角深层缺陷、反派动机等）一律留空，不准编造
❌6 | **Lv1 不做 validation** | Lv1 不需要 Phase 3 式的验证回环——后文中如果指纹不准，靠升级到 Lv2/Lv3 补正
```

### 4.4 异常场景新增

| 场景 | 触发条件 | 处理动作 |
|:-----|:---------|:---------|
| 前 10 章不足以提取 story-engine | 精读后发现核心假说不明确（如多线叙事） | Lv1 摘要标注 "story-engine 暂存疑"，建议升级到 Lv2 |
| 参考书少于 20 章 | 是短篇 | 全量精读，产出不变（6 个文件全部可填满） |
| 主角在 10 章内未出场 | 群像/多主角 | 产出多个 Lv4 参考卡（最多 3 个），标注 "主角未确定" |
| 前 10 章文风特征不显著 | 作者在前 10 章"在找状态" | 文风指纹标注 "前 X 章不稳定，建议升级 Lv2 取后 20 章重新统计" |
