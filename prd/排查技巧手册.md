## 二、输入/输出原始信息在哪（完整度最高 → 最低）

### Tier 1：runs 目录 —— 信息最完整

```
{项目}/.paopao/projects/{项目名}/runs/{run_id}/
├── input.json     ← ★ 完整 prompt（发给 DeepSeek 的最终文本）
│    ├── prompt:        system + skill + history + 用户指令（全部组裝好的文本）
│    ├── skills:        本轮注入的 skill 列表及路径
│    ├── input:         用户指令元数据（instruction, skillNames, model 等）
│    ├── project:       项目信息
│    └── injectedHistoryTurns: 注入的历史轮数
│
├── response.md    ← DeepSeek 返回的最终 assistant 可见文本
│
└── events.jsonl   ← 流式事件
     ├── thinking delta:   agent 的思考过程（完整思维链）
     ├── tool-call:        工具调用（Read/Write/Exec 的 input）
     ├── tool-result:      工具返回（Read 的输出内容 + 截断标记）
     └── file-change:      文件变更记录（create/update + diff）
```

**排查顺序**：
1. `input.json#prompt` → 本轮 system prompt 的完整内容（skill 有没有、版本对不对）
2. `input.json#skills` → 本轮注入的 skill 列表及版本路径
3. `events.jsonl` → agent 的完整 thinking 链 + 所有 tool calls + Read 输出大小（判断截断）
4. `response.md` → 最终输出
5. `input.json#prompt` 对比 `D:\popwave-skills\` 源码 → 确认注入的版本是否是最新的

### Tier 2：Conversations 目录 —— 对话摘要（适合快速浏览，缺 system prompt）

```
{项目}/.paopao/projects/{项目名}/conversations/{conv_id}.jsonl
每行一条消息：
  - user: 用户原文
  - assistant: content（最终回复）+ events（thinking/tool_call 摘要）
```

**适用场景**：快速回顾对话内容、定位用户提过的问题
**不适用**：查看 system prompt、判断 skill 版本、验证 Read 完整性

### Tier 3：创作产出目录 —— 最终成果（缺少过程信息）

见 §一。

## 三、Skill 定义在哪

| 位置 | 内容 | 何时更新 |
|:-----|:-----|:--------|
| `D:\popwave-skills\skills\{skill-id}\` | 源码（你手动编辑） | 你手动改 |
| `C:\Users\AWMPRO\AppData\Roaming\paopao\remote-skills\{skill-id}\{version}\` | 缓存（Paopao 实际加载） | Paopao 自动同步 |

**排查技巧**：
- `input.json#skills[].path` → 确认本轮从哪个路径加载、哪个版本
- 对比源码 `D:\popwave-skills\` 和缓存 `remote-skills\` 内容是否一致
- 改了源码但缓存没更新 → Paopao 用的还是旧版
- 注意版本目录名（如 `12.0.0/`）可能和 SKILL.md 内声明的版本号（如 `v15.0`）不一致——以 SKILL.md 内容为准

**Skill 文件结构**：

```
{skill-id}/{version}/
├── SKILL.md              ← 主文件（Paopao 注入的就是这个）
├── skill.json
├── CHANGELOG.md
├── steps/                ← 步骤详细指令（需 agent 手动 Read）
│   ├── step-1-xxx.md
│   └── step-2-xxx.md
├── templates/            ← 产出物模板（需 agent 手动 Read）
├── references/           ← 参考文件（需 agent 手动 Read）
└── prompt-templates/     ← 提示词模板（需 agent 手动 Read）
```

## 四、常见审计场景操作指南

### 场景 1：验证 Skill 是否被正确注入

```python
import json
d = json.load(open('runs/{run_id}/input.json', 'r', encoding='utf-8'))

# 1. 看注入的 skill 列表
print(d['skills'])  # [{name, path}, ...]
print(d['input']['skillNames'])  # ['expert-writer', 'pop-novel-writer', ...]

# 2. 看 prompt 中 skill 内容是否完整
prompt = d['prompt']
has_route = '继续前进' in prompt and 'writer' in prompt  # expert-writer 路由表
has_think = 'Think（需求审视）' in prompt
has_reflect = 'Reflect（四层递进审视）' in prompt
print(f'route={has_route} think={has_think} reflect={has_reflect}')
```

### 场景 2：查看 agent 的完整思考过程

```python
import json

thinking = []
with open('runs/{run_id}/events.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        evt = json.loads(line)
        if evt.get('type') != 'model-trace':
            continue
        for item in evt.get('items', []):
            if item.get('kind') == 'thinking':
                thinking.append(item.get('text', ''))

full = ''.join(thinking)
print(f'总思考字符数: {len(full)}')
print(full[:2000])  # 前 2000 字符
```

### 场景 3：判断 Read 工具是否截断了子文件

```python
import json, os

# 从 events.jsonl 提取 Read 调用和结果
reads = []
with open('runs/{run_id}/events.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        evt = json.loads(line)
        if evt.get('type') != 'model-trace':
            continue
        for item in evt.get('items', []):
            if item.get('kind') == 'tool-call' and item.get('toolName') == 'read':
                try:
                    inp = json.loads(item['input'])
                    reads.append({'id': item['toolCallId'], 'path': inp.get('path', '')})
                except:
                    pass
            if item.get('kind') == 'tool-result' and item.get('toolName') == 'read':
                for r in reads:
                    if r.get('id') == item.get('toolCallId'):
                        r['output_len'] = len(item.get('output', ''))
                        r['status'] = item.get('status', '')
                        r['truncated'] = '(truncated' in item.get('output', '').lower()

# 对比实际文件大小
for r in reads:
    path = r.get('path', '')
    if os.path.exists(path):
        actual = os.path.getsize(path)
        output = r.get('output_len', 0)
        ratio = output / actual * 100 if actual > 0 else 0
        trunc = '⚠️截断' if ratio < 95 or r.get('truncated') else '✅完整'
        print(f'{path}: actual={actual} read={output} ({ratio:.0f}%) {trunc}')

# 正常：100% = 完整；<95% = 被截断；~50% = 严重截断
```

### 场景 4：统计全部 runs 中 skill 注入情况

```python
import json, os, glob

runs_dir = r'{项目}/.paopao/projects/{项目名}/runs'
for run_dir in sorted(glob.glob(os.path.join(runs_dir, '*'))):
    input_file = os.path.join(run_dir, 'input.json')
    if not os.path.exists(input_file):
        continue
    d = json.load(open(input_file, 'r', encoding='utf-8'))
    skills = d.get('skills', [])
    snames = [s.get('name', s) if isinstance(s, dict) else s for s in skills]
    inp = d.get('input', {})
    inst = inp.get('instruction', '')[:50] if isinstance(inp, dict) else ''
    print(f'{os.path.basename(run_dir)[:8]} | skills: {snames} | "{inst}"')
```

### 场景 5：统计 agent 读取了哪些子 skill 文件

```python
import json, os

skill_reads = set()
with open('runs/{run_id}/events.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        evt = json.loads(line)
        if evt.get('type') != 'model-trace':
            continue
        for item in evt.get('items', []):
            if item.get('kind') == 'tool-call' and item.get('toolName') == 'read':
                try:
                    inp = json.loads(item['input'])
                    path = inp.get('path', '')
                    if 'SKILL.md' in path or 'step-' in path:
                        skill_reads.add(path)
                except:
                    pass

if skill_reads:
    print(f'读取了 {len(skill_reads)} 个 skill/step 文件:')
    for p in sorted(skill_reads):
        print(f'  {p}')
else:
    print('⚠️ 本轮没有读取任何 skill 或 step 文件')
```

## 五、关键指标速查

| 指标 | 健康值 | 警告值 | 获取方式 |
|:-----|:------|:------|:--------|
| expert-writer 注入完整性 | 5/5 | ≤4/5 | input.json prompt 中搜索 §7 个关键段落 |
| writer SKILL.md Read 完整度 | ≥95% | <80% | events.jsonl tool-result 的 output_len vs 源文件大小 |
| steps 文件读取数量 | ≥3 (step-1/2/5) | 0 | events.jsonl tool-call 中搜索 step- |
| Design 文件存活率 | 100% | <100% | 03-写作资产/ch*-design.md 是否存在 |
| 连续轮次零读取 | ≤1 | ≥2 | 连续 runs 中统计 skill_reads 为空 |
| Entity-snapshot 时效性 | total_chapters = ch文件数 | 不等 | entity-snapshot._meta.total_chapters |
