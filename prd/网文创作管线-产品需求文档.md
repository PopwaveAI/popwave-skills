# 网文创作管线产品需求文档（PRD）

> 版本：v1.0  
> 创建日期：2026-06-08  
> 文档状态：草案

---

## 1. 文档概述

### 1.1 目的

本文档定义网文创作 Skill 群的理想数据流架构，明确各阶段产物的输入输出关系，建立从灵感萌发到正文产出的完整创作管线。

### 1.2 范围

覆盖网文创作全流程：
- 灵感捕捉与创意孵化
- 故事框架与设定构建
- 大纲规划与幕纲设计
- 章纲编排与导演卡制作
- 正文渲染与风格注入

### 1.3 核心理念

**强数据流、可追溯、分级强制**

- **强数据流**：下游必须显式消费上游产物，形成不可绕过的依赖链
- **可追溯**：每个创作决策都能追溯到源头产物（设定/锚定/大纲）
- **分级强制**：P0强制（不可跳过）、P1建议（有则更好）、P2可选（自由发挥）

## 3. 各阶段详细设计

### 3.1 Phase 0：灵感与创意

#### 3.1.1 输入
- 用户提供的灵感片段、创意点子
- 参考作品（如有）

#### 3.1.2 处理流程
1. **灵感捕捉**：记录原始灵感，不评判、不筛选
2. **创意孵化**：追问2-3轮，完善核心冲突、主角特质、世界观基调
3. **可行性初判**：判断是否具备长篇潜力

#### 3.1.3 输出
| 产物 | 格式 | 下游 |
|:-----|:-----|:-----|
| 创意卡片 | Markdown | 开书Skill |
| 故事种子 | YAML | 开书Skill |

### 3.2 Phase 1：开书启动（开书Skill）

#### 3.2.1 输入
- 创意卡片、故事种子
- 锚定小说拆书产出（如有）

#### 3.2.2 处理流程
1. **故事引擎设计**（Phase 0）
   - 锁定核心设定（core_premise）
   - 用户确认后方可进入下一阶段

2. **素材采集与拆书融合**（Phase 0.3-0.6）
   - 参考书甄别
   - 跨域素材聚合（至少3个不同领域）
   - **拆书成果强制融合**：读取锚定小说拆书产出，提取可复用元素

3. **设定与验证**（Phase 1-5）
   - L1六件套：世界蓝图→力量体系→历史驱动力→物种天赋→势力格局→资源物品
   - 世界稳定性检验（7项checklist）
   - 数值体系生成
   - 宪法（constitution.yaml）制定

4. **锚点设计**（Phase 6-7）
   - 起点快照：主角初始状态、世界初始状态
   - 终点快照：主角最终状态、世界最终状态
   - **必须用户确认**后方可进入大纲阶段

### 3.4 Phase 3：大纲架构（大纲Skill）

#### 3.4.1 输入
- L0产品层、L1元设定层
- 起点/终点快照
- 锚定小说拆书产出（T4剧情全貌、T6叙事设计技法）

#### 3.4.2 处理流程
1. **锚点确认**（Phase 0）
   - 读取起点/终点快照
   - 确认跨度：卷字数预期、章数、M1/M3升级幅度

2. **关键里程碑设计**（Phase 1）
   - 从终点反推必经节点
   - 每个里程碑：成就标准、预期章节段、等级要求、朝向方向
   - **必须用户确认**

3. **情节线草案**（Phase 2）
   - 罗列候选线：M1世界危机（必选）、M2主角战斗（必选）、M3主角成长（必选）、S支线（可选）
   - 设计交叉节奏
   - 契诃夫枪备忘
   - **必须用户确认**

4. **L1设定扫描与info_release规划**（Phase 3-a）
   - 扫描L1六件套，提取本幕必须释放的设定知识点
   - P0/P1/P2分级
   - 分配到每章

5. **幕纲设计**（Phase 3-b）
   - 填写act-XX.yaml
   - 每章必须包含：emotional_goal、payoff、end_hook、plotlines_active、info_release

6. **场景卡试读**（Phase 4）
   - 选1个中爽点章节写300-400字试读
   - **用户确认**后方可继续

7. **节奏自检+平台校准**（Phase 5）
   - 7+3+3项自检
   - 番茄/起点节奏模板校准

#### 3.4.3 输出
| 产物 | 格式 | 路径 | 下游 | 强制等级 |
|:-----|:-----|:-----|:-----|:--------:|
| act-XX.yaml | YAML | 设计/幕/ | 章纲设计、正文渲染 | P0 |
| 里程碑设计.md | Markdown | 设计/ | 正文渲染 | P0 |
| 情节线草案.md | Markdown | 设计/幕/ | 章纲设计 | P1 |
| info_release规划.md | Markdown | 设计/幕/ | 正文渲染 | P0 |
| 节奏自检报告.md | Markdown | 设计/幕/ | - | P1 |

### 3.5 Phase 4：章纲设计

#### 3.5.1 输入
- act-XX.yaml（当前幕）
- global-summary.md（前文状态）
- 锚定章库（如有）

#### 3.5.2 处理流程
1. **章级导演判断**
   - 本章在幕情绪弧线中的位置
   - 与上章的情绪衔接检查
   - 爽点等级与铺垫-释放比匹配

2. **锚定章选择**（P1建议）
   - 根据本章类型（对话/战斗/悬疑/...）选择锚定章
   - 输出锚定章引用说明

3. **事实骨架设计**
   - 事件链：本章具体发生哪几件事、顺序是什么
   - 信息释放：哪些设定在本章释放、如何释放
   - 情绪节拍：每段的情绪目标

4. **登场人物卡设计**
   - 本章出场角色列表
   - 每个角色的当前状态、目标、冲突
   - 角色间关系动态

5. **导演卡整合**
   - 合并事实骨架与登场人物卡
   - 输出为Design设计包

#### 3.5.3 输出
| 产物 | 格式 | 路径 | 下游 | 强制等级 |

#### 3.6.3 处理流程（3步驱动）

**Step 1: Design（LLM）**
- 读取所有P0输入
- 产出：设计包（设计说明 + 信息释放策略 + 事件链 + 设定包 + 上下文快照）
- 输出：03-写作资产/chXXX-design.md

**Step 2: Render（LLM）**
- 输入：设计包 + 文风源（10章语料包或文风DNA）+ 宪法红线
- 产出：正文（02-正文/chXXX.md）+ 状态更新块
- 执行写后自评四问

**Step 3: State Update（零LLM）**
- 从状态更新块解析
- 追加到global-summary.md

3. 将风格规则注入Prompt
4. 在正文产出中体现风格特征

| 1 | story-engine.yaml未锁定就进L1设定 | 退回Phase 0 |

- [ ] 元数据头完整

## 7. 异常处理

### 7.1 上游产物缺失

| 场景 | 处理方式 |
|:-----|:---------|
| 可选依赖缺失（P1/P2） | 降级处理，标注"未使用XX参考" |
| 强制依赖缺失（P0） | 暂停管线，返回上游补全 |
| 依赖格式不兼容 | 尝试转换，失败则报错 |

### 7.2 产物版本冲突

| 场景 | 处理方式 |
|:-----|:---------|
| 上游产物更新 | 下游自动感知，建议重新生成 |
| 多版本并存 | 使用最新confirmed版本 |
| 版本回滚 | 保留历史版本，支持切换 |
