# 拆书元 Skill v11.0

> **定位：模板工厂。** 拆书不是目的——**把热门书的成功配方萃取成完整的项目模板包才是目的。** 产出物镜像写作端项目目录树，格式完全对齐写作端模板。创作端可以直接 cp 到新项目目录→按新书卖点调参→用。Lv1 就能开工。Lv3 能续写。
>
> **一级原则**：写作端用什么格式（.md / .yaml），拆书端就用什么格式。写作端有什么模板字段，拆书端就填什么字段。**写作端是甲方。**


## 速查表

| 级别 | 触发词 | 范围 | 耗时 | 产出形态 |
|:-----|:-------|:-----|:-----|:---------|
| **Lv1 快速扫描** | "快速拆""先拆看看""Lv1" | 前 20 章精读 + 目录跳读 | ~25min | 6 个文件——故事引擎+力量体系+角色卡+起/终点快照+拆解摘要 |
| **Lv2 标准模板** | "拆这本书""拆解XX"（默认） | Phase S + 0/1 诊断 → Lv2 主流程（前 100 章全读） | ~1-3h | ~20 个文件——完整项目模板包 |
| **Lv3 工厂级** | "全书拆解""全量""工厂" | Lv2 全部 + 正文分章 + 设计包 + entity-snapshot | ~3-8h | ~50+ 个文件——含正文的完整项目包 |
| **单维度提取** | "只要XX""只要力量体系" | Phase S + 诊断 + 指定维度 | ~30-60min | 指定维度文件 |
| **续拆** | "继续拆XX" | 检查已有产出，跳过 Phase S | 按需 | 补全缺失 |

**消费关系**：产出模板包 → 创作端 copy 到新项目 → creative 跳过 W0/W1 直接碰撞 → world 加载 L1 模板 → plot 加载卷/幕模板


## 产出 = 镜像项目目录树

拆书完成后产出物直接放在项目根目录下，**和 `pipeline-arch.md` 定义的规范目录树完全相同**：

> **命名规则**：不加 `-template` 后缀。和项目文件同名、同格式、同路径。
> **格式强制**：写作端模板用 `.md` → 拆书端产出 `.md`。写作端模板用 `.yaml` → 拆书端产出 `.yaml`。

    ↓

    ↓

    ↓

## ❌ 质量红线

| # | 标准 |
|:-:|:-----|
| ❌1 | **不只是复述剧情** — 每个产出必须回答"这个手法好在哪、我怎么用" |
| ❌2 | **格式对齐写作端** — 写作端.md → 拆书端.md；写作端.yaml → 拆书端.yaml。不可混用 |
| ❌3 | **模板字段不缩水** — 写作端模板有多少字段，必须全量填充。不可跳过 |
| ❌4 | **诊断驱动执行** — Phase 1 决定哪些产出文件值得产。不诊断就全量=浪费 |
| ❌5 | **采样已全读100章** — Lv2默认前100章全量逐章精读。未读完=不准写 |
| ❌6 | **Phase 4 验证已执行** — 至少采样3个未精读区域 + 跨文件一致性检查 |
| ❌7 | **不准凭空书写 — ETL管道级红线** — 写产出前必须先执行本 phase 指定的 `_scripts/` 提取脚本，产出 `_temp/*.json`。产出中的每个硬数据（年龄/等级/地名/职业/道具名）必须能从对应 JSON 中找到。无原文证据 = 留空标注"（原文未显式说明）"，不得凭记忆编造 |
| ❌8 | **03-pop-dna v3.0 独立执行** — 文风DNA通过 03-pop-dna skill 独立执行 |
| ❌9 | **Lv1 不编造** — 前20章未显示的信息一律留空 |
| ❌10 | **角色卡先选级再填写** — 必须先按 character-schema 选级规范确认角色 Lv |
| ❌11 | **输入文件完整性已验证** — 检查中段可读性和结尾完整性 |
| ❌12 | **数据提取脚本未执行** — 写产出前必须已运行本 phase 对应的提取脚本。未运行=❌7违规 |
| ❌13 | **JSON 中无数据仍写入产出** — `_temp/*.json` 中不存在的数据，产出中出现了且未标注「推断」→ 视为编造，严格退回 |


## WRONG 示例

### WRONG 1：产出格式不对齐写作端

> ❌ 写作端 `故事引擎.md`（5区段叙事格式）→ 拆书端产出 `story-engine-template.yaml`（键值对列表）
> ✅ 写作端 `.md` → 拆书端 `.md`。格式 1:1 对齐

### WRONG 2：复述剧情而非提取模板

> ❌ "第一章讲主角被退婚，然后遇到了老爷爷..."
> ✅ 故事引擎.md 「碰撞」段：`A × B — 退婚（外部冲突）是表面驱动，真正的碰撞是穿越者记忆 vs 废材身份的互否`

### WRONG 3：Lv1 编造未显示的信息

> ❌ Lv1 角色卡写"主角深层缺陷是自我牺牲倾向"，实际前20章未显示
> ✅ 留空，标注"前20章未显露"

### WRONG 4：跳过诊断直接全量

> ❌ 不管书的特点，每个相位都跑一遍
> ✅ Phase 1 诊断 → P0 文件全量产出 → P1 文件选产出 → P2 简要记录。全量只在 Lv3 时执行

### WRONG 5：模板字段缩水

> ❌ 故事引擎.md 不写「第五区段 结构预期」（因为"Lv1 数据不准"）
> ✅ 写。"卷1 某某状态→某某状态"。不准就标注「基于前20章推断」。但不写 = 字段缺失 = 模板不可用

### WRONG 6：跳过数据提取直接写产出

> ❌ Phase S 写完"16岁、弯刀大师专长"但未先运行 `extract-baseline.ps1`
> ✅ 先执行 `& '..\_scripts\extract-baseline.ps1' -TxtPath "..." -OutputDir ".\_temp\"` → 从 `_temp/baseline-data.json` 的 `ages[]` 字段确认"年龄原文未显式说明，推断约13-17" → 再从 `levels[]` 字段确认职业路径

### WRONG 7：JSON 中不存在但产出中写了

> ❌ `world-data.json` 的 `faction.entries[]` 中无"金溪镇商会"，产出 `05-势力格局.md` 却写了"金溪镇商会：控制南北贸易"
> ✅ `world-data.json` 中无此条目 → 金溪镇（ch40）是普通小镇，无单独势力 → 产出中不写，或标注"（前100章未显露独立势力）"


## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文文件不存在/格式不支持** | 先用 `scripts/extract.py` 尝试提取；仍失败则提示用户提供 .txt |
| **字数太少（<5万字）** | 确认是否继续；继续则缩减拆解范围 |
| **同人作品但无原著知识** | 先搜索原著百科/时间线 |
| **前20章不足以提取完整故事引擎** | 标注"基于前20章推断"，未来 Lv2 更新 |
| **主角在20章内未出场（群像/多主角）** | 产出多个 Lv4 角色卡（最多3个），标注"主角未确定" |
| **文件 >500K 字符** | 强制走 REPL 式脚本切片访问 |
| **用户 Lv1 止步后想继续深拆** | 检查 Phase S 产出路径，有则跳过直达 Phase 0 |
| **Lv3 正文分章时章节标记不连续** | 标记 skip，标注"原文缺失 chX" |
| **03-pop-dna 不可用** | 文风DNA留空，标注"03-pop-dna 未执行" |


## 阶段边界越界检测

| 边界场景 | 处理 |
|:---------|:-----|
| Phase S 未完成直接进 Phase 0 | ❌ 退回完成 Phase S |
| Phase 0 未产出采样日志 | ❌ 退回 Phase 0 |
| Phase 1 未产出诊断报告 | ❌ 退回 Phase 1 |
| Lv2 产出跳过 P0 文件 | ❌ 退回执行缺失的文件 |
| 产出格式与写作端不匹配 | ❌ 退回——`.md`不可改成`.yaml` |
| Lv1 越界做 Lv3 的活 | ❌ Lv1 不做正文分章/设计包反推/entity-snapshot |
