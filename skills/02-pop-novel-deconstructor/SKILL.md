# 拆书解构 v9.0.0

拆书的根本目的：**从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA**。
不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"**。

**默认拆书范围 = 第一卷或前100章（全读）。** 前100章全量逐章精读，确保覆盖完整的设定引入期、角色建立期和节奏成型期。仅在用户明确要求"全书拆解"时才全量走原动态密度表。

产出物供给 pop-novel-creative（创意生成）管线。

# Phase S：快速拆解（Lv1）

> **定位：** Lv1 快速扫描。前 10-20 章精读 + 目录级跳读，~25 分钟（Agent 全自动，无需人在场）。
> **产出：** 6 个轻量文件 → "老板可以开书了"。不等 Phase 0-4。
> **与 Phase 0-4 的关系：** Phase S 独立存在，不依赖也不影响后续流程。用户可在此止步直接开书，也可继续全量深拆。

## 触发条件

| 条件 | 行为 |
|:-----|:------|
| 用户说"快速拆"、"先拆看看"、"Lv1" | **只走 Phase S**，不走 Phase 0-4 |
| 用户说"拆这本书" | 默认 Phase S + Phase 0→4 全跑（Phase S 优先产出，用户先拿到 Lv1 结果）|
| 已拆过这本书 → 用户想继续 | 检查 _参考书/{书名}/ 下是否有 Phase S 产出，有则跳过直接进 Phase 0 |

## 执行流程

```
Step 1 — 确认赛道（~30秒）
  读目录/简介 → 判断题材类型

Step 2 — 精读前 10-20 章（~15分钟）
  逐章精读：主角首次登场 → 冲突建立 → 第一个小高潮
  记录：
    - 主角的初始状态（身份/驱力/能力/缺陷）
    - 卷1结束时主角的状态变化
    - 核心驱动事件（触发主角行动的那个推手）
    - 5 条可观察的文风特征（句式/对话/节奏/心理描写/感官描写）
    - 前 10 章释放了哪些关键信息

Step 3 — 目录级概览（~5分钟）
  跳读后续章节标题/开头 → 确认：
    - 全书有多少卷
    - 卷1在总章数中的占比
    - 卷1的核心矛盾是否在第1-20章就有明确暗示

Step 4 — 产出 6 个文件（~5分钟）
  按下方规范输出
```

# Phase 0：分层扫描（Layer 0）

## 目的
在不对全书精读的前提下，快速建立整体认知。**这一阶段的核心是"在哪里"和"大概是什么"，不是"细节"。**

## 格式识别与输入验收

### Step 0 — 格式识别
检查输入文件的扩展名，确定提取策略：

| 格式 | 策略 | 依赖 |
|------|------|------|
| `.txt` / `.md` / `.markdown` | 直接 UTF-8/GBK 读取（走原有逻辑） | 无 |
| `.pdf` | 用 `scripts/extract.py` 提取（技术类 → Docling，文字类 → pdftotext→PyPDF2→pdfminer 降级链） | pip install PyPDF2 / pdfminer.six |
| `.epub` | 用 `scripts/extract.py` 提取（ebooklib → stdlib zipfile 降级） | pip install ebooklib beautifulsoup4 |
| `.docx` | 用 `scripts/extract.py` 提取（python-docx → stdlib ZIP/XML 降级） | pip install python-docx |
| `.html` / `.htm` | 用 `scripts/extract.py` 提取（bs4 → stdlib html.parser 降级） | pip install beautifulsoup4 |
| `.rtf` | 用 `scripts/extract.py` 提取（striprtf → regex 降级） | pip install striprtf |
| `.mobi` / `.azw` / `.azw3` | 用 `scripts/extract.py` 调用 Calibre 提取 | 安装 Calibre |

**预检环境：** `python scripts/extract.py --check` 一键打印所有格式的提取器安装状态和缺失的安装命令。

### Step 1 — 多格式提取（非 TXT/MD）

```
INPUT_PATH 为 PDF/EPUB/DOCX/HTML/RTF/MOBI/AZW
└── 运行 python scripts/extract.py <INPUT_PATH>
    └── 读取 <tempdir>/book_skill_work/full_text.txt 作为 TEXT
    └── 读取 <tempdir>/book_skill_work/metadata.json 获取章节/字数信息
```

如果用户提供多个源文件（如 PDF + 笔记 TXT），extract.py 会自动合并为一个连贯文本。

### Step 2 — TXT/MD 验收（原有逻辑）
如果输入是 `.txt` / `.md`，检查文件是否完整可用。如果发现中间部分大段损坏，需要下载更完整的资源。

**验收流程：**
```
1. 扫描全书结构：搜索"第X章""第X卷"标记，统计章节数、卷数
2. 中段完整性检查：在文件25%/50%/75%位置各取一段文字
   - 如果三处都可读（含中文汉字）→ ✅ 通过
   - 如果任意一处乱码/网站垃圾 → ❌ 损坏，进入下载流程
3. 结尾检查：确认有正常的全书终局标记（全书完/完结/尾声），不是截断在战斗高潮
4. 如原文件损坏：
   a. 搜索"《书名》 txt 下载 全本"或"《书名》 作者 txt"
   b. 尝试多个来源（7shutxt/bqg161/xbiqupao 等）
   c. 为每个来源提取直接的TXT下载链接
   d. 下载后重新执行上述验收流程
   e. 如果下载的可用 → 替换原文件路径；仍不可用 → 标记质量问题继续拆解
```

**下载验证脚本参考（Python）：**

import re

## 采样策略
**默认策略（一卷/前100章全读）：**

| 范围 | 采样规则 |
|:-----|:---------|
| 全文 | 第1-100章（或第一卷全卷）：**全量逐章精读** |

全读覆盖率 = **100%**，确保：
- 设定引入节奏无遗漏（所有设定首次出现都在前100章）
- 角色建立期全覆盖（主角/反派的第一次亮相、第一次选择、第一次转变）
- 节奏成型期完整（从开篇到第一个高潮闭环）
- 采样无需高潮/随机采样辅助——全读本身已覆盖所有关键节点

范围边界：
- 默认拆解第一卷。若第一卷超100章，只拆前100章。
- 若用户明确要求全书拆解，全量走原动态密度表（见附录：全书拆解密度表）。

## REPL 式大书访问（新增，配合全读策略使用）

> **场景：** 超长网文（>150章）用 Read 工具逐章全读太贵，且 250 行/次的限制导致来回切换开销极大。
> **解法：** 把 `full_text.txt` 当作可查询语料库，用脚本按章节偏移切片后分批精读。

```python
# 方案 A：Python 脚本按章节切分
import re
with open('full_text.txt', 'r', encoding='utf-8') as f:
    content = f.read()
# 找到所有"第X章"标题的偏移

### 1. 赛道识别（确认 Phase 0 的判断）

### 2. 核心吸引力分析

# 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文不存在/格式不支持** | 先用 `scripts/extract.py` 尝试提取（支持 PDF/EPUB/DOCX/HTML/RTF/TXT/MD），仍失败则提示用户提供 .txt |
| **字数太少（<5万字）** | 确认是否继续，继续则缩减范围 |
| **同人但无原著知识** | 先搜索原著百科/时间线（❌4红线） |
| **发现这本书质量差** | 输出"不推荐拆解"报告 |
| **Lv1 前10章不足以提取 story-engine（多线叙事）** | Lv1 摘要标注 "story-engine 暂存疑"，建议升级 Lv2 |
| **Lv1 主角在10章内未出场** | 产出多个 Lv4 参考卡（最多3个），标注 "主角未确定" |
| **Lv1 前10章文风不显著** | 文风指纹标注 "前 X 章不稳定，建议升级 Lv2" |
| **用户 Lv1 止步后想继续** | 检查 _参考书/{书名}/ 下是否有 Phase S 产出，有则跳过直接进 Phase 0 |
