# 拆书解构 v9.0.0

> **定位：** 从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA。不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"。**
>
> **默认拆书范围 = 第一卷或前100章（全读）。** 仅用户明确要求"全书拆解"时才全量走动态密度表。

---

## 速查表

| 操作模式 | 触发词 | 范围 | 耗时 | 产出文件数 |
|:---------|:-------|:-----|:-----|:-----------|
| Lv1 快速拆解 | "快速拆""先拆看看""Lv1" | 前10-20章精读+目录跳读 | ~25min | 6个 |
| Lv2 标准拆解（默认） | "拆这本书""拆解XX" | Phase S + Phase 0→4 | ~1-3h | 6+N个 |
| Lv3 全书拆解 | "全书拆解""全量" | 全部章节（动态密度表） | ~3-8h | 6+N个 |
| 续拆已拆过的书 | "继续拆XX" | 检查已有产出，跳过Phase S | 同Lv2 | 补全缺失 |

## 操作路径一览

```
Phase S (Lv1 快速拆解)      ← 优先跑完，用户可在此止步直接开书
    ↓ 用户决定继续
Phase 0 (分层扫描)          → 采样日志
    ↓
Phase 1 (自诊断)            → 诊断报告 + T1~T6 任务清单
    ↓
Phase 2 (精读拆解)          → T1~T6 + 文风DNA
    ↓
Phase 3 (验证采样)          → 验证报告
    ↓
Phase 4 (整合输出)          → 三维拆书档案
```

## Phase S：快速拆解 Lv1 — SOP

> **定位：** Lv1 快速扫描。前 10-20 章精读 + 目录级跳读，~25 分钟（Agent 全自动，无需人在场）。
> **产出：** 6 个轻量文件 → "老板可以开书了"。不等 Phase 0-4。

### 第一步：确认赛道（~30秒）

| 输入 | 动作 | 产出 |
|:-----|:-----|:-----|
| 书名/文件 | 读目录+简介 | 题材类型判断 |

### 第二步：精读前 10-20 章（~15分钟）

**读什么：** 主角首次登场 → 冲突建立 → 第一个小高潮

**记录什么：**
- 主角的初始状态（身份/驱力/能力/缺陷）
- 卷1结束时主角的状态变化
- 核心驱动事件（触发主角行动的推手）
- 5 条可观察的文风特征（句式/对话/节奏/心理描写/感官描写）
- 前 10 章释放了哪些关键信息

### 第三步：目录级概览（~5分钟）

**读什么：** 后续章节标题/开头

**确认什么：**
- 全书有多少卷
- 卷1在总章数中的占比
- 卷1的核心矛盾是否在第1-20章就有明确暗示

### 第四步：产出 6 个文件（~5分钟）

**产出清单：**

```
📁 _参考书/{书名}/
├── 📄 Lv1-拆解摘要.md          ← 入口文件
├── 📄 story-engine.yaml        ← 核心假说（与 bookstrap Phase 0 同格式）
├── 📄 Lv4-{主角}-参考卡.md     ← 按 character-schema Lv4 精简
├── 📄 卷1-起点快照.md          ← 第 1 章开篇状态
├── 📄 卷1-终点快照.md          ← 卷末主角状态
└── 📄 快速文风指纹-top5.md    ← 5 条可执行文风规则
```

**❌ 门禁：** 前 10 章未显示的信息（主角深层缺陷、反派动机等）一律留空，不准编造。留空比编造安全。

## Phase 0：分层扫描 — SOP

### 第一步：格式识别与输入验收

| 文件格式 | 提取方式 |
|:---------|:---------|
| `.txt` / `.md` | 直接 UTF-8/GBK 读取 |
| `.pdf` | `python scripts/extract.py <文件>`（降级链：Docling→PyPdf2→pdfminer） |
| `.epub` | `python scripts/extract.py <文件>`（降级链：ebooklib→zipfile） |
| `.docx` | `python scripts/extract.py <文件>`（降级链：python-docx→ZIP/XML） |
| `.html` | `python scripts/extract.py <文件>`（降级链：bs4→html.parser） |
| `.rtf` | `python scripts/extract.py <文件>`（降级链：striprtf→regex） |
| `.mobi` / `.azw` / `.azw3` | `python scripts/extract.py <文件>`（需 Calibre） |

```powershell
# 预检提取器环境
python scripts/extract.py --check
```

**❌ 门禁：** 输入文件必须在上述格式列表中。不支持的格式 → 退回要求提供 .txt。

### 第二步：验收文件完整性

**读什么：** full_text.txt 的 25%/50%/75% + 结尾

**验收规则：**
```
三处都可读（含中文汉字 ≥20 字符）     → ✅ 通过
任意一处乱码/网站垃圾                 → ❌ 损坏
结尾有正常全书终局标记                 → ✅ 通过
截断在战斗高潮无完结标记              → ❌ 损坏
```

**❌ 门禁：** 损坏的文件 → 搜索下载更完整的资源替换后再继续。如果下载仍不可用 → 标记质量问题继续拆解。

### 第三步：全量采样（默认策略）

| 范围 | 采样规则 | 覆盖率 |
|:-----|:---------|:-------|
| 默认 | 第1-100章（或第一卷全卷）逐章精读 | 100% |
| 全书拆解（用户要求） | 走动态密度表 | 按总字数决定 |

**读什么：** >500K 字符的文件走 REPL 式访问（脚本切片，不用 Read 逐章调用）

**产出：**
```
📄 {书名}-Phase0-采样日志.md
```

## Phase 1：自诊断 — SOP

### 第一步：确认赛道（验证 Phase 0 判断）

### 第二步：三维核心吸引力分析

| 维度 | 诊断问题 |
|:-----|:---------|
| 设定（舞台） | 这本书的设定在哪方面最突出？ |
| 剧情（发生了什么+角色） | 故事和角色哪部分最抓人？ |
| 技法（如何发生的） | 作者的写法有什么独特之处？ |

### 第三步：自动生成拆解任务清单

**三维归属框架（T1~T6 + 文风DNA）：**

```
设定层（舞台）
├── T1 力量体系/规则系统
├── T2 世界观展开
└── T3 角色系统设计

剧情层（发生了什么）
└── T4 剧情全貌

技法层（如何发生的）
├── T5 叙事技法综合
├── T6 数据流/系统写法（赛道触发）
└── ★ 文风DNA → pop-dna v3.0
```

### 第四步：优先级分配

| 优先级 | 标准 | 行数要求 |
|:-------|:-----|:---------|
| P0 | 最核心的价值（3-5项） | 深度拆解，500+行 |
| P1 | 有价值的非核心卖点（3-5项） | 中等深度，300+行 |
| P2 | 有但不突出（其余项） | 简要记录，100行 |
| — | 该赛道不适用 | 跳过，标注原因 |

**产出：**
```
📄 {书名}-Phase1-诊断报告.md
```

## Phase 2：精读拆解 — SOP

### 第一步：按任务逐个产出

每个任务独立产出独立文件，不合并压缩。

| 任务 | 模板文件路径 |
|:-----|:-------------|
| T1 | `skills/pop-novel-deconstructor/templates/T1-力量体系模板.md` |
| T2 | `skills/pop-novel-deconstructor/templates/T2-世界观展开模板.md` |
| T3 | `skills/pop-novel-deconstructor/templates/T3-角色系统设计模板.md` |
| T4 | `skills/pop-novel-deconstructor/templates/T4-剧情全貌模板.md` |
| T5 | `skills/pop-novel-deconstructor/templates/T5-叙事技法综合模板.md` |
| T6 | `skills/pop-novel-deconstructor/templates/T6-数据流写法模板.md` |
| ★ 文风DNA | `skills/pop-dna/templates/style-dna-profile.md` |

### 第二步：子Agent 任务描述

**✅ 正确写法：**
```
你的任务是对这本书的力量体系进行全量结构化拆解。
请严格按以下模板中的检查清单逐项填写：
📄 skills/pop-novel-deconstructor/templates/T1-力量体系模板.md

【搜索范围】第X~X章
【产出格式】结构化Markdown，每项检查清单填充示例格式的内容
```

**❌ 错误写法：**
```
拆解这本书的力量体系。
```

**❌ 门禁：** 派给子Agent的任务必须**引用对应模板文件的完整检查清单**。不引用模板 → 退回重写任务描述。

### 第三步：赛道调整

根据赛道自动偏移侧重（玄幻→T1/T2/T3/T5/T6；都市→T2/T3/T4/T5；悬疑→T2/T3/T4/T5 等）。

## Phase 3：验证采样 — SOP

### 第一步：采样

| 采样类型 | 数量 | 目的 |
|:---------|:-----|:-----|
| 随机采样 | 3-5个未精读区域 | 验证核心发现的一致性 |
| 边缘采样 | 1-2个"可能不适用"区域 | 测试发现的边界条件 |
| 类型采样 | 每种场景类型至少1例 | 验证类型公式的普遍性 |

### 第二步：对比验证

**读什么：** 之前未精读的章节区域

**对比项：** Phase 2 发现的规律/模式是否在全书一致

### 第三步：产出报告

**❌ 门禁：** 发现不一致 → 修订 Phase 2 产出。发现新模式 → 补充。

**产出：**
```
📄 {书名}-Phase3-验证报告.md
```

## WRONG 示例

### WRONG 1：复述剧情而非提取写法

> ❌ "第一章讲主角被退婚，然后遇到了老爷爷..."
> ✅ "退婚开篇手法：用外部冲突（退婚）快速建立读者共情+铺垫金手指（老爷爷），可在开书时参照"

### WRONG 2：合并多个任务到同一个文件

> ❌ 一个文件中写了 T1（力量体系）+ T2（世界观）+ T3（角色系统）
> ✅ 每个任务独立文件：`{书名}-T1-力量体系规则手册.md`，`{书名}-T2-世界观展开.md`

### WRONG 3：子Agent 任务不引用模板

> ❌ "请拆解这本书的角色系统"
> ✅ "请严格按以下模板中的检查清单逐项填写：📄 skills/pop-novel-deconstructor/templates/T3-角色系统设计模板.md"

### WRONG 4：Lv1 编造未显示的信息

> ❌ Lv1 写"主角深层缺陷是自我牺牲倾向"，实际前10章根本未显示
> ✅ 留空，标注"前10章未显露"

### WRONG 5：Phase 3 跳过不执行

> ❌ Phase 2 → 直接 Phase 4，没有验证采样
> ✅ Phase 3 至少采样3个未精读区域

### WRONG 6：文风DNA 用旧版不用 pop-dna v3.0

> ❌ 手写文风分析，没有维度模板
> ✅ 必须按 `skills/pop-dna/templates/style-dna-profile.md` 产出

## 异常与边界条件表

| 场景 | 处理 |
|:-----|:-----|
| **正文文件不存在/格式不支持** | 先用 `scripts/extract.py` 尝试提取；仍失败则提示用户提供 .txt |
| **字数太少（<5万字）** | 确认是否继续；继续则缩减拆解范围 |
| **同人作品但无原著知识** | 先搜索原著百科/时间线（❌4红线强制触发） |
| **发现这本书质量差（不值得拆）** | 输出"不推荐拆解"报告，列出原因 |
| **前10章不足以提取 story-engine（多线叙事）** | Lv1 摘要标注"story-engine 暂存疑"，建议升级 Lv2 |
| **主角在10章内未出场（群像/多主角）** | 产出多个 Lv4 参考卡（最多3个），标注"主角未确定" |
| **前10章文风不显著** | 文风指纹标注"前 X 章不稳定，建议升级 Lv2" |
| **用户 Lv1 止步后想继续深拆** | 检查 `_参考书/{书名}/` 下是否有 Phase S 产出，有则跳过直接进 Phase 0 |
| **参考书少于20章** | 全量精读，6个文件全部可填满 |
| **文件 >500K 字符** | 强制走 REPL 式脚本切片访问，禁止直接 Read |
| **多格式输入（PDF+TXT）** | `extract.py` 自动合并为一个连贯文本 |
| **CDP 环境无可用** | Python 脚本按章节切片代替逐章 Read |

## 阶段边界越界检测

| 边界场景 | 检测条件 | 处理 |
|:---------|:---------|:-----|
| Phase S 未完成直接进 Phase 0 | 检查 `_参考书/{书名}/Lv1-拆解摘要.md` 是否存在 | ❌ 退回完成 Phase S |
| Phase 0 未产出采样日志 | 检查 `{书名}-Phase0-采样日志.md` | ❌ 退回 Phase 0 |
| Phase 1 未产出任务清单 | 检查 `{书名}-Phase1-诊断报告.md` 中的任务清单 | ❌ 退回重新诊断 |
| Phase 2 跳过赛道调整 | 检查是否按赛道配置侧重 | ❌ 按赛道调整表重新分配 |
| Phase 3 未执行直接 Phase 4 | 检查 `{书名}-Phase3-验证报告.md` | ❌ 退回 Phase 3 |
| Lv1 混入 Phase 3 验证回环 | Lv1 模式不允许验证采样 | ❌ 降级为 Phase S，不留验证环节 |
| 用户要求全书拆解但未声明 | 默认只拆前100章 | ⚠️ 确认用户是否要"全书拆解" |

## 质量红线（❌ 列表）

| # | 标准 |
