# Phase 1：自诊断 + 优先级分配

> 定位：判断这本书"哪里最值得拆"，决定 Lv2 执行哪些产出文件。

## 执行

### 第一步：确认赛道
验证 Phase S 的赛道判断。

### 第二步：三维核心吸引力分析

| 维度 | 诊断问题 |
|:-----|:---------|
| 设定（舞台） | 这本书的设定在哪方面最突出？ |
| 剧情（发生了什么） | 故事和角色哪部分最抓人？ |
| 技法（如何发生的） | 作者的写法有什么独特之处？ |

### 第三步：Lv2 产出文件优先级

| 优先级 | 标准 | Lv2 产出文件 |
|:-------|:-----|:---------|
| P0 | 最核心的价值维度 | 故事引擎.md + 六件套主维度 + 角色卡 + 全书架构.md + 卷volume-01.md + 幕act-01.yaml |
| P1 | 有价值的非核心维度 | 六件套其余 + 数值体系 + 设计包 |
| P2 | 有但不突出 | 简要记录，不产模板 |
| — | 赛道不适用 | 跳过 |

## 产出

`_参考书/{书名}/Phase1-诊断报告.md`
