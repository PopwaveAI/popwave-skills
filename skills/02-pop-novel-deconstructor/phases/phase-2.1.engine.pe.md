## 🔥 开工前必读：原文验证门禁

**第一条：原文必须已读取。** 未读完前100章就来写 → 退回。

**第二条：每一条写入产出文件的信息，都必须能在原文中找到对应章节的证据。**
- ✅ `主角阵营：守序邪恶（ch2面板显示：'阵营：守序邪恶'）`
- ❌ `前世是半步传奇盗贼`（原文未写，只写了是游戏玩家）

**第三条：找不到原文证据 → 不准写。留空比编造安全。**
**第四条：所有推断必须标注「推断」标签。** 不得把推断写成事实。

**写前自查流程：**
```
1. 这段话的每个"事实" → 找原文对应 chXX
2. 能找到 → 标注 chXX
3. 找不到/记不清 → 返回读原文再写
4. 读了原文也没有 → 留空，标注"前X章未显露"
```

## 🔧 数据提取前置命令（硬性！先执行再写产出）

故事引擎需要 ch1-100 中与"核心假说、驱力、冲突、卖点"相关的结构化数据。

### 第一步：执行基线提取（ch1-20）
```powershell
python "..\_scripts\extract.py" baseline "{$TXT_FILE_PATH}" ".\_temp\"
```
输出：`_temp/baseline-data.json` — 角色/地名/等级/事件基线

### 第二步：执行章节索引提取（ch1-100）
```powershell
python "..\_scripts\extract.py" index "{$TXT_FILE_PATH}" ".\_temp\"
```
输出：`_temp/chapter-index.json` — 全量章节标题/首句/类型标签

### 提取后手动挖掘以下数据（写入 `_temp/story-engine-data.json`）

| 字段 | 从原文提取什么 | 方法 |
|:-----|:--------------|:-----|
| 核心假说 | 本书最核心的一句话设定 | 扫描简介 + ch1-10 开篇段落 |
| 主角驱力 | 他想要什么 / 怕什么 / 为什么 | baseline-data.json + 读取关键章节全文 |
| 碰撞类型 | 核心冲突（人与世界/人与人/人与己） | 从事件链中识别碰撞模式 |
| 卖点 | 前三章有什么吸引读者的钩子 | chapter-index.json 中标记的关键章 |

### 规则
1. **产出中的每个断言必须能找到原文证据**，记录 chXX
2. **推断内容必须标注「推断」**，例如：`阵营倾向：守序邪恶推断（ch2面板显示守序邪恶）`
3. 写一半发现缺数据 → 返回去读原文对应章 → 记录到 JSON → 再写
