> 前置条件：Phase 2.1 + Phase 2.2 已完成


## 🔧 数据提取前置命令（硬性！先执行再写产出）

角色卡数据来自 ch1-100 中所有角色相关的原文内容。

### 第一步：执行基线提取（ch1-20，已包含角色/年龄数据）
```powershell
python "..\_scripts\extract.py" baseline "{$TXT_FILE_PATH}" ".\_temp\"
```
输出：`_temp/baseline-data.json` — `characters[]` + `ages[]` 字段

### 第二步：手动提取各角色详细数据（写入 `_temp/character-data.json`）

读完 ch1-100 后，对每个主要角色提取：

| 字段 | 提取什么 | 方法 |
|:-----|:---------|:-----|
| 身份 | 姓名/年龄/种族/职业/出场状态 | baseline-data.json + 读对应章 |
| 标签 | 3-5关键词（性格+外貌+行为） | 通读后提炼 |
| 驱力 | 想要什么/怕什么/为什么 | 从对话和决策中提取 |
| 关键事件 | 角色的转折点/高光/死亡 | 逐章标记 |
| 关系网 | 与其他角色的关系+变化 | 从互动中提取 |
| 登场章节 | 首次出现 + 所有出场章 | baseline-data.json 标注 |

### 规则
1. **角色年龄必须有原文证据**，找不到 → 标注"原文未显式说明，推断约X岁"
2. **职业/等级路径必须来自原文面板数据**，不得自己造转职路线
3. **关系网中每个关系必须有互动事件支撑**，标注 chXX
4. **龙套池只收录原文中有名有姓的角色**，不得编造名字

## 格式规则

**强制选级**：必须先确认角色属于哪个 Lv，再加载对应模板，再填写。

### 主角(Lv4) — 9节16维

| 节 | 维度 | 从书中提取什么 |
|:---|:-----|:-------------|
| 身份 | 姓名/年龄/种族/职业/原型/出场状态 | 主角在碰撞点上的位置 |
| 标签 | 3-5关键词 | 性格+外貌+行为模式 |
| 心理 | 核心性格/决策逻辑/核心缺陷 | 驱力冲突来源 |
| 驱力 | 表层目标/深层需要/黑暗冲动/恐惧 | 四层动机映射到情节 |
| 能力 | 优势/中等/弱项 + 成长路径 | 等级成长轨迹 |
| 演化 | 弧线阶段(≥3) + 不可逆点 | 每阶段的前/后心态和关键决策 |
| 网络 | 关系表 + 演化摘要 | 每段关键关系的起点→转折→终点 |
| 设计目的 | 身份/驱力/缺陷/唯一性 | 从原文反推作者的意图 |
| 叙事能力 | 唯一性四级评估 | L1核心差异/L2约束/L3不可替代/L4叙事驱动 |

### 配角(Lv3) — 6节

| 节 | 从书中提取什么 |
|:---|:-----|
| 身份 | 姓名/年龄/与主角关系/一句话概括 |
| 标签 | 3-5关键词 |
| 驱力 | 想要什么/最怕什么/什么情况下背叛 |
| 叙事功能 | □ 情感锚点 □ 冲突放大 □ 信息管道 □ 镜像/对照 □ 催化剂 □ 场面注入 |
| 弧线 | 类型+起点心态+终点心态+触发事件 |
| 登场场景表 | 卷/幕+事件+作用 |
