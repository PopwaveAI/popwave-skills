# Phase 3：Lv3 工厂级深度提取

> 定位：全书拆解。产出完整项目模板包（含正文分章+设计包+entity-snapshot），可直接 cp 到新项目根目录。
> 前置条件：Lv2 全部完成 + 用户确认进入 Lv3
> 耗时：~3-8h

## 🔧 数据提取前置命令（硬性！先执行再写产出）

Lv3 需要全书数据。执行以下脚本产出全部分章文本和全量索引。

### 第一步：执行章节索引提取（全书，标记卷边界）
```powershell
python "..\_scripts\extract.py" index "{$TXT_FILE_PATH}" ".\_temp\"
```
输出：`_temp/chapter-index.json` — 全书章节标题/行号/字符数

### 第二步：手动分割正文（确保每章原文完整保留）

从 TXT 中按 `^第\d+章` / `^第[一二三四五六七八九十百千]+章` 分割，写入 `正文/ch001~chN.md`。

**注意**：正文分割是 1:1 原文复制，不得修改、不得归纳、不得省略。

### 第三步：设计包反推前要求（每包写前提取 ch 原文）
```powershell
# 对每5章一个包，提取 chXX-原文.json
python "..\_scripts\extract.py" baseline "{$TXT_FILE_PATH}" ".\_temp\"  
# 然后手动从 baseline-data.json 中按 ch 范围过滤
```

### 规则
1. **正文分章 = 纯复制**，不允许任何概括或改写
2. **设计包中的事件链必须逐章标注 chXX**，每章至少一个事件
3. **entity-snapshot 中的等级/状态必须来自 ch 结尾处的原文面板**
4. 写设计包时发现缺数据 → 去原文对应章重读 → 补录 → 再写

## 产出结构

```
项目根/
├── 正文/ch001~chN.md                     ← 从TXT按章节分割，正文+_state_update块
├── 写作资产/设计包/ch001~050-设计包.md    ← 按 templates/design-pack.tpl.md 反推
├── 00-总控/entity-snapshot.yaml          ← 角色状态快照（YAML结构化）
├── 设计/卷/volume-02~N.md                ← 全部卷
├── 设计/幕/vol-02~N/act-*.yaml           ← 全部幕
└── 写作资产/文风DNA/{书名}.md            ← 03-pop-dna v4.0 场景卡格式
```

## 格式规则

- **正文 .md** — prose-render 格式（正文+`_state_update`块）。内容从 TXT 按章节分割，保留原文。
- **设计包 .md** — 按 `templates/design-pack.tpl.md` 格式从正文反推：基础信息+登场人物+事件链+情绪节拍+钩子
- **entity-snapshot .yaml** — YAML 结构化，agent 可精确 parse
