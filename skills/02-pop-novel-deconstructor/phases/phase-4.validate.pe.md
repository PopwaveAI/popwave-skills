# Phase 4：验证 + 打包

> 定位：验证模板完整性和一致性，产出打包索引。
> 前置条件：Lv2 或 Lv3 全部完成

## 🔧 验证前数据检查

Phase 4 不自行提取数据。检查各 phase 的 `_temp/` JSON 文件是否均已生成：

```powershell
# 验证所有中间数据文件存在
Test-Path '.\_temp\baseline-data.json'
Test-Path '.\_temp\chapter-index.json'
Test-Path '.\_temp\world-data.json'
Test-Path '.\_temp\story-engine-data.json'
Test-Path '.\_temp\plot-data.json'
Test-Path '.\_temp\character-data.json'
```

**验证规则**：每项产出必须能追溯到上述 JSON 中的原始数据。无法追溯的片段标记为 `⚠️ 未经验证`。

## 验证维度

### 1. 随机采样验证
- 随机3-5个未精读区域
- 边缘1-2个"可能不适用"区域
- 每种场景类型至少1例

### 2. 跨文件一致性
| 检查项 | 对比文件 | 规则 |
|:-------|:---------|:-----|
| 力量体系 vs 世界蓝图 | `02-力量体系.md` ↔ `01-世界蓝图.md` | 力量来源与核心法则一致 |
| 势力 vs 世界蓝图 | `05-势力格局.md` ↔ `01-世界蓝图.md` | 势力分布与地理一致 |
| 角色卡 vs 卷设计 | `角色卡-Lv4.md` ↔ `volume-01.md` | 角色状态与卷起点快照一致 |
| Canvas vs 数值体系 | `act-01.yaml` ↔ `combat_capability.yaml` | 战力数值与Canvas中的战斗规模一致 |

### 3. 产出打包索引
生成 `_参考书/{书名}/Phase4-产出索引.md`：
- 所有产出文件列表（路径+格式+消费端）
- Lv1/Lv2/Lv3 分级标记
- 缺失标注（如有）
