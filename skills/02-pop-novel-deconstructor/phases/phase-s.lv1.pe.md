# Phase S：Lv1 快速扫描

> 定位：Lv1。前 20 章精读 + 目录级跳读，约 25 分钟。Agent 全自动，无需人在场。
> 产出：6 个文件。产出后可止步（"老板可以开书了"），不等 Phase 0-4。

---

## 🔥 开工前必读：原文验证门禁（不满足不可开工）

**第一条：原文必须已读取。** 未读取目标 TXT → 退回。不得凭记忆编造。

**第二条：每一条写入产出文件的信息，都必须能在原文中找到对应章节的证据。** 格式：
- ✅ `核心驱力：保护妹妹薇薇安（ch1-4: 索伦昏迷中第一反应是妹妹安全）`
- ✅ `年龄：约18-20岁（原文未明确写年龄。从薇薇安8岁+ch5回忆''一个年幼的小男孩要养育一个只有三岁的妹妹''推测年龄差约5岁，推断约13-17岁）`
- ❌ `年龄：16岁`（原文未写）

**第三条：找不到原文证据 → 不准写。留空比编造安全。** 标注`（基于前X章推断，Lv2将更新）`或`（前X章未显露）`。

**第四条：所有推断必须标注「推断」标签。** 不得把推断写成事实。

---

## 🔧 数据提取前置命令（硬性！先执行再写产出）

执行以下脚本提取 ch1-20 原文数据，提取结果写入 `_temp/baseline-data.json`，此 JSON 是写本 phase 全部产出的唯一数据来源。

### 执行命令
```powershell
python "..\_scripts\extract.py" baseline "{$TXT_FILE_PATH}" ".\_temp\"
```

### 输出文件
`.\_temp\baseline-data.json`

字段说明：
| 字段 | 内容 | 用途 |
|:-----|:-----|:-----|
| `characters[]` | 所有命名角色 + 首次出现章节 | 角色卡、故事引擎 |
| `places[]` | 所有地名 + 首次出现 | 世界蓝图、故事引擎 |
| `levels[]` | 等级/职业/技能提及 + 出处 | 力量体系、角色卡 |
| `ages[]` | 年龄信息 | 角色卡 |
| `monsters[]` | 怪物/Boss + 出场章节 | 力量体系 |
| `events[]` | 逐章关键事件 | 故事引擎、起点快照 |

### 规则
1. **此 JSON 是唯一原料**：产出中的每个硬数据（年龄/等级/地名/角色名等）必须能在 JSON 中找到
2. **JSON 中标记为 `__inferred__` 的数据**：只在原文有间接证据、无显式声明的，产出时必须标注「推断」
3. **写一半发现缺数据**：返回修改脚本扩大提取范围 → 重新提取 → 再用新 JSON 写
4. **不允许凭空书写**：JSON 中没有的数据 = 原文未出现 = 不能写入产出

- `故事引擎.md` ← 按 `templates/story-engine.tpl.md` 5 区段模板

## 原文验证写前自查

写每一段之前，执行以下流程：
```
1. 这段话的每个"事实"（年龄、等级、关系、事件顺序、地名）→ 找原文对应
2. 能找到 → 标注 chXX
3. 找不到/记不清 → 返回读原文再写
4. 读了原文也没有 → 留空，标注"前X章未显露"
```
