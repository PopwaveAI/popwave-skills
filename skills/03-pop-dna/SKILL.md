# 03-pop-dna · 文风DNA蒸馏引擎 v4.0

> **定位：从原文中按场景类型提取作者笔触。** 每个场景卡 = 1-2 句观察 + 500+ 字原文 + 时间演变。产出的不是分析报告，是 prose-render 渲染时旁边打开的参考页。

---

## 速查表

| 步骤 | 操作 | 读什么 | 产出什么 | 门禁 |
|:-----|:-----|:-------|:---------|:-----|
| 步骤0 | 取样 | 全书扫描 | 采样清单（≥30章，≥20,000行） | ❌ 不够退回 |
| 步骤1 | 逐章精读 | 采样章全文 | 场景类型标记 + 候选原文段 | — |
| 步骤2 | 全书搜索验证 | 全书 | 验证过的判断 | — |
| 步骤3 | 写风格文件 | 步骤1+2 产出 | `styles/{书名}.md`（20-30K） | ❌ 原文/场景/通用维度不达标退回 |
| 步骤4 | 试写验证 | 风格文件 | 2 组 300 字对比稿 | — |

### v4.0 vs v3.0 核心变化

| v3.0 | v4.0 |
|:-----|:-----|
| "战斗" = 1 个维度 | 战斗拆为 4 个场景卡（早期/中坚/后期/暗杀） |
| 每条维度 2-3 处 1-2 行原文 | 每个场景卡 500+ 字完整原文 |
| 场景类型由 prose-render 自己判断 | 设计包 scene 字段 → DNA 场景卡 1:1 映射 |
| 产出一个"观察"文件 | 产出 prose-render 可直接对照的场景参考 |

## ❌ 质量红线（不满足即退回）

```
□ 任何维度/场景卡的原文 < 500 字 → 退回。原文不够，render 无法感受作者笔触。
□ 场景卡 < 6 个 → 退回。场景类型覆盖不足。
□ 通用维度 < 4 个 → 退回。
□ 任何项目无"时间演变"段 → 退回——不知道早期/中期/末期差异的文风认知是残废的。
□ ★ 不接受降级替换 → 场景卡原文必须与 scene 类型严格对应。
   战斗·早期遭遇战的原文不能是行军队列描述，boss 战的原文不能是万象无常牌设定说明。
   找不到合适的原文段 → 退回扩采，继续读更多章，直到找到真正的匹配内容。
   采样范围内找不到说明采样不够，不是"随便塞一段差不多的"的理由。
□ ★ 产出只留摘要 → 写入 styles/ 文件后对话中不粘贴完整风格文件。说"已写入 styles/{书名}.md。摘要：{场景卡数量及覆盖类型}。需对照渲染时调用。"
```

## 执行流程（SOP）

### 步骤0：取样

**读什么：** 全书（扫描章节目录 + 确认章节边界和全卷分布）。

**做什么：**
- 均匀采样：前 20% + 中段 40-60% + 后 20%，每段 ≥10 章
- 目标：≥30 章，≥20,000 行

**产出：** 采样清单（章号列表 + 每章行数统计）。

**❌ 门禁：** 不到 20,000 行 → **退回**，继续读更多章。

---

### 步骤1：逐章精读采样章

**读什么：** 步骤0 选出的全部采样章的**完整内容**（不跳段、不只看开头）。

**做什么：**
- 标记章内的场景类型和边界（一个章可能含多个场景类型）
- 为每种场景类型标记候选原文段

**产出：** 场景类型清单（含对应原文段的位置标注）。

---

### 步骤2：全书搜索验证

**读什么：** 全书 + 步骤1 的初步判断。

**做什么：**
- 对每个可能写成绝对结论的判断，搜全书验证
- 搜索数字标注在产出中

**产出：** 验证过的判断列表。

---

### 步骤3：写出风格文件（★ v4 核心）

**读什么：** 步骤1+2 的全部产出 + `templates/style-dna-profile.md` v4 模板。

**做什么（三步）：**

#### Step 3a：选维度 + 场景卡
- 选出 ≥4 个通用维度 + ≥6 个场景卡
- 场景卡选择法则：
  1. 读采样章 → 标记所有场景类型（战斗/对话/描写/叙述 x 阶段）
  2. 去重 → 合并同类型候选，选最典型的
  3. 选 ≥6 个最典型的 → 每个配 500+ 字原文
  4. 卡名对齐 prose-render 的 scene 字段命名

#### Step 3b：截取原文（★ 不降级）
每个场景卡/维度截取 500+ 字原文。必须是该 scene 类型下真实发生的原文：
- "战斗·早期遭遇战" → 不能塞军队行军或战术解说段
- "战斗·中坚 Boss 战" → 不能塞抽卡牌或世界设定
- "对话·极简引导词" → 不能塞黑面包生存描写

**❌ 门禁：** 采样范围内找不到匹配段 → **退回** 步骤0 扩采，不接受降级替换。

#### Step 3c：写观察 + 时间演变
- 写观察 + 时间演变
- 确认场景卡名与设计包 scene 字段对齐

**产出：** `styles/{书名}.md`，全文约 20-30K（含原文摘录）。

**❌ 最终门禁：**
- [ ] 所有维度/场景卡原文 ≥ 500 字
- [ ] 场景卡数量 ≥ 6
- [ ] 通用维度数量 ≥ 4
- [ ] 每项有时间演变段
- [ ] 无降级替换
- ❌ 以上任意一条不通过 → **退回**，不合格不出。

---

### 步骤4：试写验证

**读什么：** 步骤3 产出的风格文件。

**做什么：**
- 选 2 个场景卡
- 对同一个骨架事件各写 300 字
- 对比是否捕捉到了差异

**产出：** 2 组 300 字对比稿 + 差异分析。

## 与 prose-render 的集成

```
prose-render Step 1（读入输入）:
  ① 全量加载 styles/{书名}.md (~20-25K，Get-Content 安全范围内)
  ② 读设计包 → 获取本章 scene 字段（如 combat_early_skirmish）
  ③ 在 DNA 文件内定位对应的场景卡
  ④ 带这个场景卡的原文 + 观察进 Step 2 渲染

为什么全量加载而非按需：
  - 文风不是场景的简单拼接 — 过渡段/情绪缓冲的写法需要整体感知
  - 20-25K 在 Get-Content 安全范围内，不触发截断
  - 文件全量驻留在 context 中，允许 agent 自由对照维度
  - 单一 DNA 文件结构简单，维护成本低
```

## WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 场景卡原文 < 500 字就交差 | render 无法感受作者笔触 | 退回扩采，继续读更多章 |
| "战斗"场景卡塞进行军/战术解说段 | 场景类型不对应 | 降级替换一概不认，必须是该 scene 类型的真实原文 |
| 不写"时间演变"段 | 不知道早期/末期差异，文风认知残废 | 每个维度/场景卡必须标注时间演变 |
| 采样范围找不到匹配段就"随便塞一段" | 降级=原文覆盖不足 | 退回扩大采样，直到找到真正的匹配 |
| 只读章节开头/结尾，不读完整章 | 漏掉章内场景类型 | 章节全文阅读，不跳段 |

## 异常与边界条件

| # | 异常场景 | 处理方式 |
|:-:|:---------|:---------|
| 1 | 全书不足 20,000 行（短篇/未完结） | 降低采样目标到全书 80% 行数，标注"采样受限" |
| 2 | 全书场景类型单一（如全是对话） | 降低场景卡下限（≥3 个），用通用维度补足 |
| 3 | 找到的原文段刚好 500 字但边缘模糊 | 扩到 600-700 字保证上下文完整 |
| 4 | 多本书/多作者同时分析 | 每本书独立产出一个风格文件，不做交叉比较 |
| 5 | 作者文风在全书中发生剧烈变化（转型） | 分前/后两期各产出一份 DNA，标注转折点 |
| 6 | 书有大量设定说明/百科式章节 | 跳过此类章节，只采叙事/场景驱动的章节 |
| 7 | prose-render scene 字段在 DNA 中找不到对应卡 | 检查命名对齐，退回步骤3c 修正卡名 |
| 8 | 文件 > 30K 触发 context 截断风险 | 拆分卷/阶段为多个 DNA 文件，标注加载顺序 |
| 9 | 无模板文件 `style-dna-profile.md` | 退回，检查模板是否存在 |
| 10 | 试写验证发现 2 个场景卡差异不明显 | 退回步骤3，重新选择差异更大的场景卡 |
