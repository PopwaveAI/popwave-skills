# 文风DNA档案 · 模板 v4

> **v4 核心变更**：通用维度和场景卡全部配 ≥500 字原文摘录。prose-render 不论读哪个维度，都能直接感受作者的笔触密度，不是看 2-3 句短摘要。

---

## 格式规则

- **平铺**，无章节编号，无禁令。每个维度/场景卡一节 H2 标题。
- **每个项目 = 观察 (1-2句) + 原文 (≥500字) + 时间演变 (1-2句)**，三者不可缺一。通用维度也不例外。
- **原文保留原生段落感**——按章逐行摘录，移除全角缩进空格，保持可读性。
- **来源标注用斜体**——`> _——CH{N}，一句话说明_` 跟在原文块后，不混在正文里。
- **场景卡头标注 scene 字段**——`scene: combat_early_skirmish` 紧随 H2 标题。
- **不写绝对化结论** — 有全书搜索数字的标注数字，没有的标注"采样范围内"。
- 文件全量供 prose-render Step 1 加载（~20-30K），不在渲染时拆分。

## 一、场景卡矩阵 (★ v4 新结构)

> **prose-render 消费协议**：设计包的 `scene` 字段必须与场景卡名称严格对齐。
> 例：设计包写 `combat_mid_boss` → prose-render 匹配到此卡的原文 + 观察。

| 场景卡 | 对应 scene 字段值 | 选择标准 |
|:------|:----------------|:--------|
| 战斗·早期遭遇战 | `combat_early_skirmish` | 主角 ≤ 二阶，对手同级或杂兵，规模 ≤ 1 房间 |
| 战斗·中坚 boss 战 | `combat_mid_boss` | 主角 2-4 阶，单挑 boss，有招式名对抗，规模 = 建筑级 |
| 战斗·后期大规模 | `combat_late_war` | 主角 ≥ 4 阶，势力/军团级对抗，有神性能力展示 |
| 战斗·暗杀/偷袭 | `combat_stealth` | 背刺系统、潜行判定，暗杀公式 |
| 对话·日常兄妹 | `dialogue_daily` | 薇薇安/主角 短对话，≤2 人 |
| 对话·情感高潮 | `dialogue_emotional` | 眼泪/告白/告别/久别重逢 |
| 对话·信息交换 | `dialogue_intel` | 酒馆打听/牧师对话，功能性 |
| 描写·环境引入 | `description_environment` | 新场景首次出场，从感官进入 |
| 描写·角色外貌 | `description_character` | 首次出场/换装/受伤后 |
| 叙述·孤独紧张 | `narration_suspense` | 独处/听动静/计算——信息→评估→决定 |
| 叙述·过渡/缓冲 | `narration_transition` | 战后收尾/清晨/赶路，节奏放缓 |

> **不硬套不存在的卡。** 此书没有暗杀系统 → 不写 combat_stealth。

## 二、通用维度 (≥4个必选)

每个维度格式：

```
## {维度名}

{1-2句观察}

### 原文

{≥500字原文摘录——按原文章节截取，保留完整对话和段落，不缩写}

### 时间演变

{开篇这类写法怎么做？中期怎么变？末期是否还存在？}
```

### 通用维度池

| 维度 | 要回答的问题 |
|:-----|:------------|
| **叙事者距离** | 叙事者离角色多近？进入内心还是只跟视线？ |
| **情绪传递** | 情绪通过动作/对话/直述中的哪一种？直述情绪词全书搜索数字。 |
| **对话·引导词** | 引导词密度？零引导词/动作前置/完整引导的比例变化。 |
| **外貌描写** | 通过身体状态还是形容词直述？嵌在动作中还是独立段落？ |
| **数据嵌入** | 面板/装备/升级怎么呈现？弹出式还是叙述流？ |
| **环境展开** | 每章从什么感官启动？空间认知从哪向外推？ |
| **段落呼吸** | 独句段的使用频率和功能？开篇/中期/末期的差异。 |
| **物品/装备描写** | 物品如何呈现——视觉在前还是数据在前？格式是否统一？ |

## 五、禁止事项

- ❌ 不统计词频/句长/修辞密度
- ❌ 不写禁令清单
- ❌ 不把单章偶然细节当全书规律
- ❌ **任何维度/场景卡的原文 < 500 字 → 退回重采**
- ❌ 任何项目无"时间演变"段 → 退回
- ❌ **不接受降级替换**——找不到匹配的原文段 → 退回扩采，不准塞"差不多"的内容蒙混过关
