## 执行流程 SOP

### 第一步：判断角色叙事权重 → 选 Lv

| 该角色 | 用级别 | 维度数 |
|:-------|:------|:------:|
| 只出现 1 个场景，路人/背景板/一句话概括即可 | **Lv1** | 2 |
| 为特定叙事功能服务，出现 1-3 幕，不需要独立弧线 | **Lv2** | 5 |
| 贯穿 2+ 幕且有独立动机，有自己的弧线 | **Lv3** | 9 |
| 故事主角 / 主要反派 / 最终对手 / 贯穿全文的灵魂 | **Lv4** | 16 |

**问自己**：这个角色对故事的叙事权重有多大？

### 第二步：加载对应 Lv 模板

| 级别 | 模板路径 | 加载方式 |
|:----|:---------|:---------|
| Lv1 | `schema/Lv1-one-shot.md` | `Get-Content -Path '{skill_path}/schema/Lv1-one-shot.md' -Encoding UTF8 -Raw` |
| Lv2 | `schema/Lv2-functional.md` | 同上 |
| Lv3 | `schema/Lv3-important.md` | 同上 |
| Lv4 | `schema/Lv4-core.md` | 同上 |

### 第三步：按维度填写角色卡

| 项目 | 内容 |
|:-----|:------|
| 读什么 | 本 SKILL.md 的维度矩阵 + 对应 Lv 模板文件 |
| 做什么 | 按维度逐项填写，严格遵循"上一级全部 + 本级新增"原则 |
| 产出什么 | `设计/角色层/{角色名}-{定位}.md`（格式与模板一致） |
| 必须包含 | **设计目的** + **叙事能力**（所有级别都必须有） |

### 第四步：选填升级/降级

| 场景 | 动作 |
|:-----|:------|
| 没想清楚 | 先按较低级别写，后续升级补全 |
| 写着写着角色变重要了 | 升级后补全缺失维度（回溯填写，不重写已有信息） |
| 降级 | 几乎不发生 — 已有多维信息的角色降级无意义 |

## ❌ 质量红线

| # | 红线 | 说明 |
|:-:|:-----|:-----|
| ❌1 | **不跳过选级就填角色卡** | 必须先确认角色级别再加载对应模板。不对应级别的字段绝不填写 |
| ❌2 | **不跨级混用维度** | Lv2 不偷 Lv4 的「能力」维；Lv3 不漏 Lv2 的「标签」维 |
| ❌3 | **不对齐就输出** | 每 Lv 维度 = 上一 Lv 全部维度 + 本 Lv 新增维度。少一项即为不合格 |
| ❌4 | **设计目的和能力不可省略** | Lv1~Lv4 每级角色卡必须包含「设计目的」和「叙事能力」声明 |
| ❌5 | **产出只留摘要** — 写入角色卡后对话中不粘贴完整卡片。说"已写入 {路径}。摘要：{角色名} Lv{N}，{核心特征}。需展开告诉我。" |

## 维度递增体系

Lv1 → Lv4 维度严格做加法。每级 = 上一级**全部维度** + 本 Lv **新增维度**。

```
Lv1 ── 路人/一次性 ──── 2 维
  ├─ 一句话摘要
  └─ [可选] ★ 复用例

Lv2 ── 功能角色 ────── Lv1 全部 + 3 维
  ├─ 摘要/身份（升级）
  ├─ 标签（新增）
  ├─ 驱动·功能（新增）
  └─ [可选] ★ 复用例（继承）

Lv3 ── 重要配角 ────── Lv2 全部 + 4 维
  ├─ 身份（升级：姓名/年龄/关系）
  ├─ 标签（继承）
  ├─ 驱力（升级：想要/恐惧/背叛条件）
  ├─ 叙事功能（新增）
  ├─ 弧线（新增）
  └─ 登场场景表（新增）

Lv4 ── 主角/核心对立 ── Lv3 全部 + 7 维
  ├─ 身份（完整版：种族/职业/原型/出场状态）
  ├─ 标签（继承）
  ├─ 心理（新增：性格特质/决策逻辑/核心缺陷）
  ├─ 驱力（升级：表层/深层/黑暗/恐惧四层 + 冲突分析）
  ├─ 能力（新增：优势/中等/弱项 + 成长路径）
  ├─ 演化（新增：阶段弧线 + 弧线类型 + 不可逆点）
  ├─ 网络（新增：关系图谱 + 演化摘要）
  ├─ 叙事功能（升级：全局叙事定位）
  └─ 弧线（升级：完整类型 + 不可逆点）
```

### 维度矩阵一览

| 维度 | Lv1 路人 | Lv2 功能 | Lv3 配角 | Lv4 主角 |
|:------|:-------:|:-------:|:-------:|:-------:|
| **摘要/身份** | ✅ 一句话 | ✅ 升级 | ✅ 升级 | ✅ 完整版 |
| **标签** | — | ✅ **新增** | ✅ 继承 | ✅ 继承 |
| **驱动·功能** | — | ✅ **新增** | ✅ 升级 | ✅ 四层完整 |
| **叙事功能** | — | — | ✅ **新增** | ✅ 全局定位 |
| **弧线** | — | — | ✅ **新增** | ✅ 完整+不可逆 |
| **心理** | — | — | — | ✅ **新增** |
| **能力** | — | — | — | ✅ **新增** |
| **演化** | — | — | — | ✅ **新增** |
| **网络** | — | — | — | ✅ **新增** |
| **设计目的** | ✅ | ✅ | ✅ | ✅ |
| **叙事能力** | ✅ | ✅ | ✅ | ✅ |

## 每级设计目的 & 叙事能力

### Lv1 · 路人/一次性

| 字段 | 内容 |
|:-----|:------|
| **设计目的** | 填充世界观，营造场景真实感。让读者感觉到"这个世界不只有主角在动，还有无数普通人在生活" |
| **叙事能力** | 承载单个场景功能：传递一句关键信息、制造一个瞬间氛围、提供一个背景对比。用后即弃，不为后续情节承担责任 |

### Lv2 · 功能角色

| 字段 | 内容 |
|:-----|:------|
| **设计目的** | 为特定叙事功能服务。作者需要一个人物来完成某个固定的叙事动作，完成后角色退场或降为背景 |
| **叙事能力** | 推动单幕情节：作为信息管道释放情报、作为催化剂触发主角决策、作为冲突放大器制造阶段性紧张 |

### Lv3 · 重要配角

| 字段 | 内容 |
|:-----|:------|
| **设计目的** | 承载独立动机，跨幕存在，形成次级弧线。读者对这个角色有情感期待，角色自己的选择会产生有分量的后果 |
| **叙事能力** | 与主角产生深度互动（伙伴/导师/恋人/宿敌）；承载读者次级情感投射（安心/心疼/愤怒）；为关键情节提供不可替代的推动力 |

### Lv4 · 主角/核心对立

| 字段 | 内容 |
|:-----|:------|
| **设计目的** | 贯穿全文的灵魂。承载故事核心主题，读者通过这个角色体验整个世界的意义 |
| **叙事能力** | 驱动全篇情节走向——角色弧线 = 故事主梗；承载读者全部情感投射；所有关键转折由这个角色的决策触发；也是读者的最终认同锚点 |

## ❌ 错误示例

### WRONG 1：维度倒灌

```
一个 Lv2 功能角色的角色卡：
## 能力
| 优势项 | 战斗 S 级 |
| 弱项   | 谋略 C 级 |
```
❌ 错误：「能力」是 Lv4 独占维度，Lv2 最多写到「驱动·功能」。
✅ 正确：一个 Lv2 功能角色的角色卡只写到标签 + 驱动·功能。

### WRONG 2：选级膨胀

```
一个只出现半章的酒馆老板 → 选了 Lv3 → 写了弧线/叙事功能/心理驱力
```
❌ 错误：酒馆老板只做一件事（递消息），Lv1 足够。
✅ 正确：先问"这个角色出现几幕？有没有独立弧线？" → 判断为 Lv1 → 只写一句话摘要。

### WRONG 3：跨级混用字段

```
Lv1 路人的角色卡里写了"驱力：想要/恐惧/背叛条件"
```
❌ 错误：「驱力」是 Lv3 才出现的升级维度，Lv1 只有一句话摘要。
✅ 正确：Lv1 只写一句话摘要 + [可选] ★ 复用例，不做任何维度扩展。

### WRONG 4：Lv3 角色遗漏了 Lv2 的「标签」维

```
一个 Lv3 配角卡：写了身份+驱力+叙事功能+弧线，但没有标签
```
❌ 错误：Lv3 = Lv2 全部维度 + 新增维度，标签是 Lv2 新增维，Lv3 必须继承。
✅ 正确：逐项对照维度矩阵，确保 Lv3 包含摘要/身份 + 标签 + 驱动·功能（Lv2 全部）+ 本级新增维度。

### WRONG 5：跳过选级直接开写

```
看到一个配角 → 直接打开 Lv4 模板 → 从心理到能力写满 16 维
```
❌ 错误：未做选级判断就加载模板。
✅ 正确：先判断叙事权重 → 选 Lv → 加载对应 Lv 模板 → 逐维填写。

## 跨赛道复用说明

模板使用**通用写作术语**，不绑定特定题材：

- 如需题材特定字段（如网文的"境界/修为"、西幻的"职业/神性"、科幻的"植入体/改造"），消费方 skill 在对应 Lv 的扩展占位处按需附加
- **不改模板本身** — 模板只保留跨赛道通用字段

## 异常与边界条件

| # | 场景 | 触发条件 | 处理动作 |
|:-:|:-----|:---------|:---------|
| 1 | 级别选不定 | 角色介于两个 Lv 之间 | 先选低一级填写，末尾标注 `※ 待升级至 LvN` |
| 2 | 模板文件缺失 | schema/LvN-*.md 不存在 | 按本 SKILL.md 的维度矩阵手写，**不编造字段** |
| 3 | 消费方跳过本 skill | bookstrap/plot 未加载模板 | 不强制中断——本 skill 是推荐性标准，非阻塞 |
| 4 | 题材扩展冲突 | 消费方追加字段与已有维度重叠 | 追加字段放对应 Lv 末尾的 `【题材扩展】` 区块 |
| 5 | 角色设计中途合并 | 两个角色合并为一人 | 取较高 Lv 的模板填写，合并两者驱动 |
| 6 | 临时角色意外复出 | Lv1 角色在后续章节再次出现 | 从 Lv1 升级到 Lv2（补标签+驱动·功能），不重写摘要 |
| 7 | 用户要求批量创建角色 | 一次需要创建 5+ 角色卡 | 逐角色按叙事权重判断级别，分别加载对应模板，不批量套用同一级别 |
| 8 | 消费方 skill 不读取本 schema | 下游 skill 直接硬编码角色字段 | 本 skill 不阻塞管线，但产出标注"角色卡未使用 04-pop-novel-character-schema 标准" |
