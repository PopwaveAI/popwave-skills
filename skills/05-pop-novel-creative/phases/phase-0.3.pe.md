# Phase 0.3：参考书策略

> 该文件同时作为执行指令和参考指南，无需加载外部 ref 文件。

## 前置条件
story-engine 方向已锁定（含 fusion_method + colliding_elements + collision_point）。

## 核心定位

**Phase 0.3 不做拆书——拆书是 deconstructor 的专业。**

Phase 0.3 做的是 deconstructor 做不到的事：**基于本故事的创意方向，决定"拆什么书"和"重点看什么"**。它是 deconstructor 和 creative 之间的策略层——给 deconstructor 一个靶子，拿到结果后只提取本故事需要的部分。

---

## 执行步骤

### Step 1 — 从碰撞点推导"我需要学什么"

不是泛泛地"找同类书"——是从本故事最脆弱的地方出发，找出最需要外部验证的问题。

```
碰撞点 → 需要验证什么 → 需要看什么书

示例（修真 × 克苏鲁）：
  碰撞点 = 飞升的终点是变成旧日支配者
  → 需要验证 1：有没有类似"修炼真相是恐怖"的先例？（找有克系元素的修真/西幻）
  → 需要验证 2：克系的世界观展开节奏一般多快？（找纯克系作品做节奏参考）
  → 需要验证 3：读者对"修炼有意义吗"这种存在主义问题接受度如何？（找有类似哲学重量的网文）
  → 需要验证 4：修真 + 恐怖这种跨类型的爽点是怎么混合的？（找跨界成功案例）

示例（修真 × 公务员）：
  碰撞点 = 编制体系 vs 逆天之道
  → 需要验证 1："体制内"叙事爽点在哪？（找官场/职场/种田文——不一定是修真）
  → 需要验证 2：幽默 + 修仙的平衡点？（找《我师兄实在太稳健了》这类反套路修真）
  → 需要验证 3：有没有其他"两套评价体系并行"的成功案例？（找非修真的——如《间谍过家家》= 杀手×家庭主夫）
```

### Step 2 — 形成"观察清单"

每个验证需求对应 deconstructor 的一个具体 T 维度。不是"拆了就行"——是"看 T1 的哪个字段、T4 的哪个指标"。

| 验证需求 | 需要看 deconstructor 的 | 关键指标 |
|:---------|:---------------------|:---------|
| 克系修真有没有先例 | T1（力量体系） / T4（剧情全貌） | 世界观的"恐怖感"是不是随着力量提升而增加？ |
| 克系展开节奏 | T2（世界观展开） / T6（叙事设计） | 每卷释放多少世界真相？读者什么时候第一次感到"不对劲"？ |
| 存在主义主题接受度 | T4（剧情全貌）卷1快照 / T7（文风DNA） | 哪一章让读者第一次意识到"修炼可能没有意义"？怎么写的？ |
| 跨类型爽点混合 | T4 M1/M2 交汇频率 | 恐怖元素和升级元素是交替还是同时？ |

### Step 3 — 选书（基于战略匹配，不只是类型匹配）

```
选择标准（优先级从高到低）：
  1. 本故事最脆弱的维度有成功先例 → 必选
  2. 核心碰撞点有类似处理方式 → 必选
  3. 类型相同但碰撞逻辑不同 → 可选（用于差异化分析）
  4. 类型不同但某个维度值得借鉴 → 可选（用于跨域对照）

最终保留 2-4 本。
如用户已有明确对标书 → 直接加入列表，跳过搜索。
如搜索不可用（WebSearch 不可用） → 降级为本地知识库 + 用户推荐。
```

### Step 4 — 触发 deconstructor

```
策略：优先用 Lv1（Phase S，~25min）

场景判断：
  ├─ 选的书恰好有现成拆解产出 → 直接加载，读观察清单对应的维度
  ├─ 没有拆解产出 → 触发 deconstructor Phase S（Lv1 快速拆解）
  │    在触发时传入观察清单 —— deconstructor 知道重点看什么
  └─ 用户后续要求深拆 → 触发 deconstructor Phase 0-4（Lv2 全量）

Lv1 已经产出的内容完全够 Phase 0.3 用：
  story-engine.yaml → 参考书的核心假说和碰撞方法
  Lv4-主角参考卡   → 参考书主角的设定方式
  卷1起终点快照    → 参考书开头和卷末的状态变化幅度
```

### Step 5 — 差异化决策（基于拆书数据，不是猜测）

deconstructor 产出后，基于实际数据填写差异化表：

```
✅ 借鉴 | [具体的规则/节奏/技巧] | [来自 deconstructor 哪份产出] | [我们怎么用]
❌ 偏离 | [具体为什么不沿用的某个做法] | [来自 deconstructor 发现的问题] | [我们的替代方案]
🆕 空白 | [deconstructor 显示所有参考书都没做的事] | [为什么我们判断这里有机会]

示例（deconstructor 数据驱动）：
  ✅ 借鉴 | T4 显示《诡秘之主》的克系认知以"序列魔药"为媒介逐步释放
         | 我们也可以用一个媒介（如"修为"）作为认知载体，但媒介本身的故事不同
  ❌ 偏离 | T2 显示《诡秘之主》的信息不对称主要在"主角知道读者不知道"
         | 我们的故事需要"读者和主角一起发现"——因为未知才是恐怖的来源
  🆕 空白 | T1 显示五本参考书的力量体系都是"越强越自由"
         | 我们的力量体系是"越强越接近不是自己"——这是全新命题
```

### Step 6 — 反哺 story-engine

差异化表不是附加品——它会反哺宪法：

```
差异化发现 → 影响 story-engine 的哪个字段

"所有参考书都是越强越自由" →
  → constitutional_bounds.world_rules.hard 追加："力量提升永远伴随自主权减少"
  → selling_point_validation.differentiation_check 追加第 N 条

"参考书的信息不对称全是主角知道读者不知道" →
  → constitutional_bounds.narrative.baseline 修正："读者和主角信息同步"
```

## 产出

存入 `_参考书分析/观察清单+差异化.md`：

- 观察清单（验证需求 → deconstructor 关键指标映射）
- deconstructor Lv1 产出索引（哪些文件覆盖了哪些需求）
- 差异化三列表（每项含：发现 + 数据来源 + 对宪法的影响）
- story-engine.yaml 的修正记录（如有）

**不产出独立的"参考书分析/{书名}.md"**——那是 deconstructor 的产出。Phase 0.3 只产出**本故事需要知道什么**。

## 与 deconstructor 的分工

```
deconstructor 做的事（Phase 0.3 不做）：
  - 逐章精读
  - 产出 T1-T7 独立分析文件
  - 提取文风DNA

Phase 0.3 做的事（deconstructor 做不了）：
  - 基于本故事的碰撞点判断需要看什么
  - 从 deconstructor 产出中提取本故事相关的部分
  - 比较多个参考书在同一个维度上的差异
  - 把外部发现翻译为对本故事宪法的修正
```
