# Phase 0.5：样品试读 ★ 核心闸门

> 该文件同时作为执行指令和参考指南，无需加载外部 ref 文件。
> 
> **这是 creative 最核心的相位。样品不通过 = 退回 Phase 0。不可跳过。**

## 前置条件
- story-engine.yaml v3 已锁定（含 constitutional_bounds + selling_point_validation）
- 金手指设计已完成（_设计笔记/金手指设计.md）

## 为什么必须做这一步

追问是语言的游戏。用户说"对就是这感觉"时，他确认的是他**想象中的**故事。样品试读是把想象变成文字——只有读到文字，用户才知道真正的感觉是什么。

这个相位回答的问题不是"这个创意好不好"，而是："**我读的时候，这是我想要的那个故事吗？**"

---

## 执行步骤

### 1. 确定样品段落

从前三章的范围内，选择一个**正在发生的事**作为样品段落。优先级：

| 优先级 | 段落类型 | 为什么选它 |
|:------:|:---------|:-----------|
| ★★★ | **开头第一段** | 这是用户想象中的"第一印象"。开头的 tone 和节奏决定了整本书的感觉 |
| ★★☆ | **主角第一次用金手指** | 验证金手指的呈现方式是否符合用户期待。是爽感爆发还是渐入佳境？ |
| ★☆☆ | **第一次冲突/对抗** | 验证冲突的质感。用户要的是智斗、碾压还是势均力敌？ |

**不要写**：
- 设定介绍段落（"这个世界有三千大道..."）——那是给 world 的，不是给用户的
- 角色独白/内心戏为主——用户需要通过事件感受 tone
- 跳过时间的大纲式叙述——要写具体的、正在进行的一段时间

### 2. 写作 500-2000 字

**写作约束**：
- 严格遵守 `constitutional_bounds` 的所有边界。如果宪法说"禁止恋爱线喧宾夺主"，样品中不能出现暧昧桥段
- 体现 `selling_point_validation` 中承诺的卖点。如果承诺"每5章至少1次正面战斗"，样品段落必须包含战斗或战斗的前兆
- 500 字为硬下限，2000 字为硬上限。不追求字数，追求验证效率
- 不追求完美文笔——样品的目的不是出版，是确认方向

**写作检查（写完自查）**：
```
□ tone 是否符合 constitutional_bounds.tone.allowed？
□ 有没有触犯 constitutional_bounds.tone.forbidden？
□ 主角的行为是否符合 constitutional_bounds.protagonist？（never 和 always 都检查）
□ 有没有违反 constitutional_bounds.world_rules.hard 中的任何一条？
□ 叙事节奏是否符合 constitutional_bounds.narrative.baseline？
□ 读完这个样品，读者能否感受到 selling_point_validation 中的卖点？
```

### 3. 展示给用户

输出 `_样品试读/样品-v1.md`。展示时只问一个问题：

> **"读起来是这个感觉吗？"**

不要问"你觉得写得怎么样"、"需要调整哪里"——这些开放式问题会引入噪音。只验证**感觉是否对**。

### 4. 用户确认路径

```
用户反馈？
  ├─ "对就是这感觉" 
  │   → 样品签字（见下方格式），锁定当前 engine，进入 Phase 0.6
  │   → story-engine.yaml 尾部追加 sample_approval 块
  │
  ├─ "不太对，感觉XXX不对"
  │   → 记录偏差点（什么感觉不对？哪一段让你出戏？）
  │   → 退回 Phase 0 重新追问（方向级偏差）
  │   → 追问完 → 重写 engine → 重写样品
  │
  ├─ "方向对但写法不对"
  │   → 记录写法偏差（节奏偏快/偏慢？语气太冷/太热？视角太远/太近？）
  │   → 调整 constitutional_bounds 的 narrative 或 tone 子块
  │   → 重写样品 v2，不退回 Phase 0
  │
  └─ "再看看别的段落"
      → 写样品 v2（换一个场景），继续验证
```

### 5. 样品签字格式

样品通过后，在 story-engine.yaml 尾部追加：

```yaml
sample_approval:
  sample_version: "v{N}"
  sample_file: "_样品试读/样品-v{N}.md"
  confirmed_by: "用户"
  confirmed_at: "YYYY-MM-DD"
  note: "用户确认：对就是这感觉"
```

**签字后才能进入 Phase 0.6**。world 检查到 sample_approval 缺失 → 退回 creative。

## 样品不通过处理

### 重写循环规则

| 偏差类型 | 处理 | 退回层级 |
|:---------|:-----|:---------|
| 方向偏差（故事核心不对） | 退回 Phase 0 重新追问 | Phase 0 |
| Tone 偏差（氛围不对） | 调整 constitutional_bounds.tone → 重写样品 | Phase 0.5 |
| 主角偏差（主角行为不对） | 调整 constitutional_bounds.protagonist → 重写样品 | Phase 0.5 |
| 节奏偏差（太快/太慢） | 调整 constitutional_bounds.narrative → 重写样品 | Phase 0.5 |
| 卖点偏差（样品没体现卖点） | 检查 selling_point_validation → 可能需要调整卖点定义 | Phase 0.5 或 Phase 0 |

### 三次不通过

如果重写 3 次（v1/v2/v3）用户仍不满意 → **暂停管线**。输出"创意分歧报告"：

```
创意分歧报告
版本历史：
  v1 — 用户反馈：[XXX不对]
  v2 — 用户反馈：[YYY不对] 
  v3 — 用户反馈：[ZZZ不对]

偏差模式分析：[每次反馈的共同点]
建议：
  ① 降低创意复杂度（如果偏差是因为概念太模糊/太复杂）
  ② 换参考书对标（如果偏差是因为缺乏明确锚点）
  ③ 先写更长的样品找感觉（2000→5000字，让用户看到更多）
```

用户选择方向后，重新开始。
