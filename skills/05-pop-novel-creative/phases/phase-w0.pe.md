# Phase W0：跨域素材聚合 ★ 强制（HARD GATE）

> 该文件同时作为执行指令和参考指南，无需加载外部 ref 文件。
>
> v1.4.0: 从 world 回流 creative 并前置。跨域素材是元素碰撞的燃料——没有足够的材料，碰撞不出好东西。

## 为什么前置

创意不是凭空生成的。用户给一个碎片（"我想写战锤灰骑士 × D&D 博德之门"），Agent 的创意碰撞质量取决于它脑子里有多少跨领域知识。Phase W0 在 Phase 0 元素融合**之前**执行——先把跨域素材喂进去，让碰撞有原材料。

**没有燃料的火花 = 假火花。后面一切都是空谈。**

## 前置条件

用户已给出创作方向（碎片/想法/模糊设定均可）。不要求完整——W0 本身就是为了补全碎片。

## 执行步骤

### 1. 卖点词拆解

从用户给的碎片中提取 3-5 个核心关键词。不是 story-engine 里的词（那个还没产出来）——是用户碎片的原生关键词。

例：用户说"战锤灰骑士 × D&D 博德之门" → 关键词：`灰骑士` `帝皇` `灵能/亚空间` `博德之门` `圣者浩劫` `D&D冒险`

### 2. 跨域搜索

按以下六大领域搜索。每关键词 1-2 次 WebSearch，覆盖其中至少 4 个领域：

地理/环境 → 社会/组织 → 生物/怪物 → 技术/魔法 → 文化/哲学 → 心理/情绪

### 3. 素材蒸馏

每方向 ≥2 个种子，标注来源/关联度/可转化性。

### 4. 关联标注

每个种子末尾标注：**这个素材能点燃什么方向？**（一句话联想）。这个标注直接喂给 Phase 0 的元素联想步骤。

## 质量导向检查表

| 检查层级 | 标准 | 判断方式 |
|:---------|:-----|:---------|
| L1 可定位 | 至少 2 个种子被明确标记为"可嵌入哪个维度" | 检查种子末尾是否有维度标记 |
| L2 有结构 | 至少 1 个种子是该领域独有的结构（不是"魔法=火焰"这种通用知识） | 这个种子是"谁都知道"还是"只有这个领域的人才知道"？ |
| L3 可碰撞 | 每个种子有"能点燃什么方向"的联想标注 | Phase 0 用得上——不只是素材堆，是有方向的燃料 |

**质量标准**：一个"该领域专家才知道的冷知识"比 10 个"百度百科通用词"有用。

## HARD GATE 说明

Phase W0 不可跳过。没有跨域素材的创意碰撞 = 信息茧房里的碰撞。**先采集，后碰撞——燃料不足不进 Phase 0。**
