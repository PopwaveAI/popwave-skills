# Phase W1：拆书融合

> 该文件同时作为执行指令和参考指南。
>
> v1.4.0: 从 world 回流 creative 并前置。拆书数据是元素碰撞的另一种燃料。

## 为什么前置

用户给创意时，往往已经提到了对标书（"借鉴深渊主宰""像诡秘那种感觉"）。与其等 Phase 0 碰撞完再回去拆书——不如先拆、先吸收，让碰撞时脑子里已经有了对标书的技法参考。

**有对标书的碰撞 ≠ 没有对标书的碰撞。** Phase 0 的元素融合在 W1 之后执行，Agent 已经知道对标书的节奏/密度/分层方式，碰撞出的方向天然差异化。

## 前置条件

- 用户提到了对标书（≥1 本）
- deconstructor 已对该书产出至少 Lv1（story-engine + 主角参考卡 + 卷1快照 + 文风指纹）

> 如果 deconstructor 尚未产出：先触发 deconstructor Lv1（~25min），等产出后再执行 W1。无对标书 → 跳过，标注"无锚点书分析，跳过拆书融合"。

## 强制规则

```
有对标书 → Phase W1 强制执行（不看 = 假对标）
无对标书 → 跳过。标注"无锚点书分析，跳过拆书融合"。
```

## 目的

将 deconstructor 对锚点书的产出物进行结构化融合，提取可复用的规则注入后续 Phase 0 的碰撞。**核心动作是提取参考值**——标注"可以是这个幅度""这个结构的原理在此"等（而非照搬）。

## 执行步骤

### 1. 定位 deconstructor 产出

读取 `_参考书/` 目录下对应锚点书的文件，确认存在：
- T1-力量体系规则手册
- T2-世界观展开
- T4-剧情全貌（含卷1起点/终点快照）
- T5-人物阵营
- T6-叙事设计技法
- T7-文风DNA指纹

### 2. 逐 T 提取参考值

- **T1 力量体系**: 阶层差/通胀幅度/瓶颈设计 → 作为后续力量体系的参考幅度
- **T1 升级年表**: 提取前 50 章的升级节奏密度 → "原著平均几章一次成长反馈？前期密度和后期密度差多少？" → 标注为「节奏参考，非本项目排期」
- **T2 世界观展开**: 信息释放节奏/世界观展开速度
- **T4 剧情全貌**: 卷1起点/终点快照作为参考锚点 → 帮助理解"卷边界怎么设"
- **T5 人物阵营**: 角色密度/阵营分裂方式
- **T6 叙事设计**: 节奏模板/悬念链路
- **T7 文风DNA**: 叙事哲学原则

### 3. 产出融合摘要

存入 `_参考书分析/拆书融合摘要.md`：

```markdown
# 拆书融合摘要
> 来源：《{书名}》 | 日期：YYYY-MM-DD

## 力量体系参考
- 阶层差：[参考幅度]
- 突破节奏：[参考值]

## 世界观展开参考
- 释放节奏：[参考值]

## 卷边界参考
- 卷1起点：[关键点]
- 卷1终点：[关键点]

## 叙事设计参考
- 高潮间隔：[参考值]
- 悬念链路：[模式]

## 角色密度参考
- 核心角色：[≤N人]
- 配角：[≤N人]
```

### 4. M1/M2 转译

**为什么需要**：deconstructor T4 的 M1/M2 是对锚点书的分析——同名但不同源，直接复用会导致因果错位。

**执行**：
- 继承：节奏规律（密度/露出方式/交汇频率）
- 不继承：具体事件
- 自检：如果转译后的 M1 事件和锚点书有 3 个以上相同 → 退回

### 5. 向 Phase 0 注入

融合摘要产出后，Phase 0 的元素碰撞环节必须读取融合摘要。碰撞时的 Agent 已知道：
- 对标书的节奏密度
- 对标书的分层方式
- 对标书的"惯用手法"

**这些数据让 Phase 0 的方向 sketch 天然带着"我怎么不同"的意识。**

### 6. 红线

❌ 有对标书但拆书成果不读取 → Phase 0 不准启动
