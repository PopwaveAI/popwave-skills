## 质量红线

| # | 红线 | 说明 |
|:-:|:-----|:-----|
| ❌1 | **宪法未锁定不进 world** | 故事引擎.md 未锁定时不得进入任何 Phase |

## 核心哲学

> **不在"发明"世界——在兑现宪法。**

1. **宪法是种子** — 故事引擎.md 的核心假说和碰撞点定义了世界的根本规则
2. **跨域素材是原料** — creative 产出的跨域素材蒸馏.md 为 L1 设定提供跨领域参考
3. **拆书数据是镜鉴** — creative 产出的拆书融合摘要.md 帮助理解对标书的节奏/密度（如有）
4. **L1 设定是枝干** — 六维度是对宪法的展开
5. **数值体系是骨架** — 宪法中"力量从哪来/谁最强"的量化
6. **宪法文档是契约** — 世界宪法.md 作为 plot Step 0 的审计矩阵

**判断标准**：任何一个设定，问"这句话在宪法哪一条里有根？"——能指出来，留；指不出来，砍。

> 执行指令：`phases/phase-1.pe.md` | 深度展开：`phases/phase-1.2.pe.md` | 交叉关联：`phases/phase-1.3.pe.md`

- 故事引擎.md 已锁定（来自 pop-novel-creative）
- creative 产出就位：跨域素材蒸馏.md + 拆书融合摘要.md（如有对标书）
- deconstructor T1/T2/T4 如有即加载为参考素材

### 核心原则

3. **六件套闭环** — 01-06 形成逻辑环，断环即退回

**产出**：`00-原始设定/L1-元设定层/01-06.md`

## Phase 3：角色卡初版

> 加载 pop-novel-character-schema 的 Lv1~Lv4 模板

### 前置条件
- Phase 2 稳定性检验通过

### 核心步骤

1. **选级**：按 character-schema 确认每个角色的 Lv（Lv1=路人 / Lv2=功能性 / Lv3=重要配角 / Lv4=主角）
2. **填写**：加载对应 Lv 的模板 → 填写 → 对照拆书 T3/T5 角色参考（如有）
3. **产出**：`状态/角色/{角色名}-角色卡.md`

### 角色卡分级消费规则

| 角色级别 | 适用对象 | 字段数 |
|:---------|:---------|:------|
| Lv1 | 一句话描述即可的路人 | ~3 |
| Lv2 | 有功能但不出彩的角色 | ~8 |
| Lv3 | 重要配角 | ~15 |
| Lv4 | 主角 | 全字段 |

## 异常与边界条件

| 场景 | 触发条件 | 处理动作 |
|:-----|:---------|:---------|
| **宪法更新** | 故事引擎.md 被 creative 修改 | 标记受影响 L1 维度 → 重推 → 连锁重建下游 |
| **L1 单篇修订** | 发现 L1 矛盾 | 修改 → 重跑交叉关联 → 重跑稳定性 → 检查下游 |
| **deconstructor 缺失** | 用户无锚点书 | 数值用平台默认节奏。creative 产出的拆书融合摘要标注"无锚点书" |
| **角色卡模板缺失** | schema/LvN 不存在 | 按维度矩阵手写，不编造字段 |
| **宪法与 L1 不一致** | 提取约束时发现 L1 模糊 | 先修 L1 → 重跑稳定性 → 再提取宪法 |
| **起点快照与宪法冲突** | Phase 5 产出与宪法矛盾 | 优先修正起点快照；若宪法需修订 → 退回 creative |
