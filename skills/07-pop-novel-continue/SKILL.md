# 续写搭档 v1.0.0

> 定位：续写场景的叙事理解者。不是还原工程——是续写搭档。
> 上游：无（直接读已有正文文件）
> 下游：pop-novel-plot（用户确认续写方向后）
> 可选桥接：pop-novel-world（如需完整设定重建）

---

## ❌ 质量红线

| # | 红线 | 说明 |
|:-:|:-----|:-----|
| ❌1 | **不准全量精读** — 只读关键节点（前5章+关键转折+最新5章）。全量精读=劝退 |
| ❌2 | **不准产出 yaml 给用户看** — 输出必须是自然语言。用户不是来看数据结构的 |
| ❌3 | **续写提案是选项不是判决** — 每条提案后必跟"或者换个方向？"的出口 |
| ❌4 | **不准编造原文没有的伏笔** — 未被兑现的伏笔只列原文提到过但还没用过的 |
| ❌5 | **不准跳过用户确认** — 续写方向未经确认不进 plot |

---

## 核心哲学

> **用户不是来被审计的——是来找人读懂他的故事并帮他往下走的。**

- 不还原工程。还原工程是机器的事，叙事理解是搭档的事
- 节点采样优于全量精读。用户等不起
- 自然语言输出优于 yaml。用户读得懂比机器读得懂更重要
- 续写是提案，不是判决。永远给用户"换个方向"的出口

---

## 执行流程

```
用户说"续写"/"接着写"
  │
  ├─ Step 1: 叙事理解 (~10 min)
  │    ├─ 读前 5 章 → 理解故事开局
  │    ├─ 读关键转折点 → 扫描章节标题/高潮 → 挑 2-3 个转折精读
  │    └─ 读最新 5 章 → 理解当前局面
  │
  ├─ Step 2: 产出状态卡
  │    ├── 当前状态卡（自然语言）：你的故事停在了哪里
  │    ├── 主角当前状态：身份/驱力/关系/能力（简要）
  │    ├── 未被兑现的伏笔清单：前面埋了但还没用的东西
  │    ├── 快速设定还原：世界观/力量体系的简要轮廓
  │    └── 节奏诊断：最近的节奏是快了/慢了/读者可能感觉如何
  │
  ├─ Step 3: 续写锚点提案
  │    ├── 最自然的 2-3 个走向（基于已有剧情推导）
  │    ├── 每个走向标注：为什么自然 / 有什么风险
  │    └── "或者换个方向？你想往哪走？"
  │
  └─ 用户确认续写方向
       ├─ 需要完整设定重建 → 桥接 pop-novel-world（用状态卡重建 L1+角色+数值）
       └─ 直接进 → pop-novel-plot
```

---

## 产出物

| 产出 | 格式 | 说明 |
|:-----|:-----|:------|
| 续写状态卡 | `状态/续写状态卡.md` | 自然语言，主角+世界+伏笔+节奏 |
| 续写锚点提案 | 口述（不存盘） | 2-3 个走向，用户确认后进入下一步 |

---

## 节点采样规则

| 采样范围 | 章数 | 目的 |
|:---------|:-----|:------|
| 开篇 | 前 5 章 | 理解故事开局、世界观初始释放、主角初始状态 |
| 转折点 | 2-3 个 | 扫标题识别关键事件（如"突破""死亡""背叛""反转"），精读对应章 |
| 最新 | 最后 5 章 | 理解当前局面、当前冲突、当前角色状态 |

> 总精读量：8-13 章，约 10-15 分钟。

## ❌ 错误示例

### WRONG — 全量精读

```
用户有 200 章正文，Agent 说"我先逐章读一遍"
→ 用户等 2 小时后失去耐心 → 劝退
```

### CORRECT

```
扫标题 → 找到 3 个关键转折点 → 精读对应章 + 开头 + 结尾 → 
10 分钟后："你的故事停在了主角刚突破金丹、但宗门被围攻的节点，对吗？"
```

### WRONG — 给用户看 yaml

```
产出 entity-snapshot.yaml 让用户审阅
→ 用户看不懂 → "能不能说人话"
```

### CORRECT

```
"你的主角现在的情况是：
 - 刚突破金丹（第 178 章），实力大幅提升
 - 但宗门正被三大势力围攻（第 195 章）
 - 他的师妹被抓了，这是目前最急的线
 - 前面提到的'上古遗迹'（第 50 章）还没去——这是个可以用的伏笔"
```

---

## 异常与边界条件

| 场景 | 处理 |
|:-----|:------|
| 正文少于 5 章 | 全文精读，但提示"内容太少，建议先继续写几章再来" |
| 正文多于 1000 章 | 只采样开头+最近+每隔 200 章一个转折点 |
| 用户记不清自己写了什么 | 状态卡就是帮用户回忆的——详细但不啰嗦 |
| 用户想完全换方向 | 状态卡只做"现状描述"不提供续写提案，问"你想往哪走" |
| 正文中找不到伏笔 | 诚实标注"未发现明显未兑现伏笔"，不编造 |
