## 一、四级爽点设计总览

| 级别 | 设计者 | 设计位置 | 谁承接 | 设计本质 |
|:-----|:------|:--------|:------|:---------|
| 小 | **chapter-design**（自设计自承接） | 单章事件链 | chapter-design | 一个干脆利落的动作/事件 |
| 中 | **plot**（Canvas 描线）+ **design**（事件执行） | 跨章 Canvas 铺垫→释放 | chapter-design | 一条子线经过 ≥3 章铺垫后释放 |
| 大 | **plot**（位置+蓄力） | 幕级 Canvas | chapter-design（按位置执行） | 剧情格局发生变化 |
| 特大 | **plot**（位置+多线汇聚） | 幕级/卷级 Canvas | chapter-design（按位置执行） | 全书承诺兑现 |

---

## 二、小爽点：自设计自承接

> plot 不设计小爽点。小爽点完全由 chapter-design 在事件链层面自设计、自承接。
> 本章节只为 plot 提供理解基础——你知道小爽点是什么，但不在 Canvas 中硬性规划。

### 2.1 定义

> 小爽点 = 一次干脆利落、杀伐果决的斩杀/制服/打脸，或一次可感知的升级/变强/赚钱/地位跃迁，或一个即用即走的社会热点梗植入。主角不废话、不犹豫、不解释。读者读完 ≈ 3 秒，反应 = "嗯，爽。"

### 2.2 三种类型

| 类型 | 设计要点 | 一句话示例 |
|:-----|:--------|:---------|
| **斩杀/打击** | 一个动作终结，不要多回合 | 暗巷背刺两恶棍，捂住嘴堵回惨叫，擦干血迹摸走钱袋 |
| **打脸/压制** | 碾压性碾压，不要势均力敌 | 全场最强绝招被一耳光破掉，"废物"二字收尾 |
| **升级/觉醒** | 可感知的质变，等级/地位/金钱跳过去 | 当场升级，伤口发痒体能回满，秒杀全场剩余敌人 |

### 2.3 设计建议（plot 可选参考）

```
在 Canvas 中，如果某条线在一章内只是"干脆利落地干掉了某个杂兵"或"升了一级"，
可以给该线标记 payoff_level=小。
但这完全不是硬性要求——chapter-design 会自行保证每章 ≥ 5 个小爽点。
```

## 三、中爽点：Canvas 描线设计（plot 核心）

### 3.1 设计定义

> **中爽点设计 = 在 Canvas 中画一条线：确定这条子线的铺垫窗口和释放窗口。**
> 你不需要设计具体事件（那是 chapter-design 的事），你只需要决定：
> - 哪条线、在哪些章处于"铺垫状态"（payoff_level=空）
> - 哪条线、在哪一章"释放"（payoff_level=中）

### 3.2 设计规则

```
每条线（主线1/主线2/主线3/支线1/支线2）的铺垫→释放采用"波浪节奏"：

蓄力（空）→ 蓄力（空）→ 蓄力（空）→ 释放（中）→ 休息（空）→ 蓄力（空）→ 释放（中）

规则：
1. 每条线连续 ≥3 章空 → 下一章必须释放（不能让读者觉得"这条线忘了"）
2. 释放后允许休息 2-3 章再进入下一轮蓄力
3. 每章至少有 1 条线处于释放状态（不能出现"全空了"的章节）
4. 如果某章所有线都是空/小 → 把最近一条快释放的线提前到这里
```

### 3.3 Canvas 填写流程

```
③ 填写节奏笔记
④ ★ 标注每条线的 payoff_level：
    - 空 = 铺垫/建设/埋伏笔（这条线在动，但没到释放）
    - 小 = 干脆利落的斩杀/升级/打脸（可选标注）
    - 中 = 这条线在这一章有一次可感知释放
    确保每章至少有 1 条线 = 中
④☆ 跟踪铺垫→释放节奏:
    - 每条线连续 ≥3 章只有空/小 → 下一章必须中
    - 释放后允许该线再空 2-3 章再释放（波浪节奏）
⑤ 汇总 payoff_map：逐章统计 ≥中 → total + 线号列表
```

## 四、大爽点：格局重画设计

### 4.1 设计定义

> **大爽点设计 = 在 Canvas 中找一条 payoff_level 从"空"连续 N 章后→突然跳到"大"的线。**
> 不是一条子线收束——是多章蓄力后**故事语境变了**。

### 4.2 判定器

释放之后，读者能不能清晰地感觉到"从这一章起，故事的语境变了"？

### 4.3 设计方法

```
① 在 Canvas 中扫描所有线：
   - 哪条线的蓄力已经覆盖了半卷到一卷？
   - 这条线的释放会改变什么？（不是"主角变强了"——是"世界不一样了"）
   
② 如果认定一条线应该做"大"释放：
   - 该释放章 payoff_level 标记为 大
   - 该释放章前 2-3 章标记为"蓄力期"（叠加压缩压力）
   - 释放后，下一章该线的语境描述要不同于之前（地图重画）

③ 大爽点间隔保证：
   - 扫描本幕所有大爽点章号
   - 相邻大爽点间隔 ≤ 5 章
   - 如果超过 → 把中间某条线的释放提前
```

### 4.4 与中爽点的区别

```
中爽点：一条子线释放 → "终于"（情绪释放）
大爽点：故事格局变化 → "地图重画了"（语境变化）

区别验证：
   释放后，下一章的主角的线描述是否变了？
   "在裂缝前观察" → "穿过裂缝到了另一边"  √ 大
   "在裂缝前观察" → "退回来休整"             × 只是中
```

## 五、特大爽点：全书承诺兑现设计

### 5.1 设计定义

> **特大爽点设计 = 找到 Canvas 中多条线的 payoff_level=大 汇聚到同一章或连续两章的时刻。**
> 不是"这条线收了"——是"原来这本书写的是这个"。

### 5.2 判定器

读者读完这一章后会在心里把前面整本重新翻一遍？觉得"前面那么多章就是为了这一章"？

### 5.3 设计方法

```
① 填完 Canvas 后扫描所有 payoff_level=大 的格子
② 找多线汇聚章（≥2 条线同为大的章）→ 特大爽点候选
③ 幕约 1/2-2/3 处选一个做幕中爆发
④ 幕末最后一章做幕末收束
⑤ 每卷至少 1 个特大爽点

如果一整个幕只有 1 个多线汇聚点：
   → 选它做幕末收束
   → 在幕中约 1/2 处强行设计一个：把一条快到释放的线提前释放，或让两条中线汇聚
```

## 六、责任归属总表

| 设计层级 | plot 设计 | chapter-design 执行 |
|:---------|:---------|:-------------------|
| 小爽点 | 不参与 | 自设计自承接（≥5/章） |
| 中爽点 | Canvas 中画线（哪条线、哪章铺、哪章放） | 按 Canvas 线号设计事件，标注 ★中爽点 |
| 大爽点 | Canvas 中标记 payoff_level=大，前 2-3 章蓄力 | 按指定章号设计全幕最高密度事件链 |
| 特大爽点 | Canvas 找多线大汇聚章，标记 payoff_level=特大 | 按指定章号设计全幕最大规模事件链 |
