# 诡异游戏写作引擎 v2

> **版本：v2.0** · 2026-05-18
>
> **定位**：为「诡异游戏/系统流/规则怪谈/恐怖求生」赛道设计的专用写作引擎。AG管线经验全量吸收，POP叙事效率保留。
>
> **核心理念**：问题的关键不是「注入多少信息」，而是「注入什么性质的信息」。
> - ❌ 否定式禁令（不要/禁止/避免）→ 消耗认知，压制创作 → 14条禁令版均分4.6
> - ✅ 肯定式元数据（定位/作用/节奏/伏笔）→ 建立心智模型，不压制创作
>
> **对标参考**：基于四本已分析对标书的94-96个写作片段提取的模式库

---

## 一、你与v7.4-v7.7的区别

| 维度 | v7.4-v7.7（旧正文引擎） | v2（诡异游戏专用） |
|:---|:---|:---|
| 架构复杂度 | 4层（章纲→导演→碎片→QC） | 2层（章级Prompt→正文） |
| 注入方式 | 4字段碎片Prompt（极简+禁令） | 12字段整章元数据（肯定式+信息锚） |
| 子Agent依赖 | 依赖场景卡节奏参照 | 内置4种场景模板，无需额外资产 |
| 赛道针对 | 通用 | 诡异游戏/系统流/规则怪谈 |
| 对标书模式 | 无 | 四本书深度分析注入 |
| 写一章耗时 | 2-3轮（导演+碎片+桥接+QC） | 1轮（Prompt→正文→QC） |
| 核心问题 | 禁令越多写得越差 | 元数据越多写得越清晰 |

### 为什么不走v7.x的4层管线？

**诡异游戏赛道不需要。**

- v7.x的导演层+碎片拆分是为解决"通用引擎如何适配任意场景"而设计的
- 诡异游戏的场景类型有限（恐怖/解谜/获得能力/战斗/日常），每种有清晰的节奏模板
- 元数据锚直接告诉LLM"这是什么章节"——效果等于导演层判断，但成本更低
- 对标书分析证明：**诡异游戏的核心不在结构复杂度，而在信息差控制和节奏锚点密度**

## 三、12字段章级元数据详解

每次写作前，按以下模板构建章级Prompt。**所有字段都是肯定式的（告诉模型"要做什么"），没有一条是禁令（"不要做什么"）。**

### 模板

```markdown
# 第X章 写作指令

## 元数据锚（12字段）

### 1. 章节定位
[事件章 / 角色章 / 主题章]

### 2. 核心作用
[推进主线 / 剧情转折 / 信息揭示 / 情绪铺垫 / 爽点爆发]

### 3. 节奏类型
[渐进紧张] — 从平静→异常→危机逐级升温
[高频切换] — 短场景快节奏，每300字一个微钩子
[铺垫积蓄] — 信息释放慢，为下章爆发蓄力
[爆发释放] — 冲突/战斗/获得的高能段落

### 4. 悬念操作
[埋设 / 强化 / 回收 / 留白]
例：埋设「主角发现自己不是唯一被选中的」→ 下几章再回收

### 5. 伏笔操作
[埋设 / 强化 / 回收]
例：回收前文「碎片地图」→ 揭示碎片指向学校医院

### 6. 转折程度
★★☆☆☆ — 微转折（配角有新线索）
★★★☆☆ — 中转折（规则出现裂痕）
★★★★☆ — 大转折（世界观的裂缝）
★★★★★ — 颠覆性（系统真相/阵营逆转）

### 7. 信息密度
[≤ 2个新概念] — 超过2个读者记不住
新概念清单：① [概念名+一句话] ② [概念名+一句话]

### 8. 本章简介（一句话）
[让LLM快速定位本章在整个故事中的位置]

### 9. 角色/道具/场景
- 核心人物：[谁在本章有戏份]
- 关键道具：[什么物品被使用/发现/改变]
- 场景：[空间位置]
- 时间压力：[无 / 模糊 / 明确倒计时]

### 10. 章末情绪锚点
[读完本章读者应该停留在什么情绪上]
例：紧张（主角决定进地下室） / 悬念（手机画面突然黑了）

### 11. 下一章预告
- 标题：[下章标题]
- 定位：[下章的类型和作用]
- 承接方向：[情绪/悬念/空间的过渡方式]

### 12. 场景类型标签
[恐怖场景 / 解谜场景 / 获得场景 / 战斗场景 / 日常过渡]

## 正文
[以下写正文...]
```

## 五、四本对标书的写作模式（可注入）

从四本已下载对标书中提取的关键写作数据，可作为「样式参照」注入章级Prompt。

### 各书的TOP1学什么

| 对标书 | 最值得学的单一模式 | 一句话说明 |
|:---|:---|:---|
| **《我的诡异人生》** | **系统作为认知工具** | 模拟器不给提升只给信息差，主角靠推理不靠外挂 |
| **《诡舍》** | **冰箱法则** | 关键信息藏在日常物品中，读者自己拼出真相 |
| **《13路末班车》** | **社畜恐怖** | 普通人视角+职场压迫，日常感中的异常最吓人 |
| **《规则怪谈》** | **番茄节奏模板** | 第1章就死人，第2章就上三楼见人皮 |

### 通用铁律（四本共有的成功特征）

```
□ 对话占比 ≥ 30%（旁白不能超过60%）
    → 诡异人生 >40%，诡舍 ≈35%
    → 用对话推进信息，不要靠旁白说明

□ 信息差设计：读者始终比主角知道多一点点
    → 让读者焦虑："为什么主角还没发现？"
    → 例子：冰箱里的人肉第8章才揭晓，但读者第4章就有线索

□ 能力代价绑定
    → 每次获得必须绑定代价（寿命/理智/自由）
    → 没有免费的午餐，否则爽感变成无敌流

□ 快慢交替：不允许连续3章紧张
    → 紧张章→喘息章→紧张章
    → 喘息章也有信息量（不是注水）

□ 每章至少一个钩子（章末3行）
    → 未解答的问题 / 刚做出的危险决策 / 逼近的威胁
    → 这决定了读者会不会点"下一章"
```

## 六、QC自检清单（快速版）

写完正文后，逐项检查。不通过不交付。

```markdown
## QC自检

### 1. 信息密度
□ 新概念 ≤ 2个（超过说明信息爆炸）
□ 每个新概念都有使用场景（不是为酷而造）

### 2. 叙事效率
□ 对话占比 ≥ 30%（数一下字数比例）
□ 说明性旁白 ≤ 3行（系统提示/规则说明）

### 3. 情绪锚点
□ 每300字至少1个情绪锚点（紧张/悬念/释放/期待/恐惧）
□ 章末3行有钩子（读者想点下一章）

### 4. 写作质量
□ 无「不是…而是…」否定式描写（"不是恐惧，而是一种…"）
□ 无「感觉到/意识到」冗余前缀（直接写发生了什么）
□ 无说明性开头（"XX是什么"而非"XX的出现让…"）

### 5. 诡异感检查
□ 至少一个"日常中的异常"（温迪戈在超市排队这种）
□ 恐惧源于未知 > 源于视觉冲击（暗示 > 直白描写）
□ 读者有信息差焦虑（"为什么主角还没明白？"）

### 6. 下一章承接
□ 章末情绪锚点与下章元数据的承接方向一致
```

## 七、工作流

```
1. 准备章级元数据（12字段）
   → 从章纲/幕纲提取
   → 参考四本对标书模式（可选注入）

2. 注入场景类型模板（可选）
   → 如果本场景类型明确，在Prompt末尾加入对应模板的关键节奏标记
   → 模板是节奏参照，不是内容（LLM自己写内容）

3. 生成正文
   → 一次性输出完整章节
   → 字数：1500-2500字（看节奏类型决定）

4. QC自检
   → 见第六节清单
   → 不通过 → 修改元数据后重新生成（不修正文，修元数据）
```
