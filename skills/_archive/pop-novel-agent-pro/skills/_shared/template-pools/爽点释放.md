# 爽点释放场景模板

> 适用于：核心爽点兑现、打脸、获得、升级、逆转
> 核心原则：**铺垫短，释放快，锚定长**

---

## 段落结构

```
┌─ 段1 · 前奏（≤3句） ─────────────────────────────┐
│  功能：交代"这件事终于要来了"                        │
│  内容：一句话情境 + 一个期待主体的动作/视角          │
│  节奏：快切，不给读者反应时间                        │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段2 · 释放（核心段） ─────────────────────────────┐
│  功能：兑现读者等待的爽点                            │
│  内容：主体事件发生（打脸/获得/升级/逆转的证据）      │
│  写法：结论前置。先写结果，再写过程（或不写过程）     │
│  节奏：一句话完成核心动作，剩余补视角反应              │
│  字数：150-300字（不要拉长）                         │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段3 · 锚定（1-2段） ─────────────────────────────┐
│  功能：让读者对"这件事发生了"产生情绪确认             │
│  内容：一个第三方视角的反应（配角震惊/环境变化/旁白） │
│  节奏：比段2慢半拍，给读者"消化"的空间                │
│  注意：不拆感觉——一个反应说一次就够，不逐层解剖       │
└────────────────────────────────────────────────────┘
```

---

## 关键约束

- **不铺垫酝酿**——爽点的前奏不需要读者心理建设，直接进入
- **不解释意义**——释放后不需要"这意味着什么"的解释段，让读者自己感觉
- **不用"突然"**——爽点不是"突然"发生的，是"终于"发生的
- **锚定段不超过段2的字数**——反应不能比事件长

---

## 节奏标记

```
段1：快 → 段2：极快 → 段3：中速
总字数：500-800字
```
