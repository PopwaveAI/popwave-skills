## 写作原则

### 1. 叙事质感由文风DNA控制

- 第三人称限制视角（除非文风DNA指定全知）
- 对话一句一个信息点（不堆砌）
- 长分析段后接短句作为换气
- 新概念通过角色判断自然带出（不单独设段说明）

**禁止**：
- "不是A，而是B"的否定式描写
- 「不是A不是B是一种C」三段否定结构
- 以"不是"开头引段（不是被推开的。不是被砸开的。→ 直接写结果）
- "他感到/他意识到/仿佛"等冗余前缀
- 在对话中塞超过2句的内心分析

### 2. 笔墨分配由设计说明控制

设计说明中标记为"核心场景"的段落充分展开（3-4段），"过渡场景"点到为止（1-2段）。

### 3. 世界观插槽按perspective字段决定视角

- `perspective: 角色内心` — 通过角色的判断/评价自然带出
- `perspective: 全知旁白` — 叙述者跳出，直接向读者解释

### 4. 情绪节奏

参考设计说明中的"读者情绪时刻表"安排段落的情绪起伏：
- 开局前300字需要有抓住读者的锚点
- 每个核心场景至少有一个情绪释放点
- 章末锁定在钩子上

### 5. 上一章锚定

读上一章结尾原文，确保本章开局的语感、视角、情绪与上一章自然衔接。不重复上一章结尾已有的动作，不忽略上一章结尾留下的钩子。

## 章末钩子

阅读设计说明中的钩子类型。确保：
- 章末最后3行落在钩子上
- 不提前释放钩子的答案
- 不连续2章使用相同类型的钩子

## ⭐ 写后自评（v3.3 新增）

正文写完后，在提交给 QC 之前，自己先回答三个问题。目标是**补充不足，不重写整章**。

```
① "我写的战斗场景和{锚定章}的画面密度差距在哪里？"
   → 找到最薄弱的 1 段，补 2-3 句感官细节

② "这一章我给了读者几个'停下来'的时刻？"
   → 标记至少 1 处"读者可能滑过去"的位置 → 加一个细节锚点

③ "字数差了多少？差的篇幅本来应该写什么？"
   → 字数不足 → 补一个被压缩的场景（不是注水）
```

**自评不通过 ≠ 重写。自评 = 找到最核心的缺失，补一段。**

## 写作后自查（v3.3）

- [ ] 读者画像：我知道在给谁写吗？
- [ ] 锚定章：我的写法接近 Director 选定的参考片段吗？
- [ ] 事实骨架中的事件序列全部覆盖了吗？
- [ ] 没有新增实体/概念吗？
- [ ] 章末钩子和设计说明一致吗？
- [ ] 没有否定式结构（"不是A而是B" / "不是A不是B是一种C" / 以"不是"开段）

## ★ AC 对照（v4.0 深度整合 — 写后自评后、QC 前执行）

写完正文并完成自评后，逐条对照 spec.md 的 Acceptance Criteria 做自我检查：

```
### AC 对照检查

来源: .trae/specs/<change-id>/spec.md

□ AC-1（正文合规）:
  - 字数在目标范围 ±15% 内: [实际字数]
  - 覆盖了所有 FR-1 场景: [是/否]
  - 未涉及 Non-Goals 排除内容: [是/否]
  - {实体名} 计数 ≥ 约束要求: [实际计数]

□ AC-2（爽点落地）:
  - FR-2 设计的爽点类型在本章中有明确落点: [是/否]
  - 如果否 → 在自评中注明"爽点未落地，需 QC 确认"

□ AC-3（钩子有效）:
  - 章末最后 3 行符合 FR-5 的钩子类型: [是/否]
  - 答案未被提前释放: [是/否]

□ AC-4（风格一致）:
  - 画面密度接近锚定章水平: [自评]
  
□ AC-5（无越界实现）:
  - 没有涉及 Non-Goals 排除内容: [是/否]
```

**AC 对照规则**：
- 所有 AC 标记"是" → 通过，进入 QC
- 1-2 项"否" → 修复后重验
- ≥3 项"否" → 回退到 Director 阶段
