# QC 自检清单（v4.0 Spec 深度整合）

> 正文产出后，作者（或主Agent）过以下5项基础检查，再追加 5 项 Spec 合规检查。
> 基础 5 项全过 → 交付定稿。Spec 5 项全过 → 通过 Spec 闸门。
> 1-2项不过 → 修正后重新检查。
> ≥3项不过 → 回退诊断，检查素材完整性。

---

## 自检项

```markdown
## 快速自检

□ 新概念 ≤ 2个（超过说明信息爆炸，分2章放）
□ 无"不是…而是…"句式（否定式描写的标志）
□ 章末最后3行有钩子（问题未回答 / 决策未做 / 威胁靠近）
□ 对话占比看着合理（≥25%，肉眼估一下）
□ 无"他感到 / 他意识到 / 仿佛"类冗余前缀

5项全过 → 交付定稿，更新全局摘要
1-2项不过 → 微调后重新检查
≥3项不过 → 回退到上下文组装，检查素材完整性
```

---

## ★ Spec 合规检查（v4.0 新增 — 对应 checklist.md）

```markdown
## Spec 合规检查

□ S-AC-1: 字数在 spec.md FR-3 定义的目标范围 ±15% 内
□ S-AC-2: 正文覆盖了 spec.md FR-1 定义的所有场景
□ S-AC-3: spec.md 中 Non-Goals 排除的内容未被涉及
□ S-AC-4: {实体名} 计数 ≥ spec.md Constraints 中的要求
□ S-AC-5: 章末钩子类型与 spec.md FR-5 一致

5项全过 → Spec 闸门打开
1-2项不过 → 回退到 Pass 2 修复
≥3项不过 → 回退到 Director 阶段重审
```

---

## 不检查什么

- 情绪锚点密度 — 上下文够，Pass 2自然知道怎么给
- 比喻密度 — 由palette控制，不单独检查
- 具体字数 — 由设计说明的笔墨优先级控制
