# Phase 0：L0 灵魂层对齐

**目标**：确认作品灵魂——为什么是你来写这本书？为什么读者要看？

**前置条件**：用户已提供至少「书名」「核心创意」「核心爽点」的初步描述。

## 执行步骤

1. **灵魂三问**
   - Q1 为什么是我来写这本书？（作者独特经验/执念/知识储备）
   - Q2 读者为什么非看不可？（不可替代的体验/情感/认知）
   - Q3 这本书的核心情绪是什么？（热血/治愈/悬疑/爽/虐/幽默……）

2. **产品定位卡 (PRD)**
   - 输出 `00-原始设定/L0-产品层/PRD.md`
   - 要素：产品名称、一句话卖点、目标读者画像、核心体验承诺、差异化优势、风险预判
   - PRD 是全书最高优先级约束文件，后续所有决策须可回溯至 PRD

3. **核心创意压力测试**
   - 测试1：电梯测试——30 秒内向陌生人讲清楚
   - 测试2：长篇拉力测试——能支撑 200 万字以上？
   - 测试3：平台适配测试——创意与目标平台读者画像匹配？
   - 测试4：差异化测试——和同类型 Top 20 作品的核心差异点
   - 通过后启动 Phase 0.3

**检查清单**：
- [ ] 核心爽点已明确
- [ ] 情绪价值已定义
- [ ] 一句话pitch能打动目标读者
- [ ] 目标平台已确定
- [ ] 对标书已确定（至少1本）
- [ ] 四项压力测试全部通过
