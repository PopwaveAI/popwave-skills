# Phase 1.2：L1 深度展开 ★ 自动执行 | 禁止跳过

**为什么**：Phase 1 的 L1 文件只有骨架，每个维度需深度展开才能成为写作时的可靠参考。不接受"先这样吧"的浅层产出。

**条件**：六篇 L1 文件骨架完成，跨域素材蒸馏报告就绪。

## 核心原则
- **逐篇执行，不可并行** — 每篇单独经历展开节点，完成后再进下一篇
- **素材注入优先** — 先看 Phase 0.5 素材库里有什么能用
- **画面锚点必须存在** — 每个子维度至少 1 个记忆锚点
- **不达标则继续** — 字数门槛不够就回头展开，不进下一步

## 深度展开节点（每篇L1文件执行一次）

```
【深度展开节点】
  ├── ① 素材注入 —— 从跨域素材蒸馏中提取本维度相关种子嵌入
  ├── ② 维度展开 —— 每子维度至少3条描述(定义+表现形式+剧情影响)
  ├── ③ 画面锚点 —— 每子维度至少1个记忆点(一句话画面/比喻/场景)
  └── ④ 达标检查 —— 不达标返回②继续展开
```
