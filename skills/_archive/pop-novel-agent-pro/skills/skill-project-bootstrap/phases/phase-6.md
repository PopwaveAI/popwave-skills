# Phase 6：超越性硬检查

**目标**：对 Boss 设计和数值体系进行超越性检查，确保长期叙事的可持续性。

**条件**：数值体系模板已创建（Phase 5）。

## 执行步骤

### 6.1 Boss 超越性检查

| 维度 | 标准 |
|------|------|
| 超越时机 | 至少提前10章铺垫，Boss战前后有节奏缓冲 |
| 超越代价 | 主角付出可量化代价（受伤/损耗/失去某物/决策困境） |
| 叙事功能 | 推动至少一个长期叙事线，纯数值填充的删除或重构 |

### 6.2 数值体系超越性检查

| 维度 | 标准 |
|------|------|
| 膨胀率 | 每次升级1.5x-3x，全卷跨度不超过初始1000x |
| 跨卷曲线 | 中期不可出现战力天花板，后期有递进感 |
| 玩家感知验证 | 读者对升级仍有期待？战力增长伴随叙事意义提升？ |

**产出**：
- `00-总控/数值体系/超越性检查报告.md`
- 每个 Boss 的三轴评分表
- 数值体系膨胀曲线图
- 修复建议
