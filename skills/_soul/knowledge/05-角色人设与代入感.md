# 角色人设与代入感

> 领域：内容创作 → 角色设计
> 适用：任何需要设计角色的小说/故事创作
> 来源：整合自人物与代入感.md、ibiling 人物塑造实战、知乎代入感 3 篇、灰骑士复盘、ESM v9.0 角色问题专项
> 目的：独立可读。读完能让任何一个角色有活人感。

---

> **让读者爱上角色的不是"他很完美"，而是"他是真实的"。**

---

## 一、主角人设铁律

### 第一章必须做的三件事

| 任务 | 写法 | 检查方式 |
|:-----|:-----|:---------|
| **给主角 1 个可记忆标签** | 外貌/口头禅/行为怪癖/职业特征 | 读者能否在这一章后说出主角的一个特点 |
| **让他做一个选择** | 面临困境时的决定，展示性格 | 这个选择是否反映了主角的核心价值观 |
| **给他一个具体的渴望** | 不是"变强"，而是"逃出地下室"/"治好妹妹的病" | 读者是否清楚主角"此刻"想要什么 |

### 主角设计公式

```
主角 = 欲望 × 困境 × 选择

欲望：驱动故事的核心动力（要够具体）
困境：阻止欲望实现的障碍（要够难）
选择：面对困境时的决策（要透露性格）
```

### 主角三种共鸣特性

| 特性 | 读者共鸣点 | 示例写法 |
|:-----|:----------|:---------|
| **同处境** | "我也遇到过这种事" | 被领导穿小鞋、被同学排挤 |
| **同渴望** | "我也想这样" | 渴望被认可、渴望变强、渴望保护重要的人 |
| **同性格** | "我也是这样的人" | 嘴硬心软、表面无所谓实际上很在意 |

不需要三个全中——**中一个就够了。** 但一个都不中的主角，读者不会在乎。

## 二、角色辨识度系统

### 让角色可区分的三个维度

写好角色卡时，填充以下三个维度：

#### 1. 说话方式

| 维度 | 示例 |
|:-----|:------|
| 语速 | 语速极快（连珠炮）/ 慢吞吞 / 间或有停顿 |
| 习惯用语 | "其实吧……"/"我跟你说"/"卧槽"/沉默良久开口 |
| 语气词 | "嗯"/"呵"/"喂"/"切" |
| 句式特征 | 短句多/长句多/反问多/自问自答 |

直接给示例比描述更有效：

```
❌ "他很冷漠"
✅ 他在一章里开口三次，最长一次三个字——"嗯。""走了。""不用。"
```

#### 2. 行为模式

| 压力类型 | 可能反应（各角色选用不同） |
|:---------|:-------------------------|
| 战斗压力 | 冲上去 / 躲起来 / 仔细观察 / 喊人帮忙 |
| 社交压力 | 怼回去 / 沉默 / 冷笑 / 假装听不懂 |
| 决策压力 | 立刻做选择 / 找别人商量 / 拖到最后一刻 |

**关键：同一压力下，不同角色反应必须不同。** 如果 A 和 B 在战斗中都"冲上去"——至少要让一个冲在前面、一个绕后。

#### 3. 至少一句"能记住的台词"

每个核心角色，在前 3 章内必须有一句能让读者记住的台词或行动。

```
张北辰（北帝镇诡录）：
  "我不知道我能不能守住" → 诚实、压力、倔强

左千户（北帝镇诡录）：
  "你是第一个让我觉得走了之后还想跟你喝一杯的" → 粗犷外表下的柔软
```

**如果写完后觉得所有角色的台词可以互换 → 角色没立住。**

## 三、代入感的工程

### 代入感公式

```
代入感 = 身份降维 + 处境共鸣 + 行为可模拟
```

#### 1. 身份降维

**起点越低越容易代入。**

| 高代入 | 低代入 |
|:-------|:-------|
| 废柴/穷小子 | 贵族/富二代（开局满级） |
| 被退婚/被开除 | 万人之上 |
| 普通人得到机遇 | 天生就是天命之子 |

不是不能写强者——但强者也要有弱点。读者代入的不是"全能"，是**"全能之下藏着的脆弱"**。

#### 2. 处境共鸣

读者的代入动机不是主角的能力，而是主角的**处境**：

```
让读者想的是：
  "如果是我，我会怎么办？"
  而不是：
  "他真厉害，但跟我有什么关系"
```

#### 3. 行为可模拟

读者在读动作描写时，身体会潜意识"模拟"：

```
"他紧握刀柄，深呼吸三次" → 读者不自觉地握拳
"他屏住呼吸，将匕首推进门缝" → 读者憋住气
```

在关键节点（战斗前/强敌前/重大决定前）调动身体的模拟反应，让读者"感觉自己在做"。

### 代入感进阶：丢人的小毛病

让主角有一个无关痛痒的"丢人毛病"——这是最快建立亲切感的方式：

| 毛病 | 效果 | 注意 |
|:-----|:-----|:------|
| 怕高 | 爬墙时手抖 | 别写成"恐高症"（过度医学化） |
| 路痴 | 找不到路必须问人 | 可用于制造偶遇/剧情推动 |
| 嘴硬 | 明明在意偏说不在乎 | 制造反差萌 |
| 抠门 | 为了几块钱纠结 | 适合穷人翻身的开局 |

**关键**：毛病只能写在**安全场景**下 — 读者会笑，不会真的讨厌主角。真正的生死关头，主角不能因为怕高而掉下去（除非这是成长弧的一环）。

## 四、角色成长弧

### 成长弧的底层逻辑

不是"等级从 1 到 100"，而是**核心信念的裂变**：

```
阶段一："我不知道我能不能做"  →  犹豫
阶段二："我必须做"             →  决心（经历第一次代价）
阶段三："我一个人做不到"       →  学会信任/合作
阶段四："不是只有我才能做"     →  传承/觉醒
```

### 成长弧检查

- [ ] 第一章的主角思维方式和最后一章不一样吗？
- [ ] 这个变化是一步一步发生的（不是突然"开悟"）吗？
- [ ] 每次成长都付出了可感知的代价吗？
- [ ] 成长后主角获得了某样新东西，但也失去了某样旧东西吗？

## 五、配角设计

### 配角的三种叙事功能

| 功能 | 定义 | 示例 |
|:-----|:-----|:------|
| **镜子** | 反射主角的性格 | 胆小配角对比勇敢主角 |
| **催化剂** | 推动剧情/触发主角行动 | 配角被杀 → 主角爆发 |
| **信息源** | 补充读者不知道的信息 | 懂行的配角解释世界观 |

配角失效的最大原因：**只有功能，没有活人感。** 上面三种功能可以同时存在，但配角至少要有一种以上。

### 配角速写清单

写配角时，不需要像主角那样深挖。但至少要回答：

- 他对主角的态度是什么？（崇拜/嫉妒/利用/保护？）
- 他在压力下的第一反应是什么？
- 他有什么让读者记住的一句话/一件事？

## 六、角色常见问题与修复

### 修复方案

| 问题 | 症状 | 根因 | 修复方向 |
|:-----|:-----|:-----|:---------|
| **性格辨识度为零** | 10 章说不出主角的三个特点 | 开书阶段缺乏行为锚定 | 每章给主角一个展示性格的小场景 |
| **角色趋同** | 所有台词可以互换 | 没有为每个角色设计说话方式 | 写角色卡时加"speech_style"字段 |
| **配角打酱油** | 出场后有剧情功能，无活人感 | 只有功能没有性格 | 让配角做一件展示性格的事 |
| **成长没体现** | 主角第一章和第一百章没区别 | 无跨章的性格进化设计 | 每 10 章复盘一次核心信念变化 |
| **背景板叔公** | 重要的长辈只存在于二手材料 | 没有让角色"活着"出场 | 大段回忆/手记都换成至少一段"当时的对话" |

---

## 七、铁律

> **如果写完后觉得"所有角色的台词可以互换"——那第一章没写主角，需要重写。**

> **主角的"弱"不是真的弱——是读者觉得他弱。废物开局最有代入感，但废物到第 10 章还不翻身 → 弃书。**

> **不要试图让读者"理解"主角。让读者"感受"主角就够了。**
