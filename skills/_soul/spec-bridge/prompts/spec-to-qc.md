# Spec → QC 注入 Prompt

> 职责：让 QC Agent 在纯感受报告之外，增加一层"Spec 合规"检查
> 使用时机：QC 三层介入时（大纲层/骨架层/正文层）
> 注入方式：追加到 QC 原有 prompt 尾部

---

## 注入内容

### 1. Spec 合规检查（追加在纯感受报告之后）

```
## ★ Spec 合规检查（可选 — 仅在发现偏离时报告）

### 范围检查
□ 被检查的内容没有超出 spec.md 的 Non-Goals 边界
  如果超出 → 记录具体偏离内容

### FR 覆盖检查
□ 检查内容覆盖了 spec.md 的所有 Requirements
  如果缺失 → 记录缺失的需求

### 预期读者情绪检查
□ 基于检查内容的实际阅读体验，spec.md Goals 中定义的目标读者情绪是否可达？
  如果不可达 → 记录原因
```

> **重要**：这层检查是**可选的补充**，不替代纯感受报告。
> 如果有合规问题，在纯感受报告的末尾追加一段"【Spec 合规备注】"。
> 不要因为发现合规问题就改变纯感受报告的内容和语气。

### 2. QC 报告尾部追加块

```
---

## 【Spec 合规备注】（仅在有发现时填写）

- ⚠️ 偏离项: [如有]
  - 偏离的 spec 项: [引用 spec.md 的具体条款]
  - 影响评估: [对读者体验的影响]
  - 建议方向: [仅标注"需关注"或"无需处理"]
```
