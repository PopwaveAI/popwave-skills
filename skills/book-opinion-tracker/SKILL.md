# book-opinion-tracker · 网文舆情追踪 v1.8

> **定位：搜口碑，不搜销量。** 8 个平台扫一遍，输出标准化舆情报告。
> 依赖 `cnovel-research` 做采集，本 skill 做分析 + 出报告。

## 执行流程（SOP）

### 第一步：输入书名 + 确认范围

**读什么：** 用户输入的书名。

**做什么：** 确认舆情追踪范围。
- 默认：扫全部 8 平台
- 可选：指定平台（如"只看龙空和知乎"）

**产出：** 任务确认清单（书名 + 平台列表 + 预期耗时）。

---

### 第二步：平台采集（调用 cnovel-research）

**读什么：** 第一步确认的平台列表 + 书名。

**做什么：** 按 cnovel-research 的采集策略逐平台获取数据。

| 采集策略 | 平台 | 工具 |
|:---------|:-----|:-----|
| 全自动采集 | 晋江/贴吧/知乎/B站/微博 | 平台专用 API/爬虫 |
| 混合采集 | 龙空 | AnySearch site: 搜索 + extract |
| 人工兜底 | 豆瓣/阅文/小红书/公众号/抖音/视频号 | AnySearch 扫标题 → 用户粘贴正文 |

**门禁：**
- [ ] 每个高可采集平台已采集 ≥10 条相关帖子/内容
- [ ] 已向用户发出人工粘贴请求（涉及低可采集平台时）
- ❌ 某平台采集结果为 0 条 → **退回**。检查关键词是否匹配或是否需要人工兜底。
- ❌ 用户未完成人工粘贴 → **退回**。等待用户粘贴后继续。

**产出：** 各平台原始采集数据（结构化/原始文本）。

---

### 第三步：分析数据 + 输出报告

**读什么：** 第二步采集的原始数据。

**做什么：** 按 `舆情报告模板.md` 输出标准化报告，写入 `.trae/archives/book-opinion-reports/`。

报告结构：
```
# 《书名》舆情报告

## 一、数据来源总览
## 二、正面评价摘要
## 三、负面评价摘要
## 四、争议焦点
## 五、读者画像特征
## 六、建议
## 附录：原始数据
```

**门禁：**
- [ ] 报告包含 ≥3 个平台的独立数据
- [ ] 每类观点至少附 1 条原文引用
- [ ] 标注了数据来源平台和采集时间
- ❌ 只有 1 个平台的数据 → **退回**。平台覆盖率不足，继续采集。
- ❌ 报告无原文引用 → **退回**。没有引用 = 没有证据。

**产出：** `{书名}-舆情报告-{日期}.md`

## WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 只扫 1-2 个平台就出报告 | 覆盖面不足，结论有偏差 | 默认扫全部 8 平台 |
| 直接给结论不附原文 | 无法验证 | 每条观点附带原文引用 |
| 人工粘贴的内容不做标注 | 真假难辨 | 标注"用户提供"区分 |
| 报告写成百科/简介而非舆情 | 偏离目标 | 聚焦"读者怎么看" |

## 异常与边界条件

| # | 异常场景 | 处理方式 |
|:-:|:---------|:---------|
| 1 | 某平台 API 限流/返回空 | 换 AnySearch site: 搜索做补充；仍不行则跳过并标注 |
| 2 | 所有平台均无相关讨论 | 告知用户"该话题在网文圈讨论度低"，出空白报告 |
| 3 | 用户不愿配合人工粘贴 | 只输出自动化采集平台的报告，标注未覆盖平台 |
| 4 | 书名过于通用（如"修真"） | 建议用户提供更精确的书名或作者名 |
| 5 | 采集到大量无关内容（同名不同书） | 加作者名/关键词缩小范围 |
| 6 | 平台改版导致采集工具失效 | 标注"该平台数据不可用"，走人工兜底 |
| 7 | 报告文件写入失败（路径/权限） | 尝试写入 `d:\popwave-skills\` 兜底 |
| 8 | 舆情涉及敏感/侵权内容 | 如实收录但标注争议性，不做价值判断 |
