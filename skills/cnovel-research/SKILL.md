# cnovel-research · 网文作者社区用户调研

> **定位：网文作者圈调研的瑞士军刀。** 整合 AnySearch 搜索引擎 + 各平台专用 API/爬虫 + 反爬绕过策略 + 高效关键词库。基于 2026-05 对 8 个网文作者社区的全量调研实战经验沉淀。

## 执行流程（SOP）

### 第一步：确定调研范围

**读什么：** 用户输入的调研话题（如"网文作者对 AI 写作的态度""起点签约门槛"）。

**做什么：**
- 确定目标平台列表（默认全平台或用户指定）
- 确定搜索关键词（从关键词库选取）

**产出：** 调研计划（任务编号 + 平台列表 + 关键词 + 预期采集方式）。

---

### 第二步：工具选择 + 数据采集

**读什么：** 第一步的调研计划。

**做什么：** 按决策树选择采集工具。

**工具选择决策树：**
```
用户发起调研请求
├─ 是多平台扫描吗？
│   ├─ 是 → 用 AnySearch batch_search + site: 同时扫 3-5 个平台
│   └─ 否 → 查"平台能力矩阵"表
│
├─ 目标平台正文拿得到吗？
│   ├─ ✅ → 优先用平台专用工具（aiotieba / crawl4weibo / B站API）
│   ├─ ⚠️ → AnySearch site: 扫标题 + 交互动作让用户粘贴正文
│   └─ ❌ → 查"人工兜底方案"，指导用户操作
│
├─ 需要帖子/视频详情内容（非仅标题）？
│   ├─ 是 + B站 → B站评论 API（免登录）
│   ├─ 是 + 贴吧 → aiotieba get_posts（需 HTTP 补丁）
│   ├─ 是 + 龙空 → AnySearch extract（部分可提取）
│   ├─ 是 + 微博 → crawl4weibo 含评论模式
│   └─ 是 + 豆瓣/阅文/小红书/公众号 → 走人工兜底
│
└─ 被反爬拦截了？
    ├─ 换个 User-Agent
    ├─ 换 HTTP→HTTPS 或反过来
    ├─ 加 Referer/Origin
    └─ 还不通 → 正文拿不到 → 启动人工兜底
```

**各平台专用工具调用：**

```bash
# AnySearch 通用
python tools/anysearch/scripts/anysearch_cli.py search "<query>" --max_results 20
python tools/anysearch/scripts/anysearch_cli.py batch_search --query "site:xxx.com AI写作" --query "site:yyy.com 网文AI"
python tools/anysearch/scripts/anysearch_cli.py extract "https://example.com/article"

# B站 API
GET https://api.bilibili.com/x/web-interface/search/type?search_type=video&keyword=<关键词>&page=<页码>
GET https://api.bilibili.com/x/v2/reply/main?oid=<aid>&type=1&mode=3&next=<页码>

# 晋江碧水
https://bbs.jjwxc.net/board.php?board=17&page=<页码>
（提取表格 <table class="olt"> 中帖子）
```

**Python 调用：**
```python
# aiotieba（贴吧）
import aiotieba
async with aiotieba.Client() as client:
    threads = await client.get_threads("小说", pn=1, rn=50)
    posts = await client.get_posts(tid, pn=1, rn=30)

# crawl4weibo（微博）
from crawl4weibo import WeiboClient
client = WeiboClient()
result = client.search_posts("AI写小说", page=1)
```

**门禁：**
- [ ] 每个平台采集结果 ≥10 条内容（或全量）
- [ ] 低可采集平台已向用户发出人工粘贴请求
- ❌ 某平台采集结果为 0 → 检查关键词，仍为 0 则标注"无相关讨论"
- ❌ 所有平台均返回 0 → **退回**。告知用户话题讨论度过低。

**产出：** 各平台原始数据（结构化 JSON / 文本 / 用户粘贴内容）。

---

### 第三步：分析 + 输出报告

**读什么：** 第二步采集的全部原始数据。

**做什么：**
- 提取关键引用
- 用户态度分层（激烈反对 → 沉默实用 → 实战派 → 营销号）
- 痛点拆解（可解决/不可解决/需规避）
- 输出报告

**产出格式：**
```markdown
# <平台名> <调研主题> 调研报告

> **数据来源**：<平台名> — <搜索词/版块>
> **采集方式**：<工具名>
> **数据规模**：<X> 个帖子/视频，<Y> 条回复/评论
> **采集时间**：<日期>

## 一、采集概况
## 二、用户真实态度（分层）
## 三、使用的工具/方法
## 四、痛点拆解
## 五、产品机会
## 六、产品/营销/风险三维建议
```

**门禁：**
- [ ] 报告含 ≥3 个平台的数据（多平台调研时）
- [ ] 每类观点附 ≥1 条原文引用
- [ ] 标注了采集方式和数据来源
- ❌ 报告无原文引用 → **退回**。没有证据的结论不做。
- ❌ 用户态度未做分层 → **退回**。必须区分不同声音。

**产出：** 标准化调研报告 Markdown 文件。

## 人工兜底方案

**适用于：** 豆瓣、阅文、小红书、微信公众号、抖音、视频号（需要登录态/Cookie 的平台）。

### 标准操作流程

1. **AI 用 AnySearch site: 搜索扫标题** — 找出该平台有哪些相关帖子/笔记，列出标题、链接、互动数。
2. **AI 选出 10-30 篇高价值内容** — 优先选高互动、标题直接命中的。列出链接。
3. **AI 告知用户需要人工操作：**

   > 自动抓取被拦了（需要登录态/Cookie），但我已用搜索引擎找到了 [X] 个相关帖子。你可以打开下面的链接，各 Command+S（或 Ctrl+S）保存页面，或直接粘贴帖子正文给我。发完了告诉我。

4. **用户发来内容后** — 跟自动化采集数据一样处理。

### 各平台人工操作指南

| 平台 | 如何打开 | 如何保存 | 注意 |
|:-----|:---------|:---------|:-----|
| **豆瓣小组** | 浏览器打开帖子链接 | Ctrl+S 保存 HTML，或复制文字粘贴 | 需已登录豆瓣账号 |
| **阅文 write.qq.com** | 浏览器打开帖子链接 | 同上 | 需已登录阅文/起点账号 |
| **小红书** | 手机 App 内打开更好 | 截图+复制文字，或 PC 端 Cmd+S | PC 端经常弹验证码 |
| **微信公众号** | 微信内打开文章 | 复制全文粘贴 | 搜狗微信搜索是定位最佳免费工具 |
| **抖音** | 手机 App | 看视频+复制评论区文字 | 评论区复制有限，可截图让 AI 读取 |
| **微信视频号** | 微信内 | 只能手动浏览+口述总结给 AI | 无公开链接 |

## 踩过的坑（已知问题）

| # | 问题 | 解决方案 |
|:-:|:-----|:---------|
| 1 | 贴吧 get_posts 超时 | 改源码 `aiotieba/api/get_posts/_api.py`，`scheme="https"` → `scheme="http"` |
| 2 | 豆瓣小组详情页 403 | 用 AnySearch `site:douban.com/group` 搜列表；详情用户手动保存 |
| 3 | 阅文 write.qq.com 搜索 API 返回有限 | 搜索 API 拿标题+回复数，配合 AnySearch site 补全 |
| 4 | 小红书 extract 返回空 | 需 Cookie + 动态渲染；AnySearch site 扫标题 → 用户手动 |
| 5 | 抖音搜索返回 code 2483 需登录 | 放弃自动化。如需数据，用户手动搜索后粘贴 |
| 6 | B站搜索 API 有时返回空 | 加 0.3-1s 延迟；换 User-Agent；分批次请求 |

## WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 只爬标题不爬正文就出报告 | 结论不可信 | 正文拿不到就走人工兜底，不跳过 |
| 4 个平台爬不到就只出 1 个平台报告 | 覆盖率不足 | 标注未覆盖平台，明确告知局限性 |
| 不区分用户态度直接给"大家都说好" | 片面 | 分层呈现：反对/中性/实用/营销号 |
| 调研结论不附原文引用 | 无法验证 | 每条关键观点带原文链接或引用 |
| 采集失败直接放弃不告知用户 | 沉默失败 | 告知用户原因，提供替代方案（人工兜底） |

## 异常与边界条件

| # | 异常场景 | 处理方式 |
|:-:|:---------|:---------|
| 1 | 所有平台 API 均返回空/报错 | 告知用户"当前平台普遍不可达"，建议改日重试 |
| 2 | 关键词过于宽泛返回数万条结果 | 追加二级关键词过滤，缩小到 ≤50 条相关结果 |
| 3 | 关键词过于精确返回 0 条 | 先用一级宽泛关键词扫，再手动筛选匹配 |
| 4 | 用户提供的调研话题与网文无关 | 告知"本 skill 仅覆盖网文作者圈"，建议换话题 |
| 5 | 用户拒绝人工粘贴（不愿配合） | 只输出自动化数据报告，标注"平台覆盖不全" |
| 6 | 平台改版导致采集脚本失效 | 走人工兜底方案，同时记录失效信息供后续修复 |
| 7 | 采集数据量大（>100MB） | 分批采集，先出初步报告再补充深度数据 |
| 8 | 采集到明显 AI 生成/水帖内容 | 按互动数和账号活跃度过滤，标注"疑似低质内容" |

## 调研工作流模板

### 场景 A：全平台快速扫描（2-3h）
1. AnySearch batch_search 扫全部平台 `site:`，每平台 10 条
2. 用平台专用工具深入 Top 3 平台
3. 整理关键引用 + 出简报

### 场景 B：单平台深度调研（4-6h）
1. 查能力矩阵定位最优路径
2. 高可采集平台 → 专用工具全量爬取（翻 5-10 页）
3. 热帖爬评论
4. 挑高质量引用 + 出报告

### 场景 D：人工兜底模式
1. AI 先用 AnySearch site: 扫标题 → 输出列表
2. AI 挑 10-30 篇 → 把链接发给用户
3. 用户手动打开 → 复制粘贴
4. AI 接收 → 正常出报告

### 场景 E：竞品工具调研
重点关注"用什么AI写文"类讨论：
- 知乎 > "写小说，哪个 Ai 好用？"
- B站 > 工具对比/横评类视频评论
- 微博 > "AI工具清单"类博文
- 豆瓣 > 工具测评类帖子
