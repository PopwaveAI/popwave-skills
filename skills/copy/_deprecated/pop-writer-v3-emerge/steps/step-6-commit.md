# Step 6：落盘+项目总控更新+弧线触发检查

> 管线: pop-writer-v3-emerge v1.2.0
> 类型: **主会话**
> 消费: Step 3修订稿（已通过质检） + 活记忆 + 种子文档 + 项目总控
> 产出: 正文落盘 + 项目总控更新 + 弧线触发判定

## 目的

将质检通过的修订稿落盘到正文文件。更新项目总控（章号+1/种子版本/弧线计数）。检查弧线校准触发条件。完成本章七步循环。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | Step 3修订稿（已通过Step 4质检） | 落盘对象 |
| 2 | `活记忆/活记忆.yaml`（刚更新的） | 节奏日志趋势+种子追踪超期预警+压力状态 |
| 3 | `种子文档.md` | 活跃线索状态+压力矩阵活跃阶段+版本号 |
| 4 | `项目总控.md` | 当前章号/种子版本/弧线校准计数 |

## 做什么

### 1. 正文落盘

将Step 3修订稿（已通过Step 4质检）写入 `正文/chXXX.md`：

```markdown
# chXXX · {本章标题}

{正文全文}
```

### 2. 项目总控更新

更新 `项目总控.md`：
- 当前章号+1
- 种子版本（如本章触发了种子生长，更新为新版本号）
- 弧线校准计数（距下次校准的剩余章数-1）
- 阶段状态（涌现写作中）

### 3. 弧线校准触发检查

检查是否满足弧线校准触发条件：

**定期触发：**
- 距上次校准已满10-20章（建议值15章）

**事件触发：**
- 活记忆任意组件触发预警：
  - 伏笔超期>20章未回收
  - 战力连续跳级
  - 连续5章低密度
  - 目的地20章无进展
  - 线索10章未触及
- 种子要素last_updated_ch > 15章（休眠候选）

**如触发：**
→ 提示用户进入 `pop-writer-v3-arc` 弧线校准

**如未触发：**
→ 继续下一章，回到Step 0

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 正文未落盘 | 补落盘后再继续 |
| 项目总控未更新 | 补更新后再继续 |
| 弧线校准触发条件未检查 | 补检查后再继续 |

## 产出

1. `正文/chXXX.md` — 本章正文落盘（修订稿）
2. `项目总控.md` — 章号+1/种子版本/弧线计数更新
3. 弧线触发判定结果（触发/未触发）

## 产出自检

- [ ] 修订稿已落盘到 `正文/chXXX.md`
- [ ] 项目总控已更新（章号+1/种子版本/弧线计数）
- [ ] 弧线校准触发条件已检查

## 本章七步循环完成

七步循环：本章规划 → 信息获取(强制化) → 调度创作子skill → 调度修订子skill → 调度质检子skill → 记忆更新+方向提示 → 落盘+项目总控更新

- 如弧线校准触发 → 交接给 `pop-writer-v3-arc`
- 如未触发 → 回到 `steps/step-0-chapter-plan.md` 开始下一章
