## 一、活种子哲学

种子不是一次性产出的静态文档，是随故事生长的故事引擎。在小说不同阶段，压力源、驱动力、金手指的活跃度都不一样——开局显性的医疗缺失可能在ch15后被解决，而强化剂压力源可能在ch20才浮现。活种子允许agent在写作过程中自主发现新要素并追加，在弧线校准时自主清理不活跃要素。

**活种子 vs 活记忆：**

| 维度 | 活种子 | 活记忆 |
|:---|:---|:---|
| 性质 | 故事引擎（方向性要素） | 状态账本（append-only记录） |
| 可变性 | 可生长可修剪 | append-only（除压缩） |
| 内容 | 六要素（压力/引擎/金手指/冲突/成长/目的地） | 七组件状态记录 |
| 作用 | 生成剧情的方向 | 记录已发生的剧情 |

## 二、生长：agent自主判断

### 核心原则

**权限全给agent。** 不设严格的生长门槛——agent在涌现写作中发现新压力源、新线索、新规则、新目标时，自主决定是否追加到种子。

### 判断原则

| 情况 | 判断 | 动作 |
|:-----|:-----|:-----|
| 新发现的要素能生成未来剧情 | 追加 | 更新种子+版本号+变更日志 |
| 只是当前章节的一次性事件 | 不追加 | 只记入活记忆 |
| 不确定是否能生成未来剧情 | 追加 | 宁多勿少，弧线校准时可修剪 |

### 生长场景（示例，非穷举）

| 触发场景 | 生长要素 | 示例 |
|:---|:---|:---|
| 发现新压力源 | 压力矩阵 | ch5发现强化剂依赖风险→追加压力源"强化剂" |
| 发现新线索 | 活跃线索 | ch8发现巫毒教与裂隙关联→追加活跃线索T006 |
| 确立新世界规则 | 世界规则 | ch10发现系统等级与精神力关系→追加世界规则 |
| 成长路径细化 | 成长路径 | ch12选择巫医职业→成长路径阶段4细化 |
| 发现新目标 | 目的地 | ch15发现裂隙本质线索→追加目的地 |
| 金手指新能力解锁 | 金手指 | ch18解锁L2能力→金手指能力模块更新 |
| 主角驱动力演化 | 主角引擎 | ch20从"求生"浮现"推翻制度"→深层驱动力激活 |
| 主角行为准则演化 | 主角引擎 | ch20从"不留活口"浮现"选择性留活口"→行为准则.行为禁忌更新 |

### 生长执行流程

1. revise子skill（Step 3）在重写过程中发现新要素 → 判断是否触发种子生长
2. 主会话（Step 4）机械执行修订记录的种子生长建议 → 更新种子文档对应要素
3. 更新该要素的`last_updated_ch`字段为当前章号
4. 版本号 minor+1（如 1.0.0→1.1.0）
5. 变更日志追加记录
6. 同步更新活记忆对应组件
7. 在event的`seed_growths`字段记录生长内容

## 三、版本管理（红线4）

### 版本号规则（语义化）

| 版本变化 | 触发条件 | 执行者 |
|:---------|:---------|:-------|
| major（X.0.0） | 种子要素结构变化（新增/删除要素类型） | 弧线校准 |
| minor（1.X.0） | 生长（追加新压力源/新线索/新规则/新目的地） | revise子skill判断→Step 4主会话执行 |
| patch（1.0.X） | 修剪/调整（关闭压力源、回收线索、调整细节） | 弧线校准 |

### 变更日志格式

```markdown
| 版本 | 章号 | 类型 | 变更内容 | 触发原因 |
|:-----|:-----|:-----|:---------|:---------|
| 1.0.0 | 0 | init | 初始化六要素 | 种子设计 |
| 1.1.0 | 5 | grow | 压力矩阵追加"强化剂" | ch5发现新压力源 |
| 1.2.0 | 8 | grow | 活跃线索追加T006 | ch8发现巫毒教与裂隙关联 |
| 1.2.1 | 15 | prune | 压力源"医疗缺失"标记已关闭 | ch15弧线校准，HP已稳定 |
```

### 种子文档头部版本元数据

```markdown
# 种子文档：{书名}

> 管线：涌现式写作v3 · 活种子
> version: 1.2.0
> lastUpdatedAt: 2026-06-25T00:00
> lastUpdateReason: ch8生长-追加活跃线索T006
```

### last_updated_ch新鲜度感知（v3.1新增）

种子文档要素切片中每条要素都携带`last_updated_ch`字段，记录该要素最后被更新的章号：

| 要素 | last_updated_ch | 含义 |
|:-----|:----------------|:-----|
| 压力矩阵 | {ch} | 压力矩阵最后生长/修改的章号 |
| 主角引擎 | {ch} | 主角引擎最后生长/修改的章号 |
| 金手指 | {ch} | 金手指最后生长/修改的章号 |
| 冲突轴+活跃线索 | {ch} | 冲突轴/线索最后生长/修改的章号 |
| 成长路径 | {ch} | 成长路径最后生长/修改的章号 |
| 目的地 | {ch} | 目的地最后生长/修改的章号 |

**新鲜度规则：**
- 初始化时全部为0
- 种子生长时，被更新要素的`last_updated_ch`更新为当前章号
- `当前章号 - last_updated_ch > 15` → 要素可能需关注（休眠/进化/修剪）
- revise子skill（Step 3）在种子生长判断时检查新鲜度
- 弧线校准时作为休眠判据

## 四、活种子与活记忆的同步关系

种子生长时，必须同步更新活记忆对应组件：

| 活种子要素 | 活记忆组件 | 同步动作 |
|:---|:---|:---|
| 压力矩阵 | 压力状态 | 种子生长新压力源→活记忆增加倒计时 |
| 冲突轴+活跃线索 | 种子追踪 | 种子生长新线索→活记忆增加伏笔 |
| 世界规则 | 世界规则 | 种子生长新规则→活记忆世界规则追加 |
| 成长路径 | 战力曲线 | 种子调整成长路径→活记忆战力曲线更新基准 |
| 目的地 | 目的地进度 | 种子增减目的地→活记忆目的地进度同步 |
| 金手指 | 角色状态 | 种子更新金手指能力→活记忆角色状态更新 |
| 主角引擎 | 角色状态 | 种子更新驱动力/行为准则→活记忆角色状态更新 |

## 五、修剪：弧线校准时执行（非本步骤）

修剪在弧线校准（pop-writer-v3-arc）时执行，不在涌现写作环中执行。

修剪场景（示例，非穷举）：

| 修剪场景 | 修剪要素 | 判定依据 |
|:---|:---|:---|
| 压力源已解决 | 压力矩阵 | 倒计时归零且不再激活 |
| 活跃线索已回收 | 活跃线索 | 线索悬念已兑现 |
| 目的地已到达 | 目的地 | 故事已涌现到该路标 |
| 线索长期休眠 | 活跃线索 | 连续10章未触及 |

**修剪原则：不删除历史，只标记状态。** 已关闭/已完结的要素移入 `写作资产/设定库/已废弃/`，供版本追溯。

## 六、防膨胀机制

PRD开放问题#1指出：权限全给agent后，agent可能过度生长导致种子膨胀。

防范措施：
1. **种子规模约束**：约600-1500字。种子超1500字时agent自主优先修剪（标记不活跃要素移入设定库/已废弃/）
2. **宁多勿少+弧线校准兜底**：生长时宁多勿少，弧线校准时修剪兜底
3. **一次性事件不生长**：只是当前章节的一次性事件，只记入活记忆，不追加到种子

## 七、agent自主判断的边界

| 可以做 | 不可以做 |
|:-------|:---------|
| 追加新压力源到压力矩阵 | 删除已有压力源（修剪归弧线校准） |
| 追加新活跃线索 | 关闭已有活跃线索（修剪归弧线校准） |
| 确立新世界规则 | 修改已有世界规则（世界规则只增不改） |
| 细化成长路径阶段 | 删除成长路径阶段（修剪归弧线校准） |
| 追加新目的地 | 删除已有目的地（修剪归弧线校准） |
| 更新金手指能力模块 | 删除金手指限制条件 |
| 激活主角深层驱动力 | 修改主角表层驱动力 |
| 细化行为准则（新增禁忌/调整战斗风格） | 修改驱动力三层（驱动力归seed设计） |

> 核心原则：生长=追加/细化，修剪=关闭/标记。涌现写作环只做生长，修剪归弧线校准。
