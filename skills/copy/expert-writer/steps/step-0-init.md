# Step 0：项目初始化 — 创建完整目录骨架

> **调用时机**：开新书之前，必须执行此步骤。
> **产出**：项目目录下创建完整骨架。
> **门禁**：❌ 不可跳过。不初始化骨架就开书 = 文件散落无归档、后续 skill 找不到目标目录。

---

## ❌ 读取协议（强制）

```
工具选择：skill_view（首选）或 Get-Content -Encoding UTF8 -Raw
❌ 禁止用 Read 工具读取 skill 文件（有行数限制，会截断）
```

---

## 目录骨架（v3.5）

```text
{项目根目录}/
│
├── 卷纲/                                [v3-plot 产出 / v3-arc 更新]
│   ├── 卷N-方向锚.md                     [v3-plot] 卷定位+起终点四维度+主冲突（≤300字）
│   ├── 幕NNN-名称.md                    [v3-plot 产出 / v3-arc 更新] 幕纲（活文档，含设定引用指针）
│   └── L3-NNN-名称.md                    [v3-arc 提炼 / v3-arc 更新] L3剧情线卡（跨多L2单元的剧情线追踪）
│
├── 写作参考/                             [统一参考库（原素材库+设定库合并）]
│   ├── 索引.md                           [v3-seed 初始化 / 全管线追加] 总目录（统一索引入口）
│   ├── 设定/                             [v3-seed 产出 ground truth] {S}
│   │   ├── 主角引擎.md                    驱动力+行为准则（跨单元不变）
│   │   ├── 金手指.md                      能力+限制+行动驱动
│   │   ├── 压力矩阵全量.md                三界压力源+交叉困境
│   │   ├── 冲突轴全量.md                  四层生态+活跃线索
│   │   ├── 核心卖点.md                    一句话+核心爽点+情绪承诺
│   │   └── 世界宪法.md（可选，复杂世界观）
│   ├── 知识沉淀/                         [pop-research / expert-writer WebSearch 查询结果] {D}
│   └── 已废弃/                           [v3-arc 修剪归档] {D}
│
├── 写作资产/
│   └── 文风库/                           [v3-seed 蒸馏产出，项目资产] {S}
│       └── {书名}.md                     文风DNA档案（修订层消费）
│
├── 活记忆/                               [v3-seed 初始化 / expert-writer 追加 / v3-arc 压缩]
│   └── 活记忆.yaml                       自然语言状态日志（baseline + events，每章一段话）
│
├── 正文/                                 [expert-writer]
│   └── chXXX.md
│
├── 弧线校准报告/                         [v3-arc]
│   └── arc-XX.md
│
└── 项目总控.md                           [expert-writer 每章更新]
```

## 文件类型标记

| 标记 | 含义 |
|:-----|:------|
| **{S}** | 静态 — 一次产出，只读不写 |
| **{D}** | 动态 — 持续维护，可被追加/修改 |
| **[skill]** | 产出者 — 哪个 skill 负责产出/维护该文件 |

> **v3.5关键变化：**
> - 种子文档.md **取消** → 功能被幕纲（卷纲/幕NNN-{名称}.md）+写作参考（写作参考/设定/）吸收
> - 素材库+设定库 **合并** → 写作参考/（统一参考库）
> - 新增卷纲/目录 → 幕纲 + L3剧情线卡 + 卷方向锚
> - 活记忆格式变更 → 自然语言段落（每章一段话），非7组件结构化YAML
> - 修订记录/ 目录取消 → revise receipt直接附在产出中，不独立存档

## 初始化操作

1. 检查 `{项目根目录}/项目总控.md` 是否存在
   - 存在 → 项目已初始化，跳过本步
   - 不存在 → 继续执行

2. **创建目录骨架**：创建上述所有目录（`卷纲/`、`写作参考/设定/`、`写作参考/知识沉淀/`、`写作参考/已废弃/`、`写作资产/文风库/`、`活记忆/`、`正文/`、`弧线校准报告/`）

3. **初始化写作参考索引**：创建 `写作参考/索引.md`（统一索引入口，取代原资料总索引.md）。初始骨架含3个分区，每个分区用表格格式（`文件 | 主题标签 | 章号 | 路径`），初始为空：

   ```markdown
   # 写作参考索引

   > 项目唯一参考索引入口。信息获取环节（Step2）强制读取本文件。所有文件均可复用，不是一次性消耗品。最后更新章号：—

   ## 设定
   （写作参考/设定/ 下的全局设定文件清单）

   | 文件 | 主题标签 | 章号 | 路径 |
   |:-----|:---------|:-----|:-----|

   ## 知识沉淀
   （pop-research / expert-writer WebSearch 沉淀的查询结果）

   | 文件 | 主题标签 | 章号 | 路径 |
   |:-----|:---------|:-----|:-----|

   ## 已废弃
   （arc修剪归档的失效要素）

   | 文件 | 主题标签 | 章号 | 路径 |
   |:-----|:---------|:-----|:-----|
   ```

4. **初始化项目总控.md**：
   - 从 `references/project/master-control.tpl.md` 读取模板
   - 写入 `{项目根目录}/项目总控.md`
   - 填写管线版本戳：读取 expert-writer SKILL.md 的 `version` 字段 → 写入 `管线版本` 和 `本项目启动时版本`
   - `📊 项目现状` 全部标记为 ⬜（无文件产出）
   - `🗺️ 所属管线` 从 `references/pipeline/manifest.md` 填充理想全流程和理想目录路由
   - `管线差异映射` 初始为空（新项目无差异）

   4a. **填充 📚 知识库路径**：
   - 解析 pop-trope-library 绝对路径：从当前 skill 目录（expert-writer 所在目录）上溯找到 pop-trope-library 的实际安装路径，写入 `skill 公共库` 行
   - 扫描 `{pop-trope-library}/设定库/` 下所有子目录（书目），列出书名，填入「skill 公共库内容速览」表
   - 如本书是对标/移植某已有书目 → 在对应行标注 ★
   - 用户私藏参考路径初始留空（`❌待补充`），不猜测——由用户后续填入或 Agent 追问

   4b. **解析文风DNA路径**：
   - 扫描 `{项目根目录}/写作资产/文风库/` 目录
   - 如目录有文件 → 将路径写入项目总控的「文风DNA」字段，标记 ✅已解析；**同时同步登记到 `写作参考/索引.md`**
   - 如目录为空 → 标记 ⏳待蒸馏（文风库将在种子设计阶段由 v3-seed 产出，不阻塞初始化）
   - 文风DNA路径供 expert-writer 修订层消费

5. **写作参考/索引.md 是 Step2 信息获取的唯一入口。** 不再创建散落的 `素材库/索引.md` 或 `设定库/_index.yaml`（已合并为写作参考/）。

6. 初始化完成 → **路由到 pop-writer-v3-seed**
