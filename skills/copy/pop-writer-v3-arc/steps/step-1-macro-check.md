# Step 1：六项宏观检查

> 管线: pop-writer-v3-arc v1.4.0
> 消费: 活记忆全量 + 幕纲（当前单元） + L3卡（如已有） + 正文（本单元全部章节） + 写作参考/设定/
> 产出: 六项宏观检查结果表 + 幕纲完成度检查表 + 回退判定

## 读什么

- `活记忆/活记忆.yaml`（全量：最后baseline + 本单元全部event，自然语言格式）
- `卷纲/幕NNN-{名称}.md`（当前L2单元卡：结构分析表+嵌套子线+子线咬合+物理坐标段）
- `卷纲/L3-NNN.md`（已有的L3剧情线卡，如存在：阶段推进表+叙事债务表+力量体系快照）
- `正文/chXXX.md`（本L2单元全部章节正文，用于交叉验证）
- `写作参考/设定/`（主角引擎.md / 金手指.md / 压力矩阵全量.md / 冲突轴全量.md，用于设定一致性检查）
- `项目总控.md`（当前章号+L2单元编号+arc计数）

## 做什么

逐项执行六项宏观检查 + 幕纲完成度检查。每项检查必须引用具体数据作为证据（活记忆/幕纲/L3卡/正文），不可凭感觉判定。详细标准见 `references/宏观检查清单.md`。

### 检查1：主线进度

扫描幕纲结构分析表 + 活记忆，核对主线推进是否符合设计。

| 检查动作 | 判定 |
|:---------|:-----|
| 对照幕纲结构分析表，核对每章事件brief是否在实际正文中兑现 | 设计的事件brief未在正文中出现 → ❌ |
| 检查幕纲核心成就（单元结束时状态变化）是否达成 | 单元结束时未达成核心成就 → ⚠️ |
| 对照L3卡阶段推进表（如已有），核对本单元是否推进了对应阶段 | L3卡设计阶段未推进 → ⚠️ |
| 检查主线是否有长期停滞（整个L2单元无主线进展） | 整个单元无进展 → ❌ |

### 检查2：子线闭合

扫描幕纲嵌套子线，核对子线的实际闭合/延伸状态。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐条检查幕纲嵌套子线的状态标注（延伸/闭合） | 设计标注"闭合"但正文未闭合 → ❌ |
| 检查是否有实际闭合但未标注的子线 | 正文已闭合但幕纲未标注 → ❌状态错误 |
| 检查子线推进的章号分布 | 某子线整个单元未触及 → ⚠️遗漏 |
| 对照L3卡（如已有），检查跨单元子线延伸方向 | 子线延伸方向与L3卡记录矛盾 → ⚠️ |

### 检查3：伏笔回收

扫描L3卡叙事债务表（如已有）+ 活记忆中的伏笔记录，核对回收状态。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐条检查L3卡叙事债务表中各债务的回收状态 | 超期>20章未回收 → ⚠️超期预警 |
| 对照正文检查已回收伏笔是否在L3卡中标记 | 正文已兑现但L3卡未更新 → ❌状态错误 |
| 检查本单元新埋设的伏笔是否记入L3卡叙事债务表 | 新伏笔未记入 → ❌遗漏 |
| 检查预期回收点是否合理 | 预期回收点已过但未回收 → ⚠️ |

### 检查4：节奏连贯

分析本单元的章节节奏波形。

| 检查动作 | 判定 |
|:---------|:-----|
| 统计本单元每章的事件密度（8-12个/章为达标） | >=3章连续低密度（<8个/章）→ ⚠️ |
| 检查爽点兑现频率 | 连续3章无爽点兑现 → ❌ |
| 检查钩子类型分布 | 连续3章非即时钩子 → ⚠️ |
| 检查章节间节奏过渡 | 单元内节奏断裂（如极快→极快→极快无喘息）→ ⚠️ |
| 对照幕纲结构分析表的节奏列设计 | 实际节奏与设计节奏偏差大 → ⚠️ |

### 检查5：压力推进

扫描写作参考/设定/压力矩阵全量.md + 活记忆，核对压力源推进状态。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐条检查压力矩阵全量中各压力源的倒计时 | 倒计时已归零且连续5章未再激活 → 确认关闭 |
| 检查是否有新压力源涌现但未记入压力矩阵 | 新压力源未记入 → ❌遗漏 |
| 检查压力推进是否有长期停滞 | 某压力源整个单元未推进 → ⚠️ |
| 检查四层生态是否都有冲突覆盖 | 某层连续整个单元无冲突 → ⚠️ |

### 检查6：设定一致

扫描写作参考/设定/ + 幕纲 + 正文，核对设定穿透情况。

| 检查动作 | 判定 |
|:---------|:-----|
| 对照写作参考/设定/主角引擎.md，检查主角行为准则一致性 | 行为与行为准则矛盾且无显性破例剧情 → ❌ |
| 对照写作参考/设定/金手指.md，检查金手指限制是否被突破 | 能力超出金手指限制定义 → ❌ |
| 对照幕纲设定引用指针，检查涉及设定是否被正确读取 | 指针指向的设定在正文中被违反 → ❌ |
| 检查角色驱动力三层演化连贯性 | 驱动力断层或行为准则无理由突变 → ❌ |
| 检查力量体系数值连续性 | 能力值跳级（单章增幅>2个台阶）→ ❌ |

## 幕纲完成度检查（v3.5新增）

在本单元结束时，额外执行5项完成度检查。详细预警阈值见 `references/宏观检查清单.md`。

| 检查项 | 检查内容 | 预警条件 |
|:-------|:---------|:---------|
| 导演意图执行率 | 已写章节中导演意图5项验证（叙事功能/事件链/情绪曲线/子线推进/章末钩子）通过的比例 | <70% 触发预警 |
| 幕纲偏离度 | 实际剧情与幕纲设计方案的偏差程度（结构分析表事件brief vs 实际发生的偏差） | 单元结束偏差>40% 触发预警 |
| 子线推进覆盖率 | 幕纲设计的子线在实际写作中是否都推进了 | <60% 触发预警 |
| 事件密度达标率 | 已写章节的事件密度（8-12个/章）达标比例 | <50% 触发预警 |
| L3叙事债务回收 | L3卡追踪的伏笔是否在预期窗口内回收 | 超期>20章触发预警 |

### 幕纲偏离度计算方法

```
偏离度 = (偏离章数 / 单元总章数) × 100%

偏离章 = 该章实际发生的事件与幕纲结构分析表对应行的事件brief存在以下任一情况：
  - 核心事件未出现
  - 出现设计中没有的重大事件（涌现偏差）
  - 子线推进方向与设计矛盾
```

## 回退判定

综合六项检查结果 + 幕纲完成度检查，判定是否需要回退。详细流程见 `references/回退执行流程.md`。

| 严重度 | 判定标准 | 处理方式 |
|:-------|:---------|:---------|
| 轻微 | 单章质量问题，不影响后续 | 打补丁：重写该章 |
| 中等 | 2-3章累积偏差，活记忆可吸收 | 微调方向提示+后续章节自然纠正 |
| 严重 | 弧线级偏移（主线进度❌ / 设定一致❌ / 幕纲偏离度>40%） | 回退重写（**红线❌4：必须用户确认范围后执行**） |
| 无需 | 六项检查全部✅且完成度检查无预警 | 无动作，直接进入Step 2 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 六项检查全部完成 | 缺项补查，不可跳过 |
| 每项检查有具体数据引用（活记忆/幕纲/L3卡/正文） | 无数据引用=重做该项 |
| 检查结果状态明确（✅/⚠️/❌） | 状态模糊=重判该项 |
| 幕纲完成度5项检查全部执行 | 缺项补查 |
| 严重回退需用户确认范围（红线❌4） | 未确认=不执行，记录待确认状态 |
