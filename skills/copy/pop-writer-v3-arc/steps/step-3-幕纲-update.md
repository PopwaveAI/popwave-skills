# Step 3：幕纲更新（设计方案→实际记录）

> 管线: pop-writer-v3-arc v1.4.0
> 消费: 本单元所有章节正文 + 活记忆events + 当前幕纲（设计方案） + Step 1/2 结果
> 产出: 更新后的幕纲（实际记录，供下一个L2设计时优先级1参考）

## 读什么

- `卷纲/幕NNN-{名称}.md`（当前L2单元卡——设计阶段的原始版本）
- `正文/chXXX.md`（本单元全部章节正文，ch起始~ch结束）
- `活记忆/活记忆.yaml`（本单元全部events——自然语言状态变化日志）
- Step 1 宏观检查结果（主线进度+幕纲偏离度+子线推进覆盖率）
- Step 2 设定修剪结果（幕纲子线状态标注）

## 做什么

**红线❌2：幕纲更新必须把"设计方案"更新为"实际记录"。** 物理坐标+子线状态+咬合关系全部回填实际值，不能只有设计占位。

幕纲在plot设计阶段是"设计方案"（预测会发生什么），单元写完后arc把它更新为"实际记录"（实际发生了什么）。更新后的幕纲供下一个L2设计时优先级1参考——让后续设计基于实际而非预测。

### 1. 物理坐标段回填

逐项更新幕纲物理坐标段，用实际正文+活记忆数据替换设计占位：

| 物理坐标段 | 设计阶段（plot产出） | 更新为（arc回填） |
|:-----------|:-------------------|:----------------|
| 时间轴 | 预估每章时间节点 | 回填实际时间节点（从正文+活记忆提取） |
| 地点轨迹 | 预估场景空间变化 | 回填实际地点序列 |
| 人物表 | 预估角色+功能+状态变化 | 回填实际人物状态变化（谁出场了、状态怎么变的） |
| 力量体系现状 | 预估单元结束数值快照 | 回填实际数值（等级/血量/技能/资产等） |

回填示例：
```
设计阶段（设计占位）：
  力量体系现状：预估LV1→LV2，解锁新技能

更新为（实际记录）：
  力量体系现状：LV1→LV2（ch003升级），解锁[街头感知]技能（ch003），
  血量8/60→12/60，经验13→46/100，获得首个自由属性点
```

### 2. 嵌套子线状态回填

逐条更新幕纲嵌套子线的实际状态：

| 子线 | 设计阶段 | 更新为 |
|:-----|:---------|:-------|
| 状态标注 | 设计预估"延伸/闭合" | 回填实际状态（延伸/闭合/失效）+实际闭合章号 |
| 阶段表 | 预估每阶段的章节+事件+不可逆变化 | 回填实际发生的事件+实际不可逆变化 |
| 闭合章号 | 预估闭合章号 | 回填实际闭合章号（如未闭合，标注延伸方向） |

回填示例：
```
设计阶段：
  子线A：铁钳帮反杀（类型：对抗 | 状态：延伸）
  | 阶段 | 章节 | 事件 | 不可逆变化 |
  | ch003 | 预估废弃厂房双杀 | 预估系统升级 |

更新为：
  子线A：铁钳帮反杀（类型：对抗 | 状态：闭合 | 闭合章号：ch003）
  | 阶段 | 章节 | 事件 | 不可逆变化 |
  | ch003 | ch003 | 废弃厂房双杀→系统升级→街头感知解锁 | 系统Lv1→Lv2，解锁感知技能，首次杀人 |
```

### 3. 子线咬合回填

更新子线咬合段，用实际因果链替换设计预估：

```
设计阶段（预估因果）：
  ch003咬合逻辑:
    ⑤铁钳帮反杀 → 产生①系统升级（经验来源）
    ①系统升级 → 产生④灵能耳鸣（升级副作用）
    因果链: ⑤ → ① → ④

更新为（实际因果）：
  ch003咬合逻辑（实际记录）:
    ⑤铁钳帮反杀 → 产生①系统升级（击杀回报机制，Lv1→Lv2）
    ①系统升级 → 产生④灵能耳鸣（升级时感知解锁副作用）
    ④灵能感知 → 引导③教团纹身发现（ch003末尾感知到异常纹身）
    因果链: ⑤ → ① → ④ → ③（比设计多了一环）
```

如实际因果链与设计有偏差，标注偏差：
```
偏差记录：设计预估因果链 ⑤→①→④，实际为 ⑤→①→④→③（多出④→③一环，
原因：ch003涌现出灵能感知引导纹身发现的情节）
```

### 4. 结构分析表核对

逐行核对幕纲结构分析表的事件brief与实际正文的偏差：

| 核对项 | 方法 | 产出 |
|:-------|:-----|:-----|
| 事件brief兑现 | 对照每章事件brief，检查正文是否兑现 | 标注"✅兑现"/"⚠️偏差"/"❌未兑现" |
| 涌现偏差 | 检查正文中出现但brief中没有的重大事件 | 标注"涌现：{事件描述}" |
| 节奏偏差 | 对照节奏列，检查实际节奏与设计节奏 | 标注"节奏偏差：{设计→实际}" |
| 情感偏差 | 对照情感设计列，检查实际情感曲线 | 标注"情感偏差：{设计→实际}" |

### 5. 一句话单元更新

将幕纲的"一句话单元"段从设计预估更新为实际概括：

```
设计阶段：预估本单元发生XXX
更新为：实际本单元发生了XXX（与设计一致/有XX偏差）
```

### 6. 可迁移要点提炼

从本单元实际写作中提炼设计经验，追加到幕纲"可迁移要点"段（供后续幕纲借鉴）：

| 提炼维度 | 说明 |
|:---------|:-----|
| 有效的子线设计 | 哪些子线设计在实际写作中效果好 |
| 偏差教训 | 设计与实际偏差的原因，下次设计如何避免 |
| 节奏经验 | 本单元节奏控制的经验 |
| 涌现亮点 | 设计中未预见但效果好的涌现情节 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 物理坐标段全部回填实际值（红线❌2） | 只有设计占位=回退重填 |
| 嵌套子线状态已标注实际闭合/延伸 | 状态模糊=重判 |
| 子线咬合已回填实际因果链 | 仍是设计预估=回填实际 |
| 结构分析表已逐行核对偏差 | 未核对=补核 |
| 一句话单元已更新为实际概括 | 仍是设计预估=更新 |

## 产出

1. 更新后的 `卷纲/幕NNN-{名称}.md`（设计方案→实际记录）：
   - 物理坐标段：实际时间轴/地点/人物状态/力量体系数值
   - 嵌套子线：实际闭合/延伸状态+实际闭合章号
   - 子线咬合：实际因果链（含偏差标注）
   - 结构分析表：每行偏差核对结果
   - 一句话单元：实际概括
   - 可迁移要点：新增经验提炼
2. 幕纲更新记录表（填入校准报告草稿）：
