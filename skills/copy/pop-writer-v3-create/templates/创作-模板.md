# 创作模板

> 所属: pop-writer-v3-create v1.3.0 · templates/
> 消费: step-1 创作产出
> 红线: ❌1 必须在导演意图框架内创作（叙事功能/事件链/情绪曲线/章末钩子必须覆盖）；❌2 不加载文风DNA；❌3 仅可更新幕纲本章聚焦相关内容；❌4 正文≥2500字

## 一、上下文确认表

逐项确认输入context已加载：

| 确认项 | 来源 | 状态 |
|:-------|:-----|:-----|
| 导演意图已加载 | 主会话Step0提取（幕纲结构分析表本章行） | ☐ 已加载（7项核心维度齐全） / ☐ 缺失 |
| 状态快照已读取 | 主会话Step1产出（活记忆+幕纲物理坐标投影，≤400字） | ☐ 已读取 / ☐ 缺失 |
| 上章末尾已读取 | `正文/ch{上一章}.md` 末尾1200字 | ☐ 已读取 / ☐ 首章无上章 |
| 设定文件已读取 | 主会话Step2强制读取（settings_ref指针指向的全读） | ☐ 全读 / ☐ 缺失 |
| **行为准则已加载** | **设定文件主角引擎段 行为准则** | **☐ 已加载 / ☐ 缺失** |
| info_acquired已加载 | 主会话Step2按需获取 | ☐ 已加载 / ☐ 无需求 |
| 幕纲已加载（可选） | `卷纲/幕NNN-{名称}.md` | ☐ 已加载 / ☐ 未注入 |

> 导演意图缺失 = 退回主会话Step0重做。行为准则缺失 = 退回seed补全。设定文件未全读 = 退回主会话Step2补读。

## 二、正文涌现结构

> 在导演意图框架内自由涌现——不预设场景台词，导演意图定义'达成什么'，create决定'怎么写'。

### 开幕段落
```
{空间/时间锚定 → 情绪传递（1-2句，对齐导演意图.emotion_curve起点）}
{如有感官锚点，第一句以该感官信号开头}
```

### 场景1：{场景描述}
```
{场景目的：推进导演意图.sublines_advance哪条子线/状态快照.pressures哪个压力源逼近}
{场景内连续流动——事件有机融合，不逐事件独立成段}
{行为一致性检查：主角行为符合设定文件主角引擎.行为准则？}
{事件转折1：___}
{事件转折2：___}
{事件转折3：___}
```

### 场景间过渡
```
{1-2句自然过渡：时空跳转/感官切换/情绪转折}
```

### 场景2：{场景描述}
```
{场景目的：___（对照导演意图.event_chain）}
{场景内连续流动}
{行为一致性检查：___}
{事件转折1：___}
{事件转折2：___}
```

### 闭幕段落（章末钩子）
```
{钩子类型：落实导演意图.chapter_hook}
{钩子内容：___}
{钩子强度：即时危机型 / 远期悬念型}
```

> 场景流渲染技法、压力源逼近写法、钩子类型详解 → 参考 `references/创作指南.md` §三-§五。

## 三、创作决策记录YAML

```yaml
creation_log:
  chapter: {章号}
  director_intent_source: "卷纲/幕NNN-{名称}.md#结构分析表"
  decisions:
    - point: "子线推进"
      choice: "{推进哪条子线，推进方式（触面/深化/回收）}"
      basis: "导演意图.sublines_advance + 幕纲嵌套子线"

    - point: "压力源选择"
      choice: "{哪个压力源逼近}"
      basis: "状态快照.pressures（must_advance=true优先）"

    - point: "金手指使用"
      choice: "{金手指怎么使用}"
      basis: "设定文件金手指能力+限制（settings_ref注入）"

    - point: "角色决策"
      choice: "{主角做了什么决策}"
      basis: "设定文件主角引擎驱动力三层+缺陷+决策风格+行为准则（settings_ref注入）"

    - point: "章末钩子"
      choice: "{钩子类型和内容}"
      basis: "导演意图.chapter_hook"

    - point: "行为一致性"
      choice: "{主角行为是否符合行为准则？如有破例，说明剧情原因}"
      basis: "设定文件主角引擎.行为准则（settings_ref注入）"

    - point: "叙事功能"
      choice: "{如何体现导演意图.narrative_function}"
      basis: "导演意图.narrative_function"

    - point: "情感曲线"
      choice: "{正文情绪走向如何对齐导演意图.emotion_curve}"
      basis: "导演意图.emotion_curve"

  l2_card_chapter_row_adjustment:  # 如有微调（红线❌3：仅限本章行）
    adjusted: false  # true/false
    field: "{调整的字段，如event_brief}"
    before: "{调整前内容}"
    after: "{调整后内容}"
    reason: "{调整原因}"

  word_count: {正文字数}
  scene_count: {场景数}
  event_count: {事件转折总数}
  pressure_sources_advanced: [{本章逼近的压力源列表}]
  sublines_advanced: [{本章推进的子线列表}]
  rewards_given: [{本章给读者的爽点列表}]
  behavior_consistency_passed: true
  director_intent_coverage:  # 红线❌1 导演意图覆盖核对
    narrative_function: true
    event_chain: true
    emotion_curve: true
    sublines_advance: true
    chapter_hook: true
```

## 四、门禁表

| 门禁项 | 检查结果 | 失败动作 |
|:-------|:---------|:---------|
| **导演意图框架已覆盖（叙事功能/事件链/情绪曲线/章末钩子）（红线❌1）** | **☐ 通过 / ☐ 退回** | **退回重写** |
| **未加载文风DNA（红线❌2）** | **☐ 通过 / ☐ 退回** | **移除文风DNA注入** |
| **未修改幕纲结构分析表其他行/嵌套子线/物理坐标（红线❌3）** | **☐ 通过 / ☐ 退回** | **退回，仅保留本章行改动** |
| **正文≥2500字（字数下限门禁）（红线❌4）** | **☐ 通过 / ☐ 退回** | **退回重写** |
| 场景流渲染（非逐事件） | ☐ 通过 / ☐ 退回 | 退回重写 |
| 本章有压力源逼近 | ☐ 通过 / ☐ 退回 | 退回重写 |
| 章末有钩子 | ☐ 通过 / ☐ 退回 | 退回重写 |
| 主角行为符合行为准则 | ☐ 通过 / ☐ 退回 | 退回重写该场景 |
| 上下文确认表已填 | ☐ 通过 / ☐ 补填 | 补填后再继续 |
| 导演意图/状态快照存在 | ☐ 通过 / ☐ 终止 | 终止，提示先完成上游 |
| context receipt已填写（预留） | ☐ 通过 / ☐ 补填 | 补填后再继续 |

> 创作子skill**不做**文风DNA对齐/8020比例/AI观感词清理检查（交给revise子skill）。

## 五、context receipt 预留格式（批次2完整实现）

create产出时附带 context receipt，确认实际接收了哪些注入项。主会话产出 manifest 声明注入了什么，create产出 receipt 确认实际接收了什么，主会话Step4对照检查一致性。

```yaml
context_receipt:
  chapter: {章号}
  manifest_id: "{主会话manifest ID}"
  received:
    - item: "director_intent"
      status: "full" | "partial" | "missing"
      actual_read: "{实际读取字数}"
      key_elements_confirmed: ["narrative_function", "event_chain", "emotion_curve", "chapter_hook"]
    - item: "state_snapshot"
      status: "full" | "partial" | "missing"
      actual_read: "{实际读取字数}"
    - item: "prev_chapter_tail"
      status: "full" | "partial" | "missing"
      actual_read: "{实际读取字数}"
    - item: "settings_ref"
      status: "full" | "partial" | "missing"
      files_read: ["{实际读取的设定文件路径列表}"]
    - item: "info_acquired"
      status: "full" | "partial" | "missing" | "not_applicable"
    - item: "l2_card"
      status: "full" | "partial" | "missing" | "not_applicable"
  excluded:
    - item: "style_dna"
      status: "excluded_by_design"
      reason: "文风渲染由revise负责（红线❌2）"
  anomalies: []
```

> 批次1：预留格式填写，确认 received 各项 status + excluded 中的 style_dna。
> 批次2：完整实现receipt产出与主会话Step4一致性检查。
