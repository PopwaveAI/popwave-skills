# CHANGELOG — pop-writer-v3-plot

## [1.5.0] - 2026-06-29
### Core Change
- 新增两层故事推演机制，解决"机械填表"根因
- step-1新增卷级故事大纲（500-1000字beat级骨架+L2切分表）
- step-2新增单元故事推演（§3，每章200-400字散文体预演）
- 节拍表/子线/红线从推演中**提取**，不再凭空填写
### Added
- step-1 §2 卷级故事大纲：开场局势→3-5个转折beat→高潮→收束
- step-1 §2.3 幕纲切分表：从故事大纲中识别段落边界切分3-6个幕纲
- step-2 §3 单元故事推演：每章含开场画面+故事流动+情绪转折+角色决策+对白锚点+章末画面
- step-2 §4.2 提取规则：节拍表每个字段从推演中提取的映射关系
- 网文铁律9条前置为推演时创作约束（推演中即标注即检查）：
  - 情绪碾压逻辑（推演中标记[逻辑待圆]）
  - 默认快节奏（每章2-4个事件转折）
  - 爽感驱动（推演中标注[爽点：类型]）
  - 主角主动（推演时自问谁在驱动）
  - 危机不间断（章末标注[下章危机：XXX]）
  - 行动即回报（推演时自问主角得到了什么）
  - 悬念不闭合（每章揭示一个谜+抛出一个新谜）
  - 章末追读（推演最后一个画面是追读时刻）
  - 设定浸泡（标注[设定浸泡：通过XXX事件呈现YYY设定]）
- 幕纲结构变更：新增"单元故事推演"段作为核心内容
### Changed
- step-2 §1 新增1.1：读卷级故事大纲+L2切分表
- step-2 §4 创作节拍表从"设计"改为"从推演提取"
- step-2 §5 嵌套子线从"设计"改为"从推演提取"
- step-2 §6 单元红线从"设计"改为"从推演提取"
### Red Lines
- 红线从6条扩展到10条：
  - ❌7 新增：单元故事推演必须产出（每章200-400字）
  - ❌8 新增：节拍表内容必须从推演提取（非凭空填写）
  - ❌9（原❌7）篇幅确认硬阻塞
  - ❌10 新增：卷级故事大纲必须产出
