# Step 1：卷方向锚 + 卷级故事大纲

> 定位：产出卷方向锚和卷级故事大纲，为 step-2 的 幕纲设计提供卷级坐标和故事骨架。

## 消费
- `写作参考/设定/`（seed 产出的全局设定：主角引擎/金手指/压力矩阵/冲突轴/核心卖点）
- `写作参考/索引.md`（本地已有设定/知识沉淀总目录）
- 用户输入（本卷方向意图）

## 产出
- `卷纲/卷N-方向锚.md`（≤300字方向向量，正式产物）
- `卷纲/推演/卷N-故事大纲.md`（500-1000字beat级故事骨架，过程性文件）

## 执行流程

### 0. 篇幅确认（硬门禁）

在设计卷方向锚之前，必须先与用户确认以下信息。**禁止跳过此步直接设计。**

向用户提问：
> 在开始设计卷方向锚之前，需要确认几个关键信息：
> 1. 本书预计总篇幅（多少万字）？
> 2. 预计分几卷？
> 3. 每卷大约多少章？
> 4. 本书的终点预期是什么（主角最终达到什么状态/完成什么目标）？
> 5. 如果已有金手指设定，金手指的终态是什么（Lv?/什么境界?）？

收到用户回答后，将篇幅信息写入卷方向锚的元数据中，然后进入Step 1。

### 1. 卷方向锚设计（方向向量）

#### 1.1 卷定位
本卷兑现全书哪个承诺（1句话）。从 `写作参考/设定/核心卖点.md` 的情绪承诺出发，明确本卷兑现哪一段。

#### 1.2 起点终点
角色/世界/力量/关系四维度的起点→终点状态变化：

| 维度 | 起点 | 终点 |
|:-----|:-----|:-----|
| 角色 |  |  |
| 世界 |  |  |
| 力量 |  |  |
| 关系 |  |  |

> 起点：参考前序卷终点 / 首卷参考 `写作参考/设定/` 的初始状态。
> 终点：本卷结束时四维度应到达的状态。

#### 1.3 主冲突
谁阻碍谁，争夺什么，失败代价（1-2句）。从 `写作参考/设定/冲突轴全量.md` 提取本卷激活的冲突线。

写入 `卷纲/卷N-方向锚.md`（≤300字）。

### 2. 卷级故事大纲（故事骨架）

> **核心原则**：大纲不是事件列表，是**故事讲一遍**。想象你在给编剧讲这一卷的故事——从开场局势到高潮对决到收束余韵，一气呵成。每个beat都是一个改变故事走向的时刻，不是流水账节点。

#### 2.1 读取设定

读取以下设定文件，作为故事推演的素材库：
- `写作参考/设定/主角引擎.md`（驱动力三层+缺陷+决策风格+行为准则）
- `写作参考/设定/压力矩阵全量.md`（本卷激活哪些压力源）
- `写作参考/设定/冲突轴全量.md`（本卷激活哪条冲突线）
- `写作参考/设定/金手指.md`（能力上限+升级路径）
- `写作参考/设定/核心卖点.md`（情绪承诺）

#### 2.2 故事推演

写一段500-1000字的散文体故事大纲。要求：

**结构**：开场局势 → 3-5个转折beat → 高潮 → 收束

- **开场局势**（50-100字）：主角一开卷在什么处境，什么压力正在逼近
- **转折beat**（每个100-200字）：改变故事走向的关键事件。每个beat必须回答：
  - 发生了什么（画面级，不是"主角变强了"而是"主角第一次用背刺秒杀了一个职业者"）
  - 情绪转折是什么（从X到Y）
  - 主角做了什么决策驱动了这个转折
- **高潮**（150-300字）：本卷最爆的时刻——可以是战斗高潮、真相揭示、身份转变
- **收束**（50-100字）：高潮后主角到了什么新状态，什么悬念留给下一卷

**网文铁律植入**（推演时必须遵守）：

- **情绪碾压逻辑**：先设计情绪高潮beat，再补合理性。推演时如果某个beat逻辑不够严但情绪到位，保留它，标记"逻辑待圆"。
- **默认快节奏**：beat之间不写"过渡期平淡生活"。如果两个beat之间有时间跳跃，一句话带过。
- **爽感驱动**：每个beat必须含一个爽感释放——困境解决/弱变强/谜团揭示/被压迫后反击。推演时标注爽点类型。
- **主角主动**：每个beat的转折由主角决策驱动，不是"敌人来找他"。即使是被攻击，也是主角的主动反应改变局面。
- **危机不间断**：beat之间不设安全区。前一个危机的解决直接触发下一个。
- **行动即回报**：每个beat中主角的行动必须有即时反馈（数值/技能/物质/信息/关系变化）。
- **悬念不闭合**：每个beat揭示一个谜团的同时制造一个更大的新谜团。推演时标注"揭示了什么"和"抛出了什么新谜"。
- **章末追读**：大纲最后一个beat必须以悬念收束，不是大团圆。
- **设定浸泡**：设定信息不独立抛出，融入beat中自然展现。推演时如果需要引入设定，标注"在beatX中通过Y事件自然呈现"。

#### 2.3 幕纲切分

从故事大纲中识别天然的故事段落边界，切分为3-6个幕纲。每个幕纲对应大纲中1-2个beat。

格式：

```
| L2编号 | beat覆盖 | 情绪曲线 | 核心爽点 |
|:-------|:---------|:---------|:---------|
| L2-001 | 开场→beat1 | 压抑→希望 | 第一次系统觉醒 |
| L2-002 | beat2→beat3 | 紧张→爆发 | 首次暗杀职业者 |
| ... | ... | ... | ... |
```

写入 `卷纲/推演/卷N-故事大纲.md`。

## 门禁
- 方向锚≤300字
- 卷定位必须有
- 四维度起点终点表完整
- 主冲突含"谁阻碍谁+争夺什么+失败代价"
- **故事大纲500-1000字**
- **故事大纲含开场+3-5个beat+高潮+收束**
- **每个beat有爽点类型标注**
- **每个beat有主角主动决策**
- **幕纲切分表完整（3-6个L2）**
