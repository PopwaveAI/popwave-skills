## 结构分析表

> 每章一行。事件brief必须用动作链（具体动作+因果，禁止纯功能描述）。设定引用列标注本章涉及的设定文件路径。

| 阶段 | 功能 | 章节 | 事件brief | 情感设计 | 节奏 | 设定引用 |
|:-----|:-----|:-----|:----------|:---------|:-----|:---------|
|  |  | chXXX | 【动作1→动作2→动作3→结果】 |  |  | 写作参考/设定/{文件}.md#{锚点} |
|  |  | chXXX | 【动作1→动作2→动作3→结果】 |  |  | 写作参考/设定/{文件}.md#{锚点} |

**事件brief合格示例**：
> 索伦从系统面板看到据点坐标→趁群龙无首潜入仓库→黑暗中逐个清除3个守卫→发现账本→带走战利品→撤离时看到教团资助文件

**事件brief不合格示例**（禁止）：
> 索伦趁乱清理铁钳帮据点

**设定引用格式**：`写作参考/设定/金手指.md#升级机制, 写作参考/设定/主角引擎.md#行为准则`
> 路径指针，不是内容副本。多个用英文逗号分隔。
