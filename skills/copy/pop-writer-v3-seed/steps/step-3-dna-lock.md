# Step 3：文风DNA蒸馏→文风库文件

> 管线: pop-writer-v3-seed v1.8.0
> 消费: Step 0创意共识单（文风方向+核心卖点） / 对标书原文 / 用户指定文风参考 / pop-shared-dna / library文风库
> 产出: 写作资产/文风库/{书名}.md（文风DNA档案，项目资产）

## v3.5 说明

文风DNA是项目资产，存放在 `写作资产/文风库/{书名}.md`。v3.5中文风DNA不变，仍由seed step-3蒸馏产出。revise阶段完整加载文风DNA作为硬阻塞。

## v1.8 核心变更：强制v4格式

**v1.7及之前的问题**：step-3使用自己的"5维度拆解"格式（句式/叙事距离/感官密度/对话风格/数据嵌入），只有抽象描述没有原文摘录。agent按5维度写出纯描述性文档，revise子agent拿到后无法感受目标作者的笔触——它读到"句式特征：极短句链"，但不知道"极短句链"长什么样。

**v1.8修复**：删除自定义格式，强制执行pop-shared-dna v4模板格式——每个维度/场景卡 = 观察(1-2句) + 原文(≥500字) + 时间演变(1-2句)。agent必须从源书原文中摘录≥500字，不能只写抽象描述。

## 做什么

### 1. 文风源确认

按优先级确认文风来源：

| 优先级 | 来源 | 条件 | 原文可用性 |
|:-------|:-----|:-----|:-----------|
| 1 | 对标书原文 | pop-decon 提供了原文片段 | ✅ 可摘录原文 |
| 2 | library 文风库 | 已有v4格式文风DNA档案 | ✅ 已有原文摘录 |
| 3 | 用户指定参考文本 | 用户明确指定文风参考 | ✅ 可摘录原文 |

### 2. 文风DNA蒸馏（强制v4格式）

调用 **pop-shared-dna** 执行完整蒸馏流程（采样→精读→验证→写入→试写），产出 `写作资产/文风库/{书名}.md`。

**格式硬约束（pop-shared-dna v4模板）**：

每个维度/场景卡必须包含三段，缺一不可：

| 段 | 内容 | 红线 |
|:---|:-----|:-----|
| 观察 | 1-2句描述作者笔触特征 | — |
| 原文 | ≥500字原文摘录（按章逐行摘录，保留完整段落和对话） | ❌ <500字退回 |
| 时间演变 | 1-2句说明早期/中期/末期差异 | ❌ 无时间演变退回 |

**v4格式示例（正确）**：

```
### 战斗·暗杀/偷袭

scene: combat_stealth

**观察**：暗杀场景的核心公式是"阴影躲藏→背刺发动→数据面板弹出"。
动作描写极度简洁，全部用动词推进（掐/切/刺/捂），没有任何冗余修饰。

**原文**：

> 索伦的身影在一瞬间凭空出现，朦胧的阴影中他的身影浮现...
> （≥500字原文摘录，保留完整段落和对话格式）
> _——CH004，贫民区夜间暗杀_

**时间演变**：早期侧重偷袭突然性，中期暗杀场景几乎消失，后期复活为传奇技巧。
```

**v4格式错误示例（v1.7及之前的错误）**：

```
❌ 错误：纯抽象描述，无原文摘录
句式特征：极短句链
叙事距离：贴脸跟随
感官密度：嗅觉优先
对话风格：动作前置+引导词
数据嵌入：竖列冒号分隔
```

**为什么不能只有抽象描述**：revise子agent拿到文风DNA后，需要从原文中直接感受目标作者的笔触密度——句式节奏、段落呼吸、感官密度。"极短句链"这5个字无法让agent理解什么叫极短句链。只有读到≥500字原文，agent才能内化那种节奏。

### 3. 双源融合文风的特殊处理

如果本书文风是双源融合（如"深渊主宰战斗文风+我在美国搞内战政治讽刺文风"）：

| 步骤 | 操作 | 产出 |
|:-----|:-----|:-----|
| 3a | 为每本文风源书分别调用pop-shared-dna蒸馏v4格式文风DNA | 各自独立的v4档案（含原文摘录） |
| 3b | 基于两份v4档案编写融合方案 | 双源融合总则表（哪个环节用哪个源） |
| 3c | 融合方案中每个维度必须标注"源"和保留原文摘录 | 原文来自源书，不能删除 |

**红线**：融合方案不能替代原文摘录。即使两个源的文风已融合，最终产出的文风DNA文件中，每个维度/场景卡仍然必须有≥500字原文摘录（从源书中摘录）。

### 4. 用户QA确认

将文风DNA档案呈现给用户确认：

```markdown
## 文风DNA确认

### 文风源
{对标书/用户指定/library/双源融合}

### 格式验证
- 场景卡数量：{N}个（≥6）
- 通用维度数量：{N}个（≥4）
- 每个维度/场景卡原文摘录：{N}字（≥500字/项）
- 时间演变段：{N}个（每项都有）
- 采样范围：{章节数+行数}

### 场景卡清单
1. {场景卡名} — scene: {字段值} — 原文{N}字
2. ...

### 通用维度清单
1. {维度名} — 原文{N}字
2. ...

---
✅ 用户确认，进入step-4设定产出
❌ 用户要求调整 → 回到蒸馏步骤调整
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 文风DNA已蒸馏（红线❌1） | 扫描写作资产/文风库/，空=终止整个skill |
| **v4格式合规**：每个维度/场景卡有观察+原文≥500字+时间演变 | 退回重新蒸馏，补全原文摘录 |
| **无纯抽象描述**：不接受只有"句式特征：极短句链"这种无原文的维度 | 退回，必须从源书摘录原文 |
| **双源融合有原文**：融合方案的每个维度保留了源书原文摘录 | 退回，补全源书原文 |
| 文风库文件已落盘 | 补充蒸馏流程 |
| **用户已确认文风DNA** | **等待用户确认后再进step-4** |
