## 做什么

### 1. 主角引擎设计→写作参考/设定/主角引擎.md

设计主角引擎并落盘为独立设定文件。主角引擎是全局设定 ground truth，跨单元不变。

| 子项 | 内容 |
|:-----|:-----|
| 表层驱动力 | 活下去，活好（ch1激活） |
| 中层驱动力 | 保护身边的人（遇到需要保护的人时） |
| 深层驱动力 | 推翻/改变制度（能力成长后浮现） |
| 缺陷 | 身体/社会身份/资源的三重约束 |
| 决策风格 | 一句话描述 |
| 行为准则 | 背景映射+战斗风格+道德底线+行为禁忌 |

行为准则设计要点：
- 背景映射：主角背景→行为规则（如"退役特工→警惕性高，先观察后行动"）
- 战斗风格：一句话概括战斗方式
- 道德底线：不可逾越的红线
- 行为禁忌：绝不会做的事

写入 `写作参考/设定/主角引擎.md`，参照 `templates/写作参考结构-模板.md` 中的主角引擎模板。

### 2. 核心卖点落盘→写作参考/设定/核心卖点.md

将Step 0创意共识单中的核心卖点正式落盘为独立设定文件：

| 字段 | 内容 |
|:-----|:-----|
| 一句话 | {核心卖点一句话} |
| 核心爽点 | {读者最期待的体验} |
| 差异化 | {和同类作品的不同} |
| 情绪承诺 | {读者读完的feeling} |

写入 `写作参考/设定/核心卖点.md`。

### 3. 活记忆baseline#0初始化→活记忆/活记忆.yaml

v3.5活记忆从7组件结构化YAML改为自然语言段落格式。baseline和event都是自然语言，不做结构化字段拆分。

```yaml
baseline: |
  ch0初始：{一段话概括初始状态——主角当前处境、能力水平、面临的压力、情绪状态、资源状况。自然语言段落，不做字段拆分。}
events: []
```

baseline#0 写作要点：
- 一段话高度概括ch0（第一章开始前）的初始状态
- 包含：主角当前处境、能力水平、面临的压力、情绪状态
- 自然语言，不拆分结构化字段
- ≤200字

events 初始为空列表。emerge Step5每章追加一条event（自然语言段落），arc每单元压缩时合并baseline+events→新baseline。

可选产出 → 改为条件必须产出（v1.7新增）：
- `写作参考/设定/世界宪法.md`（世界法则+溯源标注）
- `写作参考/设定/力量体系.md`（能力分级+运作机制）
- `写作参考/设定/社会结构.md`（社会金字塔+流动规则）

**判断规则（v1.7硬门禁）：** 如果本书的核心卖点是魔改世界观（异世界/超凡复苏/邪神渗透/修仙末世等），以上三个文件从"可选"变为"必须"。agent按以下条件判断：
- 故事发生在与现实不同的世界（异世界/架空/魔改现实）→ 世界宪法必须产出
- 有超自然力量体系（修仙/魔法/异能/系统）→ 力量体系必须产出
- 有非现代社会结构（宗门/领主/奴隶制/种姓）→ 社会结构必须产出

如果以上条件均不满足（纯现实都市无超自然），则三项可选。

**设计理由：** v3.5实测中agent把这三个文件当"可选"直接跳过，导致plot设计幕纲时缺少世界观ground truth，正文写作时读者"全程懵逼"不知道这是什么世界。世界观设定是核心卖点的基础设施，不能省。
