# Skill 通用 · 思考模式注入规范

> v1.0 | 2026-05-20
> 核心原则：**先思考，再出文件。跳过思考 = 放弃质量。**

---

## 一、为什么需要思考模式

"先想清楚再动手"这个模式本身是对的。无论在哪一层，跳过思考直接产文件，质量就会打折扣。反过来，停下来想清楚再动，哪怕只多花一次 LLM 调用的成本，产出也明显不一样。

## 二、通用思考模式格式

每个 Skill 的核心产出步骤前，插入一个**思考节点**。

### 思考节点的结构

```
思考节点名称
  ├── ① 对齐检查 —— "上一步的设计和这一步要做的，一致吗？"
  ├── ② 核心判断 —— "这一步要产出的东西，第一性是什么？"
  ├── ③ 风险扫描 —— "这一步容易踩什么坑？"
  ├── ④ 策略说明 —— 输出判断链（散文式）
  └── ⑤ 待确认/疑问 —— 需要上游或用户确认的问题
```

## 三、注入方式

在现有流水线的"输入→执行→输出"之间插入思考节点：

```
旧：输入 → 执行 → 输出
新：输入 → 【思考节点】→ 执行 → 输出
```
