# 批量风格迁移 — 多章正文改写/渲染

> **定位**：从现有源文本（原文txt或已有正文）批量改写/渲染为指定文风。不走 creative→plot→chapter 标准流水线，直接从源文本提取事件链 → 创建复合设计包 → 并行渲染。
> **典型场景**：用户说"用XX风格从ch01写N章/改写成XX风格/风格迁移N章"。
> **前置条件**：源文本（原文txt文件或已有正文.md文件）存在；styles/{目标风格}.md DNA档案就绪。

---

## 1. 与单章渲染的区别

| 维度 | 单章渲染 | 批量风格迁移 |
|:-----|:---------|:-------------|
| 输入源 | 1 设计包 | 源文本（txt/md）+ 提取事件链 → 复合设计包 |
| 上游 | chapter-design（必须已产出单章设计包） | 无上游依赖 — 直接从源文本提取 |
| 设计包 | 单章独立 | 复合设计包（N章合1文件，或逐章独立） |
| 输出格式 | `正文/chXXX.md` | `正文/{风格名}v3/chXXX.md`，每章独立文件 |
| 执行方式 | 单线程 Step 0→1→2→3→4 | 并行委托（delegate_task），每批 5 章 |
| pipeline 旁路 | — | 跳过 creative→plot→chapter 管线 |

---

## 2. 批处理策略

### 2.1 批次划分

```
原文 10 章 → 分 2 批，每批 5 章
batch 1: ch01-05  → delegate_task 1
batch 2: ch06-10  → delegate_task 2
```

**经验法则**：
- 每批 = 5 章（平衡子 agent 负载和上下文窗口）
- 当 ch01-05 是紧密关联的起承转合（如第一卷上半部），用 **复合设计包**（5章合1文件）
- 当章节相对独立，用 **逐章设计包**（每章独立文件，子 agent 依次渲染）
- 5 章/批 × 3000 字/章 ≈ 15K 输出 → 子 agent 的 context 适合

### 2.2 复合设计包格式

当 N 章合1时，复合设计包的目录结构：

```markdown
# 第1-5章 综合设计包 — {卷标题}

## 卷一整体弧线
{前三章的铺垫弧线 + 后两章的小高潮}

## 逐章设计

### ch01 {标题}
- 事件链（scene/POV标注）
- 关键对白/🔒标记
- 角色状态
- 情绪弧线位置

### ch02 {标题}
...

### ch05 {标题}
...

## 全局角色状态（跨5章追踪）
{角色在5章中的变化轨迹}
```

### 2.3 逐章设计包（独立文件）

当章节相对独立时，每章一个设计包，按 `ch{NN}-设计包.md` 写入。子 agent 依次读取渲染。

---

## 3. 执行流程

### Step 1 — 源文本分析

从源文本提取：
- 章标题 + 行号区间
- 每章核心事件（3-6个主要事件）
- 关键对白（标注🔒）
- 角色状态变化
- 情绪弧线（起点→发展→转折→章末钩子）

### Step 2 — 创建复合设计包

写入 `写作资产/设计包v3/ch{NN-NN}-设计包.md`：
- L1 骨架+事件链（逐章，每事件标注 scene/POV）
- L2 爽点+情绪弧线（逐章）
- L3 角色+对白（含🔒关键对白，转换为目标风格的语义等价表述）
- L4 感官+DNA对接（附体系映射表）

**体系映射表**（跨体系迁移时必加）：

| 源体系概念 | 目标体系概念 |
|-----------|-------------|
| 预言法术（九环） | 占卦推演/窥天机 |
| 流星爆 | 天降陨星 |
| 恐怖骑士 | 邪道尸煞武士 |
| DND等级（5级平民/1级盗贼） | 初期境界（炼气期） |
| 经验值分配 | 经脉贯通/丹田突破 |

### Step 3 — 并行渲染

用 delegate_task 分发。**父agent必须按 pop-writer-prose 的 step-0-delegation-contract.md 准备 context，不可凭自己理解过滤。**

#### 父agent自检（发出 delegate 前逐项确认）：

| # | 自检项 | 不通过处理 |
|:-:|:-------|:----------|
| D1 | 已加载 pop-writer-prose 的 `steps/step-0-delegation-contract.md` | 退回加载 | 
| D2 | context 中包含子agent入口指令（`skill_view('pop-writer-prose') → 加载 step-0 到 step-4 → 按序执行`） | 缺则补 | 
| D3 | context 中包含路径矩阵（design_pack / style_dna / output / prev_chapter） | 缺则补 |
| D4 | context 中包含门禁3条（剧情不变/角色不可篡改/🔒原文照写） | 缺则补 |
| D5 | context 中包含章节连续性摘要（时间/地点/角色状态/上章末事件） | 缺则补 |
| D6 | 首章渲染标注 `is_first_chapter: true`（跳过 prev_chapter） | 检查标记 |

#### delegate context 标准模板：

```markdown
task: 渲染 ch{NN}-{标题} {风格}风格
design_pack: {项目}/写作资产/设计包v3/ch{NN}-设计包.md
style_dna: {项目}/styles/{风格}.md
output: {项目}/正文/{风格}v3/ch{NN}-{标题}.md
prev_chapter: {项目}/正文/{风格}v3/ch{NN-1}-{标题}.md   # 首章可略
is_first_chapter: true/false
gates:
  - 原文/设计包核心事件不可增删，不可改顺序
  - 角色生死/状态不可篡改
  - 🔒标记内容必须原文照写
continuity:
  - 上章末尾时间: {时间锚点}
  - 上章末尾地点: {地点}
  - 主角状态: {关键状态}
  - 配角状态: {关键状态}
  - 最后一幕事件: {简述}
  - 本章起始点: {从哪开始}
instructions: |
  1. skill_view('pop-writer-prose') 全量加载
  2. 加载 steps/step-0 到 step-4 共5个文件
  3. 按 step 1-4 顺序执行渲染
```

### Step 4 — 验证

所有子 agent 完成后：
- [ ] 每章字数在 2000-3000 字范围内
- [ ] 🔒标记在所有章中原文照写（或语义等价）
- [ ] 对话引导符合目标风格（遮天=单字"道"引导）
- [ ] 战斗风格符合目标 DNA（遮天=碾压四步 vs 深渊=潜行暗杀）
- [ ] 无 AI 观感词（他感到/他仿佛/他意识到）
- [ ] 无游戏术语侵入（经验值/等级→东方修行语言）
- [ ] 章末钩子不闭合

## 4. 风格迁移特化

### 4.1 体系映射原则

1. **语义等价**：不改变剧情功能和角色关系
2. **不做扩展**：设计包没有的概念不引入（如原文没提"修仙"就不加"金丹"）
3. **命名一致**：角色名/地名保持原文，只改力量体系术语

### 4.2 常见迁移对照（DND→仙侠/东方玄幻）

| DND/西幻概念 | 东方玄幻概念 |
|-------------|-------------|
| 等级/职业 | 境界/功法 |
| 面板数据（力量/敏捷） | 体感描写（筋骨齐鸣/身如轻燕） |
| 经验值 | 灵气积累/药力炼化 |
| 职业技能（潜行/开锁） | 技法/身法/步法 |
| 法术（预言术/流星爆） | 神通/禁术 |
| 盗贼/战士转职 | 功法晋级/洗髓 |
| 传奇领域 | 化神/大乘 |

### 4.3 无需翻译的元素

- 兄妹亲情（不变）
- 贫民区生存逻辑（不变）
- 阴谋/背叛（不变）
- 恐惧/绝望/希望（不变）
- 地名（不变，除非目标风格有对应体系）

---

## 5. 风险与红线

| # | 红线 |
|:-:|:-----|
| 🚫1 | **不跳过源文本读取** — 批量场景下 agent 容易凭"对这本书的印象"写事件链，必须逐章读原文提取。 |
| 🚫2 | **复合设计包不丢失粒度** — 5 章合1的设计包必须在每章内标注事件级 scene/POV，不能简化为"第1-5章整体是升级阶段"。 |
| 🚫3 | **风格迁移不改变剧情** — 杀科尔帮的人还是杀科尔帮，只是写法从"背刺+面板"变成"身影一闪+气势碾压"。 |
| 🚫4 | **批量≠串行** — delegate_task 并行。禁止串行逐章渲染（耗时过长）。 |
