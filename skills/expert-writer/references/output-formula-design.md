# 产出物公示表设计说明

> **状态**：设计归档。格式规范已合并入 `project-master-control.tpl.md` 模板。
> 本文档保留作为设计决策记录。具体列格式见模板。

## 历史设计

产出物公示表最初定义为 10 列表（含 `=` 和 `+` 作为独立列），格式：

```
| 产出 | = | 输入 A | + | 输入 B | + | 输入 C | 产出者 | 消费者 | 状态 |
```

**设计意图**：逐列可被正则解析（"取第 1 列为产出文件，取第 3/5/7 列为输入文件"），方便健康检查做断裂检测。

**废弃原因**：10 列在小屏幕上无法阅读；字段语义不直观；对齐线 `|:-----|:|=|:-------|:|=|...` 维护成本高。

## 当前格式（v2.0）

已合并入 `project-master-control.tpl.md` 的 `📊 项目现状 → 产出物清单` 段：

```
| 产出文件 | 上游依赖 | 产出者 | 状态 |
|:---------|:---------|:-------|:----:|
```

4 列。依赖链写成自然语言 `A + B + C` 或 `A → B → C`，`→` 表示化合方向。

**健康检查消费方式**（替代原来的列解析）：
- 取"产出文件"列 → 搜索文件系统 → 更新"状态"列
- 不再逐列解析 `=` 和 `+` 标记
