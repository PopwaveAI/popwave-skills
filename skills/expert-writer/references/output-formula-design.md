# 产出物公示表设计说明（A + B + C = D）

## 定位

产出物公示表是 `项目总控.md` 的核心全貌段之一。它取代了原来扁平的"关键产出索引"（文件→一句话），改为**公式化依赖链**展示：每个产出文件由哪些上游产物化合而成。

## 格式

```
| 产出 (D) | = | 输入 A | + | 输入 B | + | 输入 C | 产出者 | 消费者 | 状态 |
```

- **=** 左边是**结果文件**（哪个文件被产出）
- **=** 右边是**原料**（哪些上游文件/数据被消费）
- **产出者** = 哪个 skill 或阶段负责这条反应
- **消费者** = 哪个下游阶段或 skill 会读取这个文件
- **状态** = ✅ 存在 / ⬜ 待产出 / ❌ 缺失（由健康检查自动扫描）

## 健康检查如何消费此表

健康检查的 Step 1b 逐行扫描此表：

1. 取每行的"产出"列文件名 → 在文件系统中搜索
2. 文件存在 → 状态列标记 ✅
3. 文件不存在但对应阶段已完成 → 标记 ❌ 缺失
4. 文件不存在且阶段未开始 → 标记 ⬜ 待产出
5. **断裂检测**：如果某行的状态是 ❌（产出缺失），但该行的输入 A/B/C 文件全都存在 → ⚠️ "原料齐但未化合"——说明产出者跳票了

同时更新 `项目总控.md` 中的理想文件树标记，二者保持同步。

## 如何扩展

当管线新增阶段或产出文件时：

1. 确认新产出文件的输入来自哪些已有文件 → 填入输入 A/B/C
2. 确认产出者和消费者是哪个 skill
3. 在表中插入新行（保持与管线顺序一致的排序）
4. 同时在 §理想文件树 中添加对应的目录条目
