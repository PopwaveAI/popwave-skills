# PRD 变更传播协议

> **用途**：当下游 skill（world/character/plot）的产出与 PRD 核心设定不一致时，定义变更标记、传播路径和硬边界违反阻断机制。
>
> **核心矛盾**：PRD 被设计为"不可变宪法"，但真实创作是迭代探索。用户在 world/character 阶段可能做出改变 PRD 核心设定的决策（种族、名字、起点、力量体系）。本协议让这种变更有序传播，而非无声漂移。

---

## 一、PRD 核心字段清单

以下字段的变更必须触发传播协议：

| 类别 | 核心字段 | PRD 中的位置 | 典型变更场景 |
|:---|:---|:---|:---|
| **主角身份** | 姓名 / 种族 / 国籍 / 前世身份 | 第四块·美学基因 + 不可妥协DNA | 用户在 world 阶段改变主角种族 |
| **起点设定** | 起始地点 / 起始状态 | 第四块·全书色彩 | 用户在 world 阶段改变起点城市 |
| **力量体系** | 核心机制 / 等级框架 / 命名体系 | 第五块·世界宪法 + L1-02 | 用户在 world 阶段调整力量等级 |
| **世界宪法** | 不可违反规则 | 第五块·世界宪法 | 用户在 world 阶段增删宪法条目 |
| **不可妥协DNA** | 硬边界（❌条目） | 第八块·不可妥协DNA+边界 | 任何阶段试图突破硬边界 |

## 二、变更传播路径

### 2.1 下游→PRD 回传（核心新增）

```
下游 skill 产出与 PRD 不一致
    ↓
skill 在项目总控中标记"PRD 变更待审"
    ↓
skill 暂停后续执行，向用户报告：
  "检测到以下变更与 PRD 不一致：
   - {字段}：PRD 写 {旧值}，本次产出为 {新值}
   - 该变更属于 {类别}
   - PRD 硬边界 {是否违反}
   请确认：① 更新 PRD ② 撤回变更 ③ 修改后重新提交"
    ↓
用户确认 → PRD 更新 → 项目总控标记"已同步" → 下游继续
用户撤回 → 下游产出修正 → 重新校验
```

### 2.2 硬边界违反阻断

当检测到产出违反 PRD 不可妥协 DNA（第八块❌条目）时：

```
skill 检测到硬边界违反
    ↓
⛔ 立即暂停，不允许继续
    ↓
向用户报告：
  "⛔ 硬边界违反阻断
   - 违反条目：{PRD 第八块❌条目原文}
   - 违反内容：{产出中的违反内容}
   - 该条目的违反后果：{PRD 中标注的违反后果}
   必须选择：
   ① 修正产出以遵守硬边界
   ② 修改 PRD 硬边界（需用户明确确认后果）"
```

### 2.3 PRD→下游传播（已有，补充）

当 PRD 被更新时（无论由 creative 还是用户直接修改）：

| PRD 变更类别 | 需要刷新的下游 | 刷新方式 |
|:---|:---|:---|
| 主角身份 | world, character, plot, chapter | 重新读取 PRD，校验产出一致性 |
| 起点设定 | world, character, plot | 重新读取 PRD，校验产出一致性 |
| 力量体系 | world, plot, chapter | 重新读取 PRD + L1-02，校验一致性 |
| 世界宪法 | world, plot | 重新读取 PRD，校验宪法合规 |
| 不可妥协DNA | 全部下游 | 全部重新校验硬边界 |

## 三、项目总控变更标记

在项目总控.md 中新增"PRD 变更待审"状态区：

```markdown
## PRD 变更待审

| 时间 | 触发阶段 | 变更字段 | PRD旧值 | 产出新值 | 硬边界违反 | 用户决定 | 状态 |
|:---|:---|:---|:---|:---|:---|:---|:---|
| {时间} | {world/character/plot} | {字段名} | {旧值} | {新值} | 是/否 | {更新PRD/撤回/修改} | 待审/已同步/已撤回 |
```

## 四、各 skill 的校验职责

### 4.1 World Skill

**校验时机**：Phase 0b（读取 PRD 后）+ Phase 1（L1 产出后）

**校验内容**：
1. 主角姓名：L1-01 中的主角名 vs PRD 第四块
2. 主角种族：L1-01 中的种族 vs PRD 不可妥协DNA
3. 起始地点：L1-01 中的起点 vs PRD 第四块
4. 力量体系框架：L1-02 中的等级框架 vs PRD 第五块世界宪法
5. 硬边界逐条检查：L1 产出 vs PRD 第八块每条❌

**校验失败处理**：暂停 + 标记项目总控 + 报告用户

### 4.2 Character Skill

**校验时机**：Step 2（写卡后）

**校验内容**：
1. 主角卡姓名：角色卡中的姓名 vs PRD 第四块 + L1-01
2. 主角卡种族：角色卡中的种族 vs PRD 不可妥协DNA
3. 配角卡地点一致性：配角卡中的地点引用 vs L1-01 的起点
4. 角色卡完整性：无"（同前）"/"同上"/"内容同前"占位符

**校验失败处理**：暂停 + 标记项目总控 + 报告用户

### 4.3 Plot Skill

**校验时机**：Step 0（设计意图捕获后）

**校验内容**：
1. 主角姓名一致性：剧情线中的主角名 vs PRD + 角色卡
2. 起点一致性：剧情线中的起点 vs L1-01
3. 力量体系一致性：剧情线中的等级引用 vs L1-02

**校验失败处理**：暂停 + 标记项目总控 + 报告用户
