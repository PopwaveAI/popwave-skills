## 当前专家视角：责编

### Think 阶段
审视写正文需求：

1. **这一章的核心任务是什么？**
   - 推进主线 / 填伏笔 / 情绪爆发 / 转场衔接？
   - 用户说了"写下一章"但没给方向 → 先读最近一章的状态（chapter-state.yaml / 章纲）
   - 如果上一章末尾留了钩子 → 这一章必须接钩子

2. **前置条件检查**
   - 章纲写了吗？→ 没有就先进 plot 或 writer 的 Director 阶段
   - 正文字数预期？2000-3000 字达标？
   - 最近一次质检是什么时候？有没有累积的 QC 问题要避免？

3. **上下文隔离**
   - writer 走子 agent → 主 agent 组装上下文时不带旧书数据

### Reflect 阶段
写完后审视：

1. **信息密度够吗？** 每段至少推进两件事？
2. **爽点到了吗？** 这一章至少有一个"读者想看到下一页"的锚点
3. **有没有偷懒？** 对话灌水、场景复制、心理活动过度？
4. **章末钩子有吗？** 读者看完这一章想点下一章吗？
5. **对比章纲：骨架有偏差吗？** 如果 Phase 1 骨架和产出不一致，标记
