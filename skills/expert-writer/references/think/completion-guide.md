# completion-guide.md — 完成后引导

> 加载时机：Reflect 阶段末尾，渲染下一轮引导。
> 加载方式：`Get-Content -Encoding UTF8 -Raw`，不用 Read 工具。

---

## 引导模板

> 状态数据来源优先级：① 活记忆/活记忆.yaml（最后 event 的 chapter）② 正文/ch*.md 文件计数 ③ 项目总控.md

| 项目状态（读实际文件判断） | 引导语 |
|--------------------------|-------|
| v3-seed 完成，无L2卡 | 「种子设计完成。需要调整吗？可以开始L2卡设计了。」 |
| v3-plot 完成（L2卡已产出），无正文 | 「L2卡设计完成。需要调整吗？可以开始涌现写作了。」 |
| expert-writer(5步循环) 第 N 章完成 | 「第 N 章已完成。需要修改吗？继续写第 N+1 章吗？」 |
| L2单元最后一章完成 | 「L2单元「{名称}」已写完。建议触发弧线校准（pop-writer-v3-arc）。」 |
| v3-arc 刚完成 | 「校准完成。需要调整吗？继续写作吗？」 |
| 参考书分析刚完成（本轮任务为 pop-decon） | 「参考书分析已完成。可以基于分析结果开始种子设计，要开始吗？」 |
| 活记忆与实际章数不一致 | 追加：「⚠️ 活记忆与实际章数不一致，建议检查活记忆.yaml。」 |
| 导演意图未确认 | 追加：「⚠️ 本章导演意图（Step0）未经用户确认，禁止进入Step1。」 |
| 项目总控.md 有废弃文件标记 | 追加：「📦 有 {N} 个废弃文件，需要清理吗？」 |

## 引导纪律

1. 每轮先问「需要修改或调整吗？」，再建议下一步
2. 修改完成后自动重新读取文件状态再做引导（不是从上一轮记忆推导）
3. 引导是建议不是催促。用户说不需要 → 停在这里
4. revise重写稿用户验收(CHECK 2)后只问是否修改，不跳下一步
