# Step 0 - 项目初始化

仅在新项目或关键目录缺失时执行。

## 最小目录

```text
项目根/
  项目总控.md
  卷纲/
    运行/
  library/
    单元剧情卡/
    设定账本/
      研究档案/
      设定涌现/
      运行/
    文风DNA/
  正文/
  活记忆/
    活记忆.yaml
```

## 初始化原则

- `项目总控.md` 用 `references/project/master-control.tpl.md` 的结构。
- `卷纲/卷N-卷纲.md` 存 plot 正式卷纲：卷定位、主爽点、读者期待、本卷边界、本书涌现燃料、卷级外部燃料筛选、本卷选角表、可选剧情燃料池和幕功能位。
- `卷纲/运行/` 存 plot 过程资料：卷定位候选、本书涌现燃料扫描、卷级外部燃料筛选、可选剧情燃料池、幕定位候选、幕级外部燃料台、案例消化、审计、章节运行日志与状态快照。
- `卷纲/幕纲-幕XXX-{标题}.md` 存 plot 产出的唯一可用幕纲。
- `library/单元剧情卡/` 只存用户明确确认后需要沉淀的单元卡副本；未通过质量门的草案不得进入这里。
- `library/设定账本/` 存 seed 立项、world-foundation 底盘、arc/plot 增量冻结的设定。
- `library/设定账本/研究档案/` 存 seed 的输入消歧、证据绑定、种子展开图和外部母版卡。
- `library/设定账本/运行/` 存 seed 过程文件：故事概念候选PK、长篇尺度与商业爽点金字塔、PRD锁定记录、seed-to-world交接记录。不要混入 `卷纲/运行/`。
- `library/设定账本/设定涌现/` 存 world-foundation 全量追踪文件：PRD消费确认、设定问题清单、候选规则池、碰撞矩阵、Foundation外部燃料台、世界金字塔一致性审计、方案竞技、冻结决策日志、设定债务、金手指候选竞技。
- `library/设定账本/Foundation-*.md` 存 Foundation 结构工程正式产物，是 plot 前置硬门。
- `活记忆/活记忆.yaml` 只放当前写作必须知道的短状态，不替代设定账本。
