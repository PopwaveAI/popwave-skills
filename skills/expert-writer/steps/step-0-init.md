# Step 0：项目初始化 — 创建完整目录骨架

> **调用时机**：开新书之前，必须执行此步骤。
> **产出**：项目目录下创建完整骨架 + 管线模式确定。
> **门禁**：❌ 不可跳过。不初始化骨架就开书 = 文件散落无归档、后续 skill 找不到目标目录。

---

## ❌ 读取协议（强制）

```
工具选择：skill_view（首选）或 Get-Content -Encoding UTF8 -Raw
❌ 禁止用 Read 工具读取 skill 文件（有行数限制，会截断）
```

---

## 步骤 1a：管线模式选择（★v8.0.0新增）

询问用户选择写作管线模式：

> **请选择写作管线模式：**
> - **v2 雪花式**：creative→world→character→plot→chapter→prose→qa（8阶段线性）。设定驱动型，适合需要完整世界观的类型。
> - **v3 涌现式**：种子设计→涌现写作环↔弧线校准（3阶段循环）。压力驱动型，网文爽感优先，信息损失最小。
>
> 模式一旦确定，不可切换。建议：网文爽文选v3，设定重的类型文选v2。

用户选择后，将模式写入项目总控.md 的「管线模式」字段。

## 文件类型标记

| 标记 | 含义 |
|:-----|:------|
| **{S}** | 静态 — 一次产出，只读不写 |
| **{D}** | 动态 — 持续维护，可被追加/修改 |
| **{M}** | 元数据 — expert-writer 维护，其他 skill 读写 |
| **[skill]** | 产出者 — 哪个 skill 负责产出/维护该文件 |

## 初始化操作

1. 检查 `{项目根目录}/项目总控.md` 是否存在
   - 存在 → 项目已初始化，跳过本步
   - 不存在 → 继续执行

2. **按模式创建目录**：
   - v2 模式：创建上述 v2 骨架所有目录（`素材储备池/`、`研究档案/`、`小说世界设定/数值体系/`、`状态/角色/`、`剧情设计/`、`章节设计包/`、`正文/`、`写作资产/文风库/`）
   - v3 模式：创建上述 v3 骨架所有目录（`写作资产/文风库/`、`活记忆/`、`正文/`、`弧线校准报告/`）

3. **初始化项目总控.md**：
   - 从 `references/project/master-control.tpl.md` 读取模板
   - 写入 `{项目根目录}/项目总控.md`
   - 填写管线版本戳：读取 expert-writer SKILL.md 的 `version` 字段 → 写入 `管线版本` 和 `本项目启动时版本`
   - `📊 项目现状` 全部标记为 ⬜（无文件产出）
   - `🗺️ 所属管线` 从 `references/pipeline/manifest.md` 填充理想全流程和理想目录路由
   - `管线差异映射` 初始为空（新项目无差异）

   3a. **填充 📚 知识库路径**（★新增）：
   - 解析 pop-trope-library 绝对路径：从当前 skill 目录（expert-writer 所在目录）上溯找到 pop-trope-library 的实际安装路径，写入 `skill 公共库` 行
   - 扫描 `{pop-trope-library}/设定库/` 下所有子目录（书目），列出书名，填入「skill 公共库内容速览」表
   - 如本书是对标/移植某已有书目 → 在对应行标注 ★
   - 用户私藏参考路径初始留空（`❌待补充`），不猜测——由用户后续填入或 Agent 追问

   3b. **解析文风DNA路径**（★P0新增）：
   - 扫描 `{项目根目录}/写作资产/文风库/` 目录
   - 如目录为空或不存在 → ❌ 硬阻塞：提示用户先完成文风DNA锁定（用 pop-shared-dna 拆解对标书或用户指定）
   - 如目录有文件 → 将路径写入项目总控的「文风DNA」字段，标记 ✅已解析
   - 文风DNA路径未解析 = creative 阶段不可启动

4. 初始化完成 → **按模式路由**：
   - v2 模式 → 路由到 pop-writer-creative
   - v3 模式 → 路由到 pop-writer-v3-seed

## 注意

- `小说世界设定/L1-01~06.md` 是 **6 个独立文件**，不是子目录
- `剧情设计/幕/vol-XX/` 中的 `XX` 是卷号（如 `vol-01`），每卷一个子目录
- `state-log.yaml` 在 `状态/` 下，不在任何 `00-总控/` 目录里
- `文风库/` 在 `写作资产/` 下，不在 `章节设计包/` 里
- 数值体系全部使用 **.md** 格式，不使用 .yaml
- 项目只有一个控制文件：`项目总控.md`（根目录），没有 `workspace-index.yaml` 或 `project.yaml`
