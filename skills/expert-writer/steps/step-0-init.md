# Step 0：项目初始化 — 创建完整目录骨架

> **调用时机**：开新书之前，必须执行此步骤。
> **产出**：项目目录下创建完整骨架。
> **门禁**：❌ 不可跳过。不初始化骨架就开书 = 文件散落无归档、后续 skill 找不到目标目录。

---

## ❌ 读取协议（强制）

```
工具选择：skill_view（首选）或 Get-Content -Encoding UTF8 -Raw
❌ 禁止用 Read 工具读取 skill 文件（有行数限制，会截断）
```

---

## 目录骨架

```text
{项目根目录}/
│
├── 种子文档.md                           [v3-seed] 活种子（六要素+版本元数据+变更日志+已关闭区）
│
├── 写作资产/
│   └── 文风库/                           [v3-seed 从 library 获取+蒸馏] {S}
│       └── {书名}.md                     文风DNA档案（项目资产，修订层消费）
│
├── 素材库/                               [v3-seed 研究档案 / v3-emerge 知识沉淀] {D}
│   ├── 索引.md                           素材库统一索引（YAML front matter + MD，强制读取入口）
│   ├── 研究档案/                         [v3-seed 种子搜索法调研持久化] {S}
│   ├── 场景素材/                         [v3-emerge 按需生成场景素材卡] {D}
│   └── 知识沉淀/                         [v3-emerge WebSearch 搜索沉淀] {D}
│       ├── 制度流程/
│       ├── 场景技法/
│       ├── 创意参考/
│       └── 伏笔追踪/
│
├── 活记忆/                               [v3-seed 初始化 / v3-emerge 追加 / v3-arc 压缩]
│   └── 活记忆.yaml                       七组件 append-only 日志（baseline + event）
│
├── 正文/                                 [v3-emerge]
│   └── chXXX.md
│
├── 弧线校准报告/                         [v3-arc]
│   └── arc-XX.md
│
└── 项目总控.md                           [v3-emerge 每章更新]
```

## 文件类型标记

| 标记 | 含义 |
|:-----|:------|
| **{S}** | 静态 — 一次产出，只读不写 |
| **{D}** | 动态 — 持续维护，可被追加/修改 |
| **[skill]** | 产出者 — 哪个 skill 负责产出/维护该文件 |

## 初始化操作

1. 检查 `{项目根目录}/项目总控.md` 是否存在
   - 存在 → 项目已初始化，跳过本步
   - 不存在 → 继续执行

2. **创建目录骨架**：创建上述所有目录（`写作资产/文风库/`、`素材库/研究档案/`、`素材库/场景素材/`、`素材库/知识沉淀/制度流程/`、`素材库/知识沉淀/场景技法/`、`素材库/知识沉淀/创意参考/`、`素材库/知识沉淀/伏笔追踪/`、`活记忆/`、`正文/`、`弧线校准报告/`）

3. **初始化素材库索引**：创建 `素材库/索引.md`（YAML front matter + MD空索引，等待 seed 研究档案和 emerge 知识沉淀填充）

4. **初始化项目总控.md**：
   - 从 `references/project/master-control.tpl.md` 读取模板
   - 写入 `{项目根目录}/项目总控.md`
   - 填写管线版本戳：读取 expert-writer SKILL.md 的 `version` 字段 → 写入 `管线版本` 和 `本项目启动时版本`
   - `📊 项目现状` 全部标记为 ⬜（无文件产出）
   - `🗺️ 所属管线` 从 `references/pipeline/manifest.md` 填充理想全流程和理想目录路由
   - `管线差异映射` 初始为空（新项目无差异）

   4a. **填充 📚 知识库路径**：
   - 解析 pop-trope-library 绝对路径：从当前 skill 目录（expert-writer 所在目录）上溯找到 pop-trope-library 的实际安装路径，写入 `skill 公共库` 行
   - 扫描 `{pop-trope-library}/设定库/` 下所有子目录（书目），列出书名，填入「skill 公共库内容速览」表
   - 如本书是对标/移植某已有书目 → 在对应行标注 ★
   - 用户私藏参考路径初始留空（`❌待补充`），不猜测——由用户后续填入或 Agent 追问

   4b. **解析文风DNA路径**：
   - 扫描 `{项目根目录}/写作资产/文风库/` 目录
   - 如目录有文件 → 将路径写入项目总控的「文风DNA」字段，标记 ✅已解析
   - 如目录为空 → 标记 ⏳待蒸馏（文风库将在种子设计阶段由 v3-seed 产出，不阻塞初始化）
   - 文风DNA路径供 emerge 修订层消费

5. 初始化完成 → **路由到 pop-writer-v3-seed**
