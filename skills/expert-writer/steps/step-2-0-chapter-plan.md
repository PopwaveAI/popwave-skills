# Step 0：种子文档任务表更新+本章聚焦

> 参考: expert-writer 涌现写作环
> 消费: 种子文档.md + 活记忆 + 上一章正文末500字 + 方向提示 + 网文爽感机制
> 产出: 更新后的种子文档.md（任务表状态变更+本章聚焦段填写）

## 目的

写之前先想明白。读种子文档+上一章正文，判断这章要达成什么——更新任务表状态+填写本章聚焦（含三问）。本章聚焦只定义"达成什么"，把"怎么达成"（场景设计）留给 create 环节涌现。做完更新，用户确认后进 Step 1 强制化信息获取。

> v3.5 关键变化：不再产出独立的 chapter_plan 文件。叙事结构由"种子文档任务表（方向）+ 压力矩阵（节奏）+ 目的地任务链（终点）"三角约束驱动。大纲和卷纲的功能被内化为任务表结构。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `种子文档.md`（当前版本） | 任务表（主线/压力/线索/目的地）+ 本章聚焦段——这章可用的故事引擎与待填写规划 |
| 2 | `活记忆/活记忆.yaml`（最后baseline+event） | 七组件当前状态——压力倒计时到哪了、哪些线索超期、节奏趋势 |
| 3 | `正文/ch{上一章}.md`（末尾500字） | 衔接——上章留了什么悬念，本章从哪里接 |

| 5 | `references/emerge-loop/网文爽感机制.md` | **写作法则**——本章聚焦必须对照检查 |

## 做什么

### 1. 故事状态感知

从活记忆七组件提取当前状态：

| 组件 | 提取什么 | 用于判断 |
|:-----|:---------|:---------|
| 压力状态 | 哪些倒计时在走？剩多少？ | 本章必须推进哪个压力源 |
| 种子追踪 | 哪些线索活跃？哪些超期预警？ | 本章该触面哪条线索 |
| 节奏日志 | 最近3章的事件密度趋势 | 本章该加速还是减速 |
| 战力曲线 | 主角当前能力水平 | 本章能力使用边界 |
| 目的地进度 | 最近的目的地进展 | 本章是否推进 |
| 角色状态 | 主角心理/资源/关系 | 驱动力哪层在激活 |

### 2. 任务表状态更新

基于状态感知，更新种子文档任务表中各项任务的状态：

| 任务类型 | 状态变更 | 判断依据 |
|:---------|:---------|:---------|
| 主线任务 | 进行中→已完成 / 待启动→进行中 | 上一章是否达成阶段性目标 |
| 压力任务 | 进行中→已逼近 / 进行中→已爆发 | 压力倒计时是否到临界点 |
| 线索任务 | 待触面→已触面 / 已触面→已深化 / 已深化→已回收 | 上一章是否推进该线索 |

**状态更新原则：**
- 只更新已完成或状态发生变化的任务，不变的不动
- 每个状态变更必须附变更理由（引活记忆event）
- 新增任务：若状态感知发现新的压力源/线索，在任务表对应分区追加

### 3. 本章聚焦填写（≤300字）

在种子文档的「本章聚焦」段填写本章规划，包含以下要素：

**① 推进项**：本章推进哪些已有任务？
- 从任务表选≥3条进行中任务，标注推进方式（触面/深化/回收）
- 不写具体场景，只标注每条任务本章的目标状态变化

**② 新增项**：本章是否新增任务？
- 若有新增压力源/线索/目的地子目标，标注新增理由
- 无新增则标注"本章无新增任务"

**③ 钩子方向**：章末钩子方向？
- 只给方向：即时危机型（下章开头需解决），不是远期悬念型（如"7天后决战"）
- 不写具体内容

**④ 本章三问**（信息问题/压力问题/钩子问题）：
- **信息问题**：本章需要向读者释放/获取什么具体信息？（≥1个具体信息点）
- **压力问题**：本章推进哪个压力源？以什么方式推进？（≥1个压力源+推进方式）
- **钩子问题**：本章章末用什么即时危机型钩子？（1个即时危机型钩子方向）

> 本章聚焦总字数≤300字。只定义"达成什么"，不定义"怎么写场景"——场景设计交给 create 环节涌现。

### 4. 网文爽感机制对照检查

将本章聚焦对照`references/emerge-loop/网文爽感机制.md`的10条法则。每条法则必须附本章聚焦中的设计证据，不能空泛打勾：

| 法则 | 本章设计证据（取自本章聚焦，非空泛打勾） | 如不满足的调整 |
|:-----|:--------------------------------------|:-------------|
| 压力机制 | {压力源如何逼近/爆发——引三问·压力问题} | 补充压力源逼近 |
| 即时奖励循环 | {读者获得中的爽感项如何即时兑现——引推进项} | 补充爽点 |
| 不间断危机链 | {章末危机钩子如何承接上章压力——引三问·钩子问题} | 补充章末危机钩子 |
| 高事件密度(8-12) | {本章推进项数量决定的事件密度取向} | 补充密度 |
| 主角主动决策 | {推进项如何驱动本章} | 改被动为主动 |
| 养成感可见 | {推进项带来的能力资源变化} | 补充能力/资源变化 |
| 信息披露节奏 | {信息问题如何分层释放} | 补充世界信息 |
| 80%展示20%内省 | {展示与内省配比预期} | 标注内省点 |
| 多线交叉(≥3线) | {≥3条线索如何交叉——引推进项} | 补充线索推进 |
| 情绪闭环 | {本章情感维度如何闭环} | 检查情绪弧线 |

无设计证据或证据空泛的法则，回到步骤3调整本章聚焦。

### 5. 落盘+用户确认

将更新写入种子文档.md（任务表状态变更+本章聚焦段填写+法则对照记录）：

**向用户展示更新摘要并请求确认：**
- 展示任务表状态变更（哪些任务状态变了+理由）
- 展示本章聚焦（推进项+新增项+钩子方向+三问）
- 展示法则对照结果（每条附设计证据，非空泛打勾）
- 等待用户确认

**用户确认后：**
- 进入 Step 1 信息获取（Step 1 将先验证三问，再强制读资料总索引）

**用户要求修改：**
- 按用户反馈调整本章聚焦/任务表
- 更新种子文档后重新确认

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 种子文档.md不存在 | 终止，提示先完成pop-writer-v3-seed |
| 活记忆不存在 | 终止，提示先初始化活记忆 |
| 任务表状态已更新（有变更则附理由） | 补充变更理由 |
| 本章聚焦含三问（信息问题/压力问题/钩子问题） | 退回补答三问 |
| 网文爽感机制10条法则全部对照且每条附设计证据（红线❌5） | 未对照/证据空泛的法则回到步骤3调整 |
| ≥3条线索推进（推进项标注触面/深化/回收） | 补充线索 |
| 章末即时危机钩子（只给方向） | 补充钩子方向 |
| **用户未确认** | **不进入Step 1，等待确认** |
