# Step 1：信息获取（强制化）

> 参考: expert-writer 涌现写作环
> 消费: Step 0本章规划 + 种子文件夹 + 活记忆 + 写作资产/设定库（如有）
> 产出: 信息获取记录（info_acquired）
> 红线: ❌6 信息获取必须读资料总索引——资料总索引.md未读取=退回补读

## 目的

v3.0的"自主判断要不要查"导致16章跳过信息获取。v3.1改为强制化：每章必须读资料总索引，读后判断有没有匹配，有则读素材库文件，无则WebSearch并写入素材库/知识沉淀。信息获取不可跳过。v3.2新增搜索深度标准和无需求门槛提高。v3.4以资料总索引.md统一取代散落的素材库/索引.md和设定库/_index.yaml。

**与v3.0的核心差异：** 不是"判断要不要查"，而是"先读资料总索引再说"。资料总索引是每章的必经之门。

## 读什么

|:-:|:-----|:-------|

| 2 | `资料总索引.md`（项目根目录，唯一索引入口） | **强制读取（红线❌6）**——聚合素材库(知识沉淀/研究档案/场景素材)+设定库+文风DNA的统一清单，判断有没有匹配 |

## 做什么

### 0. 本章三问验证

读取种子文档本章聚焦段的三问，验证每个问题有具体回答：

| 三问 | 验证标准 | 未通过动作 |
|:-----|:---------|:-----------|
| 信息问题 | ≥1个具体信息点（非空泛描述） | 退回Step 0补答 |
| 压力问题 | ≥1个压力源+推进方式（非空泛描述） | 退回Step 0补答 |
| 钩子问题 | 1个即时危机型钩子方向（非远期悬念型） | 退回Step 0补答 |

**门禁：三问任一未回答或回答空泛=退回Step 0补答。不允许跳过。**

> 三问是Step 0本章聚焦的核心产出，Step 1以此为入口验证规划质量。三问未通过=本章规划不完整，必须退回Step 0。

### 1. 强制读资料总索引（红线❌6门禁）

**第一步永远是读资料总索引。** 打开项目根目录 `资料总索引.md`，查看已沉淀的所有资料清单。

- 资料总索引不存在 → 首次执行，退回 `step-0-init` 补建资料总索引骨架
- 资料总索引已读 → 基于 info_gaps 在总索引中查找匹配条目，进入步骤2

**门禁：资料总索引.md 未被读取，退回补读。不允许跳过。禁止再读散落的 `素材库/索引.md` 或 `设定库/_index.yaml`——它们已被资料总索引取代。**

### 1.5 查设定库条目（在资料总索引中查找）

在 `资料总索引.md` 的「设定库」分区查找涉及的力量体系 / 社会结构 / 角色档案等条目，按需读取对应设定文件：

| 判断条件 | 动作 |
|:---------|:-----|
| 涉及力量体系/能力使用 | 读 `写作资产/设定库/力量体系.md` |
| 涉及社会制度/势力格局 | 读 `写作资产/设定库/社会结构.md` |
| 涉及世界法则/规则约束 | 读 `写作资产/设定库/世界宪法.md` |
| 涉及已建档角色 | 读 `写作资产/设定库/角色档案/{角色名}.md` |
| 涉及卷纲方向 | 读 `写作资产/设定库/卷纲路标/`（参考但不强制遵循） |
| 不涉及任何设定 | 标记 `setting_lib_read: false` |

> 设定库是 ground truth 参考层，不是强制消费。读设定库是为了确保世界观一致性（力量体系怎么运作、社会结构怎么分层），剧情走向仍由种子引擎驱动。设定库条目统一登记在资料总索引的「设定库」分区，不再单独读取 `设定库/_index.yaml`。

### 2. 基于种子文档本章聚焦段做4类检查

从种子文档本章聚焦段的推进项、三问中提取信息需求。

**v3.5更新：优先处理种子文档本章聚焦段的三问（Step 0产出的信息需求）**

Step 0本章聚焦已产出三问（信息问题/压力问题/钩子问题），列出本章的信息缺口。Step 1必须优先处理这些缺口：

| 三问类型 | 处理方式 |
|:---------|:---------|
| 信息问题涉及外部资料 | **必须WebSearch**，搜索结果写入素材库/知识沉淀/对应子目录 |
| 信息问题涉及已有素材 | 先查资料总索引→有则读本地→无则WebSearch |

处理完三问信息需求后，再做4类补充检查：

| 检查类型 | 判断问题 | 信息源 | 示例 |
|:---------|:---------|:-------|:-----|
| 制度/流程细节 | 规划中的场景涉及什么我不熟悉的制度/流程？ | WebSearch | 催债人上门→查美国催债法流程 |
| 场景技法 | 规划中的场景类型需要参考技法吗？ | library套路库 | 帮派对峙→查套路库的压迫感营造 |
| 已埋伏笔/前文细节 | 规划中要推进的线索需要呼应哪些前文？ | 项目本身（活记忆+前章正文） | MS-13线索→查ch1的老约翰之死细节 |
| 创意参考 | 规划中有什么场景需要灵感参考？ | WebSearch/library | 绝境反杀→查类似场景的节奏控制 |

### 3. 资料总索引匹配判断

对每个信息需求，先查资料总索引中是否已有匹配的素材库文件：

```
信息需求 → 查资料总索引
  ├── 总索引中有匹配文件 → 读该素材库文件 → 提取关键信息 → 标记source: 素材库/知识沉淀/{文件名}
  └── 总索引中无匹配文件 → WebSearch/library查询 → 提取关键信息 → 写入素材库/知识沉淀 → 更新资料总索引
```

### 4. 信息获取（如有需求）

按需求清单逐项获取：

**4a. 资料总索引中有匹配 → 读素材库文件**
- 从 `素材库/知识沉淀/{分类}/{文件名}.md` 读取
- 提取关键信息融入写作上下文
- 在info_acquired记录中标记 `source: 素材库/知识沉淀/{文件名}.md`

**4b. 资料总索引中无匹配 → WebSearch并写入素材库/知识沉淀**
- WebSearch搜索制度/流程/创意参考
- 提取关键信息：流程步骤、关键规则、现实约束
- **写入素材库/知识沉淀**：创建 `素材库/知识沉淀/{分类}/{文件名}.md`，写入搜索结果
- **更新资料总索引**：在 `资料总索引.md` 对应分区追加条目
- 在info_acquired记录中标记 `source: WebSearch→素材库/知识沉淀/{文件名}.md`

**4c. library查询（套路库）**
- 查询 `pop-trope-library/套路库/{套路名}.md`
- 提取场景技法/渲染参考
- 在info_acquired记录中标记 `source: library套路库/{套路名}.md`

**4d. 项目本身（活记忆+前章正文）**
- 查阅活记忆种子追踪组件：已埋伏笔清单+状态
- 查阅前章正文：具体伏笔细节
- 提取待回收伏笔的内容和预期回收点
- 在info_acquired记录中标记 `source: 活记忆+前章正文`

### 4e. 搜索深度标准（v3.2新增）

WebSearch获取的信息必须满足搜索深度标准：

| 维度 | 标准 | 不达标处理 |
|:-----|:-----|:-----------|
| 文件字数 | 每篇知识沉淀文件≥500字 | 补充搜索结果至达标 |
| 内容深度 | 必须含具体案例/流程步骤/数据数值（至少其一） | 追搜补充具体细节 |
| 多轮搜索 | 首次搜索结果不足→追搜（换关键词/换角度），最多3轮 | 3轮仍不足→标注"信息有限"并继续 |
| 结构化 | 按知识沉淀文件模板结构化 | 重新整理 |

**多轮搜索策略：**
1. 首次搜索：用种子文档本章聚焦段提取的核心关键词搜索
2. 首次结果评估：是否有具体案例/流程/数据？字数是否≥500？
3. 不足→追搜：换关键词（更具体）/ 换角度（从案例反查流程）/ 换信息源
4. 最多3轮，3轮后仍不足→标注"信息有限，创作时酌情处理"

### 5. 无需求处理（v3.2门槛提高）

4类检查均无明确需求 → `needs: []` 仍合法，但**必须逐类说明不需求理由**：

```yaml
info_acquired:
  needs: []
  no_need_reasons:
    制度流程: "{具体理由}"
    场景技法: "{具体理由}"
    已埋伏笔: "{具体理由}"
    创意参考: "{具体理由}"
```

**门禁：no_need_reasons 四项未全部填写 = 退回补填。**

**连续3章 `needs: []` → 标记信息获取退化预警。**

**核心原则：允许"无需求"，但不允许"不读资料总索引"，也不允许"敷衍式跳过"。** 常规章节（日常生存、已建立机制的运作）可以无需求，但每章都必须读资料总索引确认，且必须说明为何无需求。

### 6. 登记记录

按 `templates/信息获取记录-模板.md` 格式，填写info_acquired记录（YAML），写入活记忆event的`info_acquired`字段。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| **资料总索引.md未读取（红线❌6）** | **退回补读资料总索引** |
| 种子文档本章聚焦段不存在 | 退回Step 0补做本章聚焦 |
| 信息获取记录未填写 | 补填后再进Step 2 |
| WebSearch获取的信息未写入素材库/知识沉淀 | 补写入后再继续 |
| 素材库新文件未更新资料总索引 | 补更新资料总索引后再继续 |
| **知识沉淀文件字数<500字** | **补充搜索结果至达标** |
| **needs:[]但no_need_reasons未全部填写** | **退回补填** |
| **连续3章needs:[]** | **标记信息获取退化预警** |

> 注意：本步骤有"必须读资料总索引"门禁（红线❌6），允许"无需求"（needs: []），但无需求必须逐类说明理由（no_need_reasons）。

## 产出

信息获取记录（YAML格式），写入活记忆event的`info_acquired`字段。包含：
- chapter（章号）
- index_read（资料总索引已读取标记）
- setting_lib_read（设定库索引是否已读取，如存在）
- setting_files_read（本章读取的设定库文件列表）
- needs（4类检查结果清单，或空数组）
- no_need_reasons（无需求时4类不需求理由，有需求时省略）
- merged（是否已融入写作上下文）
- kb_new_files（本章新建的素材库文件列表）
- index_updated（索引是否已更新）
