## 1. 活记忆追加

> v3.5 活记忆格式：自然语言段落，每章一段话概括状态变化。
> 不再是7组件结构化YAML。

### 1.1 读取当前活记忆

```powershell
Get-Content -Encoding UTF8 -Raw '活记忆/活记忆.yaml'
```

### 1.2 追加本章event

从本章正文 + 导演意图 + 状态快照，提炼一段自然语言状态变化记录：

```yaml
# 活记忆/活记忆.yaml 追加内容

events:
  - chapter: 3
    summary: >
      第3章：主角在废弃厂房双杀铁钳帮追兵，触发系统首次升级（Lv1→Lv2），
      解锁街头感知技能。搜索圈暂时解除，但发现命运迷宫纹出现在尸体上，
      灵能警告触发，暗示超自然威胁。主角心理从压抑转向警觉，
      保护欲驱动力增强。铁钳帮搜索线推进到"暂时解除但未根除"。
      超自然线首次浮现（命运迷宫纹+灵能警告）。系统线升级到Lv2。
    state_changes:
      protagonist_ability: "Lv1→Lv2, 新增街头感知"
      protagonist_psychology: "压抑→警觉"
      pressure_iron_claw: "搜索圈逼近→暂时解除"
      pressure_supernatural: "无→灵能警告触发"
    pending_updates:
      new_foreshadowing: ["命运迷宫纹的含义", "灵能警告的来源"]
      advanced_promises: ["系统升级的后续影响（已部分兑现）"]
      new_reveals: ["铁钳帮背后可能有超自然势力"]
    timestamp: "2026-06-28T14:30:00"
```

### 1.3 活记忆追加规则

- 每章只追加一个event
- summary 是自然语言段落（一段话），概括本章所有状态变化
- state_changes 是结构化补充（关键状态变化点）
- pending_updates 更新待兑现要素（新增/推进/兑现）
- 不删除已有events（活记忆是累积日志）
- 弧线校准时由arc负责压缩旧events

## 4. 弧线触发检查

### 4.1 判断是否为L2单元最后一章

> v3.5 弧线触发：每个L2单元结束时触发arc（完整5步全跑）。
> 不再等"每10-20章"。

1. 从L2卡结构分析表获取当前L2单元的总章数
2. 判断当前章号是否为该L2单元的最后一章
3. 如是 → 触发弧线校准
4. 如否 → 5步循环完成，等待用户"继续"指令进入下一章

### 4.2 触发arc

如本章是L2单元最后一章：

1. 更新项目总控：`管线阶段: 涌现写作环 → 弧线校准`
2. 提示用户：
   > 第{chapter}章已落盘。这是L2单元「{L2单元名称}」的最后一章。
   > 建议触发弧线校准（pop-writer-v3-arc），对L2单元进行长线结构检查。
   > 确认后开始弧线校准。
3. 用户确认 → 路由到 pop-writer-v3-arc

### 4.3 不触发arc

如本章不是L2单元最后一章：

1. 5步循环完成
2. 进入 `step-3-reflect.md`（通用审视+引导）
3. 引导语：
   > 第{chapter}章已落盘。L2单元「{L2单元名称}」进度 {已完成}/{总章数}。
   > 输入"继续"开始第{chapter+1}章。
