# Step 2 - 执行路由

按 Step 1 的判断路由到执行 skill。你只做上下文组装和交接，不在这里重新设计流程。

## 路由输入

给执行 skill 的上下文保持完整但不臃肿：

- 当前目标：用户要得到什么产物。
- 项目位置：项目根目录和目标输出路径。
- 前置材料：相关幕纲、运行日志、设定账本、文风DNA、上章末尾、活记忆。
- 公共资料库：`pop-trope-library` 路径、任务标签、候选案例、库缺口声明。
- 外部类型母版：读取 `library/设定账本/跨媒介母版雷达.md`；用户提到的影视、美剧、漫画、游戏、电影或经典桥段必须额外传给 plot；即使用户没提，plot 也要主动扫描外部母版。
- 约束：用户刚刚提出的偏好、禁止项、必须保留项。

## 执行 skill 加载规则

Pop 通常只强制注入 expert-writer。路由到执行 skill 时，不允许只凭本文件概述执行。必须读取本地对应 skill 文件：

- seed：读 `skills/pop-writer-v3-seed/SKILL.md`，再按需读它的 `steps/step-*.md`。
- world-foundation：读 `skills/pop-writer-v3-world-foundation/SKILL.md`，再按需读它的 `steps/step-*.md`。
- plot：读 `skills/pop-writer-v3-plot/SKILL.md`，再读它声明的 step 文件。
- create/revise/arc：读对应 `SKILL.md` 和 step 文件。

若运行环境无法读取本地 skill 文件，必须在回复里声明“执行 skill 未加载，只能做对齐草案”，不得落正式产物。

## 执行规则

- 调 `seed`：先做调研水下层、种子展开、外部母版、故事概念候选 PK 和用户确认，再锁定新书 PRD，最后输出 `seed-to-world交接包.md`。seed 只负责立项，不产 L1、金手指、文风DNA。正式产物写入 `library/设定账本/`，过程文件写入 `library/设定账本/运行/` 或 `library/设定账本/研究档案/`，不得写入 `卷纲/运行/`。
- 调 `world-foundation`：消费新书 PRD、研究档案和 seed-to-world 交接包，建立 `library/设定账本/设定涌现/` 全量追踪目录，产出 L1 世界包、起点快照、世界宪法、动态升级表、主角引擎、金手指候选竞技、开局契约、文风DNA、Library 查询方向和跨媒介母版雷达。缺设定涌现目录时不得称为底盘完成。
- 调 `plot`：先消费 world 产物索引、设定涌现目录、新书 PRD、L1 世界包、起点快照、世界宪法、动态升级表、主角引擎、金手指、开局契约和文风DNA，再做全书/卷级尺度契约、卷级设计意图契约、卷级战略锚、卷内幕序列、当前幕切片，然后做案例消化、外部类型母版主动扫描与移植、事件源发散、故事引擎铸造、三套同等颗粒度互斥候选竞技、章级导演包，最后压缩幕纲。除非用户明确说“无需中途确认/全自动跑完”，否则 plot 至少在“卷级尺度/设计意图”和“当前幕切片”阶段停下让用户确认。
- 调 `create`：让它根据章级导演包、章节运行日志、案例消化摘要和爽文审计结果写正文，不要求它重新设计剧情。缺章级导演包时只能标记为降级草案。
- 调 `revise`：只在用户明确要改正文时调用。
- 调 `arc`：让它事后复盘真实发生的剧情线和设定变化。

## 写作规划交互规则

- 剧情规划不是一次性黑箱生成。plot 必须把中间过程落盘到 `卷纲/运行/`。
- 不要把 plot 的 Step 3/4 私下交给子 Agent 或隐藏分支，除非用户明确要求独立验证或后台并行。写作规划需要用户能看见进度、能中途干预。
- “完整执行一场”默认含义是“按完整流程推进”，不是“跳过所有确认”。只有用户明确说“不要中途确认”才允许一口气跑完。
- 如果用户表达不满意、要求复盘、要求重跑，优先读取 run 日志和过程文件，定位流程断点，不要只加 checklist。

## Seed 立项硬要求

调 `seed` 时必须传入这条要求：seed 不是设定百科，而是研究档案 + 故事概念 PK + 新书立项 PRD + world 交接包。

seed 输出必须包含：

- `library/设定账本/研究档案/00-研究索引.md`
- `library/设定账本/研究档案/01-种子展开图.md`
- `library/设定账本/研究档案/02-外部母版卡.md`
- `library/设定账本/运行/00-故事概念候选PK.md`
- `library/设定账本/新书立项PRD.md`
- `library/设定账本/seed-to-world交接包.md`
- `library/设定账本/seed-产物索引.md`

seed 必须通过三道门：

- 研究水下门：必须有输入消歧、证据绑定、种子展开、外部母版卡。
- 概念 PK 门：必须有 2-3 个互斥故事概念和用户确认/自动选择理由。
- 新书 PRD 门：必须有篇幅体量、主角起点/终点、世界机器、元爽点和禁区。
- 交接门：必须明确不可涌现项、允许涌现项、证据索引和风险清单。

如果缺这些，expert-writer 不得说“seed 完成”，只能说“当前为 seed 草案，缺研究/概念PK/PRD/交接门”。

## Plot 候选竞技硬要求

调 `plot` 时必须传入这条要求：多方案不是 A 方案的微调，而是三套互斥故事引擎的竞技。

plot 输出必须包含：

- `卷纲/运行/幕XXX-03-候选竞技.md`
- `卷纲/运行/幕XXX-03a-方案A全章.md`
- `卷纲/运行/幕XXX-03b-方案B全章.md`
- `卷纲/运行/幕XXX-03c-方案C全章.md`
- 最终幕纲内的章级导演包，或 `卷纲/章节设计包/幕XXX-chYYY-设计包.md`
- 三套候选 A/B/C，各自六个故事引擎、完整逐章运行日志、子线咬合表。
- 三套候选必须同等颗粒度；不得出现 A 是完整稿、B/C 是概览或差异标注。
- 三套候选的主类型母版/主剧情机制、主压力、第一主动动作、收益路径、敌人误判链、幕终点收益至少 4 项不同。
- 每次 plot 都必须有外部类型母版扫描文件 `卷纲/运行/幕XXX-02b-外部母版扫描.md`；用户提出外部作品时，必须有外部类型母版卡、`M-E` 事件源，或明确淘汰理由。
- 竞技场评分、淘汰理由、胜出理由。
- `卷纲/运行/幕XXX-产物索引.md`

如果缺这些，expert-writer 不得在最终回报中称“幕纲完成”，只能说“当前为草案，缺候选竞技/质量门”。

## 禁止

- 禁止恢复历史写作环术语。
- 禁止把旧 changelog、废弃 references 当流程来源。
- 禁止在 expert-writer 内部抢执行 skill 的活。
