# step-3-reflect.md — 通用审视 + 项目总控回写 + 引导

> **读什么**：子 skill 产出物
> **产出什么**：审视结论 + 项目总控.md 更新 + 引导语

---

## ❌ 读取协议（强制）

```
工具选择：skill_view（首选）或 Get-Content -Encoding UTF8 -Raw
❌ 禁止用 Read 工具读取 skill 文件（有行数限制，会截断）
```

---

## 通用检查三问

| # | 问 | 检查方式 |
|:-:|:---|:---------|
| 1 | **产出回答了用户问的问题吗？** | 对比用户原始需求和产出结果 |
| 2 | **有没有超出 scope 的多余产出？** | 用户没要求的东西写多了 = 浪费 |
| 3 | **有没有明显的盲点被忽略？** | "看起来对但站不住"的结论 |

## 阶段级读者体验验收

> 每个子 skill 完成后，除了文件级检查，必须做一次"读者视角"验收。
> 不是检查文件格式对不对，是检查"读者读到这个产出会想继续吗？"

| 阶段完成 | 验收问题 | 不通过动作 |
|:---------|:---------|:-----------|
| v3-seed | 种子六要素能否驱动前 3 章？（核心压力+主角弧线+世界规则+爽点承诺是否自洽） | 退回 v3-seed 调整种子 |
| expert-writer(涌现写作环)（每章） | 质检子skill已完成五问反思+种子生长判断+爽点终验，expert-writer 只做宏观检查：章号连续性+活记忆一致性+方向提示存在性 | 退回 expert-writer(涌现写作环) 重写该章 |
| v3-arc | 校准报告六项检查是否通过？（节奏/伏笔/角色弧线/世界一致性/爽点密度/读者体验） | 退回 v3-arc 重新校准 |

**方向提示检查**（expert-writer 每章完成后）：
- 确认 expert-writer 已在活记忆中生成方向提示（下一章走向建议）
- 方向提示缺失 → 退回 expert-writer 补写

**弧线校准触发检查**（expert-writer 每章完成后）：
- 章号差值 ≥ 10（距上次校准）→ 建议触发 v3-arc
- 发生重大剧情转折（expert-writer 标注）→ 建议触发 v3-arc
- 用户明确要求 → 触发 v3-arc
- 未触发 → 继续涌现写作

## 质量信号

| 信号 | 行动 |
|:-----|:-----|
| 完美覆盖需求 | 直接交付，一句话总结 |
| 小细节偏了 | 标记偏差，再问用户是否接受 |
| agent 偷懒 | 人工检查 |
| 南辕北辙 | 不退——复盘路由错在哪 |

> **详细审视清单由各子 skill 自管。** expert-writer 不重复子 skill 的落盘检查、一致性校验、质检、活人感检查。如果子 skill 产出有质量问题，退回子 skill 重做。

## ⛔ 项目总控回写（强制执行 — 不更新 = 阶段未完成）

> **这是 reflect 步骤的硬性执行项，不是可选建议。**

**每次子 skill 执行完成后，必须立即执行以下操作：**

1. **读取 `项目总控.md`**（用 `Get-Content -Encoding UTF8 -Raw`）
2. **更新上半屏（📊 项目现状）**：
   - 首屏仪表盘：将刚完成的阶段标记从 ⏳/⬜ 改为 ✅
   - 实际阶段执行表：追加已完成阶段行
   - 产出物清单：扫描文件系统，更新状态列
   - 当前进度锚点：读 `活记忆/活记忆.yaml` 最后一条 baseline + 之后的 event，提取：当前章号（最后一条 event 的 chapter）、种子版本、弧线计数、方向提示状态
   - 追加执行顺序日志：`| {序号} | {阶段名} | {时间戳} | {备注} |`
3. **写入更新后的 `项目总控.md`**
4. **确认写入成功**（重新读取验证）
5. **下半屏（🗺️ 所属管线）不动** — 只有版本漂移时才刷新

**⛔ 门禁检查：**
- 项目总控.md 是否已更新？→ 未更新 = 阶段未完成，禁止进入下一阶段
- 产出物清单是否与实际文件系统一致？→ 不一致 = 补齐后再进入下一阶段

### 活记忆压缩检查

每次 reflect 时检查活记忆是否需要压缩：

1. 读活记忆最后一条 baseline 的章号 + 最后一条 event 的章号
2. 差值 ≥ 20 → 执行压缩：合并 baseline + event 为新 baseline，更新压缩计数
3. 差值 < 20 → 跳过
4. 压缩计数写入项目总控.md

## 完成后引导

### 引导纪律

1. 每轮先问「需要修改或调整吗？」，再建议下一步
2. 修改完成后重新读取文件状态再做引导
3. 引导是建议不是催促
4. 质检后只问是否修改

### 引导模板（概要）

| 项目状态 | 引导语 |
|:---------|:-------|
| v3-seed 完成 | 「种子设计完成。需要调整吗？可以开始涌现写作了。」 |
| expert-writer(涌现写作环) 第 N 章完成 | 「第 N 章已完成。需要修改吗？继续写第 N+1 章吗？」 |
| v3-arc 完成 | 「校准完成。需要调整吗？继续写作吗？」 |

> 完整引导表 → `references/think/completion-guide.md`
