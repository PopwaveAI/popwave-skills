```
工具选择：skill_view（首选）或 Get-Content -Encoding UTF8 -Raw

```

## 1. 通用3问（v3.5）

5步循环完成后，对每章产出进行通用审视：

### 1.1 门禁表遵守

- Step0 导演意图是否经用户确认？（红线4）
- Step1 状态快照是否≤400字？
- Step2 设定指针是否全部强制读取？
- Step3 create/revise子agent是否先读子SKILL.md再路由？（红线2）
- Step4 receipt检查是否6项全部通过？
- Step5 活记忆+正文+项目总控是否三文件更新？

### 1.2 导演意图达成

- 三问是否全部回答？
- event_chain是否完整推进？
- emotion_curve和pacing是否遵循？
- chapter_hook是否已留？

### 1.3 活记忆健康度

- 活记忆是否每章一段话追加？
- pending要素（伏笔/承诺/揭示）是否及时更新？
- 活记忆是否过长需要arc压缩？

## 2. 项目总控回写

> Step5已更新项目总控的基本字段。本步补充审视结论。

1. `Get-Content -Encoding UTF8 -Raw` 读取 `项目总控.md`
2. 补充以下字段：
   - `审视结论`：通过/有问题
   - `问题记录`：如有问题，记录具体问题和修复建议
   - `下一步建议`：继续下一章 / 触发arc / 暂停修复
3. 写回 `项目总控.md`

## 4. 弧线校准检查

> v3.5：每个L2单元结束时触发arc。

检查项目总控的L2单元进度：
- 如当前章是L2单元最后一章 → 引导用户触发arc
- 如当前章不是最后一章 → 引导用户继续下一章

> arc触发后，pop-writer-v3-arc 会：
> 1. 检查L2单元的结构完整性
> 2. 校准L3剧情线
> 3. 压缩活记忆（如过长）
> 4. 修剪失效要素（归档到写作参考/已废弃/）
> 5. 更新L2卡（如需调整后续章节结构）
