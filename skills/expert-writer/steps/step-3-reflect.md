# Step 3 - 简短回报

结束时告诉用户三件事即可：

1. 本次完成了什么产物。
2. 产物写到了哪里。
3. 下一步最自然推进什么。
4. 如果处于 plot 阶段且需要用户确认，明确列出“待确认项”和 2-3 个可选方向。
5. 如果处于 plot 阶段且生成了多个过程文件，必须明确告诉用户：唯一可用稿是哪一个；哪些只是过程文件；最终稿是否通过质量门；如果未通过，下一步补什么。
6. 如果处于 plot 阶段，必须说明是否完成卷定位/幕定位确认、本书涌现燃料扫描、外部燃料筛选、本卷选角表、可选剧情燃料池和章级施工卡。没有完成时，不得说“自检通过/执行完成”。
7. 如果处于 seed/world 阶段，必须说明是否通过长篇承诺门或 Foundation结构工程门；没有通过时，不得说“立项完成/底盘完成”。
8. 如果处于 create 阶段，必须说明是否拿到 plot 章级施工卡；没有拿到时，不得说“正式正文”。

不要长篇解释内部流程；除非用户要求，不展示完整校验细节。
