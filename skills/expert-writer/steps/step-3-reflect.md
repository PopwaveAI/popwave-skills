# Step 3：轻量收尾与下一步提示

> 本步只做简短状态反馈，不做旧式审稿。

## 1. 收尾检查

根据本次路由类型检查最小结果：

| 路由 | 检查 |
|:-----|:-----|
| seed | 故事 DNA、硬设定、设定生长语法、初始种子池是否产出 |
| plot | 状态契约、运行日志、设定账本、单元卡是否产出 |
| create | 正文是否落盘，活记忆是否追加 |
| revise | 是否响应用户修改目标，是否保持运行日志结算 |
| arc | 是否更新实际单元卡、设定账本、活记忆/L3 |

## 2. 项目总控回写

如存在 `项目总控.md`，只更新必要字段：

- 当前阶段。
- 当前章号/单元进度。
- 最近更新内容。
- 下一步建议。

不要写长篇审视。

## 3. 引导语

### seed 完成

> 底层故事种子已就位。下一步可以设计第一个剧情单元，让 plot 生成状态契约、运行日志和设定账本。

### plot 完成

> 剧情单元已运行完成。下一步可以从运行日志开始写第 {chapter} 章。

### create 完成

> 第 {chapter} 章已落盘。若满意，可以继续下一章；若想调整，直接说“重写/加强爽点/压缩/调文风”。

### revise 完成

> 修订已完成。若满意，可以继续；若这次修改改变了剧情事实，建议回到 plot/arc 校准设定账本。

### arc 完成

> 单元复盘已完成，真实剧情线和设定账本已更新。下一步可以设计下一个剧情单元。

## 4. 完成

等待用户下一步指令。
