## 设计原则详解

### 一、人物优先：先读懂这个人，再看数据

我们不是在做一个 YouTube 频道页面——我们在给**这个人**做个人网站。YouTube 只是我们认识她的一个窗口。

**正确的创作顺序：**

1. **人物速写** — 不看任何数据，先读她的内容。浏览视频标题、感受她的语气、理解她在做什么样的事。用一句话回答："如果向朋友介绍这个人，怎么说？"

2. **内容分析** — 从标题、描述、视频类型中提取她的核心表达：她在分享什么？她的观众是谁？她最想被记住的是什么？

3. **数据参考** — 最后再看数字。订阅数、播放量等用于验证判断，而非主导判断。

**关键判断原则：当数据信号和人物信号冲突时，优先选择符合创作者实际情况的那个。**

> 反例：Emily Chong 数据上播放量偏斜（Top2 占 62%），按数据逻辑应该做"Viral Hits 专区"。但她的内容本质是系列化旅行日记，北京视频高播放只是因为"北京"是搜索大词。按目的地组织比按播放量组织更符合她。
>
> **这个判断不是来自 data.json——它来自视频标题的语气：「超好吃的小众餐馆」「挖到宝」「CP值超高」。**

**原则：页面最终要像这个人，而不是像这个人的数据报表。**

### 二、创作者人格驱动一切

每个创作者都有独特的 personality signature。你的任务是**识别她是一个什么样的人，她想向这个世界表达什么**，然后用视觉语言放大它。

不要给创作者分类贴标签。下面的参考表仅作灵感提示，**禁止直接套用**：

| 如果你感受到的气质 | 可能的视觉方向（仅供参考，必须独立判断） |
|:-----------------|:----------------------------------------|
| 温暖分享型（像朋友在聊天） | 柔和圆角、自然纹理、暖色调、手账感 |
| 专业输出型（像老师在讲课） | 结构清晰、留白克制、网格严谨、冷静色系 |
| 感性表达型（像艺术家在创作）| 实验性排版、大胆留白、定制字体、戏剧性对比 |
| 活力躁动型（像在 party）| 高饱和、动感切角、霓虹发光、字体粗犷 |
| 沉静治愈型（像在冥想）| 深色基底、发光质感、粒子光晕、神秘氛围 |

**原则：不要给不同的创作者做出一模一样的页面。如果换了个名字和头像这个页面看起来也差不多，那设计就失败了。**

### 三、视觉表现力 = 人格翻译器

信息层级是骨架，但**视觉张力是灵魂**。一个只有骨架没有灵魂的页面，就是你说的"瑞士极简主义"——正确但无聊。

**正确的创作心态：**

1. **先感受，再规划** — 看完创作者的视频，你最直接的感受是什么？温暖？躁动？沉静？幽默？酷？记录下第一感受——这是页面的情感基调。
2. **用视觉翻译人格** — 你不是在"做页面布局"，你是在用设计语言翻译这个人的气质。她的视频是温暖的→页面用暖色调+柔和曲线。她的视频是狂野的→页面用高饱和+斜切分割。她的视频是深沉的→页面用深色基底+戏剧性光影。
3. **信息层级服务于体验** — 访客首先需要知道"这是谁/做什么/有什么作品"，这个顺序不能乱：

```
Hero（命名+头像+tagline+CTA）
   ↓
Stats（2-3 个关键数字，视觉化）
   ↓
About（频道故事 / 创作者寄语）
   ↓
Featured / Latest Videos（缩略图墙）
   ↓
Social + CTA（订阅号召、社交链接）
```

但**如何视觉化这些信息，没有标准答案**。同一个信息层级，可以有完全不同的人格化表达。

**原则：内容层级是填空题，视觉表现力才是作文题。不要做出「正确但无聊」的页面。**

### 四、色彩系统：情绪信号，不是配色公式

色彩的首要任务不是"好看"，而是**传递情绪信号**。

- **创作者的视频是什么情绪，色彩就传递什么情绪**：
  - 温暖治愈 → 米白/暖橙/大地色系，低饱和
  - 躁动活力 → 高饱和撞色、霓虹、荧光点缀
  - 沉静专业 → 深色基底 + 单一强调色
  - 浪漫梦幻 → 紫色/粉色/金色，渐变丰富
  - 幽默轻松 → 明快亮色、对比跳跃
  - 极简高级 → 黑白灰 + 少量点缀色

- **没有颜色数量限制**。2 色也可以，5 色也可以。重要的是色彩系统整体给人的感觉是否匹配创作者。
- **从频道头像/缩略图中提取主色**，但不必被它束缚。如果提取的色不符合创作者的人格，可以推翻。
- **对比度 ≥ 4.5:1**（无障碍标准，不可妥协）

**原则：色彩不传递情绪就等于失败。一个"安全配色"的页面，往往也是最无聊的页面。**

### 五、艺术多样性：没有唯一正确的视觉语言

> ⚠️ 这是最重要的原则。**如果所有页面看起来都像出自同一套模板，说明你在套用一种默认的美学，而不是在翻译不同的人格。**

核心命题：**每个创作者都值得一种独特的视觉语言。**

以下是 10 种不同的视觉表达方向 + 5 种极端风格。**每做一次页面，必须从中选一个方向（或自己定义全新的风格说明书），禁止连续两次使用同一方向。**

> 💡 每个方向都有对应的 HTML 视觉参考模板，位于 `style-refs/` 目录，双击即可在浏览器打开。创作前建议先打开对应风格的模板感受视觉语言。

具体每个方向的 CSS 规格，参考本文档后续的 [`10 个常规设计方向`](#10-个常规设计方向) 和 [`5 个极端风格`](#5-个极端风格)。

### 六、响应式即默认

所有页面必须：
- 桌面端（≥1024px）：展示完整布局，网格排列
- 平板（768-1023px）：网格减列，保持可读性
- 手机（<768px）：单列布局，touch-friendly 按钮

**原则：mobile-first 开发。先在手机上把核心信息排好，再往大屏扩展。**

### 七、技术纯洁性

页面必须是 **自包含单文件 HTML**：
- 所有 CSS 内联在 `<style>` 中
- 所有 JS 内联在 `<script>` 中
- 字体通过 Google Fonts / 系统字体加载
- 零外部 CSS/JS 库依赖
- **HTML 可见内容中禁止出现任何 Emoji Unicode 字符** —— 包括国旗、食物、动物、手势、符号、箭头等所有 Emoji 序列。替代方案见「视觉缺口处理策略」
- 禁止 `@import`
- 双击即可在浏览器打开

### 方向一：温暖手作感 · Crafted Warmth
📄 `style-refs/ghibli-journal.html`

```
题材匹配：生活分享 / 手工 / 美食 / 日常 vlog / 家居
气质关键词：亲切、真实、触感、刚出炉的面包
```

**色彩系统：**
- 底色 `#faf6f0` — 暖米白，像旧书的纸色
- 主色 `#c1694f` — 陶土红棕，有手的温度
- 辅色 `#7a9e7e` — 鼠尾草绿，自然安静
- 点缀 `#d4a373` — 蜂蜜金
- 文字 `#2d2a24` — 暖黑

**字体系统：**
- 展示字：`'Playfair Display', serif` (600-700, 标题/数字)
- 正文字：`'Inter', sans-serif` (300-400, 正文/元信息)
- 手写感：`'Caveat', cursive` (400, 引用/签名/标签) — via Google Fonts

**CSS 手法：**
- 卡片：`border-radius: 16px` · `box-shadow: 0 2px 12px rgba(45,42,36,0.06)` · `background: #ffffff`
- 纹理：`background-image: url("data:image/svg+xml,...")` — 极淡的纸纹 SVG Base64
- 按钮：圆角 50px · 背景 `#c1694f` · hover 上浮 2px
- 分割线：`1px solid rgba(193,105,79,0.12)`

**布局签名：** 非对称两栏 Hero · 图片与文字错位排列 · 卡片之间留白 > 24px · 内边距舒适 (padding > 20px)

### 方向二：暗黑质感 · Dark Atmosphere
📄 `style-refs/cyberpunk-neon.html`

```
题材匹配：音乐 / 游戏 / 电影 / 深夜内容 / 电子
气质关键词：氛围、沉浸、深邃、霓虹灯下的夜晚
```

**色彩系统：**
- 底色 `#0a0a10` — 极深炭黑，比纯黑有层次
- 卡片 `#12121c` — 深紫黑
- 主色 `#c084fc` — 薰衣草紫（音乐/浪漫）或 `#22d3ee` — 青色（科技/冷静）
- 辅色 `#a855f7` — 深紫（hover 状态）
- 点缀 `#f59e0b` — 琥珀金（播放量/徽章）
- 文字 `#f1f1f6` — 近白 · 文字辅 `#9ca3af` — 冷灰

**字体系统：**
- 展示字：`'Poppins', sans-serif` (600-800, 标题/数字)
- 正文字：`'Inter', sans-serif` (300-500, 正文/元信息)

**CSS 手法：**
- Hero 渐变：`background: linear-gradient(180deg, #0a0a10 0%, #1a0a2e 100%)`
- 卡片：`border: 1px solid #252538` · `border-radius: 14px` · hover 上浮 8px + 紫色外发光
- 粒子系统：Canvas 30-80 光点浮动 + 鼠标排斥效果
- 数字渐变：`background: linear-gradient(135deg, var(--accent), var(--gold))` + `-webkit-background-clip: text`
- 入场动画：`fadeInUp` + IntersectionObserver stagger 0.06s

**布局签名：** 深色通栏 Hero (100vh) · 数字三栏卡片(1fr 1fr 1fr) · 视频网格 3-4 列 · 圆角 12-16px · 所有区块之间有 1px 深紫灰分隔线

### 方向三：高饱和度撞色 · Pop Punch

```
题材匹配：创意 / 潮流 / 年轻化 / 穿搭 / 街头文化
气质关键词：大胆、活力、不守规矩、派对
```

**色彩系统：**
- 底色 `#ffffff` — 纯白，让撞色有视觉呼吸空间
- 主色 `#ff3366` — 霓虹粉
- 辅色 `#00ff88` — 霓虹绿
- 点缀 `#6600ff` — 深紫
- 强调 `#ffcc00` — 亮黄色
- 文字 `#111111` — 纯黑

**字体系统：**
- 展示字：`'Anton', sans-serif` (400, 标题 — 全大写粗体) 或 `'Bebas Neue', sans-serif`
- 正文字：`'Inter', sans-serif` (400-600)
- 数值：`'Space Grotesk', sans-serif` (500-700)

**CSS 手法：**
- Hero：斜切分割 `clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%)`
- 卡片：`border: 3px solid #111` · `border-radius: 4px`（小圆角/直角 = 年轻感）
- 按钮：全宽 · 粗体 · 背景填主色 · 文字白 · hover 背景切换为辅色
- 标签：纯黑色背景 + 白色文字 + 2px 边框
- 分割线：`2px dashed #111`
- 卡片旋转：某些卡片轻微 rotate(-1deg) 制造不完美感

**布局签名：** 不对称排版 · 元素重叠（文字叠图片） · 粗框围栏 · 大字号 display · 高密度信息

### 方向四：静谧留白 · Quiet Space

```
题材匹配：冥想 / ASMR / 治愈 / 极简 / 慢生活
气质关键词：呼吸感、安静、克制、禅
```

**色彩系统：**
- 底色 `#f8f8f6` — 暖白（非冷白）
- 卡片 `#ffffff`
- 主色 `#8b8b7a` — 鼠尾草灰绿
- 点缀 `#d4a373` — 陶土粉（极小面积）
- 文字 `#3a3a35` — 深灰

**字体系统：**
- 展示字：`'DM Serif Display', serif` (400, 标题)
- 正文字：`'Inter', sans-serif` (300-400, 大行距 1.8)
- 数字：`'DM Serif Display', serif` (400)

**CSS 手法：**
- 极致留白：section padding 80-120px · 卡片间 gap 40px · 行距 1.8
- Hero：文字居中 · 大量上方留白（padding-top: 15vh）· 无装饰
- 边框：`1px solid rgba(139,139,122,0.15)` — 极淡分割
- 图片：`border-radius: 24px` · 柔和阴影 `0 8px 30px rgba(0,0,0,0.04)`
- 动画：极慢 fadeIn（0.8s）· 无 bounce · 无 hover 缩放

**布局签名：** 文字居中 · 大片空白 · 极窄字体 · 单列优先 · 整个页面只有一个视觉焦点

### 方向五：图鉴式杂志 · Collector's Archive
📄 `style-refs/editorial-magazine.html`

```
题材匹配：旅行 / 探店 / 生活方式 / 收藏 / 书评
气质关键词：编辑感、策展、翻阅感、叙事
```

**色彩系统：**
- 底色 `#f5f3ef` — 米白
- 卡片 `#ffffff`
- 主色 `#3a5a40` — 深橄榄绿（自然/旅行）或 `#8b5cf6` — 紫色（创意）
- 点缀 `#c9a84c` — 金
- 文字 `#1a1a18` — 暖黑

**字体系统：**
- 展示字：`'Playfair Display', serif` (500-700, 标题)
- 正文字：`'Source Serif 4', serif` (400, 正文 — 衬线易读)
- 无衬线：`'Inter', sans-serif` (500, 标签/日期/数据)

**CSS 手法：**
- 布局：两列图文混排（图片占 60%、文字占 40% 或反过来）
- 图片：不同宽高比混合（16:9、4:3、1:1）、`object-fit: cover`
- 文字重叠：`margin-top: -40px` 让文字块轻微叠在图片上
- 标签：大写 + 间距 0.12em + `font-size: 10px`
- 引用标记：`::before { content: "—"; }` 引出引用
- 页码感：底部 `01 / 12` 式的计数

**布局签名：** 混合网格（非均匀）· 图文比例多变 · 有翻页节奏感 · 标题小于常规（杂志感来源于克制）

**字体系统：**
- 展示字：`'EB Garamond', serif` (500-700, 标题 — 经典衬线)

**布局签名：** 直白（无花哨布局）· 图片带白框像老照片 · 衬线字体为主 · 低对比 · 朴素

### 极端风格创作纪律

- 选出方向后，**必须说明为什么是这个方向而不是别的**（关联创作者的人格特征）
- 不要选择「看起来好做」的方向，选择「最像这个人」的方向
- 如果实在无法确定，问问自己：「这个创作者会说哪种视觉语言？」
- 可以从某个方向出发，但根据数据做合理调整——方向是起点不是束缚

**原则：每次创作都是一次独立的设计选择题。连续两页看起来像同一个人的，就是失败。**

## 网站设计 PRD 模板（完整版）

> Step 3 必须产出 PRD，否则不得进入 Step 4。

PRD 文件命名：`{创作者}_设计PRD.md`

### 核心原则：PRD 不是填表，是设计叙事

**禁止** 用简单表格罗列数据。每个部分必须以 **人物洞察 → 推理过程 → 设计决策** 的完整逻辑链输出。

合格标准：读完 PRD，你应该能清晰回答：
1. **这个人是谁？** 她是一个什么样的人，她想表达什么？
2. **页面给人的感觉和她本人一致吗？** 色彩/字体/布局是否在"翻译"她的人格？
3. **哪些内容信号（而非数据信号）推导出了核心设计决策？**

### 一、创作者深度理解（这是最重要的一部分）

> 这一部分的目标是：让读完的人觉得"我好像认识这个人了"。

不要只列「名称/订阅/播放量」表格。数据只是辅助，要讲的是**人**。

必须回答的问题（用 narrative 写作，禁止列表）：

- **如果用一句话向朋友介绍她，怎么说？**（来自人物速写）
- **她是一个什么样的人？** 从视频标题的语气、内容的风格、频道的氛围中感受。她兴奋吗？温暖吗？专业吗？治愈吗？像是朋友在分享还是老师在讲课？
- **她在做什么事？** 不是频道分类（"旅行"），而是她具体在做的行为（"一个新加坡女孩在分享她去过的地方、吃过的东西"）。
- **她想向这个世界表达什么？** 透过所有内容，她想被记住的核心理念是什么？（例如："最好的旅行方式是用脚丈量、用味蕾记忆"）
- **她的内容气质关键词（3-5个词）**：从标题语气、视频氛围、频道描述中提炼。

**写作示例（❌ 反对）——数据罗列：**
```
Emily Chong 是一个新加坡旅行频道，319 订阅，35 个视频。
```
这是 data.json 的内容，不是人物画像。

**写作示例（✅ 正确）——人物画像：**
```
Emily 是一个住在新加坡的女孩，她的护照比衣橱还满。她的视频语气是「你看我发现了这个！」——兴奋、分享、不怕不完美。她在做的事不是"教你怎么旅行"，而是"跟我一起感受这个地方"。关键词：真实、温暖、味蕾驱动、City Walk、发现精神。
这个判断不是来自数据——来自她的标题：「超好吃的小众餐馆」「挖到宝」「CP值超高」。
```

### 二、内容信号 → 设计策略映射

> 这是 PRD 的核心输出。你需要找出至少 **3 个关键信号**（可以是内容信号、数据信号，或两者结合），说明每个信号如何转化为具体的设计决策。

每个信号用以下格式写作：

```
### 信号 1：[信号类型：内容信号/数据信号/两者结合]

**证据：** [引用具体依据——可以是视频标题原文、标题语气、频道描述段落、或 data.json 中的具体数字]
**设计映射：** 这个信号说明了____，所以设计上应该____。
**具体措施：** 例如「按目的地组织视频而非按播放量排序，因为 Emily 的内容本质是系列化旅行日记，不是爆款驱动」
```

**寻找信号的提示（至少找到 3 个，越多越好）：**

**内容信号（优先寻找）：**
- 标题的语气语态 → 兴奋/温暖/专业/幽默？决定页面整体 tone
- 内容是否有系列/结构 → 有系列则按系列组织，不按时间
- 视频中创作者是否出镜 → 有出镜则页面更"人"化，无出镜则更"品牌化"
- 内容是否围绕一个核心理念 → 这个理念可以作为页面的设计哲学
- 标题中高频出现的词汇 → 这些是创作者自己的语言，可以融入 About 区

**数据信号（辅助参考）：**
- 播放量分布 → 哪些内容真正打动了观众？
- 频道创建时间 + 增长速度 → 新频道需要"说明这是谁"，老频道需要"展示了什么"
- 视频时长分布 → 长视频→沉浸式；短视频→快速浏览
- 语言混合 → 决定页面语言 + 字体方案
- 社交平台分布 → 观众在哪、创作者在哪

**⚠️ 当内容信号和数据信号冲突时：**
你的任务是论证为什么某一边更准确。例如：数据说播放量偏斜应做爆款区，但人物速写说这是一个系列驱动的创作者——选择更符合创作者的方案，并在 PRD 中写出你的判断理由。

### 三、视觉语言系统

这一部分输出具体的色彩/字体/质感决策，但 **每个决策必须引用第二部分的分析作为理由**。

#### 色彩方案

```
### 色彩方案

**主色推导：**
- 主色 `#xxxxxx` — 提取自 [具体来源：头像第 x 区域 / 播放量最高视频缩略图的主色调]
- 推导理由：这个颜色在频道内容中反复出现，因为它传递了____感觉，与创作者的____人格匹配
- 色彩语义：这个颜色在这个创作者语境下的含义是____

**色彩配置：**
- 主色：`#xxxxxx` — 用途：____
- 强调色：`#xxxxxx` — 用途：____
- 背景色：`#xxxxxx` — 用途：____
- 辅助点缀：`#xxxxxx` — 用途：____
- 对比度验证：____/____ = ____:1（≥ 4.5:1 ✅）
```

如果只有「主色/副色/背景色」三个，写出真实推理过程。模糊的如「推荐深色模式」直接不合格。

#### 字体系统

```
### 字体系统

| 层级 | 字体 | 字重 | 字号 | 选择理由（必须关联数据/人格） |
|:----|:-----|:-----|:-----|:---------------------------|
| 展示标题 | xxx | xxx | xxx | 这个字体传递了____感觉，因为创作者____ |
| 正文 | xxx | xxx | xxx | ____ |
| 数据/数字 | xxx | xxx | xxx | ____ |
```

#### 整体质感

描述页面的"感觉"——不是技术规格，是情绪和质感：
- **深色/浅色？** 为什么？
- **圆润/硬朗？** 为什么？
- **动态/静态？** 什么动画？为什么？
- **材质感？** 磨砂玻璃、纸纹理、霓虹光、还是纯平？

### 四、体验架构

> 描述访客自上而下浏览页面时，**情感和注意力的流动**。不是简单的 section 列表，而是一个体验序列。

用以下格式：

```
### 体验序列

如果标准五段式（Hero→Stats→About→Videos→Footer）是最优解，可以沿用。但必须说明为什么是这个顺序。

**每个 Section 的设计笔记（至少包含）：**
- 这个 section 承载什么信息？
- 用什么视觉形式呈现？（卡片/网格/全屏/列表/波形图...）

## 创作步骤详述

### 第 1 步：人物速写 + 内容分析（人物优先，双轨并行）

> 打开数据之前，先读人。

#### 1a. 人物速写（必须先做，跳过此步不得进入下一步）

在没有打开任何数据文件之前，先做这件事：

1. **浏览所有视频标题**（从 `data.json` 的 video titles 或直接从频道页面获取）。不要看播放量、不要看时长、不要看任何数字。
2. **感受语气**：这些标题的语气是什么？兴奋的？温暖的？专业的？像是在跟朋友聊天还是在讲课？
3. **一句话定义**：写一句话——"如果 Emily 是我朋友，刚旅行回来要给我看照片，她是什么语气？"
4. **写人物速写**（100-200 字），包含：
   - 这个人是做什么的（不是频道分类，而是"她在做一件什么事"）
   - 视频标题透露的语气特征
   - 如果只看标题不看数据，你感觉这个频道最有价值的是什么
   - 她想向这个世界表达什么

**人物速写是后续所有设计决策的"北极星"。如果设计决策和人物速写的直觉冲突了，你应该优先相信人物速写。**

> ✅ 正确示例（Emily Chong）：
> ```
> 人物速写：Emily 是一个住在新加坡的女孩，她的护照比衣橱还满。她的视频语气是「你看我发现了这个！」——兴奋的、分享的、不怕不完美的。她在做的事不是"教你怎么旅行"，而是"跟我一起感受这个地方"。
> 她想表达的核心：旅行最好的方式是用脚丈量、用味蕾记忆。
> ```

> ❌ 错误示例：
> ```
> Emily 是一个新加坡旅行 YouTuber，319 订阅，25K 播放。
> ```
> 这是数据罗列，不是人物速写。

#### 1b. 深度内容分析（然后做）

读取 `data.json` 和 `analysis_ready.json`。此时你已经有了人物速写的"北极星"，用数据来验证和丰富你的判断。

**必须完成的分析清单：**

| 分析维度 | 具体做法 | 为什么重要 |
|:---------|:---------|:-----------|
| **内容赛道** | 浏览所有视频标题，找共同主题/关键词/模式 | 验证人物速写是否准确 |
| **时间线索** | 频道创建日期 + 当前数据 → 增长曲线 | 新频道→需要"这是谁"；老频道→需要"积累了啥" |
| **播放量分布** | 按播放量降序排视频，算 top3 占比 | 人话就是：哪些内容真正打动了观众？ |
| **内容系列** | 标题中是否有统一后缀/前缀/命名规则 | 有系列则按系列组织，不按时间 |
| **语言判断** | 视频标题 + 频道描述的语言比例 | 决定页面语言 + 字体方案 |
| **观众情感钩子** | 从标题和描述的情绪词推断 | 这个人的什么特质让观众愿意留下来？ |
| **缩略图风格** | 粗略评估所有 maxres 缩略图的视觉一致性 | 决定卡片是否需要额外修饰 |

**产出：** 一个包含人物速写 + 交叉分析笔记的文档（为 PRD 做准备）。

⚠️ **重要提醒：如果人物速写和数据分析在某一点上冲突，你的任务不是"让数据赢"——而是判断为什么冲突，并选择更符合创作者实际情况的那个。在 PRD 中必须写出这个判断过程。**

### 第 2 步：产出网站设计 PRD

根据上面的「网站设计 PRD 模板」撰写 PRD。

**严肃警告：**
- ❌ 禁止简单罗列 data.json 中的原始字段
- ✅ 必须呈现「人物洞察/内容分析 → 推理过程 → 设计决策」的完整链条
- ❌ 禁止说「这个颜色适合这种风格」
- ✅ 必须说「从她的视频氛围中感受到温暖治愈感，所以选择暖杏色为主色」
- 至少写 3 个「信号 → 设计映射」

**产出 `{创作者}_设计PRD.md` 后，不要继续创作 HTML，将 PRD 呈现给用户过目确认。**

### 第 4 步：创作 HTML

基于已确认的 PRD 创作自包含单文件 HTML。

**⚠️ 核心要求：PRD 怎么写的，HTML 就必须怎么实现。**

**各 section 内容密度规范：**

**Hero：** 必须有层级感。至少 4-5 层视觉层次：头像/背景 → 大标题 → tagline → 描述 → CTA。

**Stats：** 数字必须加动画。使用 `requestAnimationFrame` 或者 CSS counter 实现数字从 0 滚动到终值。纯静态数字是**不合格**的。

**About：** 必须从 `channel.description` 或视频标题中提取有意义的内容，写成至少 3 段正文。About 是页面**情感连接的核心**，不是鸡肋。

**Videos：** 至少展示 6 个视频。缩略图用 `maxresdefault`。每个卡片必须包含：缩略图、标题（双行截断）、播放量、时长。hover 效果：卡片上浮 + 缩略图放大 + 阴影变化。

**Footer：** 不能只有「© 2026 xxx」。必须包含：版权 + 社交链接 + 订阅 CTA。社交链接使用内联 SVG 图标。

**质量纲领：**
1. ❌ 不允许存在的 section：Services、Testimonials、Blog、Case Studies、Team
2. ❌ 不允许占位图（SVG 生成的颜色块、unsplash 随机图）
3. ❌ 不允许中英混用（除非频道本身就是双语）
4. ❌ 不允许没有 About section
5. ✅ 优先使用 `channel.thumbnails.high` 或 `bannerUrl` 作为头像/背景
6. ✅ 所有数字直接从 `data.json` 读取，不得 hardcode 无来源数字
7. ✅ 页面必须让人感觉"花了大量时间做出来的", 不是"半小时拼凑的"

### 第 4.5 步：独立审查子 Agent

> 创作完成后，不得直接交付。**必须启动一个独立的子 Agent，对 HTML 进行冷眼审查。**

#### 审查规则

1. **上下文隔离**：审查子 Agent 仅收到 HTML 文件路径。不接收 PRD、频道数据、创作记录等任何上下文。
2. **逐条验证**：审查子 Agent 逐条对照下方「质量门禁清单」，输出审查报告。
3. **报告强制公开**：审查报告必须以明文追加到最终回复中，不得隐藏或省略。

#### 质量门禁清单

**零 Emoji 检查（优先级最高）：**
- [ ] 遍历 HTML 所有可见文本节点，确认无任何 Emoji Unicode 字符
- [ ] `style` 和 `script` 标签内的内容跳过

**页面充实度检查：**
- [ ] 至少 5 个有内容的 Section（Hero / Stats / About / Videos / Footer）
- [ ] About Section 有至少 3 段正文
- [ ] Hero 有至少 4 层视觉层次
- [ ] 去掉视频卡片后，剩余文本仍有阅读价值

**视觉方向一致性检查：**
- [ ] 页面的视觉方向与 PRD 中选型一致
- [ ] 每个视觉决策都能回溯到"这个人是谁"
- [ ] 页面看起来不像上一页的复制品

**PRD 一致性检查：**
- [ ] 主色 / 强调色 / 背景色与 PRD 一致
- [ ] 字体系统与 PRD 一致

**工匠精神检查：**
- [ ] Stats 有数字动画（requestAnimationFrame）
- [ ] 有入场动画（IntersectionObserver + stagger）
- [ ] 有交互反馈（hover 效果、按钮状态、链接动画）
- [ ] 页面整体看起来精良而非"凑合能用"

**响应式检查：**
- [ ] 桌面（≥1024px）完整布局
- [ ] 平板（768-1023px）网格减列
- [ ] 手机（<768px）单列布局

**技术检查：**
- [ ] 所有 CSS 内联
- [ ] 所有 JS 内联
- [ ] 零外部库依赖
- [ ] 视频缩略图使用 maxresdefault
- [ ] 无占位图
- [ ] 无残留注入标记

#### 审查报告模板

```
### ⚖️ 独立审查报告

**审查结论：** ✅ 通过 / ❌ 不通过

| 检查类别 | 结果 | 违规详情 |
|:---------|:-----|:---------|
| 零 Emoji | ✅/❌ | （如有违规，列出具体行号和字符内容）|
| 页面充实度 | ✅/❌ | （如不通过，指出未达标的项和当前值）|
| 视觉方向一致 | ✅/❌ | （同上）|
| PRD 一致性 | ✅/❌ | （同上）|
| 工匠精神 | ✅/❌ | （同上）|
| 响应式 | ✅/❌ | （同上）|
| 技术检查 | ✅/❌ | （同上）|

**违规明细（如有）：**
- 第 XX 行：xxx（违规类型 — 具体描述）
- ...

**审查人：**（子 Agent 标识）
```

#### 迭代规则

- 审查通过 → 进入 Step 5（可选视觉质检）
- 审查未通过 → 退回 Step 4 修改，最多 2 次重审
- 第 3 次仍未通过 → 降级处理：输出当前审查报告 + 未通过清单，交付用户人工判断

## 视觉缺口处理策略

> 当频道数据中缺少头像、banner、场景图等视觉资产时，agent 必须遵循此决策树，**不得降级使用 emoji 或占位符**。

#### 头像不可用

优先级依次为：
1. 使用 `channel.thumbnails.high` 或 `channel.thumbnails.medium`
2. 使用播放量最高视频的 `maxresdefault` 缩略图裁切为方形
3. 使用 `GenerateImage` 工具生成风格化头像
4. ❌ 禁止使用纯色渐变 + 首字母

#### Banner / 背景图不可用

1. 使用播放量最高视频的缩略图放大作为 Hero 背景
2. 使用 CSS 多层渐变 + 纹理 overlay 模拟背景深度
3. 使用 `GenerateImage` 工具生成氛围背景图
4. ❌ 禁止使用纯色背景 + emoji 替代视觉位置

#### 目的地 / 分类 / 系列视觉不可用

1. 使用该分类中播放量最高视频的缩略图
2. 使用 CSS 分层设计：渐变底色 + 纹理图案 pattern overlay + ISO 国家代码或分类名
3. 使用 `GenerateImage` 生成场景氛围图
4. ❌ 禁止使用 flag emoji 或任何其他 emoji 替代目的地视觉标识
5. ❌ 禁止使用「纯渐变色块 + emoji」的组合形式

#### 通用原则

- 视觉位置「有真实图」优于「有质感无图」优于「廉价占位」
- 如果只能使用纯 CSS 设计，确保视觉层次 ≥ 2 层
- 任何视觉位置的设计投入不应少于内容排版的时间投入

## 工匠精神

"精致"不是内容多的结果，而是**对每个视觉元素都精雕细琢**的结果。

**每个 section 的充实度参考：**

| Section | 有就行 | 更到位 |
|:--------|:-------|:-------|
| **Hero** | 头像 + 名称 + tagline + CTA | 有视觉焦点、字体渐变、入场动画、非对称布局 |
| **Stats** | 3 个数字 + 标签 | 数字渐变色、hover 反馈、计数动画 |
| **About** | 2-3 段正文 | 引用块、重点高亮、情感化叙事、签名 |
| **Videos** | 缩略图 + 标题 + 播放量 | hover 效果、入场动画、时长标签、播放量排序 |
| **Footer** | 版权 | 社交 SVG 图标、订阅 CTA |

**原则：每一个像素都可以更好。如果你觉得一个 section "差不多行了"，那它一定还能再打磨 20%。**
