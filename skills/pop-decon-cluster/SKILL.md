# pop-decon-cluster · 聚类卷幕 v1.0.0

> **定位**：拆书管线 Phase 2。从 `chapter-index.json` 的逐章标签/首句/字数中识别卷边界、幕边界、反推剧情线 Canvas、追踪契诃夫枪链。
> **方向**：逆 `plot` + `pop-writer-chapter`。只聚类原文已呈现的结构，从不「设计」剧情。
> **级别覆盖**：前N章 / 全书 — 不足100章不跑此Phase。

---

## 管线上下文

`Phase 1 事实提取 → Phase 2 聚类卷幕 ← 你在这里 → Phase 3 归纳世界观 → Phase 4 归纳故事引擎`

---

## 前置条件

- `_temp/chapter-index.json` / `baseline-data.json` / `plot-data.json` 三个 JSON 必须就位
- `写作资产/事实骨架/` + `状态/角色/` — Phase 1 产出
> 文件不全 → 退回 Phase 1。

## 步骤速查

| 步骤 | 操作 | 读什么 | 产出 | 门禁 |
|:----:|:-----|:-------|:-----|:-----|
| 1 | 识别卷边界 | chapter-index 情绪/时空跳跃 | `设计/全书架构.md` + 卷级划分 | 每个边界有地名/标签突变证据 |
| 2 | 识别幕边界 | 卷内标签密度变化 | `设计/幕/vol-01/act-*.yaml` | 幕划分有标签密度曲线支撑 |

| 4 | 契诃夫枪追踪 | 扫描设伏→回收对 | `设计/幕/vol-*/chekhov-tracker.yaml` | 每枪有 set_in/fired 章证据 |

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase 1 未完成就聚类** — `_temp/` 缺失 → 退回 Phase 1 |
| ❌2 | **卷边界无证据** — 必须有地点变更/tag密度证据 → 退回补充 |
| ❌3 | **剧情线不是设计出来的** — Canvas 行来自 events 聚类，非 agent 自拟 |
| ❌4 | **前N章只做N章** — 标注「基于前N章推断，Lv3 将修正」 |
| ❌5 | **契诃夫枪不编造** — 每枪必须有 set_in/fired 的章节证据 |

### ❌ WRONG 示例

| ❌ 错误做法 | 后果 |
|:-----------|:-----|
| 卷边界拍脑袋无引证 | 产出被退回，标注「无证据」 |
| Canvas 矩阵直接写主线而非聚类 | 剧情线不可追溯，Phase 3/4 链断裂 |
| 契诃夫枪编造设伏/回收章 | 污染追踪，整卷重跑 |

### 边界条件

| 条件 | 处理 |
|:----|:-----|
| 不足100章 | 拦截，提示"章节数不足以支撑聚类" |
| `_temp/` 缺失 | 退回 Phase 1 |
| 数据不足以判断 | 留空，标注「前 N 章数据不足以判断」 |
| 全书仅一卷 | 跳过卷边界识别，直入幕划分 |
| 契诃夫枪零匹配 | 输出空 yaml，标注"本卷未发现伏笔" |
| Lv2 后新章连载 | 增量运行，追加至现有 yaml |
