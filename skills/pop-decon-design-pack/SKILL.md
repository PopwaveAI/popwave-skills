# pop-decon-design-pack · 章节设计包 v1.1.0

> **定位：Phase 1 of deconstruction. extract.py → 按章拆分 → 5章一批 LLM 提取设计包。**
> **核心约束：ETL 硬性前置。每 5 章一次 LLM 调用，去广告+事件提取一步完成。**

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **ETL 未执行就写设计包** — extract.py 没跑，full_text.txt 不存在 → 退回 |
| ❌2 | **跳步拆分** — 全本 txt 没按章切 → 退回先拆分 |
| ❌3 | **凭空发明事件** — 事件链中出现了原文不存在的条目 |
| ❌4 | **5章一批覆盖不全** — 批内读了 5 章但只产 3-4 份设计包 → 退回 |
| ❌5 | **evidence 空** — 事件/设定/物品/人物每条必须有原文引用，null 不退库 |
| ❌6 | **设定信息蒸发** — 章内有设定/物品/重要对话释放但未提取到对应区块 |
| ❌7 | **章型误判** — 混合章型（如交锋/转折）放宽检查；单章型"战斗"中交锋占比< 60% → 退回 |
| ❌8 | **广告未去** — 设计包事件中混入非正文内容 → 退回 |
| ❌9 | **产出只留摘要** — 写入后只说「已产出 {N} 份设计包。事件{M}+设定{P}+物品{Q}项」|

| 步骤 | 操作 | 读什么 | 产出 | 门禁 |
|:-----|:-----|:-------|:-----|:-----|

## 核心流程

### Step 1: ETL + 按章拆分
详见 `steps/step-1-etl-split.md`

**读什么：** TXT/EPUB 原文文件
**做什么：** 运行 extract.py → 按章正则拆分到 `_temp/chapters/chXXX.txt`
**产出：** `_temp/metadata.json` + `_temp/full_text.txt` + `_temp/chapters/ch001.txt ~ chNNN.txt`
**❌ 门禁：** 拆分后的文件数与 metadata.json 检测到的章节数不匹配 → 退回。广告/导航/版权声明不做单独清洗（在 Step 2 LLM 中一并处理）。

### Step 2: 5章一批 LLM 提取设计包
详见 `steps/step-2-batch-process.md`

**读什么：** 按批读取 5 个章节文件（最后一批可能不足 5 章）
**做什么：** 一次 LLM 调用，完成：
1. 去广告/导航/版权声明
2. 提取每章 5-8 个原子事件（标注类型 + 参与角色 + chXX 证据）
3. 识别角色出场 + 地点 + 状态变化
4. 直接写入 5 份设计包文件

**批对齐规则：** ch1-5, ch6-10, ch11-15 … 固定窗口，不跨卷对齐。

**产出：** `写作资产/设计包/ch001-设计包.md` ～ `chNNN-设计包.md`
**❌ 门禁：** 一批内只产了 3 份设计包但读了 5 章 → 退回补齐。某章广告未去 → 退回。

### Step 3: 验证
详见 `steps/step-3-verify.md`

**做什么：** 对比 `_temp/chapters/` 和 `写作资产/设计包/` 的文件数，确保全覆盖。
**❌ 门禁：** 设计包文件数 < 章节文件数 → 退回 Step 2 补充缺失章节。

## 产出格式（完整版）

每份设计包包含以下区块，**全部从原文提取，不创造不推测**：

| 区块 | 内容 | 下游消费 |
|:-----|:-----|:---------|
| **事件链** | 3-15个原子事件（依章型），每个含类型/内容/参与角色/原文证据 | volume 剧情线聚类 |
| **设定提取** | 本章释放的世界观信息（系统规则/怪物行为/世界规律），每项带证据 | setting 世界观归纳 |
| **人物观察** | 主角性格特质+本章核心目标+关键决策，每条从行为推断但标注证据 | character 跨章分析 |
| **物品一览** | 本章出现的物品及重要性（1-3星） | setting 物品 + volume 关键道具跟踪 |
| **地点一览** | 本章场景及功能描述 | volume 卷地理 + setting 世界蓝图 |
| **重要信息/对话** | 本章的核心信息释放——台词/笔记/系统提示，标注说话者和重要性 | plot 信息节点 + setting |
| **登场人物** | 主角before/after状态+配角表（core_desire/叙事功能/与主角关系） | character + volume 角色池 |
| **情绪节拍** | 起点→推进→高潮→终点（摘自原文情绪变化） | plot 情绪弧线 + writing 参考 |
| **钩子** | 章末悬念/信息/情绪钩子（原文就有，不设计） | plot 钩子追踪 |
| **跨章弧线** | 当前章在弧线中的位置+弧内已发生事件数 | volume 幕纲划分 |
| **与上章衔接** | 上章末尾→本章起始的时间/状态衔接 | volume 情节连续性验证 |

## 边界条件

| 条件 | 处理方式 |
|:-----|:---------|
| 最后一组不足 5 章 | 按实际章数处理，不补空白章 |
| extract.py 不可用 | 终止，提示用户确认 Python 环境和依赖 |
| 章节标题无法正则匹配（如无编号） | 用「chX-首句前20字」作为文件名 |
| 某章只有寥寥几句 | 标注「本章过短」，仍然产出设计包 |
| 广告与正文难以区分 | 保留疑似行，标注「⚠️ 可能为广告」|
| 5章一批中某章事件密集 | 事件数可超 8，但每章仍保持独立事件链 |

## ❌ WRONG 示例

| 场景 | 错误做法 | 正确做法 |
|:-----|:---------|:---------|
| ETL 未运行 | 跳过 extract.py 直接写设计包 | 先执行 extract.py 得到 full_text.txt |
| 不拆分逐章 | 直接读 full_text.txt 一次 LLM 跑全书 | 按章拆分后 5 章一批处理 |
| 广告混入 | 设计包事件包含「请下载XX APP 查看更多」| LLM 调用时指示去掉非正文内容 |
| 跨批依赖 | 在批处理中引用下一批的内容 | 每批只读自己的 5 章，不跨批 |
