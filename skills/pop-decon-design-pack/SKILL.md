# pop-decon-design-pack · 章节设计包 v1.0.0

> **定位：Phase 2 of deconstruction. 从逐章 JSON 逆向产出设计包，对齐 pop-writer-chapter 的设计包格式。**
> **核心约束：仅提取原文出现的事实。不设计、不推测、不编造事件。**

## 核心流程

### Step 1: 读每章 JSON
详见 `steps/step-1-read-json.md`
- **读什么**: `_temp/chapter-data/chXXX.json`
- **做什么**: 提取 event_summaries、role_appearances、key_dialogues
- **产出**: 事件链基底（原文事件摘要列表）
- **门禁**: JSON 缺失 → 跳过该章 / 退回 Phase 1

### Step 2: 填充事件链
详见 `steps/step-2-event-chain.md`
- **读什么**: Step 1 产出的事件链基底
- **做什么**: 将原文事件摘要拆为 5-8 个原子事件，标注事件类型
- **产出**: 事件链列表（每个事件含类型标签 + 原文证据引用）
- **门禁**: 每章 <5 事件 → 退回

### Step 3: 写入设计包
详见 `steps/step-3-write-design-pack.md`
- **读什么**: Step 2 产出的事件链
- **做什么**: 按精简设计包模板（不含情绪节拍/信息释放/钩子/衔接字段）写入文件
- **产出**: `写作资产/设计包/chXXX-设计包.md`
- **门禁**: 格式不全 → 退回

---

## 产出格式（精简版）

拆书端设计包仅包含以下字段：

- **基础信息**：章号、标题、核心事件摘要
- **登场人物**：角色名 + 原文中的状态/行为（before / after 变化须有证据）
- **事件链**：5-8 个原子事件，每个含：
  - 事件内容（从原文提取，非设计）
  - 事件类型（交锋/探索/社交/成长/披露/转折/日常）
  - 参与角色
  - 原文证据引用
- **事件类型标签**：整章的事件类型分布

不产出写作端模板中的「情绪节拍/信息释放/钩子/与上章衔接」字段——这些属于写作设计范畴，拆书端不做推测。

## 落盘检查点

| 确认项 | 级别 |
|:-------|:----:|
| `_temp/chapter-data/chXXX.json` 已读取 | All |
| `写作资产/设计包/chXXX-设计包.md` 已写入 | All |
| 事件链中每事件有原文证据 | All |
| 角色 after 状态无原文证据时标注「无证据」 | All |

---

## 边界条件

| 条件 | 处理方式 |
|:-----|:---------|
| `_temp/chapter-data/` 目录不存在 | 终止，退回执行 `pop-decon-clean` Phase 1 |
| 某章 JSON 数据过少（仅 1-2 条摘要） | 标注「本章内容简略」，按实际数量产出事件链 |
| 事件类型模糊（既像探索又像社交）| 标注复合类型，如 `探索/社交` |
| 角色出场但无状态变化 | 标注「出场，无显著变化」|

## ❌ WRONG 示例

| 场景 | 错误做法 | 正确做法 |
|:-----|:---------|:---------|
| Phase 1 未完成 | 跳过 `_temp/chapter-data/` 检查直接写设计包 | 先验证 chapter-data 目录非空 |
| 编造事件 | 事件链中出现原文没有的「内心独白/闪回」| 只写原文 event_summaries 中出现的内容 |
| 无证据状态 | 角色 after = "力量提升" 但原文没写 | 标注「原文未描述本章结束状态」|
| 抄袭写作话术 | "本章情绪目标是紧张→释然" | 改为提取事实：「第3段B对A说出真相，A的反应是沉默」|
