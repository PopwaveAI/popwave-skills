# 设计包精度锚点格式 v2

> 核心发现：当设计包beat描述粒度停留在"做了什么事"级别时，prose-render 会在自身语言模型中补全"具体怎么做的"细节，产生看起来合理但不符合原文的输出。修复方式是在每个beat中加入4个精度锚点字段。

## 问题根源

### 旧版（v1）beat格式

```
| 标题 | 类型 | 内容 | 关键证据 |
```

内容示例："将150点经验分配到平民职业，升至10级满级"

渲染引擎拿到这个描述，不知道具体升级给了什么（技能点？专长？属性增益？），只能靠语言模型自行推断——于是编造了不存在的"平民大师专长"和属性增益。

### 偏差案例

| beat描述 | 渲染产出 | 原文事实 | 原因 |
|:---------|:---------|:---------|:-----|
| "分配经验升至满级" | 获得【平民大师专长】力量+1体质+1 | 只获得25技能点，无专长无属性 | 设计包未标注精确增益值 |
| "守墓老人点头同意" | 守墓老人认出索伦是盗贼 | 是冥王之女神殿的老牧师 | 设计包未标注精确身份 |
| "审视属性面板" | 完整面板重新弹出 | 心里过账，不重新弹出 | 设计包未标注"不重新弹出" |

## v2 精度锚点格式

在每个beat中增加4个字段：

### 1. scene（场景类型）

映射DNA场景卡的scene值。prose-render 据此定位DNA中的场景卡原文+观察+时间演变。

```
scene: combat_stealth
```

可取的值见 `pop-shared-dna` 的场景卡矩阵。如果beat不是纯战斗/纯对话等单一类型，取主要类型。

### 2. POV（感知锚点）

本beat从哪个角色的感知出发。叙事者锁定在该角色肩膀后侧，不进入内心。

```
POV: 索伦
```

跨角色切换的章节（如ch003墓园→酒馆双线），每beat必须显式标注POV。留空默认主角。

### 3. 关键对白/数据

必须在渲染正文中出现的原文——系统消息的精确文本、升级反馈的数值、不可替代的角色台词、面板数据的精确排版。

```
关键对白/数据: "获得25（智力值20+（智力20－10）*0.5）点技能点" | "灵魂虚弱状态解除进度45%" | 职业：5级平民/1级盗贼【一阶】
```

多条用 `|` 分隔。填空则 Render 自行补全（可能编造）。

**只填不可替换的原文**——如果一条信息在原文中是变量（如具体伤害数字12点 vs 16点），标注具体值。如果信息是模板化结构（"成功发动背刺*2，对目标造成{伤害}点伤害！"），标注模板并说明变量范围。

### 4. 感官锚点

启动本beat的感官提示，精确到具体感觉信号。

```
感官锚点: 雨声启动
感官锚点: 铁锈味启动
感官锚点: 月光下泥浆的灰色反光
```

写具体的物理感觉（"凳子腿刮地面的摩擦声""潮湿发霉的樟脑味"），不写抽象描述（"气氛压抑""环境阴沉"）。留空由 Render 从DNA感官序列规则自行发挥。

轻量写法（设计包beat表下标注）: 感官锚点(beatN): 雨声启动、铁锈味

## 完整beat示例（v2）

```
| 2 | 分配经验 | 成长 | scene: character_advancement | POV: 索伦 | 关键对白/数据: "平民职业等级提升到10级！"|"获得25点技能点。" | 感官锚点: 暖流从胸口蔓延的触感 | 内容: 索伦将150经验分配到平民职业，升至满级 | 证据: "平民职业等级提升到10级！获得25点技能点。" |
```

## 适用场景

### 拆书管线（pop-decon-design-pack）

拆书时原文已存在，精度锚点从句原文中精确提取。精度锚点 4 字段（scene/POV/关键对白/感官锚点）属于拆书管线；但完整动作序列（动作六段式 8 列表）defer 给 prose-render，设计包不再产出。

### 创作管线（pop-writer-chapter）

写新书时原文不存在，beat4字段由设计师在写设计包时同步填写。**"精确动作序列"不属于创作管线的设计包**——动作序列本身就是正文，prose-render 拿到 scene/关键对白/感官锚点后应自行生成。
