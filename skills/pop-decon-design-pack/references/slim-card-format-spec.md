# 瘦身白描卡格式规范（fast mode）

> **定位**: 单章独立 Markdown，4 段式白描卡，DS API 并发处理，全书压缩比 ~11%
> **模式**: `execution.mode = fast`
> **前身**: v4 设计包（3层+1区，~1-2K字/章）→ 瘦身白描卡（4段，~150-400字/章）
> **实测数据**: 深渊主宰第一卷187章，原文623K → 产出71K，压缩比11.3%，3分钟/10并发

---

## 1. 何时选择 fast mode

| 场景 | 推荐模式 | 理由 |
|:-----|:---------|:-----|
| 全书 300 万字+ 拆解 | fast | 10 并发 ~12 分钟完成，precision 需数小时 |
| 快速验证拆书管线可行性 | fast | 快速产出骨架，评估下游消费价值 |
| 大规模拆书 + 后续人工精修 | fast | 先跑白描卡骨架，再对关键章节精修 |
| 精品拆书 / 逐章深度分析 | precision | 需要 beat 链、感官锚点、5 表设定提取 |
| 拆书为写（正向创作参考） | precision | 需要 prose-render 可消费的精度锚点 |

**判断规则**：章数 > 100 且不需要 prose-render 直接消费 → fast mode。

---

## 2. 总体结构

每章白描卡为独立 md 文件，**4 段式结构**：

```markdown
# chXXX「标题」

POV: xxx | 章型: xxx | 原文: XXXX字

## 事件白描（3-5句，覆盖本章核心）
第一句：主角做了什么+遇到了什么。
第二句：冲突/转折是什么。
第三句：结果如何+留下了什么。
（最多5句，每句一个事件节点，不可遗漏关键转折）

## 关键数据
🔒 属性面板/关键对白原文（一行式摘要+原文定位指针）

## 爽点·钩子
爽点: 信息差/数值/打脸/情绪（无则省略整行）
钩子: 悬念内容 → 预期回收 chXXX（强度 L1-L5，无则省略整行）

## 人物关系变化（可选）
A→B: 动词（利用/背叛/信任…）（仅本章确实发生关系变化时填，无则省略整个小节）
```

## 3. 字数规则

| 指标 | 目标 | 可接受 | 上限 |
|:-----|:-----|:-------|:-----|
| 事件白描 | 150-250 字 | ≤400 字 | 500 字 |
| 关键数据(🔒) | 一行式 | ≤100 字 | 200 字（能力描述类） |
| 爽点·钩子 | 1-2 行 | ≤50 字 | — |
| 人物关系变化 | 1 行 | ≤30 字 | — |
| **单章总产出** | **150-250 字** | **≤400 字** | **500 字** |

> **迭代方案（固化）**：实测平均产出 377 字/章，略超目标但可接受。超出的主要原因是战斗/成长章的 🔒 数据块。通过以下迭代控制字数：
> 1. 🔒 数据改为「一行式摘要 + 原文定位指针」，禁止全文引用能力描述
> 2. 人物关系变化无则省略整个小节，不写"本章无"
> 3. 爽点和钩子无则省略对应行，不写"无"
> 4. 事件白描不超过 5 句，每句一个事件节点

---

## 4. 各段填写规则

### 4.1 事件白描

- 3-5 句话覆盖本章全部核心剧情转折
- 每句一个事件节点（不是一段描写）
- 不可遗漏关键转折（角色死亡、能力获得、关系变化等）
- 按原文叙事顺序排列
- 倒叙/插叙内容标注[倒叙]

**✅ 正确**：
```
索伦潜入森林发现狗头人大群。他先用淬毒弩矢射中暗红狗头人，随即近战斩杀六只。被包围后腿部受伤，使用月弧斩再杀三只。暗红狗头人释放燃烧之手，火焰吞噬索伦和周围狗头人。
```

**❌ 错误**：
```
索伦在森林里遇到了一些狗头人，打了一场很激烈的战斗。最终他凭借智慧和勇气获得了胜利，但也付出了一些代价。（太模糊，丢失全部事件节点）
```

### 4.2 关键数据（🔒）

**一行式摘要 + 原文定位指针**，禁止全文引用：

**✅ 正确**：
```
🔒 索伦属性面板：力量12/敏捷19(+1)/体质15/智力18(+1)/感知15/魅力16 | 平民5级/盗贼1级 | ch002-属性面板段
🔒 "燃烧之手！"——暗红狗头人术士释放 | ch050-战斗高潮段
```

**❌ 错误**：
```
🔒 【弱等寒冷抗性【个人专长】：在对抗寒冷的过程中，你越来越能抵抗寒冷的环境...（全文引用200+字的能力描述）
```

**🔒 标记规则**：
- 属性面板数据：摘要关键数值 + 定位指针
- 关键对白：原文引用（仅关键句，非整段）+ 定位指针
- 升级/获得能力：能力名 + 一行效果摘要 + 定位指针
- 本章无关键数据：标 `🔒 本章无`

### 4.3 爽点·钩子

**爽点**（可选，无则省略整行）：
- 类型：信息差/数值/打脸/情绪
- 一行式标注

**钩子**（可选，无则省略整行）：
- 悬念内容 → 预期回收章（强度 L1-L5）
- 钩子强度定义与 v4 相同：
  - L1: 日常悬念 | L2: 轻微悬念 | L3: 中等悬念 | L4: 强悬念 | L5: 致命悬念

### 4.4 人物关系变化（可选）

- 仅本章确实发生关系变化时填写
- 格式：`A→B: 动词（利用/背叛/信任/牺牲/敌视…）`
- 无关系变化 → **省略整个小节**，不写"本章无"或"无"

## 5. 章型判定

与 v4 共用同一套章型：

| 章型 | 定义 |
|:-----|:-----|
| 日常 | 无重大事件，角色日常活动 |
| 成长 | 主角能力/认知提升，获得新技能或属性 |
| 披露 | 重要信息揭示，世界观展开 |
| 转折 | 剧情方向改变，命运转折点 |
| 社交 | 角色间互动为主，建立/改变关系 |
| 探索 | 发现新地点/新人物/新事物 |
| 交锋 | 战斗/对抗/智斗 |

## 6. 质量卡尺（5项）

| # | 检查项 | 通过标准 | 扣分规则 |
|:-:|:-------|:---------|:---------|
| 1 | 事件白描完整 | 3-5句，覆盖全部核心转折 | 缺1转折 -1 |
| 2 | 🔒关键数据 | 有🔒标记且为一行式摘要+指针 | 缺标记 -2 |
| 3 | 章型正确 | 7型之一且与内容匹配 | 错误 -1 |
| 4 | 钩子标注 | 有钩子或合理省略 | 缺钩子且无说明 -1 |
| 5 | 字数控制 | ≤500字 | >500字 -1 |

### 评级

| 扣分 | 评级 | 判定 |
|:----:|:-----|:-----|
| 0 | 🏆 优秀 | 可直接使用 |
| 1-2 | ✅ 良好 | 可使用 |
| ≥3 | ❌ 不合格 | 退回重写 |

## 8. v4 设计包 vs 瘦身白描卡 差异对照

| 维度 | v4 设计包（precision） | 瘦身白描卡（fast） |
|:-----|:---------|:---------|
| 结构 | 3层+1区（beat链+爽点+角色+设定区） | 4段（事件白描+关键数据+爽点钩子+关系变化） |
| 单章字数 | 1-2K 字 | 150-400 字 |
| 压缩比 | 80-100% | ~11% |
| beat粒度 | 8-12个beat，表格7列 | 3-5句事件白描 |
| 设定提取 | 5表（地点/力量/角色/物品/战斗） | 融入🔒关键数据 |
| 感官锚点 | 每beat标注 | 不标注（defer给prose-render） |
| 角色对白 | 表格+语气+潜台词 | 融入🔒或事件白描 |
| 情绪弧线 | 4段（起点→推进→高潮→终点） | 融入爽点一行式 |
| 处理方式 | delegate_task 3章/批 | DS API 10并发 |
| 187章耗时 | ~35-45分钟 | ~3分钟 |
| 下游消费 | prose-render 直接消费 | volume/setting 归纳后消费 |
| 质量卡尺 | 7项 | 5项 |
| 适用场景 | 精品拆书/拆书为写 | 大规模拆书/快速验证/全书骨架 |

## 9. 下游消费说明

瘦身白描卡**不直接供 prose-render 消费**，而是供下游 Phase 2-3 归纳消费：

| 下游阶段 | 消费方式 |
|:---------|:---------|
| Phase 2（volume） | 一次性注入全卷白描卡（~70K字），AI 基于完整事件链做卷级分析 |
| Phase 3（setting） | 从🔒标记数据中归纳力量体系/物品/角色数据，无需逐章扫描 |
| Phase 4（PRD） | 全书白描卡（~256K字）可分批注入，做跨卷因果链/伏笔线分析 |
| 正向创作参考 | 作为"拆书为写"的骨架参考，比完整原文效率高 8 倍 |

> **注意**：若下游需要 prose-render 直接消费单章设计包，必须使用 precision mode（v4 设计包），不可使用 fast mode。
