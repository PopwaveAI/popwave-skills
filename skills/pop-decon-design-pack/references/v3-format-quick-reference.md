# 设计包v4 格式快照（可嵌入 delegate_task context）

> **用途**: 子agent收到此格式快照后，产出设计包必须严格匹配以下格式。
> **来源**: `references/设计包v3-格式规范.md`（完整规范，v4 内容）

---

## 文件命名

`设计包v4/ch{三位数}-设计包.md`，如 `设计包v4/ch001-设计包.md`、`设计包v4/ch037-设计包.md`

---

## 首行格式

```markdown
# 设计包 — ch001「章节标题」
```

---

## beat链格式（唯一允许的beat描述方式）

**禁止使用段落式beat描述。** beat必须放在表格中：

```markdown
| # | beat | 类型 | scene | POV | 参与角色 | 原文证据 |
|:-:|:-----|:----|:-----|:----|:---------|:---------|
| 1 | 薇薇安冒雨生火做饭 | 日常 | description_environment | 薇薇安 | 薇薇安 | "屋外传来噼里啪啦的雨声"→"点燃后轻轻地揉了揉眼睛" |
| 2 | 盘点余粮，担心断粮 | 日常 | narration_suspense | 薇薇安 | 薇薇安 | "袋子里面的粮食已经不多了" |
```

每beat完成后添加 🔒 不可替换标记 + 感官锚点：

```markdown
> 🔒 不可替换（beat2）: 关键对白（"食物已经不多了。必须要留给哥哥吃。"）
> 感官锚点(beat2): 雨声启动、柴火噼啪声
```

---

## 3层+1区最少包含的小节标题

```
## 1. beat链（beat链层）          ← 必须有表格形式的beat表
## 2. 爽点设计（爽点层）          ← 情绪弧线+钩子
## 3. 角色与人设（角色层）        ← 登场角色+关键对白
## 4. 设定/物品提取区              ← 地点/力量体系/角色怪物数据/物品资源/战斗章数据
```

---

## 产出后检查清单（子agent自检）

- [ ] 文件名格式正确（`设计包v4/chXXX-设计包.md`）
- [ ] 首行格式正确（`# 设计包 — chXXX「标题」`）
- [ ] beat链用表格，没用段落式
- [ ] 每个beat都有原文证据
- [ ] 每个beat都有 scene + POV 字段
- [ ] 每个beat都有感官锚点标注
- [ ] 4个小节标题全部存在（3层+1区）
- [ ] 设定/物品提取区存在且至少1个子区有数据
