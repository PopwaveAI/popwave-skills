# 设计包v3 格式快照（可嵌入 delegate_task context）

> **用途**: 子agent收到此格式快照后，产出设计包必须严格匹配以下格式。
> **来源**: `references/设计包v3-格式规范.md`（完整规范）

---

## 文件命名

`ch{三位数}-设计包.md`，如 `ch001-设计包.md`、`ch037-设计包.md`

---

## 首行格式

```markdown
# 设计包 — ch001「章节标题」
```

---

## 事件链格式（唯一允许的事件描述方式）

**禁止使用段落式事件描述。** 事件必须放在表格中：

```markdown
| # | 事件 | 类型 | scene | POV | 参与角色 | 字数估计 | 原文证据 |
|:-:|:-----|:----|:-----|:----|:---------|:--------|:---------|
| 1 | 薇薇安冒雨生火做饭 | 日常 | description_environment | 薇薇安 | 薇薇安 | ~250 | "屋外传来噼里啪啦的雨声"→"点燃后轻轻地揉了揉眼睛" |
| 2 | 盘点余粮，担心断粮 | 日常 | narration_suspense | 薇薇安 | 薇薇安 | ~120 | "袋子里面的粮食已经不多了" |
```

每事件完成后添加 🔒 不可替换标记：

```markdown
> 🔒 不可替换（事件2）: 关键对白（"食物已经不多了。必须要留给哥哥吃。"）
```

## 节奏分布（表格后）

```markdown
**节奏分布**:
```
事件1-3: 日常铺垫 [慢]
事件4-5: 情感深化 [慢→中]
事件6:   外部威胁插入 [快]
```
```

---

## 4层结构最少包含的小节标题

```
## 1. 事件链（骨架层）          ← 必须有表格形式的事件表
## 2. 爽点设计（爽点层）        ← 套路+情绪弧线+钩子
## 3. 角色与人设（角色层）      ← 登场角色+关键对白
## 4. 感官与画面（感官层）      ← 环境基线+DNA映射
```

---

## 产出后检查清单（子agent自检）

- [ ] 文件名格式正确（`chXXX-设计包.md`）
- [ ] 首行格式正确（`# 设计包 — chXXX「标题」`）
- [ ] 事件链用表格，没用段落式
- [ ] 每个事件都有原文证据
- [ ] 每个事件都有 scene + POV 字段
- [ ] 4个小节标题全部存在
- [ ] 本章提取了至少1个套路 → 写入了 `套路库/`
