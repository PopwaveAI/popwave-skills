# 价值点采集 → pop-trope-library 入库分流 SOP

> 用途：把设计包里采集到的优秀小说价值点，分流到 `pop-trope-library` 的五库，而不是全部塞进套路库。
> 原则：拆书产物服务写书 PRD 与后续 skill。能被写书侧复用的，才值得入库。
> v4 变更：价值点分流从逐章同步操作改为 **Step 5 批量 pass**（全部设计包完成后执行），支持跨章去重与合并。

---

## 1. 写书侧需要的价值点类型

从 `expert-writer` 写书管线倒推，library 至少要覆盖以下 10 类可复用资产：

| 写书阶段 | 需要从 library 获取什么 | 当前归属 |
|:---------|:------------------------|:---------|
| creative | 书型、读者承诺、元爽点组合、题材焊接、失败案例、外部素材如何转成剧情卡/设定卡 | `立项库/` + `设定库/` + `剧情库/` |
| world | 世界规则、力量体系、社会结构、资源经济、术语文明底色 | `设定库/` |
| character | 角色原型、身份张力、关系结构、人物弧线、说话风格 | `设定库/角色与关系/` 或 `剧情库/角色弧线/` |
| plot | 卷战略、剧情线结构、幕结构、枪链、payoff 密度 | `剧情库/结构模型/` |
| chapter | 章型、事件链、章末钩子、场景调度、信息释放 | `剧情库/章法模型/` + `套路库/` |
| prose | scene 文风、对白节奏、动作描写、感官优先级 | `文风库/` |
| qa | 使用红线、失败案例、崩坏征兆、兑现缺口 | `套路库/` + `立项库/失败案例/` |

## 2. 不要遗漏的六类“非套路”资产

### 2.1 商业定位与读者承诺

归属：`立项库/立项模式/`、`立项库/元爽点组合/`

采集标准：

1. 这本书如何在前 1-3 章告诉读者“我卖什么”。
2. 主爽点如何持续兑现，而不是只在设定里成立。
3. 哪些承诺后续会制造 plot/world 的硬约束。

### 2.2 人物与关系模式

归属：`设定库/角色与关系/` 或 `剧情库/角色弧线/`

采集标准：

1. 角色身份如何绑定世界规则。
2. 关系如何产生长期冲突和长期收益。
3. 角色说话风格是否能迁移到新书。

### 2.3 兑现节奏与 Canvas 密度

归属：`剧情库/结构模型/`

采集标准：

1. 每章小爽点 / 中爽点 / 大爽点如何排布。
2. 枪点从设伏到回收跨几章。
3. 哪些章节负责铺垫，哪些章节负责集中兑现。

### 2.4 章法与场景调度

归属：`剧情库/章法模型/`

采集标准：

1. 章首如何接上一章。
2. 事件链如何分配 scene、POV、参与角色和信息释放。
3. 章末钩子如何制造下一章行动压力。

### 2.5 信息释放与认知差

归属：`套路库/` 或 `剧情库/章法模型/`

采集标准：

1. 谁知道什么，谁不知道什么。
2. 读者何时知道，角色何时知道。
3. 信息差如何转化为情绪收益。

### 2.6 表达技术与句段手法

归属：`文风库/`

采集标准：

1. scene 对应的句式、段落呼吸、叙事距离。
2. 动作、对白、系统面板、内心独白如何嵌入。
3. 哪些原文摘录可作为文风锚点，哪些只能摘要。

## 3. 价值点分流批量 pass 产出规范

v4 将价值点分流从逐章同步操作改为 **Step 5 批量 pass**：全部设计包完成后，批量扫描全部设计包（不读原文），跨章去重与合并后产出分流清单。

### pass 输入/输出契约

| 维度 | 内容 |
|:-----|:------|
| **读什么** | `写作资产/设计包v4/ch*-设计包.md`（全文，每批 ~10 章） |
| **做什么** | 批量扫描可复用资产 → 按 SOP 分流规则分类 → 跨章去重与合并 → 产出分流清单 |
| **产出** | `写作资产/价值点分流/chXXX-价值点.md`（按批）或 `写作资产/价值点分流/汇总.md` |
| **门禁** | 每条有证据位置 / 不服务写书侧不入库 / 不新增一级库名 |

### 价值点分流清单格式

```markdown
## 价值点分流清单

| 价值点 | 类型 | 推荐入库 | 证据位置 | 复用说明 | 是否入库 |
|:-------|:-----|:---------|:---------|:---------|:---------|
| {一句话命名} | 立项/设定/角色关系/剧情结构/章法/套路/文风/QA红线 | {库路径} | chXXX-¶YY | {如何服务写书侧} | 是/否/待复盘 |
```

### 类型枚举

| 类型 | 入库位置 |
|:-----|:---------|
| 立项 | `立项库/` |
| 设定 | `设定库/{书名}/` |
| 角色关系 | `设定库/角色与关系/` 或 `剧情库/角色弧线/` |
| 剧情结构 | `剧情库/结构模型/` |
| 章法 | `剧情库/章法模型/` |
| 套路 | `原始素材/`，成熟后晋升 `套路库/` |
| 文风 | `文风库/{书名}.md` |
| QA红线 | `立项库/失败案例/` 或对应 `套路库/{模式名}.md` |

## 4. 分流判定规则

| 问题 | 是 | 否 |
|:-----|:---|:---|
| 它回答“这本书为什么成立”吗？ | 入 `立项库/` | 继续判断 |
| 它回答“这个世界怎么运转”吗？ | 入 `设定库/` | 继续判断 |
| 它回答“角色/关系如何长期产生戏”吗？ | 入 `设定库/角色与关系/` 或 `剧情库/角色弧线/` | 继续判断 |
| 它回答“剧情线/幕/章如何排布”吗？ | 入 `剧情库/结构模型/` 或 `剧情库/章法模型/` | 继续判断 |
| 它回答“具体桥段怎么爽”吗？ | 入 `原始素材/`，成熟后 `套路库/` | 继续判断 |
| 它回答“语言怎么写”吗？ | 入 `文风库/` | 不入库 |

---

## 5. 红线

> v4 变更：以下红线保留不变，执行主体从单章子 agent 改为 **Step 5 批量 pass**。

| # | 红线 |
|:-:|:-----|
| 1 | 把所有价值点都归为套路，导致人物、节奏、文风、立项经验丢失 |
| 2 | 只提取"好看的桥段"，不提取"为什么能持续兑现"的结构 |
| 3 | 角色只记名字，不提取身份张力、关系变化、说话风格 |
| 4 | 文风只记"冷峻/幽默"，不绑定 scene 和可执行句段手法 |
| 5 | 没有证据位置的价值点不得入库 |
| 6 | 不服务写书侧任何阶段的观察，不入库 |
