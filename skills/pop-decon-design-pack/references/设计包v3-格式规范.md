# 设计包v3 格式规范

> **定位**: 单章独立 Markdown，4层结构（骨架+爽点+角色+感官），适合 prose-render 直接消费
> **格式**: Markdown（非 YAML）— 段落结构天然支持自由扩展字段
> **前身**: v2（5章合一 YAML）→ v3（单章独立 MD），详见本章 v2/v3 差异表
> **示例**: `D:/workspace/深渊主宰-拆解试水/写作资产/设计包v3-demo/设计包-ch001-v3示例.md`

---

## 1. 总体结构

每章设计包为独立 md 文件，**4层结构**：

```markdown
# 设计包 — chXXX「章节标题」

> 范围: chXXX | 字数目标: XXXX | 章型: xxx | POV: xxx | 原文字数: XXXX

## 1. 事件链（骨架层）

> 做什么 —— 剧情骨架，确保不偏离原文事件

| # | 事件 | 类型 | 场景 | POV | 参与角色 | 字数估计 | 原文证据 |
|:-:|:-----|:----|:-----|:----|:---------|:--------|:---------|

事件链必须覆盖本章全部核心内容，每章 8-12 个事件。

### 事件类型

日常 / 成长 / 披露 / 转折 / 社交 / 探索 / 交锋

### 不可替换标记

事件链中每个事件下，标注"不可替换的关键对白/数据"——prose-render 必须严格遵循，不得编造替代值：
```markdown
> 🔒 不可替换: 属性面板数据（敏捷19（+1））、关键对白（"哥哥！……"）
```

## 2. 爽点设计（爽点层）

> 为什么会爽 —— 套路识别 + 情绪设计 + 钩子

### 2.1 本章套路

| 套路类型 | 结构拆解 |
|:---------|:---------|
| 打脸/反转/碾压/奇遇/危机升级/温情/信息差 | 四段式：起势→蓄势→爆发→余波 |

### 2.2 情绪弧线

`起点: xxx → 推进: xxx → 高潮: xxx → 终点: xxx`

每段情绪标注对应事件编号。

### 2.3 爽点机制

- 信息差爽点：读者知道但角色不知道
- 数值爽点：面板暴涨/技能解锁/装备升级
- 打脸爽点：弱者挑衅→反杀→数值提升→路人反应
- 情绪爽点：温馨/虐心/感动/燃

### 2.4 章末钩子

| 钩子类型 | 钩子内容 | 强度(L1-L5) | 预期回收章 |
|:---------|:---------|:-----------:|:---------|

**钩子强度定义：**
- L1: 日常悬念（明天吃什么）
- L2: 轻微悬念（陌生人的目光）
- L3: 中等悬念（发现线索但不知道意义）
- L4: 强悬念（直接威胁/重大发现）
- L5: 致命悬念（生死关头/世界观颠覆）

## 3. 角色与人设（角色层）

> 是谁在做什么、为什么这么做 —— 避免千人一面

### 3.1 登场角色 × 行为锚定

| 角色 | 本章触发的人格特质 | 内心冲突 | 行为表现 | 关系变化 |
|:-----|:------------------|:---------|:---------|:---------|

### 3.2 关键对白

> 每段关键对话标注：谁说、对谁说、语气、潜台词

- `[角色A→角色B]` 语气: xxx — "对话内容" — 潜台词: (角色真正想表达的意思)

## 4. 感官与画面（感官层）

> 怎么写 —— 让正文有画面、有温度、有张力

### 4.1 环境基线

- 时间: 夜晚/清晨/正午/黄昏
- 天气: 雨/晴/雪/阴
- 光线: 昏暗/明亮/烛光/月光
- 空间: 室内/室外/狭窄/开阔
- 气味: xxx
- 声音: xxx

### 4.2 动作场景（如有战斗）

| 阶段 | 环境 | 敌方评估 | 战术决策 | 身体反馈 | 系统反馈 | 感官锚点 | 节奏 |
|:-----|:-----|:---------|:---------|:---------|:---------|:---------|:----|
| 诱敌 | | | | | | | 紧张→激烈→松弛 |

### 4.3 氛围渲染要点

> prose-render 需要特别注意的氛围节点，标注事件号

- 事件X→X+1过渡：从X情绪到Y情绪的切换手法

### 4.4 文风DNA对接映射

| DNA维度 | 本章需求 |
|:--------|:---------|
| 感官优先顺序 | 例：听觉→嗅觉→视觉 |
| 段落呼吸 | 紧张段单句，日常段中长句 |
| 叙事距离 | 例：全程限制性POV，不进他心 |
| 对话引导 | 动作前置/裸引语，无"说" |
| 数据嵌入 | 面板/系统反馈的嵌入方式 |

## 2. v3格式质量卡尺

### 2.1 通过标准（12+2条）

| # | 检查项 | 通过标准 | 扣分规则 |
|:-:|:-------|:---------|:---------|
| 1 | 事件链完整 | 8-12事件，覆盖全部核心内容 | 缺1事件 -1 |
| 2 | 每事件有原文证据 | `chXX-XX段` 格式 | 缺证据 -1 |
| 3 | 每事件有字数预算 | `~xxx` 格式 | 缺预算 -1 |
| 4 | 不可替换标记 | 关键对白/数据标注 🔒 | 缺标记 -2 |
| 5 | 情绪弧线 | 起点→推进→高潮→终点 | 缺弧线 -1 |
| 6 | 套路识别 | 至少1个套路类型+结构拆解 | 缺套路 -2 |
| 7 | 钩子定义 | 类型+内容+强度+预期回收章 | 缺钩子 -1 |
| 8 | 角色行为锚定 | 每登场角色有特质+冲突+表现 | 缺角色 -1 |
| 9 | 关键对白标注 | 语气+潜台词 | 缺语气 -1 |
| 10 | 环境基线 | 至少4维（时间/天气/光线/空间） | 缺维度 -1 |
| 11 | 动作六段式 | 仅限战斗章，3阶段以上 | 缺段式 -2 |
| 12 | DNA对接映射 | 至少4维映射 | 缺映射 -1 |
| 13 | 套路归档长度 | 每个识别的套路都有归档段（含四段式+关键数据） | 缺一个套路 -2 |
| 14 | 套路库更新 | 归档段中的原文案例已追加到 `套路库/{套路名}.md` | 缺更新 -1 |

### 2.2 评级

| 扣分 | 评级 | 判定 |
|:----:|:-----|:-----|
| 0 | 🏆 优秀 | 可直接渲染 |
| 1-3 | ✅ 良好 | 可渲染 |
| 4-6 | ⚠️ 合格 | 需修补后渲染 |
| ≥7 | ❌ 不合格 | 退回重写 |

## 3. v2 → v3 差异对照

| 维度 | v2（现有） | v3（新规范） |
|:-----|:-----------|:-------------|
| 格式 | YAML块 | 结构化 Markdown |
| 粒度 | 5章合一 | 单章独立 |
| 每章事件数 | 3-8 | 8-12（含字数预算） |
| 动作序列 | 简述"做了什么" | 六段式（环境→评估→决策→执行→反馈→战后） |
| 对话 | 1-2句引文 | 完整序列+语气+潜台词标注 |
| 感官 | ❌ 基本缺失 | ✅ 环境基线6维+每事件关键感官锚点 |
| 爽点 | ❌ 无 | ✅ 套路类型+爽点机制+情绪弧线 |
| 角色 | ❌ 只记录出现 | ✅ 人格特质触发+内心冲突+关系变化 |
| 钩子 | 一句话摘要 | ✅ 类型+强度(L1-L5)+预期回收章 |
| POV | ❌ 无 | ✅ 每事件标注+全章POV声明 |
| 字数 | 估计值（无校准） | 目标+每事件预算+偏差跟踪 |
| 文风DNA对接 | ❌ 无 | ✅ DNA维度→本章需求映射表 |
| 不可替换标记 | ❌ 无 | ✅ 🔒标注关键对白/数据（prose不可编造） |
| **套路提取** | ❌ 无 | ✅ 识别+四段式拆解+归档 |
| **套路库** | ❌ 无 | ✅ 逐个文件聚合，覆盖度可追踪 |

## 5. v3 套路提取与归档规范

> **定位**：每个设计包产出时，同时提取本章使用的网文套路，归档到 `写作资产/套路库/`。
> **目的**：从原文长出套路库，而非从理论推导。覆盖度够了再决定是否沉淀为 skill 引用。
> **每个套路一个文件，同名套路累加原文案例。**

### 5.1 套路提取规则

每章设计包完成后，在爽点层的套路字段基础上，额外提取一节「套路归档」：

```markdown
## 5. 套路归档

> 本章适用的网文套路，每个套路转入写作资产/套路库/{套路名}.md

### 套路: 打脸装逼

原文起止章节: ch008

#### 四段式结构

| 阶段 | 原文描述 | 字数 | 感官重点 |
|:-----|:---------|:----|:---------|
| 1. 起势 | 狗头人嘲讽/挑衅/"早晚打杀了你...炖汤" | ~150 | 听觉(粗哑骂声)+光线变化(火光半明半暗) |
| 2. 蓄势 | 索伦沉默，希斯转变（老狗→凶悍） | ~100 | 身体语言（弓背/呜声）+视觉（眼睛反光） |
| 3. 爆发 | 索伦从昏迷中睁眼，薇薇安惊喜扑入 | ~200 | 慢动作特写（手指微动/眼皮微睁） |
| 4. 余波 | —（本章收于苏醒，反杀在ch008） | — | — |

#### 关键数据
- 触发条件: 反派当众威胁、主角有隐藏前提（昏迷→苏醒本身就是反转）
- 核心机制: 信息差（读者期待的反转 vs 角色不知道的隐藏前提）
- 同类案例索引: ch032(薇薇安反杀)、ch144(神子对决)
```

### 5.2 套路归档工序

每章设计包写完后，对每个识别的套路执行一次套路归档：

```
1. 检查写作资产/套路库/{套路名}.md 是否存在
2. 存在 → 追加原文案例（追加到同类案例索引）
3. 不存在 → 新建套路文件，写入四段式+关键数据+首个案例
4. 套路文件头部维护使用次数计数器
```

### 5.3 套路文件格式

```markdown
# 套路: {套路名}

> 首次收录: chXXX | 使用次数: N | 最后更新: chXXX

## 四段式模板

| 阶段 | 做什么 | 字数占比 | 感官重点 |
|:-----|:-------|:--------|:---------|
| 1. 起势 | xxx | ~15% | xxx |
| 2. 蓄势 | xxx | ~20% | xxx |
| 3. 爆发 | xxx | ~35% | xxx |
| 4. 余波 | xxx | ~30% | xxx |

## 触发条件

- 设计包事件类型需包含: {交锋/转折/披露}
- 需有: {反派/挑衅/信息差/隐藏实力/围观者任意2个}

## 核心机制

{一句话概括这个套路为什么爽}

## 原文案例

| 章节 | 书名 | 变体说明 | 四段式特点 |
|:----|:-----|:---------|:-----------|
| ch008 | 深渊主宰 | 标准版 | 起势→蓄势→爆发完整 |
| ch032 | 深渊主宰 | 蓄势偏长版 | 蓄势段多2段内心独白 |
| ch144 | 深渊主宰 | 爆发双段式 | 爆发拆为两段：先显露→再加码 |

## 注意事项

- ❌ 爆发段不要用感叹号堆砌——一个关键动作特写胜过三个形容词
- ❌ 蓄势段不要超过3段——读者耐心有限
- ✅ 余波段一定要有"立场改变"的配角反应
```

### 5.4 套路归档红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **不跳过套路提取** — 每章必须提取至少1个套路（纯过渡章可标"无"，但必须显式标注） |
| ❌2 | **不编造套路名** — 套路名必须来自已知套路列表或原文自然命名，不得自创"XX流"等外人看不懂的术语 |
| ❌3 | **四段式不省略** — 即使某阶段为空（如章节末尾没有余波），也要显式标注"本章无此阶段，关联chXXX" |
