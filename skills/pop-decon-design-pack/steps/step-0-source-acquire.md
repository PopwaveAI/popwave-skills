# Step 0: 源文件获取

> **方向**：拆书管线最前置环节。检测项目目录是否已有源 TXT；若无，委派 tool-download-webnovel 自动下载。
> **核心约束**：本步不实现下载逻辑——委派 tool-download-webnovel skill 执行。本步只做检测 + 路由 + 落位校验。

---

## I/O 注解

| 维度 | 内容 |
|:-----|:------|
| **读什么** | 项目目录（检测 .txt）+ 用户输入（书名/URL） |
| **做什么** | ① 检测源文件 ② 无源文件→委派 tool-download-webnovel ③ 有→直接进 Step 1 |
| **产出** | 项目根目录下 `{书名}.txt`，记为 `$TXT_PATH` |
| **门禁** | 产出 TXT 不存在 / 大小为 0 / 内容为 HTTP 错误页 → 退回 |

---

## 触发判断

```
项目目录是否有 .txt 文件？
  ├─ 是 → 跳过下载，记录 $TXT_PATH，进 Step 1
  └─ 否 → 进入下载流程
```

多个 .txt 时询问用户选哪个，或取最大文件。

---

## 下载流程（委派 tool-download-webnovel）

**无源文件时启用。** 加载 `tool-download-webnovel` skill，按其 SKILL.md 完整执行搜索→下载→校验三步。

### 1. 输入确认

| 输入 | 获取方式 |
|:-----|:---------|
| **书名** | ① 用户显式给出 ② 从项目目录名推断（需用户确认，目录名常为简写） |
| **URL**（可选） | 用户直链 → 直接走下载路径 |

> ⚠️ 书名歧义必须与用户确认。目录名"知否知否"≠ 全名"知否知否应是绿肥红瘦"。

### 2. 委派执行

加载 `tool-download-webnovel` skill 执行其完整流程。**委派指令必须明确：**
- 产出 TXT 落位到**当前拆书项目根目录**（非该 skill 默认的 `downloads/`）
- 文件名用 `{书名}.txt`
- 完成其 Step 3 校验（含付费墙检测）

### 3. 下载后校验

- [ ] TXT 存在于项目根目录
- [ ] 文件大小 > 50KB（过小多为错误页/付费墙截断）
- [ ] 预览前 200 字可读（非 HTML/HTTP 错误）
- [ ] 章节数 ≥ 用户指定拆章数

未通过 → 退回 tool-download-webnovel 换源重试；全部来源失败 → 终止告知用户。

---

## 交付 Step 1

```
$TXT_PATH = {项目根}/{书名}.txt
→ 进入 Step 1: ETL + 按章拆分
```

## 质量红线

| # | 红线 |
|:-:|:-----|
| ❌0a | 源文件缺失仍硬跑 Step 1 — Step 1 依赖 $TXT_PATH，未获取不得进入 |
| ❌0b | 下载失败静默继续 — 换源失败/报错 → 终止告知用户，不得用空文件进 ETL |
