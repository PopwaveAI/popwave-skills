# Step 2: 逐章v4设计包提取

> **方向**：逐章单文件提取，v4 精简模式（3层+1区）。一次 LLM 调用完成单章全量提取。
> **核心约束**：这是拆书管线唯一的**质量瓶颈**。设计包烂 = volume/setting 全烂。宁退不回。
> **Enrichment**：套路归档/价值点分流 defer 到 Step 4/5 批量 pass，单章阶段不执行。

---

## I/O 注解

| 维度 | 内容 |
|:-----|:------|
| **读什么** | 单章 `_temp/chapters/chXXX.txt`（必须已预拆分为独立文件） |
| **做什么** | 一次 LLM 调用：去广告 → 逐章结构化提取 → 写入单章 v4 设计包（3层+1区） |
| **产出** | `写作资产/设计包v4/chXXX-设计包.md`（每章 1 个独立文件） |
| **门禁** | 3层+1区结构完整、每事件有精度锚点、设定区非空 → 缺一退回 |

## 0. 前置认识：不同章型的事件粒度

同一套"5-8 个事件"不能套所有章。必须先判断章型再定粒度：

| 章型 | 特征 | 每章事件数 | 事件粒度 |
|:-----|:-----|:----------|:---------|
| **战斗** | 回合制对抗、有胜负有转折 | 8-12 | 每一轮交锋 = 一个事件 |
| **对话/信息** | 两人/多人交谈、信息交换 | 5-8 | 每轮信息释放 = 一个事件 |
| **探索/发现** | 新地点、新发现、世界观披露 | 6-10 | 每个地点/每个发现 = 一个事件 |
| **过渡/日常** | 赶路、修炼、日常 | 3-5 | 每段日常/每段修炼 = 一个事件 |
| **高潮/转折** | 重大事件、多线汇合 | 10-15 | 可拆到每人每线一个事件 |

**章型判定规则（必须明确，不留模糊空间）：**
- 章内首段提战斗/交战/对决 → 战斗章
- 章内首段提探索/密室/未知 → 探索章
- 章内 60%+ 篇幅为对话 → 对话章
- 章内包含重大死亡/背叛/揭秘 → 转折章
- 上述均不满足 → 过渡章

## 3. 事件链格式（唯一允许的事件描述方式）

**禁止使用段落式事件描述。** 事件必须放在表格中（7列，删除字数估计）：

```markdown
| # | 事件 | 类型 | scene | POV | 参与角色 | 原文证据 |
|:-:|:-----|:----|:-----|:----|:---------|:---------|
| 1 | 薇薇安冒雨生火做饭 | 日常 | description_environment | 薇薇安 | 薇薇安 | "屋外传来噼里啪啦的雨声"→"点燃后轻轻地揉了揉眼睛" |
| 2 | 盘点余粮，担心断粮 | 日常 | narration_suspense | 薇薇安 | 薇薇安 | "袋子里面的粮食已经不多了" |
```

每事件完成后标注不可替换标记 🔒 和感官锚点（与🔒并列，每事件下标注）：

```markdown
> 🔒 不可替换(事件2): 关键对白（"食物已经不多了。必须要留给哥哥吃。"）
> 🎯 感官锚点(事件2): 听觉(雨声噼啪)+触觉(手指粗糙)+视觉(火光半明半暗)
```

**「原文证据」列不是原文转录！** 只写定位信息：
- ✅ `ch003-¶12` / `地窖对峙段` / `首句·海圆历1506年`
- ❌ 摘录完整段落（如 72 行对白块）
- ❌ 文件末尾追加 `## 原文证据` 代码块

### 事件类型

| 类型 | 定义 | 示例 |
|:----|:-----|:-----|
| 交锋 | 有胜负的对抗 | 决斗、战斗、谈判对抗 |
| 探索 | 发现新事物 | 进入新地点、发现宝物、触发隐藏 |
| 社交 | 角色间互动 | 结盟、对话、交易、背叛 |
| 成长 | 能力/境界变化 | 升级、领悟新技能、突破瓶颈 |
| 披露 | 信息揭示 | 揭开真相、历史揭秘、设定说明 |
| 转折 | 剧情转向 | 死亡、重大抉择、命运翻转 |
| 日常 | 过渡内容 | 赶路、修炼、日常起居 |

### 事件质量铁律

- 每个事件是一个**独立的因果单位**，不是一段剧情。"A 拔剑攻向 B，B 闪避反击"是一个事件（交锋）。"A 和 B 打了三个回合"是三个事件（交锋×3）。
- combat 章事件粒度：每一刀/每一轮 = 一个事件。不是"整场战斗" = 一个事件。
- dialogue 章事件粒度：每一轮信息交换 = 一个事件。不是"整段对话" = 一个事件。
- evidence 必须指向可验证的原文——不说"主角很强"而是 `ch003-¶12` 或 `"一拳轰碎了城墙"`。

## 4. v4 结构（3层+1区）

每章设计包为独立 md 文件，**3层结构 + 设定/物品提取区**：

### L1 事件链层

> 剧情骨架，确保不偏离原文事件

事件链必须覆盖本章全部核心内容，每章 8-12 个事件。每事件必须表格格式（7列），含 scene/POV/原文证据/不可替换标记 🔒/感官锚点。

### L2 爽点层

> 为什么会爽 —— 套路识别 + 情绪设计 + 钩子

- **本章套路**：套路类型 + 四段式结构拆解（起势→蓄势→爆发→余波）
- **情绪弧线**：起点→推进→高潮→终点，每段标注对应事件编号
- **爽点机制**：信息差/数值/打脸/情绪
- **章末钩子**：类型+内容+强度(L1-L5)+预期回收章

### L3 角色层

> 是谁在做什么、为什么这么做 —— 避免千人一面

- **登场角色 × 行为锚定**：人格特质触发+内心冲突+行为表现+关系变化
- **关键对白**：每段标注谁说、对谁说、语气、潜台词

### S1 设定/物品提取区

> 章内设定/物品/境界/势力数据提取 —— 防止设定信息蒸发

- **设定项**：世界观/境界体系/势力/规则等章内揭示的设定
- **物品**：章内出现的物品及其新信息
- **重要信息/对话**：具有设定性质的重要信息释放

纯过渡章可标"本章无新设定"，但必须显式说明原因。

---

## 5. 裁剪优先级（上下文不足时按此顺序裁剪）

1. **L1 事件链层**（永不裁剪——这是下游 Phase 2-4 的唯一数据源）
2. **L2 爽点层**（套路识别+钩子——优先保留）
3. **L3 角色层**（关键对白+关系变化——可简化为要点）
4. **S1 设定/物品提取区**（上下文不足时可精简为一行）

> ⛔ **裁剪顺序永远是 S1→L3→L2，绝对不允许裁剪 L1 事件链层的事件链和原文证据。** 事件链缺失 = 该章对下游完全无用。

## 6. LLM 调用：完整 Prompt

```

你是一个网文拆解分析器。你的任务是从原文中提取事实，不做任何创作。

【处理原则（按优先级排序）】
P0 — 去非正文：跳过章节头部/尾部的广告、导航、作者感言、版权声明、更新通知、QQ群/公众号推广。这些内容不得出现在任何事件中。
P1 — 只提取不创作：事件内容必须来自原文。不得概括、不得推断、不得添加原文中没有的细节。
```

## 7. 输出后的质量检验（LLM 返回后必须运行）

每章 LLM 返回后，agent **必须**执行以下检查。**全部通过才能写入文件。**

### 7.1 完整性检查（3层+1区）

- [ ] 文件包含 3 个层小节标题（事件链/爽点/角色）+ 1 个设定/物品提取区
- [ ] 事件链使用表格格式（7列），不是段落式
- [ ] 事件数 ≥ [章型对应最小值]
- [ ] 设定/物品提取区非空（纯过渡章显式标"本章无新设定"并说明原因）
- [ ] 每事件有 🔒 不可替换标记
- [ ] 每事件有 🎯 感官锚点

### 7.2 证据检查

- [ ] 每事件「原文证据」非空，为定位指针格式（`chXXX-¶YY` 或关键句首词）
- [ ] 证据不是原文完整段落摘录
- [ ] 文件末尾无 `## 原文证据` 代码块
- [ ] 设定/物品每项有原文证据

### 7.3 精度锚点检查

- [ ] 每事件有 scene 字段
- [ ] 每事件有 POV 字段
- [ ] 每事件有 🔒 不可替换标记（关键对白/数据）
- [ ] 每事件有 🎯 感官锚点

### 7.4 广告检查

- [ ] 没有任何事件 content 出现广告文案
- [ ] 非正文内容（广告/导航/作者感言）未混入事件链

### 7.5 事件真实性检查

- [ ] 事件链中无原文不存在的事件
- [ ] 事件内容来自原文，不是 agent 的概括或推断

## 8. 质量卡尺（8项）

| # | 检查项 | 通过标准 | 扣分规则 |
|:-:|:-------|:---------|:---------|
| 1 | 事件链完整 | 8-12事件，覆盖全部核心内容 | 缺1事件 -1 |
| 2 | 每事件有原文证据 | `chXXX-¶YY` 或关键句首词格式 | 缺证据 -1 |
| 3 | 不可替换标记 | 关键对白/数据标注 🔒 | 缺标记 -2 |
| 4 | 情绪弧线 | 起点→推进→高潮→终点 | 缺弧线 -1 |
| 5 | 套路识别 | 至少1个套路类型+结构拆解 | 缺套路 -2 |
| 6 | 钩子定义 | 类型+内容+强度+预期回收章 | 缺钩子 -1 |
| 7 | 角色行为锚定 | 每登场角色有特质+冲突+表现 | 缺角色 -1 |
| 8 | 设定区非空 | 设定/物品提取区有内容（纯过渡章显式说明） | 缺设定 -2 |

### 评级

| 扣分 | 评级 | 判定 |
|:----:|:-----|:-----|
| 0 | 🏆 优秀 | 可直接渲染 |
| 1-3 | ✅ 良好 | 可渲染 |
| ≥4 | ❌ 不合格 | 退回重写 |

---

## 9. 质检未通过 → 退回策略

| 失败类型 | 对应检查 | 处理 |
|:---------|:---------|:-----|
| **结构缺失** | 7.1 — 3层+1区缺一 | 退回重写，补充缺失层 |
| **证据空/转录** | 7.2 — evidence 空或摘录完整段落 | 退回重写，强调"定位指针不是原文转录" |
| **精度锚点缺失** | 7.3 — 缺 scene/POV/🔒/🎯 | 退回重写，补充精度锚点 |
| **广告残留** | 7.4 — 出现广告 | 退回重写，明确指出残留的广告位置 |
| **凭空发明事件** | 7.5 — 事件不存在于原文 | 退回重写，逐事件核对原文 |
| **重跑 3 次仍不过** | 全部 | 标记为问题章节，人工介入 |

## 10. 单章写入说明

LLM 返回的 Markdown 通过质检后，按以下规则写入单章设计包文件：

**写入路径：**
```
写作资产/设计包v4/chXXX-设计包.md
```

**文件名强制规则：** `ch{三位数}-设计包.md`（如 `ch001-设计包.md`, `ch037-设计包.md`）

严禁以下变体：
- ❌ `chXXX_v4设计包.md`
- ❌ `chXXX-v4设计包.md`
- ❌ `v4_设计包_chXXX-chYYY.md`
- ❌ 合并多章到同一文件

**首行格式：**
```markdown
# 设计包 — chXXX「章节标题」
```

**标记规则：**
- 所有「原文证据」字段必须非空，为定位指针格式
- 每事件必须有 🔒 不可替换标记和 🎯 感官锚点
- 若某区块本章无数据，写`（本章无）`或显式说明原因
- **不执行套路归档和价值点分流** — 这些 defer 到 Step 4/5 批量 pass

## 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **ETL前置缺失** — 未ETL/未按章拆分/中文TXT硬跑extract.py → 退回 |
| ❌2 | **凭空发明事件** — 事件链中出现原文不存在的事件 → 退回 |
| ❌3 | **证据缺失或转录** — 每事件必须有原文证据指针(chXXX-¶YY或关键句首词)，禁止摘录完整段落，禁止末尾追加原文证据代码块 |
| ❌4 | **设定信息蒸发** — 章内设定/物品/境界/势力数据必须进入设定/物品提取区 |
| ❌5 | **精度锚点缺失** — 每事件必须有scene+POV+关键对白/数据(🔒)+感官锚点 |
| ❌6 | **结构不完整** — 必须包含3层+1区全部小节，缺一退回；事件链必须用表格格式 |
| ❌7 | **广告混入** — 设计包事件中混入非正文内容 → 退回 |
| ❌8 | **多章合并或命名违规** — 每章独立文件chXXX-设计包.md |
