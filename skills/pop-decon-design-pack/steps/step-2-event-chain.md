# Step 2: 填充事件链

> **方向**：将原文事件摘要拆解为 5-8 个原子事件，标注事件类型。
> **核心约束**：仅从原文提取。不设计、不推测、不编造。

---

## I/O 注解

| 维度 | 内容 |
|:-----|:-----|
| **读什么** | Step 1 产出的事件链基底（原始 event_summaries） |
| **做什么** | 将原文事件摘要拆为 5-8 个原子事件，标注类型标签 |
| **产出** | 事件链列表（每事件含类型标签 + 原文证据引用） |
| **门禁** | 每章 <5 事件 → 退回 |

---

## 操作步骤

### 1. 读取事件链基底

从 Step 1 拿到的每章基线数据，获取 `event_summaries` 列表。

### 2. 拆解原子事件

对于每条原文事件摘要，按以下原则拆解：

**原子事件定义**：
- 一个动作/一个观察/一个感觉变化 = 一个事件
- 战斗章：一刀一事件；对话章：一轮信息交换一事件
- 探索章：一个发现一事件；过渡章：一个移动/到达一事件

**拆解示例**：

```
原文摘要: "主角在野外遭遇妖兽袭击，被迫使用隐藏能力，被神秘人看到"
↓ 拆解
事件1: 主角在野外赶路，感知到妖兽气息
事件2: 妖兽从树丛中突袭主角
事件3: 主角闪避妖兽的首次攻击
事件4: 主角被迫使用隐藏能力反击
事件5: 隐藏能力的光芒引起神秘人注意，神秘人暗中观察
```

### 3. 标注事件类型

为每个原子事件分配类型标签：

| 类型 | 说明 | 检测关键词 |
|:-----|:-----|:-----------|
| 交锋 | 战斗/对抗/冲突 | 攻击、防御、对战、冲突、对抗 |
| 探索 | 发现/调查/解密 | 发现、探索、调查、寻找、查看 |
| 社交 | 交流/谈判/互动 | 对话、交谈、询问、讨论、协商 |
| 成长 | 修炼/突破/提升 | 修炼、突破、升级、领悟、提升 |
| 披露 | 信息揭露/转折揭示 | 揭示、坦白、暴露、知道真相 |
| 转折 | 意外/反转/关键变化 | 突然、没想到、反转、意外 |
| 日常 | 过渡/后勤/移动 | 前往、返回、休息、准备、等待 |

### 4. 标注原文证据引用

每个原子事件后标注原文证据：

```
事件: 主角感知到妖兽气息
证据: 原文第15段「空气中传来腥臭味，林越握紧了手中的剑。」
```

### 5. 门禁检查（通过/退回）

```
□ 每章事件链 ≥ 5 个原子事件
□ 每事件有类型标签
□ 每事件有原文证据引用（行号或段落简写）
□ 事件总数不超过原文 event_summaries 的合理范围
  - 3 条摘要 → 合理拆出 5-8 个原子事件
  - 5 条以上摘要 → 可拆出 8-12 个原子事件
→ 全部通过 → 进入 Step 3（写入设计包）
→ 未通过 → 退回重新拆解
```

---

## 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **凭空发明事件** — 原子事件在原文 event_summaries 中无对应 |
| ❌2 | **事件类型错误** — 原文是「探索发现」标注为「交锋」|
| ❌3 | **无证据引用** — 原子事件未标注原文出处 |
