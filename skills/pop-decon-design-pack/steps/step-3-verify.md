# Step 3: 验证

> **方向**：确认设计包v4全覆盖，无遗漏章节；检查3层+1区结构完整性。

---

## I/O 注解

| 维度 | 内容 |
|:-----|:------|
| **读什么** | `_temp/chapters/` vs `写作资产/设计包v4/` |
| **做什么** | 对比已拆分章数与设计包覆盖章数，确认无遗漏；检查3层+1区完整性 |
| **产出** | 验证报告 |
| **门禁** | 设计包覆盖总章数 < 已拆分章数 → 退回 Step 2 |

## 操作步骤

### 1. 文件数对比

对比 `_temp/chapters/` 与 `写作资产/设计包v4/` 的文件数：

```bash
# 章节文件数
ls _temp/chapters/ch*.txt | wc -l

# 设计包文件数
ls 写作资产/设计包v4/ch*-设计包.md | wc -l
```

- 设计包文件数 = 章节文件数 → 通过
- 设计包文件数 < 章节文件数 → 列出缺失章节号，退回 Step 2

### 2. 完整性检查（3层+1区）

对每份设计包文件执行以下检查：

#### 2.1 小节标题存在

每份设计包必须包含以下 4 个小节标题（3层+1区）：

| # | 小节标题 | 层/区 |
|:-:|:---------|:------|
| 1 | `## 1. beat链` 或含「beat链」 | L1 beat链层 |
| 2 | `## 2. 爽点` 或含「爽点」 | L2 爽点层 |
| 3 | `## 3. 角色` 或含「角色」 | L3 角色层 |
| 4 | `## 4. 设定` 或含「设定/物品」 | S1 设定/物品提取区 |

- 4 个小节标题全部存在 → 通过
- 缺任一小节 → 标记为缺层文件，退回 Step 2 重写

#### 2.2 beat表7列检查

抽检后10%（至少3章），确认beat链使用表格格式且为7列：

```markdown
| # | beat | 类型 | scene | POV | 参与角色 | 原文证据 |
```

- 7列完整 → 通过
- 使用段落式beat描述（非表格） → 退回重写
- 列数不匹配（如仍有「字数估计」列） → 退回重写

#### 2.3 每beat精度锚点检查

抽检后10%（至少3章），每beat必须包含以下4个精度锚点：

| # | 锚点 | 检查方式 |
|:-:|:-----|:---------|
| 1 | **scene** | beat表中有 scene 列值 |
| 2 | **POV** | beat表中有 POV 列值 |
| 3 | **原文证据** | beat表中有原文证据（`chXXX-¶YY` 或关键句首词），非空、非完整段落摘录 |
| 4 | **感官锚点** | 每beat下有 `🎯 感官锚点` 标注 |

- 4 个锚点全部存在 → 通过
- 缺任一锚点 → 退回重写

同时检查每beat下是否有 `🔒 不可替换` 标记（关键对白/数据）。

#### 2.4 设定区非空检查

每份设计包的设定/物品提取区（S1）必须非空：

- 有设定项/物品/重要信息 → 通过
- 纯过渡章标注"本章无新设定"并说明原因 → 通过
- 设定区为空且无说明 → 退回重写

### 3. 命名与首行格式检查

#### 3.1 命名一致性

全部文件名必须为 `chXXX-设计包.md`（三位数补零）。

- 全部匹配 → 通过
- 有偏差 → 统一重命名

#### 3.2 首行格式

全部文件首行必须匹配：
```markdown
# 设计包 — chXXX「章节标题」
```

- 全部匹配 → 通过
- 有偏差 → 标记差异文件，执行 post-hoc 归一化脚本修复（`references/post-hoc-format-normalization.md`）

> **⚠️ 首行格式漂移是持续顽疾：** 实测多批次并行提取后首行格式偏离率高。首行格式检查必须是全量扫描（不能仅抽检），修复策略必须是 post-hoc 脚本化归一化。**高频漂移场景改用 `scripts/normalize-headlines-from-source.py`（从源文件提取标题，更可靠）**。

### 4. beat数下限检查

| 章节位置 | beat数下限 |
|:---------|:----------|
| 前70%章节 | ≥8 |
| 后30%章节 | ≥5 |

- 达标 → 通过
- 未达标 → 标注低密度警告（不退回，但标记供下游参考）

### 5. 验证结果

全部通过 → 通知 orchestrator（pop-decon）Phase 1 设计包提取完成，可进入 Phase 2。

未通过 → 列出缺失/不合规文件清单，退回 Step 2 补充或重写。
