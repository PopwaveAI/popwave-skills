# Step 3: 验证（双模式）

> **方向**：确认设计包/白描卡全覆盖，无遗漏章节；检查结构完整性。

---

## I/O 注解

| 维度 | precision mode | fast mode |
|:-----|:---------------|:----------|
| **读什么** | `_temp/chapters/` vs `写作资产/设计包v4/` | `_temp/chapters/` vs `写作资产/白描卡/` |
| **做什么** | 对比文件数，检查3层+1区完整性 | 对比文件数，检查4段结构完整性 |
| **产出** | 验证报告 | 验证报告 |
| **门禁** | 设计包文件数 < 章节文件数 → 退回 | 白描卡文件数 < 章节文件数 → 退回 |

---

## 操作步骤

### 1. 文件数对比

**precision mode**：对比 `_temp/chapters/` 与 `写作资产/设计包v4/`
**fast mode**：对比 `_temp/chapters/` 与 `写作资产/白描卡/`

```bash
# precision: 设计包文件数
ls 写作资产/设计包v4/ch*-设计包.md | wc -l

# fast: 白描卡文件数
ls 写作资产/白描卡/ch*.md | wc -l

# 章节文件数
ls _temp/chapters/ch*.txt | wc -l
```

- 产出文件数 = 章节文件数 → 通过
- 产出文件数 < 章节文件数 → 列出缺失章节号，退回 Step 2

### 2. 完整性检查

#### 2A. precision mode（3层+1区）

每份设计包必须包含以下 4 个小节标题：

| # | 小节标题 | 层/区 |
|:-:|:---------|:------|
| 1 | `## 1. beat链` 或含「beat链」 | L1 beat链层 |
| 2 | `## 2. 爽点` 或含「爽点」 | L2 爽点层 |
| 3 | `## 3. 角色` 或含「角色」 | L3 角色层 |
| 4 | `## 4. 设定` 或含「设定/物品」 | S1 设定/物品提取区 |

同时检查：beat表7列、每beat精度锚点（scene+POV+原文证据+感官锚点）、🔒不可替换标记、设定区非空。

#### 2B. fast mode（4段式）

每份白描卡必须包含以下 3 个必需小节（第4段可选）：

| # | 小节标题 | 必需性 |
|:-:|:---------|:------|
| 1 | `## 事件白描` | 必需 |
| 2 | `## 关键数据` | 必需 |
| 3 | `## 爽点` 或含「爽点·钩子」 | 必需 |
| 4 | `## 人物关系变化` | 可选（省略=无变化） |

同时检查：
- 事件白描 3-5 句
- 🔒 数据为一行式摘要+指针格式（非全文引用）
- 章型标注正确（7型之一）
- 单章总字数 ≤500 字
- 无数据的段省略而非写"无"

### 3. 命名与首行格式

**precision mode**：
- 文件名：`chXXX-设计包.md`（三位数补零）
- 首行：`# 设计包 — chXXX「章节标题」`

**fast mode**：
- 文件名：`chXXX.md`（三位数补零）
- 首行：`# chXXX「章节标题」`

### 4. 抽检质量

#### precision mode

抽检后10%（至少3章），检查 beat 表7列、精度锚点、🔒标记。

#### fast mode

抽检后10%（至少3章），检查：
- 事件白描覆盖全部核心转折
- 🔒 数据格式正确（一行式+指针）
- 无"本章无"（应省略而非标注）

### 5. 验证结果

全部通过 → 通知 orchestrator（pop-decon）Phase 1 完成，可进入 Phase 2。

未通过 → 列出缺失/不合规文件清单，退回 Step 2。

## 验证清单汇总

### precision mode（9项）

| # | 检查项 | 通过标准 | 失败处理 |
|:-:|:-------|:---------|:---------|
| 1 | **文件数对比** | 设计包文件数 = 章节文件数 | 缺文件退回 Step 2 |
| 2 | **3层+1区小节标题** | 4 个小节标题全部存在 | 缺层退回重写 |
| 3 | **beat表7列** | 表格格式，7列 | 退回重写 |
| 4 | **每beat精度锚点** | scene + POV + 原文证据 + 感官锚点 | 缺锚点退回重写 |
| 5 | **🔒不可替换标记** | 每beat有 🔒 标记 | 缺标记 → 警告 |
| 6 | **设定区非空** | 有内容或显式说明 | 空且无说明 → 退回 |
| 7 | **命名一致性** | 全部 `chXXX-设计包.md` | 统一重命名 |
| 8 | **首行格式** | `# 设计包 — chXXX「标题」` | post-hoc 脚本修复 |
| 9 | **beat数下限** | 前70%≥8，后30%≥5 | 标注低密度警告 |

### fast mode（6项）

| # | 检查项 | 通过标准 | 失败处理 |
|:-:|:-------|:---------|:---------|
| 1 | **文件数对比** | 白描卡文件数 = 章节文件数 | 缺文件重跑 |
| 2 | **3段必需小节** | 事件白描+关键数据+爽点钩子 | 缺段退回 |
| 3 | **🔒数据格式** | 一行式摘要+指针，非全文引用 | 退回重写 |
| 4 | **命名一致性** | 全部 `chXXX.md` | 统一重命名 |
| 5 | **首行格式** | `# chXXX「标题」` | 标记差异文件 |
| 6 | **字数控制** | ≤500字/章 | 超标标记警告 |
