# Step 4: 套路归档批量 pass

> **定位**: 全部设计包完成后执行，取代原单章同步归档。
> v4 将套路归档从 Step 2 的每章同步操作 defer 到此处批量执行，消除并行写冲突，提升归档质量。

---

## I/O 契约

| 维度 | 内容 |
|:-----|:------|
| **读什么** | `写作资产/设计包v4/ch*-设计包.md`（仅读 L2 爽点层的套路类型 + 事件链） |
| **做什么** | 扫描全部设计包的套路类型 → 按套路名分组 → 跨章聚合四段式 + 关键数据 → 写入套路库 |
| **产出** | `写作资产/套路库/{套路名}.md`（每套路一个独立文件） + `写作资产/套路库/目录.md` |
| **门禁** | 每套路 ≥ 1 章证据 / 不编造套路名 / 不产出合并总表 |

---

## 流程

### 1. 扫描阶段

遍历全部设计包文件，提取 `(章节, 套路类型)` 二元组。

```
输入: 写作资产/设计包v4/ch001-设计包.md ~ chNNN-设计包.md
操作: 逐文件读取 L2 爽点层，提取「套路识别」字段中的套路名
输出: [(ch001, "扮猪吃虎"), (ch001, "打脸"), (ch002, "扮猪吃虎"), ...]
```

- 纯过渡章标注「无套路适用(纯过渡章)」的，跳过不提取
- 套路名以设计包中实际写明的为准，本阶段不做归一化

### 2. 分组阶段

按套路名分组，统计每套路涉及的章节列表。

```python
from collections import defaultdict

trope_chapters = defaultdict(list)
for ch, trope_name in scanned_pairs:
    trope_chapters[trope_name].append(ch)

# 输出示例:
# "扮猪吃虎" → [ch001, ch002, ch015, ch047, ...]
# "打脸"     → [ch001, ch003, ch022, ...]
```

- 同一章节出现多个套路是正常的，分别归入各自分组
- 分组完成后输出一份套路清单（套路名 + 涉及章节数），供归档阶段使用

### 3. 归档阶段

每个套路一个子 agent，可并行（各套路文件互不冲突，无并发写问题）。

每个子 agent 接收：
1. 套路名
2. 涉及章节列表
3. 对应设计包文件路径列表

子 agent 读取涉及章节的设计包片段（仅 L2 爽点层 + 事件链相关部分），产出该套路的四段式 + 关键数据，写入 `写作资产/套路库/{套路名}.md`。

```
子 agent 任务:
- 读取: 写作资产/设计包v4/ch001-设计包.md (仅L2爽点层+事件链)
        写作资产/设计包v4/ch002-设计包.md (仅L2爽点层+事件链)
        ...
- 产出: 写作资产/套路库/扮猪吃虎.md
- 格式: 见下方「套路文件格式」
```

**并行安全**：每个套路写入独立文件，不存在共享文件并发写问题（v3 的套路库并发写冲突在此架构下自然消除）。

### 4. 索引阶段

全部套路文件归档完成后，生成 `写作资产/套路库/目录.md`。

```markdown
# 套路库目录

> 自动生成 | 共 {N} 个套路 | 来源: {书名}

| 套路名 | 涉及章节数 | 涉及章节 | 文件 |
|:-------|:----------|:---------|:-----|
| 扮猪吃虎 | 5 | ch001, ch002, ch015, ch047, ch089 | [查看](扮猪吃虎.md) |
| 打脸 | 3 | ch001, ch003, ch022 | [查看](打脸.md) |
| ... | ... | ... | ... |
```

## 套路文件格式

每个 `写作资产/套路库/{套路名}.md` 包含以下区块：

### 四段式模板表

| 阶段 | 做什么 | 字数占比 | 感官重点 |
|:-----|:-------|:---------|:---------|
| 铺垫 | {该套路如何引入，前期如何埋线} | ~{X}% | {读者此时应感受到什么} |
| 升级 | {矛盾如何激化，压力如何递增} | ~{X}% | {感官锚点} |
| 爆发 | {爽点释放的核心时刻} | ~{X}% | {高潮感官} |
| 余韵 | {爆发后的收束与后续钩子} | ~{X}% | {情绪落点} |

> 四段式不省略空阶段——如果某阶段在该套路变体中不存在，显式标注「本变体无此阶段」并说明原因。

### 触发条件

- {该套路在什么前提下才会被激活}
- {角色/设定/剧情需要满足什么条件}

### 核心机制

- {套路运转的内在逻辑}
- {为什么读者会觉得爽——情绪收益来源}

### 原文案例表

| 章节 | 书名 | 变体说明 | 四段式特点 |
|:-----|:-----|:---------|:----------|
| ch001 | {书名} | {该案例与标准四段式的差异} | {铺垫→升级→爆发→余韵 各阶段简述} |
| ch015 | {书名} | {变体说明} | {四段式特点} |
| ... | ... | ... | ... |

### 注意事项

- {使用该套路时常见的失败模式}
- {与哪些套路搭配效果好}
- {哪些书型不适合此套路}

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | 不编造套路名——套路名必须来自设计包 L2 爽点层实际标注 |
| 2 | 四段式不省略空阶段——空阶段显式标注「本变体无此阶段」，不得直接跳过 |
| 3 | 不得产出套路归档总表——每套路一个独立文件，禁止合并为单一总表文件 |

---

## 与 Phase 2 衔接

- **必须在 Phase 2 (pop-decon-volume) 前完成**：Phase 2 的卷级分析依赖套路库的跨章聚合数据
- `pop-decon-volume` 已支持套路库为空时跳过——如果本 Step 被跳过（如纯过渡书无套路），Phase 2 不会报错，但卷级套路分析将为空
- 本 Step 产出的 `目录.md` 是 Phase 2 快速定位套路的索引
