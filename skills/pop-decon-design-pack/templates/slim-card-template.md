# chXXX 瘦身白描卡模板（fast mode）

> 产出路径: `写作资产/白描卡/chXXX.md`
> 生产方式: pop-decon-design-pack v6.0.0 (fast mode)
> 格式权威来源: `references/slim-card-format-spec.md`
> 下游消费者: pop-decon-volume → pop-decon-setting（归纳消费，不直接供 prose-render）

---

## 模板说明

瘦身白描卡 = 从原文提取的 4 段式压缩卡，DS API 并发处理，全书压缩比 ~11%。

**核心原则**：
- 事件白描必须覆盖全部核心转折（不可遗漏）
- 🔒 数据必须是一行式摘要+原文定位指针（禁止全文引用）
- 爽点/钩子/关系变化无则省略，不写"无"
- 单章总产出 150-400 字（目标 150-250，上限 500）

---

## 模板正文

```markdown
# chXXX「章节标题」

POV: xxx | 章型: xxx | 原文: XXXX字

## 事件白描（3-5句，覆盖本章核心）
第一句：主角做了什么+遇到了什么。
第二句：冲突/转折是什么。
第三句：结果如何+留下了什么。
（最多5句，每句一个事件节点）

## 关键数据
🔒 [属性面板摘要/关键对白原文/升级数据，一行式+定位指针。无则标"本章无"]

## 爽点·钩子
爽点: [信息差/数值/打脸/情绪，无则省略整行]
钩子: [悬念内容] → 预期回收 chXXX（强度 L1-L5，无则省略整行）

## 人物关系变化（可选）
[A→B: 动词（利用/背叛/信任…），仅本章确实发生关系变化时填。无则省略整个小节]
```

## 填写自检

### 结构完整性
- [ ] 事件白描存在且 3-5 句
- [ ] 关键数据(🔒)存在
- [ ] 爽点·钩子存在（可为空行省略）
- [ ] 章型标注正确（7型之一）

### 数据质量
- [ ] 事件白描覆盖全部核心转折
- [ ] 🔒 数据为摘要+指针格式，非全文引用
- [ ] 无数据的段省略而非写"无"
- [ ] 单章总字数 ≤500 字

### 章型对照
- [ ] 日常/成长/披露/转折/社交/探索/交锋 — 7选1
