# pop-decon-extract · 事实提取 v1.0.0

> **定位：ETL + 角色卡 + 事实骨架。Phase 1 of the deconstruction pipeline.**
> **核心约束：ETL 硬性前置。不归纳、不推测、不写故事引擎。每个事实必须有 chXX 证据。**

## ❌ 质量红线
| # | 红线 |
|:-:|:-----|
| ❌1 | ETL 未执行就写产出 — extract.py 还没跑，_temp/*.json 不存在 → 退回跑脚本 |
| ❌2 | JSON 无数据仍写入 — _temp/*.json 中无条目，产出中出现且未标注「推断」→ 退回 |
| ❌3 | 年龄/等级凭空书写 — 角色卡所有属性必须有 JSON 来源，找不到标注「原文未显式说明」|
| ❌4 | Lv2 角色卡混入 Lv1 内容 — Lv1 只产角色卡+龙套池，不产事实骨架 |
| ❌5 | 产出只留摘要 — 文件中写完后对话只说「已写入 {路径}。摘要：{核心}。」|

## 速查表
| 步骤 | 操作 | 读什么 | 产出 | 门禁 |
|:-----|:-----|:-------|:-----|:-----|
| Lv1 | 角色卡提取 + 龙套池 | `_temp/baseline-data.json` | 角色卡(Lv4+Lv3) + 龙套池 | ETL 未完成 → 退回 |
| Lv2 | 事实骨架 + 摘要 | `_temp/chapter-index.json` + Lv1 产出 | Lv1 + 事实骨架(ch001-050) + 摘要 | Lv1 未完成 → 退回 |
| Lv3 | 全书事实骨架 | 全部 `_temp/*.json` + Lv2 产出 | Lv2 + 完整事实骨架 | Lv2 未完成 → 退回 |

## 核心流程
### Step 1: ETL 数据提取
详见 `steps/step-1-etl.md`
- **读什么**: `{txt_path}`
- **做什么**: Run `extract.py all {txt_path} {output_dir}`
- **产出**: `_temp/baseline-data.json` + `chapter-index.json` + `world-data.json`
- **❌门禁**: 任意 JSON 缺失 → 退回

### Step 2: 角色卡提取
详见 `steps/step-2-characters.md`
- **读什么**: `_temp/baseline-data.json`
- **做什么**: 从 JSON 提取 → 主角卡(Lv4) + 配角卡(Lv3) + 龙套池
- **产出**: `状态/角色/` 下角色卡 + 龙套池
- **❌门禁**: 角色属性无 JSON 来源 → 退回

### Step 3: 事实骨架 (Lv2/Lv3)
详见 `steps/step-3-skeleton.md`
- **读什么**: `_temp/chapter-index.json` + Lv1 角色卡
- **做什么**: 逐章提取核心事件 → 写作资产/事实骨架/
- **产出**: `写作资产/事实骨架/` 下事实骨架文件
- **❌门禁**: Lv1 未完成 → 退回

### Step 4: 摘要
详见 `steps/step-4-summary.md`
- **读什么**: 全部产出文件
- **做什么**: 汇总 Phase1 产出 → 结构化摘要
- **产出**: `_参考书/{书名}/Phase1-事实提取摘要.md`
- **❌门禁**: 产出文件缺失 → 退回

## ❌ WRONG 示例
| 场景 | 错误做法 | 正确做法 |
|:-----|:---------|:---------|
| ETL 未运行 | 跳过 extract.py 直接写角色卡 | 必须先执行 ETL 得到 JSON |
| JSON 无数据 | 仍写入角色卡且不标注「推断」| 标注「原文未显式说明」|
| Lv2 混入引擎 | 在事实骨架中加入故事引擎推测 | 只提取原文显式描述的事件 |

## 边界条件
| 条件 | 处理方式 |
|:-----|:---------|
| 空文件 / 0 章 TXT | 终止，报「输入为空」|
| 仅 1 章 | 按 Lv1 执行，不升 Lv2+ |
| 角色同名不同人 | 加角色编号(r1/r2)，交叉引用段落 |
| 叙事顺序乱跳 | 按书中出场顺序建索引，覆盖 ch 标注 |
| 对话体 / 第一人称 | 从对话中提取行为，标注「推测自对话」|
| 文件中途断章 | 只提取已存在章节，不补不续 |
