# Step 2: 角色卡提取

> **方向**：从 JSON 结构数据 + 原文精读 → 标准角色卡。
> **核心约束**：每个硬数据（年龄/等级/职业/属性）必须来自 `_temp/baseline-data.json`。所有推断必须标注「推断」。
> **覆盖级别**：Lv1(ch1-20) / 前N章 / 全书

---

## I/O 注解

| 维度 | 内容 |
|:-----|:-----|
| **读什么** | `_temp/baseline-data.json`（characters[] / ages[] / levels[]）+ 逐章原文精读 |
| **做什么** | 使用角色卡模板提取主角/配角/龙套 |
| **产出** | 主角卡(Lv4) + 配角卡(Lv3) + 龙套池 |
| **门禁** | Step 1 已完成（JSON 存在）；角色卡中每个硬数据必须有 JSON 来源 |

---

## 操作步骤

### 1. 前置确认

```
□ _temp/baseline-data.json 存在
□ _temp/world-data.json 存在
→ 全部通过 → 继续
→ 未通过 → 退回 Step 1
```

### 2. 提取主角卡 (Lv4)

- 从 `baseline-data.json#characters[]` 确认主角名
- 从 `baseline-data.json#ages[]` 确认年龄
- 从 `world-data.json#class.entries[]` 确认等级/职业路径
- 从 `baseline-data.json#events[]` 提取关键事件
- 使用模板 `templates/character-lv4.tpl.md` 填写
- 产出路径：`状态/角色/{主角名}-角色卡-Lv4.md`

**填写规范：**
- 每个字段必须标注 `@evidence: chXX`（来源章节）
- JSON 中找不到的数据 → 留空标注「原文未显式说明」
- 间接推断的数据 → 标注「推断」

### 3. 提取配角卡 (Lv3)

- 从 `baseline-data.json#characters[]` 筛选主要配角（出场频次高/剧情权重大的角色）
- 使用模板 `templates/character-lv3.tpl.md` 填写
- 产出路径：`状态/角色/{配角名}-角色卡-Lv3.md`
- 每个配角独立文件

**数量规范：**
- Lv1 (ch1-20)：1-3 个配角
- Lv2 (ch1-100)：3-8 个配角
- Lv3 (全书)：所有主要配角

### 4. 提取龙套池

- 从 `baseline-data.json#characters[]` 收集所有命名但非主角/配角的角色
- 产出路径：`状态/角色/龙套池.md`

**龙套池格式：**

| 角色名 | 出场章节 | 身份/描述 | @证据 |
|:-------|:---------|:----------|:------|

### 5. 门禁检查（通过/退回）

```
□ 主角卡存在且所有字段来自 JSON
□ 每个配角卡独立文件
□ 龙套池包含所有命名角色
□ 所有硬数据标注 @chXX
□ 找不到的数据标注「原文未显式说明」
□ 推断数据标注「推断」
→ 全部通过 → Lv1 可进入 Step 4（摘要）；前N章/全书 进入 Step 3
→ 未通过 → 退回修正
```

---

## 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **凭空书写** — 年龄/等级/职业等属性必须有 JSON 来源 |
| ❌2 | **chXX 标注缺失** — 角色卡中每个事实字段必须标注原文章节 |
| ❌3 | **推断未标注** — 任何非显式声明的数据必须标注「推断」 |
| ❌4 | **Lv2 混入 Lv1 内容** — Lv1 只产角色卡+龙套池，不产事实骨架 |

└── 龙套池.md                  ← 所有命名角色清单
