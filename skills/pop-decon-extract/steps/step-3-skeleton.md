# Step 3: 事实骨架 (Lv2/Lv3)

> **方向**：逆 `pop-writer-chapter`。逐章提取核心事件链 + 角色状态变化 + 信息披露。
> **核心约束**：Lv2 覆盖 ch1-100，Lv3 覆盖全书。Lv1 不产事实骨架。
> **前置条件**：Step 1（ETL）+ Step 2（角色卡）已完成。

---

## I/O 注解

| 维度 | 内容 |
|:-----|:-----|
| **读什么** | `_temp/baseline-data.json`（events[]）+ `_temp/chapter-index.json`（逐章 tags）+ 逐章原文精读 |

| **产出** | `写作资产/事实骨架/ch001-050-事实骨架.md`（Lv2）+ `ch051-100-事实骨架.md` etc.（Lv3） |
| **门禁** | 仅 Lv2/Lv3 执行此步骤；Lv1 止步于 Step 2 |

## 操作步骤

### 1. 前置确认

```
□ Lv2：ch1-100 原文到位
□ Lv3：全书原文到位
□ Step 1 已完成（三个 JSON 存在）
□ Step 2 已完成（角色卡已产出）
→ 全部通过 → 继续
→ 未通过 → 退回
```

### 2. 逐章提取事实骨架

对每个章节执行以下操作：

**a) 读 JSON 数据**
- 从 `chapter-index.json` 获取章节 tags 和 charCount
- 从 `baseline-data.json#events[]` 提取本章事件列表
- 从 `world-data.json` 提取本章世界观条目

**b) 精读原文**
- 确认 JSON 中事件描述的准确性
- 补充 JSON 未覆盖的关键细节

**c) 产出每章 5-8 个结构化事件**

每个事件格式：

```
- [事件类型] 事件简述 @chXXX
  参与角色: [角色列表]
  状态变化: [角色状态的前后变化，无则不写]
  影响: [对后续剧情的推动作用]
```

**事件类型：**
| 类型 | 说明 | 示例 |
|:-----|:-----|:-----|
| `交锋` | 战斗/对抗 | 击杀/逃离/谈判 |
| `探索` | 发现新地点/信息 | 发现秘境/阅读古籍 |
| `社交` | 角色互动 | 结盟/背叛/对话 |
| `成长` | 角色能力提升 | 升级/领悟新技能 |
| `披露` | 世界观信息揭示 | 历史真相/境界体系 |
| `转折` | 剧情关键转折 | 生死抉择/命运转折 |
| `日常` | 过渡性内容 | 修炼/赶路/市井 |

### 3. 产出文件结构

```
写作资产/事实骨架/
├── ch001-050-事实骨架.md    ← Lv2/Lv3
├── ch051-100-事实骨架.md    ← Lv3（Lv2 止步于 ch1-100）
├── ch101-150-事实骨架.md    ← Lv3（全书覆盖）
└── ...                      ← Lv3
```

**文件格式：**

```markdown
# ch001-050 事实骨架 — {书名}

## 章节概览

| 章节范围 | 核心弧 | 主要事件数 |
|:---------|:-------|:----------|
| ch001-050 | [本段剧情的核心弧名称] | [事件总数] |

## 逐章事件

### ch001 {章节标题}

- [交锋] [事件简述] @ch001
  参与角色: [角色A], [角色B]
  状态变化: [角色A 从X→Y]
  影响: [为后续Z事件埋下伏笔]

- [探索] [事件简述] @ch001
  参与角色: [角色A]
  影响: [揭示世界观设定W]

### ch002 {章节标题}
...
```

### 4. 门禁检查（通过/退回）

```
□ 每个事件标注 @chXXX

□ Lv2 覆盖 ch1-100；Lv3 覆盖全书
→ 全部通过 → 进入 Step 4

```
