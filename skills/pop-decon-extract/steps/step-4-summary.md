# Step 4: 摘要 — Phase 1 事实提取摘要

> **方向**：汇总 Phase 1 全部产出，输出标准化摘要报告。
> **核心约束**：摘要只汇总已有数据，不新增任何分析或归纳。
> **覆盖级别**：Lv1（仅角色卡摘要）/ Lv2/Lv3（完整摘要）

---

## I/O 注解

| 维度 | 内容 |
|:-----|:-----|
| **读什么** | Step 1 产出（三个 JSON）+ Step 2 产出（角色卡）+ Step 3 产出（事实骨架，若有） |
| **做什么** | 汇总 Phase 1 全部产出为结构化摘要报告 |
| **产出** | `_参考书/{书名}/Phase1-事实提取摘要.md` |
| **门禁** | 所有前置步骤已完成；摘要中不引入任何新数据 |

---

## 操作步骤

### 1. 前置确认

```
□ Lv1: Step 1 + Step 2 已完成
□ Lv2/Lv3: Step 1 + Step 2 + Step 3 已完成
→ 全部通过 → 继续
→ 未通过 → 退回完成前置步骤
```

### 2. 收集统计数据

从 `_temp/baseline-data.json` 提取：
- 总章节数
- 总角色数（主角 + 配角 + 龙套）
- 总事件数
- 总地点数
- 等级/境界条目数

### 3. 汇总角色产出

列出已产出的角色文件：
- 主角卡（Lv4）
- 配角卡（Lv3）清单
- 龙套池（角色数量）

### 4. 汇总事实骨架（Lv2/Lv3）

列出已产出的骨架文件：
- 覆盖章节范围
- 总事件数
- 主要事件类型分布

### 5. 产出摘要文件

路径：`_参考书/{书名}/Phase1-事实提取摘要.md`

**模板：**

```markdown
# Phase 1 — 事实提取摘要

> @book: {书名} | @level: {Lv1/Lv2/Lv3} | @date: {YYYY-MM-DD}

## 1. 覆盖范围

| 维度 | 值 |
|:-----|:---|
| 章节范围 | ch001-{chN} |
| 覆盖字数 | {字数}字 |
| 处理级别 | Lv{1/2/3} |

## 2. ETL 数据

| 文件 | 状态 | 关键条目数 |
|:-----|:-----|:----------|
| `_temp/baseline-data.json` | [已完成] | characters: {N}, places: {N}, events: {N} |
| `_temp/chapter-index.json` | [已完成] | chapters: {N} |
| `_temp/world-data.json` | [已完成] | entries by category: {各分类条目数} |

## 3. 角色产出

| 类型 | 文件 | 角色 |
|:-----|:-----|:-----|
| 主角卡(Lv4) | `状态/角色/{主角名}-角色卡-Lv4.md` | {主角名} |
| 配角卡(Lv3) | `状态/角色/{配角名}-角色卡-Lv3.md` | {配角清单} |
| 龙套池 | `状态/角色/龙套池.md` | {龙套数量} 个命名角色 |

## 4. 事实骨架 (Lv2/Lv3)

| 文件 | 覆盖章节 | 事件数 |
|:-----|:---------|:-------|
| `写作资产/事实骨架/ch001-050-事实骨架.md` | ch001-050 | {N} |

## 5. 质量审计

| 检查项 | 状态 |
|:-------|:-----|
| ETL 已执行 | [通过] |
| 所有角色硬数据有 JSON 来源 | [通过] |
| 所有 chXX 标注完整 | [通过] |
| 推断已标注「推断」 | [通过] |
| 无故事引擎内容 | [通过] |
```

### 6. 门禁检查（通过/退回）

```
□ 摘要只汇总已有数据，无新增分析
□ 统计数据准确反映了各文件的实际情况
□ Lv1 摘要不含事实骨架内容
→ 全部通过 → Phase 1 完成
→ 未通过 → 退回修正
```
