## 核心哲学

**拆书 = 逆向破解。** PRD 是破解完成后的总结报告——不是快速扫描（Phase S 已废弃），而是基于全管线数据的深度综合。

PRD 回答一个核心问题：**如果我要立项写一本这样的书，我需要知道什么？**

答案来自 L1-L4 全管线产出的交叉验证，而非凭空推断。

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase 4 未完成就执行** — 创意溯源产出缺失 → 退回 Phase 4 |
| ❌2 | **核心假说无证据支撑** — 核心假说必须有 L1-L4 的 chXX 证据支撑，不得凭书名/直觉编造 |
| ❌3 | **赛道判定凭书名猜测** — 赛道必须基于实际拆解数据（设定+创意溯源），不凭书名猜测 |
| ❌4 | **可复用资产清单引用未入库条目** — 必须引用已入库的 pop-trope-library 条目，不得列出未入库的虚拟条目 |
| ❌5 | **PRD 与全管线数据矛盾** — PRD 中的结构/卖点/读者等结论必须与 L1-L4 产出一致，不得自行推翻已有结论 |

## ❌ WRONG 示例

| 错误做法 | 问题 |
|:---------|:-----|
| 凭书名「XX修仙」直接判定赛道为修仙 | 违反红线 3，赛道必须从设定+创意溯源数据提炼 |
| 核心假说写「这是一本爽文」无任何 chXX 证据 | 违反红线 2，假说需有全管线证据支撑 |
| 可复用资产清单列出「剧情库/修仙/升级流」但该目录不存在 | 违反红线 4，必须引用已入库条目 |
| Phase 4 创意溯源未完成就跳到 PRD | 违反红线 1，PRD 是最后环节 |

## 边界条件

| # | 场景 | 处理 |
|:-:|:-----|:-----|
| 1 | 某管线阶段产出不完整 | 标注「Phase X 数据不足」，基于已有数据产出，不编造缺失部分 |
| 2 | 前N章拆解（非全书） | PRD 标注拆解窗口百分比，结构预估标注「基于前N章推断」 |
| 3 | 文风DNA文件缺失 | 标注「文风DNA未产出」，目标读者部分基于设定+爽点设计推断 |
| 4 | 套路库为空 | 标注「套路库未入库」，可复用资产清单对应行标注「待入库」 |
| 5 | 创意溯源参考域单一 | 如实反映，不人为扩充参考域分布 |

> **核心理念**：PRD 是逆向破解的终点站。所有结论必须可追溯到 L1-L4 的具体产出文件。PRD 不产生新数据，只综合已有数据。填写前：扫描全管线产出 → 交叉验证关键结论 → 每结论标注数据来源。

**并行策略**：Step 1 的数据收集可并行委托 2-3 个子 agent 分组读取（如：①设计包+L2单元卡+L3剧情线 ②L4全书事件+设定+创意溯源 ③文风DNA+套路库）。Step 2 必须由主 agent 综合执行，确保交叉验证一致性。
