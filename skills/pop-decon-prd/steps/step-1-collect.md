# Step 1：收集 L1-L4 全管线产出 → `_temp/prd-collected-data.md`

**目标**：扫描全管线产出，统计关键数据，为 Step 2 综合提供数据基础。

## 收集清单

### 1. 设计包统计（Phase 1）
- 读取 `设计包v3/` 目录
- 统计：章节数 / 总字数 / 套路条目数
- 记录：拆解窗口（前N章 / 全书M章 / 百分比）

### 2. L2 单元卡统计（Phase 2）
- 读取 L2 单元卡目录
- 统计：单元总数 / 类型分布（爽点型/转折型/铺垫型/...）
- 提取：可迁移要点清单

### 3. L3 剧情线统计（Phase 2）
- 读取 L3 剧情线目录
- 统计：线数 / 线型分布（成长线/冲突线/情感线/...）
- 提取：核心剧情线摘要

### 4. L4 全书事件统计（Phase 2）
- 读取 L4 全书事件目录
- 统计：事件总数 / 置信度分布（高/中/低）
- 提取：关键事件清单（S级/A级）

### 5. 设定统计（Phase 3）
- 读取 L1 六件套目录（L1-01~06）
- 读取 `世界宪法.md`
- 读取 `combat_capability.yaml`（数值体系）
- 读取 `起点快照.md` / `终点快照.md`
- 统计：设定文件数 / 关键设定要素

### 6. 创意溯源统计（Phase 4）
- 读取创意溯源目录
- 统计：参考域数 / 条目数 / 转化深度分布
- 提取：跨域参考索引

### 7. 文风DNA
- 读取文风DNA文件
- 提取：核心文风特征 / 爽点设计模式

### 8. 套路库统计
- 读取 `pop-trope-library/原始素材/` 目录
- 统计：已入库套路卡数 / 标签分布
- 提取：已入库条目清单（书名+标签）

## 产出

`_temp/prd-collected-data.md` — 结构化数据摘要，包含以上 8 项统计结果。

> **注意**：本步骤只收集不分析。所有数据原样记录，分析判断留给 Step 2。
