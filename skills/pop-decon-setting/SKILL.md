## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase 2 未完成就归纳** — L2/L3产出缺失 → 退回 Phase 2 |
| ❌2 | **骨架层未完成就写血肉层** — L1-02 力量体系未产出时，不得开始地理/势力/历史/资源/物种等血肉层设定。血肉层必须塞入骨架层的金字塔 |
| ❌3 | **凭空发明设定** — 无设计包/chXX 证据且未标注「数据极少」的名称 |
| ❌4 | **编造境界链** — 原文从未出现的分级体系（如「凡人→超凡→破格」） |
| ❌5 | **前N章产出全书级文件** — 文件名不得含"全书"，必须有 scope 声明 |
| ❌6 | **归纳=编造** — 有 chXX 证据=归纳，没有且未标注=编造 |
| ❌7 | **Wiki 数据未标注置信度** — 从 `_temp/wiki-skeleton.md` 消费的设定必须标注「📖 Wiki来源」，不得混入精度数据中伪装成已验证 |
| ❌8 | **正文内联 chXX** — 正文中不得出现内联章节号（如"建立终焉之地（ch001）"）。证据归表格"证据章"列；非表格段落末尾以独立 `> 证据：chXXX` 行承载。保持阅读流顺畅 |

## ❌ WRONG 示例

| 错误做法 | 问题 |
|:---------|:-----|
| 跳过 L1-02 力量体系直接写地理/势力/历史 | 违反红线 2，血肉层必须塞入骨架层金字塔 |
| 凭空发明「凡人→超凡→破格→神话」境界链，原文未出现 | 编造设定，违反红线 4 |
| 前N章产出全书级文件 | 数据不足，违反红线 5 |
| 力量体系分类无设计包证据支撑 | 违反红线 6 |
| 正文写"天龙创世建立终焉之地（ch001）" | 内联 chXX 打断阅读，违反红线 8。应改为正文不标 chXX，表格证据列或段落末尾证据行承载 |

## 边界条件

| # | 场景 | 处理 |
|:-:|:-----|:-----|
| 1 | 某维度数据极少 | 注明「前 N 章此维度数据极少」，跳过该模板 |
| 2 | 原文设定前后矛盾（ch10 vs ch80） | 取最新版本，标注矛盾点及出现章节 |
| 3 | 前N章止步 | 直接产出当前可用数据 |
| 4 | 角色状态缺失 | 标注「数据不足」，引用最近可用章 |
| 5 | 战斗数据不足 | 产出可用数据，标注「样本不足」 |
| 6 | 第一人称视角局限 | 注明视角局限，区分「原文明确」和「角色推断」 |

```

[ ] （可选）_temp/wiki-skeleton.md — Wiki 全局骨架
[ ] 套路库非空
```

> **核心理念**：归纳≠编造。写作是引擎→设定→正文，拆书是正文→设定→引擎。每件套回答「走到第 N 章积累了什么原文证据」。填写前：扫描设计包 → 高频模式提取 → 每模式标注 ≥2 个 chXX 出处。
> 
> **Wiki 骨架使用时**：对精度窗口内已出现的设定标注「✅ chXX 确认」，对精度窗口外的设定标注「📖 Wiki来源，未在阅读窗口内验证」。不得用 Wiki 数据覆盖精度窗口内的原文证据。

**并行策略**：**骨架层（L1-02）必须先单独完成**，不可并行。骨架完成后，血肉层可并行委托 3-4 个子 agent 分组产出（如：①世界蓝图+历史驱动力 ②资源物品+物种天赋 ③势力格局+金手指+世界观弹性边界）。每组读各自的设计包子集 + Phase 2 L2/L3文件 + **L1-02 力量体系（骨架参考）**。context 中给出模板格式和关键数据摘要可减少子 agent 的阅读量。
