# Step 10：金手指设计 → L1-07-金手指设计.md

**数据来源**：Phase 1 设计包🔒面板/系统数据 + Phase 2 L2单元卡金手指触发事件 + 原文系统描述

**归纳逻辑**：
1. 从设计包面板数据中提取金手指的类型和核心载体（系统面板/戒指空间/重生记忆等）
2. 从 L2单元卡中提取金手指触发的关键事件，确认金手指的核心功能和升级路径
3. 从原文系统描述中提取金手指的限制条件（代价/冷却/副作用/前提/上限）
4. 交叉验证金手指与 L1-02 力量体系的交互关系（是否受境界限制/是否能越级/是否独立于功法）
5. 分析金手指的商业逻辑：爽点供给节奏/压力机制/读者代入点/可持续性
6. **禁止行为**：原文没有的金手指功能 = 不准编造。原文没有的限制条件 = 标注「原文未明确」

### 金手指类型判定标准

| 类型 | 判定特征 | 典型表现 |
|:-----|:---------|:---------|
| 系统流 | 有可视化的系统面板/任务/奖励 | 「叮！恭喜获得…」 |
| 重生流 | 主角拥有前世记忆 | 利用未来信息改变命运 |
| 签到流 | 每日/定期签到获得奖励 | 「签到成功，获得…」 |
| 随身空间 | 拥有独立存储/种植/制造空间 | 戒指/玉佩/丹田内的空间 |
| 面板流 | 有属性面板但无任务系统 | 属性点/技能树/经验值 |
| 血统流 | 特殊血脉/体质带来的能力 | 龙血/神体/圣骨 |
| 吞噬流 | 通过吞噬他人/物品获得能力 | 吸星大法/吞噬进化 |
| 模拟器 | 可模拟人生/推演未来 | 「模拟开始…」 |
| 组合型 | 同时拥有多种金手指 | 主金手指+副金手指 |

### 与 L1-02 力量体系的关系

金手指设计是 L1-02 力量体系的补充和深化：
- L1-02 回答「这个世界怎么升级」
- L1-07 回答「主角凭什么比别人升得快/强」

两者必须交叉验证：金手指的越级能力必须与 L1-02 的跨级约束一致。
