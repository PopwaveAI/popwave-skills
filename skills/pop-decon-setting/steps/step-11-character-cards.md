# Step 11：主要角色人物卡 → 主要角色人物卡.md

**数据来源**：Phase 1 白描卡🔒角色数据 + L1-04 人物动机分析 + Phase 2 L2/L3 角色行为轨迹 + 原文对白 + L1-07 金手指设计

**归纳逻辑**：
1. 从 L1-04 人物动机分析中继承核心欲望/致命缺陷/功能分类
2. 从白描卡角色数据中提取基本信息（姓名/身份/外貌/年龄/阵营）
3. 从 L2/L3 角色行为轨迹中提取行为模式（面对敌人/弱者/背叛/选择时的反应）
4. 从原文对白中提取说话方式（一句话概括语言风格）
5. 从 L2/L3 边界状态中提取人物弧光（起点→终点+转变触发事件）
6. 从 L1-07 金手指设计中关联主角的能力体系
7. **敌人专属**：从 L2/L3 冲突事件中提取威胁等级/对抗阶段/弱点/对抗策略/结局
8. **禁止行为**：原文没有的角色信息 = 不准编造。行为模式必须从原文行为归纳，不可用形容词堆砌

### 收录标准

| 等级 | 出场章数 | 处理方式 |
|:-----|:---------|:---------|
| S 核心角色 | 全书/大半 | 逐人详细建卡（主角+核心对手 1-3人） |
| A 主要角色 | ≥20章 | 逐人建卡（盟友/导师/卷级反派 3-6人） |
| B 功能角色 | 5-19章 | 摘要卡（功能/反应模式/说话方式） |
| 路人 | <5章 | 不收录 |

### 与 L1-04 人物动机分析的关系

人物卡是 L1-04 人物动机分析的展开和深化：
- L1-04 回答「这个角色想要什么/有什么缺陷/什么功能」
- 人物卡回答「这个角色是谁/怎么行动/怎么说话/怎么变化」

L1-04 的人物动机表直接继承到人物卡的「性格与驱动」模块，不重复提取。

### 敌人卡设计要点

敌人卡是本产出的核心增量，需特别注意：
- **威胁等级递进**：敌人必须按 S→A→B 递进出现，不能一开始就面对S级敌人
- **弱点可利用性**：每个敌人必须有可被主角利用的弱点（性格/能力/关系/资源）
- **叙事功能覆盖**：全书敌人需覆盖四种叙事功能（压力源/成长催化剂/镜像对照/牺牲品）
- **对抗策略多样性**：不能所有敌人都用同一方式击败

**产出**：按 `templates/主要角色人物卡.tpl.md` 填写
