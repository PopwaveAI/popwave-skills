# Step 13：角色对白风格库 → 角色对白风格库.md

**数据来源**：原文 chapters/ 目录直接提取

**归纳逻辑**：
1. 对每个 S/A 级角色，回到原文逐章扫描对白
2. 提取 5-10 条代表性对白，覆盖不同场景类型（日常/战斗/情绪爆发/关键决策/对峙）
3. 每条标注：章号 + 场景语境 + 对白原文 + 语言特征分析
4. 总结每个角色的"语言指纹"（一句话概括）
5. 横向对比各角色语言差异，生成对照表
6. 提取场景化对白模板（如"主角威胁敌人"的标准对白模式）
7. **禁止行为**：原文没有的对白 = 不准编造。对白必须逐字摘录原文

**续写/同人价值**：续写时对照本库确保角色不 OOC。同人写作时参考语言风格还原角色声音。

**工作量提示**：本步骤需要回原文扫描，是 Phase 3 中工作量最大的步骤。建议用子 agent 并行处理不同角色。

**产出**：按 `templates/角色对白风格库.tpl.md` 填写
