# Step 2：力量体系 → L1-02-力量体系.md

> **定位**：力量体系是男频全书的核心骨架，是 Phase 3 的**第一优先级**产出。本 step 必须先于其他设定 step 完成，因为地理/势力/历史/资源等血肉层设定需要塞入本卡构建的金字塔骨架。

**数据来源**：`world-data.json#class.entries[]` + `#magic.entries[]` + Phase 1 设计包 + Phase 2 L2/L3 叙事结构 + Phase 1.5 白描卡

**产出模板**：`templates/L1-02-力量体系.tpl.md`

---

## 两层结构

本 step 产出分两层，**第一层（事实）未完成不得写第二层（理解）**：

| 层 | 章节 | 任务 | 数据源 |
|:---|:-----|:-----|:-------|
| 第一层·事实 | 一~四 | 记录原文力量体系（核心力量定义/境界金字塔/主角路线图/跨级约束） | 设计包 class/magic 字段 + 原文面板 + L2/L3 + 白描卡 |
| 第二层·理解 | 五 | 拆解设计逻辑与可迁移模式 | 事实层 + L2/L3 + 白描卡 + 金手指设计交叉验证 |

## 第一层：设定事实记录（一~四）

### 一、核心力量定义

**归纳逻辑**：
1. **识别核心力量形态**：从设计包 class/magic 字段 + 白描卡的力量描写中，判断核心力量是什么形态——灵能/灵气/气/原力/魔力/信仰之力/钱/人口/官位职级/血统/科技/知识/其他。若为复合形态（如主+副），说明占比与关系
2. **追溯力量源头**：力量从哪来（天地灵气/神赐/基因/修炼积累/契约/掠夺）。源头决定可持续性与稀缺性
3. **配套世界观规则**：力量体系依附于怎样的世界观？世界为什么会有这种力量？力量体系与世界起源/规则的关系。**此项与 L1-01 世界蓝图的"力量来源"字段交叉验证，但本卡更深入——世界蓝图只记录"有什么"，本卡回答"为什么"**
4. **禁止行为**：原文没有的力量形态 = 不准编造

### 二、境界分级金字塔

**归纳逻辑**：
1. 从 class 条目提取完整等级链（如凡人→筑基→金丹…），**每个境界名必须有 chXX 原文**
2. 每个境界填一行金字塔表格，七列：
   - **质变点**：什么标志进入此级（如"丹田凝实""觉醒回响"）
   - **破坏力/影响力**：能做什么（如"一拳碎山""控制一城"）
   - **社会地位**：此级在社会中的角色（如"宗门弟子""一方诸侯"）
   - **金字塔数量**：稀缺度（大量/常见/稀少/极稀少/唯一）
   - **资源需求**：升级到此级需要什么（如"百年灵药+悟道"）
3. **非传统修炼流处理**：若原文无明确境界分级（如都市/官场文以"职级/财富"为力量），用对应社会层级替代（如：科员→科长→处长→厅长→部长）
4. **金字塔结构说明**：描述金字塔整体形态——陡峭型（少数垄断顶层）还是平缓型（层级差距小）
5. **禁止行为**：原文没有的境界名 = 不准编造。原文没有的职业名 = 不列入

> 事实层门禁：境界名/层级名必须有 chXX 出处，否则标注「数据极少」并跳过该维度。

### 三、主角晋级路线图×章节图

**归纳逻辑**：
1. 从白描卡 + L2 单元卡的力量定位列 + 角色卡成长路径中，提取主角每次境界变化的章节
2. 每个晋级阶段填一行表格，七列：
   - **起始境界→终止境界**：本阶段的境界变化
   - **章节范围**：chXXX-YYY
   - **升级触发事件**：什么事件触发升级（如"玄武自杀触发濒死"ch557）
   - **停留章数**：在此境界停留多少章（蓄力长度）
   - **全书节奏位置**：前段·铺垫 / 前段·起步 / 中段·加速 / 中段·高潮 / 后段·爆发 / 后段·收束
3. **节奏分析**：从路线图看全书节奏——匀速还是加速？哪个阶段停留最长（蓄力）？哪个阶段跳级最快（爆发）？升级节奏与剧情高潮的对应关系
4. 此表是全书节奏的"力量视角快照"，可与 Phase 2 卷纲的幕序列交叉验证

### 四、跨级约束与力量天花板

**归纳逻辑**：
1. **跨级约束**：从白描卡的战斗段落提取——差几级不可战胜？越级条件是什么（智斗/金手指/规则漏洞/特殊道具/人数优势）
2. **力量天花板**：从角色卡成长路径确认——故事结束时最强级别是什么？是设定硬上限还是主角个人极限

## 第二层：力量体系拆解理解（五）

> 从"记录原文有什么"上升到"理解作者为什么这么设计"。每条结论必须有 chXX 证据，不得凭空推断。

### 5.1 设计哲学

**归纳逻辑**：
1. 对比传统修炼流（凡人→筑基→金丹…），本书力量体系做了什么取舍？为什么？
2. 核心设计逻辑回答：作者用这套体系解决了什么叙事问题？（如：避免战力膨胀 / 制造智斗空间 / 服务主题表达）
3. 证据来源：L2 单元卡的力量定位列 + 白描卡中的力量转折点

### 5.2 叙事功能

**归纳逻辑**：从 L2/L3 叙事结构中提取力量体系承担的功能，逐项标注 chXX：
- **压力源**：力量差距如何制造压迫感
- **成长标尺**：力量层级如何标记主角进度
- **阶级隐喻**：力量体系是否映射社会阶级/权力结构
- **剧情引擎**：力量追求如何驱动剧情前进
- 发现其他功能时追加行

### 5.3 爽点驱动机制

**归纳逻辑**：从白描卡的爽点段落 + L2 卡的情感设计列中提取，力量体系如何供给爽感：
- **升级仪式感**：突破时的仪式/代价/外部反馈 + 触发频率
- **越级反杀**：什么条件允许越级 + 频率
- **能力克制链**：能力间的相生相克
- **信息差碾压**：力量认知差如何制造智斗优势
- 发现其他爽点类型时追加行

### 5.4 与其他系统的咬合关系

**归纳逻辑**：**血肉层设定如何塞入力量体系骨架**——交叉验证力量体系与其他 L1 产出的关系：
- **金手指**：读 `L1-07-金手指设计.md`，判断独立/嵌套/递进关系
- **地理蓝图**：读 `L1-01-世界蓝图.md`，判断力量体系是否有空间分布特征（灵气浓度差异/势力地盘按层级划分）
- **势力格局**：读 `L1-05-势力格局.md`，判断力量层级是否决定势力地位
- **历史驱动**：读 `L1-03-历史与驱动力.md`，判断力量体系的历史演化（衰退/复兴/变天）
- **资源经济**：读 `L1-06-资源与物品.md`，判断力量升级消耗什么资源形成循环
- **世界观设定**：读 `L1-01` + `L1-03`，判断力量体系是否与世界起源/规则绑定

### 5.5 可迁移设计模式

**归纳逻辑**：从设计哲学+叙事功能+爽点驱动中提炼可复用模式：
1. 每个模式 = 一句话设计要点 + 适用赛道 + chXX 证据
2. 判断标准：该模式是否依赖本书独有设定？不依赖 = 可迁移
3. 参考 `Phase2-叙事技法分析.md` 的可迁移技法清单交叉验证

### 5.6 不可迁移部分

**归纳逻辑**：识别与本书独有设定深度绑定的力量体系元素：
1. 每项标注"不可迁移原因"（与哪些本书独有设定绑定）
2. 参考 `Phase2-叙事技法分析.md` 的不可迁移部分清单交叉验证
3. 若全部可迁移则如实说明，不强凑不可迁移项
