# Step 4：物种与天赋 → L1-04-物种与天赋.md

**归纳逻辑**：
1. 从 species 条目中提取主要种族名称、特征、是否可玩、势力归属
2. 从怪物数据中按 CR 等级分类异兽/怪物分布与生态

4. **（六维融合）从白描卡的事件白描和人物关系变化中提取人物动机分析**

### 六维拆文法维度1融合：人物动机分析

> **融合原因**：六维维度1（人物设定：核心欲望/致命缺陷/功能分类）与 L1-04 的 NPC/角色数据描述同一批对象。
> 融入此处统一为结构化提取，消除重叠，同时解决六维产出偏浅的问题（用表格化字段替代散文式描述）。

在 L1-04 的角色/NPC 表中增加 3 列：

| 列名 | 提取规则 | 数据来源 |
|:-----|:---------|:---------|
| **核心欲望** | 驱动该角色行动的根本动力（一句话） | 事件白描中角色的反复行为模式 + 🔒关键对白 |
| **致命缺陷** | 该角色性格中的软肋/盲点（一句话） | 事件白描中角色的失败/挫折原因 |
| **功能分类** | 推动剧情类（篮子A）/ 强化主角类（篮子B）/ 暗桩角色 | 角色在本卷中的叙事功能 |

**功能分类判定标准**：
- **推动剧情类（篮子A）**：存在是为了制造剧情变化（反派、导师、盟友、关键信息携带者）→ 做了什么事故事才能往下一阶段走
- **强化主角类（篮子B）**：存在不推动剧情，但让读者更理解主角的性格脾气和软肋 → 不推动剧情但深化主角形象
- **暗桩角色**：前半段是强化主角型（篮子B），后半段变成推动剧情的关键角色 → 需标注转变点

**暗桩角色标注格式**：
```
⚠️ 暗桩角色：{角色名}
- 前半段功能：强化主角型（篮子B）— {具体表现}
- 转变点：chXXX {触发事件}
- 后半段功能：推动剧情型（篮子A）— {具体表现}
```

**提取规则**：
- 核心欲望必须从角色行为中归纳，不可凭空编造（标注证据章号）
- 致命缺陷必须从角色失败/挫折中归纳，不可使用形容词堆砌
- 功能分类基于角色在本卷中的叙事功能判定，不是基于角色好坏
- 每个主要角色（出场≥3章）必须填写这3列

**产出**：按 `templates/L1-04-物种与天赋.tpl.md` 填写
