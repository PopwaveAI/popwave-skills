# Step 5：势力格局 → L1-05-势力格局.md

**数据来源**：`world-data.json#faction.entries[]` + Phase 2 卷势力动机 + 白描卡人物关系变化

**归纳逻辑**：
1. 从 faction 条目提取主要势力名称、范围、核心目标
2. 从 Phase 2 卷幕的势力动机中提取权力分布关系
3. 标注各势力间的对抗/合作/中立关系
4. **（六维融合）从白描卡的人物关系变化中提取 A→B 动词格式的关系图谱**
5. **（六维融合）从事件白描中提取剧情冲突表（利益冲突/信念冲突）**

### 六维拆文法维度2+3融合：关系图谱+剧情冲突

> **融合原因**：六维维度2（人物关系图谱）与 L1-05 的势力关系描述同一批信息；六维维度3（剧情冲突）与 L1-05 的势力对立重叠。
> 融入此处统一为结构化提取，用 A→B 动词格式替代散文描述，用冲突表格替代笼统描述。

#### 关系图谱（维度2融合）

用动词标注每对角色/势力之间的关系：

格式：
```
A → B: 动词（利用/背叛/信任/牺牲/保护/仇视/暗恋…）— 类型：利益同盟/情感羁绊/表面朋友实际对立
```

**提取规则**：
- 动词必须从白描卡的人物关系变化段或事件白描中归纳
- 每对主要角色/势力之间必须标注关系
- 关系类型三选一：利益同盟（基于共同利益）/ 情感羁绊（基于感情）/ 表面朋友实际对立（伪装关系）
- 关系发生变化的标注变化章号

#### 剧情冲突表（维度3融合）

从事件白描中罗列所有冲突，标记类型：

| 冲突名 | 对立方A | 对立方B | 冲突类型 | 焦点（争夺什么） | 层级 | 证据章 |
|:-------|:-------|:-------|:---------|:---------------|:-----|:------:|

**冲突类型**：
- **利益冲突**：角色间对稀缺资源的争夺
- **信念冲突**：角色间在价值观/原则上的根本对立

**冲突层级**：
- **卷级**：冲突在本卷内解决
- **全书级**：冲突延续到后续卷
- **大反派标记**：某角色与主角既有利益冲突又有信念冲突 → 标记为大反派

**提取规则**：
- 冲突必须从事件白描中识别，不可凭空编造
- 每个冲突标注证据章号
- 既有利益又有信念冲突的必须标记为大反派

**产出**：按 `templates/L1-05-势力格局.tpl.md` 填写
