# Step 1：可回源性审计

**目标**：验证每个 Phase 产出的每个硬数据都有可追溯的数据来源。

| 检查项 | 对照规则 |
|:-------|:---------|
| 角色卡中的年龄/等级/职业 | → `_temp/baseline-data.json` 或 `_temp/world-data.json` 中是否存在 |
| L1 六件套中的体系名/势力名 | → `_temp/world-data.json` 对应分类中是否存在 |
| Canvas 中事件推进 | → `_temp/chapter-index.json` 对应章的事件摘要 |
| 故事引擎中核心假说 | → Phase 2 Canvas 主线 + Phase 3 世界宪法中是否有对应原文证据 |

**审计方法**：
```
1. 随机抽取 10 个硬数据（跨越不同 Phase 产出文件）
2. 每个硬数据 → 打开对应 JSON → 搜索是否存在
3. JSON 中存在但值不匹配 → 标注"数据偏离"
4. JSON 中不存在 → 标注"⚠️ 未经验证的断言"
5. JSON 中不存在的非硬数据（如"故事基调"）→ 标注"归纳自 {数据来源}"
```
