# pop-decon-volume · 幕纲卷纲 v2.1.0

> **定位**：Phase 2 of deconstruction. 从章节数据和设计包中识别卷边界、幕边界、
> 反推剧情线、追踪契诃夫枪链、聚合套路库。
> **格式变更**：v2.0.0 起幕纲由 YAML 改为 **Markdown**（增加幕级元数据、副线统计、
> 量化节奏数据），卷纲增加分支线跟踪表。
> **方向**：逆 `plot` + `pop-writer-chapter`。只聚类原文已呈现的结构，从不「设计」剧情。

---

## 层级定义（卷 vs 幕 vs 段）

**决定边界尺度的核心判据不是字数，而是力量体系是否跨越了一个代际。**

| 层级 | 功能 | 典型规模 | 核心边界信号 | 反面例子（不是卷边界） |
|:-----|:-----|:--------|:------------|:----------------------|
| **卷** | 一次力量体系代际跃迁 / 世界级冲突质变 | 80-200章 | ①力量体系代际跃迁（1阶→传奇 / 传奇→圣者 / 圣者→神性）②冲突类型质变（生存求生→战争→封神博弈）③地理版图维度切换（一座城→大陆→跨位面） | 换城市（那是幕信号）/ 出现新配角（那是段信号） |
| **幕** | 一个有始有终的行动弧 | 15-40章 | ①地点切换②新配角阵营加入③叙事基调转变（压抑→热血→黑暗）④有mini高潮收束 | 单场战斗（那是段信号）/ 新增世界观设定章（那是幕内节点） |
| **段**（可选） | 一个连续性场景块 | 3-8章 | ①时间/空间连续②同一冲突类型③单一POV或角色组 | — |

**实际操作原则：**
- 当你在设计包中看到"主角升级了" → 这是幕内事件
- 当主角的**战力定位**从"城市级冒险者"变成"区域级战力" → 这是卷边界
- 当敌人从"帮派混混"变成"教会组织"再变成"神灵" → 每次质变都是卷边界
- 不确定时优先划为幕，后续可合并为卷

**scope 声明（❗必须执行）：**
所有产出文件必须在首行之后添加以下声明：
```markdown
> ⚠️ **基于前X章数据推断**：本章只读到了全书的一部分（共约N卷）。
> 所有后续结论（剧情线走向/未解之谜估计/套路偏好）均基于此有限窗口，
> 不代表全书终态。如需更新，可在后续章节补充后修正。
```
其中 X = 实际读到的章节数，N = 用户告知或从已知信息估算的全书卷数。
「全书架构」这种命名本身就是scope claim——确认是全书还是一卷之前不要用。

## 产出目录树

```markdown
设计/
├── 卷/
│   └── 卷1.md                    ← Markdown，卷设计（当前读到的章节数据）
│                                  （全文时写作 卷/卷1.md ~ 卷/卷N.md）
│   └── 全书架构.md               ← 仅当确实确认是全书时使用，否则命名为 卷1-架构.md
└── 幕/卷1/
    ├── 幕1.md                    ← Markdown（非YAML），含幕级元数据+章级切片+节奏自检
    ├── 幕2.md
    └── chekhov-tracker.md        ← Markdown 伏笔追踪表
```

**命名规范（所有子agent严格执行）：**
- 卷文件：`# 卷X · 名称（chXXX-chYYY）` — 文件名为 `卷X.md`
- 幕文件：`# 幕X · 名称（chXXX-chYYY）` — 文件名为 `幕X.md`
- 间幕文件：`# 间幕X · 名称（chXXX-chYYY）` — 文件名为 `幕-间幕X.md`
- 伏笔追踪：`chekhov-tracker.md` — 首行 `# 契诃夫枪追踪 · 卷X（chXXX-chYYY）`
- 目录名：`幕/卷X/`（卷号与卷文件一致）

**delegate_task 时的格式锁定：**
委派子agent 时，必须在 context 中明确给出 heading 格式，例如：
```
产出文件首行必须严格为：「# 幕1 · 名称（ch001-ch020）」
禁止使用「act-01:」「act 1」「chapter 1」等变体
```

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase 1 未完成就聚类** — 设计包不存在 → 退回 |
| ❌2 | **卷边界无证据** — 必须有地点变更/情绪密度证据 → 退回补充 |
| ❌3 | **剧情线不是设计出来的** — 行来自 events 聚类，非 agent 自拟 |
| ❌4 | **前N章只做N章** — **每文件首行之后必须**标注「基于前X章推断，后续将修正」。且文件名/目录名不得出现"全书"字样 → 改用 `卷1-架构.md` |
| ❌5 | **契诃夫枪不编造** — 每枪必须有 set_in/fired 的章节证据 |
| ❌6 | **套路聚合不编造** — 仅从 `套路库/` 读取数据，不得凭空归纳 |
| ❌7 | **幕纲缺幕级元数据** — 每个 幕X.md 必须包含幕级元数据区块（戏剧功能/状态变化/冲突/副线活跃度/量化数据）→ 缺任意一项退回 |
| ❌8 | **卷纲缺分支线跟踪表** — 卷X.md 必须包含分支线跟踪表（L2 必备），记录所有≥2章的副线 |
| ❌9 | **产出格式为 Markdown** — 严禁输出 YAML 格式的幕纲。act-*.yaml 已废弃 |
| ❌10 | **设计包是唯一数据源，不是原始小说** — 子 agent 必须从 `写作资产/设计包v3/chXXX-设计包.md` 读取数据，不得绕过设计包直接从原始 TXT 提取。委派时在 context 中给出**绝对路径**防止相对路径解析失败 |
| ❌11 | **卷/幕混淆** — 15-30章规模的叙事弧是幕，不是卷。卷至少80章以上，需有力量体系代际跃迁信号。误把幕当作卷产出 → 退回。参见[层级定义](#层级定义卷-vs-幕-vs-段) |
| ❌12 | **命名不规范** — 卷文件必须 `# 卷X · 名称（chXXX-chYYY）`，幕文件必须 `# 幕X · 名称（chXXX-chYYY）`。首行格式不匹配 → 退回。参见[产出目录树](#产出目录树) |
| ❌13 | **delegate_task 未锁定格式** — 委派子agent时必须在其 context 中给出精确的 heading 格式范例。子agent 间命名不一致 → 追责到调度 agent |

## 产出格式规范

### 幕纲 — act-*.md

详见模板 `templates/act-skeleton.tpl.md`

必须包含以下区块：

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| 幕级元数据 | ✅ | 戏剧功能、状态变化(力量/关系/认知)、核心冲突、关键决策、副线活跃度、量化数据 |
| 章级切片 | ✅ | 每章字数/类型/紧张值/POV/场景/剧情线推进/登场角色/情绪目标/爽点/钩子/契诃夫枪 |
| 节奏自检 | ✅ | 剧情线活跃度、情绪节奏四段、爽点密度分布、综合评级 |

### 卷纲 — 卷X.md

详见模板 `templates/volume-design.tpl.md`

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| scope声明 | ✅ | 「基于前X章推断，后续将修正」— 文件第二行（首行之后） |
| 卷级定义 | ✅ | 核心命题、情感基调、幕数 |
| 起点/终点快照 | ✅ | 力量/关系/认知三轴变化（标注此卷在全卷中的定位） |
| 背景 | ✅ | 时间、地理、角色池、势力动机 |
| **剧情线** | ✅ | 主线1-3（含人格/阵营漂移，不只等级）、**每线≥5阶段** |
| **幕级摘要** | ✅ | 本卷包含哪些幕，每幕的范围和一句话核心（取代原来的分支线跟踪表在卷级的存在） |
| **分支线跟踪表** | ✅ | 所有≥2章的副线，含起止章/活跃幕/状态/描述 |
| 伏笔-回收对照表 | ✅ | 跨幕伏笔追踪 |
| 未解之谜清单 | ⚠️ L3推荐 | 卷末悬念 |
| 阵营漂移轨迹 | ⚠️ L2推荐 | 主角阵营变化节点 |
| **套路分析段** | ⚠️（套路库非空时强制） | 本卷使用频率Top套路（基于当前窗口） |

### 幕纲 — 幕X.md

详见模板 `templates/act-skeleton.tpl.md`

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| scope声明 | ✅ | 「基于前X章推断，后续将修正」 |
| 幕级元数据 | ✅ | 戏剧功能、状态变化(力量/关系/认知)、核心冲突、关键决策、副线活跃度、量化数据 |
| 章级切片 | ✅ | 每章字数/类型/紧张值/POV/场景/剧情线推进/登场角色/情绪目标/爽点/钩子/契诃夫枪 |
| 节奏自检 | ✅ | 剧情线活跃度、情绪节奏四段、爽点密度分布、综合评级 |

### 卷架构 — 卷1-架构.md（当只读到部分章节时）

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| scope声明 | ✅ | 「基于前X章（共约N卷），仅据此窗口划分的卷内幕结构」 |
| 幕划分 | ✅ | 每幕章范围+证据链 |
| 当前卷的主题宣言 | ✅ | 一句话+三段展开 |
| 各幕结构定位 | ✅ | 每幕在当前卷中的戏剧作用 |
| 各幕核心问题 | ✅ | 每幕的 dramatic question |
| 多幕预留接口 | ✅ | 幕与幕之间的连接点（提示可能的卷边界信号） |

| 文件 | 检查项 |
|:----|:-------|
| `设计/卷/卷1-架构.md`（或`全书架构.md`） | 有scope声明；幕划分有证据链；无空幕；有主题宣言+结构定位+核心问题；文件名与实际范围一致 |
