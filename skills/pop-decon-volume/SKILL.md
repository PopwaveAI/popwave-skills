# pop-decon-volume · 幕纲卷纲 v2.0.0

> **定位**：Phase 2 of deconstruction. 从章节数据和设计包中识别卷边界、幕边界、
> 反推剧情线、追踪契诃夫枪链、聚合套路库。
> **格式变更**：v2.0.0 起幕纲由 YAML 改为 **Markdown**（增加幕级元数据、副线统计、
> 量化节奏数据），卷纲增加分支线跟踪表。
> **方向**：逆 `plot` + `pop-writer-chapter`。只聚类原文已呈现的结构，从不「设计」剧情。

---

## 前置条件

- Phase 1 已完成：`写作资产/设计包v3/chXXX-设计包.md`（或 v2 格式）存在
- `_temp/chapter-index.json` / `plot-data.json` 就位
> 文件不全 → 退回 Phase 1。

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase 1 未完成就聚类** — 设计包不存在 → 退回 |
| ❌2 | **卷边界无证据** — 必须有地点变更/情绪密度证据 → 退回补充 |
| ❌3 | **剧情线不是设计出来的** — 行来自 events 聚类，非 agent 自拟 |
| ❌4 | **前N章只做N章** — 标注「基于前N章推断，后续将修正」|
| ❌5 | **契诃夫枪不编造** — 每枪必须有 set_in/fired 的章节证据 |
| ❌6 | **套路聚合不编造** — 仅从 `套路库/` 读取数据，不得凭空归纳 |
| ❌7 | **幕纲缺幕级元数据** — 每个 act-*.md 必须包含幕级元数据区块（戏剧功能/状态变化/冲突/副线活跃度/量化数据）→ 缺任意一项退回 |
| ❌8 | **卷纲缺分支线跟踪表** — volume-*.md 必须包含分支线跟踪表（L2 必备），记录所有≥2章的副线 |
| ❌9 | **产出格式为 Markdown** — 严禁输出 YAML 格式的幕纲。act-*.yaml 已废弃 |

## 产出格式规范

### 幕纲 — act-*.md

详见模板 `templates/act-skeleton.tpl.md`

必须包含以下区块：

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| 幕级元数据 | ✅ | 戏剧功能、状态变化(力量/关系/认知)、核心冲突、关键决策、副线活跃度、量化数据 |
| 章级切片 | ✅ | 每章字数/类型/紧张值/POV/场景/剧情线推进/登场角色/情绪目标/爽点/钩子/契诃夫枪 |
| 节奏自检 | ✅ | 剧情线活跃度、情绪节奏四段、爽点密度分布、综合评级 |

### 卷纲 — volume-*.md

详见模板 `templates/volume-design.tpl.md`

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| 卷级定义 | ✅ | 核心命题、情感基调、幕数 |
| 起点/终点快照 | ✅ | 力量/关系/认知三轴变化 |
| 背景 | ✅ | 时间、地理、角色池、势力动机 |
| **剧情线** | ✅ | 主线1-3（含人格/阵营漂移，不只等级）、**每线≥5阶段** |
| **分支线跟踪表** | ✅ | 所有≥2章的副线，含起止章/活跃幕/状态/描述 |
| 伏笔-回收对照表 | ✅ | 跨幕伏笔追踪 |
| 未解之谜清单 | ⚠️ L3推荐 | 卷末悬念 |
| 阵营漂移轨迹 | ⚠️ L2推荐 | 主角阵营变化节点 |
| **套路分析段** | ⚠️（套路库非空时强制） | 本卷使用频率Top套路 |

### 全书架构 — 全书架构.md

| 区块 | 必须 | 内容 |
|:-----|:----:|:-----|
| 卷划分 | ✅ | 每卷章范围+证据链 |
| 全书主题宣言 | ✅ | 一句话+三段展开 |
| 各卷结构定位 | ✅ | 每卷在全书中起什么戏剧作用 |
| 各卷核心问题 | ✅ | 每卷的 dramatic question |
| 多卷预留接口 | ✅ | 卷与卷之间的连接点 |
