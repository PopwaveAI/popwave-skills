# Step 2：卷纲归纳（含溯源燃料台+叙事技法分析）

> **方向**：从 L2 单元卡逆向归纳卷级结构，产出卷纲（含溯源燃料台+叙事技法分析）。
> **级别覆盖**：前N章 / 全书

---

## 数据来源

- **Step 1 产出的 L2 单元卡**（`设计/L2单元卡/L2-*.md`）— 唯一数据源
- L2 单元卡的「物理坐标」→ 确定卷的物理范围
- L2 单元卡的「结构分析」→ 归纳幕序列与功能位
- L2 单元卡的「嵌套子线」→ 识别跨卷方向（延伸子线）
- L2 单元卡的「单元边界信号」→ 确定卷边界
- **（fast mode）白描卡的事件白描+章末钩子** → 叙事技法分析的时间线对照数据

> **❗ 核心约束**：卷纲从 L2 单元卡逆向归纳，不从原文物理卷结构直接推导。
> 卷纲对齐 plot 卷纲模板结构，但适配拆书场景（删除「涌现燃料扫描」，燃料台改为溯源燃料台）。

## 归纳规则

### 1. 确定卷边界

扫描所有 L2 单元卡，按物理坐标（章节范围）连续性确定卷边界：
- 多个 L2 单元在同一物理卷内连续 → 归入同一卷纲
- 卷末有叙事弧收束或全局质变 → 卷边界确认
- L2 卡的嵌套子线「延伸」标记 → 跨卷方向的线索

### 2. 归纳本卷定位

从 L2 单元卡的结构分析和主题表达中，逆向归纳卷级定位：
- **卷定位**：一句话概括本卷在全书中干什么
- **主爽点**：从 L2 卡的「情感设计」和「读者收益」中提取 1-2 个核心爽点
- **读者期待**：从 L2 卡的主题表达中归纳
- **剧情brief**：3-5句话框选本卷走向，不写 beat

### 3. 构建舞台（7列人物表）

从 L2 单元卡的物理坐标中提取登场人物，填写 7 列人物表：

| 人物 | 定位 | 反应模式 | 说话方式 | 与主角关系 | 起点状态 | 终点状态 |
|------|------|----------|----------|:----------|----------|----------|

### 4. 填写溯源燃料台

溯源燃料台是卷纲的核心特色，用于记录「原文段落→猜测来源→置信度」。4 类溯源：

| 溯源类型 | 说明 | 示例 |
|:---------|:-----|:-----|
| **剧情溯源** | 原文中具体的剧情桥段，猜测其来源 | 「主角蛰伏后爆发复仇」→ 来源可能是某作品的类似桥段 |
| **设定溯源** | 原文中的世界观/设定元素，猜测其来源 | 「演武令金手指」→ 来源可能是某游戏机制 |
| **创意溯源** | 原文中的创意点子/反转设计，猜测其灵感来源 | 「穿越者心态蜕变」→ 来源可能是某心理学概念 |
| **质感溯源** | 原文的文风/质感/氛围，猜测其文学来源 | 「武侠质感」→ 来源可能是金庸/古龙风格 |

每类溯源格式统一为：
```
| 原文段落 | 猜测来源 | 置信度 |
|:---------|:---------|:-------|
| {原文中具体的段落或元素} | {猜测的来源作品/套路/概念} | {低/中/高} |
```

> **❗ 红线**：溯源燃料台4类溯源必须全部产出，缺任一类型退回。

### 5. 构建幕序列（含 L2 卡映射）

从 L2 单元卡逆向归纳幕序列，每幕映射到对应的 L2 单元卡：

| 幕 | 章节范围 | 功能位 | 主爽点 | 对应L2卡 | 下幕衔接 |
|:---|:---------|:-------|:-------|:---------|:---------|

### 6. 提取事件种子池

从 L2 单元卡的嵌套子线（尤其是「延伸」子线）中提取事件种子：
- **主线种子**：从 L2 卡的主结构弧中提取
- **支线/暗线种子**：从 L2 卡的嵌套子线中提取
- **世界观/设定种子**：从 L2 卡的物理坐标和主题表达中提取

### 7. 识别核心高潮点

从 L2 单元卡的结构分析中，识别本卷的核心高潮点：
- 每个高潮点标注：chXXX — 一句话描述 + 爽感类型
- 爽感类型从 L2 卡的情感设计中提取

### 8. 叙事技法分析（六维拆文法维度4+5融合）

> **来源**：六维拆文法的维度4（正常时间线）和维度5（成品剧情节奏）的分析能力。
> 融入此处是因为时间线打乱和信息差是卷级叙事技法，需从全卷 L2 单元卡中归纳。

#### 8.1 时间线对照

扫描全卷白描卡/L2 单元卡，将事件按**真实发生先后顺序**排列，与**成品章节顺序**对照：

| # | 真实时间线事件 | 对应章 | 成品顺序位置 | 是否打乱 | 打乱类型 |
|:-:|:-------------|:------:|:-----------:|:--------:|:---------|
| 1 | {真实最早发生的事件} | chXXX | 第N章 | 否 | — |
| 2 | {真实第二早的事件} | chXXX | 第M章 | 是 | 提前/延后/倒叙/插叙 |
| ... | | | | | |

**打乱类型定义**：
- **提前**：真实时间晚但成品排前 → 制造悬念（读者先看到结果，后看到原因）
- **延后**：真实时间早但成品排后 → 制造反转（读者最后才知道真相）
- **倒叙**：整段倒序讲述
- **插叙**：在主线中插入过去的事件片段

#### 8.2 信息差分析

从时间线对照中，提炼作者制造的**信息差**（六维拆文法的核心方法）：

| 信息差类型 | 具体内容 | 读者知道 vs 角色不知道 | 制造效果 | 回收章 |
|:-----------|:---------|:----------------------|:---------|:------:|
| 悬念型 | {哪些剧情被提前了} | 读者知道结果，角色不知道原因 | 悬念 | chXXX |
| 反转型 | {哪些剧情被藏到最后} | 角色知道，读者最后才知道 | 反转 | chXXX |
| 误会型 | {角色间信息不对称} | 角色A知道，角色B不知道 | 紧张/冲突 | chXXX |
| 伏笔型 | {前文暗藏线索} | 读者第一次读会忽略 | 回收时的惊喜 | chXXX |

> **precision mode**：从设计包事件链和 L2 单元卡的结构分析中提取
> **fast mode**：从白描卡的事件白描和章末钩子中提取

### 9. 估算篇幅预算

从 L2 单元卡的章节范围中，估算本卷的篇幅预算：
- 总章数：L2 卡章节范围的合集
- 各幕分配：幕序列中每幕的章数
- 总字数估算：基于章数 × 平均字数

## 产出

| 文件 | 内容 |
|:-----|:-----|
| `设计/卷纲/卷1-卷纲.md` | 按 `templates/卷纲-拆书版.tpl.md` 填写 |
| `设计/卷纲/卷2-卷纲.md` | ...（覆盖全部已识别卷） |

每个卷纲必须包含：
- **本卷定位**：卷定位/主爽点/读者期待/剧情brief（3-5句话）
- **舞台**：7列人物表 + 地图 + 时间
- **溯源燃料台**：4类溯源（剧情/设定/创意/质感），每类格式为 `| 原文段落 | 猜测来源 | 置信度 |`
- **幕序列**：含 L2 卡映射列
- **事件种子池**：主线种子/支线·暗线种子/世界观·设定种子
- **核心高潮点**：chXXX — 一句话描述 + 爽感类型
- **叙事技法分析**：时间线对照表 + 信息差分析表
- **篇幅预算**：总章数/各幕分配/总字数估算

## 与 plot 卷纲模板的差异

| 维度 | plot 卷纲模板 | 拆书版卷纲 |
|:-----|:-------------|:-----------|
| 涌现燃料扫描 | 有（本书设定→可触发事件） | **删除**（拆书不需要） |
| 燃料台 | 卷级外部燃料台 | **溯源燃料台**（原文段落→猜测来源→置信度，4类溯源） |
| 幕序列 | 5列 | **6列**（新增「对应L2卡」映射列） |
| 人物表 | 7列 | 7列（对齐） |
| 叙事技法 | 无 | **新增**（时间线对照+信息差分析，六维拆文法维度4+5融合） |
| 数据来源 | plot 设计阶段产出 | **从 L2 单元卡逆向归纳** |

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | **卷纲从 L2 卡逆向归纳** — 不得跳过 L2 直接从原文物理卷结构推导卷纲 |
| 2 | **溯源燃料台4类溯源必须完整** — 剧情溯源/设定溯源/创意溯源/质感溯源，缺任一类型退回 |
| 3 | **幕序列必须含 L2 卡映射** — 每幕必须标注对应的 L2 单元卡编号 |
| 4 | **不编造溯源来源** — 溯源猜测必须基于原文段落，标注置信度（低/中/高），不得凭空捏造 |
| 5 | **不设计剧情** — 卷纲是逆向归纳原文已呈现的结构，不是设计新剧情 |
| 6 | **舞台人物7列表完整** — 每个人物必须填写全部7列 |
| 7 | **叙事技法分析必须有时间线对照** — 至少标注3处时间线打乱点及信息差类型 |
