# Step 3：跨卷主题线追踪

> **方向**：从所有卷纲 + L2 单元卡中追踪跨卷持续演进的主题线，产出极简追踪文件。
> **级别覆盖**：前N章 / 全书

---

## 数据来源

- **所有卷纲**（`设计/卷纲/卷N-卷纲.md`）— 卷级结构视角
- **L2 单元卡**（`设计/L2单元卡/L2-*.md`）— 单元级事件视角
- L2 单元卡中标注「延伸」的嵌套子线 → 跨卷主题线追踪的种子
- 卷纲的幕序列中的 L2 卡映射 → 主题线在不同卷中的分布

> **❗ 核心约束**：跨卷主题线从 L2 单元卡的「延伸子线」和卷纲的结构中追踪，不从记忆推断。
> 本步骤产出极简 YAML 格式，不产出独立卡片。

## 追踪规则

### 1. 从 L2 单元卡的「延伸子线」提取种子

扫描所有 L2 单元卡的嵌套子线表，提取标注为「延伸」的子线：
```
L2-001 嵌套子线：
  - 子线1：武道修炼线（延伸）→ 种子
  - 子线2：演武令金手指线（延伸）→ 种子
  - 子线3：王成/王印仇怨线（闭合）→ 不追踪
```

### 2. 跨卷追踪同一主题的演进

对每个种子，在后续 L2 单元卡和卷纲中寻找同主题的子线或事件：
- 同一承压者/压力源持续施压 → 同一条主题线
- 压力源变化但主题延续 → 同一条主题线
- 主题完全切换 → 新的主题线

### 3. 确定跨卷范围

从卷纲的幕序列中，确定每条主题线跨越的卷范围：
- 主题线在哪些卷中活跃
- 主题线的触发卷和收束卷（如果已收束）

### 4. 标注承载 L2 卡

每条主题线必须标注它承载了哪些 L2 单元卡：
```
武道修炼线：
  承载L2：L2-001（ch001-ch011）→ L2-002（ch012-ch025）→ L2-003（ch026-ch040）
  跨卷范围：卷1
```

### 5. 描述演化阶段

每条主题线描述其演化过程（极简）：
- 触发→推进→高潮→收束（至少标注已有阶段）

### 6. 标注置信度

- **候选**：前N章只看到部分演化，无法确认完整走向
- **确认**：全书拆解，演化阶段完整

---

## 产出

产出单个文件 `设计/跨卷主题线.md`，使用 YAML 极简格式：

```yaml
# 跨卷主题线追踪
# 基于 {书名} 前{X}章数据推断

主题线:
  - 名称: XX线
    类型: 主线/支线/设定线
    跨卷范围: 卷1-卷3
    承载L2: [L2-001, L2-005, L2-012]
    演化: 触发→推进→高潮→收束
    置信度: 候选/确认

  - 名称: YY线
    类型: 支线
    跨卷范围: 卷1-卷2
    承载L2: [L2-002, L2-007]
    演化: 触发→推进→复杂化
    置信度: 候选
```

**字段说明：**
- **名称**：主题线名称（简洁，如「武道修炼线」「复仇线」）
- **类型**：主线/支线/设定线/背景线/关系线/成长线/反派线
- **跨卷范围**：该主题线跨越的卷范围（如「卷1-卷3」）
- **承载L2**：该主题线承载的 L2 单元卡编号列表
- **演化**：极简描述演化阶段（触发→推进→高潮→收束，标注已有阶段）
- **置信度**：候选（前N章）/ 确认（全书）

> **❗ 格式约束**：本文件使用 YAML 极简格式，不产出独立卡片。每条主题线最多 6 个字段，不展开细节。
