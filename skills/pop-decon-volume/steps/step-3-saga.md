# Step 3：L4 全书事件识别

> **方向**：从 L3 剧情线中识别跨线交汇的全局性事件，产出 L4 全书事件卡。
> **级别覆盖**：前N章（候选）/ 全书（确认）

---

## 数据来源

- **Step 2 产出的 L3 剧情线卡**（`设计/L3剧情线/L3-*.md`）— 唯一数据源
- L3 线卡中的「与其他线的咬合」表 → 跨线交汇点识别
- L3 线卡中的「阶段推进」表 → 交汇点的演化阶段

> **❗ 核心约束**：L4 全书事件从 L3 剧情线的跨线交汇中识别，不从 L2 单元或原子事件直接判断。

## L4 全书事件识别信号

| 信号 | 判断方式 | 示例 |
|:-----|:---------|:-----|
| **多线交汇** | 2 条以上 L3 线在同一章节范围内交汇 | 武道修炼线 + 演武令金手指线 + 穿越者心态线在 ch010 交汇（突破+复仇+蜕变） |
| **全局质变** | 交汇引发人物命运/世界格局/主题表达的质变 | ch010 复仇后通缉 → 主角从"新手村"进入"逃亡期" → 全局质变 |
| **演化阶段** | 事件有明确的演化过程（非一次性事件） | 触发→酝酿→爆发→余波→新均衡，至少 3 阶段 |
| **跨卷影响** | 事件的影响跨越原书的多个物理卷 | "穿越者迷失"主题从 ch001 延续到 ch098+ → 全书级事件 |

## 置信度机制

### 前 N 章拆解（候选识别）

当只读到全书的一部分时，L4 全书事件只能标注为「候选」：

| 置信度 | 条件 | 标注 |
|:-------|:-----|:-----|
| **低** | 仅有 1 条 L3 线触及此事件，交汇证据不足 | `候选（置信度：低）` |
| **中** | 有 2 条 L3 线交汇，但演化阶段不完整 | `候选（置信度：中）` |
| **高** | 有 2+ 条 L3 线交汇，演化阶段≥3段，但未看到全书终态 | `候选（置信度：高）` |

### 全书拆解（确认）

当读到全书终态时，将候选升级为确认：
- 演化阶段完整（触发→酝酿→爆发→余波→新均衡）
- 组成线全部确认（所有 L3 线的交汇点已验证）
- 全局影响已观测到终态

标注为：`确认`

> **❗ 红线**：前 N 章拆解时，L4 全书事件必须标注「候选（置信度：低/中/高）」，不得标注为「确认」。

## 识别流程

### 1. 扫描 L3 线卡的「与其他线的咬合」表

收集所有 L3 线卡中的跨线咬合关系：
```
L3-001（武道修炼线）咬合：
  - 与 L3-002（演武令线）：双向驱动（金手指加速修炼，修炼验证金手指）
  - 与 L3-004（穿越者心态线）：双向驱动（修炼膨胀催生心态危机，心态蜕变改变修炼策略）

L3-003（仇怨线）咬合：
  - 与 L3-004（穿越者心态线）：因果链（复仇导致通缉，通缉触发反思）
```

### 2. 识别交汇集群

将咬合关系聚类为交汇集群：
```
交汇集群 A（ch001-ch011）：
  组成线：L3-001 + L3-002 + L3-003 + L3-004
  交汇点：ch010（突破+复仇+蜕变同时发生）
  → 候选 L4 事件：穿越者新手村弧线
```

### 3. 验证演化阶段

检查每个候选 L4 事件是否有完整的演化阶段：
- 触发：何时开始？
- 酝酿：如何积累？
- 爆发：何时交汇？
- 余波：后果是什么？
- 新均衡：达到什么新状态？

### 4. 标注置信度

根据前 N 章的覆盖程度，标注候选/确认。

### 5. 填写事件卡

按 `templates/L4-全书事件.tpl.md` 填写每个 L4 全书事件卡。

## 产出

| 文件 | 内容 |
|:-----|:-----|
| `设计/L4全书事件/L4-001-{事件名}.md` | 按 `templates/L4-全书事件.tpl.md` 填写 |
| `设计/L4全书事件/L4-002-{事件名}.md` | ...（覆盖全部已识别全书事件） |

每个 L4 全书事件卡必须包含：
- **演化阶段**：阶段/卷/表现形态/承载线（L3编号）/核心事件
- **组成线**：线（L3编号）/阶段/功能/与其他线的咬合
- **全局影响**：人物命运/世界格局/主题表达

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | **前N章不得将候选标注为确认** — 必须标注「候选（置信度：低/中/高）」，全书拆解时才可标注「确认」 |
| 2 | **L4 组成线必须引用 L3 线卡** — 组成线表中的每条线必须标注 L3 编号，不得跳过 L3 直接引用 L2 或事件 |
| 3 | **不从 L2 或事件直接判断 L4** — L4 从 L3 跨线交汇中识别，不从 L2 单元或原子事件直接判断 |
| 4 | **不编造全书事件** — 事件来自 L3 线卡的咬合关系聚类，不是 agent 自拟 |
| 5 | **演化阶段不完整时降级** — 如果演化阶段不足 3 段，降级为 L3 线内事件，不作为 L4 全书事件 |
