# Step 3.5：提取章级剧情单元

> **方向**：从 Step 2（幕边界）+ Step 3（剧情线）的产出中，识别并提取 2-5 章的封闭叙事单元。
> **定位**：拆碎为可复用的事件链骨架——不是卷级宏大叙事，是章级的独立剧情弧线。
> **产出路径**：`写作资产/剧情单元/{单元名}.md`

## 什么是章级剧情单元

一个封闭的 2-5 章叙事弧，有独立的起承转合、因果闭环、情绪曲线。可以脱离卷/幕上下文被其他书独立复用。

**反面例子（不是单元）**：
- 单章高潮（没有完整的起承转合，只是弧线的一部分）
- 整卷主线（太大，复用=照搬原文）
- "主角升级了"（只是一个事件，不是一个弧）

**正面例子（是单元）**：
- 无间道排查内鬼（排查→自证→被出卖→重生）：ch001-004，4章
- 20年因果闭环（三层认出→截胡→世界升维→复仇）：ch013-020，8章核心段
- 卧底温水煮青蛙（信任建立→突破→暴露→对决）：ch023-028，6章

## 识别规则

扫描幕纲的章级切片，寻找以下信号：

| 信号 | 说明 |
|:-----|:-----|
| 情绪落差 ≥ 4个点 | 从紧张值5以下拉到9以上再回落（说明有一个完整的起-高潮-落） |
| 因果闭环 | 有明确的"设伏→回收"配对（可以从 chekhov-tracker 快速定位） |
| 地点/角色切换 | 进入新地点+退出该地点的完整弧 |
| 冲突升级三阶 | 事件链中冲突层级发生两次以上跃迁（如：猜疑→对抗→生死） |

## 产出格式

每个单元一个文件。格式固定：

```markdown
---
book: {书名}
author: {作者}
unit_level: {章级/幕级}
chapters: {chXXX-chYYY}
pattern: {模式关键词}
joy_points: {元爽点}
---

# {单元名}

> {一句话。字数范围+可嵌入场景}

## 事件链

```
→ {事件1}（{标注情绪/功能}）
→ {事件2}（{标注}）
→ ...
→ {终局事件}（{标注}）
```

## 结构解析

**核心设计模式**：{一个命名——要能在其他书里搜索到}

{因果链}：
```
{事件A} → {事件B} → {事件C} → {结果}
```

{反转/转折点}：
| # | 位置 | 转什么 | 功能 |
|:-:|:-----|:-------|:-----|
| 1 | {章号} | {转了什么} | {结构作用} |

## 金句

> "{原文金句}" — {谁说的}，{章号}

## 复用要点

- {要点1}
- {要点2}
- {替换建议}
```

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | **禁止原文叙事** — 事件链每行是结构描述（"内鬼提议全体自裁以自证"），不是原文叙事（"王平站起来说"） |
| 2 | **禁止写作方法论** — 不写"这个单元好在哪/为什么有效/读者感受"。只写结构 |
| 3 | **事件链来源** — 每行事件必须能在设计包的章级切片中找到对应章节证据 |
| 4 | **2-8章范围** — 单元过大=不适合独立复用，退回缩小范围 |
| 5 | **至少一个因果闭环或反转** — 只有直线推进没有结构设计的单元不入库 |

## 产出

`写作资产/剧情单元/{单元名}.md` — 每个已识别单元一个文件
