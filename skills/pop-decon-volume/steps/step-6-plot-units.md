# Step 6：提取剧情线（library 回流）

> **前置**：Step 1-5 全部完成（卷边界×幕边界×剧情线×契诃夫枪×套路聚合均已产出）
> **方向**：从完整的卷幕分析中，提取可迁移的剧情线。卷/幕/章级切片只作为证据，不进入公共 library 主资产。
> **产出路径**：`写作资产/剧情线/{线型}-{编号}-{名称}.md`

## 为什么不再提取卷/幕剧情单元

公共 `pop-trope-library/剧情库` 服务写书侧，而不是保存原书切片。

| 旧模型 | 问题 | 新模型 |
|:-------|:-----|:-------|
| 章级单元 | 太贴原文桥段，容易变成事件流水账 | 作为证据，不入公共剧情库 |
| 幕级单元 | 会把原书幕节奏带进新书 | 只用于定位剧情线阶段 |
| 卷级单元 | 容易混合多线，变成整卷照搬 | 拆成单条剧情线 |

> 核心原则：一条剧情线 = 一个入库对象。

## 识别规则

### 剧情线识别信号

| 信号 | 说明 |
|:-----|:-----|
| 有承压者/行动者 | 这条线围绕谁被逼迫、诱惑、误导或推动 |
| 有持续压力源 | 不是一次性敌人，而是制度、反派、资源缺口、关系债务或认知缺口持续施压 |
| 有阶段变化 | 触发、推进、复杂化、反转/代价、高潮、收束至少 4 段清晰 |
| 有叙事债务 | 前文递送的信息/关系/规则/道具/选择，后文必须回收 |
| 能脱离卷/幕复用 | 不依赖原书固定章数和幕名，也能给新书 plot 提供结构参考 |

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | **禁止按卷/幕入公共库** — 卷/幕是证据容器，不是写书模板 |
| 2 | **禁止多线混合** — 一条主线/支线/设定线/关系线单独成文 |
| 3 | **禁止照搬原书篇幅** — 章数、幕数、卷长必须写入“不可迁移部分” |
| 4 | **缺核心驱动力不入库** — 没有压力源和失败代价的线不能服务写书 |
| 5 | **缺叙事债务不入库** — 没有递送与回收，说明它不是长期剧情线 |
