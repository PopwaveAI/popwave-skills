# Step 6：提取剧情单元（分级）

> **前置**：Step 1-5 全部完成（卷边界×幕边界×剧情线×契诃夫枪×套路聚合均已产出）
> **方向**：从完整的卷幕分析中，按量级识别并提取封闭叙事单元——章级(2-5章)、幕级(5-15章)、卷级(15+章)
> **产出路径**：`写作资产/剧情单元/{单元名}.md`

## 剧情单元分级

| 量级 | 章数 | 识别源 | 复用场景 | 例子 |
|:-----|:----:|:-------|:---------|:-----|
| **章级** | 2-5章 | 幕纲章级切片（情绪曲线陡变段） | 小场景嵌入：排查内鬼/信任建立/单次反杀 | 无间道自证→灭口段（ch001-004） |
| **幕级** | 5-15章 | 幕边界（一个完整的幕） | 中型结构复用：完整的因果闭环/复仇弧/卧底弧 | 三层认出→截胡→世界升维→复仇（ch013-020） |
| **卷级** | 15章+ | **Step 3 剧情线的单条主线**（M1世界危机/M2主角成长/M3主角行动）中，形成独立元叙事的完整弧线 | 大型结构参考：一条主线贯穿全卷的完整叙事模式 | M1世界危机线（从内鬼暴露→到仙门收割体系揭示）/ M2主角成长线（从弃子→捕快→卧底→尸解→器灵的身份转变） |

> ⚠️ 卷级单元不提取"整卷所有线放在一起"——那是照搬。正确的卷级单元从 Step 3 产出的**单条主线**中提取：如果 M1（世界危机线）/ M2（主角成长线）/ M3（主角行动线）中某一条形成了独立完整的叙事弧，就单独提取为卷级单元。一条主线=一个单元。

## 识别规则

### 章级单元识别信号（在幕纲章级切片中扫描）

| 信号 | 说明 |
|:-----|:-----|
| 情绪落差 ≥ 4个点 | 从紧张值5以下拉到9以上再回落（完整的起-高潮-落） |
| 因果闭环 | 有明确的"设伏→回收"配对（从 chekhov-tracker 快速定位） |
| 地点/角色切换 | 进入新地点+退出该地点的完整弧 |
| 冲突升级三阶 | 事件链中冲突层级发生两次以上跃迁（猜疑→对抗→生死） |

### 幕级单元识别信号（在幕边界中扫描）

| 信号 | 说明 |
|:-----|:-----|
| 完整的幕 = 一个单元 | 该幕本身就是一个封闭的叙事弧，有独立的主角状态变化 |
| 跨章因果链 | 幕内的多个事件形成完整的因果闭环，不是零散推进 |
| 可脱离前后幕理解 | 不看前后幕，只看这一幕也能看懂完整的起承转合 |

### 卷级单元识别信号（从Step 3剧情的单条主线中扫描）

| 信号 | 说明 |
|:-----|:-----|
| 单条主线形成独立元叙事 | 从 M1/M2/M3 中任选一条，看它是否有独立的"起点→质变→终点"弧线。不是"主线做了很多事"——是"这条线讲了一个完整的故事" |
| 该主线可脱离其他两条线理解 | 不看另外两条线，只看这条线也能看懂一个完整的故事结构 |
| 主线内有明确的因果链和反转 | 不是零散事件推进——事件之间有因果逻辑，有至少一个反转或质变节点 |

## 产出格式

每个单元一个文件，格式固定。量级不同但格式相同：

```markdown
---
book: {书名}
author: {作者}
unit_level: {章级/幕级/卷级}
chapters: {chXXX-chYYY}（约{N}字）
pattern: {模式关键词——用箭头串联，如"自证→交换→被出卖灭口"}
joy_points: {元爽点}
---

# {单元名}

> {一句话。字数范围+可嵌入场景}

## 事件链

```
→ {事件1}（{标注情绪/功能}）
→ {事件2}（{标注}）
→ ...
→ {终局事件}（{标注}）
```

## 结构解析

**核心设计模式**：{一个命名——要能在其他书里搜索到}

{因果链}：
```
{事件A} → {事件B} → {事件C} → {结果}
```

{反转/转折点}：
| # | 位置 | 转什么 | 功能 |
|:-:|:-----|:-------|:-----|
| 1 | {章号} | {转了什么} | {结构作用} |

## 金句

> "{原文金句}" — {谁说的}，{章号}

## 复用要点

- {要点1}
- {要点2}
- {替换建议}
```

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | **禁止原文叙事** — 事件链每行是结构描述（"内鬼提议全体自裁以自证"），不是原文叙事（"王平站起来说"） |
| 2 | **禁止写作方法论** — 不写"这个单元好在哪/为什么有效/读者感受"。只写结构 |
| 3 | **事件链来源可追溯** — 每行事件必须能在幕纲章级切片中找到对应章节证据 |
| 4 | **章级单元：2-8章** — 单章不是单元。超过8章拆成更小单元或归入幕级 |
| 5 | **至少一个因果闭环或反转** — 只有直线推进没有结构设计的单元不入库 |
| 6 | **幕级单元优先于卷级** — 幕级可独立复用。卷级只提取从Step 3单条主线中形成独立元叙事的弧线，不提取整卷多线混合 |

## 产出

`写作资产/剧情单元/{单元名}.md` — 每个已识别单元一个文件。一卷通常产出 3-8 个章级 + 1-3 个幕级 + 0-3 个卷级（对应 M1/M2/M3 三条主线中形成独立元叙事的条数）。
