# 数值体系反拆方法论 · v1.0

> 归属：pop-decon Phase 3 的扩展维度
> 用途：从拆书数据中反向破译作者的"防力量体系写崩"工程
> 核心理念：数值体系 ≠ 战斗数值。它是全书的**社会阶层数学模型**。

---

## 冰山定位

```
战斗数值（水面之上1/8）      combat_capability.yaml、属性、HP、伤害
   │
宏观建模（水面之下7/8）      静态金字塔 + 动态升级表
```

已有的 `combat_capability.yaml` 只提纯了"水面之上的数字"，但作者真正用来控崩的是"水面之下的数学建模"——这是反拆要去破译的。

---

## 一、静态金字塔（Static Pyramid）

### 定义
把全书的力量层级建成一个**社会金字塔**，每层回答四个维度：

| 维度 | 反拆问题 | 原文证据来源 |
|:-----|:---------|:------------|
| **数量** | 这一层在这个国家/城市有多少人？| 军队规模、组织成员数、NPC密度 |
| **社会地位** | 这一层的人在城里是什么身份？能做什么不能做什么？| 社会互动、NPC对话、身份描述 |
| **生产力/破坏力** | 这一层的人能生产多少价值？能造成多大破坏？| 经济数据、战斗场景 |
| **战斗张力** | 主角和这一层的对手打起来，是什么体验？| 实际战斗描写 |

### 反拆步骤

**Step 1：从设计包和 L2 单元卡中提取"层级分界点"**
- 原文中是否有明确的阶级分界描述？（如 ch080"进阶=一阶→5级以上=二阶→10级以上=三阶→15级以上=高阶"）
- 每个层级的典型代表角色是谁？
- 主角在什么阶段能打过什么层级的对手？

**Step 2：逐层填充数据**
- 从 combat_capability.yaml 提取 HP/属性范围
- 从 L1-05-势力格局 提取各城的军力层级分布
- 从各设计包中的"城市人口/军队规模"描述推算人口密度
- 从 NPC 的社会反应（"见到传奇惊慌"/"对二阶习以为常"）倒推社会地位

**Step 3：识别"不可跨越之门禁"**
- 哪些门是硬门槛？（如"20属性=超凡"——不到就绝对不行）
- 哪些门是软门槛？（如"高阶不冒险"——是社会规约不是硬规则）
- 主角是否实际跨越过这些门？代价是什么？

**Step 4：画出金字塔图**
```
           │  层名   │ 数量级  │ HP范围  │ 社会地位 │ 主角关系
    ───────┼─────────┼────────┼────────┼─────────┼─────────
    顶层   │ 魔神    │ <5    │ 800+   │ 位面领主 │ 不可力敌
           │ 半神    │ <10   │ —     │ 国之上   │ 需传奇团
           │ 传奇    │ 0-2/城│ 500-1500│ 守护者  │ 旁观
    中高层 │ 高阶    │ 1-5/城│ 300-500│ 统治层   │ 智取/借势
           │ 三阶    │~10/城 │ 150-300│ 城市精英 │ 势均力敌
    中低层 │ 二阶    │~50/城 │ 60-150 │ 老兵/中层│ 初期对手
           │ 一阶    │~200/城│ 20-60  │ 底层兵/学徒│ 经验包
    底层   │ 凡人    │~万/城 │ 5-15   │ 无权力   │ 秒杀
```

## 二、动态升级表（Dynamic Escalation）

### 定义
主角提升能级时，他的"博弈层级/接触圈子/冲突规模"如何同步升级，同时不破坏下级金字塔。

### 核心机制：套盒子换地图

```
每个"盒子" = 一张地图 + 一套静态金字塔

盒1: 新手村      盒2: 区域地图      盒3: 王国政治      盒4: 神战位面
┌──────┐         ┌──────────┐       ┌──────────┐       ┌──────────┐
│ 传奇  │         │ 传奇     │       │ 传奇     │       │ 魔神     │
│ 高阶  │         │ 高阶     │       │ 高阶     │       │ 半神     │
│ 三阶  │         │ 三阶     │       │ 三阶     │       │ 传奇     │
│ 二阶←─┤         │ 二阶←─┤       │ 二阶     │       │ 高阶     │
│ 一阶←─│主角     │ 一阶     │       │ 一阶     │       │ ...      │
│ 凡人  │         │ 凡人     │       │ 凡人     │       │          │
└──────┘         └──────────┘       └──────────┘       └──────────┘
```

### 反拆步骤

**Step 1：标记主角的能级跃迁节点**
在时间线上标出：
- 主角的等级/能力质变点
- 同时期的地图变化
- 同时期的对手层级跃迁

**Step 2：识别"换地图的触发机制"**
每次换地图的**真实原因**是什么？
- 旧地图被物理摧毁了？ → 被动换图
- 主角太强需要新挑战？ → 主动换图（少见）
- 卷入了更高层的事件？ → 被动卷入（常见）

**Step 3：分析"新地图的重启机制"**
主角进入新地图后，哪些重置了？哪些保留了？

| 维度 | 重置程度 | 原因 |
|:-----|:---------|:-----|
| 社会地位 | 完全归零 | 新地图有新的社交体系 |
| 人际关系 | 部分重置 | 旧人脉在新地图无效 |
| 经济资产 | 轻度重置 | 钱和装备仍然有用 |
| 战斗能力 | 不变 | 这是主角的硬实力 |
| 知识/信息 | 不变 | 重生/穿越优势 |

**Step 4：提取"防崩铁律"**
从主角的整个升级路径中归纳出作者遵循的隐形规则。常见铁律：
1. **高阶不冒犯** — 顶层战力不在日常场景出现
2. **越级看代价** — 超过能级+2的胜利必须支付代价
3. **换图即重置** — 新地图主角从底层开始但保留战力
4. **传奇不对决主角** — 传奇的对手永远是另一个传奇
5. **卷末卸能** — 每卷末尾主角付出代价，保证下卷起点可控

## 三、输出物规范

### 文件结构
```
设计/数值体系反拆/
├── 卷01-静态金字塔.md      # 金字塔图 + 逐层分析
├── 卷01-动态升级表.md      # 6阶段跃迁 + 套盒子分析
└── 全卷-数值规律汇总.md    # 跨卷规律（后续卷追加后合并）
```

### 内容规范
- **必须基于原文证据**：每个数量估计、每个社会地位描述都要标注 chXX 出处
- **不编造数字**：如果原文没有准确人口数，用「~万/城」「0-2/城」的模糊描述并标注"估计值"
- **区分"原文明确"和"反推估计"**：明确标注哪些是直接数据，哪些是从行为倒推的

## 四、与已有 Pipeline 的关系

| Pipeline Phase | 已有产出 | 数值体系反拆的消费方式 |
|:--------------|:---------|:---------------------|
| Phase 1 设计包 | 每章事件+对白+数据 | 提取人口/战力/社会互动数据 |
| Phase 2 L2单元卡/L3剧情线 | L2-*.md, L3-*.md | 提供阶段划分时间线 |
| Phase 3 L1设定 | L1-02力量体系/combat_capability | **反拆原材料**——但需重新组织为金字塔视角 |
| Phase 4 创意溯源 | 跨域参考索引 | 为"这个数值设计象哪个游戏/作品"提供线索 |

**当前缺口**：Phase 3 的"数值"目前只走 pop-writer-world 的 forward-construction 模式（战斗数值榜）。需要本方法论作为扩展维度，补上 reverse-engineering 视角。
