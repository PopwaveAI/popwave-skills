# 小量级拆书 Phase 2 策略（< 100 章）

> 2026-06-17 | 实战验证于 《让魔门再次伟大》（37章）

## 问题

pop-decon 默认规则建议 Phase 2（L2单元卡/L3剧情线/L4全书事件）跳过 < 100 章的拆书。但实践表明，30-80章的完整卷同样值得跑 Phase 2——尤其当用户明确要求时。

## 并行委托模式

### 分幕策略

1. 先从 `卷1-架构.md`（或 Phase 1 数据）中识别幕边界
2. 按幕分配子 agent：
   - 幕1：ch001-ch008（8章）→ ~130s，39KB
   - 幕2：ch009-ch022（14章）→ ~177s，51KB
   - 幕3：ch023-ch032（10章）→ ~177s，46KB
   - 幕4：ch033-ch037（5章）→ ~125s，33KB
3. 全部并行执行，总耗时 ≈ 最慢子 agent 耗时（~3min）

### 子 agent 上下文模板

每个子 agent 接收以下信息：

```
项目路径：D:\workspace\让魔门再次伟大
幕范围：chXXX-chYYY（N章）
幕名：名称
卷：卷1（基于前X章数据推断）

设计包路径（绝对路径）：
D:\workspace\...\写作资产\设计包v3\chXXX-设计包.md
...

模板格式（严格给出首行范例）：
# 幕1 · 名称（ch001-ch008）

scope声明：
> ⚠️ **基于前X章数据推断**

必须包含的区块：
1. 幕级元数据（戏剧功能/状态变化/冲突/副线/量化数据）
2. 章级切片（每章：场景/剧情线推进表/角色/情绪目标/爽点/钩子/契诃夫枪）
3. 节奏自检（活跃度/情绪节奏/爽点密度/综合评级）
```

### 关键注意事项

1. **设计包格式兼容**：ch001-ch022 使用新格式（`# 设计包 — chXXX「标题」`，含事件链表格），ch023+ 使用旧格式（`# 第XX章「标题」— 设计包 v3`，含节拍序列）。子 agent 必须同时支持两种格式的解析。
2. **绝对路径**：子 agent 的工作目录可能与项目不同，必须传绝对路径。
3. **格式锁定**：在 context 中明确给出首行范例，防止不同子 agent 产出命名/格式不一致。
4. **契诃夫枪聚合**：每幕子 agent 提取契诃夫枪后，需要单独一个子 agent 聚合全量扫描设计包生成 chekhov-tracker.md。

### 产出验证

| 检查项 | 标准 |
|:-------|:-----|
| 首行格式 | `# 幕X · 名称（chXXX-chYYY）` |
| scope声明 | 第二行，`⚠️ 基于前X章数据推断` |
| 幕覆盖 | 全部幕文件章数之和 = 总章数 |
| 章级切片 | 每章都有完整的剧情线推进表 |
| 契诃夫枪 | 每枪有设伏章号和回收章号证据 |
