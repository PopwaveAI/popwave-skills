## 步骤 1：识别卷边界

**数据来源**：`_temp/chapter-index.json` → `chapters[].tags` + `chapters[].chapter` + `chapters[].charCount`

### 卷边界判断信号

| 信号 | 判断方式 | 示例 |
|:-----|:---------|:-----|
| **时空跳跃** | 连续 N 章后首句出现新地名（不在之前 20 章中） | ch20 "商队向着北方的白马城出发" → 卷结束 |
| **任务完结** | 连续 worldbuilding 标签密集章后，出现情绪低谷 + 过渡章 | 大事件后休息章 = 卷末 |
| **主角升级跨越** | 等级/阶位从一阶进入二阶 → 可能为新卷起点 | ch17 盗贼进阶二阶 |
| **地图扩展** | chapter-index 中搜索后面章节的 places 词频变化 | 新地名频率突增 |

### 卷边界标注格式

```
卷1: ch1 ～ ch20（琥珀城篇。起点：贫民区小屋 / 终点：准备离开琥珀城）
  证据：ch20《神之子》暗示世界观扩展；ch21 开始出现商队/冒险者公会
卷2: ch21 ～ ch100（荒野商路篇。起点：加入商队 / 终点：白马城区域）
  证据：ch22 冒险者公会；ch40 金溪镇；ch42 白马城；ch53 弯刀大师
...
```

> **小范围警告**：如果 ch1-100 不足以支撑卷级判断（如只有 1 卷），标注"基于前 100 章推断，Lv3 全书将修正"。

## 步骤 2：识别幕边界（卷内）

**数据来源**：卷内 chapter-index 标签的密度变化

| 幕类型 | 标签特征 |
|:-------|:---------|
| 开局幕 | worldbuilding 标签密集 → 设定介绍 |
| 推进幕 | battle + worldbuilding 交替 → 冒险探索 |
| 高潮幕 | battle 标签连续密集 → 连续战斗 |
| 收尾幕 | economy 标签增多 + 章节字数降低 → 交易/整理/过渡 |

### Canvas 矩阵反推规则

每个幕的 canvas，逐章反推：
```
主线1（世界危机）：worldbuilding 标签密集章 → 标记为世界线推进章
主线2（主角成长）：battle 标签章且字数上升 → 标记为成长线推进章  
主线3（主角行动）：首句中的主动动词（"索伦决定/前去/潜入"）
设线负载：连续 non-battle 章中新设定披露次数
```

> **ch 级 Canvas 字段**（act-01.yaml）：每章填写 payoff_summary / 设定线 / 设线负载 / 主线1/2/3推进 / 支线1/2推进 / emotional_goal / end_hook
> 推进内容来自 Phase 1 事实骨架的事件摘要

## ❌ 质量红线

| # | 标准 |
|:-:|:-----|
| ❌1 | **卷边界有原文证据** — 每个卷至少有一个时空跳跃/任务完结信号 |
| ❌2 | **Canvas 矩阵无空白列** — 每章至少有一条主线在推进。如有空白 → 标注「过渡章无主线推进」 |
| ❌3 | **节奏自检通过** — 单条线无连续 4 章以上留白（rhythm_check） |
| ❌4 | **契诃夫枪每条标注设伏 chXX** — 不得凭空编造未在原文中看到的伏笔 |
| ❌5 | **Canvas 中 payoff 标注来自原文事件** — 不得编造"payoff_level: 大"等无法在原文中找到对应高潮的评价 |

## 工作纪律

```
1. 卷边界判断 → 引证 chapter-index 中的地名变更/标签类型突变 → 标注 chXX
2. Canvas 填写 → 每章从 Phase 1 事件摘要中提取至少一个推进 → 标注 chXX  
3. 契诃夫枪 → 从事实骨架中采集"首次出现但未回收"的物品/角色/设定伏笔 → 标注设伏 chXX
4. 无法判断 → 留空，标注"（前 N 章数据不足以判断）"
```
