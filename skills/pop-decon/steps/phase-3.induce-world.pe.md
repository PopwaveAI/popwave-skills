# Phase 3：归纳世界观

> **方向**：逆 `world` + `creative`（设定层）。从累积的设定信息披露中归纳出 L1 六件套 + 数值体系 + 世界宪法。
> **级别覆盖**：Lv2(ch1-100) / Lv3(全书)
> **前置条件**：Phase 1 事实提取已完成 + Phase 2 卷幕聚类已完成
> **Lv1 不执行此 Phase**（前 20 章不够归纳世界观）

## 核心理念：归纳 vs 编造

> **写作端 world**：从故事引擎出发 → 设计 L1 设定 → 向 plot 交付世界观。"我想要一个什么样的世界？"
> **拆书端 Phase 3**：从原文累积的设定信息披露出发 → 按分类学归纳 → 结构化表达。"这本书实际是一个什么样的世界？"

每件套的填写，是在回答：**"走到第 N 章为止，我们积累了什么关于这个维度的原文证据？"**

## 步骤详解

### Step 1：地理蓝图 → `L1-01-世界蓝图.md`

**数据来源**：`world-data.json#geography.entries[]` — 所有地名及其出现章节

**归纳逻辑**：
1. 将 geography 条目按出现章节排序 → 形成地理扩展时间线
2. 按出现频次分组 → 核心区域(≥10次) / 常规区域(2-9次) / 提及区域(1次)
3. 从 Phase 2 的卷地图扩展顺序中提炼"叙事空间节奏"

**产出**：地理总览 / 位面规则 / 核心法则 / 力量来源 / 世界基调

### Step 2：力量体系 → `L1-02-力量体系.md`

**数据来源**：`world-data.json#class.entries[]` + `#magic.entries[]`

**归纳逻辑**：
1. 从 class 条目中提取完整的等级链（一阶→二阶→…→传奇）— **每个环节必须有 chXX 原文**
2. 从 class/magic 条目中提取主要职业路径（如盗贼/战士/法师/牧师等）— 只列原文中确有的
3. 从 class 条目中提取突破条件（如"血脉纯化仪式"ch17）
4. 从角色卡的成长路径确认力量天花板
5. **禁止行为**：原文没有的境界名 = 不准编造。原文没有的职业名 = 不列入

### Step 3：历史与驱动力 → `L1-03-历史与驱动力.md`

**数据来源**：`world-data.json#deity.entries[]` + `#faction.entries[]` + Phase 2 主线

**归纳逻辑**：
1. 从 deity 条目提取"动荡之年"等纪元概念 → 标记首次出现 chXX
2. 从 faction 条目和 Phase 2 主线提取核心矛盾（如"众神陨落→凡人有机会封神"）
3. 前提大事件按时间线重构

### Step 4-6：物种/势力/资源 → `L1-04/05/06`

从对应 world-data.json 分类中提取，按模板填写。

### Step 7：世界宪法

**来自原文叙事规律**，而非 world-data：
- 基调边界：主角做事下限是什么（如 ch4 杀人不眨眼→允许残忍）
- 世界底线：规则允许的最大极限
- 视角/节奏规律：wolrdbuilding/battle 章分布规律（从 chapter-index tags 统计得出）

### Step 8：数值体系

**数据来源**：world-data.json#class.entries[] + 角色卡等级路径

段位映射必须来自原文面板数据（如 ch2 "10级平民/1级盗贼【一阶】"、ch17 "5级盗贼【二阶】"）

## ❌ 质量红线

| # | 标准 |
|:-:|:-----|
| ❌1 | **每件套的数据必须能在对应 JSON 分类中找到** — L1-02 力量体系中的境界链 = world-data#class 中确有的条目 |
| ❌2 | **不准编造体系名** — "灵气九转"、"魔法十环"等原文没有的命名 → 严格退回 |
| ❌3 | **归纳 ≠ 编造** — 从数据中提取模式是归纳。凭记忆补充数据是编造。区分标准：JSON 中能找到条目 = 归纳；找不到 = 编造 |
| ❌4 | **缺失标注** — JSON 某分类条目过少 → 产出中必须标注"（前 N 章此维度数据极少）" |
| ❌5 | **世界宪法中每条必须有叙事证据** — 不能写"主角不能背叛"而原文找不到这种约束 |

## 归纳纪律

```
归纳三步法：
1. 打开 world-data.json 对应分类的 entries[]
2. 浏览全部条目 → 识别高频模式和显著异常
3. 将模式写入模板 → 每个模式标注至少 2 个 chXX 出处

反例：
  ❌ "力量体系分为：凡人/超凡/破格/神话"（原文从未出现这些词）
  ✅ "力量体系分阶：一阶(1-5级) / 二阶(6-10级) / 传奇(16-20级)  ch2/ch17"（原文有面板数据）
```
