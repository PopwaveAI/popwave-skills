## 步骤 1：可回源性审计

**目标**：验证每个 Phase 产出的每个硬数据都有可追溯的数据来源。

| 检查项 | 对照规则 |
|:-------|:---------|
| 角色卡中的年龄/等级/职业 | → `_temp/baseline-data.json` 或 `_temp/world-data.json` 中是否存在 |
| L1 六件套中的体系名/势力名 | → `_temp/world-data.json` 对应分类中是否存在 |
| Canvas 中事件推进 | → `_temp/chapter-index.json` 对应章的事件摘要 |
| 故事引擎中核心假说 | → Phase 2 Canvas 主线 + Phase 3 世界宪法中是否有对应原文证据 |

**审计方法**：
```
1. 随机抽取 10 个硬数据（跨越不同 Phase 产出文件）
2. 每个硬数据 → 打开对应 JSON → 搜索是否存在
3. JSON 中存在但值不匹配 → 标注"数据偏离"
4. JSON 中不存在 → 标注"⚠️ 未经验证的断言"
5. JSON 中不存在的非硬数据（如"故事基调"）→ 标注"归纳自 {数据来源}"
```

---

## 步骤 2：跨 Phase 一致性检查

**管线上游 → 下游数据链检查**：

| 检查项 | Phase 1 | Phase 2 | 规则 |
|:-------|:--------|:--------|:-----|
| 角色等级 | 角色卡成长路径 | volume-01.md 起点快照 | 卷1起点状态必须与 ch1 角色卡状态一致 |
| 地理范围 | L1-01 世界蓝图 | 全书架构 卷地理范围 | 卷1范围中的所有地标必须在世界蓝图中存在 |
| 力量体系 | L1-02 力量体系 | combat_capability.yaml | 段位名/突破条件必须一致 |
| 核心矛盾 | L1-03 历史与驱动力 | Canvas 主线1 | 主线1（世界危机）的根源必须在历史驱动力中描述 |

**Lv3 追加检查（故事引擎 vs 前序产出）**：

| 检查项 | 前序 Phase | 故事引擎 | 规则 |
|:-------|:----------|:---------|:-----|
| 卷数 | Phase 2 全书架构 | 结构预期表 | 卷数一致 |
| 核心假说 | Phase 3 世界宪法 + Phase 2 Canvas | 一句概括 | 假说概括的冲突必须能在 Canvas 中找到对应高潮 |
| 主角边界 | Phase 1 角色卡设计目的 | 宪法段 | 偏差 ±1 条以内为合理归纳 |

**矛盾处理**：
- 偏差 → 标注严重程度（P0/P1/P2）
- P0 → 返回对应 Phase 修正
- P1 → 标注提醒但保留
- P2 → 仅记录在产出索引中
