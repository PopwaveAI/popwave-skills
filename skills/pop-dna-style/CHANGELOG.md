# CHANGELOG

## v1.1.0 (2026-07-20)

### step1新增类型权重采样

**根因**：项目c测试发现诡舍DNA（恐怖悬疑）均匀采样导致恐怖氛围丢失——恐怖小说的核心笔触集中在揭秘/暗杀/情感高潮3类场景，但均匀采样只各采2-3章，核心笔触样本权重被日常对话/过渡缓冲等边缘场景稀释。

**改动**：
- step1.md新增1.0节：识别参考书类型→确定采样偏重
- 类型→采样偏重表：恐怖悬疑（重采揭秘/暗杀/情感高潮）、战斗爽文（重采boss战/群战）、都市异能（重采日常对话/揭秘/暗杀）、仙侠养成（重采环境引入/过渡缓冲/揭秘）
- 核心场景采3-4章，边缘场景采1-2章，总采样量保持25-30章不变
- 采样清单追加偏重说明
- skill.json：v1.1.0

## v1.0.0 (2026-07-20)

### 新建skill：文风DNA提取

填补write skill笔触DNA标准插槽的供给方空白。基于R42测试验证（玄鉴仙族+夜无疆+深渊主宰交叉测试）确认DNA可插拔方案可行。

**新增：**
- 双档位设计：
  - A·笔触DNA（必做）：句式/感官/叙事距离/对话/情绪外化/段落呼吸/系统面板格式 → `涌现/文风锚定.md`，write消费
  - B·剧情DNA brief（可选）：爽点节奏/Boss梯度/主角成长曲线3维度brief → `涌现/剧情DNA-brief.md`，seed/plot消费
- 场景定向采样策略：9种场景类型×2-3章=25-30章，替代旧的"前10+中5+后5"固定时间分段
- v4模板强制：每个维度/场景卡 = 观察(1-2句) + 原文(≥500字) + 时间演变(1-2句)
- 部署到项目本地`涌现/`，不进library库（library库已优化掉）
- 4条红线：v4模板完整性/场景覆盖≥8/只学笔触不学内容/部署到项目本地

**和decon的边界：**
- decon = 重拆（8维度全拆+Beat Sheet+全书剧情白描），用于精拆一本书做立项圣经
- pop-dna-style = 轻采（笔触DNA必做+剧情brief可选），用于拿参考书当灵感源
- 两者读同一本书但粒度不同，不冲突

**设计依据：**
- R42原版"仅DNA"高分是假象（DNA含原文+同章剧情梗概双重叠），R42f公平测试（ch50/ch075排除采样范围）验证DNA可插拔真实有效
- write v8.0.0做减法移除内置风格倾向后，笔触层全靠DNA供给，pop-dna-style填补这个供给方
