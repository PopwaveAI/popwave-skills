# pop-dna-style · 文风DNA提取

> 从参考书采样 → 按pop-shared-dna v4模板提取笔触DNA → 部署到项目本地`涌现/文风锚定.md`
> v1.0.0：新建skill，填补write skill笔触DNA标准插槽的供给方空白

---

## 做什么

输入：参考书txt文件 + 用户可选档位（A笔触必做/B剧情brief可选）
输出：`涌现/文风锚定.md`（笔触DNA）+ `涌现/剧情DNA-brief.md`（如启用档位B）

**定位**：pop-dna-style是write skill笔触DNA标准插槽的供给方。write消费`涌现/文风锚定.md`驱动笔触层，pop-dna-style负责产出并部署这个文件。

**和decon的边界**：
- decon = 重拆（8维度全拆+Beat Sheet+全书剧情白描），用于精拆一本书做立项圣经
- pop-dna-style = 轻采（笔触DNA必做+剧情brief可选），用于拿参考书当灵感源
- 两者读同一本书但粒度不同，不冲突

## 双档位设计

| 档位 | 内容 | 产出 | 消费方 | 默认 |
|:--|:--|:--|:--|:--|
| A·笔触DNA（必做） | 句式/感官/叙事距离/对话/情绪外化/段落呼吸/系统面板格式 | `涌现/文风锚定.md` | write | 默认开启 |
| B·剧情DNA brief（可选） | 爽点节奏/Boss梯度/主角成长曲线（3维度brief） | `涌现/剧情DNA-brief.md` | seed/plot | 用户开关 |

**档位B的3维度**：
1. 爽点节奏：单元剧循环模式 + 爆发间隔 + 章末钩子类型
2. Boss梯度：4层递进结构 + 出场铺垫方式
3. 主角成长曲线：破局手段演变 + 压力源切换

每维度1段话观察 + 1-2条迁移指引，不写Beat Sheet，不做压力源6字段转化。

## 采样策略：场景定向（非时间分段）

**不再用"前10+中5+后5"的固定分段**，改为按场景类型定向采样：

| 场景类型 | 采样量 | 怎么找代表章 |
|:--|:--|:--|
| 日常对话 | 2-3章 | 章名含"叙旧/来访/闲谈"或非战斗非揭秘章 |
| 环境引入 | 2-3章 | 章名含"初到/抵达/新地"或新场景首出场 |
| 早期遭遇战 | 2-3章 | 前30章内的战斗章 |
| 中坚boss战 | 2-3章 | 中段章节，单挑boss，有招式名对抗 |
| 群战/军队战 | 2-3章 | 章名含"围杀/战场/军阵"或多对一 |
| 暗杀/背刺 | 2-3章 | 章名含"刺杀/潜行/暗影"或偷袭章 |
| 揭秘/心理戏 | 2-3章 | 章名含"真相/发现/回忆"或内心戏章 |
| 过渡缓冲 | 2-3章 | 战后章/清晨/赶路章 |
| 情感高潮 | 2-3章 | 章名含"告别/重逢/告白"或眼泪章 |

**采样流程**：
1. 解析全书章节列表（章号+章名）
2. 章名关键词匹配 → 初步分类
3. 对分类不确定的章，读前50行确认场景类型
4. 每种场景类型选2-3个代表章
5. 汇总采样清单（25-30章）

**DNA场景卡矩阵要求≥8个**（覆盖主要场景类型），详见step1。

## 红线

1. **笔触DNA必须按pop-shared-dna v4模板提取**——每个维度/场景卡 = 观察(1-2句) + 原文(≥500字) + 时间演变(1-2句)，三者不可缺一
2. **采样必须场景定向覆盖≥8种场景类型**——禁止只用"前10+中5+后5"时间分段，必须按场景类型采样
3. **笔触DNA只学笔触不学内容**——观察字段只描述句式/感官/叙事距离等写作风格特征，禁止描述剧情设计/战斗表现/角色思维等内容元素
4. **产出必须部署到项目本地`涌现/文风锚定.md`**——不进library库，直接部署到项目本地供write消费
