# pop-dna · 文风DNA蒸馏引擎 v3.0

> **v1 → v3 的根本变化**：v1 统计"平均句长 18-25 字"（症状），v2 放原文片段让 Agent 感受（浸泡）。v3 直接提取**作者在每个场景里做了什么选择**——给 Agent 的不是统计数字，是可执行的结构规则。

---

## ❌ 质量红线

```
□ 不统计平均句长、词频、感叹号密度等量化指标——那是症状，不是方法
□ 不放整段原文让 Agent "感受氛围"——Agent 不知道对齐什么
□ 每条规则必须回答三个问题：作者做了什么选择？原文证据？Agent 应该怎么做？
□ 必须区分骨架层（跨赛道通用）和肉身层（换赛道要换变量）
```

---

## 一、核心方法论

### 什么是 v3？

把一位作者的写作风格，从"只可意会"变成**一组按场景类型组织的可执行决策规则**。

```
作者的 20 章原文
     ↓ 不统计、不浸泡
提取: 每个场景类型下作者做了什么选择
     ↓ 配双文对照（如有跨作品对比）
产出: 4 段结构化的风格文件
  §1  作者方法论摘要（≤5 条·贯穿全场景）
  §2  场景型决策规则（描写/对话/决策/战斗/群像·每条配原文证据+可执行指令）
  §3  跨赛道便携性标注（骨架 vs 肉身·用变量代替硬码）
  §4  风格红线（本风格绝对不会做的事）
```

### 为什么这比 v1/v2 好

| 维度 | v1 统计 | v2 浸泡 | v3 提取 |
|------|------|------|------|
| 给 Agent 的产出 | "平均句长 18-25 字" | 一段原文 + "感受节奏" | "写外貌 = 材质×版型×配饰·三层叠加·零抒情" |
| Agent 执行方式 | 数字母句长 | 凭语感猜 | 知道结构——直接执行 |
| 跨赛道 | ❌ 仙侠句长和西幻不同 | ⚠️ 能但 Agent 不知道换什么 | ✅ 每条标注骨架/肉身 + 什么变量替换 |

## 二、执行流程

```
步骤0 · 取样（~20 章，前中后均匀覆盖）
  → 前 20%: 7 章 / 中段 40-60%: 7 章 / 后 20%: 6 章
  → 20 章全部进入分析

步骤1 · 逐场景扫描
  → 不按维度分——按场景类型分：外貌描写 / 城市群像 / 对话市井 / 决策推理 / 战斗 / 世界观植入
  → 在每个场景中问三个问题：
    ① 作者做了什么结构选择？
    ② 原文证据是什么？（摘具体句子）
    ③ 如果有跨作品对比——同一作者在不同题材的同一场景做了什么不同选择？
  → 区分"风格特征"和"内容特征"（角色名/设定专名 = 内容特征，排除）

步骤2 · 提取方法论摘要
  → 从所有场景规则中提取 3-5 条贯穿全场景的写作哲学
  → 每条附原文证据

步骤3 · 写出风格文件
  → 按 4 段结构产出 `styles/{书名}.md`
  → 每段场景规则 = 规则描述 + 原文证据（双文对照如有）+ 可执行指令

步骤4 · 跨赛道便携性标注
  → 每条规则标注 portable（骨架·永久保留）或 genre_specific（肉身·换变量）
  → 用变量代替硬码：{本项目参考书} / {本世界阶层标记物} / {目标时代底层口语}

步骤5 · 试写验证
  → 用产出的风格文件试写 300-500 字
  → 验证 Agent 是否能按结构规则执行
```

## 三、产出文件结构

`styles/{书名}.md` — 4 段：

### §1 作者方法论摘要

≤5 条贯穿全场景的写作哲学。每条一句话 + 一句原文证据。不列统计数字。

```markdown
### 1. 用物件清单代替结论陈述
→ "不要写'这里很繁华'——写青衿襕衫、圆领长袍、上襦下裙。读者自己组合出阶层地图。"
证据: 诡秘贝克兰德工装清单 ↔ 剑烛定江府衣冠清单 — 同一结构
```

### §2 场景型决策规则

按场景类型组织。每条规则含：规则描述 + 原文证据（最好有双文对照）+ 可执行指令。

```
### 场景：人物外貌描写
规则: 材质 × 版型 × 配饰 三层叠加 / 零抒情 / 零模糊形容词
文A: "黑色双排扣长礼服，同色丝绸半高礼帽，镶金黑色手杖"（诡秘·英伦）
文B: "双螺髻，葱白绣银边直领对襟短衫袄，鹅黄色轻薄罗裙"（剑烛·大圣）
可执行指令: 写人物外貌时，只写材质、版型、配饰三件事。不用"英俊""气质非凡"。

### 场景：角色决策推理
规则: 条件句前置 — "在 X 的前提下，Y 是更优选择"
文A: "在确认对方并无进一步敌意……的前提下，克莱恩理智克制住了符咒。"
文B: "在处境没具体了解的前提下，说'部分真话'是更好更优的选择。"
可执行指令: 写角色做重大决定时，先写前提条件（"在……的前提下"），再写决策。不写"凭直觉""心中一横"。
```

### §3 跨赛道便携性标注

```yaml
portable（跨赛道通用——骨架）:
  - 三层叠加描写          → 可用到任何文化的角色外貌
  - 条件句决策结构        → 可用到任何题材的决策场景
  - 物品清单代替结论      → 可用到任何世界的城市/群像描写

genre_specific（换赛道换变量——肉身）:
  - 参考书                → 变量: {本项目考据参考书}
    西幻: 《英国绅士手册》 / 仙侠: 《大明会典》
  - 阶层标记物            → 变量: {本世界阶层外观标记}
    西幻: 工装/礼帽/商贩 / 仙侠: 青衿/圆领袍/直裰
  - 市井口语              → 变量: {目标时代底层黑话}
    明清: "车船店脚牙" / 现代: 工地切口
```

### §4 风格红线

本风格绝对不会做的事。禁令式。每条 ≤1 行。

## 四、使用方式

### 提取风格 DNA

```
用户：帮我分析 [书名] 的文风，我需要 Agent 模仿它
pop：收到。请提供原文文件（≥10章）或作品链接。
→ 取样 ~20 章（前中后均匀）
→ 按场景类型扫描，提取结构规则（不统计词频/句长）
→ 产出 styles/{书名}.md + 试写验证 300-500 字
```

### 对比两本书/同一作者跨题材

```
用户：[作者] 的西幻和武侠在文风上有什么不同？
→ 产出一份双文对照的文件
→ 每条规则标注骨架公共性 vs 肉身差异
```

## 五、与 writer 的集成

pop-dna 产出的 `styles/{书名}.md` 直接供 `pop-novel-writer` §Step 2 Phase 1 消费：

```
writer Phase 1（风格锚定）：
  ① 读 design 块F 的场景类型 → 在 styles/ 文件 §2 中找到对应场景规则
  ② 执行该规则的可执行指令（不是感受氛围——是按结构写）
  ③ 每 2-3 个 beat 回读一次原文证据，检查质感是否对齐
```
