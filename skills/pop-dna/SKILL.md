# pop-dna · 文风DNA蒸馏引擎

> **让 Agent 学会某位作者/某本书的写作风格。**
>
> 灵感来源：
> - [richards199999/Thinking-Claude](https://github.com/richards199999/Thinking-Claude)（系统级 Prompt 工程，14.2k ⭐）
> - Style Mimic prompt（社区流行的"喂样本→分析→Profile→写作"方法）
> - auto-claude-writing-agent-pub（写作 Agent 工作流）
> - pop-novel-writer 三层文风DNA框架（事实骨架 × 叙事策略 × 哲学DNA）

---

## 一、概述

### 什么是文风DNA蒸馏？

就是把一位作者的写作风格，从"只可意会"变成"可结构化描述、可注入给 Agent"的一组资产：

```
作者的3-5篇原文
     ↓ 分析
结构化的风格DNA档案 → 可用于 Prompt 注入 / 微调数据 / Skill 封装
     ↓ 注入
Agent 产出带上该作者的风格质感
```

### 能力边界

| 维度 | ✅ 能做到 | ❌ 做不到 |
|------|---------|---------|
| 词汇偏好 | 复刻用词习惯、句式节奏、修辞风格 | 100%还原每个遣词造句 |
| 叙事姿态 | 复刻叙事者接入深度、信息释放方式 | 完全复制作者的思维过程 |
| 节奏模式 | 复刻铺垫/释放节律、段落结构 | 预测作者未来会写什么 |
| 情感质感 | 复刻情绪表达手段、留白习惯 | 拥有作者的情感经历 |

## 二、两条路线

本 Skill 提供两条路线，可根据使用场景选择：

### 🅰️ Prompt 路线（最实用、门槛最低）

**原理**：通过精心设计的 system prompt + 结构化风格档案，让 Agent 在对话中模拟目标风格。

**适用场景**：
- 想快速让 Chat 类 Agent 模仿某风格写作
- 没有 GPU / 不想做模型训练
- 需要灵活切换多种风格

**流程**：

```
用户提供目标作者的3-5篇原文样本（每篇≥500字）
     ↓
步骤1 · 文风DNA提取
  → Agent 逐篇分析：词汇 / 句式 / 节奏 / 修辞 / 叙事者姿态 / 信息释放方式
  → 用「templates/style-dna-profile.md」模板产出结构化风格档案
     ↓
步骤2 · Prompt 组装
  → 将风格档案填入「templates/style-mimic-prompt.md」提示词模板
  → 组装为完整的 system prompt
     ↓
步骤3 · 风格写作
  → 用组装好的 Prompt 进行试写验证
  → 调整 Prompt 中的偏好权重，迭代优化
     ↓
产出
  → 风格DNA档案（templates/style-dna-profile.md）
  → 复刻提示词（templates/style-mimic-prompt.md）
  → 试写验证片段
```

### 🅱️ 微调路线（效果好、深度蒸馏）

**原理**：用目标作者的全文数据集对 LLM 做 LoRA/QLoRA 微调，将风格信息写入模型权重。

**适用场景**：
- 需要高质量、长时间的风格一致性输出
- 有充足的原文数据（≥10万字）
- 有 GPU 资源

**推荐工具**：
| 项目 | 说明 | 链接 |
|------|------|------|
| LLaMA-Factory ⭐25k+ | 有 Web UI，点鼠标就能训练 | [github.com/hiyouga/LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory) |
| Axolotl ⭐10k+ | 命令行驱动，支持 Q-LoRA | [github.com/OpenAccess-AI-Collective/axolotl](https://github.com/OpenAccess-AI-Collective/axolotl) |

**数据准备**：将作者原文整理为 JSON 格式 `{"instruction":"主题","output":"原文段落"}`，每对 N 条。

## 三、文风DNA分析框架（7维）

当 pop 执行 Style Mimic 分析时，按以下 **7 个维度** 完整扫描目标作者的风格。每个维度回答「这个作者在这个决策点上做了什么选择」。

### 维度1：词汇与语料

| 检查点 | 问题 |
|--------|------|
| 词汇丰富度 | 用词偏口语化还是书面化？生僻词频次？ |
| 特色词库 | 有无标志性高频词/短语/口头禅？ |
| 语气标记 | "了""的""吧""吗"的使用密度？感叹号/问号频次？ |
| 否定/肯定偏好 | 更多用肯定句还是否定句？有无"不是A而是B"结构？ |

### 维度2：句式与节奏

| 检查点 | 问题 |
|--------|------|
| 句长分布 | 偏好短句/长句/混合？平均句长？ |
| 句式变化 | 主谓宾结构还是倒装/插入语？ |
| 排比/重复 | 是否常用排比或重复来制造节奏？ |
| 标点偏好 | 逗号、句号、冒号、破折号的使用习惯？ |

### 维度3：叙事者姿态

| 检查点 | 问题 |
|--------|------|
| 侵入频率 | 叙事者是否经常跳出来解释/评价？每千字几次？ |
| POV 距离 | 紧贴角色内心（零距离）还是远距叙述？ |
| 因果关系 | 作者主动说"因为所以"，还是让读者自己推导？ |
| 情绪标签 | 直接写"他感到恐惧"，还是通过动作/环境暗示？ |

### 维度4：信息释放策略

| 检查点 | 问题 |
|--------|------|
| 释放节奏 | 集中爆发 / 均匀撒放 / 埋伏笔 / 层层递进？ |
| 设定密度 | 每千字引入几个新概念？每个概念如何解释？ |
| 悬念构造 | 用什么手段制造悬念？信息差？时间压迫？ |
| 章末钩子 | 每章结尾的习惯方式？悬念截断/信息炸弹/情绪总结？ |

### 维度5：对话策略

| 检查点 | 问题 |
|--------|------|
| 对话密度 | 对话占全文比例？ |
| 引导词风格 | 极简("XX道") / 偶用 / 每句必用 + 动作描写？ |
| 对话功能 | 主要推进信息 / 塑造关系 / 控制节奏 / 释放情绪？ |
| 独白比例 | 角色内心独白的使用频次？ |

### 维度6：修辞与描写

| 检查点 | 问题 |
|--------|------|
| 修辞密度 | 零修辞 / 克制 / 密集？ |
| 修辞类型 | 比喻/拟人/排比/隐喻等偏好？ |
| 感官维度 | 偏视觉/听觉/触觉/嗅觉？常用几种感官叠加？ |
| 环境描写 | 环境描写占段落比例？环境是否与情绪呼应？ |

### 维度7：情绪与张力

| 检查点 | 问题 |
|--------|------|
| 情感传递 | 靠动作/靠内心独白/靠环境/靠对话潜文本？ |
| 节奏模式 | 单峰型/多峰型/递进型/震荡型/平铺型？ |
| 铺垫-释放比 | 张力铺垫占章节比例？铺垫多久释放一次？ |
| 文风"气质" | 冷峻/温暖/戏谑/压抑/激昂/诗意/冷幽默？ |

## 四、使用方式

### 方式1：提取风格DNA

```
用户：帮我分析这本 [书名] 的写作风格，我想让 Agent 学会它的文风
pop：好的，请提供该书的 3-5 章原文（或前 3000-5000 字）

[用户提供原文]

pop 执行：
  1. 按7维分析框架逐章扫描
  2. 填入 style-dna-profile.md 模板
  3. 生成文风DNA档案
  4. 组装为 style-mimic-prompt 提示词
  5. 产出试写验证
```

### 方式2：模仿指定作者写作

```
用户：帮我写一段 [主题]，用 [某作者] 的风格
pop：收到，我先加载 [某作者] 的DNA档案
  → 如果已有 → 直接注入 Prompt → 写作
  → 如果没有 → 请提供 3-5 篇样本 → 提取DNA → 写作
```

### 方式3：多风格对比生产

```
用户：帮我用 鲁迅 和 村上春树 两种风格写同一段场景
pop：收到，分别加载两个DNA档案
  → 并行生成两版 → 输出对比
```

## 五、输出规范

### 5.1 每次提取的产出清单

| 文件 | 路径 | 说明 |
|------|------|------|
| 风格DNA档案 | `templates/style-dna-profile.md`（填充后版本） | 完整的7维风格分析 |
| 复刻提示词 | `templates/style-mimic-prompt.md`（填充后版本） | 可直接使用的 Prompt |
| 试写验证 | 产出示例 | 用提取DNA写的 300-500 字验证片段 |

提取完成的风格DNA应保存在项目目录下：

```
项目/
├── styles/
│   └── 作者名-书名.md        ← 风格DNA档案
├── prompts/
│   └── 作者名-书名-prompt.md  ← 复刻提示词
```

## 附录：Thinking-Claude 风格Prompt 核心原理

> Thinking-Claude 项目的本质是：**把一种特定的"思维方式"编码为可执行的 system prompt。**

其设计原理可迁移到文风模仿：

```
Thinking-Claude 的做法：                    pop-dna 的迁移：
┌─────────────────────────┐              ┌─────────────────────────┐
│ 分析o1模型的思维方式      │      →      │ 分析目标作者的写作风格   │
│ 提取关键思维特征          │      →      │ 提取7维风格特征         │
│ 编码为结构化system prompt │      →      │ 编码为风格档案+prompt   │
│ 注入Claude使其模仿o1思维  │      →      │ 注入Agent使其模仿作者   │
└─────────────────────────┘              └─────────────────────────┘
```

核心编码原则：
1. **具体化**：不说"模仿鲁迅的文风"，说"句长10-20字，善用隐喻，叙事者冷峻克制"
2. **结构化**：按维度分块，每个块内列具体规则
3. **示例绑定**：每条规则附原文示例
4. **否定清单**：列出"这个风格不会做的事"
