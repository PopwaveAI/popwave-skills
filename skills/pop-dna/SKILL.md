# pop-dna · 文风DNA蒸馏引擎 v3.0

> **给 writer 的不是指令——是发现。** writer Phase 1 打开 styles/ 文件后 30 秒读完，写调音叉，上正文。

---

## ❌ 质量红线

```
□ 采样量 < 10,000 行 → 退回。不足以写一份支撑 200 万字小说的文风DNA。
□ 不统计平均句长、词频、感叹号密度——那是症状，不是发现。
□ 不写禁令（"绝对不要做 X"）——在有限蒸馏下没资格说绝对。
□ 不写可执行指令（"写 X 时先 Y"）——发现本身就是指引，writer 自己判断。
□ 不把单个场景的偶然细节当全书规律——搜全书验证。
□ 每个维度必须标注开篇/中期/末期的变化趋势。没有对比就没有文风认识。
```

---

## 一、核心方法论

### 做什么

从原文中提取**作者在每个维度上的模式**——配原文证据、标全周期变化、搜全书验证。产出的不是报告，是 writer 能 30 秒读完的参考页。

### 不做什么

| 不做 | 原因 |
|------|------|
| 统计平均句长/词频/修辞密度 | 那是症状，不是方法 |
| 写禁令清单 | 在有限蒸馏下没资格说"绝对" |
| 写可执行指令 | 发现本身就是指引，writer 自己判断 |
| 放整段原文让 agent "感受" | 和 v2 浸泡没区别 |
| 贴便携性标注/骨架肉身分类 | 属于下游 writer 的判断，不属于 DNA |

## 二、执行流程

```
步骤0 · 取样
  → 全书扫描，确认章节边界和全卷分布
  → 均匀采样：前 20% + 中段 40-60% + 后 20%，每段 6-7 章
  → 目标：≥20 章，≥10,000 行
  → 不到 10,000 行 → 退回，继续读

步骤1 · 逐章精读采样章
  → 读完整章全文（不跳段、不只看开头）
  → 记录每个维度的具体表现

步骤2 · 全书搜索验证
  → 对每个可能写成绝对结论的判断，搜全书验证
  → 例：看到 "150米/120米/80米" → 搜全书 `剑势`/`重劈` 确认是否有招式名
  → 例：看到开篇几乎不用"英俊" → 搜全书 "英俊" 确认是否全书都不用
  → 搜索数字标注在产出中

步骤3 · 写出风格文件
  → 按模板 `templates/style-dna-profile.md` 产出 `styles/{书名}.md`
  → 从维度池中选择 ≥7 个维度（通用类 ≥4 + 战斗类 ≥1）
  → 维度名由实际发现决定——不硬套不存在的维度
  → 每个维度：1-2 句描述 + 时间变化 + 2-3 处原文 + 全书搜索数字（如有）

步骤4 · 试写验证
  → 用产出的风格文件试写 300-500 字
```

---

## 三、产出格式

见 `templates/style-dna-profile.md`。关键原则：

- 平铺 H2，无章节编号
- 每个维度配 2-3 处原文引用
- 标注开篇/中期/末期变化趋势
- 有全书搜索数字的标注数字
- 末尾：2 条全书稳定特征
- **不硬套不存在维度的名**——书里有什么特征就写什么维度

---

## 四、使用方式

```
用户：帮我分析 [书名] 的文风
→ 扫描全书章节边界 → 均匀采样 ≥20 章 → 逐章精读
→ 全书搜索验证关键判断
→ 产出 styles/{书名}.md
```

---

## 五、与 writer 的集成

pop-dna 产出的 `styles/{书名}.md` 供 `pop-novel-writer` §Step 2 Phase 1 消费：

```
writer Phase 1（风格锚定）：
  ① 打开 styles/ 文件 → 30 秒读完
  ② 对照 design 的场景类型找到对应维度
  ③ 用自己的语感对照原文证据，写出调音叉
```
