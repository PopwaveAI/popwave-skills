## §1 作者方法论摘要

### 1. 情绪全部外化为身体动作
→ "不要写'他害怕'——写'他缩回了手，抱紧了怀里的老狗'。所有内心状态通过外部动作和身体反应传递。"
证据: ch003: "她慌忙缩回了手，抱紧了怀里的老狗。碰到后背上的伤口，不由疼得抽泣了一下。"

### 2. 短句开篇，感官先行
→ "每一章的开头 3-5 句不放任何叙事总结——只放感官细节：声音、气味、触感。小说不始于'琥珀城是一座破败的城市'——始于'雨声先于意识到来'。"
证据: ch001: "雨声先于意识到来。"（全文第一句——不是背景介绍，是直接被雨声砸进场景）

### 3. 数据嵌入叙事，不独立弹出面板
→ "面板/系统信息不放在独立代码块里。属性用冒号连接列在段落中。面板是叙述者视线的一部分——不是打断叙事的弹窗。"
证据: ch001: 面板信息直接列在正文段落里——"姓名：江轩 / 种族：半精灵 / 职业：10级平民（满级）"——无单独代码块。

### 4. 温情和威胁高频交替
→ "不在单一情绪里让读者待太久。日常喂粥的下一段就是门外脚步声。读者的情绪像被两只手交替推——一只手推温暖，一只手推危机。"
证据: ch001 节奏序列: 做饭（日常）→ 脚步声（威胁）→ 狗吠（紧张）→ 远去（放下）→ 喂粥（温情）→ 关门回忆（低沉）→ 决意（凄凉）→ 打杀（恐惧）→ 醒来（希望）

## §2 场景型决策规则

### 场景 2.1：人物外貌描写

规则: 外貌通过身体状态传递——不写"长什么样"，写"处于什么状态"。太瘦、有伤、皮肤惨白、指尖有茧——每一处身体细节都在说话。
文A: ch001: "骨节分明。太瘦了。皮肤白得有点病态，能看清手背上的青色血管。指尖有细小的茧——不是拿刀握剑的茧，是做粗活的茧。"
可执行指令: 写人物外貌时，不写"他面容清秀""他身材魁梧"。写身体状态——瘦到什么程度？皮肤是什么颜色？手上有什么痕迹？每处细节都是一个信息——不要为描写而描写。

### 场景 2.2：环境/城市群像

规则: 从一个点展开空间——不直接说"这里是贫民区"。从棚屋内部开始：泥地→木箱→草席→破布→小女孩→向外看一眼→看到贵族区的塔楼。空间认知是主角睁开眼睛逐步发现的。
文A: ch001: "视野模糊了很久才慢慢聚焦。头顶是一根根裸露的房梁……墙上挂着几条破布……角落里缩着一个小小的身影。"
可执行指令: 环境描写从角色的感官出发——ta 先看到什么、再看到什么、最后才推出结论。不直接从结论开始写。

### 场景 2.3：角色决策推理

规则: 理性推演 + 游戏化思维——角色做决定时不写"他心中一横""他凭直觉"。他算收益、算风险、回看面板数据、回忆游戏经验。决策是成本核算，不是冲动。
文A: ch002: 江轩暗杀巴拉斯前——先确认对方人数、路线、携带武器。再算自己能承受几击。面板上的"生命值 6/12"是决策依据。
可执行指令: 写角色做重大决定时，先展示 ta 的计算过程——看到了什么信息、怎么判断风险、最后选了什么。不写"直觉""冲动""心中一横"。

### 场景 2.4：战斗

规则: 节奏序列 = 观察→计算→行动→反馈→调整。每次攻击都有明确理由（"他转身是因为听到了脚步声"），每次受伤都有面板数据跟进（生命值变化）。感官切换：视觉（场景布局）→ 触觉（刀刺进身体的阻力）→ 面板数据（数字反馈）。
文A: ch002 暗杀巴拉斯: 先观察（"他只有一个人"）→ 计算（"一刀不够，需要两刀"）→ 行动（"从背后刺进去"）→ 反馈（"面板跳出经验值"）→ 调整（"脚步声——不能恋战"）
可执行指令: 战斗不写"他狠狠砍了一刀"。写五步循环：观察（他看到了什么）、计算（他判断怎么做）、行动（具体动作）、反馈（面板/身体反应）、调整（变化后继续）。每步用短句。感官不纯视觉——加入阻力感/痛感/数字。

### 场景 2.5：对话/市井

规则: 对话极简——引导词几乎不用。市井场景用粗粝口语。妹妹的对话永远简短（"嗯""好"）——不是不会说话，是在极度疲劳和恐惧中说不出来。反派台词带酒气和暴戾。
文A: ch001: "「钱在哪？」——踢门声比这句话先到。"
文B: ch001: "「——」妹妹的回应是沉默。然后沙哑的'嗯'。"
可执行指令: 市井场景的对话用动作前置（先踢门再说台词）。对话不是由引导词展开——是由动作和环境展开。妹妹的对话≤5 个字。反派对话带身体动作（踹门/吐口水/酒气）。

### 场景 2.6：世界观植入

规则: 新概念用极短段落带出——不解释、不拓展、不展开说明段。面板第一次激活只展示了 5 行数据——足够了。琥珀城阶级分化只用了一句话："三条街的距离，两个世界——那边灯火通明，这边只有煤油灯。"不需要解释琥珀城的历史和社会结构。
文A: ch001: "姓名：江轩 / 种族：半精灵 / 职业：10级平民（满级） / 状态：虚弱（灵魂复活未完全适应） / 生命值：6/12" —— 5 行，不解释"什么是半精灵""什么是灵魂复活"。
文B: ch001: 琥珀城阶级——"从棚屋门口看出去，三条街外的贵族区塔楼亮着灯火。三条街，两个世界。"
可执行指令: 每个新概念首次出现时 ≤3 句。不写说明段落。概念和场景绑定——"半精灵"出现在面板上而非叙述者解释。阶级不是"这座城市分为上城区和下城区"——是"那边灯火通明，这边只有煤油灯"。

## §3 跨赛道便携性标注

```yaml
portable（跨赛道通用——骨架·永久保留）:
  - 情绪动作外化            → 任何题材·任何文化
  - 短句开篇·感官先行       → 任何题材——不换
  - 数据嵌入叙事（面板不独立弹出） → 任何数据流/系统流赛道
  - 战斗五步循环（观察→计算→行动→反馈→调整） → 任何需要战斗的赛道
  - 世界观用场景带出·≤3句   → 任何需要世界观铺垫的赛道
  - 温情/威胁高频交替       → 任何需要保持张力的赛道

  - 身体状态细节            → 变量: {本世界身体标记}
    西幻: 皮肤惨白/青色血管/茧 / 仙侠: 灵力枯竭/经脉受损/气息紊乱
  - 面板/数据格式           → 变量: {本项目数值格式}（扁平数值/境界比率/物理量纲）
  - 市井口语               → 变量: {目标时代底层口语}
  - 城市分区隐喻            → 变量: {本世界阶层可视标记}
    西幻: 贵族区塔楼灯火 vs 煤油灯 / 仙侠: 仙门云海 vs 集市喧闹
```

## §4 风格红线

```
本风格绝对不会做的事：
□ 写"他感到恐惧/绝望/愤怒"——永远用动作或身体反应替代
□ 面板数据独立成代码块——面板是叙述的一部分，不是弹窗
□ 开篇第一句用背景介绍——第一句必须是感官细节
□ 用"不是A而是B"的否定式描写——永远肯定陈述
□ 世界观用教材式说明段——每个新概念 ≤3 句、绑定场景
□ 战斗只用视觉描写——必须混合触觉/听觉/身体反应
□ 对话用"XX道""XX说"引导——对话由动作和环境前置
□ 温情场景超过 500 字不跟威胁——读者不能在同一情绪里待太久
```
