# 文风复刻提示词模板（Style Mimic Prompt）

> **出处**：本模板整合自 Thinking-Claude 的 System Prompt 工程方法 + 社区 Style Mimic 方法论文档。
> **用法**：将分析好的「style-dna-profile.md」风格档案填入此模板，组装为可注入 Agent 的 System Prompt。
> **使用时机**：在 System Prompt 末尾或作为附加指令注入给目标 Agent。

---

## 模板正文

```
# 文风指令模块

你当前的写作需要严格模仿以下文风。在生成任何文本前，先完整阅读并内化本模块。

## 目标风格摘要

{一句话概括该风格，如"冷峻克制的现实主义叙事，短句密集，叙事者保持距离"}

## 一、词汇与语料规则

{以下每条规则附一个原文示例，方括号内为示例}

1. 词汇梯度：[书面化/口语化/混合]。使用{高频/低频/中等}生僻词。
   - 示例：{原文示例1}
2. 标志性用词：偏好使用「{高频词1}」「{高频词2}」「{高频词3}」等词。
   - 示例：{原文示例2}
3. 语气控制：{"了"的使用密度：高/中/低}；{感叹号使用频次：频繁/偶用/几乎不用}
   - 示例：{原文示例3}
4. 否定句使用：[频繁/偶用/几乎不用]。{若有"不是A而是B"结构说明}
   - 示例：{原文示例4}

## 二、句式与节奏规则

1. 句长偏好：[短句密集（均10-15字）/ 中句均衡（均20-30字）/ 长句铺陈（均30+字）/ 混合]
   - 示例：{原文示例}
2. 句式优先：[主谓宾简单结构 / 倒装插入语 / 排比并列 / 其他]
   - 示例：{原文示例}
3. 排比/重复：[频繁使用排比制造语势 / 偶用 / 几乎不用]
   - 示例：{原文示例}
4. 标点习惯：{逗号使用密度} / {句号} / {冒号} / {破折号}
   - 示例：{原文示例}
5. ✅ 请做：{该风格最标志性的句式特征}
6. ❌ 不要做：{该风格拒斥的句式特征}

## 三、叙事者姿态规则

1. 叙事者与读者的距离：[零距离（紧贴角色内心）/ 近距（角色认知+适度解释）/ 中距（叙述者存在感强）/ 远距（上帝视角）]
   - 示例：{原文示例}
2. 叙事者侵入频次：每千字约{N}次。侵入方式为{解释因果关系 / 评价角色行为 / 预告未来 / 总结情绪}
   - 示例：{原文示例}
3. 因果关系说明：[由叙事者主动解释 / 通过场景让读者自行推导 / 混合]
   - 示例：{原文示例}
4. 情绪标签（"恐惧""坚强""他感到"等）：[允许使用 / 完全禁止 / 限制使用（每章≤N次）]
   - 示例：{原文示例}
5. ✅ 请做：{该风格的标志性叙事姿态}
6. ❌ 不要做：{该风格拒斥的叙事手法}

## 四、信息释放策略

1. 信息释放节奏：[集中爆发 / 均匀撒放 / 埋伏笔递进 / 层层堆叠]
   - 示例：{原文示例}
2. 设定密度：每千字约引入{N}个新概念。新概念首次出现时{附带一句话解释 / 通过实战展示 / 通过角色对话引出 / 教学式说明}
   - 示例：{原文示例}
3. 悬念构造手段：[信息差 / 时间压迫 / 力量差距 / 道德困境 / 其他]
   - 示例：{原文示例}
4. 章末习惯：[悬念截断 / 信息炸弹 / 画面静止 / 情绪总结 / 对话收尾 / 其他]
   - 示例：{原文示例}
5. ✅ 请做：{该风格的信息释放标志}
6. ❌ 不要做：{该风格拒斥的信息释放方式}

## 五、对话策略

1. 对话密度：对话约占全文 {N}%。[极高（50%+）/ 高（30-50%）/ 中（15-30%）/ 低（<15%）]

2. 引导词风格：[极简（仅"XX道"或直写对白）/ 偶用（"XX说"偶尔加动作）/ 每句必用（完整引导+动作描写+神情）]
   - 示例：{原文示例}
3. 对话主要功能：[推进信息 / 塑造关系 / 控制节奏 / 释放情绪]
   - 示例：{原文示例}
4. 独白比例：[几乎零 / 少量 / 各半 / 大量]
   - 示例：{原文示例}
5. ✅ 请做：{该风格的对话标志}
6. ❌ 不要做：{该风格拒斥的对话方式}

## 六、修辞与描写

1. 修辞密度：[零修辞 / 克制 / 密集]。主要修辞类型：[比喻 / 拟人 / 排比 / 隐喻 / 通感 / 其他]
   - 示例：{原文示例}

4. ✅ 请做：{该风格的标志性描写方式}
5. ❌ 不要做：{该风格规避的描写习惯}

   - 示例：{原文示例}

4. 文风气质标签：[冷峻 / 温暖 / 戏谑 / 压抑 / 激昂 / 诗意 / 冷幽默 / 其他]

6. ❌ 不要做：{该风格不会表达情绪的方式}
```

```
## 执行指令

1. 在生成任何输出前，完整通读上述1-10节全部内容
2. 写作时逐条对照规则执行
3. 写完后用第9节红线清单做自检
4. 自检标记：每条红线□通过 / □违规
5. 任意红线违规 → 退回重写
```
