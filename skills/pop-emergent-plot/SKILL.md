# plot-design

> 将故事种子和设定渲染为可供 review 消费的章节锚点。管"写什么"，不管"怎么写"。

---

## 设计哲学：故事先行 + 燃料式

**故事先行**：每一幕先想清楚"为什么需要这一幕"（定位+目的），然后直接白描剧情走向——一大段文字讲故事。剧情白描是核心，写不好后面展开全是浪费时间。维度拆解（危机/冲突/节奏/资源）从剧情中反拆，不是先设计维度再拼剧情。

**燃料式**：种子文档提供世界观层面的燃料潜力，素材收集为每幕建立原始燃料库（广撒网，只收集不选用），幕纲设计时从燃料库选取→改造→融入剧情白描。6道门禁保证质量，但不规定内容。

核心区别：
- 种子文档提供**设定和约束**，不提供**剧情**
- seed阶段二（世界展开）把种子设定转化为**可操作的设计锚点**（主角怎么主动行动/角色怎么反应/势力怎么制造事件）
- 幕定位提供**方向和目的**，不提供**事件**
- 剧情白描是**核心环节**——先有故事再拆维度，不是先拆维度再拼剧情
- 幕纲是agent创造力发挥的地方——从燃料池选燃料、白描剧情、反拆维度、分章植入
- 质量由6道前置门禁保证，不由预设剧情保证

---

## 做什么

输入：故事种子(seed，含阶段二世界展开产出：主角引擎/角色储备池/世界圣经) + 赛道文档(reference)
输出：幕纲（含幕定位 + 7节拍硬锚点 + 爽点链条 + 角色行为锚点 + 章卡边界）

**前置依赖**：plot-design 启动前，seed 必须已完成阶段一（种子文档）和阶段二（主角引擎/角色储备池/世界圣经）。如未完成，回到 seed。

两层架构：
- **SKILL.md（本文件）**：通用层——2-step SOP + 前置门禁 + 红线 + 速查表
- **references/**：赛道层——各赛道的爽点结构/节奏偏好/冲突来源/事件分量标准，按需加载

与 review 的对接：幕纲输出的7节拍硬锚点 + 爽点链条 + 章卡边界交接给 review（由 review 压缩进 current-state.md 的"下一章硬推进"字段）。

---

## 剧情体量标准

| 层级 | 章节范围 | 质变要求 | 对标参考 |
|------|---------|---------|---------|
| 幕 | 8~15章 | 主角巨大变化 + 剧情巨大变化 | BG3一个act、深渊主宰一个剧情弧 |

**体量意识**：
- 提取外部作品结构时（"提取结构→换皮→融合"），必须先评估源作品的体量层级——大于幕级的结构只提取幕级子结构或结构模式
- 幕纲必须规划8~15章的完整体量，不是3章试写
- 每幕幕末必须有可验证的质变（主角状态/关系/地位/能力的不可逆变化）

## 怎么操作（SOP骨架）

> **execution.mode**: formal（必读输入齐全落盘）/ draft（有缺口可补全继续）/ trial（快速试做不落盘）。检测 current-state.md 是否存在，不存在→formal；存在但缺字段→draft；用户说"试做"→trial。

### Step 0：加载上下文 + 选赛道

读取项目文件 + 按题材加载赛道文档。

| 文件 | 何时读 |
|------|--------|
| `seed-种子文档.md` | 每次必读——主卖点/核心承诺/不可牺牲项/禁区 |
| `主角引擎.md` | 每次必读——主角行动偏好/阵营操作性定义/第一章主动动作 |
| `角色储备池.md` | 每次必读——S/A/B/C角色+反应模式+可制造事件+主角可利用点 |
| `世界圣经.md` | 按需——力量体系/三阶社会结构/势力格局/资源物品/核心矛盾 |
| `references/通用剧情模式.md` | 每次必读——通用爽点结构/事件分量标准/低压区设计/危机多维设计/金手指巧思设计 |
| `references/{赛道}剧情模式.md` | 按题材加载——该赛道的爽点结构/节奏偏好/冲突来源 |
| `references/设计原则库/*.md` | 每次必读——开局/危机/成长曲线/情感/战斗 5个文件，提供设计判断依据 |

**红线**：如果 `主角引擎.md` / `角色储备池.md` 不存在，停止。回到 seed 完成阶段二（世界展开）后再进入 plot-design。

### Step 1：素材收集（每幕前置）→ 逐幕循环

详见 `steps/step-1-material.md`。**每幕设计前必须先做一轮素材收集。** 不是可选步骤。

产出：`燃料库/原始燃料/` 目录（多个方向文件 + 内部涌现 + 索引）

核心内容：
- 根据书的特性规划8-15个搜索方向（11维度框架：原作IP/目标书/同人改编/跨媒介/战斗成长/金手指/世界观/神话/前世身份/爽文平台/角色关系）
- 每个方向独立搜索、独立落盘（主agent召唤Explore子agent执行WebSearch）
- 外部搜索后做**内部涌现**（从设定+外部素材碰撞中推导新燃料）
- **只收集不选用**——选中燃料是Step 2幕纲设计时做的事

**为什么需要这一步**：API无联网能力，单靠内部推导的燃料容易同质化。子agent搜索能引入跨媒介IP/历史参考/神话原型/其他作品结构模式，让燃料来源多元化，避免剧情平庸。

### Step 2：幕纲（5步设计链路·逐幕循环）

详见 `steps/step-2-act.md`。**两阶段设计**：阶段A快速出2-3个候选方向让用户挑选，阶段B基于选定方向走5步链路。

**阶段A**：方向选择（简版多方案）
- 产出：`卷纲/幕NNN-候选方案.md`
- 每个方向半页纸：幕定位（作用/体验/弧线）+ 剧情方向梗概（3-5句）+ 幕末质变
- 2-3个方向必须有实质性差异（不同冲突/不同压力来源/不同角弧方向）
- **暂停，等待用户选择**

**阶段B**：5步设计链路（基于选定方向展开）
- 产出：`卷纲/幕NNN-设计笔记.md`（思考过程）+ `卷纲/幕纲-幕NNN-{标题}.md`（正式幕纲）
- 5步链路：
  1. **定位与起点快照**：本幕位置+前序回顾+起点状态7维度
  2. **确认本幕目的**：剧情推演目标+主角成长目标+读者体验目标+开局契约（第一幕）
  3. **剧情白描（核心·三段差异化→评估→整合）**：先产出3段方向不同的白描（各有侧重，每段从原始燃料库选取≥2条素材改造融入），经agent自评+作者挑选后整合为1版最终白描。整合版必须通过9项质量自检（含故事容量/成长分散/资源多元3项硬指标）。写不好禁止往下走
  4. **反拆维度**：从剧情白描反拆章节预算/地理/人物/战斗/成长/资源事件+燃料利用率检查+资源缺口分析+剧情密度测算+节奏红线+质变/门禁自检
  5. **分章概览**：把反拆信息植入每章，输出分章概览表
- 正式幕纲从5步产出中提取，控制在200行以内（含剧情白描全文）
- **不含分章施工卡**——幕纲确认后先初始化事实快照，再进入施工卡

**事实快照**：幕纲确认后的后置步骤。初始化快照→注入施工卡→注入正文→每章写完后更新。参考 `steps/step-3-snapshot.md`。

**分章施工卡**：快照初始化后的后置步骤。参考 `steps/step-3-chapters.md`。

## 前置门禁

**过不了不进 review。** 六道门在幕纲阶段逐章检验：

### 门禁1：事件分量

这个事件配独立成章吗？必须满足至少一条：
- 改变主角命运（不可逆的身份/地位/能力/关系变化）
- 打开新可能性（新的地图/势力/规则/资源解锁）
- 制造追读欲望（留下读者必须知道答案的悬念）

不满足 → 降级为章内事件，不配独立成章。

### 门禁2：角色地基

读者能用三句话说出主角是谁吗？
- 身份：主角是什么人（一句话，含社会角色 + 核心特质）
- 欲望：主角想要什么（一句话，具体到可操作目标）
- 伤口：主角怕什么/缺什么（一句话，内心脆弱点）

三句话缺任一 → 回去补角色地基，不进分章设计。

### 门禁3：期待感

读者翻到这章末尾，脑子里有没有一个"想知道但还不知道"的问题？
- 藏九露一：这章揭示什么，保留什么悬念
- 每章末尾必须有一个读者主动想追问的问题
- 问题不能是"接下来会发生什么"这种万能句

没有明确期待感设计 → 重写章末钩子。

### 门禁4：冲突驱动

冲突是两个角色的欲望撞在一起，还是作者硬塞的？
- 冲突双方各有欲望（敌方想什么，主角想什么，为什么不可调和）
- 冲突升级有内在逻辑（不是"突然来了一个更强的敌人"）
- 冲突解决方式体现角色特质（不是万能解法）

冲突无角色欲望支撑 → 重新设计冲突来源。

### 门禁5：危机多维

本幕/本章的危机是否来自多个维度？单一危机源的章节判为不合格。

至少叠加2个以上危机维度（详见 `references/通用剧情模式.md` 的"危机多维设计"模块）：
- 生存危机（不行动就死/饿/被杀）
- 情感危机（在乎的人受到威胁）
- 社会危机（身份暴露/被追杀/被排斥）
- 认知危机（世界规则和预期不符/前世知识有误）
- 资源危机（经济/物资/时间不足）

只有单一危机源 → 回到素材补强环节，补充其他维度的危机线。

### 门禁6：金手指巧思

金手指设计是否有"巧思"——不只是"能做什么"，而是设计本身能产生决策困境/剧情发动机？

不合格标准：
- 金手指只是"翻译器"（展示数据/属性/技能信息），不产生决策
- 金手指只有纯正向反馈（用→变强→用），没有使用时的选择/博弈
- 主角前5章成长速度落后于爽文节奏（前3章应有觉醒+升级+翻盘三连）

不合格 → 回到种子文档或素材补强环节，重新设计金手指的"巧思维度"。

## 红线

1. **事件分量红线**：小闭环事件（无命运影响的琐碎胜利）不配独立成章。违反=不合格。
2. **角色地基红线**：主角三句话（身份/欲望/伤口）必须在种子文档中确立。违反=不合格。
3. **期待感红线**：每章必须有明确的"藏什么/露什么"设计。章末钩子必须是读者主动想追问的问题。违反=不合格。
4. **冲突驱动红线**：冲突必须源于角色欲望对立，不能是"突然来了敌人"的公式驱动。违反=不合格。
5. **章卡边界红线**：每章必须写清"本章只写什么/禁止吞并下一章什么"。review 若因此合并章节，责任归 plot-design。违反=不合格。
6. **主角主动红线**：每章主角至少1个主动决策+1个行动。不把世界压迫当主角驱动。违反=不合格。
7. **锚点格式红线**：幕纲输出的7节拍硬锚点必须与 review 的章型节拍定义对齐。违反=不合格。
8. **危机多维红线**：每幕至少叠加2个以上危机维度。单一危机源的幕判为不合格。违反=不合格。
9. **素材收集红线**：每幕幕纲前必须先完成素材收集环节（外部搜索+内部涌现）。跳过素材收集直接写幕纲=不合格。
10. **体量红线**：幕必须规划8~15章完整体量。把大于幕级的结构压进5章或将幕写成3章以下=不合格。每幕幕末必须有可验证的质变（主角状态/关系/地位/能力的不可逆变化）。
11. **燃料落盘红线**：素材收集的调研内容必须落盘到 `燃料库/原始燃料/` 目录，每个方向独立一个文件 + 内部涌现文件 + 索引文件。只在对话中使用不落盘=不合格。
12. **主动行为密度红线**：每章至少1个主角主动行为——从{战斗/宏观层面解决问题/主动出击/收获/获得反馈/质变级反馈和成长}中选≥1项。篇幅不做固定约束，由叙事需要决定。连续2章无主动行为=不合格。
13. **seed前置红线**：plot-design 启动前 seed 必须已完成阶段一和阶段二。`主角引擎.md` + `角色储备池.md` + `世界圣经.md` 缺失时进入 plot-design = 不合格。
14. **剧情白描红线**：阶段B第3步（剧情白描）不达标时禁止进入第4步（反拆维度）。带着烂地基往后走=全盘报废。白描必须是通过质量自检的完整故事弧线，不是大纲/提纲/维度列表。
15. **剧情密度测算红线**：阶段B第4步必须做剧情密度测算。密度不达标时必须调整（反向调整白描或调整章节预算），禁止跳过密度测算直接分章。

## 速查表

### 素材收集质量门

| 项 | 合格 | 不合格 |
|----|------|--------|
| 方向规划 | 根据书特性规划8-15个搜索方向，覆盖11维度框架 | 固定5个方向，不思考书特性 |
| 外部搜索 | 每个方向独立搜索+独立落盘到 `燃料库/原始燃料/方向NN-*.md` | 聚合在一个文件或跳过搜索 |
| 内部涌现 | 从设定+外部素材碰撞中推导≥2条新燃料 | 只罗列已有燃料，无新增 |
| 燃料落盘 | `燃料库/原始燃料/` 含多方向文件+内部涌现+索引 | 只在对话中使用不落盘 |
| 只收集不选用 | Step 1不输出补强清单，选中留给Step 2 | 在Step 1就做选取决策 |

### 幕纲质量门

| 项 | 合格 | 不合格 |
|----|------|--------|
| seed阶段二前置 | 主角引擎+角色储备池+世界圣经已加载 | 缺失seed阶段二产出直接写幕纲 |
| 素材收集 | 已读取原始燃料库索引 | 跳过素材收集直接写幕纲 |
| 剧情白描 | 500-1500字完整故事弧线，通过10项质量自检（含多视角） | 大纲/提纲/维度列表，或未达标就进入反拆 |
| 剧情密度测算 | 密度测算达标或已调整 | 跳过密度测算直接分章 |
| 事件分量 | 每章通过门禁1 | 有章不通过但仍保留 |
| 角色地基 | 三句话在种子文档确立，幕纲继承 | 幕纲阶段才补 |
| 期待感 | 每章有"藏什么/露什么" | 只有事件没有期待感设计 |
| 冲突驱动 | 双方欲望对立 | 公式驱动 |
| 危机多维 | 每幕≥2个危机维度叠加 | 单一危机源 |
| **主动行为密度** | 每章≥1个主角主动行为（战斗/解决问题/出击/收获/反馈/质变成长），篇幅不固定 | 连续2章无主动行为 |
| 7节拍硬锚点 | 每章完整且与review章型对齐 | 缺节拍或格式不对 |
| 爽点链条 | 爽点具体到"对方可见反应"，有旁观者 | 只有"主角变强" |
| 章卡边界 | 每章写清终点+保留项+禁止吞并项 | 缺失 |
| 微beat施工表 | 每章8-14个beat，8种类型交替 | 纯行动beat链 |
| 角色行为锚点 | 主角+反派+配角的反应模式 | 无行为锚点 |
| 体量 | 每幕8~15章完整体量 | 幕不足3章 |
| 幕末质变 | 主角状态/关系/地位/能力不可逆变化 | 无质变 |

### 爽点链条设计速查

| 要素 | 要求 |
|------|------|
| 爽点定义 | 具体到"对方可见反应"（如当众出丑/地位翻转/被反驳到哑口无言） |
| 旁观者 | 有在场旁观者放大爽感（围观/嘲讽/震惊） |
| 低压区 | 翻盘前有低压区蓄力（被踩到泥里：当众羞辱/被当废物/被嘲笑） |
| 翻盘 | ≤3行完成翻盘动作 |
| 失态 | 对方有可见失态（不可置信/崩溃/愤怒） |
| 即时奖励 | 行动后可见回报（资源/情报/名声/能力提升） |

### 微beat类型速查

| beat类型 | 作用 | 频率参考 |
|---------|------|---------|
| 情绪beat | 共鸣/动机/张力 | 每章1-2个 |
| 行动beat | 主角主动出击 | 每章2-4个，主力 |
| 收获beat | 面板/升级/结算/资源到账 | 系统流每章1-2个 |
| 信息beat | 揭示/反转/线索 | 每章1-2个 |
| 冲突beat | 对峙/被欺/打脸 | 每章1-2个，爽点载体 |
| 损失beat | 挫折/代价/危机升级 | 每章0-1个 |
| 社交beat | 结盟/背叛/身份暴露 | 每章0-1个 |
| 日常beat | 喘息/互动/彩蛋 | 大战后1个 |

### 赛道文档速查

## 与 review 的对接

幕纲输出的每章施工卡 + 事实快照交接给 review（由 review 压缩进 current-state.md 的"下一章硬推进"字段），包含 review 需要的全部输入：

| review 输入 | 幕纲输出项 |
|-------------|-----------|
| 章型选择 | 幕纲标注的章型（opening_shift/confrontation_pressure等） |
| 7节拍硬锚点 | 每章的7节拍beat链 |
| 爽感引擎 | 每章的爽点链条设计 |
| 不可触发项 | 章卡边界的"禁止吞并项" + 种子文档的"禁区" |
| 人物状态 | 角色行为锚点 + 幕初/幕末状态 |
| **已确立事实** | **事实快照（全局设定+已确立事实+当前角色状态+待确认偏离）** |
| 微观技法推荐 | 章型对应的推荐技法组合（review自行选择） |

review 不需要读取完整幕纲，只需读取对应章节的施工卡 + 事实快照。
