# 通用剧情模式

> 所有赛道共享的剧情设计原则。每次必加载。赛道专属文档在此基础上叠加。

---

## 一、事件分量标准

### 什么事件配独立成章

一个事件必须满足以下至少一条才配独立成章：

| 条件 | 说明 | 举例 |
|------|------|------|
| 改变主角命运 | 不可逆的身份/地位/能力/关系变化 | 获得灵能核心/被暗杀者锁定/从骑手变成被追杀者 |
| 打开新可能性 | 新的地图/势力/规则/资源解锁 | 解锁新区域/获得新能力/接触新势力 |
| 制造追读欲望 | 留下读者必须知道答案的悬念 | 裂变日是人祸/暗杀者已在赶来/核心三天会爆 |

### 什么事件不配独立成章

- 小闭环事件：无命运影响的琐碎胜利（捡到一点钱/打赢无关紧要的小角色）
- 纯氛围事件：没有信息推进的场景（纯环境描写/纯心理活动）
- 重复事件：与前面章节同质的事件（又一次打脸小角色/又一次发现线索但不推进）

不配独立成章的事件 → 降级为章内事件，合并到其他章。

## 二、爽点结构（通用）

### 爽点公式

```
压力 → 突破 / 弱→强 / 未知→揭示 / 被欺→反击
```

每章至少触发1个，且有爆发感（不是温吞推进）。

### 爽点类型

| 类型 | 机制 | 适用场景 |
|------|------|---------|
| 打脸循环 | 轻视→被欺→暗蓄→翻盘→失态 | 对峙章/强弱对话 |
| 压力突破 | 绝境→发现缝隙→突破→回味 | 战斗章/追杀章 |
| 未知揭示 | 疑问累积→线索收束→真相炸裂→后果展开 | 开篇章/揭秘章 |
| 弱→强反转 | 被碾压→觉醒/升级→反杀 | 战斗章/觉醒章 |

### 低压区设计

翻盘前必须有低压区蓄力。低压区不是"主角有点紧张"，是"把主角踩到泥里"：

| 低压区类型 | 具体表现 | 厚度参考 |
|-----------|---------|---------|
| 当众羞辱 | 被当面嘲笑身份/能力/出身 | 2-3段（约400-600字） |
| 被当废物 | 被高位者无视或轻视到忽略不计 | 1-2段（约200-400字） |
| 被碾压 | 实力差距明显到绝望 | 2-3段（约400-600字） |
| 被背叛 | 信任的人突然反水 | 1-2段（约200-400字） |

低压区厚度需匹配翻盘爆发力——翻盘越爽，低压区越厚。但低压区不超过3段，否则读者弃书。

### 旁观者设计

爽点必须有旁观者在场放大爽感：

| 旁观者类型 | 作用 | 举例 |
|-----------|------|------|
| 围观群众 | 见证+传播 | 巷子里看到打脸的路人 |
| 嘲讽者 | 前期嘲讽→后期震惊 | 之前看不起主角的同事 |
| 权威者 | 侧面定级 | 比主角强的角色被结果震惊 |

## 三、期待感设计

### 藏九露一

每章揭示一部分信息，保留更大的悬念：

| 要素 | 说明 |
|------|------|
| 露什么 | 本章揭示的新信息（读者此时应该知道什么） |
| 藏什么 | 本章不给答案的悬念（读者此时想知道但还不知道什么） |
| 章末追问 | 读者读完本章后脑子里必须有的具体问题 |

### 章末钩子标准

章末钩子不能是"脚步声逼近"这种被动悬念，必须是让读者主动想追问的具体问题：
- 合格："核心三天会爆，他还没学会控制——下一个来的S级暗杀者只给他几小时"
- 不合格："接下来会发生什么？"

## 四、冲突设计

### 冲突源于欲望对立

| 要素 | 说明 |
|------|------|
| 主角欲望 | 主角在这场冲突中想要什么（具体到可操作） |
| 敌方欲望 | 敌方在这场冲突中想要什么（具体到可操作） |
| 不可调和 | 为什么两个人的欲望不能同时满足 |
| 升级逻辑 | 冲突怎么从低烈度升级到高烈度（不是"突然来了更强的敌人"） |
| 解决方式 | 冲突怎么解决（体现角色特质，不是万能解法） |

### 冲突升级路径

```
低烈度（试探/摩擦）→ 中烈度（正面冲突/代价）→ 高烈度（生死/命运对决）
```

每一步升级必须有内在逻辑——前一步的结果导致后一步的发生。

## 五、代入感五要素

读者代入角色靠的不是设定倾倒，是以下五个要素：

| 要素 | 说明 | 实现方式 |
|------|------|---------|
| 活人感 | 角色像活人不像NPC | 有小习惯/小矛盾/日常互动 |
| 熟悉感 | 读者能理解角色的处境 | 用读者熟悉的场景/关系/情感 |
| 情感共鸣 | 读者能感受角色的情绪 | 通过身体反应而非情绪描写 |
| 利益一致 | 读者希望主角赢 | 主角的欲望是读者也想要的 |
| 信息对称 | 读者和主角知道一样多 | 不让主角比读者多知道太多（制造悬念除外） |

## 六、章型与爽感引擎匹配

| 章型 | 推荐爽感引擎 | 原因 |
|------|------------|------|
| opening_shift | 未知揭示 | 开篇靠悬念驱动追读 |
| exploration_gain | 压力突破 | 探索靠突破障碍获得爽感 |
| confrontation_pressure | 打脸循环 | 对峙靠翻盘打脸获得爽感 |
| combat_reversal | 压力突破 | 战斗靠绝境翻盘获得爽感 |
| high_tier_display | 未知揭示 | 高位展示靠揭示更大世界获得爽感 |
| reveal_hook | 未知揭示 | 揭秘靠真相炸裂获得爽感 |

## 七、番茄快爽节奏约束

> 番茄读者模型：手机滑动阅读，30秒内需要新内容刺激，连续3次纯氛围则退出。前1/3章节必须出现核心爽点。

### 每章爽点分布

| 位置 | 要求 | 说明 |
|------|------|------|
| 前1/3 | 至少1个信息钩子或小爽 | 读者划30秒看不到刺激就退出。数据面板弹出/属性碾压展示/信息差优势展示 |
| 中1/3 | 核心事件推进 | 事件必须有命运影响，不能纯氛围 |
| 后1/3 | 爽感爆发+章末钩子 | 翻盘/升级/面板弹出+追问 |

### 禁止的节奏模式

| 模式 | 问题 |
|------|------|
| 全章低压蓄力 | 番茄读者不等到下一章。低压区不超过3段（400-600字），必须在本章内翻盘 |
| 铺垫超过2章才爆发 | 最多1章铺垫+1章爆发。不能ch001-002全铺垫ch003才爆发 |
| 连续2章无面板/无战斗 | 数据面板是核心爽感载体。至少每2章弹1次面板 |
| 同种战斗连续3次 | 切换战斗模式（暗杀→正面→游击→终结） |

### 章型搭配约束

| 规则 | 说明 |
|------|------|
| 不得连续2章同章型 | ch001=confrontation_pressure则ch002必须换 |
| 每3章至少1个combat_reversal | 番茄需要战斗爽感，3章不打架读者会弃 |
| opening_shift面板必须在前1/3弹出 | 不要等到章末才弹属性面板 |

### 跨幕衔接约束

| 规则 | 说明 |
|------|------|
| 幕002首章不得与幕001末章同章型 | 跨幕必须换节奏 |
| 跨幕时至少引入1个新燃料类型 | 不能完全用旧燃料 |
| 跨幕首章必须有独立爽点 | 不能是纯过渡章 |

## 八、危机多维设计

> 番茄Top20拆解发现：18/19本前三章都是"不死就死"级别冲突，且全部有多维危机叠加。单一危机源=剧情平庸。
> 危机不是单一来源，而是多维度交织施压，让主角"不得不行动"且"行动有方向"。

### 五大危机维度

| 危机维度 | 定义 | 典型表现 | 压力机制 |
|---------|------|---------|---------|
| 生存危机 | 不行动就死/饿/被杀 | 被追杀/饥饿/疾病/倒计时 | 不可逆后果+强制行动 |
| 情感危机 | 在乎的人受到威胁 | 妹妹被抓/朋友背叛/守护对象受伤 | 情感驱动+冲动风险 |
| 社会危机 | 身份暴露/被追杀/被排斥 | 有面板的秘密被发现/被通缉/被驱逐 | 社会关系崩塌+孤立 |
| 认知危机 | 世界规则和预期不符 | 前世知识有误/世界真相被揭露/信仰崩塌 | 决策依据动摇+判断失误 |
| 资源危机 | 经济/物资/时间不足 | 没钱买药/经验值不够/时间窗口关闭 | 资源稀缺+选择困境 |

### 危机叠加原则

1. **不是平行罗列，而是交织**：生存危机引发资源危机（受伤→需要药→没钱买），资源危机加剧情感危机（没钱买药→妹妹病情恶化），形成连锁反应
2. **至少1个维度有倒计时/不可逆后果**：这是"结构性强制"——主角不是"想"行动，是"必须"行动。倒计时是最好用的压力工具（2天后流放/7天后试炼/经验值递减到0）
3. **危机升级路径**：低烈度（不安/试探）→中烈度（正面冲突/代价）→高烈度（生死/命运对决），每步升级有内在逻辑

### 番茄Top20的危机叠加案例

| 小说 | 叠加维度 | 结构性强制 |
|------|---------|-----------|
| 大奉打更人 | 生存（流放）+情感（家人入教坊司）+资源（2天倒计时）+认知（妖物作祟被推翻） | 2天后流放 |
| 十日终焉 | 生存（被杀）+社会（囚徒困境）+认知（主角是说谎者） | 投票即生死 |
| 别叫我恶魔 | 生存（魔灾）+情感（夭夭生病）+资源（没钱） | 经济贫困+家人需要钱 |
| 夜无疆 | 生存（饥饿）+情感（邻居挨饿）+资源（无粮） | 不找食物就死 |
| 神明调查报告 | 生存（被反复拉回）+认知（伤害带到现实）+社会（被认为是精神病） | 诅咒强制 |

### 危机设计检查清单

- [ ] 本幕有≥2个危机维度叠加？
- [ ] 至少1个危机维度有倒计时/不可逆后果？
- [ ] 危机之间是交织的而非平行的？
- [ ] 主角是"不得不行动"而非"想行动"？
- [ ] 危机烈度达到"不死就死"级别？（前3章尤其重要）

## 九、金手指巧思设计

> 番茄Top20拆解发现：19/19本的金手指都有"巧思"——不只是"能做什么"，而是设计本身能产生决策困境/剧情发动机。
> 金手指不是"翻译器"（展示信息），而是"冲突生成器"（产生选择/博弈/风险）。

### 什么是"巧思"

巧思 = 金手指的设计本身能产生以下至少一项：

| 巧思类型 | 定义 | 番茄Top20案例 |
|---------|------|-------------|
| 决策困境 | 使用金手指时必须做选择，不是"用了就变强" | 每日一卦三选一（乱世荒年）——选了小吉就失去大凶的机会 |
| 使用博弈 | 金手指的使用本身是策略博弈 | 一分钟时停（时停起手）——什么时候用、怎么用是战略决策 |
| 风险代价 | 使用金手指有风险/代价 | 模拟死亡=现实死亡（我的诡异人生）——用之前要掂量 |
| 信息差武器 | 金手指提供的信息差能直接产生社交博弈 | 前世刑侦知识（大奉打更人）——在仙侠世界用推理碾压 |
| 自生长能力 | 金手指本身能长出无限剧情 | 说谎者规则（十日终焉）——一条规则驱动5章博弈 |
| 成长节奏控制器 | 金手指的设计控制主角成长速度，保证爽文节奏 | s级天赋唯一性（游戏入侵）——抢到就是最强，驱动倒计时抢跑 |

### 金手指设计反模式（不合格）

| 反模式 | 问题 | 番茄Top20中没有一本这样做 |
|--------|------|------------------------|
| 纯翻译器 | 只展示数据/属性/技能信息，不产生决策 | 数据面板如果只弹数据=纯翻译器 |
| 纯正向循环 | 用→变强→用→变强，没有选择/博弈/风险 | 无限刷怪升级=纯正向循环 |
| 无使用场景限制 | 任何场景都能用，没有"什么时候用"的决策 | 无限时停=无聊 |
| 成长速度过慢 | 前5章主角还在"新手村" | 前3章应觉醒+升级+翻盘三连 |

### 巧思设计检查清单

- [ ] 金手指能产生决策困境吗？（使用时要选择，不是无脑用）
- [ ] 金手指的使用有博弈/策略空间吗？（什么时候用/怎么用是决策）
- [ ] 金手指的信息差能产生社交博弈吗？（不只是战斗中用，社交/谈判/探索也能用）
- [ ] 金手指能自生长剧情吗？（一条设定能驱动多章事件）
- [ ] 主角前3章成长速度跟得上爽文节奏吗？（觉醒+升级+翻盘三连）

### 游戏入侵流金手指设计指引

游戏入侵流的金手指通常是面板/系统/数值类。容易陷入"纯翻译器"反模式。设计时注意：

1. **面板不只是展示数据，要产生决策**：面板显示敌人属性→主角要决定"打还是跑"→属性差距让决策有张力
2. **数值变化要有选择**：升级时面板显示"技能点+1"→主角要选择分配到哪个属性→不同分配导致不同build
3. **信息差要扩展到非战斗场景**：面板在社交中也能用→评估对方属性→判断对方是否在说谎→社交博弈
4. **成长节奏要匹配爽文**：前3章至少完成觉醒→首杀→升级→越级击杀四步。不能5章还在1级

## 十、微beat节奏锚点

- 开场beat不能纯氛围——第一个beat必须有信息钩子
- 情绪beat必须带钩——情绪中埋一个"为什么"
- 行动beat必须带收益——行动后必须有可数反馈
- 章末beat必须是主动决策——不是被动悬念，是主动选择
- 纯行动beat链=干瘪；纯情绪beat链=拖沓；8种交替=节奏

## 十一、资源获取设计

> 爽文的核心循环：行动→资源回报→变强→更强行动。如果资源获取没有设计好，主角的成长就是"无源之水"。

### 资源获取嵌入世界，而非任务结构

**原则**：资源获取活动应该是主角主动选择的行为，嵌入在世界场景中，而非NPC给的"任务"。

**为什么**：NPC给任务→做任务→拿奖励=MMORPG式任务结构，在小说中显得生硬。主角去码头撬锁不是因为有人给了他"撬锁任务"，是因为他是盗贼，撬锁是他变强的方式。

**怎么做**：

1. **资源获取活动来源于世界观**：码头区有走私仓库→有锁可撬+有陷阱可拆+有货物可偷。墓园区有尸体→有骷髅可打+有随葬品可拿。下水道有变异生物→有怪物可杀+有草药可采。
2. **主角主动选择去获取**：不是"接到任务去码头"，是"主角判断需要变强→知道码头有锁可撬→主动去码头"。
3. **每次获取同时产出多重收益**：撬锁不只获得职业经验，还可能翻到金币/情报/物品。杀人不只获得杀戮经验，还翻尸体获得财物+线索。

### 三级事件分类

设计章节流时，将事件分为三级，确保资源获取有节奏：

| 级别 | 特征 | 资源产出 | 出现频率 |
|------|------|---------|---------|
| 主线 | 推进核心目标，不可跳过 | 高（大量经验+关键物品/情报） | 每幕3-5个节点 |
| 支线 | 提供资源/情报/关系，可跳过但变弱 | 中（职业经验+金币+NPC关系） | 每幕2-4个 |
| 日常 | 可重复的低风险活动 | 低（少量经验+少量金币） | 嵌入章节内部，不独立成章 |

### 面板即时反馈

每次资源获取活动后弹出面板，强化"行动→回报"的即时满足感：

| 活动 | 面板弹出内容 |
|------|------------|
| 撬锁成功 | "获得15点职业经验" |
| 拆陷阱成功 | "获得50点职业经验" |
| 背刺击杀 | "获得30点杀戮经验" + 伤害结算面板 |
| 升级 | 完整属性面板 + 新技能/属性点 |
| 翻尸体 | 物品列表 + 金币计数 |

**禁止**：资源获取活动后无面板反馈=读者感受不到主角在变强。

### 世界级事件管理报警

主角的资源获取活动（尤其是杀人）会推高"系统关注度"。用世界级事件来管理关注度，比"杀多了引来增援"更优雅：

- 主角灭门帮派→传说度提升→但更大的世界事件（邪神信徒活动/妖兽入侵/神灵陨落）转移了强者注意力→主角获得安全窗口
- 这不是"杀人没有后果"，是"在更大的格局面前，贫民区的事不值一提"
