# Step 1 - 素材补强（每幕前置）

> 每幕幕纲设计前的必选环节。对本幕做一轮广撒网式的外部素材收集 + 内部涌现。
> **核心原则**：这一步只管收集不管选用。原始燃料越多越广越好。选中燃料是Step 2幕纲设计时做的事。
> **红线**：跳过素材补强直接写幕纲=不合格。

## 设计哲学

```
Step 1（本步骤）          Step 2（幕纲设计）
原始燃料库                选中燃料 → 改造 → 融入剧情白描
├── 方向1.md  (广)        ↑
├── 方向2.md  (广)        幕纲设计师从原始燃料库中
├── 方向3.md  (广)        选取对本幕有用的素材
├── ...                   深度改造后融入剧情
└── 索引.md               不是照搬，是改造
```

**为什么分开**：在燃料补强阶段就做"选中"决策太早了——此时还没有幕纲的完整上下文（幕定位/弧线/冲突），无法判断哪些素材真正有用。幕纲设计师在写剧情白描时才有完整上下文，才能做出最好的选取和改造决策。

## 1a. 加载本幕上下文

读取：
- `seed-种子文档.md`（必读——世界观/主角/金手指/禁区/核心卖点）
- `燃料库/原始燃料/`（如有前序幕——检查索引，避免重复搜索）
- 已完成的前序幕纲（如有——确保衔接）

## 1b. 方向规划

> **核心**：根据书的特性全面思考搜索方向。不固定数量，不固定类别。
> 专业作者调研可能几十万字——方向要足够多足够广。

从以下11个维度出发，根据本书特性选择8-15个方向进行搜索：

### 方向选择原则

1. **书的特性驱动**：先列出本书的所有特性标签（类型/赛道/主角身份/金手指/开局/节奏/目标读者），每个特性至少推导出1个搜索方向。不是从维度表里挑——是从书的特性出发，用维度表检查是否遗漏
2. **原作IP深度优先**：写同人，原作是最核心的燃料来源。至少3-5个方向聚焦原作（叙事结构/战斗体系/势力格局/经典场景拆解/原作时间线衔接）
3. **跨媒介IP内化**：同题材的经典影视IP不是"看看就好"，是要内化参考——提取结构模式→换皮→融入本书。至少2-3个方向搜跨媒介IP（如加勒比海盗/黑帆/刺客信条黑旗/真实海盗历史）
4. **战斗与成长必选**：爽文的核心是战斗和成长，至少2个方向聚焦此维度（战斗描写技法+成长反馈设计）
5. **主角前世身份必搜**：穿越者的前世身份是独有燃料源，至少1个方向搜前世职业技能如何在异世界发挥
6. **方向之间不重叠**：每个方向有独立的搜索目标，不重复
7. **宁多勿少**：专业作者调研可能几十万字。8-15个方向是底线，发现新方向随时补充

## 1c. 外部搜索（主agent召唤子agent执行）

> **架构约束**：API调用无联网能力。外部搜索由主agent召唤Explore子agent执行WebSearch。
> **红线**：跳过外部搜索直接写幕纲=不合格。

### 执行流程

1. **主agent按方向规划逐个搜索**：每个方向一次WebSearch调用（或WebFetch获取详情）
2. **每个方向独立落盘**：搜索结果分文件保存到 `燃料库/原始燃料/`，不要聚合在一个文件里
3. **搜索要深不要浅**：每个方向至少获取3-5条有效素材，不是2-3条就收工
4. **如果搜索结果不够**：换关键词重搜，或用WebFetch深入获取详情

### 子agent调用规范

主agent调用Explore子agent时，prompt格式：
```
搜索方向：{方向名称}
搜索关键词：{2-3个关键词}
搜索目标：获取和"{本书核心特性}"相关的{方向}素材
本书上下文：{一句话描述本书的核心设定和当前幕的冲突主题}
要求：
1. 返回3-5条搜索结果的内容摘要（每条150-300字），不是链接列表
2. 每条标注源体量层级（卷级/幕级/章级）
3. 每条标注素材类型（叙事结构/战斗技法/设定细节/冲突模式/氛围营造/角色原型）
4. 重点提取可转化为小说创作的结构性素材，不是百科科普
```

### 落盘文件结构

每个方向一个文件：`燃料库/原始燃料/方向NN-{方向名}.md`

```markdown

- **可提取的燃料**：（转化为小说创作可用的结构/元素/技法）
```

## 1d. 内部涌现

> 在外部搜索之后做。结合外部素材和已有设定，推导新可能性。
> 不是罗列已有燃料，而是**从设定+外部素材的碰撞中长出新东西**。

### 涌维度1：危机维度扫描

检查本幕当前有哪些危机维度，缺哪些：

| 危机维度 | 本幕是否有？ | 如果没有，能从设定中推导吗？ |
|---------|------------|--------------------------|
| 生存危机 | | |
| 情感危机 | | |
| 社会危机 | | |
| 认知危机 | | |
| 资源危机 | | |

**要求**：本幕至少叠加2个危机维度。

### 涌维度2：金手指巧思扫描

| 检查项 | 达标？ |
|--------|-------|
| 金手指在本幕有产生决策困境吗？ | |
| 金手指的使用本身有风险/代价/博弈吗？ | |
| 金手指的信息差优势在本幕被利用了吗？ | |
| 主角成长速度跟得上爽文节奏吗？ | |

### 涌维度3：燃料扩展推导

从种子文档设定 + 外部搜索素材的碰撞中推导新燃料：

| 已有设定/外部素材 | 碰撞产生的新燃料 | 来源 |
|---------|----------------|-----------|
| 种子设定#X + 外部素材#Y | 从两者的结合点推导出的新事件/冲突/设定 | 设定×素材涌现 |

**要求**：至少推导出2条新燃料。

### 内部涌现落盘

内部涌现结果保存到 `燃料库/原始燃料/方向00-内部涌现.md`

## 输出

1. `燃料库/原始燃料/方向NN-{方向名}.md`（每个方向一个文件，含搜索原文摘要+燃料提取）
2. `燃料库/原始燃料/方向00-内部涌现.md`（内部涌现记录）
3. `燃料库/原始燃料/索引.md`（总索引，供Step 2查阅）

**不输出补强清单**——选中燃料是Step 2幕纲设计时做的事。

输出后进入 `steps/step-2-act.md` 幕纲设计。幕纲设计时**必须读取原始燃料库**，从中选取并改造素材融入剧情白描。
