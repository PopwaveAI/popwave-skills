# Step 2 - 幕纲（5步设计链路）

> **核心原则**：先有故事，再拆维度。剧情白描是核心——写不好，后面展开全是浪费时间。
> **两阶段设计**：阶段A快速出方向让用户挑选，阶段B基于选定方向走5步链路。
> **体量要求**：每幕8~15章完整体量。幕末必须有可验证的质变。

---

## 阶段A：方向选择（简版多方案）

> 阶段A的目的是快速产出2-3个不同方向的剧情走向，让用户用人的判断力挑选最有张力的那个。
> **不展开细节**——每个方案控制在半页纸以内。不求完整，求方向清晰、张力明确。

### 2A-a. 加载上下文

读取：
- `seed-种子文档.md`（必读——世界观/主角/金手指/禁区/核心卖点/背景暗流）
- `主角引擎.md`（必读——主角行动偏好/阵营操作性定义/第一章主动动作）
- `角色储备池.md`（必读——S/A/B/C角色+反应模式+可制造事件+主角可利用点）
- `世界圣经.md`（按需——力量体系/三阶社会结构/势力格局/资源物品/核心矛盾）
- `燃料库/原始燃料/索引.md`（必读——浏览原始燃料索引，了解可用素材池全貌，快速判断哪些方向对本幕有用）
- `燃料库/原始燃料/方向NN-*.md`（按需深入——对索引中判断为有用的方向，读取详情获取可提取的燃料）
- `references/设计原则库/`（快速浏览5个文件的开局/危机/成长/情感/战斗原则标题，不求细读）
- `prose-render/references/章型定义.md`（快速浏览6种章型的使用条件，不求细读）
- 已完成的前序幕纲（如有——确保衔接）

**红线**：
- 如果原始燃料库（`燃料库/原始燃料/索引.md`）不存在，停止。回到Step 1完成素材收集后再进入幕纲。
- 如果主角引擎/角色储备池不存在，停止。回到seed完成阶段二（世界展开）后再进入幕纲。

### 2A-b. 产出2-3个候选方向

每个方向只需要以下内容（简版，不展开）：

---

#### 候选方向N

**幕定位**（3行）：
- 本幕作用：（一句话——这一幕要解决什么/建立什么/转变什么）
- 核心体验：（一句话——读者读完应该获得什么感受）
- 情感弧线：从____ → 经过____ → 以____结束

**剧情方向梗概**（3-5句话描述整幕故事弧线，不分章。这是方向感描述，不是详细白描）

**幕末质变**（1行）：主角什么不可逆变化 + 剧情什么格局变化

---

### 2A-c. 方案差异化要求

2-3个方向必须有**实质性差异**，不是同一个思路的微调：

| 差异维度 | 说明 |
|---------|------|
| 核心冲突不同 | 方案A的矛盾是X，方案B的矛盾是Y，不是同一矛盾的不同包装 |
| 压力来源不同 | 方案A的压力来自生存，方案B的压力来自情感，方案C的压力来自认知 |
| 角弧方向不同 | 方案A的主角从被动到主动，方案B的主角从冷酷到有羁绊 |
| 燃料选取不同 | 方案A选取燃料库方向03的结构，方案B选取方向05的模式，方案C纯自主涌现 |

**红线**：如果2-3个方向看起来"都差不多"，说明没有真正思考不同方向。重来。

### 2A-d. 输出与等待确认

将2-3个候选方向输出到 `卷纲/幕NNN-候选方案.md`，然后**暂停，等待用户选择**。

用户选定后进入阶段B。

---

## 阶段B：5步设计链路

> 基于用户选定的方向，走以下5步。所有思考沉淀到 `卷纲/幕NNN-设计笔记.md`。
> **设计笔记是agent的思考记录，供追溯和自检用，不直接给prose-render消费。**

---

### 第1步：定位与起点快照

**1a. 本幕位置**

- 本幕是第X卷第Y幕
- 前序剧情回顾（如有——一段话概括之前发生了什么，确保衔接）
- 本幕在全书的功能位置（开篇/发展/转折/高潮/收束）

**1b. 起点状态快照**

| 维度 | 当前状态 |
|------|---------|
| 身体 | （虚弱/健康/受伤等） |
| 面板 | （等级/属性/经验值/技能） |
| 物质 | （金币/装备/药剂/资源） |
| 情报 | （知道什么/不知道什么） |
| 社会关系 | （盟友/敌人/中立/可利用的人） |
| 外部威胁 | （当前面临的威胁） |
| 前世知识 | （知识的可靠度/已知偏差） |

---

### 第2步：确认本幕目的

**2a. 三层目标**

| 目标层 | 内容 |
|--------|------|
| 剧情推演目标 | 本幕结束时剧情格局是什么？（如"人贩子网络从隐藏威胁变成明确敌人"） |
| 主角成长目标 | 本幕结束时主角的等级/能力/资源状态？（如"1级→3-4级盗贼，建立初步情报网络"） |
| 读者体验目标 | 本幕结束时读者获得了什么体验？建立了什么期待？（如"专业猎杀的爽感+对动荡时代来临的期待"） |

**2b. 开局契约**（仅第一幕需要填写）

| 项 | 内容 |
|----|------|
| 第一章弱势 | 主角开局的弱势状态（身体/资源/情报/社会地位） |
| 触发条件 | 什么事件打破弱势状态，触发主角行动 |
| 第一主动操作 | 主角主动做的第一件事（不是"发生什么"，是"主角主动做什么"） |
| 第一可见反馈 | 主角主动操作后获得的第一可见反馈（面板/资源/情报/声名） |
| 第一收益或损失 | 主动操作带来的收益或代价 |
| 新压力 | 主动操作后引来的新压力（不是解决压力，是制造新压力） |
| 禁止提前兑现 | 本幕不能提前给出的后续收益（防止透支高潮） |

---

### 第3步：剧情白描（核心）

> **这是整个幕纲设计的核心环节，占阶段B 50%的注意力。**
> **这部分写不好，后面展开全是浪费时间。**
>
> 采用**三段差异化白描→评估→整合**流程：先产出3段方向不同的白描，经agent自评+作者挑选后整合为1版最终白描。R33测试证明：不同方向的白描各有优势（有的弧线密度高、有的挫折质量好、有的原创性强），整合后取长补短，质量远超单次产出。

#### 3a. 三段差异化白描

同时产出3段白描，每段必须有**实质性差异**（不是同一思路的微调）。

**差异化维度**（3段白描至少在以下3个维度上各自不同）：

| 差异维度 | 说明 | 示例 |
|---------|------|------|
| 核心弧线构成不同 | 不同的事件组合 | A=觉醒首杀+码头练级+灭门帮派 / B=追踪疤脸+潜入当铺+码头清剿 / C=葬狗+暗杀恶棍+练级+灭门 |
| 参考素材侧重不同 | 不同的参考作品或结构 | A侧重飓风营救的逐层追查 / B侧重原著的灭门练级 / C纯自主涌现 |
| 资源获取路径不同 | 不同的成长路线 | A靠撬锁+杀人 / B靠拆陷阱+灭门搜刮 / C靠下水道怪物+帮派据点 |
| 挫折设计不同 | 不同的挫折类型 | A=知识偏差误判 / B=情报贩子出卖遭伏击 / C=仓库暴露被迫转线 |

**每段白描的要求**：

- 800-2000字/段
- 包含起因→发展→高潮→收束的完整弧线
- 写的是"发生了什么故事"，不是"这一幕的主题是什么"
- 主角必须主动驱动剧情，不能被推着走
- 幕末必须有质变（不是原地踏步）
- 资源获取活动自然嵌入剧情
- **多视角叙事**：白描中必须包含至少1个非主角视角的叙事段落（敌人视角/旁观者视角/配角视角）。全程锁主角视角会让故事单调——在Boss战/新势力登场/主角不在场的关键事件中切换视角。切换不是"主角猜到敌人在想什么"，是真正切到敌人的眼睛看这场戏

#### 3a-燃料. 燃料选取与改造（每段白描必做）

> **这是Step 1原始燃料库的消耗端。** 燃料不是照搬——是"选取→改造→融入"。
> 3段白描各自选取不同的燃料组合，是差异化的重要来源。

**选取**：从原始燃料库中为本段白描选取≥2条素材。选取依据：本段白描的核心弧线需要什么支撑（战斗结构/冲突模式/氛围营造/角色原型/叙事技法）。

**改造**（不是照搬，是内化）：

| 改造操作 | 说明 | 示例 |
|---------|------|------|
| 提取结构 | 只取骨架模式，不取具体内容 | 从黑帆提取"海盗民主投票"的结构模式，换皮为海贼团内部的船员投票 |
| 换皮融合 | 保留结构，替换为本书世界观元素 | 从加勒比海盗的"诅咒契约"结构，换皮为海贼王的恶魔果实副作用 |
| 降维适配 | 大体量作品只取幕级子结构 | 从冰与火之歌铁群岛线只取"铁种文化"的底层逻辑，不取具体剧情 |
| 碰撞合成 | 两条燃料交叉产生新设计 | 帧级战斗拆解 × 游戏策划数值直觉 = 主角用数值思维拆解对手动作 |

**源体量层级决定提取粒度**：

| 源体量 | 可提取的粒度 | 不可提取 |
|--------|------------|---------|
| 卷级 | 模式/法则/主题处理 | 具体剧情/角色/场景 |
| 幕级 | 结构骨架/弧线模式/冲突升级路径 | 具体事件/台词 |
| 章级 | 具体技法/场景写法/节奏控制 | 原文照搬 |

**融入**：改造后的燃料自然嵌入白描叙事中，不是"这里用了一个参考"的注释式插入。读者看不到参考痕迹，只看到好故事。

**燃料选取记录**（写入设计笔记）：

| 段落 | 选取的燃料 | 来源方向 | 改造方式 | 融入位置 |
|------|-----------|---------|---------|---------|
| 白描A | 燃料1: XXX | 方向03 | 提取结构→换皮 | 弧线1的冲突升级 |
| 白描A | 燃料2: XXX | 方向05 | 碰撞合成 | 弧线2的战斗设计 |
| 白描B | 燃料3: XXX | 方向01 | 降维适配 | 弧线1的氛围营造 |
| ... | | | | |

**红线**：3段白描中至少有2段选取了不同的外部燃料。如果3段白描的燃料选取完全相同，说明差异化不足。

**故事容量要求**（R33测试发现的核心问题）：

一幕8-15章的体量，白描不能只讲一个任务。必须包含**2-4条故事弧线**，每条弧线有独立的核心事件和爽点。

| 弧线数 | 适用场景 | 示例 |
|--------|---------|------|
| 1条 | 不够——一个任务撑不起8-15章 | ❌ 只写"追踪疤脸→杀疤脸→发现更大威胁" |
| 2-3条 | 标准容量 | 情感弧线（葬狗/照顾妹妹）+ 成长弧线（撬锁练级）+ 战斗弧线（灭门帮派） |
| 4条+ | 高密度 | 葬狗+暗杀恶棍+码头练级+灭门帮派+发现世界级事件 |

**每条弧线必须具备**：
- 独立的核心事件（不是同一件事的不同阶段）
- 独立的爽点（每条弧线至少1个）
- 独立的资源获取（不同弧线获取不同类型资源）

**成长分散要求**：

主角的等级提升不能集中在一次事件中跳跃（如"一次杀戮连升3级"）。必须**分散在多个活动中**：

| 错误做法 | 正确做法 |
|---------|---------|
| 杀疤脸一次获得足够经验1→4级 | 撬锁拆陷阱升1级(职业经验) + 暗杀2恶棍升1级(杀戮经验) + 灭门帮派升1级(杀戮经验) |
| 所有经验来自杀人 | 杀戮经验(杀人) + 职业经验(撬锁/拆陷阱/偷窃) + 物质资源(翻尸/搜刮/偷窃) 三线并行 |

白描中必须体现多个资源获取节点，每个节点产出不同类型资源。

**挫折质量要求**：

挫折不能只是"换个方式再试一次"。每次挫折必须**推动故事转向新方向**：

| 低质量挫折 | 高质量挫折 |
|---------|---------|
| 第一次暗杀失败→准备毒药再试一次 | 眼线被撞死→被迫转向新线索(妓院)→救错人→被迫转向安全屋 |
| 挫折只是延迟成功 | 挫折改变故事方向，制造新的剧情分支 |

白描中至少1-2次挫折，且每次挫折推动主角转向新的故事弧线或新的解决路径。

**幕级起伏节奏要求**：

白描输出的是一幕完整剧情，此时不做章级拆分。但必须在幕级保证战斗和爽点的起伏节奏——不能平铺直叙到最后才爆发。

| 节奏维度 | 要求 | 反面案例 |
|---------|------|---------|
| 前期爆发 | 白描前1/3必须有第一个爽点爆发（首杀/面板激活/越级击杀/信息差碾压） | 前1/3全是铺垫和探索，到中期才打第一场 |
| 战斗分布 | 白描中至少3场战斗，分布在前期/中期/高潮三个位置 | 全部战斗集中在幕末高潮 |
| 爽点分布 | 白描中至少4个爽点节点，分布在前/中/后/末，不得集中在一处 | 爽点全堆在高潮段 |
| 起伏交替 | 必须有"低压→爆发→低压→更大爆发"的交替节奏 | 从头到尾一个节奏，或持续高压无喘息 |
| 首杀时机 | 首杀（无论杀骷髅还是杀人）必须在白描前期完成 | 首杀放在幕末高潮 |

**每段白描的质量自检**（3段都需通过）：

| 检查项 | 不达标则重写 |
|--------|-------------|
| 剧情有内在逻辑吗？ | 不是事件的堆砌，是因果链 |
| 主角主动吗？ | 主角在驱动事件，不是事件推着主角走 |
| 有爽点吗？ | 每条弧线至少1个让读者"爽"的时刻 |
| 有挫折和转折吗？ | 每次挫折推动故事转向新方向，不是"换个方式再试" |
| 幕末有质变吗？ | 主角或格局有不可逆变化 |
| 读起来像故事吗？ | 不是大纲/提纲/维度列表，是故事 |
| **故事容量够吗？** | **2-4条故事弧线，不是1个任务撑全幕** |
| **成长分散吗？** | **等级提升分散在多个活动中，不是一次跳跃** |
| **资源多元吗？** | **杀戮经验+职业经验+物质资源三线并行，不是只靠杀人** |
| **有多视角吗？** | **至少1个非主角视角段落（敌人/旁观者/配角），不是全程锁主角** |
| **前期有爆发吗？** | **白描前1/3有爽点爆发（首杀/面板/越级击杀），不是慢热到最后** |
| **战斗分布合理吗？** | **≥3场战斗分布在前/中/末，不是全堆在高潮** |

#### 3b. 评估

对3段白描做结构化评估，为整合提供依据。

**agent自评**：逐段分析以下维度，标注每段的优势和弱点：

| 评估维度 | 说明 |
|---------|------|
| 弧线构成 | 哪几条弧线？密度够不够？哪段最好？ |
| 成长曲线 | 升级节点分散度？最终等级是否合理？哪段最合理？ |
| 挫折质量 | 挫折次数？是否推动转向？哪段挫折最好？ |
| 资源多元 | 杀戮/职业/物质三线是否并行？哪段最多元？ |
| 爽点密度 | 爽点数量和质量？哪段最爽？ |
| 原创性 | 是否有从设定自然涌现的原创设计？哪段最新颖？ |
| **燃料改造质量** | **选取了哪些燃料？改造方式是否到位（不是照搬）？融入是否自然（读者看不到参考痕迹）？哪段改造最好？** |
| **燃料多样性** | **3段白描是否选取了不同方向的燃料？还是都集中在同一方向？** |

**作者评估**：将3段白描+agent自评呈现给作者，由作者：
- 挑选最认可的方向作为整合基底
- 指定要从其他段中吸收的元素
- 或提出新的调整方向

#### 3c. 整合

基于agent自评+作者反馈，整合出1版最终白描。

**整合原则**：
- 选1段作为基底（通常选综合评分最高的）
- 从其他段中吸收其优势元素（弧线/挫折/资源路径/原创设计）
- **燃料整合**：从3段白描的燃料选取中，保留改造质量最好的燃料组合，放弃生硬照搬的。整合后的燃料使用必须自然融入故事，读者看不到参考痕迹
- 整合后不是简单拼接，是重新写一版完整故事
- 整合版必须通过10项质量自检
- 整合版字数800-2000字

**整合版示例**（R33测试中E/F/G的整合思路）：
- 基底选F（3条线索链逐层追查，挫折质量最高）
- 从E吸收"4条弧线密度"（E的清除同伙弧线可补入F）
- 从G吸收"下水道变异老鼠+剥皮事件"（原创资源区+世界级事件管理报警）
- 整合后：F的3条线索链 + E的清除同伙弧线 + G的下水道资源区和剥皮事件 = 4-5条弧线的完整白描

**红线**：整合版白描不达标时禁止进入第4步。带着烂地基往后走=全盘报废。

### 第4步：反拆维度

> 从第3步的剧情白描反拆幕纲维度。不是先设计维度再拼剧情，是**先有故事再拆维度**。
> 如果反拆过程中发现剧情有缺口（密度不够/资源缺口/人物不够），评估是**反向调整第3步的剧情白描**，还是**调整第4步的维度设计**。

| 时间范围 | 剧情在多长时间内发生？ |
| 出场人物 | 谁必须出场？ |
| 潜在可出场人物 | 角色储备池中谁可以出场？ |
| 主角战斗 | 哪些战斗？对手什么等级？ |
| 主角成长 | 哪些成长节点？等级变化曲线？ |
| 资源事件 | 剧情中嵌入了哪些资源获取活动？（撬锁/拆陷阱/偷窃/杀人/搜刮） |

#### 4a-燃料. 燃料利用率检查

> 检查原始燃料库中的素材在剧情白描中的利用情况。不是每条燃料都要用——是确保有用的燃料没有遗漏，没用到的明确标注原因。

| 燃料库方向 | 选取的燃料 | 改造方式 | 融入白描的哪个弧线 | 利用质量 |
|-----------|-----------|---------|------------------|---------|
| 方向01 | | | | 已用/未用（原因） |
| 方向03 | | | | 已用/未用（原因） |
| ... | | | | |

**检查标准**：
- 白描中选取的燃料≥2条（与3a-燃料的选取记录对齐）
- 每条选取的燃料都经过了改造（不是照搬）
- 未选取的燃料有明确原因（"本幕不需要"或"留给后续幕"）
- 如果发现白描中遗漏了高价值燃料：反向调整第3步白描，补充融入

**红线**：如果白描中完全没有使用原始燃料库中的任何外部素材，说明Step 1的燃料收集白做了或Step 2没有读取燃料库。回到2A-a重新加载燃料库。

#### 4g. 幕级密度测算

**核心检查**：把剧情白描的事件铺到章节预算上，测算幕级密度是否达标。

> 白描阶段尚未分章，章级指标（每章事件密度/每章爽感频率等）在第5步分章时检查。此处只检查幕级。

| 密度指标 | 要求 | 达标？ |
|---------|------|-------|
| 白描故事弧线数 | 2-4条 | |
| 白描战斗总数 | ≥3场 | |
| 白描爽点总数 | ≥4个 | |
| 前期爆发节点 | 前1/3有爆发 | |
| 起伏交替节奏 | 有低压→爆发的交替 | |

**如果密度不够**：
- 选项A：反向调整第3步的剧情白描——增加事件/转折/挫折
- 选项B：调整第4步的章节预算——减少章数让事件更密集
- 评估哪个调整更合理，不要只调一个维度

> 章级节奏红线（前3章首杀/连续2章无面板/章型交替等）移至第5步分章时执行——白描阶段尚未分章，无法检查章级节奏。此处只检查幕级节奏。

| 红线 | 要求 | 达标？ |
|------|------|-------|
| 前期爆发 | 白描前1/3有爽点爆发 | |
| 战斗分布 | ≥3场，分布在前/中/末 | |
| 爽点分布 | ≥4个，不集中在一处 | |
| 首杀时机 | 首杀在白描前期 | |
| 起伏交替 | 有低压→爆发的交替节奏 | |

#### 4i. 幕末质变自检

5项检验（体量/主角质变/剧情质变/质变铺垫/未透支后续）。

#### 4j. 6道门禁自检

事件分量/角色地基/期待感/冲突驱动/危机多维/金手指巧思。

#### 4k. POV规划

从白描中提取自然出现的POV切换 + 补充规划。此表直接植入施工卡的"角色行为锚点"部分，prose-render照执行。

| 章 | 主视角 | 需切换的POV | 切换目的 | 切换时机 |
|----|--------|------------|---------|---------|
| | | | 侧面定级/放大爽感/制造信息差/建立情感 | |

**POV切换原则**：
- 从白描中自然出现的视角切换优先提取——白描里写了"疤脸在酒馆不知道死神靠近"，就提取为chXX的敌人视角
- 补充规划时参照流派skill的 `多线叙事.md` 规则（如boss战必切boss视角/新势力登场必切1段/主角不在场必切在场者）
- 每幕至少1个非主角视角段落（与白描自检第10项对齐）
- 切换目的必须明确——没有目的的切换不规划

**设计笔记末尾附一行**：`已通过全部检验（密度/节奏/质变/门禁/POV）。`

### 第5步：分章概览

> 把第4步的反拆信息植入每一章，实现每章剧情完整。
>
> **白描是骨架，分章是长肉。** 白描提供一幕的完整故事弧线和核心事件，但不涵盖每章的具体细节。分章时创作者有充分自由度补充内容，同时必须满足番茄节奏红线。

#### 5a. 创作自由度

分章时创作者可自行补充以下内容（白描中未明确但分章需要的）：

| 可补充类型 | 说明 | 示例 |
|-----------|------|------|
| 小BOSS/精英遭遇 | 白描只写了大事件，分章时可在过程中插入精英怪/小头目的遭遇战 | 白描写"灭血帮小队"，分章补充血帮有一个2级精英打手拦路 |
| 额外剧情起伏 | 白描是主线弧线，分章时可补充支线起伏让节奏更紧凑 | 白描写"墓园练级"，分章补充墓园中遇到一个神秘NPC |
| POV切换点 | 白描只在关键事件处标注视角切换，分章时根据每章内容规划具体POV | 白描写"敌人视角看索伦猎杀"，分章确定这是ch6的哪个位置切入 |
| 钩子/悬念设计 | 白描关注大弧线，分章时每章的章末钩子由创作者设计 | 白描写"主角暴露身份"，分章设计暴露后的章末悬念 |

**约束**：补充内容必须服务于白描弧线和节奏红线，不得偏离白描主线，不得为凑字数注水。

#### 5b. 番茄节奏红线执行

分章时必须逐项检查并满足以下红线。这些红线在白描阶段无法检查（因为尚未分章），在分章阶段是硬约束。

| 红线 | 要求 | 达标？ |
|------|------|-------|
| 前3章完成觉醒+首杀 | 觉醒→首杀→升级在前3章 | |
| 每3章至少1个combat_reversal | 3章不打架读者会弃 | |
| 前5章有升级 | 前5章无升级=过慢 | |
| 铺垫不超过2章才爆发 | 最多1章铺垫+1章爆发 | |
| 连续2章无面板/战斗 | 至少每2章弹1次面板 | |
| 不得连续2章同章型 | 章型交替 | |
| 战斗模式不连续3章同种 | 切换战斗模式 | |

**如果红线不达标**：
- 通过5a的创作自由度补充内容来满足节奏（如前3章缺首杀→在ch2补充一场遭遇战）
- 不得通过删减白描核心事件来满足节奏
- 不得为满足节奏而注水（如硬塞一场无意义的战斗）

#### 5c. 分章概览表

| 章 | 章名 | 核心事件 | 章型 | 主动行为 | 资源获取 | 倒计时 |
|----|------|---------|------|---------|---------|--------|
| | | | | | | |

**分章原则**：
- 每章必须有完整的核心事件（不是片段）
- 每章必须有主动行为（主角在驱动）
- 资源获取活动自然嵌入章节（不是单独列出的"练级章"）
- 章型交替（不得连续2章同章型）
- 战斗模式交替（不得连续3章同种）
- **白描中的核心事件是硬锚点，不可删减；分章补充的内容是软填充，可调整**

## 正式幕纲输出

> 从5步链路的产出中提取正式幕纲，输出到 `卷纲/幕纲-幕NNN-{标题}.md`。
> **只包含下游需要的信息，不包含思考过程。** 控制在200行以内（含剧情白描全文）。
> 严格按照 `templates/幕纲-模板.md` 的结构输出。以下为各区块的内容来源：

#### 一、幕定位
从阶段A选定的方向继承。包含：本幕作用、核心体验、情感弧线、**幕末质变**（从4i提取，前置到此处）。

#### 二、剧情白描
第3步3c的整合白描全文（~1500-2000字）。不是压缩梗概。必须包含多弧线交织、多视角段落、世界观线索埋入、后续幕剧情线索埋入。

#### 三、六要素
合并原"主角收益"+"主动行为"+"核心爽点"为统一表格：

| 要素 | 内容 |
|------|------|
| **时间** | 本幕时间跨度 |
| **地点** | 本幕涉及场景 |
| **人物** | 所有有名角色（标注阵营归属） |
| **主要矛盾** | 核心冲突（一句话） |
| **主角收益** | 等级/技能/装备/金币/情报/人脉/身份变化（含具体数值和获得章节） |
| **核心爽点** | 1-3个核心爽点（标注类型和所在章节） |

#### 四、多弧线规划
从白描和4k POV规划中提取：

| 弧线类型 | 内容 | 本幕状态 |
|---------|------|---------|
| **主线** | 本幕主线故事弧 | 展开/推进/收束 |
| **支线/暗线** | 支线剧情 | 埋线/推进/收束 |
| **世界观线索** | 本幕揭示/埋入的世界观信息 | 埋线/揭示 |
| **后续幕线索** | 为后续幕埋入的契诃夫之枪 | 埋线 |

#### 五、POV规划
从4k提取。每章主视角/需切换的POV/切换目的/切换时机。

#### 六、篇幅与节奏设计
从第5步5b继承番茄节奏红线对齐表。

#### 七、章节流概览
从第5步5c继承分章概览表。

#### 八、prose-render交接
- 文风注意（DNA版本/面板格式/主角笔触/设定强调纪律）
- **不可改动事件**（硬锚点列表，编号。必须包含：势力归属/面板数值/角色名称等关键约束）
- 可自由发挥空间
- 爽点外显要求（面板弹出时机/旁观者反应/知识偏差外显）
- 角色行为锚点（主角+配角+反派，含阵营归属和反应模式）

**不输出**：燃料清单/空间设计表/NPC反应表/资源缺口表/危机维度表/冲突设计表/压力链路表/密度测算表/门禁检验表。这些都在设计笔记里。
