# step-1 找燃料 + 题材机制分流

> 本文件是 pop-emergent-research 第一步执行指令。进入前已由 SKILL.md 强保障注入红线。

## 目标

找能入场面的写作燃料，并对参考书里的题材机制做分流：能进本书的转成可触发事件/主角操作点/可外显爽点；不能直接迁移的写入 content-mechanics。本步只产出对话内草稿，不落盘。

## 必读输入与 execution.mode

execution.mode 三档切换条件统一引用 PRD §4.5，本 skill 不重复定义三档表。

- **formal 必读输入**：已读 `涌现/seed-种子文档.md` 或用户提供的临时 seed；燃料不少于 3 条；主燃料有事件形状/主角操作点/可外显爽点。齐全且达标才正式落盘。
- **draft**：有燃料但来源/入戏方式/主角操作点不足，产出标记 draft 仍落盘。
- **trial**：快速补燃料，不落盘，仅对话内。

读取 seed（如已存在）：

```powershell
Get-Content -Encoding UTF8 -Raw 涌现/seed-种子文档.md
```

## 执行

### 1. 燃料采集

从用户提供的资料/参考书/外部事件/行业机制中找可入场面的燃料。每条燃料必须落到三要素，缺一不可：

- **事件形状**：什么事件、在什么场面发生、怎么触发
- **主角操作点**：主角能做什么、怎么主动介入（不是被动遭遇）
- **可外显爽点**：读者能直接看到的爽感外显（不是内心活动）

### 2. 题材机制分流

参考书里的武学招式、系统面板、诡异规则、职业数值、战斗升级路径、组织运营模型，先当"内容机制"处理：

| 判定 | 处理 |
|:-----|:-----|
| 能进入本书 | 转成可触发事件 + 主角操作点 + 可外显爽点，进入本书涌现燃料表 |
| 不能直接迁移 | 记入 content-mechanics.md（来源/能否迁移/正确路由/禁止误用） |
| 疑似文风特征 | 不得交给 soul/write，先在 content-mechanics 标注"禁止误用" |

红线：不把内容机制伪装成文风特征交给 soul/write。

### 3. 燃料分级

- **近期可用**：可直接筛入 current-state 的近期燃料
- **中期保留**：暂不入场但保留
- **禁用**：与本书冲突或风险过高

## 质量门

- 至少 3 条可写燃料（formal）。
- 每条主燃料有事件形状、主角操作点、可外显爽点。
- 内容机制已分流，不进入 soul。
- 不排剧情，不写正文。
