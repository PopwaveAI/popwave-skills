# Pop Emergent Seed

seed 只回答“这个故事靠什么成立、靠什么长期爽、哪些判断不能丢”。它帮助作者判断，不替作者锁死剧情。

## 第一性边界

seed 必须把参考对象拆成三层，不得混写：

| 层 | 问题 | 下游 |
| --- | --- | --- |
| 题材机制 | 这个故事靠什么规则运转？如武学升级、诡异污染、DND面板、职业数值、金手指反馈 | research / 设定库 / content-mechanics |
| 剧情承诺 | 主角会怎样行动，读者期待什么长期变化？ | write 写作包 |
| 表达 soul | 叙事人格、句子气口、对白和信息释放有什么偏好？ | soul.md |

seed 不直接写正文，也不把题材机制写进 soul。若用户给的是“武林半侠传 + 深渊主宰 + 诡异流”这类混合参考，seed 必须先分层：哪些是机制，哪些是剧情承诺，哪些才是表达魂。

## 执行模式

| 模式 | 条件 |
| --- | --- |
| `formal` | 用户输入、不可牺牲项、候选碰撞、推荐理由和待 research 问题齐全 |
| `draft` | 有方向但候选、禁区、主角驱动或世界压迫缺口仍在 |
| `trial` | 只做快速脑暴，不落盘，不称种子完成 |

## 输入

- 用户原始 idea、参考作品、现实事件、角色关系、题材母版。
- 已有项目 seed/世界观/正文锚定，如存在。
- 用户不可牺牲项和禁区。

## 执行

1. 输入消歧：区分参考作品、本书设定名、现实机制、用户原创点子。
2. 种子展开：主角种子、世界种子、爽点种子、类型种子、反差种子。
3. 相似/相反/盲区：只保留会改变故事判断的内容。
4. 候选碰撞：给 2-3 个互斥概念，不能让 B/C 陪跑。
5. 推荐方案：说明胜出理由、败者强点是否吸收、哪些不吸收。
6. 用户断点：除非用户说自动继续，候选后停下确认。

## 质量门

- 有 2-3 个互斥候选，或用户明确只要单方案。
- 有核心快感、主角主动方式、世界压迫和禁区。
- 有待 research 问题。
- 混合参考已分清“题材机制 / 剧情承诺 / 表达 soul”。
- 没有把 seed 越级写成世界设定、卷纲、章纲或正文。
