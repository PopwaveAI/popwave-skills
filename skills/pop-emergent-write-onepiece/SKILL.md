# pop-emergent-write-onepiece

> 海贼王世界冒险流写作引擎。将剧情锚点渲染为番茄小说平台可消费的章节正文。
> **赛道级skill**——只管"海贼王世界的战斗/场景/叙事怎么写"。
> 书级内容（笔触DNA/金手指系统/世界圣经）从项目目录加载，不在skill内。

---

## execution.mode

引用 PRD §4.5。本 skill 的 formal 必读输入：current-state.md（含下一章硬推进+人物状态+燃料队列+伏笔债务）+ dna/ 笔触DNA文件 + 用户本轮要求 齐全且达标。缺 current-state 不得标 formal。三档切换条件见 PRD §4.5，不在此重复。

| mode | 全局条件 | 落盘 |
| --- | --- | --- |
| `formal` | 本 skill 必读输入齐全且达标 | 正式落盘产出 |
| `draft` | 必读输入有缺口但可补全继续 | 产出标记 draft，落盘 |
| `trial` | 快速试做，不称完成 | 不落盘，仅对话内 |

正文完成后，落盘到 `涌现/正文/{书名}-第{N}章-{标题}.txt`，对话中只回摘要+钩子+创作记录。trial 模式不落盘，仅对话内。

## 赛道定义

海贼王世界冒险流的赛道级特征（不依赖具体书籍，任何海贼王世界小说都适用）：

1. **霸气体系战斗**：见闻色（预感/感知）、武装色（硬化/触碰果实者）、霸王色（威压/清场），三色各有写法
2. **恶魔果实战斗**：自然系/动物系/超人系各有战斗逻辑，果实能力者对决有特殊规则（海楼石/海水弱点）
3. **海上战斗**：风浪/船只/接舷/海兽——船舶级战斗是海贼王世界的特色战斗形态
4. **势力交织**：海军/海贼/革命军/世界政府多势力并存，天然支持多线叙事
5. **悬赏金体系**：侧面定级的核心参照物——悬赏金额=实力+威胁度的综合标尺
6. **船员羁绊**：小团体（3-5人）的信任与成长是海贼王世界的核心叙事动力
7. **海贼王战斗风格**：海贼王的战斗有独特质感——**体型差碾压**（巨汉vs瘦小少年）、**夸张体术**（一拳打飞/踩碎地面/牙齿咬断刀）、**意志力外放**（不死不休的拼劲）、**招式即性格**（每个人的战斗方式反映其人设）。即使是底层战斗，也要有海贼王式的"蛮荒感"——这不是武侠的优雅拆招，是街头搏命+超人体质的世界
8. **底层战斗力基线**：海贼王世界的普通人并非弱者——海贼世界的底层杂役也经历过海上搏杀，体格远超现代普通人。主角即使开局底层，其身体素质也应在一个"能挨打、能跑路、会基本搏斗"的基线上，而非手无缚鸡之力。参考海贼法典：主角威廉开局就是海贼团底层，但能挥刀、能游泳、能翻墙、能格挡——他是"弱海贼"而非"普通人"

**不在赛道定义中的**（由project层决定）：主角性格、道德观、金手指类型、笔触风格、力量分级的具体数字。

## 怎么操作（SOP）

### Step 1：加载项目上下文

读取项目的核心文件：

| 文件 | 位置 | 内容 |
|:----|:-----|:-----|
| `seed-种子文档.md` | 项目目录 | 主角/卖点/金手指/笔触配置/三阶段走向 |
| `世界圣经.md` | 项目目录 | 力量体系/等级划分/地理/势力/经济 |
| `dna/笔触DNA.md` | 项目目录 | 叙事距离/句式/对话力场/情绪处理/章节过渡/感官序列 |
| `systems/金手指系统.md` | 项目目录（如有） | 金手指的触发规则/表现形式/限制条件 |
| `幕纲-幕NNN-{标题}.md` | 项目目录 | 本幕章节流概览 + prose-render交接 + 主角收益 |
| `current-state.md` | 项目目录 | 下一章硬推进 + 人物状态 + 燃料队列 + 伏笔债务 |

**红线**：笔触DNA和世界圣经必须加载。缺少任一=不可写作。

### Step 2：章意图思考 + 金手指触发判断

**读完施工卡后、选择章型之前，先想清楚这章到底要给读者什么体验。**

加载 `爽文章节类型池.md`（来自plot-design/references/），然后回答：

```
## 章意图思考
- 本章定位：（从10类章节类型中选1个主类型，可选1个副类型）
- 体验设计：为了让读者获得____体验，我需要重点写好____场景
- 信息植入：本章需要传递____信息，用____方式植入（回忆/对话/事件/场景，禁止旁白）
- 角色连接：本章需要加深读者对____角色的感情，通过____手段
- 爽感设计：本章的爽感点在____，压力在前____，释放在后____
- 主动行为核验：本章主角的主动行为类型是____
- 金手指判断：本章是否触发金手指？→ 查阅项目systems/金手指系统.md的触发规则
- 本章必须避免的：（不要走偏到什么方向，1~2条）
```

**金手指触发判断**：

金手指的触发规则不在skill中——由项目的 `systems/金手指系统.md` 定义。本skill读取该文件后，按其规则判断本章施工卡中的事件是否触发金手指。

| 判断步骤 | 操作 |
|:--------|:----|
| 1. 读取金手指系统文件 | 加载项目 `systems/金手指系统.md` |
| 2. 查触发条件表 | 系统文件中应定义"什么事件触发什么效果" |
| 3. 比对施工卡事件 | 本章硬锚点中是否有匹配触发条件的事件 |
| 4. 确定触发内容 | 如果触发，确定金手指在本章的具体表现形式 |
| 5. 标记在执行报告中 | 记录触发的金手指效果和段落位置 |

**红线**：
1. 意图思考必须在写作之前完成，不可跳过
2. 如果施工卡中已有"本章意图声明"，则核验并深化，不重复填写
3. 金手指触发判断必须依据项目的系统文件，不可自行发明触发规则

### Step 3：选择章型

根据 Step 2 的意图思考 + 施工卡的本章硬锚点，选择最合适的章型。章型完整定义在 `../pop-emergent-write/references/章型定义.md`，必读。

6种章型速查：

| 章型 | 使用条件 | 读者期待 |
|:----|:--------|:--------|
| opening_shift | 第一章/世界规则松动/金手指首次出现 | 先信现实→被打开更大世界 |
| exploration_gain | 进入新地点/发现资源线索/情报收集 | 跟着主角探索未知 |
| confrontation_pressure | 强弱位阶对话/势力试探/布局算计 | 看打脸反转 |
| combat_reversal | 被压制/追杀/需打穿误判/首杀/暗杀 | 看绝境翻盘 |
| high_tier_display | 远超主角层级的战斗/人物出现（大将/四皇级） | 看格局碾压+立志 |
| reveal_hook | 规则揭露/旧伏笔变义/新敌人/悬赏令/升级揭秘 | 看真相炸裂 |

### Step 4：加载赛道技法 + 项目资源

**4a. 笔触DNA**：从项目目录加载 `dna/笔触DNA.md`。笔触DNA提供：
- 叙事距离/句式段落呼吸/对话力场/情绪处理/章节过渡/感官序列
- 6种章型的节拍笔触绑定（如DNA文件中包含）

**4b. 世界圣经**：从项目目录加载 `世界圣经.md`。从中读取：
- 力量体系等级划分 → 确定主角当前处于哪个战斗阶段
- 战斗阶段映射到skill的战斗模式文件：

| 世界圣经定义的主角阶段 | 加载skill文件 |
|:-------------------|:------------|
| 无霸气/无果实 | `流派技法/战斗模式-无霸气阶段.md` |
| 已获霸气/未吃果实 | `流派技法/战斗模式-霸气阶段.md` |
| 已吃果实（可觉醒） | `流派技法/战斗模式-果实阶段.md` |

- 阶段名称由世界圣经定义，skill不硬编码等级数字。如果世界圣经的阶段划分与上表不完全对应，取最接近的阶段。

**4c. 金手指系统**：从项目目录加载 `systems/金手指系统.md`（如有）。Step 2已读取触发规则，此处加载完整格式（表现形式/限制/写法指引）。

**4d. 赛道技法按需加载**：根据施工卡标注 + 章型需求：

| 条件 | 加载文件 |
|:----|:--------|
| 海上战斗/海战/接舷战 | `流派技法/海上战斗.md` |

**4e. 场景卡全量加载**：`流派技法/场景卡.md`（23种场景公式，每条1行）

**4f. references按需加载**（位于 `../pop-emergent-write/references/`）：

### Step 5：微观技法事前选择

从技法文件（通用技法6 + 情境技法6 + 流派专属5）中选择**2-3类**应用于本章。选定后**必须加载对应的技法文件**。

#### 17类速查

**通用技法**（`../pop-emergent-write/references/通用技法/`）：

| 技法 | 一句话定义 |
|------|----------|
| 信息差博弈 | 围绕"谁知道什么"设计博弈，三方信息不对等产生快感 |
| 后果链设计 | 章内事件不闭环，产生可见后果涟漪 |
| 人设微操 | 通过小动作/小习惯让角色立体，靠行为不靠描述 |
| 节奏微操 | 一章内快慢交替、张弛有度 |
| 预期违背 | 故意建立读者预期，然后打破它 |
| 镜头切换 | 短暂切换POV到配角/敌人/旁观者 |

**情境技法**（`../pop-emergent-write/references/情境技法/`）：

| 技法 | 一句话定义 |
|------|----------|
| 社会梗融入 | 把现实流行梗/热点/知识融入小说 |
| 感官锚点 | 用具体生理感官细节建立代入感 |
| 情感锚点设计 | 用一个"关系"作为情感引擎 |
| 世界观渐进揭示 | 通过角色经历让概念一层层展开 |
| 恐怖奇观美学化 | 将恐怖/怪诞用美学语言包装 |
| 高位视角插入 | 短暂切换到幕后人视角展示全局 |

**流派专属**（`../pop-emergent-write/references/流派专属/`）：

| 技法 | 一句话定义 |
|------|----------|
| 规则博弈与限制创意 | 冲突靠理解规则、利用规则漏洞 |
| 数值面板叙事 | 用数值变化驱动叙事（如金手指系统有数值化呈现） |
| 非人视角参与 | 非人主角的局限性和独特感知制造信息差 |
| 仪式感叙事 | 用仪式化动作和语言建立庄重感 |
| 知识降维科普 | 用专业知识建立可信度 |

### Step 6：写作

按章型的7节拍推进剧情。每个节拍执行五层指导：

| 层 | 来源 | 决定 |
|:--|:-----|:----|
| 笔触层 | 项目笔触DNA | 怎么写（叙事距离/句式/对话气口/战斗表现/POV规则） |
| 节奏层 | `../pop-emergent-write/references/节奏约束.md` | 多快写（事件密度/感官上限/爽感触发/微钩子） |
| 格局层 | `../pop-emergent-write/references/格局表达.md` | 写多大（空间跃迁/时间纵深/侧面定级/规则对抗/史诗口吻） |
| 爽感层 | `../pop-emergent-write/references/爽点设计.md` | 怎么爽（打脸循环/小爽链/爆发后三段式/即时奖励） |
| 微观层 | Step 5选定的技法文件 | 单章怎么精彩（17类技法中选定的2-3类） |

**五层冲突时的优先级**：爽感层 > 节奏层 > 微观层 > 格局层 > 笔触层。

**微观层执行要求**：选定的2-3类技法必须在正文中可见落地。写完后在执行报告中标注落地位置。

### Step 7：验收

写完后逐项检查验收表（见速查表）。在正文末尾追加执行报告（含微观技法选择卡 + 技法落地标注 + 金手指触发记录）。

### Step 8：移交review

正文渲染完成后，移交review skill进行编辑审核。
- "正文采纳" → 下一章渲染（使用review产出的状态快照）
- "正文需修改" → 按修改项清单重写 → 再移交review
- "人工裁决" → 暂停等待人工判断

## API管道注入规范

当通过API管道调用本skill时（主agent调度+脚本调API模式），信息注入遵循以下规范。

### 全链路架构

```
seed阶段一 → 种子文档
seed阶段二 → 主角引擎 + 角色储备池 + 世界包     （1次API）
    ↓
plot-design → 幕纲
    ↓
事实快照初始化 → 从幕纲提取全局设定/人物表/战斗列表  （无API，脚本提取）
    ↓
施工卡生成 → 每章一张施工卡                        （1次API批量）
    ↓
prose渲染 → 逐章正文                              （10次API + 10次快照更新API）
```

### 设计原则

1. **System Prompt烘焙稳定上下文**：笔触DNA/金手指/种子/主角引擎/角色储备池/世界包全部烘焙进system prompt，全书章节共用，不重复传输
2. **User Prompt只传施工卡+快照**：幕纲不直接传给prose——施工卡已从幕纲提取必要信息
3. **权威标注**：system prompt中的角色名/设定为权威。前章结尾标注为"AI生成参考"
4. **不传递AI产出作为权威**：前章结尾不作为角色名/设定来源
5. **事实快照每章更新**：每章写完后提取新事实，注入下一章，确保信息一致

### System Prompt（固定，全书章节共用）

从以下来源组装：

| 来源 | 内容 | 加载方式 |
|:-----|:-----|:--------|
| 本skill SOP+红线+章型骨架+速查表 | 赛道级技法 | 内联 |
| `参考DNA/{书名}/笔触DNA.md` | 完整笔触规范 | 脚本加载完整文件追加 |
| `参考DNA/{书名}/金手指系统.md` | 完整金手指系统 | 脚本加载完整文件追加 |
| `产出/种子/seed-种子文档.md` | 种子文档全文 | 脚本加载追加 |
| `产出/world-build/主角引擎.md` | 主角行为模式表 | 脚本加载追加 |
| `产出/world-build/角色储备池.md` | S/A/B/C角色定义（角色名以此为准） | 脚本加载追加 |
| `产出/world-build/世界包.md` | 力量体系/势力/社会/矛盾 | 脚本加载追加 |

### User Prompt（每章不同）

| 注入项 | 内容 | 来源 |
|:------|:-----|:-----|
| 本章施工卡 | 6部分：意图声明/7节拍锚点/爽点危机/微beat表/章卡边界/prose交接 | `产出/施工卡/chNNN-施工卡.md` |
| 事实快照 | 全局设定/已确立事实/当前角色状态 | `产出/事实快照/事实快照-幕NNN.md`（每章更新） |
| 前章衔接 | 上一章正文最后~500字，标注为AI参考 | 上一章产出文件 |
| 写作要求 | 本章特定约束 | 脚本生成 |

### User Prompt结构模板


## 6种章型骨架

每种章型7个节拍。节拍功能定义是通用的，笔触绑定在项目笔触DNA中。

### opening_shift（开篇/世界松动章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 日常秩序/角色处境建立 |
| 2 第一信息变化 | 不合常理的痕迹/异常目击/金手指首次出现 |
| 3 压力/诱因 | 权威或群体无法解释/异常超出常识 |
| 4 主角动作或判断 | 接近异常/做出行动 |
| 5 阻碍升级 | 异常是更大物象的一角（爽感爆发点） |
| 6 可见反馈 | 常识崩裂/金手指激活/世界真相一角 |
| 7 章末钩子 | 异常继续逼近或呼叫下一层真相 |

### exploration_gain（探索/资源露头章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 环境反常或情报线索出现 |
| 2 资源/线索露头 | 发现目标（宝藏/情报/人物/训练机会） |
| 3 危险阻挡 | 或别人不敢拿/信息不完整需验证 |
| 4 判断收益，冒险 | 选择深入 |
| 5 危险扩大或深处异动 | 爽感爆发点 |
| 6 资源到手/能力变化 | 获得回报 |
| 7 更深处还有更大变化 | 章末钩子 |

### confrontation_pressure（对峙/压力章）

| 节拍 | 功能 |
|:----|:----|
| 1 敌方/高位者给压力 | 出场即轻视 |
| 2 对方暴露目的/误判 | 筹码出现 |
| 3 语言羞辱/身份压制 | 被欺蓄力 |
| 4 沉默/反问/亮筹码 | 暗蓄 |
| 5 对方加码/旁人围观 | 爽感爆发点（被欺→反击） |
| 6 对方失态/局势微反转 | 打脸反馈 |
| 7 矛盾升级到下一场行动 | 章末钩子 |

### combat_reversal（战斗/反击章）

| 节拍 | 功能 |
|:----|:----|
| 1 明确对方优势/主角劣势 | 出场即轻视 |
| 2 主角发现缝隙/变量 | 战术机会出现 |
| 3 对方攻击/羞辱/封路 | 压制蓄力 |
| 4 不解释，直接执行 | 暗蓄 |
| 5 对方后手/位阶压制超预期 | 爽感爆发点（压力→突破）。内部=压制段≤3段+爆发段≤3行 |
| 6 身体损伤/武器破碎/敌人失态 | 代价必须可见且不可逆 |
| 7 反击引来更高后果（悬赏/追杀/名声传播） | 章末钩子 |

### high_tier_display（高位展示章）

| 节拍 | 功能 |
|:----|:----|
| 1 高位存在出场/气息压场 | 威压即事件（大将/四皇/传说级） |
| 2 普通判断失效 | 独句段 |
| 3 旁观者/环境承受不住 | 侧面定级 |
| 4 观察/避让/借势/立志 | 主角动作 |
| 5 高位行为造成大范围后果 | 爽感爆发点（未知→揭示） |
| 6 天地/组织/群众影响变化 | 反馈 |
| 7 主角未来必须抵达此层级 | 章末钩子 |

### reveal_hook（揭秘/章末钩子章）

| 节拍 | 功能 |
|:----|:----|
| 1 上一场结果/异常余波 | 余波开场 |
| 2 新解释出现 | 信息句清楚 |
| 3 解释带来更大危险 | 短句施压 |
| 4 选择追查/隐瞒/反击 | 主角动作 |
| 5 新信息牵出更高层 | 爽感爆发点（未知→揭示） |
| 6 关系/组织判断改变 | 反馈 |
| 7 下一章必须进入新场面 | 章末钩子 |

## 红线

### 赛道级红线（skill强制）

1. **世界观红线（同人模式）**：本项目为海贼王同人，使用《海贼王》原作世界观，包括原作地点（东海/伟大航路/新世界/马林梵多等）、原作势力（海军/世界政府/革命军/四皇等）、原作体系（恶魔果实/霸气/悬赏金/海贼团结构等）。原作角色可以作为背景人物出场或被提及。但**禁止主角与原作核心角色产生深度互动或加入原作主角团**——主角拥有独立的故事线，原作角色是世界的组成部分而非主角的依附对象。违反=不合格。
2. **锚点红线**：施工卡的硬锚点必须按序踩中。不可触发项不得触发。违反=不合格。
3. **节奏红线**：事件密度≥8/章。爽感≥1/章。微钩子每3-5行一个。违反=不合格。
4. **篇幅红线**：3000-5000中文字。
5. **微观技法红线**：Step 5必须完成。选择卡必须填写。选定的2-3类技法必须在正文中可见落地。必须加载对应技法文件。未填选择卡=不合格。
6. **战斗多样性红线**：连续3章不得使用同一种战斗模式。连续2章不得使用相同章节过渡方式。
7. **项目资源红线**：笔触DNA和世界圣经必须加载后才能写作。缺少任一=不可开始写作。

### 书级红线（由项目笔触DNA/金手指系统补充）

书级红线由项目的笔触DNA文件和金手指系统文件定义。例如：
- 穿越者视角红线（如项目DNA中有穿越者设定）
- 灰色道德红线（如项目DNA中有灰色道德设定）
- 金手指触发红线（由项目金手指系统文件定义）
- 战术战斗红线（如项目DNA中有战术驱动设定）

**本skill在Step 1加载项目DNA后，同时加载其定义的书级红线，与赛道级红线合并执行。**

## 速查表

### 节奏约束速查

| 指标 | 约束 | 豁免 |
|:----|:----|:-----|
| 事件间隔 | ≤20行 | — |
| 感官连续 | ≤3行 | 金手指空间描写≤5行（如系统文件有此设定） |
| 情绪连续 | ≤2行 | — |
| 事件密度 | 8-12个/章 | — |
| 爽感触发 | ≥1个/章 | — |
| 蓄力上限 | ≤2段 | — |
| 信息释放 | ≤3段 | — |
| 氛围段 | ≤1段(≤3行) | — |
| 微钩子 | 每3-5行 | — |

### 格局五支柱速查

| 支柱 | 每章最低 | 表达方式 |
|:----|:--------|:--------|
| 空间尺度跃迁 | ≥1次 | 独句段，从"这个码头"扩展到"东海"甚至"伟大航路" |
| 时间纵深 | ≥1次 | 1-2句嵌入，放入大海贼时代历史线 |
| 侧面定级 | ≥2个 | 物理层+名气层（悬赏金是核心参照物），具体现象非形容词 |
| 规则对抗 | 主角行动有规则层意义 | ≤3行 |
| 史诗口吻 | 1-2次 | 独句段拉远，拉完立刻回贴身 |

### 爽感速查

| 机制 | 要求 |
|:----|:----|
| 打脸循环 | 轻视从出场启动≥2层，翻盘≤3行，对方有可见失态 |
| 小爽链 | 每3-5事件≥1个小爽，大爽前≥2个铺垫，≥2种类型 |
| 爆发后三段式 | 爆发→反馈≤2段→余韵≥1段→下一事件 |
| 即时奖励 | 每章≥1次行动→可见回报闭环 |
| 能力可视化 | 能力状态有物理标志（悬赏金/霸气显现/果实变化），读者能判断当前水平 |

### 战斗阶段映射速查

| 世界圣经定义的阶段 | skill加载文件 | 核心特征 |
|:-----------------|:------------|:--------|
| 无霸气/无果实 | `战斗模式-无霸气阶段.md` | 战术驱动，武器+地形+配合 |
| 已获霸气 | `战斗模式-霸气阶段.md` | 霸气增强战术，非替代 |
| 已吃果实 | `战斗模式-果实阶段.md` | 果实是工具，非身份 |

**注意**：阶段划分由世界圣经定义，skill只提供对应阶段的战斗写法。如果世界圣经的阶段不完全是这三档，取最接近的。

### 验收表

| 项 | 合格 | 不合格 |
|:---|:-----|:------|
| 笔触 | 笔触DNA的风格特征在正文中可见 | 只换宏大词 |
| 章型套路 | 7节拍完整，有推进非纯氛围 | 只有氛围没有推进 |
| 节奏 | 8-12事件/爽感≥1/微钩子每3-5行 | 事件<8或大段空转 |
| 格局 | 5支柱全落地 | 全程锁在一个物理空间 |
| 爽感 | 打脸循环+小爽链+爆发后三段式+即时奖励 | 无爽感或爽感温吞 |
| 主动行为 | 每章≥1个主角主动行为 | 连续2章无主动行为 |
| 世界观（同人） | 使用海贼王原作世界观，原作角色可作为背景 | 主角与原作核心角色深度互动/加入原作主角团 |
| 微观技法 | 选择卡已填+已加载技法文件+2-3类技法落地 | 未填/未加载/未落地 |
| 金手指 | 按项目系统文件规则触发/不触发 | 漏触发或乱触发 |
| 项目红线 | 书级红线全部通过 | 违反书级红线 |
