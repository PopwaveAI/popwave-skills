# 爽点链条矩阵

> 不是配额表，是章型×引擎的匹配模板。锚点定义阶段查表选引擎，填具体剧情点，链条结构是现成的。
>
> 设计原理：第8轮实验发现——爽感是叙事架构问题，不是文笔问题。把爽点链条前置到锚点定义阶段后，两组agent都踩中了完整链条。本矩阵把这个过程从"手动设计"升级为"查表填空"。

---

## 矩阵总表

| 章型 | 主引擎 | 辅引擎 | 调味 | 链条环节数 |
|:----|:------|:------|:----|:----------|
| combat_reversal | 压力突破 | 打脸循环 | 小爽×3 + 即时奖励 | 12 |
| confrontation_pressure | 打脸循环 | 信息优势 | 小爽×2 + 旁观者反应 | 10 |
| exploration_gain | 未知揭示 | 即时奖励 | 小爽×2 + 能力成长 | 10 |
| high_tier_display | 弱→强展示 | 旁观者震惊 | 小爽×1 + 格局跃迁 | 8 |
| reveal_hook | 未知揭示 | 悬念钩子 | 反转×1 + 新谜题 | 8 |
| opening_shift | 未知揭示 | 能力觉醒 | 悬念铺垫（本章不爆，埋雷） | 6 |

## 各章型链条模板

### combat_reversal（战斗/反击章）

**引擎选型理由**：战斗章的核心爽感是"绝境翻盘"。压力突破给翻盘的释放感，打脸循环给翻盘的反差感。小爽沿途让读者"预感"主角要赢，即时奖励让读者看到代价背后的成长。

**链条模板**（12环节）：

```
[小爽1·判断正确] — 主角从细节判断出对手的弱点/模式
[轻视启动] — 对方嚣张/误判，语气像在说一件无聊的事
[小爽2·信息优势] — 主角从对方的轻视中提取关键信息
[压制] — 对方用实际行动碾压，主角陷入绝境
[小爽3·主动选择] — 主角在压力下做出疯狂但主动的决定
[绝境顶点] — 最黑暗的时刻，读者相信主角真的可能输
[暗蓄] — 发现缝隙/变量/关键信息（1句闪回或1个观察）
[突破爆发] — 1-3行翻盘，越绝境配越短
[失态] — 对方崩溃/尖叫/否认，有台词有物理描写
[即时奖励] — 主角获得可见回报（能力升级/新装备/新情报）
[回味·停一拍] — 主角喘气/环视/感受身体变化，1-2句
[章末钩子] — 新威胁/新悬念/更大存在
```

**适用条件**：对方是智能生物（能嚣张/能失态），或非智能但有可发现的弱点。

---

### confrontation_pressure（对峙/压力章）

**引擎选型理由**：对峙章的核心爽感是"打脸"——看不起主角的人输了。打脸循环是主引擎，信息优势做辅（主角靠信息翻盘而非蛮力），旁观者反应放大爽感。

**链条模板**（10环节）：

```
[轻视启动] — 对方出场即压制，语气/姿态/信息全方位轻视
[小爽1·信息优势] — 主角知道对方不知道的事（暗藏筹码）
[羞辱加码] — 对方加码羞辱/威胁/展示权力差
[暗蓄] — 主角不辩解，默默准备（沉默本身就是蓄力）
[小爽2·判断正确] — 主角判断出对方的真实目的/弱点
[翻盘爆发] — 主角亮筹码/反问/揭露信息，1-3行翻转局势
[对方失态] — 对方哑口/愤怒/重新评估（必须有可见反应）
[旁观者反应] — 在场第三方震惊/改观/重新定位主角
[即时奖励] — 主角获得信息/地位/资源
[章末钩子] — 矛盾升级到下一场行动
```

**适用条件**：多方在场（至少主角+对手+旁观者），强弱位阶对话。

---

### exploration_gain（探索/资源章）

**引擎选型理由**：探索章的核心爽感是"发现"——未知变已知的那一刻。未知揭示是主引擎，即时奖励做辅（探索有回报），能力成长让追读有跨章动力。

**链条模板**（10环节）：

```
[疑问1] — 环境反常，第一个不对劲的细节
[疑问2] — 第二个反常，与疑问1形成呼应或矛盾
[小爽1·判断正确] — 主角判断出探索方向/方法
[危险阻挡] — 进入更深处的代价/风险出现
[深入选择] — 主角主动选择冒险（不是被动推进）
[疑问3] — 最关键的线索出现，3条线即将串联
[真相炸裂] — 1-2句揭示，颠覆之前的认知
[即时奖励] — 获得资源/能力/情报（可见回报）
[小爽2·技能展示] — 用新获得的东西做了一件专业的事
[章末钩子] — 更深处还有更大变化
```

**适用条件**：主角进入新环境，有明确的探索目标但过程充满未知。

---

### high_tier_display（高位展示章）

**引擎选型理由**：高位展示章的核心爽感不是主角赢，是"格局碾压+立志"——读者看到远超主角层级的存在，产生敬畏感和期待感。弱→强展示让读者感受差距，旁观者震惊放大格局，格局跃迁拉大尺度。

**链条模板**（8环节）：

```
[威压出场] — 高位存在出现，不需要攻击，存在本身就是事件
[普通判断失效] — 主角/旁观者的常识被打碎（独句段）
[侧面定级] — 旁观者/环境承受不住（物理层+灵能层）
[小爽1·主角观察到的] — 主角看到了别人没看到的细节（灵能视角优势）
[大范围后果] — 高位存在的一个动作/一句话造成大范围影响
[格局跃迁] — 独句段拉远尺度，让读者意识到"比想象的大"
[立志/借势] — 主角从敬畏转化为行动决心
[章末钩子] — 主角未来必须抵达此层级
```

**适用条件**：远超主角当前层级的存在出场，主角是旁观者而非战斗参与者。

---

### reveal_hook（揭秘/钩子章）

**引擎选型理由**：揭秘章的核心爽感是"原来如此"——之前埋的线索在这一刻串联炸裂。未知揭示是主引擎，悬念钩子确保炸裂后立刻产生新悬念。

**链条模板**（8环节）：

```
[余波开场] — 上一场结果/异常余波，读者还在消化上一章
[新解释出现] — 一个新的解释框架出现（不是最终真相）
[小爽1·信息串联] — 主角把2-3条之前的线索串起来
[解释带来危险] — 新解释意味着更大的威胁
[选择追查] — 主角主动选择追查真相（不是被动卷入）
[真相炸裂] — 1-2句颠覆性揭示，改变一切
[关系/判断改变] — 揭示改变了主角对某人/某事的基本判断
[章末钩子] — 下一章必须进入新场面
```

**适用条件**：有多条前文埋设的线索需要收束，揭示后产生新方向。

---

### opening_shift（开篇/世界松动章）

**引擎选型理由**：开篇章的核心不是"爆爽"而是"埋雷"——建立世界观、制造悬念、让读者好奇。未知揭示做主引擎但本章不给大爆，悬念铺垫让读者带着问题进入第2章。

**链条模板**（6环节）：

```
[常识建立] — 现实世界的正常秩序，读者感到安全
[第一异常] — 不合常理的数据/痕迹/目击
[权威失效] — 权威机构/常识无法解释异常
[主角接近] — 主角主动接近异常（不是被动卷入）
[异常升级] — 异常不是孤例，是更大物象的一角
[章末钩子] — 异常继续逼近，但不解释——留给下一章
```

**适用条件**：第一章/新卷第一章/世界观转换章。本章不爆爽，只埋悬念。

---

## 使用方法

1. **锚点定义阶段**：根据本章章型查表，选择对应的链条模板
2. **填空**：把模板中的每个环节替换为具体剧情点（环节结构不变，内容替换）
3. **验证**：检查链条是否完整（环节数匹配）、引擎是否匹配章型
4. **交付**：锚点定义文档中包含"爽点链条"章节，写作agent按链条渲染

## 与v8 DNA的关系

- 矩阵在锚点定义阶段使用，不在写作阶段使用
- 写作agent收到的是已填好具体剧情的爽点链条，不需要自己选引擎
- v8 DNA的笔触+references的手法负责"怎么写"，矩阵负责"写什么爽感结构"
