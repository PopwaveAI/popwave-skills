# pop-fanqie-character · 番茄角色管理引擎

> Phase 3.5——plot完成后、write之前。把plot产出的毛坯角色塑造成有灵魂的角色卡。v1.1.0

## 做什么

| 输入 | 输出 | 下游 |
|:-----|:-----|:-----|
| 分幕设计 + 全书设定/ + 创意.md + ch001.md | `设计/角色库/角色库.md` | write（角色唯一源）+ review |

**定位**：character不前置建库——plot写完剧情白描后才知道第一卷需要多少角色、每幕谁出场。
**与world分工**：world做设定（力量体系/地图/势力），character做角色（性格/小传/成长数据）。

| 全书设定/ | pop-fanqie-world | `设计/全书设定/`（力量体系.md+势力.md+全书配角.md） | 敌人梯度+势力格局+配角设定 |

## 红线

1. **读取协议**：读取 skill 文件用 `Get-Content -Encoding UTF8 -Raw`，禁用 Read 工具
2. `设计/角色库/角色库.md`是write的唯一角色状态源——write写正文时所有角色信息以角色库为准
3. 建库规模由分幕设计驱动——不预设配角/反派数量，按plot产出的出场角色清单按需建库
4. 性格描述禁止泛泛词——禁止"善良""勇敢"等标签，必须是具体行为倾向（≥1个场景化描述）
5. 核心角色必须有小传——主角+所有配角+反派，每个≥200字背景故事
6. write增量更新必须回写角色库——write引入新角色或更新状态时同步更新角色库.md
