# Step 0: 导入/续写模式

> 当用户已有历史资料/设定/正文时，跳过从零初始化，重建项目状态并路由到正确Phase。

## 触发条件（任一满足即进入本Step）

1. 用户明确说"导入/续写/已有/接续/迁移"
2. Step 1初始化时检测到 `正文/` 或 `设计/` 已有文件
3. 项目目录已有内容但无project-state.md

---

## 执行

### 0a. 资产扫描

用LS扫描项目目录（或用户指定的源目录），将已有文件映射到标准结构：

| 标准位置 | 检测文件 | 对应Phase |
|:--|:--|:--|
| 素材/用户意图.md | 用户意图.md / 意图.md | Phase 0 |
| 素材/赛道调研.md | 赛道调研.md / 市场调研.md | Phase 0 |
| 素材/文风锚定.md | 文风锚定.md / 文风DNA.md | Phase 0 |
| 素材/decon-lite-*.md | decon-lite-*.md | Phase 0 |
| 设计/创意.md | 创意.md | Phase 1 |
| 设计/全书设定/ | 全书设定/ 目录有文件 | Phase 2 |
| 设计/第一卷剧情/剧情白描.md | 剧情白描.md | Phase 3 |
| 设计/角色库/角色库.md | 角色库.md | Phase 3.5 |
| 正文/ch*.txt | ch*.txt / ch*.md | Phase 4 |
| 审核/小说快照.md | 小说快照.md | Phase 5 |

→ 输出：**资产清单**（逐项标记 ✅已有 / ❌缺失）

**非标准文件名处理**：用户文件名可能不规范（如"我的设定.docx""power-system.txt"）。扫描时对文件名做模糊匹配，匹配不上的归入"未分类资产"并询问用户映射到哪个标准位置。

### 0b. 缺口分析

对照依赖链，逐Phase检查产出是否就绪：

| Phase | 产出 | 就绪判定 |
|:--|:--|:--|
| Phase 0 | 用户意图.md + 赛道调研.md | 两者都有=✅ |
| Phase 1 | 设计/创意.md + 正文/ch001.txt | 两者都有=✅ |
| Phase 2 | 设计/全书设定/ 有文件 | 目录有文件=✅ |
| Phase 3 | 设计/第一卷剧情/剧情白描.md | 存在=✅ |
| Phase 3.5 | 设计/角色库/角色库.md | 存在=✅ |
| Phase 4 | 正文/ch*.txt | 有正文=✅ |
| Phase 5 | 审核/小说快照.md | 存在=✅ |

→ 输出：**缺口报告**（哪些Phase已就绪/哪些缺失）

**正文进度检测**：如果 `正文/` 有文件，提取最大章节号作为 `current_chapter`。例如有 ch001.txt~ch015.txt → current_chapter = ch016（下一章待写）。

### 0c. 落地Phase决策

根据缺口报告 + 正文进度，按下表决定落地Phase：

| 条件 | mode | 落地Phase | 说明 |
|:--|:--|:--|:--|
| 正文有 + 小说快照有 | resume | Phase 4（续写下一章） | 状态完整，直接接续 |
| 正文有 + 小说快照无 | resume | 先执行0e正文反推 → Phase 4 | 需补建快照 |
| 正文无 + Phase 3.5就绪 | import | Phase 4（write） | 设定+剧情+角色完整 |
| 正文无 + Phase 3就绪 | import | Phase 3.5（character） | 缺角色库 |
| 正文无 + Phase 2就绪 | import | Phase 3（plot） | 缺剧情白描 |
| 正文无 + Phase 1就绪 | import | Phase 2（world） | 缺世界设定 |
| 正文无 + Phase 0就绪 | import | Phase 1（seed） | 缺创意+首章 |
| 全无 | fresh | Phase 0 | 从零开始（走正常step1流程） |

→ **向用户展示资产清单 + 缺口报告 + 落地Phase建议，用户确认后进入0d**

### 0d. 状态文件重建

#### 创建/更新project-state.md

如果 `project-state.md` 不存在，创建标准格式。如果已存在，直接SearchReplace更新。

用SearchReplace更新以下字段：

```markdown
# 项目：{项目名}

> 管线：番茄skill群 | 创建：{时间} | 更新：{时间}

## 当前阶段
mode: {import/resume/fresh}
phase: {落地Phase}
current_chapter: {current_chapter}

## 阶段完成情况
- [x] Phase 0: 用户意图 + 并发前置准备（如就绪）
- [x] Phase 1: Seed → 设计/创意.md + 正文/ch001.txt（如就绪）
- [x] Phase 2: World → 设计/全书设定/（如就绪）
- [x] Phase 3: Plot → 设计/第一卷剧情/剧情白描.md（如就绪）
- [x] Phase 3.5: Character → 设计/角色库/角色库.md（如就绪）
- [x] Phase 4: Write → 正文/chXXX.txt（如就绪）
- [x] Phase 5: Review → 审核/review-chXXX.md（如就绪）
```

**规则**：已就绪的Phase标记 `[x]`，未就绪的标记 `[ ]`，落地Phase为当前进行中。

#### 更新底牌就绪区块

将已有素材对应的底牌从❌改为✅或done/skipped。

#### 生成project-state.html

运行脚本生成可视化HTML：

```bash
python skills/pop-fanqie-pipeline/scripts/generate-state-html.py {projectDir}/project-state.md
```

#### 资产归位

如果资产文件在非标准位置（如根目录而非 `设计/`），移动或复制到标准位置。询问用户是否保留原文件。

### 0e. 正文反推（仅当有正文但缺关键状态文件时执行）

> 触发条件：`正文/` 有文件 且 `审核/小说快照.md` 不存在

**目标**：从已有正文反推生成 review/write 所需的最小状态文件。

#### 0e-1. 生成小说快照

读取已有正文（全部章节，或量大时取最近5章+第一章），提取以下信息生成 `审核/小说快照.md`：

```markdown
# 小说快照（导入反推生成）

> ⚠️ 本快照由正文反推生成，未经逐章review校验，需人工审核确认。

## 全书进度
- 已完成：chXXX（最大章节号）
- 当前卷：第X卷（从正文推断）
- 当前幕：第X幕（从正文推断）

## 涌现设定累计
- （从正文中提取已建立的设定/规则/力量体系信息）

## 角色状态总表
| 角色 | 当前位置 | 能力等级 | 关系 | 当前目标 |
|:--|:--|:--|:--|:--|
| （从正文最后几章提取各角色最新状态） |

## 剧情线进度
| 剧情线 | 状态 | 进度 |
|:--|:--|:--|
| （从正文提取主线/支线进度） |

## 读者已知信息池
- （截至最后一章读者已知的信息）

## 待回收伏笔总表
| 伏笔 | 埋设章号 | 待回收 | 状态 |
|:--|:--|:--|:--|
| （从正文提取未回收的伏笔） |
```

#### 0e-2. 生成current-state.md

基于小说快照，生成 `current-state.md`（下一章写作入口包）：
- 当前章位 = 最大章节号 + 1
- 人物状态 = 快照中的角色状态总表
- 不可改事实 = 从正文提取的已发生关键事件
- 下一章硬推进 = 从最后一章的章末钩子推断
- 伏笔债务 = 快照中的待回收伏笔

#### 0e-3. 缺失设计文档处理

如果关键设计文档缺失，按以下方式处理：

| 缺失文档 | 处理方式 |
|:--|:--|
| 创意.md | 从正文提取故事核心 → 生成最小创意文档（标注"反推，需校验"） |
| 全书设定/ | 从正文提取世界规则/力量体系 → 生成最小设定（标注"反推"） |
| 角色库.md | 从正文提取出场角色 → 生成角色库（标注"反推"） |
| 剧情白描.md | 无法从正文反推 → **必须询问用户**：①提供大纲 ②跳过plot直接续写（降级模式） |

**降级模式说明**：如果用户选择跳过plot直接续写，write将无剧情白描约束。需在 `current-state.md` 中手动指定下一章核心事件和爽感节点，write按current-state指导续写。后续可补跑plot生成正式剧情白描。
