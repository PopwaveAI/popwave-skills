# CHANGELOG

## v3.0.0 (2026-07-14)

### 重构：从"场景标签生成器"改为"故事推导引擎"

**根因**：v2.0.0的plot产出的全是元数据标签（变体编号/章型/爽感类型/钩子），不包含故事本身。write拿到标签只能自己脑补故事，质量全看运气。实测项目c中4章全是"看见问题→打脸→赢了"的重复循环，场景不够痛、章节之间无连续性。

**改动**：
- 新增 Step 1 燃料注入：每批次生成前自动搜索3类燃料（现实事件/爆款参考/情绪触发），提炼≤500字燃料摘要注入故事生成
- 新增 Step 2 四象限散弹枪：每章出4个故事草案（保守×最好 / 保守×最坏 / 激进×最好 / 激进×最坏），覆盖行为×结果四象限
- 草案必须回答5问：处境/行动/转折/变化/读者幻想——缺任何一个=废稿
- 新增 Step 3 用户选定：用户从4个草案中选1个，选定后才补全元数据（先有故事再贴标签）
- 场景卡格式改为故事优先：故事全文在前，元数据在后
- 每批从5-10章减为3章（四象限产出量大）
- 新增红线：连续2章不得使用相同象限
