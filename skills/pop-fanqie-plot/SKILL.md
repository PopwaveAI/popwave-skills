# pop-fanqie-plot · 番茄场景推导引擎

> 从概念种子推导每章场景卡。极轻量——不建幕纲、不建施工卡，只产出1页场景卡驱动正文写作。
> 核心逻辑：概念变体池 → 选取变体 → 场景卡。每章场景卡≤500字。

---

## SOP骨架

### Step 1: 概念变体池维护

从概念种子的100章推导中选取当前阶段的变体，并补充新变体。

变体池格式：
```
| 变体编号 | 变体描述 | 生成场景类型 | 爽感类型 | 预计使用章节 |
```

变体来源：1.概念种子100章推导 2.已写章节的涌现信息 3.外部搜索补强

**爽感类型**（11种，从种子文档的节奏模型中选取）：
压力突破 / 弱强反转 / 未知揭示 / 被欺反击 / 悬疑反转 / 知识碾压 / 甜宠反差 / 搞笑反套路 / 虐心释放 / 奇遇悬念 / 宿命改写

### Step 2: 场景卡批量生成

每章1张场景卡，≤500字。每次生成5-10章批次。

场景卡格式：
```
## chNNN · 章名

概念变体：V0XX - 变体描述
章型：从13种章型中选1（详见 references/章型定义.md）
爽感闭环：从11种中选1 | 触发(1句) | 爆发(1句) | 后果(1句)
钩子：章末留什么悬念（1句话）
节奏约束：事件密度X（按种子节奏模型）/ 概念交互≥X次 / 开篇规则(A级或B级)
边界：写到哪停 / 不能碰什么（各1句话）
前章衔接：上一章结尾的最后一件事（1句话）
```

**场景卡自检**：
- 概念变体是否明确？无变体=废章
- 爽感闭环是否完整？触发→爆发→后果三段缺一=废章
- 钩子是否有追读动力？无钩子=废章
- 与前章衔接是否自然？
- 章型是否与前一章不同？（连续2章不得使用相同章型）

---

## 番茄节奏红线（7条）

| 红线 | 要求 |
|------|------|
| 前3章完成概念展示+首爽 | ch1概念激活或延后≤1章，ch3首爽释放 |
| 每3章≥1高潮 | combat_reversal或mystery_deduction或等效爆发 |
| 前5章有概念升级 | ch5前概念必须进化1次 |
| 铺垫≤1章 | 番茄不允许2章纯铺垫 |
| 连续2章无概念交互=不合格 | 每章必须有概念使用（悬疑流可改为"概念线索推进"） |
| 不得连续2章同章型 | 13种章型轮换 |
| 每章必须有章末钩子 | 不允许无钩子章 |

**节奏模型适配**（从种子文档读取，不同模型用不同密度标准）：

| 节奏模型 | 事件密度 | 替代密度指标 |
|---------|---------|------------|
| 爽文快节奏 | 8-12 | — |
| 悬疑钩子节奏 | 4-6 | 悬念密度≥3/章 |
| 情感蓄力节奏 | 5-7 | 情感密度≥2/章 |
| 智力碾压节奏 | 3-5 | 信息密度≥4/章 |

章型速查（详见 references/章型定义.md）：
- 原始6种：opening_shift / exploration_gain / confrontation_pressure / combat_reversal / high_tier_display / reveal_hook
- 新增7种：mystery_deduction / atmosphere_horror / emotional_pressure / sweet_interaction / knowledge_display / comedic_subversion / time_skip_montage

## 红线

1. 场景卡≤500字/章（不预写正文）
2. 每章必须标注概念变体+章型+爽感类型
3. 番茄节奏红线7条全量执行
4. 连续2章不得使用相同概念变体
5. 连续2章不得使用相同爽感类型
6. 连续2章不得使用相同章型
