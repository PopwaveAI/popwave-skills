# pop-fanqie-plot · 番茄骨架引擎

> 加载创意文档+黄金首章 → 世界构筑（四张地图+人物包+敌人梯度） → 第一卷详规 → 四图叠加推演剧情白描 → 落盘
> 产出：1-骨架/骨架.md + 1-骨架/剧情白描.md + 1-骨架/章锚点表.md

---

## 定位

plot帮助用户把创意从idea变成一个**完整的小说故事世界**。

seed产出的是"创意文档+黄金首章"——方向已定但世界还没构筑。plot负责：
1. **世界构筑**：基于创意文档展开四张地图+完整人物包+第一卷敌人4层梯度
2. **第一卷详规**：幕序列/悬念/高潮/势力/反派
3. **剧情白描**：用四张地图叠加推演法推演整卷故事流

> 剧情白描是plot层最重要的产出，投入50%注意力。write层最关键输入是剧情白描。

## 红线

1. 必须加载seed产出的完整创意文档+黄金首章——禁止凭空编造世界观
2. **四张地图必须独立产出且互相自洽**——地理/力量/势力/危机各一张，不能散落
3. **第一卷敌人必须4层梯度**——街头层/地方势力层/执法系统层/幕后金主层，每层至少1个具名反派
4. **剧情白描必须用四张地图叠加推演法**——禁止凭空一口气写，必须基于四图+人物包推演
5. 第一卷终点必须与世界危机挂钩——不能只是打完boss就结束
6. 幕序列第一幕必须遵循黄金三章法则（ch01已由seed产出，从ch02开始详规）
