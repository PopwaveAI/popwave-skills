# pop-fanqie-plot · 番茄故事推导引擎

> 脑洞长出故事。plot从概念推导全局危机阶梯，再逐章生成故事草案。
> 核心逻辑：全局危机阶梯 → 燃料注入 → 四象限散弹枪 → 用户选定 → 场景卡定稿。

---

## SOP骨架

### Step 0: 全局危机阶梯

从概念种子推导全书危机递增曲线。这一步只做1次（项目启动时），后续每批次生成时对照更新。

**0a. 危机阶梯设计**

番茄的"敌人"不一定是人——是社会不公、信任崩塌、生存倒计时、系统限制。从概念种子推导3-5个危机阶段，量级递增：

```
| 阶段 | 危机类型 | 主角面临什么 | 量级 | 预计章节 |
|------|---------|-------------|------|---------|
| 1 | 个人恩怨 | 被个体欺压 | 小 | ch1-5 |
| 2 | 群体对抗 | 被组织/群体围攻 | 中 | ch6-15 |
| 3 | 体制碾压 | 被制度/权力压制 | 大 | ch16-30 |
| 4 | ... | ... | 更大 | ... |
```

每个阶段必须比上一阶段**量级更大**——不是"更强的敌人"，是"更大的压迫感"。

**0b. 因果链设计**

每个阶段的解决方式必须产生下一阶段需要的条件或触发下一阶段的危机：

```
阶段1解决 → 获得了什么/暴露了什么 → 恰好触发阶段2的危机
阶段2解决 → 获得了什么/暴露了什么 → 恰好触发阶段3的危机
```

不是孤立事件的堆叠，是因果链条。

**0c. 情绪曲线设计**

每个阶段对应一种主导情绪，不能从头到尾都是"爽"：

```
| 阶段 | 主导情绪 | 读者状态 |
|------|---------|---------|
| 1 | 憋屈→爆发 | "终于有人替我出气" |
| 2 | 危机→反击 | "这次没那么容易了" |
| 3 | 绝望→逆转 | "完了→不可能→他怎么做到的" |
```

**0d. 部分解决比例**

约30-40%的章节是"部分解决"——打退但没打死，或解决了眼前问题但引出更大危机。不是每章都要完全爽完。

**产出**：`危机阶梯.md`，含阶段表+因果链+情绪曲线。

### Step 1: 燃料注入

每批次生成前，必须执行**3次真实搜索**（不是凭记忆概括，必须调用搜索工具）：

| 搜索序号 | 搜索词公式 | 提取什么 |
|---------|---------|---------|
| 搜索1 | 概念关键词 + 当前阶段危机类型 + "真实事件/案例" | 至少1个真实事件的**具体细节**（谁做了什么、结果如何） |
| 搜索2 | 概念关键词 + "小说/网文/番茄/起点" + "热门/高赞" | 至少1个爆款作品的**具体剧情骨架**（不是简介，是章节怎么写的） |
| 搜索3 | 概念关键词 + "梗/meme/流行/讨论" | 至少1个**可用的梗或流行元素**（能融入剧情的，不是挂着的标签） |

搜索结果落盘为燃料摘要，格式：

```
## 燃料摘要（batchNN · 阶段N）
### 现实事件
- [事件名]：[一句话发生了什么] → 可用元素：[具体动作/细节/转折]
### 爆款参考
- [作品名]ch[XX]：[剧情骨架] → 可学技法：[怎么写冲突/怎么收尾]
### 梗/流行元素
- [梗名]：[一句话解释] → 融入方式：[怎么编进剧情]
```

**红线**：燃料摘要必须引用搜索到的具体内容，不得用"通常""一般来说""现实中常见"等概括性语言代替。没有搜索结果=该类燃料标注"未找到"并继续，不得编造。

### Step 2: 四象限故事草案

每章生成4个故事草案，覆盖行为×结果四象限：

| | 保守行为 | 激进行为 |
|---|---------|---------|
| **最好结果** | Q1: 规则/礼貌→对方认错妥协 | Q3: 掀桌/极端→全面胜利 |
| **最坏结果** | Q2: 规则/礼貌→反被压制 | Q4: 掀桌/极端→付出惨痛代价 |

每个草案必须回答6问：
- **处境**：主角这章处于什么状态，什么打破了他
- **行动**：他做了什么（具体动作，不是"打脸"三个字）
- **转折**：什么是他没预料到的
- **变化**：章末跟章初有什么不同
- **因果**：本章变化如何触发下章处境
- **读者幻想**：读者代入的是"我也想___但我不敢，他敢"

草案长度：每条150-300字，写故事不写标签。

### Step 3: 用户选定+场景卡定稿

用户从4个草案中选1个。选定后补全元数据，落盘为场景卡。

场景卡格式：
```
## chNNN · 章名

### 故事（定稿）
（用户选定的草案全文）

### 元数据
危机阶段：第N阶段 - 危机类型
概念变体：V0XX
章型：从13种中选1
爽感闭环：类型 | 触发(1句) | 爆发(1句) | 后果(1句)
解决程度：完全解决 / 部分解决（部分解决需注明引出了什么）
钩子：章末留什么悬念（1句话）
节奏约束：事件密度X / 概念交互≥X次 / 开篇规则(A级或B级)
边界：写到哪停 / 不能碰什么
前章衔接：上一章结尾的最后一件事（1句话）
因果指向：本章变化如何触发下章危机（1句话）
```

用户说"你选"或"自动"时：选最具戏剧张力（转折最意外+读者幻想最强）的那个。

## 番茄节奏红线（7条）

| 红线 | 要求 |
|------|------|
| 前3章完成概念展示+首爽 | ch1概念激活或延后≤1章，ch3首爽释放 |
| 每3章≥1高潮 | combat_reversal或等效爆发 |
| 前5章有概念升级 | ch5前概念必须进化1次 |
| 铺垫≤1章 | 不允许2章纯铺垫 |
| 连续2章无概念交互=不合格 | 每章必须有概念使用 |
| 不得连续2章同章型 | 13种章型轮换 |
| 每章必须有章末钩子 | 不允许无钩子章 |

节奏模型适配（从种子文档读取）：

| 节奏模型 | 事件密度 | 替代指标 |
|---------|---------|---------|
| 爽文快节奏 | 8-12 | — |
| 悬疑钩子 | 4-6 | 悬念密度≥3/章 |
| 情感蓄力 | 5-7 | 情感密度≥2/章 |
| 智力碾压 | 3-5 | 信息密度≥4/章 |

章型速查：opening_shift / exploration_gain / confrontation_pressure / combat_reversal / high_tier_display / reveal_hook / mystery_deduction / atmosphere_horror / emotional_pressure / sweet_interaction / knowledge_display / comedic_subversion / time_skip_montage

## 红线

1. **Step 0必须先做**——没有全局危机阶梯不得直接生成场景卡
2. 每章必须出4个象限草案——不得只出1个
3. 草案必须回答6问（处境/行动/转折/变化/因果/读者幻想）——缺任何一个=废稿
4. 燃料注入必须调用搜索工具执行3次真实搜索——禁止凭记忆概括，禁止用"通常/一般来说"代替
5. 用户选定后才补全元数据——不得先定标签再编故事
6. 番茄节奏红线7条全量执行
7. 连续2章不得使用相同象限（避免连续激进或连续保守）
8. 连续2章不得使用相同解题方式（规则怼/掀桌/证据链/情感攻势等需轮换）
9. 部分解决占比≥30%（不得每章都完全解决）
