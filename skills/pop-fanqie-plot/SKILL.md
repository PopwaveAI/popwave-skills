# pop-fanqie-plot · 番茄叙事引擎

> 加载骨架.md（world产出）→ 叙事流剧情白描 → 章锚点表 → 落盘
> v3.0.0：从 plot 拆出 world skill，plot 瘦身为专注叙事创作。设定设计（力量体系→地图→势力→危机→弧线）移至 pop-fanqie-world。

---

## 定位

plot 消费 world 产出的骨架.md，专注**叙事创作**——把结构化的世界设定变成"给朋友讲故事"的叙事流白描。

**与 world 的分工**：world 专注设定设计（理性推导），产出骨架.md；plot 专注叙事创作（感性创作），消费骨架.md 产出剧情白描.md + 章锚点表.md。

---

## 管线总览

| 步骤 | 做什么 | 产出 | 怎么执行 |
|:--|:--|:--|:--|
| Step 1 | 加载骨架.md | 骨架就绪 | **内化在本文件，直接执行** |
| Step 2 | 叙事流剧情白描 | 剧情白描.md | **读取 `steps/step2.md`，执行后读取 step3.md** |
| Step 3 | 落盘+章锚点表 | 三件套 | **读取 `steps/step3.md`，执行后交付** |

# Step 1 · 加载骨架.md（内化）

| 输入 | 来源 | 路径 | plot用途 |
|:--|:--|:--|:--|
| 骨架.md | pop-fanqie-world | `1-骨架/骨架.md` | 力量体系+地图+势力+危机+第一卷弧线——白描的世界地基 |
| 创意+首章 | pop-fanqie-seed | `0-立项/创意.md` + `2-正文/ch001.md` | 创意/金手指/行为引擎/主角轮廓——白描的角色驱动 |
| 参考书DNA | pop-dna-style | `涌现/文风锚定.md` | 参考书的叙事节奏DNA（爽点节奏/Boss梯度）——白描的节奏参考 |

### 从骨架.md提取白描输入

| 骨架章节 | 白描用途 |
|:--|:--|
| 力量体系 | 确定主角当前对赌难度 |
| 城市舞台 | 确定场景发生的地标 |
| 势力结构 | 确定碰撞的势力 |
| 危机压力 | 确定推进的倒计时 |
| 幕序列 | 按幕分段写白描 |
| 核心高潮点 | 展开为三段式高潮场景 |
| 悬念分层 | 标注信息差/伏笔/钩子 |

### 从ch001提取起点

核心事件 → 章末钩子( **ch002必须从这里延续** ) → 金手指激活状态 → 已出场角色(不可冲突)

缺失骨架标注缺失，不阻塞流程（但质量会下降）。

---

## 红线

1. **必须加载骨架.md**——禁止凭空编造世界。骨架缺失标注缺失，不阻塞但质量下降
2. **剧情白描必须是叙事流，不是注水提纲**——每段必须有氛围/情绪弧/关键画面，标注信息差/伏笔/钩子
3. **第一卷终点必须与世界危机挂钩**——不能只是打完boss就结束
4. **Step文件链式加载**——Step1内化在本文件直接执行→读取steps/step2.md→读取steps/step3.md
