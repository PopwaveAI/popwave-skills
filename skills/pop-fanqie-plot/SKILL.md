# pop-fanqie-plot · 番茄长篇骨架设计器

> 消费seed立项包，产出第一卷执行骨架——幕序列+悬念分层+高潮三段式+分层反派。
> 给方向不给细节。write在骨架内自由发挥。

---

## SOP骨架

### Step 1: 加载立项包

读取`创意.md`。提取全部7项：
- 一句话故事 + 草纲（方向已定，plot不重复）
- 金手指（功能/入口能力+触发条件+限制边界+进化方向含衍生分支）
- 主角设定卡（职业/主性格/记忆点/缺点/成长弧线）
- 世界观骨架（地图/等级/进度3段/核心规则+世界自洽四问答案）
- 长线线索（主线悬念+长线伏笔）
- 番茄简介（基调参考）

立项包已定方向，plot做的是**第一卷执行方案**——不是重新设计方向。

### Step 2: 第一卷详规

**2a. 第一卷终点和boss**

第一卷的终点事件（一段话）+ boss是谁（一句话+核心欲望+致命弱点）。

boss优先从seed的分层反派低级或中期层选取。

**2b. 配角设计（2-4个）**

主角设定卡已由seed产出，plot只设计配角。每个配角：
- 名字
- 核心欲望（一句话）
- 功能分类（情感锚点/终极对手/镜像对手/逆徒/助力者）
- 与主角的关系张力（一句话——什么让他们冲突或绑定）

**2c. 幕序列（3-5幕）**

每幕：
- 幕名
- 功能位（建置/冲突升级/高潮/余波）
- 一句话弧线（这幕发生了什么核心事件）
- 章节预算（约X章）

第一幕参考——黄金三章法则（番茄通用，给write的方向参考）：
- ch01：核心冲突爆发——读者知道"发生了什么事"
- ch02：金手指激活——读者知道"主角凭什么翻盘"
- ch03：短期目标明确——读者知道"接下来看什么"
- ch04-07：第一次公开打脸——兑现第一个爽点
- ch08-10：绑定强力助攻——埋下长线悬念

**2d. 悬念分层设计（方法论5.2）**

升级版信息差——分3层：

| 悬念层级 | 说明 | 数量 |
|---------|------|------|
| 核心悬念 | 全书要解答的大问题（继承seed主线悬念） | 1条 |
| 场景悬念 | 某个场景中的具体障碍/未解问题 | 2-3条 |
| 段落悬念 | 一连串场景的共同未知/伏笔 | 1-2条 |

每条标注触发章（大约第几章揭示）。

**2e. 核心高潮点（2-3个，三段式）**

每个高潮点按方法论8三段式设计：
- **低谷**：主角遭遇什么困境/压制（一句话）
- **高潮**：主角如何爆发/反转（一句话）
- **变化**：高潮后主角/格局发生了什么变化（一句话）

**2f. 分层反派设计（3层）**

优先继承seed世界观骨架中的设定，补充细节：

| 层级 | 定位 | 解锁时机 | 核心欲望 | 致命弱点 |
|------|------|---------|---------|---------|
| 低级 | 第一卷前半段对手 | 第一卷前半段 | | |
| 中期 | 第一卷后半段-第二卷对手 | 第一卷后半段 | | |
| 终极 | 全书终局黑手 | 全书终局 | | |

### Step 3: 落盘

产出`骨架.md`，包含Step 1提取确认+Step 2全部内容。

---

## 红线

1. 主角设定卡从seed继承——plot不重新设计主角，只设计配角
2. 悬念必须分3层——核心/场景/段落，不能只有信息差
3. 核心高潮点必须是三段式——低谷→高潮→变化，缺一段不可
4. 给方向不给细节——每幕一句话弧线，不写章节级规划（R17验证：重骨架导致模板化）
5. 只产出`骨架.md`——不写单元模板/变体池/场景卡
