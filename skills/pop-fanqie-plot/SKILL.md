# pop-fanqie-plot · 番茄单元剧蓝图设计器

> 梗拿到手，plot设计一个可复用的模具，不是规划100章。
> 一次设计：单元模板+长线种子+变体池。之后每章取变体+套模板+推进长线。

---

## SOP骨架

### Step 1: 单元模板设计

从梗.md读取金手指+行为引擎+单元剧结构，设计**一个可复用的单元骨架**。

top20拆解验证——5本偏单元剧的top20全部遵循同一个骨架：可变输入×固定流程。plot的职责是把梗里的行为引擎固化为模板。

**单元模板格式**：

```
## 单元模板（每个单元3-8章）

| 节拍 | 内容 | 章数 | 爽感类型 |
|------|------|------|---------|
| 输入 | 新问题出现（新客户/新怪物/新病症） | 1章 | 悬念型——读者好奇这个单元是什么 |
| 识别 | 辨认问题类型和规则 | 1章 | 信息型——读者获得新世界观碎片 |
| 破解 | 主角用金手指（受限制）解决 | 1-3章 | 爽感闭环——触发→爆发→后果 |
| 奖励 | 获得新能力/资源/信息/声望 | 0.5章 | 收获型——读者获得满足感 |
| 钩子 | 埋一点长线信息，引出下个单元 | 0.5章 | 追读型——读者想看下一个 |
```

**设计原则**：
- 模板必须可复用——换一个变体就能跑一个新的单元
- 破解环节是核心——金手指的限制在这里制造张力
- 每个单元必须有完整的爽感闭环（触发→爆发→后果）
- 钩子必须指向长线种子，不能是独立小事件

### Step 2: 长线种子设计

设计**一条贯穿全书的主线悬念**。top20四种埋法（至少用一种）：

| 埋法 | 说明 | top20示例 |
|------|------|----------|
| 奖励即碎片 | 每个单元的奖励本身就是主线拼图的一块 | 749收容怪物每个能力指向更大真相 |
| 规则即暗示 | 每个单元的规则揭示一层世界观真相 | 异兽迷城每遇到新兽揭开一层"世界怎么了" |
| 客户即线索 | 客户的背后故事连接主线 | 风水之王林家诅咒指向更大谜团 |
| 限制即悬念 | 金手指限制的未解之谜持续制造追读 | 时停为什么只有1分钟 |

**长线种子格式**：

```
## 长线种子

- 主线悬念：（一句话——全书要解答的大问题）
- 埋法：（用哪种/哪几种）
- 推进节奏：每___个单元推进一次长线
- 第一次揭示：第___章首次触及长线
- 长线爆发点：第___章长线悬念大爆发（约全书2/3处）
```

### Step 3: 变体池设计

批量生成**10-15个单元变体**，每个变体就是一个可取用的"案件/客户/怪物"。

**变体池格式**：

```
## 变体池（10-15个，按需取用）

| # | 变体名 | 类型 | 痛感级别 | 长线碎片 | 爽感设计 |
|---|-------|------|---------|---------|---------|
| 1 | 例：医院地缚灵 | 怪谈客户 | 4级 | 无（纯单元） | 帮鬼完成遗愿→鬼自愿离开→获得能力 |
| 2 | 例：校园笔仙 | 怪谈客户 | 3级 | 有（笔仙知道一些关于世界的真相） | 拆穿笔仙骗局→获得真实信息 |
| ... | | | | | |
```

**变体设计原则**：
- 痛感级别1-5递增，前期以3-4级为主，偶尔5级
- 每3-4个变体中有1个带长线碎片
- 类型要多样化——不能10个都是同一种客户/怪物
- 爽感设计要差异化——不能每个都是"打脸"或"获得能力"

### Step 4: 燃料搜索（按需触发）

当变体池设计遇到"类型太单一""痛感不够""想不出新变体"时触发。

搜索3类素材：

| 类型 | 搜索方向 | 找什么 |
|------|---------|------|
| 真实事件 | 赛道关键词+"真实事件/案例" | 具体的真实戏剧动作 |
| 爆款参考 | 赛道关键词+"小说/网文/热门章节" | 具体的剧情骨架 |
| 梗/元素 | 赛道关键词+"梗/meme/流行" | 能融入剧情的流行元素 |

搜索结果直接用于丰富变体池，不单独落盘。

3. 变体池（10-15个可取用变体）

---

## 红线

1. 只设计模具，不规划具体章节——不写"第1章写什么、第2章写什么"
2. 单元模板必须可复用——换变体就能跑新单元
3. 长线种子必须至少用一种top20埋法
4. 变体池必须≥10个，类型多样化，痛感递增
5. 每3-4个变体中至少1个带长线碎片
6. 燃料搜索必须真实调用搜索工具——禁止凭记忆概括
