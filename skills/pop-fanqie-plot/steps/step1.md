# Step 1: 加载创意文档 + 黄金首章

读取 `0-立项/创意.md` 和 `2-正文/ch001.md`。提取创意文档全部字段+黄金首章锚点。

---

## 1a. 加载创意文档

seed v13产出的`创意.md`包含以下字段，plot全部消费：

| 产出字段 | 提取内容 | plot用途 |
|---------|---------|---------|
| 一句话简介 | 主角是谁，在做什么，遇到了什么 | 方向已定，plot不重复 |
| 梗 | 一句话梗——一个具体行为×一个反差场景 | 剧情白描必须延续这个梗的调性 |
| 金手指（梗×机制×限制） | 金手指的表现形态/动力/张力 | 力量地图的金手指升级路径基础 |
| 行为引擎 | 主角每章在做什么具体行为 | 幕序列设计依据——每幕围绕行为引擎展开 |
| 轻量主角轮廓 | 名字/身份/核心动机/记忆点 | 人物包设计起点（plot补全配角+反派） |
| 番茄简介 | 散文体基调 | 剧情白描基调参考 |

**创意文档已定方向和金手指**，plot做的是**完整世界构筑+第一卷执行方案**——不是重新设计方向，也不重新设计金手指。

**plot补全seed没做的**：
- 四张地图（地理/力量/势力/危机）——seed不做，plot Step 2产出
- 完整人物包（配角+反派）——seed只给主角轮廓，plot补全
- 第一卷敌人4层梯度——seed不做，plot Step 2产出
- 世界观骨架（世界格局/力量体系/核心规则）——seed不做，plot Step 2基于金手指+行为引擎推导

## 1b. 加载黄金首章

读取 `2-正文/ch001.md`（seed的Step 3产出），提取：

| 要素 | 提取内容 |
|------|---------|
| ch001核心事件 | 首章发生了什么（3-5句） |
| ch001章末钩子 | 首章留下的悬念——ch002必须回收 |
| ch001金手指状态 | 首章结束时金手指激活到什么程度 |
| ch001主角状态 | 首章结束时主角的处境/位置 |
| ch001已出场角色 | 首章已经出场的角色——不能在plot中冲突 |
| ch001反差画面 | 首章的"生活化场景×金手指"反差画面——剧情白描要延续这个调性 |

**黄金首章是plot推演的起点**——ch02开始的剧情白描必须从ch001的章末钩子延续，不能另起炉灶。
