# Step 4: 四图叠加推演剧情白描

> 剧情白描是plot层最重要的产出，投入50%注意力。它是write层的最关键输入。
> 必须用"四张地图叠加推演法"推演——禁止凭空一口气写。

---

## 推演方法：四张地图叠加推演法

### 核心公式

```
剧情白描 = L2单元卡 × 四张地图叠加 × 因果链推理
```

### 推演流程

1. **从Step 3幕序列提取L2单元卡**（每幕拆成3-5个L2单元，每个单元覆盖2-3章）
2. **对每个L2单元卡执行6步推演**：
   - 查地理地图 → 确定摄像头位置（这段故事发生在哪个地标）
   - 查力量地图 → 确定主角等级变化（这段故事主角从LVx到LVy）
   - 查势力地图 → 确定碰撞和演变（这段故事哪些势力碰撞）
   - 查危机地图 → 确定倒计时推进（这段故事哪些倒计时推进）
   - 因果链串联 → 生成叙事段（把以上四图信息串成故事流）
   - 标注信息差 → 增加"读者知道但角色不知道"的信息差标注

3. **ch01特殊处理**：ch01已由seed的黄金首章产出，不重新推演。从ch01的章末钩子开始推演ch02。

---

## API推演（推荐）

> **重要**：推演过程推荐走API而非agent。R35测试验证：agent有上下文污染风险（会把不相关作品的剧情混入），API只看到prompt更纯净。R35 v1(agent)得分3.9 vs v2(API)得分9.1。

### API推演步骤

1. **构造prompt**：把四张地图+人物包+第一卷详规+黄金首章锚点作为system prompt注入
2. **调用DeepSeek API**：要求基于四图叠加推演法推演ch02-ch末的剧情白描
3. **落盘API产出**：API产出的剧情白描直接落盘到 `1-骨架/剧情白描.md`

### API prompt模板

```
你是一位资深网文剧情设计师，精通"四张地图叠加推演法"。

任务：基于以下世界构筑文档，推演第一卷剧情白描（ch02-ch末）。

世界构筑文档：
[注入四张地图+人物包+第一卷详规]

黄金首章锚点（ch01）：
[注入ch001核心事件+章末钩子+主角状态]

推演方法：
对每个L2单元卡执行6步：
1. 查地理地图 → 确定摄像头位置
2. 查力量地图 → 确定主角等级变化
3. 查势力地图 → 确定碰撞和演变
4. 查危机地图 → 确定倒计时推进
5. 因果链串联 → 生成叙事段
6. 标注信息差 → 增加"读者知道但角色不知道"

要求：
- ch02开始推演，从ch01章末钩子延续
- 整卷一口气写完，不拆章节标签
- 以"给朋友讲故事"的方式，自然流畅
- 融入叙事：画面/情绪/内心活动都融入叙事，不单独标注
- 每段标注信息差/伏笔/钩子
```

## 写作要求

1. **整卷一口气写完**：不拆章节/不拆标签，整卷故事流一口气写下来
2. **以"给朋友讲故事"的方式**：自然流畅，确保故事本身好看
3. **融入叙事**：画面/情绪/内心活动都融入叙事，不单独标注
4. **消费世界构筑**：把四张地图+人物包的关键信息融入故事流，write层不再重复注入

---

## 内容要求

每段故事流必须包含：
- **事件推进**：主角做了什么，发生了什么变化
- **信息差标注**：`(读者知道但角色不知道：XXX)`
- **伏笔标注**：`(伏笔：XXX，chXX回收)`
- **钩子标注**：`(钩子：XXX)`

---

## 世界构筑消化规则

剧情白描里必须消化以下设定（write层不再注入）：
- **力量地图**：战斗时融入等级/技能/金手指的具体表现（不是列数值，而是写感受）
- **地理地图**：场景迁移时自然带出地标特征
- **势力地图**：组织出现时自然带出势力背景
- **危机地图**：倒计时推进时自然带出压力感
