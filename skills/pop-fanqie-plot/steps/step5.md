## 5c. 落盘章锚点表.md

> 章锚点表是write层的"每章导航"。write层需要这个表来明确每章边界+锚点+预期回收章。

### 写作要求

从剧情白描中提取每章的锚点信息，整理成表格。每章一行，包含：
- **章号**：ch001, ch002...
- **POV**：视角人物（主角/配角/反派）
- **章型**：开篇转折/成长/披露/对抗/揭示（映射番茄write的4种章型）
- **核心事件**：3-5句覆盖本章核心
- **爽点**：爽感类型+具体内容
- **钩子**：章末钩子内容
- **预期回收章**：这个钩子在chXX回收
- **关键数据**：本章出现的属性/技能/装备变化

### 格式

```markdown
# 《书名》· 第一卷章锚点表

| 章号 | POV | 章型 | 核心事件 | 爽点 | 钩子 | 预期回收章 | 关键数据 |
|------|-----|------|---------|------|------|-----------|---------|
| ch001 | 主角 | 开篇转折 | （从seed黄金首章提取） | | | ch002 | |
| ch002 | 主角 | 成长 | | | | ch003 | |
| ch003 | 主角 | 披露 | | | | ch004 | |
```

**注意**：ch001的锚点从seed的黄金首章提取，不重新推演。
