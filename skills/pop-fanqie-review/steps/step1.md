# Step 1-3：正向符合性 + 正文质量 + 反向充足性

> 触发：写完一章正文后进入review。
> 目标：三步审计正文，产出审稿判断供step-2落盘。

## 前置确认（execution.mode）

- formal：已读待审正文、current-state、骨架/设定齐全，输出审稿判断并落盘。
- draft：缺部分材料，但能从正文生成临时判断，产出标记draft。
- trial：只做快速体感，不落盘。

## 必读输入

- 正文 `正文/chXXX.md`
- write交付面板（核心事件/爽感闭环/章末钩子/笔触状态）
- 剧情白描本章段
- 骨架/设定（如涉）+ 角色库
- 上一版 `current-state.md`（如存在）
- 笔触DNA（三态协议）：检查 `素材/文风锚定.md` 是否存在

---

## Step 1：正向符合性检查（正文 vs 参考资料匹配）

> 第一道闸门：正文是否兑现了剧情白描承诺的故事。主线丢失=废章。

### 1a. 核心事件逐beat对照

打开剧情白描本章段，提取核心事件beat（通常2-4个），逐条对照正文：

| beat | 白描承诺 | 正文是否兑现 | 偏离类型 |
|:--|:--|:--|:--|
| 1 | ... | ✓/✗ | 缺失/替换/增加/顺序变/强度变 |

**判定**：全部兑现→进1b。有偏离但主线未丢→标注偏离进1b。主线丢失→**废章**。

### 1b. 设定一致性

| 检查项 | 判定标准 | 不合格处理 |
|:--|:--|:--|
| 金手指规则 | 使用是否符合"梗×机制×限制"？是否加了代价（禁止）？ | 标注违反点 |
| 力量体系 | 主角能力是否越级？等级是否与骨架一致？ | 标注越级点 |
| 世界观规则 | 是否违反世界自洽（能力来源/触发条件/作用边界）？ | 标注违反点 |
| 势力格局 | 势力关系/实力数值是否与设定一致？ | 标注不一致 |

### 1c. 角色一致性

检查配角/反派行为是否符合角色卡设定，是否OOC。标注OOC点。

### 1d. 爽感闭环检查

- 本章主推爽感类型：______
- 触发段：有/无（位置）
- 爆发段：有/无（位置）
- 后果段：有/无（位置）
- 三段缺一 → **废章**

### 1e. 番茄底线检查

| 检查项 | 标准 | 不合格处理 |
|:--|:--|:--|
| 字数 | 2000-2500字，不准超过2500 | 超出→REJECT |
| 章末钩子 | 主动决策型，禁止被动接收型 | 违规→REJECT |
| 节奏密度 | 8-12个事件/章 | 不足→标注 |
| 章型7节拍 | 开场炸弹→冲突→金手指→应用→爽感→危机→钩子 | 缺节拍→标注 |
| 前后章衔接 | 是否回收前章钩子？ | 未回收→标注 |
| 章名格式 | 10字以上，有噱头/反差/创意点 | 违规→标注 |
| 合规底线 | 政治敏感/红线词/未成年保护/价值观 | 命中任一→直接REJECT |

## Step 2：正文质量检查（正文本身）

### 2a. AI味7项检查

> 定性判断，逐条检查，**每项必须给具体位置**。

| 检查项 | 判定标准 | 不合格处理 |
|:--|:--|:--|
| 句式重复 | 同一句式连续使用2次以上？ | 标注位置+建议替换 |
| 情绪直给 | "他感到XX""心里一XX""不由得"？ | 标注位置+建议用动作展示 |
| 结尾收束 | 章末停在总结性旁白？ | 标注+建议停在动作或对话 |
| 描写堆叠 | 连续3行以上纯感官描写无动作推进？ | 标注+建议插入动作 |
| 信息重复 | 同一信息一章内出现2次以上？ | 标注+建议删除 |
| 对话死板 | 连续6行以上纯对话无动作？ | 标注+建议穿插动作 |
| 高疲劳词 | 冷笑/蝼蚁/轰然炸裂/倒吸凉气/瞳孔骤缩/满场死寂等单章≥2次？ | 标注+建议替换 |

### 2b. 笔触DNA一致性检查

**启用态**（`素材/文风锚定.md`存在）：

| 检查项 | 判定标准 |
|:--|:--|
| 句式节奏 | 正文句式是否符合DNA描述的特征？ |
| 叙事距离 | 是否遵从DNA（近距跟随/中距旁观）？ |
| 对话引导词 | 是否遵从DNA（省略/用"道"/其他）？ |
| 感官顺序 | 是否遵从DNA（如触觉→视觉→听觉）？ |
| 情绪外化 | 是否遵从DNA（动作外化/物象外化/直述）？ |
| 系统面板格式 | DNA含面板→正文是否遵从？DNA无面板→正文是否擅自加？ |
| 章名 | 贴合内容命名，无噱头章名 |
| 章末钩子 | 主动决策型，禁止被动接收型 |

**trial模式**（无DNA）：检查全章风格自洽统一+未套用错误风格。

输出：本章DNA生效等级（强/中/弱/未生效）+ 下一章DNA执行包建议。

### 2c. 好看度4问

> 最主观也最重要。技法全✓不代表好看。**每问必须给具体证据**。

1. **这章有没有让人想往下翻的劲？**
   - 有 → 引用最抓人的1-2句正文
   - 没有 → 问题在哪？（节奏太慢/信息太少/主角太怂/废话太多/场景太常见）

2. **最抓人的画面/台词是什么？**
   - 找得到 → 引用原文
   - 找不到 → 标注"无记忆点"（缺乏记忆点=本章最大问题）

3. **哪里无聊了？**
   - 标注具体段落位置
   - 建议：删/压缩/插入动作/换场景

4. **代入感够不够？**
   - 读者能想象出画面吗？
   - 主角的选择和读者会重叠吗？
   - 配角像活人吗？

## Step 3：反向充足性检查（参考资料是否足以指导后续创作）

> 审完正文后反向评估：手上的参考资料是否足以指导下一章创作？

### 3a. 剧情白描充足性

| 检查项 | 标准 | 缺项处理 |
|:--|:--|:--|
| 下一章核心事件 | 是否有明确的beat？ | 缺→建议plot补下一章剧情白描 |
| 下一章爽感节点 | 是否已设计爽感类型+三段式？ | 缺→建议plot补爽感设计 |
| 下一章章末钩子 | 是否已规划？ | 缺→建议plot补钩子设计 |
| 当前卷幕序列 | 剩余章节是否有明确方向？ | 缺→建议plot补幕纲 |

### 3b. 设定库充足性

| 检查项 | 标准 | 缺项处理 |
|:--|:--|:--|
| 正文涉及但未定义的设定 | 设定被正文使用但设定库无定义？ | 缺→建议world补设定 |
| 下一章可能需要的设定 | 下一章场景涉及的设定是否已有？ | 缺→建议world补设定 |
| 力量体系完整性 | 当前境界后续是否有明确阶位？ | 缺→建议seed补骨架 |

### 3c. 角色卡充足性

| 检查项 | 标准 | 缺项处理 |
|:--|:--|:--|
| 新登场角色 | 是否有完整角色卡？ | 缺→建议character补角色卡 |
| 已有角色行为模式 | 是否足以指导后续写作？ | 缺→建议character补行为模式 |
| 角色关系网络 | 当前角色间关系是否明确？ | 缺→建议character补关系网络 |

### 3d. 燃料库充足性

| 检查项 | 标准 | 缺项处理 |
|:--|:--|:--|
| 可用燃料队列 | 是否有足够素材供后续章节？ | 缺→建议research补燃料 |
| 外部信息注入 | 是否需要新一轮WebSearch/调研？ | 缺→建议research执行调研 |

### 产出：补强建议清单

```markdown
## 反向充足性检查结论

| 维度 | 状态 | 补强建议 |
|:--|:--|:--|
| 剧情白描 | ✅充足 / ⚠️需补强 | （具体建议） |
| 设定库 | ✅充足 / ⚠️需补强 | （具体建议） |
| 角色卡 | ✅充足 / ⚠️需补强 | （具体建议） |
| 燃料库 | ✅充足 / ⚠️需补强 | （具体建议） |
```

**红线**：即使全部充足也要逐项确认"充足"。不能跳过反向检查。
