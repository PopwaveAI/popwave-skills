# CHANGELOG

## v11.2.0 (2026-07-16)

### DNA提取钩子vs核心方法论区分

**根因**：老板发现R21测试中DNA把参考书的前期钩子（斩杀线）误当成了核心系统设计。斩杀线只在ch01出现制造紧迫感，ch03回血后从未复现——后续压力全来自外部（债主/警察/帮派/boss）。但DNA提取把斩杀线写成"关键设计"，迁移指引建议"需加HP/斩杀线机制"，导致seed把斩杀线设计成了金手指核心功能。

**改动**：
- step0新增"提取原则：区分前期钩子和核心方法论"——对每个元素标注出现章节，只在前1-2章出现的标注`[钩子]`，跨章节复现的标注`[核心]`
- step0新增"0b-后检：钩子vs核心方法论分类清单"——落盘前对所有元素做分类检查
- step0维度1观察问题改为"系统在多个章节反复使用的核心机制是什么？哪些只在开篇出现后续不复现？"
- step0维度3加入"压力源是否持续？还是只作为开篇钩子后续被替代？"检查
- step2 DNA约束表拆为"核心方法论→作为设计约束"和"前期钩子→只参考不复制"两部分
- step2删除"主角生存压力源"作为独立约束行——压力节奏合入单元剧循环约束
- reference/dna-example.md更新：斩杀线标注`[钩子]`，功能模块表加持续性列，压力曲线表加持续性列，总结表拆为钩子vs核心两部分
- SKILL.md红线第7条：DNA提取必须区分`[钩子]`和`[核心]`
