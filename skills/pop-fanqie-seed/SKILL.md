# pop-fanqie-seed · 番茄创意搜索引擎

> 从真实top20文本反推追读驱动，10个创意选3融合1，结构化验证后落盘。
> R16测试验证：真实文本反推（G6）6:0完胜搜索摘要反推（G5）。

---

## SOP骨架

### Phase 1: 真实文本反推 → 创意发散 → 选+融合 → 展开

**1a. 问需求**

| 用户回答 | 路径 |
|---------|------|
| 直接给了创意/画面/一句话 | 跳过Phase 1，直接进Phase 2验证 |
| 给了方向/偏好（"想写都市异能""想写神医文"） | 进1b真实文本反推 |
| 完全开放（"你看着来"） | 扫番茄/起点热门榜单定方向，再进1b |

**1b. 真实文本反推**

从本地番茄top20小说文本中选3本与用户方向最相关的作品，读取每本开头文本（前3000-5000字），逐本分析。

**番茄top20小说文本目录**：`D:\workspace\参考小说txt\番茄top20\`

**选本逻辑**：优先选赛道内作品；赛道内不足3本时，补充跨赛道top20（R16证明跨赛道借鉴反而能提取更底层的追读驱动）。

**逐本分析三要素**（从实际文本提取，禁止印象流编造）：
- **追读驱动**：读者翻页是因为想知道什么具体问题？
- **爽感来源**：什么体验让读者爽？
- **情绪钩子**：开头用什么抓住了读者？

**赛道追读驱动总结**（1b的核心产出）：
1. 读者真正在追的体验（从3本共同点归纳）
2. 最有效的爽感类型（按效果排序）
3. 还没用过的手法（3本都没用过的=新作品机会）

**借鉴与避雷**：借鉴有效手法，避雷已占的设定/金手指/开局模式。

**1c. 创意发散**

基于1b的市场认知，发散10个一句话创意。每个创意一句话，带画面，带限制感。
- 10个方向必须差异最大化
- 每个要有具体行为×反差场景
- 优先填充1b"还没用过的手法"

**1d. 选+融合**

从10个创意中选3个融合为1：
- **A的画面**：视觉冲击最强的
- **B的限制**：张力最大的
- **C的反差**：认知颠覆最强的

融合方法：A画面作开篇，B限制作金手指约束，C反差作核心冲突。融合后必须是一个连贯的故事前提。

**无用户交互时**：自动选最优融合。
**有用户交互时**：把10个创意呈现给用户选择。

**1e. 展开**

把融合后的创意展开为完整种子：合成金手指（创意×机制×限制）+ 核心爆点 + 番茄简介。
- 核心爆点：一句话定位读者付费核心——读者为什么追这本书？最爽的那个点是什么？
详见 reference/showcase.md 的合成示例库和 reference/synopsis-guide.md 的简介写法参考。

**1f. 用户确认**

把展开后的种子呈现给用户。确认→进Phase 2。修改→回到1d或1e调整。
**无用户交互时**：自动继续Phase 2。

---

### Phase 2: 验证

**2a. 四眼法验证**

验证对象是展开后的金手指：

1. **有没有画面？** — 读完脑子里立刻浮现场景
2. **限制条件是不是天然存在？** — 不是硬加的
3. **能不能想象出十几种不同场景？** — 第3个场景不会腻
4. **有没有行为引擎？** — 主角每章在做一件具体的事（谈判/把脉/写手册），还是在操作一个系统（抽卡/升级）？

四项全过→放行，进Phase 3。任何一项不过→打回，重新展开。

---

### Phase 3: 落盘

**只产出一个文件**：`创意.md`

```
# 创意

## 金手指
- 创意：（一句话）
- 创意来源：（融合来源：A画面+B限制+C反差）
- 核心爆点：（一句话——读者为什么追这本书？最爽的那个点）
- 入口能力：（最劲爆的那个能力，写进简介的）
- 进化方向：（一句话——能力体系往哪发展）
- 机制：（一句话）
- 限制：（一句话）
- 合成金手指：（一句话——创意×机制×限制的产物）

## 番茄简介
【关键词1】+【关键词2】+【关键词3】+【关键词4】

（150-300字散文体简介，必须包含：创意展示、限制暗示、第一个爽点的期待感、至少2句角色台词、情绪弧线）
```

不写种子文档。不写100章推导。不写世界规则。不写节奏框架。不写笔触风格。
**创意就是种子。金手指就是引擎。拿到创意直接去写。**

## 红线

1. Phase 1必须从真实top20文本反推追读驱动——禁止印象流编造
2. 逐本分析三要素必须引用具体文本细节——禁止泛化描述
3. 创意发散必须产出≥10个差异化方向——不得少于10个，不得同质化
4. 必须10选3融合1——A画面+B限制+C反差，不得直接用10个中的某一个
5. 金手指三要素齐全——创意×机制×限制缺一不可
6. 四眼法四项全过才放行
7. 只产出`创意.md`——不写种子文档/世界规则/节奏框架/笔触风格
8. 创意不得与番茄Top20核心机制雷同
9. 番茄简介必须是散文体——带对话、带情绪弧线、带钩子，不是概述
