# pop-fanqie-seed · 番茄概念种子构建器

> 梗是种子，标签是土壤，脑洞是芽。好梗自带画面、自带反差、自带传播力。
> seed的核心能力是搜梗——从全网找到那个让人一看就想读的梗，融合标签形成完整脑洞，三眼法验证能不能跑长线。

---

## SOP骨架

### Step 1: 搜梗

梗 = 一个具体的、有反差的、能让人脑补画面的点子。不是抽象标签，不是概念描述，是那个让人一看就"卧槽这个有意思"的东西。

**1a. 接收用户输入（三条路径）**

**路径A：用户直接给梗**
用户给了一个具体的梗。
- "走夜路被黄鼠狼拦住问话，用公文套话回答"
- "上辈子救落水女子而死，此生要复仇人间一切女拳"
- "合乎周礼——用古代周礼句式怼现代场景"

→ 跳到Step 2融合。

**路径B：用户给方向/标签**
用户给了题材方向或关键词标签，但没给具体梗。
- "我想写个都市+规则怪谈的"
- "有没有校园题材的爆点"

→ agent搜索找梗（见1b）。

**路径C：用户什么都没给**
→ agent自主搜梗（见1b）。

**1b. 搜索找梗（核心能力）**

当用户没给具体梗时，必须执行搜索找梗。不是凭记忆编，是去全网找正在传播的、有热度的、有反差的梗。

**搜索策略——3轮搜索覆盖3个来源：**

| 轮次 | 搜索词公式 | 找什么 |
|------|---------|------|
| 第1轮 | 题材方向 + "梗/热梗/meme/流行" | 找正在传播的梗（B站/抖音/贴吧/知乎） |
| 第2轮 | "番茄小说/网文" + 题材方向 + "热门/高赞/推荐" | 找已验证的爆款概念 |
| 第3轮 | 题材方向 + "反差/反转/搞笑/离谱" + "真实事件/案例" | 找现实中的反差素材 |

**搜梗标准——好梗必须满足3条：**
- **有画面**：读完脑子里立刻浮现场景，不用想
- **有反差**：两个不该在一起的东西放在一起了（古代×现代、严肃×日常、高贵×卑微）
- **有传播力**：不是agent觉得有趣，是网上已经有人在讨论/转发/二创

**搜梗产出——至少给用户3个梗候选：**

```
## 梗候选

### 梗1：[梗名/一句话]
来源：[哪里搜到的——B站/抖音/知乎/番茄小说/新闻]
反差点：[什么和什么放在一起了]
画面：[一句话描述脑补的场景]
可扩展方向：[校园/都市/商战/玄幻...]（标注能跑多远）

### 梗2：...
### 梗3：...
```

用户选1个，或要求重新搜。

### Step 2: 融合

选定梗之后，融合标签形成完整脑洞。不是简单贴标签，是让标签和梗产生化学反应。

**2a. 标签融合**

从梗出发，匹配3-5个标签：

| 标签类型 | 数量 | 作用 |
|---------|------|------|
| 题材/流派 | 2个 | 定位赛道（如【都市】【规则怪谈】） |
| 核心元素 | 1-2个 | 梗的具象化（如【阴间体制】【搞笑】） |
| 卖点 | 0-1个 | 读者钩子（如【无女主】【杀伐果断】） |

标签来源：梗自带的方向 + 用户给的偏好 + agent判断的最佳匹配。

**2b. 脑洞成型**

梗 + 标签融合后，形成一句话完整脑洞：

```
脑洞 = 梗（核心反差点）
     × 标签（赛道+元素+卖点）  
     × 限制（"但是"——概念自带的约束）
     = 一句话能让人想读的故事前提
```

例子：
- 梗"合乎周礼" × 标签【都市】【搞笑】【古文×现代】 × 限制"只能说合乎周礼的话，不合礼数的话说了会遭反噬"
- → "一个现代人被迫只能用周礼文言文说话，在校园、职场、商战里用古代礼仪暴打现代人，但说错一句就遭反噬"

**2c. 三眼法验证**

一眼看三个维度，任何一个不过就打回换梗：

**2c-1. 有没有画面？**
- ✅ "黄鼠狼拦住问话" → 画面立刻浮现
- ❌ "获得成长型系统" → 没画面

**2c-2. 限制条件是不是天然存在？**
- ✅ "时停1分钟" → 1分钟就是限制
- ❌ "无敌系统" → 没限制

**2c-3. 能不能想象出十几种不同场景？**
- ✅ "合乎周礼" → 校园怼老师、职场怼领导、商战怼对手、相亲怼嘉宾、直播怼喷子……
- ❌ "获得读心术" → 第3个场景就腻

**验证结果**：
- 三项全过 → 放行，进Step 3
- 任何一项不过 → 打回，换梗或调整融合方式，重新验证

### Step 3: 落盘

**3a. 番茄简介（首要产出）**

```
【关键词1】+【关键词2】+【关键词3】+【关键词4】

简介（150-300字，必须包含：梗展示+限制暗示+第一个爽点的期待感）
```

简介写法：不要写大纲，写"钩子"。读完简介读者应该想知道"然后呢？"。

**3b. 种子文档**

写入 `概念种子文档.md`：

```
# 概念种子

## 番茄简介
（3a的产出）

## 梗
- 核心梗：（一句话——那个让人想读的反差点）
- 来源：（用户给的/搜索找到的——哪里来的）
- 反差结构：（什么×什么）

## 概念拆解
- 核心机制：（1句话——概念怎么运作）
- 限制条件：（1句话——概念的"但是"）
- 画面：（1句话——第一个浮现场景）

## 兑现者
- 主角：姓名 | 概念关系 | 每章核心行为模式 | 一句话性格标签
- 配角2-3个：姓名 | 与概念的关系 | 功能定位

## 最小世界规则（≤5条）
| 规则 | 内容 | 概念如何利用 |

## 节奏框架
- 节奏模型：爽文快节奏 / 悬疑钩子 / 情感蓄力 / 智力碾压（选1）
- 爽感公式：从11种中选3-5种适合本概念的
- 开篇规则：A级(冲突+概念前300字) / B级(冲突前300字，概念延后≤1章)
- 12岁读者模型：每30秒需新内容，连续3次纯氛围则退出

## 可扩展方向
- 阶段1：[场景类型]（ch1-N）
- 阶段2：[场景类型]（chN+1-M）
- 阶段3：[场景类型]（chM+1-...）
（给plot Step 0危机阶梯做参考，不强制）

## 笔触风格
- 关键词（3-5个）：
- 叙事人称：
- 对话风格：
- 例外条款（不写什么）：
```

**3c. 防雷验证**

与番茄Top20核心概念比对，雷同=打回重选。不雷同=通过。

## 红线

1. 搜梗必须调用搜索工具——禁止凭记忆编造梗，禁止用"通常/一般来说"代替
2. 必须给用户≥3个梗候选——不得只给1个让用户没得选
3. 三眼法三项全过才放行——任何一项不过就换梗
4. 不建世界圣经/力量体系/角色储备池/不做100章推导
5. 梗必须有天然限制条件——无限制=无张力
6. 梗不得与番茄Top20核心机制雷同
7. 番茄简介必须有≥3个关键词标签+完整简介段落
