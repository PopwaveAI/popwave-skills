# pop-fanqie-seed · 番茄创意引擎

> 自由出创意，结构做打磨。从无到有把创意打磨成可试读的开篇。v13.18.0

## 做什么

| 输入 | 输出 | 下游 |
|:-----|:-----|:-----|
| 用户一句话方向/偏好/参考书意图 | `设计/创意.md` + `正文/ch001.md` | pop-fanqie-world, pop-fanqie-plot |

**定位**：seed负责创意到首章。世界构筑移交plot，seed不碰。用户提到参考书时，seed 1a 路由到前置供给（tool-download-webnovel+pop-dna-style/pop-decon），完成后回 1b。

## 怎么操作

> execution.mode: 三阶段串行 | 强保障：本 SKILL.md 由 host 层每次 run 强制注入 | 弱保障：steps/ + reference/ 需 agent 主动 readFile

| 阶段 | 做什么 | 产出 | step 文件 |
|:-----|:-----|:-----|:----------|
| Phase 1 | 七维底牌摸底→参考书闸门→市场调研→题材深度调研(条件)→改编策略分析→双轨发散(王道×猎奇各5)→用户选1 | 选定方向+策略+校准清单 | `steps/step1.md` |
| Phase 2 | 行为引擎检查→合成金手指→四眼法验证→故事纲领(三核心+营销层)→散文体简介→轻量主角轮廓 | `设计/创意.md` | `steps/step2.md` |
| Phase 3 | 基于创意.md写2000-2500字试读章 | `正文/ch001.md` | `steps/step3.md` |

## 红线

1. **读取协议**：读取 skill 文件用 `Get-Content -Encoding UTF8 -Raw`，禁用 Read 工具
2. 发散prompt必须纯自由——不注入搜索数据/格式模板/结构引导/借鉴点/避雷点，只给一句话方向+策略框架
3. 市场调研必须做——发散前WebSearch 3轮+借鉴点/避雷点+爽点类型清单，发散后事后校准（踩避雷点=淘汰）
4. 1d-0改编策略必须做——5+5发散前组装3-5个策略组装卡，用户选1-2个后才发散
5. Phase 1必须产出≥5个差异化方向——不得只给3个，不得同赛道，每句话读完浮现场景
6. Phase 2只在用户选定后启动——不在发散阶段做结构化
7. 金手指三要素齐全——创意×机制×限制缺一不可，禁止只加限制不加代价
8. 四眼法四项全过才放行——画面/限制/场景量/行为引擎
9. 只产出创意.md+ch001.md——不写骨架/世界规则/四张地图/敌人梯度（移交plot）
10. 首章必须好看——必须有"生活化场景×金手指"反差画面，纯战斗/纯穿越不算合格
11. 番茄简介必须是散文体——带对话、带情绪弧线、带钩子，不是概述/设定说明书
12. 故事纲领必须用三核心+营销层格式——世界4层/舞台·危机3要素/主角5要素/最大钩子≤20字/即时兑现感。走形式=废纲领
13. 全书最大卖点必须收集——1a必问，处置驱动型触发1c-5题材深度调研，禁止逃避"吞噬变强"
14. 创意不得与番茄Top20核心机制雷同——市场校准的避雷点校验必须执行
