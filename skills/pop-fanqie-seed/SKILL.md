# pop-fanqie-seed · 番茄立项引擎

> 线上搜索番茄top小说反推追读驱动 → 创意融合 → 完整立项包（金手指+主角+世界观+长线线索+简介）
> 方法论融合：吸收PRD立项环节（主角设定卡/金手指衍生性/总纲递进/世界观骨架/长线线索）

---

## SOP骨架

### Phase 1: 线上搜索 → 创意发散 → 选+融合 → 立项展开

**1a. 问需求**

| 用户回答 | 路径 |
|---------|------|
| 直接给了创意/画面/一句话 | 跳过1b，直接进1d（用用户创意为A画面，仍需补B限制+C反差） |
| 给了赛道/关键词 | 进1b线上搜索 |
| 完全开放 | 扫番茄热门榜单定方向，再进1b |

**1b. 线上番茄搜索**

用WebSearch按用户赛道+关键词搜索番茄热门小说：
- 搜"[赛道] 番茄小说 排行/热门/推荐"
- 找3本热门作品的：书名、简介、标签、评分
- 不需要读全文——从简介和标签提取信息

逐本分析三要素（从搜索到的公开信息提取，禁止印象流编造）：
- 追读驱动：读者翻页是因为想知道什么？
- 爽感来源：什么体验让读者爽？
- 情绪钩子：简介/标签用什么抓住了读者？

赛道总结：读者在追什么 + 最有效爽感类型 + 还没用过的手法。借鉴有效手法，避雷已占设定。

**1c. 创意发散**

基于1b市场认知，发散10个一句话创意。差异最大化。每个要有具体行为×反差场景。优先填充"还没用过的手法"。

**1d. 选+融合**

10选3融合1：A画面（视觉冲击最强）+ B限制（张力最大）+ C反差（认知颠覆最强）。融合后必须是连贯的故事前提。
无用户交互时自动选最优。有用户交互时呈现10个让用户选。

**1e. 立项展开**

把融合创意展开为完整立项包（7个产出）：

1. **一句话故事**（谁+想做什么+为什么）
2. **草纲**（谁+想做什么+为什么+如何做+阻力+结局方向）
3. **金手指**（创意×机制×限制 + 核心爆点 + 入口能力 + 进化方向含≥3种衍生分支）
4. **主角设定卡**（职业/主性格/记忆点[外貌特征或口头禅或习惯动作]/缺点/成长弧线[从→到]/描写方法）
5. **世界观骨架**（世界地图一句话/等级体系一句话/进度3段/核心规则1-2条与金手指相关）
6. **长线线索**（主线悬念1条+长线伏笔1-2条）
7. **番茄简介**（150-300字散文体，含创意展示/限制暗示/爽点期待/2句台词/情绪弧线）

详见 reference/showcase.md 合成示例库和 reference/synopsis-guide.md 简介写法。

**1f. 用户确认**

呈现立项包。确认→Phase 2。修改→回1d或1e。无用户交互时自动继续。

---

### Phase 2: 验证

**2a. 四眼法**

1. 有没有画面？— 读完脑子里立刻浮现场景
2. 限制条件天然存在？— 不是硬加的
3. 十几种不同场景？— 第3个不腻
4. 有行为引擎？— 主角每章在做具体的事（谈判/把脉/写手册），不是操作系统

**2b. 金手指衍生性检查**（方法论3.3）

评估金手指能生长出多少种不同类型的能力分支：
- ≥3种质变型分支（新类型能力，不只是更强/更快/更多）→ 合格
- 只有量的变化（更强/更快/更多）→ 不合格，打回重设计

四眼法+衍生性全过→放行。任何不过→打回。

## 红线

1. Phase 1必须从线上番茄搜索反推追读驱动——禁止印象流编造
2. 金手指必须有衍生性——能生长出≥3种质变型能力分支，不只是量的变化
3. 主角设定卡必须包含记忆点+缺点——缺一不可（代入感源头）
4. 四眼法四项全过+衍生性合格才放行——任何一项不过打回重设计
