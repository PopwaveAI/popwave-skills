## SOP骨架

### Phase 1: 标签搜索 → 创意发散 → 选+融合 → 立项展开

**1a. 问需求**

| 用户回答 | 路径 |
|---------|------|
| 直接给了创意/画面/一句话 | 跳过1b，直接进1d（用用户创意为A画面，仍需补B限制+C反差） |
| 给了赛道/关键词 | 进1b标签搜索 |
| 完全开放 | 扫番茄热门榜单定方向，再进1b |

**1b. 标签搜索**

用WebSearch按用户赛道+关键词搜索番茄热门小说。搜索策略：

第一步——用番茄标签精准定位细分赛道。标签体系三层：
- 主题标签：玄幻脑洞/都市异能/东方仙侠/异世大陆/都市高武等
- 角色标签：反派/天才/腹黑/大佬/全能等
- 情节标签：系统/重生/穿越/升级流/扮猪吃虎/打脸等

搜索词格式：`[主题标签] [情节标签] 番茄小说 热门/推荐/排行 2025 2026`

例如用户说"玄幻"：搜"东方玄幻 升级流 番茄小说 热门 2025"、"玄幻脑洞 系统 番茄小说 推荐"、"传统玄幻 扮猪吃虎 番茄小说 排行"——三次搜索覆盖不同细分。

第二步——找5-8本作品（不只3本头部，找中腰部作品也有参考价值）。提取：书名、简介、标签、评分。

逐本分析三要素（从搜索到的公开信息提取，禁止印象流编造）：
- 追读驱动：读者翻页是因为想知道什么？
- 爽感来源：什么体验让读者爽？
- 情绪钩子：简介/标签用什么抓住了读者？

赛道总结：读者在追什么 + 最有效爽感类型 + 还没用过的手法。借鉴有效手法，避雷已占设定。

**1c. 创意发散**

基于1b市场认知，发散10个一句话创意。差异最大化。每个要有具体行为×反差场景。优先填充"还没用过的手法"。

创意发散法（吸收换元法指令的置换思路）：
- 一度置换：把现实世界的行为框架（借贷/投资/看病/考试/打官司）置换到修仙/玄幻世界
- 二度置换：在一度置换基础上，改变行为主体或对象（把"借钱"变成"借灵根"）
- 极端法：把正常逻辑推到极端（"如果灵气要还利息呢"）
- 相反法：把常规设定反转（"如果越弱越强呢"）

**1d. 选+融合**

10选3融合1：A画面（视觉冲击最强）+ B限制（张力最大）+ C反差（认知颠覆最强）。融合后必须是连贯的故事前提。
无用户交互时自动选最优。有用户交互时呈现10个让用户选。

**1e. 立项展开**

把融合创意展开为完整立项包。每个产出都有推导逻辑，不是填空。

**产出1·一句话故事**：谁+想做什么+为什么

**产出2·草纲**：谁+想做什么+为什么+如何做+阻力+结局方向

**产出3·金手指**（吸收升级流指令的金手指设计框架）：

金手指设计铁律——**只加限制不加代价**：
- 限制 = 使用条件/边界（只能在子时用/只能对低于自己境界的人生效/每次间隔三天）
- 代价 = 用一次扣一年寿命/用一次忘一段记忆 ← **禁止**。代价让读者不爽
- 限制随剧情逐渐解锁——初始限制大，随主角成长逐步解除

金手指5要素：
1. 创意：一句话
2. 功能：能做什么（入口能力——最劲爆的那个，写进简介）
3. 触发条件：什么情况下能用（自然限制，不是惩罚）
4. 限制边界：使用条件/范围/冷却——初始限制严格，随成长解锁
5. 进化方向：含≥3种衍生分支（质变型，不是量的变化）

**产出4·主角设定卡**（吸收雪花写作法人物塑造+番茄过稿指令"人物深度/亮点/对比"）：

推导逻辑——从金手指反推主角：
1. 什么人最适合这个金手指？（把脉诊股→中医背景/灵债→精算师思维）
2. 记忆点：从职业/性格推导一个视觉化特征（习惯动作/口头禅/外貌标记）
3. 缺点：从金手指的限制推导——限制会诱发什么弱点？（精算→贪婪/把脉→多疑）
4. 成长弧线：从金手指进化方向推导——主角从"被金手指驱动"到"驾驭金手指"再到"超越金手指"

5要素：职业/身份、主性格、记忆点、缺点、成长弧线（从→到）

**产出5·世界观骨架**（世界自洽核心）：

推导逻辑——从金手指推导世界规则：
1. 金手指存在的前提是什么世界？（灵债→灵气是借贷体系/把脉→万物有可诊断的"脉"）
2. 这个世界怎么运转？（谁掌握资源/力量如何分配/普通人和修士什么关系）
3. 金手指与世界规则的挂钩点在哪？（灵债→天地法则强制执行/把脉→天道有因果线）

世界自洽四问（必须全部回答）：
- 能力从何来？— 金手指的起源有世界规则支撑
- 触发条件是什么？— 不是随意的，有世界内在逻辑
- 作用边界在哪？— 不能万能，有明确的天花板
- 世界规则如何执行？— 不是主角自己扣自己，是世界机制在运转

4要素：世界地图（一句话）、等级体系（一句话）、进度框架（3段式）、核心规则（1-2条与金手指挂钩的世界规则，必须回答自洽四问）

**产出6·长线线索**：

推导逻辑——从金手指进化方向推导长线：
1. 主线悬念：金手指的终极秘密是什么？（灵债→谁在收债/把脉→脉的尽头是什么）
2. 长线伏笔：金手指进化方向暗示的终极格局（灵债→飞升者被收割/把脉→天道本身就是病人）

2要素：主线悬念（1条）、长线伏笔（1-2条）

**产出7·番茄简介**（150-300字散文体，含创意展示/限制暗示/爽点期待/2句台词/情绪弧线）

详见 reference/showcase.md 合成示例库和 reference/synopsis-guide.md 简介写法。

**1f. 用户确认**

呈现立项包。确认→Phase 2。修改→回1d或1e。无用户交互时自动继续。

### Phase 2: 验证

**2a. 四眼法**

1. 有没有画面？— 读完脑子里立刻浮现场景
2. 限制条件天然存在？— 不是硬加的
3. 十几种不同场景？— 第3个不腻
4. 有行为引擎？— 主角每章在做具体的事（谈判/把脉/写手册），不是操作系统

**2b. 金手指衍生性检查**

评估金手指能生长出多少种不同类型的能力分支：
- ≥3种质变型分支（新类型能力，不只是更强/更快/更多）→ 合格
- 只有量的变化（更强/更快/更多）→ 不合格，打回重设计

**2c. 世界自洽检查**

回答四问：
1. 能力从何来？— 有世界规则支撑，不是天上掉下来的
2. 触发条件是什么？— 有世界内在逻辑，不是随意触发
3. 作用边界在哪？— 有明确天花板，不能万能
4. 世界规则如何执行？— 是世界机制运转，不是主角自己扣自己

四问任何一问答不上来或逻辑断裂→打回1e重设计。

**2d. 金手指代价检查**

检查金手指是否含"代价"（用一次扣XX/扣寿命/扣记忆等负反馈）：
- 有代价→打回重设计为限制（使用条件/边界/冷却）
- 只有限制→通过

四眼法+衍生性+世界自洽+代价检查全过→放行。任何不过→打回。

## 红线

1. Phase 1必须从线上番茄搜索反推追读驱动——禁止印象流编造
2. 金手指只加限制不加代价——使用条件/边界/冷却可以，扣寿命/扣记忆等负反馈禁止
3. 世界自洽四问必须全部回答——能力从何来/触发条件/作用边界/世界规则如何执行
4. 金手指必须有衍生性——≥3种质变型能力分支
5. 主角设定卡必须包含记忆点+缺点——从金手指反推，不是凭空编
