# Step 0: 参考书DNA提取（可选·有参考书时执行）

> 有参考书txt时，在Step 1之前执行本步。无参考书则跳过。
> 完整提取示例见 `reference/dna-example.md`（以《我在美国搞内战》为例）

---

## 0a. 连续读取参考书前100章

> 连续读取前100章（约一卷），不做间隔采样。DS 1M上下文可容纳约150万字，100章约80-100万字，完全够用。

### 读取范围

- **前100章全文**（或全书前1/3，取较小值）
- 如果全书不足100章，读完全书
- txt可能有网页杂质（分页标记/广告行），读取时自动过滤

### 读取执行

1. 扫描txt章节标记，确定总章数
2. 提取前100章正文（按章节标记分割）
3. 如果txt是单大文件无章节标记，按字符数估算：每章约5000-10000字（因平台而异），取前80-100万字

### 为什么是100章而不是更多

- **一卷的完整性**：100章约等于一卷，包含完整的开篇→发展→中段→高潮弧线
- **上下文预算**：100章约80-100万字，加上skill指令约2万字，总计不超过120万字，DS 1M token上下文可以容纳
- **方法论把控力**：100章连续阅读可以看到完整的节奏演变曲线、系统功能扩展轨迹、长线伏笔埋设与回收情况——间隔采样会遗漏中间的渐变过程

## 0b. 提取8维DNA

逐维度提取，每个维度包含：观察（1-2句）+ 原文证据（摘录关键段落）+ 迁移指引（如何用到我们的项目）。

### ⚠️ 提取原则：区分"前期钩子"和"核心方法论"

对每个提取的元素，标注它**在哪几章出现**：
- 只在开篇1-5章出现，后续不复现 = **前期钩子**（标注`[钩子]`）→ 只学方法论（"用系统机制制造前期紧迫感"），不复制具体机制
- 在多个章节反复出现 = **核心方法论**（标注`[核心]`）→ 作为设计约束迁移
- 前期频繁出现，中期减弱但不消失 = 标注`[核心·衰减]`

> 典型例子：参考书的"斩杀线"（血量低于5就死）只在ch01出现制造紧迫感，ch02回血后到ch100再没复现→这是前期钩子，不是核心系统设计。应该学的是"开篇需要一个系统级紧迫感钩子"，不是"设计斩杀线"。

### 维度1·系统设计模式

观察参考书的金手指/系统怎么设计：
- 系统在多个章节反复使用的核心机制是什么？（不只是战斗——还有什么是系统管的？）
- 哪些机制只在开篇出现后续不复现？（前期钩子）
- 系统面板包含哪些字段？面板字段是否在中后期扩展或缩减？
- 系统如何驱动主角决策？这个驱动力是否持续？ch50/100的系统驱动力是否跟ch01一致？
- 系统的"非战斗功能"有哪些？（商店/任务/被动技能/社交技能/其他？）非战斗功能是否跨章节持续使用？还是前期用完后期废弃？

原文证据：摘录2-3段系统面板弹出的段落（含开篇和中后期各至少1段）
迁移指引：核心方法论→我们的金手指需要补充什么维度？前期钩子→我们的开篇需要什么紧迫感钩子（设计自己的，不复制参考书的具体机制）？

### 维度2·剧情节奏样本

拆成两层：

**单元剧循环**：参考书的"变种事件"循环模式是什么？每个单元剧的固定流程是什么？
- 每个单元剧的起止节点是什么？（遇到什么开始，获得什么结束？）
- 单元剧之间怎么变种？（换NPC类型/换交互方式/换资源类型/换系统功能？）
- 前10章的单元剧模式 vs ch50的单元剧模式 vs ch100的单元剧模式——变种了什么？保留了什么？
- 每几个单元剧后引入新元素？

**长线剧情推进**：参考书怎么在单元剧之间推进长线？
- 长线线索什么时候埋？什么时候揭示？是否回收？
- 单元剧和长线怎么交织？
- boss什么时候首次暗示？什么时候正式出场？出场后是否持续存在？

逐章拆解beat sheet（按10章一组分块，每组列核心事件+系统交互+战斗比例+章末钩子）：
- 每章的核心事件（1句话）
- 每章的系统交互次数
- 每章的战斗vs非战斗比例
- 章末钩子是什么类型？
- 标注：单元剧循环章 / 长线推进章

迁移指引：我们的plot骨架需要调整什么节奏？

### 维度3·主角生存压力源（合入节奏样本）

观察主角为什么非干不可——压力是节奏的一部分，不是独立维度：
- 生存压力是什么？（病/饿/被追杀/资源耗尽/其他？）
- 压力是系统级的还是外部级的？（系统级=系统机制逼的；外部级=被人追杀）
- **这个压力源是否持续？** 还是只作为开篇钩子后续被其他压力替代？ch50/100的压力来源跟ch01是否一致？
- 每章的压力变化轨迹（压力升降曲线）——注意标注压力来源是否切换
- 压力与单元剧循环的关系：每个单元剧是否改变压力状态？

原文证据：摘录1-2段主角计算生存资源的段落（开篇和中后期各至少1段）
迁移指引：核心压力机制→我们的主角危机怎么设计？前期钩子→开篇用什么紧迫感（设计自己的，不复制参考书的具体机制）？

### 维度4·社会交互模式

观察非战斗社会交互怎么写：
- 主角跟谁交互？（不是敌人的人）
- 交互中系统是否参与？（被动技能/任务触发/交易结算？）
- 社会交互解决了什么问题？（获取资源/获取信息/建立关系？）
- NPC的"系统标注"怎么设计？（系统是否给NPC加标签？标签是否影响主角决策？）

迁移指引：我们的plot骨架缺什么社会交互场景？

### 维度5·资源经济系统

观察资源怎么创造张力：
- 主角缺什么资源？
- 资源如何获取？（战斗/交易/任务/探索？）
- 资源稀缺如何驱动决策？
- 资源是否有替代品？（低成本替代品 vs 高成本正品？）
- 资源是否可转换？（A→B→C→D的转换链？）
- 信息本身是否是资源？

原文证据：摘录1段主角盘点资源的段落
迁移指引：我们的世界观需要补充什么经济系统？

### 维度6·世界观转译机制

观察系统如何把现实世界转译成另一个认知框架——这是很多系统流的核心设计：
- 系统是否给现实世界起了"另一个名字"？
- 现实中的角色被转译成什么？（基于什么标准判定？社会角色/恶意程度/其他？）
- 现实中的物品/事件被转译成什么？
- 转译是纯装饰还是有功能？（转译是否影响主角决策？）

> 注意：并非所有参考书都有转译机制。如果没有，记录"无转译"并跳过此维度。

原文证据：摘录2-3段系统转译的段落
迁移指引：我们的系统需要什么转译机制？（结合我们的世界观背景设计）

### 维度7·主角设计

观察参考书的主角怎么设计——提取设计方法不是复制人设：
- 主角的前世/既往身份如何赋能？（身份不只是记忆点——它给了主角什么能力和社交筹码？）
- 主角的初始弱点是什么？（极端弱势还是一般弱势？）
- 主角的核心生存策略是什么？（战斗力？交易？信息差？社交？其他？）
- 主角怎么利用系统规则做非战斗决策？
- 主角的成长方向是什么？（变强？建势力网？获取信息？控制资源？其他？）

原文证据：摘录1段主角做决策的段落（展示生存策略）
迁移指引：我们的主角设计需要调整什么？

### 维度8·Boss设计

观察参考书的boss怎么设计：
- boss的首次暗示在什么时候？（通过什么方式暗示？NPC提及/环境暗示/系统标注？）
- boss正式出场前有多少层铺垫？
- boss的设计维度：身份/核心欲望/致命弱点/与主角的关系/系统标注（如有）
- boss与主角的第一次交互方式是什么？（战斗？谈判？测试？利用？）
- boss的势力结构是什么？（手下/盟友/敌人）
- boss出场后是否持续存在？还是出场即消失？

原文证据：摘录boss首次出场的段落
迁移指引：我们的boss设计需要什么维度？

## 0b-后检. 钩子vs核心方法论分类清单

落盘前，对8个维度提取的所有元素做一次分类检查。**关键依据：前100章的持续性验证**。

| 元素 | 开篇(ch01-10) | 中段(ch40-60) | 后段(ch80-100) | 分类 | 迁移方式 |
|------|---------------|---------------|---------------|------|---------|
| （如：斩杀线） | ch01出现 | 不复现 | 不复现 | `[钩子]` | 只学"开篇需系统级紧迫感"，不复制斩杀线 |
| （如：NPC标注） | ch01-10频繁 | ch40-60复现 | ch80-100复现 | `[核心]` | 作为设计约束迁移 |
| （如：搜尸获得技能） | ch01/03/05 | ch32复现 | ch90复现 | `[核心]` | 作为设计约束迁移 |
| ... | | | | | |

**判断规则**：
- 开篇出现 + 中段不复现 = `[钩子]`
- 开篇出现 + 中段复现 = `[核心]`
- 开篇出现 + 中段复现 + 后段不复现 = `[核心·衰减]`——方法论有效但需注意后续可能弱化

**原则**：`[钩子]`元素不进入step2的DNA约束表，只在"开篇钩子参考"中列出。`[核心]`元素才作为设计约束。
