# Step 1: 市场调研 + 纯自由发散（Phase 1）

> **核心原则：发散prompt不注入任何搜索数据，但发散方向必须经过市场校准。**
> R9测试验证——纯自由模式（11字符system+一句话user）产出的创意质量远高于注入搜索数据/格式模板/结构引导的模式。"中医把脉诊断股票""殡葬直播间全网等投胎"这些最佳梗都是纯自由模式下诞生的。
>
> 但R13/R14的《诡编》级别好创意之所以能踩在读者爽点上，还有另一个关键——**市场调研做校准**：先WebSearch摸清当前什么在火+已有模式是什么，产出"借鉴点+避雷点"，发散时避开已有模式（避雷）+借鉴已验证的爽感机制（借鉴）。
>
> **两者不矛盾**：R9反对的是"把搜索结果塞进发散prompt"（约束发散），R13做的是"先市场分析再指导发散方向"（校准方向，不约束发散）。

---

## 1a. 问需求

第一步问用户想写什么。根据回答分路径：

| 用户回答 | 路径 |
|---------|------|
| 直接给了梗/画面/一句话 | 跳过Phase 1，直接进Step 2打磨 |
| 给了方向/偏好（"想写都市异能""想写神医文"） | 进1b市场调研→1c纯自由发散 |
| 完全开放（"你看着来"） | 进1b市场调研（扫榜定方向）→1c纯自由发散 |
| 提到参考书（"参考《深渊主宰》的笔触写一本XXX"） | 路由前置供给→完成后回1b市场调研 |
| 想精拆做立项圣经（"我想精拆《玄鉴仙族》"） | 路由pop-decon→完成后回1b市场调研 |

**前置供给路由**（当用户提到参考书时）：

| 用户意图 | 路由 | 说明 |
|:--|:--|:--|
| 参考XX的笔触 | tool-download-webnovel下载→pop-dna-style提取DNA | 产出`涌现/文风锚定.md`供write消费 |
| 精拆XX做立项圣经 | pop-decon Phase 1-4精拆 | 产出设定库/剧情库供plot消费 |
| 两者都要 | download→decon+dna-style并行 | 两个产出都部署到项目本地 |

前置供给完成后，回到seed继续1b市场调研。DNA产出部署到`涌现/文风锚定.md`，write在主线渲染时消费（seed阶段不消费DNA，DNA是write的笔触层供给）。

**阻塞性步骤**：必须等用户回答后才能继续。禁止直接进入1b。

## 1b. 市场调研（WebSearch + 借鉴点/避雷点分析）

> R13/R14核心环节。**这一步的产出是"借鉴点+避雷点清单"，不是搜索数据本身**。搜索数据消化完后不进入发散prompt。

### 1b-1. WebSearch（3轮）

搜索3轮，找当前赛道什么在火+已有模式是什么：

- 第1轮：`"番茄小说/起点中文网" + "{赛道} + 热门/排行榜/2026"` → 找当前榜单热门作品
- 第2轮：`"B站/抖音/知乎" + "{赛道} + 小说推荐/出圈" + "2026"` → 找正在出圈的概念
- 第3轮：`"网文" + "{赛道} + 新题材/新趋势/审美疲劳" + "2026"` → 找正在冒头的新方向+读者已疲劳的老模式

**用户完全开放时**：先扫番茄/起点总榜定方向，再用方向做1b。

### 1b-2. 市场分析（借鉴点 + 避雷点）

对每部热门作品/每种热门模式，分两层标注：

```markdown
1. 《书名/模式》
   借鉴：[可借鉴的机制/爽感/节奏/读者期待]
   避雷：[必须避开的已有设定/金手指/开局模式]
```

**借鉴点示例**（R13诡异流分析）：
- 《诡舍》：借鉴"规则怪谈+无限轮回的生死游戏张力"，避雷"密室逃生+规则怪谈"已有开局
- 《捞尸人》：借鉴"冷门行当的新鲜感+中式民俗恐怖"，避雷"捞尸行当+克苏鲁"已有设定
- 趋势借鉴："主动对抗规则"是已验证爽感方向（"被动求生"已审美疲劳）

### 1b-3. 产出校准清单

市场分析消化后，产出一张**借鉴点+避雷点清单**：

```markdown
## 市场校准清单（赛道：{用户方向}）

### 借鉴点（可借鉴的机制/爽感/节奏）
1. [机制/爽感1] —— 来源：《XXX》
2. [机制/爽感2] —— 来源：趋势
...

### 避雷点（必须避开的已有模式）
1. [已有设定/金手指/开局1] —— 来源：《XXX》
2. [已有设定/金手指/开局2] —— 来源：趋势
...
```

**这份清单给agent看，不进入1c发散prompt**。agent在1c发散后用这张清单做校验。

## 1c. 纯自由发散

> **核心：发散prompt不注入任何搜索数据/借鉴点/避雷点。** 只给一句话方向，让模型自由发挥。市场校准在发散**之后**做，不在发散**之中**做。

### 1c-1. 发散（纯自由）

**发散指令**（这是给子agent/API的全部输入）：

```
system: 你是一个网文创意编辑。
user: 我想写{用户方向}，请帮我生成{N}个完全不同方向的开书创意种子。
```

N = 5-6（不是3。R9证明模型在多产出时创意发散性更强，3个太少容易同质化）

**关键约束（给agent的，不给API的）**：
- 5-6个方向必须差异最大化——不同切入点、不同主角类型、不同情绪基调
- 不套框架，不加限制，不想机制——只管有没有意思
- 参考"给怪谈写员工手册""中医把脉诊断股票"这种好梗的特质：**一个具体行为×一个反差场景，一句话就能让人脑补画面**
- **画面优先**：每句话读完脑子里立刻浮现场景。没有画面的创意直接淘汰

### 1c-2. 好梗标准（agent自检，不通过重新发散）

每个发散的梗必须满足：
- **有画面**：读完脑子里立刻浮现场景，不用想
- **有反差**：两个不该在一起的东西放在一起了（古代×现代、严肃×日常、高贵×卑微、恐怖×生活化）
- **有意思**：读者看完会不会"卧槽这个我想看"？如果只是"逻辑成立但没意思"，淘汰

**反面教材（禁止产出的类型）**：
- "主角穿越到XX世界，获得XX系统，开始升级" — 纯事件，无画面无反差
- "主角有XX能力，限制是XX，通过XX变强" — 纯机制，无生活化场景
- "主角在XX势力中崛起，对抗XX敌人" — 纯势力操盘，无有趣行为

**正面教材（R13/R14级别的好梗）**：
- 推开凶宅大门，女鬼扑来，我在她脑门上看到属性面板，把"暴躁"改成"正在织毛衣"
- 被黄鼠狼拦路讨封，用红头文件格式回答，它必须照办
- 殡葬直播间，全网围观亡魂投胎
- 给怪谈写员工手册——禁止食用同事

### 1c-3. 市场校准（用1b清单做校验）

发散出5-6个梗后，用1b的市场校准清单做**事后校验**：

| 梗 | 踩避雷点？ | 用到借鉴点？ | 校验结果 |
|----|----------|------------|---------|
| 梗1 | ✗（避开已有模式） | ✓（用了XX机制） | 放行 |
| 梗2 | ✓（踩了《XXX》已有设定） | — | 淘汰，重新发散 |
| 梗3 | ✗ | ✗ | 放行（不强制用借鉴点） |

**校验规则**：
- **踩避雷点=淘汰**：梗的核心设定/金手指/开局与已有热门作品雷同，淘汰重发散
- **用到借鉴点=加分**：梗借鉴了已验证的爽感机制（如"主动对抗规则"），优先选入1d
- **不强制用借鉴点**：梗没踩避雷点也没用借鉴点，只要画面/反差/有意思达标，照样放行

**为什么事后校准而不是事前注入**：事前注入=把搜索数据塞进发散prompt=约束发散=R9验证反对的。事后校准=发散完再用清单筛=不约束发散只过滤=保持纯自由发散的画面优先优势。

## 1d. 用户选择

把校验通过的梗呈现给用户，让用户选1个。格式：

```
## 创意方向

### 创意1：[名称]
一句话描述：[主角是谁，在做什么，遇到了什么]
画面：[脑补的场景，最好带一句台词]
为什么有意思：[吸引人的点]
市场定位：[借鉴了什么已验证机制 / 避开了什么已有模式]

### 创意2：...
...
```

用户选1个，或要求重新发散。

**无用户交互时**（API直调/自动化测试）：自动选最有画面感+反差感+用到借鉴点的1个，继续Step 2。
