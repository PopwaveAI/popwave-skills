# Step 1: 参考书摸底 + 市场调研 + 双轨发散（Phase 1）

> **核心原则：双轨发散——A轨王道赛道扫描排行榜借鉴+融合微调（5个），B轨猎奇赛道纯自由发散（5个）。共10个立项。**
> R9验证纯自由发散创意质量高但偏向猎奇（"殡葬直播间""中医诊断股票"），R13/R14验证借鉴排行榜是出好创意的重要校准。两者互补：王道给安全牌，猎奇给出圈牌。

---

## 1a. 问需求

第一步同时摸排两个维度：**方向摸底** + **参考书摸底**。两个维度独立执行，但参考书摸底必须在进入1b市场调研前完成。

### 方向摸底

| 用户回答 | 路径 |
|---------|------|
| 直接给了梗/画面/一句话 | 跳过Phase 1发散，直接进Step 2打磨 |
| 给了方向/偏好（"想写都市异能""想写神医文"） | 进1b参考书闸门→1c市场调研→1d双轨发散 |
| 完全开放（"你看着来"） | 进1b参考书闸门→1c市场调研（含扫榜定方向）→1d双轨发散 |

### 参考书摸底（必问，pipeline Phase 0入口）

> **红线**：方向摸底后的第一步不是进市场调研，是**问参考书**。参考书未就绪且用户未明确拒绝，不得进入1c市场调研。

主动问用户：
> "开工前先确认一下——有没有想参考笔触的书？比如觉得哪本书的节奏/爽感/写法对胃口？如果还没想好，我可以帮你搜几本同赛道卖得好的参考。"

根据回答路由：

| 用户回答 | 路由 |
|---------|------|
| 给了书名（"参考《深渊主宰》的笔触"） | → tool-download-webnovel 下载 → pop-dna-style 提取笔触DNA（档位A必做+档位B剧情brief可选）→ 完成后进1c市场调研 |
| 没想好，需要推荐 | → pop-research（种子级）调研同赛道热门 → 产出推荐书单 → 用户选1-2本 → download + dna-style → 完成后进1c市场调研 |
| 明确拒绝（"不需要参考书"） | → 再次确认风险（write将因缺失DNA进trial模式，笔触质量由API自由发挥）。用户坚持拒绝→标注风险→进1c市场调研 |
| 想在发散前先精拆一本书 | → tool-download-webnovel 下载 → pop-decon Phase 1-4 精拆 → 产出沉淀到 `设定库/剧情库/立项库/` → 完成后回1b参考书摸底（仍需提取笔触DNA） |

**参考书摸底完成后**，更新project-state.md中的参考书条目：
- 书名 / 本地文件路径 / 笔触DNA状态 / 档位

## 1b. 参考书闸门（pipeline Phase 0 出口）

进入1c市场调研前的最后检查：

- ✅ 参考书已下载 + 笔触DNA已部署（涌现/文风锚定.md存在） → 放行，进1c
- ✅ 用户已明确拒绝参考书（project-state.md标注"无参考书风险"） → 放行，进1c
- ❌ 参考书未就绪且用户未拒绝 → **回到1a参考书摸底，不得跳过**

## 1c. 市场调研（WebSearch + 借鉴点/避雷点分析）

> R13/R14核心环节。**这一步的产出是"借鉴点+避雷点清单"，不是搜索数据本身**。搜索数据消化完后不进入发散prompt。

### 1c-1. WebSearch（3轮）

搜索3轮，找当前赛道什么在火+已有模式是什么：

- 第1轮：`"番茄小说/起点中文网" + "{赛道} + 热门/排行榜/2026"` → 找当前榜单热门作品
- 第2轮：`"B站/抖音/知乎" + "{赛道} + 小说推荐/出圈" + "2026"` → 找正在出圈的概念
- 第3轮：`"网文" + "{赛道} + 新题材/新趋势/审美疲劳" + "2026"` → 找正在冒头的新方向+读者已疲劳的老模式

**用户完全开放时**：先扫番茄/起点总榜定方向，再用方向做1c。

### 1c-2. 市场分析（借鉴点 + 避雷点）

对每部热门作品/每种热门模式，分两层标注：

```markdown
1. 《书名/模式》
   借鉴：[可借鉴的机制/爽感/节奏/读者期待]
   避雷：[必须避开的已有设定/金手指/开局模式]
```

**借鉴点示例**（R13诡异流分析）：
- 《诡舍》：借鉴"规则怪谈+无限轮回的生死游戏张力"，避雷"密室逃生+规则怪谈"已有开局
- 《捞尸人》：借鉴"冷门行当的新鲜感+中式民俗恐怖"，避雷"捞尸行当+克苏鲁"已有设定
- 趋势借鉴："主动对抗规则"是已验证爽感方向（"被动求生"已审美疲劳）

### 1c-3. 产出校准清单

市场分析消化后，产出一张**借鉴点+避雷点清单**：

```markdown
## 市场校准清单（赛道：{用户方向}）

### 借鉴点（可借鉴的机制/爽感/节奏）
1. [机制/爽感1] —— 来源：《XXX》
2. [机制/爽感2] —— 来源：趋势
...

### 避雷点（必须避开的已有模式）
1. [已有设定/金手指/开局1] —— 来源：《XXX》
2. [已有设定/金手指/开局2] —— 来源：趋势
...
```

**这份清单给agent看，不进入发散prompt**。agent在发散后用这张清单做校验。

## 1d. 双轨发散：王道×猎奇

> **核心：不再只有一条纯自由发散管线。拆为双轨——A轨"王道赛道"扫描排行榜top书提取核心卖点→融合新元素产出5个方向，B轨"猎奇赛道"纯自由发散产出5个方向。共10个立项，用户从两条赛道里选。**
>
> 设计依据：纯自由发散优势是反差/画面/原创性，劣势是容易全部偏向猎奇（"殡葬直播间""中医诊断股票"），缺少王道选项。用户选不到靠谱的安全牌会全部推翻重来。排行榜借鉴弥补这个缺陷——用已验证的卖点做基底，加新元素做微调，产出"安全但有趣"的选项。

---

### 1d-A. 王道赛道：排行榜借鉴→融合新元素（产出5个）

> **目标**：5个"王道"方向——有排行榜验证的卖点做基底，但加了新元素不撞车。

#### 1d-A-1. 扫榜（WebSearch 2轮，找top书）

搜索目标赛道的番茄/起点排行榜top作品：

- 第1轮：`"番茄小说" + "{赛道}" + "排行榜/热销/2026"` → 找top 5
- 第2轮：`"{赛道}" + "起点" + "精品/万订/2026"` → 补起点top 3（如果番茄结果不够）

**用户完全开放时**：用1c市场调研时已经扫过总榜，直接复用那个方向。

#### 1d-A-2. 提取核心卖点（每本书1句话提炼为什么能卖）

对每本top书，提取**可迁移的核心卖点**（不是抄设定，是理解为什么读者买单）：

| top书 | 书名 | 核心卖点（1句话） | 可迁移到什么方向 |
|:--|:--|:--|:--|
| 1 | 《XXX》 | 冷门行当的新鲜感+中式民俗恐怖 | 其他冷门行当+任意类型 |
| 2 | 《XXX》 | 主动对抗规则的爽感+生死游戏张力 | 任意规则体系+主动对抗 |
| ... | ... | ... | ... |

**提炼原则**：
- 只提取"可迁移的卖点机制"，不提取"具体设定"（如《诡舍》的卖点是"规则怪谈+主动对抗"，不是"密室逃生场景"）
- 每本书最多提取1-2个卖点，只提取最核心的那个
- 卖点必须能用1句话讲清楚"读者为什么买单"

#### 1d-A-3. 融合发散（取卖点→加新元素→产出5个方向）

基于提取的5-8个核心卖点，用以下融合策略产出5个方向：

**融合策略**：
1. **卖点平移**：把A书的卖点机制平移到B书的场景/行当里（如"规则怪谈+主动对抗"平移到"外卖骑手+规则怪谈"）
2. **卖点叠加**：把两本书的不同卖点叠在一起（如"冷门行当新鲜感"+"主动对抗规则"→"殡葬师+规则怪谈+主动对抗"）
3. **反向操作**：把某本书的核心卖点翻过来用（如"被动求生"翻成"主动狩猎诡异"）
4. **设定替换**：保留卖点机制，换成全新设定（如"规则怪谈"改为"合同陷阱"→"公司法务给妖怪审合同"）
5. **情绪基调换挡**：同一个卖点机制换一种情绪基调（恐怖→爽文，悬疑→喜剧，黑暗→温馨）

**发散指令**（给子agent的全部输入）：

```
system: 你是一个网文创意编辑，擅长从热门作品提炼可迁移的卖点并微调出新方向。
user: 赛道：{用户方向}。已验证的核心卖点（可迁移）：{卖点列表}。避雷点（必须避开）：{避雷点列表}。请用融合策略（卖点平移/卖点叠加/反向操作/设定替换/情绪基调换挡）产出5个王道方向的开书创意。每个方向保留已验证卖点做基底，但加上新元素做差异化。
```

#### 1d-A-4. 王道方向质量门禁

每个王道方向必须满足：
- **有卖点基底**：能说清楚借了哪个top书的哪个已验证机制
- **有差异化**：加了至少1个新元素（新场景/新行当/新情绪基调），不是换皮
- **不踩避雷点**：核心设定不和已有热门撞车
- **有画面**：读完脑子里能浮现场景

不通过→重新融合发散。

---

### 1d-B. 猎奇赛道：纯自由发散（产出5个）

> **目标**：5个"猎奇"方向——不借鉴排行榜，纯自由发散，追求反差/画面/原创性。保持R9验证的最佳实践。

**发散指令**（给子agent/API的全部输入）：

```
system: 你是一个网文创意编辑。
user: 我想写{用户方向}，请帮我生成5个完全不同方向的开书创意种子。
```

**关键约束（给agent的，不给API的）**：
- 5个方向必须差异最大化——不同切入点、不同主角类型、不同情绪基调
- 不套框架，不加限制，不想机制——只管有没有意思
- **画面优先**：每句话读完脑子里立刻浮现场景。没有画面的创意直接淘汰

**好梗标准（agent自检，不通过重新发散）**：
- **有画面**：读完脑子里立刻浮现场景，不用想
- **有反差**：两个不该在一起的东西放在一起了
- **有意思**：读者看完会不会"卧槽这个我想看"？

**反面教材（禁止产出）**：
- "主角穿越到XX世界，获得XX系统，开始升级" — 纯事件
- "主角有XX能力，限制是XX，通过XX变强" — 纯机制
- "主角在XX势力中崛起，对抗XX敌人" — 纯势力操盘

### 1d-C. 双轨市场校准（用1c清单统合校验）

A轨和B轨各产出5个梗后，用1c的市场校准清单做**统合校验**：

| 轨道 | 梗 | 踩避雷点？ | 用到借鉴点？ | 校验结果 |
|:--|:--|:--|:--|:--|
| 王道 | 梗A1 | ✗ | ✓ | 放行 |
| ... | ... | ... | ... | ... |
| 猎奇 | 梗B1 | ✗ | ✗ | 放行 |
| ... | ... | ... | ... | ... |

**校验规则**：
- **踩避雷点=淘汰**（两轨统一规则）
- **用到借鉴点=加分**（两轨统一规则）
- 王道赛道额外校验：**有卖点基底+有差异化**，不通过淘汰不补（最多淘汰2个）
- 猎奇赛道额外校验：**有画面+有反差+有意思**，不通过重新发散补足5个
- 最终呈现：王道5个+猎奇5个=10个

### 1d-D. 双轨要点

| | 王道赛道（A） | 猎奇赛道（B） |
|:--|:--|:--|
| 产出数 | 5个 | 5个 |
| 发散方式 | 扫榜→提取卖点→融合新元素 | 纯自由发散（不注入任何数据） |
| 发散prompt注入 | 卖点列表+避雷点+融合策略 | 只给赛道方向 |
| 质量侧重 | 安全有趣（有卖点基底+差异化） | 原创有趣（画面+反差+有意思） |
| 用户场景 | "不想踩雷，想要靠谱的" | "要出圈的，赌一把" |

## 1e. 用户选择（10选1，双轨展示）

把校验通过的10个梗呈现给用户，让用户选1个。分两条赛道展示：

```
## 创意方向（共10个）

### 🔷 王道赛道（排行榜验证 + 微调，适合稳扎稳打）

#### 王道1：[名称]
一句话描述：[主角是谁，在做什么，遇到了什么]
画面：[脑补的场景，最好带一句台词]
为什么有意思：[吸引人的点]
市场定位：[借鉴了《XXX》的[卖点]机制，做了[差异化]]

###（王道2-5同上格式）

### 🔶 猎奇赛道（纯自由发散，适合博出圈）

#### 猎奇1：[名称]
一句话描述：[主角是谁，在做什么，遇到了什么]
画面：[脑补的场景，最好带一句台词]
为什么有意思：[吸引人的点]
市场定位：[原创度高，没撞车已有热门]
```

用户选1个，或要求某条赛道重新发散（如"王道都不满意，重新扫榜""猎奇太疯，补2个保守的"）。

**无用户交互时**（API直调/自动化测试）：默认从王道赛道选1个（验证过的卖点+差异化达标），继续Step 2。不选猎奇赛道默认——猎奇是高方差选项，无人值守不安全。
