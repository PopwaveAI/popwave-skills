# Step 1: 纯自由发散（Phase 1）

> **核心原则：不加任何约束。** R9测试验证——纯自由模式（11字符system+一句话user）产出的创意质量远高于注入搜索数据/格式模板/结构引导的模式。"中医把脉诊断股票""殡葬直播间全网等投胎"这些最佳梗都是纯自由模式下诞生的。

---

## 1a. 问需求

第一步问用户想写什么。根据回答分路径：

| 用户回答 | 路径 |
|---------|------|
| 直接给了梗/画面/一句话 | 跳过Phase 1，直接进Step 2打磨 |
| 给了方向/偏好（"想写都市异能""想写神医文"） | 进1b纯自由发散 |
| 完全开放（"你看着来"） | 进1c扫榜定方向再发散 |

**阻塞性步骤**：必须等用户回答后才能继续。禁止直接进入1b。

---

## 1b. 纯自由发散

拿到用户方向后，**不给API/agent任何额外约束**——不注入搜索数据，不给格式模板，不给结构引导，不提示"3选1"。只给一句话方向，让模型自由发挥。

**发散指令**（这是给子agent/API的全部输入）：

```
system: 你是一个网文创意编辑。
user: 我想写{用户方向}，请帮我生成{N}个完全不同方向的开书创意种子。
```

N = 5-6（不是3。R9证明模型在多产出时创意发散性更强，3个太少容易同质化）

**关键约束（给agent的，不给API的）**：
- 5-6个方向必须差异最大化——不同切入点、不同主角类型、不同情绪基调
- 不套框架，不加限制，不想机制——只管有没有意思
- 参考"给怪谈写员工手册""中医把脉诊断股票"这种好梗的特质：**一个具体行为×一个反差场景，一句话就能让人脑补画面**
- **画面优先**：每句话读完脑子里立刻浮现场景。没有画面的创意直接淘汰

### 好梗标准（agent自检，不通过重新发散）

每个发散的梗必须满足：
- **有画面**：读完脑子里立刻浮现场景，不用想
- **有反差**：两个不该在一起的东西放在一起了（古代×现代、严肃×日常、高贵×卑微、恐怖×生活化）
- **有意思**：读者看完会不会"卧槽这个我想看"？如果只是"逻辑成立但没意思"，淘汰

**反面教材（禁止产出的类型）**：
- "主角穿越到XX世界，获得XX系统，开始升级" — 纯事件，无画面无反差
- "主角有XX能力，限制是XX，通过XX变强" — 纯机制，无生活化场景
- "主角在XX势力中崛起，对抗XX敌人" — 纯势力操盘，无有趣行为

**正面教材（R13/R14级别的好梗）**：
- 推开凶宅大门，女鬼扑来，我在她脑门上看到属性面板，把"暴躁"改成"正在织毛衣"
- 被黄鼠狼拦路讨封，用红头文件格式回答，它必须照办
- 殡葬直播间，全网围观亡魂投胎
- 给怪谈写员工手册——禁止食用同事

---

## 1c. 扫榜定方向（用户完全开放时）

用户完全开放时，先扫番茄/起点热门榜单找到当前火的题材，再用1b纯自由发散。

搜索3轮：
- "番茄小说/起点中文网" + "热门/排行榜/2025" → 找当前什么在火
- "B站/抖音/知乎" + "小说推荐/出圈" + "2025" → 找正在出圈的概念
- "网文" + "新题材/新趋势" + "2025" → 找正在冒头的新方向

**扫榜结果只用于确定方向**，不注入发散prompt。确定方向后走1b。

---

## 1d. 用户选择

把5-6个方向呈现给用户，让用户选1个。格式：

```
## 创意方向

### 创意1：[名称]
一句话描述：[主角是谁，在做什么，遇到了什么]
画面：[脑补的场景，最好带一句台词]
为什么有意思：[吸引人的点]

### 创意2：...
...
### 创意6：...
```

用户选1个，或要求重新发散。

**无用户交互时**（API直调/自动化测试）：自动选最有画面感和反差感的1个，继续Step 2。
