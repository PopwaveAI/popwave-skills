# Step 2: 结构化打磨（Phase 2）

> 用户选定方向后，进入结构化打磨。这一阶段的作用不是改创意，而是**给选定的创意装上引擎、验证质量、写成简介**。

---

## 2a. 行为引擎检查

检查选定创意是否已有行为引擎——主角每章在做一件具体的事（谈判/把脉/写手册），还是在操作一个系统（抽卡/升级/兑换）。

- **已有行为引擎** → 跳过碰撞，直接进2b
- **没有行为引擎** → 用行为框架碰撞补上

行为框架来源——与赛道无关的日常领域，自带流程、甲乙方关系、可博弈规则：

| 素材池 | 示例行为 |
|-------|---------|
| 职场行为 | 签合同/写手册/定KPI/走流程 |
| 社会服务 | 接投诉/送快递/做客服 |
| 法律框架 | 维权/执法/合规审查 |
| 民间习俗 | 讨封问话/红白事流程 |
| 日常生活 | 做饭/装修/带娃/相亲 |

碰撞方法：把行为框架的流程步骤逐条映射到创意场景。例——"殡葬直播间"碰撞"客服"框架：接单→亡魂提需求→走流程完成→满意度评分→获得功德值。每条映射=一个章节。

## 2b. 合成金手指

```
金手指 = 梗（提供形式——怎么表现）
       × 机制（提供动力——为什么有效果）
       × 限制（提供张力——为什么不能随便用）
```

**合成示例库（showcase）**：

| 梗 | 机制 | 限制 | 合成金手指 |
|----|------|------|-----------|
| 中医把脉×金融分析 | 望闻问切诊断一切 | 主角不懂金融只懂中医 | 用中医把脉诊断股票走势，准确但不懂金融术语无法解释 |
| 扶不扶（老人摔倒社会梗） | 善行值兑换 | 只有扶了被讹的人才有值 | 扶被讹的老人获得善行值兑换能力，越被骂越强 |
| 打工人魂穿老板 | 魂穿控制 | 只能穿24小时且对方必须是压榨过自己的人 | 魂穿压榨过自己的老板24小时，只限仇人 |
| 全网黑转粉 | 黑粉值转化 | 必须真的被误解不能自导自演 | 被全网黑时获得能力加成，自导自演无效 |
| 黄鼠狼讨封 | 话术成真 | 只能用公文套话格式 | 被妖怪拦住问话时用红头文件格式回答，对方必须照办——但格式错了也照办 |
| 篡改诡异代码 | 属性编辑 | 改完不能立即生效，有24小时冷却 | 看到诡异属性面板，能改状态，但改完24小时后才生效，期间诡异会暴走 |

合成要求：
- **梗提供形式**：金手指的表现形态（怎么展现）
- **机制提供动力**：为什么金手指有效果
- **限制提供张力**：为什么不能随便用，制造选择压力
- **禁止只加限制不加代价**：限制必须让主角做选择，不能是单纯削弱

## 2c. 四眼法验证

对合成后的金手指做四眼法验证：

| 眼 | 检查什么 | 通过标准 |
|----|---------|---------|
| 画面眼 | 闭上眼能不能看到主角用金手指的具体画面？ | 能说出1个具体场景 |
| 限制眼 | 限制能不能制造选择压力？ | 主角用金手指时要权衡，不能无脑用 |
| 场景眼 | 这个金手指能跑10个不同的场景吗？ | 能列出10个章节场景 |
| 引擎眼 | 主角每章在做什么具体行为？ | 能说出行为引擎，不是纯升级 |

- 四项全过 → 放行，进2d
- 任何一项不过 → 打回，换机制或调整限制，重新合成

## 2e. 故事纲领（五维框架·500字）

> **故事纲领不决定世界细节（那是plot的活），它决定这本书的灵魂。plot拿到的不只是配置项，还有这个世界的呼吸。**

基于已打磨的梗/金手指/行为引擎/主角轮廓，写半页纸故事纲领。五个维度，每个≤3句话：

### A. 世界底色（1-2句）

> 前3章让读者呼吸到的空气——不是设定说明，是情绪质感。

写法：用一句话描述读者翻第一页时感受到的世界基调。不是"这是DND西幻世界"，是"这是一个没有人相信未来的世界——圣者浩劫三年后到，每个人都在拼命活过今天"。

| 类型 | 世界底色示例 |
|:--|:--|
| 恐怖悬疑 | 深夜的运河底有东西在走，老一辈讳莫如深但真实存在的民俗恐怖 |
| 末世生存 | 永夜，漆黑如深渊，火泉是唯一的光和热。饿和冷是空气的本味 |
| 都市异能 | 日常的裂缝——你身边任何人都有可能是披着人皮的另一个物种 |
| 系统流 | 规则的冷酷竞技场——输了死，没有协商余地 |

### B. 危机基调（1-2句）

> 悬在整本书头上的那一把剑——全书最大的悬念或威胁是什么？不是具体倒计时，而是读者在追读过程中持续感受到的压迫。

写法：一句话说清"这本小说最大的危机本质是什么"。不是"boss要毁灭世界"，而是具象的、与主角的核心矛盾相关的威胁。

| 类型 | 危机基调示例 |
|:--|:--|
| 金手指反噬 | 金手指本身也许站在敌人那边——系统的终极秘密比任何敌人都危险 |
| 世界倒计时 | 圣者浩劫三年后到，连神明都会陨落——一个凡人怎么扛 |
| 身份危机 | "我到底是什么"——主角的身份是一个他自己都不知道的谜，而且谜底可能比所有敌人都可怕 |
| 信任崩塌 | 五口之家里也许就有一只披着人皮的兽——最大的危险不在外面，在餐桌上 |

### C. 主角一生弧线（2-3句）

> 这个人的故事到底在讲什么——像小说腰封上那句最勾人的话。不是"从LV1到LV10"，而是他这一生要解决什么矛盾、学会什么、变成什么。

写法：三句话——起点（他最初是什么状态/困境）、拐点（什么事件迫使/激励他改变）、终点（他最终变成了什么样的人，或解决了什么问题）。

| 要素 | 示例（命运卡牌系统） |
|:--|:--|
| 起点 | 一个破产贵族末裔，在贫民区用前世记忆赌博养妹妹 |
| 拐点 | 发现命运卡牌系统不仅是对赌工具——它本身就是圣者浩劫的一部分 |
| 终点 | 他用三年攒够了活下去的资本，但真正学会的是怎么活——系统不是武器，是枷锁，而他要打破枷锁 |

### D. 最大钩子（≤20字）

> 和主角弧线不同。主角弧线是"这本书在讲什么"，最大钩子是"为什么路人会停下来"——一个路人在书店/App里翻三页能被什么概念暴力抓住。

写法：一句话，像书名旁边那行小字。不需要上下文就能看懂"这讲什么"。

| 类型 | 最大钩子示例 |
|:--|:--|
| 概念反转 | "穿越成一面镜子——不能说话、不能动、不能被人发现你是活的" |
| 装疯反杀 | "被关进精神病院的天才棋手，不是受害者而是猎手" |
| 内鬼视角 | "10人密室1个说谎者——主角就是那个必须骗过所有人的人" |
| 体制复仇 | "被自己效忠的国家枪决的英雄之子，重生回到一切开始前" |

### E. 即时兑现感（2-3句）

> 前5章如何让读者感受到"已经在路上了"——每一章结尾都在推动核心事件，不纯铺垫。

写法：描述前5章的节奏模式。不是列事件，而是描述"每章给了读者什么信号"。

| 模式 | 节奏描述示例 |
|:--|:--|
| 剥洋葱式 | 第1章赌场翻盘→第2章被堵门反杀→第3章公会接单→第4章下水道发现尸体→第5章第一次和制度权力博弈。每章一个新矛盾，层层加深 |
| 两线交织 | 现实线（求生/经营）和梦境线（剧场/观众）交替推进，每章两个世界的拼图各多一块 |
| 技能树开枝 | 每章激活金手指的一个新功能，同时引入一个新对手——"成长"和"威胁"同步升级 |

**硬约束**：
- 五个维度加起来≤500字——每个维度都是信息密度极高的"浓缩液"，不是展开分析
- 每个维度必须给出具体答案——不是"主角会成长"这种废话，是"他从独狼变成必须保护公会的人"
- 最大钩子必须能以≤20字一句话讲清——讲不清说明还没想明白

## 2f. 落盘创意.md（含故事纲领）

产出 `0-立项/创意.md`：

```markdown
# 创意名

## 一句话简介
（一句话说清这个创意是什么——主角是谁，在做什么，遇到了什么）

## 梗
（一句话梗——一个具体行为×一个反差场景）

## 梗来源
（用户选定的方向 + Phase 2打磨过程）

## 金手指
- 梗：（一句话）
- 机制：（一句话）
- 限制：（一句话）
- 合成：（梗×机制×限制的完整表述）

## 行为引擎
（主角每章在做什么具体行为）

## 轻量主角轮廓
- 名字：
- 身份：（一句话职业/社会角色）
- 核心动机：（一句话——为什么主角会卷入故事）
- 记忆点：（一句话——读者怎么记住这个主角）

## 番茄简介
（150-300字，第一人称散文体，带对话，带情绪弧线，带钩子。不是描述创意，是在讲创意的故事）
```

### 番茄简介写法参考（来自R13/R14的高质量产出）

> 我叫李想，是个普通人。
>
> 直到有一天，我推开一间贴满符咒的凶宅大门——那只披头散发的女鬼正朝我扑来，我却在她脑门儿上看到了几行字：
>
> 【对象：厉鬼·张翠花】【当前状态：暴躁】【是否编辑？是/否】
>
> 我鬼使神差点了"是"，打了一行字："当前状态：正在为你织毛衣，还有半小时完工。"
>
> 女鬼僵在原地，低头看了看自己手里的毛线球和两根棒针，愣住了。
>
> 三秒后，她面无表情地飘到角落，开始……织毛衣。

这段好在哪里：第一人称代入、对话展现人设、从日常到异常的转折自然、连环梗轰炸（女鬼织毛衣→画皮鬼面膜美容→古神变AI客服）、结尾留悬念。**不是描述创意，是在讲创意的故事。**

### 番茄简介硬约束

- **至少3个连环梗**：每一句都是一个梗，读者读完想笑想看
- **至少1句荒谬台词**：主角用金手指时的具体台词
- **1个认知反转收尾**：结尾抛出反转/悬念/威胁，让读者想看正文
- **禁止设定说明书式写法**：不准"主角是谁+金手指是什么+限制是什么+进化路径"这种概述
- **必须第一人称**：代入感
