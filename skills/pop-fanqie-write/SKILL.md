# Step 1 · 加载燃料（内化）

## 1a. 全文加载项目文件

**必须加载，禁止用摘要替代**（按优先级排序）：

1. `1-骨架/剧情白描.md` — **最关键输入**
2. `1-骨架/骨架.md` — 第一卷详规
3. `0-立项/创意.md` — 立项包
4. `current-state.md` — 当前状态
5. 前一章正文（`2-正文/chXXX.txt`）——回收章末钩子
6. 本章在骨架中的锚点（章号+章意图+配角+场景）

**输入优先级**：剧情白描 > 骨架 > 创意 > 设定精选

## 1b. 设定库精选注入（禁止全文注入）

**红线：设定库不全文注入，只精选与本章相关的设定**

| 本章特征 | 注入的设定维度 | 说明 |
|:--|:--|:--|
| 战斗章 | 力量体系（属性/技能/专长）+ 物种与天赋 | — |
| 探索章 | 世界蓝图（地理）+ 势力格局（组织） | — |
| 交易章 | 资源与物品（价格/物品） | — |
| 对话章 | 术语与文明底色（专有名词）+ 历史与驱动力（背景） | — |
| 成长章 | 力量体系（升级/职业）+ 物种与天赋（天赋觉醒） | — |

- 每个维度只提取与本章事件直接相关的段落（≤500字/维度）
- 最多注入2个维度（≤1000字总设定）
- 禁止注入"与本章无关的设定历史/势力详情/物品清单"

## 1c. 加载笔触DNA（标准插槽·三态协议·按需加载）

**DNA源文件**：`项目空间/涌现/文风锚定.md`（第一优先级）

### 三态协议

**启用态**（`涌现/文风锚定.md` 存在）：
- **按需加载，禁止全文注入**：
  1. 先读DNA文档的结构索引
  2. **必须全读**：通用维度（≥4个，全部读取）+ 全书稳定特征（全部读取）
  3. **定向读取**：根据本章场景类型，只读对应的场景卡（1-3张），不读无关场景卡
  4. **禁止使用limit参数截断读取**——每个区块必须完整读取到底
- 笔触层包括：叙事距离/句式节奏/物象/对话引导词/情绪外化/感官顺序/段落呼吸/系统面板格式（如有）

### 场景卡加载对照表

| 本章特征 | 加载的场景卡 |
|:--|:--|
| 日常/过渡/对话密集 | 对话·日常 + 描写·环境引入 + 叙述·过渡缓冲 |
| 战斗/Boss战 | 战斗·中坚boss战（+ 战斗·群战 如有） |
| 揭秘/真相/心理戏 | 叙述·揭秘心理戏 + 对话·信息交换 |
| 情感高潮/告别/牺牲 | 对话·情感高潮 + 叙述·孤独紧张 |
| 暗杀/偷袭/潜行 | 战斗·暗杀背刺 + 描写·环境引入 |
| 开场/新场景 | 描写·环境引入 + 描写·角色外貌 |
| 多场景混合 | 最多加载3张场景卡，取最核心的3种 |

**缺失态**（DNA不存在）：
- Step1 必须提示用户："未检测到笔触DNA，建议先用 pop-dna-style 提取并部署"
- 用户拒绝部署 → 进入trial模式

**trial模式**（用户拒绝部署）：
- write skill不内置任何默认风格
- 用户须显式声明基础风格倾向，或接受API自由发挥
- 交付面板必须标注"trial模式·无DNA驱动"

**消费规则**：只学笔触（句式节奏/感官顺序/叙事距离/段落呼吸/对话引导词习惯/系统面板格式），禁止从DNA原文摘录中提取剧情设计/战斗表现/角色思维

## 1d. 回收前章钩子

如非第一章：读取前章章末钩子 → 本章前300字必须回收 → 禁止跳过直接开新场景

|:----|:--------|:--------|:------------|

# Step 3 · 写正文（内化·三层指导）

> 笔触层从DNA取（启用态）或用户显式声明（trial模式）。write skill不内置任何笔触约束。

## 第1层·结构层

按章型7节拍骨架组织结构，字数分配：

| 段落 | 字数 | 内容 |
|------|------|------|
| 节拍1-2 | 300-400字 | 起笔+第一信息变化 |
| 节拍3-4 | 400-500字 | 压力/诱因+主角动作 |
| 节拍5 | 500-700字 | 阻碍升级/爽感爆发点 |
| 节拍6 | 400-500字 | 可见反馈 |
| 节拍7 | 200-300字 | 章末钩子 |
| **合计** | **2000-2500字** | |

## 第2层·节奏层

番茄12岁小学生读者模型：

| 指标 | 约束 |
|:----|:----|
| 事件间隔 | ≤20行 |
| 事件密度 | 8-12个/章 |
| 蓄力上限 | ≤2段 |
| 信息释放 | ≤3段 |
| 微钩子 | 每3-5行 |

## 第3层·爽感层

- **主推爽感必须触发**（章型绑定的主推爽感）
- **爽感闭环**：触发→爆发→后果
- **爆发后三段式**：爆发≤3行 → 反馈≤2段 → 余韵≥1段
- **小爽沿途**：每3-5事件1个小爽
- **即时奖励**：每章≥1次行动→可见回报

详见 `references/爽点引擎.md`。

## 写作禁区

禁止：写思考过程/检查表/分析框架 / 章末没有钩子 / 超过2500字

---

## 红线

1. **每章必须全文加载剧情白描.md+骨架.md+current-state.md**——禁止用current-state.md替代立项包
2. **设定库禁止全文注入**——只精选与本章相关的2个维度（≤1000字）
3. **第一章(ch001)由seed产出**——write从ch002开始
4. **篇幅硬限制**：2000-2500字，**不准超过2500字**。超2500字=废章，必须裁剪
5. **章型必选**：每章必须从4种章型中选1个，按7节拍骨架组织
6. **爽感≥1/章**：每章至少1个爽感触发，用爽点引擎设计
7. **章名**：每章正文开头必须有正常章节名
8. **笔触DNA标准插槽**（三态协议）：启用态从DNA取/trial模式用户声明
9. **章末有钩子**：每章必须以主动钩子或威胁逼近收尾
10. **不写思考过程/检查表**——直接写正文
11. **Step文件链式加载**——Step1+3内化在本文件→读取steps/step2.md→读取steps/step4.md→读取steps/step5.md

## 速查表

### 节奏约束

| 指标 | 约束 |
|:----|:----|
| 事件间隔 | ≤20行 |
| 事件密度 | 8-12个/章 |
| 蓄力上限 | ≤2段 |
| 信息释放 | ≤3段 |
| 微钩子 | 每3-5行 |

### 爽感速查

| 机制 | 要求 |
|:----|:----|
| 打脸循环 | 轻视≥2层，翻盘≤3行，对方有可见失态 |
| 小爽链 | 每3-5事件≥1个小爽，大爽前≥2个铺垫 |
| 爆发后三段式 | 爆发→反馈≤2段→余韵≥1段→下一事件 |
| 即时奖励 | 每章≥1次行动→可见回报闭环 |

### 篇幅控制

| 字数区间 | 判断 | 处理 |
|:--|:--|:--|
| 2000-2500字 | 合格 | 完成 |
| 1800-2000字 | 偏短 | 检查信息密度 |
| 2500-3000字 | 超标 | 必须裁剪描述性文字 |
| 超过3000字 | 废章 | 重写 |
