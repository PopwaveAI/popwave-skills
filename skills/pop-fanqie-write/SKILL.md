# pop-fanqie-write · 番茄正文渲染引擎

> 拿到梗直接写。写之前想4个问题，写的时候守6条笔触规则，写完检查篇幅。
> 不加载场景卡。不加载种子文档。不加载危机阶梯。不加载元数据。梗就是输入。

---

## SOP骨架

### Step 1: 章意图思考

写之前强制回答4问（不是走形式，防止"节拍都在但没灵魂"）：
- 本章读者体验什么情绪？（不是"爽"，是具体的：憋屈→释放？危机→逆转？）
- 本章的信息炸弹是什么？（读者读完这章获得了什么新信息？）
- 本章主角的哪句话/哪个动作会让读者记住？（不是"打脸"，是具体的那句话/那个画面）
- 本章跟前2章的情感差在哪？（不能是同一种情绪循环）

### Step 2: 正文渲染

**2a. 基本规则**
- 篇幅：2000-2500中文字，不准超过3000字
- 12岁读者模型全程约束（每30秒需新内容）
- 概念兑现：金手指在本章可见且推动剧情
- 爽感闭环：每章≥1个（触发→爆发→后果三段完整可见）
- 对话引导词只用"道"
- 每章必须有章末钩子

**2b. 去AI味6条笔触规则（核心——违反=不合格）**

| 规则 | 要求 | 典型AI味（禁止） |
|------|------|-----------------|
| 句式不重复 | 同一句式不得连续使用2次以上 | "他XX了。他XX了。他XX了。" |
| 情绪不直给 | 用行动/对话/环境展示情绪，不写"他感到绝望" | "越算越绝望""心里一沉""不由得" |
| 结尾不收束 | 章末停在动作或对话上，不写总结性旁白 | "他知道，这只是开始""一切才刚刚开始" |
| 描写不堆叠 | 感官描写≤3行后必须有动作推进 | 连续5行环境描写后才有角色动作 |
| 信息不重复 | 同一信息全章只出现1次 | 同一个设定/情绪/状态在一章内被提及3次 |
| 对话不死板 | 对话间穿插动作/环境，不写纯对话墙 | 连续8行对话没有任何动作描写 |

**2c. 章节过渡**
- 连续2章不得使用相同过渡方式
- 过渡方式：时间跳跃 / 场景切换 / 动作衔接 / 对话引入

### Step 3: 验收

| 维度 | 合格标准 |
|------|---------|
| 篇幅 | 2000-2500字，不准超过3000字 |
| 章意图4问 | 已回答 |
| 去AI味·句式 | 无同一句式连续2次以上 |
| 去AI味·情绪 | 无直给式情绪词 |
| 爽感闭环 | 触发→爆发→后果三段均可见 |
| 章末钩子 | 有追读动力 |
