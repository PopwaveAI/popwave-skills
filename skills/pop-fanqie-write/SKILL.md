# pop-fanqie-write · 番茄正文渲染

> 心法引导方向，笔触DNA驱动笔触，章型骨架驱动结构，状态驱动连续性。

---

## 做什么

输入：项目上下文（**剧情白描.md**+骨架.md+current-state.md+前章正文+**设定库精选**）+ 本章锚点 + 笔触DNA（可选）
输出：2000-2500字的章节正文，**不准超过2500字**，附交付面板

**定位**：write从ch002开始写正文（ch001已由seed的黄金首章产出）。write消费plot产出的剧情白描+章锚点表，逐章渲染正式正文。

文件架构：
- **SKILL.md（本文件）**：4种章型骨架 + SOP（含笔触DNA加载+微观技法选择） + 红线 + 速查表
- **references/章型定义.md**：4种章型的7节拍骨架 + 密度约束
- **references/爽点引擎.md**：5种爽感引擎（打脸/压力突破/未知揭示/小爽沿途/即时奖励）
- **references/technique.md**：写作技法参考（卡壳时翻阅）
- **dna/**：笔触DNA文件（decon拆书产出或手动创建）——叙事距离/句式/物象/对话/情绪外化/章型节拍绑定

## 4种章型骨架

番茄适配的4种章型（简化自起点6种，适配12岁小学生滑动阅读）：

| 章型 | 使用条件 | 读者期待 | 主动行为类型 |
|:----|:--------|:--------|:------------|
| opening_shift | 第一章/世界规则松动/金手指激活 | 先信现实→被打开更大世界 | 收获/质变级反馈 |
| confrontation_pressure | 对峙/打脸/信息差博弈/势力试探 | 看打脸反转 | 宏观层面解决问题 |
| combat_reversal | 战斗/绝境翻盘/首杀/暗杀 | 看绝境翻盘 | 战斗 |
| reveal_hook | 揭秘/新敌人/升级/章末钩子 | 看真相炸裂 | 获得反馈/质变级反馈 |

> 探索章和高位展示章合入confrontation_pressure或reveal_hook，番茄读者不需要太细的章型区分。

每种章型7节拍骨架详见 `references/章型定义.md`。

## 红线

1. **每章必须全文加载剧情白描.md+骨架.md+current-state.md**——禁止用current-state.md替代立项包
2. **设定库禁止全文注入**——只精选与本章相关的2个维度（≤1000字），设定库是"查字典"不是"读全书"
3. **第一章(ch001)由seed产出**——write从ch002开始，ch002必须回收ch001黄金首章的章末钩子
4. **篇幅硬限制**：2000-2500字，**不准超过2500字**。超2500字=废章，必须裁剪
5. **章型必选**：每章必须从4种章型中选1个，按7节拍骨架组织结构
6. **爽感≥1/章**：每章至少1个爽感触发，用爽点引擎设计
7. **微观技法选择**：Step 2必须选3-4类微观技法并落地（含v7.2.0新增动作链+多感官优先级，必选）
8. **动作链硬约束**：禁止用1个动词概括连续动作，必须拆成2-3个细动作。每章≥3处，战斗章≥5处
9. **多感官硬约束**：每场景至少覆盖3种感官，每章≥2处完整多感官描写，战斗章≥3处
10. **章名设计**：每章正文开头必须有10字以上设计章名（有噱头/反差/玩梗）
11. **笔触DNA绑定**（如启用）：叙事距离/句式/物象/对话/情绪外化必须从笔触DNA中来
12. **章末有钩子**：每章必须以主动钩子或威胁逼近收尾
13. **不写思考过程/检查表**——直接写正文

## 速查表

### 节奏约束速查

| 指标 | 约束 |
|:----|:----|
| 事件间隔 | ≤20行 |
| 感官连续 | ≤3行 |
| 情绪连续 | ≤2行 |
| 事件密度 | 8-12个/章 |
| 爽感触发 | ≥1个/章 |
| 蓄力上限 | ≤2段 |
| 信息释放 | ≤3段 |
| 微钩子 | 每3-5行 |

### 爽感速查

| 机制 | 要求 |
|:----|:----|
| 打脸循环 | 轻视≥2层，翻盘≤3行，对方有可见失态 |
| 小爽链 | 每3-5事件≥1个小爽，大爽前≥2个铺垫 |
| 爆发后三段式 | 爆发→反馈≤2段→余韵≥1段→下一事件 |
| 即时奖励 | 每章≥1次行动→可见回报闭环 |

### 微观技法速查（8类，v7.2.0新增动作链+多感官）

| 技法 | 一句话定义 | 何时选 |
|------|----------|--------|
| 信息差博弈 | 围绕"谁知道什么"设计博弈 | 对峙章/开篇章 |
| 感官锚点 | 用具体生理感官细节建立代入感 | 所有章型可选 |
| **动作链** ⭐ | 拆解连续动作成2-3个细动作，禁止概括式 | 战斗章/动作密集章必选 |
| **多感官优先级** ⭐ | 每场景至少覆盖3种感官 | 所有章型必选 |
| 预期违背 | 故意建立读者预期，然后打破它 | 揭秘章/开篇章 |
| 节奏微操 | 一章内快慢交替、张弛有度 | 战斗章/对峙章 |
| 人设微操 | 通过小动作/小习惯让角色立体 | 对峙章 |
| 后果链设计 | 章内事件不闭环，产生可见后果涟漪 | 战斗章/揭秘章 |

### 篇幅控制速查

| 字数区间 | 判断 | 处理 |
|---------|------|------|
| 2000-2500字 | 合格 | 完成 |
| 1800-2000字 | 偏短 | 检查信息密度 |
| 2500-3000字 | 超标 | 必须裁剪描述性文字 |
| 超过3000字 | 废章 | 重写，保留剧情推进和对话 |
