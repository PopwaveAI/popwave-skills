# pop-fanqie-write · 番茄正文渲染引擎

> 将故事卡渲染为番茄小说平台可消费的章节正文。故事是内容，元数据是约束——先服从故事，再用元数据校验。
> 输入是plot v3.1.0产出的故事卡（含故事草案+元数据+危机阶段+因果指向）。渲染时必须体现因果衔接和解决程度。

---

## SOP骨架

### Step 1: 加载上下文+章意图思考

**1a. 加载**
- 概念种子文档（概念定义+生成机制+节奏框架+笔触风格+节奏模型+爽感公式）
- 危机阶梯.md（全局结构，确认当前章节处于哪个危机阶段）
- 故事卡（当前章，含故事草案+元数据+因果指向）——故事草案是渲染的核心输入，元数据是约束
- 前章结尾（800字）
- 事实快照（当前状态，无快照则从已写章节末尾推断）

**1b. 章意图思考**

渲染前强制回答4问（不是走形式，防止"节拍都在但没灵魂"）：
- 本章读者体验什么情绪？（不是"爽"，是具体的：憋屈→释放？危机→逆转？）
- 本章的信息炸弹是什么？（读者读完这章获得了什么新信息？）
- 本章主角的哪句话/哪个动作会让读者记住？（不是"打脸"，是具体的那句话/那个画面）
- 本章跟前2章的情感差在哪？（不能是同一种情绪循环）

**1c. 章型+爽感读取**
- 从故事卡元数据读取章型，确认7节拍骨架
- 从故事卡元数据读取爽感类型
- 确认概念兑现点（在哪个节拍兑现概念）

### Step 2: 正文渲染+执行报告

**2a. 正文渲染**
- 篇幅：2000-2500中文字，不准超过3000字
- 12岁读者模型全程约束（每30秒需新内容）
- 概念兑现：本章概念变体必须在正文中可见且推动剧情
- 爽感闭环：每章≥1个爽感闭环（触发→爆发→后果三段必须完整可见）
- 节奏物理量（按种子节奏模型执行）：

| 节奏模型 | 事件密度 | 感官 | 情绪 | 替代指标 |
|---------|---------|------|------|---------|
| 爽文快节奏 | 8-12个 | ≤3行 | ≤2行 | — |
| 悬疑钩子节奏 | 4-6个 | ≤4行 | ≤3行 | 悬念密度≥3/章 |
| 情感蓄力节奏 | 5-7个 | ≤4行 | ≤4行 | 情感密度≥2/章 |
| 智力碾压节奏 | 3-5个 | ≤3行 | ≤2行 | 信息密度≥4/章 |

**2b. 笔触执行（去AI味核心）**

以下6条笔触规则强制执行，违反=不合格：

| 规则 | 要求 | 典型AI味（禁止） |
|------|------|-----------------|
| 句式不重复 | 同一句式不得连续使用2次以上 | "他XX了。他XX了。他XX了。" |
| 情绪不直给 | 用行动/对话/环境展示情绪，不写"他感到绝望" | "越算越绝望""心里一沉""不由得" |
| 结尾不收束 | 章末停在动作或对话上，不写总结性旁白 | "他知道，这只是开始""一切才刚刚开始" |
| 描写不堆叠 | 感官描写≤3行后必须有动作推进 | 连续5行环境描写后才有角色动作 |
| 信息不重复 | 同一信息全章只出现1次 | 同一个设定/情绪/状态在一章内被提及3次 |
| 对话不死板 | 对话间穿插动作/环境，不写纯对话墙 | 连续8行对话没有任何动作描写 |

**2c. 章节过渡**
- 连续2章不得使用相同过渡方式
- 过渡方式：时间跳跃 / 场景切换 / 动作衔接 / 对话引入
- 对话引导词只用"道"
- 首次杀人：3-5行心理波动，后续递减到一句冷评

**2d. 执行报告**
- 章意图思考4问的回答
- 验收检查表（14维度）：

| 维度 | 合格标准 |
|------|---------|
| 笔触 | 500字内看出风格/句式/对话气口 |
| 章型 | 7节拍完整 |
| 节奏 | 按节奏模型密度达标 / 爽感≥1 |
| 概念兑现 | 概念变体在正文中可见且推动剧情 |
| 爽感闭环 | 触发→爆发→后果三段均可见，缺后果=不合格 |
| 因果衔接 | 本章结局必须触发下章危机（按故事卡因果指向） |
| 解决程度 | 按故事卡标注：完全解决/部分解决（部分解决必须引出更大危机） |
| 主动行为 | 主角≥1决策+1行动 |
| 章末钩子 | 有追读动力 |
| 面板/金手指 | 按概念种子定义的交互方式 |
| 篇幅 | 2000-2500字，不准超过3000字 |
| 开篇规则 | A级(冲突+概念)或B级(冲突，概念延后≤1章) |
| 去AI味·句式 | 无同一句式连续2次以上 |
| 去AI味·情绪 | 无直给式情绪词（绝望/沉重/不由得/不禁） |

## 红线

1. 开篇规则：A级前300字必出冲突+概念；B级前300字必出冲突，概念可延后≤1章
2. 每章必须兑现故事卡定义的概念变体
3. 节奏密度按种子节奏模型执行，爽感闭环必须完整（触发→爆发→后果）
4. 连续2章不得使用相同章节过渡方式
5. 对话引导词只用"道"
6. 概念交互必须有"限制条件"展示——无限制=无张力
7. 每章必须有章末钩子——不允许无钩子章
8. **章意图思考4问不得跳过**——不回答直接写=不合格
9. **去AI味6条笔触规则全量执行**——任一违反=不合格
