# pop-fanqie-write · 番茄正文渲染引擎

> 将场景卡渲染为番茄小说平台可消费的章节正文。概念驱动写作——每章必须兑现概念承诺。
> 章型系统+微观技法复用自prose-render体系，输入从"施工卡"改为"场景卡"。

---

## SOP骨架

### Step 1: 加载上下文+章型选择

**1a. 加载**
- 概念种子文档（概念定义+生成机制+节奏框架+笔触风格+节奏模型+爽感公式）
- 场景卡（当前章）
- 前章结尾（800字）
- 事实快照（当前状态）

**1b. 章型选择**
- 从13种章型中选择主章型（见 references/章型定义.md）
- 确认7节拍骨架
- 确认概念兑现点（在哪个节拍兑现概念）

**1c. 爽感+技法选择**
- 从11种爽感公式中选择本章爽感类型（按种子流派选取）
- 选2-3类技法（通用/情境/流派）
- 填写选择卡：技法名 | 落位置 | 预期效果

### Step 2: 正文渲染+执行报告

**2a. 正文渲染**
- 篇幅：2000-2500中文字，不准超过3000字
- 12岁读者模型全程约束（每30秒需新内容）
- 概念兑现：本章概念变体必须在正文中可见且推动剧情
- 爽感公式：每章≥1个爽感触发，且有爆发感
- 节奏物理量（按种子节奏模型执行）：

| 节奏模型 | 事件密度 | 感官 | 情绪 | 替代指标 |
|---------|---------|------|------|---------|
| 爽文快节奏 | 8-12个 | ≤3行 | ≤2行 | — |
| 悬疑钩子节奏 | 4-6个 | ≤4行 | ≤3行 | 悬念密度≥3/章 |
| 情感蓄力节奏 | 5-7个 | ≤4行 | ≤4行 | 情感密度≥2/章 |
| 智力碾压节奏 | 3-5个 | ≤3行 | ≤2行 | 信息密度≥4/章 |

- 章节过渡：连续2章不得使用相同过渡方式
- 对话引导词只用"道"
- 首次杀人：3-5行心理波动，后续递减到一句冷评

**2b. 执行报告**
- 微观技法选择卡（技法名+落地标注）
- 验收检查表（10维度）：

| 维度 | 合格标准 |
|------|---------|
| 笔触 | 500字内看出风格/句式/对话气口 |
| 章型 | 7节拍完整 |
| 节奏 | 按节奏模型密度达标 / 爽感≥1 |
| 概念兑现 | 概念变体在正文中可见且推动剧情 |
| 爽感 | 爽感公式触发≥1次且有爆发感 |
| 主动行为 | 主角≥1决策+1行动 |
| 章末钩子 | 有追读动力 |
| 面板/金手指 | 按概念种子定义的交互方式 |
| 篇幅 | 2000-2500字，不准超过3000字 |
| 开篇规则 | A级(冲突+概念)或B级(冲突，概念延后≤1章) |

---

## 产出

`chNNN.md`（正文+执行报告）

---

## 红线

1. 开篇规则：A级前300字必出冲突+概念；B级前300字必出冲突，概念可延后≤1章
2. 每章必须兑现场景卡定义的概念变体
3. 节奏密度按种子节奏模型执行，爽感≥1个
4. 连续2章不得使用相同章节过渡方式
5. 对话引导词只用"道"
6. 概念交互必须有"限制条件"展示——无限制=无张力
7. 每章必须有章末钩子——不允许无钩子章
