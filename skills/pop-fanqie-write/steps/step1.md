# Step 1: 加载燃料

> 全文加载项目上下文 + **笔触DNA标准插槽（三态协议）**

## 1a. 全文加载项目文件

**必须加载，禁止用摘要替代**（按优先级排序）：

1. `1-骨架/剧情白描.md` — **最关键输入**，plot产出的整卷故事流（含信息差/伏笔/钩子标注）
2. `1-骨架/骨架.md` — 第一卷详规（幕序列+L2线+高潮点+悬念分层）
3. `0-立项/创意.md` — 立项包7产出（金手指+主角+世界观骨架）
4. `current-state.md` — 当前状态
5. 前一章正文（`2-正文/chXXX.txt`）——回收章末钩子
6. 本章在骨架中的锚点（章号+章意图+配角+场景）

**输入优先级**：剧情白描 > 骨架 > 创意 > 设定精选

## 1b. 设定库精选注入（禁止全文注入）

**红线：设定库不全文注入，只精选与本章相关的设定**

检测路径：`0-立项/参考书拆书/设定库/` 或 decon产出的Phase3 L1六件套

### 精选规则

根据本章白描卡/场景卡的章型+事件，只注入本章会用到的设定维度：

| 本章特征 | 注入的设定维度 |
|---------|--------------|
| 战斗章 | 力量体系（属性/技能/专长）+ 物种与天赋（种族特征） |
| 探索章 | 世界蓝图（地理）+ 势力格局（组织） |
| 交易章 | 资源与物品（价格/物品） |
| 对话章 | 术语与文明底色（专有名词）+ 历史与驱动力（背景） |
| 成长章 | 力量体系（升级/职业）+ 物种与天赋（天赋觉醒） |

### 注入方式

- 每个维度只提取与本章事件直接相关的段落（≤500字/维度）
- 最多注入2个维度（≤1000字总设定）
- 禁止注入"与本章无关的设定历史/势力详情/物品清单"

**原则：设定库是write层的"查字典"，不是"读全书"。write需要什么查什么，不查的就不注入。**

**如设定库不存在**：跳过本步，依赖剧情白描（剧情白描应该已经消化了设定库的关键信息）。

## 1c. 加载笔触DNA（标准插槽·三态协议·按需加载）

**DNA源文件**：`项目空间/涌现/文风锚定.md`（第一优先级，10万字裁剪时也不裁剪）

> 该文件由 pop-dna-style 产出并部署。**DNA保持全量文档不动，write按本章场景类型定向消费。不精简DNA，只精准加载。**

### 三态协议

**启用态**（`涌现/文风锚定.md` 存在）：
- **按需加载，禁止全文注入**：
  1. 先读DNA文档的结构索引（通用维度列表 + 场景卡矩阵列表）
  2. **必须全读**：通用维度（≥4个，全部读取）+ 全书稳定特征（全部读取）
  3. **定向读取**：根据本章场景类型，只读对应的场景卡（1-3张），不读无关场景卡
  4. **禁止使用limit参数截断读取**——每个区块必须完整读取到底
- Step3 写正文时，笔触层从DNA取，不得绕过
- 笔触层包括：叙事距离/句式节奏/物象/对话引导词/情绪外化/感官顺序/段落呼吸/系统面板格式（如有）

### 场景卡加载对照表

根据本章章型+事件特征，定向加载对应场景卡：

| 本章特征 | 加载的场景卡 | 说明 |
|:--|:--|:--|
| 日常/过渡/对话密集 | 对话·日常 + 描写·环境引入 + 叙述·过渡缓冲 | 首章偏重这个组合 |
| 战斗/Boss战 | 战斗·中坚boss战（+ 战斗·群战 如有） | 战斗章核心卡 |
| 揭秘/真相/心理戏 | 叙述·揭秘心理戏 + 对话·信息交换 | 信息密度高时用 |
| 情感高潮/告别/牺牲 | 对话·情感高潮 + 叙述·孤独紧张 | 情绪张力最大时用 |
| 暗杀/偷袭/潜行 | 战斗·暗杀背刺 + 描写·环境引入 | 感官详细度要求高 |
| 开场/新场景 | 描写·环境引入 + 描写·角色外貌 | 首章或新地图 |
| 多场景混合（如首章含日常+揭秘+对话） | 最多加载3张场景卡 | 取前3个最相关的 |

> 场景卡加载上限：每章最多3张。如本章涉及超过3种场景类型，只取最核心的3种。

### 加载流程

1. 读DNA的结构索引（判断有哪些通用维度、哪些场景卡）
2. 全量读入所有通用维度 + 全书稳定特征
3. 根据本章章型+场景，从对照表中确定要加载的场景卡
4. 定向读入对应场景卡（每张读到底，禁止limit截断）
5. 本章用不到的场景卡**跳过不读**

**缺失态**（`涌现/文风锚定.md` 不存在）：
- Step1 必须提示用户："未检测到笔触DNA，建议先用 pop-dna-style 提取并部署"
- 用户拒绝部署 → 进入trial模式
- 用户同意部署 → 暂停write，等用户跑完 pop-dna-style 再回来

**trial模式**（用户拒绝部署）：
- **write skill不内置任何默认风格**（v8.0.0移除）
- 用户须显式声明基础风格倾向，或接受API自由发挥的质量波动
- 交付面板必须标注"trial模式·无DNA驱动"

### 消费规则（强制区分笔触与内容）

- **只学笔触**：句式节奏、感官顺序、叙事距离、段落呼吸、对话引导词习惯、系统面板格式
- **禁止学内容**：不得从DNA原文摘录中提取剧情设计、战斗表现、角色思维等内容元素
- **禁止limit截断**：每个读入的DNA区块必须完整读完，不得设limit参数中途截断
