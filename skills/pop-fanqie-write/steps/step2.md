# Step 2: 章型+技法锁定

> 选章型 + 章意图思考 + 微观技法选择（含动作链+多感官优先级）

---

## 2a. 选章型

从4种章型中选1个（详见 `references/章型定义.md`）：

| 章型 | 使用条件 |
|:----|:--------|
| opening_shift | 第一章/世界规则松动/金手指激活 |
| confrontation_pressure | 对峙/打脸/信息差博弈/势力试探 |
| combat_reversal | 战斗/绝境翻盘/首杀/暗杀 |
| reveal_hook | 揭秘/新敌人/升级/章末钩子 |

**选择依据**：骨架中本章的章意图+本章核心事件类型。

**锁定密度约束**：选定章型后，自动锁定该章型的战斗密度/系统交互密度/冲突密度。

---

## 2b. 章意图思考

回答3个问题（想清楚再写，不写在正文里）：

1. **本章核心事件是什么**？（1句话）
2. **本章爽感爆发点在哪**？（对应章型7节拍的哪个节拍）
3. **本章章末钩子是什么**？（主动钩子或威胁逼近）

## 2c. 微观技法选择

从8类技法中选3-4类（**v7.2.0新增动作链+多感官优先级**）：

| 技法 | 一句话定义 | 何时选 |
|------|----------|--------|
| 信息差博弈 | 围绕"谁知道什么"设计博弈 | 对峙章/开篇章 |
| 感官锚点 | 用具体生理感官细节建立代入感 | 所有章型可选 |
| **动作链** ⭐NEW | 拆解连续动作成2-3个细动作，禁止概括式描写 | 战斗章/动作密集章必选 |
| **多感官优先级** ⭐NEW | 每场景至少覆盖3种感官（视觉+听觉+触觉/嗅觉/味觉任选） | 所有章型必选1次 |
| 预期违背 | 故意建立读者预期，然后打破它 | 揭秘章/开篇章 |
| 节奏微操 | 一章内快慢交替、张弛有度 | 战斗章/对峙章 |
| 人设微操 | 通过小动作/小习惯让角色立体 | 对峙章 |
| 后果链设计 | 章内事件不闭环，产生可见后果涟漪 | 战斗章/揭秘章 |

**选择依据**：章型推荐组合 + 本章事件特征。

**落地要求**：选定技法必须在正文中可见。不能"选了感官锚点但全章没有感官细节"。

### 动作链落地规范（v7.2.0核心迭代）

**核心原则**：禁止用1个动词概括连续动作，必须拆成2-3个细动作构成链条。

**对比示例**：

| 概括式（禁止） | 动作链（要求） |
|--------------|--------------|
| 他杀死了光头。 | 他踏前一步，左手扣住光头的下颌，右手将匕首送进喉咙，刀尖从颈后穿出。 |
| 他甩了甩匕首上的血。 | 他甩动手腕，血珠顺着刀刃滑落，滴在泥地里，发出轻微的嗒嗒声。 |
| 她喂哥哥喝粥。 | 她舀起一勺粥，吹了几口气，用另一只手掰开他的嘴，小心翼翼把粥灌进去。 |
| 他站起来，走到门口。 | 他撑着床板坐起来，脚踩在地上试了试力气，扶着墙一步步挪到门口。 |

**适用场景**：
- 战斗动作（刺/砍/挡/闪）必须动作链
- 日常动作（吃/走/拿/放）关键节点必须动作链
- 情绪动作（哭/笑/怒/怕）必须动作链

**数量约束**：每章至少3处动作链，战斗章至少5处。

### 多感官优先级落地规范（v7.2.0核心迭代）

**核心原则**：每个场景至少覆盖3种感官，不堆叠（每种感官≤3行）。

**感官优先级表**：

| 场景类型 | 主感官 | 次感官 | 辅感官 |
|---------|-------|-------|-------|
| 室内日常 | 视觉 | 听觉 | 嗅觉 |
| 战斗场景 | 触觉 | 视觉 | 听觉 |
| 户外场景 | 视觉 | 听觉 | 嗅觉/触觉 |
| 情绪场景 | 触觉 | 听觉 | 视觉 |
| 神秘场景 | 听觉 | 嗅觉 | 视觉 |

**对比示例**：

| 单感官（不足） | 多感官（要求） |
|--------------|--------------|
| 雨下了一整夜。 | 雨砸在破木板上，噼啪响（听觉）。屋顶的漏洞滴着水，在地上积成小洼（视觉）。屋里弥漫着霉味和湿气（嗅觉）。 |
| 他杀死了光头。 | 匕首刺穿喉咙的瞬间，他感觉到刀刃卡在软骨里的阻力（触觉）。血溅在手上，温热黏腻（触觉）。光头的喉咙发出咯咯声（听觉）。 |

**数量约束**：每章至少2处完整多感官描写（3种感官以上），战斗章至少3处。

## 2d. 爽感引擎选择

从5种爽感引擎中选1-2个（详见 `references/爽点引擎.md`）：

| 章型 | 主推爽感 | 次推爽感 |
|------|---------|---------|
| opening_shift | 压力突破 | 未知揭示 |
| confrontation_pressure | 打脸循环 | 即时奖励 |
| combat_reversal | 压力突破 | 小爽沿途 |
| reveal_hook | 未知揭示 | 小爽沿途 |

**主推必触发**，次推可选。

**爽感必须闭环**：触发→爆发→后果，缺一不可。

## 2e. 章名设计（v7.2.0新增）

每章正文开头必须有章名，章名要求：
- **10字以上的一句话句子**
- **有噱头/反差/玩梗**
- 能展现本章的爽点或核心事件

**对比示例**：

| 平淡章名（禁止） | 设计章名（要求） |
|---------------|---------------|
| 第五章 一个清晨 | 当我发现妹妹背上的伤痕时，整条街都要付出代价 |
| 第二章 墓园 | 老狗的血，还有她最后一把米 |
| 第一章 穿越 | 雨夜中，她将最后的米粥喂进哥哥嘴里 |

**章名设计原则**：
- 用具体物象（米粥/血/伤痕）而非抽象概念
- 有情绪张力或反差
- 可以玩梗但不要生硬
- 章名放在正文最开头，用 # 标题格式

## 2f. 笔触DNA绑定（如启用）

如Step 1b加载了笔触DNA，本章必须绑定：
- 叙事距离从DNA中来
- 句式节奏从DNA中来
- 物象从DNA中来
- 对话风格从DNA中来
- 情绪外化方式从DNA中来

如未加载笔触DNA，用通用笔触约束：
- 叙事距离：近为主，关键场景切近
- 句式：短句为主，长句用于铺垫
- 物象：每章1-2个反复出现的物象
- 对话：对话引导词只用"道"
- 情绪外化：通过动作和物象，禁止直接写"他感到..."
