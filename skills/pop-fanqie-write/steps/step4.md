# Step 4: 篇幅硬限制检查+落盘

> 字数硬限制：2000-2500字，不准超过2500字

---

## 4a. 字数检查

**统计正文字数**（不含交付面板，不含markdown标记）：

| 字数区间 | 判断 | 处理 |
|---------|------|------|
| 2000-2500字 | ✅ 合格 | 进入4b落盘 |
| 1800-2000字 | ⚠️ 偏短 | 检查信息密度，补充爽感反馈或微观技法落地 |
| 2500-3000字 | ❌ 超标 | **必须裁剪**：删除描述性文字、压缩铺垫段、保留剧情推进和对话 |
| 超过3000字 | ❌❌ 废章 | **重写**：保留核心剧情推进和对话，删除所有非必要描写 |

**裁剪优先级**（从先删到后删）：
1. 环境描写（不是物象的环境描写）
2. 人物外貌描写
3. 内心独白
4. 铺垫段
5. 对话（最后删，对话是番茄读者最爱看的）

**禁止裁剪**：
- 剧情推进点
- 爽感爆发段
- 章末钩子

> 注：系统面板格式由DNA决定（如有）。trial模式不内置面板格式。

**自动化字数自检**：模型自报字数不可信（R29测试10章全部虚报，平均虚报1637字）。**必须调用字数自检脚本验证**：

```bash
python skills/pop-fanqie-write/scripts/wordcount_check.py <章节文件.md> --auto
```

脚本会：
1. 统计实际中文字符数（不含标点/空格/markdown标记）
2. 对比自报字数，标记虚报
3. 如超标(>2500字)，调用API裁剪到2000-2500字
4. 如一次裁剪未达标，自动二次裁剪
5. 更新交付面板的字数字段

**红线：落盘前必须执行字数自检脚本，禁止直接用模型自报字数落盘。**

---

## 4b. 章型节拍检查

**快速自检**：本章是否按7节拍骨架组织？

- 节拍1-7是否都有？
- 节拍5（爽感爆发点）是否到位？
- 节拍7（章末钩子）是否有主动钩子？

如缺失节拍，补齐。

---

## 4c. 落盘

将正文写入 `正文/chXXX.txt`。

**落盘格式**：
- 纯正文，不含思考过程/检查表
- 章标题在第一行
- 系统面板格式由DNA决定（如有）
- 不含markdown标记（除了正文需要的）
