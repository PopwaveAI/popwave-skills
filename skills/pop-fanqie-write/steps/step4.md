# Step 4: 交付 + 人机共创

写完并检查字数后，在章节正文末尾附加"人机共创面板"。

---

## 交付面板格式

在正文写完并存盘后，向用户呈现以下面板：

```
## 📋 本章交付面板

### 当前伏笔清单
| 编号 | 伏笔 | 埋设章 | 状态 |
|------|------|--------|------|
| 1 | xxx | ch01 | 未揭示/已揭示/推进中 |
| 2 | xxx | ch02 | 未揭示 |
| ... | ... | ... | ... |

### 下一章方向（请选择或补充）

**方向A：** xxx
（基于xxx线索推进，预期爽点：xxx）

**方向B：** xxx
（引入xxx新冲突，预期爽点：xxx）

**方向C：** xxx
（回收xxx伏笔，预期爽点：xxx）

---
请选择一个方向（A/B/C），或提出你自己的想法。
```

---

## 设计原则

1. **伏笔清单**：列出所有当前悬空伏笔（已揭示的不列），标注状态。超过10条的只列最近5条+总数提示
2. **方向设计**：3个方向必须覆盖不同类型——推进现有线索 / 引入新冲突 / 回收旧伏笔
3. **预期爽点**：每个方向标注预期爽点类型，帮用户判断
4. **用户选择**：必须等用户选择后才进入下一章写作。用户也可自定义方向
