# Step 5: 交付面板

> 输出交付面板，供用户/review检查

---

## 交付面板格式

写完正文后，在正文末尾附上交付面板：

```
---

## 交付面板

| 字段 | 值 |
|------|-----|
| 章号 | chXXX |
| 章名 | （10字以上的设计章名） |
| 章型 | opening_shift / confrontation_pressure / combat_reversal / reveal_hook |
| 字数 | XXXX字 |
| 核心事件 | （1句话） |
| 爽感引擎 | （主推+次推） |
| 微观技法 | （选了哪3-4类，含动作链/多感官标注） |
| 动作链数量 | X处（战斗章≥5，日常章≥3） |
| 多感官场景数量 | X处（每章≥2，战斗章≥3） |
| 章末钩子 | （1句话） |
| 推进L2线 | （推进了哪条剧情线） |
| 笔触DNA | 启用/未启用 |
| 前章钩子回收 | 是/否（第一章为N/A） |
```

---

## 人机共创

如用户在场，可询问：
- 这章的爽感爆发点是否到位？
- 章末钩子是否够吸引？
- 是否需要调整章型？
- 动作链是否够细？有没有概括式动作？
- 多感官是否到位？有没有场景只覆盖1种感官？

如用户不在场（自动模式），跳过本步。

---

## 完成

本章交付完成。进入下一章或等待用户指示。
