# 对话博弈场景模板

> 适用于：以对话为主推进情节的场景（谈判/对峙/信息交换/关系推进）
> 核心原则：**每句对话必须推进至少一个维度（信息/关系/地位）**

---

## 段落结构

```
┌─ 段1 · 情境（1段，3-5句） ───────────────────────┐
│  功能：定位——谁在场、什么态势、对话的目的             │
│  写法：动作描述带出情境，不单独写"背景介绍"            │
│  示例：他用指节敲了三下桌面。"合同第七条。"            │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段2 · 对话核心（3-8轮交替） ──────────────────────┐
│  每轮结构：                                          │
│    ① 一方的发言（1-2句）                            │
│    ② 对方的反应（动作/表情/沉默）+ 回应             │
│  每轮功能标签（选1）：                               │
│    ▶ 信息交换 —— 有新信息放出                      │
│    ▶ 地位博弈 —— 谁压谁一头                        │
│    ▶ 关系推进 —— 信任/对立/暧昧的变化               │
│    ▶ 潜台词战 —— 表面的意思和实际的意思不一致        │
│  节奏：信息交换轮快（1-2句），地位博弈轮可拉长        │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段3 · 收尾（1段） ───────────────────────────────┐
│  功能：本轮对话的落点，不总结                       │
│  内容：一个画面/一个动作/一句话，暗示对话结果         │
│  示例："他端起茶杯，已经凉了。"                      │
│  注意：不写"他们达成了共识"——用画面说话               │
└────────────────────────────────────────────────────┘
```

---

## 关键约束

- **不写无信息对话**——"你还好吗？""还行。"这种不推进任何维度的对话不出现
- **不写提问式信息揭示**——"你知道XX吗？"不如"XX已经在路上了"
- **对话中的人不重复说同一件事**——第一句没说清楚就换角度，不说第二遍
- **动作标签优先于对话标签**——"他说"少用，用动作带出谁在说话

---

## 节奏标记

```
情境：慢速 → 对话：快（信息轮）/ 中（博弈轮） → 收尾：慢
总轮数：3-8轮（超过8轮需要作为独立场景处理）
```
