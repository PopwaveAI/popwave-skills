# 悬疑揭示场景模板

> 适用于：信息逐步释放、读者认知不断刷新、伏笔回收
> 核心原则：**读者知道得比角色少，但比角色想知道的多**

---

## 段落结构

```
┌─ 段1 · 入口（1段） ───────────────────────────────┐
│  功能：触发揭示的事件/线索出现                        │
│  内容：一个异常点、一个新发现、一条新线索             │
│  写法：客观描写事件本身，不附带"这很奇怪"的旁白       │
│  示例：不是"他发现了异常之处"——                       │
│        是"地砖上有一道裂缝，裂缝里渗着黑色液体。"      │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段2 · 探索（1-3段，逐层加深） ────────────────────┐
│  功能：读者跟随角色一步步接近真相                     │
│  每段结构：                                          │
│    ① 角色动作（查看/询问/尝试）                     │
│    ② 获得的新信息（比上一段更深一层）               │
│    ③ 读者的认知更新（不是角色的认知更新）           │
│  层数控制：                                           │
│    Layer 1: 表层信息（"哦？"）                       │
│    Layer 2: 矛盾信息（"等等，不对。"）               │
│    Layer 3: 真相（"原来如此！"）                     │
│  注意：不到 Layer 3 不揭示全貌                       │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段3 · 反转/刷新（1段，核心） ────────────────────┐
│  功能：让读者对之前的信息产生完全不同的理解            │
│  内容：一个信息让前面所有线索产生新意义               │
│  写法：不解释——让信息本身产生冲击                     │
│  示例：不是"原来凶手是XX"——                           │
│        是"法医摘下手套说：'死者的DNA和你的一样。'"     │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段4 · 余波（1段） ───────────────────────────────┐
│  功能：给读者消化反转的时间，同时带出新悬念           │
│  内容：角色对真相的反应（不讨论/不分析） + 新问题     │
│  写法：一个画面/一个动作就够了                       │
└────────────────────────────────────────────────────┘
```

---

## 关键约束

- **信息分层不能跳层**——Layer 1 没建立就不能到 Layer 3
- **不早泄**——不到反转段不释放核信息
- **不叙述者剧透**——"他不知道…"是作弊，用事件本身限制读者视角
- **Layer 3 不能是"原来都是一场梦"**——任何类型的揭示都必须对后续情节产生实质影响

---

## 节奏标记

```
入口：中速 → Layer 1：慢 → Layer 2：慢+堆积 → Layer 3/反转：极快（信息密度峰值）
总字数：500-1000字
```
