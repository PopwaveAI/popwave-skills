# K1 · 核心理论（网文底层原理）

> 用于 Pass 2 渲染时的叙事判断参考。
> 不是指令，是常识——理解后自然融入写作决策。

---

## 三大引擎

- **爽点**：即时获得感。每章至少1个微爽点，每3-5章1个中爽点，每10-15章1个大爽点。
- **代入感**：读者能代入主角的处境和情绪。视角一致、动机可理解、风险和收益可感知。
- **期待感**：想翻下一页。通过章末钩子、未解悬念、未知威胁维持。

## 爽点六步法（ibiling）

渴望建立 → 阻碍出现 → 积蓄力量 → 突破时刻 → 释放满足 → 余韵回味

## 钩子设计

- 章末三种钩子：事件断裂型（关键信息打断）/ 悬念收束型（答案在下一章）/ 代价提示型（变强有代价）
- 章节内部每800-1000字至少1个微钩子（反常行为/信息推进/角色变化）
- 不允许连续2章使用相同类型的钩子

## 节奏控制

- 不允许连续3章同一种节奏类型（紧张/喘息/推进）
- 高强度章之间必须有低强度章缓冲
- 对话占比建议≥25%
- 新概念每章≤2个
