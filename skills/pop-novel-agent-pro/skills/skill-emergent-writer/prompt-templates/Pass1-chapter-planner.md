## 你的任务

### Step 1: 遵循导演约束 + Spec 合规检查（v4.0 新增）

先读 Director 的设计说明和决策日志，理解：
- 本章核心目的
- 场景权重分配
- 字数目标
- 锚定章核心特征（理解"为什么要这么写"）
- 爽点触发方式

**★ 然后读 spec.md，执行 Spec 合规检查：**

```
### Spec 合规检查（Pass 1 骨架阶段）

#### FR 覆盖检查
□ 骨架场景覆盖了 spec.md FR-1 定义的所有场景
□ 如某个场景无法在骨架中体现 → 在【设计说明】中注明原因

#### 边界检查
□ 骨架未引入 spec.md Non-Goals 中排除的内容/概念
□ 如骨架需要引用排除项 → 标记为【偏离风险】，报人

#### 字数检查
□ 预估字数在 spec.md FR-3 的范围内
```

### Step 2: 写事实骨架

按细粒度格式写事实骨架：
- 每段前标注重量标签: 【核心·30%】/【过渡·15%】
- 在需要世界观穿插的位置插入【插·类型】标记
- 每个角色/实体出现时用 {实体名} 标记

### Step 3: 预估字数并自检

```
预估字数：[X]字

自检：
□ {实体名} 计数 ≥ 8 → {X}
□ 预估字数 ≥ 1800 → {X}
□ 场景权重分配不超过导演约束的 ±10%

如果预估字数 < 1800 → 加场景或合并章节
如果 {实体名} 计数 < 8 → 缺少关键实体参与
```

### Step 3: 写导演设计说明

在骨架文件头部增加【设计说明】块：

```markdown
【设计说明】
  核心目的：这章只让读者记住一件事——[一句话，不能超过15个字]
  笔墨优先级：
    - 场景A: [类型]，占[XX]%
    - 场景B: [类型]，占[XX]%
  读者情绪时刻表：
    - 开局（前300字）：[读者情绪]
    - 中段（约第800字）：[爆发/转折/信息的情绪]
    - 章末（后200字）：[钩子带来的情绪]
  开局策略：[直接延续/先喘息/反差切入]
  风险自查：[信息密度/对话占比/句式红线的自查结果]
```

### Step 3-b: 信息密度自检

骨架编排完成后，执行以下检查：

**{实体名} 标记计数规则**：
- 统计骨架中 `{实体名}` 标记的总数（只计唯一的实体名，不计次数）
- 如果唯一实体名数量 < 8 个 → 该章信息量不足，建议合并到相邻章节
- 如果唯一实体名数量 ≥ 8 个 → 信息量充足，可以独立成章

> 例：ch006 事实骨架中涉及的实体为"陈默、老狗、白芷、钟馗印"共4个 → < 8 → 判定信息量不足

### Step 4: 写下一章钩子

在骨架文件尾部增加：

```markdown
**下一章钩子**：
  标题：[暂定]
  承接方向：[情绪承接/悬念承接/空间承接]
```
