## 设计原则

### market-test 不是什么

- ❌ **不是宪法对齐检查**（那是写作 Agent 自查的事）
- ❌ **不是拆书技能**（拆书是"研究别人的书写法"，market-test 是"验证我们自己的书面市质量"）
- ❌ **不是编辑评分**（不模拟编辑签约标准）

### market-test 是什么

当一本新书还没有真实读者点评时，market-test 模拟一个具备以下三层知识的读者来读稿：

1. **品类常识** — 网文写作的通用底线（黄金三章、钩子设计、对话占比…）
2. **赛道对标** — 同类爆款怎么做、踩过什么坑（来自开书阶段的参考书分析）
3. **自设承诺** — 这本书自己在前几章建立了什么期待，后续有没有兑现

每一层独立运作，不互相污染。最终结论来自三层交叉验证。

## 审稿流程

```
用户请求："审第X章" / "审前X章"

  【节点D · 启动思考】（v1.2新增）
    ① 对齐检查：本次要审的材料属于什么阶段？（开篇/中期/幕末）
    ② 核心判断：这轮验证的目标是什么？
       - 检查品类常识底线？（重点Layer 1）
       - 检查赛道对标？（重点Layer 2）
       - 检查自设承诺兑现？（重点Layer 3）
    ③ 风险扫描：是否有上次审稿反馈的系统性问题需要复检？
    ④ 产出：验证路径说明（散文式）
    ⑤ 如有需要确认的问题 → 「待确认/疑问」输出给作者
  │
  │  ↓ 确认路径
  │
  1. 加载品类常识基线
     → 从学习资料提取通用底线检查清单
  
  2. 如果存在 _参考书分析/ 目录
     → 加载参考书分析，生成赛道对标检查项
  
  3. 如果存在前几章正文
     → 提取自设承诺：这本书承诺了读者什么
  
  4. 执行审稿（走三层，子Agent）
     → Layer 1: 品类常识
     → Layer 2: 赛道对标
     → Layer 3: 自设承诺
  
  5. 输出市场验证报告
  
  6. （可选）输出向上游反馈
```

## Layer 1：品类常识（通用底线）

**来源**：`novel-agent-pro/学习资料/` 中的网文基础知识（黄金三章理论、网文底层原理、钩子设计指南、爽点与节奏系统、编辑签约标准等）

**检查项**（随平台不同而调整）：

```
番茄品类常识基线：
  ✅ 前300字有冲突/危机/反转
  ✅ 主角在前300字内出场
  ✅ 每章至少1个章内微爽点
  ✅ 对话占比建议 25-40%
  ✅ 章末有信息差或半路阻断型钩子
  ✅ 连续无爽点章节 ≤ 2 章
  ⚠️ 主角不能连续3章被动
  ⚠️ 前500字专有名词 ≤ 3 个（防止堆设定）
  ❌ 章末是内心独白收尾（伪钩子）

起点品类常识基线：
  ✅ 前1000字建立核心悬念或冲突
  ✅ 主角动机"普通"（活下去/保护家人/证明自己）
  ✅ 世界观信息通过行动而非说明释放
  ✅ 每卷至少1个情绪高潮点
  ⚠️ 可以接受较低对话占比，但信息释放效率必须高
  ⚠️ 设定说明不能超过连续2段
```

**输出格式**：

```
## Layer 1 · 品类常识检查

平台：番茄

| 检查项 | 结果 | 说明 |
|-------|:---:|------|
| 前300字有冲突 | ✅ | 第1段就出现了规则触发 |
| 主角出场 | ✅ | 第1段 |
| 对话占比 | ⚠️ 18% | 品类基线 25-40%，偏低 |
| 章末钩子 | ✅ | 信息差型 |
| 连续无爽点 | ❌ 3章 | 品类安全上限2章，当前超标 |
| 前500字专有名词 | ✅ 2个 | 在安全范围内 |

结论：1项红线预警（连续无爽点3章），1项黄线（对话占比偏低）。
```

## Layer 2：赛道对标（复用参考书分析）

**来源**：`00-原始设定/_参考书分析/{书名}.md`（开书 Phase 0.3 产出）

如果目录不存在，本层跳过并提示"本层需要开书阶段 _参考书分析/ 目录，跳过"。

**运作方式**：

```
遍历 _参考书分析/ 目录下所有的参考书报告
  对每本参考书：
    读取「可学清单」和「避坑清单」
    对照当前章节，检查哪些做到了、哪些没做到

不要求"做得比参考书好"——而是"已知的风险有没有踩到、已知的成功技法有没有使用"
```

**输出格式**：

```
## Layer 2 · 赛道对标

参考书分析来源：00-原始设定/_参考书分析/

### 诡异人生 赛道对标
| 检查项 | 结果 | 说明 |
|-------|:---:|------|
| SL循环每轮有增量 | N/A | 本项目无SL机制 |
| 设定通过行动释放 | ⚠️ | 当前章节有一段纯说明性文字(约200字) |
| 对话占比不照抄低对话 | ✅ | 本章对话占比22%，已有意识避免 |

### 诡舍 赛道对标
| 检查项 | 结果 | 说明 |
|-------|:---:|------|
| 规则驱动叙事 | ✅ | 规则A在本章被触发 |
| 规则展示方式 | ⚠️ | 规则B是通过对话展示的，但触发条件不清晰 |
| 中段节奏控制 | N/A | 当前仅到第8章 |

### 赛道对标综合结论
- 参考书已知风险已规避：✅（对话占比、中段塌陷已有意识）
- 参考书成功技法已使用：⚠️（规则展示方式需优化，建议改为「违反→后果」的顺序）
```

## Layer 3：自设承诺（书自身的读者预期）

**来源**：项目已写好的前5-10章正文。不依赖宪法或章纲。

**原理**：每一本书只要写了几章，就在读者脑中建立了预期。这些预期是客观存在的（读者读到的东西），不是作者想给的。Layer 3 做的就是提取这份"承诺"，然后检查当前章节有没有兑现。

**运作方式**：

```
Step 1: 读前5章（或已有章节），提取「读者已建立的预期」
  - 类型预期：读者会以为这是"规则驱动型"还是"信息释放型"？
  - 节奏预期：前5章的节奏是快还是慢？读者已经适应了什么节奏？
  - 角色预期：某个角色前5章一直在做什么？读者会期待他继续做什么？
  - 悬念预期：前5章埋了什么未回答的问题？

Step 2: 读当前章，检查：
  - 有没有违背已建立的预期？违背是否有意为之（反转）还是失误？
  - 有没有兑现已建立的期待？
  - 有没有损害已有的角色认知？
```

**输出格式**：

```
## Layer 3 · 自设承诺

### 读者已建立的预期（从前5章提取）
- 类型：规则驱动型恐怖，每章会展示一条规则的触发/后果
- 节奏：前5章平均每章1.5个信息释放点
- 主角：警觉型，遇事会观察再行动
- 未答悬念：房东的身份、楼上的敲门声

### 当前章检查
| 检查项 | 结果 | 说明 |
|-------|:---:|------|
| 规则触发 | ✅ | 展示了规则C的触发后果 |
| 节奏匹配 | ⚠️ | 本章信息释放点仅1个（低于前5章均值1.5） |
| 主角行为一致 | ✅ | 仍然是观察后行动 |
| 旧悬念推进 | ❌ | 房东身份和敲门声在本章未被提及 |
| 新悬念建立 | ✅ | 章末建立了新悬念（规则D的暗示） |

### 自设承诺综合结论
- 类型承诺：✅ 维持
- 节奏承诺：⚠️ 偏慢（信息释放量下降）
- 角色承诺：✅ 一致
- 悬念管理：⚠️ 旧悬念未推进（连续X章未提及）
```

## Part B（可选）：向上游反馈

**触发条件**：当 market-test 在多次审稿中识别出系统性缺陷时，可选择性输出此部分。

**不随单次审稿自动输出**——避免"一次异常就改技能"的反应过度。建议积累3次以上同类问题再触发。

**输出格式**：

```
## Part B · 向上游反馈（可选）

### 系统性缺陷清单
| 缺陷 | 出现频次 | 涉及章节 | 根因推测 | 建议修改的Skill |
|------|---------|---------|---------|----------------|
| 系统面板式规则说明 | 4章 | ch3/ch5/ch7/ch8 | 写作Agent倾向于直接输出系统面板文字 | emergent-writer |
| 章末内心独白收尾 | 3章 | ch4/ch6/ch9 | prompt中缺少"钩子类型"约束 | emergent-writer |

### 具体修改建议
1. **emergent-writer**：在写规则说明时，禁止使用系统面板文字格式，改为「玩家对话发现规则」或「违反→后果展示」
2. **emergent-writer**：章末检查"收尾段落是否是主角内心独白"，如果是则标记为伪钩子，强制替换为信息差或半路阻断

### 不建议动的
- 主角的警觉性表达（当前稳定，不要因为一次异常就改）
```
