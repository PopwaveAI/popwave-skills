# 开书启动 — 项目初始化（完整版）

> 版本：v2.9 | 日期：2026-06-03 | 分类：bootstrap | 执行顺序：Phase 1

---

## v2.9 核心变化

1. **Phase 0.5 强制执行**：跨域素材聚合从"可选"升级为"强制禁止跳过"，增加量化产出标准（覆盖领域≥4、种子≥8、字数≥1500），不达标自动补充。
2. **Phase 1.2 L1深度展开**：新增逐篇扩写阶段，为每篇L1文件设定最少子维度数、每子维度描述行数、总字数门槛，配合跨域素材注入和画面锚点要求。不达标禁止进入下一步。
3. **Phase 1.3 L1交叉关联矩阵**：六篇L1文件逐对交叉检查14对关联关系，强制补写交叉引用，确保设定有机整体而非独立文档。
4. **产出深度标准全面量化**：每篇L1文件有具体的字数门槛、子维度数量、描述行数要求，将"丰富度"从主观判断变成可量化检查。


### v2.8 保留特性（v2.9 继承并增强）

1. **reader_profile 嵌入**：Phase 4 新增，在 project.yaml 中嵌入读者画像字段（人口统计/阅读偏好/爽点敏感度/弃书阈值），确保设定层始终对齐目标读者。
2. **数值体系模板升级**：Phase 5 新增 combat_capability / monster_rank_map / act_rank_schedule 三组标准化模板，规范战力-怪物-节奏映射。
3. **超越性硬检查**：Phase 6 新增 Boss 超越性检查（超越时机/代价/叙事功能三轴）与数值体系超越性检查（膨胀率/跨卷曲线/玩家感知验证）。
4. **产出规则统一**：所有 Phase 的输出产物明确标注路径与格式，接入 novel-agent-pro 的 manifest 追踪体系。

## 什么时候使用

| 场景 | 说明 |
|------|------|
| **新书开坑** | 从零到一搭建完整的小说项目骨架 |
| **创意可行性验证** | 不确定核心创意能不能撑起一部长篇，需要系统检验 |
| **设定规范化** | 已有零散设定，需要将其结构化、分层化、标准化 |
| **需要写作资产体系** | 项目需要包含数据库、HTML追踪面板、SVG可视化等配套 |

**不要使用此 Skill 的场景：**
- 只想写一个短篇/中篇（< 10 万字）→ 用 light-bootstrap
- 只需要生成 world stability check → 用 world-stability-check 独立 skill
- 已有完整项目结构，只想更新局部设定 → 用对应的设定 skill

## 完整工作流程

### Phase 0：L0 灵魂层对齐

**目标**：确认作品灵魂——为什么是你来写这本书？为什么读者要看？

前置条件：用户已提供至少「书名」「核心创意」「核心爽点」的初步描述。

执行步骤：

1. **灵魂三问**
   - Q1 为什么是我来写这本书？（作者独特经验/执念/知识储备）
   - Q2 读者为什么非看不可？（不可替代的体验/情感/认知）
   - Q3 这本书的核心情绪是什么？（热血/治愈/悬疑/爽/虐/幽默……）
   
2. **产品定位卡 (PRD)**
   - 输出 `00-原始设定/L0-产品层/PRD.md`
   - 内容要素：产品名称（书名）、一句话卖点、目标读者画像（年龄/性别/阅读场景）、核心体验承诺、差异化优势、风险/弱点预判
   - PRD 是全书最高优先级约束文件，后续所有设定决策须可回溯至 PRD

3. **核心创意压力测试**
   - 测试1：「电梯测试」——30 秒内向陌生人讲清楚这本书
   - 测试2：「长篇拉力测试」——核心创意是否能支撑 200 万字以上
   - 测试3：「平台适配测试」——创意与目标平台的主流读者画像是否匹配
   - 测试4：「差异化测试」——和同类型 Top 20 作品的核心差异点
   - 测试4通过后，启动 Phase 0.3

---

### Phase 0.3：参考书甄别

**目标**：系统筛选对标参考书，为后续设定提供借鉴与差异化锚点。

条件：Phase 0 灵魂对齐完成，PRD.md 已存在。

执行步骤：

1. **参考书粗筛**
   - 基于核心创意+核心爽点，搜索 5-10 本潜在对标作品（同类型/同赛道/同金手指变体）
   - 排除标准：数据太差（均订 < 500）、风格差异过大、年代久远（> 10 年且无翻新）
   - 保留 2-4 本进入深度分析

2. **深度分析**（调用 novel-deconstructor skill）
   - 对每本参考书执行模式B「对标拆解」
   - 提取：节奏地图、核心设定规则、爽点触发模式、读者评价的痛点/槽点
   - 输出 `00-原始设定/_参考书分析/{书名}.md`

3. **差异化决策**
   - 对比 PRD 中的差异化优势，确认「我不做什么」
   - 生成差异化清单：明确标注「借鉴点」「偏离点」「空白创新点」
   - 输出：差异化决策备忘录（嵌入 PRD.md 尾部）

### Phase 0.4：金手指概念设计

**目标**：设计有约束、有代价、有成长弧线的金手指系统。

条件：参考书分析完成，差异化方向已明确。

执行步骤：

#### 节点 A：金手指导演判断

**核心问题**：作品类型是否需要金手指？

| 作品类型 | 金手指必要性 |
|----------|-------------|
| 系统流/无限流/穿越重生 | **必须** — 金手指是剧情发动机 |
| 都市/现实/历史 | **可选** — 可以用天赋/知识/技能替代 |
| 纯文学/悬疑推理 | **不推荐** — 破坏悬念/代入感 |

> 如果判断为「不推荐」，跳至 Phase 0.4.2（无金手指路线替代方案）。

#### 四级评估

| 级别 | 评估项 | 说明 |
|------|--------|------|
| L1 | 核心功能 | 金手指做什么？一句话说清楚 |
| L2 | **约束条件** | 使用限制是什么？（冷却/代价/条件/副作用） |
| L3 | **成长弧线** | 金手指自身是否成长/进化？升级条件？ |
| L4 | **叙事功能** | 金手指如何驱动剧情？（解决问题/制造冲突/暴露性格） |

> L2-L4 是金手指设计的质量分水岭。L2 缺乏约束 → 无敌流风险；L3 缺乏成长 → 前期爽点耗尽后空转；L4 缺乏叙事功能 → 沦为机械降神。

#### 三必答问题

1. **代价是什么？** — 使用金手指的代价必须可感知、可累积、不可轻易消除。
2. **成长空间是什么？** — 金手指是否有明确的升级路径？升级是否伴随新限制？
3. **什么情况下金手指会失效？** — 必须有「金手指失灵场景」作为剧情张力来源。

#### 文化根系检查

- 金手指的文化原型/隐喻是什么？（例如：中国传统修仙体系中的「丹田」；西方奇幻中的「血脉觉醒」）
- 是否与作品的世界观基调一致？
- 是否与目标读者的文化认知框架兼容？

#### 番茄视角验证

模拟番茄小说读者视角，检查：
- 前 3 章：金手指是否出现？是否足够吸引人？
- 前 10 章：金手指是否持续驱动剧情？是否制造了明确的期待？
- 前 50 章：金手指成长曲线是否可见？是否仍有新意？

**产出**：
- `00-原始设定/L1-元设定层/04-金手指.md`
- 包含：核心功能定义、约束条件矩阵、成长树（SVG/ASCII）、失效场景清单

#### Phase 0.4.2（替代路线）：无金手指替代方案

如果作品不需要金手指，需回答：
1. 主角的核心驱动力是什么？（欲望/使命感/诅咒/债务……）
2. 剧情推进的发动机是什么？（解谜/追凶/生存/复仇……）
3. 替代金手指的「不对称优势」——主角拥有什么别人没有的？
   - 知识差异（穿越者/行业专家）
   - 认知差异（更高维度的思考方式）
   - 资源差异（人脉/遗产/秘密组织）
   - 体质差异（特殊血统/天赋/变异）

**产出**：`00-原始设定/L1-元设定层/04-无金手指替代方案.md`

### Phase 0.5：跨域素材聚合 ★ 强制 | 禁止跳过 | 深度设定之根基

**目标**：从 L0 灵魂对齐 + 核心卖点出发，跨越多个知识领域搜索素材，将外部研究转化为 L1 设定层的设计养分。

> ⚠️ **关于设定浅与深的真相**：本书现在开书阶段的设定之所以"浅"，根因就在于这一步被跳过了。Phase 1 写的 L1 文件只靠 LLM 内部知识——那是"泛泛的常识"。跨域搜索的作用是把"泛常识"变成"有根系的设定"。跳过这一步，后面的 L1 无论如何展开都是空中楼阁。

**条件**：PRD 已定稿，金手指设计完成。

**● 强制规则** —— 这一步是把"一个好创意"变成"一个可落地的世界观"的关键转换阶段，也是决定设定深度的核心关口。
**禁止跳过**。L1 设定层的丰富度依赖此阶段的素材质量。

**执行步骤**：

1. **卖点词拆解**
   - 从 PRD 的核心卖点中提取 3-5 个关键词/短语
   - 示例：书名《深渊归途》→ 「深渊」「归途」「回家」「层」「怪物」「进化」

2. **跨域搜索**（每个关键词执行 1-2 次 WebSearch）
   - 领域参考矩阵（至少覆盖 4 个不同领域）：

   | 关键词类别 | 搜索方向 | 示例 |
   |------------|---------|------|
   | 地理/环境 | 真实地理/建筑/生态 | 深海热泉生态系统、地下城结构、喀斯特地貌 |
   | 生物/怪物 | 古生物/神话生物/生态位 | 寒武纪奇虾、北欧神话中庭巨蛇、深海生物发光机制 |
   | 社会/组织 | 历史组织/秘密社团 | 共济会结构、中世纪行会、忍者组织 |
   | 技术/魔法 | 真实科技/伪科学/神秘学 | 卡巴拉生命之树、炼金术原理、量子隧穿 |
   | 文化/哲学 | 哲学概念/文化原型 | 尼采永恒轮回、佛教六道、英雄之旅原型 |
   | 心理/情绪 | 认知科学/行为模式 | 斯德哥尔摩综合症、峰终定律、习得性无助 |

3. **素材蒸馏**
   - 每个搜索方向提取：**至少 2 个**可直接使用的「设定种子」
   - 标注：种子来源、与核心卖点的关联度、可转化性（直接使用/改编使用/仅作灵感）
   - 产出：`00-原始设定/_素材聚合/跨域素材蒸馏.md`

4. **设定转化**
   - 将素材种子映射到 L1 设定层的六个维度：
     - 01-世界观底层逻辑 → 自然法则/宇宙规则
     - 02-世界观表层规则 → 社会制度/文化习俗/力量体系规则
     - 03-种族/物种设定 → 基于素材的物种设计
     - 04-金手指 → 素材对金手指系统的补充/限制
     - 05-势力组织 → 基于历史/社会素材构建组织
     - 06-关键物品/资源 → 基于科技/魔法素材构建物品体系
   - **最少有 3 个种子最终转化到 L1 文件的具体段落**，不可零转化

**产出**：
- `00-原始设定/_素材聚合/跨域素材蒸馏.md`
- L1 各文件的「素材来源注释」

**产出量化标准（自动检查，不达标则补充搜索）**：
| 检查项 | 门槛 | 不达标处理 |
|--------|:---:|:----------:|
| 覆盖领域数 | ≥ 4 个不同领域 | 补充未覆盖的领域 |
| 设定种子总数 | ≥ 8 个 | 补充搜索 |
| 转化到 L1 的种子数 | ≥ 3 个 | 补充转化映射 |
| 蒸馏报告字数（中文） | ≥ 1500 字 | 补充素材描述 |

### Phase 1.2：L1 深度展开 ★ 自动执行 | 禁止跳过 | 逐篇扩写

**为什么需要这一步**：Phase 1 产出的 L1 文件完成了骨架搭建，但每个维度仍需深度展开才能真正成为写作时的可靠参考。本阶段自动对每一篇 L1 文件进行逐项展开，不接受"先这样吧"的浅层产出。

**条件**：六篇 L1 文件骨架已完成，跨域素材蒸馏报告已就绪。

**核心原则**：
- **逐篇执行，不可并行**——每篇 L1 文件单独经历「深度展开节点」，完成后再进入下一篇
- **素材注入优先**——先看看 Phase 0.5 的跨域素材库里有什么是这篇能用的，再用
- **画面锚点必须存在**——每个子维度至少配 1 个让读者能"看到"的记忆锚点
- **不达标则继续**——字数门槛是底线，不够就回头展开，不进下一步

#### 深度展开节点（每篇 L1 文件执行一次）

```
【深度展开节点】
  ├── ① 素材注入 —— 从跨域素材蒸馏报告中提取与本维度相关的种子，嵌入对应段落
  ├── ② 维度展开 —— 每个子维度展开到至少 3 条具体描述，每条描述包含：
  │       - 定义（它是什么）
  │       - 表现形式（在故事中读者会看到什么）
  │       - 对剧情的影响（它如何推动/约束情节）
  ├── ③ 画面锚点 —— 每个子维度至少配 1 个记忆锚点（一句话画面/比喻/典型场景）
  └── ④ 扩充程度达标检查 —— 不达标则返回 ② 继续展开
```

#### 各 L1 文件的深度标准

### Phase 6：超越性硬检查

**目标**：对 Boss 设计和数值体系进行超越性检查，确保长期叙事的可持续性。

条件：数值体系模板已创建（Phase 5）。

执行步骤：

#### 6.1 Boss 超越性检查

对每个关键 Boss 逐项评估：

| 检查维度 | 说明 | 检查标准 |
|----------|------|----------|
| 超越时机 | Boss 出现的时机是否合理 | 至少提前 10 章铺垫；Boss 战前后应有节奏缓冲 |
| 超越代价 | 击败 Boss 的代价是否可感知 | 主角必须付出可量化的代价（受伤/损耗/失去某物/决策困境） |
| 叙事功能 | Boss 在叙事中的角色 | Boss 必须推动至少一个长期叙事线；纯数值填充的 Boss 需要删除或重构 |

**检查结果记录**：
- 通过 → 标记 green
- 有条件通过 → 标记 yellow + 改进建议
- 不通过 → 标记 red + 必须重做

#### 6.2 数值体系超越性检查

| 检查维度 | 说明 | 检查标准 |
|----------|------|----------|
| 膨胀率 | 力量体系的数值增长速度是否合理 | 主角每次升级的战斗力增幅保持在 1.5x-3x；全卷跨度内不超过初始的 1000x |
| 跨卷曲线 | 力量等级在全书跨度的变化 | 中期不可出现「战力天花板」；后期应有递进感而非数值膨胀 |
| 玩家感知验证 | 模拟读者对战力增长的感知 | 中期和后期读者是否仍对升级有期待？战力增长是否伴随叙事意义的提升？ |

**产出**：
- `00-总控/数值体系/超越性检查报告.md`
- 每个 Boss 的三轴评分表
- 数值体系膨胀曲线图（建议用 SVG 或 ASCII 图表）
- 修复建议（如有不通过项）
