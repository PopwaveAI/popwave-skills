# Phase 3：项目骨架创建

**目标**：生成完整的小说项目骨架——目录结构、总控文件、角色卡、数据库、追踪面板。

**条件**：卷级设计至少完成卷01。

## 执行步骤

1. **目录结构创建**（参见 `references/产出目录结构.md`）

2. **总控配置** → `00-总控/project.yaml`
   - 项目元信息（书名/作者/平台/开始日期/目标字数）
   - Phase 进度标记
   - 文件索引
   - reader_profile

3. **宪法文件** → `constitution.yaml`
   - 从 PRD 提取的不可违背创作原则
   - 核心情绪基调
   - 禁止清单

4. **角色卡创建** → `02-大纲/L3-角色层/`
   - 主角：满配（背景/动机/成长弧线/关系网）
   - 重要配角：中配
   - 龙套：文中动态塑造，不创建文件

5. **数据库初始化** → `04-数据库/novel.db`
   - 表：projects / chapters / characters / items / events / notes

6. **追踪面板** → `00-总控/project-status.html`

7. **章状态文件** → `00-总控/chapter-state.yaml`
   - 模板：见 templates/ 目录
   - 作用：写作时作为"单点真理源"，写前读取当前状态，写后自动更新
