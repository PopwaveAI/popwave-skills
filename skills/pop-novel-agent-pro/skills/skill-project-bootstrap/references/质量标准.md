# 质量标准

| 维度 | 标准 | 验证方式 |
|------|------|----------|
| 完整性 | 所有 Phase 的产出文件均已创建 | 检查文件清单与 Phase 产出一致 |
| 一致性 | L0-L1-L2 设定无矛盾 | 交叉引用检查（Phase 1.5） |
| **深度** ⭐ | **每篇 L1 文件达到字数门槛+子维度数+描述行数要求，14对交叉关联全部通过** | **Phase 1.2 展开完成验证 + Phase 1.3 关联矩阵检查** |
| 可追溯性 | 每个设定可回溯至 PRD | 设定文件中标注 PRD 引用 |
| 读者对齐 | 设定符合 reader_profile | Phase 4 对齐检查 |
| 爽点密度 | 符合 Phase 2 爽点密度标准 | 爽点分布检查 |
| 金手指约束 | L2-L4 评估均通过 | 金手指设计文档检查 |
| 数值合理性 | Phase 6 超越性检查通过 | 超越性检查报告 |
| 宪法遵守 | 所有设定不违背 constitution.yaml | 偏离度检查 |
