# L1 · 元设定层通用模板

> 开书阶段必填。每个"有名字的、会在书里作为独立实体出现的"BOSS/怪物填一份此模板。
> 填完后数据进入 v3.db weirds/skills 表，写作时 ESM before 自动加载到 context。

---

## Boss/怪物设计模板（通用版）

适用所有类型小说中任何一个"需要被主角对抗的实体"——妖兽/诡异/异形/机械/人类/神明。

```yaml
boss_profile:
  # ── 基础身份 ──
  name: "正式名称"                    # 读者记住的名字。不能是"那个东西"
  alias: "别称"                       # 不同势力对它的称呼差异也是世界观
  category: "妖兽/诡异/异形/机械/人类/神明"  # 按项目题材写

  # ── 行为系统（决定它在故事中的存在方式）──
  behavior_logic:
    活动范围: "它出现在什么地方？"
    活动时段: "它什么时候活动？"
    触发条件: "主角做什么会遇到它？"
    行为模式: "它在没有敌人时做什么？"   # 游荡/休眠/筑巢/巡逻/进食

  # ── 战斗系统 ──
  combat:
    攻击仪式:                          # 每次攻击的标志性画面
      - "第一次攻击时的前摇是什么？"     # 让读者能预判
      - "攻击时的标志性画面"             # 必须有画面感
      - "击中的后果"                    # 具体身体变化
    攻击方式: []                        # 2-3种，不贪多
    防御方式: "怎么规避伤害？"
    弱点: "什么能真正伤到它？"
    战斗节奏: "它会变招吗？会狂暴吗？会逃跑吗？"

  # ── 世界观生态位 ──
  ecological_role:
    - "它为什么存在？（不是剧情原因，是在世界里的自然位置）"
    - "它和其他物种的关系？"
    - "杀死它会破坏什么？"

  # ── 叙事功能 ──
  narrative_role:
    type: "精英怪/BOSS/世界BOSS/多次遭遇/解密型"
    用途: "展示世界观/测试新能力/制造转折"
    战后影响: "杀掉它之后世界有什么变化？"

  # ── 感官锚点（读者记住它靠这个）──
  sensory_signature:
    视觉: "最大的视觉特征一句话"
    声音: "它发出什么声音？"            # 沉默也是一种声音
    气味: ""                            # 可选
    触感: "被它碰到的感觉？"

  # ── 文化/原型锚点（可选但强烈推荐）──
  archetype:
    culture: "源自什么文化/传说？"
    reference: "参考了什么经典怪物设计？"
    deviation: "和原型的差异是什么？（创新的来源）"
```
