# 分阶段学习路径

> 按写作 Pipeline 的每个环节，标注需要读什么学习资料、重点看什么。
> 不是"学一遍就没用了"——是每次进入该环节前复习对应的资料。
> 创建日期：2026-05-19

---

## Pipeline 总览

```
开书启动 → 剧情架构（+爽点版场景卡）→ 正文写作（幕纲数据直读）→ market-test
   │           │                            │                     │
   │           │                            │                     └─ 写完审
   │           │                            │                     └─ 复盘沉淀
   │           │                            │
   │           │                            └─ 轻量自检（爽点/情绪/钩子）
   │           │
   │           └─ 爽点分布/情绪叠加/叠加压缩/节奏诊断
   │
   └─ L0灵魂对齐（六件必出）→ 参考书分析 → L1-L3设定
```

> **v7.8 管线变化**：移除了章纲生成环节。上下文组装器直接从幕纲读取数据（emotional_goal / payoff / reader_emotion_path / end_hook），跳过中间层。

## 关键原则

1. **不是一次学完** — 每个阶段只看当前阶段需要的资料。开书时不看"QC怎么写"，正文写作时不看"钩子体系怎么设计"。
2. **读重点、不读全** — 每个文件标注了"重点看什么"，不要求通读后才干活。
3. **拆书报告是常读文件** — 密藏域报告在不同阶段有不同的读法：开书读世界观设计方法，正文写作读写法规则，market-test 读赛道对标。
4. **复盘资料只在复盘节点读** — 写作过程中不读复盘资料，避免"上次踩坑"的阴影影响当前章节的判断。
