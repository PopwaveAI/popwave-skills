# 学习资料整合方案

> 目标：将网文行业通用知识（黄金三章/爽点系统/钩子设计/编辑标准/代入感）吸收进 6 个 Skill，填补"只会写但不知道什么算好"的空白。
> 数据源：学习资料/ 内 7 份行业文档

---

## 总览：各 Skill 的缺口与吸收点

| Skill | 当前强项 | 当前缺什么 | 吸收什么 | 改动量 |
|:---|:---|:---|:---|:---:|
| **project-bootstrap** | L0-L3分层设定、跨域研究 | 目标读者画像无行业标准参考，读者定位靠感觉 | 平台读者画像（番茄vs起点vs晋江）、三大引擎（爽点/代入感/期待感）作为L0对齐输入 | 中 |
| **plot-architecture** | 情绪曲线锚定、偏离检测 | 平台节奏参考表笼统、缺乏三幕式结构指导 | 三幕式篇章配比（25%/55%/20%）、情绪叠加模型、平台忍耐曲线细化 | 中 |
| **chapter-outline** | 事实链、信息差设计、锚定检索 | 钩子只有字段没有类型体系、微观节奏无指导 | 6种钩子类型+连续两章同类型禁令、起承转合模板、每章情绪升维设计 | 中 |
| **emergent-writer** | 导演层/碎片写作/QC双Layer | QC检查缺少行业参考区间、导演层爽点判断力弱 | 对话占比参考区间25-40%、爽点类型判断维度、QC追加代入感检查 | 小 |
| **reader-qa** | SOP-R/SOP-C/SOP-P 三模式完善 | 评分维度缺对话/爽点/代入感、问题标签不全 | 六维编辑评分模板、新增对话密度/爽点节奏/代入感维度、新增人设模糊/钩子疲劳标签 | 中 |
| **book-deconstructor** | beat_label 16类、质量门三级、去重 | beat_label与行业术语脱节、节奏地图缺读者感受标注 | 新增"爽点释放""追更驱动"等节拍标签、三幕式章节定位 | 小 |

## 各 Skill 具体修改方案

### 1. skill-project-bootstrap

**修改位置**：Phase 0 L0灵魂层对齐

**当前**：核心爽点（先知优势/割韭菜/建设反馈）、情绪价值（紧张/轻松/爽感/悬疑）

**补入**：
```
新增 L0必备输入：
  - 目标平台（番茄/起点/晋江）→ 自动加载对应读者画像
  - 目标读者画像模板（从行业数据提炼）：
     番茄读者：碎片化阅读、3秒决定去留、喜欢快节奏
     起点读者：老书虫、容忍铺垫但必须有回报、付费意愿高
     晋江读者：人设驱动、情感线优先、节奏可慢但人物不能崩
  - 三大引擎自检：爽点类型是什么？代入感靠什么？期待感怎么制造？
```

### 2. skill-plot-architecture

**修改位置**：幕纲生成 + 平台节奏参考表

**当前**：情绪曲线锚定参考书、平台节奏参考表（番茄/起点/晋江三列）

**补入**：
```
幕纲情绪曲线增加：
  - 三幕式配比：第一幕（20-25%）、第二幕（50-60%）、第三幕（15-20%）
  - 情绪叠加标注：幕内每章可标2种情绪（不是单一标签）
    例：ch06 = 紧张+好奇、ch12 = 爽+不安

平台节奏参考表细化为：
  - 番茄：前10章生死线（≤3000字/章）、前3章留人率≥30%、爽点每章1个
  - 起点：前3章建置期（≤3500字/章）、每8-12章一个大爽点
  - 晋江：前5章人设期、每10-15章一次高潮
```

### 3. skill-chapter-outline

**修改位置**：章纲格式中的 end_hook 升级 + 新增起承转合指导

**当前**：end_hook 只是一个字段，没有类型标记

**补入**：
```
end_hook 升级为对象（非字符串）：
  - type: 悬念型/事件断裂型/倒计时型/情绪冲击型/信息炸弹型/反转型（六选一）
  - content: 具体钩子内容
  - 检查：连续两章不能用同类型

新增"章节节奏建议"（可选）：
  - 起（前500字）：钩子/冲突
  - 承（500-1500字）：推进/积累
  - 转（1500-2500字）：转折/爆发
  - 合（2500-3000字）：收尾+新悬念
```

### 4. skill-emergent-writer

**修改位置**：QC Layer 2 检查项

**当前**：Payload交付、体验目标达成、新概念≤2、碎片配比（对话≥1）、连续8段无时序信号

**补入**：
```
QC Layer 2 新增检查：
  - 对话占比：建议25-40%（从数据统计），低于20%或高于50% → ⚠️ 提醒
  - 爽点节奏：本章有无至少1个微爽点？
  - 代入感检查：主角动机是否"普通"（保护家人/活下去/证明自己）？
  - 情绪曲线：本章读者情绪是否经历了"起"（不是一直平）？

导演Agent判断维度新增：
  - 爽点类型预判：这章的核心爽点是哪种？（打脸/获得/升级/守护/复仇/装逼）
  - 铺垫-释放比评估：铺垫是否足够？是否还没铺垫就直接爽了？
```

### 5. skill-reader-qa

**修改位置**：SOP-R 评分维度 + 问题标签库

**当前**：五维评分（信息密度/情绪曲线/追更意愿/困惑点/关键画面）、10类问题标签

**补入**：
```
评分维度升级为七维（新增2个）：
  + 对话密度：是否在合理区间？（1-10分）
  + 代入感：读者能代入主角吗？动机是否"普通"？（1-10分）
  + 爽点节奏：本章有爽点吗？铺垫够吗？（1-10分）

问题标签库新增（3类）：
  + 人设模糊：看完记不住主角特质
  + 代入感弱：主角动机离读者太远
  + 爽点不足：铺垫太长或爽点太弱

编缉视角评分可作"第二意见"：
  SOP-R 完成后，可运行一次"编辑视角评分"（六维评分+过稿线判断）
  作为参考，不作为否决标准
```

### 6. skill-book-deconstructor

**修改位置**：beat_label 16类

**当前**：开篇建立/金手指亮相/危机植入/情感兑现/世界观种子/能力验证/日常推进/情感锚点/能力突破/铺垫/爽点交付/身份确立/新地图进入/信息差激活/代价具体化/幕末高潮

**补入**（新增3个节拍标签，不删除现有的）：
```
+ 追更驱动：章末钩子特别强的章节（对应钩子设计指南的6种类型）
+ 读者情绪拐点：读者情绪从低到高或从高到低的转折章（对应情绪叠加模型）
+ 代入感锚定：章节重点在建立读者与主角的情感连接（对应代入感三要素）
```

## 执行顺序

| 批次 | 内容 | 工作量 | 优先级 |
|:---:|:---|:---:|:---:|
| 1 | **emergent-writer** — QC追加对话占比/爽点/代入感检查 | 小（改QC部分） | 🔴 最高 |
| 1 | **reader-qa** — 评分维度升级 | 中（改模板+标签库） | 🔴 最高 |
| 2 | **chapter-outline** — 钩子类型体系+起承转合 | 中（改章纲格式） | 🟡 |
| 2 | **plot-architecture** — 三幕式+平台节奏细化 | 中（改幕纲+参考表） | 🟡 |
| 3 | **project-bootstrap** — 读者画像+平台定位 | 中（改L0阶段） | 🟢 |
| 3 | **book-deconstructor** — beat_label 扩充 | 小（加标签枚举） | 🟢 |
