# 钩子设计指南

> 来源：公众号"我在写故事" + 多篇爆款拆解
> 核心：钩子的底层逻辑是"未完成感"——人在阅读时追求预期被满足的过程

---

## 一、钩子的底层原理

### 常见误解

❌ "在章末写个人突然晕倒/传来敲门声"
❌ "主角说了一句'完了'"
❌ "留下一个语气悬念"

这些不是真正的钩子。读者现在的阈值极高，生硬的转折只会让人觉得故弄玄虚。

### 钩子的本质

```
读者在阅读时，是在追求"预期被满足"
优秀作者极度擅长控制"预期的发放节奏"
他们在读者即将获得满足的前一秒，按暂停键
```

**真正的钩子 = 未完成感 = 读者的小手不受控制地划向下一页**

## 二、三种高转化钩子

### 2.1 信息差钩子（推荐最高）

核心：让主角和读者拥有不对等信息。

**手法 A：主角先看到真相，读者被晾在半空**

```
平庸写法：
主角打开保险箱，拿出日记，读完 → 哦，原来如此 ✓ 信息完整 ✗ 没有钩子

带钩子写法：
主角颤抖着翻开日记的最后一页，
目光停留在落款的那个名字上。
他猛地站起身，碰倒了手边的咖啡杯。
竟然是她。
↑ 主角知道了答案，但读者不知道"她"是谁
→ 读者为了填平信息差，划向下一页
```

**手法 B：读者知道真相，主角不知道**

```
读者通过一个切场景，看到反派的密谋
而主角还在按原计划行动
→ 读者产生焦虑："别去啊！你不知道那是陷阱！"
→ 焦虑驱动阅读
```

### 2.2 半路阻断钩子

核心：在情绪顶点时截断。

```
主角终于掌握了反派做假账的铁证
准备冲进董事会当场揭发

平庸：他推开门，把文件拍在桌上 ✓ 爽感 ✗ 缺乏余味

带钩子：
主角大步走向会议室的红木大门，
手已经按在了金属门把上。
只要推开这扇门，里面那个伪善的人就会身败名裂。
就在这时，他口袋里的备用手机振动了。
屏幕上跳动的，是失踪五年的亲妹妹的来电。
↑ 一个旧的冲突还没释放，量级更大的新悬念砸下来
```

### 2.3 微钩子（每 300 字一个）

手机阅读一屏大约 300 字。优秀的作者会在脑中内置"字数感应器"，每 300 字递进一个小转折。

微钩子不需要惊天动地的反转：
- 主角一个迟疑的眼神
- 一句没说完的话
- 一个突如其来的细微动作

> "多去体会那种把话说到一半的克制感。等你熟练了，钩子就会变成你行文时的呼吸。"

## 三、章节级的钩子系统

### 章首钩子（前 300 字）

| 类型 | 写法 | 效果 |
|:---|:---|:---|
| 冲突重启 | "他没死。" | 立刻重启上一章的悬念 |
| 时间压力 | "还剩三个小时。" | 制造紧迫感 |
| 场景跳跃 | 直接从上一章结尾跳到关键场景 | 跳过无聊的过渡 |

### 章中钩子（每 800-1000 字一个）

- 信息释放 → 引出更大的疑问
- 小爽点 → 暗示还有更爽的在后头
- 角色反常行为 → 为什么他这么做？

### 章末钩子（最后 200-500 字）

**六大钩子类型**（不可连续两章同类型）：

| 类型 | 效果 | 案例 |
|:---|:---|:---|
| **悬念型** | 制造疑问 | "他愣住了——房间里站着一个人" |
| **事件断裂型** | 在最关键的时刻切断 | "他举起剑，用力劈下——" |
| **倒计时型** | 制造紧迫感 | "距离爆炸还有三分钟" |
| **情绪冲击型** | 用情绪收尾 | "原来从头到尾，都是他设计的" |
| **信息炸弹型** | 抛出一个重磅信息 | "爸——我是你儿子" |
| **反转型** | 颠覆读者认知 | "他不是来救我的，他是来杀我的" |

**硬规则**：
- 连续两章不能用同类型钩子
- 纯氛围式收束不算钩子（"夜很深了""他闭上了眼"）

## 四、钩子在网文 QA 中的应用

QA 可以检查的问题清单：

```
□ 每一屏（~300字）是否有一个微钩子/信息推进？
□ 章末钩子是否有明确类型标记？
□ 连续两章钩子类型是否重复？
□ 钩子是"真正的事件断裂"还是"生硬的'啊'式悬念"？
□ 是否有利用"信息差"制造阅读焦虑？
□ 主角知道某事的章节，是否给读者留下了不知道的盲区？
```
