# 爽点评估框架 · v0.1（子Agent专用）

> 用途：对章纲/正文进行爽点质量评分
> 主Agent行为边界：主Agent负责生成章纲和正文。本框架由**独立子Agent**运行，不参与写作过程。
> 介入时机：章纲定稿后 → 子Agent评估 → 不通过则退回调整 → 通过才进入正文写作

---

## 一、评分维度（6项，每项1-10分）

### 1. 读者渴望清晰度（权重×2）

**核心问题**：释放前读者脑中是否有一个**具体的话**？

**检查方法**：读完章纲/正文后，试着写出一句"读者在翻到这一页时最想看到的话"。

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 说不清读者想看什么 / 渴望模糊（"想知道后面怎么样"） |
| 4-6 | 能说出大概方向，但不够具体（"希望主角能赢"） |
| 7-8 | 有具体的、可描述的问句（"标记到底能不能打上"） |
| 9-10 | 渴望极其具体，且带有情绪色彩（"别让他白死啊！"） |

### 2. 阻碍强度（权重×1.5）

**核心问题**：读者是否在**真正担心**"这事不会成"？

**检查方法**：阻碍是单一的（一个困难）还是复合的（多个困难叠加）？读者在压抑段是否有"要完了"的感觉？

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 没有真正的阻碍，读者默认"肯定能成" |
| 4-6 | 有一个阻碍，但不够强，读者不太担心 |
| 7-8 | 有复合阻碍，读者会短暂紧张 |
| 9-10 | 极其强的阻碍+不知道会不会成功的悬念 |

### 3. 释放定位准确性（权重×2）

**核心问题**：章纲/正文是否正确识别了"释放点"并给了它对应的篇幅？

**检查方法**：找出全章/全段落中字数最多的位置——它是不是释放点？

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 最长的段落在压抑段/过渡段，释放点被压缩到几乎没有 |
| 4-6 | 释放点有一定篇幅，但不是全章重心 |
| 7-8 | 释放点占全章30-40%，字数分配合理 |
| 9-10 | 释放点明确作为全章重心，压抑段压缩到最低 |

### 4. 等级判断准确性（权重×1.5）

**核心问题**：章纲标注的等级（微/中/大/终极）与"释放后处境改变程度"是否匹配？

**判定标准**：
- **微** = 释放后处境没变，有情绪峰值
- **中** = 释放后当前局势发生实质性改变
- **大** = 释放后角色基本状态/关系发生不可逆变化
- **终极** = 释放后世界运行规则/格局发生变化

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 等级判断完全错误（微写成大、中写成毒点） |
| 4-6 | 等级偏高或偏低，但不算离谱 |
| 7-8 | 等级判断准确 |
| 9-10 | 等级精准，且设计预期与交付能力匹配 |

### 5. 名场面/金句锚点（权重×1.5）

**核心问题**：读者一周后能记住什么？

**检查方法**：读完章纲/正文，能否凭记忆说出一个具体的画面或一句台词？

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 没有任何可记忆的画面或台词 |
| 4-6 | 有一个画面但不够强，容易被忘记 |
| 7-8 | 有一个清晰的名场面或金句 |
| 9-10 | 名场面+金句双重锚点，读者能记住 |

### 6. 释放后处理（权重×1）

**核心问题**：释放后跟了什么？

**不允许**：解释性文字（"他意识到""这意味着""他明白了"）
**允许**：展示代价 / 展示新威胁 / 展示后果

| 分数 | 标准 |
|:---:|:---|
| 1-3 | 释放后跟了解释/对话/闲聊/总结 |
| 4-6 | 释放后没有明显解释，但有"多余的过渡" |
| 7-8 | 释放后直接展示代价或新威胁 |
| 9-10 | 释放后的下一句就是代价或新威胁，无任何过渡 |

---

## 二、总分计算

**总分** = (渴望清晰度×2) + (阻碍强度×1.5) + (释放定位×2) + (等级判断×1.5) + (名场面×1.5) + (释放后×1)

**满分** = 10×(2+1.5+2+1.5+1.5+1) = 10×9.5 = 95分

**评级**：
- 85+ = 优秀，可交付
- 70-84 = 良好，小幅调整后可交付
- 55-69 = 及格，需要调整
- <55 = 不及格，退回重修
