# Goal Mode — novel-agent-pro 全链路自动化方案 V3

> 理念：**目标驱动 + 代码闸门 + 违规审计**  
> 借鉴 Codex /goal：给一个目标 + 验收条件，agent 自己推进，直到完成标准全部满足。

---

## 一、核心架构

```
【你】设定目标:
  "写 30 章，每章遵循 emergent-writer v9.3 六阶段管线"

       │
       ▼
【我（SOLO）】进入 Goal Loop:
  ┌───────────────────────────────────────────────────────┐
  │ while True:                                           │
  │   step = progress.find_next_pending()                 │
  │   if not step: break                                  │
  │                                                        │
  │   # 1. 读上下文（一次缓存，持续使用）                   │
  │   # 2. spawn subagent → 执行 skill                    │
  │   # 3. python pipeline_gate.py --step X --file Y      │
  │   #    ↑ 我无权修改输出，只能读结果                     │
  │   # 4. progress.record(gate_result)                    │
  │   # 5. if gate == FAIL:                               │
  │   #      counter[step] += 1                            │
  │   #      if counter[step] >= 3:                        │
  │   #         mark BLOCKED → 跳过，不卡死管线            │
  │   # 6. continue                                        │
  └───────────────────────────────────────────────────────┘

       │
       ▼
【你】N 小时后验收:
  progress.json → 进度摘要
  violation_log.json → 每步违规记录
  → 一眼看出哪个环节在什么位置开始崩
```

## 二、目标设定

### 你设定的目标

```
/goal
项目: 灰骑士之主杀穿博德之门3 v2
平台: 起点中文网
全链路: 拆书 → 设定 → 幕纲 → 30 章正文（含 QC 三层介入）
验收标准:
  - 每步产出物存在且格式正确
  - pipeline_gate.py 全部通过（PASS 或 WARN 都接受，FAIL 必须记录）
  
执行规则:
  - 不要跳过任何步骤
  - 每步完成后必须运行 pipeline_gate.py
  - pipeline_gate.py 的输出不可修改
  - 连续 3 次 FAIL → 标记 BLOCKED，跳过继续，不卡死
  - 全部完成后打印进度总结
```

## 四、pipeline_gate.py（代码闸门）

这是**不可绕过的硬闸门**。

### 设计原则

1. **纯代码** — 无 LLM 参与，不依赖任何 AI 判断
2. **硬编码阈值** — 不允许外部参数改变判定标准
3. **仅读文件** — 不写任何内容，没有副作用
4. **JSON 输出** — stdout 输出结构化的 gate 结果，供编排层读取
5. **固定退出码** — 0=全部通过，1=有 WARN，2=有 FAIL

### 调用方式

```bash
python pipeline_gate.py --step B06 --file 03-正文/ch005.md --prev-wc 2400
# 输出: {"step":"B06","status":"PASS","issues":[],"word_count":2350,"duration_ms":45}
# 或:   {"step":"B06","status":"FAIL","issues":[
#         {"level":"FAIL","code":"WC_TOO_LOW","detail":"字数不足: 1500 (目标≥1800)"},
#         {"level":"WARN","code":"NO_HOOK","detail":"章末无钩子"}
#       ],"word_count":1500,"duration_ms":42}
```

### 各步骤验证项

#### B01-Director
- 文件存在性检查
- 字数 ≥ 500
- 6 项前置检查关键词（读者画像/经验日志/锚定章/字数目标/原始设定/爽点等级）
- 场景权重分配标记
- 锚定章引用文件名

#### B03-Pass1-骨架
- 文件存在性检查
- {实体名} 正则计数 ≥ 8
- 预估字数标记 ≥ 1800
- 【设计说明】块存在

#### B06-Pass2-正文（最严格）
- 文件存在性检查
- 字数 ≥ 1800（FAIL）/ ≥ 2200（WARN）
- 字数趋势：对比上章下跌 ≥ 20% → WARN
- 写后自评存在 + ≥ 100 字
- 违禁句式 7 变体正则匹配 → 有命中就 WARN
- 段落数 ≥ 8
- 平均段落长度 ≤ 400
- 章末钩子（最后 500 字包含 ? ! 但 然而 不知道 还没 没有 不是 …… ——）

#### B02/B04/B07-QC
- 文件存在性检查
- 含 PASS 或 NOT_PASS 判定
- B07 额外：含 5 个问答标记

## 五、自纠正机制

### 为什么需要自纠正

纯代码闸门只能"发现"问题，不能"解决"问题。如果某步持续 FAIL，需要机制让 agent 知道前一次失败了、原因是什么。

### 自纠正流程

```
subagent 执行 B06-ch008
  → pipeline_gate.py FAIL（字数 1500）
  → progress 记录 fail
  → 下次轮到同一章 B06 时：
    progress 告诉编排层："ch008 B06 上次 fail，原因是字数不足"
    编排层 spawn 新 subagent 时附带：
      "【前次失败: 字数 1500，目标 ≥ 1800。请补足字数再提交。】"
  → subagent 重做
  → 再次跑 gate
  → 如果连续 3 次仍 FAIL → 标记 BLOCKED，跳过，不卡管线
```

### 自纠正信息传递

每次 subagent 的 prompt 里附带：

```
## 进度上下文
当前进度: ch007 全部完成
当前步骤: ch008 B06-Pass2-正文

## 本章前序步骤审核
B01-ch008 ✅ PASS
B02-ch008 ✅ PASS  
B03-ch008 ✅ PASS
B04-ch008 ✅ PASS
B05-ch008 ✅ PASS
B06-ch008 ⚠️ 前次 FAIL（字数 1500，目标 ≥ 1800；缺少章末钩子）

## 全链路违规摘要
ch005 B06 WARN 字数偏低: 2030
ch007 B01 WARN 前置检查缺失: 爽点等级确认
```

这样 subagent 知道：
- 前面几步都过了
- 自己这步上一次出了什么问题
- 整本书目前有哪些违规趋势

## 六、Goal Mode 工作流程（完整伪代码）

```
# 这次是我（SOLO）执行的逻辑，不是脚本

def goal_loop(project_path, total_chapters):
    
    # 第 0 步：一次性加载静态资源
    cache = {
        "skill_md": read_file("emergent-writer/SKILL.md"),
        "templates": {
            "director": read_file("Director-prompt.md"),
            "pass1": read_file("Pass1-chapter-planner.md"),
            "pass2": read_file("Pass2-renderer.md"),
        },
        "reader_profile": read_yaml("project.yaml#reader_profile"),
        "settings": read_dir("00-原始设定/"),
    }
    
    # 第 1 步：加载/初始化进度
    progress = ProgressManager(project_path)
    
    # 第 2 步：主循环
    while True:
        step = progress.next_pending()
        if not step:
            break  # 全部完成
        
        # 2a. 构建 subagent prompt
        prompt = build_prompt(step, cache, progress)
        
        # 2b. spawn subagent（Task 工具）
        result = Task(query=prompt, subagent_type="general_purpose_task")
        
        # 2c. 跑代码闸门
        gate_cmd = (
            f"python pipeline_gate.py"
            f" --step {step.id}"
            f" --file {result.output_path}"
            f" --prev-wc {progress.prev_word_count}"
        )
        gate_output = run_command(gate_cmd)
        gate_result = json.loads(gate_output)
        
        # 2d. 我无权修改 gate_result
        progress.record(step, gate_result, result)
        
        # 2e. 更新上章字数（用于趋势检查）
        if step.type == "B06":
            progress.prev_word_count = gate_result.get("word_count", 0)
        
        # 2f. 自纠正：同一 step 连续 3 次 FAIL
        if gate_result["status"] == "FAIL":
            progress.fail_count[step.id] += 1
            if progress.fail_count[step.id] >= 3:
                progress.mark_blocked(step.id)
                log(f"⚠️ {step.id} 连续 3 次 FAIL → BLOCKED，跳过")
        else:
            progress.fail_count[step.id] = 0
    
    # 第 3 步：全部完成，打印总结
    summary = progress.summary()
    print_summary(summary)
    # 保存 violation_log
    violations = progress.violations()
    save_json("violation_log.json", violations)
```

## 八、应对偷懒的三层防御

| 层 | 机制 | 谁执行 | 可绕过 |
|:---|:-----|:-------|:------|
| **层1** | **会话隔离**：每步新 subagent，不知道总量 | 编排层 | ❌ subagent 不继承会话 |
| **层2** | **代码闸门**：pipelin_gate.py 独立运行，硬编码阈值 | pipeline_gate.py | ❌ 编排层只能读输出、不能改 |
| **层3** | **违规审计**：violation_log 记录全部 WARN/FAIL | 编排层 | ❌ progress.json 每步都写 |
