## 🔧 当前管线的实际瓶颈（按阻塞程度排序）

1. **跨书叙事语感控制** — 海贼法典写对了"站着讲世界"，但如果换成深渊主宰，同样写着写着会偏。管线不解决"不同书默认语感不同"的问题。LoRA是方向但太远。

2. **知识的可测量影响力** — K1-K4、场景模板、经验日志——这三样东西写进了输入包，但没有任何方法知道它们到底起了多少作用。如果它们其实没用，我们背了三项维护成本。

3. **Pass 1自动化缺失** — 目前骨架全部是人写。如果管线最终目标是"人写幕纲→管线自动产出骨架+正文"，那Pass 1 Agent的可靠性必须验证。

4. **骨架的QC标准未建立** — 骨架本身的质量谁来把关？如果Pass 1漏了重要事件或标注了错误的场景权重，Pass 2写得再好也是错的骨架约束出来的"好东西"。

## 🧠 根因复盘：场景类型预判不足与涌现式设计

### 发现的漏洞

ch392（生死战斗章）暴露了两个 schema 级别的缺失：

**漏洞一：交叉引用无双向约束。** 实体卡设计阶段只定义了"单个实体长什么样"（identity/palette/speech_style），没有定义"实体之间的关系由谁持有且如何验证一致性"。`inventory` 和 `skills.mastered` 是后来塞进角色卡的字段，但对应的 item/skill 卡没有反向校验。

**漏洞二：战斗/能力缺案例集。** 对话有 `speech_style.examples`（"你有女儿吗？""算你走运"），所以 Pass 2 写对话有素材参考。战斗的能力只有名称列表（"元素化""过热形态"），没有"这个能力在战斗中具体怎么表现"的案例数据。这不是字段的缺失——是 schema 设计从一开始就没有为"战斗渲染"准备素材层。

### 根因链条

```
设计实体卡 schema 时，只覆盖了"角色描述"（identity/speech_style/palette）
  → 因为第一轮测试跑的是深渊主宰（文学化感知型）和政治对话型（海贼法典ch389）
    → 对话型场景有 speech_style 就够了，战斗型场景还没测过
      → 直到 ch392 才暴露：能力只有名称列表，战斗无素材可参考
        → 同理：如果下轮测试跑"修炼突破"或"解谜推演"型场景，还会暴露新的缺失
```

**这不是"当时没写好"——这是只测了一种场景类型就认为 schema 够用。** 验证计划应该覆盖至少3种场景类型，但实际只覆盖了1-2种。

### 长期方案：通用起点 + 涌现式扩展

解决方案不是"先设计完整的 schema 覆盖所有场景类型"——这做不到，因为在写战斗章之前你不知道战斗章会缺什么，更不知道下下一个项目（深渊主宰的战斗/修炼/解谜）会缺什么。

**正确的设计策略：一个最小的通用起点 + 涌现式记录新增字段。**

```
起点 schema：
  角色 → { identity / state / relationships }
  物品/技能/地点 → { name / type / owner / description }
  ↑ 这是一个几乎什么书都能填的骨架

遇到新场景类型时（如战斗）：
  → 在技能卡上涌现式新增 combat_profile 字段
  → 在验证记录里记一笔："战斗场景类型 → 新增 combat_profile 字段"
  → 下次另一个项目遇到战斗场景时复用

而不是：
  一开始设计完整的 abilities.combat_use / limits / power_scale
  因为那时候你根本不知道这些字段在真实写作中会怎么用
```

**核心原则：管线只提供骨架和数据通路，schema 字段由场景类型驱动涌现。** 今天新增了 `combat_profile`，下次写修炼场景可能新增 `training_profile`，再下次解谜场景可能新增 `puzzle_profile`。每个字段的诞生都有一个具体的原因——不是因为"设计文档写了"，而是因为"不写这笔，Pass 2 渲染不了这种场景"。

### 验证标准的修正

当前升格条件：
```
- "ESM 在真实项目跑通 before/after 全流程≥3章"
- "连续写5章不出现状态冲突"
- "实体卡数量达到30+仍可正常维护"
```

应该追加：
```
- "至少覆盖3种不同场景类型并验证 schema 字段完备" ← NEW
```

因为 "5章不冲突" 只能测状态一致性，测不了 schema 的场景覆盖率。今天 4 章就已经覆盖了政治对话和生死战斗两种类型——如果 ch392 是 ch393，等 5 章验证通过之后再打斗，那么"战斗 schema 缺失"这个问题会被埋到升格后才暴露。
