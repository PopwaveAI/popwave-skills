### 为什么能提速

v2.0 前三章的节奏："做梦→醒来→去老宅→第二夜→第三夜→终于有人来了"——前3章全是铺垫，没有任何"让读者翻页"的爽点。

v2.1 前三章的节奏：
- CH001：冷开场（冲击力）+ 金手指亮相 + 被困 + 三重钩子
- CH002：老狗直接进场 + 核心卖点（"NPC？！"）+ 后院异常 + 手抄本自动显字
- CH003：阴影攻击 + 化煞刃首秀 + 门开了 + 棺材在等

**v2.0 用2.5章做到的铺垫，v2.1用0.5章完成了。** 多出来的2章用来释放——CH002完成核心卖点、CH003完成第一次战斗和能力觉醒。

## 四、导演 Agent 的贡献总结

### v2.0 管线（无导演层）

```
幕纲 → 章纲 → 碎片写作 → 碎片桥接 → 手动调整
```

- 没有人在每章写作前思考"这一章什么重要、什么可砍"
- 没有知识注入（LLM不知道番茄读者的阅读习惯）
- 没有经验传递（每次调用都是独立上下文）
- 没有反问机制（发现文件冲突时默默执行）
- 没有自检红线（AI味问题只有用户反馈后才修复）

### v2.1 管线（v8.0 导演 Agent）

```
幕纲 → 素材准备 → 导演判断(1次LLM) → 用户确认 → 组装 → 生成 → 自检 → QA
```

| 导演 Agent 贡献 | 具体体现 |
|:---|:---|
| **发现文件冲突** | CH001：章纲单线vs三份文件双线，建议以剧情脉络为准 |
| **修正CH002映射** | act-01.yaml旧定义"遗产"已过时，实际应该是碰撞章 |
| **笔墨优先级分配** | "金手指确认"核心写400-500字、"手抄本规则"可砍到1条 |
| **开局策略** | "我在跑"零前置语，第一人称冷开场 |
| **章末钩子类型检查** | CH002原设计"事件断裂型"与CH001冲突，主动修正为"悬念收束型" |
| **风险识别** | "不是A而是B"高风险预警（CH001检出7处+修复、CH002检出4处+修复、CH003检出11处+修复） |
| **知识注入** | K1-K4让导演知道番茄读者前300字决策、新概念≤2/章、对话≥25% |
| **经验日志** | 已写入1条经验记录（CH002战术判断），后续章节可参考 |

## 五、一句话总结

**v2.0 是用"堆文件+靠LLM自由发挥"的方式在写——管线提供的是格式，不是判断。v2.1 是用"先思考再写"的方式——导演 Agent 提供的是判断，不是流程。**

前三章的对比已经能看到差异：v2.1 在同一章数内（CH003就完成了战斗首秀）比 v2.0（CH003还在"两人终于见面了"）的叙事密度提升了约2-3倍。而且这些密度来自**正确的信息压缩**——砍掉了不该写的内容（两整夜的独处、重复的氛围描写），把笔墨压到了最核心的场景上（冷开场、金手指亮相、核心卖点碰撞、战斗首秀）。
