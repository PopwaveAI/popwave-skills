# 番茄化节奏修正复盘

> 时间：2026-05-18
> 涉及：V2 第一卷 act-01.yaml 结构重构
> 诱因：ch1正文被指出"番茄读者不会看完"

---

## 问题根因

**没有在卷纲设计阶段定义目标读者画像。**

设计第一卷时，参考了《我的诡异人生》的节奏——氛围慢热、压抑积累、27章才第一次大胜利。
但《我的诡异人生》的读者群体是"能接受慢节奏恐怖氛围"的悬疑读者——不是番茄读者。

番茄读者前三章决定去留。ch1如果主角被动、没有进展、没有小胜利——读者直接划走。

### 3. 项目复盘沉淀

此文档 + 索引更新。

## 可复用规则（跨项目）

1. **Skill 级** — 所有 plot/outline 类技能必须将目标读者画像作为必填输入
2. **项目级** — constitution.yaml 必须包含 reader_persona 节，影响所有设计决策
3. **检查级** — 章纲/正文生成前，先自检：这个节奏是否符合目标平台读者的忍耐曲线？
4. **差异化** — 参考书可以是不同平台的，但**我们自己的节奏必须与目标平台对齐**
