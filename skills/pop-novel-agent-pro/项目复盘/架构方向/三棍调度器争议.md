# 三棍调度器争议

> 三棍（崩坏度/应对力/差距）是否成立？
> 这是当前整个系统中最大的O（Open）——尚未闭合。

---

## 一、三棍从哪来

三棍不是拍脑袋的。它来自两条独立路径的汇合：

### 路径A：StoryOps沙盒推演

诡异游戏的 `005-causal-engine-十轮推演复盘.md` 记录了一次大规模因果推演实验：

1. **拆诡异人生的payload网络**：66节点、113因果边
2. **发现本作因果密度不到1/3**：诡异人生每3章就有1次"原因→结果"，本作要8-10章
3. **尝试用数值做推演**：赋予每个payload"崩坏度贡献""应对力贡献"
4. **实验结果**：推演结果与诡异人生实际曲线吻合度约80%

这给了"崩坏度/应对力"这个概念一个实证基础。

### 路径B：诡异人生的"A好+B好"模式

诡异人生的好不是单一原因，而是多层叠加：

```
A好（因果密度高）→ 读者觉得"合理"
B好（对话占比合理）→ 读者觉得"热闹"
C好（信息差控制）→ 读者觉得"焦虑"
D好（能力代价绑定）→ 读者觉得"有起伏，不是种马文"
```

三棍试图把"B好+C好+D好"中的节奏部分定量化。**这个动机是合理的。**

## 二、当前对三棍的质疑

### 质疑1：数字是否能翻译成好故事？

三棍本质上是将"节奏"数值化。但诡异人生的反例是这个：

```
诡异人生ch01-ch10中：
  第1章：主角濒死 → 崩坏度骤升 → 按理说要加压
  但第1章读者感受是"好奇+紧张"叠加，不是纯"被压着打"
  
  第4章：获得能力 → 应对力上升 → 按理说要推进
  但第4章读者感受是"爽+不安"，因为代价还没揭示
```

**问题**：一个数字（崩坏度=40）不能同时代表"危险等级"和"读者焦虑感"。前者是客观设定，后者是主观体验。三棍把两者混在一起了。

### 质疑2：情绪叠加怎么办

```
三棍假设：每章基调唯一（加压/突破/推进/喘息）
  但实际好章节往往是混合情绪：
  - "紧张+好奇"（新威胁登场时）
  - "爽+不安"（主角变强但代价未兑现时）
  - "绝望+希望"（逆境但留有反杀线索）

一个基调标签无法表达这种叠加态。
```

### 质疑3：代价对位是好的，但和三棍捆绑对吗

诡异人生中代价对位（每次变强绑代价）确实有效。但这是**独立于三棍的设计规则**，不依赖三棍数值系统也可以执行。把两者捆绑在 "三棍调度器" 中，增加了整体风险：

```
如果三棍被证伪 → 代价对位也会跟着被放弃 → 可惜

应该解耦：代价对位作为独立规则，三棍作为可选校准工具
```

## 三、三棍的"可验证"vs"不可验证"拆分

如果保留三棍，应该把可验证部分和不可验证部分清晰拆开：

### ✅ 可验证（有数据支撑）

| 规则 | 证据 | 可靠度 |
|:---|:---|:---:|
| 成长的每几步绑定一次代价 | 诡异人生数据：13次能力获取，11次绑代价 | 高 |
| 信息密度≤2新概念/章 | 诡异人生ch01只有3个概念 | 高 |
| 对话占比应该增加 | 诡异游戏15% vs 诡异人生40% | 高 |
| 连续N章无变化 = 无效章 | 灰骑士5-10章复盘发现 | 高 |
| 每8-12章允许1次"绝望章" | 诡异人生节奏模式 | 中 |

### ❓ 存疑（无实证支撑）

| 规则 | 问题 | 结论 |
|:---|:---|:---|
| 崩坏度/应对力独立数值化 | 数值=设定还是=体验？ | **待定** |
| 差距稳定在15-25 | 这个区间从哪来？诡异人生的数据？还是拍脑袋？ | **待定** |
| 基调判定表（差距变化→基调） | 场景是唯一基调吗？ | **待定** |
| 作者只答3个白话问题 | 如果作者意图无法被3个问题覆盖？ | **待定** |

### 独立规则（不依赖三棍）

以下规则直接来自诡异人生分析，**不需要三棍数值系统也能执行**：

1. 对话碎片≥1/章 ✅ 已在v7.6 QC层实现
2. 成长的payload必须绑代价 ✅ 已设计，待融入导演层
3. 新概念≤2/章 ✅ 已在v7.6 QC层设计
4. "读者明确知道X"替代"读者感觉到X" ✅ 已在v7.6实现
5. 连续3章动作+独白=不合格 ✅ 已在v7.6 QC层设计

## 四、建议的处理方式

### 短期：剥离三棍，保留独立规则

```
v7.6 当前架构中已经实现/设计了：
  - 碎片类型配比硬约束 ← 解决"对话太少"
  - 信息密度配额 ← 解决"信息密度失衡"  
  - 代价对位强制 ← 解决"修炼段不恐怖"
  - "读者明确知道X"可验证目标 ← 解决"战斗无参与感"
  - 场景卡入库5问质检 ← 解决"一眼看不懂"
  - QC双Layer ← 解决"红线落实不了"

这些已经覆盖了 v7.6.1-PRD 声称要解决的7个问题中6个。
唯一剩下的"场景核心=事件序列"是导演层设计问题，不依赖三棍。
```

**结论**：三棍对当前v7.6来说并非必需。已有的独立规则已经覆盖了大部分PRD声称要解决的问题。

### 中期：把三棍降级为可选诊断工具

```
三棍不做"调度器"（不控制写作），做"诊断器"（复盘时给出建议）：
  - 跑完每章后，反向标注三棍值（崩坏度+? 应对力+?）
  - 10章后出节奏诊断报告："差距连续8章在5以下，建议加压"
  - 给作者参考，不自动调整
```

这样三棍的"可验证部分"能保留，"不可验证的主动控制"不造成伤害。

### 长期：如果要落地，需要什么

1. **数据校准**：用3-5本成熟作品做全卷三棍标注，验证差距的合理区间
2. **A/B测试**：同章节用三棍版vs无三棍版对比，看读者体验差异
3. **情绪叠加模型**：至少支持2个基调叠加（如"紧张+好奇"），而不是单一标签

在这三点完成前，三棍调度器不建议落地。

---

## 五、总结

| 立场 | 论据 | 支持者 |
|:---|:---|:---:|
| 三棍应该做 | 节奏需要定量工具，纯靠感觉不靠谱 | PRD设计者 |
| 三棍有问题 | 数字无法代表情绪叠加，15-25区间无实证 | 复盘分析 |
| 三棍不是必须 | 独立规则已覆盖大部分问题 | v7.6现状 |

**当前建议：三棍不做调度器，做诊断器。保留为可选复盘工具，不进入写作流程。**
