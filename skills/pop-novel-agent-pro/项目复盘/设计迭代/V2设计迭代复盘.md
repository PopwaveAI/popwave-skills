## 二、关键决策（值得记录为跨项目经验）

### 2.1 第一卷结构：日常裂缝 → 秘境 Boss

**V1的错误**：第一幕直接以纸人巷开场，读者进入时没有任何世界观铺垫，纸扎手艺体系一次性灌入，认知负担过高。

**V2的修正**：第一卷拆为 5 个阶段——

```
Phase 0 · 日常裂缝（1章）      入门锚点：日常生活有东西不对
Phase 1 · 诡异渐进（2-5章）    低认知负荷的规则怪谈，勾住读者
Phase 2 · 召唤降临（6-12章）    玩家视角切入，双线叙事启动
Phase 3 · 纸人巷攻略战（13-28章） 第一条主线（Boss/秘境/副本）
Phase 4 · 庇护所建立（29-40章） 第一幕收束，下幕预告
```

**核心增量**：纸人巷从"开局就直接喂"变成"贯穿第一卷的秘境 Boss"。不是改内容——是**改节奏**。同样的素材，放在不同的节奏位置，读者感受完全不同。

**跨项目规则**：
> 第一卷的前5章是"读者签约期"。不要急于展示核心设定——先让读者觉得"这个世界有意思"，再慢慢打开。核心设定展示得越晚，读者接受阈值越低。

### 2.2 角色设定：三分预设，七分涌现

**来源**：这是 v7.2-v7.6 约束实验中已验证的原则在角色设计上的映射。

| 实验结论 | 在角色设计上的应用 |
|:---|:---|
| 约束不能创造好内容 | 角色卡不写具体行为，不写对话草稿 |
| 肯定式元数据比否定式禁令有效 | 角色卡写"声线/动机/边界"，不写"第X章必须做Y" |
| 类型定位是最大贡献项 | 角色卡的主要功能是"定位"——告诉导演 Agent 这个角色在场景中是什么角色 |

**实测效果**：阿疯（探路先锋）→ 写的时候只定了"声线话多+动机是把游戏当真实/不是真的想死+边界是不知道真相"。导演 Agent 写玩家线时，阿疯的"死前留遗言"和"复活后第一句'爽！'"不是预设的——是涌现出来的。涌现后回头锁定行为模式，补进角色卡。

**跨项目规则**：
> 角色卡 = 声线 + 动机 + 边界 + 卷级功能。不要写具体行为。  
> 涌现出彩了 → 回来补角色卡。不出彩 → 下章换一个人物推上去。

### 2.3 陈默的"冷"不是问题——是读者的签到卡

**发现**：读者初看陈默，会觉得他冷、功利、把玩家当工具。这不全是节奏问题——**这就是人物设计**。

读者的情感旅程应该是：
```
前期：这人怎么这么冷 → 
中期：哦原来他死过一次 → 
后期：他的冷不是他的选择，是他的代价
```

如果陈默一开始就是"热心的好人"，那他的成长弧线就断了。读者陪着他从冷到热的过程——才是这个角色真正的魅力。

**跨项目规则**：
> 主角的"缺点"不是 bug，是 feature。问题不是"怎么让他变好"，是"在正确的节奏上让读者理解他为什么是这样"。

### 2.4 苏晚的定位修正：从功能角色到核心人物

**V1的定位**：苏晚 ≈ 系统 UI。她的存在意义是"召唤玩家"。

**V2的定位修正**：苏晚是**第一卷的剧情核心**——纸人巷的钥匙、反转的触发器、玩家系统的运营层。她不是配角，她和陈默是平等的双核。

这个修正来自一个简单的追问："如果苏晚只是一个功能NPC，那主角和她之间的张力哪里来？"  
答案：如果她不只是功能——她有自己的目的（从豆包里出来）、有自己的代价（反转后永不能恢复人身）、和自己的情感（"恨过陈默，因为他来得太晚了"）——那她和陈默的每一次对话都有潜台词。

**跨项目规则**：
> 问自己三次："如果把这个角色删除，故事会怎样？"  
> 一次 → 不能删。两次 → 确认。三次 → 这个角色在故事里是有分量的。  
> 如果答案是"删了也没区别"——那不是配角，那是功能性道具。重新设计他/她。

## 三、meta 层面：这一轮沉淀了什么

### 3.1 文档定位共识（可写入 SKILL.md 设计原则）

本轮确认了 L1 元设定文件的**三个检查维度**：

> **完整性** — 该定义的都定义了吗？（有没有遗漏的关键概念？）  
> **精确性** — 定义的内容是否自洽、无矛盾？（逻辑闭环了吗？）  
> **可操作性** — 读完后能直接指导写作吗？（还是只停留在"好看"的层面？）

L1 文件的质量 = 完整性 × 精确性 × 可操作性。  
三者缺一不可。"写得好看但写的时候不知道怎么写" = 可操作性不足，需要补。

### 3.2 "写前复盘"流程（可写入 SKILL.md 或项目启动步骤）

本轮证明了**写卷纲前做一次全材料复盘**的价值：

```
Step 0: 列出所有设计材料清单
Step 1: 逐件读通（不跳读）
Step 2: 对照三个维度评分（完整性/精确性/可操作性）
Step 3: 标出 P0/P1/P2 问题
Step 4: 修复 P0 → 确认方向 → 修复 P1
Step 5: T8：进入卷纲设计
```

这个流程应该成为 **novel-agent-pro 标准工作流的前置步骤**，不是可选项。

### 3.3 涌现角色的管理机制

```
涌现（AI 写作中自然出现）
    ↓
"这个人有戏"（作者或审稿发现）
    ↓
补 L1 角色卡（三分预设：声线+动机+边界+卷级功能）
    ↓
下一章正式可用（导演 Agent 有参考依据）
    ↓
如果行为矛盾 → 追加边界拦截（只在角色卡里，不在写作 Prompt 里）
```
