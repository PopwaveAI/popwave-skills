# AG管线经验吸收报告

> 分析对象：[AI_NovelGenerator](https://github.com/Dirtfy/AI_NovelGenerator) 开源小说生成管线
> 分析版本：诡异游戏 AG版（全10章）
> 吸收日期：2026-05-18
> 吸收版本：v7.6 → v7.7 整合

---

## 一、关键发现

### 1.1 核心结论

> **问题的关键不是「注入多少信息」，而是「注入什么性质的信息」**

AG的写作Prompt注入12+个字段，远多于v7.6碎片Prompt的4字段。但AG的效果更好（"读起来更轻松"），且没有走回「约束越多写得越差」的老路。

根本原因：**AG注入的是「肯定式元数据」，旧版POP注入的是「否定式禁令」。**

两者虽然都是往Prompt里塞很多东西，但东西的性质完全不同：

| 信息性质 | 效果 | 机制 |
|:---|:---:|:---|
| 否定式约束（不要/禁止/避免） | ❌ 越加越差 | 消耗认知带宽、压制创作流、需要额外审查环节 |
| 肯定式元数据（定位/作用/节奏/伏笔） | ✅ 越多越清晰 | 建立心智模型、减负LLM的判断成本、不压制创作 |
| 肯定式素材（前文摘要/角色状态/知识库） | ✅ 越多越丰富 | 给LLM更多"可用的原料"，但不限制"怎么做" |

### 1.2 AG管线的三个结构特征

#### 特征一：元数据锚代替禁令约束

AG的每个章节写作Prompt包含：

```
├── 章节定位（事件/角色/主题）
├── 核心作用（推进/转折/揭示）
├── 悬念密度（紧凑/渐进/爆发）
├── 伏笔操作（埋设/强化/回收）
├── 转折程度（★☆☆☆☆）
├── 章节简述（一句话）
├── 字数要求
├── 核心人物/关键道具/场景地点/时间压力
└── 下一章信息（标题/定位/作用/简述）
```

**这些字段的行为逻辑是「告诉LLM目标方向」，而非「禁止LLM走错路」。** LLM读完这些元数据后，能够在脑海中建立清晰的「这一章的地图」——知道自己写的是什么类型的章节、什么节奏、什么功能、下一章往哪走。

对比旧版POP的14条禁令——每条都是"不要X"，LLM在写每一句话之前都要过一遍审查环节。禁令越多，创作流断得越碎。

#### 特征二：下一章前瞻锚定

AG的写作Prompt永远带着**下一章的信息**（标题+定位+作用+简述）。

效果：写本章结尾时，LLM不只考虑"这一章要结束"，还考虑"下一章要接什么"。因此章末不易出现"突然断掉"的感觉，情绪锚点是顺滑过渡的。

对比v7.6的碎片写作：每个碎片写完就完，桥接靠主Agent事后缝——但主Agent只缝碎片之间的过渡，不管章末到下一章的情绪锚定。这正是CH03→CH04断层问题的根因之一。

#### 特征三：全局摘要持续累积

AG每写完一章，就自动更新 `global_summary.txt`。这是一个累积的全书进展描述（≤2000字），下一次写作时直接注入。

效果：解决了"不看上一章正文"的问题。即使导演层判断失误，全局摘要也能提供基本的上下文连续性保障。

对比v7.6：导演层靠"读整章正文"做判断——理论上更完整，但执行成本高。AG的"全局摘要"方案虽然信息有损耗，但成本低、持续性强、不依赖LLM的阅读精度。

## 二、AG的局限（同样重要）

AG并不是完美的，它的问题：

| 问题 | 表现 | 根因 |
|:---|:---|:---|
| 叙事效率低 | 10章讲的内容，POP 5章就能讲完 | 每章3-4个事件 vs POP的5-6个，差距40-50% |
| 节奏平 | 缺少高频悬念钩子 | 无系统性的章末情绪设计 |
| 解释性过强 | 让角色对话来解释设定 | 无"展示vs告知"的质量控制 |
| 缺乏系统爽感 | 没有数值/成长公式/资源累积 | 无Payload网络设计 |
| 无QC层 | 质量完全依赖第一次生成 | 无回退机制 |

**AG的策略是通过降低叙事密度来换取「阅读轻松感」。** 这是一种取舍，不是绝对优势。v7.6追求的是高频爽感+信息密度，不应该为了追求"轻"而牺牲叙事效率。

## 四、保留不动（AG有但我们不学）

| AG的做法 | 不学的理由 |
|:---|:---|
| 整章一次性生成 | 已被v7.6的碎片写作证明更优 |
| 向量知识库检索 | 已被场景卡替代，检索不稳定 |
| 无QC层 | 已被v7.6的QC双Layer证明必要 |
| 降低事件密度 | v7.6的定位是高频网文，不应牺牲信息密度 |
| 长段落叙事 | 已被v7.6的短句快节奏证明更符合番茄读者 |
