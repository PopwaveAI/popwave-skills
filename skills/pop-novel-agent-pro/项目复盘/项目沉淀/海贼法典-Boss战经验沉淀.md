## 一、核心发现：锚定章机制

### 1.1 什么流程出了好结果

```
老板指定参考章（273-282 BIG MOM战役）→
全量读入上下文 →
不经过骨架设计，直接写正文 →
一次过，老板说"对劲了"
```

### 1.2 之前失败的流程

```
自己选参考章（选错了——第318章）→
只读片段（没发现那章不是战斗章）→
自己写正文 →
反复改 →
越改越差
```

### 1.3 锚定章的正确用法

| 条件 | 正确做法 | 错误做法 |
|:----|:--------|:--------|
| 谁选 | 老板指定 | 自己猜 |
| 怎么读 | 全量读入上下文 | 读片段就动笔 |
| 选什么 | 场景匹配的连续章节 | 单章或无关章节 |
| 选错时 | 自己发现不了 | 需要老板纠正 |

## 二、ch397写对了什么

### 2.1 每句话都是画面，一个信息只给一次

**对的写法（ch397）：**
> 两把黑刀相撞的瞬间，冲击波没有向四周扩散——是向上。
> 碰撞点上方的云层被冲击波撕开了一个圆形的缺口。
> 月光从缺口中倾泻下来，照亮了整片码头。

三句话，三个画面：冲击波向上→云层被撕开→月光倾泻。读者眼睛从头扫到底，不需要停下来理解。

**错的写法（之前版本）：**
> 一道笔直的切口从威廉左侧延伸出去，切穿了石板，切穿了碎石层，一直延伸到码头边缘的阴影中才消失。边缘平滑，没有裂纹——像地面自己长出了一道裂缝。

三句话在说同一件事：切口很平滑。读者在同一行信息上停了三次。

**规则：写完一句话，问"这句话给新信息了吗"。没给就删掉。**

### 2.2 每一刀都有环境反馈

**对的写法（ch397）：**
```
威廉出刀 → 冲击波撕云层 → 地面碎裂 → 观战者后退
米霍克出刀 → 撞击声像地底碎裂 → 海面产生波纹
第三刀 → 米霍克从格挡转为对攻
连续碰撞 → 码头全碎 → 岸壁崩塌
```

每一刀后面跟着一个后果——劈下去不是"被接住了"，是"这一刀改变了什么"。

**错的写法（之前版本）：**
```
威廉第一刀 → 米霍克接住
威廉第二刀 → 米霍克再接住
威廉第三刀 → 米霍克再接住
```

七刀全被接住，反馈是"米霍克姿态不变"。读者读了七刀得到同一个信息。

**规则：如果连续三刀都不改变战场状态，这三刀不应该出现在正文里。**

### 2.3 战场空间持续降级

**ch397的空间变化：**
```
码头地面碎裂 → 石板全碎 → 岸壁崩塌 → 码头沉入海中 → 海岸线被切开
```

同一个空间越打越严重，读者感受到"再打下去整座岛都要没了"。

**之前版本的码头上始终只有"一道切口"——战斗结束了码头还在。读者不会觉得"这一战很重"。**

**规则：战斗不能打在同一个状态的地面上。战场自己就在讲故事。**

### 2.4 观战者反应是刻度尺

**ch397的观战者递进：**
```
"退后——" → 所有人后退
"这两个人疯了。"
多弗朗明哥前倾，抓住船舷
甚平右手从胸前放下
克洛克达尔雪茄滑落
```

老沙雪茄滑落比甚平放手重，甚平放手比多弗前重视觉重。不是随机分配——是递进的。读者通过他们的身体幅度判断"现在到什么程度了"。

**规则：观战者的反应要有梯度。最强的反应留给最强的场面。**

### 2.5 不解释画面

ch397没有这些句子：
- "这意味着威廉的蒸气加速处于极限状态"
- "他意识到米霍克的见闻色提前锁定了他的位置"
- "这不是反应，是预判"

全是画面直接告诉读者。如果画面本身不够传达信息，就去加强画面，而不是在画面后面加一句解释。

**规则：不出现在画面里的东西，不出现在文字里。**

## 三、核心变量：参考文本在上下文里

### 3.1 为什么ch397突然写对了

不是突然"会写了"。是273-282 BIG MOM战役的完整节奏被加载进了上下文：

> 每一句的结构、每个攻防循环的长度、环境破坏的递进方式——全部被输出对齐到了输入信号的节奏上。

之前写ch394时上下文里只有"自己写的废稿+设计文档"，所以输出是对已有文本的微调。不是创作，是修改。

**规则：写战斗之前，上下文里必须有一份用户认可的、场景匹配的战斗文本。不靠记忆写。**

### 3.2 这个机制的局限性

| 局限 | 表现 | 解决办法 |
|:----|:-----|:--------|
| 自己选不对参考 | 自己选的章节不是战斗章 | 老板指定参考 |
| 读不完整 | 读了三段就觉得懂了 | 全量读，不截断 |
| 跨类型迁移失败 | 海贼的写法不能直接套诡异 | 需要该类型自己的参考 |
| 无法自检 | 写了烂章也不自知 | 老板反馈是唯一质量信号 |

## 四、与事实骨架的关系

ch397写的时候没有读事实骨架。但回合设计（"第3轮·过热带·米霍克全力回应"）和能级升温（七武海高段→四皇级门槛）已经在对话历史里残留——它不是主动加载的，但它在上下文里。

**事实骨架的真正作用：**
- 当没有参考文本时，防止写出烂文（保险，不是发动机）
- 当有参考文本时，提供结构精度和升温标记（与参考不冲突）

**规则：参考文本提供画面和节奏，骨架提供结构精度，互不冲突。如果有参考，参考优先。如果没有参考，走骨架管线。**

## 五、可以带走的写作规则（战斗篇）

1. **一个信息只给一次。** 写完一句问自己：这句给新信息了吗？
2. **每一击必须改变环境。** 一刀下去战场状态没变，这刀不写。
3. **战场状态持续降级。** 完整→损坏→崩塌→消失。
4. **观战者反应递进。** 最重的反应留给最强的场面。
5. **不解释画面。** 画面传递不了的，不是靠解释补——是靠换一个更准确的画面。
6. **写之前读参考。** 不靠记忆写战斗。
7. **参考由老板指定。** 自己选会选错。
