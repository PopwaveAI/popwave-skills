## 一、四轮对照实验（最重要的经验来源）

这是整个系统方法论层面的核心资产。四轮实验验证了一个铁律：**约束越多写得越差。**

### 实验设计

```
对照组：v5（52章人工反馈迭代，均分8.8）
实验组A：v7.1（仅4条红线，无章纲，无类型定位）
实验组B：v7.2旧版（14条约束，含章纲）
实验组C：v7.2新版（4条红线 + 章纲 + 类型定位"网文爽文"）
```

### 实验结果

| 版本 | 均分 | 说明 |
|:---|:---:|:---|
| v5（基准） | 8.8 | 52章迭代累积 |
| v7.2新版（章纲+类型） | **7.1** | 冷启动第一稿，已接近可用 |
| v7.1（仅红线） | 6.3 | 缺方向，但比加约束好 |
| v7.2旧版（14条约束） | **4.6** | 约束越多写得越差 |

### 逐项发现的因果关系

| 变量 | 效果 | 影响幅度 |
|:---|:---|:---:|
| 添加"网文爽文"类型定位 | 爽点+2.2，动作节奏+2.8 | **最大贡献项** |
| 添加章纲事件序列（8-12条） | 信息方向+0.8 | 稳定方向 |
| 添加14条详细约束 | 质量 -2.5 | **负面** |
| 移除文风锚点片段 | 质量稳定，未下降 | 锚定原文不是必需品 |
| 保留4条基本红线 | 防止致命错误 | 必要但不充分 |

### "14条约束"为什么那么差

1. **注意力稀释**：子Agent同时处理"避免不是A是B"+"避免比喻"+"维持人物口吻"+"对齐节奏"——顾此失彼
2. **合规心态**：子Agent优先确保"不违规"而非"写得好"
3. **句式僵硬**：约束互斥（要短句←→要有细节），模型无法同时满足

源文件：[灰骑士]`v5_vs_v7.1_对比分析.md`、`v5_vs_v7.2_对比分析.md`、`v5_vs_v7.2final_对比分析.md`

## 二、锚定系统的问题

灰骑士的锚定系统（scene_fragments.db 检索）暴露了严重问题：

### 已知技术问题（来自09-已知问题追踪）

| ID | 问题 | 严重度 | 状态 |
|:---|:---|:---:|:---:|
| BUG-001 | Write vs Python换行不一致 → 格式崩溃 | 阻塞 | 已规避 |
| BUG-002 | split/join 吞 `\n\n` → 单行文件 | 阻塞 | 已规避 |
| BUG-003 | redline_scan --fix 破坏已排版章节 | 阻塞 | 已规避 |
| BUG-004 | 子Agent幻觉——战锤人名泄露 | 阻塞 | 未处理 |
| BUG-005 | 玄幻风泄露——川字/水潭（灰骑士专属） | 阻塞 | 部分修复 |
| BUG-006 | 双逗号污染 `，，` | 影响体验 | 已规避 |
| BUG-007 | 自动化子Agent单行输出 | 阻塞 | 未处理 |

### 锚定检索的根本缺陷

1. **同场景≠同节奏**：查到的"战斗"片段可能是慢热型，不适用于本段"快节奏"
2. **跨书匹配不稳定**：深渊主宰的句法风格迁移到灰骑士——感觉不对
3. **片段质量波动**：锚定库中的片段未经质检，良莠不齐

**这些缺陷直接导致了 v7.5 场景卡方案的诞生，也间接导致了 v7.6 的"场景卡=节奏参照样本"修正。**

### 从锚定到场景卡的演进逻辑

```
锚定片段（scene_fragments.db）
  跨项目共享，从参考书提取 → 质量不稳定
    ↓
场景卡 v7.5（可调用资产）
  项目独有，从自己已写的高质量段提取 → 但子Agent复制粘贴输出僵硬
    ↓
场景卡 v7.6（节奏参照样本）
  项目独有，走5问入库质检 → 子Agent模仿节奏后自建 → 灵活
```

## 三、reader-qa 审稿发现

### 灰骑士前5章审稿（SOP-R）

**6维度评分**：
| 维度 | 评分 |
|:---|:---:|
| 总体 | 7.5 |
| 战斗 | 9.0（强项） |
| 内心独白 | 6.5（弱项） |

**核心问题**：
1. **节奏拖沓**：前5章信息密度过高，读者需要"停下解码"
2. **章末钩子失效**：多章节末尾以"旁白总结"收束，不是事件断裂
3. **信息动机弱**：读者知道"主角要做什么"，但不知道"为什么现在做"
4. **内心独白语气不统一**：时而战锤士兵、时而网文男主

源文件：`skill-reader-qa/reviews/greyknights-ch001-005-20260502.md`

### 灰骑士5-10章复盘（SOP-P）

**发现**：
- 跨章情绪曲线图的张力维持不足
- 第5-10章缺乏"危机升级"曲线——主角一直在打杂兵
- 信息获取动机偏弱——没有任何"迟了就来不及了"的紧迫感

源文件：`skill-reader-qa/retro/retro-20260502-greyknights-ch5-10.md`

### 审稿模式的有效性验证

```
SOP-R（单章审稿）：有效——6维度评分+问题清单
SOP-S（存档）：有效——历史数据可查
SOP-C（还原度校验）：有效——发现"声音偏离"的量化方法
SOP-P（复盘）：最有用——跨章的趋势发现（如"连续5章无危机升级"）
```
