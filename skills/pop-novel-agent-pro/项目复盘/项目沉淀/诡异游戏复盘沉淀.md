## 一、根因发现链

### 第一条链：展示vs告知（最大单一发现）

```
v2正文均分5.5 vs 场景卡均分8.5-9.0 vs 诡异人生均分9.0
    ↓
同样的大纲、同样的设定——差距在与"怎么写"而非"写什么"
    ↓
场景卡（自己写的、质量高）vs 正文（同一批人写的、质量低）
    ↓
根因：正文中没执行场景卡的"展示"技法
    ↓
解决方案：把场景卡的写法注入正文写作流程 → 碎片写作方案
```

源文件：`001-v2正文根因定位.md` + `002-横向对比分析.md` + `003-场景碎片写作方案.md`

### 第二条链：约束减负

```
第一步诊断方向错误："加约束控制质量"
    ↓
用户指正："减才是正路"
    ↓
灰骑士四轮实验验证：14条约束版均分4.6
    ↓
迁移到诡异游戏：v2 (5.5) → v3 (7.0)，+1.5进步
```

源文件：`001-v2正文根因定位.md`（含用户指正记录）

### 第三条链：对齐度≠质量（反直觉发现）

```
v2 对齐度 100%（总分5.5）
v3 对齐度 72.5%（总分7.0）
    ↓
完美执行大纲≠好正文
    ↓
"对"是必要条件，不是充分条件
    ↓
写作质量来自场景卡级别的"展示"执行，不是大纲对齐
```

源文件：`005-意图对齐度分析.md`

## 二、竞品对标关键结论（诡异人生）

### 核心差距

| 维度 | 诡异游戏 v2 | 诡异人生 | 差距 |
|:---|:---:|:---:|:---:|
| 信息密度（首章新概念） | 13个 | 3个 | **4倍** |
| 信息呈现（五维评分） | 4.5 | 9.0 | -4.5 |
| 节奏密度 | 高但拖沓 | 高且紧凑 | — |
| 因果密度 | 1/3 | 1（基准） | -2/3 |
| 对话占比 | 15% | >40% | -25% |

### 诡异人生的设计手法（从96个片段中提取）

1. **悬念递进结构**：每10章一个生存/规则/能力/代价周期
2. **信息差设计**：读者始终比主角知道多一点点——焦虑驱动
3. **对话密度**：对话占比 >40%，旁白不超60%
4. **能力代价绑定**：每次变强必绑代价（鬼手容纳→寿命缩短）
5. **快慢交替**：紧张章→喘息章→紧张章，不允许连续3章紧张

源文件：`我的诡异人生-设计手法分析报告.md` + `诡异游戏vs诡异人生_前10章深度对比PK.md`

### 这个PK为什么重要

这是整个项目最有力的外部校准。诡异人生是同一赛道的成熟作品，它的评分（9.0）是系统的**天花板参照**。v2（5.5）→ v3（7.0）→ v7.6（CH01实测通过）——每一步都能在这个参照系中找到具体差距。

## 三、场景卡实验关键记录

### 场景卡注入实验（CH06 封字符碎裂段）

```
v6原文：19行密集感官描写，多用否定句式"不是…是…"
注入版：用诡异人生「战斗」场景卡节奏重写
结果：注入版信息释放更清晰，节奏更紧凑
```

实验结论：**高质量场景卡 → 节奏参照 → 正文质量提升。** 但 v7.5 走过头了（把场景卡当可调用资产，子Agent复制粘贴）。

源文件：`场景对照卡注入实验记录.md`

### 场景卡的正确用法（v7.6 修正）

```
v7.5错误：场景卡=可调用资产，子Agent复制
v7.6修正：场景卡=节奏参照样本，子Agent模仿节奏后自建内容

区别：
- v7.5：子Agent读→复制→输出 → 僵硬
- v7.6：子Agent读→模仿节奏→基于情节自建→输出 → 灵活
```

### CH02/CH03 A/B测试

| Prompt写法 | 子Agent输出 | 结论 |
|:---|:---|:---:|
| "读者感觉到玩家上线了" | 信息缺失、隐晦 | ❌ 模糊 |
| "读者明确知道①陈默主动拉人②苏晚牵线③3名玩家已进入" | 所有信息完整交付 | ✅ 精准 |

**铁律验证**："读者明确知道X" > "读者感觉到X"

### CH03→CH04修复

```
问题：CH03结尾渲染"荒地有危险"，CH04开场写"拔草"
根因：导演Agent只读了大纲，没读上一章正文
修复：导演Agent先读上一章正文 → 发现结尾情绪在紧张锚点
       → CH04开场先处理读者预期（不是直接"拔草"）
```

**铁律验证**：上一章正文权重 > 大纲权重（大纲是静态计划，读者情绪是动态变量）。

## 五、三棍的起源（诡异游戏贡献）

三棍并不是PRD凭空发明的。它来自诡异游戏的两个前置探索：

1. **StoryOps沙盒推演**（005-causal-engine-十轮推演复盘.md）
   - 拆解诡异人生payload网络：66节点、113因果边
   - 发现本作因果密度不到1/3
   - 尝试用"崩坏度/应对力"做节奏推演

2. **诡异人生的"A好+B好"模式**
   - 分解后发现：诡异人生的好不是单一原因
   - 是"A好（因果密度高）+ B好（对话占比合理）+ C好（信息差控制）"的叠加
   - 三棍试图把"B好+C好"定量化

**所以三棍的动机是合理的**（想把节奏量化），**但方法是否有效尚未验证**。

详见 `三棍调度器争议.md`。
