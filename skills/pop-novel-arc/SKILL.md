# Pop Writer V3 Arc

arc 负责事后复盘：从已经发生的正文和运行日志里识别真实剧情线、设定变化和下一单元种子。

## 输入

- 本单元幕纲和章节运行日志。
- 本单元正文。
- 当前设定账本。
- 当前活记忆。

## 执行

1. 读 `steps/step-1-macro-check.md` 检查单元完成度。
2. 读 `steps/step-2-setting-prune.md` 修剪并冻结设定。
3. 读 `steps/step-3-幕纲-update.md` 回填真实偏差。
4. 读 `steps/step-4-l3-card-distill.md` 沉淀跨单元剧情线。
5. 读 `steps/step-5-memory-compress.md` 压缩活记忆。
6. 输出本单元对 library 的反向建议：哪些任务索引应新增、哪些案例消化有效、哪些爽点证据失效。

输出按 `templates/弧线校准报告-模板.md` 和 `templates/L3卡-模板.md`。

## 红线

- 不把预想剧情线强塞进正文已经发生的事实。
- 不冻结一次性细节。
- 不保留历史复盘资料。
- 不替 plot 设计下一单元，只提供可被 plot 使用的新状态和新种子。
- 不直接修改公共 library，除非用户明确要求；只输出可审核的任务索引建议。
