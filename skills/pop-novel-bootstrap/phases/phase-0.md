# Phase 0：L0 灵魂层对齐

## 这份文档存在的意义

写一本小说之前，最危险的事不是设定没想好——是**你不知道自己为什么要写**。

PRD（产品定位文档）存在的意义是：在动笔之前回答三个所有人都会问的问题——为什么是你写？读者为什么看？这本书到底是什么？

**没有 PRD 的后果**：写到 50 章发现情绪基调飘了、爽点类型变了、读者画像模糊了。所有决策没有锚点可回溯。

---

## 必须回答的问题

1. **为什么是我来写这本书？** — 只有你能写的东西是什么？（你的独特经验/执念/知识储备/情感连接）
2. **读者为什么非看不可？** — 这本书提供了什么其他书给不了的体验？（不可替代的情感/认知/爽点体验）
3. **这本书的核心情绪是什么？** — 读完第一个感觉。（热血/治愈/悬疑/爽/虐/幽默……不是"都有"，是**排第一的那个**）
4. **目标读者是谁？** — 他们的年龄、性别、阅读场景、弃书阈值。
5. **这本书和同类型 Top 20 最核心的差异在哪？** — 不是"我更好"，是"我不同"。

---

## 执行步骤

### 1. 灵魂三问
依次回答 Q1、Q2、Q3。每个问题不加限制地发散回答，再精炼成一句话。

### 2. 产品定位卡（PRD）
输出 `00-原始设定/L0-产品层/PRD.md`，包含：
- 产品名称（书名）、一句话卖点、目标读者画像
- 核心体验承诺（读者读完后记得什么）
- 差异化优势（和同类作品的区别）
- 风险/弱点预判（诚实面对问题）

> PRD 是全书最高优先级约束文件。后续所有设定决策如果和 PRD 冲突，以 PRD 为准。

### 3. 核心创意压力测试
| 测试 | 标准 | 通过条件 |
|:-----|:-----|:---------|
| 电梯测试 | 30 秒内向陌生人讲清楚这本书 | 对方能复述核心设定和爽点 |
| 长篇拉力测试 | 核心创意能支撑 200 万字以上 | 有明确的扩展路径（不是"写不动了就完结"） |
| 平台适配测试 | 创意与目标平台读者画像匹配 | 在平台找到≥3 本同类爆款作为参照 |
| 差异化测试 | 与 Top 20 同类作品的核心差异 | 至少有 1 个"别人没有"的元素 |

全部通过 → 启动 Phase 0.3。任一不通过 → 回到灵魂三问重新对齐。
