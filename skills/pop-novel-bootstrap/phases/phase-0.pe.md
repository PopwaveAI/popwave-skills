# Phase 0：故事引擎设计

## 前置条件
用户已提供至少：一个故事想象（场景/概念/世界观/角色片段，不拘）。

## 加载参考
先加载 `phases/phase-0.ref.md`，了解 story-engine.yaml 的 schema 和追问方法。

## 核心原则

Phase 0 不是让用户填空，是 agent 接住用户的想象，帮用户一起完善它。

用户说"我想写灰骑士穿越博德之门3"——这是一个画面，不是一个填空题。
Agent 的任务是：
1. 接住它
2. 追问细节
3. 锁定方向
4. 产出结构化数据

## 执行步骤

### 1. 接住 + 追问
接住用户的话，不要转换成三个抽象问题。用具体的追问帮用户展开：

| 用户说了 | 不要问 | 要问 |
|:--------|:-------|:-----|
| "灰骑士穿越博德之门3" | "核心情绪是什么？" | "灰骑士的哪些特质你想保留？灵能？审判庭？忠诚？" |
| "修仙世界" | "读者为什么非看不可？" | "这个修仙世界的独特点是什么？修炼体系有什么不同？" |
| "末世丧尸" | "为什么是你写？" | "你想写生存本身，还是人与人之间的博弈？" |

追问方向（不全部需要，随对话自然流淌）：
- 场景追问："这个世界最关键的一个画面是什么？"
- 冲突追问："主角面临的最大困难是什么？来自哪里？"
- 角色追问："主角是个什么样的人？他想做什么？"
- 基调追问："这个故事的整体氛围是黑暗还是轻松？"

追问 2-3 轮，用户确认基本方向清晰了，进入第 2 步。

### 2. 产出 story-engine.yaml
根据追问结果，结构化输出 `00-原始设定/L0-产品层/story-engine.yaml`：

```yaml
engine:
  version: 1
  # 一句话：一个XX的人，想做XX，但XX挡着
  core_premise: "一个灰骑士审判官穿越到博德之门3的世界，发现混沌的污染已经渗入费伦，他必须在被本地势力视为疯子之前，组织起对抗混沌四神的防线"

  # 世界的时空和力量定位
  world_anchor:
    time: "动荡之年/博德之门3时期"
    space: "费伦大陆"
    power_tier: "超凡→破格"  # 凡人 / 超凡 / 破格 / 神话

  # 核心冲突类型（可多选）
  core_conflict_type:
    - 生存        # 主角面临持续的生命威胁
    - 战斗        # 物理/魔法对抗是核心驱动力

  # 读者承诺：读者每5章靠谱到什么
  reader_promise: "每5章至少有一次正面战斗或关键情报揭露"

  # 主角
  protagonist:
    one_line: "一个坚定的灰骑士审判官，穿越后失去了大部分力量，但保留了灵能和对混沌的认知"
    driving_force: "他要阻止混沌四神在这个世界的入侵，即使所有人都不相信他"

  # 读者画像（直接产出，不再绕道 PRD）
  reader_profile:
    demographic:
      age_range: "18-30"
      reading_scenario: "睡前/通勤"
    preferences:
      - 战斗描写详细
      - 设定严谨
      - 主角有信念
    drop_threshold:
      - 连续2章无战斗则弃书
```

### 3. story-engine 确认
向用户展示 story-engine.yaml，用户确认后锁定。
- 用户说"对，就是这感觉" → 锁定，进下一步
- 用户说"不太对" → 回到第 1 步追问环节

## 检查清单
