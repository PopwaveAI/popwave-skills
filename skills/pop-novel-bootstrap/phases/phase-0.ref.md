# Phase 0 参考：story-engine.yaml 结构与追问方法

## story-engine.yaml schema

```yaml
engine:
  version: 1

  # 一句话：一个XX的人，想做XX，但XX挡着
  core_premise: ""

  # 世界的时空和力量定位
  world_anchor:
    time: ""     # 什么时代/纪元
    space: ""    # 什么地域/位面
    power_tier: "凡人→超凡→破格→神话"  # 故事主要落在这个范围的哪一档

  # 核心冲突类型（可多选，不超过3个）
  core_conflict_type:
    # 可选：战斗 / 解谜 / 生存 / 种田 / 复仇 / 守护 / 探索

  # 读者承诺：读者每5章靠谱到什么
  reader_promise: ""

  # 主角
  protagonist:
    one_line: ""        # 一句话描述
    driving_force: ""   # 他想什么/怕什么/要什么

  # 读者画像（直接产出，下游plot/writer/QC共享）
  reader_profile:
    demographic:
      age_range: ""
      reading_scenario: ""
    preferences:          # 爽点敏感度，列表
    drop_threshold:       # 什么情况下读者会弃书，列表
```

## 后附说明

### 追问方向参考

| 追问方向 | 示例问题 |
|:---------|:---------|
| 场景 | "这个世界最关键的一个画面是什么？" |
| 冲突 | "主角面临的最大困难是什么？来自哪里？" |
| 角色 | "主角是个什么样的人？他想做什么？怕什么？" |
| 基调 | "这个故事的整体氛围是黑暗还是轻松？" |
| 设定边界 | "这个世界跟现实世界最大的不同是什么？" |
| 读者预期 | "读者看完前三章，应该记住什么？" |

### core_premise 写法

不要在 Phase 0 追求"一句话卖点"级别的精炼。用户刚说出来的想法应该是自然语言的：

- ✅ "一个灰骑士穿越到博德之门3，混沌四神要入侵费伦，他得组织防线"
- ❌ "混乱与秩序碰撞下的末世救赎史诗"

追问 2-3 轮后整理成 `一个XX的人，想做XX，但XX挡着` 格式即可。

### power_tier 选择

| 层级 | 读者体感 | 示例 |
|:-----|:---------|:-----|
| 凡人 | 主角跟普通人差不多强 | 低魔/历史/悬疑 |
| 超凡 | 主角有明显超越常人的能力 | 修仙中前期/低魔/异能 |
| 破格 | 主角能影响世界规则 | 修仙后期/神灵 |
| 神话 | 概念层面操作 | 完结收尾阶段 |

### reader_profile

不要猜"年龄/性别/阅读场景"，从故事本身推断。

- 主角是坚韧型战斗角色 → 偏好战斗描写 → 弃书阈值：连续无战斗
- 世界观有独特设定 → 偏好设定严谨 → 弃书阈值：设定矛盾
- 节奏偏慢 → 偏好慢热 → 弃书阈值：长期无推进

### 与旧版本的差异

- 不再有 PRD 六要素——"风险预判/差异化优势"等商业文档内容不属于这里
- 不再有灵魂三问——Q1/Q2 是作者自我安慰，无下游消费方
- 不再有压力测试——"30秒讲清楚/能撑200万字"无人能判断
- engine.yaml 各字段可被下游管线直接消费（plot/writer/QC）
