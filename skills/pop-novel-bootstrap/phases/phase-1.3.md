# Phase 1.3：L1 交叉关联 enrichment ★ 自动执行 | 禁止跳过

## 这份文档存在的意义

六篇 L1 各自展开后最容易出的问题：**各说各的**。

世界观底层逻辑说"魔法来自星界"，表层规则说"法师用魔网施法"，两个文件之间没有交叉引用。Agent 写第一章时参考 02-表层规则，它不知道 01-底层逻辑里还有星界的设定。

交叉关联存在的意义是：**确保六篇设定是一个有机整体，而不是六份独立文档**。每一对文件之间必须有明确的关系声明。

**没有交叉关联的后果**：正文中出现设定冲突——比如角色能做的事被 L1-04 金手指允许，但在 L1-02 表层规则里说不通。

---

## 必须回答的问题

1. **每对 L1 文件之间的关联说清楚了吗？** — 14 对关联，逐对检查。每一对都必须能回答"文件 A 的核心陈述在文件 B 里是如何体现/约束/扩展的？"
2. **有没有冲突？** — 如果两篇 L1 对同一个概念有不同的描述，是冲突还是互补？
3. **有没有孤立的设定？** — 有没有某个设定只在一篇文件里出现，在其他文件里完全没有呼应？

---

## 执行步骤

### 1. 关联矩阵校准
逐对检查每两篇 L1 文件之间的关联：

| 文件A | 文件B | 必须存在的关联 |
|-------|-------|--------------|
| 01-底层逻辑 | 02-表层规则 | 表层规则是底层逻辑的具体表现？ |
| 01-底层逻辑 | 03-种族 | 种族的生理根基与底层法则一致？ |
| 01-底层逻辑 | 04-金手指 | 金手指来源在底层逻辑中有位置？ |
| 01-底层逻辑 | 05-势力 | 各势力力量来源与底层法则兼容？ |
| 01-底层逻辑 | 06-物品 | 物品运作不违反底层法则？ |
| 02-表层规则 | 03-种族 | 种族的日常生活在表层规则中有体现？ |
| 02-表层规则 | 04-金手指 | 金手指使用受表层规则约束？ |
| 02-表层规则 | 05-势力 | 各势力活动遵循表层规则？ |
| 02-表层规则 | 06-物品 | 物品获取/使用符合表层规则？ |
| 03-种族 | 05-势力 | 种族分布与势力版图吻合？ |
| 03-种族 | 06-物品 | 物品产地/用途与种族特征匹配？ |
| 04-金手指 | 05-势力 | 金手指使用影响势力关系？ |
| 04-金手指 | 06-物品 | 金手指能否/如何与关键物品互动？ |
| 05-势力 | 06-物品 | 势力控制哪些关键资源/物品？ |

### 2. 缺失关联补写
发现缺失关联 → 在对应文档中添加交叉引用段落。
格式：`【关联：XX文件】—— 这里描述的XX，在《YY文件》中体现为ZZ`
每篇文件至少增加 2 条指向其他文件的交叉引用。

### 3. 一致性冲突修复
发现冲突 → 标记为冲突。
冲突≤2 处 → 直接在本文修复并记录。
冲突>2 处 → 标记为系统性冲突，退回 Phase 0 重新对齐。

**产出**：
- 所有 L1 文件增加交叉引用标注
- `00-原始设定/L1-元设定层/_交叉引用记录.md`
