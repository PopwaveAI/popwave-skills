# Phase 1.3：L1 交叉关联

## 前置条件
Phase 1.2 全部通过。

## 核心原则

不在每篇 L1 文件中插入 `【关联：XX】` 污染性标注。改为产出独立记录文件，记录三个维度的交叉检查结果。

## 执行步骤

### 1. 交叉检查
逐对检查六篇 L1 之间是否存在矛盾或不一致。重点关注：

| 检查对 | 关注点 |
|:-------|:-------|
| 01↔02 | 世界法则和力量体系一致？02中的境界突破条件与01中设定的法则冲突？ |
| 01↔04 | 物种的分布符合世界蓝图？04中的种族出现在01设定的正确时空？ |
| 02↔04 | 种族的天赋/血脉与力量体系一致？ |
| 03↔01/02/04/05/06 | 大事件对这个世界产生了什么影响？ |
| 05↔01/02 | 势力掌控的资源/地盘与01/02一致？ |

### 2. 产出 `_交叉引用记录.md`

记录三类信息：

```
## 已确认一致
- [设定1] ↔ [设定2] — 一致，无矛盾

## 矛盾点（需修复）
- [矛盾描述] — 涉及文件：[文件1] + [文件2] — 建议修复：[方向]

## 待确认
- [超出当前幕范围的设定] — 需在后续幕中验证
```

### 3. 修复
- 矛盾点 ≤ 3 处 → 直接修复对应 L1 文件
- 矛盾点 > 3 处 → 退回 Phase 1.2 重推
