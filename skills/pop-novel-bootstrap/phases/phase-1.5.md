# Phase 1.5：世界稳定性检验

## 这份文档存在的意义

设定写完很容易自我感觉良好。世界稳定性检验存在的意义是：**在进入大纲和正文之前，先把设定放到一起"碰撞"一下**，看有没有矛盾、有没有漏洞、有没有写死自己的陷阱。

**没有稳定性检验的后果**：写到第 100 章发现力量体系有逻辑漏洞——得回去改设定，但读者已经接受了之前的版本，改不了了。

---

## 必须回答的问题

1. **L1 设定之间有没有不可修复的矛盾？** — 力量体系是否与世界观底层逻辑一致？种族设定是否与势力关系一致？物品体系是否与力量体系兼容？
2. **每个设定的"为什么"能回溯到 PRD 吗？** — 如果一个设定存在，但它不服务于 PRD 的任何爽点，为什么要存在？
3. **有没有"作者知道但角色不应该知道"的信息暴露？** — 这是新手最常犯的错误：角色知道了他不应该知道的事情，只因为作者想让他知道。
4. **当前设定是否限制了未来剧情发展的可能性？** — 有没有某个设定写得太死，导致后续卷写不下去了？
5. **预留了后续卷的"新发现/新规则"空间了吗？** — 第一卷的设定不能穷尽所有可能性——要留一些"读者以为是这样，第二卷才发现完全不是"的空间。

---

## 执行步骤

### 1. 一致性检查
交叉验证所有 L1 文档：
- 力量体系是否与世界观底层逻辑一致？
- 种族设定是否与势力关系一致？
- 物品体系是否与力量体系/经济体系一致？

### 2. 逻辑闭环检查
- 每个设定的「为什么」是否可回溯至 L0 PRD？
- 是否存在无法解释的矛盾/漏洞？
- 是否存在「作者知道但角色不应该知道」的信息暴露？

### 3. 可扩展性检查
- 当前设定是否限制了未来剧情发展的可能性？
- 是否存在「写死自己」的设定陷阱？
- 是否预留了后续卷的「新发现/新规则」空间？

### 4. 外部一致性验证（可选）
- 查阅同类型经典作品的设定处理方式作为参照
- 确保差异点是有意识的设计而非遗漏

**产出**：
- `00-原始设定/world-stability-check.md`
- 修复方案（如需修改 L1 则标记变更）
