# Phase 1.5：世界稳定性检验

## 前置条件
Phase 1.3 通过。

## 执行步骤

逐项检查以下 checklist。全部通过 → 进 Phase 3。任意一项不通过 → 修复对应 L1 文件。

## 稳定性 checklist

```
□ 力量体系与世界蓝图矛盾？
   检查：02中的境界突破条件，是否与01中设定的世界法则冲突？
   示例：如果01说"这个世界灵气稀薄"，但02说"人人都可筑基"→ 矛盾

□ 历史事件中的因果链完整？
   检查：03中的大事件是否有明确的触发→发展→结果？
   示例：如果03说"诸神陨落"，但01中没有任何证据显示神曾经存在 → 断裂

□ 势力之间的关系有张力？
   检查：05中的势力不是"各管各的"，而是有互动/对抗
   示例：两个势力在同一块地盘上却没有冲突 → 无张力

□ 物种的分布符合世界蓝图？
   检查：04中的种族知道现在01设定的正确时空
   示例：如果01说"北域终年严寒"，但04说"北域的精灵族住在热带雨林" → 矛盾

□ [主角] 特殊能力在该世界中存在理论支撑？
   检查：主角的独特能力在01/02中有对应的规则或例外说明
   示例：遮天的荒古圣体被天地压制 —— 这是有理论支撑的（天地法则对特殊体质有反应）
        如果主角的挂毫无来源 → 断裂

□ 是否有"一次性设定陷阱"？
   检查：是否存在某个设定在L1中出现一次、为剧情服务一次、然后用不再用？
   示例："地底有一条通往另一个世界的河"——如果主角只用一次且再无影响 → 陷阱

□ 力量体系是否有明确的天花板和突破路径？
   检查：02的力量天花板和突破条件是否清晰？
   示例：遮天的大帝→红尘仙路径清晰；如果只说"主角很强但没有最终目标" → 模糊
```

## 产出
- `world-stability-check.md`（逐项记录检查结果 + 修复方案）
- 修复方案应精确到"修改 L1 哪个文件的哪个子字段"
