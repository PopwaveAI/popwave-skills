# Phase 3：项目骨架创建

## 这份文档存在的意义

前面所有的设定和设计，最后需要有一个**具体的、可执行的工程骨架**。

项目骨架不是"写文档"——是**把前面的所有产出变成可以直接开始写作的基础设施**：目录结构、总控文件、角色卡、数据库、追踪面板。

**没有项目骨架的后果**：写正文时找不到参考文件在哪、依赖关系混乱、修改了设定但没人知道改到哪了。

---

## 必须回答的问题

1. **所有前面阶段产出的文件是否已经归位？** — 目录结构是否和 Phase 2 产出的地理/势力/角色等一一匹配？
2. **"单点真理源"建好了吗？** — project.yaml（元配置）、constitution.yaml（宪法约束）、chapter-state.yaml（章状态追踪）这三个文件是否已创建且已填入初始数据？
3. **主角和重要配角的角色卡完成了吗？** — 不能让正文都开始写了还不知道配角长什么样。

---

## 执行步骤

### 1. 目录结构创建
按照 `references/产出目录结构.md` 创建完整目录树。

### 2. 总控配置 → `00-总控/project.yaml`
- 项目元信息（书名/作者/平台/开始日期/目标字数）
- Phase 进度标记
- 文件索引（所有已创建文件的路径清单）
- reader_profile

### 3. 宪法文件 → `constitution.yaml`
从 PRD 提取的不可违背的创作原则：
- 核心情绪基调（不能被任何剧情"带偏"）
- 禁止清单（什么不能写——比如不杀友军、不虐主过度等）
- 这条文档全流程通用，任何 agent 写入前都需对照

### 4. 章状态文件 → `00-总控/chapter-state.yaml`
写作时的"单点真理源"。写前读取当前状态，写后自动更新。

### 5. 角色卡创建 → `02-大纲/L3-角色层/{角色名}.md`
- 主角角色卡：满配（含背景/动机/成长弧线/关系网/金手指联动）
- 重要配角：中配（含背景/动机/与主角关系）
- 龙套角色：不创建角色卡，文中动态塑造

### 6. 数据库初始化 → `04-数据库/novel.db`
表结构：projects / chapters / characters / items / events / notes

### 7. 追踪面板 → `00-总控/project-status.html`
HTML 进度追踪面板（进度条 + 文件清单 + Phase 状态）
