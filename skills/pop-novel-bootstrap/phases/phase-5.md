# Phase 5：数值体系构建

## 这份文档存在的意义

一本书没有好的力量体系，就像一座建筑没有承重墙。读者看不到，但每一场战斗、每一次升级、每一个"打不过就跑"的决策，都在依赖这个系统的支撑。

力量体系存在的意义不是"让小说看起来有设定"——是**让读者在每一场战斗中直观感受到"主角现在有多强、敌人有多强、这场战斗有多少胜算"**。没有这个参照系，战斗就变成了"作者说打得过就打，打不过就跑"。

**没有力量体系的后果**：
- 前期打刀虫艰难，50章后打刀虫依然艰难——读者感觉不到成长
- 前期一刀秒小怪，100章后一刀秒Boss——读者感觉不到挑战
- Boss战没有"悬"的感觉——读者反正知道作者不会让主角输
- 升级变成了"数字变大了"，但读者不觉得主角真的变强了

## 经典力量体系的共同特征

见 `references/网文力量体系大全.md` — 从凡人修仙、遮天、斗破、盘龙、诡秘之主、一世之尊、全职法师、大奉打更人、全球高武、牧神记、诛仙、巫师之旅、龙蛇演义等 20+ 个经典体系中提取的结构规律。

核心提炼——所有好的体系都共享以下特征：

### 凡人→超凡→超规格→神话 的四段结构

| 阶段 | 凡人级 | 超凡级 | 规格级 | 神话级 |
|:-----|:-------|:-------|:-------|:-------|
| **凡人修仙** | 炼气 | 筑基→金丹→元婴 | 化神→炼虚→合体 | 大乘→渡劫 |
| **遮天** | 轮海 | 道宫→四极 | 化龙→仙台1-3 | 仙台4+→大帝 |
| **巫师之旅** | 学徒 | 正式巫师(1-4环) | 真灵巫师 | 深渊巫师 |
| **诡秘之主** | 序列9-7 | 序列6-4 | 序列3-1 | 序列0 (神) |

### 关键设计原则

**1. 断级差（量子跃迁不是线性增长）**

金丹和筑基的差距不是"多50%法力"——是**量级碾压**。一个金丹修士可以秒杀十个筑基修士。这就是断级差。

设计时应明确的断级效应：
```
筑基打筑基 → 五五开
筑基打金丹 → 必败（除非有特殊装备/计谋）  
金丹打筑基 → 碾压
金丹打金丹 → 五五开
金丹打元婴 → 必败
```

没有断级差的力量体系是"所有人都在同一围城里打架"——读者分不清强弱。

**2. 标志性节点（每个级别都有读者"看得见"的信号）**

好的体系里，每次升级都有一个具体的、可描述的画面：

| 体系 | 升级节点标志 |
|:-----|:------------|
| 凡人修仙·结丹 | 丹田中凝结一颗金色内丹 |
| 凡人修仙·元婴 | 内丹碎裂，一个迷你版自己从丹田中飞出 |
| 遮天·轮海 | 开辟苦海→涌出命泉→架设神桥→达彼岸 |
| 巫师之旅·一环 | 构建第一个法术模型，精神力突破阈值 |
| D&D | 获得新法术位，职业特性出现质变 |

**没有标志性节点 = 升级只是跑条，读者记不住。**

**3. 代价递进（越强越危险）**

好的力量体系不是"越升级越爽"——是"越升级，代价越大"。

| 体系 | 代价递进 |
|:-----|:---------|
| 凡人修仙 | 结丹=寿元大涨但天劫初现；元婴=可夺舍但天劫加重；化神=开始感知天地法则但被飞升通道限制 |
| 诡秘之主 | 序列越高能力越强，但失控风险越大，人性越少 |
| D&D | 高级法术威力巨大但施法次数有限，且引起更高层次存在注意 |

**没有代价递进的力量体系 = 无敌流，读者3卷后弃书。**

**4. 跨级战需要合理解释（不是"主角光环"）**

同级打赢 → 装备+战术，读者可以接受
越一级打赢 → 天时地利人和+主角付出重大代价，读者勉强能接受
越两级打赢 → 需要极特殊情境说明（金手指全力爆发/对方严重削弱），读者半信半疑
越三级打赢 → 这是崩了

## 必须回答的问题

1. **你的小说有几种力量类型？** — 魔法/武功/修仙/科技/血脉……每种类型在体系中的位置是什么？
2. **段位之间的断级差有多明显？** — 二级能否打一级？能打几个？需要什么条件跨级战？
3. **每个段位的标志性节点是什么？** — 读者能看到什么、听到什么、感受到什么来说明"他升级了"？
4. **越强代价越大吗？** — 高级段位有什么副作用/风险/限制？如果完全没有，这就是无敌流。
5. **同段位内有没有变体？** — 不是所有的筑基都长一样——有功法差异、有血脉差异、有装备差异。

## 执行步骤

### combat_capability
定义角色在每个段位的战斗能力边界：
- 段位名称、突破瓶颈条件
- 攻击范围（近战/远程/范围）
- 单挑战力定位、群战战力定位
- 弱点或冷却条件
- **标志性画面**：读者靠什么"看到"这个段位？

### monster_rank_map
将每个敌人类型映射到读者的认知坐标系：
- 敌人名称 → 对应段位
- 跨级对照（对低一级=碾压？同级=苦战？高一级=逃跑？）
- 至少设计 3 档难度差，让读者感受到成长

### act_rank_schedule
规划每段战力成长的节奏：
- 每幕/每卷结束时主角应达到的段位
- 关键里程碑章节
- 跨卷不可出现"中期就已经无敌了"的战力天花板

- `00-总控/数值体系/monster_rank_map.yaml`  
- `00-总控/数值体系/act_rank_schedule.yaml`
