# Phase 6：起点状态设计（NEW v4.0）

> 该文件同时作为执行指令和参考指南。

## 前置条件
Phase 5 数值体系生成完成。L1 设定已锁定。Phase 0.6 拆书融合已完成（如有锚点书）。

## 目的
产出 `设计/起点快照.md`。写第1章之前，明确故事开始时世界的完整状态。
**产出后必须经用户确认才能进 Phase 7。**

## 执行步骤

### 1. 读取上下文
- L1 元设定层（世界蓝图/力量体系/历史驱动力/物种天赋/势力格局/资源物品）
- story-engine.yaml (core_premise)
- 数值体系 (combat_capability.yaml)
- deconstruct 融合摘要 (如有)

### 2. 产出起点快照

输出 `design/起点快照.md`，按以下结构：

```
主角初始状态：
  等级/职业/装备/金钱/能力/对世界的认知

配角的初始状态：
  每个主要配角在故事开始时是什么位置、什么心态

城市/世界的初始状态：
  表面是什么样、暗面是什么样
  M1（世界级压力源）已经露出到什么程度

伏笔初始状态：
  哪些线已经埋好但还没触发
  哪些事件已经在发生但主角不知道

和其他角色的关系：
  同一时间线不同切面的相互关系
```

### 3. 闸门
输出 `起点快照.md` 内容展示给用户 → 用户说"对" → 进入 Phase 7。
❌ 起点快照未经用户确认不进 plot。
