# Phase 7：终点展望（NEW v4.0）

> 该文件同时作为执行指令和参考指南。

## 前置条件
Phase 6 起点快照已锁定（用户确认）。

## 目的
基于 story-engine + L1 设定 + 起点快照，反推产出 `设计/终点快照.md`。
**产出后必须经用户确认才能进 plot。**

## 执行步骤

### 1. 读取上下文
- Phase 6 产出: `design/起点快照.md`
- story-engine.yaml
- L1 元设定层
- 数值体系 act_rank_schedule.yaml (每卷结束时主角段位，来自 Phase 5)

### 2. 反推终点状态

输出 `design/终点快照.md`，按以下结构：

```
主角终点状态：
  这一卷结束时主角什么等级/职业/属性/装备/专长
  心智和心态发生了什么变化
  未解决的核心问题是什么

配角的终点状态：
  每个主要配角活了还是死了、变了什么

城市的终点状态：
  M1 推进到了什么程度
  社会秩序还剩多少

情节线的终点状态：
  哪些线收束了、哪些线留给下一卷
  最大的悬念钩子是什么

读者感受：
  读完这一卷读者应该记住的感觉
```

### 3. 闸门
输出 `终点快照.md` 内容展示给用户 → 用户说"对" → 进入 plot。
❌ 终点快照未经用户确认不进 plot。
