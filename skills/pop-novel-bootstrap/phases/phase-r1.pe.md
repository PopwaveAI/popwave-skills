# Phase r1：逆向工程 — 事件日志

## 前置条件
用户提供了已有正文（.txt/.md 文件）。
环境检测：扫描所有已有文件（PRD / L1 / 宪法 / 卷大纲 / 事件日志）并输出检测报告。

## 加载参考
先加载 `phases/phase-r1.ref.md`，了解事件日志和批次摘要的格式。

## 核心原则
- 本相位只读，不修改任何正文文件
- 每批最多 10 章，每批处理完用户确认后再继续
- 事件日志是后续所有逆向相位的基础——质量越高，后续越准

## 执行步骤

### 0. 环境检测
检查已有文件，输出检测报告：
- 已有正文：N 章（ch001 ~ chNNN）
- 已有事件日志：有/无
- 已有 PRD：有/无
- 已有 L1：有/无
- 已有宪法：有/无

### 1. 分批处理
按每批 10 章划分，逐批执行：
1. 读取本批 N 章原文
2. 读取前序批次摘要（如有）
3. 按 phase-r1.ref.md 格式输出逐章事件日志
4. 输出本批批次摘要

### 2. 用户确认
每批完成后暂停，展示关键发现，用户确认后继续下一批。

## 产出
- `01-写作资产/事件日志/ch{NNN}-eventlog.md`（每章一份）
- `01-写作资产/事件日志/batch{N}-summary.md`（每批一份）
