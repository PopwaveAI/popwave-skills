# Phase r1 参考：逐章事件日志模板

## 单章事件日志格式

```markdown
# ch{NNN} 事件日志

## 1. 关键场景
- [场景名]：[核心事件，1-2句]

## 2. 状态变化
- [角色名]：[关键获得/失去/关系变化]

## 3. 剧情动力
- 推进线索：
- 新开线索：
- 关闭线索：

## 4. 信息不对称
- 读者知道但角色不知道：
- 角色知道但读者不知道：
- 无明显信息差：

## 5. 情绪账户
- 本章走向：[上升/下降/反转]
- 章末读者状态：[期待/担心/愤怒/满足]
- 已兑现承诺：
- 新建承诺：

## 6. 文风指纹
- 平均句长（估）：~[X] 字
- 对话占比（估）：~[X]%
- 本章特色句式/高频词：[1-2个]
```

## 批次摘要格式

```markdown
# Batch {N} 摘要（ch{NNN}-ch{NNN}）

## 弧线概述
[本批10章的故事弧，3-5句]

## 主角状态（批次结束时）
- 关键资源/装备：
- 核心关系变化：
- 心理状态：

## 活跃势力
| 势力 | 立场 | 与主角关系 | 当前动向 |

## 跨批伏笔追踪
| 伏笔 | 埋入章 | 本批状态 |

## L1 候选（从本批提取的永久规则，按新六件套分类）

```yaml
world_blueprint:      # 01-世界蓝图 候选
power_system:         # 02-力量体系 候选
history_drivers:      # 03-历史与驱动力 候选
species_traits:       # 04-物种与天赋 候选
power_structure:      # 05-势力格局 候选
resources_items:      # 06-资源与物品 候选
```

> 每条候选规则标注出现章号。出现 1 次的标注"待确认"，出现 ≥2 次的标注"可信"。
```

## 处理规则

- 每批最多 10 章，不足 10 章也处理
- 每批处理完向用户展示关键发现，确认后继续
- 事件日志写入 `01-写作资产/事件日志/`
- 批次摘要写入 `01-写作资产/事件日志/batch{N}-summary.md`
