# 信息释放落地参考（Step 2 消费）

> 所属: pop-novel-chapter-design v1.4 · references/
> 说明: 此文件不独立执行。Agent 在 Step 2 逐事件设计时，参考本章的检查规则。

---

## 每设计一个事件时检查

### 1. 本章的 info_release 清单来自 act-XX.yaml

从 `info_release` 字段提取所有 P0 + P1 项。

### 2. 每个信息点三要素

| 问题 | 约束 |
|:-----|:-----|
| **在哪个事件释放？** | 标注在事件的信息释放字段 |
| **怎么释放？** | 实战展示 / 角色对话 / 叙事者说明 / 探索发现 —— 与 act-XX.yaml 的 release_method 一致 |
| **在哪一句/哪个场景释放？** | 一句话描述——"主角推开布道站的门时"、"保罗修女包扎时随口说" |

### 3. 信息密度检查

- 一个事件最多 2 个新信息点
- 章首事件不强制放信息
- 高复杂度设定必须分到不同事件逐步释放
- 读完所有事件后汇总为「信息释放落地清单」写入骨架

### 4. 按需读 L1 设定原文

按 info_release[].source_doc 指向的路径读取。读完即用，不预读全部。

---

## 红线

- ❌ P0 信息点未分配到任何事件 → 退回
- ❌ 一个事件 ≥ 3 个信息点 → 拆分
- ❌ 释放方式与 act-XX.yaml 不一致 → 退回
