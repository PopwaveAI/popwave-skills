| 级别 | 是什么 | 密度 | 判定器 | 负责 |
|:-----|:------|:----:|:------|:----:|
| **小** | 干脆利落的斩杀/升级/打脸/梗 | 每章 ≥ 5 | 读者 3 秒读完，反应"嗯爽" | **chapter-design** |

## 二、小爽点

### 2.1 定义

> 小爽点 = 一次干脆利落、杀伐果决的斩杀/制服/打脸，或一次可感知的升级/变强/赚钱/地位跃迁，或一个即用即走的社会热点梗植入。主角不废话、不犹豫、不解释。读者读完 ≈ 3 秒，反应 = "嗯，爽。"

**关键**：小爽点是一个完整的、需要设计的事件。它必须在事件链层面被规划——非文本脉冲，也非"写得好一点"。

### 2.3 在事件链中设计

```
Step 2 逐事件设计时：

事件 N:
  内容    ← 「角色做了什么 → 什么结果」。一句话
  情绪目标
  冲突层次
  中爽点? ← 读者认知有变化？
  小爽点? ← 这个事件是否是一次干脆的斩杀/升级/打脸/梗植入？
            是 → 标记为 ◆小爽点
  信息释放
  字数

全部事件写完后：
- [ ] ◆小爽点 ≥ 5 个？
- 不足 → 追加：过渡场景中插入一个干净利落的斩杀或处境逆转
      或 → 把一个铺垫事件升级为有"即时有结果"的小爽点事件
```

## 三、中爽点

### 3.1 定义

> 中爽点 = 有铺垫 + 有释放。铺垫和释放之间读者感知得到"蓄力"的过程。释放那一刻，读者不只在爽——读者在想"终于"或者"我就知道"。

与小爽点的区别：小爽点 = 即时有结果；中爽点 = 前面 N 章在铺，这一章收。

### 3.3 Canvas 消费指南

> ⚠️ 中爽点位置规划已归 plot。Canvas payoff_map 是 plot 交付给你的"各章中爽点位置图"。
> 你拿到的是：哪条线在第几章释放，级别是什么。你的职责：把这条线设计为一个完整的事件。

```
Step 1 读入 act-XX.yaml 的 canvas.payoff_map[本章]：

total_payoff ≥ 1 → Canvas 已供给
  lines[] 中标注了释放的线号和级别（如 "M1:中", "M3:小"）
  → 在 volume-XX.md §四 找到对应线定义，为它设计一个让读者认知变化的事件
  → 标记为 ★中爽点事件

total_payoff = 0 → 本章 plot 未能安排任何线释放
  ⚠️ 这是 plot 层面的失误。但本章仍需 ≥ 1 中爽点。
  → 进入 §四 的独家中爽点制造（兜底）
```

### 3.4 无供给时的中爽点制造

**策略 B（首选）：已有 M/S 线的微释放。**

```
逐线扫 Canvas.entries[本章]：
- M1 做了什么？→ "观察裂缝变化" → 升级为 "发现裂缝只对左侧有反应" → 信息揭示
- M3 做了什么？→ "和队友商量" → 升级为 "队友说漏了一个秘密" → 信息揭示
- S1 做了什么？→ "和妹妹吃饭" → 升级为 "妹妹问了一个不该她知道的问题" → 关系转折
```

**策略 C（兜底）：独立中爽点事件。**
- 对话揭示——两个角色在过渡场景"顺带说破"前面埋了多章的信息
- 内部抉择——主角在独处时做了一个决定（心理推进，而非打斗）

**策略 A（上游反馈）：连续 ≥ 3 章无释放 → 建议 plot 增加 S 线。**

## 四、责任归属

| chapter-design 负责 | chapter-design 不负责 |
|:---------------------|:----------------------|
| 逐事件标记 ◆小爽点（≥5/章） | 中爽点位置规划（由 plot Canvas 指定） |
| Canvas 有供给时按线号设计中爽点事件 | 大爽点位置（由 plot Canvas 指定） |
| Canvas 无供给时自行制造中爽点 | 特大爽点位置（由 plot Canvas 指定） |
| 连续无释放 → 建议 plot 增加 S 线 | 文本脉冲（归 prose-render 写作质量检查） |
