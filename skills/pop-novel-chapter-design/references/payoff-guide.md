# payoff-guide.md — 中爽点 · 事件级设计

> 管线: pop-novel-chapter-design v1.5
> 定位：chapter-design 专属。中爽点的定义、Case、事件链中的标记方法、以及当 Canvas 未供给中爽点时的自制造策略。
> 大爽点+终极爽点 → `pop-novel-plot/templates/payoff-guide.md`
> 微爽点 → `pop-novel-prose-render/references/payoff-guide.md`

---

## 一、定义

**中爽点 = 读者读完这个事件后，对故事的认知发生了可感知的变化。**

不是"这段写得好"——是"知道了之前不知道的事"、"角色关系变了"、"障碍被打破了"、"情绪落地了"。

| 级别 | 密度 | 是什么 | 读者反应 |
|:-----|:----:|:------|:--------|
| 中爽点 | 每章 ≥ 1 | **一次可感知的情节推进或情绪转折** | 眼睛睁大、调整了坐姿 |

---

## 二、四种类型 + 案例

| 类型 | 说明 | 案例 |
|:-----|:-----|:-----|
| **信息揭示** | 读者知道了之前不知道的关键信息 | "那个一直不说话的小孩，是城主十年前丢的儿子。" |
| **关系转折** | 两个角色的关系发生质变——升级/破裂/反转 | "她收了刀。不是因为打不过——是因为看到了他脖子上的旧疤。" |
| **障碍突破** | 主角闯过了一道关卡 | "第三扇门打开时，整个地下室亮了——他找到了。不只是武器。还有地图。" |
| **情绪落地** | 积蓄的情绪终于释放——读者和角色一起感受到了 | "他蹲下来，把戒指放在墓碑前。没说一句话。但走的时候，背直了。" |

## 三、事件链中逐事件标记

**在 Step 2 逐事件设计时，每个事件末尾加一行判断。**

Step 2 的执行流程（新增第 N+1 行）：

```
事件N:
  事件内容  ← 「角色做了什么 → 什么结果」
  情绪目标
  冲突层次
  中爽点标记 ← 读者读完这个事件后，对故事的认知有没有变化？
    - 有变化 → ★中爽点事件（注明类型：信息揭示/关系转折/障碍突破/情绪落地）
    - 无变化（纯过渡/纯环境/纯铺垫/纯确认）→ 不标记
  信息释放
  估计字数
```

**逐事件扫完后的自检**：

```
Step 3 产出前验证：
- [ ] 本章 ★中爽点事件 ≥ 1 个？
  不足 → 回到最平淡的过渡事件，将其升级为有信息揭示或情绪转折的事件。
  或者 → 把某个铺垫事件与后面的释放事件合并到本章。
```

## 四、Canvas 消费指南

**plot 完成的 act-XX.yaml 中，Canvas 每章有一条 `payoff_map` 汇总线。**

这是 chapter-design Step 1 必须读入的关键字段：

### 4.1 读取流程

```
Step 1 读入 act-XX.yaml 时，专门看 Canvas 段的 payoff_map：

1. 看本章是否有 payoff_map 记录：
   - 有 → 看哪些线（M1/M2/M3/S1/S2）在 payoff_map 中被标记为 中 或 大
   - 无 → Canvas 未供给，本章需要自行制造

2. 有供给的情况下：
   - 按线号在 Canvas.entries[本章].{线号} 中找到该线的摘要 → 这是中爽点事件的核心素材
   - 在事件链中为这条线设计一个"读者认知发生变化"的事件
   - 该事件可能发生在章首（开门见山）或章中（先铺后揭），但不该在章末最后一行（章末留给钩子）

3. 无供给的情况下（总 payoff=0）：
   → 进入 §五 的"独家中爽点制造"流程
```

### 4.2 Canvas 供给分级处理

| Canvas 本意 payoff 供给 | 本章需产出 |
|:------------------------|:----------|
| 大爽点章（某线 payoff=大） | 全幕最高密度事件链。大爽点事件 = 章中高潮，前后围绕它设计 |
| 终极爽点章（多线汇聚） | 全幕最大规模事件链。每条 payoff=大 的线各分配至少一个高潮事件 |
| 中爽点供给 ≥ 1 | 按 Canvas 线号设计对应的事件。至少 1 个中爽点事件已兜底 |
| 中爽点供给 = 0 | ⚠️ **必须自救** → 进入 §五 |

## 五、Canvas 未供给时的独家中爽点制造

**有些章 Canvas 上没有任何线在该章释放——都是铺垫/建设/过渡。但中爽点密度要求每章 ≥ 1。这里必须自己造。**

### 5.1 策略选择

按优先级排列：

**策略 A：增加 S 线（推荐，上游反馈）**

如果连续多章 Canvas 无释放，且 S1/S2 还有空线——向 plot 提出：在此卷 volume-XX.md §四 增加一条 S 线，拉高全卷的事件密度。

```
在 chapter-design 执行中发现 Canvas 连续 ≥ 3 章无 payoff → 
通知 plot："建议增加 S3 线，主要关注 XX 角色的个人线或 XX 设定线索"
→ plot 回改 volume-XX.md §四 + Canvas
```

**策略 B：增加已有线的事件密度（推荐，自闭环）**

在无释放章中，从已有 M/S 线中提取一个"微型释放"：
- M1 线在铺垫章中，让主角发现一个细节——虽然裂缝没封，但发现了新的弱点 → 这是信息揭示
- M3 线在建设中，让一个配角说漏了嘴——虽然没推进主线，但透露了关键信息

```
逐线扫 Canvas.entries[本章]：
- M1 做了什么？→ "观察裂缝变化" → 可升级：加上"发现裂缝只对左侧有反应" → 信息揭示
- M3 做了什么？→ "和队友商量下步" → 可升级：商量的过程中队友说漏了一个秘密 → 信息揭示
- S1 做了什么？→ "和妹妹吃晚饭" → 可升级：晚饭时妹妹问了一个不该她知道的问题 → 关系转折
```

**策略 C：独立事件（备用）**

不依赖任何线，插入一个结构独立的中爽点事件：
- 对话揭示——两个角色在过渡场景中"顺带说破"了之前埋了多章的信息
- 环境暗示——主角在过场中注意到一个细节，改变了之前的判断
- 内部抉择——主角在独处时做了一个决定（不是打斗，是心理推进）

### 5.2 设计原则

```
1. 不从零造线——在已有 M/S 线上做微型释放
2. 不改变 Canvas 的大方向——只是把铺垫升级为"铺垫中有一次微释放"
3. 优先策略 B（已有线微释放）→ B 不够再策略 C（独立事件）→ 长期缺再策略 A（建议加线）
```

## 六、责任归属

| chapter-design 负责 | chapter-design 不负责 |
|:---------------------|:----------------------|
| 逐事件标记中爽点 ★ | 大爽点位置（由 plot Canvas 指定） |
| Canvas 有供给时按线号设计中爽点事件 | 终极爽点位置（由 plot Canvas 指定） |
| Canvas 无供给时自行制造 ≥ 1 中爽点 | 微爽点（归 prose-render） |
| 连续无释放 → 建议 plot 增加 S 线 | |
