# Step 2：事件链设计

> 管线: pop-novel-chapter-design v1.3
> 模板: `templates/fact-skeleton.md`

---

## 目的

将本章拆解为事件序列。每个事件 = 一个「有叙事意义的剧情节点」。事件链定义「这场戏发生了什么」，不定义「怎么演、什么质感」。

---

## 什么是 Design 事件

### 一句话定义

> **一个事件 = 一个回合 = 一个独立的因果单位。**

一个事件是：一个角色做了（或遭受了）一件事，这件事产生了某个结果，读者读完这个结果后情绪发生了一次变化。

### 事件不是

```
❌ "他在科尔帮据点里暗杀了二楼的三个人"        ← 三个回合合并，太粗
❌ "推开门。门轴发出微响。他走进去。"          ← 句子级动作，太细——这是 Render 的微节拍
❌ "一场激烈的战斗在这个夜晚爆发了"            ← 说了等于没说
```

### 事件是

```
"他逐一清除了二楼三个目标："  ← 这是一个镜头描述，不是事件

✅ "推门进入科尔办公室——科尔正在翻抽屉，背对着门"        ← 第一个回合：接近+发现
✅ "科尔转身拔剑——已经来不及了，刀从侧面切入他的右腕"    ← 第二个回合：交锋+结果
✅ "科尔惯用手被废，退到墙角——江轩正面逼近"              ← 第三个回合：追击
✅ "正面格杀——科尔倒下，三刀补命，确认死亡"              ← 第四个回合：终结
```

**一次战斗的 N 个回合 = N 个 Design 事件。** 每个回合有「谁做了什么事 → 什么结果 → 读者什么感觉」。

### 不同场景下的事件粒度

| 场景 | 一个事件 = | 示例 |
|:-----|:----------|:-----|
| **战斗** | 一个回合/一次交锋/一个战术决策 | 主动出击→结果 / 躲避→反击 / 发现弱点→利用弱点 |
| **对话** | 一次姿态变化或信息揭示 | 试探→对方沉默 / 摊牌→对方情绪崩溃 / 随口一句话→暗藏的信息被主角捕捉 |
| **探索** | 一次发现或一次判断 | 看到鳞片→判断非天然 / 蹲下查看拖痕→推断身长 / 发现血迹→判断是人类血 |
| **过渡** | 一次情绪转变或新目标确立 | 回到家看到妹妹→决定带她去治伤 / 站在巷口看到新面孔→判断秃鹫帮在摸底 |

### 每个 Design 事件回答

| 问题 | 约束 |
|:-----|:-----|
| **在哪？** | 必须可追溯到 act-XX-地图.md |
| **谁？** | 必须可追溯到 act-XX-人物.md |
| **发生了什么？** | 角色做了什么 → 什么结果。一个句子。不写具体动作细节/感官/氛围 |
| **读者感受到什么？** | 情绪目标 ← 这是 Render 的创作靶心。Render 必须让读者达到这个情绪 |
| **估计字数** | 100-250 字（Render 在此范围内把"结果"展开成具体的动作/感官/氛围段落） |

> **情绪目标是给 Render 的靶心，不是给 Design 的免责声明。**
> Design 提供了足够数量的有情绪靶心的事件 → Render 才有材料去激发读者的情绪波动。
> 如果 Design 事件太少、太粗、太模糊 → Render 拿不到靶心 → 逐句渲染时自己也不知道该往哪个方向写 → **读者感受就平了。**

## 事件数量：硬性下限 + 复杂度上浮

```
Step 1：算下限
  Design 事件数 ≥ 章预期字数 ÷ 200

Step 2：算额外需求（多线场景）
  每条并行的叙事线各需要 2-4 个独立事件做切换
  → 双线 +4 事件，三线 +6，四线以上 +8

Step 3：取两者之大值
```

### 不同场景的下限与参考范围

| 篇幅 | 硬性下限 | 常见范围 | 说明 |
|:-----|:------:|:------:|:-----|
| 2000 字过渡章 | **≥10** | 10-14 | 六个事件只够写一段开幕式 |
| 2500 字常规章 | **≥12** | 12-18 | 单线叙事为主 |
| 2500 字战斗章（单线） | **≥12** | 16-24 | 暗杀/决斗——每回合 = 一个事件 |
| 3000 字战斗章（多线） | **≥15** | 20-30 | 城市战/军团战——多线各有独立回合链 |
| 2500 字对话章 | **≥12** | 12-16 | 每轮信息交换/情绪摊牌 = 一个事件 |
| 2500 字探索章 | **≥12** | 12-18 | 每步发现/判断 = 一个事件 |

> `÷200` 是硬性下限。低于此值是设计缺陷——不是"风格选择"，是给 Render 的材料不够。

## Design 事件 vs Render 微节拍

```
                 Design 事件                                Render 微节拍
                ────────────                               ────────────
                
 管什么           "科尔转身拔剑——已经来不及了，              Render 拿到这个事件，自己决定：
                 刀从侧面切入他的右腕"                       科尔转身的第一反应是什么 →
                                                            拔剑时的细节（剑卡住了/拔出来了） →
                 "正面格杀——科尔倒下，                       怎么切入右腕（从哪个角度/什么感觉） →
                 三刀补命，确认死亡"                           倒下的方式/三刀的顺序/确认死亡的细节
                
 数量            12-30 个/章                                40-60 个/章（Design 不关心）
                
 情绪目标          有 ← Design 标注                          据此选句式和节奏 →
                
 信息释放          标注 info_release item_id ←               按 release_method 执行 →
```

**Design 给 Render 一套「靶心」。Render 据此设计每个段落的写作方案。** 如果靶心太少，Render 自己也不知道往哪打——结果就是写得平。

## 跨章剧情弧线识别

如果本章的核心剧情在给定字数内无法完成一个完整的「起→承→转→合」，**不要压缩事件**，而是：

1. **判定为跨章弧线** → 标注 `arc_span: [N, M]`
2. **为每章标注弧内位置** → `决战①` / `决战②` / `决战③`
3. **为每章标注情绪分工** → 铺垫→交战→高潮收束

跨章弧线的事件分配：
1. 对整个弧线设计完整事件链
2. 按章节字数预算切分
3. 每个切分点找一个情绪落点

---

## 执行

### 1. 判断跨章弧线

```
检查 act-XX.yaml 中相邻章的标题和 emotional_goal：
  - 标题连续且相关（"决战①""决战②""决战③"）→ 跨章
  - 情绪目标递进（紧张→临界→释放）→ 跨章
  - 字数预算 > 3500，剧情复杂度 > 大纲预估 → 跨章
```

### 2. 设计事件链

1. 从本章第一个叙事时刻开始，逐个列出每个回合/每次发现/每轮对话
2. 每个事件标注地点、角色、内容（做什么→什么结果）、情绪目标、估计字数
3. 写完检查：
   - 事件数 ≥ 硬性下限？
   - 事件之间有因果链（所以→然后）？
   - 情绪没有来回跳极？
   - 章末事件是钩子？

### 3. 战斗章事件链示例（科尔帮据点暗杀，~2500 字）

```
事件1:  暗巷观察据点 — 数清门口两人、二楼窗口一人、烟囱有烟 — 警觉 — 120
事件2:  绕到后巷 — 木箱堆叠，可攀上二楼 — 判断 — 100
事件3:  翻窗上二楼 — 窗台湿滑，左手打滑→抓稳 — 紧张 — 120
事件4:  落地无声 — 确认没有惊动任何人 — 警觉 — 80
事件5:  听到鼾声 — 第一个目标在睡觉，门虚掩 — 判断 — 100
事件6:  反手握住匕首 — 推门→门轴轻响→鼾声停了 — 紧张上升 — 120
事件7:  不等对方完全清醒 — 左手捂嘴→右手匕首刺入喉咙 — 冷酷 — 140
事件8:  确认死亡 — 血沫从嘴角溢出，没有挣扎 — 冷酷+确认 — 80
事件9:  从尸体上跨过 — 走廊地板在脚下轻响 — 过渡 — 60
事件10: 第二个房间有光 — 一个醒着，正趴在桌上写信 — 观察 — 100
事件11: 背对门口 — 油灯的光拉长了他的影子 — 利用环境 — 80
事件12: 刀从侧面切入 — 油灯被撞倒，房间瞬间黑暗 — 果断 — 120
事件13: 黑暗中不动 — 五秒，确认楼下无动静 — 警觉 — 80
事件14: 第三个房间空 — 床铺没叠，窗户开着 — 疑问 — 100
事件15: 走廊尽头有烟味 — 楼下大厅有人说话 — 新信息 — 100
事件16: 压低身形下楼 — 看见大厅火光，两个人围炉喝酒 — 观察 — 120
事件17: 判断：从背后解决 — 第一个人，刀从后颈切入 — 果断 — 120
事件18: 第二个人刚站起来 — 正面短刀交锋，三个回合→割断喉咙 — 激烈 — 160
事件19: 整栋楼静了 — 只有雨声和远处狗叫 — 喘息 — 100
```

**19 个事件 × ~110 字均值 = ~2100 字战斗骨架。** 每个回合是独立事件，Render 在每个事件 110 字的空间内写具体动作/氛围/感官。

---

## 红线

- ❌ 事件数 < 章预期字数 ÷ 200 → 退回（"靶心不够，Render 没材料"）
- ❌ 战斗事件写成模糊概括 → 退回（"开始攻击对方"——哪一刀？什么结果？谁的回合？）
- ❌ Design 事件写成 Render 微节拍 → 退回（"他的匕首划破空气发出尖锐的啸声，刀尖刺入布料"——这是 Render 的活）
- ❌ 事件链因果断裂 → 退回（没有"所以"或"然后"）
- ❌ 跨章弧线未标注 → 退回
- ❌ 跨章弧线的章末事件不是自然情绪落点 → 警告
