# Step 2：事件链设计

> 管线: pop-novel-chapter-design v1.4
> 模板: `templates/fact-skeleton.md`
> 参考: `references/character-scheduling.md` `references/location-orchestration.md` `references/info-release.md` `references/emotional-beats.md`

---

## 目的

将本章拆解为事件序列。**每个事件设计时同步确定：在哪、谁、发生什么、读者什么感觉、要不要放设定。** 不是先设计事件后补填表——补填必然导致错配。

---

## 一、什么是 Design 事件

> **一个事件 = 一个回合 = 一个独立的因果单位。**

角色做了（或遭受了）一件事 → 产生了某个结果 → 读者读完情绪发生一次变化。

```
❌ "他在据点里暗杀了二楼三个人"        ← 三个回合合并
❌ "推开门。门轴微响。他走进去。"       ← Render 微节拍
❌ "一场激烈的战斗爆发了"              ← 说了等于没说

✅ "科尔转身拔剑——刀从侧面切入他右腕"   ← 一个回合：交锋+结果
```

**一次战斗 N 个回合 = N 个 Design 事件。**

### 不同场景粒度

| 场景 | 一个事件 = | 示例 |
|:-----|:----------|:-----|
| 战斗 | 一个回合/一次交锋 | 出击→结果 / 躲避→反击 |
| 对话 | 一次姿态变化或信息揭示 | 试探→对方沉默 / 摊牌→情绪崩了 |
| 探索 | 一次发现或一次判断 | 看到鳞片→判断非天然 |
| 过渡 | 一次情绪转变或新目标 | 看到妹妹的伤→决定带她去治伤 |

---

## 二、事件数量

```
硬性下限: Design 事件数 ≥ 章预期字数 ÷ 200
多线补充: 双线+4 / 三线+6 / 四线+8
```

| 篇幅 | 下限 | 常见 |
|:-----|:----:|:----:|
| 2000 字过渡章 | ≥10 | 10-14 |
| 2500 字常规/对话/探索章 | ≥12 | 12-18 |
| 2500 字战斗章（单线） | ≥12 | 16-24 |
| 3000 字战斗章（多线） | ≥15 | 20-30 |

## 三、执行流程

### 3.1 判断跨章弧线

```
检查 act-XX.yaml 相邻章的标题和 emotional_goal：
  - 标题连续（"决战①""②""③"）→ 跨章
  - 情绪递进（紧张→临界→释放）→ 跨章
  - 字数预算 > 3500 且剧情 > 一章量 → 跨章
```

跨章则：对整个弧线设计事件链 → 按章切分 → 每章找情绪落点。标注 `arc_span / 本章位置 / 情绪分工`。

### 3.2 逐事件设计

**从本章第一个叙事时刻开始，逐个事件设计。每个事件同步确定所有字段：**

```
事件N:
  地点    ← 先查 act-XX.yaml 的 locations → 查 act-XX-地图.md 取描述 → 写在事件里
  参与角色 ← 先查 act-XX.yaml 的 characters_active → 查 entity-snapshot 取 before 状态 → 写在事件里
  事件内容  ← 「角色做了什么 → 什么结果」。一个句子。不写具体动作/感官/渲染细节
  情绪目标  ← 每轮取一个词（参考 references/emotional-beats.md 词汇表）
  冲突层次  ← external / internal / interpersonal — 可选 1-3 层。思考：这个事件在哪个层面产生张力？外在对抗？内心挣扎？还是人际摩擦？过渡事件可为空
  信息释放  ← 如本事件触发 info_release，标注 item_id（检查: ≤2条/事件, 不重复, 与 release_method 一致）
  估计字数  ← 100-250 字
```

> 设计事件3 时发现需要一个角色不在 characters_active 中 → 退回，不要编。需要的地名不在地图 → 退回，不要编。

### 3.3 全部事件写完后的全局检查

| 维度 | 检查 | 不通过 |
|:-----|:-----|:------|
| **情绪弧线** | 逐步递进？章首非高潮？无连续3事件同一情绪？ | 退回 |
| **冲突层次** | 战斗/对话/危机章大多数事件标注了冲突层次？过渡章可少标 | 警告 |
| **信息释放** | P0 全部分配？无事件≥3条？释放方式一致？ | 退回 |
| **空间** | 连续事件间地理可达？地点选择服务情绪目标？ | 退回/警告 |
| **角色** | before 来自 entity-snapshot？台词风格是具体词？ | 退回 |
| **因果** | 每个事件到下一个有"所以"或"然后"？ | 退回 |
| **钩子** | 章末事件是钩子？ | 退回 |

### 3.4 整理为情绪节拍 + 信息释放清单

用全部事件的情绪目标汇总出：起点 → 推进 → 高潮 → 终点。用全部事件的信息释放字段汇总出落地清单。

## 四、战斗章事件链示例（科尔帮据点暗杀，~2500 字）

```
事件1:  暗巷观察据点 — 数清门口两人/二楼窗口一人/烟囱有烟 — 警觉 — 120
事件2:  绕到后巷 — 木箱堆叠，可攀上二楼 — 判断 — 100
事件3:  翻窗上二楼 — 窗台湿滑，左手打滑→抓稳 — 紧张 — 120
事件4:  落地无声 — 确认没惊动任何人 — 警觉 — 80
事件5:  听到鼾声 — 第一个目标在睡，门虚掩 — 判断 — 100
事件6:  反手握匕首 — 推门→门轴轻响→鼾声停了 — 紧张上升 — 120
事件7:  不等对方清醒 — 左手捂嘴→右手匕首刺入喉咙 — 冷酷 — 140
事件8:  确认死亡 — 血沫从嘴角溢出，没挣扎 — 冷酷+确认 — 80
事件9:  跨过尸体 — 走廊地板在脚下轻响 — 过渡 — 60
事件10: 第二个房间有光 — 一个醒着，趴在桌上写信 — 观察 — 100
事件11: 背对门口 — 油灯拉长影子 — 利用环境 — 80
事件12: 刀从侧面切入 — 油灯撞倒，房间黑暗 — 果断 — 120
事件13: 黑暗中不动五秒 — 确认楼下无动静 — 警觉 — 80
事件14: 第三个房间空 — 床铺没叠，窗户开着 — 疑问 — 100
事件15: 走廊尽头烟味 — 楼下大厅有人说话 — 新信息 — 100
事件16: 压低下楼 — 大厅火光，两人围炉喝酒 — 观察 — 120
事件17: 从背后解决第一个 — 刀从后颈切入 — 果断 — 120
事件18: 第二个正面交锋 — 三个回合→割断喉咙 — 激烈 — 160
事件19: 整栋楼静了 — 雨声和远处狗叫 — 喘息 — 100
```

**19 事件 × ~110 字 = ~2100 字骨架。** Render 在每个事件的 110 字空间内自主拆 3-5 个微节拍。

---

## 红线

- ❌ 事件数 < 章字数 ÷ 200 → 退回
- ❌ 战斗事件写成模糊概括 → 退回
- ❌ Design 事件写成 Render 微节拍 → 退回
- ❌ 事件链因果断裂 → 退回
- ❌ 跨章弧线未标注 → 退回
