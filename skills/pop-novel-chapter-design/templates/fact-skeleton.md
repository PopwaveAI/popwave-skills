# chXXX-事实骨架模板

> 产出路径: `03-写作资产/chXXX-事实骨架.md`
> 生产方式: pop-novel-chapter-design v1.1+
> 下游消费者: pop-novel-prose-render

---

## 模板说明

事实骨架 = 本章的剧情结构设计。它定义「发生什么」，不定义「怎么发生」。

**粒度标准**：每个事件 = 一个动作/一个观察/一个感觉变化/一句话。不是「整个战斗」是一个事件，而是「刺出第一刀」是一个事件。

**关键原则**：
- 事件只描述**内容**（谁、在哪、做什么），不描述**方式**（怎么写的、什么语气）
- 每个事件 150-400 字渲染量。战斗动作事件取低值（150-200），描写/对话事件取中高值（250-400）
- 章预期字数 ÷ 200 ≈ 基准事件数
- 每个事件标注情绪目标 → Render 据此选句式和节奏

## 填写自检

### 事件数量（硬性下限）

- [ ] **事件数 ≥ 章预期字数 ÷ 200** — 低于此值是设计缺陷，"靶心不够，Render 没材料"
- [ ] 每个事件 = 一个回合/一个因果单位（战斗章：一刀一事件；对话章：一轮信息交换一事件）

### 事件质量

- [ ] 每个事件标注了情绪目标 ← 这是 Render 的创作靶心
- [ ] 事件内容是「谁做了什么 → 什么结果」— "科尔转身拔剑——刀从侧面切入他右腕"，不是 "开始攻击"
- [ ] 事件内容**不是** Render 微节拍 — 不写具体动作细节/感官/氛围
- [ ] 事件链因果不断裂 — 每个事件和下一个事件之间有"所以"或"然后"
- [ ] 章末事件是钩子事件

### Canvas 对齐

- [ ] 每个事件的「参与角色」在 act-XX-人物.md 中存在
- [ ] 每个事件的「地点」在 act-XX-地图.md 中存在
- [ ] 「信息释放落地清单」与 info-release-XX.md 对齐
- [ ] 「与上章衔接」与 entity-snapshot 对上章的状态一致
- [ ] 情绪节拍「终点」能自然衔接到下一章的起点
- [ ] 跨章弧线如存在，arc_span + 本章位置 + 情绪分工 三个字段必填
- [ ] 每个事件有估计字数（100-250字）
