# 拆书解构

拆书是"配钥匙"——从优秀作品中提取可复用的写法规则和节奏模板。**不只是复述剧情。**

产出物供给 pop-novel-writer（正文写作）和 pop-novel-bootstrap（开书设定）管线。

---

## ❌ 质量红线（开工前→完工后自检）

| # | 标准 | 完工确认 |
|:-:|:-----|:--------:|
| ❌1 | **不只是复述剧情** — 理解剧情是必要前提，但产出必须回答"这个手法好在哪里、我怎么用" | [ ] |
| ❌2 | **锚定章来源与场景匹配** — 新书→参考书；续写→自身前文。不确定问用户 | [ ] |
| ❌3 | **每条特征提炼答"好在哪"而非"做了什么"** — "环境压制战斗节奏"→❌；"闷响替代视觉描写"→✅ | [ ] |
| ❌4 | **已主动搜索关联资料** — 同人→查原著百科/时间线；特殊题材→搜领域素材 | [ ] |
| ❌5 | **P0 级规则全部写入，P2 级已丢弃** — 手法独特+有片段佐证→必须写入。泛泛而谈→丢弃 | [ ] |
| ❌6 | **每个产出至少含 1 条可复用规则** | [ ] |
| ❌7 | **覆盖场景类型 ≥ 2 种（拆解时），锚定章覆盖 ≥ 3 种（提取时）** | [ ] |
| ❌8 | **不准用二手资料代替原文** — 分析必须基于原文 | [ ] |
| ❌9 | **文风DNA提取完整** — Step 3-b 必须产出 5 条原则，每条含信念陈述+证据链+跨章验证+应用规则，不允许缺项 | [ ] |

## 什么时候用

| 场景 | 推荐模式 |
|:-----|:---------|
| 画整本书的节奏地图 | 模式A — 结构拆解 |
| 学习作者的写法技巧，提取可复用规则 | 模式B — 对标拆解 |
| 只需要某一类内容 | 模式C — 主题定向提取 |
| 提取设定层面的逻辑 | 模式D — 体系拆解 |
| 只有大纲/章纲，无正文 | 模式E — 大纲密度拆解 |

## 拆解流程（3步）

### Step 1：拆解前思考——想清楚为什么拆

1. **这本书在什么方面值得学？** — 节奏？写法？设定？体系？
2. **我的书需要什么？** — 带着具体问题去拆
3. **选择哪个模式？** — 参考上方表格

输出一句话拆解目标，例如：
- ✅ "我要学这本书每15-20章放一个高潮的节奏控制手法"
- ❌ "我要拆这本书"

然后按选择进入对应的模式。

### Step 2：执行拆解（按模式走）

#### 模式A — 结构拆解

全书通读 → 每章标记节拍 → 绘制情绪曲线 → 总结节拍模式

产出：`素材库/拆解报告/节奏地图-{书名}-{日期}.md`

**❌ 常见错误**：写成章节摘要而非功能节点标记。

#### 模式B — 对标拆解

全书扫描 → LLM 切分场景并分类 → 提取写法规则 → 新颖性去重 → 规则入库

产出：`素材库/拆解报告/对标拆解-{书名}-{日期}.md`

**❌ 常见错误**："环境压制战斗节奏"这种泛泛规则 → P2丢弃。
**✅ 好规则**："用'闷响'替代视觉描写——读者'听到'了战斗"

#### 模式C — 主题定向提取

明确目标主题 → 全局搜索匹配场景 → 仅对匹配场景执行模式B提取

产出：`素材库/拆解报告/主题提取-{主题}-{书名}-{日期}.md`

#### 模式D — 体系拆解

按卷扫描收集设定元素 → 归纳层级/晋升条件/克制关系 → 成长时间线 → 跨卷演化分析

产出：`素材库/拆解报告/设定-{书名}-{日期}.md`

#### 模式E — 大纲密度拆解

按章扫描大纲标注意事件类型 → 计算事件密度+爽点间隔 → 绘制节奏曲线

产出：`素材库/拆解报告/大纲-{书名}-{日期}.md`

### Step 3：提取锚定章片段

所有模式完成后追加此步骤。从拆解素材提取 3-5 个高光片段，按场景类型分类写入锚定章库。

**提取标准**：

| 等级 | 标准 | 处理 |
|:----|:-----|:-----|
| P0 | 情绪反应强烈 + 写法有独特技巧 | 必须提取 |
| P1 | 写法有技巧或场景精彩 | 推荐提取 |
| P2 | 场景和写法都普通 | 跳过 |

**文件名格式**：`{场景类型}-{来源书名}-{场景特征}.md`
**来源决策**：新书→参考书；续写→自身前文；不确定问用户。

---

### Step 3-b：提取叙事哲学DNA（v3.0 新增）

在锚定章片段提取完成后执行此步骤。基于 Step 3 产出的锚定章片段 + 再次扫描原文，按 v3.0 文风锚定包模板产出 5 条叙事哲学原则，组装为完整的 `01-写作资产/文风锚定包.md`。

**输入**：
- Step 3 产出的锚定章片段文件（`素材库/锚定章库/`）
- 原文全文（用于跨章验证）

**输出**：`01-写作资产/文风锚定包.md`
- Layer 1：锚定章片段（从 Step 3 文件按模板格式重组，含微观质感标注 + DNA印证）
- Layer 3：5 条叙事哲学原则（本条新增）

**参考格式**：`skills/pop-novel-writer/styles/文风锚定包模板.md`（v3.0版本）

**5 个叙事哲学维度**：

| # | 维度 | 核心问题 |
|:-:|:-----|:---------|
| 1 | 信息释放哲学 | 作者对信息释放的根本态度是什么？ |
| 2 | 叙事者姿态哲学 | 叙事者与读者之间的距离关系是什么？ |
| 3 | 情感表达哲学 | 情绪主要靠什么传达？ |
| 4 | 对话策略哲学 | 对话在小说中扮演什么角色？ |
| 5 | 张力-释放哲学 | 节奏控制的底层逻辑是什么？ |

**每条原则的完整结构**：

```
#### 原则 N：[维度名称]

**信念陈述**：[一句话回答作者的叙事信念——不是做了什么，而是为什么这么做]

> "作者认为..."

**证据链**（来自锚定章）：
- {锚定章文件名} → [该片段在该维度上的具体表现]
- {锚定章文件名} → [该片段在该维度上的具体表现]

**跨章验证**：该特征在 CH{..} 和 CH{..} 中一致出现？
- CH{..}：{一致/不一致，表现为}
- CH{..}：{一致/不一致，表现为}

**应用规则**（可执行指引）：
- [可执行规则1]
- [可执行规则2]
- [可执行规则3]
```

**提取步骤**：

1. **扫描锚定章片段** — 阅读 Step 3 产出的所有锚定章 .md 文件，理解微观质感特征
2. **逐维分析** — 对 5 个维度分别从锚定章片段中寻找证据
3. **跨章验证** — 再次扫描原文 2-3 个不同章节，验证该特征是否一致
4. **提炼信念** — 从证据上升到信念层，回答「作者为什么这么做」
5. **推导应用规则** — 从信念拆解为 3-5 条写作者可直接执行的规则
6. **组装输出** — 按模板格式写入 `01-写作资产/文风锚定包.md`，合并锚定章片段 + DNA 原则

**❌ 常见错误**：
- 信念陈述写成「作者做了X」而非「作者认为Y」
- 证据链没有引用具体锚定章文件名
- 应用规则抽象不可执行（如"控制好节奏"→❌；"每章铺垫不超过70%"→✅）
- 跳过跨章验证直接写结论（单个章节的特征可能是特例，不是DNA）

## ❌ 错误示例

### WRONG 1：只停在复述剧情

```
用户：拆一下《诡舍》
Agent：输出"这本书讲了主角穿越到恐怖世界的故事"
❌ 错误：只复述了剧情
✅ 正确：先理解剧情 → 再提取手法：
   "每15-20章放一个高潮，铺垫期对话占比25-30%"
```

### WRONG 2：特征提炼≈剧情复述

```
核心特征：①环境压制战斗节奏 ②战斗中有明确阶段转换
❌ 任何战斗都符合，不具体
✅ 正确：
   ①"闷响"替代视觉描写——读者"听到"了战斗
   ②雨水落在通红脚上滋滋作响——物理细节建立可信度
```

### WRONG 3：不搜索关联资料

```
用户：拆《海贼法典》（海贼王同人）
Agent：只读了给的txt
❌ 错误：未搜索百科对照原著时间线
✅ 正确：搜索百科 → 对照txt判断偏离度 → 提取"再创作"的独特写法
```

### WRONG 4：只产锚定章，不产文风DNA

```
用户：拆《斩神》
Agent：产出 3 个锚定章片段到素材库/锚定章库/，然后结束
❌ 错误：只有微观质感实例，没有叙事哲学DNA。writer v10.0 消费端要求
    01-写作资产/文风锚定包.md 必须含 5 条 DNA 原则
✅ 正确：在锚定章基础上，按 v3.0 模板提取 5 条DNA原则：
    原则1（信息释放哲学）：战斗前集中释放设定，战斗中零新信息——读者注意力不被分散
    原则2（叙事者姿态哲学）：叙事者完全隐身，不介入不评价——读者自行推导因果关系
    原则3（情感表达哲学）：情绪通过身体反应和环境投射传达，不写情绪名词
    原则4（对话策略哲学）：对话每轮推进信息+关系两个维度，不浪费在寒暄
    原则5（张力-释放哲学）：每20章一个高潮，铺垫期对话占比25-30%
    写入 01-写作资产/文风锚定包.md
```

## 异常与边界条件

拆解环境假设理想，但实操常遇异常。以下预定义 fallback，保证过程不会"一跑就卡住"。

| 场景 | 触发条件 | 处理动作 |
|:-----|:---------|:---------|
| **正文文件不存在/格式不支持** | `ch001.txt` 或用户提供的文件找不到 | 提示用户提供 .txt 或 .md 格式。不接受 pdf/docx → 引导用户转换后再提供。**不退回到编造内容** |
| **参考书字数太少（<5 万字）** | 全书通读后估算总字数不足 5 万字 | 拆解价值有限。与用户确认是否继续。用户坚持 → 缩减拆解范围，只做模式 B 或模式 C，标注"短篇参考书" |
| **同人题材但无原著知识** | 检索引擎返回用户只提供了同人 txt，无原著参考 | **必须先搜索原著百科、设定、时间线**（❌4 红线）。搜索不到 → 标注"无原著参考，拆解限于同人正文内部一致性分析" |
| **模式B 所有规则都是 P2** | 新颖性去重后全部规则被标记为 P2 | 回退到模式A（先画节奏地图），然后再做模式B。如果模式A充分，能发现 P0 级规则的概率从 <10% 提升到 >40% |
| **锚定章提取发现无片段达到 P0** | Step 3 扫描所有高光场景，均不满足"情绪强烈+写法独特" | 降低标准到 P1，但输出报告标注"参考书本身质量一般，锚定章仅供参考"。不让管线空转 |
| **WebSearch 搜索关联资料时平台不可用** | 搜索同人百科/民俗素材时特定站点返回不可用 | 降级为可用平台。搜索仍不可用 → 标注"未做关联搜索"，不阻塞拆解 |
| **fragment-pipeline 运行失败** | Python 脚本报错，db 未生成 | 标注"片段库构建失败"，但不阻塞锚定章提取和拆解报告。后续手动修复 db 或重跑 |
| **拆解中发现这本书质量差/不适用** | 模式B 前 20 章扫描后全部场景类型单一（如只有战斗），或写法无特色 | 输出"不推荐拆解"报告，解释为什么这本书没有参考价值。让用户决定是否强行拆解 |
| **用户要求拆自身作品** | 用户提供的是自己写的正文 | **按场景判断**：续写阶段（writer 管线内）→ 允许从自身前文提取锚定章。新书开坑阶段（bootstrap 阶段）→ 退回，拆书只拆参考书 |
| **多个模式产出物路径冲突** | 用户同时要求模式A+模式D，产出文件名可能重复 | 文件名后追加模式标识（如 `节奏地图-{书名}-{日期}-A.md`）。不相互覆盖 |

**原则**：异常先告知用户，再按规则处理。绝不静默跳过或编造拆解数据。
