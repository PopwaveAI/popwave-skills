# 拆书解构 v7.0.0

拆书的根本目的：**从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA**。
不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"**。

**默认拆书范围 = 第一卷或前100章（全读）。** 前100章全量逐章精读，确保覆盖完整的设定引入期、角色建立期和节奏成型期。仅在用户明确要求"全书拆解"时才全量走原动态密度表。

产出物供给 pop-novel-writer（正文写作）和 pop-novel-bookstrap（开书设定）管线。

# Phase 0：分层扫描（Layer 0）

## 目的
在不对全书精读的前提下，快速建立整体认知。**这一阶段的核心是"在哪里"和"大概是什么"，不是"细节"。**

## 输入文件验收（新增）
检查输入的 .txt 文件是否完整可用。如果发现中间部分大段损坏，需要下载更完整的资源。

**验收流程：**
```
1. 扫描全书结构：搜索"第X章""第X卷"标记，统计章节数、卷数
2. 中段完整性检查：在文件25%/50%/75%位置各取一段文字
   - 如果三处都可读（含中文汉字）→ ✅ 通过
   - 如果任意一处乱码/网站垃圾 → ❌ 损坏，进入下载流程
3. 结尾检查：确认有正常的全书终局标记（全书完/完结/尾声），不是截断在战斗高潮
4. 如原文件损坏：
   a. 搜索"《书名》 txt 下载 全本"或"《书名》 作者 txt"
   b. 尝试多个来源（7shutxt/bqg161/xbiqupao 等）
   c. 为每个来源提取直接的TXT下载链接
   d. 下载后重新执行上述验收流程
   e. 如果下载的可用 → 替换原文件路径；仍不可用 → 标记质量问题继续拆解
```

**下载验证脚本参考（Python）：**

## 采样策略
**默认策略（一卷/前100章全读）：**

| 范围 | 采样规则 |
|:-----|:---------|
| 全文 | 第1-100章（或第一卷全卷）：**全量逐章精读** |

全读覆盖率 = **100%**，确保：
- 设定引入节奏无遗漏（所有设定首次出现都在前100章）
- 角色建立期全覆盖（主角/反派的第一次亮相、第一次选择、第一次转变）
- 节奏成型期完整（从开篇到第一个高潮闭环）
- 采样无需高潮/随机采样辅助——全读本身已覆盖所有关键节点

范围边界：
- 默认拆解第一卷。若第一卷超100章，只拆前100章。
- 若用户明确要求全书拆解，全量走原动态密度表（见附录：全书拆解密度表）。

## 采样类型
| 类型 | 方式 | 目的 |
|------|------|------|
| **开篇采样** | 前N章全读 | 建立基线，理解初始设定和风格 |
| **间隔采样** | 每N章读1章 | 追踪主线进展，感受节奏变化 |
| **高潮采样** | 自己判断"可能的高潮"位置 | 不遗漏关键高潮节点 |
| **结尾采样** | 最后N章全读 | 理解结局和收线方式 |
| **随机采样** | 随机选N章 | 验证整体一致性，发现意外模式 |

# Phase 1：自诊断（Self-Diagnosis）

## 目的
Agent 基于 Layer 0 的阅读体验，自己判断"这本书的核心价值是什么"，并自动生成拆解任务清单。不需要人类选模式。

## 诊断内容

### 1. 赛道识别（确认 Phase 0 的判断）

### 2. 核心吸引力分析
| 维度 | 诊断问题 |
|------|---------|
| **设定**（舞台） | 这本书的设定在哪方面最突出？ |
| **剧情**（发生了什么 + 角色） | 故事和角色哪部分最抓人？ |
| **技法**（如何发生的） | 作者的写法有什么独特之处？ |

### 3. 拆解任务自动生成
**三维归属框架（v8.0.0 — T1~T7 精简结构）：**
```
设定层（舞台）—— 故事在哪里发生、靠什么运转
├── T1 力量体系/规则系统
├── T2 世界观展开
└── T3 角色系统设计（原T3+T5+T9合并升级）

剧情层（发生了什么）—— 故事说了什么
└── T4 剧情全貌

技法层（如何发生的）—— 故事怎么被写出来的
├── T5 叙事技法综合（原T6+T8+T9合并）
├── T6 数据流/系统写法（赛道触发，原T7）
└── T7 文风DNA（原T10）
```

| 任务 | 维度 | 默认检查重点 | 模板文件 |
|------|------|-------------|---------|
| T1 | 设定·力量体系/规则 | 职业/神性/静态层级/动态通胀/领域/专长 | `templates/T1-力量体系模板.md` |
| T2 | 设定·世界观展开 | 地图展开/社会/经济/政治/种族/位面/技术 | `templates/T2-世界观展开模板.md` |
| T3 | 设定·角色系统设计 | 主角驱动力/决策节点/成长弧线/情感绑定/配角生态/反派体系/关系网 | `templates/T3-角色系统设计模板.md` |
| T4 | 剧情·剧情全貌 | 主线/支线/成长线/情感线/逐卷活动表/收线评估 | `templates/T4-剧情全貌模板.md` |
| T5 | 技法·叙事技法综合 | 开篇/节奏/悬念/战斗高潮/特殊魅力，五合一综合 | `templates/T5-叙事技法综合模板.md` |
| T6 | 技法·数据流写法 | **赛道触发**：面板放置/融合策略/显示逻辑/演变/体验评估 | `templates/T6-数据流写法模板.md` |
| T7 | 技法·文风DNA | 赛道定位/6种场景类型写前规则/锚定章片段/量化参考 | `templates/T7-文风DNA模板.md` |

### 4. 优先级
| 优先级 | 标准 | 处理 |
|--------|------|------|
| P0 | 这本书最核心的价值（3~5项） | 深度拆解，独立文件，500+行 |
| P1 | 有价值但不是核心卖点（3~5项） | 中等深度，300+行 |
| P2 | 有但不突出（其余项） | 简要记录，100行即可 |
| — | 该赛道不适用 | 跳过，标注原因 |

# 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文不存在/格式不支持** | 提示用户提供 .txt 或 .md，不接受 pdf/docx |
| **字数太少（<5万字）** | 确认是否继续，继续则缩减范围 |
| **同人但无原著知识** | 先搜索原著百科/时间线（❌4红线） |
| **发现这本书质量差** | 输出"不推荐拆解"报告 |
