# 拆书解构 v6.0.0

拆书的根本目的：**从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA**。
不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"**。

产出物供给 pop-novel-writer（正文写作）和 pop-novel-bootstrap（开书设定）管线。

---

## 管线总览

```
Phase 0: 分层扫描（Layer 0）
  ├── 采样策略：开篇全读 + 间隔采样 + 高潮采样 + 结尾全读
  ├── 赛道识别：自动判断题材类型
  ├── 产出：故事骨架 + 关键节点清单 + 采样日志
  └── 产出物：{书名}-Phase0-采样日志.md
        ↓
Phase 1: 自诊断（Self-Diagnosis）
  ├── 分析 Layer 0 产出
  ├── 按设定/剧情/技法三维判断核心吸引力
  ├── 自生成拆解任务清单（T1~T9，三维全覆盖）
  ├── 产出：诊断报告 + 任务清单 + 优先级
  └── 产出物：{书名}-Phase1-诊断报告.md
        ↓
Phase 2: 精读拆解（Layer 1）
  ├── 基于关键节点，精读相关章节范围
  ├── 按 T1~T9 任务逐个独立产出
  ├── 每个任务产出独立文件，不合并压缩
  └── 产出物：{书名}-T{N}-{任务名}.md × N
        ↓
Phase 3: 验证采样（Layer 2）
  ├── 随机采样 3~5 个之前未精读的区域
  ├── 检验 Phase 2 发现的规律是否在全书中一致
  ├── 发现不一致则修订，发现新模式则补充
  └── 产出物：{书名}-Phase3-验证报告.md
        ↓
Phase 4: 整合输出
  ├── 汇总 T1~T9 所有独立产出
  ├── 提取文风DNA（5条叙事哲学原则）
  ├── 提取核心发现（3~5个最值得学的点）
  ├── 整合为赛道适配配置
  └── 产出物：{书名}-三维拆书档案.md（索引）+ 各独立文件
```

# Phase 0：分层扫描（Layer 0）

## 目的

在不对全书精读的前提下，快速建立整体认知。**这一阶段的核心是"在哪里"和"大概是什么"，不是"细节"。**

## 采样策略

根据字数动态调整密度：

| 总字数 | 采样密度 | 开篇 | 结尾 | 随机采样 |
|--------|---------|------|------|---------|
| <50万字 | 每5章/全读 | 前10章全读 | 后5章全读 | 无 |
| 50~150万字 | 每10章采1章 | 前10章全读 | 后5章全读 | 5章 |
| 150~300万字 | 每15章采1章 | 前15章全读 | 后5章全读 | 5章 |
| 300~500万字 | 每20章采1章 | 前20章全读 | 后10章全读 | 8章 |
| >500万字 | 每30章采1章 | 前30章全读 | 后10章全读 | 10章 |

## 采样类型

| 类型 | 方式 | 目的 |
|------|------|------|
| **开篇采样** | 前N章全读 | 建立基线，理解初始设定和风格 |
| **间隔采样** | 每N章读1章 | 追踪主线进展，感受节奏变化 |
| **高潮采样** | 自己判断"可能的高潮"位置 | 不遗漏关键高潮节点 |
| **结尾采样** | 最后N章全读 | 理解结局和收线方式 |
| **随机采样** | 随机选N章 | 验证整体一致性，发现意外模式 |

## 赛道识别

通读前3章，自动判断题材类型。从以下赛道知识库中匹配：

| 赛道 | 判定关键词 | 拆解关注重点 |
|------|-----------|-------------|
| 玄幻修仙 | 灵根/修炼/突破/境界 | 力量体系/升级节奏/战斗场面 |
| 都市职场 | 公司/谈判/职场/阶层 | 社会规则/人际关系/反转技巧 |
| 悬疑推理 | 案件/线索/凶手/谜题 | 线索管理/误导设计/节奏紧张感 |
| 言情耽美 | 情感/关系/误解/纠缠 | 情感曲线/心理描写/亲密尺度 |
| 西幻/DND风 | 属性/面板/职业/传奇 | 力量体系/战斗描写/设定释放 |
| 末世废土 | 丧尸/变异/生存/资源 | 生存压力/资源管理/道德抉择 |
| 系统/数据流 | 面板/经验/升级/抽奖 | 数据嵌入/成长体系/系统设计 |
| 历史/架空 | 朝堂/战争/权谋/宫廷 | 政治体系/社会结构/权谋设计 |
| 通用（无法判断） | — | 三维全覆盖，无重点偏向 |

书名：

# Phase 1：自诊断（Self-Diagnosis）

## 目的

Agent 基于 Layer 0 的阅读体验，自己判断"这本书的核心价值是什么"，并**自动生成拆解任务清单**。不需要人类选模式。

## 诊断内容

### 1. 赛道识别（确认 Phase 0 的判断）
- 确认题材类型
- 置信度
- 依据

### 2. 核心吸引力分析

分析这本书最值得拆的三个维度分别"强在哪里"：

| 维度 | 诊断问题 | 示例回答 |
|------|---------|---------|
| **设定**（舞台） | 这本书的设定在哪方面最突出？ | DND力量体系完整自洽，神性/杀戮之子/封神机制有原创性 |
| **剧情**（发生了什么 + 角色） | 故事和角色哪部分最抓人？ | 开篇薇薇安+老狗希斯的情感冲击力极强，薇薇安身份悬念贯穿全书，索伦守序邪恶的人格魅力 |
| **技法**（如何发生的） | 作者的写法有什么独特之处？ | 数据面板嵌入叙事不打断阅读、邪恶主角读者共情机制 |

### 3. 拆解任务自动生成

Agent 根据诊断结果，生成 T1~T9 任务清单。**注意：T1~T9 是固定框架，但每个任务的重点随赛道和核心吸引力动态调整。每个任务必须独立产出文件。**

**三维归属：**

```
设定层（舞台）—— 故事在哪里发生、靠什么运转
├── T1 力量体系/规则系统
├── T2 世界观展开
└── T3 主角金手指/特殊天赋

剧情层（发生了什么 + 角色）—— 故事说了什么
├── T4 剧情全貌（故事本身）
└── T5 人物/阵营

技法层（如何发生的）—— 故事怎么被写出来的
├── T6 叙事设计技法（开篇/节奏/悬念）
├── T7 数据流/系统写法
├── T8 战斗/高潮场面
└── T9 特殊魅力塑造
```

| 任务 | 维度 | 默认检查重点 | 赛道调整示例 |
|------|------|-------------|-------------|
| T1 | 设定·力量体系/规则 | 职业/神性/领域/专长 | 修仙→灵根/功法/境界 |
| T2 | 设定·世界观展开 | 地图展开/社会结构/经济/种族/位面 | 都市→社会阶层/行业规则 |
| **T3** | **设定·金手指/天赋** | **能力规则/限制/成长路径/与主线的绑定** | 系统→具体系统功能说明 |
| **T4** | **剧情·剧情全貌** | **主线/支线/成长线/情感线的全量拆解，逐卷线活动表** | **任何赛道→必做，先回答讲了什么** |
| T5 | 剧情·人物/阵营 | 主角弧线/配角功能/反派分层/关系网 | 言情→情感关系/成长阶段 |
| **T6** | **技法·叙事设计** | **开篇技法/节奏技法/悬念技法** | 悬疑→悬念设计优先 |
| T7 | 技法·数据流/系统写法 | 面板放置/信息嵌入/演变规律 | 仅数据流赛道 |
| T8 | 技法·战斗/高潮描写 | 战斗公式/分布统计/疲劳管理 | 玄幻/西幻→必做 |
| T9 | 技法·特殊魅力塑造 | 邪恶主角/反讽/黑色幽默等 | 按核心吸引力定 |

### 4. 优先级

| 优先级 | 标准 | 处理 |
|--------|------|------|
| P0 | 这本书最核心的价值（3~5项） | 深度拆解，独立文件，500+行 |
| P1 | 有价值但不是核心卖点（3~5项） | 中等深度，300+行 |
| P2 | 有但不突出（其余项） | 简要记录，100行即可 |
| — | 该赛道不适用 | 跳过，标注原因 |

## 诊断报告产出格式

```
📄 {书名}-Phase1-诊断报告.md

【赛道识别】

【核心吸引力（设定/剧情/技法三维）】

【拆解任务清单】

- ...

- ...

【预计产出文件清单】
1. {书名}-T1-力量体系手册.md

3. ...
```

# Phase 2：精读拆解（Layer 1 → T1~T9）

## 核心原则

**每个任务独立产出，不压缩合并。** T1 就是 T1，不会在"三维档案"里一笔带过。每个任务都有自己的 .md 文件。

## 子Agent 任务描述规范

**这是 v6.0.0 最重要的改动：派给子Agent的任务描述必须附完整拆解检查清单。**

### 正确写法（v6.0.0）

```
派遣子Agent拆解设定层T1力量体系：

【任务描述】
你的任务是对这本书的力量体系进行全量结构化拆解。

【必须覆盖的维度】（检查清单）
□ 职业体系（阶层结构/进阶条件/双职业规则）
□ 神性/神格系统（三要素/神格等级/领域获取/封神仪式）
□ 专长/技能系统（专长分类/获取时机/与故事的关系）
□ 施法体系（魔网机制/崩溃前/崩溃后/替代方案）
□ 杀戮经验机制（获取/转化/与节奏的关系）

【搜索范围】
在此范围章节中查找：...
```

### 错误写法

```
拆解这本书的力量体系。
```

## T1~T9 完整检查清单

### T1：力量体系/规则系统（设定层）

```
□ 职业体系（阶层结构/进阶条件/双职业规则）
□ 神性/神格系统（三要素/神格等级/领域获取/封神仪式）
□ 专长/技能系统（专长分类/获取时机）
□ 施法体系（魔网机制/崩溃前/崩溃后）
□ 杀戮经验机制（获取/转化/与节奏的关系）
□ 升级年表（每次升级的位置/变化/新能力）
□ 领域词典（所有领域名称/获取方式/效果）
□ 专长词典（所有专长名称/类型/效果）
□ 神祇数据库（所有出场的/提及的神祇）
□ 体系演变总结（各阶段的不同）
```

### T2：世界观展开（设定层）

```
□ 地图展开顺序（逐个卷/阶段记录解锁的新区域）
□ 信息释放时间表（每卷给出的新信息/隐藏的信息）
□ 社会结构（阶层/教会/商会/工会/帮派）
□ 经济体系（货币/贸易/财富流动/资源分布）
□ 政治格局（王国/势力/联盟/战争）
□ 种族体系（主要种族/关系/文化）
□ 位面法则（物质位面/深渊/冥界/星界各自规则）
□ 技术设定（奥术/魔像/密瑟能核/浮空城）
```

### T3：主角金手指/特殊天赋（设定层）← NEW

```
□ 金手指/天赋的完整规则说明（是什么、怎么用、有什么限制）
□ 获取方式（天生/获得/激活/绑定）
□ 成长路径（金手指是否进化、升级条件）
□ 与主线的绑定关系（金手指如何推动/限制剧情）
□ 限制条件（冷却/代价/反噬/失效条件）
□ 与其他设定体系的交互（金手指在力量体系中的定位）
□ 同类作品中该金手指的独特性/创新点
```

### T4：剧情全貌（剧情层）← 全量剧情线拆解

```
【核心原则：故事不是一条线走到底。一卷内有多个剧情线并行——主线可能不止一条，
还有支线、成长线、情感线等。每条线有独立的弧线、自己的契诃夫枪，线之间汇聚/交叉形成高潮。】

□ 全书一句话梗概（一条线说清楚"这本书在讲什么"）
□ 核心冲突矩阵（主角想要什么 × 什么在阻止他 × 代价是什么）

□ 全卷剧情线清单 —— 识别并列出全书所有剧情线
   每条线记录：
    ├─ 线ID（如：主线-力量/主线-身份/支线-政治/感情线-XX/成长线-主角）
    ├─ 线类型（主线/支线/成长线/感情线/世界观线/悬念线）
    ├─ 一句话定义"这条线在讲什么"
    ├─ 覆盖章节范围（起止卷/章）
    ├─ 核心问题（这条线回答哪个读者问题）
    └─ 线状态（引入→发展→高潮→解决/已解决/未解决/烂尾）

□ 逐卷剧情线活动表
    格式示例：
    卷名 | 活跃线ID列表 | 线汇聚高潮章节 | 本卷新开线 | 本卷收束线
    ----|-------------|--------------|-----------|-----------
    卷1 | 主线-生存,支线-神子,感情线-兄妹 | 第8章(主线+感情线汇聚) | 主线-生存,支线-神子 | —
    卷2 | 主线-势力,支线-身份,感情线-歌莉娅 | 第43章(三线汇聚) | 主线-势力,感情线-歌莉娅 | 主线-生存
    ...

□ 每条剧情线的独立生命周期
    格式示例（以主线-身份为例）：
    引入：卷1 第20章 —— 薇薇安可能是"神子"
    发展：卷1 第26章（术士觉醒）→ 卷1 第45章（第一滴血）→ 卷1 第92章（神子确认）
    转化：卷1 第176章 —— 莉莉安出现，线从"她是谁"变为"她们是谁"
    高潮：卷8 第13章 —— 一体双魂揭示
    解决：全书尾声 —— 恐惧双子封神，与索伦一起进入神国

□ 主线与支线的交汇关系
    - 每条支线在哪些章节与主线交汇
    - 交汇产生的效果（推动主线/提供信息/改变方向/制造障碍）
    - 无交汇的孤立线 → 标记为潜在问题

□ 成长线（主角/重要角色的能力、心智、地位的阶段性变化）
    格式示例：
    主角索伦：
      阶段1（卷1）：贫民区盗贼 → 生存驱动
      阶段2（卷2-3）：海外提督 → 扩张驱动
      阶段3（卷4-5）：半神 → 权力驱动
      阶段4（卷6-8）：神灵 → 秩序驱动
      阶段5（卷10）：死亡之主 → 法则驱动

□ 情感线（爱情/亲情/友情的情感变化轨迹）
    - 每条情感线的关键节点（初遇→深化→考验→稳定）
    - 情感线如何影响/被剧情线推动

□ 关键转折点清单
    （改变故事方向的节点——不是小高潮，是"读完这章后故事就不一样了"）
    格式示例：
    K1: 卷1第45章 —— 薇薇安第一次杀人，揭示神子天赋 → 冲突从"生存"变为"身份"
    K2: 卷3第97章 —— 索伦弑神，获得弑神者专长 → 冲突从"凡人争霸"变为"神战"
    K3: 卷8第1章 —— 魔网崩溃 → 力量体系底层重构
    ...

□ 结局收线质量评估
    - 哪些线收得好（有铺垫有回收）
    - 哪些线没收（未解决/悬空）
    - 哪些线收得勉强（工具化收尾）
    - 总体评价：收线覆盖率
```

### T5：人物/阵营（剧情层）

```
□ 主角完整档案（性格/动机/弧线/多阶段变化）
□ 配角功能分类（导师/盟友/反派/工具人/搞笑担当）
□ 角色关系网络（人与人之间的连接/冲突/联盟）
□ 反派分层（初级/中级/终极/宿命敌手）
□ 阵营规则（该书世界的善恶法则/道德边界）
□ 血统/神子/特殊身份设定
```

### T6：叙事设计技法（技法层）← 合并原T4+T5+T6

```
【注意：这里分析的是"作者怎么设计叙事"，不是复述剧情】

□ 开篇设计技法
   └ 前1000字分析（情感钩子四层拆解）
   └ 第1~30章逐章功能标记（每章在结构中的作用）
   └ 世界信息分层引入表（每章释放了什么/隐藏了什么）
   └ 冲突升级路径（生存→个人→群体→超自然→命运）
   └ 节奏张力曲线（逐章张力打分，标注松弛点）
   └ 开篇做出的承诺清单（短期/中期/长期回报周期）

□ 长篇节奏技法
   └ 逐卷核心冲突表（每卷的核心矛盾）
   └ 高潮分布图（全书的S/A/B级高潮位置）
   └ 卷结构规划（各卷字数/章节数/功能）
   └ 嵌套节奏分析（大/中/小节奏单元）
   └ 节奏特征总结（双峰/三幕/多浪/平稳等）

□ 悬念管理技法
   └ 主悬念生命周期（引入→维持→转化→解决的时间线）
   └ 全体次要悬念追踪（每个悬念的独立生命周期）
   └ 悬念密度漏斗（各卷活跃悬念数量变化）
   └ 悬念类型分布（身份/能力/关系/结局四类占比）
   └ 悬念回收质量评估（哪些回收得好/哪些烂尾）
```

### T7：数据流/系统写法（技法层）

□ 面板放置规律（位置/长度/占章节比例统计）

□ 面板技法演变（各阶段的差异）

# Phase 3：验证采样（Layer 2）

## 目的

检验 Phase 2 的发现是否具有**全书的代表性**。上一轮验证发现的典型问题需要在此阶段纠正。

## 采样规则

| 采样类型 | 数量 | 目的 |
|----------|------|------|
| 随机采样 | 3~5个未精读区域 | 验证核心发现的一致性 |
| 边缘采样 | 1~2个"可能不适用"区域 | 测试发现的边界条件 |
| 类型采样 | 每种战斗/对话/场景类型至少1例 | 验证类型公式的普遍性 |

## 验证 Checklist

```
□ Phase 2 发现的规律在采样章节中成立？
□ 是否有意外的新模式出现？
□ Phase 2 的结论是否需要修订？
   - 成立（原结论不变）
   - 需修订（标注具体修订内容）
   - 推翻（说明原因并提供替代解释）
```

## 产出格式

```
📄 {书名}-Phase3-验证报告.md

【验证章节清单】
1. 第X~X章（卷名）：[选择原因]
2. ...

【验证结果】
结论1：[原始结论]
  □ 验证通过 ✅
  □ 需修订 ⚠️ → 修订为：[新结论]
  □ 不成立 ❌ → 替代解释：[...]

结论2：...
...

【新增发现】
如验证中发现新模式，记录在此。
```

# 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文文件不存在/格式不支持** | 提示用户提供 .txt 或 .md 格式。不接受 pdf/docx |
| **字数太少（<5万字）** | 拆解价值有限。与用户确认是否继续。继续则缩减范围 |
| **同人题材但无原著知识** | 先搜索原著百科/时间线（❌4红线） |
| **Phase 2 中发现这本书质量差** | 输出"不推荐拆解"报告，让用户决定 |
