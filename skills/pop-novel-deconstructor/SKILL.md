# 拆书解构 v6.0.0

拆书的根本目的：**从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA**。
不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"**。

产出物供给 pop-novel-writer（正文写作）和 pop-novel-bootstrap（开书设定）管线。

---

## 管线总览

```
Phase 0: 分层扫描（Layer 0）
  ├── 采样策略：开篇全读 + 间隔采样 + 高潮采样 + 结尾全读
  ├── 赛道识别：自动判断题材类型
  ├── 产出：故事骨架 + 关键节点清单 + 采样日志
  └── 产出物：{书名}-Phase0-采样日志.md
        ↓
Phase 1: 自诊断（Self-Diagnosis）
  ├── 分析 Layer 0 产出
  ├── 按设定/剧情/技法三维判断核心吸引力
  ├── 自生成拆解任务清单（T1~T9，三维全覆盖）
  ├── 产出：诊断报告 + 任务清单 + 优先级
  └── 产出物：{书名}-Phase1-诊断报告.md
        ↓
Phase 2: 精读拆解（Layer 1）
  ├── 基于关键节点，精读相关章节范围
  ├── 按 T1~T9 任务逐个独立产出
  ├── 每个任务产出独立文件，不合并压缩
  └── 产出物：{书名}-T{N}-{任务名}.md × N
        ↓
Phase 3: 验证采样（Layer 2）
  ├── 随机采样 3~5 个之前未精读的区域
  ├── 检验 Phase 2 发现的规律是否在全书中一致
  ├── 发现不一致则修订，发现新模式则补充
  └── 产出物：{书名}-Phase3-验证报告.md
        ↓
Phase 4: 整合输出
  ├── 汇总 T1~T9 所有独立产出
  ├── 提取文风DNA（5条叙事哲学原则）
  ├── 提取核心发现（3~5个最值得学的点）
  ├── 整合为赛道适配配置
  └── 产出物：{书名}-三维拆书档案.md（索引）+ 各独立文件
```

# Phase 0：分层扫描（Layer 0）

## 目的
在不对全书精读的前提下，快速建立整体认知。**这一阶段的核心是"在哪里"和"大概是什么"，不是"细节"。**

## 采样策略
根据字数动态调整密度：

| 总字数 | 采样密度 | 开篇 | 结尾 | 随机采样 |
|--------|---------|------|------|---------|
| <50万字 | 每5章/全读 | 前10章全读 | 后5章全读 | 无 |
| 50~150万字 | 每10章采1章 | 前10章全读 | 后5章全读 | 5章 |
| 150~300万字 | 每15章采1章 | 前15章全读 | 后5章全读 | 5章 |
| 300~500万字 | 每20章采1章 | 前20章全读 | 后10章全读 | 8章 |
| >500万字 | 每30章采1章 | 前30章全读 | 后10章全读 | 10章 |

## 采样类型
| 类型 | 方式 | 目的 |
|------|------|------|
| **开篇采样** | 前N章全读 | 建立基线，理解初始设定和风格 |
| **间隔采样** | 每N章读1章 | 追踪主线进展，感受节奏变化 |
| **高潮采样** | 自己判断"可能的高潮"位置 | 不遗漏关键高潮节点 |
| **结尾采样** | 最后N章全读 | 理解结局和收线方式 |
| **随机采样** | 随机选N章 | 验证整体一致性，发现意外模式 |

## 赛道识别
通读前3章，自动判断题材类型：

| 赛道 | 判定关键词 | 拆解关注重点 |
|------|-----------|-------------|
| 玄幻修仙 | 灵根/修炼/突破/境界 | 力量体系/升级节奏/战斗场面 |
| 都市职场 | 公司/谈判/职场/阶层 | 社会规则/人际关系/反转技巧 |
| 悬疑推理 | 案件/线索/凶手/谜题 | 线索管理/误导设计/节奏紧张感 |
| 言情耽美 | 情感/关系/误解/纠缠 | 情感曲线/心理描写/亲密尺度 |
| 西幻/DND风 | 属性/面板/职业/传奇 | 力量体系/战斗描写/设定释放 |
| 末世废土 | 丧尸/变异/生存/资源 | 生存压力/资源管理/道德抉择 |
| 系统/数据流 | 面板/经验/升级/抽奖 | 数据嵌入/成长体系/系统设计 |
| 历史/架空 | 朝堂/战争/权谋/宫廷 | 政治体系/社会结构/权谋设计 |
| 通用（无法判断） | — | 三维全覆盖，无重点偏向 |

## 产出格式
```
📄 {书名}-Phase0-采样日志.md

【基本信息】书名/作者/总字数/总章节数/卷数/赛道识别
【采样日志】卷名 | 章节范围 | 采样章节 | 核心事件 | 阅读感受 | 新信息获取
【关键节点清单】K1: 第X章（事件）→ 需要精读 ...
【初步故事骨架】整体结构 / 阶段识别 / 核心冲突
```

# Phase 1：自诊断（Self-Diagnosis）

## 目的
Agent 基于 Layer 0 的阅读体验，自己判断"这本书的核心价值是什么"，并自动生成拆解任务清单。不需要人类选模式。

## 诊断内容

### 1. 赛道识别（确认 Phase 0 的判断）

### 2. 核心吸引力分析
| 维度 | 诊断问题 |
|------|---------|
| **设定**（舞台） | 这本书的设定在哪方面最突出？ |
| **剧情**（发生了什么 + 角色） | 故事和角色哪部分最抓人？ |
| **技法**（如何发生的） | 作者的写法有什么独特之处？ |

### 3. 拆解任务自动生成
**三维归属框架：**
```
设定层（舞台）—— 故事在哪里发生、靠什么运转
├── T1 力量体系/规则系统
├── T2 世界观展开
└── T3 主角金手指/特殊天赋

剧情层（发生了什么 + 角色）—— 故事说了什么
├── T4 剧情全貌（故事本身）
└── T5 人物/阵营

技法层（如何发生的）—— 故事怎么被写出来的
├── T6 叙事设计技法（开篇/节奏/悬念）
├── T7 数据流/系统写法
├── T8 战斗/高潮场面
└── T9 特殊魅力塑造
```

| 任务 | 维度 | 默认检查重点 | 模板文件 |
|------|------|-------------|---------|
| T1 | 设定·力量体系/规则 | 职业/神性/静态层级/动态通胀/领域/专长 | `templates/T1-力量体系模板.md` |
| T2 | 设定·世界观展开 | 地图展开/社会/经济/政治/种族/位面/技术 | `templates/T2-世界观展开模板.md` |
| T3 | 设定·金手指/天赋 | 规则/限制/成长/与主线绑定/叙事功能 | `templates/T3-金手指天赋模板.md` |
| T4 | 剧情·剧情全貌 | 主线/支线/成长线/情感线/逐卷活动表/收线评估 | `templates/T4-剧情全貌模板.md` |
| T5 | 剧情·人物/阵营 | 主角档案/配角分类7种/关系网/反派4层 | `templates/T5-人物阵营模板.md` |
| T6 | 技法·叙事设计 | 开篇/节奏/悬念三模块，每模块含子项清单 | `templates/T6-叙事设计技法模板.md` |
| T7 | 技法·数据流/系统写法 | 面板放置/融合策略/显示逻辑/演变/体验评估 | `templates/T7-数据流写法模板.md` |
| T8 | 技法·战斗/高潮描写 | 6种类型公式/全本统计/通用公式/疲劳管理 | `templates/T8-战斗高潮模板.md` |
| T9 | 技法·特殊魅力塑造 | 同情机制/道德边界/关怀系统/升级路径/可复制性 | `templates/T9-特殊魅力模板.md` |

### 4. 优先级
| 优先级 | 标准 | 处理 |
|--------|------|------|
| P0 | 这本书最核心的价值（3~5项） | 深度拆解，独立文件，500+行 |
| P1 | 有价值但不是核心卖点（3~5项） | 中等深度，300+行 |
| P2 | 有但不突出（其余项） | 简要记录，100行即可 |
| — | 该赛道不适用 | 跳过，标注原因 |

## 诊断报告产出格式
```
📄 {书名}-Phase1-诊断报告.md

【赛道识别】类型/置信度/依据
【核心吸引力】设定/剧情/技法三维
【拆解任务清单】P0/P1/P2 + 对应模板文件路径
【预计产出文件清单】{书名}-T{N}-{任务名}.md
```

# Phase 2：精读拆解（Layer 1 → T1~T9）

## 核心原则
**每个任务独立产出，不合并压缩。** 每个任务有自己的 .md 文件。

## 子Agent 任务描述规范
**派给子Agent的任务描述必须引用模板文件中的完整检查清单。**

### 正确写法
```
你的任务是对这本书的力量体系进行全量结构化拆解。
请严格按以下模板中的检查清单逐项填写：
📄 skills/pop-novel-deconstructor/templates/T1-力量体系模板.md

【搜索范围】第X~X章
【产出格式】结构化Markdown，每项检查清单填充示例格式的内容
```

### 错误写法
```
拆解这本书的力量体系。
```

## 模板引用路径
```
skills/pop-novel-deconstructor/templates/T1-力量体系模板.md
```

# Phase 3：验证采样（Layer 2）

## 目的
检验 Phase 2 的发现是否具有全书的代表性。

## 采样规则
| 采样类型 | 数量 | 目的 |
|----------|------|------|
| 随机采样 | 3~5个未精读区域 | 验证核心发现的一致性 |
| 边缘采样 | 1~2个"可能不适用"区域 | 测试发现的边界条件 |
| 类型采样 | 每种场景类型至少1例 | 验证类型公式的普遍性 |

## 产出格式
```
📄 {书名}-Phase3-验证报告.md

【验证章节清单】...
【验证结果】结论1: ✅通过 / ⚠️需修订 / ❌不成立
【新增发现】...
```

# 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文不存在/格式不支持** | 提示用户提供 .txt 或 .md，不接受 pdf/docx |
| **字数太少（<5万字）** | 确认是否继续，继续则缩减范围 |
| **同人但无原著知识** | 先搜索原著百科/时间线（❌4红线） |
| **发现这本书质量差** | 输出"不推荐拆解"报告 |
