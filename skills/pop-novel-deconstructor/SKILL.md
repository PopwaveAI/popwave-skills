# Book-Deconstructor v4.8 — 一体化拆书 × 五模式设计

> **定位**：拆书是"配钥匙"——从优秀作品中提取可复用的写法规则、节奏模板、设定逻辑和事件密度模式，而非简单摘要。
>
> **核心原则**：不输出"这本书讲了什么"，只输出"这本书为什么好看"以及"我可以用什么手法达到类似效果"。

---

## 协作边界

> ⚠️ **拆书定位声明：** 本 skill（book-deconstructor）是 **novel-agent-pro 内部拆书引擎**，面向"拆书为写"——产出供给 pop-novel-writer 的 14 项输入包和 scene_fragments.db。
>
> 同级 skill `pop-reader-making` 是 **独立拆书管线**，面向"拆书为读"——产出供读者查阅的叙事笔记 and 供 pop-html-anything 渲染的结构化 YAML。
>
> **分工：**
> - book-deconstructor → 消费方：pop-novel-writer / pop-novel-bootstrap → 产出：scene_fragments.db + 节奏地图 + 对标规则
> - pop-reader-making → 消费方：读者 / pop-html-anything → 产出：叙事笔记.md + 结构化YAML（含 image_prompt）

## 版本概要

**v4.8 核心升级**：
- **新增模式D「体系拆解」**：从卷级数据采集到跨卷演化分析，专攻设定逻辑归纳与诡异分级体系
- **新增模式E「大纲密度拆解」**：以大纲/章纲为素材，分析事件密度、节奏分布与爽点间隔
- 所有模式产出物统一路径至 `01-写作资产/拆解报告/`
- 引入质量门规则（P0/P1/P2），确保产出物可用性

## 五模式选型决策树

在开始拆解之前，先确认你要用哪种模式。以下决策树帮助你快速选型：

```
节点E·拆解前思考
│
├─ 你有完整的小说全文（txt）吗？
│   ├─ YES → 进入节点1
│   └─ NO  → 进入节点3
│
├─ 节点1：你的目的是什么？
│   ├─ 想画整本书的节奏地图，看高潮分布 → 模式A「结构拆解」
│   ├─ 想学习作者的写法技巧、提取可复用规则 → 模式B「对标拆解」
│   ├─ 只想提取某一类特定内容（如：打斗场景/情感戏/世界观描写）→ 模式C「主题定向提取」
│   ├─ 想做设定层面的逻辑归纳（诡异体系/成长体系）→ 模式D「体系拆解」
│   └─ 以上多个需求 → 组合模式（A+B / A+D / B+D 等）
│
├─ 节点2：你有大纲或章纲文件吗？
│   ├─ YES → 想分析事件密度和节奏分布 → 模式E「大纲密度拆解」
│   └─ NO  → 需要先获取素材
│
└─ 节点3：你的素材只有大纲/章纲，没有正文？
    └─ 只有这个素材 → 模式E「大纲密度拆解」
```

## 模式A：结构拆解（画节奏地图）

> **目的**：为整本书画出节拍序列与情绪曲线，定位高潮、低谷、过渡段落的分布规律。

### 核心理念

结构拆解不是做章节摘要，而是标记**功能节点**。将每个章节定位为一个或多个"节拍"，每个节拍带有功能标签和强度评分。

### 执行流程（4步）

| 步骤 | 操作 | 产出 |
|------|------|------|
| **Step 1** | 全书通读，标记节拍转折点 | 原始节拍标注 |
| **Step 2** | 每章提取情绪锚（兴奋度/悬念度/信息密度）并打分（1-10） | 情绪锚数据表 |
| **Step 3** | 按卷绘制情绪曲线，识别"高潮区""平原区""低谷区" | 情绪曲线图（文字描述） |
| **Step 4** | 总结节拍模式——高潮间距规律、铺垫节奏、收尾模式 | 节奏地图报告 |

### 产出物格式

文件路径：`01-写作资产/拆解报告/节奏地图-{书名}-{日期}.md`

内容结构：
- 全书节拍一览表（章节号 / 功能标签 / 情绪强度 / 字数）
- 分卷情绪曲线（文字描述版）
- 节拍模式总结（高潮间距 / 铺垫模式 / 收尾模式）
- 可复用节奏模板（供下游章纲使用）

---

## 模式B：对标拆解（配钥匙）

> **目的**：从优秀作品中提取"写法规则"——非预设维度，LLM 自动识别场景类型，基于多维度提取规则，经过新颖性去重后入库。

### 核心理念

"配钥匙"的比喻：不关心锁长什么样，只关心这把钥匙能开什么锁。
- 不预设分析维度（避免思维定式）
- LLM 自动识别场景类型（战斗/情感/心理/世界观透露/过渡……）
- 每个场景自动提取多维度规则（句式/节奏/信息释放/情绪控制……）
- 多源片段聚合 → 新颖性去重 → 规则入库

### 执行流程

| 步骤 | 操作 | 产出 |
|------|------|------|
| **Step 1** | 全书扫描，LLM 自动切分场景并分类 | 场景类型清单 |
| **Step 2** | 对每个场景提取写法规则（句式特征 / 节奏控制 / 信息释放时机 / 情绪锚点） | 多维度规则列表 |
| **Step 3** | 跨场景新颖性去重，保留差异化规则 | 去重后规则集 |
| **Step 4** | 规则 + 多源片段聚合 → 入库 | 规则库条目 + 质检条目 |

### 产出物格式

文件路径：`01-写作资产/拆解报告/对标拆解-{书名}-{日期}.md`

内容结构：
- 场景类型分布统计
- 各场景类型的核心写法规则（含原文片段引用）
- 新颖性去重说明
- 质检条目（可供下游 QA 使用）
- 回流至宪法的建议条款

## 模式C：主题定向提取

> **目的**：当不需要全量拆解，只想提取某一类特定内容时使用（如：只提取打斗场景的写法，只分析感情线的节奏）。

### 使用方式

1. 在 pop 声明中明确目标主题（如："只提取悬疑氛围营造手法"）
2. 系统全局搜索包含该主题的场景片段
3. 仅对匹配场景执行模式B的规则提取流程
4. 产出定向主题报告

### 产出物格式

文件路径：`01-写作资产/拆解报告/主题提取-{主题}-{书名}-{日期}.md`

内容结构：
- 主题定义与搜索条件
- 匹配场景清单
- 主题相关写法规则
- 片段示例集合

---

## 模式D：体系拆解（设定逻辑归纳）

> **目的**：从设定层面拆解作品的底层逻辑——诡异/能力/力量体系的分级逻辑、主角成长节奏、跨卷演化路径。适合世界观驱动型作品。

### 执行流程（4步）

| 步骤 | 操作 | 产出 |
|------|------|------|
| **Step 1 — 卷级数据采集** | 按卷扫描，收集每卷出现的设定元素（诡异实体 / 能力等级 / 关键道具 / 势力关系） | 卷级设定数据表 |
| **Step 2 — 诡异分级体系归纳** | 归纳诡异/力量体系的层级关系、晋升条件、克制链条 | 诡异分级体系图（文字版） |
| **Step 3 — 成长节奏提取** | 标记主角各阶段的实力等级、突破节点、能力获取时间线 | 成长节奏时间线 |
| **Step 4 — 跨卷演化分析** | 分析设定体系在各卷之间的演化——哪些设定被抛弃、哪些被强化、哪些被修正 | 设定演化分析报告 |

### 产出物格式

文件路径：`01-写作资产/拆解报告/设定-{书名}-{日期}.md`

内容结构：
- 卷级数据采集总表
- 诡异分级体系（层级结构 / 晋升条件 / 克制关系）
- 成长节奏时间线（主角实力 vs 剧情进度对照）
- 跨卷演化分析
- 可供下游使用的设定模板

---

## 模式E：大纲密度拆解

> **目的**：当素材不是正文而是大纲/章纲时，分析事件密度、节奏分布、爽点间隔。适合在开书前对标竞品大纲结构。

### 执行流程

| 步骤 | 操作 | 产出 |
|------|------|------|
| **Step 1** | 按章扫描大纲条目，标记事件类型（铺垫/冲突/反转/高潮/收尾） | 事件类型标记表 |
| **Step 2** | 计算事件密度（每千字事件数）和爽点间隔（章节间距） | 密度与间隔数据 |
| **Step 3** | 绘制节奏曲线（基于事件强度+类型） | 大纲节奏曲线 |
| **Step 4** | 与模式A/模式B产出交叉比对，提出结构优化建议 | 密度优化建议 |

### 产出物格式

文件路径：`01-写作资产/拆解报告/大纲-{书名}-{日期}.md`

内容结构：
- 事件类型分布统计
- 事件密度热力图（按卷/按章节）
- 爽点间隔分析
- 结构优化建议

## 质量门规则

所有产出物在交付前必须经过以下质量门过滤：

| 等级 | 定义 | 处理方式 |
|------|------|----------|
| **P0（保留）** | 直接可用，规则清晰，有片段佐证 | 正常入库，写入最终报告 |
| **P1（暂存）** | 有一定价值但不够完整，需要补充素材 | 放入暂存区，标记待补 |
| **P2（丢弃）** | 泛泛而谈、与其他规则重复、无法验证 | 直接丢弃，不进报告 |
