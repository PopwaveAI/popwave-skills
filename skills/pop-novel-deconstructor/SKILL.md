# 拆书解构 v6.0.0

拆书的根本目的：**从优秀作品中提取可复用的写法规则、节奏模板、体系设计和文风DNA**。
不只是复述剧情，不只是记流水账。**每个产出都必须回答"这个手法好在哪、我怎么用"**。

产出物供给 pop-novel-writer（正文写作）和 pop-novel-bootstrap（开书设定）管线。

---

## 管线总览

```
Phase 0: 分层扫描（Layer 0）
  ├── 采样策略：开篇全读 + 间隔采样 + 高潮采样 + 结尾全读
  ├── 赛道识别：自动判断题材类型
  ├── 产出：故事骨架 + 关键节点清单 + 采样日志
  └── 产出物：{书名}-Phase0-采样日志.md
        ↓
Phase 1: 自诊断（Self-Diagnosis）
  ├── 分析 Layer 0 产出
  ├── 按设定/剧情/技法三维判断核心吸引力
  ├── 自生成拆解任务清单（T1~T9，三维全覆盖）
  ├── 产出：诊断报告 + 任务清单 + 优先级
  └── 产出物：{书名}-Phase1-诊断报告.md
        ↓
Phase 2: 精读拆解（Layer 1）
  ├── 基于关键节点，精读相关章节范围
  ├── 按 T1~T9 任务逐个独立产出
  ├── 每个任务产出独立文件，不合并压缩
  └── 产出物：{书名}-T{N}-{任务名}.md × N
        ↓
Phase 3: 验证采样（Layer 2）
  ├── 随机采样 3~5 个之前未精读的区域
  ├── 检验 Phase 2 发现的规律是否在全书中一致
  ├── 发现不一致则修订，发现新模式则补充
  └── 产出物：{书名}-Phase3-验证报告.md
        ↓
Phase 4: 整合输出
  ├── 汇总 T1~T9 所有独立产出
  ├── 提取文风DNA（5条叙事哲学原则）
  ├── 提取核心发现（3~5个最值得学的点）
  ├── 整合为赛道适配配置
  └── 产出物：{书名}-三维拆书档案.md（索引）+ 各独立文件
```

# Phase 0：分层扫描（Layer 0）

## 目的

在不对全书精读的前提下，快速建立整体认知。**这一阶段的核心是"在哪里"和"大概是什么"，不是"细节"。**

## 采样策略

根据字数动态调整密度：

| 总字数 | 采样密度 | 开篇 | 结尾 | 随机采样 |
|--------|---------|------|------|---------|
| <50万字 | 每5章/全读 | 前10章全读 | 后5章全读 | 无 |
| 50~150万字 | 每10章采1章 | 前10章全读 | 后5章全读 | 5章 |
| 150~300万字 | 每15章采1章 | 前15章全读 | 后5章全读 | 5章 |
| 300~500万字 | 每20章采1章 | 前20章全读 | 后10章全读 | 8章 |
| >500万字 | 每30章采1章 | 前30章全读 | 后10章全读 | 10章 |

## 采样类型

| 类型 | 方式 | 目的 |
|------|------|------|
| **开篇采样** | 前N章全读 | 建立基线，理解初始设定和风格 |
| **间隔采样** | 每N章读1章 | 追踪主线进展，感受节奏变化 |
| **高潮采样** | 自己判断"可能的高潮"位置 | 不遗漏关键高潮节点 |
| **结尾采样** | 最后N章全读 | 理解结局和收线方式 |
| **随机采样** | 随机选N章 | 验证整体一致性，发现意外模式 |

## 赛道识别

通读前3章，自动判断题材类型。从以下赛道知识库中匹配：

| 赛道 | 判定关键词 | 拆解关注重点 |
|------|-----------|-------------|
| 玄幻修仙 | 灵根/修炼/突破/境界 | 力量体系/升级节奏/战斗场面 |
| 都市职场 | 公司/谈判/职场/阶层 | 社会规则/人际关系/反转技巧 |
| 悬疑推理 | 案件/线索/凶手/谜题 | 线索管理/误导设计/节奏紧张感 |
| 言情耽美 | 情感/关系/误解/纠缠 | 情感曲线/心理描写/亲密尺度 |
| 西幻/DND风 | 属性/面板/职业/传奇 | 力量体系/战斗描写/设定释放 |
| 末世废土 | 丧尸/变异/生存/资源 | 生存压力/资源管理/道德抉择 |
| 系统/数据流 | 面板/经验/升级/抽奖 | 数据嵌入/成长体系/系统设计 |
| 历史/架空 | 朝堂/战争/权谋/宫廷 | 政治体系/社会结构/权谋设计 |
| 通用（无法判断） | — | 三维全覆盖，无重点偏向 |

## 产出格式

```
📄 {书名}-Phase0-采样日志.md

【基本信息】
书名： 
作者： 
总字数： 
总章节数：

【关键节点清单】
K1: 第X章（{事件名}）→ 疑似高潮/转折，需要 Layer 1 精读
K2: ...

【初步故事骨架】
整体结构：
```

# Phase 1：自诊断（Self-Diagnosis）

## 目的

Agent 基于 Layer 0 的阅读体验，自己判断"这本书的核心价值是什么"，并**自动生成拆解任务清单**。不需要人类选模式。

## 诊断内容

### 1. 赛道识别（确认 Phase 0 的判断）
- 确认题材类型
- 置信度
- 依据

### 2. 核心吸引力分析

分析这本书最值得拆的三个维度分别"强在哪里"：

| 维度 | 诊断问题 | 示例回答 |
|------|---------|---------|
| **设定**（舞台） | 这本书的设定在哪方面最突出？ | DND力量体系完整自洽，神性/杀戮之子/封神机制有原创性 |
| **剧情**（发生了什么 + 角色） | 故事和角色哪部分最抓人？ | 开篇薇薇安+老狗希斯的情感冲击力极强，薇薇安身份悬念贯穿全书，索伦守序邪恶的人格魅力 |
| **技法**（如何发生的） | 作者的写法有什么独特之处？ | 数据面板嵌入叙事不打断阅读、邪恶主角读者共情机制 |

### 3. 拆解任务自动生成

Agent 根据诊断结果，生成 T1~T9 任务清单。**注意：T1~T9 是固定框架，但每个任务的重点随赛道和核心吸引力动态调整。每个任务必须独立产出文件。**

**三维归属：**

```
设定层（舞台）—— 故事在哪里发生、靠什么运转
├── T1 力量体系/规则系统
├── T2 世界观展开
└── T3 主角金手指/特殊天赋

剧情层（发生了什么 + 角色）—— 故事说了什么
├── T4 剧情全貌（故事本身）
└── T5 人物/阵营

技法层（如何发生的）—— 故事怎么被写出来的
├── T6 叙事设计技法（开篇/节奏/悬念）
├── T7 数据流/系统写法
├── T8 战斗/高潮场面
└── T9 特殊魅力塑造
```

| 任务 | 维度 | 默认检查重点 | 赛道调整示例 |
|------|------|-------------|-------------|
| T1 | 设定·力量体系/规则 | 职业/神性/领域/专长 | 修仙→灵根/功法/境界 |
| T2 | 设定·世界观展开 | 地图展开/社会结构/经济/种族/位面 | 都市→社会阶层/行业规则 |
| T3 | **设定·金手指/天赋** | **能力规则/限制/成长路径/与主线的绑定** | 系统→具体系统功能说明 |
| T4 | **剧情·剧情全貌** | **主线/支线/成长线/情感线的全量拆解，逐卷线活动表** | **任何赛道→必做，先回答讲了什么** |
| T5 | 剧情·人物/阵营 | 主角弧线/配角功能/反派分层/关系网 | 言情→情感关系/成长阶段 |
| T6 | **技法·叙事设计** | **开篇技法/节奏技法/悬念技法** | 悬疑→悬念设计优先 |
| T7 | 技法·数据流/系统写法 | 面板放置/信息嵌入/演变规律 | 仅数据流赛道 |
| T8 | 技法·战斗/高潮描写 | 战斗公式/分布统计/疲劳管理 | 玄幻/西幻→必做 |
| T9 | 技法·特殊魅力塑造 | 邪恶主角/反讽/黑色幽默等 | 按核心吸引力定 |

### 4. 优先级

| 优先级 | 标准 | 处理 |
|--------|------|------|
| P0 | 这本书最核心的价值（3~5项） | 深度拆解，独立文件，500+行 |
| P1 | 有价值但不是核心卖点（3~5项） | 中等深度，300+行 |
| P2 | 有但不突出（其余项） | 简要记录，100行即可 |
| — | 该赛道不适用 | 跳过，标注原因 |

## 诊断报告产出格式

```
📄 {书名}-Phase1-诊断报告.md

【赛道识别】
类型：
置信度：

【核心吸引力（设定/剧情/技法三维）】

技法（写法）突出点：

【拆解任务清单】
P0:
- T{N}: {任务名} → 独立文件
- T{N}: {任务名} → 独立文件
P1:
- ...
P2:
- ...

【预计产出文件清单】
1. {书名}-T1-力量体系规则手册.md
2. {书名}-T2-世界观展开.md
3. ...
```

# Phase 2：精读拆解（Layer 1 → T1~T9）

## 核心原则

**每个任务独立产出，不压缩合并。** T1 就是 T1，不会在"三维档案"里一笔带过。每个任务都有自己的 .md 文件。

## 子Agent 任务描述规范

**这是 v6.0.0 最重要的改动：派给子Agent的任务描述必须附完整拆解检查清单。**

### 正确写法（v6.0.0）

```
派遣子Agent拆解设定层T1力量体系：

【任务描述】
你的任务是对这本书的力量体系进行全量结构化拆解。

【必须覆盖的维度】（检查清单）
□ 职业体系（阶层结构/进阶条件/双职业规则）
□ 神性/神格系统（三要素/神格等级/领域获取/封神仪式）
□ 专长/技能系统（专长分类/获取时机/与故事的关系）
□ 施法体系（魔网机制/崩溃前/崩溃后/替代方案）
□ 杀戮经验机制（获取/转化/与节奏的关系）

【搜索范围】
在此范围章节中查找：...

【产出格式】
输出为结构化Markdown文档，每项检查清单给出示例格式的内容。
```

### 错误写法

```
拆解这本书的力量体系。
```

## T1~T9 完整检查清单

### T1：力量体系/规则系统（设定层）

```
【核心原则：不只是记录"有什么规则"，要梳理"规则之间的逻辑关系"]
         ——如"灵根决定修炼速度"和"修炼速度决定社会地位"之间的因果链】

□ 力量体系概览
   - 该世界有几个独立的力量体系？（如：东方修仙体系 vs 西方魔法体系）
   - 各体系之间的交互/冲突/压制关系
   - 体系在全文中的优先性和出现频率

□ 职业/阶位体系
   - 阶层结构：完整等级划分（如：一阶~九阶/凡人→半神→真神）
   - 进阶条件：每阶需要什么条件（经验/血脉/资源/仪式）
   - 进阶效果：每阶解锁的新能力、属性增幅
   - 兼职/双职业规则：允许几种职业同时修炼？冲突点在哪？
   - 升级年表：主角及关键角色每次升级的位置/章节/变化/新能力

□ 属性/数值系统（如存在）
   - 六维/多维属性定义（力量/敏捷/智力等的作用）
   - 属性增长机制（升级加/装备加/特殊事件加）
   - 属性阈值（多少点进入超凡领域）
   - 面板数据结构：什么数据总是显示/选择性显示/始终不显示

□ 施法/能量系统
   - 能量来源（魔力/灵气/神力/内力等）
   - 施法机制（记忆法术/即兴施法/蓝条施法）
   - 系统崩溃/变革（如魔网崩溃）及其叙事影响
   - 替代方案（系统崩溃后角色如何应对）

□ 经验/成长系统
   - 经验获取方式（击杀/任务/修炼/感悟）
   - 经验转化规则（怎么花、升级曲线）
   - 经验与故事节奏的匹配（早期升级快 vs 后期漫长积累）

□ 装备/物品系统
   - 物品等级划分（普通→超凡→传奇→神器）
   - 装备的获取/升级/绑定机制
   - 关键装备清单及其在剧情中的作用

□ 特殊规则（领域/神性/法则等）
   - 领域词典：所有领域名称/获取方式/效果
   - 神性/神格/神力三要素的关系
   - 神格等级划分及对应的能力解锁
   - 封神仪式/修行突破的完整流程
   - 体系演变总结：随故事推进，力量规则本身发生了什么变化

□ 专长/技能词典
   - 所有专长分类（个人/职业/传奇/史诗）
   - 获取时机（升级/事件/领悟）
   - 与故事的关系（专长如何在关键时刻扭转战局）

□ 神祇数据库（如存在）
   - 所有出场/提及的神祇：名称/神格等级/神职/阵营/关键行动
   - 神系结构：各大神系的关系网
```

### T2：世界观展开（设定层）

```
【核心原则：世界观不是一次性给出来的，是按节奏逐步展开的。
         记录"每卷解锁了什么新区域/新信息"，以及"什么信息被刻意隐藏了"】

□ 地图展开顺序
    格式示例：
    阶段 | 解锁区域 | 章节范围 | 叙事功能
    ----|---------|---------|---------
    卷1 | 琥珀城贫民区 | 1-187章 | 微型视角切入，让读者从最底层了解世界
    卷2 | 海外群岛 | 1-167章 | 离开大陆文明踏入蛮荒
    卷4末 | 无底深渊 | 第92章起 | 第一次离开物质位面
    卷5 | 印记城 | 第63章 | 揭示多元宇宙终极城市
    卷8 | 冥界 | 第15章 | 终极阶层解锁
    - 核心规律：每个新地图都比前一层更宏大，同心圆式展开

□ 信息释放时间表
    格式示例：
    卷/章 | 释放的新信息 | 隐藏的信息 | 释放方式
    ------|-------------|----------|---------
    卷1 第2章 | 属性面板、职业体系、穿越真相 | 穿越原因 | 主角苏醒时自动浮现
    卷1 第3章 | 诸神体系、圣者浩劫预兆 | 浩劫具体内容 | NPC对话
    卷1 第10章 | 剥皮事件、邪神信徒 | 恐惧神子 | 事件发生
    ...
    - 核心规律：每2-3章释放一个核心设定，通过"主角感知/回忆/推算"自然暴露
    - 隐藏机制：始终保持"平民视角"限制信息获取量

□ 社会结构
   - 阶层体系（皇室→贵族→平民→奴隶 或 宗门→散修→凡人）
   - 教会/宗教组织（教廷结构、神灵信仰体系）
   - 商会/经济组织（贸易线路、行会、商业联盟）
   - 工会/职业组织（冒险者工会、法师协会、盗贼公会）
   - 帮派/地下势力（黑市、暗杀组织、情报网络）

□ 经济体系
   - 货币体系（铜银金/灵石/信用点等）
   - 贸易流动（核心贸易路线、稀缺资源分布）
   - 财富等级（主角的财富变化轨迹——从穷到富的几个阶段）
   - 经济学设定如何影响剧情（没钱买装备→冒险赚钱→有钱招兵买马）

□ 政治格局
   - 王国/势力列表（名称/统治者/国力/立场）
   - 势力关系网（联盟/敌对/中立/附属）
   - 关键政治事件（战争/政变/联姻/背叛）
   - 主角势力发展轨迹（从个人→小团体→势力→国度）

□ 种族/物种体系
   - 主要种族列表（人类/精灵/矮人/恶魔/魔鬼等）
   - 种族特征（天赋/文化/寿命/居住地）
   - 种族关系（友好/敌对/奴隶制/共治）
   - 混血/特殊物种设定（半精灵/半恶魔/龙裔等）

□ 位面/地理法则
   - 位面结构图（物质位面→半位面→下层位面→上层位面→星界）
   - 各位面的通行规则（传送/投影/化身降临）
   - 特殊地域法则（某地禁魔、某地时间流速不同）
   - 地图的物理边界（世界是有限还是无限？探索尽头是什么？）

□ 技术/文明设定
   - 文明水平（中世纪/蒸汽/魔法科技/星际）
   - 核心技术（奥术/炼金/魔像/密瑟能核/浮空城）
   - 技术发展的限制和代价
   - 重要发明的时间线及其叙事影响
```

### T3：主角金手指/特殊天赋（设定层）

```
【核心原则：金手指是这本书最核心的"设定卖点"之一。
         必须完整记录它的规则、限制、演变和对剧情的推动作用】

□ 金手指/天赋完整档案
   - 名称与类型（系统/血脉/神器/重生记忆/特殊体质）
   - 一句话说清楚"这个金手指是什么"
   - 获取方式（穿越自带/血脉觉醒/契约绑定/捡到/被选中）
   - 激活时机（立即生效/条件触发/逐步解锁）
   - 同类作品中该金手指的独特性/创新点

□ 能力规则清单
   - 核心能力（逐条列出金手指提供的各项能力）
   - 每项能力的触发条件和使用方式
   - 能力上限/极限（能成长到什么程度）
   - 能力衰减/失效机制（什么情况下不能用）

□ 成长路径
   - 金手指本身是否有进化/升级？
   - 升级条件（吸收能量/完成特定任务/达到某个阶位）
   - 每个阶段解锁的新能力
   - 成长与故事阶段的匹配（早期弱后期强）

□ 限制条件（非常重要）
   - 冷却时间（多久能用一次）
   - 能量消耗（使用需要消耗什么资源）
   - 代价/反噬（使用过度会怎样——失忆/虚弱/失控）
   - 失效条件（什么情况下金手指不工作——禁魔区域/被封印）
   - 不可为规则（金手指明确禁止做的事情）

□ 与主线的绑定关系
   - 金手指如何推动主线（没有金手指就推不动剧情）
   - 金手指如何限制主线（不能太强导致无冲突）
   - 金手指与核心矛盾的关系（解决核心矛盾的钥匙/关键障碍）
   - 如果去掉金手指，故事还能成立吗？

□ 与其他设定体系的交互
   - 金手指在力量体系中的定位（外挂/辅助/核心）
   - 与世界观规则的兼容性（是否破坏原有平衡）
   - 与同类角色的关系（别人有没有类似的天赋）
   - 交互冲突点（金手指触犯了哪些世界规则）

□ 金手指的叙事功能分析
   - 早期：降低死亡风险/提供成长动力/制造独特性
   - 中期：提供变强路径/制造选择困境/引发外部觊觎
   - 后期：代价显现/升级瓶颈/终极意义揭示
```

### T4：剧情全貌（剧情层）

```
【核心原则：故事不是一条线走到底。一卷内有多个剧情线并行——主线可能不止一条，
还有支线、成长线、情感线等。每条线有独立的弧线、自己的契诃夫枪，线之间汇聚/交叉形成高潮。】

□ 全书一句话梗概（一条线说清楚"这本书在讲什么"）

□ 核心冲突矩阵
   主角想要什么 × 什么在阻止他 × 代价是什么
   冲突升级轨迹（低级冲突→中级冲突→终极冲突）

□ 全卷剧情线清单 —— 识别并列出全书所有剧情线
   每条线记录：
    ├─ 线ID（如：主线-力量/主线-身份/支线-政治/感情线-XX/成长线-主角）
    ├─ 线类型（主线/支线/成长线/感情线/世界观线/悬念线）
    ├─ 一句话定义"这条线在讲什么"
    ├─ 覆盖章节范围（起止卷/章）
    ├─ 核心问题（这条线回答哪个读者问题）
    ├─ 线状态（引入→发展→高潮→解决 / 已解决 / 未解决 / 烂尾）
    └─ 线优先级（P0主线/P1重要支线/P2补充线）

□ 逐卷剧情线活动表
    格式示例：
    卷名 | 活跃线ID列表 | 线汇聚高潮章节 | 本卷新开线 | 本卷收束线
    ----|-------------|--------------|-----------|-----------
    卷1 | 主线-生存,支线-神子,感情线-兄妹 | 第8章(主线+感情线汇聚) | 主线-生存,支线-神子 | —
    卷2 | 主线-势力,支线-身份,感情线-歌莉娅 | 第43章(三线汇聚) | 主线-势力,感情线-歌莉娅 | 主线-生存
    ...

□ 每条剧情线的独立生命周期
    格式示例（以主线-身份为例）：
    引入：卷1 第20章 —— 薇薇安可能是"神子"
    发展：卷1 第26章（术士觉醒）→ 卷1 第45章（第一滴血）→ 卷1 第92章（神子确认）
    转化：卷1 第176章 —— 莉莉安出现，线从"她是谁"变为"她们是谁"
    高潮：卷8 第13章 —— 一体双魂揭示
    解决：全书尾声 —— 恐惧双子封神，与索伦一起进入神国

□ 主线与支线的交汇关系
    - 每条支线在哪些章节与主线交汇
    - 交汇产生的效果（推动主线/提供信息/改变方向/制造障碍）
    - 交汇的节奏（多久交汇一次？哪些阶段密集交汇？）
    - 无交汇的孤立线 → 标记为潜在问题

□ 成长线（主角/重要角色的能力、心智、地位的阶段性变化）
    格式示例：
    主角索伦：
      阶段1（卷1）：贫民区盗贼 → 生存驱动
      阶段2（卷2-3）：海外提督 → 扩张驱动
      阶段3（卷4-5）：半神 → 权力驱动
      阶段4（卷6-8）：神灵 → 秩序驱动
      阶段5（卷10）：死亡之主 → 法则驱动

□ 情感线（爱情/亲情/友情的情感变化轨迹）
    - 每条情感线的关键节点（初遇→深化→考验→稳定→升华）
    - 情感线与剧情线的交互（情感线如何被剧情推动/推动剧情）
    - 情感线收束质量（自然/生硬/未完成）

□ 关键转折点清单
    （改变故事方向的节点——不是小高潮，是"读完这章后故事就不一样了"）
    格式示例：
    K1: 卷1第45章 —— 薇薇安第一次杀人，揭示神子天赋 → 冲突从"生存"变为"身份"
    K2: 卷3第97章 —— 索伦弑神，获得弑神者专长 → 冲突从"凡人争霸"变为"神战"
    K3: 卷8第1章 —— 魔网崩溃 → 力量体系底层重构
    ...

□ 伏笔管理
    - 契诃夫枪清单（设伏位置→预期的回收位置→实际回收位置）
    - 设伏类型（对话暗示/物品埋伏/事件预设/角色伏线）
    - 伏笔密度（每多少章设一个新伏笔）
    - 回收质量（哪些收得好、哪些浪费了、哪些忘了）

□ 结局收线质量评估
    - 哪些线收得好（有铺垫有回收）
    - 哪些线没收（未解决/悬空）
    - 哪些线收得勉强（工具化收尾）
    - 总体评价：收线覆盖率（某宝视角，如"整本书收线率90%"）
```

### T5：人物/阵营（剧情层）

```
【核心原则：角色是故事的灵魂。不只是记录"有谁"，要分析"他们在故事中扮演什么功能、
         如何互相关联、如何随故事演变"】

□ 主角完整档案
   - 基本信息（姓名/种族/年龄/身份/阵营）
   - 性格核心标签（3-5个词，如：冷酷/务实/智慧/忠诚/果断）
   - 核心动机（驱动主角行动的根本原因——保护妹妹/复仇/成神）
   - 性格多阶段变化（每个阶段的性格特征和驱动力的变化）
   - 决策模式（主角做决定的方式——理性分析/直觉/情感驱动）
   - 道德系统（什么是主角的底线、什么情况下会打破）

□ 配角功能全量分类
   每条配角的记录：
    ├─ 角色名与身份
    ├─ 功能分类：
    │   ├─ 导师型（提供知识/指导方向）
    │   ├─ 盟友型（并肩作战/资源支持）
    │   ├─ 反派型（制造冲突/对立）
    │   ├─ 情感型（爱情/亲情/友情纽带）
    │   ├─ 工具人型（一次性推动剧情→标记为潜在问题）
    │   ├─ 搞笑担当（缓解节奏/制造轻松）
    │   └─ 信息员（提供剧情线索/背景信息）
    ├─ 与主角的关系及演变
    ├─ 在剧情中的功能（推动哪条线？提供什么？）
    ├─ 出场频率（常驻/阶段性/一次性）
    └─ 人设质量评估（立体/工具人/模板化）

□ 角色关系网络图（文本描述版）
    格式示例：
    索伦 → 薇薇安（兄妹/保护者与被保护者/依赖关系核心）
    索伦 → 歌莉娅（伴侣/事业伙伴/情感锚点）
    索伦 → 莉莉安（另一个妹妹/一体双魂中的特殊关系）
    薇薇安 ↔ 莉莉安（同一身体的两个灵魂/既竞争又共存）
    反派关系网：恐惧魔神（终极源头）→ 恐惧神子（众多个体）→ 吸血鬼神子/恐惧魔女
    ...

□ 反派分层体系
    ├─ 初级反派（前期的对手——帮派头目/人贩子/小Boss）
    │   功能：给主角练手、展示主角能力、制造早期紧张感
    ├─ 中级反派（中期的敌人——海盗王/半神/敌对势力）
    │   功能：推动主线进展、制造阶段性高潮、展示主角成长
    ├─ 终极反派（全书最强敌人——恐惧魔神/地狱之主/希瑞克）
    │   功能：全书终极冲突的目标、考验主角的终极意义
    └─ 宿命敌手（与主角有特殊因缘的反派——吸血鬼神子/猎杀之神）
        功能：镜像对比、哲学对立、宿命对决

□ 阵营/善恶法则
   - 该世界的阵营体系（如DND九宫格/修仙正邪/末世秩序混沌）
   - 善恶的判定标准（行为决定/血脉决定/神祇决定）
   - 阵营转换的规则（可以转换吗？代价是什么？有先例吗？）
   - 主角的阵营定位及其演变
   - 阵营冲突如何制造剧情张力

□ 血统/特殊身份设定
   - 血统体系（如神子/返祖/特殊血脉）
   - 每个血统的能力/限制/代价
   - 血统觉醒的方式和效果
   - 血统在剧情中的作用（推动主线/创造冲突/决定结局）
```

### T6：叙事设计技法（技法层）

```
【核心原则：这里分析的是"作者怎么设计叙事"，不是复述剧情。
         记录的每一句都要能回答"这个设计好在哪，我怎么学"】

□ 开篇设计技法
   └ 前1000字分析
       - 第一句话：什么内容、什么语气、什么效果
       - 前100字建立了什么（情绪/设定/悬念/场景）
       - 前1000字的情感钩子拆解（四层逐层分析）
       - 读者在前1000字内产生的心理变化路径

   └ 第1~30章逐章功能标记
       格式示例：
       章号 | 功能标签 | 核心事件 | 张力分(1-10) | 是否高潮
       ----|---------|---------|-------------|--------
       1 | 情感锚定 | 薇薇安视角建立共情 | 6 | 否
       4 | 第一次释放 | 暗杀帮派成员 | 8 | 是
       8 | 身份确立 | 割喉者之夜 | 9 | 是
       14-16 | 三连高潮 | 蛇魔战 | 10 | 是 ✅
       25 | 情感缓冲 | 阳光下的微笑 | 2 | 否
       ...
       - 节奏公式推导：大约每X章一个张力高峰，高峰之间设Y章松弛

   └ 世界信息分层引入表
       章号 | 释放的信息 | 隐藏的信息 | 释放方式 | 信息量
       ----|-----------|----------|---------|------
       1 | 贫民区生存法则 | 穿越真相 | 环境描写 | 低
       2 | 属性面板、前世记忆 | 穿越原因 | 内心独白 | 中
       ...
       - 核心规律：每2-3章释放一个核心设定，信息总量逐步增加

   └ 冲突升级路径
       - 记录冲突从低级到高级的逐层升级
       - 格式：生存→个人→群体→超自然→身份→命运
       - 每级冲突的章节范围、触发事件、升级原因

   └ 开篇承诺清单
       承诺类型 | 承诺内容 | 回报预期 | 实际回报
       --------|---------|---------|---------
       成长承诺 | 属性面板+升级路线 | 卷内 | ✅
       冒险承诺 | 地下城探索+宝物 | 卷内 | ✅
       情感承诺 | 兄妹羁绊 | 贯穿全书 | ✅
       史诗承诺 | 圣者浩劫+诸神陨落 | 长期 | ✅
       ...

□ 长篇节奏技法
   └ 逐卷核心冲突表
       卷序 | 核心冲突 | 冲突层级 | 主角状态 | 篇幅
       ----|---------|---------|---------|----
       1 | 生存vs压迫 | 个体 | 贫民区盗贼 | ~40万
       ...

   └ 高潮分布图
       - 标注全书所有S/A/B级高潮位置和内容
       - 高潮特征总结（双峰/三幕/多浪/平稳）
       - 高潮间隔规律（每多少章一个高潮？间隔是否变化？）

   └ 嵌套节奏分析
       - 大节奏单元（60-100章）的中高潮分布
       - 中节奏单元（15-25章）的小高潮分布
       - 小节奏单元（4-8章）的爽点分布
       - 三层嵌套如何服务于速读和细读的不同需求

□ 悬念管理技法
   └ 主悬念生命周期
       格式：引入（章节）→ 维持（手法）→ 转化（新形式）→ 解决（章节）
       - 维持手法分析：信息延迟/误导/部分揭示/分段揭示

   └ 全体次要悬念追踪
       - 每个次要悬念的独立生命周期
       - 悬念之间的交互（一个悬念解决是否引出另一个）
       - 悬念密度漏斗：逐卷活跃悬念数量统计

   └ 悬念类型分布
       - 身份类（占比/处理节奏）
       - 能力类（占比/处理节奏）
       - 关系类（占比/处理节奏）
       - 结局类（占比/处理节奏）

   └ 悬念回收质量评估
       - 哪些回收得好、哪些烂尾、哪些悬空
       - 总体评价：读者读完后的"未解答问题"数量
```

### T7：数据流/系统写法（技法层）

```
【核心原则：数据流（面板/系统提示/数值变化）是这类小说的核心技法。
         分析的不是"有什么数据"，而是"数据怎么融入叙事"。】

□ 面板/数据窗口分类统计
   - 面板类型清单（属性面板/技能面板/系统提示/状态变化等）
   - 每种类型出现的频率和场景
   - 面板长度分布（短:3-5行/中:10-15行/长:20-30行/超长:30+行）

□ 面板放置规律统计分析
   - 面板在章节中的位置统计（开头/中段/末尾/战斗前/战斗后/升级时）
   - 核心规则提炼：面板从不放在章节首尾，永远在中段
   - 例外情况分析（什么时候打破了规则？效果如何？）

□ 面板与叙事融合策略
   - 策略A-面板确认+叙事解释（占比/示例/效果评估）
   - 策略B-叙事中断式（占比/示例/效果评估）
   - 策略C-双线同步式（占比/示例/效果评估）
   - 各种策略在不同阶段的使用偏好变化

□ 数据显示/隐藏的选择逻辑
   - 总是显示的数据（姓名/种族/阵营/核心属性）
   - 选择性显示的数据（神性值/神格值/传说度等——在封神后才出现）
   - 始终不显示的数据（身高体重/人际关系数值）
   - 选择逻辑提炼：只显示读者"需要知道"的信息

□ 面板与技法的演变
   记录从早期到后期的数据流变化：
   阶段 | 面板长度 | 功能 | 占章节比 | 读者期待
   ----|---------|------|---------|---------
   早期 | 10-15行 | 读者理解工具 | ~20% | 新鲜好奇
   中期 | 15-20行 | 角色成长确认 | ~15-20% | 期待变化
   后期 | 20-30行 | 身份宣告工具 | ~20-50% | 见证巅峰

□ 数据流对阅读体验的影响评估
   - 数据流是打断叙事还是增强沉浸？
   - 数据分析：哪些位置的数据插入提升了阅读体验，哪些降低了
   - 优化建议：如果自己做类似写法，哪些设计值得借鉴，哪些要避免
```

### T8：战斗/高潮场面（技法层）

```
【核心原则：战斗不是"谁打了谁"，是"作者怎么让读者感受到紧张/爽/震撼"。
         每种战斗类型有自己的节奏公式。】

□ 战斗类型分类
   类型 | 定义 | 代表章节 | 出现频率
   ----|------|---------|---------
   暗杀 | 快速隐蔽击杀 | 第8章割喉者之夜 | 66场(10%)
   单挑 | 1v1长时间对决 | 第14-16章蛇魔战 | 11场(2%)
   群战 | vs多个敌人 | 少量 | 10场(2%)
   战争 | 有组织的军团战 | 第三卷海战 | 158场(25%)
   神战 | 神灵级对抗 | 第84章水漫三千里 | 242场(38%)
   未分类 | 其他 | — | 151场(24%)

□ 每种战斗类型的固定公式
   格式示例（暗杀型）：
   战前准备：极短(10行) → 环境铺垫
   战斗过程：快速击杀每个敌人(2-5行/个)
   数据反馈：击杀提示面板
   战后处理：简洁搜刮
   感官调用：视觉(血迹)+听觉(惨叫声)+触觉(匕首切入)
   读者感受目标：冷酷/高效/爽

   其他几种类型同样逐项分析

□ 全本战斗分布统计
   - 逐卷统计：战斗数/战斗行数/战斗占比/主导类型
   - 战斗类型转移规律（前期暗杀为主→中期战争→后期神战）
   - 战斗占比曲线：密度最高卷 vs 密度最低卷

□ 通用战斗公式提炼
   战前准备(10-50行) → 接触/对峙(20-30行) → 战术决策(5-15行)
   → 数据反馈(面板插入) → 击杀 → 战后处理(战利品评估/叙事间奏)

□ 战后模式分析
   - 战后处理方式分类（战利品评估/面板确认/叙事间奏/无处理）
   - 面板覆盖率逐卷统计（早期~90%, 后期~93%）
   - 完整模式（面板+战利品）的覆盖率

□ 战斗疲劳管理机制
   - 全书连续战斗区间统计
   - 作者使用的手法：
     ① 种田穿插（低战斗密度卷作为"呼吸空间"）
     ② 战斗类型切换（暗杀↔战争↔神战交替）
     ③ 战后叙事间奏（战斗后必然插入剧情推进）
   - 疲劳管理效果评估：读者在哪个阶段最容易感到厌倦？
```

### T9：特殊魅力塑造（技法层）

```
【核心原则：有些书的魅力来自"怎么写"而不是"写了什么"。
         反英雄/邪恶主角/黑色幽默/独特文风等，都需要单独分析。】

□ 读者同情机制（针对反英雄/邪恶主角）
   - 保护壳层级分析：
     第1层：环境决定（"别无选择"——贫民区不冷酷活不下去）
     第2层：受害者正当化（杀的都是更恶的人——打妹妹主意的人贩子）
     第3层：底线保留（仍有在乎的东西——花1银德勒安葬老狗）
     第4层：保护弱者（邪恶的价值选择——保护薇薇安是绝对命令）
   - 每层对应的具体章节和效果评估
   - 如果多于一层的保护壳，层与层之间的递进关系

□ 道德边界管理
   - 不可跨越的铁律清单（如：不伤害无辜者/不违背契约/不抛弃主角在乎的人）
   - 每条边界在什么情况下受到考验、如何坚守
   - 边界外的"安全区"（作者允许主角做的恶：杀敌人/复仇/计谋）

□ 关怀系统（主角在乎什么）
   - 主角的情感锚点（一个人/一个理念/一个物品/一个地方）
   - 锚点的展示方式（行为展示/不直接写内心独白）
   - 锚点在故事不同阶段的变化（早期是保护→中期是守护→后期是传承）
   - 锚点被威胁时的主角反应模式（暴走/计谋/交易/牺牲）

□ 邪恶/反英雄的升级路径
   阶段 | 邪恶表现 | 读者态度 | 叙事策略
   ----|---------|---------|---------
   生存型 | 偷窃/暗杀/欺骗 | 理解 | 环境决定论
   扩张型 | 斩首/战争/统治 | 接受 | 敌人先动手
   法则型 | 弑神/改写规则 | 敬畏 | 冲突升级自然
   框架型 | 战略克制/与正神联手 | 尊重 | 角色成熟

□ 其他特殊魅力
   - 幽默/反讽/黑色幽默的类型和分布
   - 反差萌（强大者的弱点/冷酷者的温情）
   - 观点金句/名台词（被读者记住和传播的句子）
   - 独特文风（如：口语化/古雅/技术性/诗性）
   - 读者共鸣点（哪些内容让读者在评论区/弹幕中强烈讨论）

□ 每种特殊魅力的"可复制性"评估
   - 这个魅力是作者个人天赋还是可学的技法？
   - 如果可学，具体怎么学？（给出可执行建议）
   - 不可学的魅力，作为"识别特征"记录即可
```

# Phase 3：验证采样（Layer 2）

## 目的

检验 Phase 2 的发现是否具有**全书的代表性**。上一轮验证发现的典型问题需要在此阶段纠正。

## 采样规则

| 采样类型 | 数量 | 目的 |
|----------|------|------|
| 随机采样 | 3~5个未精读区域 | 验证核心发现的一致性 |
| 边缘采样 | 1~2个"可能不适用"区域 | 测试发现的边界条件 |
| 类型采样 | 每种战斗/对话/场景类型至少1例 | 验证类型公式的普遍性 |

## 验证 Checklist

```
□ Phase 2 发现的规律在采样章节中成立？
□ 是否有意外的新模式出现？
□ Phase 2 的结论是否需要修订？
   - 成立（原结论不变）
   - 需修订（标注具体修订内容）
   - 推翻（说明原因并提供替代解释）
```

## 产出格式

```
📄 {书名}-Phase3-验证报告.md

【验证章节清单】
1. 第X~X章（卷名）：[选择原因]
2. ...

【验证结果】
结论1：[原始结论]
  □ 验证通过 ✅
  □ 需修订 ⚠️ → 修订为：[新结论]
  □ 不成立 ❌ → 替代解释：[...]

结论2：...
...

【新增发现】
如验证中发现新模式，记录在此。
```

# 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| **正文文件不存在/格式不支持** | 提示用户提供 .txt 或 .md 格式。不接受 pdf/docx |
| **字数太少（<5万字）** | 拆解价值有限。与用户确认是否继续。继续则缩减范围 |
| **同人题材但无原著知识** | 先搜索原著百科/时间线（❌4红线） |
| **Phase 2 中发现这本书质量差** | 输出"不推荐拆解"报告，让用户决定 |
