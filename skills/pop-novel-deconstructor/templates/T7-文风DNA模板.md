# T7 文风DNA - 拆解模板

> **@consumed_by**: pop-novel-writer.step-2-render(Phase 1 风格锚定 — 直接作为 styles/*.md 使用)
> **@produced_as**: `styles/{书名简称}.md` — 产出后直接复制到 writer 的 styles/ 目录

## 核心原则

不要分析"便"出现几次、平均句长多少、比喻密度几回。这些是给统计看的，不是给 Agent 用的。
Agent 需要的是**原文段落的质感**——它读一段原文，用自己的语感对齐，然后写出来。

## 检查清单

### □ 战斗场景 —— 选 1-3 段原文

从全书中选最能代表战斗文风的段落。优先选：
- 首次战斗（看作者怎么引入战斗节奏）
- 死战/越级战（看作者怎么写压迫感和逆转）
- 碾压战（看作者怎么写"很快"的爽感）

每段标注来源章节、场景类型和 1 句说明。

### □ 对话场景 —— 选 1-2 段原文

从全书中选最能代表对话/谈判/交涉风格的段落。优先选：
- 非战斗解决冲突（看作者怎么用对话代替打斗）
- 关键信息在对话中释放的段落

### □ 发现/探索场景 —— 选 1-2 段原文

从全书中选最能代表发现/探索/世界观展开节奏的段落。

### □ 危机/悬念场景 —— 选 1-2 段原文

从全书中选最能代表"读者想知道然后呢"的段落。

### □ 风格红线清单

从原文中提取的硬性约束（不超过 10 条）。格式：
```
□ 规则 → 0容忍/标记
```

## 示例格式（参考 abyss.md）

```
## 一、战斗

### 【战斗·暗杀·一对多】第X章 XXX

{3-5段原文}

每一段之间用 `---` 分隔。
```

不要保留旧模板的任何量化分析内容（如"便"44次、短句占比31.4%等）。不要保留优先级分级（P0/P1/P2）。只保留原文片段 + 场景标注 + 红线清单。
