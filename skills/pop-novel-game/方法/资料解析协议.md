# 资料解析协议 · Phase A

## 目标

将用户提供的创作资料（格式不限）解析为结构化数据，为后续文游设计提供完整的世界观底座。

## 输入格式支持

支持但不限于以下格式：

- Markdown 文档（.md）
- Word 文档（.docx）
- Pure text（.txt）
- 笔记软件导出（Notion/飞书/幕布等）
- 结构化 YAML/JSON
- 图片（含文字的截图，需 OCR 阅读）

## 提取要素

### 静态层（世界观设定）

```
地理
  ├─ 大陆/国家/区域名称及描述
  ├─ 重要地点（城市/门派/秘境/基地等）
  └─ 地图层级关系

势力/阵营
  ├─ 组织名称、类型（国家/门派/家族/公司等）
  ├─ 宗旨、立场、目标
  ├─ 内部结构、层级
  └─ 势力间关系（同盟/敌对/中立）

规则体系
  ├─ 力量体系（修仙境界/魔法体系/科技等级等）
  ├─ 社会规则（阶级/法律/禁忌/风俗）
  └─ 特殊规则（书特有的机制，如天命之子/灵根/基因锁）

关键名词
  ├─ 专有名词及定义
  ├─ 器物/法宝/科技造物
  └─ 特殊生物/种族
```

### 角色层（人物设定）

```
每个角色提取：
  ├─ 基本信息：姓名/别名/年龄/性别/外貌
  ├─ 性格特点：核心性格标签、行为模式、语言风格
  ├─ 背景故事：出身、经历、关键转折
  ├─ 能力体系：修为/技能/特殊能力
  ├─ 人际关系：与其他角色的关系（亲情/友情/爱情/仇敌等）
  ├─ 角色弧光：成长变化轨迹
  └─ 关键事件：该角色经历的重要事件

整体产出：
  ├─ 角色列表（含角色分类：主角/配角/反派/NPC）
  └─ 关系网络（谁认识谁、谁恨谁、谁爱谁）
```

### 剧情层（故事框架）

```
核心冲突
  ├─ 故事的核心矛盾（正邪对抗/生存危机/权力之争等）
  └─ 主线驱动力

关键事件
  ├─ 故事开端/转折/高潮/结局事件
  └─ 影响世界观走向的重大变故

分支可能性
  ├─ 可以 diverged 的决策点
  ├─ 隐藏剧情/支线线索
  └─ 潜在的多结局路径
```

## 输出格式

Phase A 产出单一 JSON 文件：

```json
{
  "world": {
    "name": "世界观名称",
    "type": "仙侠/科幻/都市/西幻...",
    "summary": "一句话世界观",
    "geography": [...],
    "factions": [...],
    "rules": [...],
    "glossary": [...]
  },
  "characters": [
    {
      "id": "char_001",
      "name": "角色名",
      "type": "主角/配角/反派/NPC",
      "traits": [...],
      "background": "...",
      "abilities": [...],
      "relationships": [
        {"target": "char_002", "type": "师徒", "description": "..."}
      ],
      "arc": "...",
      "key_events": [...]
    }
  ],
  "plot": {
    "core_conflict": "...",
    "key_events": [...],
    "branch_points": [...],
    "endings": [...]
  }
}
```

## 纪律

- **Phase A 不做创造**——只提取，不添加原始资料中没有的东西
- **不遗漏重要角色**——出场超过 3 次或对剧情有实质影响的角色都应收录
- **关系必须双向记录**——如果 A 是 B 的师父，B 也要记录为 A 的徒弟
