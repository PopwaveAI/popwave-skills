# 过渡推进场景模板

> 适用于：章与章之间的桥接、新信息引入、为下一章铺垫
> 核心原则：**过渡不是"休息"——是带着读者往前走**

---

## 段落结构

```
┌─ 段1 · 回响（1段，2-3句） ───────────────────────┐
│  功能：确认上一章的落点在读者记忆中还在                │
│  内容：上一章的结果的一个片段画面/感受                │
│  写法：不总结，不解释——直接切一个画面                 │
│  示例："虎口的血已经干了。"                           │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段2 · 新信息引入（1-2段） ────────────────────────┐
│  功能：引入本章/本幕需要的新设定、新角色、新线索       │
│  内容：一个新信息（不是一大段背景介绍）                │
│  写法：通过事件/对话/环境带入，不单独写设定段          │
│  字数：100-200字                                      │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段3 · 下一章钩子（1段） ─────────────────────────┐
│  功能：让读者想点下一章                              │
│  内容：一个新矛盾 / 一个新悬念 / 一个即将发生的事     │
│  写法：事件的一半自然露出，不人工制造悬念             │
│  示例：不是"他不知道，危险正在逼近"——                 │
│        而是"手机震了一下。一条消息：'他们到了。'"      │
└────────────────────────────────────────────────────┘
```

---

## 关键约束

- **过渡章没有独立存在的价值**——删除后，上一章到下一章的阅读体验不能断裂
- **不写"喘口气"的纯停顿**——网文读者不需要喘息，需要理由继续看
- **新信息≤2个**——超过2个新信息就不是过渡章，是设定章
- **钩子不来自叙述者**——来自事件本身，不是"他不知道…"

---

## 节奏标记

```
回响：慢（1拍） → 新信息：中速 → 钩子：快（切入）
总字数：300-500字
```
