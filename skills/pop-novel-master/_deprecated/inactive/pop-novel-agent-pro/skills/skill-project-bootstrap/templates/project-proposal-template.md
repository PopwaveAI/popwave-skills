# 项目计划书模板

> 用途：【开书启动】阶段，向用户展示项目投资价值  
> 时机：创意初步确认后，正式创建项目前  
> 原则：用"找投资"的视角打动用户，而非执行细节

---

## 一、一句话卖点

> 格式：[核心冲突] + [独特解决方案] + [终极愿景]
> 
> 示例：
> - "诡异入侵现实，主角召唤'第四天灾'玩家，用骚操作和无限复活对抗不可名状的恐怖，建立人类最后庇护所。"
> - "穿越战锤40K底巢，获得塔科夫系统，从拾荒鼠鼠到帝皇神选，刮地皮也能拯救帝国。"

---

## 二、市场机会（为什么现在做）

| 维度 | 分析框架 |
|:---|:---|
| **市场空白** | 该题材/融合方向是否有成功案例？是蓝海还是红海？ |
| **元素验证** | 拆解的单一元素是否有爆款验证？（如第四天灾已验证，诡异已验证） |
| **读者需求** | 目标读者群体的痛点是什么？现有作品是否满足？ |
| **改编潜力** | 视觉化程度、审核风险、IP衍生空间 |

| 爽点层级 | 具体表现 | 对标案例 |
|:---|:---|:---|
| **即时爽** | 每章/每段的小高潮，如反转、打脸、发现 | 参考作品 |
| **成长爽** | 肉眼可见的进步，如等级、装备、势力扩张 | 参考作品 |

## 使用说明

1. **填充顺序**：
   - 必须先有一句话卖点（核心锚定）
   - 然后填充市场机会（为什么值得做）
   - 再填充爽点设计（读者为什么爽）
   - 最后风险评估（诚实面对问题）

2. **避免陷阱**：
   - 不要写具体章节内容（那是【章纲生成】的事）
   - 不要写详细世界观（那是【设定采集】的事）
   - 不要过度承诺（留有余地，后续迭代）

3. **目标读者**：
   - 这份计划书是写给"决策者"（用户/作者）看的
   - 不是写给"执行者"（写作子Agent）看的
   - 要打动人，不是指导人
