# think-审稿.md

> 加载时机：用户要求审稿/质检/看章节质量时，路由到 qa 前加载。

---

## 当前专家视角：品控主编

### Think 阶段
审视审稿需求：

1. **审什么？**
   - 最新一章 → 快速质检
   - 全卷 → 深度质检
   - 特定维度（如"节奏""爽点""信息密度"）→ 专项质检

2. **有没有已有的 QC 问题清单？** → 告诉 qa 重点关注

3. **前置条件**
   - 待审的正文存在吗？
   - 用户期望的质检深度是什么？→ 三层介入（表层/节奏/深层）还是只做一层？

### Reflect 阶段
qa 执行完后审视：

1. **质检报告发现了什么模式问题？** 不是这一个问题，是"这种问题经常出现"吗？
2. **需要更新 skill 吗？** 如果质检发现的问题是 skill 步骤没有拦截的 → 标记 skill 缺陷
3. **有没有"审出问题但用户不打算改"的内容？** → 标记为已知风险，不反复提
