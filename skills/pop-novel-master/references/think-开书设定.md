## 当前专家视角：资深编辑

### Think 阶段
审视用户需求，拆解后再路由：

1. **用户的「开新书」需求够具体吗？**
   - 有一个故事想象（场景/概念/角色片段）→ 直接路由 bootstrap
   - 只有模糊方向（"我想写本系统文"）→ 追问：什么场景？主角是谁？ → 再路由
   - 过去是否已拆过参考书？数据还新鲜吗？需要重做吗？

2. **前置条件检查**
   - 如果用户已经写过正文（续写场景）→ 不是开书，是 reverse 模式
   - 如果用户只是"想设定着玩"→ 路由 bootstrap 的部分 phase 即可

### Reflect 阶段
bootstrap 执行完成后，审视产出：

1. **story-engine.yaml 的 core_premise 真的能撑起一个故事吗？** 还是一句话说了等于没说？
2. **L1 设定有没有"看起来对但站不住"的幻想设定？**（比如"主角很特别"但没说明为什么）
3. **有没有不该此时暴露的信息？**（第一卷就让读者看到了最终 Boss）
4. **力量体系有没有本质矛盾？**（装备给的成长 vs 自身成长的冲突）
5. **如果发现关键盲点 → 决定修复优先级：P0 现在修 / P1 写完这章修 / P2 下一卷再说**
