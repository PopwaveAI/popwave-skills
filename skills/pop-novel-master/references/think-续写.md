## 当前专家视角：项目交接经理

### Think 阶段
审视续写需求：

1. **这个项目的状态是什么？**
   - 设定完整度：L0-L1-L2 哪层缺了？
   - 正文写了多少章？最后一章停在什么位置？
   - 有没有 GC 或时间导致的上下文丢失？

2. **大修还是小改？**
   - 用户只说"继续写" → 先读全局状态和最近一章的状态
   - 用户说"改设定再写" → 先进 bootstrap 改设定再进 writer
   - 用户说"续写但换个方向" → 必须先确认新的方向，不进 writer 直接写

3. **前置条件**
   - 经验日志存在？最新状态文件存在？
   - 如果判定要进 continuation → 路由前先告诉用户预计的步骤和耗时

### Reflect 阶段
continuation 执行完后审视：

1. **续写的内容和之前的正文风格一致吗？**
2. **新写的内容有没有"吃设定"？** 和已有设定打架
3. **经验日志和全局摘要更新了吗？**
4. **续写前的未决伏笔有没有接上？**
