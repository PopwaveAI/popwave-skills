# 黄金三章引擎 v1.2

> 版本：**v1.2** · 2026-05-25
>
> **v1.2 核心变化（v3.3 对齐）**：
> - novel_agent_version 升级到 v3.3
> - 新增 reader_profile 感知——在节点C·黄金思考中检查读者画像（平台/年龄/阅读习惯）
> - 章节字数基线明确为 2200-2500 字
> - QC 指引更新：写完后输出 pop-novel-qa 三层质检（大纲层/骨架层/正文层纯感受报告）
>
> 核心原则：前三章不是"起头"，是**整本书的浓缩核**。读者用前三章决定追不追，编辑用前三章决定签不签。
>
> 正常章节可以"慢慢释放"，前三章不行。前三章必须在极短的篇幅内完成：**把读者拉进来 → 压住 → 释放但留下更大的悬念**。

## 与正常章节管线的区别

| 维度 | 正常章节（CH4+） | 黄金三章（CH1-CH3） |
|:---|:---|:---|
| 爽点等级 | 微爽点打底，中爽点每3-5章1个 | **中爽点起步，三章内至少1个大爽点** |
| 爽点版场景卡 | 每等级写1段验证 | **每章核心事件各写1段释放段，总共≥6段** |
| 简略章纲分析 | 可选 | **强制**，每章都做 |
| 风格种子 | CH1注入参考书原文，CH2+不注入 | CH1强制注入参考书原文；**CH2-CH3上章结尾原文承担** |
| 自检 | 5项轻量 | 5项 + **额外3项（首屏/节点/三章连读）** |
| 先写场景卡再写正文 | 可选 | **强制**——必须先写场景卡验证爽点，通过才能转入正文 |

## 黄金三章的情绪弧线

前三章不是三篇独立文章，是一条连续的读者情绪管理曲线：

```
CH1 · 拉进来
    情绪：压迫/好奇/微爽 → 期待
    目标：让读者产生"我要看下去"的冲动
    钩子：必须是情绪驱使型（悬念/信息炸弹/半路阻断），不能用信息差弱钩子

CH2 · 压住
    情绪：紧张/焦虑/资源不足感 → 憋屈但不绝望
    目标：让读者为第三章蓄力——"这章憋着，第三章应该能释放"
    钩子：必须在情绪最高点截断，不能给读者"喘口气"的机会

CH3 · 释放 + 新钩子
    情绪：释放/爽 → 震撼/期待
    目标：让读者获得第一次真正满足，同时抛出一个更大的问题
    完美收尾：主角获得阶段性成果 ✅ → 但发现这只是一个开始 ⚠️ → 更大的威胁出现 ❓
```

## 管线流程

```
L0灵魂对齐（来自 skill-project-bootstrap）
    │
    ▼
黄金三章·幕纲设计
  │  参照 pop-novel-plot v2.0 输出格式
  │  但 payff_distribution 不适用"微/中/大/终极"
  │  改为三章专属分布：
  │    CH1：中爽点×2（打脸/获得/解谜各一）+ 大爽点级章末钩子
  │    CH2：中爽点×2 + 大爽点×1（能力质变/信息反转）+ 大爽点级章末钩子
  │    CH3：中爽点×1 + 大爽点×1 + 终极爽点级章末钩子
  │  情绪路径三章连成一条弧线（见上）
    │
    ▼
爽点版场景卡·全量验证
  │  不写"每等级各1段"
  │  而是：每章的核心爽点事件各写 1 段释放段（300-500字/段）
  │  CH1：至少 2 段 → 中爽点释放段
  │  CH2：至少 3 段 → 中爽点×2 + 大爽点×1
  │  CH3：至少 2 段 → 大爽点 + 终极爽点级钩子
  │  写入 01-写作资产/场景卡/ 目录
  │
  验证：
    │  作者读完所有场景卡
    │  → "读者读完后，能产生'我要看第四章'的冲动吗？"
    │  → "三章的情绪弧线完整吗？"
    │  → 如未通过 → 回黄金幕纲调整
    │
    ▼ 通过
循环执行（CH1 → CH2 → CH3）：
  │
  【节点C·黄金 · 每章写作前思考】（v1.2 升级版）
   ① 对齐检查：这章在黄金三章弧线中的位置——拉人/压住/释放？
      情绪起点是否衔接上一章结尾？
      ⭐ 读者画像检查：从 project.yaml 读取 reader_profile —— 这章写给谁看的？
   ② 核心判断：这章如果只让读者记住一件事，是哪件？
   ③ 首屏检查：前300字能留住人吗？(快速验证法——单独读一遍)
   ④ 风险扫描：信息密度≥2个新概念？章末钩子类型和上一章不同吗？
   ⑤ 产出：本章设计说明（散文式判断链）
   ⑥ 如有需要确认的问题 → 「待确认/疑问」输出给作者
  │
  Step 1  简略章纲分析（强制，不可跳过）
  │        输入：上一章结尾正文（CH1用风格种子）+ 幕纲本章数据
  │        输出：3-4行→ 上章情绪承接 + 本章开头策略
  │
  Step 2  上下文组装（复用 pop-novel-writer output-template.md）
  │        6块素材：
  │          ① 全局摘要（含前三章累积摘要）
  │          ② 本章爽点兑现（幕纲锁定）
  │          ③ 角色状态
  │          ④ 风格种子（CH1注入参考书原文 / CH2-CH3注入上章结尾原文）
  │          ⑤ 本章设计（情绪承接+阶段定位+核心事件+情绪路径+关键要素）
  │          ⑥ 下一章钩子
  │
  Step 3  正文写作（单次LLM调用）
  │        字数：**2200-2500 字**（v1.2 基线，不低于 2000，不高于 2800）
  │
  Step 4  自检（7项）
  │        5项通用（爽点兑现/情绪路径/钩子延续/无否定式/无感到类前缀）
  │        + 3项黄金三章专项：
  │          □ 前300字是否真正抓住了人？
  │          □ 本章是否至少有2个"读者想划到下一页"的节点？
  │          □ 三章连起来读，节奏是否稳定？(CH1拉人→CH2压住→CH3释放)
  │
  Step 5  QC 质检（v1.2 新增）
  │        ⭐ 调用 pop-novel-qa v0.4.1 执行正文层 QC：
  │          "读完了还是刷过去了？会弃书吗？"
  │          红线：想跳过≥2处或会弃书 → 退回重写
  │        可选：大纲层 QC（在写作前检查幕纲计划）
  │
  Step 6  更新全局摘要
  │        python scripts/update_global_summary.py ...
  │
  Step 7  继续下一章
    │
    ▼ CH3写完后
转入 pop-novel-writer 正常循环
  │  前三章的 global_summary + character_state 已积累
  │  从 CH4 起按 v7.8 普通流程运行
```

## 自检清单

写作完成后逐项检查：

```
### 通用（5项）
□ 核心爽点是否兑现？读者读完后真的感到了"爽"吗？
□ 情绪路径完整？起点≠终点？
□ 章末钩子是情绪延续，不是强行插入？
□ 无"不是…而是…"句式
□ 无"他感到/他意识到/他评估"类冗余前缀

### 黄金三章专项（3项）
□ 前300字是否真正抓住了人？
  快速验证：把前300字单独读一遍，如果感觉"可以划走"→ 重写
□ 本章是否至少有2个"读者想划到下一页"的节点？
  （每800-1000字一个——微钩子/信息推进/角色反常行为）
□ 三章连起来读，节奏是否稳定？
  CH1拉人 → CH2压住 → CH3释放
  如果CH2和CH1一个节奏→CH2拖了
  如果CH3释放没到位→CH3写了但没做功
```
