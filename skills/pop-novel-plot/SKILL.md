# 剧情架构

小说的本质是让读者经历情绪变化。幕纲的第一性 = 爽点分布与情绪弧线，不是功能标签。

产出物供给 pop-novel-writer（正文写作）管线。

---

## ❌ 质量红线（开工前→完工后自检）

每条开工前确认 → 画 [ ]；完工自检 → 改 [x]。全部通过才能输出。

- [ ] **PRD + L1 设定已存在** —— 必须先过 bootstrap 开书启动。不过开书不进幕纲
- [ ] **节点B导演判断已输出** —— 写幕纲前先思考"这个幕的第一性是什么"
- [ ] **每章有 emotional_goal + payoff + end_hook** —— 章级切片齐备
- [ ] **爽点等级与铺垫-释放比匹配** —— 微 2:1 / 中 4:1 / 大 8-10:1 / 终极 20:1
- [ ] **幕内无连续 3 章同一情绪叠加组合** —— 违反则替换至少 1 章
- [ ] **节奏自检 7+3+3 项全部通过**
- [ ] **场景卡交互闸门已通过** —— 老板点头才能继续
- [ ] **平台节奏校准已完成** —— 不做校准直接进 writer → 退回
- [ ] **Boss 战有三阶段结构（如有）** —— P1→P2→P3，缺阶段退回
- [ ] **act-XX.yaml 已输出到 `设计/幕/`**
- [ ] ★ **情节线规划已完成** —— 幕级定义中 plotlines ≥ 2 条，每章 plotlines_active 不为空

## 什么时候用

| 场景 | 说明 |
|:-----|:------|
| 设计幕/卷的爽点分布和情绪弧线 | 新书开书完成后、需规划整体节奏时 |
| 细化单幕的章级切片 | 已有高层大纲但缺每章情绪目标时 |
| 设计卷末 Boss 战 | 幕级高潮需要独立设计三阶段结构时 |
| 续写场景下重新规划下一幕 | bootstrap reverse 完成后续写、需补充后续幕纲时 |
| 检查现有幕纲节奏问题 | 写作过程中发现节奏塌了、需回头修正时 |

## 核心流程（5步）

### Step 1：前置条件 + 节点B导演判断

**前置条件检查**：
- [x] project 已通过 bootstrap 开书启动（L0 + PRD + L1）
- [x] 世界稳定性检验已通过
- [x] reader_profile 已嵌入 project.yaml
- ❌ 不满足 → 先调 bootstrap。**不允许跳过**。

**节点B·幕设计前思考**：写幕纲之前先停下来想清楚。输出一段散文式判断链，覆盖：

1. 对齐检查：上游 L0/L1 设计与这个幕一致吗？
2. 核心判断：这个幕的第一性是什么？在全书扮演什么角色？
3. 情绪弧线：起点 → 顶点 → 终点 合理吗？
4. 双轴检查：对抗有"强度 + 深度"两个维度？
5. 风险扫描：平台节奏满足？爽点分布有连续空白？
6. ★ **情节线规划**：这个幕涉及多少条主/支线？每条线的期望出现频率？契诃夫枪在哪些章节设伏、哪些章节回收？

❌ 跳过节点B直接填 act-XX.yaml 模板 → 退回。

### Step 2：设计幕纲（act-XX.yaml）

按情绪驱动原则设计。每章以 emotional_goal + payoff 为第一性。

**幕级定义（act-XX.yaml 顶部）**：
```yaml
act: 1
title: 觉醒之章
plotlines:                      # ← 新增：本幕涉及的支线定义
  - id: 主线
    desc: 主角升级打怪
    expected_frequency: 每章    # 期望出现频率
    chekhov_guns:               # 契诃夫之枪
      - name: 金手指来源
        setup_ch: 1
        payoff_ch: 20
        desc: 穿越的真正原因是XXX
      - name: 最终Boss身份
        setup_ch: 5
        payoff_ch: 30
  - id: 感情线
    desc: 与女主角的关系
    expected_frequency: 每2章
```

**每个章级切片必须包含**：
- emotional_goal（情绪目标，第一性）
- payoff（爽点类型 + 等级 + 触发方式 + 读者感受）
- reader_emotion_path（读者情绪变化路径）
- end_hook（钩子类型 + 驱动力 + 内容）
- ★ **plotlines_active**：本章节推进了哪些支线（列表，对应 plotlines.id）
- ★ **chekhov_set**：本章节设伏了哪些契诃夫枪（列表）
- ★ **chekhov_fire**：本章节回收了哪些契诃夫枪（列表）

**爽点四级体系**：
- 微爽点 2:1 / 中爽点 4:1 / 大爽点 8-10:1 / 终极爽点 20:1

**情绪叠加模型**：每章推荐 2 种混合情绪（紧张+好奇 / 爽+不安 / 绝望+希望 / 释放+期待 / 震撼+敬畏）

❌ 铁律：幕内不允许连续 3 章同一情绪叠加组合。

### Step 3：交互闸门——场景卡试读

幕纲设计完成后。选 1 个中爽点章节，写 300-400 字正文释放段。

**选段标准（按优先级）**：
1. 这个幕最核心的爽点兑现时刻
2. 与 L0 爽点类型最相关
3. 正文中最可能出问题的位置

输出给老板确认。点头 → 进 Step 4。退回 → 回 Step 2。

❌ 写 3 段而不是 1 段。场景卡是交互产物，不归档。

### Step 4：节奏自检 + 平台校准

**节奏自检（7+3+3 项）**：
```
□ 连续无爽点章节 ≤ 3 章？
□ 连续无新信息章节 ≤ 2 章？
□ 高强度章间距 ≥ 3 章？
□ 情绪低谷长度 ≤ 2 章？
□ 终极爽点预留了整卷铺垫？
□ 大爽点前 2-3 章用叠加压缩蓄力了？
□ 幕的情绪弧线有"起"？
⭐ 核心三要素密度：连续 2 章同时缺(战斗/恐怖/成长) → 重设计
⭐ fun_level 零爽点：连续 2 章"零"→ 改
⭐ 事件数/多样性/字数基线
★ 支线空白检查：每条 plotline 的连续空白章数是否超过期望间隔的 2 倍？
★ 契诃夫枪检查：已设伏未回收的枪中，是否有超过 10 章仍未回收的？
★ 汇总表一致性：情节线纲汇总表与 act-XX.yaml 数据一致？
```

任何一项不通过 → 回 Step 2。

**平台校准**：

| 节点 | 番茄 | 起点 |
|:---|:----|:----|
| 第1章钩子 | 必须有 | 第5章前 |
| 核心卖点亮相 | 第5章内 | 第10章内 |
| 第一次战斗 | 第10章内 | 第20章内 |
| 第一个副本闭环 | 第15章内 | 第30章内 |
| 最大纯铺垫区间 | ≤ 1章 | ≤ 3章 |

不通过 → 节奏重排（不删内容，只调顺序）。重排后重跑核心三要素密度检查。

### Step 5：输出

- `设计/幕/act-XX.yaml` —— 幕纲设计（含 plotlines 定义 + 每章 plotlines_active / chekhov_set / chekhov_fire）
- `设计/幕/节奏自检报告.md` —— 自检结果（含支线空白 + 契诃夫枪检查）
- `设计/幕/情节线纲汇总表.md` —— 从 act-XX.yaml 自动渲染的罗琳式汇总表（可选产出）

**情节线纲汇总表格式参考**（从 act-XX.yaml 数据自动生成）：

```markdown
# 第1幕·觉醒之章 — 情节线纲汇总表

| CH | 主线 | 感情线 | 预言 | 契诃夫之枪 |
|:--:|:---:|:-----:|:---:|:---------|
|  1 | ■   | ☐     | ☐   | 🔫金手指来源设伏 |
|  2 | ■   | ■     | ☐   | |
|  3 | ■   | ☐     | ■   | |
| ...|     |       |     | |
| 20 | ■   | ☐     | ■   | 💥金手指来源回收 |
```

> ■ = 该支线在本章推进 / ☐ = 未出现 / 🔫 = 设伏 / 💥 = 回收
>
> 由 ESM before 阶段或用户手动触发渲染，不强制产出。

**Boss 战设计通则（需要时引用）**：
- P1 环境压制：展示 Boss 压迫感
- P2 全力对抗：常规手段不够用
- P3 打破极限：代价/外力/牺牲完成逆转
- 次级角色作为"情绪温度计"
- 环境作为"第二对手"

## ❌ 错误示例

### WRONG 1：跳过节点B直接填模板

```
用户：设计第一幕的幕纲
Agent：直接填 act-01.yaml 字段
❌ 错误：没有先思考"这个幕的第一性是什么"
✅ 正确：先停一下，输出节点B思考说明 →
   "这个幕是全书第一幕，核心任务是建立世界观和代入感"
   然后再填 act-XX.yaml
```

### WRONG 2：幕纲完成直接跳正文

```
Agent：幕纲产出 → 直接进 writer
❌ 错误：跳过了场景卡闸门 + 节奏自检 + 平台校准
✅ 正确：幕纲 → 场景卡试读 → [通过] 自检+校准 → writer
                                        [退回] 回幕纲调整
```

### WRONG 3：Boss 战只有一阶段

```
Ch20：主角和 Boss 对打，主角爆发取胜
❌ 错误：没有三阶段结构，缺少代价展示
✅ 正确：P1 被压制 → P2 常规手段不够 → P3 代价逆转
   战后影响后续 3 章
```

### WRONG 4：用户一开口就直接写多幕

```
用户：帮我做前 3 幕大纲
Agent：一口气创建 act-01.yaml / act-02.yaml / act-03.yaml
❌ 错误：违反单幕闸门原则，用户/老板尚未确认前一幕就提前规划
✅ 正确：先完成第 1 幕 → 场景卡试读 → 确认 → 再进第 2 幕
```

### WRONG 5：设计幕纲时不规划情节线

```
Agent：产出 act-01.yaml，每章只有 emotional_goal + payoff
❌ 错误：没有定义支线、没有分配每章推进的线、没有契诃夫枪规划
✅ 正确：先列出这个幕涉及的 plotlines → 定义每条线的期望频率 →
         规划契诃夫枪设伏/回收章号 → 每章标注 plotlines_active +
         chekhov_set / chekhov_fire
```

## 异常与边界条件

| # | 场景 | 触发条件 | 处理方式 |
|:-:|:-----|:---------|:---------|
| 1 | act 文件不存在 | 调用 act-XX.yaml 时发现文件缺失 | 先检查 project/design 结构是否完整；若无则从 Step 2 重建 |
| 2 | L1 设定未就绪 | 团队试图在 bootstrap 未完成时进入幕纲 | 驳回请求，返回 bootstrap 管线，输出"L1 未就绪清单" |
| 3 | 节点B 被跳过 | Agent 直接产出 act-XX.yaml 而没有节点B判断 | 人工拦截退回，要求补充节点B思考说明，不补不继续 |
| 4 | 场景卡被老板退回 | 老板看试读后不满意，退回 Step 2 | 记录退回原因，对照修改 emotional_goal 或 payoff 结构，改完再送审 |
| 5 | 节奏自检未通过 | 7+3+3 项有一项未达标 | 在自检报告中标红失败项，回 Step 2 重排爽点分布/支线分布，重排后重跑自检 |
| 6 | 平台校准未通过 | 番茄/起点节奏模板不匹配 | 只调章节顺序不改内容，重排后重跑核心三要素密度检查 |
| 7 | Boss 战缺少阶段 | act-XX.yaml 中 boss_fight 字段不足 3 个 phase | 退回补充 P1/P2/P3，确保代价展示 |
| 8 | 用户中途更换发布平台 | 已完成番茄校准后要求切起点（或反向） | 清空平台校准结果，重新加载目标平台节奏模板从头校准 |
| 9 | 场景卡存档冲突 | 场景卡被错误保存到 `设计/幕/` 造成命名冲突 | 不归档场景卡，移入 `_temp/` 目录，确保 `设计/幕/` 只放正式产出 |
| 10 | 多幕 YAML 命名冲突 | 多个 act-XX.yaml 的 XX 编号重复或跳号 | 校验命名连续性：从 act-01 开始、不重复、不跳号。冲突则重编号 |
| 11 | 老板设计中缺少阶段指示 | 项目无 Boss 战设计，但老板未明确标注"无 Boss" | 默认检查卷末高潮合理性；若高潮级别 ≥ 大爽点则要求补 Boss 战设计 |
| 12 | 用户中途更换项目 | 幕纲设计中突然要求切换到另一本书 | 冻结当前产出不删除，在新项目目录重建 pipeline，不允许跨项目复用 |
