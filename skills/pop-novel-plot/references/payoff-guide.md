# payoff-guide.md — 中爽点 + 大爽点 + 特大爽点 · 位置规划

> 管线: pop-novel-plot v6.1
> 定位：plot 专属。中爽点（新增责任）、大爽点、特大爽点的定义、位置规划方法、Canvas 联动。
> 小爽点 + 中爽点(事件执行) → `pop-novel-chapter-design/references/payoff-guide.md`

## 一、四级爽点总表

| 级别 | 是什么 | 密度 | 判定器 | 负责 |
|:-----|:------|:----:|:------|:----:|
| 小 | 干脆利落的斩杀/升级/打脸/梗 | 每章 ≥ 5 | 读者 3 秒读完，反应"嗯爽" | chapter-design |
| 中 | 有铺垫的释放。一条子线收束 | 每章 ≥ 1 | 读者想"终于"或"我就知道" | **plot (位置)+design(事件)** |
| **大** | **剧情层面的重大转折点。格局重画** | 间隔 ≤ 5 章 | **"故事进入新阶段了"** | **plot** |
| **特大** | **全书承诺兑现** | 每卷 ≥ 1 | **"原来前面就是为了这一章"** | **plot** |

## 二、中爽点（★ plot 新增责任）

### 2.1 定义

> 中爽点 = 一条子线经过 ≥3 章铺垫后的释放。释放那一刻，读者想的是"终于"，不是"嗯爽"。

小爽点是即时有结果（一刀一个）；中爽点前面 N 章在铺，这一章收。作为 plot agent，你的职责是：在 Canvas 中确保每条线的铺垫→释放节奏合理，每章至少有一条线处于"释放"状态。

### 2.2 责任调整（重要）

中爽点涉及跨章铺垫→释放结构，chapter-design 只看单章，无法编排"哪条线第几章铺、第几章收"——这是 plot 的职责。

| plot 负责 | chapter-design 负责 |
|:----------|:-------------------|
| Canvas 逐线标记 payoff_level=中 | 按 Canvas 线号设计具体事件 |
| 确保每章至少有一条线 payoff_level=中 | 标记该事件为 ★中爽点 |
| 跟踪每条线的铺垫→释放节奏 | Canvas 无供给时 → 自行制造 |
| 某条线连续 3+ 章只有铺垫无释放 → 协调提前释放或加线 | |

### 2.3 Canvas 填写流程

```
③ 填写节奏笔记
④ ★ 标注每条线的 payoff_level：
    - 中 = 这条线在这一章有一次可感知释放（信息揭示/关系转折/障碍突破/情绪落地）
    确保每章至少有 1 条线 = 中（不能存"本章各线全在铺垫"的情况）
    如果某章所有线都是空/小 → 把最近一条快释放的线提前到这里
④☆ 跟踪铺垫→释放节奏:
    - 每条线（M1/M2/M3/S1/S2）连续 ≥3 章只有铺垫无释放 → 下一章必须释放
    - 释放后允许该线再铺垫 2-3 章再释放（波浪节奏）
⑤ 汇总 payoff_map：逐章统计 ≥中 → total + 线号列表
```

### 2.4 设计案例（11 个——理解"一条线在 Canvas 中如何描"）

以下案例不是小说片段，而是每条子线的**设计逻辑**和**Canvas 描线方式**。你读出的是：
- 这条线在 Canvas 中如何分配铺垫章和释放章（payoff_level=空→中）
- 这条线释放后对故事格局的**设计意义**（不是文本效果）
- 当你在 act-skeleton.yaml 中填 canvas.entries 时，这个案例告诉你每条线的概述该写什么

> 11 个案例分 A/B/C 三类，分别对应 Canvas 中的不同线角色。

#### A. 战斗线：Canvas 中 M3（外部冲突）/ S（支线）的设计

*设计逻辑：创造威胁→堆砌准备→设置释放窗口*

| # | 案例名 | Canvas 线 | 铺垫章（payoff_level=空）的设计 | 释放章（payoff_level=中）的设计 | 释放的设计意义 |
|:-:|:-------|:-------:|:-------------------------------|:-------------------------------|:-------------|
| 1 | **被废→隐忍→爆发** | M2 成长 | ch1-ch7 该线持续空。设计手法：第1章给线一个高起点(被废)→ch3-5给隐忍窗口(奇遇但不暴露)→ch6-7加外部施压(兄弟中毒+家族嘲笑)，让读者觉得"这条线蓄了很久" | ch8 该线=中。设计手法：选择"在家族会议上当众爆发"而非私下打脸——所有关键人物都在场。先师弟挑衅→再一拳结束→最后对父亲说一句证明 | 这条线释放标志M2第一个里程碑完成。之后可以转去打元老线 |
| 2 | **元老逼宫→群压→一挑五** | M3 外部冲突 | ch43-ch46 该线持续空。设计手法：4 章内让三个不同层级的人连续施压(元老→家族第一高手→隐世长辈)，形成"所有人都针对主角"的错觉 | ch47 该线=中。设计手法：一掌震退第一高手为"量变到质变"的信号，再用一招控群技(春蚕丝雨捆绑五人)做"全面碾压"的视觉证明 | 这条释放闭合了"三月赌约"这条跨40章的长线，M3 外部冲突线升格为主角不可再被轻视 |
| 3 | **兄弟被杀→布局→伏击** | S 兄弟仇线 | ch53-ch60 该线连续空。设计手法：第1章给线一个情感锚点(瓦隆被杀)，不在下一章立即复仇——先压情绪，转去布局(分析+联盟+做局) | ch61 该线=中。设计手法：选择"巷战埋伏+二人配合"而非主角单枪匹马杀上门——前者有"部署被验证"的认知释放，后者只是打架赢了 | 这条线释放闭合一个"复仇子线"，同时建立主角的盟友圈(哲普)和团队战斗力认证 |
| 4 | **发现威胁→筹备→猎杀** | M3 外部线 | ch5-ch11 该线持续空。设计手法：先给一个"时间压力"→boss 在晋升传奇，拖久会输；再给"资源不足"→主角打不过，需要雇佣帮手 | ch12 该线=中。设计手法：让主角和帮手以"两阶段配合"(主坦+跳斩)完成击杀，而非主角单人爆发——强调"充分准备后的胜利" | 这条线释放证明了：对于超出当前等级的敌人，正确的设计方法是"准备线≥战斗线" |
| 5 | **地下城探索→传送到深渊→苦战蛇魔** | M3 探索线 | ch11-ch15 该线连续空。设计手法：设计了多段"推进"但不释放——发现入口(不释放)→过陷阱(不释放)→抽牌传送到深渊(不释放)→被蛇魔重伤(不释放) | ch16 该线=中。设计手法：让主角在"精心准备后(分析弱点+升级)"用一次战术动作(跳斩)终结强敌，而非硬碰硬磨死 | 这条线释放同时产出"二阶进阶"——释放不止一个事件，而是事件+成长打包，释放感更强 |

#### B. 成长线：Canvas 中 M2（主角成长）的设计

*设计逻辑：给主角一个"快要跨过去的槛"，再在压力最大化时拆掉它*

| # | 案例名 | Canvas 线 | 铺垫章设计 | 释放章设计 | 意义 |
|:-:|:-------|:-------:|:----------|:----------|:----|
| 6 | **第一次实战→紧张→一刀** | M2 成长线 | ch28-ch29 连续空。设计手法：不放实战——先做"考前紧张"(ch28)，再做"环境渲染+教官叮嘱"(ch29)，让读者感受到"这是第一次，可能会出事" | ch30 该线=中。设计手法：怪物的压迫感要推到极限(重量级描写：悍马撞击感+獠牙如弯刀)，让读者以为会有一场苦战→然后一刀解决 | 这条线释放第一次实战的"圆满完成"，M2 成长线进入新阶段——从"训练"到"实战" |
| 7 | **觉醒→隐藏30章→绝境公开** | M2 成长线 | ch19-ch49 持续空。设计手法：不给任何公开使用的机会——每次战斗都用刀法/身法解决，只让读者知道"主角有底牌但不用" | ch50 该线=中。设计手法：选择"全队生死绝境"做为释放窗口——队友以为必死无疑、铺了上千头怪兽的压力、敌人已经撤退等着收尸 | 这条线释放的设计意义：读者憋了 30 章，看到六柄飞刀悬空那一刻是"终于！"而非"哦" |
| 8 | **有力量没功法→众矢之的→烧圣旨悟道** | M2 成长线 | ch5-ch7 连续空。设计手法：前 3 章先给力量(熊狼蟒三力)但不给功法，外部压力逐章递增(十皇子威胁→十八皇子警告→被克扣用度) | ch8 该线=中。设计手法：用一个"消耗性资源(烧圣旨)"而非"正常修炼"解锁——让读者觉得"这是聪明人找到的捷径"而非"运气好捡到" | 这条线释放直接从凡境→宗师，M2 线完成一次质变。后续设计空间打开(天子封神术的后续境界) |

#### C. 复仇/打脸线：Canvas 中 S 线（羁绊）/ M1线（世界危机）的设计

*设计逻辑：让"受欺"重复多次但每次都差一点，在读者开始不耐烦前一次解决*

| # | 案例名 | Canvas 线 | 铺垫章设计 | 释放章设计 | 意义 |
|:-:|:-------|:-------:|:----------|:----------|:----|
| 9 | **被当众欺压→被封国公成靶子→狩猎证明** | M3+S | ch17-ch19 连续空。设计手法：十皇子当众出手(被化解但没反击)→被赐爵位反而成靶子→蛮族大将入林，形成"众目睽睽之下必须证明自己"的公开压力 | ch20 该线=中。设计手法：选公开狩猎场做释放场景，让所有王公贵族当观众。九十九掌不是"打了很久"，是"打完还气定神闲"才是释放点 | 这条线释放同时解决了三条压力线(十皇子施压/朝野质疑/猎杀考验)，一拖三是最经济的释放设计 |
| 10 | **人贩子密谋→妹妹反杀→深夜复仇** | S 守护线 | ch45-ch47 连续空。设计手法：威胁从"密谋"(ch45)→"动手"(ch46)→"妹妹独自解决"(ch47)，逐章升级但不给主角出手机会 | ch48 该线=中。设计手法：主角不是打架式复仇——用毒弩(远程)→尸体处理(专业)→弯刀收割(快速)，体现"冷静的猎人"而非"愤怒的莽夫" | 这条线释放的设计意义：让读者看到主角处理威胁的方式不同于小爽点(一刀一个)——有计划、有毒药、有收尾 |
| 11 | **荒野遇怪物→科普→组队歼灭** | M3 外部线 | ch30-ch33 连续空。设计手法：先展示怪物如何可怕(杀死马匹)→再科普其生物习性(让读者知道这不是杂兵)→然后组织队伍(展示主角的战术判断力) | ch34 该线=中。设计手法：不是主角 solo 清场——是学主角上次的方式"从甲壳缝隙刺入大脑"(验证主角的方案有效)，再收战利品和经验 | 这条线释放证明了主角的"野外生存+团队领导"能力，M3 线从"个人行动"进入"组织队伍"阶段 |

**plot 理解要点**：以上 11 条子线的共同设计模式是：铺垫 N 章（≥3，逐章叠加压力但不释放）→ 设置 1 个释放窗口（该章 payoff_level=中）。在 Canvas 中描线时：铺垫阶段该线 payoff_level 全为 空，释放章写 中。释放后允许该线再空 2-3 章再放一次中（波浪节奏）。

## 三、大爽点

### 3.1 定义

> 大爽点 = 一条或多条主线的释放改变了故事的格局。释放后，下一章的语境已经不一样了——地图被重画了。

**判定器**：释放之后，读者能不能清晰地感觉到"从这一章起，故事的语境变了"？

### 3.2 案例

> ⏳ **待从参考书中提取。** 大爽点须逐卷通读才能精确定位"剧情格局变化"的关键转折点。大爽点案例库将按同一批书逐本提取。

**提取标准**：
- 释放后，故事语境发生根本性变化（下一章读者已经知道"不一样了"）
- 读者能清晰感觉到"从这一章起进入新阶段"
- 铺垫至少覆盖半卷到一卷的长度

### 3.3 Canvas 联动设计方法

**核心逻辑**：Canvas 矩阵中某条线的 payoff_level=大 → 那一章就是大爽点章。

```
plot Step 2 设计 Canvas 时逐线标记 payoff_level：
  - 空 = 铺垫/建设/埋伏笔
  - 小 = 干脆利落的斩杀/升级/打脸/梗植入
  - 中 = 这条线在这里有一次可感知的释放（信息揭示/关系转折）
  - 大 = ★ 这条线在这章爆发——前面的蓄力全部释放。故事从这章起进入新阶段

填完 Canvas 后扫描：
  → 所有 payoff_level=大 的格子 → 大爽点章号
  → 验证间隔 ≤ 5 章
  → 大爽点前 2-3 章标记蓄力
```

**大爽点间隔保证**：
- 扫描本幕所有大爽点章号
- 如果相邻大爽点间隔 > 5 章 → 把中间某条线的释放提前
- 大爽点前 2-3 章做叠加压缩蓄力

## 四、特大爽点

### 4.1 定义

> 特大爽点 = 全卷乃至全书的终极释放。不是一条线收束，不是格局变化——是**作者从第 1 章就在铺垫的那个承诺，终于兑现了**。

**判定器**：读者读完这一章后会不会在心里把前面整本重新翻一遍？会不会觉得"前面那么多章就是为了这一章"？

### 4.2 案例

> ⏳ **待从参考书中提取。**

**提取标准**：
- 读者读完这一章后会在心里把前面整本重新翻一遍
- 作者从第 1 章就在铺垫的那个答案终于揭晓
- 释放不是"这条线收了"，而是"原来这本书写的是这个"

### 4.3 Canvas 联动设计方法

**特大爽点 = Canvas 中多条线 payoff_level=大 汇聚到同一章或连续两章 + 全卷情绪极值。**

```
plot 填完 Canvas 后：
1. 扫描所有 payoff_level=大 的格子
2. 找多线汇聚章（≥ 2 条线同为 大 的章）→ 特大爽点候选
3. 幕约 1/2-2/3 处选一个做幕中爆发
4. 幕末最后一章做幕末收束
5. 每卷至少 1 个特大爽点
```

### 4.4 责任归属

| plot 负责 | plot 不负责 |
|:----------|:-----------|
| Canvas 逐线标注 payoff_level（中/大/特大） | 具体事件内容（归 chapter-design） |
| 确保每章 ≥ 1 条线有中级别释放 | |
| 跟踪各线铺垫→释放节奏 | |
| Canvas 汇总 payoff_map 交给 design | |
| 大爽点间隔 ≤ 5 章 / 特大 ≥ 1 个/卷 | |

## 五、Canvas payoff_map 交付物

```
payoff_map:
  ch1:  total=0 → ⚠️ 自行制造 ≥ 5 小爽点 + ≥ 1 中爽点
  ch3:  total=1 (lines=["M1:中"]) → ✅ 供给 1 中爽点，按线号定位
  ch7:  total=1 (lines=["M1:大"]) → ★ 大爽点章，全幕最高密度事件链
  ch11: total=1 (lines=["M3:大"]) → ★ 大爽点章
  ch14: total=3 (lines=["M1:大","M2:大","M3:中"]) → ★★ 特大爽点候选（2线大汇聚）
```

**消费规则**：
- total=0 → chapter-design 自行制造
- 某线=大 → 大爽点章
- ≥2 线=大 → 特大爽点章
