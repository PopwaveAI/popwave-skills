## 一、四级爽点总表

| 级别 | 是什么 | 密度 | 判定器 | 负责 |
|:-----|:------|:----:|:------|:----:|
| 小 | 干脆利落的斩杀/升级/打脸/梗 | 每章 ≥ 5 | 读者 3 秒读完，反应"嗯爽" | chapter-design |
| 中 | 有铺垫的释放。一条子线收束 | 每章 ≥ 1 | 读者想"终于"或"我就知道" | **plot (位置)+design(事件)** |
| **大** | **剧情层面的重大转折点。格局重画** | 间隔 ≤ 5 章 | **"故事进入新阶段了"** | **plot** |
| **特大** | **全书承诺兑现** | 每卷 ≥ 1 | **"原来前面就是为了这一章"** | **plot** |

## 二、中爽点（★ plot 新增责任）

### 2.1 定义

> 中爽点 = 一条子线经过 ≥3 章铺垫后的释放。释放那一刻，读者想的是"终于"，不是"嗯爽"。

小爽点是即时有结果（一刀一个）；中爽点前面 N 章在铺，这一章收。作为 plot agent，你的职责是：在 Canvas 中确保每条线的铺垫→释放节奏合理，每章至少有一条线处于"释放"状态。

### 2.2 责任调整（重要）

中爽点涉及跨章铺垫→释放结构，chapter-design 只看单章，无法编排"哪条线第几章铺、第几章收"——这是 plot 的职责。

| plot 负责 | chapter-design 负责 |
|:----------|:-------------------|
| Canvas 逐线标记 payoff_level=中 | 按 Canvas 线号设计具体事件 |
| 确保每章至少有一条线 payoff_level=中 | 标记该事件为 ★中爽点 |
| 跟踪每条线的铺垫→释放节奏 | Canvas 无供给时 → 自行制造 |
| 某条线连续 3+ 章只有铺垫无释放 → 协调提前释放或加线 | |

### 2.3 Canvas 填写流程

```
③ 填写节奏笔记
④ ★ 标注每条线的 payoff_level：
    - 中 = 这条线在这一章有一次可感知释放（信息揭示/关系转折/障碍突破/情绪落地）
    确保每章至少有 1 条线 = 中（不能存"本章各线全在铺垫"的情况）
    如果某章所有线都是空/小 → 把最近一条快释放的线提前到这里
④☆ 跟踪铺垫→释放节奏:
    - 每条线（M1/M2/M3/S1/S2）连续 ≥3 章只有铺垫无释放 → 下一章必须释放
    - 释放后允许该线再铺垫 2-3 章再释放（波浪节奏）
⑤ 汇总 payoff_map：逐章统计 ≥中 → total + 线号列表
```

## 三、大爽点

### 3.1 定义

> 大爽点 = 一条或多条主线的释放改变了故事的格局。释放后，下一章的语境已经不一样了——地图被重画了。

**判定器**：释放之后，读者能不能清晰地感觉到"从这一章起，故事的语境变了"？

### 3.2 案例

> ⏳ **待从参考书中提取。** 大爽点须逐卷通读才能精确定位"剧情格局变化"的关键转折点。大爽点案例库将按同一批书逐本提取。

**提取标准**：
- 释放后，故事语境发生根本性变化（下一章读者已经知道"不一样了"）
- 读者能清晰感觉到"从这一章起进入新阶段"
- 铺垫至少覆盖半卷到一卷的长度

### 3.3 Canvas 联动设计方法

**核心逻辑**：Canvas 矩阵中某条线的 payoff_level=大 → 那一章就是大爽点章。

```
plot Step 2 设计 Canvas 时逐线标记 payoff_level：
  - 空 = 铺垫/建设/埋伏笔
  - 小 = 干脆利落的斩杀/升级/打脸/梗植入
  - 中 = 这条线在这里有一次可感知的释放（信息揭示/关系转折）
  - 大 = ★ 这条线在这章爆发——前面的蓄力全部释放。故事从这章起进入新阶段

填完 Canvas 后扫描：
  → 所有 payoff_level=大 的格子 → 大爽点章号
  → 验证间隔 ≤ 5 章
  → 大爽点前 2-3 章标记蓄力
```

**大爽点间隔保证**：
- 扫描本幕所有大爽点章号
- 如果相邻大爽点间隔 > 5 章 → 把中间某条线的释放提前
- 大爽点前 2-3 章做叠加压缩蓄力

## 四、特大爽点

### 4.1 定义

> 特大爽点 = 全卷乃至全书的终极释放。不是一条线收束，不是格局变化——是**作者从第 1 章就在铺垫的那个承诺，终于兑现了**。

**判定器**：读者读完这一章后会不会在心里把前面整本重新翻一遍？会不会觉得"前面那么多章就是为了这一章"？

### 4.2 案例

> ⏳ **待从参考书中提取。**

**提取标准**：
- 读者读完这一章后会在心里把前面整本重新翻一遍
- 作者从第 1 章就在铺垫的那个答案终于揭晓
- 释放不是"这条线收了"，而是"原来这本书写的是这个"

### 4.3 Canvas 联动设计方法

**特大爽点 = Canvas 中多条线 payoff_level=大 汇聚到同一章或连续两章 + 全卷情绪极值。**

```
plot 填完 Canvas 后：
1. 扫描所有 payoff_level=大 的格子
2. 找多线汇聚章（≥ 2 条线同为 大 的章）→ 特大爽点候选
3. 幕约 1/2-2/3 处选一个做幕中爆发
4. 幕末最后一章做幕末收束
5. 每卷至少 1 个特大爽点
```

### 4.4 责任归属

| plot 负责 | plot 不负责 |
|:----------|:-----------|
| Canvas 逐线标注 payoff_level（中/大/特大） | 具体事件内容（归 chapter-design） |
| 确保每章 ≥ 1 条线有中级别释放 | |
| 跟踪各线铺垫→释放节奏 | |
| Canvas 汇总 payoff_map 交给 design | |
| 大爽点间隔 ≤ 5 章 / 特大 ≥ 1 个/卷 | |

## 五、Canvas payoff_map 交付物

```
payoff_map:
  ch1:  total=0 → ⚠️ 自行制造 ≥ 5 小爽点 + ≥ 1 中爽点
  ch3:  total=1 (lines=["M1:中"]) → ✅ 供给 1 中爽点，按线号定位
  ch7:  total=1 (lines=["M1:大"]) → ★ 大爽点章，全幕最高密度事件链
  ch11: total=1 (lines=["M3:大"]) → ★ 大爽点章
  ch14: total=3 (lines=["M1:大","M2:大","M3:中"]) → ★★ 特大爽点候选（2线大汇聚）
```

**消费规则**：
- total=0 → chapter-design 自行制造
- 某线=大 → 大爽点章
- ≥2 线=大 → 特大爽点章
