在全卷视角下建立 N 卷蓝图，回答"这本书整体长什么样"。后续 volume-XX.md 从本书架中拆取单卷战略。

## 执行

### 1. 卷拆分

从 story-engine 的核心假说 + 终点快照反推全卷轮廓。

| 字段 | 说明 | 示例 |
|:-----|:-----|:------|
| 全书总章数 | 预估 | ch001 — ch800 |
| 总卷数 | N 卷 | 4-7 卷 |
| 每卷章节范围 | 卷 1 从 chX 到 chX | 卷1: ch001-080 / 卷2: ch081-200 / ... |

**每卷定义：**

| 字段 | 说明 |
|:-----|:-----|
| 卷号 | 第 N 卷 |
| 章节范围 | chX — chX |
| 核心命题 | 一句话：这卷的核心剧情是什么 |
| 读者感受 | "从 X 到 Y" |
| 主角起点/终点等级 | 对应 act_rank_schedule 的阶段 |
| 主角心理变化 | 从什么心态到什么心态 |
| 核心矛盾 | 本卷的 A vs B |
| 下一卷的钩子 | 卷末留下什么悬念 |

> 卷 1-4 级架构：故事从哪里开始（起点快照）→ 中间有哪些大阶段 → 到哪里结束（终点快照）。
> 卷 5-7 级细节：起始在操作中确认后延伸细化，初始为"待定 N 章"。

### 5. 首卷黄金窗口（卷 1 硬性闸门）

> 番茄平台标准。卷 1 前 2 章决定读者是否弃书。

| 节点 | 硬性要求 | 不满足→ |
|:-----|:--------|:-------|
| ch01 核心卖点亮相 | **第 1 章内** 必须亮出本书独一无二的卖点 | P0 退回——读者不知道"这本书是什么" |
| ch01 章末钩子 | **必须有** 可感知的悬念驱动力 | P0 退回——读者没有翻下一章的理由 |
| ch02 第一次战斗/重大矛盾 | **第 2 章内** 必须发生可感知的冲突 | P0 退回——读者没尝到爽感 |
| ch01-ch02 纯铺垫区间 | **≤ 1 章** 连续无事件推进 | P0 退回——读者走神 |

> 全书架构阶段即须确认 ch01/ch02 的核心事件落入上述窗口。不能在幕纲阶段才发现前两章没有卖点或战斗——那已经太晚了。
