## 执行流程

### 1. 前置条件检查

在进入本幕设计前，必须先确认以下条件全部满足：

- [ ] project 已通过 bootstrap 开书启动（L0 + PRD + L1）
- [ ] 世界稳定性检验已通过
- [ ] reader_profile 已嵌入 project.yaml
- [ ] constitution.yaml 已加载

❌ 任何一项不满足 → **先调 bootstrap**，不允许跳过。

### 2. 节点B · 幕设计前思考

写幕纲之前先停下来想清楚。按 `templates/checkpoint-b.md` 模板，输出散文式判断链，覆盖：

1. **对齐检查**：上游 L0/L1 设计与这个幕一致吗？
2. **核心判断**：这个幕的第一性是什么？在全书扮演什么角色？
3. **情绪弧线**：起点 → 顶点 → 终点 合理吗？
4. **双轴检查**：对抗有"强度 + 深度"两个维度？
5. **风险扫描**：平台节奏满足？爽点分布有连续空白？
6. **情节线预感**：直觉上这个幕可能涉及哪几条线？

### 3. 输出

产出 `设计/幕/节点B-XX.md`，按 `templates/checkpoint-b.md` 格式。
