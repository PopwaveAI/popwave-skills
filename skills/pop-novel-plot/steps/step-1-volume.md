# Step 1：卷设计

> 产出: `设计/卷/volume-XX.md`
> 模板: `templates/volume-design.md`
> 管线: pop-novel-plot v5.0

---

## 目的

一卷 = 80-150 章，一个大剧情的完整呈现。本步骤产出卷级战略（目标/背景/剧情线/快照/势力动机），供后续幕纲编排和下游 chapter-design 消费。

---

## 前置条件

- [ ] project 已通过 bookstrap 开书启动（L0 + PRD + L1）
- [ ] 状态/角色/角色卡已加载
- [ ] 起点快照 + 终点快照已用户确认

---

## 执行

### 1. 卷级定义

确定一卷的框架性信息（口述给用户确认，不写文件）：

| 字段 | 说明 | 示例 |
|:-----|:-----|:------|
| 卷号 | 第几卷 | 第 1 卷 |
| 章节范围 | 从 chX 到 chX | ch1 — ch100 |
| 幕划分 | 本卷含几幕、每幕的章段 | 幕1: ch1-ch25 / 幕2: ch26-ch50 / 幕3: ch51-ch75 / 幕4: ch76-ch100 |
| 本卷核心命题 | 一句话 | "主角从荒村少年到南疆城站稳脚跟" |
| 读者结束感受 | "从 X 到 Y" | "从'这世界好可怕'到'主角要硬刚回去了'" |

### 2. 填充卷背景

按 `templates/volume-design.md` 模板逐节填充：

**§一 卷级定义** ← 从上一步口述结果写入
**§二 快照** ← 从起点快照 + 终点快照 + act_rank_schedule 提取
**§三 背景** ← 设计：
  - 时间范围（起始/结束/节奏提示）
  - 地理范围（本卷活动区域列表 + 叙事功能）
  - 角色池（主角从 Lv1 角色卡继承 + 核心配角 + 功能角色）
  - **势力动机（★ v6.0）**：本卷活跃势力各想达成什么、动机是什么
**§四 剧情线** ← M1/M2/M3 必选，S1/S2 可选。每条线一句话定义+本卷推进+预期频次

### 3. 用户确认闸门

展示 `设计/卷/volume-XX.md` 给用户 → 用户点头后才进 Step 2 的幕纲编排。

> ⚠️ 用户点头前不进幕纲设计。卷定义错了，后面的幕纲全是错的。

---

## 产出

`设计/卷/volume-XX.md` — 单文件，下游 chapter-design 的卷级 Canvas。
