# Step 11：节奏自检 + 平台校准

> 所属管线: pop-novel-plot v4.1+
> 模板: `templates/rhythm-check.md`

---

## 执行

### 节奏自检（7+3+3 项）

按 `templates/rhythm-check.md` 模板逐项检查：

**基础 7 项**：
- 连续无爽点章节 ≤ 3 章
- 连续无新信息章节 ≤ 2 章
- 高强度章间距 ≥ 3 章
- 情绪低谷长度 ≤ 2 章
- 终极爽点预留了整卷铺垫
- 大爽点前 2-3 章用叠加压缩蓄力了
- 幕的情绪弧线有"起"

**核心三要素**：
- 连续 2 章同时缺(战斗/恐怖/成长) → 重设计
- fun_level 零爽点连续 2 章 → 改
- 事件数/多样性/字数基线

**线-里程碑交叉**：
- 所有里程碑在30章内有对应推进节点
- 无里程碑无任何线与之对应
- 无线贯穿始终但从未触达任何里程碑
- 每个 MK 至少有1条 M 线与之对应
- M1/M2/M3 各自至少触达1个 MK

**契诃夫枪**：
- 已设伏未回收的枪中，无超过 10 章仍未回收的
- 情节线纲汇总表与 act-XX.yaml 数据一致

**信息释放**：
- 连续无新信息章节 ≤ 2 章
- P0 级信息全部在本幕内分配
- 第1章新概念 ≤ 2 个
- 每个 info_release 标注了 source_doc

任何一项不通过 → 回 Step 3-b（爽点/支线/信息释放重排）。

### 平台校准

| 节点 | 番茄 | 起点 |
|:---|:----|:----|
| 第1章钩子 | 必须有 | 第5章前 |
| 核心卖点亮相 | 第5章内 | 第10章内 |
| 第一次战斗 | 第10章内 | 第20章内 |
| 第一个副本闭环 | 第15章内 | 第30章内 |
| 最大纯铺垫区间 | ≤ 1章 | ≤ 3章 |

不通过 → 节奏重排（不删内容，只调顺序）。重排后重跑核心三要素密度检查。
