## 一、四级爽点总表

| 级别 | 是什么 | 密度 | 判定器 | 负责 |
|:-----|:------|:----:|:------|:----:|
| 小 | 干脆利落的斩杀/升级/打脸/梗 | 每章 ≥ 3 | 读者 3 秒读完，反应"嗯爽" | chapter-design |
| 中 | 有铺垫的释放。一条子线收束 | 每章 ≥ 1 | 读者想"终于"或"我就知道" | chapter-design |

## 二、大爽点

### 2.1 定义

> 大爽点 = 一条或多条主线的释放改变了故事的格局。释放后，下一章的语境已经不一样了——地图被重画了。

**判定器**：释放之后，读者能不能清晰地感觉到"从这一章起，故事的语境变了"？

### 2.2 案例

> ⏳ **待从参考书中提取。** 大爽点须逐卷通读才能精确定位"剧情格局变化"的关键转折点。当前已完成小爽点首轮提取（10 个），大/特大爽点案例库将按同一批书逐本提取。

**提取标准**：
- 释放后，故事语境发生根本性变化（下一章读者已经知道"不一样了"）
- 读者能清晰感觉到"从这一章起进入新阶段"
- 铺垫至少覆盖半卷到一卷的长度

### 2.3 Canvas 联动设计方法

**核心逻辑**：Canvas 矩阵中某条线的 payoff_level=大 → 那一章就是大爽点章。

```
plot Step 2 设计 Canvas 时逐线标记 payoff_level：
  - 空 = 铺垫/建设/埋伏笔
  - 小 = 干脆利落的斩杀/升级/打脸/梗植入
  - 中 = 这条线在这里有一次可感知的释放（信息揭示/关系转折）
  - 大 = ★ 这条线在这章爆发——前面的蓄力全部释放。故事从这章起进入新阶段

填完 Canvas 后扫描：
  → 所有 payoff_level=大 的格子 → 大爽点章号
  → 验证间隔 ≤ 5 章
  → 大爽点前 2-3 章标记蓄力
```

**大爽点间隔保证**：
- 扫描本幕所有大爽点章号
- 如果相邻大爽点间隔 > 5 章 → 把中间某条线的释放提前
- 大爽点前 2-3 章 Accumlation（叠加压缩蓄力）

## 三、特大爽点

### 3.1 定义

> 特大爽点 = 全卷乃至全书的终极释放。不是一条线收束，不是格局变化——是**作者从第 1 章就开始铺垫的那个承诺，终于兑现了**。

**判定器**：读者读完这一章后会不会在心里把前面整本重新翻一遍？会不会觉得"前面那么多章就是为了这一章"？

### 3.2 案例

> ⏳ **待从参考书中提取。** 特大爽点须全书通读才能精确定位"全书承诺兑现"时刻。当前已完成小爽点首轮提取（10 个），大/特大爽点案例库将按同一批书逐本提取。

**提取标准**：
- 读者读完这一章后会在心里把前面整本重新翻一遍
- 作者从第 1 章就在铺垫的那个答案终于揭晓
- 释放不是"这条线收了"，而是"原来这本书写的是这个"

### 3.3 Canvas 联动设计方法

**特大爽点 = Canvas 中多条线 payoff_level=大 汇聚到同一章或连续两章 + 全卷情绪极值。**

```
plot 填完 Canvas 后：
1. 扫描所有 payoff_level=大 的格子
2. 找多线汇聚章（≥ 2 条线同为 大 的章）→ 特大爽点候选
3. 幕约 1/2-2/3 处选一个做幕中爆发
4. 幕末最后一章做幕末收束
5. 每卷至少 1 个特大爽点
```

### 3.4 责任归属

| plot 负责 | plot 不负责 |
|:----------|:-----------|
| Canvas 逐线标注 payoff_level | 小爽点+中爽点具体事件（归 chapter-design） |
| Canvas 汇总 payoff_map 交给 design | 具体事件内容 |
| 大爽点间隔 ≤ 5 章 / 特大 ≥ 1 个/卷 | |
| 大爽点前蓄力标记 / 特大爽点多线汇聚设计 | |

## 四、Canvas payoff_map 交付物

```
payoff_map:
  ch1:  total=0 → ⚠️ 自行制造 ≥ 3 小爽点 + ≥ 1 中爽点
  ch3:  total=1 (lines=["M1:中"]) → ✅ 供给 1 中爽点，按线号定位
  ch7:  total=1 (lines=["M1:大"]) → ★ 大爽点章，全幕最高密度事件链
  ch11: total=1 (lines=["M3:大"]) → ★ 大爽点章
  ch14: total=3 (lines=["M1:大","M2:大","M3:中"]) → ★★ 特大爽点候选（2线大汇聚）
```

**消费规则**：
- total=0 → chapter-design 自行制造
- 某线=大 → 大爽点章
- ≥2 线=大 → 特大爽点章
