# 正文渲染 / 上色表达 v2.0

消费 pop-novel-chapter-design 的骨架产物，直接应用 `00-原始设定/文风DNA/` 文风DNA 渲染为可读正文。

**核心约束：不碰剧情。** 不读上游 Canvas、不验证设定、不判断角色出场是否合理。Design 说了这章发生什么 → 我只管写好。

**v2.0 核心变更：文风DNA从项目本地读取。**  
`00-原始设定/文风DNA/*.md` 已经在 Step 1 读入 context，渲染时直接应用，不经过中间风格契约。

---

## ❌ 质量红线（开工前→完工后自检）

- [ ] **设计包存在** — chXXX-设计包.md 完整可读
- [ ] **设计包已读取** — 角色状态 + 事件链 + 场景类型已提取
- [ ] **文风DNA 已读取** — `00-原始设定/文风DNA/` 下有对应档案（Step 1 已读入 context）
- [ ] **锚定章已感知** — 本章场景类型对应的锚定章特征已建立（如有）
- [ ] **所有事件已渲染** — 每个骨架事件在正文中有对应段落
- [ ] **风格验证通过** — 无风格偏差 ≥ 2 处，对照 `00-原始设定/文风DNA/` 原文验证
- [ ] **constitution 红线未触发**
- [ ] **叙事者不解释** — 没有"他意识到/他感到/他仿佛"等 AI 观感词

## 执行流程

```
Step 1 — 读入输入
  读设计包 + 00-原始设定/文风DNA/*.md + 锚定章 + 宪法
  ↓
  （文风DNA 进入 context，直接用于后续渲染。不产出中间文件）

Step 2 — 逐事件渲染（★ 核心）
  开幕段落 → 逐个事件（理解意图→选 POV→写段落→过渡）→ 闭幕钩子
  直接应用文风DNA中对应维度的模式
  ↓

Step 3 — 风格验证
  对照文风DNA原文（不是对照规则列表）→ 标记偏差 → 最小修补
  ↓

Step 4 — 最终输出
  完整性检查 → 写入 03-正文/chXXX.md + 章末状态更新块
```

## ❌ 错误示例

### WRONG 1：叙事者跳出来解释
→ 「他手里的剑很锋利——这把武器是他三年前从师门继承的，那场变故之后他一直在熟悉它的重量。」  
→ ✅ 正确：「剑刃在月光下泛着冷白。缠绳握柄上被拇指磨出了一道光滑的凹痕。」

### WRONG 2：因为风格规则而跳过骨架事件
→ 骨架有「主角向盟友提问」事件，但 Render 觉得「对话太长不符合简洁风格」就直接删了。  
→ ✅ 正确：骨架说了有对话就必须写对话。语言可以简洁，事件不可跳过。

### WRONG 3：读上游 Canvas
→ Render 读了 act-XX.yaml 或 L1 设定。  
→ ✅ 正确：Render 不知道这些文件的存在。只读骨架 + 人物卡 + 风格材料。

### WRONG 4：重新提取风格契约
→ 先写一份"风格契约"，再用契约渲染。这是多此一举——文风DNA文件已经完整可读。  
→ ✅ 正确：Step 1 读完文风DNA，context 中已有。Step 2 直接渲染。
