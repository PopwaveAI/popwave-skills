# payoff-guide.md — 文本脉冲 · 写作质量检查

> 管线: pop-novel-prose-render v2.0
> 定位：prose-render 专属。**文本脉冲不是爽点层级**——它是写作质量检查。
> 爽点体系四档（小/中/大/特大）全部在 plot + chapter-design 中规划。

---

## 一、重要说明

二轮对齐后确认：所有四级爽点（小/中/大/特大）都是**事件级或更高层级的产物**，必须在 chapter-design 的事件链或 plot 的幕纲中规划。

prose-render 不承担任何爽点级别的规划。但有一个独立的写入质量检查：**文本脉冲**。

---

## 二、文本脉冲定义

> 文本脉冲 = 一个段落内，一次期待推翻或感官锐化。读者不会停下来想，但会因为这些微脉冲持续往下扫。

与小爽点的区别：
- 小爽点 = 一个完整的"一刀斩杀"/"一次升级"事件 → chapter-design 设计
- 文本脉冲 = 一句"他以为门锁了一推就开" → prose-render 写作时自然产生

---

## 三、五种类型 + 案例

| 类型 | 一句话案例 |
|:-----|:---------|
| **小反转** | "他以为门锁了——一推就开。" "刀拔出来的瞬间他看到了刀背。" |
| **信息碎片** | "血的流向不对。不是从伤口往外流，是从门口往伤口流。" |
| **角色微表情** | '"你走吧。"她没抬头，但手里的茶杯抖了一下。' |
| **环境锐化** | "风从北边来，带的不只是沙子——还有铁锈味。" |
| **节奏剪刀** | "三。二。一。火光亮了。" |

---

## 四、密度自检

**在 Step 3 风格验证中执行。**

```
逐 500 字块扫描：
- [ ] 每个 500 字块内至少有一次"期待推翻"或"感官锐化"？
- [ ] 是否有超过 500 字连续无微脉冲的段落？
- [ ] 章末倒数 500 字密度最高？（读者最后看到的决定"下一章点不点"）

不通过 → 回到最平淡的段落，插入小反转/微表情/环境锐化。
不增删事件、不改角色行为、不改信息——只加文本技艺。
```

---

## 五、责任归属

| prose-render 负责 | prose-render 不负责 |
|:-------------------|:--------------------|
| 写作中自然产生文本脉冲 | 小爽点/中爽点/大爽点/特大爽点（全部归上游） |
| Step 3 扫描脉冲密度 | 改剧情、改事件（不碰设计包） |
