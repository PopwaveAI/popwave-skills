# Step 1：读入输入

> 管线: pop-novel-prose-render v1.0

---

## 目的

读取 Design 的骨架产物 + 风格材料，为渲染建立基线。

---

## 前置条件

- [ ] `03-写作资产/chXXX-事实骨架.md` 存在
- [ ] `03-写作资产/chXXX-登场人物卡.md` 存在
- [ ] styles/ 目录下有文风DNA 档案
- [ ] constitution.yaml 存在

---

## 执行

### 1. 读事实骨架

从 `chXXX-事实骨架.md` 提取：
- 本章核心目的 → 理解这场戏的叙事意图
- 场景类型（combat/dialogue/discovery/crisis/transition/mix）→ 选择风格规则
- 事件链（数量、内容、情绪目标、信息释放）→ 逐事件渲染的蓝图
- 情绪节拍（起点→发展→高潮→终点）→ 指导句式节奏
- 钩子 → 章末段落的写法

### 2. 读登场人物卡

从 `chXXX-登场人物卡.md` 提取：
- 每个角色的 before 状态 → 事件的起点状态
- 每个角色的 after 状态 → 事件的结束状态
- 每个角色的台词风格 → 对话写法
- 角色关系动态 → 对话潜文本

### 3. 读风格材料

**文风DNA 档案**（`styles/`）：
- 读全文，重点提取本章场景类型对应的维度

**锚定章库**（如有，`01-写作资产/锚定章库/`）：
- 找到与本章场景类型匹配的锚定章
- 只取「核心特征提炼」+「可复用规则」，不取原文摘录

**上一章正文最后 800 字**（`03-正文/ch{上一章}.md`）：
- 语感衔接——不是复读内容，是感受句子的质感：
  - 现在的句子是长的还是短的？
  - 上一章结尾是安静还是紧张？
  - 叙事者的声音是近的还是远的？

### 4. 读宪法

`constitution.yaml`：提取红线，在 Phase 3 验证。
