# Step 1：读入输入

> 管线: pop-novel-prose-render v1.0

---

## 目的

读取 Design 的骨架产物 + 风格材料，为渲染建立基线。

---

## 前置条件

- [ ] `03-写作资产/chXXX-设计包.md` 存在
- [ ] styles/ 目录下有文风DNA 档案
- [ ] constitution.yaml 存在

---

## 执行

### 1. 读设计包

从 `chXXX-设计包.md` 提取：
- 本章核心目的 → 理解这场戏的叙事意图
- 场景类型（combat/dialogue/discovery/crisis/transition/mix）→ 决定选 styles/ 的哪些维度
- 登场人物状态 → 每个角色的 before/after 状态 + 台词风格 + 关系动态
- 事件链（数量、内容、情绪目标、冲突层次、信息释放）→ 逐事件渲染的蓝图
- 情绪节拍（起点→发展→高潮→终点）→ 指导句式节奏
- 钩子 → 章末段落的写法

### 2. 读风格材料 + 感知锚定

**文风DNA 档案**（`styles/`）——读全文，在脑中建立风格感知：

不是提取规则列表（规则必然丢信息），是用原文证据感受：
- 这个作者的句子是长的还是短的？
- 叙事者站得近还是远？
- 战斗怎么切节奏的？对话怎么留白的？

**筛选本章相关维度**：根据骨架的场景类型，定位 styles/ 中对应的维度：
| 场景类型 | 重点关注的 styles/ 维度 |
|:--------|:----------------------|
| combat | 战斗场景写法、句式节奏、动作描写 |
| dialogue | 对话策略、叙事者姿态、信息释放方式 |
| discovery | 描写策略、环境功能、氛围建立 |
| crisis | 紧迫感写法、悬念节奏、短句爆发 |
| transition | 过渡写法、段落切换、时间推进 |

**锚定章库**（如有，`01-写作资产/锚定章库/`）：
- 找到与本章场景类型匹配的锚定章
- 读「核心特征提炼」+「可复用规则」——不读原文摘录
- 问自己：**如果这段场景的作者来写我的这个事件，他会怎么写？**

**上一章正文最后 800 字**（`03-正文/ch{上一章}.md`）：
- 语感衔接——不是复读内容，是感受句子的质感：
  - 现在的句子是长的还是短的？
  - 上一章结尾是安静还是紧张？
  - 叙事者的声音是近的还是远的？

### 4. 读宪法

`constitution.yaml`：提取红线，在 Phase 3 验证。
