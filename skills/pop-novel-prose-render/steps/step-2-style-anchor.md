# Step 2：风格锚定

> 管线: pop-novel-prose-render v1.0
> 模板: `templates/style-contract.md`

---

## 目的

从文风DNA + 锚定章中，为「本章这场戏」提取可执行的风格规则。不是翻译成规则列表（那必然丢信息），而是用自己的语感对齐目标质感的句子。

---

## 执行

### Phase 1.1：读入

1. 读文风DNA 档案全文（8 维）
2. 找到与本章场景类型（combat/dialogue/discovery/crisis）对应的维度：
   - combat → 战斗文风维度
   - dialogue → 对话策略 + 叙事者姿态维度
   - discovery → 信息释放策略 + 修辞描写维度
3. 读匹配的锚定章的「核心特征提炼」+「可复用规则」

### Phase 1.2：提取为风格契约

按 `templates/style-contract.md` 模板，输出：

1. **叙事哲学**：叙事者距离 / 叙事者声音 / 情感传递方式
2. **句式与节奏**：句长 / 段落切换 / 情绪-节奏对应
3. **描写策略**：感官优先级 / 环境功能 / 战斗描写方式（如有）/ 禁止的描写
4. **对话策略**：引导词风格 / 对话密度 / 对话功能 / 禁止的对话方式
5. **本章红线**：从文风DNA 的 writing_redlines 提取

### Phase 1.3：锚定语言

**这不是提取规则——规则必然丢信息。** 读锚定章的原文片段，用自己的语感感受句子的重量、节奏、留白。然后问自己：

> "如果这段场景的作者来写我的这个事件，他会怎么写？"

答案不是规则列表。是**写作时的第一感受**——具体到：
- "这个战斗场景的动词都很短，一段只有一个动作"
- "这个对话场景每个人说完话都有一个沉默的间隔"
- "叙事者从来不进角色脑子，所有心理都是外部显示的"

把这些感受写进风格契约，作为 Phase 2 渲染时的「质感锚点」。

### Phase 1.4：风格契约不落盘

风格契约是内存产物，不写入文件。Phase 2 消费完即销毁。

---

## 红线

- ❌ 风格锚定未完成就进 Phase 2 → 退回
- ❌ 风格契约 = 复述文风DNA 的字段名（如"句子以短句为主"不加具体怎么短）→ 退回
