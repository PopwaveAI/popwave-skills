# Step 5：最终输出

> 管线: pop-novel-prose-render v1.0

---

## 目的

组装完整的章节正文文件，附加状态更新块，写入磁盘。

---

## 执行

### 1. 组装正文

```
# 第X章 章标题

[开幕段落]

---
[事件1 段落]

---
[事件2 段落]

---
...

---
[闭幕段落]

# === 状态更新 ===
chapter: X
summary: "..."  ← 一句话总结本章
style_report:
  phase1_contract: "已锚定（{文风DNA来源}·{场景类型}·{锚定章}）"
  phase3_result: "✅ 全部通过" / "⚠️ N处偏差·已修补"
  p0_violations: N
  deviations: [] / ["偏差1", "偏差2"]
entity_updates:
  角色A:
    status: "..."  ← 从人物卡 after 状态取
    location: "..."
    key_items: [...]
    mental: "..."
event_log:
  - "事件1摘要"
  - "事件2摘要"
```

### 2. 写入

写入 `03-正文/chXXX.md`

### 3. 写后自评四问

- [ ] **事件覆盖**：骨架的每个事件都在正文中有对应段落？
- [ ] **情绪对齐**：读完本章的实际情绪体验 ≈ 骨架的情绪节拍？
- [ ] **人物一致**：角色的台词、行为和反应符合人物卡的状态？
- [ ] **风格一致**：风格契约的规则是否贯穿了整章（不是前两段有后两段丢）？

---

## 产出自检

- [ ] 章节号与文件名 chXXX 一致
- [ ] 章节号与事实骨架中的章节号一致
- [ ] 正文末尾有 `# === 状态更新 ===` 块
- [ ] 状态更新块中的角色状态与人物卡的 after 状态一致
- [ ] style_report 中的偏差数与 Phase 3 验证结果一致
- [ ] 正文文件编码为 UTF-8
