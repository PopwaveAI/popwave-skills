# 正文写作引擎

六阶段管线（5步驱动）：Director 设计说明+信息释放策略 → 上下文搜集+世界快照 → 骨架Agent产出事实骨架+QC → 渲染（骨架+DNA）输出正文+状态更新块 → 状态更新维护 global-summary + entity-state。

产出物供给 pop-novel-qa（质检）管线。

---

## ❌ 质量红线（开工前→完工后自检）

开工前：先看红线，知道什么不能碰。完工后：勾 [ ]，确认达标。不过则退回重改。

| # | 标准 | 完工确认 |
|:-:|:-----|:--------:|
| ❌1 | **act-XX.yaml 已存在并通过节奏自检** — 无幕纲不进正文写作 | [ ] |
| ❌2 | **spec.md 已审批就绪** — 规格文档是写前硬闸门，必须通过审批 | [ ] |
| ❌3 | **写前必读已执行** — 读 global-summary + chapter-state + 当前幕纲 + L1 设定 + 数值体系 + 读者画像。无"我记得"只有"我确认过" | [ ] |
| ❌4 | **Director 必须输出决策日志** — 含锚定章引用、字数决策、爽点触发方式。不能只有设计说明没有"为什么这么选" | [ ] |
| ❌5 | **骨架层QC已通过** — 骨架Agent产出的事件链完整、设定包完整、密度标记完整、实体计数≥8、预估字数≥1800 | [ ] |
| ❌6 | **写后自评必须执行** — 自问四问：信息释放按骨架执行、读者停下来的时刻、字数差、文风DNA原则遵守。补充不重写 | [ ] |
| ❌7 | **QC 红线触发 → 退回 Pass 2 重写** — "想跳过"≥2 或 "会弃书" 不通过 | [ ] |
| ❌8 | **黄金三章特有** — 中爽点起步、前300字首屏检查、三章情绪弧线完整（CH1拉→CH2压→CH3释放+新钩子） | [ ] |
| ❌9 | **ESM after 必须更新 global-summary + 状态变更** — 不更新则下游管线无数据可用 | [ ] |
| ❌10 | **引用锚定章时确认来源** — 新书→参考书；续写→自身前文。锚定章来源必须与场景匹配 | [ ] |

## 写前必读清单

写正文前（Phase 1 Director 之前），按此清单顺序读取确认。不可跳过。

```
□ global-summary.md + chapter-state.yaml → 确定本章衔接点
□ 当前幕纲（act-XX.yaml）→ 确认本章情绪目标、爽点类型、章末钩子
□ ★ info_release 清单（自 act-XX.yaml 本章切片）→ 确认本章释放哪些设定信息、释放方式
□ project.yaml#reader_profile → 确认写给谁看
□ L1 元设定层 → 确认力量体系/金手指约束/势力关系
□ 数值体系（combat_capability + act_rank_schedule）→ 确认能力边界
□ 地理+时间轴（如存在）→ 确认场景地理和时间点
□ ★ 叙事策略指令（ESM before 生成）→ 确认本章的信息释放节奏/叙事者姿态/情感表达方式
```

读完后的 thinking 中显式确认：

```
读前状态：[我从对话记忆中"记得"的设定摘要]
读后状态：[与实际文件的差异/需修正的认知]
```

如果无差异 → 继续。如有差异 → 以文件为准。

❌ 常见错误：agent 倾向于"我记得"而非"我确认过"。读前确认不可跳过。

## 六阶段管线（5步驱动 — 3次LLM调用）

先判断是否启用黄金三章模式：

```
用户的章节？
  ├─ CH1–CH3 → 黄金三章模式（见下文）
  └─ CH4+ → 正常管线
```

### 正常管线（CH4+）

#### Step 1：Director（LLM）

**输入**：
- act-XX.yaml 当前章切片（含 emotional_goal + info_release + plotlines_active）
- 读者画像（project.yaml reader_profile）
- L1 元设定层

**输出**：设计说明 + 信息释放策略

**设计说明**包含：
- 本章核心目的（一句话）
- 场景权重分配 + 字数目标
- 爽点触发方式（视觉冲击/信息反转/代价展示/碾压释放/情绪积累释放）
- 描写密度指南

**信息释放策略**（关键产出）：
- 本章释放哪些设定信息（从 info_release 读取）
- 每项信息的释放方式（实战展示/角色对话/叙事者说明/探索发现）
- 信息密度控制（本章新概念 ≤ N 个）

**☆ Director 不读文风DNA** — DNA 由渲染器（Step 4）直接消费。

**前置检查**：
```
□ 读者画像已读入
□ 爽点等级已确认
□ info_release 已确认（本章释放哪些设定信息）
□ 字数目标已定
```

→ 大纲层 QC 通过才能进 Step 2。

---

#### Step 2：上下文搜集（零 LLM）

按 Director 的信息释放策略，从文件系统读取：

1. **L1 设定** — 按 info_release 中的 source_doc 路径读取
2. **上一章结尾** — 读取上一章正文最后 800 字
3. **entity-state.yaml** — 读取当前实体状态（如不存在则跳过）
4. **global-summary.md** — 读取全书进展摘要（≤1500字）

**输出**：世界快照（结构化 YAML，格式参考 `templates/entity-state-schema.md`）

→ 进 Step 3。

---

#### Step 3：骨架 Agent（LLM）

**输入**：Step 1 设计说明 + Step 2 世界快照

**输出**：事实骨架（格式参考 `templates/事实骨架模板.md`）

**执行**：
1. 按信息释放策略中的 release_method 确定事件节点嵌入方式
   - "实战展示" → 创建"该设定被自然验证"的事件节点
   - "角色对话" → 创建"角色通过对话带出"的节点
   - "叙事者说明" → 标记为【插·设定说明】
   - "探索发现" → 创建"主角通过观察发现"的节点
2. 按 density 确定展开程度（集中爆发→独立节点 / 均匀撒放→分散嵌入 / 埋伏笔→只标现象不标解释）
3. 按模板格式组织事件链 + 设定包 + 密度标记
4. 填充统计数据（事件节点总数、唯一实体计数、设定包条目数、预估字数）

**骨架层 QC**：
```
□ 事件链完整覆盖本章全部不可变事件
□ 每个 info_release 在骨架中有对应嵌入位置
□ {实体名} 计数 ≥ 8
□ 预估字数 ≥ 1800
□ 无叙事结构决策（"此处应为高潮"）
□ 无叙事者评价（"主角很愤怒"）
□ 设定包内容全部从 L1 提取（非编造）
```
不通过 → 退回修改。通过 → 进 Step 4。

---

#### Step 4：渲染（LLM）

**输入**（Everything Bundle）：
| # | 项目 | 来源 |
|:-:|:-----|:------|
| 1 | 事实骨架 | 骨架 Agent（Step 3） |
| 2 | 世界快照 | 上下文搜集（Step 2） |
| 3 | 设计说明 + 信息释放策略 | Director（Step 1） |
| 4 | 文风DNA（5条叙事哲学原则） | 文风锚定包（deconstructor 产出） |
| 5 | 宪法红线 | constitution.yaml |
| 6 | 全局摘要 | global-summary.md |
| 7 | 上一章结尾原文 | 正文上一章 |

**三层框架消费**：
```
【Layer 1：事实骨架】
  优先级最高。骨架中的每一个事件节点必须在正文中出现。
  - 有设定包的节点：对应设定必须在正文中释放，方式遵守 release_method
  - 密度标记：极高/高→充分展开，中/低→点到为止

【Layer 3：文风DNA（5条叙事哲学原则）】
  控制句子质感、情感表达方式、对话策略。
  - DNA 原则是信念层——"为什么要这么写"
  - 直接应用，不需要中间翻译层
```

**输出**：
1. **成品正文** — 本章完整正文。禁止否定式描写、"他感到/仿佛"等前缀。
2. **状态更新块** — 正文后追加以下结构：

```yaml
# === 状态更新 ===
chapter: {编号}
summary: "一句话总结本章核心进展"
entity_updates:
  {角色名}:
    status: {新状态}
    location: {新位置}
    key_items: [{关键物品变更}]
  {角色名}:
    ...
world_updates:
  时间: {新时间点}
  地点: {新地点}
  状态: {全局状态变化}
event_log:
  - {核心事件1}
  - {核心事件2}
```

**写后自评**：
- "本章的信息释放按骨架执行了吗？" → 对照骨架中的设定包和密度标记
- "这一章我给了读者几个'停下来'的时刻？" → 标记至少 1 处补一个细节锚点
- "字数差了多少？差的篇幅应该写什么？" → 字数不足 → 补一个被压缩的场景
- "文风DNA的原则我遵守了吗？" → 检查叙事者姿态/情感表达/对话策略是否偏离

→ QC 通过才能进 Step 5。红线标记 "想跳过"≥2 或 "会弃书" → 退回 Step 4 重写。

---

#### Step 5：状态更新（零 LLM）

从渲染器产出的状态更新块中解析，写入文件：

1. **追加到 global-summary.md**
   - 在"追加记录"表中新增一行
   - 更新"全书进展"顶层字段（主角状态、最新事件）

2. **更新 entity-state.yaml**
   - 按 entity_updates 逐条更新角色状态
   - 按 world_updates 更新时间/地点

3. **更新伏笔状态**（如有）

**输出文件**：
- `design/global-summary.md` — 追加后
- `design/entity-state.yaml` — 更新后

### 黄金三章模式（CH1–CH3）

前三章不是"起头"，是整本书的浓缩核。同一套管线开到 120% 强度。

#### 与正常管线的差异

| 维度 | 正常章节（CH4+） | 黄金三章（CH1–CH3） |
|:---|:---|:---|
| 爽点等级 | 微爽点打底 | **中爽点起步，三章内至少1个大爽点** |
| 字数目标 | 1800-2500 | **2200-2500（最低2000）** |
| 场景卡 | 可选 | **强制**——每章核心事件写1段释放段 |
| 写前思考 | Director | Director + **节点C·黄金** 5项检查 |
| 自检 | 5项轻量 | 5项通用 + **3项专项** |

#### 情绪弧线

```
CH1 · 拉进来 → 让读者产生"我要看下去"的冲动
              钩子：情绪驱使型（悬念/信息炸弹），不用弱钩子

CH2 · 压住 → 为第三章蓄力
              钩子：在情绪最高点截断

CH3 · 释放 + 新钩子 → 第一次真正满足 + 抛出更大问题
              完美收尾：阶段性成果 → 但刚开个头 → 更大威胁出现
```

#### 爽点分布

| 章号 | 爽点 |
|:---|:-----|
| CH1 | 中爽点×2 + 大爽点级章末钩子 |
| CH2 | 中爽点×2 + 大爽点×1 + 大爽点级章末钩子 |
| CH3 | 中爽点×1 + 大爽点×1 + 终极爽点级章末钩子 |

#### 节点C·黄金（写前检查，替代 Director）

```
1. 位置对齐：这章在弧线中是拉人/压住/释放？情绪起点衔接上一章？
2. 读者画像：从 project.yaml 读 reader_profile
3. 核心判断：这章只让读者记住一件事，是哪件？
4. 首屏风险：前300字单独读一遍，能留住人吗？
5. 信息密度：≥2个新概念？章末钩子类型和上一章不同？
```

#### 专项自检（写完追加）

- □ 前300字是否真正抓住了人？（感觉"可以划走"→重写）
- □ 本章是否至少有 2 个"读者想划到下一页"的节点？
- □ 三章连起来读，节奏稳定？（CH1拉人→CH2压住→CH3释放）

前三章写完后，CH4 起自动恢复为正常管线。

## ❌ 错误示例

### WRONG 1：不读文件，靠"我记得"写正文

```
Agent：我记得主角的力量体系是XXX…
❌ 错误：没有读取 L1 设定层和数值体系就直接写正文
✅ 正确：先读 global-summary + L1 设定 + 数值体系，
   在 thinking 中确认"读前记得什么 / 读后修正了什么"
```

### WRONG 2：Director 不输出信息释放策略

```
设计说明：本章写主角和 Boss 战斗，目标2300字
❌ 错误：没有信息释放策略——"这章放哪几个设定？什么方式放？"
✅ 正确：附上信息释放策略——
   "本章释放金手指绑定规则（叙事者说明）+ 力量体系升级条件（实战展示）；
   新概念 ≤ 2 个"
```

### WRONG 3：黄金三章当普通章写

```
CH1：主角在家起床→吃饭→出门遇熟人
❌ 错误：微爽点打底，前300字没抓住人
✅ 正确：CH1 第一段就上冲突/悬念/信息炸弹，
   中爽点起步，章末情绪驱使型钩子
```

## 异常与边界条件

| 场景 | 触发条件 | 兜底行为 |
|------|----------|----------|
| `act-XX.yaml` 不存在 | Director 执行前检测到缺失 | 基于 `spec.md` 自动生成最低可行 act 骨架，记录警告日志 |
| `spec.md` 尚未审批 | 读取 `spec.md` 时 `approved: false` | 降级为"草案模式"，仅生成试写段落，不提交正式章节 |
| L1 设定/数值体系文件缺失 | 注入 ESM 前检测 `01-写作资产/` 下无设定文件 | 跳过设定项注入，仅用通用常识兜底，记录缺失清单 |
| QC-renderer.md 模板缺失 | QC 阶段找不到 `QC-renderer.md` | 使用内置最低 QC 标准（字数达标 + 无重复段落），降低质检粒度 |
| 文风锚定包.md 不存在 | ESM before 阶段检查三项优先级均命中失败 | 降级为 styles/default.md，若 default 也不存在则使用最简散文叙事 |
| 骨架实体计数 < 8 | 骨架生成完毕后实体列表 < 8 项 | 检索上章 + spec 高频名词补至 8 项，附标记 `[auto-filled]` |
| 写后自评发现重大缺陷 | 自评 QC 分数 < 60 或含红色缺陷 | 不自动修复，立即中断输出并弹出"QC 拒稿报告"，要求用户确认修正方向 |
| 用户中途要求改平台/风格 | 写作流程中收到平台/风格切换指令 | 冻结当前草稿，回退至 Director 阶段重新做设计决策，保持已写片段为参考 |
| 上章 global-summary 损坏/为空 | 读取 `global-summary.json` 为空或 JSON 解析失败 | 重新扫描最近 3 章正文提取关键事件链，重新生成 summary 后继续 |
| 章节段落字数 < 1800 | 渲染完成后逐段检测字数 | 该段落标记 [short]，自动触发局部扩写迭代（至多 2 轮），仍不足则合并至相邻段落 |

> **核心原则：绝不静默编造内容。** 所有兜底行为必须输出清晰的警告/标记/报告，确保可追溯。任何因异常产生的自动生成内容必须显式标注其来源（如 `[auto-filled]`、`[草案]`、`[short]`），不得以"看起来自然"为由掩盖异常路径。

## 文风锚定包系统（v3.0 — 三层框架）

文风控制通过「文风锚定包」注入。此文件由 deconstructor 在拆书阶段产出，包含 Layer 1（锚定章片段实例）和 Layer 3（5条叙事哲学DNA）。

### 全链路

```
【拆书阶段 · deconstructor Step 3-b】
  拆书完成后 → 基于锚定章片段 + 原文
  按5个叙事哲学维度分析 → 提取信念陈述 + 证据链 + 应用规则
  填入文风锚定包模板 v3.0 → 产出 01-写作资产/文风锚定包.md
    ↓
【ESM 阶段 · 读取优先级】
  ① 01-写作资产/文风锚定包.md（项目级，用户书的专属文风DNA）
  ② 降级 → styles/{writing_style}.md（内部预置参考，v3.0 DNA格式）
  ③ 进一步降级 → styles/default.md（通用兜底）
    ↓
  Step 4 渲染器读取文风DNA（5条叙事哲学原则）作为 Layer 3 消费
  （锚定章片段不注入渲染器，仅 DNA 原则参与渲染）
```

### 为什么改为三层

| 层级 | 位置 | 内容 | 谁产出 | 何时使用 |
|:----|:----|:------|:------|:--------|
| **Layer 1** | 文风锚定包（锚定章节） | 战斗/对话/悬疑原文片段 + 微观质感标注 | deconstructor 从参考书提取 | 提取DNA时的实例参考，不注入渲染器 |
| **Layer 3** | 文风锚定包（DNA节） | 5条叙事哲学原则（信念+证据链+应用规则） | deconstructor 从参考书提取 | 渲染时控制句子质感（唯一注入渲染器的文风输入） |

### 文风DNA读取路径

```
Step 4 渲染器：
  仅读取 01-写作资产/文风锚定包.md 中的"二、叙事哲学 DNA"节（5条原则）
  不读取锚定章片段——DNA 已蒸馏，不需要原文支持

| ID | 定位 | 适用场景 | 不适合 |
|:---|:-----|:--------|:------|
| `default` | **兜底**——bootstrap 未提取时降级使用 | 所有网文题材通用兜底 | — |
| `tomato` | 番茄快节奏：短句+对话驱动+极简描写 | 日更平台、快节奏爽文 | 西幻/悬疑慢热 |
