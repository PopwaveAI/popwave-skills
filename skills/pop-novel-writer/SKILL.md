# 正文写作引擎

六阶段管线：Director 设计说明 + 决策日志 → 事实骨架 → ESM 输入包注入（含锚定章+风格） → 正文渲染 + 自评 → QC → 状态更新。

产出物供给 pop-novel-qa（质检）管线。

---

## ❌ 质量红线（开工前→完工后自检）

开工前：先看红线，知道什么不能碰。完工后：勾 [ ]，确认达标。不过则退回重改。

| # | 标准 | 完工确认 |
|:-:|:-----|:--------:|
| ❌1 | **act-XX.yaml 已存在并通过节奏自检** — 无幕纲不进正文写作 | [ ] |
| ❌2 | **spec.md 已审批就绪** — 规格文档是写前硬闸门，必须通过审批 | [ ] |
| ❌3 | **写前必读已执行** — 读 global-summary + chapter-state + 当前幕纲 + L1 设定 + 数值体系 + 读者画像。无"我记得"只有"我确认过" | [ ] |
| ❌4 | **Director 必须输出决策日志** — 含锚定章引用、字数决策、爽点触发方式。不能只有设计说明没有"为什么这么选" | [ ] |
| ❌5 | **事实骨架实体计数 ≥ 8，预估字数 ≥ 1800** — 低于则退回合并或追加 | [ ] |
| ❌6 | **写后自评必须执行** — 自问三段：与锚定章差距、读者停下来的时刻、字数差。补充不重写 | [ ] |
| ❌7 | **QC 红线触发 → 退回 Pass 2 重写** — "想跳过"≥2 或 "会弃书" 不通过 | [ ] |
| ❌8 | **黄金三章特有** — 中爽点起步、前300字首屏检查、三章情绪弧线完整（CH1拉→CH2压→CH3释放+新钩子） | [ ] |
| ❌9 | **ESM after 必须更新 global-summary + 状态变更** — 不更新则下游管线无数据可用 | [ ] |
| ❌10 | **引用锚定章时确认来源** — 新书→参考书；续写→自身前文。锚定章来源必须与场景匹配 | [ ] |

## 什么时候用

- 用户要求写第 N 章正文
- 用户要求续写当前章节
- 用户要求修改已写章节
- 前三章自动启用黄金三章模式（中爽点起步、场景卡强制）

## 写前必读清单

写正文前（Phase 1 Director 之前），按此清单顺序读取确认。不可跳过。

```
□ global-summary.md + chapter-state.yaml → 确定本章衔接点
□ 当前幕纲（act-XX.yaml）→ 确认本章情绪目标、爽点类型、章末钩子
□ ★ info_release 清单（自 act-XX.yaml 本章切片）→ 确认本章释放哪些设定信息、释放方式
□ project.yaml#reader_profile → 确认写给谁看
□ L1 元设定层 → 确认力量体系/金手指约束/势力关系
□ 数值体系（combat_capability + act_rank_schedule）→ 确认能力边界
□ 地理+时间轴（如存在）→ 确认场景地理和时间点
□ ★ 叙事策略指令（ESM before 生成）→ 确认本章的信息释放节奏/叙事者姿态/情感表达方式
```

读完后的 thinking 中显式确认：

```
读前状态：[我从对话记忆中"记得"的设定摘要]
读后状态：[与实际文件的差异/需修正的认知]
```

如果无差异 → 继续。如有差异 → 以文件为准。

❌ 常见错误：agent 倾向于"我记得"而非"我确认过"。读前确认不可跳过。

## 六阶段管线（三层框架驱动）

先判断是否启用黄金三章模式：

```
用户的章节？
  ├─ CH1–CH3 → 黄金三章模式（见下文）
  └─ CH4+ → 正常管线
```

### 正常管线（CH4+）

#### Step 1：Director（设计说明 + 决策日志 + 信息释放策略）

读 act-XX.yaml + 读者画像 + 经验日志 + 锚定章库。

**★ 三层框架输入（新增）**：
1. 从 act-XX.yaml 读取当前章的 **info_release** 数组 → 确定本章释放哪些设定信息
2. 从文风锚定包读取 **Layer 3 文风DNA**（5条叙事哲学原则）→ 确定本章的叙事基调和策略偏好
3. **info_release 中的 source_doc 路径** → 指引后续骨架Agent从L1提取具体内容

**输出设计说明**：
- 本章核心目的（一句话）
- 场景权重分配 + 字数目标
- 锚定章引用（从库中选同类型片段）
- 爽点触发方式（视觉冲击/信息反转/代价展示/碾压释放/情绪积累释放）
- 描写密度指南
- ★ **信息释放策略**（基于 info_release）：
  - 本章释放哪些设定信息（从 info_release 字段读取）
  - 每项信息的释放方式（实战展示/对话揭示/叙事者说明/探索发现）
  - 信息密度控制（本章新概念 ≤ N 个）

**输出决策日志**（含"为什么"）：
- 锚定章选择 + 原因 + 核心特征提炼
- 字数决策 + 原因
- 爽点触发方式 + 原因
- ★ 信息释放策略决策 + 原因

→ 大纲层 QC 通过才能进 Step 2。

#### Step 2：骨架 + ESM 注入（三层框架核心）

**Layer 1：骨架 Agent 产出纯事实骨架**

遵循 Director 约束 + 消费 info_release：

1. **按 info_release.source_doc 从 L1 设定文件提取具体内容**
   - 读取 Director 设计说明中的"信息释放策略"
   - 对每条 info_release，按 source_doc 路径读取 L1 设定文件
   - 获取该设定的实际文本内容

2. **按 release_method 嵌入骨架**
   - "实战展示" → 在骨架中创建 1 个"该设定被自然验证"的事件节点
   - "角色对话" → 在骨架中创建 1 个"角色通过对话带出该设定"的节点
   - "叙事者说明" → 在骨架中标记为【插·设定说明】
   - "探索发现" → 在骨架中创建 1 个"主角通过观察/实验发现"的节点

3. **按 density 决定展开程度**
   - "集中爆发" → 该设定在骨架中有独立的事件节点
   - "均匀撒放" → 该设定分散到多个事件节点的描述中
   - "埋伏笔" → 在骨架中只标记现象，不标记解释

4. **产出格式**：事件链 + 设定包 + 密度标记
   ```
   事件节点1：[事件]
     设定包：{
       挂载的 info_release: {从L1提取的具体内容}
     }
     密度标记：[极高/高/中/低]
   ```

5. **设定实体追加到 {实体名} 计数** — 确保骨架实体计数 ≥ 8

骨架实体计数 ≥ 8 通过。预估字数 < 1800 退回合并。
→ 骨架层 QC 通过才能进渲染。

**Layer 2：ESM before 生成叙事策略指令**

在组装输入包之前，从文风DNA动态生成5条叙事策略指令：

读取文风锚定包中的 DNA（5条叙事哲学原则） → 按当前章节语境生成指令：

① **信息释放节奏指令**（从DNA原则1推导）
   - 根据 info_release 项的数量和 density 决定
   - 输出：本章信息释放的总节奏 + 每项信息的释放位置 + 遮蔽策略

② **叙事者姿态指令**（从DNA原则2推导）
   - 根据本章场景类型（战斗/对话/悬疑）决定
   - 输出：POV 距离 + 叙事者是否介入 + 本章不做什么

③ **情感表达指令**（从DNA原则3推导）
   - 根据本章 emotional_goal 决定
   - 输出：情感传达手段 + 高强度段落禁止项 + 动情时刻环境策略

④ **对话策略指令**（从DNA原则4推导）
   - 根据本章对话场景预期占比决定
   - 输出：对话功能 + 引导词频率 + 每轮对话推进维度上限

⑤ **张力控制指令**（从DNA原则5推导）
   - 根据本章在幕情绪弧线中的位置决定
   - 输出：铺垫-释放目标比例 + 章内张力峰值位置 + 章末钩子情感锚点

输出格式：YAML 或 Markdown 块，作为第 14 项注入 Pass 2 渲染。

**ESM before（零LLM）**：组装 15 项输入包：

| # | 来源 | 内容 |
|:-:|:----|:------|
| 0 | project.yaml | 读者画像 reader_profile |
| 1-5 | SQLite / 文件 | 主角/技能/物品/状态变更 |
| 6-8 | 骨架+文件 | 骨架推断 + global-summary + 上一章结尾 |
| 9 | Director | 设计说明 + 决策日志（含锚定引用） |
| 10 | ★ **文风锚定包 v3.0** | **Layer 1 锚定章片段 + Layer 3 文风DNA（5条叙事哲学原则+应用规则）** |
| 11 | 经验日志 | 自动匹配的经验教训 |
| 12 | QC 反馈 + spec.md | 上一轮 QC 报告 + 规格约束 |
| **13** | ★ **骨架 Agent** | **info_release 实体内容（从 L1 按 source_doc 提取的具体设定文本）** |
| **14** | ★ **ESM before** | **叙事策略指令（从 DNA 按本章语境生成的 5 条策略指令）** |

第10项读取流程（文风锚定包 v3.0，兼容 v2.0）：
  1. `01-写作资产/文风锚定包.md`（bootstrap 从参考书提取，项目级优先）
  2. 降级 → `styles/{writing_style}.md`（内部预置参考）
  3. 进一步降级 → `styles/default.md`（通用兜底）

> 兼容性：v2.0 参数式锚定包 → ESM before 检测后走旧逻辑生成近似指令。
> v3.0 DNA 式（本框架）→ 按上面的 Layer 2 生成逻辑输出真实指令。

#### Step 3：渲染 → 自评 → QC → 状态更新

**★ 三层框架消费指南**：

写作时按以下优先级消费三层输入：

```
【Layer 1：事实骨架 + info_release 实体内容】
  优先级最高。先确保骨架事件序列全部覆盖，再考虑修饰。
  - 有 info_release 的章节：对应设定必须在正文中出现，方式遵守 release_method
  - 无 info_release 的章节：不引入新的世界观设定

【Layer 2：叙事策略指令】
  优先级次之。在骨架落定后，按指令控制信息释放节奏和情感表达。
  - 指令中的"禁止项"优先于一切
  - 指令中的"推荐项"作为段落编排的参考
  - 如果指令与文风DNA冲突 → 以文风DNA为准

【Layer 3：文风DNA + 锚定章片段】
  最低层但最根本。决定句子的质感。
  - DNA 原则是信念层（"为什么要这么写"）
  - 锚定章片段是实例层（"写成什么样"）

三层冲突时的解决规则：
  1. 骨架（Layer 1）必须完整 → 不因风格原因删事件
  2. 叙事策略（Layer 2）可降级 → 如指令与DNA冲突，以DNA为准
  3. 文风DNA（Layer 3）可弹性适配 → 同一作者在不同场景可以有风格偏移
```

**渲染**：接收 15 项输入（含 Layer 1 info_release 实体内容 + Layer 2 叙事策略指令 + Layer 3 文风DNA）→ 输出成品正文。禁止否定式描写、"他感到/仿佛"等前缀。

**写后自评**：
- "我写的战斗场景和{锚定章}的画面密度差距在哪里？"
- "这一章我给了读者几个'停下来'的时刻？"
- "字数差了多少？差的篇幅应该写什么？"
- ★ "本章的信息释放按计划执行了吗？" → 对照 info_release 确认每条设定信息是否在正文中出现。如有未释放的设定 → 标注"信息延迟"，在 QC 阶段确认是否合理
- 补充最核心的缺失，不重写整章。

**QC**：正文层 QC → 纯感受报告。红线标记 "想跳过"≥2 或 "会弃书" → 退回 Pass 2 重写。

**ESM after**：
- 更新 global-summary.md（追加本章记录）
- 更新 chapter-state.yaml
- 如 QC 有红线 → 追加经验日志条目

### 黄金三章模式（CH1–CH3）

前三章不是"起头"，是整本书的浓缩核。同一套管线开到 120% 强度。

#### 与正常管线的差异

| 维度 | 正常章节（CH4+） | 黄金三章（CH1–CH3） |
|:---|:---|:---|
| 爽点等级 | 微爽点打底 | **中爽点起步，三章内至少1个大爽点** |
| 字数目标 | 1800-2500 | **2200-2500（最低2000）** |
| 场景卡 | 可选 | **强制**——每章核心事件写1段释放段 |
| 写前思考 | Director | Director + **节点C·黄金** 5项检查 |
| 自检 | 5项轻量 | 5项通用 + **3项专项** |

#### 情绪弧线

```
CH1 · 拉进来 → 让读者产生"我要看下去"的冲动
              钩子：情绪驱使型（悬念/信息炸弹），不用弱钩子

CH2 · 压住 → 为第三章蓄力
              钩子：在情绪最高点截断

CH3 · 释放 + 新钩子 → 第一次真正满足 + 抛出更大问题
              完美收尾：阶段性成果 → 但刚开个头 → 更大威胁出现
```

#### 爽点分布

| 章号 | 爽点 |
|:---|:-----|
| CH1 | 中爽点×2 + 大爽点级章末钩子 |
| CH2 | 中爽点×2 + 大爽点×1 + 大爽点级章末钩子 |
| CH3 | 中爽点×1 + 大爽点×1 + 终极爽点级章末钩子 |

#### 节点C·黄金（写前检查，替代 Director）

```
1. 位置对齐：这章在弧线中是拉人/压住/释放？情绪起点衔接上一章？
2. 读者画像：从 project.yaml 读 reader_profile
3. 核心判断：这章只让读者记住一件事，是哪件？
4. 首屏风险：前300字单独读一遍，能留住人吗？
5. 信息密度：≥2个新概念？章末钩子类型和上一章不同？
```

#### 专项自检（写完追加）

- □ 前300字是否真正抓住了人？（感觉"可以划走"→重写）
- □ 本章是否至少有 2 个"读者想划到下一页"的节点？
- □ 三章连起来读，节奏稳定？（CH1拉人→CH2压住→CH3释放）

前三章写完后，CH4 起自动恢复为正常管线。

## ❌ 错误示例

### WRONG 1：不读文件，靠"我记得"写正文

```
Agent：我记得主角的力量体系是XXX…
❌ 错误：没有读取 L1 设定层和数值体系就直接写正文
✅ 正确：先读 global-summary + L1 设定 + 数值体系，
   在 thinking 中确认"读前记得什么 / 读后修正了什么"
```

### WRONG 2：Director 只有设计说明没有决策日志

```
设计说明：本章写主角和 Boss 战斗，目标2300字
❌ 错误：没有决策日志——"为什么选这个锚定章？为什么定2300字？"
✅ 正确：附上决策日志——
   "锚定章选'天蓬咒驱邪'，因为本章的战斗节奏和该片段一致；
   字数定2300因为三场景需要展开；爽点选视觉冲击型"
```

### WRONG 3：黄金三章当普通章写

```
CH1：主角在家起床→吃饭→出门遇熟人
❌ 错误：微爽点打底，前300字没抓住人
✅ 正确：CH1 第一段就上冲突/悬念/信息炸弹，
   中爽点起步，章末情绪驱使型钩子
```

## 异常与边界条件

| 场景 | 触发条件 | 兜底行为 |
|------|----------|----------|
| `act-XX.yaml` 不存在 | Director 执行前检测到缺失 | 基于 `spec.md` 自动生成最低可行 act 骨架，记录警告日志 |
| `spec.md` 尚未审批 | 读取 `spec.md` 时 `approved: false` | 降级为"草案模式"，仅生成试写段落，不提交正式章节 |
| L1 设定/数值体系文件缺失 | 注入 ESM 前检测 `01-写作资产/` 下无设定文件 | 跳过设定项注入，仅用通用常识兜底，记录缺失清单 |
| QC-renderer.md 模板缺失 | QC 阶段找不到 `QC-renderer.md` | 使用内置最低 QC 标准（字数达标 + 无重复段落），降低质检粒度 |
| 文风锚定包.md 不存在 | ESM before 阶段检查三项优先级均命中失败 | 降级为 styles/default.md，若 default 也不存在则使用最简散文叙事 |
| 骨架实体计数 < 8 | 骨架生成完毕后实体列表 < 8 项 | 检索上章 + spec 高频名词补至 8 项，附标记 `[auto-filled]` |
| 写后自评发现重大缺陷 | 自评 QC 分数 < 60 或含红色缺陷 | 不自动修复，立即中断输出并弹出"QC 拒稿报告"，要求用户确认修正方向 |
| 用户中途要求改平台/风格 | 写作流程中收到平台/风格切换指令 | 冻结当前草稿，回退至 Director 阶段重新做设计决策，保持已写片段为参考 |
| 上章 global-summary 损坏/为空 | 读取 `global-summary.json` 为空或 JSON 解析失败 | 重新扫描最近 3 章正文提取关键事件链，重新生成 summary 后继续 |
| 章节段落字数 < 1800 | 渲染完成后逐段检测字数 | 该段落标记 [short]，自动触发局部扩写迭代（至多 2 轮），仍不足则合并至相邻段落 |

> **核心原则：绝不静默编造内容。** 所有兜底行为必须输出清晰的警告/标记/报告，确保可追溯。任何因异常产生的自动生成内容必须显式标注其来源（如 `[auto-filled]`、`[草案]`、`[short]`），不得以"看起来自然"为由掩盖异常路径。

## 文风锚定包系统（v3.0 — 三层框架）

文风控制通过「文风锚定包」（第 10 项）注入。锚定章片段（Layer 1 实例）和叙事哲学DNA（Layer 3 原则）合并为一个文件。

### 全链路

```
【开书阶段 · bootstrap / deconstructor】
  用户选参考书（约3本） → bootstrap 读前 3-5 章
    从原文摘录战斗/对话/悬疑三类片段（各 300-400 字）
    按5个叙事哲学维度分析原文实际表现
    提取信念陈述 + 证据链 + 应用规则
    填入文风锚定包模板 v3.0 → 产出 01-写作资产/文风锚定包.md
    ↓
【ESM before 阶段 · 读取优先级】
  ① 01-写作资产/文风锚定包.md（项目级，用户的专属文风DNA）
  ② 降级到 styles/{writing_style}.md（内部预置参考，v3.0 DNA格式）
  ③ 进一步降级 → styles/default.md（通用兜底）
    ↓
  作为第10项注入
  ↓
【ESM before 阶段 · 生成 Layer 2】
  从文风DNA（5条叙事哲学原则）动态生成5条叙事策略指令
  → 作为第14项注入 Pass 2 渲染
```

### 为什么改为三层

| 层级 | 位置 | 内容 | 谁产出 | 何时使用 |
|:----|:----|:------|:------|:--------|
| **Layer 1** | 文风锚定包（锚定章节） | 战斗/对话/悬疑原文片段 + 微观质感标注 | bootstrap 从参考书提取 | 渲染时作为实例参考 |
| **Layer 2** | ESM before 动态生成 | 5条叙事策略指令（信息释放/叙事者姿态/情感表达/对话策略/张力控制） | ESM before 从DNA生成 | 渲染时控制信息分布和节奏 |
| **Layer 3** | 文风锚定包（DNA节） | 5条叙事哲学原则（信念+证据链+应用规则） | bootstrap 从参考书提取 | 生成Layer 2 + 渲染时控制句子质感 |
