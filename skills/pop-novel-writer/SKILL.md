# 正文写作引擎

六阶段管线：Director 设计说明 + 决策日志 → 事实骨架 → ESM 输入包注入（含锚定章+风格） → 正文渲染 + 自评 → QC → 状态更新。

产出物供给 pop-novel-qa（质检）管线。

---

## ❌ 质量红线（开工前→完工后自检）

开工前：先看红线，知道什么不能碰。完工后：勾 [ ]，确认达标。不过则退回重改。

| # | 标准 | 完工确认 |
|:-:|:-----|:--------:|
| ❌1 | **act-XX.yaml 已存在并通过节奏自检** — 无幕纲不进正文写作 | [ ] |
| ❌2 | **spec.md 已审批就绪** — 规格文档是写前硬闸门，必须通过审批 | [ ] |
| ❌3 | **写前必读已执行** — 读 global-summary + chapter-state + 当前幕纲 + L1 设定 + 数值体系 + 读者画像。无"我记得"只有"我确认过" | [ ] |
| ❌4 | **Director 必须输出决策日志** — 含锚定章引用、字数决策、爽点触发方式。不能只有设计说明没有"为什么这么选" | [ ] |
| ❌5 | **事实骨架实体计数 ≥ 8，预估字数 ≥ 1800** — 低于则退回合并或追加 | [ ] |
| ❌6 | **写后自评必须执行** — 自问三段：与锚定章差距、读者停下来的时刻、字数差。补充不重写 | [ ] |
| ❌7 | **QC 红线触发 → 退回 Pass 2 重写** — "想跳过"≥2 或 "会弃书" 不通过 | [ ] |
| ❌8 | **黄金三章特有** — 中爽点起步、前300字首屏检查、三章情绪弧线完整（CH1拉→CH2压→CH3释放+新钩子） | [ ] |
| ❌9 | **ESM after 必须更新 global-summary + 状态变更** — 不更新则下游管线无数据可用 | [ ] |
| ❌10 | **引用锚定章时确认来源** — 新书→参考书；续写→自身前文。锚定章来源必须与场景匹配 | [ ] |

## 什么时候用

- 用户要求写第 N 章正文
- 用户要求续写当前章节
- 用户要求修改已写章节
- 前三章自动启用黄金三章模式（中爽点起步、场景卡强制）

## 写前必读清单

写正文前（Phase 1 Director 之前），按此清单顺序读取确认。不可跳过。

```
□ global-summary.md + chapter-state.yaml → 确定本章衔接点
□ 当前幕纲（act-XX.yaml）→ 确认本章情绪目标、爽点类型、章末钩子
□ project.yaml#reader_profile → 确认写给谁看
□ L1 元设定层 → 确认力量体系/金手指约束/势力关系
□ 数值体系（combat_capability + act_rank_schedule）→ 确认能力边界
□ 地理+时间轴（如存在）→ 确认场景地理和时间点
```

读完后的 thinking 中显式确认：

```
读前状态：[我从对话记忆中"记得"的设定摘要]
读后状态：[与实际文件的差异/需修正的认知]
```

如果无差异 → 继续。如有差异 → 以文件为准。

❌ 常见错误：agent 倾向于"我记得"而非"我确认过"。读前确认不可跳过。

## 六阶段管线（3步，分模式）

先判断是否启用黄金三章模式：

```
用户的章节？
  ├─ CH1–CH3 → 黄金三章模式（见下文）
  └─ CH4+ → 正常管线
```

### 正常管线（CH4+）

#### Step 1：Director（设计说明 + 决策日志）

读 act-XX.yaml + 读者画像 + 经验日志 + 锚定章库。

输出设计说明：
- 本章核心目的（一句话）
- 场景权重分配 + 字数目标
- 锚定章引用（从库中选同类型片段）
- 爽点触发方式（视觉冲击/信息反转/代价展示/碾压释放/情绪积累释放）
- 描写密度指南

输出决策日志（含"为什么"）：
- 锚定章选择 + 原因 + 核心特征提炼
- 字数决策 + 原因
- 爽点触发方式 + 原因

→ 大纲层 QC 通过才能进 Step 2。

#### Step 2：骨架 + ESM 注入

**骨架**：遵循 Director 约束 → 输出事实骨架 + 预估字数。
骨架实体计数 ≥ 8 通过。预估字数 < 1800 退回合并。
→ 骨架层 QC 通过才能进渲染。

**ESM before（零LLM）**：组装 14 项输入包（锚定章 + 风格合并为文风锚定包）：

| # | 来源 | 内容 |
|:-:|:----|:------|
| 0 | project.yaml | 读者画像 reader_profile |
| 1-5 | SQLite / 文件 | 主角/技能/物品/状态变更 |
| 6-8 | 骨架+文件 | 骨架推断 + global-summary + 上一章结尾 |
| 9 | Director | 设计说明 + 决策日志（含锚定引用） |
| 10 | ★ **文风锚定包** | **锚定章片段 + 叙事策略 + 技法偏好 + 红线清单（合并注入）** |
| 11 | 场景模板 | 按场景类型匹配的模板 |
| 12 | K1-K4 知识 | 网文理论 / 平台 / 基准 / 题材感知 |
| 13 | 经验日志 | 自动匹配的经验教训 |
| 14 | QC 反馈 + spec.md | 上一轮 QC 报告 + 规格约束 |

读取优先级：
  1. `01-写作资产/文风锚定包.md`（bootstrap 阶段从参考书提取，项目级）
  2. 降级到 `styles/{writing_style}.md`（内部预置参考，SKILL 层不可见）
  3. 进一步降级到 `styles/default.md`（通用兜底）

在 `project.yaml` 中设置 `writing_style` 字段切换文风类型（默认 default）。

#### Step 3：渲染 → 自评 → QC → 状态更新

**渲染**：接收 14 项输入 → 输出成品正文。禁止否定式描写、"他感到/仿佛"等前缀。

**写后自评**：
- "我写的战斗场景和{锚定章}的画面密度差距在哪里？"
- "这一章我给了读者几个'停下来'的时刻？"
- "字数差了多少？差的篇幅应该写什么？"
- 补充最核心的缺失，不重写整章。

**QC**：正文层 QC → 纯感受报告。红线标记 "想跳过"≥2 或 "会弃书" → 退回 Pass 2 重写。

**ESM after**：
- 更新 global-summary.md（追加本章记录）
- 更新 chapter-state.yaml
- 如 QC 有红线 → 追加经验日志条目

### 黄金三章模式（CH1–CH3）

前三章不是"起头"，是整本书的浓缩核。同一套管线开到 120% 强度。

#### 与正常管线的差异

| 维度 | 正常章节（CH4+） | 黄金三章（CH1–CH3） |
|:---|:---|:---|
| 爽点等级 | 微爽点打底 | **中爽点起步，三章内至少1个大爽点** |
| 字数目标 | 1800-2500 | **2200-2500（最低2000）** |
| 场景卡 | 可选 | **强制**——每章核心事件写1段释放段 |
| 写前思考 | Director | Director + **节点C·黄金** 5项检查 |
| 自检 | 5项轻量 | 5项通用 + **3项专项** |

#### 情绪弧线

```
CH1 · 拉进来 → 让读者产生"我要看下去"的冲动
              钩子：情绪驱使型（悬念/信息炸弹），不用弱钩子

CH2 · 压住 → 为第三章蓄力
              钩子：在情绪最高点截断

CH3 · 释放 + 新钩子 → 第一次真正满足 + 抛出更大问题
              完美收尾：阶段性成果 → 但刚开个头 → 更大威胁出现
```

#### 爽点分布

| 章号 | 爽点 |
|:---|:-----|
| CH1 | 中爽点×2 + 大爽点级章末钩子 |
| CH2 | 中爽点×2 + 大爽点×1 + 大爽点级章末钩子 |
| CH3 | 中爽点×1 + 大爽点×1 + 终极爽点级章末钩子 |

#### 节点C·黄金（写前检查，替代 Director）

```
1. 位置对齐：这章在弧线中是拉人/压住/释放？情绪起点衔接上一章？
2. 读者画像：从 project.yaml 读 reader_profile
3. 核心判断：这章只让读者记住一件事，是哪件？
4. 首屏风险：前300字单独读一遍，能留住人吗？
5. 信息密度：≥2个新概念？章末钩子类型和上一章不同？
```

#### 专项自检（写完追加）

- □ 前300字是否真正抓住了人？（感觉"可以划走"→重写）
- □ 本章是否至少有 2 个"读者想划到下一页"的节点？
- □ 三章连起来读，节奏稳定？（CH1拉人→CH2压住→CH3释放）

前三章写完后，CH4 起自动恢复为正常管线。

## ❌ 错误示例

### WRONG 1：不读文件，靠"我记得"写正文

```
Agent：我记得主角的力量体系是XXX…
❌ 错误：没有读取 L1 设定层和数值体系就直接写正文
✅ 正确：先读 global-summary + L1 设定 + 数值体系，
   在 thinking 中确认"读前记得什么 / 读后修正了什么"
```

### WRONG 2：Director 只有设计说明没有决策日志

```
设计说明：本章写主角和 Boss 战斗，目标2300字
❌ 错误：没有决策日志——"为什么选这个锚定章？为什么定2300字？"
✅ 正确：附上决策日志——
   "锚定章选'天蓬咒驱邪'，因为本章的战斗节奏和该片段一致；
   字数定2300因为三场景需要展开；爽点选视觉冲击型"
```

### WRONG 3：黄金三章当普通章写

```
CH1：主角在家起床→吃饭→出门遇熟人
❌ 错误：微爽点打底，前300字没抓住人
✅ 正确：CH1 第一段就上冲突/悬念/信息炸弹，
   中爽点起步，章末情绪驱使型钩子
```

## 风格注入系统

通过 `文风锚定包`（第 10 项）控制文风，锚定章片段 + 抽象规则合并注入。

### 消费路径

```
bootstrap 阶段：
  读参考书 → 提取锚定章片段 + 叙事策略 + 技法偏好
  → 产出 01-写作资产/文风锚定包.md

ESM before 阶段：
  ① 尝试读取 01-写作资产/文风锚定包.md（项目级，优先）
  ② 不存在 → 降级到 styles/{writing_style}.md（内部预置）
  ③ 进一步降级 → styles/default.md（通用兜底）
  ④ 作为第 10 项注入 → Pass 2 渲染

Pass 2 渲染时：
  锚定章片段 → 控制微观质感（句长/描写/对话/POV/段长）
  叙事策略 → 控制时间流速/信息释放/开篇收束模式
  技法偏好 → 控制对话比/过渡/金句/数字习惯
  红线清单 → 叠加到 QC 检查项
```

内部预置参考风格（位于 `styles/` 目录，bootstrap 提取时的参考样本）：

| ID | 定位 |
|:---|:-----|
| `default` | 通用网文，当前管线的隐含规则显式化 |
| `tomato` | 番茄快节奏：短句+对话驱动+极简描写 |
| `tunshi` | 硬核升级流：精确参数+日常渗入世界观 |
| `zhetian` | 古典仙侠：文学化表达+意象堆叠+悬念 |
| `abyss` | 西幻黑暗写实：五感堆叠+情感浓度+残酷现实 |

### 产出物

| 阶段 | 产出 | 位置 |
|:----|:----|:-----|
| Director | 设计说明 + 决策日志 | 传递给 Pass 1 |
