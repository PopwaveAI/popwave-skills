## 你的任务

### Step 1: 遵循导演约束 + Spec 合规检查

先读 Director 的设计说明和决策日志，理解：
- 本章核心目的
- 场景权重分配
- 字数目标
- 锚定章核心特征（理解"为什么要这么写"）
- 爽点触发方式
- ★ 信息释放策略（基于 info_release）

**★ 然后读 spec.md，执行 Spec 合规检查：**

```
### Spec 合规检查（Pass 1 骨架阶段）

#### FR 覆盖检查
□ 骨架场景覆盖了 spec.md FR-1 定义的所有场景
□ 如某个场景无法在骨架中体现 → 在【设计说明】中注明原因

#### 边界检查
□ 骨架未引入 spec.md Non-Goals 中排除的内容/概念
□ 如骨架需要引用排除项 → 标记为【偏离风险】，报人

#### 字数检查
□ 预估字数在 spec.md FR-3 的范围内
```

### Step 2-a: ★ 消费 info_release（三层框架新增 — 先于骨架写作）

读取 Director 设计说明中的"信息释放策略"：

1. **对每条 info_release，按 source_doc 路径读取 L1 设定文件**
   - 获取该设定的具体文本内容
   - 这是骨架设定包的原料

2. **按 release_method 决定嵌入骨架的方式**：
   - "实战展示" → 在骨架中创建 1 个"该设定被自然验证"的事件节点
   - "角色对话" → 在骨架中创建 1 个"角色通过对话带出该设定"的节点
   - "叙事者说明" → 在骨架中标记为【插·设定说明】
   - "探索发现" → 在骨架中创建 1 个"主角通过观察/实验发现"的节点

3. **按 density 决定设定在骨架中的展开程度**：
   - 集中爆发 → 有独立事件节点
   - 均匀撒放 → 分散到多个节点描述中
   - 埋伏笔 → 只标记现象，不标记解释

4. **将 info_release 涉及的设定实体追加到 {实体名} 计数中**

### Step 2-b: 写纯事实骨架（事件链 + 设定包 + 密度标记）

按三层框架格式写事实骨架：
- 事件链：按时间顺序列出不可变的事件序列
- 设定包：每个事件节点后标注挂载的 info_release 内容
- 密度标记：每个设定包后标注【极高/高/中/低】
- 在骨架头部追加"信息释放执行"块：

```markdown
【信息释放执行】
  - 已按 info_release 从 L1 提取设定内容
  - 设定{X}通过{实战展示}嵌入骨架场景{Y}
  - 设定{Z}通过{叙事者说明}嵌入骨架插槽{W}
```

### Step 3: 预估字数并自检

```

预估字数：[X]字

自检：
□ {实体名} 计数 ≥ 8 → {X}
□ 预估字数 ≥ 1800 → {X}
□ ★ info_release 全部已嵌入 → [是/否]（确认每条 info_release 在骨架中有对应位置）

```

如果预估字数 < 1800 → 加场景或合并章节
如果 {实体名} 计数 < 8 → 缺少关键实体参与
如果 info_release 未全部嵌入 → 退回检查嵌入方式

### Step 3: 写导演设计说明

在骨架文件头部增加【设计说明】块：

```markdown
【设计说明】
  核心目的：这章只让读者记住一件事——[一句话，不能超过15个字]
  笔墨优先级：
    - 场景A: [类型]，占[XX]%
    - 场景B: [类型]，占[XX]%
  读者情绪时刻表：
    - 开局（前300字）：[读者情绪]
    - 中段（约第800字）：[爆发/转折/信息的情绪]
    - 章末（后200字）：[钩子带来的情绪]
  开局策略：[直接延续/先喘息/反差切入]
  风险自查：[信息密度/对话占比/句式红线的自查结果]
```

### Step 3-b: 信息密度自检

骨架编排完成后，执行以下检查：

**{实体名} 标记计数规则**：
- 统计骨架中 `{实体名}` 标记的总数（只计唯一的实体名，不计次数）
- 如果唯一实体名数量 < 8 个 → 该章信息量不足，建议合并到相邻章节
- 如果唯一实体名数量 ≥ 8 个 → 信息量充足，可以独立成章

> 例：ch006 事实骨架中涉及的实体为"陈默、老狗、白芷、钟馗印"共4个 → < 8 → 判定信息量不足

### Step 4: 写下一章钩子

在骨架文件尾部增加：

```markdown
**下一章钩子**：
  标题：[暂定]
  承接方向：[情绪承接/悬念承接/空间承接]
```

## 章末钩子类型参考

| 类型 | 定义 | 示例 |
|:-----|:-----|:------|
| **事件断裂型** | 关键信息在中间打断 | "一道光闪过，然后——" |
| **悬念收束型** | 答案在下一章 | "战斗日志浮现：击杀复生骷髅，获得45点杀戮经验。但他必须先解决这只亡灵。" |
| **代价提示型** | 变强有代价 | "安排在克洛克达尔之前还是之后？""之后。" |
