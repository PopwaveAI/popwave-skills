# Step 1：设计（LLM）

> 职责：产出"这章要写什么" + "渲染时要注意什么"。
> 设计与渲染的边界：Design 管「结构正确」——事件顺序、信息释放、爽点定位。Render 管「质感正确」——句子节奏、叙述者声音、风格执行。

---

## 输入

- act-XX.yaml 当前章切片（含 emotional_goal + info_release + plotlines_active + payoff）
- 读者画像 reader_profile（来自 project.yaml）
- L1 元设定层（按 info_release 中标注的 source_doc 路径读取）
- global-summary.md（获取上章结束时的状态）
- 上一章正文最后 800 字（获取衔接点）
- **续写/长篇小说：倒数20章全文精读 + 事实提取报告**（闸门由 expert-writer 强制）
- design/里程碑设计.md（如有）→ 确认本章对应哪个 MK 及该 MK 的状态
- design/终点快照.md（如有）→ 确认当前进度与终点距离

---

## 执行步骤

### 1. 读入上下文

按 info_release 中的 source_doc 路径逐一读取对应的 L1 设定文件。
读取 global-summary.md 和上一章结尾 800 字。
读取 design/里程碑设计.md 和 design/终点快照.md（如果存在）。

### 2. 产出设计包（一份文件，含四块内容）

---

#### 块A：设计说明

```
本章核心目的：[一句话，读者读完这章后应该感受到什么]

★ 爽点定位（NEW v13.0）：
  金手指展示：[本章展示了主角的什么优势/能力？]
  期待感构建：[读者在期待什么发生？从什么时候开始期待？]
  爽点释放：[期待在哪个事件节点被兑现？读者感受到什么情绪？]
  → 公式：{金手指} × {期待感} → {爽点释放}

爽点等级：[微/小/中/大/终极 — 从 act-XX.yaml 读取]
字数目标：[目标字数]

★ 情绪弧线（本章内部）：
  起点情绪 → 发展 → 高潮 → 终点情绪
  （四个节点必须不同，形成弧线）

★ 信息释放策略：
  本章释放设定：
    - {item_id} — 释放方式：{实战展示/角色对话/叙事者说明/探索发现}
    - {item_id} — 释放方式：{实战展示/角色对话/叙事者说明/探索发现}
  信息密度控制：本章新概念 ≤ {N} 个

★ 钩子设计：
  类型：{危机预告/行动锁定/威胁升级/悬念/信息反转/选择关口}
  内容：{一句话钩子}
  放置位置：最后 3 行

★ 里程碑对齐：
  本章对应里程碑：{MK-01/MK-02/MK-03/无}
  里程碑推进状态：{start/mid/complete}
  与终点快照的对齐度：
    预计本章后 M1（世界危机）进度：
    预计本章后 M2（战斗/行动）进度：
    预计本章后 M3（成长/蜕变）进度：
```

#### 块B：事件链

按序列出本章所有不可变事件：

```
### 事件 1：[事件标题]

**事件描述**：[发生了什么，一句话]

**设定包**：
- {key}: {从 L1 提取的具体文本内容}

**密度标记**：[极高/高/中/低]

**在爽点链中的角色**：[铺垫/蓄力/释放/收束]

### 事件 2：[事件标题]
...
```

**QC 标准**：
- 事件链完整覆盖本章全部不可变事件
- 每个 info_release 在事件链中有对应的嵌入位置
- 事件链至少有一个"释放"节点（爽点兑现）
- 唯一实体计数 ≥ 8
- 预估字数 ≥ 1800
- 无叙事结构决策（"此处应为高潮"等——这是渲染层的事）
- 设定包内容全部从 L1 提取

#### 块C：渲染指引（NEW v13.0 — 给 Render 步骤的风格约束）

> 块C 不决定"写什么"，只决定"怎么写"。它是设计层和渲染层之间的契约。
> 渲染步骤拿到块C后，用风格锚定（step-2-render Phase 1）验证并增强。

```
★ 本章风格焦点（从文风DNA中选取2-3个本章最关键的维度）：
  1. {维度名，如"句子节奏"} — 本章为什么需要它？{理由}
  2. {维度名，如"叙述者介入"} — 本章为什么需要它？{理由}
  3. {维度名，如"对话策略"} — 本章为什么需要它？{理由}

★ 感官维度优先级（本章以哪种感官为主）：
  主导感官：{视觉/听觉/触觉/嗅觉/味觉}（占比~{N}%）
  辅助感官：{...}（占比~{N}%）
  → 战斗场景禁用纯视觉，必须混入触觉/听觉

★ 情绪传递方式（本章）：
  - 优先：{外部动作展示 / 感官描写 / 对话 / 内心独白}
  - 禁用：直述情绪（"他很愤怒"）；禁用"他感到/他意识到"

★ 对话-叙述比目标：
  本章预期对话占比：~{N}%（战斗章≤20% / 日常章30-40% / 会议章≤50%）

★ 章节呼吸节奏（4段结构）：
  开局{N}字 — 锚点（感官切入/行动切入）
  发展{N}字 — 事件推进+信息释放
  高潮{N}字 — 爽点释放
  收束{N}字 — 钩子锁定
```

#### 块D：上下文快照

```
时间/地点承接: {从上一章结尾提取}
角色当前状态: {从 global-summary 提取关键状态}
已释放伏笔: {哪些伏笔在本章被回收}
新设伏笔: {本章埋下了什么}
```

---

## 输出

`03-写作资产/chXXX-design.md`（Render完成后保留供回溯，正文本不包含）

---

## 不要包含的内容

- ❌ 文风DNA读取（渲染器在 step-2-render Phase 1 自己做）
- ❌ 锚定章引用和选择
- ❌ 长段世界观背景铺陈
- ❌ 正文草稿
- ❌ 状态更新块（Step 3 处理）
- ❌ 渲染层的句子级决策（"这里用短句""这里加个比喻"）
