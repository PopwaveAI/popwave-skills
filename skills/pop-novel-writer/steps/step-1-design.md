# Step 1：设计（LLM）

> 职责：产出"在什么空间、谁在打、打什么、怎么打、渲染要注意什么"。
> 设计与渲染的边界：Design 管「结构正确」——事件顺序、信息释放、爽点定位、空间约束、人设底线。Render 管「质感正确」——句子节奏、叙述者声音、风格执行。

---

#### 块B：本章空间（NEW v14.0）

> 消费 act-XX-地图.md。把 plot 层的一句「locations: ["废弃货栈"]」展开为可用的战斗/探索空间。
> 事件链设计时必须对照此块——每个事件发生在哪个 zone、从哪个 zone 进入哪个 zone。

```yaml
locations:
  - name: "{地点名}"
    source: "{act-XX-地图.md 中的条目路径}"
    primary_use: "{战斗/探索/对话/过渡}"   # 本章这个空间主要用来做什么
    description: "{2-3 句散文式描写，定调}"  # 例："货栈已废弃多年，海风从墙缝灌入，将烛光吹得忽明忽暗。"
    layout:
      入口: "{从哪里进入·门的特征·进入时的感官}"
      内部:
        zone_a: "{名称·大小约N米·特征·适合做什么}"
        zone_b: "{名称·大小约N米·特征·适合做什么}"
        # 战斗章 ≥3 个 zone，非战斗章 ≥1 个
      出口: "{怎么离开·通往哪里}"

    # 仅战斗章展开：
    combat_features:
      - zone: "{zone_a}"
        cover: "{掩体类型}"
        hazard: "{危险物·如有}"
        tactical_note: "{近战/远战/利用环境}"
      - zone: "{zone_b}"
        cover: "{掩体类型}"
```

**空间设计约束**：
- 战斗章 → 空间必须有 ≥3 个 distinct zone（入口→中央→纵深/出口），支持战术移动
- 所有 zone 必须足以容纳每个 beat 的动作需求（蹲/侧身/后退/冲刺）
- 光源/听觉特征必须标注（"单烛·半径3米·其余全黑" / "脚步声在石墙上回弹"）

#### 块C：登场人物（NEW v14.0）

> 消费 act-XX-人物.md + constitution.yaml。每人 §前后·弧线·约束·反应 四维度。
> 事件链设计时必须对照此块——谁在哪个事件做什么、有什么底线不能破。
> **before 状态从 entity-snapshot.yaml 读取（唯一 canon），不靠 memory。**

```yaml
characters:
  - name: "{角色名}"
    source: "{act-XX-人物.md 中的条目路径}"
    
    before:                          # 本章开始前的状态 — 从 entity-snapshot.yaml 对应角色条目读取
      level: "{等级/能力基准}"
      mental: "{情绪·想法·心态}"
      physical: "{健康/受伤/疲劳}"
      relation_to_protagonist: "{和主角的关系·本章开始前的状态}"
      
    after:                           # 本章结束后的状态
      level: "{是否有变化}"
      mental: "{情绪·想法·心态变化}"
      physical: "{伤情变化}"
      
    constraint:                      # ← 从 constitution 或人设读取——渲染层不能越界的底线
      - "{约束1}"                    # 例："从未杀过人·打架的目的是让人倒下，不是让人死"
      - "{约束2}"                    # 例："不恋战·任务优先"
      
    arc_this_chapter:                # 本章内部弧线（≤1句话）
      "{从 X → Y}"
      
    role_this_chapter:               # 本章功能定位
      "{主角/反派/反应者/信息源/背景板/不登场(仅提及)}"
      
    reaction_beat:                   # 如果 role=反应者 → 绑定到事件链的哪个事件后
      "{事件 N 后·具体反应动作}"
```

**人设约束规则**：
- `constraint` 从 constitution.yaml 中的角色相关条款提取，不可在章纲层自编
- 如果 constitution 未定义该角色的 constraint → 查 story-engine.yaml 的 core_premise 中关于该角色的描述
- 如果两处都无定义 → **标记空白，不编造**。渲染层看到空白 = 该角色本章无硬约束
- 主角的 constraint 对应"他在这个世界观里的行为底线"，一旦设定，事件链不能设计违反底线的节拍

#### 块D：事件链

> 消费块B（空间）+ 块C（人物约束）。按空间 zone 和人物 constraint 限制来设计节拍。

按序列出本章所有不可变事件：

```
### 事件 1：[事件标题]

**地点**：{块B 中的 zone 名}
**登场**：{块C 中的角色名}
**事件描述**：[发生了什么，一句话]

**密度标记**：[极高/高/中/低]
**在爽点链中的角色**：[铺垫/蓄力/释放/收束]
**预期字数**：{N}
```

**★ 释放级事件特殊要求**：

当事件在爽点链中的角色为"释放"时，必须展开为子节拍：

```
### 事件 N：[标题] ☆☆☆

**地点**：[zone 链——"入口→中央→木箱区→后门"]
**登场**：[角色列表]
**在爽点链中的角色**：释放

**子节拍设计**（4-7 个节拍，每个 = 动作 + 情绪 + 空间移动）：
  beat 1: {动作} → {情绪}  [地点: {zone}]
  beat 2: {动作} → {情绪}  [地点: {zone}]
  beat 3: {动作} → {情绪}  [地点: {zone}]
  beat 4: {动作} → {情绪}  [地点: {zone}]           ← 触底
  beat 5: {动作} → {情绪}  [地点: {zone}]           ← 转折（金手指/觉醒）
  beat 6: {动作} → {情绪}  [地点: {zone}]           ← 反制
  beat 7: {动作} → {情绪}  [地点: {zone}]           ← 余震

**关键转折点**：[在哪个 beat、主角做了什么选择、结果是什么]
**代价**：[这场战斗/事件结束后主角付出了什么——不能只有"赢了"，必须有对应代价]
```

**节拍设计原则**：
```
战斗：接触→第一回合(被压制)→中盘(困境·不能退)→触底→转折(金手指)→反制(逆转)→余震(代价)
对话：试探→施压→反制→摊牌→结果
发现：寻找→发现→消化→复盘→新选项
危机：威胁逼近→逃不掉→被迫卷入→第一反应
```

**★ 反应事件（每章至少 1 个）**：

事件链末尾必须有反应事件——非主角角色看到主角的变化后做了什么：

```
### 事件 {M}：反应节点

**谁看到**：{块C 中的角色名，role_this_chapter=反应者}
**看到了什么**：{主角身上的具体变化/异常}
**反应动作**：{动作·表情·沉默·退缩·追问}
```

> 如果没有角色能"看到"（例如主角全程独处），必须到块C 自检：如果所有角色的 role_this_chapter 都不包括反应者 → 退回，加一个反应角色。

**QC 标准**：
- 事件链完整覆盖本章全部不可变事件
- 每个 info_release 在事件链中有对应的嵌入位置
- 事件链至少有一个"释放"节点（爽点兑现）
- 释放级事件必须有子节拍设计
- 必须有至少 1 个反应事件，且反应者存在于块C
- 所有事件的"地点"可在块B 中找到对应 zone
- 所有事件的"登场"可在块C 中找到对应角色
- 主角的所有动作不违反块C 中对主角的 constraint
- 唯一实体计数 ≥ 8
- 预估字数 ≥ 1800
- 无叙事结构决策（"此处应为高潮"）
- 设定包内容全部从 L1 提取

#### 块E：设定嵌入表

> 每个 info_release 绑定到块D 的具体事件/节拍。不是"本章释放什么设定"，而是"在哪个节拍、通过什么触发动作、以什么方式释放"。

| 设定项 | 来源 | 嵌入节拍 | 嵌入方式 | 触发动作 |
|--------|------|---------|---------|---------|
| {item_id} | {source_doc} | {事件 N · beat X} | {release_method} | {1-2句：具体什么动作触发} |

**规则**：
- 同一条设定最多在 2 个章节中释放（第 2 次 = 强化，第 3 次 = 冗余）
- P0 设定必须绑定到"释放"或"蓄力"级事件
- P1 设定可绑定到"铺垫"级事件
- 触发动作必须具体——不是"战斗时展示"，是"beat 5 转折·江轩主动触碰血脉·面板隐现力量+2"
