# Step 1：设计（LLM）

> 职责：合并原 Director（设计意图+信息释放策略）+ 骨架（事件链+设定包）为单一设计包。
> 一次性产出："这章要写什么" + "怎么写这些事" + "需要读什么文件"。

---

## 输入

- act-XX.yaml 当前章切片（含 emotional_goal + info_release + plotlines_active）
- 读者画像 reader_profile（来自 project.yaml）
- L1 元设定层（按 info_release 中标注的 source_doc 路径读取）
- global-summary.md（获取上章结束时的状态）
- 上一章正文最后 800 字（获取衔接点）
- **续写/长篇小说：倒数20章全文精读 + 事实提取报告**（闸门由 expert-writer 强制，不通过不进 writer）
- design/里程碑设计.md（如有）→ 确认本章对应哪个 MK 及该 MK 的状态
- design/终点快照.md（如有）→ 确认当前进度与终点距离

## 执行步骤

### 1. 读入上下文

按 info_release 中的 source_doc 路径逐一读取对应的 L1 设定文件。
读取 global-summary.md 和上一章结尾 800 字。
读取 design/里程碑设计.md 和 design/终点快照.md（如果存在）。

### 2. 产出设计包（一份文件，含三块内容）

#### 块A：设计说明

```
本章核心目的：[一句话，读者读完这章后应该感受到什么]

爽点等级：[从 act-XX.yaml 读取当前章的 level]
字数目标：[目标字数]

★ 信息释放策略：
  本章释放设定：
    - {item_id} — 释放方式：{实战展示/角色对话/叙事者说明/探索发现}
    - {item_id} — 释放方式：{实战展示/角色对话/叙事者说明/探索发现}
  信息密度控制：本章新概念 ≤ {N} 个

★ 钩子设计：
  类型：{危机预告/行动锁定/威胁升级/悬念/信息反转/选择关口}
  内容：{一句话钩子}

★ 里程碑对齐（NEW v12.0）：
  本章对应里程碑：{MK-01 / MK-02 / 无（纯过渡）}
  里程碑推进状态：{start / mid / complete}
  与终点快照的对齐度：
    预计本章后 M1（世界危机）进度：
    预计本章后 M2（战斗/行动）进度：
    预计本章后 M3（成长/蜕变）进度：
  距离终点还需：{N} 个里程碑未完成 / 估算剩余 {N} 章
```

#### 块B：事件链

按序列出本章所有不可变事件：

```
### 事件 1：[事件标题]

**事件描述**：[发生了什么，一句话]

**设定包**：
- {key}: {从 L1 提取的具体文本内容}

**密度标记**：[极高/高/中/低]

### 事件 2：[事件标题]
...
```

**QC 标准**：
- 事件链完整覆盖本章全部不可变事件
- 每个 info_release 在事件链中有对应的嵌入位置
- 唯一实体计数 ≥ 8
- 预估字数 ≥ 1800
- 无叙事结构决策（"此处应为高潮"等）
- 设定包内容全部从 L1 提取

#### 块C：上下文快照

```
时间/地点承接: {从上一章结尾提取}
角色当前状态: {从 global-summary 提取关键状态}
已释放伏笔: {哪些伏笔在本章被回收}
```

---

## 输出

`03-写作资产/chXXX-design.md`（Render完成后保留供回溯，正文本不包含）

**注意**：设计包不写入正文。它是 Render 步骤的输入。

---

## 不要包含的内容

- ❌ 文风DNA读取（渲染器消费）
- ❌ 锚定章引用和选择
- ❌ 长段世界观背景铺陈
- ❌ 正文草稿
- ❌ 状态更新块（Step 3 处理）
