# Step 2：渲染（LLM）

> 职责：消费设计包 → 输出正文 + 状态更新块。

---

## 输入（Everything Bundle）

| # | 项目 | 来源 |
|:-:|:-----|:------|
| 1 | 设计包（事件链+设定包+上下文快照） | Step 1 设计 |
| 2 | 文风源（10章语料包或文风DNA） | 文风锚定 |
| 3 | 宪法红线 | constitution.yaml |
| 4 | 全局摘要 | global-summary.md |

---

## 两层框架消费

```
【Layer 1：事件链】
  优先级最高。事件链中的每一个事件节点必须在正文中出现。
  - 有设定包的节点：对应设定必须在正文中释放
  - 释放方式遵守 release_method
  - 密度标记：极高/高→充分展开，中/低→点到为止

【Layer 2：文风控制】
  由文风源控制句子质感、情感表达方式、对话策略。
```

## 写作原则

### 1. 叙事质感由文风源控制

- 信息释放哲学 → 控制每项设定的释放时机和方式
- 叙事者姿态哲学 → 控制叙事者是否介入、POV 距离
- 情感表达哲学 → 控制情绪传达手段
- 对话策略哲学 → 控制对话功能、引导词频率
- 张力-释放哲学 → 控制章内铺垫-释放节奏

**禁止**：
- "不是A，而是B"的否定式描写
- 「不是A不是B是一种C」三段否定结构
- 以"不是"开头引段
- "他感到/他意识到/仿佛"等冗余前缀
- 在对话中塞超过2句的内心分析

### 2. 笔墨分配

设计包中标记为"密度标记：极高/高"的段落充分展开（3-4段），"中/低"点到为止（1-2段）。

### 3. 设定包嵌入方式

| release_method | 操作 |
|:-------------|:------|
| 实战展示 | 通过角色在行动中验证该设定 |
| 角色对话 | 通过角色之间的对话带出该设定 |
| 叙事者说明 | 叙事者直接向读者解释（仅density为极高时用）|
| 探索发现 | 主角通过观察/实验发现该设定 |

### 4. 情绪节奏

- 开局前300字需要有抓住读者的锚点
- 每个核心场景至少有一个情绪释放点
- 章末锁定在钩子上

### 5. 上一章锚定

确保本章开局的语感、视角、情绪与上一章自然衔接。上下文快照中的"时间/地点承接"字段用于此。

---

## 字数参考

| 节奏类型 | 推荐字数 |
|:---------|:---------|
| 渐进紧张/博弈 | 1,800-2,200字 |
| 高频切换/爆发释放 | 1,500-1,800字 |
| 重要转折/首章 | 2,200-2,500字 |

---

## 章末钩子

确保：
- 章末最后3行落在钩子上
- 不提前释放钩子的答案
- 不连续2章使用相同类型的钩子

---

## 输出格式

**正文**（完整章节正文，写入 `02-正文/chXXX.md`）

**状态更新块**（正文后追加）：

```yaml
# === 状态更新 ===
chapter: {编号}
summary: "一句话总结本章核心进展"
entity_updates:
  {角色名}:
    status: {新状态}
    location: {新位置}
    key_items: [{关键物品变更}]
world_updates:
  时间: {新时间点}
  地点: {新地点}
  状态: {全局状态变化}
event_log:
  - {核心事件1}
  - {核心事件2}
```

## 写后自评

```
① "本章的信息释放按设计包执行了吗？"
   → 对照事件链中的设定包和密度标记
② "这一章我给了读者几个'停下来'的时刻？"
   → 标记至少 1 处补一个细节锚点
③ "字数差了多少？差的篇幅本来应该写什么？"
   → 字数不足 → 补一个被压缩的场景
④ "文风原则我遵守了吗？"
```

**自评不通过 ≠ 重写。自评 = 找到最核心的缺失，补一段。**

### ★ 写后自评 — 里程碑感知（NEW v12.0）

读完本章正文，回答：

□ 本章推进的里程碑是否被读者感知到了？
  - 如果 MK 是 M1 型（世界危机推进）→ 读者能否感受到"世界又变了"？
  - 如果 MK 是 M2 型（战斗/行动）→ 读者能否感受到"主角做了件有代价的事"？
  - 如果 MK 是 M3 型（成长/蜕变）→ 读者能否感受到"主角跟之前不一样了"？
□ 如果本章无 MK（纯过渡章），是否有足够的情绪波折和信息量支撑阅读体验？
