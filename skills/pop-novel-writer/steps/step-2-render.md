# Step 2：渲染（LLM）— 三阶段

> 职责：消费设计包 → 锚定风格 → 产出正文 → 验证风格执行。
> 这是决定「读者读到的句子是什么质感」的步骤。

---

## 三阶段总览

```
Phase 1 — 风格锚定（读DNA → 提取可执行规则 → 产生风格契约）
Phase 2 — 正文渲染（设计包 × 风格契约 → 逐事件写入正文）
Phase 3 — 风格验证（对照契约自检 → 标记偏差 → 最小修补）
```

---

## 输入

| # | 项目 | 来源 |
|:-:|:-----|:------|
| 1 | 设计包（块A 设计说明 + 块B 事件链 + 块C 渲染指引 + 块D 上下文快照） | Step 1 |
| 2 | 文风DNA档案 | styles/ 或项目 styles/ |
| 3 | 宪法红线 | constitution.yaml |
| 4 | 全局摘要 | global-summary.md |
| 5 | 上一章正文最后 800 字 | 02-正文/ |

## Phase 1：风格锚定

> 目的：把文风DNA从"参考文档"变成"可执行的写作约束"。
> 不锚定不进渲染。锚定产出是一份 ≤8 条的「风格契约」。

### 1.1 读入

- 读取文风DNA档案（如 `styles/abyss.md`）
- 读取设计包块C（渲染指引）中标注的本章风格焦点
- 读取上一章正文最后 800 字（获取衔接点的实际语感）

### 1.2 提取风格契约

从 DNA 中提取本章最需要执行的规则。**不是全部规则——是本章最关键的 5-8 条**。

按以下模板产出，口头输出即可（不写文件）：

```
【风格契约 — ch{XXX}】

1. 句子节奏规则：
   - 短句(<20字)占比目标：{N}% （从DNA sentence_rhythm 提取）
   - 段落首句：{必须为短句 / 可用长句}
   - 节奏模式：{短起长展 / 递进重复 / 数据穿插}

2. 叙述者声音规则：
   - 介入频率：{每{N}段介入一次 / 仅在关键时刻}
   - 介入方式：{推测副词("恐怕""似乎") / 直接评价 / 补充信息}
   - 禁止：{总结性收束("这就是…") / 过度解释}

3. 对话规则：
   - 引导词：{单字"道"为主 / "说"+"道"混用}
   - 修饰语偏好：{沉声道/缓缓道/轻声道/冷声道}
   - 单次对话长度：{≤{N}字}
   - 对话-叙述比目标：~{N}%（从块C提取）

4. 感官规则：
   - 主导感官：{视觉/听觉/触觉}（占比~{N}%）
   - 每{N}字至少一次非视觉感官
   - 战斗场景必须混入：{触觉(疼痛/冲击)+听觉(碰撞/呼吸)}

5. 信息释放规则：
   - 模式：{三段式(宣告→解释→展开) / 先果后因}
   - 新概念引入时必须有：{1句读者已知的锚点作为前置}

6. 情绪传递规则：
   - 优先：{外部动作展示/感官描写/对话语气}
   - 禁止：{直述情绪词("愤怒""悲伤""恐惧") / "他感到""他意识到"}

7. 禁用句式：
   - "不是A，而是B" / "不是A不是B" / 以"不是"开头引段
   - "他感到…" / "他意识到…" / "他仿佛…"
   - "首先…其次…" / "总结来说…" / "显而易见…"

8. {DNA 特有的 1-2 条高优先级规则}
   - 例-深渊主宰："便"的密度 ≥{N}次/千字
   - 例-遮天：禁止形容词堆叠式描写
   - 例-番茄：每段 ≤3 句话
```

### 1.3 风格契约优先级

按影响程度排序：

| 优先级 | 类型 | 说明 |
|:------:|:-----|:------|
| P0 | 禁用句式 | 违反即不合格，必须清除 |
| P0 | 情绪传递规则 | 违反 = AI 观感暴露 |
| P1 | 句子节奏规则 | 违反 = 读感不对，但不致命 |
| P1 | 叙述者声音规则 | 违反 = 风格不统一 |
| P2 | 感官规则 | 违反 = 描写单薄 |
| P2 | 对话规则 | 违反 = 对话质感不对 |

## Phase 2：正文渲染

> 两层执行：Layer 1（事件链 = 骨架强制覆盖）+ Layer 2（风格契约 = 质感控制）。

### 2.1 写作前准备

1. 风格契约已锚定（Phase 1 完成）
2. 上一章结尾 800 字已读（锚定语感）
3. 宪法红线已确认（constitution.yaml 中与本章相关的条款）

### 2.2 Layer 1：事件链强制覆盖

事件链中的**每一个事件节点必须在正文中出现**。逐一写入，不可跳过。

| 节点类型 | 密度标记 | 展开要求 |
|:---------|:--------:|:---------|
| 铺垫 | 中/低 | 1-2段，点到为止 |
| 蓄力 | 高 | 2-3段，充分渲染张力 |
| 释放 | 极高 | 3-5段，爽点场景完整展开 |
| 收束 | 中 | 1-2段，干净利落地锁住 |

每个事件节点写入时：
- 确认设定包已嵌入（按 release_method 执行）
- 确认在爽点链中的角色被兑现
- 确认密度与展开要求匹配

### 2.3 Layer 2：风格契约执行

**风格契约不是"写完检查"——是"写时执行"。** 写每一段时，在心里过：

```
这句违反 P0 禁用句式吗？
这句的情绪是"展示"还是"直述"？
这句的节奏和 DNA 的 sentence_rhythm 一致吗？
这段有叙述者声音吗？（如果需要介入）
这段有非视觉感官吗？（如果密度标记为高/极高）
```

**风格契约如何影响写作（具体操作）**：

| 契约条目 | 写时怎么办 |
|:---------|:----------|
| 句子节奏：短句起势 | 每段开头第一句控制在 15 字以内 |
| 叙述者介入 | 高密度段落写完主角视角后，追加半句叙述者评价（"这码头上的规矩，是用拳头写的。"） |
| 感官混入 | 每写完 200 字视觉描写，补 1 句听觉/触觉/嗅觉 |
| 对话密度 | 写完对话段后心算字数，超了就砍 |
| "便"的密度 | 书面语处用"便"替代"就"（深渊主宰DNA） |

### 2.4 写作中的情绪节奏控制

对照设计包块A中的「情绪弧线」：

```
起点情绪 → [写第1-N段] → 读者应该在此时感到{X}
  ↓
发展 → [写第N-M段] → 推情绪往上或往下
  ↓
高潮 → [写第M-K段] → 爽点释放，情绪峰值
  ↓
终点情绪 → [写最后3段] → 落在钩子上，不给答案
```

### 2.5 字数分配

| 节奏段 | 占比 | 2000字章 | 2500字章 |
|:------|:----:|:--------:|:--------:|
| 开局锚点 | 10-15% | 200-300 | 250-375 |
| 发展（事件推进） | 40-50% | 800-1000 | 1000-1250 |
| 高潮（爽点释放） | 25-30% | 500-600 | 625-750 |
| 收束（钩子） | 10-15% | 200-300 | 250-375 |

## Phase 3：风格验证

> 写完正文后即时自检。不是 QA（那是 pop-novel-qa 的事），是**风格契约执行度检查**。

### 3.1 快速 P0 扫描

```
□ ctrl-F 搜索"不是A而是B"/"不是A不是B" → 0 处
□ ctrl-F 搜索"他感到"/"他意识到"/"他仿佛" → 0 处  
□ ctrl-F 搜索"首先""其次""总结来说" → 0 处
□ 对话中是否有"他感到/他意识到/他心中" → 0 处（内心独白除外）
```

任何 P0 违规 → 立即删除/改写该句，不进下一步。

### 3.2 风格契约对照

```
□ 句子节奏：抽样 5 段首句 → 是否 ≤15 字？
□ 叙述者声音：本章有 ≥{N} 处叙述者介入？（N=块C 预期）
□ 感官混入：战斗/高密度段落中是否有 ≥2 种感官？
□ 对话密度：全文对话占比 ≈{N}%？（快速估算）
□ DNA 特有规则：{如"便"的密度检查}
```

### 3.3 情绪弧线验证

```
□ 读一遍开局 300 字 → 读者能被抓住吗？如果不能，前 3 句是什么？
□ 爽点释放节点 → 读者能感受到"爽"吗？还是只是"知道了发生了什么"？
  → 如果只是"知道了"：补 1 句感官细节（痛/声音/温度变化）
□ 章末最后 3 行 → 落在钩子上吗？还是提前给了答案？
```

### 3.4 验证结论

```
✅ 全部通过 → 正文合格，进 Step 3
⚠️ P1/P2 偏差 → 标记到状态更新块，不阻塞。下一章 Render 时优先修正。
❌ P0 违规 → 立刻修补违规句，重新检查，直到 P0 = 0
```

## 输出格式

**正文**（`02-正文/chXXX.md`）：

- 标题格式：`# 第{N}章 {标题}`
- 正文：纯叙述，无 YAML 头
- 章末追加 `# === 状态更新 ===` 块（Step 3 消费）

**状态更新块**（正文末尾追加）：

```yaml
# === 状态更新 ===
chapter: {编号}
summary: "一句话总结本章核心进展"
style_report:
  phase1_contract: "已锚定"
  phase3_result: "✅ / ⚠️ / ❌"
  p0_violations: {N}
  deviations: [{规则}：{偏差描述}]
entity_updates:
  {角色名}:
    status: {新状态}
    location: {新位置}
world_updates:
  时间: {新时间点}
  地点: {新地点}
event_log:
  - {核心事件1}
  - {核心事件2}
```

新增 `style_report` 字段供 expert-writer Reflect L1 消费，写入 `workspace-index.yaml#file_registry[].style_executed`。

## 写后自评

```
① "事件链每个节点都覆盖了吗？"
   → 对照设计包块B，逐条确认
② "爽点兑现了吗？读者感受到的是'爽'还是'知道'？"
   → 如果是"知道"：爽点释放节点的描写缺了什么感官细节？
③ "风格契约 P0 项 0 违规了吗？"
   → P0 ≠ 0 → 修补后重检
④ "章末钩子有效吗？"
   → 最后 3 行读者会想知道下一章吗？
```
