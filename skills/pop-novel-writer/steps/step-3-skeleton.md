# Step 3：骨架 Agent（LLM）

> 职责：基于 Director 设计说明 + 世界快照，产出本条的事件链+设定包+密度标记。
> 按 `templates/事实骨架模板.md` 格式化产出。

---

## 输入

- Director 设计说明 + 信息释放策略（Step 1 产出）
- 世界快照（Step 2 产出）
- L1 设定文档（Step 2 已读取的源文件）

## 输出

事实骨架（格式参考 `templates/事实骨架模板.md`）

## 执行步骤

### 1. 按 source_doc 从 L1 提取具体内容

对信息释放策略中列出的每条 info_release：
1. 读取对应 source_doc 路径下的设定文本
2. 将设定文本挂载到骨架的事件节点作为设定包

### 2. 按 release_method 确定嵌入方式

| release_method | 操作 |
|:-------------|:------|
| 实战展示 | 创建 1 个"该设定被自然验证"的事件节点 |
| 角色对话 | 创建 1 个"角色通过对话带出该设定"的节点 |
| 叙事者说明 | 标记为【插·设定说明】 |
| 探索发现 | 创建 1 个"主角通过观察/实验发现"的节点 |

### 3. 按 density 决定展开程度

| density | 展开方式 |
|:--------|:---------|
| 集中爆发 | 该设定有独立的事件节点，充分展开 |
| 均匀撒放 | 该设定分散到多个事件节点的描述中 |
| 埋伏笔 | 只标记现象，不标记解释 |

### 4. 按模板格式组织

```
### 节点 1：[事件简述]

**事件**：[一句话描述发生了什么事]

**设定包**：
- {key}: {从 L1 提取的具体文本内容}

**密度标记**：[极高/高/中/低]

**涉及实体**：{实体名}、{实体名}
```

### 5. 填充统计数据

- 事件节点总数
- 唯一实体计数（要求 ≥ 8）
- 设定包条目数
- 预估字数（要求 ≥ 1800）

---

## 骨架层 QC

```
□ 事件链完整覆盖本章全部不可变事件
□ 每个 info_release 在骨架中有对应的嵌入位置
□ {实体名} 计数 ≥ 8
□ 预估字数 ≥ 1800
□ 无叙事结构决策（"此处应为高潮"）
□ 无叙事者评价（"主角很愤怒"）
□ 设定包内容全部从 L1 提取（非编造）
```

**不通过** → 退回修改。
**通过** → 进 Step 4。
