# 风格注入包 · tomato（v9.5）

> 番茄快节奏文风。适用于日更短章节平台。
> 在 `project.yaml` 中配置 `writing_style: tomato` 启用。
>
> **与 default 的核心差异**：对话驱动（60%+）、极短段、余白写满（不给读者停下来想的时间）、多峰式情绪起伏。

---

## 第1层 · 微观质感

### 1.1 句长分布

```
[x] 短句密集：平均 8-20 字，超过 30 字的句子占比 < 10%
```

**句长变化模式**：

```
[x] 碎片型——大量超短句穿插，制造断裂节奏
```

### 1.2 描写密度与感官偏好

```
[x] 极低密度（白描）——对话+动作驱动，几乎不写环境
```

**感官偏好**：

```
[x] 视觉优先——偏重"看到"什么（颜色/形状/光影）
```

> 只写角色能看到的。不写"感觉"。不写"空气中有一种……的味道"。环境描写只出现在动作间隙中，不独立成段。

### 1.3 对话风格

```
[x] 极简——不用引导词，读者通过内容分辨说话人
```

**独白 vs 对白比例**：

```
[x] 几乎零独白——全是人物之间的对话
```

> 严格规则：每 500 字最多 1 句内心独白。独白不超过 15 字。

### 1.4 POV 距离

```
[x] 零距离——完全贴死在主角大脑里。读者获得的信息=主角获得的信息。
```

> 不写"他不知道的是""与此同时""另一边"。读者知道的=主角知道的。

**POV 切换频率**：

```
[x] 从不切换——全书锁死一个视角
```

### 1.5 段长与分段模式

```
[x] 极短段——每段 1-2 句，关键动作独占一行
```

**分段模式**：

```
[x] 碎段收束——段落前半部分长，后半部分越切越碎
```

### 1.6 修辞偏好

```
[x] 零修辞——完全平铺直叙，不用任何修辞格
```

> 不用比喻。不用拟人。不用排比。信息密度优先。

### 1.7 词汇层级

```
[x] 口语化——日常用词，短缩写法，接近说话
```

**术语密度**：

```
[x] 零术语——所有概念用白话解释
```

> 如果必须出现术语，通过对话自然带出。不在叙述中解释。

---

## 第2层 · 叙事策略

### 2.1 时间流速

```
[x] 快速推进——1章≈2-7天
```

> 不写"几分钟内的战斗"，按番茄标准一章覆盖一个完整的剧情节点。
> 慢镜头战斗只在终极爽点（幕末大高潮）使用。

**时间处理**：

```
[x] 顺序叙事——完全按时间顺序
[x] 跳跃式——章间可跳数天（用"三天后"硬切）
```

### 2.2 信息释放模式

```
[x] 集中爆发——每章集中释放 1-2 个重磅信息
```

**信息密度**：

```
[x] 高密度——每章 ≥6 个
```

### 2.3 叙事声音侵入度

```
[x] 零侵入——纯故事，没有任何叙事者声音
```

### 2.4 开篇模式

```
[x] 钩子先行——第一句话就是剧情推进或悬念
```

> 前 300 字必须有钩子。不铺环境，不建世界观，不设内心独白。第一行即推进。

### 2.5 收束模式

```
[x] 悬念截断——在高潮/对话中间截断
```

> 绝不用情绪总结收尾。不给读者"喘口气"的机会。最后一句话必须让人想点下一章。

### 2.6 情绪节奏模式

```
[x] 多峰型——每 800-1000 字一个小起伏
```

> 番茄读者刷一屏约 5 秒。如果连续 2 屏没有情绪波动，读者就划走了。
> 每 800-1000 字必须出现：一个反转/一个微爽点/一个新信息/一个情绪冲击。

### 2.7 余白密度

```
[x] 写满——心理/环境/解释全部写清楚
```

> 不给读者"自己脑补"的机会。番茄读者不脑补——他们没时间。
> 心理给全、情绪给全、结果给全。唯一留白的是"下一章会发生什么"（章末钩子）。

## 第3层 · 技法偏好

### 3.1 对话/叙述比

```
[x] 对话驱动：对话占全文 60%+
```

### 3.2 心理-动作比

```
[x] 动作驱动——写做不写想，读者通过行为理解角色
```

### 3.3 环境-剧情耦合度

```
[x] 零耦合——不写环境，只写事件
```

### 3.4 过渡处理

```
[x] 硬切——直接进入新场景，不用过渡段
[x] 空行分隔——用"---"或空行分段
```

### 3.5 感叹号/问号密度

```
[x] 密集——每段都有，快节奏章节更密
```

### 3.6 金句倾向

```
[x] 常规——每章 1-2 个金句
```

**金句类型偏好**：

```
[x] 反转型——前一句和后一句形成反差
[x] 感叹型——"这就是XX"
```

### 3.7 数字使用习惯

```
[x] 精确数字——"17分钟后""300米""3.5秒"
```

## 第4层 · 红线清单

### 4.1 句式红线

```
□ 否定句结构（不是A而是B / 不是A不是B是一种C / 以"不是"开段）→ 0容忍
□ "他感到/他意识到/他觉得/他评估"类前缀 → 0容忍
□ 超过 40 字的句子 → 标记退回
□ 连续 2 句超过 30 字 → 标记
□ 首 300 字无钩子 → 退回 Director 阶段
□ 出现"像/如同/仿佛"类比喻词 → 标记
```

### 4.2 段落红线

```
□ 连续 2 段纯环境描写 → 退回
□ 单段超过 5 句 → 标记
□ 单段出现 2 个以上逗号断句 → 标记（碎句不允许逗号连接）
□ 连续 3 段无对话 → 退回
□ 段首出现环境铺陈（"天空是……""空气中……"）→ 标记
```

### 4.3 技法红线

```
□ "过渡段"（不推进剧情也不推进情绪的段落）→ 0容忍
□ 对话引导词 → 0容忍（极简对话，不用"他说/她说"）
□ 连续 2 章同一种开篇模式 → 需修改
□ 连续 2 章同一种收束模式 → 需修改
□ 内心独白连续超过 2 句 → 退回
□ 每章独白总数超过 3 句 → 退回
□ 出现"然后""接着""就在这时"等口语过渡词 → 标记
□ 战后/事件后的情绪回收段落 → 0容忍（直接切到下一段）
```

### 4.4 红线触发后的动作

```
[x] 任何红线 → 退回重写
[x] 标记项 ≥ 2 → 退回
```

> ⚠️ 番茄标准比 default 更严。1 处"想跳过"就算红线，不需要 2 处。

## 场景类型 vs 写法映射

| 场景类型 | 句长 | 描写密度 | 段长 | 节奏 | 特殊要求 |
|:--------|:----|:--------|:----|:----|:--------|
| 战斗对抗 | 极短（5-15） | 极低 | 1句/段 | 极快 | 一句一拳。不写招式名和架势。写"一拳打碎了他的牙" |
| 对话博弈 | 极短（5-20） | 0 | 1-2句/段 | 快 | 零引导词。纯对白推进。省略微动作 |
| 悬疑揭示 | 极短（8-25） | 低 | 1-2句/段 | 快 | 不写感官堆叠。写"当X发生时Y明白了" |
| 压抑蓄力 | 极短（5-20） | 极低 | 1句/段 | 加速 | 压抑感通过事件节奏制造，不是通过描写 |
| 爽点释放 | 极短（3-15） | 0 | 1句/段 | 爆发 | 一句定乾坤。不给读者"喘口气"的机会 |
| 轻松调剂 | 短句（10-25） | 极低 | 2-3句/段 | 中 | 对话推进为主，穿插反转型微爽点 |
| 过渡推进 | 短句（10-20） | 极低 | 1-2句/段 | 快 | 省略过渡。用"三天后"或空行替代 |
