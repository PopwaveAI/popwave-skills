# 风格注入包 · tomato（v9.5）

> 番茄快节奏文风。适用于日更短章节平台。
> 在 `project.yaml` 中配置 `writing_style: tomato` 启用。

---

## 核心维度

| 维度 | 设定 | 说明 |
|:---|:---|:---|
| 句长分布 | 短句为主（8-20 字） | 极限情况下可以连续 3 句不超过 15 字。长句（>40 字）零容忍。 |
| 描写密度 | 低密度 | 不写环境描写。不写"他看见/他听见"。直接写动作和对话。一个场景的描写不超过 3 个感官点。 |
| 对话风格 | 极简 | 几乎不用"他说/她说"。每句最多带一个微动作区分说话人。读者通过内容判断谁在说话。 |
| POV 距离 | 零距离 | 完全贴死在主角大脑里。读者的认知宽度=主角的认知宽度。不写任何主角"正在走神"的内容。 |
| 段长 | 短段（1-3 句） | 关键动作独占一段。频繁换段制造视觉节奏感。每屏（手机 5-8 行）至少换一次段。 |
| 修辞偏好 | 极克制 | 不用比喻，不用排比，不用拟人。句子信息密度优先。 |
| 词汇层级 | 口语化 | 可以用日常用词和短缩写法。但不用网络梗和地域方言。 |

---

## 写法铁则（10 条）

1. **前 300 字必须有钩子**：不铺环境，不设世界观。第一句话就是剧情推进的开始。
2. **每 800-1000 字一个微钩子**：信息推进/角色反常行为/章内悬念——读者每刷一屏都有"再来一行"的冲动。
3. **不要过渡段**：场景切换直接用"---"或空行。不在过渡段里做心理总结和情绪回收。
4. **对话驱动**：正文中对话占比不低于 60%。大段独白/心理分析/世界观解释全部打散嵌入对话。
5. **不写内心 OS**：读者不需要知道主角在想什么——他们看主角做什么。用行动表达心态，不用"他觉得/他认为"。
6. **否定句禁止**：同 default 风格——5 种否定式正文零容忍。
7. **战斗写脸写血**：不写招式名和术语，写"一拳打碎了他的牙"。先写伤害结果，再写怎么打的。
8. **每章至少 2 个反转节点**：读者以为 A 会发生 → 发生的是 B。预期差是番茄最核心的爽点机制。
9. **章末钩子必须是情绪驱使型**：不能用"下一章揭晓"这种弱钩子。要让读者"知道不该追但忍不住点下一章"。
10. **字数压到 1800-2200**：不追求长章。短章=高更新频率=高留存。

## 场景类型 vs 写法映射

| 场景类型 | 句长 | 描写密度 | 节奏 | 特殊要求 |
|:--------|:----|:--------|:----|:--------|
| 战斗对抗 | 极短（5-15 字） | 极低 | 极快 | 一句一拳或一刀。不写"他后退两步，稳住身形"——写"后退。站稳。" |
| 对话博弈 | 短句（8-20 字） | 极低 | 快 | 省略引导词和微动作。纯对白推进。 |
| 悬疑揭示 | 短句（10-25 字） | 中 | 中→快 | 不写感官堆叠，写"当 X 发生时，Y 明白了"。 |
| 压抑蓄力 | 短句（8-20 字） | 低 | 快→加速 | 压抑感通过事件节奏制造，不是通过描写。 |
| 爽点释放 | 极短（5-15 字） | 极低 | 爆发 | 一句定乾坤。不给读者"喘气"的机会。 |
| 轻松调剂 | 中句（15-25 字） | 低 | 中 | 以对话为主，穿插反转型微爽点。 |
| 过渡推进 | 短句（10-20 字） | 极低 | 快 | 省略过渡。用"三天后"或空行替代。 |

|:----|:--------|:----------------|

## 红线清单（QC 检查项）

```
□ 否定句（5 变体）→ 0 容忍
□ 感到类前缀 → 0 容忍
□ 碎句独占行（<8字，非引号内，非核心意象句）→ 0 容忍
□ 连续 2 段以上环境描写 → 0 容忍
□ "过渡段"（心理总结/情绪回收/场景转换文字）→ 0 容忍
□ 出现长句（>40 字）→ 标记
□ 首 300 字内无钩子 → 退回
□ 对话占比 < 60% → 砍描写加对话
□ 连续同种临场类型超过 2 章 → 需要替换
□ 字数低于 1500 或高于 2500 → 退回
□ "想跳过"≥1 或 "会弃书" → 退回重写  （注：番茄标准更严，1 处想跳过就算红线）
```
