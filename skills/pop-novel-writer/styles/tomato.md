# 文风参考 · tomato（v3.0）

> 番茄快节奏文风参考。适用于日更短章节平台。
> 在 `project.yaml` 中配置 `writing_style: tomato` 启用。
>
> **适用场景**：日更平台（番茄/七猫）、快节奏爽文、升级流、对话驱动型
> **不适合**：需要氛围沉浸的西幻/悬疑、慢热型叙事、出版级文学

---

## 一、锚定章原文参考

> 以下原文来自 tomato 文风的真实产出（深渊主宰测试）。非独立参考书，是番茄平台优化规则的实例化。

### [战斗·镇口对峙]

**来源章节**：ch001 测试产出

```
艾伦背着猎弓走进琥珀镇。

"站住。"

守门的老头抬起眼皮。看清是他，又缩回棚子底下。

"今天收成怎么样？"

"两只兔子。"

"行。"

老头点了点头。

"走了。"

艾伦穿过镇口。手里提着两只野兔。兔子还温着，脖子上的箭口凝了血痂。

他经过老水井。井绳又松了一圈。

"得紧了。"
```

**微观质感标注**：
- 句长：**极短**（3-12字），碎句推进
- 描写密度：**极低**——不写环境、不写五感，只写动作和对话
- 对话：**零引导词**——通过内容分辨谁在说话
- POV：**零距离**——艾伦的所见所做
- 段长：**1句/段**，关键动作独占一行
- 修辞：**零修辞**——无比喻、无拟人

**叙事哲学印证**：原则1（信息释放——每句释放最小信息量，不铺垫直接给结果）、原则2（叙事者隐身——纯动作对白，零叙事声音侵入）、原则4（对话策略——零引导词纯对白推进）、原则5（张力释放——快速推进，不给读者间歇）

### [对话·铁匠铺前]

**来源章节**：ch001 测试产出

```
铁匠铺的炉火还亮着。老汤姆蹲在铺子门口，擎着烟斗。

"回来了？"

"嗯。"

老汤姆的目光在野兔上停了半秒。

"明天还去？"

"看天气。"

"行。"
```

**微观质感标注**：
- 引导词密度：**零**——纯对白
- 独白比例：零——不写任何内心活动
- 推进维度：**信息推进**——每句推进最小信息量
- 字数：每个话轮不超过 5 个字

**叙事哲学印证**：原则2（叙事者姿态——纯对白，叙事者完全隐退）、原则4（对话策略——话轮极短，每句推进最小信息量，对话即叙事引擎）

### [悬疑·空屋]

**来源章节**：ch001 测试产出

```
他推开门。屋子里是空的。

灶台是凉的。炉膛里只剩死灰。

窗户开着的。风从窗口灌进来，吹得干薄荷叶子满地打转。

他站了一秒。

然后转身出了门。
```

**微观质感标注**：
- 感官密度：低——只写视觉
- 信息释放：**集中爆发**——一推开就给结果（空的/凉的），不给过渡
- 留白度：**写满**——不留给读者猜测时间
- 情绪节奏：**多峰型**——推门（紧张）→ 空（冲击）→ 转身出门（下一个开始）

**叙事哲学印证**：原则1（信息释放——集中爆发，一推开就给结果）、原则3（情感表达——情绪通过动作外显："站了一秒"传达失落/警惕，不写内心OS）、原则5（张力释放——多峰节奏，每句一个微冲击）

## 二、叙事哲学 DNA

> 番茄DNA的核心哲学 = 快节奏爽文：极短段、对话驱动、零修辞、不铺垫直接上结果、情绪外显不内挖。

### 原则 1：信息释放哲学

**信念陈述**：作者认为信息应该集中爆发式释放，因为番茄读者每刷一屏约5秒，连续2屏无信息推进读者就会划走；不铺垫环境、不建世界观，每章集中释放1-2个重磅信息。

**应用规则**：
1. 每章集中释放 1-2 个重磅信息，不稀释在琐碎描写中
2. 前 300 字必须有钩子（剧情推进或悬念），第一句话就是信息推进
3. 信息密度保持每章 ≥6 个信息点，连续2屏必须有新信息
4. 推门就给结果——不给过渡、不给氛围铺垫
5. 留白度"写满"——唯一留白是"下一章会发生什么"，不给读者自己脑补的机会

### 原则 2：叙事者姿态哲学

**信念陈述**：作者认为叙事者应该完全隐身，因为零侵入叙事让读者沉浸在故事本身；纯故事，没有任何叙事者声音，让动作和对白自己说话。

**应用规则**：
1. 零引导词——通过内容分辨谁在说话，不使用"他说""她问"类标签
2. 零侵入——不使用"读者可能不知道……""值得一提的是……"等叙事者干预
3. 心理活动零描写——不写"他心想""他在想什么"
4. POV零距离——紧贴主角的所见所做，不跳跃、不评论
5. 硬切过渡——直接进入新场景，不用过渡段，空行分隔即可

### 原则 3：情感表达哲学

**信念陈述**：作者认为情绪应该通过动作和行为外显，因为写做不写想，读者通过行为理解角色；不写内心独白、不写情绪分析，只用动作和反应传递情感。

**应用规则**：
1. 情感必须通过可观察的行为传递——"他站了一秒"而非"他感到不安"
2. 内心独白连续不超过 2 句，整章零内心OS为佳
3. 0容忍"他感到/他意识到/他觉得/他评估"类前缀
4. 战后/事件后不写情绪回收段落——情绪通过下一个动作延续
5. 否定句结构 0容忍——写"他走了"而非"他没有停留"

### 原则 4：对话策略哲学

**信念陈述**：作者认为对话是推动叙事的主要引擎，因为番茄爽文是对话驱动的，对话占全文60%以上；每个话轮极短（不超过5字），只推进最小信息量，通过对话本身的内容区分说话者。

**应用规则**：
1. 对话占全文 60%+，每3段之内必须有对话
2. 每个话轮不超过 5 个字，一句话一个信息
3. 零引导词——纯对白，不用"他说""她问""XX说"
4. 连续 2 段纯环境描写退稿——必须用对话或动作打断
5. 段首不出现环境铺陈（"天空是……""空气中……"）

### 原则 5：张力-释放哲学

**信念陈述**：作者认为张力应该高频小周期释放，因为番茄读者需要每800-1000字获得一个微爽点/反转/新信息/情绪冲击；不在高潮处情绪总结收尾，而是在高潮/对话中间截断制造悬念。

**应用规则**：
1. 每 800-1000 字必须有一次反转/微爽点/新信息/情绪冲击
2. 情绪节奏为多峰型——不断制造小起伏，不让读者松气
3. 悬念截断收尾——在高潮/对话中间截断，最后一句话必须让读者想点下一章
4. 快速推进——1章≈2-7天，不写几分钟内的慢镜头战斗（慢镜头只在幕末终极爽点使用）
5. 顺序叙事 + 跳跃式——章间可跳数天（用"三天后"硬切），不做时间线性填充

## 三、红线清单

### 3.1 通用红线

```
□ 否定句结构 → 0容忍
□ "他感到/他意识到/他觉得/他评估"类前缀 → 0容忍
```

### 3.2 本风格特有红线

```
□ 超过 40 字的句子 → 标记退回
□ 连续 2 段纯环境描写 → 退回
□ 对话引导词 → 0容忍
□ 首 300 字无钩子 → 退回 Director 阶段
□ 内心独白连续超过 2 句 → 退回
□ 出现"像/如同/仿佛"类比喻词 → 标记
□ 出现"然后""接着""就在这时"等口语过渡词 → 标记
□ 战后/事件后的情绪回收段落 → 0容忍
□ 连续 3 段无对话 → 退回
□ 段首出现环境铺陈（"天空是……""空气中……"）→ 标记
```

### 3.3 红线触发后的动作

```
[x] 任何红线 → 退回重写
[x] 标记项 ≥ 2 → 退回
```
