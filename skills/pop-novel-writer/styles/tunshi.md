# 文风参考 · tunshi（v1.0）

> 以《吞噬星空》（我吃西红柿）为样本提取的文风参考。科幻升级流。
> 在 `project.yaml` 中配置 `writing_style: tunshi` 启用。
>
> **特征**：平实流畅、技术参数精确、对话带微动作和神态、慢热。

---

## 一、锚定章参考模式

> 实际产出时替换为参考书对应场景的原文片段。

### [战斗] — 参数辅助型

**微观质感**：
- 句长：中句（15-30字），均匀分布
- 描写：中密度，附带精确速度和力量数据
- 对话：中等引导，对话带动作神态
- POV：近距离，偶尔切换外部视角
- 段长：中段（3-6句），均匀切分

### [对话] — 引导丰富型

**微观质感**：
- 引导词密度：每句对话带说话人动作或表情
- 独白比例：各半，角色通过独白做决策推演
- 推进维度：每句对话推进信息+情绪

### [悬疑/发现] — 数据揭示型

**微观质感**：
- 感官密度：中，视觉+听觉
- 信息释放：通过发现/分析/数据逐步揭示
- 留白度：中度留白

## 二、叙事策略

### 2.1 时间流速

```
[x] 正常流速——1章≈半天到一天
```

时间处理：顺序叙事

### 2.2 信息释放模式

```
[x] 均匀撒放——信息持续不断地穿插在每段中
[x] 对话揭示型——通过角色对话和独白双重释出
```

信息密度：中（每章 3-5 个新概念，逐步展开世界观）

参考锚定章：对话场景 → 通过对话释放信息的方式

### 2.3 叙事声音侵入度

```
[x] 低侵入——偶尔有"现如今""此刻"类引导
```

### 2.4 开篇模式

```
[x] 场景铺陈——先给环境/时间/地点再推进剧情
```

> 典型开篇：先描写时间+场景+氛围，再引入人物和对话。

### 2.5 收束模式

```
[x] 情绪总结——给读者"喘口气"的余地再结束
[x] 对话收尾——以一句有分量的话或日常对话结束
```

### 2.6 情绪节奏模式

```
[x] 多峰型——每 1200-1500 字一个小起伏
```

### 2.7 余白密度

```
[x] 中度留白——战斗和参数写满，日常和情感留有余白
```

参考锚定章：悬疑场景 → 留白处理方式

## 四、红线清单

### 4.1 通用红线

```
□ 否定句结构 → 0容忍
□ "他感到/他意识到/他觉得/他评估"类前缀 → 0容忍
```

### 4.2 本风格特有红线

```
□ 超过 60 字的句子 → 标记退回
□ 首次出现术语不带解释 → 标记
□ 连续 3 段纯环境描写 → 标记
□ 战斗场景无精确数字 → 标记（风格特色缺失）
□ 连续 3 章无技术参数/数据 → 标记
□ 内心独白无实际行动衔接 → 标记
```

### 4.3 红线触发后的动作

```
[x] 标记项 ≥ 3 → 退回
```
