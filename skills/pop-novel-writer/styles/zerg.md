# 文风参考 · zerg（v3.0 — 叙事哲学DNA）

> 以《虫族帝国》为样本提取的叙事哲学DNA。异兽流·游戏系统·种田发展。
> 在 `project.yaml` 中配置 `writing_style: zerg` 启用。
>
> **适用场景**：异兽流/虫族/星际/种田发展、游戏系统向、基地建设
> **不适合**：纯对话驱动型、古典仙侠、慢热情感向
>
> **取样范围**：CH1（覆盖系统绑定/创造/探索）
> **取样章节数**：1章
> **叙事者侵入频率**：约 0.5次/1000字
> **对话占比范围**：25%-35%（含系统语音+主角吐槽）

---

## 一、锚定章片段

> 以下原文摘自《虫族帝国》CH1，作为本DNA的具体实例。

### [系统绑定·KEY降临]

**来源章节**：第 1 章

```
key是宇宙超级文明月神帝国花费数百万年创造出的科技结晶，他的研究方向
就是被称为"神之领域"的生命科技。所谓生命科技并不是低级的生物科技，
而是只有神才能拥有的力量——创造生命。

"警告！警告！能量不足，key将进入休眠状态，请及时充能。"
说完还不等楚恒问怎么充能，便已化为一道流光没入楚恒眉心。

"次奥！太尼玛不智能了吧，能量用完会进入休眠状态也不提前告诉我一声，
还好意思说自己是高科技！"楚恒无语的对着key消失的地方伸出了中指。
```

**微观质感标注**：
- 句长：中句（15-30字），系统语音与内心吐槽交替
- 描写密度：低——不写环境，纯设定+对话
- 对话：**系统语音+主角吐槽**双线推进
- POV：**零距离**——紧贴主角接收信息的认知过程
- 段长：中段（3-5句）
- 修辞：克制——无比喻，平实叙述游戏化设定
- **叙事哲学印证**：原则1（系统揭示型信息释放）+ 原则3（吐槽外化情感）

### [创造·第一只小狗]

**来源章节**：第 1 章

```
"给我创造一个zergling。"

key闪动了两下道："搜索到类似生物，需要消耗低级能量10点，因为缺乏
基因密码，系统将自行组成基因，预计力量比理想中消弱百分之五十，
请先为生物命名后创造，是或否？"

"次奥，一个小狗就把仅有的能量用完了，命名为小狗，创造！"

只见这时自己面前的空间突然一阵扭曲，一个丑陋而充满侵略性的生物出现在
自己面前，而后如同一只猎犬般爬到了楚恒脚下，静静地等待着楚恒的命令。
```

**微观质感标注**：
- 引导词密度：系统语音自带引导，主角吐槽无引导词
- 独白比例：**高**——所有判断/吐槽都是内心OS
- 推进维度：**信息+决策**——每个系统提示后接主角决策
- 叙事节奏：设定介绍→系统响应→主角反应→创造完成
- **叙事哲学印证**：原则4（系统语音+吐槽双线）+ 原则5（多峰型节奏）

### [探索·陌生星球]

**来源章节**：第 1 章

```
楚恒跌跌撞撞的跟着小狗，终于在前面看到了一丝亮光。于是兴奋的加快了速度，
终于冲出了这个漆黑的无底深渊。

耀眼的光芒将楚恒眼睛刺的生疼，好半天才恢复视力的楚恒才震惊的发现这
是一个怎样的个世界。

自己站在一座高山的山腰处，背后是自己刚刚走出的山洞，而面前入眼之处
尽是如同海洋般无边无际的密林，密密麻麻的树木多的让人窒息。
```

**微观质感标注**：
- 感官密度：中——视觉（光刺眼/密林）+ 体感（窒息感）
- 信息释放：**探索发现型**——跟着主角一起"开地图"
- 留白度：中度留白——写环境不写系统提示，让读者先感受
- POV：零距离——只写主角看到的
- **叙事哲学印证**：原则1（探索发现型信息释放）+ 原则5（探索即是张力）

## 二、叙事哲学 DNA（5条原则）

### 原则 1：信息释放哲学

**信念陈述**：

> "作者认为信息应该通过系统提示/面板释放和主角探索发现双线推进。游戏化数据是核心叙事驱动力——系统面板的每一次弹出都是一次爽点脉冲。信息在本书中是资源——玩家用能量/基因等资源换取信息，信息又解锁新的资源。"

**证据链**（来自锚定章）：
- 系统绑定·KEY降临 → KEY的背景说明一次性释放（宇宙超级文明/生命科技）
- 创造·第一只小狗 → 系统提示精确给出消耗（10能量/50%削弱）+ 创造规则
- 探索·陌生星球 → 信息通过主角的感官体验逐步发现（先黑暗→光刺眼→密林）

**跨章验证**：
- CH1：系统揭示+探索发现双线，每段至少有1个系统信息或环境发现

**应用规则**（可执行指引）：
- 系统信息 → 通过系统提示/面板释放，必须附带精确数据
- 世界观信息 → 通过主角探索新地图发现，不给教材式说明
- 每章至少 5 个系统提示+新信息
- 系统面板数据必须精确（不精确 = 标记，风格特征缺失）
- 连续 3 段无系统交互/创造/升级 → 标记

---

### 原则 2：叙事者姿态哲学

**信念陈述**：

> "叙事者完全隐身。纯故事呈现，没有任何叙事者声音。信息通过系统面板和主角的认知过程传递——系统说'消耗能量10点'，角色说'次奥'，叙事者不在中间做任何解释或评价。"

**证据链**（来自锚定章）：
- 系统绑定·KEY降临 → KEY的背景（"key是宇宙超级文明……"）以准系统说明口吻带出，非叙事者评价
- 创造·第一只小狗 → "只见这时自己面前的空间突然一阵扭曲"——纯场景呈现，无叙事者介入
- 探索·陌生星球 → "楚恒跌跌撞撞的跟着小狗"——紧贴主角视角，无跳出解释

**跨章验证**：
- CH1：叙事者侵入约 0.5次/1000字，基本不介入

**应用规则**（可执行指引）：
- POV 距离默认模式 → **零距离**（紧贴主角接收信息的认知过程）
- 叙事者侵入场景 → 禁止介入（全风格零容忍叙事者主观评价）
- 因果关系解释 → 由读者通过场景自行推导
- 系统说明 → 以系统语音/面板形式出现，不以叙事者口吻

---

### 原则 3：情感表达哲学

**信念陈述**：

> "情绪主要通过吐槽直接外化。主角对系统提示的反应、对环境的震撼、对自身处境的焦虑全部通过吐槽表达。与此同时，成功创造/升级/发现时通过感叹句释放成就感——'次奥''这就是XX！'"

**证据链**（来自锚定章）：
- 系统绑定·KEY降临 → "次奥！太尼玛不智能了吧"——吐槽表达挫败感
- 创造·第一只小狗 → "次奥，一个小狗就把仅有的能量用完了"——吐槽表达无奈
- 探索·陌生星球 → "耀眼光芒刺的生疼""震惊""窒息"——身体反应代替情绪标签

**跨章验证**：
- CH1：情感表达的主要手段=吐槽+身体反应

**应用规则**（可执行指引）：
- 高强度情绪（挫败/无奈/震惊）→ 通过吐槽表达，不写直白情绪名词
- 成就感/兴奋 → 通过感叹句表达（"这就是XX！""终于！"）
- 焦虑/紧迫感 → 通过资源不足/能量危机来建立（ "仅有的能量用完了"）
- 主角连续 3 段无吐槽/反应 → 风格特征缺失，标记

---

### 原则 4：对话策略哲学

**信念陈述**：

> "对话由系统语音和主角吐槽构成双线驱动。系统语音承担信息释放功能（面板数据/创造提示），主角吐槽承担情绪释放功能（对系统的吐槽/对处境的评论）。两者交替是本书最具辨识度的风格特征。"

**证据链**（来自锚定章）：
- 系统绑定·KEY降临 → 系统警告（信息）→ 主角吐槽（情绪）交替
- 创造·第一只小狗 → 系统提示消耗（信息）→ 主角决定（决策）→ 创造完成（结果）
- 全章估算：系统语音+主角吐槽合计占 25%-35%

**跨章验证**：
- CH1：对话结构=系统提示↔主角吐槽↔系统提示↔主角决策

**应用规则**（可执行指引）：
- 每轮对话推进的维度 → 1-2 个（信息+决策并进）
- 引导词使用频率 → 系统语音自带引导，主角吐槽无引导词
- 独白比例 → 高（主角的判断+吐槽全部通过内心OS）
- 系统语音风格 → 扁平、精确、机械化
- 主角吐槽风格 → 口语化、夸张、网络化

---

### 原则 5：张力-释放哲学

**信念陈述**：

> "张力主要通过资源稀缺（能量不足/基因缺失）和未知环境（陌生星球/危险生物）建立，释放通过创造成功/发现新资源/升级解锁完成。节奏呈多峰型——每 800-1000 字一个系统升级/创造/发现小高潮，不给读者冷却期。"

**证据链**（来自锚定章）：
- 系统绑定·KEY降临 → KEY降临（铺垫）→ 能量不足的警告（限制建立）→ 自动绑定（小释放）
- 创造·第一只小狗 → 创造命令（铺垫）→ 确认条件（限制）→ 创造成功（释放）
- 探索·陌生星球 → 黑暗深渊（压抑）→ 光刺眼（转折）→ 无尽密林（释放+新悬念）

**跨章验证**：
- 整体旋律：**多峰型**——每 800-1000 字一个小高潮
- 章内铺垫-释放比：铺垫不超过整章的 50%
- 连续张力上限：可持续 2-3 章（系统升级+资源收集可以维持多峰节奏）

**应用规则**（可执行指引）：
- 章末钩子类型 → **系统提示收尾**（最后一句是新的系统信息/升级提示）或 **悬念截断**（在创造完成/变强边缘截断）
- 大爽点前的蓄力 → 不超过 1 章
- 连续无系统交互/创造/升级的章节上限 → 0 章（每章必须包含系统交互）
- 开篇策略 → **金手指先行**（先给系统/金手指再进入剧情）或 **场景切入**（先写困境再揭示金手指）
- 收束策略 → 系统提示收尾（新的系统信息/升级提示）

## 三、红线清单

### 3.1 通用红线

```
□ 否定句结构 → 0容忍
□ "他感到/他意识到/他觉得/他评估"类前缀 → 0容忍
```

### 3.2 本风格特有红线

```
□ 超过 60 字的句子 → 标记退回
□ 系统面板数据不精确 → 标记
□ 连续 3 段无系统交互/创造/升级 → 标记（风格特征缺失）
□ 连续 2 章无新生物/新能力出现 → 标记
□ 主角连续 3 段无吐槽/反应 → 标记（虫族帝国风格需要主角反应）
```

### 3.3 红线触发后的动作

```
[x] 标记项 ≥ 3 → 退回
```
