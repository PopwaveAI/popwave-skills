# 文风参考 · zerg（v1.0）

> 以《虫族帝国》为样本提取的文风参考。异兽流·游戏系统·种田发展。
> 在 `project.yaml` 中配置 `writing_style: zerg` 启用。
>
> **适用场景**：异兽流/虫族/星际/种田发展、游戏系统向、基地建设
> **不适合**：纯对话驱动型、古典仙侠、慢热情感向

## 一、锚定章原文参考

### [系统绑定·KEY降临]

**来源章节**：第 1 章

```
key是宇宙超级文明月神帝国花费数百万年创造出的科技结晶，他的研究方向
就是被称为"神之领域"的生命科技。所谓生命科技并不是低级的生物科技，
而是只有神才能拥有的力量——创造生命。

"警告！警告！能量不足，key将进入休眠状态，请及时充能。"
说完还不等楚恒问怎么充能，便已化为一道流光没入楚恒眉心。

"次奥！太尼玛不智能了吧，能量用完会进入休眠状态也不提前告诉我一声，
还好意思说自己是高科技！"楚恒无语的对着key消失的地方伸出了中指。
```

**微观质感标注**：
- 句长：中句（15-30字），系统语音与内心吐槽交替
- 描写密度：低——不写环境，纯设定+对话
- 对话：**系统语音+主角吐槽**双线推进
- POV：**零距离**——紧贴主角接收信息的认知过程
- 段长：中段（3-5句）
- 修辞：克制——无比喻，平实叙述游戏化设定

### [创造·第一只小狗]

**来源章节**：第 1 章

```
"给我创造一个zergling。"

key闪动了两下道："搜索到类似生物，需要消耗低级能量10点，因为缺乏
基因密码，系统将自行组成基因，预计力量比理想中消弱百分之五十，
请先为生物命名后创造，是或否？"

"次奥，一个小狗就把仅有的能量用完了，命名为小狗，创造！"

只见这时自己面前的空间突然一阵扭曲，一个丑陋而充满侵略性的生物出现在
自己面前，而后如同一只猎犬般爬到了楚恒脚下，静静地等待着楚恒的命令。
```

**微观质感标注**：
- 引导词密度：系统语音自带引导，主角吐槽无引导词
- 独白比例：**高**——所有判断/吐槽都是内心OS
- 推进维度：**信息+决策**——每个系统提示后接主角决策
- 叙事节奏：设定介绍→系统响应→主角反应→创造完成

### [探索·陌生星球]

**来源章节**：第 1 章

```
楚恒跌跌撞撞的跟着小狗，终于在前面看到了一丝亮光。于是兴奋的加快了速度，
终于冲出了这个漆黑的无底深渊。

耀眼的光芒将楚恒眼睛刺的生疼，好半天才恢复视力的楚恒才震惊的发现这
是一个怎样的个世界。

自己站在一座高山的山腰处，背后是自己刚刚走出的山洞，而面前入眼之处
尽是如同海洋般无边无际的密林，密密麻麻的树木多的让人窒息。
```

**微观质感标注**：
- 感官密度：中——视觉（光刺眼/密林）+ 体感（窒息感）
- 信息释放：**探索发现型**——跟着主角一起"开地图"
- 留白度：中度留白——写环境不写系统提示，让读者先感受
- POV：零距离——只写主角看到的

## 二、叙事策略

### 2.1 时间流速

```
[x] 弹性——前期探索详细写，后期种田快速跳
```

时间处理：顺序叙事

### 2.2 信息释放模式

```
[x] 系统揭示型——信息通过系统提示/面板释放
[x] 探索发现型——信息通过主角探索新地图发现
```

信息密度：高（每章 ≥5 个系统提示+新信息）

参考锚定章：系统绑定场景 → 信息释放节奏

### 2.3 叙事声音侵入度

```
[x] 零侵入——纯故事，叙事者不出现
```

### 2.4 开篇模式

```
[x] 金手指先行——先给系统/金手指再进入剧情
[x] 场景切入——先写困境再揭示金手指
```

### 2.5 收束模式

```
[x] 系统提示收尾——最后一句是新的系统信息/升级提示
[x] 悬念截断——在创造完成/变强边缘截断
```

### 2.6 情绪节奏模式

```
[x] 多峰型——每 800-1000 字一个系统升级/创造/发现小高潮
```

### 2.7 余白密度

```
[x] 写满——系统面板/数据/基因信息全部给读者
```

参考锚定章：系统绑定场景 → 数据密度

## 三、技法偏好

### 3.1 对话/叙述比

```
[x] 均衡型——系统语音算对话，主角吐槽算独白
```

### 3.2 心理-动作比

```
[x] 心理辅助——动作在前，系统判断在后
```

### 3.3 过渡处理

```
[x] 空行分隔——用"---"分隔场景
[x] 时间标记——"X天后"
```

### 3.4 金句倾向

```
[x] 偶尔——每 2-3 章 1 个
```

金句类型偏好：**感叹型**（"这就是XX！"）、反转型

### 3.5 数字使用习惯

```
[x] 精确数字——"10能量""50%消弱""百分之九十九点九九"
```

> 系统数据是核心阅读驱动力，每个数字必须精确。

## 四、红线清单

### 4.1 通用红线

```
□ 否定句结构 → 0容忍
□ "他感到/他意识到/他觉得/他评估"类前缀 → 0容忍
```

### 4.2 本风格特有红线

```
□ 超过 60 字的句子 → 标记退回
□ 系统面板数据不精确 → 标记
□ 连续 3 段无系统交互/创造/升级 → 标记（风格特征缺失）
□ 连续 2 章无新生物/新能力出现 → 标记
□ 主角连续 3 段无吐槽/反应 → 标记（虫族帝国风格需要主角反应）
```

### 4.3 红线触发后的动作

```
[x] 标记项 ≥ 3 → 退回
```
