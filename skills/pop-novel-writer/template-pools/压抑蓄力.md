# 压抑蓄力场景模板

> 适用于：情绪低谷、资源不足、主角受压——为后续释放蓄力
> 核心原则：**压抑不是目的，压抑是为释放购买的期票**

---

## 段落结构

```
┌─ 段1 · 压迫施加（1段，2-4句） ────────────────────┐
│  功能：告诉读者情况在变差                             │
│  内容：一个具体的损失/威胁/限制，而不是"感觉很糟"     │
│  写法：用事件/信息施加压力，不用情绪描述施加压力       │
│  示例：不是"他感到绝望"——                             │
│        是"手机屏幕碎了。通讯录备份？一周前的。"         │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段2 · 应对尝试（1-2段） ──────────────────────────┐
│  功能：主角在做些什么，但不够                        │
│  内容：主角的行动尝试 + 结果（不够/不行/差一点）     │
│  节奏：一次不行，再来一次，第二次也不行——但第三次有转机 │
│  注意：不能让读者觉得"主角在犯蠢"——                   │
│        主角做的选择在当时是合理的，只是不够            │
└────────────────────────────────────────────────────┘
                      ↓
┌─ 段3 · 释放前兆（1段） ───────────────────────────┐
│  功能：给读者一个"还有希望"的信号                    │
│  内容：一个微小的转机/新信息/角色的一句话             │
│  写法：不要刻意——不要让读者觉得"有救了"              │
│        让读者觉得"等等，也许还有办法"                 │
│  字数：1-3句                                          │
└────────────────────────────────────────────────────┘
```

---

## 关键约束

- **不小于2次挫折**——1次不是蓄力，是开局不利。至少2次才叫"压抑"
- **不放空枪**——压抑章节必须有对应的释放点（当前章末/下一章/幕末），压抑本身不独立存在
- **不写心理活动累积**——不做"他越来越焦虑"这种线性递减，用事件推
- **不重复同一个压迫源**——两次挫折必须是不同维度（资源/关系/信息/安全）

---

## 节奏标记

```
压迫：中速（具体） → 应对1：中速 → 应对2：快（但不够） → 前兆：慢（停顿感）
总字数：400-700字
```
