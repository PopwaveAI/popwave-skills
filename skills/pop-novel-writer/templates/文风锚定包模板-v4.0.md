# 文风锚定包设计蓝图 v4.0

## ——面向一本书的全量 DNA 结构

---

## 设计原则

1. **场景优先** — 同一本书战斗/情感/对话写法不同，DNA必须按场景类型切分
2. **可执行** — 每条规则必须是渲染器能直接照着写的指令，不是文学评论
3. **有实例** — 每个场景类型配锚定章片段，规则从实例推导出来
4. **有跨章验证** — 不靠单章节断言，明确说"这个特征在CH1/CH5/CH15一致/不一致"
5. **有密度标记** — 每类场景能回答"这章有多少信息要放、放多密"

### Part 1：核心叙事哲学DNA（全书不变层）

> 全书所有场景共享的底层信念。不随场景类型变化。

每条原则结构：

```
#### 原则 N：[名称]

**信念陈述**：[一句话回答作者为什么这么做]

> "作者认为..."

**证据链**（来自锚定章）：
- {锚定章名} → {该片段的对应表现}
- {锚定章名} → {该片段的对应表现}

**跨章验证**：
- CH1：{表现}
- CH5：{表现}
- CH15：{表现}
- → 结论：[一致 / 有偏移]

**默认应用规则**（全书通用）：
- [规则1]
- [规则2]
- [规则3]
```

5条原则与当前一致：
| 原则 | 核心问题 |
|:----|:---------|
| 1 | 信息释放哲学——作者对信息释放的根本态度 |
| 2 | 叙事者姿态哲学——叙事者与读者之间的距离 |
| 3 | 情感表达哲学——情绪靠什么传达 |
| 4 | 对话策略哲学——对话扮演什么角色 |
| 5 | 张力-释放哲学——节奏控制的底层逻辑 |

### Part 2：场景型DNA（按场景类型细化）

> 核心DNA中每条原则在不同场景类型下的具体应用规则。
> 渲染器先确定"这一章主要是哪个场景类型"，然后加载对应的场景型DNA。

#### 场景类型目录

| # | 场景类型 | 定义 | 适用情境 |
|:-:|:--------|:-----|:---------|
| 1 | **战斗/对抗** | 一对一或多对多的战斗对抗 | 野外遭遇、竞技场、偷袭、复仇战 |
| 2 | **情感/悲伤** | 以情绪释放为核心驱动力 | 离别、死亡、回忆、重逢、告白失败 |
| 3 | **对话/信息** | 以信息交换为主要功能 | 谈判、咨询、日常交谈、情报交换 |
| 4 | **悬疑/压迫** | 以未知恐惧为驱动力 | 潜伏、追逐、密室、神秘现象、怪物未出现时 |
| 5 | **碾压/爽点** | 以力量不对等为快感来源 | 升级突破、秒杀、装逼打脸、碾压战斗 |
| 6 | **设定揭示** | 以系统/世界观信息释放为主 | 系统激活、穿越记忆、世界观揭秘、面板出现 |
| 7 | **决战/大场面** | 宏大场景下的多角色对抗 | Boss战、团战、攻城战、神战 |
| 8 | **过渡/缓冲** | 低张力的场景衔接 | 赶路、休息、日常、场景切换、战利品盘点 |

## 渲染器消费方式

```
Step 1 Director：
  确定"本章以{战斗+情感}为主"
  → 传递给 Step 4 渲染器

Step 4 渲染器：
  → 读核心DNA（全书信念）→ 确定底层哲学
  → 读场景型DNA 2.1（战斗）+ 2.2（情感）→ 确定本场景的具体写法
  → 读节奏地图 → 确定阶段密度基准
  → 共加载 1（核心）+ 2（场景）= 3 组规则
  → 正文产出
  → 写后自查对照红线清单
```
