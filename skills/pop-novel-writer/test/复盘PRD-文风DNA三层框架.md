# PRD · 文风DNA分离实验复盘与三层框架定义

> 状态：实验验证通过，待纳入pop-novel-writer管线设计
> 日期：2026-06-05
> 实验负责人：pop（老板江轩决策，pop执行）

---

## 一、背景：我们遇到了什么瓶颈

pop-novel-writer从v8.0迭代到v9.7.1，六阶段管线经过多轮打磨，但遇到了一个核心瓶颈：

**剧情设计质量和正文质量的提升空间越来越小。**

过去几个版本的改进尝试：
- 增加骨架实体计数要求（≥8）
- 细化文风锚定包（锚定章+风格规则合并注入）
- 升级QC红线
- 强化Director设计说明

效果都不显著。产出的正文质量稳定在"不错"但无法突破到"惊艳"。

## 二、根因诊断

老板在2026-06-05提出了一个核心洞察：

> "把一本书或一章内容，事实部分完全剥离出来当事实骨架，文风DNA相关的当血肉填充。然后尝试切换不同的文风DNA来验证是否可行。"

围绕这个洞察，我们展开了三轮实验。

## 三、三轮实验过程

### 实验材料
- **参考书**：《深渊主宰》（诸生浮屠）第1-3章
- **测试案例**：现有深渊主宰文风对比测试
- **数据**：原文完整前三章 + 五种文风对比产出

### 第1轮：粗事件链 + 叙事哲学DNA（v1）

**设计**：
- 骨架：仅含事件链（8节点/5节点/7节点），去叙事标签
- DNA：叙事哲学（5条原则），无参数规则
- 控制：并行子agent，同一时间窗口

**结果**：✅ 验证通过，差异存在
- 三版在新增内容、信息释放路径、结尾策略上有结构性差异
- 但在事件覆盖和顺序上高度一致
- **发现**：骨架粒度决定了DNA发挥空间——5节点 > 7节点 > 8节点+标签

**瓶颈发现**：5-8节点骨架丢失了大量原文信息量（游戏系统、世界观背景、角色历史），导致DNA没有"信息素材"可以差异化处理。

### 第1.5轮：原文DNA逆向揭示

**设计**：
- 将原文与三版DNA产出逐段对比

**关键发现**：
- 原文的叙事者声音**非常主动**——大量"叙事者跳出来解释"（"她今年才八岁，生火对她来说很困难"）
- 原文对待世界观的方式是**教材式先解释再叙事**——CH2塞了700字游戏系统说明，CH3塞了500字全书世界观预告
- 原文的情感表达**直白外露**——"恐惧。""她坚强地擦去泪水。""它有这样的资格。"
- 我们提炼的"深渊DNA"实际上比原文更加克制——说明我们在提取时做了"理想化"过滤

**结论**：原文的叙事者声音既不属于纯事件，也不属于纯风格——它是一个中间层。

### 第2轮：三层骨架 + 叙事策略DNA（v2）

**设计**：
- 骨架：事件链 + 设定包 + 信息释放决策点
- DNA：叙事哲学 + 叙事策略维度（明确指导面对高密度信息时的处理方式）
- 控制：并行子agent

**结果**：✅✅ 差异显著拉大
- 三版在"穿越说明"上做出了本质不同的处理（深渊：线索化；番茄：功能化；遮天：意象化）
- 三版在"属性面板展示"上做出了不同的取舍（深渊：省略明细；番茄：全面板；遮天：回忆包裹）
- 三版在"世界观预告"上做出了不同的保留决策（深渊：砍成空气暗示；番茄：砍成一行系统提示；遮天：意象化保留）
- 三版的结尾类型完全不同（深渊：动作闭环；番茄：钩子；遮天：意象定格）

**核心结论**：**差异从"同一故事不同语气"升级为"同一事实不同叙事者结构决策"**。

## 四、三层框架定义

### 总览

```
┌─────────────────────────────────────────────┐
│              ① 纯事实骨架                      │
│     ├─ 事件链（不可变的因果序列）               │
│     └─ 设定包（每个事件上挂载的世界/角色信息）   │
├─────────────────────────────────────────────┤
│              ② 叙事策略指令                     │
│     ├─ 信息释放节奏（一次性/分散/埋伏笔）       │
│     ├─ 叙事者姿态（侵入/零侵入/场景化）         │
│     ├─ 情感表达方式（直白标签/动作暗示/意象）   │
│     └─ 设定密度控制（信息释放决策点标记）       │
├─────────────────────────────────────────────┤
│              ③ 文风DNA                         │
│     ├─ 叙事哲学（5-7条原则，如"如实写不评价"）  │
│     └─ 技法偏好（对话密度/修辞倾向/段落节奏）   │
└─────────────────────────────────────────────┘
```

### 4.1 第一层：纯事实骨架

**职责**：记录"发生了什么"和"存在什么信息需要传递"。

**包含**：
- **事件链**：不可变的因果序列，按时间顺序排列
- **设定包**：每个事件节点上原文挂载的世界信息、角色背景、系统数据——不含"怎么讲"的决策
- **信息释放决策点标记**：密度评级（极高/高/中/低），作为策略调用的触发标记

**设计原则**：

| 过去的做法（错误） | 现在的做法（正确） |
|:-----------------|:-----------------|
| 骨架只包含事件，不记录信息量 | 骨架同时记录事件链+设定包 |
| 骨架内嵌叙事结构标签（权重/情绪/钩子） | 骨架不嵌叙事决策 |
| 骨架隐含"所有的信息都要在这次释放" | 骨架标记密度，让策略决定释放策略 |

**示例**（CH2节点2-穿越）：
```
节点2：索伦发现属性面板，认出游戏系统
  设定包：
    - 属性全套数据（12属性+5技能+传奇技能+专长）
    - 穿越者身份说明
    - 原主消散事实
    - 前世最后一刻记忆（养生仓→闪电）
  密度标记：极高
```

### 4.2 第二层：叙事策略指令

**职责**：决定"信息怎么被讲出来"。

**四个维度**：

| 维度 | 选项 | 决策依据 |
|:----|:----|:--------|
| 信息释放节奏 | 一次性释放 / 分散到后续 / 埋伏笔 | DNA偏好 + 当前密度标记 |
| 叙事者姿态 | 主动解释 / 零侵入 / 场景化 | DNA偏好 + 目标平台偏好 |
| 情感表达方式 | 直白标签 / 动作暗示 / 意象投射 | DNA偏好 |
| 设定密度控制 | 全放 / 部分放 / 转化为行动 | DNA偏好 + 节点在故事中的位置 |

**设计原则**：这个层的指令由DNA驱动生成，但它是一个**独立的中间产物**——因为它可以被复用、被覆盖。

### 4.3 第三层：文风DNA

**职责**：决定"作者怎么看世界、怎么对待读者"。

**构成**：
- **叙事哲学**（5-7条原则）：抽象的方向性指导，如"如实写、不评价"
- **技法偏好**（可选）：对话密度倾向、修辞倾向、段落节奏倾向

**设计原则**：
- ❌ 不包含参数规则（"句长10-25字""段长2-4句"）
- ❌ 不包含否定性约束（"禁止XX""不要XX"——除非是明确红线）
- ✅ 每一条原则都在回答一个问题：**「这个作者在面对一个叙事决策时，脑子里想的是什么？」**

### 4.4 三层的交互流程

```
Director阶段：
  读project.yaml + 幕纲 + 设定
  → 产出：① 纯事实骨架（事件链+设定包+密度标记）

ESM before阶段：
  读DNA → 由DNA生成 ② 叙事策略指令
  → 将①+②+③打包注入

渲染阶段：
  接收①+②+③
  → 针对每个高密度标记节点，按策略指令做出结构决策
  → 按DNA哲学完成微观质感写作
```

## 五、实验数据证据

### 5.1 控制变量有效性

| 要求 | 实现方式 |
|:----|:--------|
| 骨架相同 | v1: 同一份事件链；v2: 同一份三层骨架 |
| DNA不同 | 三套独立DNA（深渊/番茄/遮天） |
| 无交叉污染 | 并行子agent，同一时间窗口，彼此不知道对方存在 |
| 无上下文干扰 | 每个agent只收到骨架+DNA，不知有其他版本 |

### 5.2 差异量化

| 维度 | v1差异程度 | v2差异程度 | 变化 |
|:----|:----------|:----------|:----|
| 开篇策略 | 中（都从雨声/环境开篇） | **大**（深渊：线索；番茄：数据；遮天：意象） | ↑ |
| 穿越处理 | 小 | **大** | ↑↑ |
| 属性展示方式 | 极小 | **中等** | ↑ |
| 世界观释放 | 极小 | **大** | ↑↑ |
| 希斯之死写法 | 小 | **中等** | ↑ |
| 结尾类型 | 中 | **大**（动作闭环/钩子/意象定格） | ↑ |
| 整体读感差异 | "同故事不同语气" | "同事实不同叙事者" | **质变** |

### 5.3 原文DNA逆向提取

通过对比原文与三版DNA产出，提取出原文真实DNA：

```
原文叙事者DNA（以《深渊主宰》CH1-3为样本）：
  1. 叙事者声音主动侵入——经常跳出来解释因果关系和角色动机
  2. 设定教材式释放——先解释世界观，再进入叙事
  3. 情感直白外露——用情绪标签告诉读者"这里应该感动/恐惧"
  4. 信息密度极高——前三章就释放全书级世界观预告
  5. POV可在角色内心和叙事者之间灵活切换
```

这与我们之前从原文"理想化提取"的深渊DNA（如实写/留白/不评价）存在本质差异。说明：**DNA提取必须以原文实际文本为基准，而不是以"我对这个风格的理解"为基准。**

## 六、设计落地建议

### 6.1 pop-novel-writer 管线改动

**当前管线**：
```
Director（设计+决策）→ 骨架（含结构决策）→ ESM注入（含文风锚定包）→ 渲染 → QC
```

**新管线**：
```
Director（设计+决策）→ 三层骨架（事件链+设定包+密度标记）
    ↓
ESM before：
  Step A：读取DNA → 生成叙事策略指令
  Step B：骨架 + 策略 + DNA + 其他10项 → 打包注入
    ↓
渲染：按策略指令处理信息释放决策点 → 按DNA哲学完成正文
    ↓
QC → 状态更新
```

### 6.2 骨架格式升级

骨架Agent的产出格式需要从：
```markdown
# chXXX 事实骨架
## 场景序列
## 核心事件
## 实体清单
## 字数统计
```

升级为：
```markdown
# chXXX 三层事实骨架

## 事件节点

### 节点1：[事件简述]
**设定包**：
- {key}: {value}
- {key}: {value}
**信息密度标记**：[极高/高/中/低]

### 节点2：[事件简述]
...
```

### 6.3 DNA格式升级

DNA需要新增一个**叙事策略维度的模板**，包含：
- 信息释放节奏偏好
- 叙事者姿态偏好
- 情感表达方式偏好
- 高密度节点的处理指引

### 6.4 实验验证阶段

建议管线改完之后，用**同一套测试用例**（深渊主宰CH1-3）重新产出三版，进行回归验证：
- 是否还有"信息泄漏"——骨架中没有设定包导致信息缺失
- 叙事策略是否被正确执行——高密度节点是否按DNA偏好的方式处理
- 跨DNA差异是否维持——三版之间是否仍能保持结构差异

## 七、关键认知记录

1. **骨架的粒度决定了DNA的发挥空间**：骨架越粗（事件点越少），DNA在"怎么讲"上发挥空间越大；骨架越细（包含设定包），DNA在"处理什么信息"上发挥空间越大。

2. **叙事者声音是一个独立的DNA维度**：它既不属于事件（骨架层），也不属于风格（DNA层）——它是策略层。要明确给DNA授权"是否使用主动叙事者声音"。

3. **DNA提取不应理想化**：从参考书提取DNA时，应以原文文本的实际特征为基准，而不是以"我对这个风格的理解"为基准。原文包含的"过时写法"（教材式设定、情绪标签）也是其DNA的一部分。

4. **信息释放决策点是连接骨架和DNA的桥**：在事件节点上标记密度，让DNA获知"这里有决策要做"——这是三层框架中最关键的机制。

5. **控制变量实验的方法论**：
   - 并行agent避免串扰
   - 骨架层级一致避免信息差异
   - DNA作为唯一变量
   - 多轮验证（CH1×CH2×CH3×CH2v2=四轮）
