# 落地研究：高质量事实骨架设计 + 稳定DNA逆向提取

基于复盘PRD的三层框架，往下推两个落地问题。

---

## 问题一：从零开始如何写出高质量事实骨架？

### 问题的本质

我们现在能写出高质量事实骨架，是因为**有原文可以反推**——原文CH2那700字游戏系统说明，我们知道它"挂在节点3的设定包里"。但写新书时没有原文，设定包里的信息从哪来？

真相是：**骨架的质量上限 = 上游设计文档的丰富度 × 骨架Agent提取设定包的能力。**

### 1.1 当前管线的问题

```
当前：
  幕纲(act-XX.yaml)  →
  角色卡               →  Director →  骨架Agent  →  产出事件链（5-8条）
  设定文档              →  
  数值体系              →
  几章纲                →

问题：幕纲只描述了"情绪目标"和"爽点类型"，不描述"这一章要传递哪些设定信息"。
      设定文档有，但骨架Agent没有被要求"从设定文档中提取当前章节需要的信息，挂载到事件节点上"。
```

### 1.2 解决方案：在幕纲中新增「信息释放计划」

幕纲目前的内容（以北帝镇诡录为例）：

```yaml
act-01:
  chapters:
    ch001:
      emotion: 拉人
      fun_level: 中爽点
      hook: 情绪驱使型
```

需要升级为：

```yaml
act-01:
  chapters:
    ch002:
      emotion: 压住
      fun_level: 中爽点
      hook: 悬念收束型
      info_release:  # ★ 新增
        - system_intro:  # 要释放的系统信息
            items: [属性面板, 职业体系, 技能体系, 传奇技能]
            density: 极高
            reason: "主角刚穿越，需要建立系统认知"
            release_decision: "DNA决定一次性/分散"
        - world_foreshadowing:  # 要释放的世界观信息
            items: [诸神黄昏, 圣者时代, 三条命]
            density: 极高
            reason: "钓读者用的世界观钩子"
            release_decision: "DNA决定是否砍掉"
```

### 1.3 骨架Agent的升级指令

骨架Agent接收到幕纲（含info_release）后：

```
Step 1: 读幕纲 → 知道本章事件链 + 信息释放计划
Step 2: 读设定文档 → 从设定文档中提取info_release中每项的具体内容
Step 3: 组装骨架：
  事件节点1：[事件]
    设定包：{
      挂载的info_release项: {提取后的具体内容}
    }
    密度标记：[来自info_release]
```

这样骨架Agent就不需要"猜"本章要传什么信息——幕纲已经告诉它了。

### 1.4 设定包的输入源映射

| 设定包类型 | 输入源 | 由谁维护 |
|:----------|:------|:--------|
| 系统/数值信息 | L1设定层 + 数值体系文档 | 产品/设定管线 |
| 世界观预告 | L1设定层 | 产品/设定管线 |
| 角色背景 | 角色卡 | 角色设计管线 |
| 势力关系 | 势力文档 | 产品/设定管线 |
| 地理/时间信息 | 地图 + 时间线 | 产品/设定管线 |
| 前文线索 | 全局摘要 + 事件日志 | ESM after阶段 |

### 1.5 关键前提

要把骨架做好，**必须先有完善的幕纲设定**。目前pop-novel-writer管线中，幕纲层（act-XX.yaml）由pop-novel-architect或pop-novel-plot产出。如果幕纲本身就不含info_release，骨架Agent就没有原料。

**所以真正要改的是幕纲的设计模板**——在act-XX.yaml中新增info_release字段，让上游管线产出这个信息。

## 问题二：文风DNA逆向提取如何稳定？

### 2.1 我们第一轮提取犯的错误

| 做了什么 | 应该是 |
|:--------|:-------|
| 只读了CH1开头1000字就提取 | 至少读3-5章，覆盖不同场景类型 |
| 凭"我对深渊风格的理解"来写（如实写/不评价） | 以原文的实际文本行为为基准 |
| 没区分"作者的写作习惯"和"这个场景的特殊写法" | 需要跨章节验证，确认某条规则是稳定的习惯 |
| 没对比同一作者的不同书 | 单书提取的DNA需要标注"样本局限性" |

### 2.2 需要多少样本

**最低要求**：3章（覆盖叙事/系统展示/情感场景各1）

**推荐数量**：5-7章（覆盖开篇、系统说明、对话、战斗、情感高潮、过渡、结尾各至少1例）

**验证要求**：提取完成后，用提取的DNA写一段新内容，与原文比对——看"像不像"。不像则追加2章继续提取。

### 2.3 提取模板

```markdown
# 文风DNA逆向提取报告

## 样本信息
- 书名：《深渊主宰》
- 作者：诸生浮屠
- 取样章节：CH1-CH3
- 取样原因：前三章，覆盖日常叙事(CH1)/系统说明(CH2)/情感场景(CH2-CH3)

---

## 一、叙事者声音分析

【方法】取样每章的开头500字+中间500字，统计叙事者"跳出来"的频率。

| 样本点 | 叙事者解释因果关系 | 叙事者替角色判断 | 叙事者直接陈述世界观 |
|:------|:-----------------|:---------------|:------------------|
| CH1开头500字 | 1次（"她今年才八岁，生火对她来说还很困难"） | 1次（"薇薇安坚强地擦去泪水"） | 0次 |
| CH1中间500字 | 0次 | 1次（"她远比同龄人懂得更多"） | 1次（"这里是琥珀城的贫民区……"大段说明） |
| CH2开头500字 | 2次（"不行！我要醒来！"内心OS也算叙事者主动表达） | 1次（"这是一个罪恶的世界！"） | 0次 |
| CH2中间500字 | 0次 | 0次 | 1次（"万能巧手……是游戏中让人咬牙切齿的无耻技能！"） |
| CH3开头500字 | 0次 | 1次（"它有这样的资格！"） | 1次（"贫民区里面永远都是一片烂泥路……"环境+评论） |
| CH3中间500字 | 0次 | 0次 | 2次（诸神黄昏世界观段落+乱葬岗说明段）

统计结论：叙事者解释因果关系频率 ≈ 每1000字0.5次
         叙事者替角色判断频率 ≈ 每1000字0.6次
         叙事者直接陈述世界观 ≈ 每1000字0.8次
评级：**中高侵入**——叙事者声音活跃但不主导

---

## 二、信息释放模式分析

【方法】识别每章中"设定说明段"的有无、长度、位置。

CH1：
  - 长设定段：1处（第60-73行，贫民区背景+索西亚介绍，约400字）
  - 短设定穿插：多处（偷盐/母亲历史/帮派纷争——在动作中自然带出）
  - 模式：**混合型**——一个长段集中释放 + 多处短穿插

CH2：
  - 长设定段：2处（属性面板说明约300字 + 游戏世界观说明约400字）
  - 短设定穿插：0处
  - 模式：**教材式集中释放**——两大段说明文

CH3：
  - 长设定段：1处（诸神黄昏世界观~500字）
  - 短设定穿插：多处（偷钱/帮派/乱葬岗）
  - 模式：**混合型**——一个长段集中释放 + 多处短穿插

统计结论：全书的整体模式是**混合型**——关键设定用教材段集中释放，细节设定用动作穿插。
         前三章共有4处长设定段，总字数约1600字。
评级：**设定说明式——被动的老网文模式**

---

## 三、情感表达方式分析

【方法】取样3个情感关键点，分析表达方式。

### 情感点1：薇薇安呼唤索伦（CH1）
原文：→ "呜呜呜！你终于好了！……" + 眼泪 + 动作（扑入怀中）
表达方式：对话 + 动作 + 情绪形容词
评级：**直白外露**

### 情感点2：希斯之死（CH2）
原文：→ 薇薇安崩溃，叙事者说明"它很疲惫，也很饿"，"它的任务完成了"
表达方式：叙事者主动总结 + 动作（抱+哭）+ 拟人化（"睡得很安详"）
评级：**叙事者替角色表达情绪**

### 情感点3：墓园告别（CH3）
原文：→ "希斯！……" 蹲着小坟前低声说话，站起来握住手
表达方式：对话 + 动作，**比前两章克制**
评级：**动作暗示为主**

统计结论：情感表达方式不稳定——CH1-2直白外露，CH3稍克制。
         原文在不涉及宏大设定的情感场景中更克制，在涉及设定说明的场景中更直白。
评级：**情境依赖型**——设定场景时叙事者主动，纯粹情感场景时动作化

---

## 四、对话策略分析

【方法】取样每章500字分析对话占比。

| 样本点 | 对话占比 | 对话功能 |
|:------|:--------|:--------|
| CH1中间500字 | ~15%（薇薇安对希斯/索伦对话） | 情绪释放 |
| CH2开头300字 | 0%（全是系统说明和内心OS） | — |
| CH3 | ~10%（薇薇安问"去哪里"+牧师对话） | 推进剧情+情绪 |

统计结论：对话占比整体偏低（10-15%），主要用于情绪释放。
         推进剧情的对话极少——大部分剧情通过叙述者描写推进。
评级：**叙事驱动型，非对话驱动型**

## 五、结论：原文DNA汇总

### 5.1 叙事哲学（5条原则）

1. **叙事者主动解释**——当作者认为读者需要"被指导"时，会跳出来说"这个设定是这样的""她很坚强""他有资格"。
   
2. **设定教材式释放**——重要世界观信息（游戏系统、诸神背景）采用集中说明段释放，不做分散处理。
   
3. **情感直白为主，情境克制为辅**——大部分情感场景中直接写"她很恐惧""她哭了"，但在告别等场景中能做到动作化。
   
4. **叙事驱动，对话为辅**——绝大部分剧情通过叙述者描写推进，对话占比低（10-15%）。
   
5. **信息密度前高后低**——前三章集中释放全书级世界观信息（1600+字设定段），后续章节设定密度会下降。

### 5.2 技法偏好

- 段落长度：中长段为主（3-6句），短段用于情绪转折
- 对话引导词：适度使用（"XX道"为主）
- 修辞：克制，偶有比喻（"像一块大石头""宛若饿狼"）
- 数字使用：游戏系统用精确数字，日常用模糊表达
- POV：Sauron为主，但可在角色内心和叙事者之间切换

## 六、提取方法论总结

### 稳定提取的步骤

```
Step 1：取样（3-5章，覆盖至少3种场景类型）
Step 2：按叙事者声音 / 信息释放 / 情感表达 / 对话策略 四个维度逐章统计
Step 3：跨章验证——某条特征是否在所有章节中一致出现
Step 4：写结论——叙事哲学（5条原则）+ 技法偏好（5-7项）
Step 5：验证——用提取的DNA写一段新内容，与原文对比
          → "像" → 提取完成
          → "不像" → 追加2章，回到Step 2
```

### 关键原则

1. **统计不猜测**——不写"我觉得这个作者喜欢留白"，要写"取样1000字中，留白出现X次"
2. **跨章验证**——某条规则如果在CH1出现但CH2-CH3都没有，那是个例不是DNA
3. **区分"习惯"和"意外"**——如果某条特征只在战斗场景出现而不在日常场景出现，这是一个"条件性DNA"
4. **标注局限性**——"本DNA基于深渊主宰CH1-3提取，不适用于该作品后期章节的叙事风格变化"
5. **双人校验**——同一份原文由两个独立agent提取，比对差异，不一致的维度重新统计
```
