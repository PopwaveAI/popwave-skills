# Step 0 - 交互式角色库决策

> **v1.2.0新增**：在角色库生成之前，通过2轮交互式决策采集用户偏好，产出`角色库决策表.md`。
> **核心原则**：给选项不给空白——每轮提供2-3个结构化选项+1个自定义入口。
> **杠杆位置**：NPC roster定下后，plot的Boss设计和NPC登场顺序都基于此。

## 前置条件

- Phase 3 world完成
- 全书设定/ + 世界圣经.md 齐全
- 骨架.md（众生攀登分层）+ 主角设计.md 齐全

## 产出

`设计/角色库/角色库决策表.md`——后续角色库生成的硬输入

## 2轮交互决策

### C1 NPC roster+反派设计（核心轮·必答）

**底牌消费**：全书设定/ + 世界圣经.md + 骨架.md众生攀登分层 + decon-plot NPC登场效果（如有）

**Agent动作**：
1. 读取世界圣经，提取活跃层NPC+势力结构
2. 读取骨架.md，提取众生攀登方式分层（4类）
3. 生成3个NPC roster方案，每个标注：核心NPC清单/反派设计/关系网/引入节奏

**选项生成规则**：
- 方案A：丰富型（8-10人，多层反派梯队，参考主流网文角色结构）
- 方案B：精简型（5-7人，集中反派，聚焦少数角色深度刻画）
- 方案C：反转型（6-8人，含盟友反转/中立方反转，增加意外性）

**输出格式**：
```
## C1 NPC roster+反派设计

基于{世界结构}，提供3个NPC roster方案：

A) {方案A名}（{人数}人）
   → 核心：{NPC1(定位)} / {NPC2(定位)} / {NPC3(定位)}
   → 反派：{反派1(底层)} → {反派2(中层)} → {反派3(关底)}
   → 关系网：{核心关系}
   → 参考：{参考书角色结构}

B) {方案B名}（{人数}人）
   → ...

C) {方案C名}（{人数}人）
   → ...

请选择，或调整NPC清单。
```

**影响标注**：NPC roster定下后，plot的Boss设计和NPC登场顺序都基于此。

### C2 NPC关系网细节（可选轮·可跳过）

**底牌消费**：C1选择的roster + 世界圣经.md势力结构

**Agent动作**：
1. 基于选定的roster，提出2-3个关系网方案
2. 每个方案标注：核心关系/暗线关系/冲突关系

**跳过规则**：用户说"跳过"或"默认"→ agent按角色库模板自动生成关系网

## 决策表产出格式

```markdown
# 角色库决策表
> 交互完成时间：{YYYY-MM-DD HH:mm}

## C1 NPC roster+反派设计
- 方案：{用户选定的方案}
- 核心NPC：{清单}
- 反派梯队：{清单}
- 人数：{N人}
- 选择：{A/B/C/自定义}

## C2 NPC关系网细节
- 核心关系：{关系描述}
- 暗线关系：{关系描述}
- 冲突关系：{关系描述}
- 选择：{A/B/跳过}

## 后续约束
- 角色库生成必须遵守本决策表
- 如需调整，需重新走对应轮次的交互
```

## 红线

1. **每轮必须给选项**——必须给2-3个结构化选项
2. **每轮必须有自定义入口**——选项末尾必须有"或调整XXX"
3. **选项必须在世界设定约束内**——NPC的攀登方式和等级必须匹配骨架.md
4. **核心轮不可跳过**——C1必答，C2可跳过
5. **决策表是角色库生成的硬输入**
