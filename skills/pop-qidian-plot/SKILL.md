# plot-design

> 将故事种子渲染为可供下游消费的幕纲+施工卡。管"写什么"，不管"怎么写"。v4.0.0：消费骨架.md（力量体系+动力引擎）+主角设计.md（爽感矛盾公式），Step 2重组为四层结构（2a本卷设定快照→2b分支剧情线≥6条→2c分幕设计→2d按幕白描），新增困难三层面（坐标系阻力+引擎代价+天赋限制），场景表爽感闭环引用主角设计.md的爽感矛盾公式。

---

## 做什么

输入：故事种子（seed-种子文档 + 骨架.md + 主角设计.md + 角色储备池 + 世界圣经）

> **世界圣经由 pop-qidian-world 产出**，含三层骨架：力量体系(广义坐标系)+动力引擎(众生攀登系统)+金手指(助力定位)。世界圣经.md结构：舞台/历史/神系/力量体系(含三段式骨架+跃迁条件)/敌人生态图/世界矛盾轴+附动力引擎。
> **plot消费世界圣经时新增消费"动力引擎"部分**：每卷从动力引擎全书演化表取本卷的引擎状态切片（引擎状态+演化节点+众生攀登方式变化+代价结构变化+主角在引擎中的位置），作为本卷剧情的底层驱动力。同时消费敌人生态图（取本卷碰撞的敌人层级）和世界矛盾轴（取本卷的矛盾状态）。

> **v4.0.0新增消费骨架.md+主角设计.md**（seed产出，消费前置）：
> - **设计/骨架.md**（seed Phase 1产出）：力量体系（广义坐标系）+动力引擎（众生攀登系统六组成）。plot消费时取本卷**坐标系位置变化区间**（主养成线起点阶位→终点阶位）+**引擎状态切片**（引擎状态+演化节点+众生攀登方式变化+代价结构变化+主角在引擎中的位置）。困难设计的"坐标系阻力"层+"引擎代价"层从骨架.md取。
> - **设计/主角设计.md**（seed Phase 2产出）：主角定位+金手指设计+**爽感矛盾公式**（坐标系门槛×天赋加速×代价约束）+金手指限制+爽感类型清单+爽感节奏建议。plot消费时取爽感矛盾公式，用于2d场景表爽感闭环设计+爽感因子标注。困难设计的"天赋限制"层从主角设计.md金手指限制取。

输出：幕纲 + 事实快照 + 分章施工卡

| 产出文件 | consumer | 核心内容 |
|---------|----------|---------|
| `卷纲/幕纲-幕NNN-{标题}.md` | review/write | 幕定位+剧情白描+六要素+章节流概览+交接 |
| `产出/正文/事实快照-幕NNN.md` | review/write | 动态状态载体 |
| `卷纲/chNNN-施工卡.md` | review/write | 每章锚点交接（v4.0.0新增爽感因子标注） |

前置依赖：seed必须完成Phase 1（骨架.md+创意.md+首章）+ Phase 2（主角设计.md）+ pop-qidian-world产出世界圣经。缺失→回seed（世界圣经缺失→回world）。

## 核心理念

**故事先行**：每一幕先想清楚"为什么需要这一幕"，然后直接白描剧情——一大段文字讲故事。维度拆解从剧情中反拆，不是先设计维度再拼剧情。

**危机螺旋**：幕=1个完整循环（危机→手段→变强→解决→新危机）。幕间切换=新敌人/新危机。手段轮换（不得连续2幕同一种）。

**困难三层面**（v4.0.0新增）：每幕的剧情困难必须来自三个层面——坐标系阻力（跨越阶位的难度，骨架.md力量体系）+引擎代价（众生攀登的代价，骨架.md动力引擎）+天赋限制（金手指的限制，主角设计.md）。三层面缺一=困难设计不成立。

**三层世界**：法则层（seed建骨架.md）→活跃层（plot每幕建）→快照层（review每章更）。

不做的事：
- 不写正文（write的活）
- 不建系统/数值（seed的活）
- 不编排章级节奏红线（review的活）

## 怎么操作（SOP骨架）

> **execution.mode**: formal（输入齐全落盘）/ draft（有缺口可补全）/ trial（不落盘）。

### Step 1：卷序规划 + 素材收集（每卷/每幕前置）→ `steps/step-1-material.md`

**每卷开始时**：设计全书敌人阶梯表（三段式骨架映射+卷级敌人量级递增+因果链验证+跨卷伏笔埋设）。产出`卷纲/全书敌人阶梯表.md`。

**v4.0.0新增加载**（每卷开始时）：
- **设计/骨架.md**（seed Phase 1产出）：取本卷坐标系位置变化区间（主养成线起点阶位→终点阶位）+动力引擎本卷切片（引擎状态+演化节点+众生攀登方式变化+代价结构变化+主角在引擎中的位置）。作为2a本卷设定快照的底层输入，也是2c分幕困难设计的坐标系阻力+引擎代价两层来源。
- **设计/主角设计.md**（seed Phase 2产出）：取爽感矛盾公式（坐标系门槛×天赋加速×代价约束）+金手指限制（天赋限制层来源）+爽感类型清单+爽感节奏建议。作为2d按幕白描场景表爽感闭环设计+爽感因子标注的依据。

**每幕设计前**：根据书特性规划6-10个搜索方向→外部搜索+内部涌现→落盘`燃料库/原始燃料/`。只收集不选用。

降级：无法搜索时用agent内部知识生成（标注quality:low）。素材不足时提醒用户。

### Step 2：幕纲（四层结构·逐卷循环）→ `steps/step-2-act.md`

> v4.0.0重组为四层结构（参考番茄plot v3.5.2但适配起点特色：保留1版直出白描+全书敌人阶梯表+施工卡+候选方向）。每卷按2a→2b→2c→2d顺序执行。

#### 2a 本卷设定快照

从骨架.md+世界圣经圈出+充实本卷设定。快照通过后，2b/2c/2d都基于本卷设定快照，不再直接引用骨架.md/世界圣经。

```markdown
### 本卷（第X卷）设定快照

#### 坐标系位置变化区间（v4.0.0新增·从骨架.md取）
- 主角起点阶位：阶位N（开卷时主角在坐标系的位置——具体到能做什么/不能做什么）
- 主角终点阶位：阶位M（卷末主角的坐标系位置——本卷成长目标）
- 卷级敌人力量范围：阶位X-阶位Y（本卷主要敌人的力量分布，和主角的差距在哪）

#### 引擎状态切片（v4.0.0新增·从骨架.md动力引擎取）
- 本卷引擎状态：（引擎处于什么阶段——稳定运转/即将演化/演化中/演化后）
- 演化节点：（本卷是否触发引擎演化？何时触发？触发事件是什么？）
- 众生攀登方式变化：（本卷众生攀登方式有无变化？哪种攀登方式成为主流？）
- 代价结构变化：（本卷攀登代价有无变化？新增什么代价？）
- 主角在引擎中的位置：（主角采用哪种众生攀登方式？在引擎中的位置变化）

#### 登场势力/角色（从世界圣经取+充实）
- 本卷碰撞势力：势力A（第N层）+势力B（第M层）——每个标注控制资源/弱点/和上层关系
- 本卷登场角色：主角+配角A+配角B+反派A+反派B——每个标注核心欲望/势力归属/和主角的关系张力

#### 涉及地图（填充势力和矛盾，不是空地图）
- 区域A：[描述/盘踞势力/内部矛盾/和区域B的矛盾]
- 区域间矛盾网络：区域A的势力A ↔ 区域B的势力B（矛盾核心：XXX）——本卷白描的冲突地基

#### 整卷主要矛盾/主要剧情走向
- 主要矛盾：（一句话——本卷主角要解决的核心问题）
- 主要剧情走向：（3-5句——本卷从开头到结尾的大致走向，不是分幕，是方向）
```

#### 2b 分支剧情线（≥6条）

从2a快照推导，先有线网再组织成幕。每幕内部多线交织，不是单线跑。

比例分布：主线1~2条 + 支线3~4条 + 暗线1~2条。

每条线标注：
- 类型（主线/支线/暗线）
- 起止状态（具体到主角/势力的状态变化）
- 涉及势力/地图（从2a快照取）
- 与主要矛盾的关系（主线如何推进矛盾/支线如何影响主线/暗线埋什么种子）
- **对应的众生攀登方式**（v4.0.0新增·与骨架.md动力引擎组成3"众生攀登方式分层"对齐）：标注这条线体现了哪种攀登方式（有资源/没资源/爬到顶/掉下来的人分别怎么爬）——主角线对应主角采用的攀登方式，支线可对应其他群体的攀登方式

暗线本卷只埋种子不揭示，标注预计揭示卷。

#### 2c 分幕设计

卷40~60章/幕8~12章/4~5幕。基于2b分支线分幕——每幕至少推进2~3条分支线（不是只推主线）。

每幕设计：
- 起承转合（起→承→转→合）
- 本幕推进的分支线（≥2条，从2b取）
- **困难来源三层面**（v4.0.0新增·缺一层=困难设计不成立）：
  - 坐标系阻力：本幕主角要跨越的阶位差距（从2a快照坐标系位置取——主角当前阶位vs目标阶位vs敌人阶位）
  - 引擎代价：本幕主角攀登付出的代价（从2a快照引擎状态取——众生攀登的通用代价+群体差异代价+累积效应）
  - 天赋限制：本幕金手指的限制如何制造困难（从主角设计.md金手指限制取——"不能取消坐标系本身的意义"如何在本幕体现）
- 碰撞势力+场景地标（从2a快照取）
- 幕末钩子（读者在想什么+主角面对什么新困境）
- **出场角色清单**（保留·character消费这个建角色库）：{角色名}({定位}·{势力归属}), {角色名}({定位}·{势力归属})——定位取值：主角/配角/反派·{层级}；势力归属从世界圣经势力格局取

对赌难度逐幕升级，不能平级或倒退。

#### 2d 按幕白描

保留v3.0.0的"剧情白描1版直出800-2000字"方法论——这是起点特色，不丢弃。

**每幕白描前·候选方向（简版多方案）**：
- 产出2-3个候选方向，每个半页纸：幕定位+剧情梗概(3-5句)+对手(谁+欲望)+情感钩子+幕末质变
- 2-3个方向必须有实质性差异——**不同对手类型**（数值型/机制型/信息型/关系型/系统型），不是同一对手的不同打法
- 暂停等用户选择

**3步设计链路**：
1. **定位与起点**：本幕位置+起点状态快照+本幕目的(剧情/成长/体验三层目标)+危机螺旋定义(敌人-成长耦合)
2. **剧情白描（核心·1版直出）**：
   - 先建活跃层(NPC/地点/组织)→直接写1版800-2000字完整白描(含2-4条故事弧线+多视角段落+燃料改造融入)
   - **v4.0.0新增·场景表爽感闭环**：白描后产出场景表，每个场景标注爽感闭环（触发→爆发→后果），引用主角设计.md的爽感矛盾公式，标注本次爽感触发了哪个因子组合（坐标系门槛/天赋加速/代价约束中的哪个组合）
   - 3项核心自检(主角主动/故事容量/循环完整)
3. **反拆分章**：从白描反拆章节预算/人物/战斗/成长/资源→分章概览表→幕末质变自检→正式幕纲输出(≤200行)

**场景表格式**（v4.0.0新增）：

| 场景 | 地点 | 人物 | 事件 | 冲突 | 爽感闭环 | 爽感因子组合 |
|:--|:--|:--|:--|:--|:--|:--|
| 1 | [地点] | [人物] | [事件] | [冲突] | 触发→爆发→后果 | 坐标系门槛×天赋加速 / 坐标系门槛×代价约束 / 三因子全触发 |

爽感因子组合取值（从主角设计.md爽感矛盾公式三因子中取）：
- 坐标系门槛×天赋加速：跨越阶位+金手指加速=突破爽
- 坐标系门槛×代价约束：跨越阶位+付出代价=张力爽
- 天赋加速×代价约束：金手指加速+付出代价=博弈爽
- 三因子全触发：坐标系门槛×天赋加速×代价约束=大爽点（每幕1个）

**后置**：事实快照初始化(`steps/step-3-snapshot.md`) → 分章施工卡(`steps/step-3-chapters.md`)
- **v4.0.0新增**：施工卡新增爽感因子标注——每章标注本章爽感节点触发的爽感因子组合，供write层实现爽感节奏。

## 红线

1. **故事先行红线**：白描是完整故事弧线，不是大纲/提纲/维度列表。白描不达标禁止进入反拆。
2. **对手驱动红线**：候选方向必须有明确对手+对手欲望。3候选必须不同对手类型。环境逼迫无对手=不合格。
3. **主角主动红线**：每章主角至少1决策+1行动。不把世界压迫当主角驱动。白描中主角必须驱动事件。
4. **危机螺旋红线**：每幕完整循环（危机→手段→变强→解决→新危机）。缺新危机=循环断裂。手段不得连续2幕同一种。敌人从全书阶梯表推导。
5. **活跃层红线**：白描前建本幕活跃层（NPC/地点/组织）。白描中现编不落盘=不合格。
6. **困难三层面红线**（v4.0.0新增）：剧情困难必须来自三个层面（坐标系阻力+引擎代价+天赋限制），缺一层=困难设计不成立。坐标系阻力←骨架.md力量体系 / 引擎代价←骨架.md动力引擎 / 天赋限制←主角设计.md金手指限制。
7. **爽感因子标注红线**（v4.0.0新增）：爽感节点必须标注触发的爽感因子（坐标系门槛×天赋加速×代价约束中的哪个组合）。无标注=爽感设计不可追溯。
8. **章末钩子决策型红线**（v4.0.0新增）：章末钩子必须是主角主动决策型，且与坐标系位置跃迁或引擎演化相关。环境压迫型钩子/与坐标系无关的钩子=不合格。

## 速查表

| 项 | 合格 | 不合格 |
|----|------|--------|
| 候选差异 | 不同对手类型(数值/机制/信息/关系/系统) | 同一对手不同打法 |
| 剧情白描 | 800-2000字完整故事弧线，2-4条弧线 | 大纲/提纲/维度列表 |
| 主角驱动 | 主角主动决策+行动驱动事件 | 世界压迫推主角走 |
| 对手设计 | 有欲望+与主角欲望对立 | 纯环境逼迫/无对手 |
| 危机螺旋 | 5环节完整+手段轮换 | 缺新危机/手段重复 |
| 活跃层 | 白描前建NPC/地点/组织 | 现编不落盘 |
| 燃料融入 | 选取≥2条素材改造融入白描 | 凭脑补/照搬 |
| 体量 | 每幕8-15章+幕末质变 | 不足3章/无质变 |
| 敌人阶梯 | 从全书阶梯表推导 | 凭空捏造 |
| 正式幕纲 | ≤200行，只含下游需信息 | 含思考过程 |
| 人物原型 | 幕级+角色标注原型(出处+参考+体感) | 只有功能描述 |
| seed前置 | 骨架.md+主角设计.md+世界圣经已加载 | 缺失直接写幕纲 |
| 分支剧情线 | ≥6条(主线1~2+支线3~4+暗线1~2)，标注众生攀登方式 | 单线推进/无攀登方式标注 |
| 困难三层面 | 坐标系阻力+引擎代价+天赋限制三层齐全 | 缺一层/困难来源不明 |
| 爽感因子 | 每个爽感节点标注触发的因子组合 | 无标注/爽感不可追溯 |
| 章末钩子 | 主角主动决策型+与坐标系跃迁/引擎演化相关 | 环境压迫型/与坐标系无关 |
| 场景表 | 爽感闭环(触发→爆发→后果)+爽感因子组合 | 只有事件无爽感闭环 |
