# 开局设计原则

> 通用设计智慧。适用于所有爽文开篇（至少同赛道通用）。
> 不是格式模板，是设计判断依据——面对多个选项时，凭什么选A不选B。

---

## 原则1：首杀目标必须有因果关联

**原则**：主角的第一次杀人/战斗，杀的目标必须和主角的核心欲望或伤口有因果关系，不能是无名路人。

**为什么**：首杀是读者建立对主角认同感的关键时刻。如果杀的是无名路人（随机入侵者/路遇劫匪），读者只会感受到"主角能打"，不会产生情感投入。如果杀的是威胁主角核心欲望的人（要卖妹妹的人贩子/伤害主角亲人的恶棍），读者会感受到"该杀"——情感上和主角站在一起。

**怎么做**：
1. 从种子文档中找到主角的核心欲望和伤口
2. 设计一个"威胁核心欲望或触发伤口"的角色作为首杀目标
3. 首杀目标必须有身份/动机——读者能一句话说清"这人为什么该死"
4. 首杀目标身上携带的物品/信息要引出下一条危机线（翻尸体不只是搜钱，更是发现线索）

**判断标准**：如果首杀目标换成另一个无名路人，对剧情没有任何影响=不合格。

**正例**：主角的伤口是"失去亲人"，核心欲望是"保护妹妹"。首杀目标是收了人贩子定金来确认妹妹情况的帮派混混。翻尸体发现纸条"半精灵丫头，活的值更多"→引出人贩子危机线。

**反例**：主角首杀目标是半夜撬门偷东西的无名贼。翻尸体搜到3铜币。杀他对剧情没有任何推动。

## 原则2：开局面板/属性要给"天才感"

**原则**：爽文开篇，主角的面板/属性弹出时，读者的第一感受应该是"这主角牛逼"，不是"这主角不太行"。

**为什么**：面板首次弹出是金手指亮相的高光时刻。如果属性偏低（只比普通人强一点），读者感受到的是"主角还得慢慢练"——爽感被稀释。如果属性偏高（远超普通人、接近专业级），读者感受到的是"主角开局就比别人强"——爽感即时释放。成长空间不靠压低开局属性来实现，而靠拉高天花板（20级→传奇→封神→微弱→强大神力，三大阶段足够长）。

**怎么做**：
1. 主角核心属性（根据职业定）至少达到"专业级"参照物——盗贼敏捷≥15（超过绝大多数街头混混）、战士力量≥14（码头苦力级以上）、法师智力≥15（超过大多数识字的人）
2. 面板弹出时用参照物梯子让读者直观感受"这个数值意味着什么"
3. 属性来源要有合理性——半精灵天赋+前世意识融合+街头生存训练=高敏捷
4. 天花板要远——开局属性和巅峰属性的差距要足够大，让读者看到成长空间
5. 知识偏差/能力缺陷不在这时候提——等首杀后再通过事件触发，不在面板弹出的爽感时刻冲淡体验

**判断标准**：面板弹出时如果读者的反应是"就这？"=不合格。如果读者的反应是"卧槽这主角开局就挺强"=合格。

**正例**：面板弹出敏捷17。参照物：普通人10，码头苦力力量12但敏捷只有9，街头混混11-13。17意味着"超过绝大多数混混，接近专业盗贼"。读者感受：主角是天才。

**反例**：面板弹出敏捷14。参照物：比普通人快一点。读者感受：主角还得练很久才能打。

## 原则3：世界观植入要有载体，禁止旁白

**原则**：世界观信息必须通过载体（回忆/对话/事件/场景）自然带出，禁止用整段旁白讲解。

**为什么**：读者读爽文时追求快节奏和高密度。整段世界观旁白会打断节奏，读者会跳读或流失。通过载体植入时，世界观信息是"顺便获得的"——主角在回忆时想到、在对话中听到、在事件中体验到——读者不知不觉就理解了世界。

**怎么做**：
1. 第一幕前5章至少2章承担世界展开功能
2. 每章展开的世界观信息不超过3条核心点（背景/规则/势力/历史）
3. 植入方式选择：
   - **回忆触发**：面板激活/受伤/看到某物→触发前世记忆碎片→带出世界观
   - **对话传递**：NPC的抱怨/威胁/炫耀→自然带出势力关系/社会结构
   - **事件展示**：被帮派收保护费→展示贫民区的权力结构
   - **场景沉浸**：走过某条街→通过环境描写建立空间感
4. **绝对禁止**：用一整段或一整章做世界观百科式讲解

**判断标准**：如果删掉世界观信息段落，读者完全不知道在说什么=旁白式植入（不合格）。如果世界观信息嵌在角色行动/对话/回忆中，删掉后行动/对话/回忆仍然完整=载体式植入（合格）。

## 原则4：第一章可以用非主角视角建立情感连接

**原则**：第一章可以用非主角视角（配角/妹妹/旁观者）开场，先建立读者对配角的情感连接，再切入主角。

**为什么**：如果全程主角视角，读者只能从主角行动中理解世界。用配角视角开场，读者可以先爱上配角——当主角行动时，读者已经在乎配角了，主角保护配角的行动就有了情感重量。这是"先让读者在乎人，再让读者看主角怎么保护人"的设计。

**怎么做**：
1. 评估第一章是否适合用配角视角——如果主角第一章昏迷/缺席/迟到，配角的视角可以填补这段时间
2. 配角视角中展示：配角的处境/恐惧/坚强/对主角的依赖→建立读者情感
3. 配角视角中自然带出世界观信息（配角看到的威胁=世界观信息）
4. 切回主角视角时，读者已经带着对配角的关心→主角的每个行动都有了情感重量

**判断标准**：用配角视角开场后，读者能用一句话说清"这个配角是什么样的人，我希望主角保护她"=合格。

**注意**：不是每本爽文都必须用配角视角开场。这是一种可选技法，适用于"主角第一章无法直接展示全部处境"的情况。

## 原则5：首杀后翻尸体要引出下一条危机线

**原则**：杀完人翻尸体不只搜钱，还要发现线索/物品/信息，引出下一条危机线。

**为什么**：翻尸体是爽文的即时奖励时刻（钱/装备/物资），但如果只有物质奖励就浪费了叙事机会。在翻尸体时发现线索（帮派标记/纸条/信物/地图），可以自然地将危机从"当前事件"升级到"更大的威胁"——首杀不是结束，而是更大危机的起点。

**怎么做**：
1. 翻尸体获得物质奖励（钱/装备/食物）——即时满足
2. 同时获得信息奖励（帮派标记→帮派会追查/纸条→人贩子网络/信物→某势力介入）
3. 信息奖励直接引出下一章/下一幕的危机线
4. 翻尸体的过程展示主角性格——面无表情/务实/冷酷

**判断标准**：翻尸体只搜到钱和装备=不合格（浪费叙事机会）。翻尸体同时发现引出下一条危机线的线索=合格。

## 原则6：危机倒计时要在第一章建立

**原则**：第一章就要建立至少1个有倒计时的危机——主角不得不行动的结构性强制。

**为什么**：没有倒计时的危机=主角可以不行动=没有追读动力。倒计时让读者知道"时间在流逝，主角必须在X之前做Y"——每章的推进都有紧迫感。

**怎么做**：
1. 从种子文档中找到最紧迫的生存压力（保护费/病情/食物/敌人逼近）
2. 给这个压力加上时间限制（3天后到期/药只够2天/明天帮派来人）
3. 倒计时在第一章通过事件/对话建立（帮派隔门喊"三天后拿不出钱就滚"）
4. 后续每章推进时倒计时在收紧（第2章变成2天/第3章变成1天）

**判断标准**：读者读完第一章后知道"主角必须在X之前做Y"=合格。读者不知道主角为什么着急=不合格。

## 原则7：双面性格要在一个场景中展示反差

**原则**：主角的"对外冷酷"和"对内温柔"要在同一个场景中形成反差，不能分开展示。

**为什么**：如果第一章全写主角冷酷杀人，第二章才写他对妹妹温柔，读者无法在一章内建立"这个杀伐者有柔软面"的印象。反差在同场景中才最有冲击力——杀完人面无表情擦匕首→转身给妹妹掰碎面饼泡水喂她，手指刻意放轻。

**怎么做**：
1. 在同一个场景中安排"冷酷行动"和"温柔行动"的对比
2. 冷酷行动：杀人/翻尸体/面无表情
3. 温柔行动：紧接着对妹妹/老狗/弱者的细微温柔（语气变轻/动作放轻/多说一句话）
4. 反差不通过旁白说明，通过行动对比让读者自己感受

**判断标准**：同一场景中读者能感受到"这个杀人魔竟然对妹妹这么温柔"=合格。需要旁白解释"他虽然冷酷但对妹妹很好"=不合格。

## 使用方式

本文件位于 `references/设计原则库/开局设计原则.md`。

### 在plot-design的Step 3（幕纲）中

设计每章施工卡时，agent应加载本文件，对照原则检验设计决策：
- 首杀目标是否符合原则1？
- 开局属性是否符合原则2？
- 世界观植入是否符合原则3？
- 翻尸体是否引出下一条危机线（原则5）？
- 倒计时是否建立（原则6）？

### 在prose-render的Step 3.5（章意图思考）中

渲染前思考时，核验施工卡是否遵循了设计原则。如果施工卡中有违反原则的设计，在修正指导中指出。
