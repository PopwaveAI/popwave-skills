# Step 2 - 幕纲（3步设计链路）

> **核心原则**：先有故事，再拆维度。剧情白描是核心——写不好，后面展开全是浪费时间。
> **体量要求**：每幕8~15章完整体量。幕末必须有可验证的质变。

---

## 阶段A：方向选择（简版多方案）

> 快速产出2-3个不同方向的剧情走向，让用户用人的判断力挑选最有张力的那个。
> **不展开细节**——每个方案控制在半页纸以内。

### 2A-a. 加载上下文

读取：
- `seed-种子文档.md`（必读——世界观/主角/金手指/禁区/核心卖点/背景暗流）
- `主角引擎.md`（必读——主角行动偏好/阵营操作性定义/第一章主动动作）
- `角色储备池.md`（必读——S/A/B/C角色+反应模式+可制造事件）
- `世界圣经.md`（必读——法则层：力量体系/地理/历史/神系）
- `卷纲/全书敌人阶梯表.md`（必读——本卷卷级敌人+因果链）
- `燃料库/原始燃料/索引.md`（必读——浏览可用素材池）
- 已完成的前序幕纲（如有——确保衔接）

**红线**：主角引擎/角色储备池/全书敌人阶梯表不存在→停止。

### 2A-b. 产出2-3个候选方向

每个方向需要以下内容（简版，不展开）：

---

#### 候选方向N

**幕定位**（3行）：
- 本幕作用：（一句话——这一幕要解决什么/建立什么/转变什么）
- 核心体验：（一句话——读者读完应该获得什么感受）
- 情感弧线：从____ → 经过____ → 以____结束

**对手**（2行）：
- 对手是谁：（姓名/类型——从全书敌人阶梯表取本卷卷级敌人拆解）
- 对手欲望：（一句话——对手想要什么，必须与主角欲望对立）

**剧情方向梗概**（3-5句话描述整幕故事弧线，不分章）

**情感钩子**（1行）：读者追读这一幕的具体理由——不是"想知道接下来发生什么"，是具体的好奇/担忧/期待

**幕末质变**（1行）：主角什么不可逆变化 + 剧情什么格局变化

---

### 2A-c. 候选差异化要求（硬约束）

2-3个方向必须有**实质性差异**——**不同对手类型**：

| 对手类型 | 说明 | 示例 |
|---------|------|------|
| 数值型 | 敌人纯粹比主角强，主角需要变强才能打 | 等级碾压的BOSS |
| 机制型 | 敌人有特殊机制/能力，主角需要找到破解方法 | 不可触碰的能力/特殊规则 |
| 信息型 | 敌人掌握主角不知道的信息，主角需要获取情报 | 潜伏的叛徒/幕后操控者 |
| 关系型 | 敌人与主角有情感羁绊，打起来有情感代价 | 前盟友/亲人/恩人 |
| 系统型 | 敌人利用系统/规则压迫主角，主角需要利用规则缝隙 | 体制压迫/合法恶人 |

**红线**：3候选不能全是同一对手类型。"不同开局事件但同一对手类型"=换皮=不合格。如果3个方向看起来"都差不多"，重来。

### 2A-d. 输出与等待确认

将2-3个候选方向输出到 `卷纲/幕NNN-候选方案.md`，然后**暂停，等待用户选择**。

用户选定后进入阶段B。

---

## 阶段B：3步设计链路

> 基于用户选定的方向，走以下3步。所有思考沉淀到 `卷纲/幕NNN-设计笔记.md`。

### 第1步：定位与起点快照

**1a. 本幕位置**
- 本幕是第X卷第Y幕
- 前序剧情回顾（一段话）
- 本幕功能位置（开篇/发展/转折/高潮/收束）

**1b. 起点状态快照**

| 维度 | 当前状态 |
|------|---------|
| 身体 | |
| 面板 | （等级/属性/技能） |
| 物质 | （金币/装备/资源） |
| 情报 | （知道什么/不知道什么） |
| 社会关系 | （盟友/敌人/中立） |
| 外部威胁 | |

**1c. 三层目标**

| 目标层 | 内容 |
|--------|------|
| 剧情推演 | 本幕结束时剧情格局是什么？ |
| 主角成长 | 本幕结束时主角等级/能力/资源状态？ |
| 读者体验 | 本幕结束时读者获得了什么体验？ |

**1d. 危机螺旋定义**

| 环节 | 内容 |
|------|------|
| 危机 | 本幕起始危机 |
| 手段 | 主角用什么突破？（与上一幕不同） |
| 变强 | 主角获得了什么？ |
| 解决 | 全部解决还是部分解决？ |
| 新危机 | 本幕结束时的下一个危机？（来源类型与上一卷同期不同） |

**敌人-成长耦合**（从全书敌人阶梯表推导）：

| 项 | 内容 |
|----|------|
| 本幕敌人 | 从卷级敌人拆解（姓名/类型/量级） |
| 主角为什么打不过 | 敌人核心优势？主角缺什么？ |
| 主角如何变强 | 需要获得什么？ |
| 打完获得什么 | 具体收益→是打下一个敌人的什么前提？ |

**1e. 开局契约**（仅第一幕需要）

| 项 | 内容 |
|----|------|
| 第一章弱势 | 主角开局弱势状态 |
| 触发条件 | 什么事件打破弱势 |
| 第一主动操作 | 主角主动做的第一件事 |
| 第一可见反馈 | 主动操作后的第一可见反馈 |
| 新压力 | 主动操作引来的新压力 |
| 禁止提前兑现 | 本幕不能提前给出的后续收益 |

### 第2步：剧情白描（核心）

> **这是整个幕纲设计的核心环节，占阶段B 50%的注意力。**
> **v3.0.0改动**：从"3段差异化白描→评估→整合"改为"1版直出"。原因：3段流程太重，实测agent直接跳过。1版直出+3项核心自检=更可执行。

#### 2a. 本幕世界展开（活跃层建设）

> 白描前必须建活跃层。不建活跃层就写白描=现编=后续漏洞。

从法则层推导本幕具体细节：

| 活跃层项 | 推导来源 | 产出 |
|---------|---------|------|
| 本幕地点 | 法则层地理→具体场景 | 地名/布局/环境 |
| 本幕NPC | 角色储备池B/C级→激活 | 姓名/身份/反应模式 |
| 本幕组织 | 法则层社会结构→具体势力 | 组织名/规矩/与主角关系 |
| 本幕资源 | 法则层经济→具体可获取资源 | 资源类型/获取方式/价值 |

活跃层追加到`世界圣经.md`的活跃层部分（append）。白描中出现的任何具体地名/NPC名/组织名必须先在活跃层落盘。

#### 2b. 燃料选取与改造

从`燃料库/原始燃料/`中为本段白描选取≥2条素材。不是照搬——是"选取→改造→融入"：

| 改造操作 | 说明 |
|---------|------|
| 提取结构 | 只取骨架模式，不取具体内容 |
| 换皮融合 | 保留结构，替换为本书世界观元素 |
| 降维适配 | 大体量作品只取幕级子结构 |
| 碰撞合成 | 两条燃料交叉产生新设计 |

燃料选取记录（写入设计笔记）：

| 选取的燃料 | 来源方向 | 改造方式 | 融入位置 |
|-----------|---------|---------|---------|
| | | | |

**红线**：白描中完全没有使用原始燃料库中的任何外部素材→说明Step 1燃料收集白做了。

#### 2c. 写1版完整白描

直接写1版800-2000字的完整剧情白描。不是大纲，是故事。

**白描必须包含**：
- 起因→发展→高潮→收束的完整弧线
- **2-4条故事弧线**（不是1个任务撑全幕——情感弧线+成长弧线+战斗弧线交织）
- 主角主动驱动剧情（不能被推着走）
- 幕末有质变（不可逆变化）
- 资源获取活动自然嵌入（撬锁/杀人/搜刮等主角主动行为）
- **多视角叙事**：至少1个非主角视角段落（敌人视角/旁观者视角）。不是"主角猜到敌人在想什么"，是真正切到敌人的眼睛看这场戏
- 燃料改造融入（读者看不到参考痕迹）
- 成长分散在多个活动中（不是一次杀戮连升3级）
- 至少1-2次挫折，且每次挫折推动故事转向新方向
- 前期有爆发（白描前1/3有爽点——首杀/面板激活/越级击杀）

**故事容量要求**：

| 弧线数 | 适用 |
|--------|------|
| 1条 | 不够——一个任务撑不起8-15章 |
| 2-3条 | 标准容量（情感+成长+战斗交织） |
| 4条+ | 高密度 |

每条弧线必须有独立的核心事件+独立的爽点+独立的资源获取。

#### 2d. 3项核心自检

白描写完后，执行3项核心检查（不是13项——精简到最关键的3项）：

| 检查项 | 问题 | 不达标=重写 |
|--------|------|------------|
| **主角主动？** | 主角在驱动事件还是被推着走？每条弧线的主角是否在做决策+行动？ | 主角被事件推着走→重写 |
| **故事容量够？** | 有2-4条故事弧线吗？还是一个任务撑全幕？成长分散在多个活动中吗？ | 只有1条弧线/成长一次跳跃→补弧线 |
| **循环完整？** | 危机→手段→变强→解决→新危机五环节齐全？手段与上一幕不同？ | 缺新危机/手段重复→补 |

**红线**：自检不达标→重写白描，不进第3步。带着烂地基往后走=全盘报废。

---

### 第3步：反拆分章

> 从第2步的剧情白描反拆维度。先有故事再拆维度。

#### 3a. 基础维度反拆

| 维度 | 从剧情白描中提取 |
|------|----------------|
| 章节预算 | 8-15章 |
| 地理范围 | 剧情涉及哪些区域 |
| 出场人物 | 谁必须出场（幕级+角色标注原型） |
| 主角战斗 | 哪些战斗？对手什么等级？ |
| 主角成长 | 成长节点？等级变化曲线？ |
| 资源事件 | 嵌入了哪些资源获取活动？ |
| 敌人-成长耦合 | 本幕敌人逼主角获得了什么→打下一个敌人的前提？ |

#### 3b. 幕末质变自检

- [ ] 主角有不可逆变化（等级/能力/关系/身份）
- [ ] 剧情格局有变化（新地图/新势力/新信息）
- [ ] 没有提前透支后续高潮收益

#### 3c. 分章概览表

| 章 | 章名 | 核心事件 | 章型 | 主动行为 | 资源获取 | 倒计时 |
|----|------|---------|------|---------|---------|--------|

分章原则：
- 每章有完整核心事件（不是片段）
- 每章有主角主动行为
- 章型交替（不得连续2章同章型）
- 白描中的核心事件是硬锚点不可删减

#### 3d. POV规划

| 章 | 主视角 | 需切换的POV | 切换目的 | 切换时机 |
|----|--------|------------|---------|---------|

每幕至少1个非主角视角段落。

## 正式幕纲输出

> 从3步产出中提取正式幕纲，输出到 `设计/第一卷剧情/卷纲.md`（v4.1.0改名：原"剧情白描.md"→"卷纲.md"）。
> 全卷四层（2a+2b+2c+2d）合并为一个文件。只包含下游需要的信息，不包含思考过程。
> 严格按照以下结构输出。

### 卷纲.md 文件结构

```markdown
# 第X卷卷纲 · {卷名}

## 2a 本卷设定快照
（坐标系位置变化区间+引擎状态切片+登场势力/角色+涉及地图+整卷主要矛盾）

## 2b 分支剧情线
（≥6条线，每条标注类型/起止/众生攀登方式/与主要矛盾关系）

## 2c 分幕设计
（每幕含起承转合四段式+困难三层面+碰撞势力+出场角色清单+幕末钩子）

## 2d 按幕白描
（每幕800-2000字完整白描+场景表爽感闭环）

## 卷纲精彩度自检
（五问自检结果——见下方模板）
```

### 起承转合四段式模板（v4.1.0新增·2c每幕必须填写）

每幕的起承转合不能只写标签，必须填写以下内容：

```markdown
### 第N幕 · {幕名}（chXXX-chXXX）

#### 起（开篇·2-3章）
- 主角起始状态：{一句话描述主角开篇时的状态}
- 打破事件：{什么事件打破了主角的平静/计划}
- 新危机：{本幕起始危机是什么}
- 读者钩子：{读者为什么要追这一幕——具体的好奇/担忧/期待}

#### 承（发展·3-4章）
- 应对手段：{主角用什么手段应对危机——与上一幕不同}
- 关键阻力：{遇到什么阻力——困难三层面在此展开}
  - 坐标系阻力：{跨越什么阶位差距}
  - 引擎代价：{付出什么攀登代价}
  - 天赋限制：{金手指的什么限制制造了困难}
- 关键获得：{主角获得什么关键资源/信息/盟友}

#### 转（转折·2-3章）
- 方案被打破：{主角的方案被什么打破}
- 意外反转：{出现什么意料之外的转折}
- 被迫新决策：{主角被迫做什么新决策}
- 反转关联：{反转与坐标系跃迁/引擎演化的关联}

#### 合（高潮+收束·2-3章）
- 破局方式：{主角如何破局}
- 大爽点：{三因子全触发的大爽点是什么}
- 幕末质变：{主角什么不可逆变化+剧情什么格局变化}
- 新危机钩子：{本幕结束时的下一个危机}
```

**红线**：起承转合四段任一段缺内容=分幕设计不成立。只写"起：开局；承：发展；转：转折；合：收束"这种标签=不合格。

## 卷纲精彩度五问自检（v4.1.0新增）

> 全卷四层（2a+2b+2c+2d）完成后，必须执行此自检。
> 任一问不通过=重做对应层，不得进入章锚点表。

### 五问

| # | 问题 | 通过标准 | 不通过→重做哪层 |
|:--|:--|:--|:--|
| 1 | **读者追读理由**：如果读者读到第3章，他有什么具体理由继续追到第10章而不是弃书？ | 能说出≥2个具体的追读理由（不是"想知道接下来发生什么"这种废话） | 2c起段+2d白描前1/3 |
| 2 | **意外性**：全卷有没有至少2个让读者"没想到"的转折？转折是不是都在读者能猜到的范围内？ | ≥2个意外转折，且至少1个与坐标系跃迁/引擎演化相关 | 2c转段 |
| 3 | **情感投入**：读者会不会为某个角色担心/愤怒/期待？有没有一个角色让读者产生情感投射？ | 能指出≥1个情感锚点角色+具体的情感投射理由 | 2b分支线+2d白描 |
| 4 | **爽感密度**：全卷40章中有多少章有明确的爽感节点？爽感节奏是否波浪式递进（不是平铺或集中在卷末）？ | ≥60%的章节有爽感节点，且爽感强度逐幕递进 | 2d场景表 |
| 5 | **卷末高潮**：最后一幕的高潮是否是全卷最精彩的段落？读者读完会不会产生"下一卷什么时候出"的冲动？ | 合段的大爽点（三因子全触发）是全卷最强爽点+卷末钩子有明确下一卷期待 | 2c合段+2d白描末段 |

### 自检结果模板（写入卷纲.md末尾）

```markdown
## 卷纲精彩度自检

| # | 问题 | 结论 | 通过？ |
|:--|:--|:--|:--|
| 1 | 读者追读理由 | {具体回答} | ✅/❌ |
| 2 | 意外性 | {具体回答} | ✅/❌ |
| 3 | 情感投入 | {具体回答} | ✅/❌ |
| 4 | 爽感密度 | {具体回答} | ✅/❌ |
| 5 | 卷末高潮 | {具体回答} | ✅/❌ |

总结：{X/5通过} {如未全通过，说明哪层需重做}
```

**红线**：五问全通过才可进入章锚点表。任一问不通过=重做对应层后再自检。
