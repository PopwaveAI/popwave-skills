# Step 3 - 章锚点表

> **v4.1.0重构**：从"每章一张施工卡（5部分×2000字）"简化为"全卷一张章锚点表（4硬锚点+3软指导）"。
> 原因：v4.0.0的施工卡过度约束write——9字段全硬约束，write没有发挥空间。v4.1.0只锁死plot必须控制的4个锚点，其余3个字段为软指导（write可调整）。
> **核心原则**：锚点表是"轨道+留白"——锁死剧情轨道（不能跑偏），留出写法空间（怎么写由write决定）。

## 前置条件

- `设计/第一卷剧情/卷纲.md`已通过精彩度五问自检
- `幕NNN-设计笔记.md`（参考——提取本章相关信息）

## 产出

全卷一张表：`设计/第一卷剧情/章锚点表.md`

## 章锚点表格式

```markdown
# 章锚点表 · 第X卷

> 每章锚点供write消费：4个硬锚点（必须遵守）+ 3个软指导（可调整）。
> 硬锚点=plot锁死的剧情轨道。软指导=write可以发挥的空间。

| 章 | 章型🔒 | 核心事件🔒 | 爽感节点🔒 | 章末钩子🔒 | 主角变化💡 | 爽感因子💡 | 章名💡 |
|:--|:--|:--|:--|:--|:--|:--|:--|
| ch001 | opening_shift | {本章必须发生的事} | {本章爽点} | {章末悬念} | {角色弧光位移} | {因子组合} | {暂定名} |
| ch002 | exploration_gain | ... | ... | ... | ... | ... | ... |
| ... | | | | | | | |

> 🔒 = 硬锚点（write必须遵守，不可更改）
> 💡 = 软指导（write可调整，但调整后需在执行报告中说明原因）
```

## 4个硬锚点（write必须遵守）

| 锚点 | 含义 | 为什么锁死 |
|:--|:--|:--|
| **章型** | 6种枚举值之一（opening_shift/exploration_gain/confrontation_pressure/combat_reversal/reveal_hook/high_tier_display） | 章型决定叙事节奏波形，是全书节奏控制的核心 |
| **核心事件** | 本章必须发生的剧情事件（1-2句话） | 这是plot的剧情轨道，write不能跑偏到别的事件上 |
| **爽感节点** | 本章的爽点触发点 | 爽感节奏是波浪式递进的，每章的爽点位置由plot控制 |
| **章末钩子** | 章末必须抛出的悬念/驱动力 | 钩子是章间衔接的纽带，必须与下一章的核心事件对接 |

**红线**：write更改4个硬锚点中的任何一个=不符合plot设计，review应打回。

## 3个软指导（write可调整）

| 指导 | 含义 | 为什么不锁死 |
|:--|:--|:--|
| **主角变化** | 角色弧光的阶段性位移建议 | write在具体场景中可能发现更自然的弧光表达方式，只要方向一致即可 |
| **爽感因子** | 爽感来源标签（坐标系门槛×天赋加速×代价约束的组合） | write可以根据具体场景调整爽感因子的权重，只要爽感节点本身不变 |
| **章名** | 暂定章名 | 章名是write的创意空间，plot只给参考 |

**规则**：write调整软指导时，需在执行报告中说明"调整了什么+为什么调整"。

## 生成规则

1. 从卷纲.md的2c分幕设计+2d白描反拆每章锚点
2. 章型交替（不得连续2章同章型）
3. 核心事件是白描中的硬锚点，不可删减
4. 爽感节点的分布必须是波浪式递进（不是平铺或集中在卷末）
5. 章末钩子必须是主角主动决策型，且与坐标系跃迁或引擎演化相关
6. 全卷章数与卷纲2c分幕设计的章数一致

## 自检

全表生成后检查：
1. 章型分布有节奏感？（不是连续同一种）
2. 爽感节点覆盖≥60%的章节？
3. 章末钩子链是否形成信息释放链条？（每章钩子→下章核心事件）
4. 硬锚点是否清晰到write不需要猜？（核心事件1-2句话能说清）
5. 软指导是否给write留了足够空间？（不是把每章都写成详细大纲）

## 红线

1. **硬锚点不可更改**——write更改章型/核心事件/爽感节点/章末钩子=不合格
2. **章锚点表不是施工卡**——不写7节拍/beat表/场景描述。只写锚点+指导
3. **锚点表≤全卷章数×1行**——每章一行，不展开。详细内容在卷纲.md
