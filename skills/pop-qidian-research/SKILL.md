## 流程·燃料+题材机制

> **触发**：用户说"找燃料/补燃料/题材机制/涌现式research"时启用。
> **产出**：`涌现/research-写作燃料.md` + `涌现/content-mechanics.md`
> **执行步骤**：`steps/step-1-find.md` → `steps/step-2-output.md`

找能入场面的写作燃料，并对参考书里的题材机制做分流：能进本书的转成可触发事件/主角操作点/可外显爽点；不能直接迁移的写入 content-mechanics。

### 核心约束

- 每条燃料必须落到三要素：事件形状 / 主角操作点 / 可外显爽点
- 内容机制分流：能进本书→燃料表；不能迁移→content-mechanics.md；疑似文风特征→禁止交给 soul/write
- 燃料分级：近期可用 / 中期保留 / 禁用
- formal 必读输入：已读 seed/临时 seed；燃料不少于 3 条；主燃料有事件形状/主角操作点/可外显爽点

详细执行步骤见 `steps/step-1-find.md` 和 `steps/step-2-output.md`。

## 流程·decon-lite

> **触发**：pipeline Phase 0 Stage 2。用户给了力量体系参考书。
> **产出**：`素材/decon-lite-{书名}.md`。消费方：seed 1d 力量体系设计 + seed 1e 动力引擎设计 + seed 1d-0 改编策略层 + seed 故事纲领（节奏参考）。

**v4.0.0引入**：从番茄research v2.2.0完整移植9表拆解。表1拆到规则级（主养成线+子养成线抽象，子养成线数量由书决定不固定字段）。表6-8服务seed 1d-0改编策略层。表9服务seed 1e动力引擎设计。所有表字段用**抽象结构**（"这本书有几层X"），不用具体内容模板（"4层势力"）。

### DL1 · 接收需求

输入：书名 + 本地txt路径 + 采样范围（默认前 300 章）

### DL2 · 九表拆解

#### 表1：力量体系结构（规则级深度）

> **规则级要求**：从描述级升级到规则级。引入"主养成线+子养成线"抽象——主养成线=位阶（区分境界的核心刻度），子养成线=位阶内的平行成长维度（每本书的子养成线数量和内容不同，由书决定，不固定字段）。

| 字段 | 说明 | 必须给出具体内容 | 抽象要求 |
|:--|:--|:--|:--|
| 力量体系类型 | 等级制/修炼境/规则系/百分比/职业树/... | 有明确判断理由 | 通用字段 |
| **主养成线（位阶）** | 这本书的位阶名称/数量/每位阶的章节范围/每位阶的质变标志 | 至少覆盖前4阶，每阶标注质变标志 | 抽象结构：列位阶名+章范围+质变标志 |
| **子养成线清单** | 这本书有几条平行的子养成线？每条叫什么？（数量由书决定） | 列出所有子养成线名称 | 抽象结构：先识别数量再列名称 |
| **子养成线1规则** | 第1条子养成线的获取规则+消耗规则+上限+与主养成线的关系 | 有具体数值/条件+原文chXX出处 | 逐条拆，不固定字段名 |
| **子养成线2规则** | 第2条... | 同上 | 同上 |
| **子养成线N规则** | 第N条...（N由这本书决定） | 同上 | 同上 |
| **子养成线交叉规则** | 子养成线之间怎么互相影响？有没有兼职惩罚/互相制约/互相增益？ | 有具体规则+原文出处 | 抽象结构：列交叉关系 |
| 升级触发 | 主角靠什么行为升级？和子养成线什么关系？ | 写具体行为，不写"打怪升级" | 通用字段 |
| 强度膨胀 | 位阶间的实力差距多大？线性/跳跃/指数？ | 有具体对比（如LV1可以1v1街头混混，LV5可以1v15） | 通用字段 |
| 破局手段演变 | 不同位阶主角各靠什么赢？ | 至少覆盖3个位阶 | 通用字段 |
| 设计亮点 | 最值得本项目借鉴的结构性设计是什么？ | 1句话讲清楚 | 通用字段 |

**拆解要点**：
- 子养成线数量不固定——深渊主宰可能有5条（职业等级/属性点/专长/技能点/兼职），修仙书可能有4条（法力/神识/肉身/法宝），系统流可能有3条（技能树/属性点/装备）。先识别这本书有几条，再逐条拆
- 每条子养成线的规则字段不固定——职业等级拆"等级上限/进阶前置/兼职方向"，法力拆"增长方式/容量上限/恢复机制"。由这条子养成线的性质决定拆什么
- **数值锚点**：每条规则标注原文chXX出处，便于world消费时查证

#### 表9：动力引擎提取表

> 服务seed 1e 动力引擎设计 + seed 1d 力量体系设计 + seed 故事纲领。基于24本男频小说验证：动力引擎是男频小说的核心（非等级系统），决定剧情驱动力、世界辨识度和叙事节奏。

| 字段 | 说明 | 抽象要求 |
|:--|:--|:--|
| 驱动逻辑 | 为什么攀登？世界级驱动力是什么？（必须是世界级，不是个人动机） | 标注驱动力覆盖范围（全员/部分/仅主角）+来源+原文chXX |
| 运转机制 | 靠什么攀登？闭环结构（输入→转化→输出→反馈） | 标注加速器+减速器+闭环是否自洽+原文chXX |
| 代价结构 | 攀登代价是什么？代价清单+主要代价 | 标注代价是否真实可感+原文chXX |
| 范式归类 | 主范式：飞轮型/压迫型/探索型/对抗型/混合型 | 标注范式依据+是否存在演化+演化触发事件 |
| 引擎与力量体系关系 | 引擎是骨架本身/骨架外挂/与骨架平行 | 标注与世界观同构度（高/中/低）+与赛道关系 |
| 三组成合一性 | 驱动逻辑+运转机制+代价结构是否合一为"力量体系+社会结构+爽感引擎" | 完全合一/部分合一/分离+证据 |
| 辨识度评估 | 只看位阶有辨识度吗？加入引擎后变化？ | 标注反例风险 |

**拆解要点**：
- 范式判据（基于24本验证）：飞轮型=正反馈闭环+外部输入维持（29.2%）；探索型=靠发现新区域/新规则驱动（29.2%）；混合型=两种以上范式叠加（25%）；压迫型=外部压力+倒计时驱动（12.5%）；对抗型=靠与同位竞争者博弈驱动（4.2%）
- 飞轮型和探索型是并列最多的两种核心驱动力——"正反馈闭环"和"发现新区域/新规则"是男频网文最核心的两种动力来源
- 长篇普遍存在范式演化（62.5%）：触发事件通常是"主角实力提升导致旧范式不足以驱动新挑战"或"新规则被发现导致探索型增强"
- **力量体系广义化**：力量体系泛指广义的阶层金字塔（包括修仙位阶/社会地位/经济等级/术法品质/强度梯度等），不限于修仙境界。所有男频小说都有广义力量体系，历史赛道的门阀社会地位金字塔、都市赛道的经济等级都是力量体系的不同形态。动力引擎始终挂在（广义）力量体系上运转
- 引擎独特性决定辨识度，比引擎清晰度更重要——引擎独特的书即使力量体系形态非传统（如门阀社会地位）也能达到极高辨识度

## 流程·赛道定位调研

> **触发**：pipeline Phase 0 Stage 2 自动派发子agent。所有项目必有。
> **产出**：`素材/赛道调研.md`。消费方：seed 1d 双轨发散（王道赛道借鉴+避雷）+ plot 三源合流（赛道参考第一源）。

### S1 · 接收需求

输入：
- 赛道方向（来自 `素材/用户意图.md`）
- 用户已给的参考书信息（如有，需要在调研中标注避免撞车）
- 用户指定对标书（如有）

### S2 · 四轮搜索

**每轮不是"搜一下"，是搜到底（至少看前 5 条结果的摘要+内容）：**

#### 第1轮：赛道格局

```
目标：摸清这个赛道当前什么在火、什么在衰落

搜索词：
1. "{赛道} 番茄小说 排行榜 热销 2026" — 找 top 5-8 + 核心卖点
2. "{赛道} 起点 精品 万订 2026" — 补起点 top 3（如果番茄不够）

对每本 top 书提取可迁移核心卖点（1句话，不抄设定）：
| 排名 | 书名 | 核心卖点（为什么读者买单） | 可迁移方向 |
```

#### 第2轮：读者偏好

```
目标：这个赛道的读者在找什么、骂什么

搜索词：
1. "{赛道} 小说 读者 推荐 出圈 2026" — B站/抖音/知乎上的推荐+讨论
2. "{赛道} 小说 不爽 雷点 审美疲劳 2026" — 读者喷什么、避雷什么
3. "{赛道} 爽点 名场面 好评 2026" — 名场面 = 核心爽点的具体化

对每轮至少看前 5 条的内容摘要，产出借鉴点和避雷点：
- 借鉴：什么爽点/机制/节奏在读者中形成了共识好评
- 避雷：什么设定/金手指/开局模式已经被喷烂了
```

#### 第3轮：赛道演变

```
目标：这个赛道正在往哪个方向变

搜索词：
1. "{赛道} 新题材 新趋势 2026" — 冒头的新方向
2. "{赛道} 传统题材 衰落 无人问津 2026" — 已经在走下坡的老模式
3. "{赛道} 细分 垂直 蓝海 2026" — 大品类下的细分赛道

产出：细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |
```

#### 第4轮：对标分析（如有用户指定对标书）

```
目标：用户指定的对标书在赛道中的位置

对每本用户指定的对标书：
1. 搜 "{书名} 作者 风格 读者评价"
2. 判断这本书在赛道的细分位置（是哪个方向的标杆？读者的核心认可点？）
3. 确认本项目不会和它撞车（设定/金手指/开局模式）
```
