# step-decon-lite 9表拆书

> 本文件是 pop-qidian-research decon-lite 档位执行指令。进入前已由 SKILL.md 强保障注入红线。
> **触发**：pipeline Phase 0 Stage 2。用户给了力量体系参考书。
> **产出**：`素材/decon-lite-{书名}.md`。消费方：seed 1d 力量体系设计 + seed 1e 动力引擎设计 + seed 1d-0 改编策略层 + seed 故事纲领（节奏参考）。

## DL1 · 接收需求

### 1.1 必读输入

- 书名
- 本地txt路径（参考书全文）
- 采样范围（默认前 300 章，用户可指定）
- 本项目赛道方向（来自 `素材/用户意图.md`，用于判断"可迁移方向"）

### 1.2 采样策略

- 默认采样前 300 章：开篇（前10章，服务表5开篇解剖）+ 中段（取间隔章数，服务表2节奏+表3情绪）+ 关键高潮点（从目录抓章节标题判断高潮）
- 若用户给了采样范围，按用户指定
- 采样声明必须落到产出文件的元数据

## DL2 · 九表拆解

> 9张表必须全拆，不能只拆部分表（红线❌8）。每张表的字段定义见 SKILL.md "流程·decon-lite DL2"章节，本 step 不重复字段定义，只讲拆解执行要点。

### 表1：力量体系结构（规则级深度）

> 红线❌9：表1必须拆到规则级——主养成线+子养成线+chXX出处。

执行步骤：
1. 先识别力量体系类型（等级制/修炼境/规则系/百分比/职业树/...）
2. 识别主养成线（位阶）：至少覆盖前4阶，每阶标注章节范围+质变标志
3. 识别子养成线数量——先识别这本书有几条，再逐条拆规则
4. 每条子养成线规则标注原文chXX出处
5. 拆子养成线交叉规则（有没有兼职惩罚/互相制约/互相增益）
6. 升级触发/强度膨胀/破局手段演变/设计亮点

拆解陷阱：
- 不要只列阶位骨架（描述级），必须拆规则（规则级）
- 子养成线数量不固定——先识别数量再拆，不要预设"4条"
- 每条子养成线的规则字段不固定——由这条子养成线的性质决定拆什么

### 表2-5：剧情节奏/情绪弧线/人称视角/开篇解剖

执行要点：
- 表2爽点间隔必须有具体章数（不是"快/慢"）
- 表3情绪弧线每10章标注情绪高点位置
- 表5第1章第1句必须有原句摘录（不仅仅是分析）

### 表6：势力格局

执行要点：
- 先识别势力分层结构（层数由书决定）
- 每层列具名势力+主角关系
- 服务seed 1d-0改编策略层的"势力替换"维度

### 表7：金手指深度

执行要点：
- 机制/限制/代价/成长路径四项必须齐全
- "与主养成线耦合"是关键——金手指加速哪条养成线？
- 服务seed 1d-0改编策略层的"金手指形式"维度

### 表8：世界危机引擎

执行要点：
- 先识别危机分层结构（层数由书决定）
- 终极危机明确（1-2句话概括）
- 服务seed 1d-0改编策略层的"世界危机"维度

### 表9：动力引擎提取表

> 红线❌10：表9动力引擎三组成（驱动逻辑/运转机制/代价结构）必须齐全。

执行要点：
- **驱动逻辑**：必须是世界级驱动力（不是个人动机），标注覆盖范围+来源+原文chXX
- **运转机制**：闭环结构（输入→转化→输出→反馈），标注加速器+减速器+闭环是否自洽
- **代价结构**：代价清单+主要代价，标注代价是否真实可感
- 范式归类：基于5种范式（飞轮型/压迫型/探索型/对抗型/混合型）+范式依据
- 引擎与力量体系关系：骨架本身/骨架外挂/与骨架平行
- 三组成合一性：驱动逻辑+运转机制+代价结构是否合一为"力量体系+社会结构+爽感引擎"
- 辨识度评估：只看位阶有辨识度吗？加入引擎后变化？

范式判据参考（基于24本男频小说验证）：
- 飞轮型=正反馈闭环+外部输入维持（29.2%）
- 探索型=靠发现新区域/新规则驱动（29.2%）
- 混合型=两种以上范式叠加（25%）
- 压迫型=外部压力+倒计时驱动（12.5%）
- 对抗型=靠与同位竞争者博弈驱动（4.2%）

## DL3 · 质量门禁

在落盘前逐项检查：

| 检查项 | 合格标准 |
|:--|:--|
| 9张表全部完整 | 每张表字段齐全，不缺项（红线❌8） |
| 表1力量体系规则级深度 | 主养成线≥4阶 + 子养成线≥2条（每条有规则+chXX出处）+ 交叉规则有（红线❌9） |
| 表6势力格局分层 | ≥2层具名势力 + 层间矛盾有 |
| 表7金手指深度 | 机制/限制/代价/成长路径四项齐全 + 与主养成线耦合关系明确 |
| 表8世界危机引擎 | ≥2层危机 + 终极危机明确 + 驱动机制有 |
| 表9动力引擎提取表 | 三组成（驱动逻辑/运转机制/代价结构）齐全 + 范式归类有 + 合一性评估有（红线❌10） |
| 爽点间隔有章数 | 不是"快/慢" |
| 开篇第1句有原文摘录 | 不仅仅是分析，有原句 |
| 采样范围已声明 | "基于前N章采样" |

不合格项必须补拆后才能进入DL4。
