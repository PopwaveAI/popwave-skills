# step-track-research 赛道定位调研

> 本文件是 pop-qidian-research 赛道定位调研档位执行指令。进入前已由 SKILL.md 强保障注入红线。
> **触发**：pipeline Phase 0 Stage 2 自动派发子agent。所有项目必有。
> **产出**：`素材/赛道调研.md`。消费方：seed 1d 双轨发散（王道赛道借鉴+避雷）+ plot 三源合流（赛道参考第一源）。

## S1 · 接收需求

### 1.1 必读输入

- 赛道方向（来自 `素材/用户意图.md`）
- 用户已给的参考书信息（如有，需要在调研中标注避免撞车）
- 用户指定对标书（如有）

读取用户意图：

```powershell
Get-Content -Encoding UTF8 -Raw 素材/用户意图.md
```

### 1.2 调研边界

- 赛道定位调研不做深层世界观调研（那是三界框架的事）
- 不拆书（那是decon-lite的事）
- 只做赛道格局+读者偏好+赛道演变+对标分析四轮搜索

## S2 · 四轮搜索

> **每轮不是"搜一下"，是搜到底（至少看前 5 条结果的摘要+内容）**

### 第1轮：赛道格局

```
目标：摸清这个赛道当前什么在火、什么在衰落

搜索词：
1. "{赛道} 番茄小说 排行榜 热销 2026" — 找 top 5-8 + 核心卖点
2. "{赛道} 起点 精品 万订 2026" — 补起点 top 3（如果番茄不够）

对每本 top 书提取可迁移核心卖点（1句话，不抄设定）：
| 排名 | 书名 | 核心卖点（为什么读者买单） | 可迁移方向 |
```

执行要点：
- 至少找到 5 本 top 作品
- 核心卖点用1句话讲清楚"读者为什么买单"
- 可迁移方向标注"能迁移到本项目的是什么"

### 第2轮：读者偏好

```
目标：这个赛道的读者在找什么、骂什么

搜索词：
1. "{赛道} 小说 读者 推荐 出圈 2026" — B站/抖音/知乎上的推荐+讨论
2. "{赛道} 小说 不爽 雷点 审美疲劳 2026" — 读者喷什么、避雷什么
3. "{赛道} 爽点 名场面 好评 2026" — 名场面 = 核心爽点的具体化

对每轮至少看前 5 条的内容摘要，产出借鉴点和避雷点：
- 借鉴：什么爽点/机制/节奏在读者中形成了共识好评
- 避雷：什么设定/金手指/开局模式已经被喷烂了
```

执行要点：
- 每轮至少看前5条结果的内容摘要
- 借鉴点和避雷点必须有来源（书名/趋势）
- 借鉴点≥3条，避雷点≥3条

### 第3轮：赛道演变

```
目标：这个赛道正在往哪个方向变

搜索词：
1. "{赛道} 新题材 新趋势 2026" — 冒头的新方向
2. "{赛道} 传统题材 衰落 无人问津 2026" — 已经在走下坡的老模式
3. "{赛道} 细分 垂直 蓝海 2026" — 大品类下的细分赛道

产出：细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |
```

执行要点：
- 细分格局≥3个方向
- 每个方向含热度+饱和度判断
- 机会判断要具体（不是"有空间"这种废话）

### 第4轮：对标分析（如有用户指定对标书）

```
目标：用户指定的对标书在赛道中的位置

对每本用户指定的对标书：
1. 搜 "{书名} 作者 风格 读者评价"
2. 判断这本书在赛道的细分位置（是哪个方向的标杆？读者的核心认可点？）
3. 确认本项目不会和它撞车（设定/金手指/开局模式）
```

执行要点：
- 如用户无指定对标书，跳过本轮，但在产出中注明"无对标分析"
- 撞车确认要具体到设定/金手指/开局模式三层

## S3 · 产出落盘

落盘到 `素材/赛道调研.md`：

```markdown
# 赛道定位调研（赛道：{用户方向}）

> pop-qidian-research 赛道定位调研 · {timestamp}
> 消费方：seed 1d双轨发散 + plot 三源合流
> 用户意图：{引用素材/用户意图.md中的赛道+标签}
> 用户对标书：{书名}（如有）

## 一、赛道格局
### top作品核心卖点表
| 排名 | 书名 | 核心卖点 | 可迁移方向 |

### 赛道盘子
- 总体：{热度判断}
- 主要细分方向
- 正在衰落的老模式

## 二、借鉴点与避雷点
### 借鉴点（已验证的爽感机制/节奏模式/读者共识好评）
1. {机制/爽感} — 来源：{书名/趋势}
2. ...

### 避雷点（已被喷烂的模式/让读者疲劳的老套路）
1. {设定/模式} — 来源：{书名/趋势}
2. ...

## 三、细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |

## 四、对标分析（如有用户指定对标书）
| 对标书 | 细分位置 | 读者核心认可 | 本项目避免撞车策略 |

## 五、落地建议
- 力量体系方向建议：{从赛道格局推导——赛道主流是什么体系？有什么差异化空间？}
- 剧情节奏建议：{从读者偏好推导——这个赛道的爽点间隔、钩子类型偏好}
- 风险提示
```

## S4 · 质量门禁

在落盘后逐项检查：

| 检查项 | 合格标准 |
|:--|:--|
| top作品≥5本 | 有书名列卖点 |
| 借鉴点≥3条 | 每条有来源 |
| 避雷点≥3条 | 每条有来源 |
| 细分格局≥3个方向 | 含热度+饱和度判断 |
| 落地建议有具体内容 | 不是"注意差异化"这种废话 |

不合格项必须补搜后才能返回。

