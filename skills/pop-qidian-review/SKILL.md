# review

> 起点章节审核引擎。四维审核：符合性→笔触→好看度→沉淀。v3.2.0。

---

## 做什么

输入：本章施工卡（plot-design产出）+ 本章正文（write产出）+ DNA笔触 + 骨架.md + 主角设计.md + 种子文档 + 前章状态快照

| 产出文件 | 核心内容 |
|:--|:--|
| `审核/chNNN-审核报告.md` | 四维审核结论+gap清单+gap决策+好看度4问+主角变化五项+钩子追踪+红线检查+本章摘要+状态快照 |
| `current-state.md`（更新） | 下一章硬推进+人物状态+燃料队列+伏笔债务+活跃层索引 |
| `审核/小说快照.md`（replace） | 全书累计视图：进度/涌现设定/角色状态总表/剧情线/读者已知信息池/待回收伏笔 |
| `审核/review-沉淀.md`（追加） | 每章审稿判断+问题+修正规则（append-only） |

下游对接：review完成→write按决策采纳/修改/裁决；current-state供下一章write消费；小说快照供全书级状态查询。

## 怎么操作（SOP骨架）

> execution.mode: formal（落盘）/ draft（补全继续）/ trial（不落盘）。
> 强弱加载：SKILL.md=完整骨架（必读），steps/下step文件=详细操作（按Step强加载全文），templates/=模板参考（落盘时加载）。

### Step 1：符合性检查 → `steps/step-1-audit.md`
核心事件逐beat对照（gap 5类型）→骨架一致性（力量体系+动力引擎）→天赋约束（金手指不喧宾夺主）→剧情映射（血肉映射坐标系）→角色一致性→爽感闭环（三段式+三因子）。不通过=废章。

### Step 2：笔触检查 → `steps/step-1-audit.md`
AI味7项（句式重复/情绪直给/结尾收束/描写堆叠/信息重复/对话死板/高疲劳词）+ 笔触DNA 8维度一致性检查。

### Step 3：好看度检查 → `steps/step-1-audit.md`
定性4问：有没有劲/记忆点/哪里无聊/代入感。必须给具体证据，不能泛泛说"还行"。

### Step 4：剧情沉淀 → `steps/step-2-commit.md`
输出审核报告（审核-chNNN.md）+更新current-state.md（状态快照+下一章硬推进+伏笔债务+活跃层索引）+更新小说快照.md（replace模式）+追加review-沉淀.md+归档旧current-state。

## 红线

1. **读取协议**：执行任何Step前先读SKILL.md获取骨架，再按Step加载对应step文件（强加载全文）。禁止跳过SKILL.md直接读step文件。
2. **gap逐beat检查**：必须逐个beat对比施工卡与正文，跳过=偏离未发现=下游错误决策。
3. **骨架一致性不通过=废章**：越级不付代价/攀登逻辑断裂/驱动逻辑脱钩=废章打回重写。
4. **金手指取消坐标系意义=废章**：金手指喧宾夺主让坐标系失去意义=废章。金手指是加速器不是引擎本身。
5. **主角变化五项每章沉淀**：位置/能力/资产/心态/关系缺一不可，不沉淀=状态断裂，下游write丢失上下文。
6. **钩子追踪列待回收列表**：超3章未回收的钩子必须标红预警，否则伏笔债务失控。
7. **小说快照每章必更新**：review完成后必须更新`审核/小说快照.md`（replace模式），不更新=全书累计视图断裂。
