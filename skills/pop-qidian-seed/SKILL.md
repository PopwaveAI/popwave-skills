## 怎么操作（SOP骨架）

> execution.mode: formal（落盘）/ draft（补全继续）/ trial（不落盘）。用户给任何方向→立即执行Phase 0，不先问问题。
> 强弱加载：SKILL.md=完整骨架（必读），steps/下step文件=详细操作（按Phase强加载全文），decon-power library=范例参考（无参考书时弱加载按赛道索引）。

### Phase 0：七维底牌摸底 → `steps/phase0-brief.md`

七维逐项摸底（①方向②笔触参考书③力量体系参考书④赛道经典⑤现有设定⑥情绪基调⑦最大卖点），缺的标缺不阻塞。底牌就绪后触发pipeline子agent并发（dna-style/research/赛道调研）。

### Phase 1：骨架层 + 创意 + 首章

**Step 0 交互式立项决策** → `steps/step-0-interactive.md`
**v9.0.0引擎三要素重构**：基于"引擎三要素"（①定位②好坏标准③达成方法）重构S1/S2/S3。S1增量吸引力≥3+4种组合模式+好坏标准显式化。S2适配度6维判断标准+好坏标准显式化。S3从7字段升级为完整主角引擎（三定位+好坏标准+金手指5方法论+追读钩子5类型+危机关系4模式）。完整版本历史见CHANGELOG.md。

**1d 力量体系设计（坐标系）** → `steps/phase1-skeleton.md`
五层结构（源力量/主养成线/子养成线+境界涌现/交叉规则）。有参考书消费decon-lite表1；无参考书从8种类型清单选3-5种+按赛道索引选decon-power范例。

**1e 动力引擎设计（众生攀登系统）** → `steps/phase1-skeleton.md`
六组成（驱动逻辑/运转机制/众生攀登分层+掉落机制/代价三层次/范式选择/演化节点）。有参考书消费decon-lite表9。

**1f 骨架自洽检查** → `steps/phase1-skeleton.md`
10项检查（含境界涌现+掉落机制），不通过不进1g。

**1g 双轨发散** → `steps/phase1-diverge.md`
王道轨5个（赛道扫榜融合）+猎奇轨5个（跨领域碰撞），三眼法判断，PK推荐1个等用户确认。

**1h 故事纲领（三核心+营销层）** → `steps/phase1-story-brief.md`
三核心（世界4层/舞台危机3要素/主角5要素）+营销层（最大钩子≤20字/即时兑现感），同时产出创意.md。

**1i 黄金首章** → `steps/phase1-first-chapter.md`
DNA笔触，2000-2500字，体现坐标系展示+金手指激活，产出`正文/ch001.txt`。

### Phase 2：主角层

**2a 主角设计** → `steps/phase2-protagonist.md`
坐标系起点位置/身份底色/性格特质/核心欲望（对齐引擎组成1）/与众生关系（对齐组成3）+9字段行为模式表。

**2b 金手指设计** → `steps/phase2-protagonist.md`
类型/加速机制/限制（必须声明"不能取消坐标系意义"）/代价（含代价转嫁）/与引擎关系。遵循"钥匙非生成器"+"赋能不越级"原则。梗×机制×限制公式+反模式检查+喧宾夺主检查。

**2c 爽感矛盾设计** → `steps/phase2-protagonist.md`
公式（坐标系门槛×天赋加速×代价约束）+闭环模板（触发→爆发→后果）+类型清单+节奏建议。

## 红线

1. **读取协议**：执行任何Phase前先读SKILL.md获取骨架，再按Phase加载对应step文件（强加载全文）。decon-power library按需弱加载。禁止跳过SKILL.md直接读step文件。
2. **骨架先行**：骨架（力量体系+动力引擎）必须在创意发散（1g）前定稿。骨架自洽检查（1f）不通过不进1g。
3. **动力引擎世界级**：必须设计众生攀登方式分层+掉落机制，不能只设计主角攀登逻辑。引擎是世界级的，不是主角专属。
4. **金手指不喧宾夺主**：钥匙非生成器+赋能不越级。限制必须显性声明"不能取消坐标系意义"。纯翻译器/纯正向循环/无限制=不合格。
