# Phase 0 · 七维底牌摸底

> 本文件是 seed v8.0.0 的起点——替代 v7.0.0 的 Step 0 轻量燃料注入。
> 七维底牌逐项摸底，缺的标缺不阻塞。底牌就绪后触发 pipeline Phase 0 Stage 2 子 agent 并发。
> 设计依据：参考番茄 seed v13.17.0 的七维底牌设计（六维→七维加全书最大卖点），适配起点架构（起点强调力量体系+动力引擎骨架，七维中③力量体系参考书是骨架设计的核心素材源）。

---

## 摸底原则

1. **七维逐项摸底**——不跳过任何一维，缺的标缺不阻塞
2. **从用户输入提取优先**——用户给的信息直接用，不重复搜索
3. **WebSearch 补缺**——用户没给的维度，用 1 次 WebSearch 补（不做重型调研）
4. **标缺不阻塞**——七维中任何一维缺失都允许进 Phase 1，但骨架设计（1d/1e）会受影响
5. **冷启动加速**——用户给任何方向（哪怕只说"海贼王同人500章"）立即摸底，不先问用户问题

## 七维底牌摸底表

### 维度①：方向

> 用户想写什么感觉的故事——不是题材标签，是情感体验+赛道定位。

| 用户回答 | 摸底方式 | 路径 |
|:--|:--|:--|
| 直接给了创意/画面/一句话 | 从用户输入提取 | 跳过 Phase 1 创意发散，直接进 1d 骨架设计（骨架仍要做） |
| 给了方向/偏好（"想写都市异能""想写海贼同人"） | 从用户输入提取 | 进 1d 骨架设计 → 1g 双轨发散 |
| 完全开放（"你看着来"） | 标缺 | 1c 赛道调研时扫榜定方向，1g 发散时覆盖多个方向 |

**方向提取示例**：

| 用户说的 | 方向（提取后） |
|:--|:--|
| "海贼王同人500章" | 在残酷海贼世界用现代智慧建立秩序的枭雄体验 |
| "修仙但是穷鬼" | 在资源垄断的修仙世界靠精打细算逆袭的爽感 |
| "都市异能搞笑" | 现代人获得异能后日常翻车的反差喜感 |

### 维度②：笔触参考书

> 有没有觉得哪本书的写法/节奏/笔触对胃口？这决定首章笔触 DNA 来源。

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了书名 | 记录书名 → 触发 dna-style 子 agent | `素材/文风锚定.md`（笔触 DNA） |
| 没给 | 标缺 → 提醒用户"首章笔触将由你显式声明" | 无（trial 模式） |

**用途**：笔触参考书 → dna-style 提取笔触 DNA → 1i 黄金首章从第 1 句开始用 DNA 笔触写。

### 维度③：力量体系参考书（起点架构核心）

> 有没有哪本书的力量体系/升级节奏可以参考？这是起点架构的核心——骨架设计（1d 力量体系+1e 动力引擎）的核心素材源。

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了书名 | 记录书名 → 触发 decon-lite 子 agent | `素材/decon-lite-{书名}.md`（9 表拆解） |
| 没给 | 标缺 → 骨架设计从赛道惯例+用户意图推导 | 无（1d 三源合流时缺第二源） |

**用途**：力量体系参考书 → decon-lite 9 表拆解 → 1d 力量体系设计（表 1 力量体系结构）+ 1e 动力引擎设计（表 9 动力引擎提取表）。

**起点架构适配说明**：番茄 seed 的②笔触参考书和③力量体系参考书是分开的两维（番茄不强调骨架前移）。起点 seed v8.0.0 保留这两维分开，因为③直接喂骨架设计——这是起点架构与番茄架构的关键差异。

### 维度④：赛道经典

> 同赛道有没有标杆书想避开或借鉴？

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了书名 | 记录到 `素材/市场校准.md` 头部（"用户指定对标书：{书名}"） | 市场校准头部追加 |
| 没给 | 标缺 → 1c 赛道调研时从排行榜找 | 无（赛道调研时补） |

**用途**：赛道经典 → 1g 王道轨扫榜时的对标参考 + 创意校准（不撞车）。

### 维度⑤：现有设定

> 你心里已经有想好的力量体系/世界规则/主角设定吗？

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了设定 | 记录到 `设计/创意.md`（"用户预设设定"章节），1d 骨架设计直接继承 | 创意.md 新增章节 |
| 没给 | 标缺 → 1d 骨架设计从零推导 | 无 |

**用途**：现有设定 → 1d 力量体系设计直接继承用户预设，不重设计。

### 维度⑥：情绪基调

> 你想写什么情绪基调的故事？

情绪基调清单（不限于）：
- 热血逆袭（弱→强，打脸翻盘）
- 暗黑复仇（被背叛→复仇，冷酷烧脑）
- 悬疑烧脑（解谜推理，反转不断）
- 都市恐怖（民俗/规则恐怖，细思极恐）
- 情感治愈（温情救赎，以情动人）
- 荒诞轻喜剧（反差幽默，荒谬对话）
- 末世求生（资源管理，生存压力）
- 权谋博弈（势力操盘，智斗）

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了偏好 | 记录到 `设计/创意.md`，1g 发散时用 | 创意.md 新增章节 |
| 没给 | 标缺 → 1g 发散时根据赛道推荐 | 无（1g 时补） |

**用途**：情绪基调 → 1g 双轨发散时约束创意的情绪方向 + 1h 故事纲领的"什么样的主角"维度参考。

### 维度⑦：全书最大卖点

> 这本书靠什么吸引读者翻页？

卖点类型清单：
- **故事驱动型**：恐怖烧脑的故事/剧情反转/悬疑揭秘——读者为故事翻页
- **设定驱动型**：有趣精彩的设定/力量体系精巧/世界观宏大——读者为设定翻页
- **处置驱动型**：主角如何处置诡/敌人/问题——处置方式本身是爽点（封印/谈判/利用/转化/共生/驱使，每个问题不同处置）

| 用户回答 | 摸底方式 | 产出 |
|:--|:--|:--|
| 给了卖点类型 | 记录到 `设计/创意.md`，约束 1g 发散方向 | 创意.md 新增章节 |
| 没给 | 标缺 → 1g 发散时根据赛道推荐 | 无（1g 时补） |

**用途**：全书最大卖点 → 约束 1g 发散方向（处置驱动型→发散必须回答"怎么处置"，不能逃避到"吞噬变强"）+ 1h 故事纲领的"最大钩子"维度参考。

## 底牌就绪后：触发 pipeline Phase 0 Stage 2 子 agent 并发

> **底牌就绪后，七维中"有但未就绪"的任务全部并发执行，不等串行。**

根据七维摸底结果，组装并发任务列表。以下三路任务互不依赖，可以同时派发：

| 编号 | 任务 | 触发条件 | 子 agent 执行什么 |
|:--|:--|:--|:--|
| ① 笔触 DNA | 用户给了笔触参考书 + 未就绪 | `你扮演 pop-dna-style，读取 skills/pop-dna-style/SKILL.md 了解你的 SOP。参考书：{书名}，本地路径：素材/downloads/{书名}.txt。提取笔触 DNA → 落盘到素材/文风锚定.md。返回：落盘路径+版本+采样章数。` |
| ② 力量体系(decon-lite) | 用户给了力量体系参考书 + 未就绪 | `你扮演 pop-research decon-lite 档位，读取 skills/pop-research/SKILL.md 了解 decon-lite SOP。参考书：{书名}，本地路径：素材/downloads/{书名}.txt。拆 9 表（力量体系结构/剧情节奏/情绪弧线/人称视角/开篇解剖/势力格局/金手指深度/世界危机引擎/动力引擎提取表）→ 落盘到素材/decon-lite-{书名}.md。返回：落盘路径+9 表摘要。注意：表 1 必须拆到规则级（主养成线+子养成线+交叉规则），不能只列阶位骨架；表 9 必须拆到动力引擎六组成。` |
| ③ 赛道调研 | 必有（所有项目都需要） | `你扮演 pop-qidian-seed 的赛道调研，读取 skills/pop-qidian-seed/steps/phase0-brief.md 了解赛道调研 SOP。赛道：{用户方向}。执行 3 轮 WebSearch（赛道热门钩子/跨赛道机制灵感/现实矛盾点）→ 产出借鉴点/避雷点清单 → 落盘到素材/市场校准.md。返回：落盘路径+借鉴点/避雷点摘要。注意：用户指定了对标书{书名}（如有），需在校准清单头部标注。` |

**并发派发规则**：
- ①②③互不依赖——三个子 agent 可以同时启动
- 三路全部返回后，主 agent 继续 1d 骨架设计

**缺失跳过**：用户没给笔触参考书→跳过①。用户没给力量体系参考书→跳过②。全部返回（含跳过）后进入 1d 骨架设计。

**子 agent 描述原则**：
- 告诉子 agent"你扮演哪个 skill 的哪个档位"
- 让它自己读对应的 SKILL.md——不把 SOP 内容塞进 prompt
- 给具体参数（书名/路径/赛道），不给完整 SOP

## 赛道调研 SOP（③号任务的详细方法）

> 如③号任务由子 agent 执行，等待返回后直接消费落盘文件。如未由子 agent 执行，则完整执行以下步骤。

### 1. 搜索赛道热门钩子（燃料 A）

用 WebSearch 搜索用户赛道的当前热门作品和钩子模式：

```
搜索词：{用户题材方向} 起点 热门 推荐 2025 2026
```

从搜索结果摘要中提取：
- 当前赛道什么书火？书名+一句话卖点
- 这些书凭什么钩子抓读者？前3章发生了什么？
- 常见金手指类型有哪些？
- 常见主角原型有哪些？

产出 1-2 条燃料条目。不需要下载全文，搜索摘要够用。

**示例**（用户说"海贼王同人"）：
```
燃料A-1：海贼同人热门钩子
- 《海贼法典》88万字太监：写实风枭雄种田，普通白领穿越罗杰死后东海
- 《海贼之黑伯爵》193万字完结：死灵法师在海贼世界，亡灵天灾流
- 常见钩子：穿越者利用剧情预知卡位 / 枭雄经营势力 / 体系流战斗
- 常见金手指：命运之线/异空间/特殊果实/系统面板
- 常见主角原型：实用主义者/枭雄/反英雄
```

### 2. 搜索跨赛道机制灵感（燃料 B）

用 WebSearch 搜索**不限赛道**的有趣机制——目标是从其他赛道"借"一个机制，制造陌生感：

```
搜索词：小说 最有趣的系统设定 金手指 机制 2025
或：小说 奇葩能力 有趣设定 推荐
```

从搜索结果中提取 1-2 个有趣的机制/能力设计，标注它来自什么赛道、什么书。

**关键**：这个燃料必须来自与用户赛道**不同**的领域。如果用户写海贼同人，这里找修仙/都市/游戏/科幻的机制。跨领域才能制造碰撞张力。

**示例**：
```
燃料B-1：跨赛道机制——记忆消耗型鉴定（来自修仙赛道《玄鉴仙族》）
- 机制：鉴定一切物品真伪，但每次消耗自身一段记忆
- 张力点：最需要信息差的人，获取信息差的代价是失去记忆
- 可碰撞方向：海贼世界中，鉴定恶魔果实真伪/历史正文含义，代价是遗忘同伴

燃料B-2：跨赛道机制——规则制定者系统（来自策略游戏赛道）
- 机制：主角定义一套行为规则，遵守获加成，违反扣属性
- 张力点：规则可以修订但修订有代价
- 可碰撞方向：海贼世界中制定自己的海盗法典，约束自己也约束部下
```

### 3. 搜索现实矛盾点（燃料 C）

用 WebSearch 搜索与用户题材形成反差或互补的**真实事件/社会现象**——目标是提供现实质感和决策困境：

```
搜索词：{与题材相关的现实关键词} 真实事件 社会现象
```

搜索方向举例：
- 海贼题材 → 搜索"现代海盗 索马里 真实" / "海上法权 海洋法律 现实"
- 修仙题材 → 搜索"古代修炼 真实历史 道教炼丹" / "阶级固化 古代科举"
- 都市题材 → 搜索"职场PUA 真实案例" / "贫富差距 现实数据"

产出 1 条燃料条目，标注现实矛盾的核心张力。

**示例**：
```
燃料C-1：现实矛盾——索马里海盗的根源
- 现实：索马里海盗源于外国渔船滥捕+政府无力管辖，渔民被迫武装
- 矛盾点："海盗"在特定语境下是弱者的生存策略，不是纯粹的恶
- 可碰撞方向：海贼世界中主角面对"是当海盗还是等死"的选择，灰色地带
```

## 降级策略

| 环境 | 动作 |
|:--|:--|
| 可搜索 | 执行 3 次 search，提取燃料 |
| 搜索失败/无网络 | 用 agent 内部知识生成 3 类燃料（标注 quality:low），提醒用户燃料质量受限 |
| 用户提供了参考书单 | 跳过 search A（赛道热门），直接从用户书单提取燃料 A；仍执行 B 和 C |

## 门禁（进 1d 骨架设计前检查）

- [ ] 七维底牌逐项摸底（有/标缺，不跳过任何一维）
- [ ] 如有笔触参考书 → dna-style 已触发或就绪
- [ ] 如有力量体系参考书 → decon-lite 已触发或就绪
- [ ] 赛道调研已执行（3 类燃料各≥1 条，或标缺降级生成）
- [ ] 燃料 B（跨赛道机制）来自与用户赛道不同的领域（如已执行赛道调研）
- [ ] 每条燃料标注了"可碰撞方向"（不是照搬，是改造融合的入口）

任一不满足：降级生成或提醒用户手动提供参考方向。**标缺不阻塞**——七维任何一维缺失都允许进 1d，但骨架设计会受影响。
