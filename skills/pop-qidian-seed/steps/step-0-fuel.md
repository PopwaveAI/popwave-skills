# Step 0：轻量燃料注入

> 触发：用户带任何方向/想法来，立即执行。不先问用户问题。
> 目标：3次web search收集3类燃料，为Step 1跨领域碰撞提供创意基因。5分钟内完成。
> 核心原则：燃料多样性 > 燃料完整性。3条跨领域燃料 > 30条同赛道燃料。

## 为什么不下载10本书

v6.0.0的燃料收集要求"搜索10本热门书 → 下载前20章 → 5维度提取 → 汇总基因库"。实测5轮从未执行。原因：操作量太重（10本书×20章=200章文本处理），agent直接跳过。

v7.0.0改为3次轻量web search，每次搜索1类燃料，只取搜索结果摘要中的关键信息，不下载全文。

## 执行步骤

### 1. 搜索赛道热门钩子（燃料A）

用WebSearch搜索用户赛道的当前热门作品和钩子模式：

```
搜索词：{用户题材方向} 起点 热门 推荐 2025 2026
```

从搜索结果摘要中提取：
- 当前赛道什么书火？书名+一句话卖点
- 这些书凭什么钩子抓读者？前3章发生了什么？
- 常见金手指类型有哪些？
- 常见主角原型有哪些？

产出1-2条燃料条目。不需要下载全文，搜索摘要够用。

**示例**（用户说"海贼王同人"）：
```
燃料A-1：海贼同人热门钩子
- 《海贼法典》88万字太监：写实风枭雄种田，普通白领穿越罗杰死后东海
- 《海贼之黑伯爵》193万字完结：死灵法师在海贼世界，亡灵天灾流
- 常见钩子：穿越者利用剧情预知卡位 / 枭雄经营势力 / 体系流战斗
- 常见金手指：命运之线/异空间/特殊果实/系统面板
- 常见主角原型：实用主义者/枭雄/反英雄
```

### 2. 搜索跨赛道机制灵感（燃料B）

用WebSearch搜索**不限赛道**的有趣机制——目标是从其他赛道"借"一个机制，制造陌生感：

```
搜索词：小说 最有趣的系统设定 金手指 机制 2025
或：小说 奇葩能力 有趣设定 推荐
```

从搜索结果中提取1-2个有趣的机制/能力设计，标注它来自什么赛道、什么书。

**关键**：这个燃料必须来自与用户赛道**不同**的领域。如果用户写海贼同人，这里找修仙/都市/游戏/科幻的机制。跨领域才能制造碰撞张力。

**示例**：
```
燃料B-1：跨赛道机制——记忆消耗型鉴定（来自修仙赛道《玄鉴仙族》）
- 机制：鉴定一切物品真伪，但每次消耗自身一段记忆
- 张力点：最需要信息差的人，获取信息差的代价是失去记忆
- 可碰撞方向：海贼世界中，鉴定恶魔果实真伪/历史正文含义，代价是遗忘同伴

燃料B-2：跨赛道机制——规则制定者系统（来自策略游戏赛道）
- 机制：主角定义一套行为规则，遵守获加成，违反扣属性
- 张力点：规则可以修订但修订有代价
- 可碰撞方向：海贼世界中制定自己的海盗法典，约束自己也约束部下
```

### 3. 搜索现实矛盾点（燃料C）

用WebSearch搜索与用户题材形成反差或互补的**真实事件/社会现象**——目标是提供现实质感和决策困境：

```
搜索词：{与题材相关的现实关键词} 真实事件 社会现象
```

搜索方向举例：
- 海贼题材 → 搜索"现代海盗 索马里 真实" / "海上法权 海洋法律 现实"
- 修仙题材 → 搜索"古代修炼 真实历史 道教炼丹" / "阶级固化 古代科举"
- 都市题材 → 搜索"职场PUA 真实案例" / "贫富差距 现实数据"

产出1条燃料条目，标注现实矛盾的核心张力。

**示例**：
```
燃料C-1：现实矛盾——索马里海盗的根源
- 现实：索马里海盗源于外国渔船滥捕+政府无力管辖，渔民被迫武装
- 矛盾点："海盗"在特定语境下是弱者的生存策略，不是纯粹的恶
- 可碰撞方向：海贼世界中主角面对"是当海盗还是等死"的选择，灰色地带
```

## 降级策略

| 环境 | 动作 |
|------|------|
| 可搜索 | 执行3次search，提取燃料 |
| 搜索失败/无网络 | 用agent内部知识生成3类燃料（标注quality:low），提醒用户燃料质量受限 |
| 用户提供了参考书单 | 跳过search A（赛道热门），直接从用户书单提取燃料A；仍执行B和C |

## 门禁（进Step 1前检查）

- [ ] 有≥3条燃料条目（赛道钩子/跨赛道机制/现实矛盾各≥1条）
- [ ] 燃料B（跨赛道机制）来自与用户赛道不同的领域
- [ ] 每条燃料标注了"可碰撞方向"（不是照搬，是改造融合的入口）

任一不满足：降级生成或提醒用户手动提供参考方向。
