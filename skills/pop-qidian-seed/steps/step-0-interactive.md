# Step 0 - 交互式立项决策

> **v8.5.0重构**：故事先行——S1先对齐"写什么故事"，S2再配套力量体系。
> **核心原则**：给选项不给空白——每轮提供2-3个结构化选项+1个自定义入口。
> **交互格式**：每轮Agent输出选项→暂停等待用户选择→确认后进入下一轮。
> **设计哲学**：用户先有的是"故事创意"而不是"力量体系"。力量体系是配套故事的——故事决定需要什么力量体系，不是反过来。

## 前置条件

- Phase 0-Stage1深问完成（用户意图.md已有赛道/标签/参考书/现有设定）
- 拆书可并行进行（decon-lite/decon-plot结果可实时喂给后续轮次选项）

## 产出

`设计/立项决策表.md`——后续骨架生成+创意+主角设计的硬输入

## 5轮交互决策

### S1 故事创意对齐（核心轮·必答·第一轮）

> **v8.5.0调整**：从"力量体系选择"改为"故事创意对齐"。
> 用户视角先有的是"要写什么故事"，不是"用什么力量体系"。

**底牌消费**：用户意图.md（赛道+标签+参考书+现有设定）+ 赛道调研.md（读者偏好+对标分析）

**Agent动作**：
1. 基于用户意图中的赛道和标签，提出3个故事方向方案
2. 每个方案标注：故事类型/核心冲突/主角定位/爽感基调/参考书/与赛道主流的差异化

**选项生成规则**：
- 方案A：赛道主流方向（安全牌，读者基础好但可能同质化）
- 方案B：差异化方向（同赛道但走冷门路线，蓝海但有风险）
- 方案C：跨赛道融合方向（赛道A的设定+赛道B的叙事方式）

**输出格式**：
```
## S1 故事创意对齐

基于你的赛道（{赛道}）和标签（{标签}），提供3个故事方向：

A) {方案A名}
   → 故事类型：{类型}
   → 核心冲突：{一句话冲突}
   → 主角定位：{主角是什么人，要做什么}
   → 爽感基调：{打脸/升级/谋略/探索}
   → 参考书：{书名}
   → 差异化：{与赛道主流的差异}

B) {方案B名}
   → ...

C) {方案C名}
   → ...

请选择（A/B/C），或描述你想要的故事方向。
```

**影响标注**：故事方向定下后，S2力量体系配套/S3金手指/S4主角性格都基于此。改故事方向=重做S1-S5。

### S2 力量体系配套（核心轮·必答·基于S1故事方向）

> **v8.5.0调整**：从"第一轮"后移到"第二轮"，且从"选体系"改为"为故事配套体系"。

**底牌消费**：S1选定的故事方向 + decon-lite表1（力量体系）+表9（动力引擎）+ decon-plot力量体系对比（如有，拆书结果可实时喂入）

**Agent动作**：
1. 基于S1的故事类型和核心冲突，提出3个力量体系方案
2. 每个方案标注：主养成线（几阶？什么名称？）+ 子养成线（几条？）+ 引擎范式（探索/压迫/混合）+ 与故事的适配度

**选项生成规则**：
- 方案A：最适配故事的力量体系（故事冲突→体系特性匹配分析）
- 方案B：变体体系（同类型但有差异化设计）
- 方案C：跨类型体系（用非传统体系支撑这个故事）

**输出格式**：
```
## S2 力量体系配套

基于S1选定的故事方向（{故事方向}），这个故事需要什么力量体系？

A) {方案A名}
   → 主养成线：{阶位数+名称}
   → 子养成线：{条数+名称}
   → 引擎范式：{探索/压迫/混合}
   → 与故事的适配度：{为什么这个体系适合这个故事}

B) {方案B名}
   → ...

C) {方案C名}
   → ...

请选择（A/B/C），或描述你想要的力量体系。
```

**影响标注**：力量体系定下后，后续所有Phase都在此体系内生成。骨架.md基于此生成。

### S3 金手指设计方向（核心轮·必答）

**底牌消费**：S1故事方向 + S2力量体系 + decon-lite表7（金手指）+ decon-plot金手指对比（如有）

**Agent动作**：
1. 基于故事方向和力量体系，提出3个金手指方案
2. 每个方案标注：类型/加速机制/限制/代价/与故事冲突的关系

**选项生成规则**：
- 方案A：复合型金手指（2-3件套，加速强但限制多）
- 方案B：单件型金手指（1件，加速适中但限制少）
- 方案C：天赋型金手指（无外部系统，靠先天优势）

### S4 主角性格底色（核心轮·必答）

**底牌消费**：S1故事方向 + S3金手指 + 用户意图.md（风格关键词）+ 赛道调研.md（读者偏好）

**Agent动作**：
1. 基于故事方向和金手指，提出3个主角性格方案
2. 每个方案标注：核心欲望/恐惧/底线/行动偏好/长篇穿透方式+参考人物

**选项生成规则**：
- 方案A：与故事方向最匹配的性格（如复仇故事→冷硬理性型）
- 方案B：反差型性格（同故事但主角性格出人意料）
- 方案C：成长型性格（性格在故事中发生显著变化）

### S5 主角弧光+爽感权重（可选轮·可跳过）

**底牌消费**：S4主角性格 + 骨架.md（已基于S1-S2生成）+ 主角设计模板

**Agent动作**：
1. 基于性格底色，提出3个弧光+爽感权重组合方案
2. 每个方案标注：第一卷弧光变化/情感锚点/坐标系门槛权重/天赋加速权重/代价约束权重

**跳过规则**：用户说"跳过"或"默认"→ agent按默认权重（4/3/3.5）+标准弧光生成

## 红线

1. **每轮必须给选项**——不能开放式提问，必须给2-3个结构化选项
2. **每轮必须有自定义入口**——选项末尾必须有"或描述你想要的XXX"
3. **S1故事先行**——力量体系（S2）为故事（S1）配套，不是故事为力量体系服务
4. **核心轮不可跳过**——S1-S4必答，S5可跳过
5. **决策表是骨架+创意+主角设计的硬输入**——后续生成必须遵守
