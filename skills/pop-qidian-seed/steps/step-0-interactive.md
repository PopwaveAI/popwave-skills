# Step 0 - 交互式立项决策

> **v8.4.0新增**：在骨架生成之前，通过5轮交互式决策采集用户偏好，产出`立项决策表.md`。
> **核心原则**：给选项不给空白——每轮提供2-3个结构化选项+1个自定义入口。
> **交互格式**：每轮Agent输出选项→暂停等待用户选择→确认后进入下一轮。
> **杠杆位置**：seed是最有杠杆的Phase——力量体系/金手指/主角性格定下后，后续world/character/plot都在此框架内生成。

## 前置条件

- Phase 0完成
- 用户意图.md + research产出（如有）齐全

## 产出

`设计/立项决策表.md`——后续骨架生成+创意+主角设计的硬输入

## 5轮交互决策

### S1 力量体系+引擎范式（核心轮·必答）

**底牌消费**：用户意图.md（赛道类型）+ research-写作燃料.md（事件形状）+ decon-lite表1+表9 + decon-plot力量体系对比（如有）

**Agent动作**：
1. 基于赛道类型，提出3个力量体系方案
2. 每个方案标注：主养成线（几阶？什么名称？）+ 子养成线（几条？）+ 引擎范式（探索/压迫/混合）+ 参考书

**选项生成规则**：
- 方案A：赛道主流体系（如DND西幻→阶位制面板流）
- 方案B：跨赛道体系（如DND西幻但用修仙境界）
- 方案C：创新体系（如DND西幻但用游戏等级制）

**输出格式**：
```
## S1 力量体系+引擎范式

基于你的赛道（{赛道}），提供3个力量体系方案：

A) {方案A名}
   → 主养成线：{阶位数+名称}
   → 子养成线：{条数+名称}
   → 引擎范式：{探索/压迫/混合}
   → 参考书：{书名}

B) {方案B名}
   → ...

C) {方案C名}
   → ...

请选择（A/B/C），或描述你想要的力量体系。
```

**影响标注**：力量体系定下后，后续所有Phase都在此体系内生成。改力量体系=重做Phase 1-4。

### S2 金手指设计方向（核心轮·必答）

**底牌消费**：S1选择的力量体系 + decon-lite表7（金手指）+ decon-plot金手指对比（如有）

**Agent动作**：
1. 基于力量体系，提出3个金手指方案
2. 每个方案标注：类型/加速机制/限制/代价/与引擎的关系

**选项生成规则**：
- 方案A：复合型金手指（2-3件套，加速强但限制多）
- 方案B：单件型金手指（1件，加速适中但限制少）
- 方案C：天赋型金手指（无外部系统，靠先天优势）

### S3 主角性格底色（核心轮·必答）

**底牌消费**：S2选择的金手指 + 用户意图.md（风格关键词）+ 赛道调研.md（读者偏好）

**Agent动作**：
1. 提出3个主角性格方案
2. 每个方案标注：核心欲望/恐惧/底线/行动偏好/长篇穿透方式+参考人物

**选项生成规则**：
- 方案A：冷硬理性型（务实/功利/底线明确）
- 方案B：热血感性型（正义/重情/越战越强）
- 方案C：腹黑谋略型（控制/布局/借刀杀人）

### S4 主角弧光细化（核心轮·必答）

**底牌消费**：S3选择的主角性格 + 骨架.md（已基于S1-S2生成）

**Agent动作**：
1. 基于性格底色，提出3个核心欲望的具体表达方案
2. 每个方案标注：第一卷角色弧光变化+情感锚点

**选项生成规则**：
- 方案A：孤狼路线（独行→学会合作）
- 方案B：结盟路线（利用→真信任）
- 方案C：隐忍路线（隐藏→被迫暴露→学会在暴露中收益）

### S5 爽感公式权重（可选轮·可跳过）

**底牌消费**：S2金手指 + S4主角弧光 + 主角设计模板

**Agent动作**：
1. 提出3个爽感权重分配方案
2. 每个方案标注：坐标系门槛/天赋加速/代价约束的权重+节奏建议

**跳过规则**：用户说"跳过"或"默认"→ agent按默认权重（4/3/3.5）生成

## 红线

1. **每轮必须给选项**——不能开放式提问，必须给2-3个结构化选项
2. **每轮必须有自定义入口**——选项末尾必须有"或描述你想要的XXX"
3. **选项必须在赛道约束内**——不可行的方案不会出现在选项中
4. **核心轮不可跳过**——S1-S4必答，S5可跳过
5. **决策表是骨架+创意+主角设计的硬输入**——后续生成必须遵守
