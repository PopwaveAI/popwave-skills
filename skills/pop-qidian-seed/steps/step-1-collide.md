# Step 1：张力驱动碰撞

> 触发：Step 0燃料就绪后立即执行。
> 目标：用跨领域燃料碰撞产生有张力的候选概念，三眼法筛选，PK后推荐1个，等用户确认。
> 核心原则：碰撞的本质是寻找结构性矛盾，不是排列组合。好的碰撞 = 组合本身就能产生决策困境。

## 为什么旧公式产出换皮

v6.0.0的碰撞公式：`题材 × 主角原型 × 金手指类型 × 世界压迫`。所有元素来自同一赛道池子 → 产出必然"不同身份同一结构"。实测Run 4产出的3个候选（奴隶/海军/酒馆老板）被agent自己判定为换皮。

v7.0.0改为跨领域张力碰撞 + 四象限散布，强制引入赛道外元素制造陌生感。

---

## 执行步骤

### 1. 提取用户核心欲望

从用户输入中提取"想写什么感觉的故事"——不是题材标签，是情感体验。

| 用户说的 | 核心欲望（提取后） |
|---------|-----------------|
| "海贼王同人500章" | 在残酷海贼世界用现代智慧建立秩序的枭雄体验 |
| "修仙但是穷鬼" | 在资源垄断的修仙世界靠精打细算逆袭的爽感 |
| "都市异能搞笑" | 现代人获得异能后日常翻车的反差喜感 |

如果用户输入足够具体（如"我要写一个海贼法典风格的同人"），直接提取。如果太模糊（如"海贼王同人"），从燃料A的热门钩子中反推用户可能想要的方向，在碰撞时覆盖多个方向。

### 2. 跨领域张力碰撞

读取Step 0的3类燃料，执行碰撞公式：

```
用户核心欲望
  × 燃料A（赛道内热门机制，改造非照搬）
  × 燃料B（跨赛道/跨领域元素，制造陌生感）
  × 燃料C（现实矛盾点，提供决策困境）
  → 张力测试：组合本身是否产生结构性矛盾？
```

**张力测试标准**：组合本身就产生决策困境或矛盾。不是"主角获得能力后变强"，而是"主角获得能力的同时也获得了一个两难选择"。

**碰撞方法**：
1. 从3类燃料中各选1个元素
2. 与用户核心欲望碰撞
3. 检查是否产生天然张力——组合本身就矛盾
4. 天然张力的标志：读者看完概念能立刻想到"那主角会怎么选？"

**碰撞示例**（用户：海贼王同人，燃料A=枭雄种田/系统面板，燃料B=记忆消耗型鉴定，燃料C=索马里渔民被迫成海盗）：
```
碰撞1：实用主义穿越者 × 记忆消耗鉴定 × 渔民被迫武装
→ 主角穿越成东海渔民，获得鉴定能力（能鉴定果实真伪/人物忠诚度）
→ 代价：每次鉴定消耗一段记忆
→ 张力：他需要信息差生存，但获取信息差的代价是遗忘——遗忘同伴、遗忘前世剧情
→ 决策困境：用能力保命但慢慢忘记一切，还是不用能力在残酷世界裸奔？
→ 读者会问："他什么时候会为了记住一个人而放弃鉴定？"
```

```
碰撞2：枭雄经营 × 规则制定者系统 × 海盗是弱者生存策略
→ 主角获得"法典系统"——自己定义规则，遵守获加成，违反扣属性
→ 张力：他可以定义"不杀平民"，但东海的现实是——不杀人就被人杀
→ 决策困境：坚持法典但实力增长慢可能被灭，还是打破法典快速变强但永远弱一截？
→ 读者会问："他会为了活命修订自己的法典吗？修订后他还是原来的他吗？"
```

### 3. 四象限散布

对每个碰撞结果，用四象限散布扩大创意空间——想象主角采取保守或激进行为，搭配最好或最坏结果：

|  | 保守行为 | 激进行为 |
|--|---------|---------|
| **最好结果** | 主角谨慎经营，法典成为铁律，势力稳步扩张 | 主角以战养战，法典随实力修订，成为东海最强 |
| **最坏结果** | 主角过于保守，被激进对手碾压，法典成空谈 | 主角为快速变强不断修订法典，最终法典面目全非，失去自我 |

从4个变体中选**最有故事张力**的1个作为候选方向。不是选"最好的结果"——是选"最能产生冲突和追读欲望的"。

通常最有张力的是"激进行为+最坏结果"或"保守行为+最好结果"——前者是悲剧英雄的弧线，后者是忍辱负重的弧线。

### 4. 生成3个候选

执行3轮碰撞（每轮选不同的燃料组合），产出3个候选。每个候选填全：

| 候选 | 一句话卖点 | 核心爽感机制 | 主角驱动 | 使用的燃料(改造方式) | 最容易写崩的点 |
|-----|---------|------------|---------|-------------------|-------------|

字段要求：
- **一句话卖点**：主角靠什么主动，一句话说清
- **核心爽感机制**：读者追读的具体驱动力（不是"变强"，是具体的爽感机制——信息差碾压/规则博弈/经营扩张/以弱胜强）
- **主角驱动**：主角自己想干嘛、怎么捅破现状，不能只写世界很惨
- **使用的燃料**：标注从哪个燃料提取了什么元素 + 怎么改造融合
- **最容易写崩的点**：这个方向最大的风险

**候选差异化要求**：3个候选必须在**核心爽感机制**上实质不同。不是开局身份不同（奴隶/海军/酒馆老板=换皮），是读者追读的根本驱动力不同：
- 候选A爽感=信息差碾压 → 候选B爽感=规则博弈 → 候选C爽感=以弱胜强 ✓
- 候选A爽感=信息差碾压 → 候选B爽感=信息差碾压(但主角是海军) → 候选C爽感=信息差碾压(但主角是奴隶) ✗ 换皮

### 5. 三眼法判断

对每个候选执行三眼法——3项全过才进入PK：

| 眼 | 问题 | 通过标准 | 不通过=淘汰 |
|----|------|---------|------------|
| 第一眼：画面 | 读完概念，脑子里是否立刻浮现一个具体场景？ | 能描述出1个具体画面（谁在哪做什么，有冲突） | 只浮现"又是一本XX文" |
| 第二眼：限制 | 主角/金手指是否有有意义的约束？ | 有明确的代价/限制/博弈条件 | 无限变强无代价 |
| 第三眼：场景 | 这个设定能跑10个不同类型的场景吗？ | 能列出10个不同的冲突场景 | 3个场景后就不知道写什么 |

**三眼法执行方式**：对每个候选，逐眼判断。任何一眼不通过→淘汰该候选，回到Step 2重新碰撞补一个。

**三眼法示例**（候选：记忆消耗型鉴定）：
```
第一眼-画面：主角站在果实堆前，手里攥着一颗恶魔果实。他知道鉴定就能知道真伪——但上一次鉴定让他忘记了前世女友的脸。他还会鉴定吗？→ 通过（有具体画面+冲突）
第二眼-限制：鉴定消耗记忆，记忆不可逆。主角必须在"信息"和"记忆"间做取舍。→ 通过（有明确代价）
第三眼-场景：①鉴定果实真伪 ②鉴定NPC忠诚度 ③鉴定历史正文含义 ④鉴定武器品质 ⑤鉴定悬赏令真假 ⑥鉴定霸气类型 ⑦鉴定隐藏能力 ⑧鉴定势力底牌 ⑨鉴定地图真伪 ⑩鉴定自己被篡改的记忆 → 通过（10个不同场景）
→ 三眼全过，进入PK
```

### 6. 候选PK + 推荐

对三眼法通过的候选逐项对比：

| 维度 | 候选A | 候选B | 候选C |
|------|-------|-------|-------|
| 核心爽感机制 | | | |
| 主角主动性 | | | |
| 长线续航(250章+) | | | |
| 独特性(同赛道100本) | | | |
| 执行难度 | | | |
| 张力强度 | | | |

推荐1个 + 理由 + 吸收败者强点 + 淘汰项。

**推荐方案必须满足**：
- 主卖点一句话说清且主角主动方式明确
- 不把"世界压迫"当主角驱动
- 核心爽感机制与同赛道主流作品实质不同

## 金手指设计公式

候选确定后，用以下公式设计金手指：

```
金手指 = 梗 × 机制 × 限制
```

| 要素 | 作用 | 来源 |
|------|------|------|
| 梗 | 提供形式——金手指"长什么样" | 燃料B(跨赛道机制)或用户输入 |
| 机制 | 提供动力——金手指"怎么运转" | 碰撞结果中的核心爽感机制 |
| 限制 | 提供张力——金手指"为什么不能随便用" | 燃料C(现实矛盾)或碰撞中的决策困境 |

**示例**：
- 梗=鉴定能力（来自修仙赛道） × 机制=鉴定一切物品真伪（信息差碾压） × 限制=每次消耗记忆（现实矛盾：信息获取有代价）
- 梗=规则制定者系统（来自策略游戏） × 机制=定义规则遵守获加成（规则博弈） × 限制=规则修订有代价（现实矛盾：原则妥协有后果）

**反模式检查**（4种必须避免）：
- 纯翻译器（只翻译信息，不产生决策）
- 纯正向循环（用了只变强，没有代价）
- 无使用场景限制（随时随地都能用）
- 成长速度过慢（读者等不到爽点）

## 门禁（进Step 2前检查）

- [ ] 有≥3个候选且填全表格
- [ ] 每个候选从不同类型燃料碰撞产生（不是同赛道排列组合）
- [ ] 3个候选的核心爽感机制实质不同
- [ ] 每个候选通过三眼法（画面/限制/场景）
- [ ] 推荐方案的主卖点一句话说清且主角主动方式明确
- [ ] 推荐方案不靠"世界压迫"冒充主角驱动
- [ ] 金手指通过梗×机制×限制公式+反模式检查
- [ ] 用户已确认推荐方案（或指定调整后方案）

任一不满足：回到碰撞生成，不进Step 2。
