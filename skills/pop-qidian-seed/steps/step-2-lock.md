# Step 2：锁定种子文档

> 触发：Step 1推荐方案经用户确认后进入。
> 目标：填充7维模板 → 种子压力测试 → 落盘`seed-种子文档.md` + `soul.md`。
> 前置：Step 1已完成，用户已确认推荐方案（或指定调整后方案）。金手指已通过梗×机制×限制公式设计。

## 前置确认

- Step 1已完成，用户已确认方案。
- execution.mode判定：
  - formal：核心承诺、主卖点、禁区、待research问题齐全。
  - draft：有缺口但可补全继续，产出标记draft，落盘。
  - trial：不落盘，仅对话内。本步骤落盘必须达formal或draft。

## 执行步骤

### 1. 填充7维模板

参照 `templates/seed-doc.tpl.md` 模板，从确认的候选概念出发，填充以下7个维度：

**维度1：核心卖点**

| 项 | 要求 |
|----|------|
| 一句话卖点 | 主角靠什么主动+为什么好看，一句话说清 |
| 元爽点 | 读者追读的核心驱动力（不是"变强"，是具体的爽感机制） |
| 追读理由 | 读者翻到下一章的具体理由 |
| 爽点外显方式 | 面板/参照物梯子/打脸/信息差碾压——选哪些 |
| 独特性声明 | 同赛道100本书里读者为什么选这本 |
| 核心悬念 | 贯穿全书250章的一个具体谜题 |

**维度2：角色地基**

不只建主角——还要建对手和关系动力学。

| 项 | 要求 |
|----|------|
| 主角身份 | 一句话：社会角色+核心特质 |
| 主角欲望 | 一句话：具体到可操作目标 |
| 主角伤口 | 一句话：内心脆弱点 |
| 核心对手身份 | 一句话：对手是谁+为什么与主角形成结构性冲突 |
| 核心对手欲望 | 一句话：必须与主角欲望形成对立，不是"坏人" |
| 关系动力学 | 张力来自什么？（利益冲突/理念对立/信息差/信任博弈） |

**维度3：世界观4系统**

每条世界规则必须回答："这条规则能反复产生什么类型的冲突？"

| 设定项 | 内容 | 冲突潜力 | 主角可利用的缝隙 |
|--------|------|---------|---------------|
| 背景设定 | 世界怎么变成现在这样的 | 产生什么结构性矛盾？ | |
| 力量体系 | 主角能做什么/不能做什么 | 产生什么决策困境？ | |
| 社会结构 | 谁在上谁在下，为什么 | 阶级/势力矛盾点？ | |
| 金手指 | 主角的独特优势 | 金手指本身就是冲突发动机 | |

必须额外覆盖（每个标注主角可利用的缝隙）：
- **经济系统**：钱从哪来、到哪去、主角怎么获取
- **政治系统**：谁定规则、规则怎么压迫主角、主角怎么利用规则缝隙
- **文化规范**：什么行为在这个世界正常但读者会震惊（代入落差）
- **信息不对称**：什么信息分层流动、主角的信息差优势在哪

**维度4：金手指巧思**

从Step 1的碰撞结果直接提取，正式填入模板：

| 设计项 | 内容 |
|--------|------|
| 金手指是什么 | 一句话（来自Step 1的梗×机制×限制） |
| 梗 | 提供形式的元素 |
| 机制 | 提供动力的运转方式 |
| 限制 | 提供张力的约束条件 |
| 巧思维度 | 决策困境/使用博弈/风险代价/信息差武器/自生长/成长节奏控制（至少1种） |
| 反模式检查 | 4种已避免：纯翻译器/纯正向循环/无场景限制/成长过慢 |
| 与世界/角色的焊接 | 在世界观的哪条冲突潜力上产生困境？和角色的哪个欲望/伤口形成张力？ |

**维度5：流派选择 + 笔触DNA + 三阶段走向**

| 项 | 内容 |
|----|------|
| 流派目录 | 如`prose-render-dndlike/`。无匹配则留空用通用prose-render |
| 笔触DNA模板 | 从流派目录的`dna/`中选择 |
| 风格关键词 | 3-5个词概括本书风格 |
| 数值表达方式 | 面板/参照物梯子/感官化——选哪些 |
| 例外条款 | 本项目中哪些笔触DNA红线让位于种子需求 |

三阶段走向：

| 项 | 内容 |
|----|------|
| 开局阶段 | 凡人期/超凡期 |
| 起点等级 | 主角开局等级 |
| 终点阶段 | 凡人期结束/超凡期结束/神明级 |
| 终点等级 | 主角全书最终等级段 |

三段式骨架（跃迁条件=最大的敌人/危机，逼主角质变的世界级压力）：

| 段 | 名称 | 跃迁条件 |
|----|------|---------|
| 段1 | | |
| 段2 | | 什么经历/代价/认知突破才能跨入 |
| 段3 | | 全书终点状态 |

**维度6：不可牺牲项与禁区**

| 项 | 数量 | 内容 |
|----|------|------|
| 不可牺牲项 | 3-5条 | 核心约束 |
| 禁区 | 3-5条 | 明确不写的方向 |

**维度7：体量规划 + 长线伏笔 + 全书故事弧**

体量规划：

| 卷 | 方向（一句话） | 卷级敌人 | 卷末质变 | 新元素引入 | 悬念递进 |
|----|--------------|---------|---------|-----------|---------|

长线伏笔清单（≥5条）：

| # | 伏笔内容 | 埋设卷 | 预计揭晓卷 | 回声节奏 |
|---|---------|-------|-----------|---------|

全书故事弧：

| 项 | 内容 |
|----|------|
| 全书核心矛盾 | 一句话 |
| 全书故事弧 | 3-5句话框选大致走向 |
| 全书敌人阶梯 | 3-5句话概述敌人量级递进脉络 |
| 终局画面 | 一句话 |

### 2. 种子压力测试

7维模板填充完成后，执行压力测试。5项全过才进落盘：

- [ ] **脑洞张力**：这个概念的核心矛盾是否让读者想知道"主角会怎么选"？
- [ ] **冲突密度**：从种子能直接推导出50章以上的不同冲突吗？
- [ ] **世界纵深**：世界有≥3个可反复利用的独立冲突源吗？（经济/政治/势力/文化各算一个）
- [ ] **角色生态**：主角+对手+至少2个关键关系都有地基吗？
- [ ] **长线续航**：有≥5条跨卷伏笔吗？每卷方向和卷末质变都清晰吗？

任一不满足：针对性回到对应维度补强，不落盘。

### 3. 落盘种子文档

将7维模板内容写入 `seed-种子文档.md`（参照 `templates/seed-doc.tpl.md` 格式）。

元数据块要求：
- doc_type: seed
- read_policy: full-if-targeted
- primary_consumer: review

### 4. 首版落盘 soul.md

soul回答"这是一本什么味道的小说"，管文风笔触风格，不管内容情节卖点。

参照 `templates/soul.tpl.md`，填充6项必填字段：
1. 叙事人格
2. 句子气口和段落呼吸
3. 对白方式
4. 信息释放
5. 氛围基调
6. 文风禁区

元数据块要求：
- doc_type: soul
- read_policy: full-required
- primary_consumer: write
- source_of_truth: true

soul不记录主卖点、元爽点、追读承诺、爽点外显方式（那些属于seed）。soul也不记录可变事实、具体剧情节点、人物状态、数值、招式名、系统规则。
