## 三层骨架理论（本skill的设计基底·消费视角）

男频小说的世界由三层骨架构成，缺一不可：

| 层 | 概念 | 定义 | 作用 | 设计方 |
|:--|:--|:--|:--|:--|
| 1 舞台 | **力量体系**(广义坐标系) | 任何形式的阶层金字塔 | 划出舞台，定义强弱刻度和位置空间 | **seed Phase 1·1d** → 骨架.md |
| 2 动力 | **动力引擎**(众生攀登系统) | 众生为什么爬+怎么爬+付出什么的系统 | 让坐标系活起来的世界级动力系统 | **seed Phase 1·1e** → 骨架.md |
| 3 助力 | **金手指**(助力定位) | 帮助主角合理化跨越坐标系的工具 | 降低跨越难度，但不能取消坐标系本身的意义 | **seed Phase 2·2b** → 主角设计.md |

> **v2.0.0核心变化**：三层骨架的设计已前移到seed（Phase 1骨架层 + Phase 2主角层）。world不再自行设计力量体系/动力引擎/金手指，改为消费骨架.md（力量体系+动力引擎）+ 主角设计.md（金手指）生长血肉（地图/势力/危机）。world保留对骨架的消费验证能力——发现问题报错回seed修改，不在world层自行补全或修改骨架。

> 剧情(表现) = 众生攀登过程的碰撞，交给 plot 层。world 只负责血肉(地图/势力/危机)。

**关键原则**（贯穿全skill）：
- **world不发明骨架，只消费骨架**——力量体系/动力引擎/金手指以骨架.md+主角设计.md为准
- 力量体系广义化——所有男频小说都有，历史赛道的门阀社会地位金字塔也是力量体系
- 动力引擎设计的是**众生如何爬**，不是主角如何爬——主角是众生之一
- 引擎是世界级的，不是主角专属
- 众生攀登方式的差异本身就是势力/人物的来源——不要脱离引擎凭空设计势力
- 金手指是助力，不能喧宾夺主

## 幕/卷统一格式定义（world+plot共用）

| 单元 | 定义 | 章数 |
|:--|:--|:--|
| **幕** | 完整起承转合的剧情单元 | 8~12章 |
| **卷** | 多个幕，主角/世界完成一次大的推进 | 40~60章（即4~5幕） |

**换算**：一卷=4~5幕（8~12章×4~5=32~60章，落在40~60区间）。

**world视角**：全书3-8卷×40~60章=120~480章（200万字级）。
**plot视角**：每卷分4~5幕写白描，每幕8~12章的起承转合。

## 定位

world 帮助用户构筑**有根的起点世界**。消费 设计/骨架.md（力量体系+动力引擎）+ 设计/主角设计.md（金手指定位），产出 `世界圣经.md`（对齐起点plot输入格式）。支持三种模式：消费骨架生长血肉(创作)、从成品书拆出三件套(拆解)、从多本拆解结果融合新世界(融合)。

**与起点群组的对接**：
- 上游：pop-qidian-seed Phase 1产出 `设计/骨架.md`（力量体系坐标系+动力引擎众生攀登系统），Phase 2产出 `设计/主角设计.md`（金手指设计+爽感矛盾公式）——world消费这两个文件作为生长起点
- world 专注**血肉生长**（地图/势力/危机/全书设定）——消费骨架.md的坐标系和引擎定义，生长地图空间分布/势力分层/危机体系，产出 `世界圣经.md`（主文件，含舞台/历史/神系/力量体系含三段式骨架+跃迁条件/敌人生态图/世界矛盾轴）+ 可选多文件拆分
- 下游：pop-qidian-plot 消费 骨架.md+主角设计.md+角色库+世界圣经——plot每卷从世界圣经切片消费，包括动力引擎的本卷引擎状态切片
- **world不做**：力量体系设计/动力引擎设计/金手指设计（seed的职责）；幕序列弧线、高潮场景展开、悬念分层、主角靠什么赢（剧情问题）

## 2c. 地图（从坐标系空间分布生长）

> 地图从力量体系(坐标系)的空间分布生长，不是凭空设计。坐标系在空间上怎么分布，地图就怎么画。
>
> **v2.0.0强制约束：地图的空间分布必须映射坐标系（骨架.md的力量体系），空间法则（力量禁区/阶位壁垒）由力量体系决定。**

推导逻辑（MMO式）：
```
骨架.md主养成线阶位 → 每个阶位的"区域主题" → 城市/世界地图
  阶位1："新手区" → 贫民区+赌场
  阶位2："城镇区" → 码头+商业街+公会
  阶位3："高级区" → 贵族区+神殿区+地下城
```

每个区域回答：
- 对应哪个力量阶位？（坐标系空间化——引用骨架.md主养成线阶位）
- 这个区域提供什么**攀登资源**？（从骨架.md动力引擎运转机制推导——众生在不同区域攀登方式不同）
- 有什么**空间法则**？（力量禁区/阶位壁垒/不可逾越区——**由骨架.md力量体系决定**）
- 主角的**核心行为**是什么？（从骨架.md动力引擎推导——主角是众生之一，遵循众生攀登逻辑）
- 有什么**信息差**可以利用？

输出：城市结构表（区域/对应阶位/攀登资源/空间法则/主角行为/信息差来源/势力归属/出现卷——标注哪卷进入此区域）

## 2d. 势力（从众生攀登方式差异生长）

> 势力从众生攀登方式的差异中生长，不是从力量体系+地图凭空设计。势力 = 众生中**同一种攀登方式的群体**——骨架.md动力引擎组成3"众生攀登方式分层"直接生长出势力。
>
> **v2.0.0强制约束：势力从骨架.md动力引擎组成3（众生攀登方式分层）生长，每个势力标注对应的攀登方式类型。**

推导逻辑：
```
骨架.md动力引擎组成3·众生攀登方式分层：
  爬到顶层的人怎么维持 → 顶层势力（垄断资源/封锁门槛/打压后来者）
  有资源的人怎么爬     → 中层势力（资源积累/联姻/投靠）
  没资源的人怎么爬     → 底层势力（命换/投机/依附）
  掉下来的人怎么办     → 散人或反派（背景/反例/同路人）
```

每层势力必须回答（设定视角，不回答剧情怎么赢）：
1. **这个势力是哪种攀登方式的群体？**（回溯骨架.md动力引擎组成3——不能脱离引擎凭空设计）
2. **标注对应的攀登方式类型**（v2.0.0强制——有资源型/没资源型/爬到顶层型/掉下来型）
3. **这个势力控制什么资源？**（资源=进入下一阶位的钥匙）
4. **这个势力有什么弱点/裂缝？**（弱点=world给plot提供的可利用设定，不是剧情怎么利用）
5. **和上层势力什么关系？**（从属/同盟/对抗——决定主角打这层时上层什么态度）

**全书要求**：4层势力全部具名设计，每层标注哪卷碰撞（第一卷通常碰撞前2层，后2层暗示；后续卷逐层展开）。势力天花板=阶位天花板（每个势力的力量上限由骨架.md坐标系决定）。

# Step 2 · 拆解模式（Decon Mode）

> **v2.0.0说明：拆解模式不受骨架前移影响——它是独立能力，不消费骨架.md，而是自己从成品书拆解出骨架。** 从成品书拆出完整的"力量体系+动力引擎+世界观设定"三件套，产出格式直接对齐创作模式——拆出来的东西可以直接作为创作模式的参考。

## 拆解流程

### Step D1：读取成品书

- 输入：成品书txt
- 读取范围：**前150章**（建立力量体系+动力引擎+金手指+地图/势力/危机）+ **中段50章**（验证引擎演化+势力深化+危机升级）
- 如有 decon-lite 拆书结果（`素材/decon-lite-{书名}.md`），优先消费其表1（力量体系结构）和表9（动力引擎）作为拆解起点

### Step D2：拆解力量体系（广义坐标系）

识别这本书的力量体系形态，拆解四层结构：

| 拆解项 | 内容 |
|:--|:--|
| 坐标系形态 | 修仙位阶/社会地位/经济等级/术法品质/强度梯度——**广义识别，不限修仙** |
| 世界源力量 | 源力量名称/来源/转化链/敌分级标准 |
| 主养成线 | 5-7阶完整表（阶位名/文化渊源/社会地位参照） |
| 子养成线 | N条（自身阶位+主养成线挂钩） |
| 交叉规则 | N条（协同/制约） |

### Step D3：拆解动力引擎（众生攀登系统）

拆解六组成：

| 拆解项 | 内容 |
|:--|:--|
| 驱动逻辑 | 世界级驱动力——众生为什么爬 |
| 运转机制 | 闭环：输入→转化→输出→反馈（全员通用，非主角专属） |
| 众生攀登方式分层 | 有资源/没资源/爬到顶层/掉下来的人分别怎么爬——**直接对应势力来源** |
| 代价结构 | 通用代价+群体差异代价+累积效应 |
| 范式归类 | 飞轮型/探索型/混合型/压迫型/对抗型 |
| 演化节点 | 长篇中引擎是否演化+何时演化+触发事件 |

### Step D4：拆解金手指定位

| 拆解项 | 内容 |
|:--|:--|
| 金手指类型 | 系统/穿越知识/重生信息差/特殊血脉/异宝 |
| 加速机制 | 如何合理化加速主角跨越坐标系 |
| 限制 | 防止喧宾夺主的限制设计 |
| 代价 | 使用金手指的代价 |
| 是否喧宾夺主 | 判定——金手指是否取消了坐标系意义 |

### Step D5：拆解世界观设定

| 拆解项 | 内容 |
|:--|:--|
| 地图 | 坐标系空间分布+区域对应阶位+空间法则 |
| 势力 | 从众生攀登方式差异回溯——每层势力是哪种攀登方式的群体 |
| 危机 | 三层面：坐标系内压+引擎代价+金手指限制 |

### Step D6：产出

落盘 `decon-world-{书名}.md`，包含完整的"力量体系+动力引擎+金手指定位+世界观设定"，格式对齐创作模式的产出格式。

# Step 2 · 融合模式（Fuse Mode）

> **v2.0.0说明：融合模式不受骨架前移影响——它是独立能力，不消费骨架.md，而是从多本拆解结果融合出新骨架。** 从2-4本成品书的拆解结果(decon-world)，融合出一个新的"力量体系+动力引擎+世界观设定"。

## 融合流程

### Step F1：加载拆解结果

- 输入：2-4本 `decon-world-{书名}.md`（拆解模式产出）
- 逐本提取三件套：力量体系+动力引擎+世界观设定

### Step F2：融合策略

> **关键原则：融合不是机械拼接，必须自洽。** 取A书的某组成+B书的某组成，必须检查兼容性。

| 融合维度 | 策略 | 自洽要求 |
|:--|:--|:--|
| 力量体系融合 | 取A书的坐标系形态+B书的子养成线结构+C书的交叉规则 | 坐标系形态与子养成线必须兼容——不能修仙位阶配经济子养成线(除非有合理化解释) |
| 动力引擎融合 | 取A书的驱动逻辑+B书的运转机制+C书的代价结构 | 驱动逻辑→运转机制→代价结构必须形成闭环——不能生存压力驱动配飞轮型运转配零代价 |
| 金手指融合 | 取A书的金手指类型+B书的限制设计 | 金手指类型与限制设计必须匹配——系统类配系统限制，穿越知识类配信息差限制 |
| 世界观融合 | 从融合后的力量体系+动力引擎，**重新生长**地图/势力/危机 | 不机械拼接各书地图——从融合后的骨架重新推导 |

### Step F3：融合自洽检查（红线级）

| 检查项 | 检查内容 | 不通过的后果 |
|:--|:--|:--|
| 坐标系与引擎匹配 | 坐标系形态与攀登方式是否兼容？ | 修仙位阶配经济攀登=不兼容，需改 |
| 引擎三组成自洽 | 驱动逻辑→运转机制→代价结构是否形成闭环？ | 缺环=引擎转不起来 |
| 金手指不喧宾夺主 | 融合后的金手指是否取消坐标系意义？ | 喧宾夺主=需加强限制 |
| 众生攀登方式自洽 | 不同群体的攀登方式是否与坐标系匹配？ | 不匹配=势力来源断裂 |

### Step F4：产出

落盘 `fuse-world-{项目名}.md`，包含融合后的"力量体系+动力引擎+金手指定位+世界观设定"，格式对齐创作模式的产出格式。可直接作为创作模式的骨架输入。

## 红线

1. **world禁止自行发明或修改力量体系**——必须消费骨架.md的坐标系定义（四层结构）。world发现问题报错回seed修改，不在world层自行补全或修改骨架。

2. **world禁止自行发明动力引擎**——必须消费骨架.md的引擎定义（六组成）。world发现问题报错回seed修改，不在world层自行补全或修改骨架。

3. **势力设计必须从众生攀登方式分层中生长**——势力从骨架.md动力引擎组成3（众生攀登方式分层）生长，每个势力标注对应的攀登方式类型。不脱离引擎凭空设计势力。全书4层势力全具名。

4. **地图空间法则必须由力量体系决定**——地图的空间分布必须映射坐标系（骨架.md力量体系），空间法则（力量禁区/阶位壁垒）由力量体系决定。

5. **危机与动力引擎驱动逻辑对齐**——危机是引擎运转的阻力表现，三层面（坐标系内压/引擎代价/金手指限制）必须与骨架.md+主角设计.md对齐。

6. **world禁止自行设计金手指**——消费主角设计.md的金手指定义。金手指是助力不喧宾夺主——如果金手指强到让坐标系失去意义，报错回seed修改。

7. **世界纵深红线**——经济/政治/文化/信息不对称全覆盖，不只地理+势力+力量体系。地图按力量层级空间化，势力按攀登方式分层，危机从三层面推导。
