# Step 0 - 交互式世界决策

> **v2.2.0新增**：在世界圣经生成之前，通过2轮交互式决策采集用户偏好，产出`世界决策表.md`。
> **核心原则**：给选项不给空白——每轮提供2-3个结构化选项+1个自定义入口。
> **杠杆位置**：世界结构定下后，plot的分幕走向和write的空间尺度都基于此。

## 前置条件

- Phase 2 seed完成
- 骨架.md + 主角设计.md + ch001.txt 齐全

## 产出

`设计/世界决策表.md`——后续世界圣经生成的硬输入

## 2轮交互决策

### W1 世界结构偏好（核心轮·必答）

**底牌消费**：骨架.md + 主角设计.md + ch001.txt + decon-lite表6（势力）+ decon-plot（如有）

**Agent动作**：
1. 读取骨架.md，提取力量体系的空间分布需求
2. 读取主角设计.md，提取主角的移动模式偏好（据点型/游击型/扩张型）
3. 生成3个世界结构方案，每个标注：地图布局/势力层数/危机类型/空间叙事策略

**选项生成规则**：
- 方案A：据点型（1城多区域，从内向外扩张）
- 方案B：游击型（多城+道路，移动叙事）
- 方案C：争霸型（1大区域多子区域，棋盘式多势力交叉）

**输出格式**：
```
## W1 世界结构偏好

基于{力量体系}+{主角性格}，提供3个世界结构方案：

A) {方案A名}
   → 地图：{布局}
   → 势力：{层数+数量}
   → 危机：{类型}
   → 空间叙事：{策略}

B) {方案B名}
   → ...

C) {方案C名}
   → ...

请选择，或描述你想要的世界结构。
```

**影响标注**：世界结构定下后，plot的分幕走向和write的空间尺度都基于此。

### W2 危机体系偏好（可选轮·可跳过）

**底牌消费**：W1选择 + 骨架.md引擎演化节点

**Agent动作**：
1. 基于选定的世界结构，提出2-3个危机体系方案
2. 每个方案标注：终极危机/阶段性危机/危机驱动机制

**跳过规则**：用户说"跳过"或"默认"→ agent按骨架.md的引擎节点自动生成

## 决策表产出格式

```markdown
# 世界决策表
> 交互完成时间：{YYYY-MM-DD HH:mm}

## W1 世界结构偏好
- 结构：{用户选定的结构}
- 地图布局：{布局}
- 势力层数：{层数}
- 危机类型：{类型}
- 空间叙事：{策略}
- 选择：{A/B/C/自定义}

## W2 危机体系偏好
- 终极危机：{描述}
- 阶段性危机：{描述}
- 驱动机制：{描述}
- 选择：{A/B/跳过}

## 后续约束
- 世界圣经生成必须遵守本决策表
- 如需调整，需重新走对应轮次的交互
```

## 红线

1. **每轮必须给选项**——必须给2-3个结构化选项
2. **每轮必须有自定义入口**——选项末尾必须有"或描述你想要的XXX"
3. **选项必须在骨架约束内**——世界结构必须匹配力量体系的空间需求
4. **核心轮不可跳过**——W1必答，W2可跳过
5. **决策表是世界圣经生成的硬输入**
