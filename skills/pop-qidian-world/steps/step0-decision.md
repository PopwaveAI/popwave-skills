# Step 0 - 交互式世界决策

> **本步骤定位**：world的设计大脑。在世界圣经生成前，通过2轮交互采集用户对世界结构和危机体系的偏好，产出`世界决策表.md`作为Step 1-3的硬输入。
> **核心原则**：给选项不给空白——每轮提供2-3个结构化选项+1个自定义入口。选项必须标注好坏标准，让用户知道每个选择的质量后果。

## 前置条件

- Phase 2 seed完成
- 骨架.md + 主角设计.md + ch001.txt 齐全

## 产出

`设计/世界决策表.md`——后续世界圣经生成的硬输入

---

## W1 世界结构偏好（核心轮·必答）

**底牌消费**：骨架.md（坐标系空间需求+引擎众生攀登分层）+ 主角设计.md（移动模式偏好）+ ch001.txt + decon-lite表6（势力）

**Agent动作**：
1. 读取骨架.md，提取坐标系阶位结构→空间分布需求
2. 读取主角设计.md，提取主角核心欲望→移动模式（据点型/游击型/争霸型）
3. 生成3个世界结构方案

### 好坏标准

| 维度 | 好世界结构 | 坏世界结构 |
|:--|:--|:--|
| 空间叙事 | 每个区域有戏剧潜力（资源争夺/势力交叉/信息差/危险梯度） | 地名列表（只有名字没有叙事价值） |
| 坐标系映射 | 空间分布映射阶位（高阶位区=危险区/核心区） | 空间和力量体系脱节 |
| 主角适配 | 地图结构匹配主角移动模式 | 地图和主角行为模式矛盾 |
| 区域差异 | 区域间有明确差异（资源/法则/势力/氛围全不同） | 所有区域长得一样 |

### 选项生成规则

**方案A：据点型**（1城多区域，从内向外扩张）
→ 适合：主角欲望是"扎根/经营/保护"，坐标系是层级嵌套型
→ 地图：1核心城+3-5外围区域，阶层从内到外递减
→ 势力：层数4层（城内3层+外围1层）
→ 危机：扩张型（向外探索=接触更高阶位）
→ 空间叙事：从小处着眼逐渐展开世界

**方案B：游击型**（多城+道路，移动叙事）
→ 适合：主角欲望是"逃离/追查/流浪"，坐标系是线性推进型
→ 地图：3-5城+连接道路，每城对应不同阶位区间
→ 势力：层数3-4层（每城1-2层势力）
→ 危机：追猎型（移动中不断遭遇不同层面危机）
→ 空间叙事：公路片式，每城一个完整弧线

**方案C：争霸型**（1大区域多子区域，棋盘式多势力交叉）
→ 适合：主角欲望是"征服/改变/掌权"，坐标系是平行竞争型
→ 地图：1大区域+5-8子区域，多势力棋盘交叉
→ 势力：层数4层（每子区域1-2层，跨区域势力联盟）
→ 危机：博弈型（多势力制衡+坐标系天花板压迫）
→ 空间叙事：棋盘博弈，势力交叉产生冲突

**输出格式**：
```
## W1 世界结构偏好

基于{坐标系类型}+{主角移动模式}，提供3个世界结构方案：

A) {方案A名}
   → 地图：{布局}
   → 势力：{层数+数量}
   → 危机：{类型}
   → 空间叙事：{策略}
   → 好坏标准对照：{该方案在4个维度的表现}

B) {方案B名}
   → ...

C) {方案C名}
   → ...

请选择，或描述你想要的世界结构。
```

**影响标注**：世界结构定下后，Step 1的地图空间化和势力具名化都基于此。plot的分幕走向和write的空间尺度也基于此。

## W2 危机体系偏好（可选轮·可跳过）

**底牌消费**：W1选择 + 骨架.md引擎演化节点 + 主角设计.md金手指限制

**Agent动作**：
1. 基于选定的世界结构，从骨架.md引擎三层面推导危机体系方案
2. 每个方案标注：终极危机/阶段性危机/危机驱动机制

### 好坏标准

| 维度 | 好危机体系 | 坏危机体系 |
|:--|:--|---|
| 引擎对齐 | 危机是引擎运转的必然阻力（坐标系内压/引擎代价/金手指限制三层面） | 随机外部威胁（和引擎无关） |
| 全书覆盖 | 1条终极+2-3条阶段性，分卷爆发形成危机链 | 只有第一卷的危机 |
| 升级路径 | 危机会恶化（不解决越来越糟），有明确升级节点 | 一次性事件，解决就没了 |
| 层面归属 | 每条危机标注属于哪个层面 | 不知道是坐标系问题还是引擎代价 |

### 选项生成规则

**方案A：引擎代价主导型**
→ 终极危机：引擎闭环断裂（代价清算）
→ 阶段性危机：每卷爆发一种引擎代价（通用代价→群体差异→代价突变）
→ 驱动机制：主角越爬越高，代价越大

**方案B：坐标系内压主导型**
→ 终极危机：坐标系顶层对决（强者碾压）
→ 阶段性危机：每卷突破一层阶位壁垒
→ 驱动机制：上面有人压制，不爬就被踩

**方案C：金手指限制主导型**
→ 终极危机：金手指终局抉择（代价转嫁到极致）
→ 阶段性危机：金手指限制逐卷暴露
→ 驱动机制：金手指是双刃剑，越用越危险

**跳过规则**：用户说"跳过"或"默认"→ agent按骨架.md的引擎节点+三层面自动分配

## 红线

1. **每轮必须给选项**——必须给2-3个结构化选项，不能开放式提问
2. **每轮必须有自定义入口**——选项末尾必须有"或描述你想要的XXX"
3. **选项必须在骨架约束内**——世界结构必须匹配力量体系的空间需求
4. **选项必须标注好坏标准**——让用户知道每个选择的质量后果
5. **核心轮不可跳过**——W1必答，W2可跳过
6. **决策表是世界圣经生成的硬输入**——Step 1-3消费决策表展开，不另起炉灶
