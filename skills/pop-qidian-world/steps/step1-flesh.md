## 1b · 地图空间化

> **定位**：把骨架.md的坐标系（力量体系阶位）展开为可叙事的空间。地图不是地名列表——是坐标系的空间投影，每个区域都有戏剧潜力。
> **消费源**：骨架.md主养成线阶位（阶位→空间区域）+ 世界决策表W1（地图布局选择）

### 好坏标准

| 维度 | 好 | 坏 |
|:--|:--|:--|
| 空间叙事 | 每个区域有戏剧潜力（资源争夺/势力交叉/信息差/危险梯度） | 地名列表（只有名字没有叙事价值） |
| 坐标系映射 | 空间分布映射阶位（高阶位区=危险区/核心区） | 空间和力量体系脱节 |
| 空间法则 | 力量禁区/阶位壁垒由力量体系决定（坐标系投影） | 空间法则随便编 |
| 区域差异 | 资源/法则/势力/氛围全不同 | 所有区域长得一样 |
| 主角适配 | 地图结构匹配主角移动模式（W1决策） | 地图和主角行为模式矛盾 |

### 达成方法

从骨架.md主养成线提取阶位→空间分布→空间法则→叙事价值注入→移动模式匹配。

#### 产出：城市舞台表

| 区域 | 对应阶位 | 攀登资源 | 空间法则 | 主角核心行为 | 信息差来源 | 势力归属 |
|:--|:--|:--|:--|:--|:--|:--|
| 区域A | 阶位1 | （从引擎组成2推导：众生攀登投入什么） | （坐标系决定：力量禁区/阶位壁垒） | （从引擎推导：主角在这里做什么） | （主角知道而NPC不知道的） | （从1c势力产出） |
| 区域B | 阶位2 | | | | | |
| ... | | | | | | |

**字段说明**：
- **对应阶位**：从骨架.md主养成线取——每个区域对应哪个阶位区间
- **攀登资源**：从骨架.md动力引擎组成2（运转机制·输入）推导——众生在这里投入什么攀登
- **空间法则**：从骨架.md力量体系决定——阶位壁垒（低阶进不去高阶区）/力量禁区（某阶位以下无法使用力量）
- **主角核心行为**：从动力引擎推导——主角在这个空间做什么（搜刮/修炼/交易/战斗）
- **信息差来源**：主角凭金手指知道而NPC不知道的信息——空间叙事的爽感来源

## 1c · 势力具名化

> **定位**：把骨架.md动力引擎组成3（众生攀登方式分层）展开为具名势力。势力=同一种攀登方式的群体——不是凭空设计的组织，是引擎运转的社会产物。
> **消费源**：骨架.md引擎组成3（四类攀登群体）+ 世界决策表W1（势力层数）

### 好坏标准

| 维度 | 好 | 坏 |
|:--|:--|:--|
| 引擎溯源 | 每个势力标注攀登方式来源（回溯引擎组成3） | 脱离引擎凭空设计 |
| 全层覆盖 | 4层全具名（底层/中层/高层/顶层+散人） | 只做前2层 |
| 结构完整 | 每个势力有：名称+控制资源+弱点/裂缝+和上层关系 | 只有名字没有结构 |
| 碰撞设计 | 每层标注碰撞卷（哪卷和主角碰撞） | 势力平行存在不交叉 |
| 弱点可利用 | 弱点是结构性的（攀登方式导致的必然弱点） | 弱点是随机编的 |

### 达成方法

从引擎组成3四类群体→每类→1层势力→每层具名（名称+资源+弱点+关系+碰撞卷）。

#### 产出：势力结构表

| 势力层 | 攀登方式来源 | 势力名 | 控制什么资源 | 弱点/裂缝 | 和上层关系 |
|:--|:--|:--|:--|:--|:--|
| 第1层·底层 | 没资源的人怎么爬（命换型/投机型） | | | | 被中层剥削 |
| 第2层·中层 | 有资源的人怎么爬（资源型/联姻型） | | | | 受高层压制 |
| 第3层·高层 | 有资源的人怎么爬（进阶：垄断型） | | | | 受顶层制约 |
| 第4层·顶层 | 爬到顶的人怎么维持（垄断/封锁/打压） | | | | 规则制定者 |
| 散人/反派 | 掉下来的人怎么办 | | | | 各卷穿插 |

**攀登方式来源硬约束**：每个势力的"攀登方式来源"必须回溯骨架.md引擎组成3——这个势力是哪种攀登方式的群体。禁止脱离引擎凭空设计势力。

**弱点设计原则**：弱点必须是攀登方式的结构性弱点——命换型的弱点是资源匮乏/健康透支，资源型的弱点是资源依赖/联姻脆弱，垄断型的弱点是僵化/信息封闭。

## 1d · 危机引擎化

> **定位**：把骨架.md动力引擎的运转阻力展开为全书危机链。危机不是随机威胁——是引擎运转的必然阻力，是坐标系内压/引擎代价/金手指限制三层面的具象化。
> **消费源**：骨架.md引擎组成4（代价结构）+组成5（范式归类）+ 主角设计.md金手指限制 + 世界决策表W2（危机体系选择）

### 好坏标准

| 维度 | 好 | 坏 |
|:--|:--|:--|
| 引擎对齐 | 危机是引擎运转的必然阻力（三层面） | 随机外部威胁 |
| 三层面覆盖 | 坐标系内压+引擎代价+金手指限制各有危机 | 只有一种层面的危机 |
| 全书覆盖 | 1条终极+2-3条阶段性，分卷爆发 | 只有第一卷的危机 |
| 升级路径 | 危机会恶化，有明确升级节点 | 一次性事件 |
| 层面归属 | 每条危机标注属于哪个层面 | 不知道是什么层面的危机 |

### 达成方法

从引擎三层面提取危机来源→设计终极+阶段性→每条标注层面+触发条件+爆发卷。

#### 三层面危机来源

| 危机层面 | 来源 | 危机表现 |
|:--|:--|:--|
| 坐标系内压 | 骨架.md主养成线阶位壁垒 | 强者注目/门槛阻隔/天花板压迫——主角爬到某阶位后触发的系统性阻力 |
| 引擎代价 | 骨架.md引擎组成4代价结构 | 通用代价显现/群体差异代价/代价突变——攀登付出的代价具象化为危机 |
| 金手指限制 | 主角设计.md金手指限制+代价 | 被发现风险/使用代价/终局抉择——金手指双刃剑的危机化 |

#### 产出：全书危机体系

| 危机层面 | 危机描述 | 触发条件 | 后果 |
|:--|:--|:--|:--|
| **终极危机**（标注层面） | 全书终卷爆发的世界级威胁 | | |
| **阶段性危机1**（标注层面） | 第一卷爆发的引擎阻力 | | |
| **阶段性危机2**（标注层面） | 第二卷爆发的引擎阻力 | | |
| **阶段性危机3**（标注层面） | 第三卷爆发的引擎阻力 | | |

**危机设计硬约束**：每条危机必须标注属于三层面中的哪个。危机是引擎运转的必然阻力——不是外部随机威胁，是"主角爬到这个位置就一定会遇到这个阻力"。

## 1e · 敌人生态化

> **定位**：把骨架.md动力引擎组成3（众生攀登方式分层）展开为4层敌人生态。敌人不是脸谱反派——是不同攀登方式的代表者，每个敌人的动机和弱点都来自其攀登方式。
> **消费源**：骨架.md引擎组成3（四类攀登群体）+ 1c势力结构表

### 好坏标准

| 维度 | 好 | 坏 |
|:--|:--|:--|
| 攀登溯源 | 每个敌人是某种攀登方式的代表者 | 脸谱反派（只有"邪恶"） |
| 四层梯度 | 底层(阶位1-3)/中层(4-5)/顶层(6-7)/散人 | 敌人梯度和阶位不对应 |
| 动机结构 | 动机来自攀登方式（资源争夺/垄断维护/生存压力） | 动机随机编 |
| 弱点引擎化 | 弱点是攀登方式的结构性弱点（引擎视角可利用） | 弱点是主角打败的 |
| 碰撞设计 | 每层标注碰撞卷 | 敌人没有出场规划 |

### 达成方法

从引擎组成3四类攀登群体→每类→1层敌人→每层具名（代表+量级+动机+弱点+碰撞卷）。

#### 产出：敌人生态图

| 敌人层级 | 攀登方式来源 | 代表敌人 | 量级（对应阶位） | 动机 | 弱点（引擎视角） |
|:--|:--|:--|:--|:--|:--|
| 顶层敌人 | 爬到顶的人怎么维持（垄断者） | | 阶位6-7 | 维护垄断/打压后来者 | 垄断导致的僵化/信息封闭 |
| 中层敌人 | 有资源的人怎么爬（资源型） | | 阶位4-5 | 资源争夺/地位保卫 | 资源依赖/联姻脆弱 |
| 底层敌人 | 没资源的人怎么爬（命换型/投机型） | | 阶位1-3 | 生存压力/投机抢夺 | 资源匮乏/命换代价 |
| 散人/变节者 | 掉下来的人怎么办 | | 各阶位 | 嫉妒/报复/投靠 | 被淘汰的执念/无退路 |

**敌人设计硬约束**：每个敌人的"攀登方式来源"必须回溯骨架.md引擎组成3。敌人动机=其攀登方式的必然冲突——垄断者打压后来者是因为"爬到顶的人怎么维持"的逻辑，命换型抢夺是因为"没资源的人怎么爬"的逻辑。弱点必须是攀登方式的结构性弱点，不是主角打败的——是引擎视角下可利用的设定。

## 1f · 矛盾轴化

> **定位**：把骨架.md动力引擎组成5（范式归类）+组成6（演化节点）展开为世界矛盾轴。矛盾轴是世界的底层张力——不是随机矛盾，是引擎运转的必然张力。
> **消费源**：骨架.md引擎组成5（范式归类）+组成6（演化节点）+ 1d危机三层面

### 好坏标准

| 维度 | 好 | 坏 |
|:--|:--|:--|
| 引擎溯源 | 矛盾从引擎范式推导（飞轮型=加速vs减速） | 矛盾和引擎脱节 |
| 三线覆盖 | 坐标系内压/引擎代价/金手指限制各有张力线 | 只有一条张力线 |
| 演化对齐 | 矛盾升级节点对齐引擎演化节点 | 矛盾静态不变 |
| 全书贯穿 | 每卷矛盾状态不同，有升级有终态 | 全书一个基调 |

### 达成方法

从引擎范式→世界基调矛盾→演化节点→矛盾升级节点→三层面→三条张力线。

#### 产出：世界矛盾轴

| 要素 | 来源 | 产出 |
|:--|:--|:--|
| 世界基调矛盾 | 引擎组成5范式归类 | 飞轮型=加速vs减速 / 压迫型=生存vs灭亡 / 探索型=已知vs未知 |
| 矛盾升级节点 | 引擎组成6演化节点 | 每个演化触发事件=矛盾升级节点 |
| 坐标系内压张力线 | 1d危机·坐标系内压层面 | 强者注目→门槛阻隔→天花板压迫 |
| 引擎代价张力线 | 1d危机·引擎代价层面 | 通用代价→群体差异→代价突变 |
| 金手指限制张力线 | 1d危机·金手指限制层面 | 被发现风险→使用代价→终局抉择 |

**矛盾轴设计硬约束**：矛盾不是自说自话——必须从引擎范式推导。飞轮型引擎的矛盾是"加速和减速的拉锯"，压迫型引擎的矛盾是"生存和灭亡的赛跑"。矛盾升级节点必须对齐引擎演化节点——引擎演化时矛盾必然升级。
