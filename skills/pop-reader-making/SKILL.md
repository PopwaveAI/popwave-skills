# pop-reader-making · 网文拆书技能 v0.14.0

帮读者拆解长篇小说。**产出双格式：人类可读的叙事笔记 + 机器可消费的结构化 YAML。**

---

## 速查表

| 阶段 | 核心动作 | 产出 | ❌ 门禁 |
|:-----|:---------|:-----|:--------|
| **Phase 0 — 书稿预检** | 通读目录 → 定分卷 → 抽样验证 | 拆书方案 | 用户未确认→不进 Phase A |
| **Phase A — 深度阅读 + 结构化标注** | 逐章读 → 叙事笔记 + YAML标注 → 关键实体 | 笔记.md + 结构化.yaml | 单卷质量未确认→不进下一卷 |
| **Phase B — 跨卷关联** | 打通整书理解 | 跨卷关联笔记 | — |
| **Phase C — 按需结构化** | 只在需要时做 | 知识图谱（可选） | — |

### 协作边界

> **拆书定位声明：** 本 skill 面向"拆书为读"——产出供读者查阅的叙事笔记和供 pop-html-anything 渲染的结构化 YAML（含 image_prompt）。
>
> **分工：**
> - pop-reader-making → 消费方：读者 / pop-html-anything → 产出：叙事笔记.md + 结构化YAML（含 image_prompt）
> - book-deconstructor（pop-novel-agent-pro 内部）→ 面向"拆书为写"→ 产出供给 emergent-writer 的 14 项输入包

## 执行流程 SOP

### 第一步：书稿预检（Phase 0）

| 项目 | 内容 |
|:-----|:------|
| 读什么 | 用户提供的完整小说目录 |
| 做什么 | 通读目录定分卷 → 密集抽样验证分卷是否合理 → 产出拆书方案 |
| 产出什么 | `拆书方案.md`（核心：分卷定界表） |
| ❌ 门禁 | **用户未确认方向 → 不退进 Phase A**。必须用户说"可以"才进入下一步 |

#### Step 0.1 — 通读目录定分卷
#### Step 0.2 — 密集抽样验证分卷
#### Step 0.3 — 产拆书方案（核心：分卷定界表）
#### Step 0.4 — 用户确认方向（第一道门禁）

---

### 第二步：按卷深度阅读 + 结构化标注（Phase A）

> 两条线并行，互不替代。
> **叙事线**（给人读）→ 保持章节顺序、叙事节奏、阅读判断
> **结构线**（给机器）→ 每章的 entities/events/metrics/signals/tone + 角色/名场面/共现

#### Step A.1：产出分卷阅读笔记（叙事线）

| 项目 | 内容 |
|:-----|:------|
| 读什么 | 当前卷全部正文 |
| 做什么 | 按章节顺序阅读，每 3-5 章为一节撰写【摘要】+【执行观察】 |
| 产出什么 | `{书名}_卷{序号}_{卷名}_笔记.md` |

**笔记四部分**：

##### 一、魔改对照（IP衍生/无限流/诸天流类小说必做）
1. 这卷借用了什么原作？
2. 作者改了什么？
3. 改得怎么样？

##### 二、章节叙事流（核心）
每3-5章为一节，每节包含【摘要】+【执行观察】。

执行观察类型：节奏判断 / 魔改效果 / 信息密度 / 结构评价 / 作者意图 / 阅读体验

##### 三、本卷关键实体（精简版）
只记重要角色和设定。

##### 四、本卷质量判断
```
阅读体验：[流畅/拖沓/起伏大]
魔改质量：[精彩/一般/生硬]
最佳段落：____
最弱段落：____
一句话总结：____
```

#### Step A.1.5：逐章结构化标注（结构线）

| 项目 | 内容 |
|:-----|:------|
| 读什么 | 同一卷正文（与叙事线同步） |
| 做什么 | 为每一章产出结构化元数据 |
| 产出什么 | 放入叙事流之后，顺序一致 |

**每章标注模板**：
```yaml
ch1:
  title: "大脑超频"
  entities: ["陈昂", "露西", "理查德"]
  events:
    - type: "穿越"
      summary: "陈昂自永无止境世界返回现实"
      participants: []
    - type: "对话"
      summary: "拦截露西和理查德，夺取CPH4"
      participants: ["陈昂", "露西", "理查德"]
  metrics:
    出场角色数: 3
    事件数: 2
    信息密度: 高
  signals: ["NZT副作用未提及—陈昂已适应"]
  tone: "兴奋好奇"
  key_entities: ["陈昂"]
```

- 每章 1-5 条 events
- `events.type` 从以下取：`战斗 / 对话 / 探索 / 设定揭示 / 伏笔 / 转折 / 日常 / 穿越`

#### Step A.2：读完一卷后，产出关键实体列表（增强版）

| 项目 | 内容 |
|:-----|:------|
| 读什么 | 本卷全部叙事笔记 + YAML 标注 |
| 做什么 | 产出角色表 + 名场面 + 共现矩阵 + 卷统计 |
| 产出什么 | 结构化数据（嵌入笔记 + 独立 YAML） |

**角色表（含 image_prompt）**：

| 角色 | 身份 | 本卷关键事件 | 章节 | 出场章节数 | 角色重要性 | 肖像 prompt |
|:----|:----|:-----------|:----:|:---------:|:---------:|:------------|
| 陈昂 | 主角 | 获得NZT、CPH4、进入笑傲 | 1-26 | 26 | 主角 | Chinese xianxia novel protagonist portrait: ... |

> **image_prompt 必须写。** 这是 pop-html-anything 生图的直接输入。prompt 结构：`[内容类型]: [视觉描述], [氛围], high quality digital illustration, cinematic lighting`

**本卷名场面**（3-8 个）：

| 场景 | 章节 | 类型 | 画面描述 | image_prompt |
|:----|:----:|:----:|:---------|:------------|
| 场景名 | 章节 | 类型 | 画面简述 | Chinese novel cinematic scene: ... |

**名场面判断标准**：①有强烈视觉画面感 ②推动关键剧情转折 ③角色情感高潮点

**实体共现矩阵**：

| 共现对 | 共同出场章节数 | 关系强度 |
|:------|:-------------:|:--------|
| 陈昂 ↔ 丁勉 | 8 | 大量 |

**本卷关键数据统计**：
```
卷统计：
  总章节数: 26
  总出场角色数: 18
  核心角色(出场>50%章): 4
  事件总数: 62
  战斗事件: 18
  对话事件: 15
  情绪曲线: 兴奋好奇 → 试探布局 → ... → 从容收官
```

#### Step A.3：用户确认本卷质量后，进入下一卷

| ❌ 门禁 |
|:--------|
| 用户未确认本卷质量 → 不退进下一卷 |
| 叙事笔记未完成 + YAML 标注未完成 → 两条线缺一不可，退回补全后再确认 |

## 产出文件清单

```
小说项目/<书名>/
├── 拆书方案.md                     ← Phase 0
├── {书名}_卷{序号}_笔记.md          ← Phase A（叙事线）
├── {书名}_卷{序号}_结构化数据.yaml   ← Phase A（结构线，供 pop-html-anything 消费）
├── {书名}_跨卷关联笔记.md            ← Phase B
```

> `*_结构化数据.yaml` 包含 volume_stats + chapters[]（含 entities/events/metrics/signals/tone）+ characters[]（含 image_prompt）+ scenes[]（含 image_prompt）+ entity_cooccurrence[]。pop-html-anything 直接读取此文件渲染。

## ❌ 质量红线

| # | 红线 | 确认 |
|:-:|:-----|:----:|
| ❌1 | **分卷定界未经用户确认不进 Phase A** | [ ] |
| ❌2 | **叙事线 + 结构线双产出缺一不可** — 不能只有笔记没有 YAML，也不能只有 YAML 没有笔记 | [ ] |
| ❌3 | **每卷必须产出结构化 YAML（含 image_prompt）** — 角色肖像 prompt + 名场面 prompt 不可省略 | [ ] |
| ❌4 | **魔改对照仅限 IP 衍生/无限流/诸天流** — 其他类型不强制，但如有改编痕迹必须分析 | [ ] |
| ❌5 | **不准用梗概代替逐章阅读** — Phase A 必须逐章读原文 | [ ] |
| ❌6 | **名场面 image_prompt 必须写** — 这是 pop-html-anything 生图的直接输入 | [ ] |
| ❌7 | **产出只留摘要** — 写入笔记/YAML 后对话中不粘贴完整产出。说"已写入 {路径}。摘要：{卷数×章节数}，核心发现：{一句}。需展开告诉我。" |

## ❌ 错误示例

### WRONG 1：跳过 Phase 0 直接读正文

```
用户：帮我拆这本《无限恐怖》
Agent：直接开始读第一章写笔记
```
❌ 错误：没有先定分卷策略，直接进入阅读，中途才发现分卷不合理需要重来。
✅ 正确：先通读目录定分卷 → 抽样验证 → 用户确认 → 再开始读。

### WRONG 2：只有叙事笔记，没有结构线 YAML

```
Agent：写了 3000 字的阅读笔记，但没有产出结构化数据 YAML
```
❌ 错误：双格式缺一不可。没有 YAML，pop-html-anything 无法消费。
✅ 正确：读完一卷后，先写叙事笔记 → 再逐章做 YAML 结构化标注 → 两个一起交付。

### WRONG 3：角色表中遗漏 image_prompt

```
Agent：角色表只写了出场章节和重要性，没有写肖像 prompt
```
❌ 错误：image_prompt 是 pop-html-anything 生图的直接输入，不可省略。
✅ 正确：每个重要角色必须写 image_prompt，结构为 `[内容类型]: [视觉描述], [氛围], high quality digital illustration, cinematic lighting`。

### WRONG 4：用梗概代替逐章阅读

```
Agent：看了一遍章节目录和梗概 → 直接写笔记
```
❌ 错误：没有逐章阅读原文就产出笔记，遗漏大量细节和信号。
✅ 正确：Phase A 必须逐章阅读原文，每 3-5 章产出摘要 + 执行观察。

### WRONG 5：跨卷关联写成了单卷总结

```
Agent：Phase B 产出的跨卷关联笔记 = 把各卷笔记拼在一起
```
❌ 错误：跨卷关联应该是因果链、伏笔线、剧情线在全书层面的打通，不是单卷摘要的拼接。
✅ 正确：按因果链/伏笔线/剧情线/书特有机制重新组织，体现跨卷的因果关系。

## 异常与边界条件

| # | 场景 | 触发条件 | 处理动作 |
|:-:|:-----|:---------|:---------|
| 1 | 小说文件不存在或格式不支持 | Phase 0 入口检查发现无可用文件 | 提示用户提供 TXT/EPUB/PDF 格式文件，**不退回到编造数据** |
| 2 | 小说篇幅过长（>500 章） | Phase 0 发现总章数超过 500 | 建议用户先选前 N 卷或前 200 章重点阅读，剩余部分按需选读，避免 token 耗尽 |
| 3 | 分卷方案用户不满意 | Phase 0.4 用户认为分卷不合理 | 返回 Phase 0.1 重新通读目录定分卷，调整后再次提交确认 |
| 4 | 章文件残缺（跳章/乱序） | 阅读时发现章节编号不连续 | 按现有章节继续，在笔记中标注"章节不连续，疑有缺失" |
| 5 | image_prompt 生成不符合预期 | pop-html-anything 渲染后发现肖像/名场面不匹配 | 回到结构化 YAML 修订 image_prompt，调整视觉描述和氛围关键词 |
| 6 | Phase B 发现矛盾（伏笔未回收） | 跨卷关联时发现前面卷的伏笔在后续卷未被处理 | 在跨卷关联笔记中标注"未回收伏笔清单"，不做主观修正 |
| 7 | 用户中途要求换书 | Phase A 已开始但用户要求拆另一本 | 先完成当前卷的笔记和 YAML 产出，断点保存后切换到新书。回到第一步重新开始 |
| 8 | 多卷并行阅读（用户要求一次性读 3 卷） | 用户要求不按卷确认直接读多卷 | 解释单卷确认的必要性——每卷有独立的质量门禁，不确认不退进；如坚持，标注"用户要求跳过单卷确认" |
