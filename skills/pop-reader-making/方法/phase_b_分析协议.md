# Phase B 分析协议 v1.0

> 跨卷关联阶段的结构化产出标准。
> 适用于 Phase B 跨卷关联（多卷读完后打通全貌）时使用。

---

## 一、定位

Phase B 的原始定义是"跨卷关联"，但只有模糊方向没有结构化产出标准。本协议为 Phase B 增加**可复用的 4 类分析产物模板**，让跨卷关联的产出质量可预期、可迭代。

### 核心纪律

| 纪律 | 说明 |
|:----|:------|
| **不捏造** | 所有分析结论必须基于阅读笔记或原文可验证 |
| **不感想** | 不允许"这段写得好""这章很精彩"等评价 |
| **标注来源** | 每条分析必须标注章节号 |

---

## 二、四类 Phase B 产出

### 2.1 因果链 (CausalChain)

**定义**：一组按因果顺序排列的事件，A 导致 B，B 导致 C。

```yaml
- id: "CC-001"
  name: "李项平之死 → 李通崖性格转变 → 李通崖坐化"
  chain:
    - step: 1
      ch: "第100章"
      summary: "李项平被箓巫咒杀，李家失去重要战力"
    - step: 2
      ch: "第150-200章"
      summary: "李通崖从激进转为保守，不再冒险"
    - step: 3
      ch: "第299章"
      summary: "李通崖为家族拼死强攻忿怒摩诃，道途断绝"
    - step: 4
      ch: "第320章"
      summary: "李通崖伤势恶化，坐化身亡"
  significance: "贯穿全书前半部的核心人物弧线"
```

**产出规则**：
- 每条必须有 `step` 序号和 `ch` 章节
- 链条长度不限，但必须保证每一步之间都有合理的因果逻辑

### 2.2 伏笔线 (ForeshadowingLine)

**定义**：早期埋设、后期揭晓的信息线索。

```yaml
- id: "FS-001"
  name: "法鉴碎片"
  setup:
    - ch: "第1章"
      summary: "法鉴正面碎裂成七八块，靠铁箍箍着"
    - ch: "第85章"
      summary: "李通崖意外发现第一块碎片，修为暴涨"
  payoff:
    - ch: "第377章"
      summary: "法鉴碎片全部集齐，揭示完整能力"
  status: "已揭晓"  # 已揭晓/待揭晓/部分揭晓
```

**产出规则**：
- 必须区分 `setup`（铺设）和 `payoff`（揭晓）
- 如果读了但没读到 pay off，标注"待揭晓"

### 2.3 剧情线 (PlotThread)

**定义**：贯穿多章的独立叙事线，可能有多个参与方。

```yaml
- id: "PT-001"
  name: "宗泉岛争夺"
  participants: ["李家", "袁家", "散修联盟"]
  ch_range: "第400-450章"
  key_events:
    - ch: "第400章" ; summary: "李家发现宗泉岛灵脉"
    - ch: "第415章" ; summary: "袁家宣称主权，两家对峙"
    - ch: "第448章" ; summary: "李家以联姻换取袁家退出"
  result: "李家实际控制宗泉岛，袁家获得贸易权"
  significance: "展示李家从武力到外交的实力跃升"
```

**产出规则**：
- 一条剧情线的跨度至少 10 章
- `result` 不可为空

### 2.4 书特有机制 (UniqueMechanism)

**定义**：这本书独有的世界观规则或系统。

```yaml
- id: "UM-001"
  name: "仙基变异"
  category: "修炼体系"
  first_hint_ch: "第377章"
  description: |
    筑基时凝聚的仙基并非固定。在特定条件下（如融合特殊灵气、
    吞噬异种能量），仙基可能发生变异，产生预期外的效果。
  examples:
    - char: "李渊蛟"
      mutated_to: "泾龙王"
      trigger: "吸收湖中金秋灵气"
      ch: "第377章"
  significance: "打破了筑基即定型的传统设定"
```

**产出规则**：
- 只有这本书"独有"的机制才归此类
- 每个机制必须附原文可验证的例子

## 三、工作流

```
Step 1: 全表扫描
  ├─ 翻阅已有阅读笔记，标记明显的事件模式
  └─ 记录直觉（什么让你觉得"这两件事有关联"）

Step 2: 因果链搜索
  ├─ 找"事件 A → 事件 B"的关联对
  └─ 确认 A 在章节上早于 B，回原文确认因果成立

Step 3: 伏笔线搜索
  ├─ 找"早期出现、后期才有意义"的信息
  └─ 标注未揭晓的伏笔（继续追后续卷）

Step 4: 剧情线梳理
  ├─ 识别独立叙事线
  └─ 确定参与方、起止范围、结果

Step 5: 书特有机制发现
  ├─ 找"这本书和别人不一样"的设定
  └─ 附原文可验证的例子
```

## 四、产出文件

产出放在 `_小说项目/<书名>/` 目录下（如需要）：

```
├── phase_b/
│   ├── 因果链.yaml
│   ├── 伏笔线.yaml
│   ├── 剧情线.yaml
│   └── 书特有机制.yaml
```

```yaml
# 文件: 因果链.yaml
# 阅读范围: 第1-500章
# 分析日期: 2026-06-02
```
