# Pop 拆书方法论 v0.5

> v0.5 改版：Phase A 从「机械提取」重构为「按卷深度阅读」；Phase C 改为按需。

---

## 一、流程全景：五段式

```
Phase 0 ─ 书稿预检（先定分卷，再定拆法）
  ├─ Step 0.1：通读目录 → 定分卷草案
  ├─ Step 0.2：密集抽样验证分卷
  ├─ Step 0.3：产出本书价值判断 Top 5
  ├─ Step 0.4：产出拆书方案
  └─ Step 0.5：用户确认方向  ← 第一道门禁

Phase A ─ 按卷深度阅读（保持叙事完整性）
  ├─ Step A.1：逐章阅读 → 产出分卷阅读笔记
  ├─ Step A.2：产出关键实体列表
  └─ Step A.3：用户确认质量 → 进入下一卷

Phase B ─ 跨卷关联（多卷读完后打通全貌）
  ├─ 跨卷角色轨迹串联
  ├─ 跨卷设定演变
  ├─ 宏观叙事结构
  └─ 伏笔追踪

Phase B+ ─ 方案复盘验证
  ├─ 对比 Phase 0 分卷 vs 阅读后的实际感受
  └─ 如有偏差，更新分卷后再次确认  ← 第二道门禁

Phase C ─ 按需结构化（问过"用户需要吗"才做）
  → 如果不需要：阅读笔记即交付
  → 如果需要：从已有笔记生成 DB + HTML
```

> ⚠️ **铁律：Phase 0 未经确认不得进 Phase A。Phase B+ 未经确认不得进 Phase C。**

---

## 二、核心认知：为什么废掉机械提取？

### 经验教训

之前的 Phase A 是把小说切成 nouns/events/relationships 三个碎片池，期望用数据库拼回去。但这套方案对手上有 19 个坑点，根本问题只有一个：

**你拆得越碎，读者越看不懂。**

一本长篇小说不是"角色清单 + 事件列表 + 关系连线"就能还原的。叙事节奏、伏笔链、角色弧光——这些必须在阅读的连贯性中才能被感知。碎片化提取把这一切都摧毁了。

### 新思路

读的时候像一个人一样读。写笔记的时候像一个人一样写。

结构化只有在两个条件都满足时才做：
1. 角色/设定多到需要检索
2. 用户明确说要

## 四、Phase A — 按卷深度阅读（v0.5 修订版）

### 4.1 核心原则

- **保持叙事完整性** — 像一个人读一本书一样读
- **核心任务不是记录"发生了什么"，而是回答"写得怎么样、魔改了什么"**
- **写笔记不写数据** — 用自然语言，禁止YAML/标签/结构化标注
- **每卷读完即确认** — 不给用户闷头读完 500 章的机会

### 4.2 产出格式

每卷产出 `{书名}_卷{序号}_{卷名}_笔记.md`，包含四部分：

#### 第一部分：魔改对照

判断这卷是否需要魔改对照（仅IP衍生/无限流/诸天流类小说需要）。如果需要，回答三个问题：
1. **这卷借用了什么IP原作？**
2. **作者改了什么？**（设定、角色命运、力量体系、剧情走向）
3. **改得怎么样？**（效果判断，不是列差异清单）

#### 第二部分：章节叙事流（★ 核心）

每3-5章为一节。每节包含两段：
- **【摘要】** 发生了什么（1-3句）
- **【☝️ 执行观察】** 这写法怎么样？（1-2句判断，必须写）

禁止批量压缩超过5章。

执行观察类型：节奏判断、魔改效果、信息密度、结构评价、作者意图、阅读体验。

#### 第三部分：关键实体列表（精简版）

只记重要角色。IP原作角色标注魔改状态。去掉肖像prompt、出场章节数、共现矩阵、情绪曲线等冗余内容。

#### 第四部分：本卷质量判断

```
阅读体验：[流畅/拖沓/起伏大]
魔改质量：[精彩/一般/生硬]  ← 核心指标
最佳段落：____（哪几章写得最好，为什么）
最弱段落：____（哪几章最拉胯，为什么）
一句话总结：____
```

### 4.3 每卷确认

读完一卷后将笔记呈现给用户。用户确认质量后进入下一卷。

## 五、Phase B — 跨卷关联（v0.5 全新）

至少有 2-3 卷阅读笔记产出后触发。

产出 `{书名}_跨卷关联笔记.md`：

- **跨卷角色轨迹**：同一角色在不同卷的状态变化表
- **跨卷设定演变**：关键设定在多卷中的展开
- **宏观叙事结构**：分卷之间的衔接方式
- **伏笔/未解问题**：尚未回收的伏笔

> 如需结构化产出（因果链、伏笔线、剧情线、书特有机制），详见 `phase_b_分析协议.md`。

---

## 六、Phase B+ — 方案复盘验证

对比 Phase 0 分卷 vs 阅读后的实际感受。

三种结果：
- ✅ 分卷准确 → 进 Phase C
- ⚠️ 边界偏差 → 微调分卷
- ❌ 重大偏差 → 回到 Phase A 补读

---

## 七、Phase C — 按需结构化（v0.5 改为可选）

### 判断标准

问：把笔记变成数据库/HTML，对用户有实质帮助吗？

回答"有"：
- 角色极多需要检索
- 设定复杂需要交叉查询
- 用户明确说要

回答"没有"：
- 书不长、角色少
- 阅读笔记已经够用

### 如做结构化

从已有笔记生成，不做"从原文重新提取"。

## 八、效果验证标准

| 标准 | 说明 |
|:----|:----|
| ✅ Phase 0 分卷定界已完成（有抽样验证） | 核心产出 |
| ✅ Phase A 阅读笔记保持章节顺序 | 不跳章、不打乱 |
| ✅ 每卷已确认质量 | 用户确认后才进下一卷 |
| ✅ Phase B+ 复盘已做 | 对比分卷 vs 实际感受 |
| ✅ Phase C 按需判断 | 问了才做，不做也行 |
| ✅ 不依赖二手资料 | 所有信息来自原文 |
