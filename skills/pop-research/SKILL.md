## 定位

pop-research 是管线的**独立调研 skill**，三种主流使用场景：

| 调用方 | 触发时机 | 档位 | 产出 |
|:--|:--|:--|:--|
| **pipeline Phase 0** | 项目初始化，自动触发 | **赛道定位调研** | `写作参考/赛道调研.md` |
| **pipeline Phase 0** | 用户给了力量体系参考书 | **decon-lite** | `写作参考/decon-lite-{书名}.md` |
| write | 写作质感缺口 | **采风** | `写作参考/知识沉淀/{主题}-采风.md` |
| seed/plot（深层需求） | 需要世界观/压力矩阵/行业深度 | **三界框架**（可选） | `写作参考/知识沉淀/{主题}.md` |

**为什么重构**：旧版 v1.4.0 310 行中 309 行是三界框架，但它被调用的频次远低于赛道定位调研和 decon-lite。新版把高频场景提到前面，三界框架降为"有需要才用"的深层选项。

# 主流程 · 赛道定位调研

> **触发**：pipeline Phase 0 Stage 2 自动派发子agent。所有项目必有。
> **产出**：`写作参考/赛道调研.md`。消费方：seed 1d 双轨发散（王道赛道借鉴+避雷）+ plot 三源合流（赛道参考第一源）。

## S1 · 接收需求

输入：
- 赛道方向（来自 `写作参考/用户意图.md`）
- 用户已给的参考书信息（如有，需要在调研中标注避免撞车）
- 用户指定对标书（如有）

## S2 · 四轮搜索

**每轮不是"搜一下"，是搜到底（至少看前 5 条结果的摘要+内容）：**

### 第1轮：赛道格局

```
目标：摸清这个赛道当前什么在火、什么在衰落

搜索词：
1. "{赛道} 番茄小说 排行榜 热销 2026" — 找 top 5-8 + 核心卖点
2. "{赛道} 起点 精品 万订 2026" — 补起点 top 3（如果番茄不够）

对每本 top 书提取可迁移核心卖点（1句话，不抄设定）：
| 排名 | 书名 | 核心卖点（为什么读者买单） | 可迁移方向 |
```

### 第2轮：读者偏好

```
目标：这个赛道的读者在找什么、骂什么

搜索词：
1. "{赛道} 小说 读者 推荐 出圈 2026" — B站/抖音/知乎上的推荐+讨论
2. "{赛道} 小说 不爽 雷点 审美疲劳 2026" — 读者喷什么、避雷什么
3. "{赛道} 爽点 名场面 好评 2026" — 名场面 = 核心爽点的具体化

对每轮至少看前 5 条的内容摘要，产出借鉴点和避雷点：
- 借鉴：什么爽点/机制/节奏在读者中形成了共识好评
- 避雷：什么设定/金手指/开局模式已经被喷烂了
```

### 第3轮：赛道演变

```
目标：这个赛道正在往哪个方向变

搜索词：
1. "{赛道} 新题材 新趋势 2026" — 冒头的新方向
2. "{赛道} 传统题材 衰落 无人问津 2026" — 已经在走下坡的老模式
3. "{赛道} 细分 垂直 蓝海 2026" — 大品类下的细分赛道

产出：细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |
```

### 第4轮：对标分析（如有用户指定对标书）

```
目标：用户指定的对标书在赛道中的位置

对每本用户指定的对标书：
1. 搜 "{书名} 作者 风格 读者评价"
2. 判断这本书在赛道的细分位置（是哪个方向的标杆？读者的核心认可点？）
3. 确认本项目不会和它撞车（设定/金手指/开局模式）
```

## S3 · 产出落盘

落盘到 `写作参考/赛道调研.md`：

```markdown

## 一、赛道格局
### top作品核心卖点表
| 排名 | 书名 | 核心卖点 | 可迁移方向 |

### 赛道盘子

- 主要细分方向

## 二、借鉴点与避雷点

1. {机制/爽感} — 来源：{书名/趋势}
2. ...

### 避雷点（已被喷烂的模式/让读者疲劳的老套路）
1. {设定/模式} — 来源：{书名/趋势}
2. ...

## 三、细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |

## 四、对标分析（如有用户指定对标书）
| 对标书 | 细分位置 | 读者核心认可 | 本项目避免撞车策略 |
```

## S4 · 质量门禁

| 检查项 | 合格标准 |
|:--|:--|
| top作品≥5本 | 有书名列卖点 |
| 借鉴点≥3条 | 每条有来源 |
| 避雷点≥3条 | 每条有来源 |
| 细分格局≥3个方向 | 含热度+饱和度判断 |
| 落地建议有具体内容 | 不是"注意差异化"这种废话 |

# 流程 · decon-lite 增强版

> **触发**：pipeline Phase 0 Stage 2。用户给了力量体系参考书。
> **产出**：`写作参考/decon-lite-{书名}.md`。消费方：plot 三源合流（第二源）+ seed 1d-0改编策略层 + seed 故事纲领（节奏参考）。

**v2.1.0升级**：旧版5表拆解深度不足——表1只到"描述级"（这本书有什么），没到"规则级"（规则怎么运转），下游world消费时只有骨架没有规则。新增表6-8补势力/金手指/危机三个维度，服务seed 1d-0改编策略层。所有表字段用**抽象结构**（"这本书有几层X"），不用具体内容模板（"4层势力"）。

## DL1 · 接收需求

输入：书名 + 本地txt路径 + 采样范围（默认前 300 章）

## DL2 · 八表拆解

### 表1：力量体系结构（规则级深度）

> **v2.1.0核心升级**：从描述级升级到规则级。引入"主养成线+子养成线"抽象——主养成线=位阶（区分境界的核心刻度），子养成线=位阶内的平行成长维度（每本书的子养成线数量和内容不同，由书决定，不固定字段）。

| 字段 | 说明 | 必须给出具体内容 | 抽象要求 |
|:--|:--|:--|:--|
| 力量体系类型 | 等级制/修炼境/规则系/百分比/职业树/... | 有明确判断理由 | 通用字段 |
| **主养成线（位阶）** | 这本书的位阶名称/数量/每位阶的章节范围/每位阶的质变标志 | 至少覆盖前4阶，每阶标注质变标志 | 抽象结构：列位阶名+章范围+质变标志 |
| **子养成线清单** | 这本书有几条平行的子养成线？每条叫什么？（数量由书决定） | 列出所有子养成线名称 | 抽象结构：先识别数量再列名称 |
| **子养成线1规则** | 第1条子养成线的获取规则+消耗规则+上限+与主养成线的关系 | 有具体数值/条件+原文chXX出处 | 逐条拆，不固定字段名 |
| **子养成线2规则** | 第2条... | 同上 | 同上 |
| **子养成线N规则** | 第N条...（N由这本书决定） | 同上 | 同上 |
| **子养成线交叉规则** | 子养成线之间怎么互相影响？有没有兼职惩罚/互相制约/互相增益？ | 有具体规则+原文出处 | 抽象结构：列交叉关系 |
| 升级触发 | 主角靠什么行为升级？和子养成线什么关系？ | 写具体行为，不写"打怪升级" | 通用字段 |
| 强度膨胀 | 位阶间的实力差距多大？线性/跳跃/指数？ | 有具体对比（如LV1可以1v1街头混混，LV5可以1v15） | 通用字段 |
| 破局手段演变 | 不同位阶主角各靠什么赢？ | 至少覆盖3个位阶 | 通用字段 |
| 设计亮点 | 最值得本项目借鉴的结构性设计是什么？ | 1句话讲清楚 | 通用字段 |

**拆解要点**：
- 子养成线数量不固定——深渊主宰可能有5条（职业等级/属性点/专长/技能点/兼职），修仙书可能有4条（法力/神识/肉身/法宝），系统流可能有3条（技能树/属性点/装备）。先识别这本书有几条，再逐条拆
- 每条子养成线的规则字段不固定——职业等级拆"等级上限/进阶前置/兼职方向"，法力拆"增长方式/容量上限/恢复机制"。由这条子养成线的性质决定拆什么
- **数值锚点**：每条规则标注原文chXX出处，便于world消费时查证

### 表2：剧情节奏

| 字段 | 说明 | 必须给出具体数值 |
|:--|:--|:--|
| 爽点间隔 | 大爽点（高潮/打脸/重大反转）间隔？小爽点（日常打脸/小突破）间隔？ | 具体章数 |
| 钩子分布 | 章末钩子以什么类型为主？分前/中/后期变化 | 分类型占比 |
| 场景切换 | 同一场景最久持续几章？战斗/日常/博弈各多久？ | 具体章数范围 |
| Boss铺垫 | 新Boss从提到到正面交手间隔？铺垫手法？ | 具体章数+铺垫方式 |
| 单元结构 | 有没有"几章一个小闭环"的单元剧模式？ | 如有，标注小闭环的章节范围和闭环方式 |

### 表3：情绪弧线（新增）

| 字段 | 说明 | 必须给出具体数值 |
|:--|:--|:--|
| 情绪高低潮分布 | 前50章高潮/低谷的分布模式 | 每10章标注情绪高点位置 |
| 压抑→释放节奏 | 加压多久释放一次？（如"压4章爆1章"） | 具体章数规律 |
| 泪点燃点 | 情感高潮的类型和触发方式 | 至少标注 2 个 |
| 幽默缓解 | 高强度段落的"出气口"怎么设计？ | 战斗后/重大事件后有没有日常/搞笑/温馨缓冲 |

### 表4：人称与视角策略（新增）

| 字段 | 说明 |
|:--|:--|
| 基础人称 | 第几人称？（1st/3rd 限知/3rd 全知/混用） |
| POV切换 | 有没有多POV？切换频次？切换时的信号（章节分隔/空行/视角锚点词） |
| 视角距离 | 近距（跟着主角的感官走）/ 中距（能看到主角但也能看别处）/ 远距（全知视角） |
| 内心戏处理 | 怎么展现主角内心？直接内心独白/行动外化/对话暗示？ |
| 值得借鉴 | 这个人称策略对你的项目有什么参考价值？ |

### 表5：开篇解剖（新增）

| 字段 | 说明 |
|:--|:--|
| 第1章第1句 | 原句摘录→分析：这一句建立了什么期待？ |
| 前3章埋了什么 | 悬念/钩子/核心矛盾各是什么？哪个最早被读者接收到？ |
| 前3章兑现了什么 | 有没有让读者看到"已经在路上"的信号？ |
| 金手指激活章 | 第几章激活？激活方式是什么？（开局就有 / 事件触发 / 隐藏但已生效） |
| 开篇节奏标签 | 用5个字概括——"冷开人文感"/"热开直给杀戮"/"悬疑慢炖" |

### 表6：势力格局（v2.1.0新增）

> 服务seed 1d-0改编策略层的"势力替换"维度。字段用抽象结构——拆"这本书的势力分层"，不写死"4层"。

| 字段 | 说明 | 抽象要求 |
|:--|:--|:--|
| 势力分层结构 | 这本书的势力分几层？每层叫什么？（层数由书决定） | 先识别层数再列名称 |
| 每层势力具名 | 每层有哪些具名势力/组织？主角和每层什么关系？ | 列具名势力+主角关系 |
| 层间矛盾 | 层与层之间的核心矛盾是什么？同层内的矛盾是什么？ | 列矛盾关系 |
| 势力博弈关系 | 势力之间怎么博弈？同盟/对抗/利用/制衡？ | 列博弈类型 |
| 主角在势力格局中的位置 | 主角起始在哪层？终点到哪层？靠什么往上爬？ | 标注起点+终点+路径 |
| 设计亮点 | 最值得借鉴的势力结构设计是什么？ | 1句话 |

**拆解要点**：
- 势力层数不固定——深渊主宰可能是帮派/工会/教会/神殿/诸神5层，修仙可能是家族/宗门/王朝/天道4层，都市可能是街头/帮派/商会/政界4层。先识别这本书有几层，再逐层拆
- 服务1d-0策略层：用户选"势力替换"策略时，这张表提供"借什么势力结构、换什么势力"的素材

### 表7：金手指深度（v2.1.0新增）

> 服务seed 1d-0改编策略层的"金手指形式"维度+world力量体系设计的金手指耦合。

| 字段 | 说明 | 抽象要求 |
|:--|:--|:--|
| 金手指类型 | 这本书的金手指属于什么类型？（重生记忆/系统面板/血脉觉醒/神器绑定/...） | 有明确判断 |
| 机制 | 金手指具体能做什么？核心功能是什么？ | 列核心功能 |
| 限制 | 金手指有什么使用限制？（冷却/消耗/条件/副作用） | 列所有限制条件+原文出处 |
| 代价 | 使用金手指要付出什么代价？（身体/精神/时间/因果） | 列代价+原文出处 |
| 成长路径 | 金手指本身会成长吗？成长条件是什么？ | 列成长阶段+条件 |
| 与主养成线耦合 | 金手指加速哪条主养成线/子养成线？还是开了一条新养成线？ | 标注耦合关系 |
| 与世界观耦合 | 金手指和世界的力量体系/规则有什么关系？是世界内的还是外挂？ | 标注耦合程度 |
| 设计亮点 | 最值得借鉴的金手指设计是什么？ | 1句话 |

**拆解要点**：
- "与主养成线耦合"是关键——深渊主宰的金手指（重生记忆）不直接给属性，而是"知道怎么练"，加速的是职业等级这条子养成线。修仙书的金手指可能直接给法力，加速主养成线。耦合方式不同，设计思路不同
- 服务1d-0策略层：用户选"金手指形式"策略时，这张表提供"借什么金手指机制、改什么限制/代价"的素材

### 表8：世界危机引擎（v2.1.0新增）

> 服务seed 1d-0改编策略层的"世界危机"维度+world危机设计的结构参考。

| 字段 | 说明 | 抽象要求 |
|:--|:--|:--|
| 危机分层结构 | 这本书的危机分几层？每层叫什么？（层数由书决定） | 先识别层数再列名称 |
| 终极危机 | 全书最终的大危机是什么？驱动全书的核心矛盾？ | 1-2句话概括 |
| 阶段性危机 | 每卷/每阶段的危机是什么？和终极危机什么关系？ | 列阶段危机+与终极危机关系 |
| 危机驱动机制 | 危机怎么驱动剧情持续运转？倒计时/衰减/惩罚/递进？ | 列驱动机制类型 |
| 危机与主角关系 | 主角和每层危机什么关系？是被动卷入还是主动对抗？ | 标注关系类型 |
| 危机升级路径 | 危机怎么升级？从小危机到终极危机的 escalation 路径？ | 列升级路径 |
| 设计亮点 | 最值得借鉴的危机引擎设计是什么？ | 1句话 |

**拆解要点**：
- 危机层数不固定——深渊主宰可能是剥皮案(卷1)/上古红龙(卷2)/诸神黄昏(全书)3层，修仙可能是宗门大比(卷1)/域外天魔(卷2)/天道崩塌(全书)3层，末日流可能是丧尸潮(卷1)/变异体(卷2)/文明覆灭(全书)3层。先识别这本书有几层危机，再逐层拆
- 服务1d-0策略层：用户选"世界危机"策略时，这张表提供"借什么危机引擎、换什么危机"的素材

## DL3 · 质量门禁

| 检查项 | 合格标准 |
|:--|:--|
| 8张表全部完整 | 每张表字段齐全，不缺项 |
| 表1力量体系规则级深度 | 主养成线≥4阶 + 子养成线≥2条（每条有规则+chXX出处）+ 交叉规则有 |
| 表6势力格局分层 | ≥2层具名势力 + 层间矛盾有 |
| 表7金手指深度 | 机制/限制/代价/成长路径四项齐全 + 与主养成线耦合关系明确 |
| 表8世界危机引擎 | ≥2层危机 + 终极危机明确 + 驱动机制有 |
