# pop-research · 独立调研 skill

> v2.0.0：聚焦管线真实使用场景。主流程：赛道定位调研（pipeline Phase 0自动触发）+ decon-lite增强（5表拆解）+ 采风（write触发）。三界框架降为深层调研可选档位。
> 被pipeline Phase 0 / seed / plot / write 按需调用

---

## 定位

pop-research 是管线的**独立调研 skill**，三种主流使用场景：

| 调用方 | 触发时机 | 档位 | 产出 |
|:--|:--|:--|:--|
| **pipeline Phase 0** | 项目初始化，自动触发 | **赛道定位调研** | `写作参考/赛道调研.md` |
| **pipeline Phase 0** | 用户给了力量体系参考书 | **decon-lite** | `写作参考/decon-lite-{书名}.md` |
| write | 写作质感缺口 | **采风** | `写作参考/知识沉淀/{主题}-采风.md` |
| seed/plot（深层需求） | 需要世界观/压力矩阵/行业深度 | **三界框架**（可选） | `写作参考/知识沉淀/{主题}.md` |

**为什么重构**：旧版 v1.4.0 310 行中 309 行是三界框架，但它被调用的频次远低于赛道定位调研和 decon-lite。新版把高频场景提到前面，三界框架降为"有需要才用"的深层选项。

# 主流程 · 赛道定位调研

> **触发**：pipeline Phase 0 Stage 2 自动派发子agent。所有项目必有。
> **产出**：`写作参考/赛道调研.md`。消费方：seed 1d 双轨发散（王道赛道借鉴+避雷）+ plot 三源合流（赛道参考第一源）。

## S1 · 接收需求

输入：
- 赛道方向（来自 `写作参考/用户意图.md`）
- 用户已给的参考书信息（如有，需要在调研中标注避免撞车）
- 用户指定对标书（如有）

## S2 · 四轮搜索

**每轮不是"搜一下"，是搜到底（至少看前 5 条结果的摘要+内容）：**

### 第1轮：赛道格局

```
目标：摸清这个赛道当前什么在火、什么在衰落

搜索词：
1. "{赛道} 番茄小说 排行榜 热销 2026" — 找 top 5-8 + 核心卖点
2. "{赛道} 起点 精品 万订 2026" — 补起点 top 3（如果番茄不够）

对每本 top 书提取可迁移核心卖点（1句话，不抄设定）：
| 排名 | 书名 | 核心卖点（为什么读者买单） | 可迁移方向 |
```

### 第2轮：读者偏好

```
目标：这个赛道的读者在找什么、骂什么

搜索词：
1. "{赛道} 小说 读者 推荐 出圈 2026" — B站/抖音/知乎上的推荐+讨论
2. "{赛道} 小说 不爽 雷点 审美疲劳 2026" — 读者喷什么、避雷什么
3. "{赛道} 爽点 名场面 好评 2026" — 名场面 = 核心爽点的具体化

对每轮至少看前 5 条的内容摘要，产出借鉴点和避雷点：
- 借鉴：什么爽点/机制/节奏在读者中形成了共识好评
- 避雷：什么设定/金手指/开局模式已经被喷烂了
```

### 第3轮：赛道演变

```
目标：这个赛道正在往哪个方向变

搜索词：
1. "{赛道} 新题材 新趋势 2026" — 冒头的新方向
2. "{赛道} 传统题材 衰落 无人问津 2026" — 已经在走下坡的老模式
3. "{赛道} 细分 垂直 蓝海 2026" — 大品类下的细分赛道

产出：细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |
```

### 第4轮：对标分析（如有用户指定对标书）

```
目标：用户指定的对标书在赛道中的位置

对每本用户指定的对标书：
1. 搜 "{书名} 作者 风格 读者评价"
2. 判断这本书在赛道的细分位置（是哪个方向的标杆？读者的核心认可点？）
3. 确认本项目不会和它撞车（设定/金手指/开局模式）
```

## S3 · 产出落盘

落盘到 `写作参考/赛道调研.md`：

```markdown
# 赛道定位调研（赛道：{用户方向}）

> pop-research 赛道定位调研 · {timestamp}
> 消费方：seed 1d双轨发散 + plot 三源合流
> 用户意图：{引用写作参考/用户意图.md中的赛道+标签}
> 用户对标书：{书名}（如有）

## 一、赛道格局
### top作品核心卖点表
| 排名 | 书名 | 核心卖点 | 可迁移方向 |

### 赛道盘子
- 总体：{热度判断}
- 主要细分方向
- 正在衰落的老模式

## 二、借鉴点与避雷点
### 借鉴点（已验证的爽感机制/节奏模式/读者共识好评）
1. {机制/爽感} — 来源：{书名/趋势}
2. ...

### 避雷点（已被喷烂的模式/让读者疲劳的老套路）
1. {设定/模式} — 来源：{书名/趋势}
2. ...

## 三、细分赛道格局
| 细分方向 | 代表作品 | 热度 | 饱和度 | 机会判断 |

## 四、对标分析（如有用户指定对标书）
| 对标书 | 细分位置 | 读者核心认可 | 本项目避免撞车策略 |

## 五、落地建议
- 力量体系方向建议：{从赛道格局推导——赛道主流是什么体系？有什么差异化空间？}
- 剧情节奏建议：{从读者偏好推导——这个赛道的爽点间隔、钩子类型偏好}
- 风险提示
```

## S4 · 质量门禁

| 检查项 | 合格标准 |
|:--|:--|
| top作品≥5本 | 有书名列卖点 |
| 借鉴点≥3条 | 每条有来源 |
| 避雷点≥3条 | 每条有来源 |
| 细分格局≥3个方向 | 含热度+饱和度判断 |
| 落地建议有具体内容 | 不是"注意差异化"这种废话 |

# 流程 · decon-lite 增强版

> **触发**：pipeline Phase 0 Stage 2。用户给了力量体系参考书。
> **产出**：`写作参考/decon-lite-{书名}.md`。消费方：plot 三源合流（第二源）+ seed 故事纲领（节奏参考）。

**增强点 vs 旧版 v1.4.0**：旧版只拆两表（力量结构+剧情节奏），新版加三表：**情绪弧线**（如何让读者又哭又笑）、**人称与视角策略**（第一/三人称怎么切换、POV 如何调度）、**开篇解剖**（前3章到底做了什么让读者留下来）。

## DL1 · 接收需求

输入：书名 + 本地txt路径 + 采样范围（默认前 300 章）

## DL2 · 五表拆解

### 表1：力量体系结构

| 字段 | 说明 | 必须给出具体数值 |
|:--|:--|:--|
| 力量体系类型 | 等级制/修炼境/规则系/百分比/职业树 | 有明确判断理由 |
| 阶位骨架 | 全书有多少个大阶位？每阶的名称和对应章节范围 | 至少覆盖前 4 阶 |
| 升级触发 | 每阶的升级条件是什么？和主角的行为有什么关系？ | 不写"打怪升级"这种废话，写具体行为 |
| 强度膨胀 | 每阶之间的实力差距多大？线性/跳跃/指数？ | 有具体对比（如LV1可以1v1街头混混，LV5可以1v15带武器） |
| 破局手段演变 | 不同阶位主角各靠什么赢？ | 至少覆盖 3 个阶位 |
| 设计亮点 | 最值得本项目借鉴的结构性设计是什么？ | 1句话讲清楚 |

### 表2：剧情节奏

| 字段 | 说明 | 必须给出具体数值 |
|:--|:--|:--|
| 爽点间隔 | 大爽点（高潮/打脸/重大反转）间隔？小爽点（日常打脸/小突破）间隔？ | 具体章数 |
| 钩子分布 | 章末钩子以什么类型为主？分前/中/后期变化 | 分类型占比 |
| 场景切换 | 同一场景最久持续几章？战斗/日常/博弈各多久？ | 具体章数范围 |
| Boss铺垫 | 新Boss从提到到正面交手间隔？铺垫手法？ | 具体章数+铺垫方式 |
| 单元结构 | 有没有"几章一个小闭环"的单元剧模式？ | 如有，标注小闭环的章节范围和闭环方式 |

### 表3：情绪弧线（新增）

| 字段 | 说明 | 必须给出具体数值 |
|:--|:--|:--|
| 情绪高低潮分布 | 前50章高潮/低谷的分布模式 | 每10章标注情绪高点位置 |
| 压抑→释放节奏 | 加压多久释放一次？（如"压4章爆1章"） | 具体章数规律 |
| 泪点燃点 | 情感高潮的类型和触发方式 | 至少标注 2 个 |
| 幽默缓解 | 高强度段落的"出气口"怎么设计？ | 战斗后/重大事件后有没有日常/搞笑/温馨缓冲 |

### 表4：人称与视角策略（新增）

| 字段 | 说明 |
|:--|:--|
| 基础人称 | 第几人称？（1st/3rd 限知/3rd 全知/混用） |
| POV切换 | 有没有多POV？切换频次？切换时的信号（章节分隔/空行/视角锚点词） |
| 视角距离 | 近距（跟着主角的感官走）/ 中距（能看到主角但也能看别处）/ 远距（全知视角） |
| 内心戏处理 | 怎么展现主角内心？直接内心独白/行动外化/对话暗示？ |
| 值得借鉴 | 这个人称策略对你的项目有什么参考价值？ |

### 表5：开篇解剖（新增）

| 字段 | 说明 |
|:--|:--|
| 第1章第1句 | 原句摘录→分析：这一句建立了什么期待？ |
| 前3章埋了什么 | 悬念/钩子/核心矛盾各是什么？哪个最早被读者接收到？ |
| 前3章兑现了什么 | 有没有让读者看到"已经在路上"的信号？ |
| 金手指激活章 | 第几章激活？激活方式是什么？（开局就有 / 事件触发 / 隐藏但已生效） |
| 开篇节奏标签 | 用5个字概括——"冷开人文感"/"热开直给杀戮"/"悬疑慢炖" |

## DL3 · 质量门禁

| 检查项 | 合格标准 |
|:--|:--|
| 5张表全部完整 | 每张表 ≥ 3个字段有具体数值 |
| 力量体系阶位≥4 | 覆盖第一卷范围 |
| 爽点间隔有章数 | 不是"快/慢" |
| 开篇第1句有原文摘录 | 不仅仅是分析，有原句 |
| 采样范围已声明 | "基于前N章采样" |

## DL4 · 存盘+返回

落盘 `写作参考/decon-lite-{书名}.md`，返回路径指针。

# 流程 · 采风

> **触发**：write 写作中遇到质感缺口（如急诊室/黑帮谈判/法庭质证等需要真实质感的场景）。

## C1 · 接收需求

输入：主题 + 需要什么质感 + 感官侧重（可选）

## C2 · 真实质感采集（2-4轮搜索）

| 采集方向 | 搜索目标 | 示例搜索词 |
|:--|:--|:--|
| 一手资料 | 从业者口述/纪录片/纪实文学/行业访谈 | "{职业} 从业者 口述" |
| 感官细节 | 视听嗅触味的描写素材 | "{场景} 声音/气味" |
| 职业黑话 | 行话/俚语/习惯用语 | "{职业} 黑话 术语" |
| 场景物理布局 | 空间结构/流程动线 | "{场所} 内部结构" |

## C3-C4 · 片段整理+质感标注+存盘

每主题 3-8 个片段（50-200字/片段），标注质感标签+感官覆盖+适用场景。
落盘 `写作参考/知识沉淀/{主题}-采风.md`。

**质量门禁**：≥3个可复用片段 | 每片段≥1种感官 | 含质感标签+适用场景

# 流程 · 三界框架（可选深层调研）

> **触发**：seed/plot 在需要世界观深度调研时调用（不是每个项目都需要）。
> **跳过**：赛道定位调研和 decon-lite 是主流程，不需要三界框架打底。

## T1-T2 · 接收需求 + 三界定界

领域 + 具体问题 + 期望深度。三界定界：社会界/自然界/超自然界（可多选）。

## T3 · 分界调研（17维度）

社会界6维度（权力/经济/法律/文化/阶层/组织）、自然界6维度（地理/气候/资源/生态/疾病/灾害）、超自然界5维度（力量体系/规则禁忌/存在层级/组织/历史）。每个维度：有需求→深度调研；无需求→逐类说明理由。

## T4-T6 · 压力源转化 + 交叉困境 + 质量门禁 + 存盘

压力源6字段结构化 + 6种交叉困境类型识别 + 多元素整合。落盘 `写作参考/知识沉淀/{主题}.md`。

---

## 红线

1. **赛道定位调研是必选项**——pipeline Phase 0 自动触发，不做=后续缺赛道约束
2. **decon-lite 5张表必须全拆**——不能只拆2张（旧版2表版本太薄，没覆盖情绪/视角/开篇）
3. **调研结果必须存盘**——所有产出明确路径，禁止只存在 agent 思维里
4. **子agent执行时直接读本文件 + 参考书txt**——不给完整 SOP 塞参数
