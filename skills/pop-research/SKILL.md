# pop-research · 独立调研 skill

> v1.4.0：新增decon-lite档位（轻量拆书——只拆力量体系结构+剧情节奏两表，供plot世界构筑参考）
> 三界框架深度调研+采风级质感采集+decon-lite轻量拆书。被seed/plot/pop-novel/write四环节按需调用

---

## 定位

pop-research 是管线的**独立调研 skill**，不归属任何单一环节：

| 调用方 | 调研需求 | 期望深度 | 产出 |
|:--|:--|:--|:--|
| seed | 世界观+压力矩阵调研 | 种子级 | 写作参考/知识沉淀/{主题}.md + 压力矩阵全量.md |
| plot | 幕纲设计时的场景调研 | 场景级 | 写作参考/知识沉淀/{主题}.md |
| pop-novel | 写作中发现信息缺口 | 缺口级 | 写作参考/知识沉淀/{主题}.md |
| write | 写作质感缺口 | 采风级 | 写作参考/知识沉淀/{主题}-采风.md |
| **seed/plot** | **力量体系参考书拆解** | **decon-lite** | **写作参考/decon-lite-{书名}.md** |

**为什么独立**：WebSearch 的痛点是 agent 临时搜两下就完事，没有深度标准。三界框架保证调研不浮于表面。decon-lite 补完整 decon 太重的问题——用户只想拿一本书做力量体系参考时，不需要跑 4 个 Phase。

---

## 调用方式（被调用方）

```
输入（调用方传入）：
  - 领域：调研涉及的领域（如"中世纪封建社会"/"修仙力量体系"）
  - 具体问题：要回答的核心问题（decon-lite不需要具体问题，传入书名+本地路径即可）
  - 期望深度：种子级（三界全量）/ 场景级（单界深入）/ 缺口级（定向补缺）/ 采风级（真实质感采集）/ decon-lite（轻量拆书）

输出（返回给调用方）：
  - 路径指针：写作参考/知识沉淀/{主题}.md 或 写作参考/decon-lite-{书名}.md
  - 索引已更新标记
  - 质量检查结果
```

# 三界框架全流程

## Step 1 · 接收调研需求

解析调用方传入的需求，确定：领域 + 具体问题 + 期望深度 + 附加参数。记录到产出文件头部。

---

## Step 2 · 三界框架定界

```
三界定义：
  - 社会界：人类社会内部的权力/经济/文化/法律/阶层/组织
  - 自然界：物理环境的资源/气候/地理/生态/疾病/灾害
  - 超自然界：超越物理法则的力量/规则/存在/禁忌

定界输出：
  - 涉及界：{社会界/自然界/超自然界}（可多选）
  - 定界理由：为什么涉及/为什么不涉及
```

---

## Step 3 · 分界调研（17维度）

### 社会界6维度（如涉及）

| 维度 | 调研问题 | 产出 |
|:--|:--|:--|
| 权力结构 | 谁掌权？权力如何分配与传承？ | 权力图谱+权力来源 |
| 经济体系 | 财富如何生产、分配、流通？ | 经济链条+资源节点 |
| 法律制度 | 规则如何制定与执行？谁执法？ | 法律框架+执法缺口 |
| 文化传统 | 主流价值观/禁忌/信仰体系？ | 文化张力点+禁忌清单 |
| 阶层关系 | 社会分层结构？流动规则？ | 阶层金字塔+流动壁垒 |
| 组织势力 | 有组织势力？利益诉求？ | 势力格局+势力关系网 |

### 自然界6维度（如涉及）

| 维度 | 调研问题 | 产出 |
|:--|:--|:--|
| 地理环境 | 地形/地貌/交通条件？ | 地理制约因素 |
| 气候条件 | 气候特征/季节变化/极端天气？ | 气候压力源 |
| 资源分布 | 关键资源分布？稀缺性？ | 资源争夺焦点 |
| 生态系统 | 动植物/生态链/生态平衡？ | 生态威胁来源 |
| 疾病疫情 | 流行病/地方病/医疗条件？ | 疾病压力源 |
| 自然灾害 | 常发灾害类型/频率/破坏力？ | 灾害威胁等级 |

### 超自然界5维度（如涉及）

| 维度 | 调研问题 | 产出 |
|:--|:--|:--|
| 力量体系 | 力量来源/分级/运作机制？ | 力量体系框架 |
| 规则禁忌 | 超自然法则/使用代价/禁忌？ | 禁忌清单+代价机制 |
| 存在层级 | 超自然存在的层级/关系？ | 存在层级图谱 |
| 超自然组织 | 超自然势力组织/利益诉求？ | 超自然势力格局 |
| 超自然历史 | 超自然事件的历史脉络？ | 历史因果链 |

每个维度：有需求→深度调研（多轮搜索）；无需求→逐类说明理由。

---

## Step 3.5 · 资料覆盖率声明

对所有来源做覆盖率标注（小说拆书/wiki/百科/书评/模型常识）：

| 字段 | 含义 |
|:--|:--|
| 资料ID | R-01, R-02... |
| 来源类型 | library拆书/wiki/百科/书评/模型常识 |
| 覆盖范围 | 第一卷/开局/某桥段/全书概述 |
| 可信度 | 高/中/低 |
| 缺口处理 | 补wiki/补目录/声明样本限制 |

若资料只覆盖局部，结论必须写"样本显示"，不得写"该作品整体证明"。

---

## Step 4 · 压力源转化（6字段结构化）

将调研发现转化为结构化压力源：

```
压力源 #N：
  1. 来源界属：社会界/自然界/超自然界
  2. 具体描述：这个压力源是什么
  3. 显性表现：读者可见的具体场景/现象——不是抽象概念
  4. 对主角的压力方式：如何逼迫主角行动
  5. 可量化指标：衡量压力大小/紧迫程度的标准
  6. 交叉可能性：可能与哪些其他压力源产生交叉
```

---

## Step 5 · 交叉困境识别（6种类型）

| 类型 | 说明 |
|:--|:--|
| 社会界内部交叉 | 两个社会界压力源之间的困境 |
| 自然界内部交叉 | 两个自然界压力源之间的困境 |
| 超自然界内部交叉 | 两个超自然界压力源之间的困境 |
| 社会×自然跨界 | 社会界与自然界压力源交叉 |
| 社会×超自然跨界 | 社会界与超自然界压力源交叉 |
| 自然×超自然跨界 | 自然界与超自然界压力源交叉 |

每个交叉困境记录：压力源A × 压力源B / 交叉类型 / 困境描述 / 道德张力 / 可衍生剧情线

---

## Step 5.5 · 多元素整合

1. 元素抽取：每个压力源标明表层事实和底层机制
2. 相似/相反/盲区：为关键压力源补同类变体、反向情境、逻辑缺口
3. 机制转译：说明该事实如何变成小说里的压力/收益/误判/代价/选择
4. 交叉碰撞：识别相似强化、相反冲突、盲区补洞和规则错位
5. 反证：说明哪些调研发现有趣但不适合当前项目

---

## Step 6 · 质量门禁

### 三界框架门禁

| 检查项 | 合格标准 | 失败动作 |
|:--|:--|:--|
| 三界框架已定界 | 涉及界+理由 | 退回Step 2 |
| 每涉及界已分界调研 | 维度覆盖完整 | 退回Step 3 |
| 压力源数量 | ≥3个 | 补充调研 |
| 交叉困境数量 | ≥3个（至少1个跨界） | 补充交叉识别 |
| 每篇知识沉淀字数 | ≥500字+含案例/步骤/数值 | 补充内容 |
| 资料覆盖率已声明 | 局部样本未冒充全篇结论 | 补充覆盖表 |
| 索引.md已更新 | 含新增条目 | 执行存盘 |

---

## Step 7-9 · 存盘+索引+返回

存盘到 `写作参考/知识沉淀/{主题}.md` → 更新索引.md → 返回路径指针给调用方。

# 采风级流程（真实质感采集）

> **跳过三界框架Step1-6**。目标：收集可复用的真实细节片段。

### 采风Step 1 · 接收采风需求

输入：主题（"急诊室"/"黑帮谈判"）+ 需要什么质感 + 感官侧重（可选）

### 采风Step 2 · 真实质感采集（2-4轮搜索）

| 采集方向 | 搜索目标 | 示例搜索词 |
|:--|:--|:--|
| 一手资料 | 从业者口述/纪录片/纪实文学/行业访谈 | "{职业} 从业者 口述" |
| 感官细节 | 视听嗅触味的描写素材 | "{场景} 声音/气味" |
| 职业黑话 | 行话/俚语/习惯用语 | "{职业} 黑话 术语" |
| 场景物理布局 | 空间结构/流程动线 | "{场所} 内部结构" |

### 采风Step 3-4 · 片段整理+质感标注

每主题3-8个片段（50-200字/片段），标注：质感标签 + 感官覆盖 + 适用场景

### 采风Step 5 · 质量门禁（采风级专用）

| 检查项 | 合格标准 |
|:--|:--|
| 可复用片段 | ≥3个 |
| 每片段感官 | 至少1种 |
| 质感标签 | 含标签+适用场景 |
| 片段字数 | 50-200字 |

### 采风Step 6-7 · 存盘+返回

存盘到 `写作参考/知识沉淀/{主题}-采风.md` → 更新索引 → 返回路径指针。

# decon-lite流程（轻量拆书）

> **定位**：完整 decon 太重（4个 Phase）——用户只想拿一本书做力量体系参考时，不需要跑全量拆书。decon-lite 只拆两张表给 plot 三源合流用。
> **触发**：seed 1a 五维摸底时用户选了"力量/体系参考书"→ 调用 pop-research decon-lite。

### decon-lite Step 1 · 接收拆书需求

输入：
- 书名
- 本地txt路径（由 tool-download-webnovel 下载后传入）
- 调用方标注（seed/plot）
- 可选：采样范围（默认前300章，可指定"前30章看开局"或"全书看演变"）

### decon-lite Step 2 · 两表拆解

#### 表1：力量体系结构

从前N章提取力量体系的**结构性信息**（学设计，不抄设定）：

| 字段 | 说明 | 例 |
|:--|:--|:--|
| 力量体系类型 | 等级制/修炼境/规则系/百分比/职业树 | 规则系（命运卡牌式对赌） |
| 阶位结构 | 一共有几层？每层叫什么？ | LV1-20，每4级一个大节点 |
| 阶位触发条件 | 升到下一阶需要什么？（不是打怪升级，是对赌/经营/探索？） | LV1→LV2：完成首次对赌并抽牌。LV4→LV5：接手管理公会 |
| 强度膨胀节奏 | 每阶之间的实力差距多大？是线性还是跳跃？ | LV1-3线性增长（街头混混到小头目），LV4-5跳跃（个人→团队），LV8+指数（接近神力） |
| 破局手段演变 | 主角在不同阶位各靠什么赢？ | 阶位1靠硬刚，阶位2靠信息差，阶位3靠制度漏洞，阶位4靠神性代价 |
| 设计亮点 | 这个力量体系中，最值得本项目借鉴的结构性设计是什么？ | 每阶升级不是靠"打怪"，而是靠"完成一次对赌"——升级触发条件和行为引擎耦合 |

#### 表2：剧情节奏

从采样范围提取节奏信息（学推进模式，不抄剧情）：

| 字段 | 说明 | 例 |
|:--|:--|:--|
| 爽点间隔 | 大爽点间隔章数？小爽点间隔？ | 大爽点20-30章，小爽点3-5章。打脸间隔≤15章 |
| 钩子类型分布 | 章末钩子以什么类型为主？ | 前50章威胁型占60%（派人堵门/暗算/要挟），50章后揭示型增多（发现秘密/系统新功能/身世线索） |
| 场景切换节奏 | 同一场景持续几章后切换？ | 战斗场景≤2章（不拖），日常场景1-3章，势力博弈持续3-5章 |
| Boss出场铺垫 | 新Boss从提到到正面交手间隔几章？ | 灰袍从ch020提到到ch024正面交手，间隔4章。幕间铺垫方式：信使/谣言/手下挑衅 |
| 节奏标签 | 这本书的节奏可以用什么词概括？ | "短章密集战，不打的时候也在算牌" |

### decon-lite Step 3 · 质量门禁

| 检查项 | 合格标准 |
|:--|:--|
| 力量体系类型已判断 | 有明确类型+理由 |
| 阶位结构完整 | 至少提取了前4阶（覆盖第一卷） |
| 破局手段演变已标注 | 至少覆盖3个阶位的对应手段 |
| 爽点间隔有数值 | 有具体章数间隔，不是"快/慢" |
| 钩子类型分布有占比 | 有分类型的占比 |
| 采样范围已声明 | "基于前N章采样" |

### decon-lite Step 4 · 存盘+返回

存盘到 `写作参考/decon-lite-{书名}.md` → 返回路径指针给调用方（seed/plot）。

**产出不进入知识沉淀目录**（decon-lite是结构化拆两表，不是叙事型知识沉淀），索引单独标注 `[decon-lite] {书名} 写作参考/decon-lite-{书名}.md`

## 红线

1. **调研必须用三界框架定界**——禁止无框架直接搜索（采风级和decon-lite走独立流程，免除本条）
2. **每篇知识沉淀≥500字+必须含具体案例/流程步骤/数据数值**——采风级：≥3个可复用片段+感官细节；decon-lite：≥2张完整表+门禁通过
3. **调研结果必须存盘**——到写作参考/知识沉淀/或写作参考/decon-lite-{书名}.md，并更新索引
4. **无需求需逐类说明理由**——每个维度标注"有需求/无需求（理由）"，连续3次无需求标记退化预警
