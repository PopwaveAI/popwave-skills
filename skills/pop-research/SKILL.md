# pop-research · 独立调研 skill v1.0.0

> v3.5涌现式写作管线的通用调研能力。从 pop-novel-seed 的种子搜索法SOP提取为独立skill，三环节共用。核心方法论：三界框架（社会界/自然界/超自然界）深度调研流程 + 公共多元素整合范式。

## skill 定位

pop-research 是 v3.5 管线的**独立调研 skill**，不归属任何单一环节，被三个环节按需调用：

| 调用方 | 调研需求 | 传入参数 | 产出 |
|:-------|:---------|:---------|:-----|
| seed | 世界观+压力矩阵调研 | 书型+核心卖点 | 写作参考/设定/压力矩阵全量.md + 写作参考/知识沉淀/研究档案 |
| plot | 幕纲设计时的场景调研 | 事件brief+涉及领域 | 写作参考/知识沉淀/{主题}.md + 幕纲设定引用指针更新 |
| pop-novel | 写作中发现信息缺口 | 缺口描述 | 写作参考/知识沉淀/{主题}.md + 索引更新 |
| write（采风级） | 写作质感缺口（如某职业/某场景的真实细节） | 主题+需要什么质感 | 写作参考/知识沉淀/{主题}-采风.md + 索引更新 |

**为什么独立**：WebSearch 的痛点是流程问题——agent 临时搜两下就完事，没有深度标准，容易停在浅层。seed 环节的调研之所以好，是因为有完整SOP约束（三界框架+分界调研+交叉困境+质量检查）。v3.5 将这套SOP提取为独立skill，让三个环节都能调用同等的深度调研能力。

## 调用方式

pop-research 是**被调用方**，不主动触发。调用方传入调研需求，pop-research 执行完毕后返回路径指针。

```
调用接口：
  输入（调用方传入）：
    - 领域：调研涉及的领域（如"中世纪封建社会"/"修仙力量体系"/"末世生存"）
    - 具体问题：要回答的核心问题
    - 期望深度：种子级（三界全量）/ 场景级（单界深入）/ 缺口级（定向补缺）/ 采风级（真实质感采集）
    - 附加参数（可选）：书型、核心卖点、事件brief、涉及领域等上下文

  输出（pop-research返回）：
    - 路径指针：写作参考/知识沉淀/{主题}.md（供调用方读取）
    - 索引已更新标记
    - 质量检查结果（压力源数/交叉困境数/是否含案例）
```

## 调研流程（9步）

> **档位分流**：种子级/场景级/缺口级 走三界框架全流程（步骤2~6）；**采风级 跳过步骤2-6，直接走采风流程**（见下方"采风级流程"），产出可复用细节片段，非结构化压力源。

```
1. 接收调研需求（调用方传入：领域+具体问题+期望深度）
   ├─ 种子级/场景级/缺口级 → 走三界框架（步骤2~6）
   └─ 采风级 → 走采风流程（见下方）
2. 三界框架定界（社会界/自然界/超自然界）—— 判断本次调研涉及哪些界
3. 分界调研（社会界6维度/自然界6维度/超自然界5维度）—— 每个涉及到的界执行对应维度调研
4. 压力源转化（6字段结构化）—— 每个调研发现转化为结构化压力源
5. 交叉困境识别（6种交叉类型）—— 识别压力源之间的交叉困境
6. 质量检查（≥3个压力源、≥3个交叉困境、含具体案例）
7. 存盘到写作参考/知识沉淀/{主题}.md
8. 更新写作参考/索引.md
9. 返回路径指针给调用方
```

> 详见 `steps/step-1-research.md`。

## 采风级流程（真实质感采集）

> 与三界框架不同，采风级的目标是**收集可复用的真实细节片段**（医生怎么查房、警察怎么办案、某地市井长什么样、某职业的黑话和习惯），供 write 层渲染质感使用。**不做压力源结构化、不做交叉困境识别**。

```
1. 接收采风需求（主题+需要什么质感）
2. 真实质感采集（2-4轮搜索）：
   · 找一手资料（从业者口述、纪录片片段、纪实文学、行业访谈）
   · 找感官细节（视觉/听觉/嗅觉/触觉/味觉的具体描写素材）
   · 找职业黑话/行话/习惯用语
   · 找典型场景的物理布局和流程
3. 片段整理：每个采风主题整理为3-8个可复用细节片段，每片段50-200字
4. 质感标注：每片段标注可用于什么场景（如"急诊室值班"/"黑帮谈判"/"工地生活"）
5. 存盘到 写作参考/知识沉淀/{主题}-采风.md
6. 更新 写作参考/索引.md
7. 返回路径指针给调用方
```

采风级产出格式：
```markdown
# {主题}采风

> 采风: pop-research · {timestamp}
> 调用方: {write/plot}
> 采风目的: {需要什么质感}

## 可复用细节片段

### 片段1：{场景/行为名称}
- 质感标签：{急诊室值班/黑帮谈判/工地生活...}
- 感官覆盖：{视觉+听觉+嗅觉...}
- 片段内容：{50-200字的真实细节描写}
- 适用场景：{write渲染什么场景时可用}

### 片段2：...
```

> 采风级不做压力源/交叉困境的质量门禁，只检查"是否有≥3个可复用片段+是否标注质感标签+是否含感官细节"。

## ❌ 质量红线

| 编号 | 红线 | 检查方式 |
|:-----|:-----|:---------|
| ❌1 | 调研必须用三界框架定界，禁止无框架直接搜索（采风级除外，走采风流程） | 调研产出含三界定界记录（涉及哪些界+为什么）；采风级含采风流程记录 |
| ❌2 | 每篇知识沉淀≥500字+必须含具体案例/流程步骤/数据数值（采风级：≥3个可复用片段+感官细节） | 字数统计+案例/步骤/数值抽查；采风级抽查片段数+感官覆盖 |
| ❌3 | 调研结果必须存盘到写作参考/知识沉淀/并更新索引 | 文件存在检查+索引.md含新增条目 |
| ❌4 | 无需求需逐类说明理由，连续3次无需求标记退化预警 | 每个维度标注"有需求/无需求（理由）"，累计3次无需求→标记退化预警（采风级不适用） |
