# 调研深度标准

> pop-research v1.0.0 质量保障规范
> 定义调研深度标准、质量检查规则、退化预警机制

---

## 1. 调研深度标准

### 1.1 字数标准

| 产出类型 | 最低字数 | 检查方式 |
|:---------|:---------|:---------|
| 每篇知识沉淀 | ≥500字 | 字数统计 |
| 压力源描述 | 每个压力源≥100字（6字段完整填写） | 字段完整性检查 |
| 交叉困境描述 | 每个困境≥80字（含道德张力+可衍生剧情线） | 字段完整性检查 |

### 1.2 内容标准

每篇知识沉淀必须包含以下至少一项具体内容：

| 内容类型 | 说明 | 示例 |
|:---------|:-----|:-----|
| 具体案例 | 真实历史事件或同类作品中的案例 | "明朝宗藩制度导致的地方资源垄断" |
| 流程步骤 | 机制运作的具体步骤 | "权力更迭流程：老宗主闭关→代掌事接管→弟子试探→正式交接" |
| 数据数值 | 可量化的指标 | "散修平均寿命32年 vs 宗门弟子平均寿命68年" |

> 纯抽象描述（如"社会存在阶层压迫"）不合格。必须有具体的案例/步骤/数值支撑。

### 1.3 搜索标准

| 标准 | 要求 | 说明 |
|:-----|:-----|:-----|
| 多轮搜索 | 最多3轮 | 不少于2轮，不超过3轮 |
| 第1轮 | 广度搜索 | 覆盖所有涉及维度的关键词搜索 |
| 第2轮 | 深度搜索 | 针对第1轮发现的关键压力源深挖细节 |
| 第3轮 | 交叉验证 | 确认数据准确性+补充案例+搜索交叉困境素材 |

**搜索策略：**

```
第1轮（广度）：
  - 按三界框架维度逐个搜索
  - 目标：覆盖所有涉及维度，识别潜在压力源
  - 每个维度1-2个搜索查询

第2轮（深度）：
  - 针对第1轮发现的关键压力源深挖
  - 目标：获取具体案例/流程步骤/数据数值
  - 每个关键压力源1个搜索查询

第3轮（交叉验证）：
  - 验证数据准确性
  - 补充缺失的案例
  - 搜索交叉困境的素材
  - 1-2个搜索查询
```

### 1.4 无需求处理

某个调研维度被判定为无需求时，不跳过，必须逐类说明理由：

```
无需求记录格式：
  - {界名} · {维度名}：无需求
    - 理由：{为什么不涉及这个维度——具体说明，不能用"不相关"一笔带过}
```

**无需求理由示例：**

```
合格：
  - 自然界 · 气候条件：无需求
    - 理由：本书设定为室内密室推理，故事发生在封闭空间内，
      气候条件不构成压力源。如后续剧情扩展到室外，需重新评估。

不合格：
  - 自然界 · 气候条件：不相关
    （没有说明为什么不相关，一笔带过）
```

## 2. 质量检查规则

### 2.1 数量检查

| 检查项 | 合格标准 | 不合格处理 |
|:-------|:---------|:-----------|
| 压力源数量 | ≥3个 | 退回第4步补充压力源 |
| 交叉困境数量 | ≥3个 | 退回第5步补充交叉困境 |
| 跨界交叉困境 | ≥1个 | 退回第5步优先识别跨界困境 |
| 知识沉淀字数 | ≥500字 | 退回第3步补充内容 |
| 具体案例/步骤/数值 | 每篇≥1项 | 退回第3步补充案例 |

### 2.2 结构检查

| 检查项 | 合格标准 | 不合格处理 |
|:-------|:---------|:-----------|
| 三界定界记录 | 含涉及界+定界理由 | 补充定界记录 |
| 分界调研维度 | 每个涉及界执行对应维度调研 | 补充遗漏维度 |
| 压力源6字段 | 每个压力源6字段完整填写 | 补充缺失字段 |
| 交叉困境4字段 | 每个困境含交叉类型+描述+道德张力+剧情线 | 补充缺失字段 |
| 无需求逐类说明 | 每个无需求维度标注理由 | 补充理由 |

### 2.3 内容检查

| 检查项 | 合格标准 | 不合格处理 |
|:-------|:---------|:-----------|
| 显性表现 | 每个压力源有读者可见的具体场景 | 补充显性场景 |
| 可量化指标 | 每个压力源有可衡量的指标 | 补充量化数据 |
| 道德张力 | 每个交叉困境有道德选择（两难） | 补充道德张力 |
| 可衍生剧情线 | 每个交叉困境标注剧情方向 | 补充剧情线方向 |

### 2.4 质量检查流程

```
质量检查执行顺序：
  1. 数量检查（压力源≥3 / 交叉困境≥3含1跨界 / 字数≥500 / 案例≥1）
  2. 结构检查（三界定界 / 分界调研 / 6字段完整 / 4字段完整 / 无需求说明）
  3. 内容检查（显性表现 / 可量化指标 / 道德张力 / 剧情线方向）
  
  全部通过 → 进入第7步存盘
  任一不通过 → 退回对应步骤补充
```

## 3. 退化预警机制

### 3.1 什么是退化

退化指 pop-research 在连续多次调研中表现出"越来越浅"的趋势——无需求维度越来越多，压力源和交叉困境数量逐步减少，案例越来越抽象。

### 3.2 退化判定

| 触发条件 | 预警级别 | 处理 |
|:---------|:---------|:-----|
| 连续3次调研出现同一界无需求 | 退化预警 | 在产出中标记 `degradation_warning: true` |
| 连续3次调研压力源数量=3（刚好及格线） | 质量下滑预警 | 在产出中标记 `quality_decline: true` |
| 连续2次调研交叉困境无跨界 | 跨界缺失预警 | 在产出中标记 `cross_boundary_missing: true` |

### 3.3 预警处理

当触发退化预警时：

1. 在知识沉淀文件头部标注预警标记
2. 在返回给调用方的路径指针中包含预警状态
3. 强制重新审视无需求的界——是否真的无需求，还是偷懒跳过
4. 如确认为真实无需求，记录理由并重置计数

```
预警标记格式：
  ---
  degradation_warning: true
  warning_reason: "连续3次自然界无需求（气候/生态/灾害维度）"
  warning_date: {timestamp}
  ---
```

### 3.4 退化计数规则

- 计数范围：同一项目内的连续调研
- 计数重置：当某个界从"无需求"变为"有需求"时，该界的无需求计数归零
- 跨项目不累计：每个新项目从0开始计数

## 4. 深度等级

根据调用方传入的期望深度，调研的覆盖范围不同：

| 深度等级 | 调用方 | 覆盖范围 | 压力源要求 | 交叉困境要求 |
|:---------|:-------|:---------|:-----------|:-------------|
| 种子级 | seed | 三界全量（涉及的所有界+所有维度） | ≥5个 | ≥3个（含1跨界） |
| 场景级 | plot | 单界深入（特定场景涉及的维度） | ≥3个 | ≥2个 |
| 缺口级 | pop-novel | 定向补缺（仅缺失的信息） | ≥1个 | 不强制（视情况） |

> 种子级调研是最完整的调研，覆盖三界全量维度。场景级和缺口级是按需调研，覆盖范围较小但深度不减——每个涉及的维度仍然需要具体案例/流程步骤/数据数值。
