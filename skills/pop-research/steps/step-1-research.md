# Step 1：调研执行流程（9步）

> 管线: pop-research v1.0.0
> 消费: 调用方传入的调研需求（领域+具体问题+期望深度）
> 产出: `写作参考/知识沉淀/{主题}.md` + `写作参考/索引.md` 更新 + 路径指针返回

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `references/三界框架SOP.md` | 三界17维度调研方法论+压力源6字段转化+交叉困境6种类型——调研的核心方法论 |
| 2 | `references/调研深度标准.md` | 深度标准（≥500字/案例/多轮搜索）+质量检查规则+退化预警机制 |
| 3 | `../pop-novel/references/多元素整合范式.md` | 将调研发现做相似/相反/盲区发散、机制转译、交叉碰撞，避免资料堆积 |
| 4 | `../pop-novel/references/资料覆盖率协议.md` | 标注资料覆盖范围，避免把局部样本当成全篇结论 |
| 5 | 调用方传入的调研需求 | 领域+具体问题+期望深度+附加参数（书型/核心卖点/事件brief等） |

## 做什么

### 第1步：接收调研需求

解析调用方传入的调研需求，确定调研范围：

```
输入参数解析：
  - 领域：{调研涉及的领域}
  - 具体问题：{要回答的核心问题}
  - 期望深度：
    · 种子级（三界全量）—— seed调用，需要完整的三界压力矩阵
    · 场景级（单界深入）—— plot调用，需要特定场景/领域的深度调研
    · 缺口级（定向补缺）—— pop-novel调用，写作中发现的信息缺口
  - 附加参数（可选）：
    · 书型、核心卖点（seed调用时传入）
    · 事件brief、涉及领域（plot调用时传入）
    · 缺口描述（pop-novel调用时传入）
```

记录调研需求到产出文件头部。

### 第2步：三界框架定界

根据调研需求，判断本次调研涉及哪些界，并记录定界理由：

```
三界定义：
  - 社会界：人类社会内部的权力/经济/文化/法律/阶层/组织
  - 自然界：物理环境的资源/气候/地理/生态/疾病/灾害
  - 超自然界：超越物理法则的力量/规则/存在/禁忌

定界输出：
  - 涉及界：{社会界/自然界/超自然界}（可多选）
  - 定界理由：{为什么涉及这些界，为什么不涉及其他界}
```

> 详见 `references/三界框架SOP.md` §1 三界定义与定界规则。

### 第3步：分界调研

对每个涉及的界，执行对应维度的深度调研：

#### 社会界6维度（如涉及社会界）

| 维度 | 调研问题 | 产出 |
|:-----|:---------|:-----|
| 权力结构 | 谁掌权？权力如何分配与传承？ | 权力图谱+权力来源 |
| 经济体系 | 财富如何生产、分配、流通？ | 经济链条+资源节点 |
| 法律制度 | 规则如何制定与执行？谁执法？ | 法律框架+执法缺口 |
| 文化传统 | 主流价值观/禁忌/信仰体系？ | 文化张力点+禁忌清单 |
| 阶层关系 | 社会分层结构？流动规则？ | 阶层金字塔+流动壁垒 |
| 组织势力 | 有组织势力？利益诉求？ | 势力格局+势力关系网 |

#### 自然界6维度（如涉及自然界）

| 维度 | 调研问题 | 产出 |
|:-----|:---------|:-----|
| 地理环境 | 地形/地貌/交通条件？ | 地理制约因素 |
| 气候条件 | 气候特征/季节变化/极端天气？ | 气候压力源 |
| 资源分布 | 关键资源分布？稀缺性？ | 资源争夺焦点 |
| 生态系统 | 动植物/生态链/生态平衡？ | 生态威胁来源 |
| 疾病疫情 | 流行病/地方病/医疗条件？ | 疾病压力源 |
| 自然灾害 | 常发灾害类型/频率/破坏力？ | 灾害威胁等级 |

#### 超自然界5维度（如涉及超自然界）

| 维度 | 调研问题 | 产出 |
|:-----|:---------|:-----|
| 力量体系 | 力量来源/分级/运作机制？ | 力量体系框架 |
| 规则禁忌 | 超自然法则/使用代价/禁忌？ | 禁忌清单+代价机制 |
| 存在层级 | 超自然存在的层级/关系？ | 存在层级图谱 |
| 超自然组织 | 超自然势力组织/利益诉求？ | 超自然势力格局 |
| 超自然历史 | 超自然事件的历史脉络？ | 历史因果链 |

> 每个维度：有需求→深度调研（多轮搜索，详见 `references/调研深度标准.md`）；无需求→逐类说明理由。

### 第3.5步：资料覆盖率声明

对所有来源做覆盖率标注，尤其是小说拆书、案例卡、wiki、百科、书评和模型常识：

| 资料ID | 作品/来源 | 来源类型 | 覆盖范围 | 可用于 | 不可用于 | 可信度 | 缺口处理 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| R-01 |  | library拆书/L2卡/L3卡/wiki/百科/书评/用户描述/模型常识 | 第一卷/开局/某桥段/全书概述/未知 | 开局机制/世界规则/长线结构/文风/复刻路由 | 全书结论/终局判断/设定细节/剧情链 | 高/中/低 | 补wiki/补目录/补书评/声明样本限制 |

若资料只覆盖第一卷、第二卷或少量桥段，调研结论必须写“样本显示”，不得写“该作品整体证明”。seed 调用需要全书卖点、长线结构、世界机器时，必须补全篇轮廓；wiki/百科可用于补轮廓，但不能替代具体剧情拆解。

### 第4步：压力源转化（6字段结构化）

将调研发现转化为结构化压力源，每个压力源包含6个字段：

```
压力源 #N：
  1. 来源界属：社会界/自然界/超自然界
  2. 具体描述：{这个压力源是什么，从调研中得出}
  3. 显性表现：{读者可见的具体场景/现象——不是抽象概念}
  4. 对主角的压力方式：{这个压力源如何逼迫主角行动}
  5. 可量化指标：{可以用什么标准衡量压力大小/紧迫程度}
  6. 交叉可能性：{这个压力源可能与哪些其他压力源产生交叉}
```

> 详见 `references/三界框架SOP.md` §3 压力源转化6字段。

### 第5步：交叉困境识别（6种交叉类型）

识别压力源之间的交叉困境，共6种类型：

```
同界内交叉（3种）：
  - 社会界内部交叉：社会界压力源之间产生的困境
  - 自然界内部交叉：自然界压力源之间产生的困境
  - 超自然界内部交叉：超自然界压力源之间产生的困境

跨界交叉（3种）：
  - 社会×自然跨界：社会界与自然界压力源交叉
  - 社会×超自然跨界：社会界与超自然界压力源交叉
  - 自然×超自然跨界：自然界与超自然界压力源交叉
```

每个交叉困境记录：
```
困境 #N：{压力源A} × {压力源B}
  - 交叉类型：同界/跨界（具体标注哪种）
  - 困境描述：{两个压力源交叉产生的两难处境}
  - 道德张力：{主角面临的道德选择}
  - 可衍生剧情线：{方向}
```

> 跨界困境通常比同界困境产生更强的道德张力，优先识别。

### 第5.5步：多元素整合

按 `pop-novel/references/多元素整合范式.md`，把调研所得压力源、真实案例、制度机制、环境因素和超自然规则放入同一张整合记录中：

1. 元素抽取：每个压力源都必须标明表层事实和底层机制。
2. 相似/相反/盲区：为关键压力源补出同类变体、反向情境和未被调研材料直接覆盖的逻辑缺口。
3. 机制转译：说明该事实如何变成小说里的压力、收益、误判、代价或选择。
4. 交叉碰撞：识别相似强化、相反冲突、盲区补洞和规则错位。
5. 反证：说明哪些调研发现看似有趣但不适合当前项目。

该步骤的输出写入知识沉淀中的“多元素整合记录”小节。

### 第6步：质量检查

按质量红线和深度标准检查调研产出：

| 检查项 | 合格标准 | 不合格处理 |
|:-------|:---------|:-----------|
| 压力源数量 | ≥3个 | 补充调研 |
| 交叉困境数量 | ≥3个（至少1个跨界） | 补充交叉识别 |
| 每篇知识沉淀字数 | ≥500字 | 补充内容 |
| 具体案例/流程步骤/数据数值 | 每篇至少含1项 | 补充案例或数据 |
| 三界定界记录 | 含涉及界+定界理由 | 补充定界记录 |
| 无需求逐类说明 | 每个无需求维度标注理由 | 补充理由说明 |
| 退化预警 | 连续3次无需求→标记 | 触发退化预警 |

> 详见 `references/调研深度标准.md` §2 质量检查规则。

### 第7步：存盘到写作参考/知识沉淀/

将调研结果写入 `写作参考/知识沉淀/{主题}.md`：

```markdown
# {主题}调研

> 调研: pop-research · {timestamp}
> 调用方: {seed/plot/pop-novel}
> 期望深度: {种子级/场景级/缺口级}

## 调研需求
{领域+具体问题+附加参数}

## 三界框架定界
- 涉及界：{...}
- 定界理由：{...}

## 分界调研
### 社会界（如涉及）
{6维度调研结果}
### 自然界（如涉及）
{6维度调研结果}
### 超自然界（如涉及）
{5维度调研结果}

## 压力源（6字段结构化）
{压力源清单}

## 资料覆盖率声明
{资料覆盖表 + 不可外推项}

## 交叉困境
{交叉困境清单}

## 多元素整合记录
{元素抽取 + 相似/相反/盲区 + 机制转译 + 交叉碰撞 + 反证}

## 质量检查结果
{各项检查通过情况}
```

### 第8步：更新写作参考/索引.md

在 `写作参考/索引.md` 中追加本次调研产出的条目：

```markdown
- [{主题}] 知识沉淀/{主题}.md | {调研日期} | {调用方} | {涉及界}
```

如索引.md不存在，则创建。

### 第9步：返回路径指针

向调用方返回路径指针：

```
返回：
  - 路径指针: 写作参考/知识沉淀/{主题}.md
  - 索引已更新: 是
  - 质量检查: {通过/不通过}
  - 压力源数: {N}
  - 交叉困境数: {N}（其中跨界 {N}）
  - 退化预警: {无/已触发}
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 三界框架已定界（涉及界+理由） | 退回第2步 |
| 每个涉及界已执行分界调研 | 退回第3步 |
| 压力源≥3个 | 退回第4步补充 |
| 交叉困境≥3个（至少1个跨界） | 退回第5步补充 |
| 每篇知识沉淀≥500字+含案例/步骤/数值 | 退回第3步补充 |
| 资料覆盖率已声明，局部样本未冒充全篇结论 | 补充覆盖表或降级结论 |
| 无需求维度已逐类说明理由 | 补充理由 |
| 已存盘到写作参考/知识沉淀/ | 执行第7步 |
| 索引.md已更新 | 执行第8步 |
| 路径指针已返回给调用方 | 执行第9步 |
