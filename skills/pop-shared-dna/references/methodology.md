# 文风DNA蒸馏方法论文档

> 本文档记录 pop-shared-dna 的核心方法论、从开源项目借鉴的设计思路、以及实验验证建议。

---

## 一、核心方法论来源

### 1.1 Thinking-Claude（richards199999/Thinking-Claude ⭐14.2k）

**核心洞察**：通过精心设计的 System Prompt，让 Claude 模仿另一种 AI 的思维方式（o1 的链式推理）。

**迁移到文风模仿**：
| Thinking-Claude 的做法 | pop-shared-dna 的迁移 |
|------------------------|----------------|
| 分析 o1 的思维方式特征 | 分析目标作者的写作风格特征 |
| 提取关键思维特征词 | 提取 7 维风格特征 |
| 编码为结构化 block | 编码为风格档案 + Prompt |
| System Prompt 附加指令 | 注入给 Agent 作为风格指令 |

**关键设计原则提取**：
1. **具体化**：不说"模仿鲁迅"，说"句长10-20字，善用反讽，叙事者冷峻克制"
2. **结构化**：按维度分块，每个块内列具体规则
3. **示例绑定**：每条规则附原文示例
4. **否定清单**：列出"这个风格不会做的事"

### 1.2 Style Mimic Prompt（社区方法）

**核心洞察**：先让 AI 自己分析样本，生成风格 Profile，再用 Profile 指导写作。

**Step-by-step 流程**：

```
Round 0：批量阅读（20章） + 分前中后期标注
  → Agent 读取连续 20 章原文
  → 以 7:7:6 比例标记前/中/后期，用于跨时期一致性验证
  → 20 章全部进入下一轮分析

Round 1：分析
  → 对 20 章逐篇分析——
     用词习惯、句式节奏、叙事者态度、修辞手法、情绪表达方式
  → 产出：风格分析报告

Round 2：Profile 生成
  → 用分析报告填入 template
  → 产出：结构化的风格档案（style-dna-profile）

Round 3：风格写作
  → 加载风格档案的写前规则作为写作指令
  → 给出写作主题
  → 产出：带有该风格质感的内容

Round 4：验证迭代
  → 与原文对比，检查"偏差点"
  → 调整规则中的偏好权重
  → 重复 Round 3
```

### 1.3 auto-claude-writing-agent-pub（写作 Agent 工作流）

**核心洞察**：写作是一套可执行工作流，而非单次生成。

**可迁移的工作流设计**：

```
brief（用户需求）
  → 调研（收集素材）
  → 规划（确定结构和风格）
  → 写作（按规则生成）
  → 质检（对照红线检查）
  → 发布（输出成品）
```

pop-shared-dna 集成这个工作流的方式：在 **规划和写作** 两个环节注入风格DNA。

### 1.4 三层文风DNA框架（pop-writer-chapter → pop-writer-prose）

**核心洞察**：风格由三个独立层次构成，而非单一的"好不好读"维度：

```
Layer 1：纯事实骨架（发生了什么）
Layer 2：叙事策略指令（信息怎么讲出来）
Layer 3：文风DNA / 叙事哲学（作者为什么这样写）
```

**pop-shared-dna 的定位**：主要涵盖 **Layer 2 + Layer 3** 的提取和应用。Layer 1（事实骨架）由 pop-writer-chapter 处理，产出设计包供 pop-writer-prose 消费。

## 二、8维分析框架的理论基础

8 维框架的设计参考了多个语言学 / 写作理论：

| 维度 | 理论基础 | 关联理论 |
|------|---------|---------|
| 词汇与语料 | 文体学 | 词汇密度分析、语域分析 |
| 句式与节奏 | 句法学 | 句长分布、句式复杂度分析 |
| 叙事者姿态 | 叙事学 | Genette 叙事距离理论、POV 分析 |
| 信息释放策略 | 信息理论 | 信息熵、渐进揭示、悬念理论 |
| 对话策略 | 语用学 | Grice 合作原则、对话分析 |
| 修辞与描写 | 修辞学 | 修辞格分类、感官描写理论 |
| 情绪与张力 | 接受美学 | 读者情感曲线、张力-释放理论 |
| 战斗文风 | 动作叙事学 | 战斗节奏理论、信息释放与感官叠加 |

## 三、提取精度影响因子

| 因子 | 影响 | 建议 |
|------|------|------|
| 样本总量 | 样本越少，分析越可能偏离真实风格；网文作者的风格会随字数增长自然演化 | ≥1万字总有效字数，至少覆盖3个不同章节 |
| 样本位置 | 只读开头（CH1-10）只能捕捉"开书模式"——节奏、信息密度、句子复杂度与中后期差异显著 | 前/中/后期各取1-2章：初期（1-30章）→ 中期（100-300章）→ 后期（500+章或最新章节） |
| 样本场景多样性 | 只看一种场景（如战斗）会遗漏对话/叙事特征 | 覆盖 3+ 种场景（日常/战斗/对话/情绪高潮） |
| 样本质量 | 机器翻译文本会掩盖原始风格特征 | 使用原文，不用译文 |
| 分析深度 | 只统计表层特征（句长、词频）会遗漏深层叙事策略 | 按 8 维框架逐层分析 |

## 四、风格档案的复用场景

提取完成的文风DNA档案（style-dna-profile）可用于以下场景：

| 场景 | 用法 | 效果预期 |
|------|------|---------|
| AI 写作 | 注入 style-mimic-prompt 作为写作指令 | 产出具备目标风格质感的文本 |
| 内容质检 | 将风格档案作为 QC 标准 | 自动检查产出是否符合目标风格 |
| 模型微调 | 用档案指导训练数据构建 | 微调后的模型天然具备风格倾向 |
| 教学分析 | 作为文学风格分析的辅助工具 | 结构化呈现作者写作技法 |
| 翻译质量控制 | 在翻译时对齐目标语言的风格偏好 | 译文保留原文风格质感 |

## 五、验证方法

### 5.1 A/B 测试

```
A 组：无风格注入的 Agent 产出
B 组：注入 style-mimic-prompt 的 Agent 产出

评估维度：
1. 盲测：让人类判断哪段更像目标作者的风格
2. 量化：句长分布、词汇分布是否接近目标作者
3. 违规检查：红线违规次数
```

### 5.2 控制变量实验

```
固定：写作主题、字数要求、Agent 基础能力
变量：注入的风格DNA（不同作者）
验证：产出之间是否体现出结构性差异
```

## 六、已知限制

1. **样本偏差**：提取的 DNA 反映的是"作者在特定样本中的表现"。样本位置（只读开头）会导致分析结果偏向"开书模式"，忽略作者在中期和后期自然演化的风格。同一作者在100章前后的句长分布、信息释放策略、战斗写法往往有显著差异
2. **模型上限**：最终输出质量受限于底层 LLM 的能力（8k vs 128k 上下文，"懂"修辞的程度不同）
3. **文化隔阂**：某些风格特征（如方言、双关、文化隐喻）难以通过 Prompt 方式有效复刻
4. **风格漂移**：在长文本生成中，风格一致性会随上下文变化逐渐衰减
5. **无法感知"突破"**：作者的风格有时会因为题材变化而主动突破自己的常规写法，DNA 档案无法预判这种突破
