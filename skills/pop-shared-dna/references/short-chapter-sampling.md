# 短章书目采样策略

> 当全书章节平均行数 < 100行时，纯靠逐章精读无法达到行数目标，需要混合策略。

## 触发条件

- 采样30章后行数 < 5,000 行
- 或全书总行数 < 20,000 行且平均每章 < 100行

## 混合采样方案

放弃"全部通过逐章精读来获取原文摘录"的策略，改为两阶段：

### 阶段A：逐章精读（30章）

按标准流程（前20%+中40-60%+后20%）选30章，逐章完整精读。目标：
- 识别全书可用的**场景类型种类**（标记每种场景出现在哪些章）
- 提取**核心场景的完整原文段**（每个场景卡至少锚定3-5个候选章）
- 建立对全书文风的整体认知（叙事距离、对话模式、段落呼吸等）

### 阶段B：全书搜索验证

使用 `search_files` 工具对全书所有章节做模式匹配验证：

| 验证项 | 搜索模式 | 验证目标 |
|:-------|:---------|:---------|
| 数据面板频率 | `姓名：\|属性：\|职业：\|经验值` | 确认分布密度 |
| 对话引导词 | `沉声道\|喃喃道\|轻声道\|缓缓道` | 确认引导词使用模式 |
| 感官关键词 | `雨声\|月光\|寒风\|血腥` | 确认感官序列一致性 |
| 偷战斗技能 | `背刺\|潜行\|阴影\|暗影\|袭杀` | 确认战斗公式一致 |
| 专长/升级 | `增强.*点\|提升.*级\|获得.*专长` | 确认数据嵌入模式 |

### 各阶段产出占比

- 精读阶段贡献：各场景卡/维度的 **观察判断** + **核心原文段**
- 搜索阶段贡献：**时间演变**段中的频率分布数据 + 全书匹配验证
- 原文摘录总量比例：精读30章约提供2,000-3,000行原文，搜索验证约提供100-200个样本点

## 注意事项

- 场景卡的原文摘录必须来自精读（确保上下文完整），不引用搜索命中片段
- 时间演变段的"早期/中期/末期的频率变化"可以引用搜索数据
- 如果全书统一的文风特征在搜索中验证通过，时间演变段可以写"全书范围内无变化"
- 搜索工具通过 `execute_code` 调用 `search_files` 时可能返回空count结果——直接用 `search_files` 工具调用

## 边界条件

| 场景 | 处理 |
|:-----|:------|
| 全书不足100章 | 全部精读，搜索可跳过 |
| 全书200章但仅2万行 | 精读30章 + 搜索验证，标注"采样受限" |
| 章节行数差异极大（50-500行跨越） | 优先选长章精读，短章用搜索补 |
