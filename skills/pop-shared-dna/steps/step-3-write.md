# 步骤3：写出章内 DNA 文件（v5.1 核心）

## 输入

- 步骤1+2 的全部产出。
- `templates/style-dna-three-layer-profile.md` v5.1 模板。
- 必要时参考 `templates/style-dna-profile.md` 的原文证据格式。

## 做什么（五步）

### Step 3a：写采样与边界声明

- 列出采样章节，并按章型标注：开篇/探索/对峙/战斗/过渡/揭秘/高潮。
- 标明不进入文风DNA的内容：全书架构、卷级节奏、世界观体系、长线角色成长。
- 标明需要隔离的内容：特殊桥段、特定角色口癖、旁观者震惊、作者坏习惯/高频词。

### Step 3b：抽章内笔触 DNA

选出 >=4 个笔触维度，例如：

- 叙事距离
- 句式与段落呼吸
- 感官顺序
- 动作写法
- 对话写法
- 情绪外化
- 物象写法

每个维度必须写：

- 适用章型
- 表现方式
- 执行约束
- 禁止项
- 证据章节

### Step 3c：截取原文证据（不降级）

每个笔触维度/章型卡截取原文证据或明确章节定位。证据必须来自对应章型：

- "战斗章"不能塞行军说明。
- "对峙章"不能塞纯环境描写。
- "开篇章"不能冒充所有章节。

采样范围内找不到匹配段 → 退回步骤0扩采，不接受降级替换。

### Step 3d：抽单章剧情套路 DNA

建立 >=5 个章型卡。每张卡只回答"这一章怎么展开"，不回答全书怎么发展。

章型卡格式：

```markdown
### {章型卡名}
chapter_type: {chapter_type_id}

使用条件：
章内读者期待：

章内节拍：
1. 起笔：
2. 第一信息变化：
3. 压力/诱因：
4. 主角动作或判断：
5. 阻碍升级：
6. 可见反馈：
7. 章末钩子：

笔触绑定：
- 叙事距离：
- 句式段落：
- 对话方式：

禁止误用：
证据章节：
```

推荐章型：

- `opening_shift`
- `exploration_gain`
- `confrontation_pressure`
- `combat_reversal`
- `transition_buffer`
- `reveal_hook`

### Step 3e：写涌现式章内消费契约

- `soul.md`：只写长期笔触偏好和禁止迁移项。
- `current-state.md`：写本章章型、笔触目标、章内套路、禁止误用、验收点。
- write 门禁：500 字内看出笔触目标；章内节拍完整；不误迁移原作名词/口癖/全书架构。
- review 门禁：检查笔触和章内套路是否生效。

## 产出

`写作资产/文风库/{书名}.md`，全文建议 12-25K。不要为了装下全书架构而膨胀。

## 最终门禁

- [ ] 笔触维度数量 >= 4
- [ ] 章型卡数量 >= 5
- [ ] 有原文证据或明确章节定位
- [ ] 无降级替换
- [ ] 有采样与边界声明
- [ ] 有单章剧情套路，不写全书架构
- [ ] 有角色/口癖隔离
- [ ] 有涌现式章内消费契约
- 任一条不通过 → 退回，不合格不出。
