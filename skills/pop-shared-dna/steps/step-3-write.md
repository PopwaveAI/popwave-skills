# 步骤3：写出风格文件（v4 核心）

## 输入

- 步骤1+2 的全部产出 + `templates/style-dna-profile.md` v4 模板

## 做什么（三步）

### Step 3a：选维度 + 场景卡

- 选出 >=4 个通用维度 + >=6 个场景卡
- 场景卡选择法则：
  1. 读采样章 → 标记所有场景类型（战斗/对话/描写/叙述 x 阶段）
  2. 去重 → 合并同类型候选，选最典型的
  3. 选 >=6 个最典型的 → 每个配 500+ 字原文
  4. 卡名对齐 prose-render 的 scene 字段命名

### Step 3b：截取原文（不降级）

每个场景卡/维度截取 500+ 字原文。必须是该 scene 类型下真实发生的原文：
- "战斗·早期遭遇战" → 不能塞军队行军或战术解说段
- "战斗·中坚 Boss 战" → 不能塞抽卡牌或世界设定
- "对话·极简引导词" → 不能塞黑面包生存描写

**门禁：** 采样范围内找不到匹配段 → **退回** 步骤0 扩采，不接受降级替换。

### Step 3c：写观察 + 时间演变

- 写观察 + 时间演变
- 确认场景卡名与设计包 scene 字段对齐

## 产出

`pop-trope-library/文风库/{书名}.md`，全文约 20-30K（含原文摘录）。

## 最终门禁

- [ ] 所有维度/场景卡原文 >= 500 字
- [ ] 场景卡数量 >= 6
- [ ] 通用维度数量 >= 4
- [ ] 每项有时间演变段
- [ ] 无降级替换
- 以上任意一条不通过 → **退回**，不合格不出。
