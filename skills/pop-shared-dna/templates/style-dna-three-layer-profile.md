# 章内文风DNA档案 · 模板 v5.1

> v5.1 目标：把文风DNA聚焦到"一章内怎么写"。主内容只管两件事：文笔笔触 + 单章剧情套路。
>
> 不再把全书架构、长线商业模型、世界观设计写进文风DNA。那些属于 seed / 剧情线 / content-mechanics。

---

## 1. 采样与边界声明

### 采样范围

列出采样章节，并按"章内类型"标注：

- 开篇章
- 探索章
- 对峙章
- 战斗章
- 过渡章
- 揭秘章
- 高潮章

### 不进入文风DNA的内容

- 全书主线架构。
- 卷级节奏。
- 世界观设定体系。
- 长线角色成长路线。
- 作者某个阶段的特殊大高潮。

### 需要隔离的内容

- 单章特殊桥段，不能泛化为所有章节。
- 特定角色口癖，不能泛化为全员说话方式。
- 作者坏习惯/高频词，不能当成有效笔触。
- 旁观者震惊，只能作为章内反馈手段，不能当成主角表达方式。

## 2. 章内笔触 DNA

> 管这一章读起来的"手感"：叙事距离、句式、段落、感官、动作、对话、修辞。

每个维度格式：

```markdown
### {笔触维度}

适用章型：
表现方式：
执行约束：
- 必须：
- 禁止：
证据章节：
原文短证据：
```

建议维度：

- 叙事距离：贴身 / 拉远 / 群体视角 / 历史视角。
- 句式与段落呼吸：短句链、长句铺陈、独句段功能。
- 感官顺序：先看见、先听见、先闻到、先身体反应。
- 动作写法：动作 -> 声音 -> 结果，还是判断 -> 动作 -> 反馈。
- 对话写法：短对峙、插科打诨、信息交换、威胁试探。
- 情绪外化：通过动作、身体、环境、对白还是旁白。
- 物象写法：物件如何从局部细节变成信息或压力。

## 3. 单章剧情套路 DNA

> 管这一章内部怎么展开。不是全书结构，只是"一章怎么起、怎么转、怎么给反馈、怎么收钩子"。

### 3.1 章型卡矩阵

每张章型卡格式：

```markdown
### {章型卡名}
chapter_type: {chapter_type_id}

使用条件：
章内读者期待：

章内节拍：
1. 起笔：
2. 第一信息变化：
3. 压力/诱因：
4. 主角动作或判断：
5. 阻碍升级：
6. 可见反馈：
7. 章末钩子：

笔触绑定：
- 叙事距离：
- 句式段落：
- 对话方式：

禁止误用：
证据章节：
```

推荐章型：

- `opening_shift`：开篇/世界规则松动。
- `exploration_gain`：探索/发现/资源露头。
- `confrontation_pressure`：对峙/谈判/位阶施压。
- `combat_reversal`：战斗/反杀/打穿误判。
- `transition_buffer`：过渡/休整/信息沉淀。
- `reveal_hook`：揭秘/规则揭露/章末大钩子。

### 3.2 单章反馈方式

反馈是章内套路的一环，不单独扩成"商业DNA"。

可选反馈：

- 敌人失态。
- 旁观者重新判断。
- 资源到手或资源线索出现。
- 组织/规则/权限发生变化。
- 主角获得新信息。
- 更高压力逼近。

### 3.3 章末钩子方式

钩子只回答"下一章为什么要看"：

- 更危险的东西出现。
- 胜利引发后果。
- 新线索指向更深地点。
- 旧伏笔换了意义。
- 主角做出主动选择。

## 4. 角色与口癖隔离

```markdown
### {角色/功能类型}

功能：
可迁移：
不可迁移：
误用风险：
```

必须至少区分：

- 主角表达。
- 调节角色表达。
- 敌人表达。
- 旁观者/群众表达。
- 作者口癖或坏习惯。

## 5. 跨题材使用说明

只允许迁移：

- 章内展开方式。
- 章内反馈方式。
- 笔触手感。
- 角色功能。

不允许迁移：

- 原作专有名词。
- 原作世界观机制。
- 原作全书架构。
- 原作角色口癖。

## 6. 涌现式写作消费契约

### 6.1 soul 只写长期笔触偏好

可写：

- 叙事距离偏好。
- 句子气口。
- 对话气质。
- 氛围边界。
- 禁止迁移项。

不写：

- 单章具体场景卡。
- 具体剧情节点。
- 全书结构判断。

### 6.2 current-state 写章内DNA执行包

```markdown
## 本章DNA执行包
- DNA源：
- 本章DNA模式：适配 | 强嫁接 | 轻触
- 本章章型：
- 本章笔触目标：
- 本章章内套路：
  1. 起笔：
  2. 第一信息变化：
  3. 压力/诱因：
  4. 主角动作或判断：
  5. 阻碍升级：
  6. 可见反馈：
  7. 章末钩子：
- 禁止误用：
- 验收点：
```

### 6.3 write 验收

- 500 字内能否看出笔触目标。
- 章内节拍是否完整。
- 是否只润色而没有改变章内展开。
- 是否误迁移原作名词/口癖/全书架构。
