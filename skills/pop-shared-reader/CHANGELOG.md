## v0.14.0 (2026-06-02)

**类型**: enhancement
**改动**:

## v0.13.1 (2026-05-31)

**根因**: 旧版产出只有叙事文本，无结构化数据 → pop-html-anything 拿到的素材太浅，只能做纯文字卡片
**类型**: architecture
**改动**: 新增 Step A.1.5 逐章 YAML 结构化标注（entities/events/metrics/signals/tone）；Step A.2 角色表新增 image_prompt 列 + 出场章节数 + 角色重要性；新增「本卷名场面」表（含场景级 image_prompt）；新增实体共现矩阵 + 卷统计；新增产出文件 `*_结构化数据.yaml`
**效果**: 产出管线双格式化——叙事笔记(MD)给人读，结构化数据(YAML)给 pop-html-anything 消费渲染

## v0.12.0 (2026-05-31)

**根因**: 旧版机械提取（nouns/events/relationships）拆得太碎 → 花了大量时间但用户还是不知道书讲了什么
**类型**: architecture
**改动**: Phase A 从"机械提取"改为"按卷深度阅读"——保持叙事完整性；新增魔改对照维度（对 IP 原作的魔改效果判断）；叙事流增加执行观察要求（每节末尾 ☝️ 判断）；禁止批量压缩；Phase C 从"必须做"改为"按需做"
**效果**: 阅读笔记质量从"剧情摘要"升级为"阅读判断"
