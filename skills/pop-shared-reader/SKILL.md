# pop-shared-reader v0.15.0

帮读者拆解长篇小说。**产出双格式：人类可读的叙事笔记 + 机器可消费的结构化 YAML。**

---

## 速查表

| 阶段 | 核心动作 | 产出 | 步骤文件 | 门禁 |
|:-----|:---------|:-----|:---------|:------|
| **Phase 0** | 书稿预检 | 拆书方案 | [steps/step-phase-0-书稿预检.md](steps/step-phase-0-书稿预检.md) | 用户确认后才进 Phase A |
| **Phase A** | 深度阅读 + 结构化标注 | 笔记.md + 结构化.yaml | [steps/step-phase-a-深度阅读.md](steps/step-phase-a-深度阅读.md) | 单卷质量确认后才进下一卷 |
| **Phase B** | 跨卷关联 | 跨卷关联笔记 | [steps/step-phase-b-跨卷关联.md](steps/step-phase-b-跨卷关联.md) | -- |
| **Phase C** | 按需结构化 | 知识图谱（可选） | [steps/step-phase-c-按需结构化.md](steps/step-phase-c-按需结构化.md) | 问了才做，不做也行 |

## 协作边界

> **拆书定位声明：** 本 skill 面向"拆书为读"——产出供读者查阅的叙事笔记和供 pop-shared-html 渲染的结构化 YAML（含 image_prompt）。
>
> **分工：**
> - pop-shared-reader -> 消费方：读者 / pop-shared-html -> 产出：叙事笔记.md + 结构化YAML（含 image_prompt）
> - book-deconstructor（pop-shared-reader 内部）-> 面向"拆书为写"-> 产出供给 pop-novel 的 14 项输入包

---

## 执行流程

按顺序执行 Phase 0 -> A -> B -> C。每一步的详细 SOP 见对应 `steps/` 文件。每阶段末尾有一道门禁，用户确认后才进入下一阶段。

---

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | 分卷定界未经用户确认不进 Phase A |
| 2 | 叙事线 + 结构线双产出缺一不可 -- 不能只有笔记没有 YAML，也不能只有 YAML 没有笔记 |
| 3 | 每卷必须产出结构化 YAML（含 image_prompt） -- 角色肖像 prompt + 名场面 prompt 不可省略 |
| 4 | 魔改对照仅限 IP 衍生/无限流/诸天流 -- 其他类型不强制，但如有改编痕迹必须分析 |
| 5 | 不准用梗概代替逐章阅读 -- Phase A 必须逐章读原文 |
| 6 | 名场面 image_prompt 必须写 -- 这是 pop-shared-html 生图的直接输入 |
| 7 | 产出只留摘要 -- 写入笔记/YAML 后对话中不粘贴完整产出 |

| 检查项 | 路径 | 状态要求 |
|:-------|:-----|:---------|

| 叙事笔记 | `{书名}_卷{N}_笔记.md` | 每卷存在（叙事线） |
| 结构化数据 | `{书名}_卷{N}_结构化数据.yaml` | 每卷存在，含 image_prompt（角色肖像 + 名场面） |

| 章节覆盖 | 各卷内章节 | 无跳章，每章有对应记录（叙事 + 结构） |

## 错误示例

| # | 错误 | 正确 |
|:-:|:-----|:-----|
| 1 | 跳过 Phase 0 直接读正文 | 先通读目录定分卷 -> 抽样验证 -> 用户确认 -> 再开始读 |
| 2 | 只有叙事笔记，没有结构线 YAML | 双格式缺一不可。读完一卷后两条线一起交付 |
| 3 | 角色表遗漏 image_prompt | 每个重要角色必须写，结构为 `[内容类型]: [视觉描述], [氛围], high quality digital illustration` |
| 4 | 用梗概代替逐章阅读 | Phase A 必须逐章阅读原文，每 3-5 章产出摘要 + 执行观察 |
| 5 | 跨卷关联写成了单卷总结拼接 | 按因果链/伏笔线/剧情线/书特有机制重新组织，体现跨卷因果关系 |

## 异常与边界条件

| # | 场景 | 触发条件 | 处理动作 |
|:-:|:-----|:---------|:---------|
| 1 | 小说文件不存在或格式不支持 | Phase 0 入口检查发现无可用文件 | 提示用户提供 TXT/EPUB/PDF 格式文件，不退回到编造数据 |
| 2 | 小说篇幅过长（>500 章） | Phase 0 发现总章数超过 500 | 建议用户先选前 N 卷或前 200 章重点阅读，剩余部分按需选读 |
| 3 | 分卷方案用户不满意 | Phase 0.4 用户认为分卷不合理 | 返回 Phase 0.1 重新通读目录定分卷，调整后再次提交确认 |
| 4 | 章文件残缺（跳章/乱序） | 阅读时发现章节编号不连续 | 按现有章节继续，在笔记中标注"章节不连续，疑有缺失" |
| 5 | image_prompt 生成不符合预期 | pop-shared-html 渲染后发现肖像/名场面不匹配 | 回到结构化 YAML 修订 image_prompt |
| 6 | Phase B 发现矛盾（伏笔未回收） | 跨卷关联时发现前面卷的伏笔在后续卷未被处理 | 在跨卷关联笔记中标注"未回收伏笔清单"，不做主观修正 |
| 7 | 用户中途要求换书 | Phase A 已开始但用户要求拆另一本 | 先完成当前卷的产出，断点保存后切换到新书 |
| 8 | 多卷并行阅读 | 用户要求不按卷确认直接读多卷 | 解释单卷确认的必要性；如坚持，标注"用户要求跳过单卷确认" |
