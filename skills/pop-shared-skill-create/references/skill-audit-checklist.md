# 存量 Skill 改造审计清单

> 所属: pop-shared-skill-create · references/
> 用途: 改造存量 skill 前，按此清单全量扫描。对照 PRD、管线规范、skill 写作规范逐项检查。
> 来源: 2026-06-22 session — pop-writer-chapter/plot/world/prose/expert-writer 全量审计实战总结。

## 审计步骤

### 1. 版本一致性（5 处对齐）

| 检查点 | 文件 |
|:-------|:-----|
| SKILL.md 标题 | 标题中的版本号 |
| SKILL.md frontmatter | `version:` 字段 |
| SKILL.md 页脚 | `## 版本` 段 |
| skill.json | `version` 字段 |
| CHANGELOG.md | 最新条目版本号 |

> 五处不一致 → 统一到最新实际版本。

### 2. PRD 对齐（输入/输出/上下游/用途）

对照 `prd/01-管线架构/01-写作专家全链路依赖图-PRD.md` 中的该 skill 行：

| 检查项 | 问 |
|:-------|:---|
| 入 | skill 实际读的文件列表和 PRD 「入」列一致吗？PRD 列了但 skill 没读？skill 读了但 PRD 没列？ |
| 出 | skill 实际产出的文件和 PRD 「出」列一致吗？ |
| 上下游 | frontmatter pipeline 字段和 PRD 一致吗？ |
| 用途 | skill 的核心定位和 PRD 「用途」列一致吗？ |

### 3. 路径一致性（全文件扫描）

搜索过时路径关键词，替换为当前 PRD 规范：

| 过时路径 | 当前规范 |
|:---------|:---------|
| `00-总控/` | `状态/` |
| `设计/卷/volume-XX.md` | `剧情设计/卷/卷{N}-卷纲.md` |
| `设计/幕/act-XX.yaml` | `剧情设计/幕/vol-XX/act-YY.md` |
| `设计/剧情线/` | `剧情设计/剧情线/` |
| `写作资产/设计包/` | `章节设计包/` |
| `写作资产/设计包v3/` | 已废弃（统一为 `章节设计包/`） |

### 4. 文件索引完整性（无死文件）

- 列出 `skills/{skill-name}/` 下所有文件
- 对照 SKILL.md 的速查表，确认每个文件都有条目
- 无条目的文件 = 死文件 → 删除或补条目
- 速查表格式：分层引导（steps/templates/references/外部依赖），每行含「什么时候→读什么→产出」

### 5. 层架构/幽灵文件检测

搜索 `层架构` 关键词——如果命中，该文件已废弃（creative 已删除）。全部引用清除。

### 6. 内联模板提取

搜索 SKILL.md 中是否内联了产出格式/模板 → 提取到 `templates/`。

### 7. Step 负载检查

| 信号 | 动作 |
|:-----|:-----|
| Step 的「读什么」列了 >4 个文件 | 拆分为预加载 + 按需查 |
| Step 做了两件不同性质的事 | 拆分为骨架步 + 润色步 |
| Step 产出没有自查 | 加质量门禁（对照上游约束逐项检查，不通过回退指定步骤） |

### 8. 速查表格式

对齐 pop-writer-creative 格式：

```markdown
## 速查表（完整目录引导）

### steps/ — 执行层
| 什么时候 | 读什么文件 | 产出 | 门禁 |

### templates/ — 模板层
| 什么时候 | 读什么文件 | 产出 |

### references/ — 知识层（读后理解，指导操作）
| 什么时候 | 读什么文件 | 产出 |

### 外部依赖
| 什么时候 | 读什么文件 | 产出 |
```

### 9. 关联 skill 同步

skill 之间共享概念（如 entity-snapshot 所有权、chekhov-tracker 合并）→ 涉及的所有 skill 一并修改，不留不一致。

### 10. SKILL.md 行数

对照 `01-skill写作规范.md §10.9`：
- 路由型 ≤120 行
- 产文件型 ≤150 行
- 超标 → 先下放 steps/，再下放 references/，最后裁剪案例
