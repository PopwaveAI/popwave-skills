# Step 2：改造模式 — 改造存量 Skill

> **前置条件：** 目标 skill 的 SKILL.md 可读。
> **产出：** 改造后的文件 + BEFORE/AFTER 对比报告。
> **核心约束：改造不改逻辑内容。** 只改结构/格式/红线位置，不动核心方法论。

---

## ❌ 质量红线（改造模式）

| # | 红线 |
|:-:|:-----|
| ❌B1 | 不扫描就动手改 — 先读完整文件，输出扫描报告，用户确认后再改 |
| ❌B2 | 改后不更新版本号和 CHANGELOG — 版本 +1，CHANGELOG 追加 |
| ❌B3 | 改后不输出 BEFORE/AFTER 报告 — 用户需要知道改了什么 |
| ❌B4 | 保留独立的质量红线文件 — 红线必须内联到正文 |
| ❌B5 | 改造范围超出当前 skill — 只改目标 skill 的文件 |
| ❌B6 | 改造改了逻辑内容 — 只改结构/格式/红线位置，不动方法论 |

---

## 改造流程（8 步）

### Step 1：扫描红线违规

**读什么：** 目标 SKILL.md 全文 + skill.json + CHANGELOG.md
**做什么：** 对照设计红线（❌D1-D13）+ 改造红线（❌B1-B6）逐项标记
**产出：** 扫描报告（违规项列表 + 证据行号）
**❌ 门禁：** 不读完全文不动手。扫描报告交用户确认后再改造。

### Step 2：精简 frontmatter

**读什么：** 当前 frontmatter
**做什么：** 砍到 ≤4 行，元数据迁移到 skill.json
**产出：** 精简后的 frontmatter

迁移映射：

| 原位置 | 迁移目标 |
|:-------|:---------|
| version | 正文末尾 `## 版本` |
| orchestration / dependencies | skill.json |
| display_name / tags / category | skill.json |
| produces | 正文末尾 `## 产出物` |

### Step 3：重写 description

**读什么：** 当前 description
**做什么：** 改为触发条件式
**产出：** 新 description

格式：`"当用户说'X/Y/Z'时启用。产出来物供给下游管线。"`

### Step 4：重写正文结构

**读什么：** 当前正文
**做什么：**
1. ❌ 质量红线提至第一屏
2. 嵌套 Phase 体系拍平为 3-6 步扁平流程
3. 每步嵌入 ❌ 错误做法 / ✅ 正确做法
4. 定义式规则改为禁令式
5. 执行层细节下沉到 steps/，知识层下沉到 references/

**产出：** 重写后的正文
**❌ 门禁：** 不改核心方法论和业务逻辑，只改结构和表达方式。

### Step 5：补充 WRONG 错误示例

**读什么：** 当前正文
**做什么：** 至少补充 1 个 agent 实际会犯的错误
**产出：** WRONG ❌ + CORRECT ✅ 完整对比区块

### Step 6：补充异常与边界条件表

**读什么：** 当前正文
**做什么：** 补充至少覆盖以下维度的异常处理表
**产出：** 异常与边界条件表

必覆盖维度：
- 文件/依赖缺失 → 不准编造数据
- 外部工具不可用 → 降级方案
- 用户中途改变需求 → 退回或暂停
- 子 agent 启动失败 → 手动执行 + 红线自检

### Step 7：输出对比报告

**读什么：** 改造前后的文件
**做什么：** 逐维度对比
**产出：** BEFORE/AFTER 对比表

```markdown
| 维度 | BEFORE | AFTER |
|:-----|:-------|:------|
| frontmatter 行数 | N 行 | N 行 |
| SKILL.md 总行数 | N 行 | N 行 |
| 红线位置 | 第 XXX 行 | 第一屏 |
| 红线格式 | 定义式 | 命令式 ❌ |
| 错误示例 | N 个 | N 个 |
| 边界条件 | 无/有 N 种 | 有 N 种 |
| steps/ 引用完整性 | N 个缺失 | 全部存在 |
| 三层架构 | 全堆在 SKILL.md | 路由/执行/知识分离 |
```

### Step 8：版本归档 + CHANGELOG

**读什么：** 当前 CHANGELOG.md
**做什么：** 版本号 +1，追加改造记录
**产出：** 更新后的 CHANGELOG.md + SKILL.md 版本区

## ❌ WRONG 示例

```
❌ WRONG: 改造 skill 时不读完整文件，扫一眼前几行就动手 → 遗漏正文中的重要方法论
✅ CORRECT: 先 skill_view 全文，输出红线扫描报告交用户确认后，再执行改造

❌ WRONG: 改造时把方法论也改了 → 破坏了 skill 的核心价值
✅ CORRECT: 只改结构/格式/红线位置，方法论和业务逻辑原封不动

❌ WRONG: 改造后不更新 CHANGELOG → 用户不知道改了什么
✅ CORRECT: 版本 +1，CHANGELOG 追加完整改造记录
```

## 改造红线速查

| 红线 | 检查方法 | 违规后果 |
|:-----|:---------|:---------|
| 无读取协议红线 | 搜"禁止用 Read" | 补充为红线第一条 |
| 速查表非目录引导 | 检查是否列出所有 steps/+references/ 文件 | 改为全文件目录引导 |
| frontmatter > 4 行 | 数行数 | 退回精简 |
| description 无触发词 | 搜"当用户说/Invoke when" | 退回重写 |
| 红线不在第一屏 | 检查第一屏内容 | 提至第一屏 |
| 红线是定义式 | 搜"P0/P1/P2/建议/推荐" | 改为禁令式 |
| 无 WRONG 示例 | 搜"❌ WRONG" | 补充至少 1 个 |
| 正文深层嵌套 | 数嵌套层级 > 2 | 拍平为 3-6 步 |
| 用第二人称 | 搜"You should/你需要" | 改为祈使句 |
| 红线是建议 | 搜"建议/推荐/应该" | 改为"❌ 不准…" |
| 速查表缺列 | 检查列数 < 5 | 补齐五列 |
| SOP 缺 I/O | 检查每步有无读什么/做什么/产出/门禁 | 补齐四标签 |
| 无落盘检查点 | 搜"落盘" | 补充检查点表 |
| 无越界检测 | 搜"越界" | 补充越界检测表 |
| 缺消费关系 | 搜"消费关系" | 补充消费关系行 |
| SKILL.md 过长 | 行数 > 60 | 执行层下沉到 steps/ |
| 知识混在路由层 | SKILL.md 含方法论/原则/历史 | 下沉到 references/ |
