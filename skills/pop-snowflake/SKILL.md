# pop-snowflake · 雪花式写作方法

> 从一句话到完整小说的迭代式扩展法。10步双轨并行，从25词种子逐步扩展为完整长篇。v1.1.0

## 做什么

| 输入 | 输出 | 下游 |
|:-----|:-----|:-----|
| 创意方向/故事种子 | 完整大纲 + 人物宝典 + 场景清单 + 小说正文 | 无（独立创作法） |

## 怎么操作

> execution.mode: 两阶段串行 | 强保障：本 SKILL.md 由 host 层每次 run 强制注入 | 弱保障：steps/ + templates/ 需 agent 主动 readFile

| 步骤 | 做什么 | 产出 | step 文件 |
|:-----|:-----|:-----|:----------|
| Step 1 | 雪花生长·设计阶段：预备→一句话→一段式三幕→人物介绍→一页纸大纲→人物大纲→四页纸大纲 | 完整大纲+人物大纲 | `steps/step1.md` |
| Step 2 | 雪花落地·执行阶段：人物宝典→场景清单→规划场景→写小说 | 人物宝典+场景清单+正文 | `steps/step2.md` |

## 红线

1. **读取协议**：读取 skill 文件用 `Get-Content -Encoding UTF8 -Raw`，禁用 Read 工具
2. 每一步的产出必须基于前一步扩展——禁止跳步或凭空重写，雪花法核心是迭代生长
3. 三次灾难性事件缺一不可——第一幕末、第二幕中、第二幕末各一次
4. 每个场景必须包含矛盾冲突——无冲突的场景要么加冲突要么删除
5. 主动场景与被动场景必须交替串联——决定即下一幕主动场景的目标
