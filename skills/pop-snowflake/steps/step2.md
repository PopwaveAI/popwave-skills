# Step 2 · 雪花落地·执行阶段

> 从完整大纲到小说正文。4个子步骤：人物宝典→场景清单→规划场景→写小说。
> 前置输入：Step 1 产出的四页纸完整大纲 + 人物大纲。

---

## 2a 第七步：人物宝典

**做什么**：为每个重要人物建立四维深度档案。

**怎么做**：
1. 取出Step 1的人物介绍卡和人物大纲作为基础。
2. 为每个人物填写四大信息维度：

### 外部信息（生理特征）
姓名、年龄、出生日期、身高、体重、民族、发色、眼睛颜色、外表描写、穿衣风格。

### 性格信息（心理特质）
幽默感、性格类型、宗教、党派、习惯、喜欢的音乐/书籍/电影/颜色、钱包里的物件。

### 环境信息（社会关系）
住所描写、教育程度、工作经历、家庭、最好的朋友、男性朋友、女性朋友、对头或敌人。

### 心理信息（深层心理）
- 童年最美好或最悲惨的记忆（一行式概括）
- 最突出/最不突出的性格特征
- 性格中的矛盾性
- 最强烈的希望
- 最深刻的恐惧
- 人生哲学
- 如何看待自己
- 被其他角色如何看待

3. 参考 `templates/人物宝典.tpl.md`
4. 可找气质相近的照片帮助可视化。
5. 设定需符合年代与人物背景。

**产出**：人物宝典×N

---

## 2b 第八步：场景清单

**做什么**：罗列出小说中的所有场景。

**怎么做**：
1. 从Step 1的四页纸大纲出发，拆分出所有场景。
2. 在表格中为每个场景标注：

| 列 | 说明 |
|----|------|
| 场景编号 | 按顺序编号 |
| 视点人物 | 通过谁的视角描述该场景 |
| 主要事件 | 概述场景核心事件 |
| 时间线 | 场景发生的时间/顺序 |
| 预估字数 | 该场景预计篇幅 |
| 备注 | 其他信息 |

3. 参考 `templates/场景清单.tpl.md`

**核心规则**：
- 每个场景**必须**包含矛盾冲突
- 没有冲突的场景 → 要么加上冲突，要么删除
- 禁止仅为"烘托气氛""解释背景""阐明动机"而添加场景
- 矛盾冲突才是推动故事发展的原动力

**视点人物** = 通过他的视角来描述该场景的人。

**产出**：完整场景清单表

---

## 2c 第九步：规划场景

**做什么**：为场景清单中的每个场景设计内部结构。

**怎么做**：
1. 将每个场景分为**主动场景**或**被动场景**：

### 主动场景（三段式）
| 段 | 作用 |
|----|------|
| 目标 | 视点人物在该场景中想达成什么 |
| 冲突 | 什么阻碍了他 |
| 挫折 | 结果——他失败了，或付出了代价 |

### 被动场景（三段式）
| 段 | 作用 |
|----|------|
| 反应 | 对前一幕挫折的情感反应 |
| 困境 | 面对困境的权衡 |
| 决定 | 做出选择——这成为下一幕主动场景的目标 |

2. **交替串联**：主动场景 → 被动场景 → 主动场景 → 被动场景……

```
主动场景          被动场景          主动场景
目标→冲突→挫折 → 反应→困境→决定 → 目标→冲突→挫折 → ...
         ↑_________________________|
         决定 = 下一幕主动场景的目标
```

3. 注意：不是所有情绪反应都构成完整被动场景。大多数小说的主动场景远多于被动场景。部分场景可跳过被动场景直接进入下一个主动场景。

**产出**：场景设计卡×N

---

## 2d 第十步：写小说

**做什么**：基于以上所有产出，逐幕写小说正文。

**怎么做**：
1. 加载以下全部产出作为写作依据：
   - 四页纸完整大纲（Step 1）
   - 人物宝典（2a）
   - 场景清单（2b）
   - 场景设计卡（2c）
2. 从第一幕开始，按场景顺序逐个写作：
   - 每个场景遵循其场景设计卡的结构（主动/被动三段式）
   - 以场景的视点人物视角写作
   - 保持人物的说话方式、行为逻辑与人物宝典一致
3. 写完一幕后接下一幕，直到全书完成。
4. 遇到卡壳时，回到大纲和人物宝典寻找灵感。

**写作原则**：
- 跟着场景设计写，不想规则，想什么好看
- 每个场景的矛盾冲突必须实质化——不能只是内心独白
- 视点人物的选择影响信息量——选择"信息最少"的人作为视点，悬念最强
- 主动场景快节奏推进，被动场景给呼吸空间

**产出**：小说正文

**确认门**：
- 每个场景是否都有实质性的矛盾冲突？
- 视点人物是否一致？
- 人物行为是否与宝典吻合？
- 三次灾难性事件是否在正文中得到体现？
→ 通过则完成。否则回到对应步骤修改。
