# pop-trope-library

## 一句话定位

这是写作管线和拆书管线共享的公共素材库。

- 原始资产按模块和书名高保真保存，不压缩 L2/L3 大卡。
- 写作消费按“任务”检索，不只按书名或粗标签检索。
- Agent 读案例后必须产出“案例消化”，不能只搬运结构。

## 强制入口

### 写作消费侧

当 seed/plot/create/revise/arc 需要参考 library 时，按这个顺序执行：

1. 判断写作任务：开局、成长、冲突、设定揭示、关系、势力、收束等。
2. 读 `任务索引/00-索引.md`，优先进入匹配任务索引。
3. 读任务索引推荐的 3-5 个案例，优先读 L2 单元卡；需要长线咬合时再读 L3。
4. 按 `references/案例消化卡-标准格式.md` 输出案例消化。
5. 再读取必要的套路库、设定库、金手指库、文风库。
6. 声明库缺、跨赛道化用、不可照搬项。

不要直接从 `剧情库/00-索引.md` 随机挑书。按书索引只用于确认库存，不是写作消费入口。

### 入库生产侧

当拆书/分析/复盘产物要进入 library 时，按这个顺序执行：

1. 按 `references/入库清洗SOP.md` 判定模块和质量门槛。
2. 剧情资产原文保持高保真：L2 入 `剧情库/{书名}/L2/`，L3 入 `剧情库/{书名}/L3/`。
3. 给 L2/L3 补充 frontmatter 与消费摘要，或新建 sidecar/任务索引条目。
4. 更新 `剧情库/00-索引.md` 与相关 `任务索引/` 文件。
5. 如果没有合适任务索引，按 `references/任务索引设计规范.md` 新建或扩展。

## 目录模型

| 模块 | 作用 | 主消费方式 |
|:-----|:-----|:-----------|
| `任务索引/` | 写作任务入口，反向组织案例 | plot/create 优先读取 |
| `立项库/` | PRD 级经验、书型、题材焊接、失败案例 | creative/seed 读取 |
| `设定库/` | 书级 L1 设定包、世界宪法、PRD | seed/world/plot 读取 |
| `文风库/` | 文风 DNA 与场景卡 | create/prose 读取 |
| `剧情库/` | L2 单元卡与 L3 剧情线原始高保真资产 | 通过任务索引间接读取 |
| `套路库/` | 抽象叙事模式和爽点公式 | plot/create/revise 读取 |
| `金手指库/` | 金手指 WHAT 层设计素材 | seed/plot 读取 |
| `references/` | 入库、检索、分类、消化规范 | 需要时读取 |

## 关键红线

- 不压缩原始 L2/L3 卡。剧情库的价值在细节密度、子线咬合、情绪曲线和可迁移要点。
- 不把“按书归档”当作“写作检索”。写作检索必须走任务索引。
- 不只迁移结构。案例消化必须同时提取爽点证据、内容肌理、不可照搬项和本项目映射。
- 不混淆设定库和剧情库：设定库回答世界怎么运转，剧情库回答剧情怎么推进。
- 不混淆金手指库和套路库：金手指库回答设计什么外挂，套路库回答怎么写外挂展开。
- 不用 `_LEGACY/` 作为消费入口。legacy 只保留证据，不服务新写作。

## 输出要求

写作消费后，至少输出：

```markdown
## Library 查询与消化
- 写作任务：{任务}
- 读取索引：{任务索引路径}
- 精读案例：{3-5 个路径}
- 采用套路/设定/金手指：{路径}
- 案例消化摘要：{按案例消化卡压缩}
- 库缺声明：{如有}
- 不可照搬声明：{如有}
```

入库维护后，至少输出：

```markdown
## Library 入库更新
- 新增/更新资产：{路径}
- 更新索引：{路径}
- 新增任务标签：{标签}
- 消费摘要状态：{已补/待补}
- 遗留风险：{如有}
```
