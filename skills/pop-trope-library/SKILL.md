## 入库哲学（关键原则）

### 晋升不走「≥2本书」门槛

抽象模式卡（套路库）和设定卡（设定库）的晋升**不设跨书次数门槛**。规则只有一条：**不重复即可**。

- 以前：同一模式必须在≥2本书出现才能晋升抽象模式
- 现在：**独特模式比常见模式更有入库价值**。如果一张卡的套路/设定在已有库中找不到语义重复，直接晋升。只有真正重复（同一个模式的不同书案例）才需要归并。

这条修正来自实际教训：≥2门槛会把最具原创性的单书模式卡死在拆书复盘阶段——而恰恰是这些独特模式最值得被其他作者看到。

### ⚠️ 设定库 vs 剧情库：根本区分（常见混淆点）

最容易装错的两个模块，记住一句话：

> **设定库 = 世界怎么运转**（规则、制度、体系、命名、力量框架）
> **剧情库 = 剧情线怎么推进**（长期叙事线、冲突结构、兑现节奏、角色弧线）

| 问自己 | 属于设定库 | 属于剧情库 |
|:-------|:-----------|:-----------|
| 这是描述世界的一个规则/制度/体系吗？ | ✅ 修仙金融化、筑基资格证、分数种姓、神职腐败 | ❌ |
| 这是描述一条长期叙事线怎么推进吗？ | ❌ | ✅ 主线/支线/设定线/关系线的启动、升级、兑现、收束 |
| 这是命名/术语/氛围底色的创意吗？ | ✅ 质感层（命名策略、文明底色） | ❌ |
| 这是原书卷/幕/章级切片吗？ | ❌ | ⚠️ 只作 legacy 证据，不作为主入库资产 |

**常见错误**：把世界观制度（法力贷体系、筑基资格证制度）放进剧情库。这些是设定——它们描述「世界怎么运转」而不是「故事怎么推进」。如果你在写卡时觉得「这像在介绍一个规则」，它属于设定库。

### ⚠️ 金手指库 vs 套路库：WHAT vs HOW（常见混淆点）

> 记住一句话：
> **金手指库 = 设计什么金手指**（核心机制、代价设计、成长路径、赛道适配）
> **套路库 = 怎么写金手指的叙事节奏**（觉醒节奏、进化节拍、展示时机）

| 问自己 | 属于金手指库 | 属于套路库 |
|:-------|:-----------|:-----------|
| 这是描述一个金手指的核心机制和代价吗？ | ✅ 系统面板型、隐藏血脉觉醒型 | ❌ |
| 这是描述金手指在叙事中怎么展开的吗？ | ❌ | ✅ 金手指体系与外挂进化、隐藏天赋觉醒 |
| 这是金手指的设计参考（能干什么/代价/成长）吗？ | ✅ | ❌ |
| 这是金手指的叙事节奏（什么时候觉醒/怎么进化/怎么展示）吗？ | ❌ | ✅ |

**常见错误**：把金手指的叙事节奏（觉醒4阶段、进化5步公式）放进金手指库。这些属于套路库——它们描述"怎么写"而不是"设计什么"。金手指库的设计卡只回答"这个金手指有什么机制、什么代价、怎么成长、适配什么赛道"。

### 六个模块各自独立入库

| 模块 | 入库时机 | 输入来源 | 格式要求 |
|:-----|:---------|:---------|:---------|
| 立项库/ | creative 复盘 / QA 归因 / 拆书复盘后 | PRD 模式、元爽点组合、题材焊接、失败案例 | PRD级模式卡，不写具体桥段 |
| 套路库/ | 发现有独立价值的不重复模式 | 拆书复盘后提炼 | 10字段抽象模式卡（见 抽象模式卡-标准格式.md） |

| 文风库/ | pop-shared-dna 运行后 | 原文采样 | v4格式：场景卡+通用维度+时间演变 |
| 剧情库/ | pop-decon-volume / plotline 复盘后 | 原书设计包的 L2 单元卡 + L3 剧情线卡 | L2 保留拆书原始格式直接入库（不压缩），按标签入库至 `{标签}/L2/{来源书}-{编号}-{名称}.md`；L3 保留独立文件入 `{标签}/L3/{来源书}-{编号}-{名称}.md`，同时在 `{标签}/00-索引.md` 维护速查表 |
| 金手指库/ | 拆书复盘后 / creative 金手指设计文档产出后 | 拆书的金手指设计分析 / creative 的金手指设计文档 | 金手指设计卡（7字段：核心机制/代价设计/成长路径/赛道适配/元爽点关联/使用红线/案例积累，见 金手指设计卡-标准格式.md） |

### 各赛道素材不强行归并

不同赛道（西幻/修仙/都市/科幻/武侠）的素材即使结构相似也不强行合并为一张卡。一只 DND 西幻力量体系卡和一只修仙资本化体系卡各自独立——作者按赛道查询时靠索引找到同类参考。只有同赛道＋同核心机制时才合并为「跨书案例」。

## 抽象模式卡标准格式

抽象模式卡 ≠ 具体套路卡。具体卡是"某本书某个桥段怎么写"，抽象卡是"这个模式家族的N种变体及其跨书实现"。

模板结构：
```
# {模式名（核心机制·一句话定调）}
> 核心理念（2-3句） / 为什么爽（1-2句心理机制）
> 这是一个模式家族，有 N 种实现。

## 心理学原理（表格）
## 通用公式（题材无关，5-7步）
## 变体家族（N个变体，每个含对比表+案例）
## 使用红线（✅ 必须做 / ❌ 不要做）
## 节奏控制（按篇幅的策略）
## 套路链推荐（前后可接什么模式）
## 跨书案例对比（每拆一书新增一列）
## 案例积累（新案例追加到此，不开新文件）
```

参考示例：`套路库/扮猪吃老虎（隐藏实力·认知差碾压）.md`
格式模板：`references/抽象模式卡-标准格式.md`

## 剧情线入库路径（已有拆书资产时）

> 当拆书项目已完成 pop-decon-volume（卷纲+L2单元卡+L3剧情线），直接入库，不压缩。

```text
1. 读 {项目}/设计/L2单元卡/L2-{编号}.md       ← 设计包 L2 卡（原始格式，含情绪弧线/节奏控制/子线咬合等全量分析）
2. 读 {项目}/设计/L3剧情线/L3-{编号}.md        ← L3 剧情线（完整阶段推进/叙事债务/线间咬合）
3. 加 frontmatter（三标签：layer/track/meta_joy）+ library 消费说明头部
4. 按内容标签写入 `剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md`（保留原始格式，不压缩）
5. 按内容标签写入 `剧情库/{标签}/L3/{来源书}-{编号}-{名称}.md`（保留独立文件）
6. 更新对应标签的 `00-索引.md`：L3 速查表 + L2 索引
```

**L2 卡不压缩原则：** 拆书产出的 L2 卡包含事件链、套路分析、情绪弧线、节奏控制、子线咬合等全量分析，这些恰恰是写作管线（plot/chapter）消费时最需要的技法参考。压缩会丢失可迁移的核心价值。原始格式直接入库，零信息损失。

**L3 剧情线保留独立文件：** L3 的阶段推进表、叙事债务回收表、线间咬合矩阵是 plot 设计剧情线时的核心参考。L3 独立文件入 `{标签}/L3/`，同时在 `{标签}/00-索引.md` 维护速查表（1-2 行/条）作为快速检索入口。速查表是目录，L3 文件是正文，两者互补。

**核查清单（入库前自检）：**
- [ ] 设计包 L2 单元卡有完整结构分析（起承转合四段）？
- [ ] 嵌套子线有清晰的闭合/延伸状态标记？
- [ ] 可迁移要点 ≥5 条且不绑定原书专属前提？
- [ ] L3 剧情线有阶段推进表 + 叙事债务 + 线间咬合？
- [ ] 对应标签的 00-索引.md 的 L3 速查表已更新？

> 不满足 → 退回补全拆书资产，不入库。

**入库数量底线（从设计包到 library）：**
- 一个完整卷（150-250章）通常产出 10-12 张 L2 单元卡，全部直接入库
- 少于 6 张 → 检查是否漏了关键单元
- 标签分布参考：崛起征服 3-4 张 + 情感联盟 2-3 张 + 其他线 1-2 张

**Windows write_file 陷阱：** write_file 工具在 Windows 上对大文件（>~500 字节）会截断。写剧情线卡等长内容时，使用 `terminal` 的 heredoc 方式（`cat > file << 'EOF' ... EOF`）代替 write_file。
