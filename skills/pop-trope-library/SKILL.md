# pop-trope-library · 标准化网文套路库 v1.0.0

> **定位**：全局共享的套路知识库。三层架构，跨 skill 引用。
> 
> **核心原则**：入库的不是案例，是公式。每个套路必须抽象到跨题材通用的理念和原理层面，才能入库公式库。原始案例只归档在原始素材/ 目录，不污染公式层。

---

## 目录结构

```markdown
pop-trope-library/
├── SKILL.md                  ← 本文件
├── 公式库/                   ← L1 跨题材抽象公式（核心资产，几乎不改）
│   ├── 00-索引.md            ← 按使用频率排序的公式索引
│   ├── 打脸装逼.md
│   ├── 绝境反杀.md
│   └── ...
├── 变体库/                   ← L2 题材级具体实现（持续扩展）
│   ├── 西幻/
│   │   ├── 打脸装逼.md       ← 深渊主宰版变体案例
│   │   └── ...
│   ├── 修仙/                 ← 待填充
│   └── 都市/                 ← 待填充
└── 原始素材/                 ← L3 按书归档的原始套路卡
    ├── 深渊主宰/
    │   ├── 打脸装逼.md       ← 含原文案例
    │   └── ...
    └── {下一本书}/
```

## 三层入库标准

### L1 公式库 — 门槛最高

| 检查项 | 通过标准 |
|:-------|:---------|
| 题材无关 | 公式中不出现任何特定书名/角色名/地名。主角=主角、反派=反派、城市=城市 |
| 理念提炼 | 有 "这个套路为什么爽" 的底层原理说明（1-2句） |
| 结构拆解 | 四段式：起势/蓄势/爆发/余波，每段标注核心机制 |
| 通用红线 | 跨题材都适用的 ❌ 不要做 / ✅ 应该做 |
| 变体参考 | 至少填写西幻/修仙/都市/科幻四栏变体对照表 |

**入库触发**：套路在≥2本书/≥5案例中被验证有效后，可提升到公式库。首次入库可放宽（单书案例≥3即可）。

**不达标示例**：
```markdown
❌ 索伦反杀狗头人后属性暴涨 → 路人震惊
   （绑定了特定角色和场景，不算公式）
✅ 主角在绝境中利用环境优势以弱胜强 → 系统数据反馈暴涨 → 旁观者态度翻转
   （抽象为通用结构）
```

### L2 变体库 — 题材级

| 检查项 | 通过标准 |
|:-------|:---------|
| 题材对齐 | 明确标注题材标签（西幻/修仙/都市/科幻） |
| 案例完整 | 含原书原案例引用 |
| 变体说明 | 标注"这个题材下这个套路有什么特殊注意事项" |

### L3 原始素材 — 最低门槛

| 检查项 | 通过标准 |
|:-------|:---------|
| 来源可溯 | 文件名包含书名+套路名 |
| 内容完整 | 含原文案例+四段式拆解 |
| 按书归档 | 每本书一个子目录 |

原始素材是"矿"，变体库是"精炼品"，公式库是"成品"。**只有成品才被 skill 引用，矿石不进 pipeline。**

## 谁在哪个环节调用

| Skill | 调用时机 | 读什么 | 用于 |
|:------|:---------|:-------|:-----|
| **pop-writer-chapter** | 设计章节事件链时 | `公式库/{套路名}.md` | 获取抽象公式和通用红线，填入本章具体设定 |
| **pop-writer-prose** | 渲染具体事件时 | `变体库/{题材}/{套路名}.md` | 获取题材级的具体写作示例参考 |
| **pop-decon-design-pack** (v3) | 产出设计包时写套路归档 | 写 `原始素材/{书名}/` | 每章拆解产出套路卡 + 追加案例 |

## 调用示例

```python
# pop-writer-chapter 创作时
# 设计包标注了 trope: "打脸装逼"
# 1. 读公式层 -> 获取抽象结构和红线
skill_view("pop-trope-library", "公式库/打脸装逼.md")
# 2. 读变体层 -> 获取本题材的具体示例
skill_view("pop-trope-library", "变体库/西幻/打脸装逼.md")
# 3. 填入本章具体设定 -> 输出高密度事件链

# pop-decon-design-pack 拆书时
# 产出套路卡 -> 写入原始素材
write_file(
    "skills/pop-trope-library/原始素材/深渊主宰/打脸装逼.md",
    content
)
```

## 清洗规则：什么能进公式库

### ✅ 可以入库

```
打脸装逼
  → 底层原理：利用信息差制造满足感。
     读者知道主角有隐藏实力/身份，但反派和围观者不知道。
     当差距被揭露的瞬间，信息差释放为爽感。
  
  抽象公式：反派挑衅 → 主角隐忍 → 实力揭露 → 围观反转
  通用红线：蓄势段不超过3段。爆发段一个动作特写胜于三个形容词。
```

### ❌ 不入库，留在原始素材

```
困境建置（绝境开局）
  → 这个其实不是一个独立的"套路"
  → 它是"绝境反杀"的前置条件的一部分
  → 留在原始素材，待积累更多案例后再判断是否升格
```

### ❌ 不入库（太窄）

```
银月行者（准传奇·低阶选民·月神侍奉者）
  → 这是深渊主宰的独特设定，涉及具体世界观
  → 跨题材无法复用 → 留在原始素材
```
