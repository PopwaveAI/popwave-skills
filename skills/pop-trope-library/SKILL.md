# pop-trope-library

> **一句话定位**：四模块公共知识库，写作管线的公共素材源。
>
> **Agent 强制流程**：
> - 消费侧（writer skill 查素材）→ 按 `references/调用匹配SOP.md` 三维查询
> - 生产侧（拆书/分析产出入库）→ 按 `references/入库清洗SOP.md` 清洗入库

## 入库哲学（关键原则）

### 晋升不走「≥2本书」门槛

抽象模式卡（套路库）和设定卡（设定库）的晋升**不设跨书次数门槛**。规则只有一条：**不重复即可**。

- 以前：同一模式必须在≥2本书出现才能晋升抽象模式
- 现在：**独特模式比常见模式更有入库价值**。如果一张卡的套路/设定在已有库中找不到语义重复，直接晋升。只有真正重复（同一个模式的不同书案例）才需要归并。

这条修正来自实际教训：≥2门槛会把最具原创性的单书模式卡死在原始素材层——而恰恰是这些独特模式最值得被其他作者看到。

### 四个模块各自独立入库

| 模块 | 入库时机 | 输入来源 | 格式要求 |
|:-----|:---------|:---------|:---------|
| 原始素材/ | 拆书管线 Phase 1 完成后 | 设计包套路归档 | 8字段（见 trope-entry-standard-ideal.md） |
| 套路库/ | 发现有独立价值的不重复模式 | 原始素材/ → 晋升 | 10字段抽象模式卡 |
| 设定库/ | 拆书 Phase 3 完成后 | L1世界观文件 + 创意溯源 | 源域 + 转化逻辑 + 适用场景 + 赛道 |
| 文风DNA/ | pop-shared-dna 运行后 | 原文采样 | v4格式：场景卡+通用维度+时间演变 |
| 剧情库/ | 拆书 Phase 4（创意溯源）完成后 | 创意溯源跨域参考 + 设计包 | 原剧情 + 改建思路 + 改建理由 + 适用场景 |

### 各赛道素材不强行归并

不同赛道（西幻/修仙/都市/科幻/武侠）的素材即使结构相似也不强行合并为一张卡。一只 DND 西幻力量体系卡和一只修仙资本化体系卡各自独立——作者按赛道查询时靠索引找到同类参考。只有同赛道＋同核心机制时才合并为「跨书案例」。

## 两个核心 SOP

| SOP | 用途 | 谁执行 |
|:----|:-----|:-------|
| `references/调用匹配SOP.md` | 三维查询（层×赛道×元爽点） | 各 writer skill 消费素材时 |

## 抽象模式卡标准格式

抽象模式卡 ≠ 具体套路卡。具体卡是"某本书某个桥段怎么写"，抽象卡是"这个模式家族的N种变体及其跨书实现"。

模板结构：
```
# {模式名（核心机制·一句话定调）}
> 核心理念（2-3句） / 为什么爽（1-2句心理机制）
> 这是一个模式家族，有 N 种实现。

## 心理学原理（表格）
## 通用公式（题材无关，5-7步）
## 变体家族（N个变体，每个含对比表+案例）
## 使用红线（✅ 必须做 / ❌ 不要做）
## 节奏控制（按篇幅的策略）
## 套路链推荐（前后可接什么模式）
## 跨书案例对比（每拆一书新增一列）
## 案例积累（新案例追加到此，不开新文件）
```

参考示例：`套路库/扮猪吃老虎（隐藏实力·认知差碾压）.md`
格式模板：`references/抽象模式卡-标准格式.md`
