# pop-trope-library

> **一句话定位**：四模块公共知识库，写作管线的公共素材源。
>
> **Agent 强制流程**：
> - 消费侧（writer skill 查素材）→ 按 `references/调用匹配SOP.md` 三维查询
> - 生产侧（拆书/分析产出入库）→ 按 `references/入库清洗SOP.md` 清洗入库

## 目录结构

```
pop-trope-library/
├── 目录.md                 ← 模块导览
├── SKILL.md
├── 文风DNA/                 ← 文风笔触特征（pop-shared-dna 产出到此）
├── 设定库/                   ← 赛道设定创意
│   ├── 00-索引.md
│   ├── 框架/                ← 框架层设定（力量/数值/规则）
│   └── 质感/                ← 质感层设定（术语/命名/文明底色）
├── 剧情库/                   ← 剧情改建思考
├── 套路库/                   ← 抽象叙事模式（已启动12模板）
│   ├── 00-总索引.md
│   └── {模式名}.md
├── 原始素材/                 ← L3 具体卡（拆书中间产物，非消费层）

    ├── 调用匹配SOP.md        ← ★ 消费侧查询流程
    ├── 入库清洗SOP.md        ← ★ 生产侧入库流程
    ├── 创意溯源-设计模式SOP.md
    ├── 元爽点-变体映射表.md
    ├── 抽象模式卡-标准格式.md
    └── trope-entry-standard-ideal.md
```

## 两个核心 SOP

| SOP | 用途 | 谁执行 |
|:----|:-----|:-------|
| `references/调用匹配SOP.md` | 三维查询（层×赛道×元爽点） | 各 writer skill 消费素材时 |
| `references/入库清洗SOP.md` | 质量门槛+三标签+去重+晋升 | 拆书管线/分析skill 产出入库时 |

## 抽象模式卡标准格式

抽象模式卡 ≠ 具体套路卡。具体卡是"某本书某个桥段怎么写"，抽象卡是"这个模式家族的N种变体及其跨书实现"。

模板结构：
```
# {模式名（核心机制·一句话定调）}
> 核心理念（2-3句） / 为什么爽（1-2句心理机制）
> 这是一个模式家族，有 N 种实现。

## 心理学原理（表格）
## 通用公式（题材无关，5-7步）
## 变体家族（N个变体，每个含对比表+案例）
## 使用红线（✅ 必须做 / ❌ 不要做）
## 节奏控制（按篇幅的策略）
## 套路链推荐（前后可接什么模式）
## 跨书案例对比（每拆一书新增一列）
## 案例积累（新案例追加到此，不开新文件）
```

参考示例：`套路库/扮猪吃老虎（隐藏实力·认知差碾压）.md`
格式模板：`references/抽象模式卡-标准格式.md`
