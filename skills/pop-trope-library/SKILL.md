# pop-trope-library · 标准化网文套路库 + 跨域元素库 v1.3.1

> **定位**：全局共享的知识库。三层套路架构 + 一套跨域元素交叉索引。
>
> **为什么拆这个库**：拆书时发现的每一个套路、每一个跨域元素来源，如果不记下来，下次开新书就找不到了。**入库的不是案例，是可复用的模式。**
>
> **谁能用**：任何 agent 在任何项目中——打开就能查"我想要的这个写法有没有人做过"、"我想融合的这个IP以前怎么焊的"。
>
> **v1.3.1 变化**：新增 `references/元爽点-变体映射表.md`（7元爽点×60变体套路映射表），衔接 creative 的爽点引擎产出与 plot 的套路配套路。`references/` 目录下新增该文件，供 creative/plot/chapter-design 跨 skill 引用。
>
> **v1.3.0 变化**：入库让魔门再次伟大 81 条 craft recipe 级套路文档（原始素材/让魔门再次伟大/），含触发条件/公式/核心张力/情绪收益/运作机制/使用注意/下游期待/复用记录。新增让魔门套路索引（00-索引.md）。plot v7.0 Step 1 通过本库配套路。

## 四层入库标准

### L1 公式库 — 门槛最高

| 检查项 | 通过标准 |
|:-------|:---------|
| 题材无关 | 公式中不出现任何特定书名/角色名/地名。主角=主角、反派=反派 |
| 理念提炼 | 有"这个套路为什么爽"的底层原理说明（1-2句） |
| 结构拆解 | 四段式：触发/盘点/诊断/规划（或等效分段），每段标注核心机制 |
| 通用红线 | 跨题材都适用的 ❌ 不要做 / ✅ 应该做 |
| 变体对照 | 至少填写西幻/修仙/都市三栏 |

**入库触发**：套路在≥2本书/≥5案例中被验证有效。

### L2 变体库 — 题材级

| 检查项 | 通过标准 |
|:-------|:---------|
| 题材对齐 | 明确标注题材标签（西幻/修仙/都市/科幻） |
| 案例完整 | 含原书原案例引用 |
| 变体说明 | 标注"这个题材下这个套路有什么特殊注意事项" |

### L3 原始素材 — 最低门槛（推荐达到理想级）

| 检查项 | 通过标准 |
|:-------|:---------|
| 来源可溯 | 文件名包含书名+套路名 |
| 内容完整 | 含原文案例+四段式拆解+注意事项+跨域来源 |
| ★ 元爽点标注 | 必须标注这个套路属于哪个元爽点家族（信息差/阶级革命/绝境反杀/身份跃迁/世界观震撼/智斗博弈/情感羁绊）。一个套路可标注多个（如\"信息差+智斗\"）。元爽点库详见 pop-writer-creative SKILL.md |
| 理想级 | 新建条目推荐参考 `references/trope-entry-standard-ideal.md`，含触发条件/公式/核心张力/情绪收益/运作机制/使用注意/下游期待/复用记录 |

**推荐流程**：新建 L3 条目前读标准模板，尽量向它靠拢。已有条目可逐步升级。

### L4 元素交叉 — 独立索引

| 检查项 | 通过标准 |
|:-------|:---------|
| 来源明确 | 素材来源（作品名/事件名/热梗名）+ 来源IP类型（影视/游戏/现实事件/热梗） |
| 焊接点 | 焊在原作的哪个结构上？替换了什么？插入在什么间隙？ |
| 改动量 | 直接取用/轻度改编/重度重构/仅保留核心意象 |
| 改动方法 | 具体写了怎么改：如"Frozen的冰封王国→作者把冲突从内心挣扎改为外敌入侵" |
| 复用建议 | 下次想用这个元素，有什么注意事项 |

## 跨域元素：什么算"来源"

| 来源类型 | 例子 | 在元素交叉/怎么记 |
|:---------|:-----|:-----------------|
| **知名IP** | Frozen, D&D, 战锤40k | `影视→Frozen→深渊主宰.md` |
| **现实事件** | 某次战争、经济危机、自然灾难 | `现实事件→XX战争→应用书.md` |
| **网络热梗** | 废话文学、躺平学、XX文学 | `热梗→{梗名}→应用书.md` |
| **社会现象** | 996、内卷、考公热 | `社会现象→{现象}→应用书.md` |
| **历史典故** | 某次变法、某个历史人物原型 | `历史→{典故}→应用书.md` |
| **其他作品** | 不出名但有用的设定 | `其他→{作品名}→应用书.md` |

## 原始素材模板

每张原始素材套路卡在拆书时填写，格式如下。新建条目推荐参考 `references/trope-entry-standard-ideal.md` 的理想级格式。

```markdown
# 套路: {套路名}

> 首次收录: chXXX | 使用次数: N | 最后更新: chXXX

## 四段式模板 / 核心结构
（同现有格式）

## 原文案例
（同现有格式）

## 注意事项
（同现有格式）

## 跨域灵感来源

> 这个套路的灵感来自哪里？不是每张卡都有来源，但如果有就必须写。

- **来源**：Frozen（冰雪奇缘）
- **来源类型**：影视
- **提取了什么**：冰封港口的视觉冲击 + 白衣少女的"以美展示毁灭" + 平凡外表→环境异变的渐进展开
- **改了什么**：
  - Frozen中冰封是失控 → 这里改为反派主动入侵
  - Frozen中艾莎是主角 → 这里寒冬少女是反派，长公主变为守护者
  - 冰封规模从一国缩为一港（适配DND世界的格局）
- **保留了**：冰从脚下蔓延的视觉序列 + 普通人在冰封瞬间的错愕感 + "神性冷漠"的气质
- **弃用了**：姐妹情感线、童年创伤、真爱之吻
- **如果想复用到自己的书**：这个框架（神灵以美示人→无声毁灭→凡人受害者特写）在任何西幻/神话类题材中都能用。关键在于"渐进式展开"——不要一次展示全部力量，让读者自己拼图。
```
