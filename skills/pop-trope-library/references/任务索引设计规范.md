# 任务索引设计规范

> 用途：把按书保存的高保真素材，改造成写作侧能直接检索的任务入口。

---

## 1. 为什么需要任务索引

剧情库按书保存是正确的，但写作侧按任务工作：

- 我现在要写开局。
- 我现在要写第一次收益闭环。
- 我现在要写绝境反杀。
- 我现在要写设定震撼。
- 我现在要写关系羁绊。

所以任务索引不是替代剧情库，而是给剧情库增加“写作入口”。

---

## 2. 推荐目录

```text
任务索引/
├── 00-索引.md
├── 开局/
├── 成长/
├── 冲突/
├── 设定/
├── 关系/
├── 势力/
└── 收束/
```

任务域可以扩展，但不要把任务名写成抽象空话。优先使用能指导写作的名称：

- `穿越求生-系统觉醒`
- `贫民区绝境-首次杀人`
- `第一次数值收益闭环`
- `绝境反杀-以弱胜强`
- `世界规则震撼揭示`
- `守护羁绊建立`
- `据点建立-资源经营`

---

## 3. 任务索引字段

每个任务索引至少包含：

```markdown
# {任务名}

## 适用场景
- 当前写作在解决什么问题。

## 检索标签
- task_tags:
- track:
- meta_joy:
- unit_stage:

## 必读案例
| 优先级 | 路径 | 读哪些段 | 学什么 | 不要照搬 |

## 配套素材
| 类型 | 路径 | 用途 |

## 消化问题
1. 主角在本任务里承受什么压力？
2. 主角的主动判断是什么？
3. 读者看到的可数收益是什么？
4. 代价或新压力是什么？
5. 哪些内容肌理能迁移到本项目？

## 输出要求
- plot/create 使用后必须输出案例消化。
```

---

## 4. 必读案例选择规则

一个任务索引推荐 3-5 个案例，不要无限堆链接。

优先级：

1. 同任务 + 同赛道 + 同元爽点。
2. 同任务 + 跨赛道但结构强。
3. 同任务 + 内容肌理强。
4. 相邻任务，作为补充。

每条案例必须说明“读哪些段”。不要只给路径，否则 agent 会被几万字大卡淹没。

## 5. 任务标签建议

| 任务域 | 常用任务标签 |
|:-------|:-------------|
| 开局 | 穿越求生 / 系统觉醒 / 低资源生存 / 第一次主动行动 / 第一次收益闭环 / 新手村逃离 |
| 成长 | 技能验证 / 数值收益 / 职业转职 / 装备获得 / 金手指认知突破 |
| 冲突 | 绝境反杀 / 追杀逃亡 / 暗杀清算 / 智斗谈判 / BOSS 战 |
| 设定 | 世界观震撼 / 规则揭露 / 制度黑暗 / 力量体系首展 |
| 关系 | 守护之约 / 同乡善意 / 战宠命名 / 追随者建立 / 情感羁绊 |
| 势力 | 据点建立 / 团队招募 / 资源经营 / 声望扩散 / 组织雏形 |
| 收束 | 债务回收 / 阶段胜利 / 新地图开启 / 更大敌人露面 |

---

## 6. 维护红线

- 不把任务索引写成书单。每个链接都要有“为什么读”和“读哪段”。
- 不复制 L2/L3 全文。索引只做导航和消化提示。
- 不用粗标签替代任务标签。`崛起征服` 是功能标签，不等于 `穿越求生-系统觉醒`。
- 不要求每张卡只有一个任务标签。一张 L2 可以出现在多个任务索引中。
