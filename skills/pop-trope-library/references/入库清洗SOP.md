# 入库清洗 SOP

> 用途：把拆书产出的 L3 具体卡 / 跨域设定素材 / 文风分析 / 剧情改建思考，清洗入库到 pop-trope-library 对应模块。
> 原则：一张高质量的抽象模式卡 > 100张散装具体卡片。精做，不做多。
> 执行者：拆书管线 / 分析 skill 产出入库时。

---

## 入库前置：质量门槛

### L3 具体套路卡（→ 原始素材/）

检查 `references/trope-entry-standard-ideal.md` 的 8 字段是否齐全：

1. 触发条件
2. 套路公式
3. 核心张力
4. 情绪收益
5. 运作机制
6. 使用注意
7. 下游期待
8. 复用记录

**不达标 → 退回拆书管线补全，不入库。**

### 设定素材（→ 设定库/）

设定素材不入库为单张碎片卡——以**整本 L1 设定**为单位入库。

入库要求：
1. 拆书项目必须有完整的 L1 六件套（L1-01~06 + 世界宪法）
2. 六件套经 world skill 产出并通过稳定性检验
3. 附带一份 PRD.md（含卖点摘要 + 匹配逻辑）

**不达标 → 先补 L1 设定，不入库。**

### 文风DNA（→ 文风DNA/）

必须满足 pop-shared-dna 的 6 条红线：
- 原文采样 ≥ 500字
- 场景卡 ≥ 6 个
- 通用维度 ≥ 4 个
- 有时间演变段
- 无降级替换
- 只留摘要不留原文

**不达标 → 退回 pop-shared-dna 扩采，不入库。**

### 剧情改建思考（→ 剧情库/）

必须有：
- 原剧情概述：被改建的原始剧情是什么
- 改建思路：怎么改的
- 改建理由：为什么这么改
- 适用场景：什么类型的作品能用这个改建思路

**不达标 → 退回，不入库。**

## Step 1：打三标签（仅原子化素材——套路卡/剧情库）

> 整本 L1 设定导入不需要打标签——卖点摘要和匹配逻辑在 PRD.md 中。

每条原子化素材（套路卡、剧情库条目）打三个标签，写入文件 frontmatter：

```yaml
---
layer: 框架          # 或 质感。机制层和调味层素材一般入套路库和文风DNA，不进设定库
track: 修仙          # 修仙/西幻/都市/科幻/武侠/通用
meta_joy: 信息差     # 信息差/阶级革命/绝境反杀/身份跃升/世界观震撼/智斗博弈/情感羁绊/无
---
```

| 标签 | 取值 | 判定方式 |
|:-----|:-----|:---------|
| `layer` | 框架 / 质感 | 素材描述的是力量/数值/规则→框架；术语/命名/文明底色→质感 |
| `track` | 修仙/西幻/都市/科幻/武侠/通用 | 从素材来源书的类型判定。跨赛道通用素材标"通用" |
| `meta_joy` | 7元爽点之一 / 无 | 从套路卡"元爽点家族"字段判定。设定素材一般标"无" |

### 批量打标（≥50卡时使用）

当一批拆书产出 50+ 张卡需要同时入库时，逐卡人工打标不可行。使用**内容关键词启发式打标**：

```python
# 元爽点检测规则（优先级从高到低，第一条匹配即锁定）
heuristics = {
    '信息差':     ['信息差', '隐藏', '秘密', '先知', '情报'],
    '阶级革命':   ['阶级', '阶层', '穷人', '剥削', '压迫', '底层'],
    '绝境反杀':   ['反杀', '反转', '逆袭', '以弱胜强', '绝境'],
    '身份跃迁':   ['身份', '跃升', '跨越', '上升', '阶级流动'],
    '世界观震撼': ['世界观', '体系', '设定', '制度', '规则', '揭露', '真相'],
    '智斗博弈':   ['智斗', '博弈', '算计', '心理', '骗', '谈判'],
    '情感羁绊':   ['关系', '感情', '羁绊', '信任', '牺牲', '守护'],
}
# 层检测规则
layer_map = {
    '框架': ['设定', '世界观', '体系', '制度', '规则', '数值', '等级', '升级', '战斗'],
    '质感': ['命名', '术语', '称呼', '名相', '氛围', '文明'],
}
```

**注意**：批量打标的 `meta_joy` 标签是首次近似值，不做绝对正确承诺。单卡精读时应修正。**禁止用批量打标结果做出"本书元爽点分布"等统计结论。**

## Step 2：去重检查

在目标模块内搜索同名/同义素材：

### 套路库去重（同一套路从不同书拆出）

| 情况 | 处理 |
|:-----|:-----|
| 已有抽象模式卡，新卡是已有变体的新案例 | 在抽象模式卡的"跨书案例对比"表新增一列，变体表"案例"行追加 |
| 已有抽象模式卡，新卡是已有变体的新亚型 | 在"变体家族"区新增一行 |
| 无抽象模式卡 | 提炼新抽象模式卡，放入 套路库/ |

### 设定库去重（同一设定创意从不同书拆出）

| 情况 | 处理 |
|:-----|:-----|
| 已有同类设定，新卡是同赛道的新案例 | 在原设定卡的"跨书案例"段追加 |
| 已有同类设定，新卡是不同赛道 | 在原设定卡的"跨赛道变体"段追加 |
| 无同类设定 | 新建设定卡 |

**不强行归并**：一条具体卡可以属于零个或一个抽象模式。

---

## Step 3：抽象层级判定

| 层级 | 位置 | 标准 |
|:-----|:-----|:-----|
| L3 具体卡 | 原始素材/ | 单书单桥段，拆书管线产出 |
| 抽象模式卡 | 套路库/ | 去重通过后提炼，无需跨书次数门槛 |

**晋升规则**：原始素材中的模式只要不重复（和已有抽象模式卡无语义重叠），就可提炼为抽象模式卡。独特模式比常见模式更有入库价值——优先入库。

晋升时按 `references/抽象模式卡-标准格式.md` 的 10 字段填写：
1. 标题（模式名·核心机制定调）
2. 引言块（核心理念/为什么爽/模式家族声明）
3. 心理学原理表
4. 通用公式（题材无关5-7步）
5. 变体家族（N变体×要素表）
6. 使用红线（✅必须做/❌不要做）
7. 节奏控制（篇幅×前置期×密度×位置）
8. 套路链推荐（流程图）
9. 跨书案例对比（每拆一书增一列）
10. 案例积累（追加规则+覆盖计数+入库日期）

## Step 4：写入对应模块

| 素材类型 | 写入路径 | 说明 |
|:---------|:---------|:-----|
| L3 具体套路卡 | `原始素材/{书前缀}_{套路名}.md` | 拆书管线产出 |
| 抽象模式卡 | `套路库/{模式名}.md` | 去重+晋升后 |
| **整本 L1 设定** | **`设定库/{书名}/`** | **L1-01~06.md + 世界宪法.md + PRD.md** |
| 文风DNA | `文风DNA/{书名}.md` | pop-shared-dna 产出 |
| 剧情改建思考 | `剧情库/{剧情名}.md` | 原子化叙事模板 |

### 设定库入库流程（按书分目录）

```
拆书项目完成 world 阶段 → L1 六件套已存在
    ↓
1. 复制 {项目}/设计/L1-世界观/L1-01~06.md + 世界宪法.md → 设定库/{书名}/
2. 撰写 PRD.md（含卖点摘要 + 匹配逻辑）
3. 卖点摘要：写出本书的核心卖点 1~N，每个卖点 2-3 句，标注对应 L1 文件
4. 匹配逻辑：写出"如果你的书需要 XX → 看这个文件"的速查表
5. 更新 设定库/00-索引.md
```

### 文风DNA的源文件查找路径

文风DNA档案由 pop-shared-dna 产出，存放在项目工作区。入库时按以下优先级查找：

1. `{项目目录}/写作资产/文风DNA/{书名}.md`（标准路径）
2. `{项目目录}/styles/{书名}.md`（旧版路径）
3. `{项目目录}/**/文风DNA/*{书名}*.md`（模糊搜索）

**找不到文风DNA** → 跳过该模块，写入 `目录.md` 状态为「缺文风DNA」。

### 批量写入套路卡（≥50张时）

当整个拆书项目的套路产出（~100-200张）一次入库时，逐卡检查8字段+格式化+写入不可行。使用**批量工作流**：

1. **源路径**：`{项目目录}/写作资产/套路库/*.md`
2. **排除**：归档总表文件（`套路归档_ch*` `*章套路归档总表*`）
3. **前缀标准化**：文件名前加 `{书前缀}_`（如 `没钱修什么仙_`）
4. **三标签批量打标**：按 Step 1 的启发式规则批量写入 frontmatter
5. **写入**：全部复制到 `原始素材/`
6. **索引**：创建/更新 `原始素材/00-索引.md`

---

## Step 5：更新索引

| 新增位置 | 更新动作 |
|:---------|:---------|
| 原始素材新增 | 创建/更新 `原始素材/00-索引.md`（按书分表 + 各书卡片数 + 入库日期） |
| 套路库新增 | 更新 `套路库/00-总索引.md`（追加行 + 变更记录） |
| 设定库新增 | 更新 `设定库/00-索引.md`（按书追加入库行） |
| 文风DNA新增 | 更新 `目录.md` 文风DNA模块状态 |
| 剧情库新增 | 更新 `目录.md` 剧情库模块状态 |

### 原始素材索引格式

```markdown
# 原始素材 · 总索引

## 入库概览

| 来源书 | 卡片数 | 文件前缀 | 入库日期 |
|:-------|:------:|:---------|:---------|
| 深渊主宰 | 113 | `深渊主宰_` | 2026-06-19 |

## 按书详细列表

### 深渊主宰（113张）
- `深渊主宰_系统觉醒（游戏面板）.md`
- `深渊主宰_隐藏天赋觉醒（神术天赋）.md`
- ...
```

`原始素材/00-索引.md` 不存在时须新建，存在时追加行。

## 入库红线

| # | 红线 |
|:-:|:-----|
| 1 | 质量门槛不达标就入库 — 宁可不入，不可降级入库 |
| 2 | 三标签缺失 — 无标签的素材无法被调用匹配SOP检索 |
| 3 | 强行归并不同模式 — 一条具体卡只属于零个或一个抽象模式 |
| 4 | 已有重复模式不归并 — 同赛道+同核心机制才需合并，不同赛道独立入库 |
| 5 | 入库不更新索引 — 索引不更新=素材不可被发现=白入库 |
