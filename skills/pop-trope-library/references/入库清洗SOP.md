# 入库清洗 SOP

> 用途：把拆书、写书、QA、复盘产出的 PRD 级经验 / 设定包 / 文风档案 / 剧情线，清洗入库到 pop-trope-library 对应模块。
> 原则：一张高质量的抽象模式卡 > 100 张散装具体卡片。精做，不做多。
> 执行者：拆书管线 / 分析 skill / 写书复盘产出入库时。

---

## 入库前置：质量门槛

### PRD 级经验（→ 立项库/）

立项库只收“书怎么立”的经验，不收具体桥段。

可入库类型：

1. 立项模式卡
2. 元爽点组合卡
3. 题材焊接卡
4. 加工哲学卡
5. 失败案例卡

入库要求：

1. 必须说明服务哪个 creative 决策。
2. 必须写明适用赛道 / 元爽点 / 风险。
3. 必须能被下一次 PRD 推导复用。
4. 失败案例必须写清“在哪个下游阶段暴露问题”。

**不达标 → 不入库。不要把立项库变成随笔仓库。**

### 设定素材（→ 设定库/）

设定素材不入库为单张碎片卡，以**整本 L1 设定**为单位入库。

入库要求：

1. 拆书项目必须有完整的 L1 六件套（L1-01~06 + 世界宪法）。
2. 六件套经 world skill 产出并通过稳定性检验。
3. 附带一份 `PRD.md`（含卖点摘要 + 匹配逻辑）。

**不达标 → 先补 L1 设定，不入库。**

### 文风档案（→ 文风库/）

必须满足 pop-shared-dna 的 6 条红线：

1. 原文采样 ≥ 500 字。
2. 场景卡 ≥ 6 个。
3. 通用维度 ≥ 4 个。
4. 有时间演变段。
5. 无降级替换。
6. 对话输出只留摘要，文件内保留必要原文摘录。

路径规则：

1. 唯一路径：`文风库/{书名}.md`

**不达标 → 退回 pop-shared-dna 扩采，不入库。**

### 剧情线（→ 剧情库/，L2 + L3 双轨，均保留原始格式）

剧情库收 L2 单元卡和 L3 剧情线，均保留拆书原始格式，不压缩。必须有：

**L2 单元卡**（保留原始格式直接入库）：
1. **单元信息**：编号、标题、章节范围、类型、字数、POV、情绪弧、核心驱动。
2. **结构分析**：主阶段表（阶段/功能/章节/事件brief/情感设计/节奏）+ 嵌套子线表。
3. **事件链**：逐章逐事件展开（编号/事件/类型/参与角色/功能）。
4. **套路分析**：套路名/所在章节/触发条件/情绪收益/回收预期。
5. **可迁移要点**：分维度（结构设计/情绪操控/角色塑造/信息释放/单元边界），≥5 条。
6. **不可迁移部分**：原书专属设定前提、角色关系。

**L3 剧情线**（保留独立文件）：
1. **线信息**：线编号、线名称、覆盖章节、线型、跨越单元、时间跨度、核心驱动。
2. **阶段推进表**：5-8 阶段，每阶段有承载单元+事件brief+不可逆变化+读者收益。
3. **叙事债务与回收**：至少1条前置递送+回收的债务。
4. **线间咬合**：与其他 L3 线的交叉点。
5. **可迁移/不可迁移**：各3-5条。
6. **起点/终点快照**：身份/战力/金手指/社会关系等维度的变化幅度。

**红线**：不按卷/幕/章级切片直接入库。L2 卡必须含结构分析和可迁移要点。L3 必须含阶段推进表和叙事债务。否则不入公共剧情库。

**不达标 → 退回重提取，不入公共剧情库。**

**📂 数据源查找路径（拆书项目已有 pop-decon-volume v3 产出时）：**
1. 读 `{项目}/设计/L2单元卡/L2-*.md` → 获取 L2 单元卡原始全量分析
2. 读 `{项目}/设计/L3剧情线/L3-*.md` → 获取 L3 剧情线完整分析
3. 加 frontmatter（三标签）+ library 消费说明头部，内容不删不改
4. 写入 `剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md` 和 `剧情库/{标签}/L3/{来源书}-{编号}-{名称}.md`
5. 更新对应标签的 `00-索引.md`：L3 速查表 + L2 索引

> 不满足前置条件（L2 单元卡缺结构分析）→ 先完成 pop-decon-volume 的 L2 提取，再入库。

## Step 1：打三标签（原子化素材 + 立项库部分条目）

> 整本 L1 设定导入不需要打标签，卖点摘要和匹配逻辑在 `PRD.md` 中。
> 文风库主检索键是书名 + `scene`，不强制三标签。

每条原子化素材（套路卡、剧情库条目、立项库可检索卡）打三个标签，写入文件 frontmatter：

```yaml
---
layer: 机制
track: 修仙
meta_joy: 信息差
---
```

| 标签 | 取值 | 判定方式 |
|:-----|:-----|:---------|
| `layer` | 立项 / 框架 / 质感 / 机制 / 调味 | 素材服务的下游层级 |
| `track` | 修仙 / 西幻 / 都市 / 科幻 / 武侠 / 通用 | 从来源书或目标题材判定。跨赛道通用素材标“通用” |
| `meta_joy` | 7 元爽点之一 / 无 | 从套路卡“元爽点家族”或立项卡服务目标判定 |

### 批量打标（≥50 卡时使用）

当一批拆书产出 50+ 张卡需要同时入库时，逐卡人工打标不可行。使用**内容关键词启发式打标**：

```python
# 元爽点检测规则（优先级从高到低，第一条匹配即锁定）
heuristics = {
    '信息差':     ['信息差', '隐藏', '秘密', '先知', '情报'],
    '阶级革命':   ['阶级', '阶层', '穷人', '剥削', '压迫', '底层'],
    '绝境反杀':   ['反杀', '反转', '逆袭', '以弱胜强', '绝境'],
    '身份跃迁':   ['身份', '跃升', '跨越', '上升', '阶级流动'],
    '世界观震撼': ['世界观', '体系', '设定', '制度', '规则', '揭露', '真相'],
    '智斗博弈':   ['智斗', '博弈', '算计', '心理', '骗', '谈判'],
    '情感羁绊':   ['关系', '感情', '羁绊', '信任', '牺牲', '守护'],
}
layer_map = {
    '立项': ['立项', '书型', '题材', '爽点组合', '加工哲学', '失败案例'],
    '框架': ['设定', '世界观', '体系', '制度', '规则', '数值', '等级', '升级', '战斗'],
    '质感': ['命名', '术语', '称呼', '名相', '氛围', '文明'],
    '机制': ['套路', '反转', '冲突', '事件链', '节奏', '钩子'],
    '调味': ['文风', '语气', '句式', '场景卡', '对白'],
}
```

**注意**：批量打标的 `meta_joy` 标签是首次近似值，不做绝对正确承诺。单卡精读时应修正。**禁止用批量打标结果做出“本书元爽点分布”等统计结论。**

## Step 2：去重检查

在目标模块内搜索同名/同义素材：

### 立项库去重

| 情况 | 处理 |
|:-----|:-----|
| 已有同类立项模式，新条目只是同赛道案例 | 在原条目的“跨书案例”段追加 |
| 已有同类立项模式，新条目改动了核心决策机制 | 新增变体，并写明差异 |
| 失败案例和已有失败案例原因相同 | 合并到同一失败模式下 |

### 套路库去重

| 情况 | 处理 |
|:-----|:-----|
| 已有抽象模式卡，新卡是已有变体的新案例 | 在抽象模式卡的“跨书案例对比”表新增一列，变体表“案例”行追加 |
| 已有抽象模式卡，新卡是已有变体的新亚型 | 在“变体家族”区新增一行 |
| 无抽象模式卡 | 提炼新抽象模式卡，放入 `套路库/` |

### 设定库去重

| 情况 | 处理 |
|:-----|:-----|
| 同一本书已有 L1 套件 | 不重复入库，只补 `PRD.md` 或索引 |
| 不同书的设定创意相近 | 不合并 L1 文件，在 `PRD.md` 互相引用 |
| 旧版 `设定库/框架` 或 `设定库/质感` 存量素材 | 标记为 legacy，迁移时并入按书目录 |

**不强行归并**：一条具体卡可以属于零个或一个抽象模式。

## Step 3：抽象层级判定

| 层级 | 位置 | 标准 |
|:-----|:-----|:-----|
| PRD 级经验 | `立项库/` | 能影响 creative 立项决策 |
| L1 整书设定 | `设定库/{书名}/` | 能被 world/creative 按书读取 |
| 文风档案 | `文风库/{书名}.md` | 能被 prose 按书名和 `scene` 读取 |
| L3 剧情线 | `剧情库/{标签}/L3/{来源书}-{编号}-{名称}.md` + `{标签}/00-索引.md` 速查表 | 保留独立文件，同时在标签索引维护速查表 |
| L2 单元卡 | `剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md` | 保留原始格式，可被 plot 改建的行动弧模式 |
| 抽象模式卡 | `套路库/` | 去重通过后提炼，无需跨书次数门槛 |

**晋升规则**：拆书复盘中发现的模式只要不重复（和已有抽象模式卡无语义重叠），就可提炼为抽象模式卡。独特模式比常见模式更有入库价值，优先入库。

晋升时按 `references/抽象模式卡-标准格式.md` 的 10 字段填写：

1. 标题（模式名 · 核心机制定调）
2. 引言块（核心理念 / 为什么爽 / 模式家族声明）
3. 心理学原理表
4. 通用公式（题材无关 5-7 步）
5. 变体家族（N 变体 × 要素表）
6. 使用红线（必须做 / 不要做）
7. 节奏控制（篇幅 × 前置期 × 密度 × 位置）
8. 套路链推荐（流程图）
9. 跨书案例对比（每拆一书增一列）
10. 案例积累（追加规则 + 覆盖计数 + 入库日期）

## Step 4：写入对应模块

| 素材类型 | 写入路径 | 说明 |
|:---------|:---------|:-----|
| 立项模式卡 | `立项库/立项模式/{模式名}.md` | creative 前置参考 |
| 元爽点组合卡 | `立项库/元爽点组合/{组合名}.md` | PRD 爽点组合参考 |
| 题材焊接卡 | `立项库/题材焊接/{焊接名}.md` | 跨题材融合参考 |
| 加工哲学卡 | `立项库/加工哲学/{原则名}.md` | 从拆书到新书的转化原则 |
| 失败案例卡 | `立项库/失败案例/{案例名}.md` | PRD 级风险沉淀 |
| 抽象模式卡 | `套路库/{模式名}.md` | 去重 + 晋升后 |
| 设定素材（整书） | `设定库/{书名}/` | L1 六件套 + 世界宪法 + PRD |
| 角色关系模式 | `设定库/角色与关系/{模式名}.md` | 角色身份张力、关系结构、说话风格 |
| 文风档案 | `文风库/{书名}.md` | 唯一路径 |
| L3 剧情线 | `剧情库/{标签}/L3/{来源书}-{编号}-{名称}.md` | 保留独立文件，按内容标签分目录 |
| L2 单元卡 | `剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md` | 保留原始格式，放入标签的 L2/ 子目录 |
| 剧情线（待定标签） | `剧情库/_INBOX/{来源书}-{名称}.md` | 标签未确定时的暂存区，定期分类迁移 |

### 设定库入库流程（按书分目录）

```text
拆书项目完成 world 阶段 → L1 六件套已存在
    ↓
1. 复制 {项目}/设计/L1-世界观/L1-01~06.md + 世界宪法.md → 设定库/{书名}/
2. 撰写 PRD.md（含卖点摘要 + 匹配逻辑）
3. 卖点摘要：写出本书的核心卖点 1~N，每个卖点 2-3 句，标注对应 L1 文件
4. 匹配逻辑：写出“如果你的书需要 XX → 看这个文件”的速查表
5. 更新 设定库/00-索引.md
```

### 文风档案源文件查找路径

文风档案由 pop-shared-dna 产出，入库时按以下路径查找：

1. `{项目目录}/文风库/{书名}.md`
2. `{项目目录}/**/文风库/*{书名}*.md`

找到后写入 library 的 `文风库/{书名}.md`。

**找不到文风档案 → 跳过该模块，写入 `目录.md` 状态为“缺文风档案”。**

## Step 5：更新索引

| 新增位置 | 更新动作 |
|:---------|:---------|
| 立项库新增 | 更新 `立项库/00-索引.md`（按子目录追加条目 + 风险标签） |
| 套路库新增 | 更新 `套路库/00-总索引.md`（追加行 + 变更记录） |
| 设定库新增 | 更新 `设定库/00-索引.md`（按书追加入库行） |
| 角色关系新增 | 更新 `设定库/角色与关系/00-索引.md` |
| 文风档案新增 | 更新 `文风库/00-索引.md` |
| 剧情库新增 | 更新对应标签目录的 `00-索引.md` + `剧情库/00-索引.md` |

## 入库红线

| # | 红线 |
|:-:|:-----|
| 1 | 质量门槛不达标就入库，宁可不入，不可降级入库 |
| 2 | 立项库收桥段碎片，导致 PRD 层被素材噪音污染 |
| 3 | 三标签缺失，无标签的机制素材无法被调用匹配 SOP 检索 |
| 4 | 文风库强制三标签，导致 prose 按书名和 scene 的主路径失效 |
| 5 | 设定库继续写入 `框架/`、`质感/` 新素材，破坏按书读取 |
| 6 | 剧情库继续把卷/幕/章级切片当主资产，或 L2/L3 未双轨入库，污染写书侧剧情线设计 |
| 7 | 强行归并不同模式，一条具体卡只属于零个或一个抽象模式 |
| 8 | 入库不更新索引，索引不更新 = 素材不可被发现 = 白入库 |
