# 入库清洗 SOP

> 用途：把拆书、写书、QA、复盘产出的素材清洗入库到 pop-trope-library。
> 核心原则：原始资产高保真保存，写作入口通过任务索引组织。

---

## 0. 总流程

```text
1. 判定素材类型
2. 检查质量门槛
3. 写入对应模块
4. 补 frontmatter / 消费摘要
5. 更新模块索引
6. 更新或新建任务索引
7. 刷新 目录.md 状态
```

---

## 1. 素材类型与路径

| 素材类型 | 写入路径 | 必须更新 |
|:---------|:---------|:---------|
| PRD 级经验 | `立项库/{子目录}/{条目}.md` | `立项库/00-索引.md` |
| 书级设定包 | `设定库/{书名}/` | `设定库/00-索引.md` |
| 文风档案 | `文风库/{书名}.md` | `文风库/00-索引.md` |
| L2 单元卡 | `剧情库/{书名}/L2/{书名}-{编号}-{名称}.md` | `剧情库/00-索引.md` + 相关任务索引 |
| L3 剧情线 | `剧情库/{书名}/L3/{书名}-{编号}-{名称}.md` | `剧情库/00-索引.md` + 相关任务索引 |
| 抽象套路卡 | `套路库/{模式名}.md` | `套路库/00-总索引.md` |
| 金手指设计卡 | `金手指库/{类型}/{条目}.md` | `金手指库/00-索引.md` |
| 角色关系模式 | `设定库/角色与关系/{模式名}.md` | `设定库/角色与关系/00-索引.md` |

剧情库不再按粗标签建主目录。粗标签进入 frontmatter，写作检索进入 `任务索引/`。

---

## 2. 质量门槛

### 立项库

只收“书怎么立”的 PRD 级经验，不收具体桥段。

必须说明：

1. 服务哪个 creative/seed 决策。
2. 适用赛道、元爽点、风险。
3. 会影响哪些下游：world/plot/create/revise。

### 设定库

以整书 L1 套件入库：

1. `PRD.md`
2. `L1-01~06`
3. `世界宪法.md`
4. 可选 `起点快照.md`

不把设定拆成散装碎片卡。

### 文风库

必须满足：

1. 原文采样不少于 500 字。
2. 场景卡不少于 6 个。
3. 通用维度不少于 4 个。
4. 有时间演变。
5. 无降级替换。
6. 对话输出保留摘要，文件内保留必要原文摘录。

### 剧情库 L2

L2 必须是完整行动弧，至少包含：

1. 单元信息。
2. 物理坐标。
3. 结构分析。
4. 嵌套子线。
5. 事件链或运行日志。
6. 爽点/套路分析。
7. 可迁移要点。
8. 不可迁移部分。

### 剧情库 L3

L3 必须是跨单元长期线，至少包含：

1. 线信息。
2. 阶段推进表。
3. 叙事债务与回收。
4. 线间咬合。
5. 起点/终点快照。
6. 可迁移/不可迁移。

## 3. frontmatter 与消费摘要

机制层素材建议使用：

```yaml
---
layer: 机制
track: 西幻
meta_joy:
  - 信息差
source_book: 深渊主宰
function_tag: 崛起征服
task_tags:
  - 穿越求生
  - 系统觉醒
  - 第一次收益闭环
best_sections_to_read:
  - 结构分析
  - 嵌套子线
  - 可迁移要点
---
```

已有大卡不必为了补 frontmatter 大规模重写。优先通过 `任务索引/` 和 sidecar 补消费入口。

---

## 4. 去重与晋升

### 套路库

| 情况 | 处理 |
|:-----|:-----|
| 新案例属于已有模式 | 追加到已有模式的案例/变体 |
| 新案例形成新变体 | 在已有模式内新增变体 |
| 无语义重复 | 新建抽象模式卡 |

独特模式不需要“至少两本书出现”才晋升。

### 设定库

不同书的设定包不强行合并。相似设定可在 `PRD.md` 互相引用。

### 剧情库

同一 L2/L3 不重复入库。若同一案例适配多个任务，通过多个任务索引引用同一原文件。

---

## 5. 任务索引更新

每次入库高价值 L2/L3 后，执行：

1. 判断它能服务哪些写作任务。
2. 查 `任务索引/00-索引.md` 是否已有入口。
3. 已有则追加到“必读案例”或“补充案例”。
4. 没有则按 `references/任务索引设计规范.md` 新建。
5. 每条引用写清“读哪些段、学什么、不要照搬什么”。

如果不更新任务索引，这张卡对写作侧基本等于不可见。

## 6. 入库确认清单

```markdown
## 入库确认
- [ ] 目标模块路径正确
- [ ] 质量门槛达标
- [ ] frontmatter/消费摘要已补或声明待补
- [ ] 模块索引已更新
- [ ] 任务索引已更新或声明不适合任务索引
- [ ] 目录.md 状态已刷新
- [ ] 未引用 `_LEGACY/` 作为消费入口
```

|:-:|:-----|
