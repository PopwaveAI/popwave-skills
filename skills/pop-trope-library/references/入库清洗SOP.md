# 入库清洗 SOP

> 用途：把拆书、写书、QA、复盘产出的 PRD 级经验 / L3 具体卡 / 设定包 / 文风分析 / 剧情线，清洗入库到 pop-trope-library 对应模块。
> 原则：一张高质量的抽象模式卡 > 100 张散装具体卡片。精做，不做多。
> 执行者：拆书管线 / 分析 skill / 写书复盘产出入库时。

---

## 入库前置：质量门槛

### PRD 级经验（→ 立项库/）

立项库只收“书怎么立”的经验，不收具体桥段。

可入库类型：

1. 立项模式卡
2. 元爽点组合卡
3. 题材焊接卡
4. 加工哲学卡
5. 失败案例卡

入库要求：

1. 必须说明服务哪个 creative 决策。
2. 必须写明适用赛道 / 元爽点 / 风险。
3. 必须能被下一次 PRD 推导复用。
4. 失败案例必须写清“在哪个下游阶段暴露问题”。

**不达标 → 不入库。不要把立项库变成随笔仓库。**

### L3 具体套路卡（→ 原始素材/）

检查 `references/trope-entry-standard-ideal.md` 的 8 字段是否齐全：

1. 触发条件
2. 套路公式
3. 核心张力
4. 情绪收益
5. 运作机制
6. 使用注意
7. 下游期待
8. 复用记录

**不达标 → 退回拆书管线补全，不入库。**

### 设定素材（→ 设定库/）

设定素材不入库为单张碎片卡，以**整本 L1 设定**为单位入库。

入库要求：

1. 拆书项目必须有完整的 L1 六件套（L1-01~06 + 世界宪法）。
2. 六件套经 world skill 产出并通过稳定性检验。
3. 附带一份 `PRD.md`（含卖点摘要 + 匹配逻辑）。

**不达标 → 先补 L1 设定，不入库。**

### 文风档案（→ 文风库/，兼容 文风DNA/）

必须满足 pop-shared-dna 的 6 条红线：

1. 原文采样 ≥ 500 字。
2. 场景卡 ≥ 6 个。
3. 通用维度 ≥ 4 个。
4. 有时间演变段。
5. 无降级替换。
6. 对话输出只留摘要，文件内保留必要原文摘录。

路径规则：

1. 新规范主路径：`文风库/{书名}.md`
2. 当前兼容路径：`文风DNA/{书名}.md`
3. 已存在于 `文风DNA/` 的档案暂不强制移动，避免打断 prose fallback。

**不达标 → 退回 pop-shared-dna 扩采，不入库。**

### 剧情线（→ 剧情库/）

剧情库主入库对象是剧情线，不是卷/幕/章级切片。必须有：

1. **线型**：主线 / 支线 / 设定线 / 背景线 / 关系线 / 成长线 / 反派线。
2. **核心驱动力**：承压者、压力源、行动方向、失败代价、终局变化。
3. **阶段推进**：触发 → 首次推进 → 复杂化 → 反转/代价 → 高潮释放 → 收束/余波。
4. **叙事债务与回收**：前置递送、必须回收的原因、回收方式。
5. **可迁移/不可迁移边界**：明确哪些能服务新书，哪些只是原书专属卷幕节奏。

**红线**：不按卷/幕/章级切片直接入库。卷/幕只作证据来源或 legacy。

**不达标 → 退回重提取剧情线，不入公共剧情库。**

### 价值点分流清单（→ 五库二级入口）

来自 `pop-decon-design-pack` 每章设计包第 6 节。必须有：

1. 价值点一句话命名。
2. 类型：立项 / 设定 / 角色关系 / 剧情结构 / 章法 / 套路 / 文风 / QA红线。
3. 推荐入库路径。
4. 证据位置：`chXXX-¶YY` 或稳定段落定位。
5. 复用说明：明确服务 creative/world/character/plot/chapter/prose/qa 哪个阶段。

**不达标 → 不入库。价值点不是读后感，必须能被写书侧复用。**

## Step 1：打三标签（原子化素材 + 立项库部分条目）

> 整本 L1 设定导入不需要打标签，卖点摘要和匹配逻辑在 `PRD.md` 中。
> 文风库主检索键是书名 + `scene`，不强制三标签。

每条原子化素材（套路卡、剧情库条目、立项库可检索卡）打三个标签，写入文件 frontmatter：

```yaml
---
layer: 机制
track: 修仙
meta_joy: 信息差
---
```

| 标签 | 取值 | 判定方式 |
|:-----|:-----|:---------|
| `layer` | 立项 / 框架 / 质感 / 机制 / 调味 | 素材服务的下游层级 |
| `track` | 修仙 / 西幻 / 都市 / 科幻 / 武侠 / 通用 | 从来源书或目标题材判定。跨赛道通用素材标“通用” |
| `meta_joy` | 7 元爽点之一 / 无 | 从套路卡“元爽点家族”或立项卡服务目标判定 |

### 批量打标（≥50 卡时使用）

当一批拆书产出 50+ 张卡需要同时入库时，逐卡人工打标不可行。使用**内容关键词启发式打标**：

```python
# 元爽点检测规则（优先级从高到低，第一条匹配即锁定）
heuristics = {
    '信息差':     ['信息差', '隐藏', '秘密', '先知', '情报'],
    '阶级革命':   ['阶级', '阶层', '穷人', '剥削', '压迫', '底层'],
    '绝境反杀':   ['反杀', '反转', '逆袭', '以弱胜强', '绝境'],
    '身份跃迁':   ['身份', '跃升', '跨越', '上升', '阶级流动'],
    '世界观震撼': ['世界观', '体系', '设定', '制度', '规则', '揭露', '真相'],
    '智斗博弈':   ['智斗', '博弈', '算计', '心理', '骗', '谈判'],
    '情感羁绊':   ['关系', '感情', '羁绊', '信任', '牺牲', '守护'],
}
layer_map = {
    '立项': ['立项', '书型', '题材', '爽点组合', '加工哲学', '失败案例'],
    '框架': ['设定', '世界观', '体系', '制度', '规则', '数值', '等级', '升级', '战斗'],
    '质感': ['命名', '术语', '称呼', '名相', '氛围', '文明'],
    '机制': ['套路', '反转', '冲突', '事件链', '节奏', '钩子'],
    '调味': ['文风', '语气', '句式', '场景卡', '对白'],
}
```

**注意**：批量打标的 `meta_joy` 标签是首次近似值，不做绝对正确承诺。单卡精读时应修正。**禁止用批量打标结果做出“本书元爽点分布”等统计结论。**

## Step 2：去重检查

在目标模块内搜索同名/同义素材：

### 立项库去重

| 情况 | 处理 |
|:-----|:-----|
| 已有同类立项模式，新条目只是同赛道案例 | 在原条目的“跨书案例”段追加 |
| 已有同类立项模式，新条目改动了核心决策机制 | 新增变体，并写明差异 |
| 失败案例和已有失败案例原因相同 | 合并到同一失败模式下 |

### 套路库去重

| 情况 | 处理 |
|:-----|:-----|
| 已有抽象模式卡，新卡是已有变体的新案例 | 在抽象模式卡的“跨书案例对比”表新增一列，变体表“案例”行追加 |
| 已有抽象模式卡，新卡是已有变体的新亚型 | 在“变体家族”区新增一行 |
| 无抽象模式卡 | 提炼新抽象模式卡，放入 `套路库/` |

### 设定库去重

| 情况 | 处理 |
|:-----|:-----|
| 同一本书已有 L1 套件 | 不重复入库，只补 `PRD.md` 或索引 |
| 不同书的设定创意相近 | 不合并 L1 文件，在 `PRD.md` 互相引用 |
| 旧版 `设定库/框架` 或 `设定库/质感` 存量素材 | 标记为 legacy，迁移时并入按书目录 |

**不强行归并**：一条具体卡可以属于零个或一个抽象模式。

## Step 3：抽象层级判定

| 层级 | 位置 | 标准 |
|:-----|:-----|:-----|
| PRD 级经验 | `立项库/` | 能影响 creative 立项决策 |
| L1 整书设定 | `设定库/{书名}/` | 能被 world/reservoir 按书读取 |
| 文风档案 | `文风库/{书名}.md` | 能被 prose 按书名和 `scene` 读取 |
| L3 具体卡 | `原始素材/` | 单书单桥段，拆书管线产出 |
| 剧情线 | `剧情库/{书名}/剧情线/` | 可被 plot 改建的长期叙事线 |
| 抽象模式卡 | `套路库/` | 去重通过后提炼，无需跨书次数门槛 |

**晋升规则**：原始素材中的模式只要不重复（和已有抽象模式卡无语义重叠），就可提炼为抽象模式卡。独特模式比常见模式更有入库价值，优先入库。

晋升时按 `references/抽象模式卡-标准格式.md` 的 10 字段填写：

1. 标题（模式名 · 核心机制定调）
2. 引言块（核心理念 / 为什么爽 / 模式家族声明）
3. 心理学原理表
4. 通用公式（题材无关 5-7 步）
5. 变体家族（N 变体 × 要素表）
6. 使用红线（必须做 / 不要做）
7. 节奏控制（篇幅 × 前置期 × 密度 × 位置）
8. 套路链推荐（流程图）
9. 跨书案例对比（每拆一书增一列）
10. 案例积累（追加规则 + 覆盖计数 + 入库日期）

## Step 4：写入对应模块

| 素材类型 | 写入路径 | 说明 |
|:---------|:---------|:-----|
| 立项模式卡 | `立项库/立项模式/{模式名}.md` | creative 前置参考 |
| 元爽点组合卡 | `立项库/元爽点组合/{组合名}.md` | PRD 爽点组合参考 |
| 题材焊接卡 | `立项库/题材焊接/{焊接名}.md` | 跨题材融合参考 |
| 加工哲学卡 | `立项库/加工哲学/{原则名}.md` | 从拆书到新书的转化原则 |
| 失败案例卡 | `立项库/失败案例/{案例名}.md` | PRD 级风险沉淀 |
| L3 具体套路卡 | `原始素材/{书前缀}_{套路名}.md` | 拆书管线产出 |
| 抽象模式卡 | `套路库/{模式名}.md` | 去重 + 晋升后 |
| 设定素材（整书） | `设定库/{书名}/` | L1 六件套 + 世界宪法 + PRD |
| 角色关系模式 | `设定库/角色与关系/{模式名}.md` | 角色身份张力、关系结构、说话风格 |
| 文风档案 | `文风库/{书名}.md` | 新规范主路径 |
| 文风档案（兼容） | `文风DNA/{书名}.md` | 旧资产 fallback，暂不强制迁移 |
| 剧情线 | `剧情库/{书名}/剧情线/{线型}-{编号}-{名称}.md` | 主线/支线/设定线/关系线/成长线/反派线 |
| 剧情线结构模型 | `剧情库/结构模型/剧情线/{模型名}.md` | 跨书提炼后的剧情线模式 |
| 剧情结构模型 | `剧情库/结构模型/{模型名}.md` | 枪链、payoff 密度、长线兑现等辅助结构 |
| 章法模型 | `剧情库/章法模型/{模型名}.md` | 章型、事件链密度、章首承接、章末钩子、信息释放 |
| 角色弧线 | `剧情库/角色弧线/{弧线名}.md` | 主角/配角/关系变化路径 |

### 设定库入库流程（按书分目录）

```text
拆书项目完成 world 阶段 → L1 六件套已存在
    ↓
1. 复制 {项目}/设计/L1-世界观/L1-01~06.md + 世界宪法.md → 设定库/{书名}/
2. 撰写 PRD.md（含卖点摘要 + 匹配逻辑）
3. 卖点摘要：写出本书的核心卖点 1~N，每个卖点 2-3 句，标注对应 L1 文件
4. 匹配逻辑：写出“如果你的书需要 XX → 看这个文件”的速查表
5. 更新 设定库/00-索引.md
```

### 文风档案源文件查找路径

文风档案由 pop-shared-dna 产出，入库时按以下优先级查找：

1. `{项目目录}/写作资产/文风DNA/{书名}.md`
2. `{项目目录}/章节设计包/文风DNA/{书名}.md`
3. `{项目目录}/styles/{书名}.md`
4. `{项目目录}/**/文风DNA/*{书名}*.md`

找到后优先写入 `文风库/{书名}.md`。若当前 prose 仍只识别 `文风DNA/`，可同步保留 `文风DNA/{书名}.md` fallback。

**找不到文风档案 → 跳过该模块，写入 `目录.md` 状态为“缺文风档案”。**

### 批量写入套路卡（≥50 张时）

当整个拆书项目的套路产出（约 100-200 张）一次入库时，逐卡检查 8 字段 + 格式化 + 写入不可行。使用**批量工作流**：

1. **源路径**：`{项目目录}/写作资产/套路库/*.md`
2. **排除**：归档总表文件（`套路归档_ch*`、`*章套路归档总表*`）
3. **前缀标准化**：文件名前加 `{书前缀}_`，如 `没钱修什么仙_`
4. **三标签批量打标**：按 Step 1 的启发式规则批量写入 frontmatter
5. **写入**：全部复制到 `原始素材/`
6. **索引**：创建/更新 `原始素材/00-索引.md`

## Step 5：更新索引

| 新增位置 | 更新动作 |
|:---------|:---------|
| 立项库新增 | 更新 `立项库/00-索引.md`（按子目录追加条目 + 风险标签） |
| 原始素材新增 | 创建/更新 `原始素材/00-索引.md`（按书分表 + 各书卡片数 + 入库日期） |
| 套路库新增 | 更新 `套路库/00-总索引.md`（追加行 + 变更记录） |
| 设定库新增 | 更新 `设定库/00-索引.md`（按书追加入库行） |
| 角色关系新增 | 更新 `设定库/角色与关系/00-索引.md` |
| 文风档案新增 | 更新 `文风库/00-索引.md`；若写入 fallback，同时更新 `目录.md` 的文风DNA兼容状态 |
| 剧情库新增 | 更新 `目录.md` 剧情库模块状态 |
| 剧情库二级资产新增 | 更新对应 `剧情库/{结构模型,章法模型,角色弧线}/00-索引.md` |

### 原始素材索引格式

```markdown
# 原始素材 · 总索引

| 来源书 | 卡片数 | 文件前缀 | 入库日期 |
|:-------|:------:|:---------|:---------|

## 按书详细列表

- `深渊主宰_隐藏天赋觉醒（神术天赋）.md`
- ...
```

`原始素材/00-索引.md` 不存在时须新建，存在时追加行。

## 入库红线

| # | 红线 |
|:-:|:-----|
| 1 | 质量门槛不达标就入库，宁可不入，不可降级入库 |
| 2 | 立项库收桥段碎片，导致 PRD 层被素材噪音污染 |
| 3 | 三标签缺失，无标签的机制素材无法被调用匹配 SOP 检索 |
| 4 | 文风库强制三标签，导致 prose 按书名和 scene 的主路径失效 |
| 5 | 设定库继续写入 `框架/`、`质感/` 新素材，破坏按书读取 |
| 6 | 剧情库继续把卷/幕/章级切片当主资产，污染写书侧剧情线设计 |
| 7 | 强行归并不同模式，一条具体卡只属于零个或一个抽象模式 |
| 8 | 入库不更新索引，索引不更新 = 素材不可被发现 = 白入库 |
