# 写书 PRD 依赖覆盖矩阵

> 用途：从写书管线反推 `pop-trope-library` 应覆盖的模板/灵感类型。
> 结论：当前五库一级结构成立，但必须补足若干二级资产，否则拆书采集会把价值点误塞进套路库。

---

## 1. 写书阶段 → library 依赖

| 写书阶段 | 核心问题 | 需要的 library 资产 | 归属 |
|:---------|:---------|:--------------------|:-----|
| creative | 这本书为什么成立 + 外部素材如何内化 | 书型、读者承诺、元爽点组合、题材焊接、失败案例、加工哲学、跨域改编案例、素材安全红线 | `立项库/` + `设定库/` + `剧情库/` |
| world | 世界怎么搭 | L1 设定包、力量体系、经济结构、术语文明底色 | `设定库/` |
| character | 人物怎么立得住 | 角色原型、身份张力、关系结构、说话风格 | `设定库/角色与关系/` |
| plot | 剧情线怎么推进 | 剧情线卡（六段格式，含阶段推进+叙事债务+线间咬合） | `剧情库/{标签}/` |
| chapter | 单章怎么落地 | 章型、事件链、章末钩子、信息释放、场景调度 | `套路库/` |
| prose | 语言怎么写 | scene 文风、对白节奏、动作描写、感官优先级 | `文风库/` |
| qa | 怎么判断没写崩 | 使用红线、失败案例、兑现缺口、模式误用案例 | `立项库/失败案例/` + `套路库/` |

---

## 2. 当前遗漏风险

| 遗漏类型 | 如果不补，会发生什么 | 推荐处理 |
|:---------|:----------------------|:---------|
| 人物关系模式 | character/chapter 只能凭临场经验写人物 | 新增 `设定库/角色与关系/` |
| QA 红线与失败案例 | 只沉淀成功套路，不沉淀为什么失败 | `立项库/失败案例/` + `套路库/{模式名}.md` 使用红线 |
| 表达技术 | 文风只剩形容词，prose 缺可执行句段手法 | `文风库/{书名}.md` 内按 scene 维护 |

---

## 3. 一级库不扩张原则

短期不新增 `人物库/`、`节奏库/`、`场景库/` 三个一级目录。

原因：

1. 写书 skill 仍在高速迭代，过早新增一级库会造成路径震荡。
2. 现有五库已经能表达“书怎么立 / 世界怎么搭 / 语言怎么写 / 故事怎么推 / 模式怎么复用”。
3. 遗漏类型本质是二级粒度不足，不是一级域缺失。

升级策略：

1. 当前剧情库已从「按书分」重构为「按内容标签分」（崛起征服/血脉觉醒/情感联盟/势力建设/反派对抗/设定揭示），标签集随入库迭代。
2. 章法、结构、角色弧线等二级资产暂不设独立目录——有内容后按其本质入库（结构模型→套路库，章法→pop-writer-chapter references，角色弧线→设定库/角色与关系/）。
3. 若某类资产连续 3 本书以上独立增长，并被多个 writer skill 稳定消费，再考虑拆成一级库。

---

## 4. 拆书侧回流要求

`pop-decon-design-pack` 每章设计包必须输出 `价值点分流清单`：

| 字段 | 说明 |
|:-----|:-----|
| 价值点 | 一句话命名，不超过 20 字 |
| 类型 | 立项/设定/角色关系/剧情结构/章法/套路/文风/QA红线 |
| 推荐入库 | 五库路径或二级目录 |
| 证据位置 | `chXXX-¶YY` 或稳定段落定位 |
| 复用说明 | 说明服务哪个写书阶段 |
| 是否入库 | 是/否/待复盘 |

没有证据位置、不能服务写书阶段、只能解释“我觉得好看”的观察，不入库。
