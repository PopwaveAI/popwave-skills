# 创意溯源 → 设计模式SOP

> **用途**：把拆书产生的"创意溯源"数据（融合案例/原创性类型/参考域图谱）转化为可复用的**卷/幕级设计模式**，再下钻到章级套路选择。
>
> **核心理念**：创意溯源产出的是"这个好创意是怎么产生的"分析报告。设计模式是"下次我可以怎么产生这种好创意"的操作指南。
>
> **前置条件**：已完成一本小说的拆书 → 产出了创意溯源目录（含参考域文件 + 作者原创贡献 + 跨域参考索引）

---

## 一、核心框架：双尺度创作模型

```
卷/幕级（格局层）                           章级（执行层）
┌─────────────────────────┐         ┌────────────────────────┐
│  创意溯源 → 设计模式      │ 分解 →  │  套路库 → 套路选择     │
│                         │         │                        │
│  作用：回答               │         │  作用：回答              │
│  "这一卷/幕的核心冲突      │         │  "这一章具体怎么写       │
│   是什么？"               │         │   能兑现设计承诺？"       │
│  "为什么这个设定是原创的？" │         │  "用什么套路单元         │
│  "需要融合哪些参考域？"    │         │   能让读者爽到/痛到？"    │
└─────────────┬───────────┘         └───────────┬────────────┘
              │                                 │
              └──────── 映射：设计→执行 ──────────┘
                    每个卷/幕设计模式
                    下钻为套路链（3-5个套路）
```

| 尺度 | 输入 | 产出 | 消费方 |
|:-----|:-----|:-----|:-------|
| **卷/幕级** | 创意溯源的融合案例 + 原创性类型 + 参考域图谱 | 设计模式（可复用的设定+冲突模板） | pop-writer-plot（设计剧情线）|
| **章级** | 套路库原始素材 + 元爽点-变体映射表 | 套路选择（情绪曲线+节奏安排） | pop-writer-chapter（设计章纲）|

## 二、SOP Step 1：从融合案例提炼设计模板

从创意溯源的"参考域-*.md"文件中提取融合案例，每个案例可以提炼为一个设计模板。

### 模板提取公式

```
原始数据（创意溯源记录）：
  参考域A × 参考域B × 参考域C → 融合结果
  （如：阴符经哲学 × 开源文化 × 阶级革命 → 五贼盗天功法）

提取为设计模板：
  模板名称：基于融合手法命名（如"哲学×现代机制×社会隐喻三重融合模板"）
  焊接点：三个域的共享底层语义是什么？
  转换操作：具体怎么焊的？（抽象→具象 / 结构移植 / 功能反转）
  适用尺度：卷级（全卷核心设定）/ 幕级（一幕内完成的设定突破）
  避坑红线：这种融合什么情况下会生硬？
```

### 设计模板格式

```markdown
## 设计模板：[名称]

### 融合配方
- **域A**：[来源域 + 提取了什么核心元素]
- **域B**：[来源域 + 提取了什么核心元素]
- **域C**：（可选）[来源域 + 提取了什么核心元素]

### 焊接点（为什么这三个域能焊在一起？）
[共享的底层语义。如果三个域没有共享语义，融合会生硬]

### 转换操作
- [原件A] → [转化后呈现]（操作类型：抽象→具象 / 结构移植 / 功能反转 / 概念跨界）
- [原件B] → [转化后呈现]
- [原件C] → [转化后呈现]

### 参考案例
- 书名/设定：[具体案例]
- 原文表现：这个融合在原文中如何呈现的？
- 效果验证：达到了什么效果？（原创性？设定深度？读者反馈？）

### 适用判断

| 判断 | 标准 |
|:----|:-----|
| ✅ 适合用这个模板 | [什么场景适合] |
| ❌ 不适合 | [什么场景不适合] |

### 避坑红线
- ❌ [常见错误1]
- ❌ [常见错误2]

### 化用示例（假设场景）
[用这个模板为一个假设的新书写一个设计的例子，让使用者能照着做]
```

### 从"让魔门"提取的融合模板示例

## 三、SOP Step 2：从原创性类型到Plot化用

创意溯源的"作者原创贡献"分类了5种原创性类型。这5种类型不应只是事后标签——它们是**创作前的生成策略**。

### 5种原创性类型 → 生成策略

#### 策略1：颠覆性重构

> **定义**：对经典概念进行根本性的功能/价值反转。

**操作指南**：
```
1. 选一个读者熟悉的"传统设定"
2. 写下它的"核心功能/价值"（在读者预期中它用来做什么）
3. 问"如果核心功能完全反过来会怎样？"
4. 验证反转后是否产生新的叙事可能性
5. 检查是否与世界观自洽（不能为了反而反）
```

**让魔门案例**：
- 传统JOJO梗（一个搞笑台词）→ 正反转为全书主题锚点（"不做人才能突破灵根限制"）
- 传统《阴符经》（顺天应道）→ 反转为"盗天为己"
- 传统挂蜡工艺（保存药性）→ 反转为悬疑叙事装置

**plot化用方式**：
- 当某一卷需要"突破性设定"时使用
- 适合放在**卷中段**（传统设定已经建立，可以做了）

#### 策略2：概念跨界

> **定义**：将A领域的结构移植到B领域并产生质变。

**操作指南**：
```
1. 选两个独立的域（A和B）
2. 找到A领域的一个"结构/机制"（如借贷、开源、存档）
3. 找到B领域的一个"命名体系/设定框架"（如修仙、武侠、系统流）
4. 问"如果把这个结构翻译到那个命名体系里，会变成什么？"
5. 验证质变是否产生（不是简单替换名字，而是机制移植后产生新功能）
```

**让魔门案例**：
- 金融借贷结构（贷款/透支/复利）→ 修仙命名体系（一诺千金/与日偕亡）
- 游戏存档-读档机制 + 开局选种族 → 修仙世界观（锚定身份系统）
- 开源文化（代码共享）→ 仙侠世界（功法开源 = 生产资料公有化）

**plot化用方式**：
- 当某一幕需要"经济驱动型冲突"时使用
- 适合作为**幕级驱动引擎**（而非一次性的章设定）

#### 策略3：三重融合

> **定义**：三种或以上独立参考源的概念融合成全新体系。

**操作指南**：
```
1. 选三个独立的域（A、B、C）
2. 从每个域提取一个核心元素
3. 找到三个元素的"共享底层语义"——如果它们有一个共同的主题/逻辑
4. 检查三个元素之间是否有冲突（如果有，需要牺牲一个或找到兼容方案）
5. 命名融合结果（名称要兼顾三个域的辨识度）
```

**让魔门案例**：
- 域外天魔系统 = 网文系统流（死后复活+身份兑换）× 佛教佛学（他化自在天）× 仙侠修真（器灵修炼）
- 共享语义："非人状态获得突破性力量"
- 六欲天自在成就法 = 佛教六欲天（命名）× 御兽流（炼化）× 废灵根逆袭（叙事命题）

**plot化用方式**：
- 全书级核心设定适合用三重融合
- 适合放在**卷首或卷末**（要么一上来就用，要么作为终极揭秘）

#### 策略4：系统化重组

> **定义**：将散落于各参考域的部分碎片整合为完整自洽的新系统。

**操作指南**：
```
1. 在多个参考域中找出"有隐式关联但没被整合"的碎片
2. 为它们设计统一的维度框架
3. 定义维度之间的关系（互斥？并行？递进？）
4. 检查系统完整性：是否有角色用不上？是否有缺口？
```

**让魔门案例**：
- 外功六要素：将武侠"金钟罩铁布衫"的笼统概念拆解为6个独立可量化维度（皮肉筋骨血髓）
- "伥与食"双用内劲：将传统吸功大法的"吸收=拥有"拆分为两阶段（吸收≠拥有）
- 太平间：将系统流的功能性面板升级为叙事结算舞台

**plot化用方式**：
- 当需要一个"让读者有在玩RPG的感觉"的体系时使用
- 适合**幕到卷之间的过渡**（格局判断：出场人物变多→需要一个统一的量化体系来管理复杂度）

#### 策略5：无直接参考

> **定义**：在前37章可考参考域中无对应原型。

**操作指南**：
```
1. 回顾所有已查明的参考域
2. 找出"任何现有参考域都无法解释"的设定
3. 确认它真的无参考（还是参考域没覆盖到？）
4. 记录"如果以后要教别人这么做，核心独创点在哪"
```

**让魔门案例**：
- 因果闭环式"信息差"叙事模式（多重信息不对称驱动的三层嵌套叙事）
- 阶级批判嵌入爽文框架（双重叙事结构）
- 王平三重背叛叙事（制度性背叛三部曲）

**plot化用方式**：
- 这些通常不是单个设定，而是**叙事结构/叙事策略/全书级手法**
- 更适合作为**创作理念**而非具体设定来化用

## 四、SOP Step 3：从参考域图谱到设计选择策略

创意溯源的跨域参考索引将14个参考域按"层位"分类。这个图谱是**写作时的参考域选择策略工具**。

### 各层在创作中的作用

| 层位 | 包含参考域 | 写作时的作用 | 典型触发场景 |
|:-----|:----------|:-----------|:------------|
| **世界观层**（设定基底） | 阴符经哲学、丹道内丹术、仙侠修真、武侠传统 | 提供世界运转的核心概念和修炼体系 | 设计力量体系/宗门结构/修炼规则时 |
| **叙事结构层**（情节驱动） | 古典小说历史、金融借贷、阶级批判、网文系统流、三十六计 | 提供情节设计和角色博弈的策略工具 | 设计冲突/布局/反转/压迫时 |
| **氛围层**（风格色彩） | 穿越者网络梗、炼丹传统、佛教佛学 | 提供叙事风格和读者共鸣 | 需要吐槽缓解/古风命名/美学包装时 |
| **基础设施层**（底层哲学） | DND 3.5e、策略游戏模拟经营 | 提供系统性规则框架和游戏化思维 | 设计量化体系/资源管理/科技树时 |

### 卷级设计时的域选择策略

基于"让魔门"的经验，不同卷主题应该加载不同参考域组合：

| 你想写的是 | 优先加载的参考域 | 重点关注 | 产出案例 |
|:----------|:----------------|:---------|:---------|
| **"制度压迫+底层反噬"卷** | 阶级批判文学 + 金融借贷概念 + 古典小说历史 | 剥削机制设计、压迫叙事节奏、官逼民反母题 | 让魔门卷一/卷二 |
| **"功法揭秘+体系突破"卷** | 阴符经哲学 + 网文系统流 + 仙侠修真 | 哲学根基、金手指进化、体系裂缝 | 让魔门卷三 |
| **"权谋博弈+多方制衡"卷** | 三十六计 + 古典小说历史 + 穿越者网络梗 | 计谋布局、势力博弈、吐槽解构 | 让魔门ch009-ch018钓鱼博弈 |
| **"新地图/新体系"卷** | 仙侠修真 + 武侠传统 + 策略游戏 | 体系对比、设定拓展、资源管理 | 让魔门赤土→仙门篇 |
| **"角色成长+情感弧线"卷** | 武侠传统 + 佛教佛学 + 穿越者网络梗 | 心法突破、符号美学、孤独感 | — |

### 跨域融合的"三明治原则"（从让魔门提炼）

```
表层：传统/东方命名（建立读者熟悉感）
中层：现代/西方机制（提供创新性）
底层：阶级批判/哲学根基（赋予深度）

例：一诺千金
  → 表层 = 古典典故（读者熟悉）
  → 中层 = 金融贷款机制（创新性）
  → 底层 = 信用即权力的阶级分析（深度）
```

**使用方式**：设计任何一个核心设定时，检查是否有这三层。缺一层的设定容易出现"方向正确的废话"。

## 五、SOP Step 4：从卷/幕设计到章级套路链

卷/幕级的设计模式（来自Step 1-3）需要下钻到章级执行。这个"设计→执行"的映射是**创意溯源和套路库的结合点**。

### 映射工作流

```
卷/幕设计决策
  │
  ├── 确定"这一卷/幕需要让读者体验什么情绪"（来自设计模式的核心张力）
  │     ↓
  ├── 按元爽点筛选套路变体（查 元爽点-变体映射表.md）
  │     ↓
  ├── 选出3-5个套路组成"套路链"（考虑情绪曲线 + 节奏安排）
  │     ↓
  └── 按章分配（每章1-2个套路 → 填写章节设计）
```

### 卷/幕设计 → 套路链映射示例

#### 示例1：让魔门"秋税真相"幕（ch019-020）

| 设计决策 | 来源 | 映射到套路 |
|:---------|:-----|:----------|
| 卷级：阶级压迫制度揭露 | 创意溯源→阶级批判文学 + 金融借贷 | **黑暗真相·系统揭秘**（幕级消化） |
| 幕级：主角从个人情感到阶级正义 | 创意溯源→作者原创贡献#5 | **阶级跃升·反向坠落**（幕级弧线） |
| 章级：每章具体套路 | 套路库 | ch019: 黑暗真相+阶级跃升反向坠落 |
| | | ch020: 先扬后抑+章末悬念炸弹 |

#### 示例2：让魔门"钓鱼博弈"幕（ch009-018）

| 设计决策 | 来源 | 映射到套路 |
|:---------|:-----|:----------|
| 卷级：权谋博弈+信息差叙事 | 创意溯源→三十六计 + 古典小说 | **钓鱼博弈**母套路（跨章部署）|
| 幕级：三层嵌套博弈 | 创意溯源→因果闭环信息差模式 | **套娃卧底** + **双钩布局** |
| 章级：每章具体套路 | 套路库 | ch009: 钓鱼博弈（建立） |
| | | ch014: 钓鱼博弈（升级三人博弈）|
| | | ch017: 双线叙事+章末钩子 |
| | | ch018: 钓鱼博弈（闭环+第四方截胡）|

### 套路链设计规则

| 规则 | 说明 | 违反后果 |
|:----|:-----|:---------|
| **情绪不可单调** | 3-5个套路尽量覆盖不同情绪（爽/反转/压迫/宣泄） | 读者情绪疲劳 |
| **大小套路搭配** | 大套路（跨章钓鱼博弈）套小套路（单章章末钩子） | 只有大套路缺执行细节 |
| **上游套路给下游套路留钩子** | 每个套路结束后留下个套路的触发条件 | 章节间断裂 |
| **卷首不扔最大爽点** | 最大爽点放在卷中或卷末 | 后续乏力 |
| **元爽点不要重复** | 连续两章用同一个元爽点稀释效果 | 审美疲劳 |

```
│  【阶段B：创意溯源→设计模式】（本文SOP核心）                  │
│                                                            │
│  输入：创意溯源数据                                          │
│  执行：                                                    │
│   Step 1：融合案例 → 设计模板（提取焊接点+转换操作+红线）      │
│   Step 2：原创性类型 → 生成策略（5种类型→5种操作指南）        │
│   Step 3：参考域图谱 → 选择策略（按卷主题选参考域组合）        │
│  产出：设计模式库（卷/幕级）                                 │
└──────────────────────────┬─────────────────────────────────┘
                           ↓
┌──────────────────────────┴─────────────────────────────────┐
│  【阶段C：卷/幕设计 → 套路链】                               │
│                                                            │
│  输入：设计模式 + 元爽点-变体映射表                           │
│  执行：                                                    │
│   Step 4.1：从设计模式提取"核心情绪目标"                      │
│   Step 4.2：按元爽点筛选套路变体                             │
│   Step 4.3：选3-5个套路组成套路链（情绪曲线+节奏）            │
│   Step 4.4：按章分配（每章1-2个套路）                        │
│  产出：套路链（章级执行计划）                                │
└──────────────────────────┬─────────────────────────────────┘
                           ↓
┌──────────────────────────┴─────────────────────────────────┐
│  【阶段D：套路库→写章】                                      │
│                                                            │
│  输入：套路卡片（原始素材/让魔门再次伟大/*.md）                │
│  执行：按套路卡片的"触发条件/公式/核心张力/使用注意"写章       │
│  产出：小说章节正文                                          │
└────────────────────────────────────────────────────────────┘
```

## 七、新手使用指南

如果你是第一次接触这套方法论，按以下顺序使用：

### 场景1：我有拆书数据了，想给我的新书做设计

```
1. 打开创意溯源的跨域参考索引 → 看"风格特征总结"（理解这本书的创作方法论）
2. 打开作者原创贡献 → 看"全书级原创贡献Top 5"（找最牛逼的设定）
3. 运行 Step 1：把Top 5融合案例提炼为设计模板
4. 运行 Step 2：选择你想用的原创性类型（推荐从"概念跨界"或"颠覆性重构"开始）
5. 运行 Step 3：按你想写的卷主题，选参考域组合
6. 打开套路库 00-索引.md → 按情绪收益速查 → 选3-5个套路
7. 运行 Step 4：设计套路链 → 开始写章
```

### 场景2：我已经有一章要写，只是想找合适的套路

```
1. 打开套路库 00-索引.md → 按情绪收益速查
2. 找"我现在要让读者有什么感受"
3. 打开对应的套路卡片 → 读"触发条件"和"公式"
4. 复用到你的章里（注意改"使用注意"的红线）
```

### 场景3：我想创建一个新的创意溯源（拆一本新书）

```
1. 读创意溯源的README.md → 理解14个参考域的分类层级
2. 按"参考域-*.md"的格式开始记录（每条记录格式：章节 | 参考来源 | 原文触发→转化后呈现 | 备注）
3. 写完所有参考域 → 写跨域参考索引（聚合分析）
4. 写完跨域参考索引 → 写作者原创贡献（提取5种原创性类型的案例）
5. 运行本文SOP → 提炼设计模式 → 入库套路库
```

## 附录：术语表

| 术语 | 定义 |
|:----|:-----|
| **创意溯源** | 拆书产生的"一个元素来自哪个参考域"的记录。事后分析，读者能看。 |
| **设计模板** | 从创意溯源的融合案例中提炼的可复用配方。创作前能用。 |
| **原创性类型** | 5种创造力策略（颠覆性重构/概念跨界/三重融合/系统化重组/无直接参考）。 |
| **参考域** | 一个外部知识领域（如阴符经哲学、金融借贷）在本小说中作为创意来源被引用。 |
| **域选择策略** | 给定写作目标，应该加载哪些参考域来获取设计灵感。 |
| **套路链** | 按情绪曲线排列的3-5个套路序列，覆盖一章到一幕的剧情推进。 |
| **套路卡片** | 单个叙事单元的craf recipe（触发条件/公式/核心张力/情绪收益/运作机制/使用注意/下游期待）。 |
