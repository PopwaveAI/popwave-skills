# 剧情库入库标准（L2+L3）

> 用途：规范剧情库以 L2 可迁移单元卡为主入库资产，L3 以速查表形式维护在标签索引中，避免卷/幕/章级切片污染写书侧。
> **5级结构对齐**：L2 单元卡（有起承转合的完整行动弧）为主入库对象，L3 剧情线（跨单元的主题追踪）只维护速查表在标签的 00-索引.md 中。

---

## 1. 为什么不按卷/幕入库

卷和幕是原书的物理组织方式，不是新书可直接复用的创作资产。

如果按卷/幕入库，写书侧会被三类噪声带偏：

1. **篇幅污染**：误以为原书 30 章一幕、250 章一卷就是本书也该采用的节奏。
2. **结构污染**：把原书多线混合状态当作单一可复用模式。
3. **目标污染**：为了复刻原书幕效果，反而偏离本书 PRD 的读者承诺。

所以公共 library 中，卷/幕只能作为证据来源、拆书中间产物或 legacy，不作为主消费入口。

---

## 2. 什么是可入库剧情线

剧情线是一条长期叙事逻辑：

```text
承压者/行动者 → 压力源 → 行动方向 → 阶段变化 → 反转/代价 → 兑现/收束
```

它必须能被 `pop-writer-plot` 用来设计剧情线。当前标签集（可迭代）：
- 崛起征服 / 血脉觉醒 / 情感联盟 / 势力建设 / 阶级揭露 / 反派对抗 / 设定揭示

---

## 3. 主路径

单书拆出的 L2 可迁移单元卡按内容标签入库，L3 不独立入库：

```text
剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md          ← L2 可迁移单元卡（主入库资产，~50-70 行）
剧情库/{标签}/00-索引.md 的 L3 速查表                 ← L3 剧情线（不建独立文件，以 1-2 行表格维护在标签索引）
```

标签从当前标签集中选一个；跨标签的 L2 卡选最匹配的主标签。L3 速查表格式：

```markdown
## L3 剧情线速查（索引层参考，不作为独立文件入库）

| 书名 | 编号 | 线名 | 跨L2单元 | 一句定调 | 可跨书复用的模式 |
|:-----|:----:|:-----|:--------:|:---------|:----------------|
| {来源书} | {编号} | {线名} | {L2-001~L2-012} | {一句话} | {模式1/模式2} |
```

标签未确定时，L2 卡先入 `_INBOX/`。

---

## 4. 标准格式

### L2 单元卡（保留拆书原始格式，不压缩）

拆书产出的 L2 卡直接入库，不做压缩。原始格式包含以下全部内容：

```markdown
# L2-{编号} · {名称}（chXXX-chYYY）

## 1. 单元信息（编号/标题/章节/类型/字数/POV/情绪弧/核心驱动）
## 2. 结构分析
  ### 2.1 主阶段（阶段/功能/章节/事件brief/情感设计/节奏）
  ### 2.2 嵌套子线（子线/阶段覆盖/章节/事件brief/情感设计/节奏）
## 3. 事件链（逐章逐事件：编号/事件/类型/参与角色/功能）
## 4. 套路分析（套路名/所在章节/触发条件/情绪收益/回收预期）
## 5. 可迁移要点（分维度：结构设计/情绪操控/角色塑造/信息释放/单元边界）
## 6. 不可迁移部分（世界观专属/角色绑定/专属设定）
```

> 入库时仅做两件事：① 加 frontmatter（三标签）② 在文件顶部加 library 消费说明（来源书/标签/入库目的）。内容不删不改。

### L3 剧情线（保留独立文件 + 速查表）

L3 保留独立文件，入 `{标签}/L3/{来源书}-{编号}-{名称}.md`。同时在对应标签的 `00-索引.md` 维护速查表（1-2 行/条）作为快速检索入口。L3 文件包含完整的阶段推进表、叙事债务、线间咬合、起点/终点快照等分析内容。速查表是目录，L3 文件是正文，两者互补。

## 5. 入库判定

### L2 可迁移单元卡判定

| 问题 | 是 | 否 |
|:-----|:---|:---|
| 它是有完整起承转合的行动弧吗？ | 继续 | 不入剧情库，重审素材类型 |
| 它有可迁移的结构模式（不被原书独有设定绑定）吗？ | 继续 | 退回设计包补「可迁移要点」 |
| 物理坐标（时间/地点/人物）可清晰界定？ | 入库 | 退回补全坐标 |
| 嵌套子线有明确的闭合/延伸/待续状态？ | 入库 | 退回补全子线状态 |
| 它只是单个桥段或世界规则吗？ | 入 `套路库/` 或 `设定库/` | 继续 |

### L3 剧情线速查判定（是否写入标签索引的 L3 表）

| 问题 | 是 | 否 |
|:-----|:---|:---|
| 它是一条持续推进的长期叙事线？ | 继续 | 不入速查表 |
| 它跨 ≥4 个 L2 单元？ | 继续 | 线太短，不单独标注 |
| 它有承压者、压力源、失败代价和终局变化？ | 入库速查表 | 只写线名，不写定调 |
| 它只是单个 L2 单元的内容？ | 不入速查表 | — |

---

## 6. 存量处理

现有 `剧情库/{书名}/` 下的幕级/卷级文件全部标记为 legacy 证据，移入 `剧情库/_LEGACY/{书名}/`。

迁移时不要直接删除；从中提取 L2 单元卡和 L3 剧情线，按原始格式写入：

```text
剧情库/{标签}/L2/{来源书}-{编号}-{名称}.md   ← L2 单元卡（原始格式）
剧情库/{标签}/L3/{来源书}-{编号}-{名称}.md   ← L3 剧情线（独立文件）
```

---

## 7. 写作侧消费映射

写作侧 L2 单元卡引用 library 的 L2 卡编号；L3 剧情线通过标签索引速查表获得跨单元认知，需要深度参考时读 L3 独立文件。

| 拆书产出 | library 形态 | 写作侧消费方式 |
|:---------|:------------|:--------------|
| L3 剧情线（跨单元追踪） | `{标签}/L3/{来源书}-{编号}-{名称}.md`（独立文件）+ `{标签}/00-索引.md` 速查表 | plot 读标签索引速查表 → 获得线名称+跨L2范围+一句定调 → 需要深度参考时读 L3 独立文件 |
| L2 单元卡（行动弧） | `{标签}/L2/{来源书}-{编号}-{名称}.md`（原始格式） | plot 直接读 L2 卡的结构分析+事件链+套路分析+可迁移要点 → 改造为新书的 L2 单元卡 |
