# 剧情库入库标准（L2+L3）

> 用途：规范剧情库收 L2 可迁移单元卡 + L3 剧情线（双轨），避免卷/幕/章级切片污染写书侧。
> **5级结构对齐**：L2 取代幕纲（有起承转合的完整行动弧），L3 取代卷纲剧情线（跨单元的主题追踪）。两者并行入库。

---

## 1. 为什么不按卷/幕入库

卷和幕是原书的物理组织方式，不是新书可直接复用的创作资产。

如果按卷/幕入库，写书侧会被三类噪声带偏：

1. **篇幅污染**：误以为原书 30 章一幕、250 章一卷就是本书也该采用的节奏。
2. **结构污染**：把原书多线混合状态当作单一可复用模式。
3. **目标污染**：为了复刻原书幕效果，反而偏离本书 PRD 的读者承诺。

所以公共 library 中，卷/幕只能作为证据来源、拆书中间产物或 legacy，不作为主消费入口。

---

## 2. 什么是可入库剧情线

剧情线是一条长期叙事逻辑：

```text
承压者/行动者 → 压力源 → 行动方向 → 阶段变化 → 反转/代价 → 兑现/收束
```

它必须能被 `pop-writer-plot` 用来设计剧情线。当前标签集（可迭代）：
- 崛起征服 / 血脉觉醒 / 情感联盟 / 势力建设 / 阶级揭露 / 反派对抗 / 设定揭示

---

## 3. 主路径

单书拆出的剧情线按内容标签入库：

```text
剧情库/{标签}/{来源书}-{编号}-{名称}.md
```

标签从当前标签集中选一个；跨标签的线选最匹配的主标签。标签未确定时先入 `_INBOX/`。

## 5. 入库判定

| 问题 | 是 | 否 |
|:-----|:---|:---|
| 它是一条持续推进的叙事线吗？ | 继续 | 不入剧情库 |
| 它有承压者、压力源、失败代价和终局变化吗？ | 继续 | 退回补全 |
| 它能脱离原书卷/幕结构被复用吗？ | 入库 | 降级为证据，不入公共库 |
| 它只是单个桥段吗？ | 入 `套路库/` | 继续 |
| 它只是世界规则吗？ | 入 `设定库/` | 继续 |

---

## 6. 存量处理

现有 `剧情库/{书名}/` 下的幕级/卷级文件全部标记为 legacy 证据，移入 `剧情库/_LEGACY/{书名}/`。

迁移时不要直接删除；从中重提取 L2 单元卡和 L3 剧情线，按双轨格式写入：

```text
剧情库/{标签}/{来源书}-L2-{编号}-{名称}.md   ← L2 可迁移单元卡
剧情库/{标签}/{来源书}-{编号}-{名称}.md      ← L3 剧情线
```
