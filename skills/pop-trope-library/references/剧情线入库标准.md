# 剧情线入库标准

> 用途：规范剧情库只收“剧情线”，避免卷/幕/章级切片污染写书侧。

---

## 1. 为什么不按卷/幕入库

卷和幕是原书的物理组织方式，不是新书可直接复用的创作资产。

如果按卷/幕入库，写书侧会被三类噪声带偏：

1. **篇幅污染**：误以为原书 30 章一幕、250 章一卷就是本书也该采用的节奏。
2. **结构污染**：把原书多线混合状态当作单一可复用模式。
3. **目标污染**：为了复刻原书幕效果，反而偏离本书 PRD 的读者承诺。

所以公共 library 中，卷/幕只能作为证据来源、拆书中间产物或 legacy，不作为主消费入口。

---

## 2. 什么是可入库剧情线

剧情线是一条长期叙事逻辑：

```text
承压者/行动者 → 压力源 → 行动方向 → 阶段变化 → 反转/代价 → 兑现/收束
```

它必须能被 `pop-writer-plot` 用来设计：

- 主线
- 支线
- 设定线 / 背景线
- 关系线
- 成长线
- 反派线

---

## 3. 主路径

单书拆出的剧情线：

```text
剧情库/{书名}/剧情线/{线型}-{编号}-{名称}.md
```

跨书提炼后的剧情线结构模型：

```text
剧情库/结构模型/剧情线/{模型名}.md
```

## 5. 入库判定

| 问题 | 是 | 否 |
|:-----|:---|:---|
| 它是一条持续推进的叙事线吗？ | 继续 | 不入剧情库 |
| 它有承压者、压力源、失败代价和终局变化吗？ | 继续 | 退回补全 |
| 它能脱离原书卷/幕结构被复用吗？ | 入库 | 降级为证据，不入公共库 |
| 它只是单个桥段吗？ | 入 `套路库/` | 继续 |
| 它只是世界规则吗？ | 入 `设定库/` | 继续 |

---

## 6. 存量处理

现有 `剧情库/{书名}/幕*.md`、`卷*.md`、`Uxx-chxxx.md` 文件全部标记为 legacy 证据。

迁移时不要直接删除；从中重提取剧情线，写入：

```text
剧情库/{书名}/剧情线/
```
