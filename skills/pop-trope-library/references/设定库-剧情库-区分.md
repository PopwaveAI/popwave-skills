# 设定库 vs 剧情库：根本区分

> **一句话**：设定库 = 世界怎么运转。剧情库 = 故事怎么推进。

---

## 判定流程图

```
你这张卡描述的是什么？
        │
        ▼
┌─ 世界的一个规则/制度/体系/命名/力量框架？ ──→ 设定库
│
└─ 一段剧情弧线/事件链/冲突结构/叙事节奏？ ──→ 剧情库
```

---

## 对比表

| 维度 | 设定库 | 剧情库 |
|:-----|:-------|:-------|
| **回答的问题** | 这个世界有什么规则？ | 这个故事怎么展开？ |
| **典型内容** | 力量体系、数值曲线、修炼制度、命名策略、文明底色、腐败体系 | 章级事件链、冲突结构、情绪弧线、悬念布局、多线交织方式 |
| **入库来源** | L1 世界观文件、创意溯源参考域、设计包中的制度性描述 | pop-decon-volume v2.2.1+ Step 3.5 从设计包+L2单元卡提取 |
| **格式** | 源域 + 转化逻辑 + 适用场景 + 赛道归属 + 核心机制 | 结构描述事件链 + 结构解析 + 金句 + 复用要点 |
| **样例** | 「法力贷体系：修仙者用未来修为抵押借款」「筑基资格证制度：三关考试+资格证附带权益」 | 「三级竞赛递进弧线：法赛→武道赛→筑基考试的三级阶梯骨架」「信息差截胡式剧情链：先知先觉→暗赚反转→阶级觉醒」 |
| **常见误用** | ❌ 把「筑基资格证」放进剧情库——这描述的是世界制度 | ❌ 把「W型情绪曲线」放进设定库——这描述的是剧情结构 |

---

## 自我检查清单

把卡写完后，逐条问自己：

- [ ] 这张卡的主语是「这个世界有XX规则」还是「这个故事有XX结构」？
- [ ] 如果去掉所有具体书名/角色名，剩下的内容是通用规则还是通用结构？
- [ ] 如果读卡的人想「在我的小说里用这个」，他会去改设定还是改剧情？
- [ ] 这张卡介绍的是一个「可以存在的东西」还是一个「可以发生的事」？

**如果答案是「存在的东西」→ 设定库。如果是「发生的事」→ 剧情库。**

---

## 历史教训

**本 session 教训**：最初把21个设定文件误放入剧情库。法力贷体系、筑基资格证、分数种姓、正神腐败、牛马经济学——这些都是世界观制度，不是剧情结构。导致剧情库需要全部清空重建。

**根因**：入库时只看到了「创意溯源」「跨域融合」的标签，没有问「这描述的是规则、剧情线，还是原书切片」。修正后剧情库主模型改为剧情线；旧章级/幕级切片仅作为 legacy 证据，设定库补回22+11张卡。
