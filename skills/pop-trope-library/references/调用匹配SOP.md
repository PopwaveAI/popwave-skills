# 调用匹配 SOP

> 用途：各 writer skill 按三维（层×赛道×元爽点）系统化查询 pop-trope-library。
> 替代当前"读目录.md→找需求"的非 SOP 逻辑。
> 执行者：各 writer skill 消费素材时。

---

## 匹配模型

```
设定层素材：查 PRD 卖点 → 匹配书目 → 读对应 L1 文件
套路层素材：三维查询（层×赛道×元爽点）→ 读套路库/剧情库
```

---

## Part A：设定层匹配（先查PRD再读L1）

> 设定库按书分目录，每本书有完整的 L1 六件套。不能用一条 tag 找到——先看 PRD 卖点找对书，再读 L1 文件。

### Step A1：查 PRD 卖点匹配书目

1. 读 `设定库/00-索引.md` — 三本书一行摘要
2. 根据你的书需要什么，查对应的 PRD.md 卖点摘要
3. 用"匹配逻辑"表定位——"需要名器分级→让魔门PRD卖点1→读L1-02+L1-06"

| 你的书需要 | 看哪本书的 PRD |
|:----------|:-------------|
| 东方武道力量体系 / 名器分级 / 修仙经济 / 阶层压迫 | 让魔门再次伟大 |
| 硬规则西幻 / 法术复杂度量化 / 封神体系 / 随机奖励 | 深渊主宰 |
| 修仙金融化 / 人体商品化 / 阶级物理化 / 赛博修仙 | 没钱修什么仙 |

### Step A2：读 L1 文件

PRD 的每个卖点都标注了对应 L1 文件。按标注路径读取。

### Step A3：根据本书 PRD 改写转化

L1 文件是原书的设定——不是直接复制。根据本书 PRD 的加工哲学改写。

## Part B：套路层匹配（三维查询）

| skill | 所属层 | 读取模块 | 读取目标 |
|:------|:-------|:---------|:---------|
| creative | 全层（主查元爽点） | `套路库/00-总索引.md` + `references/元爽点-变体映射表.md` | 元爽点匹配，确定本书主元爽点 |
| reservoir | 框架+质感 | `设定库/`（先看PRD卖点→读L1文件） | 跨域设定素材源 |
| world | 框架+质感 | `设定库/`（先看PRD卖点→读L1文件） | 设定创意池 |
| plot | 机制层 | `套路库/` + `剧情库/` + `references/元爽点-变体映射表.md` | 套路链配套 + 剧情改建参考 |
| chapter | 机制层 | `套路库/{具体套路名}.md` | 事件链设计的套路公式+节奏控制 |
| prose | 调味层 | `文风DNA/{书名}.md` | 风格渲染场景卡 |
| qa | 全层 | `套路库/{具体套路名}.md` 的"使用红线"段 | L3 原文对照 |

---

## Step 2：确定赛道维度（筛选同赛道）

1. 从 `创意种子/PRD.md` 的"类型锚点"提取赛道（修仙/西幻/都市/科幻/武侠/...）
2. 在目标模块内按赛道标签（frontmatter `track` 字段）筛选
3. 匹配优先级：
   - 同赛道素材 > 通用素材 > 跨赛道素材（需标注"跨赛道化用"）
4. 无同赛道素材时：降级到通用素材，标注"本赛道库缺，用通用素材"

---

## Step 3：确定元爽点维度（筛选变体，适用于机制层 skill）

1. 从 `创意种子/爽点引擎.md` 取本书 2-3 个主元爽点
2. 从 plot 幕锚点 `act-YY.md` 取本卷/本章爽点目标
3. 查 `references/元爽点-变体映射表.md` 匹配变体家族
4. 在 Step 2 筛选结果中按元爽点标签（frontmatter `meta_joy` 字段）进一步筛选
5. 无元爽点匹配时：标注"本元爽点库缺变体"，回退到通用套路

> 7个元爽点：信息差 / 阶级革命 / 绝境反杀 / 身份跃迁 / 世界观震撼 / 智斗博弈 / 情感羁绊

---

## Step 4：输出匹配清单

```markdown
# pop-trope-library 查询结果 · {skill} · {日期}

## 查询条件
- 层维度：{框架/质感/机制/调味}
- 赛道维度：{修仙/西幻/...}
- 元爽点维度：{信息差/.../无}

## 匹配结果
| # | 素材名 | 模块 | 层 | 赛道 | 元爽点 | 匹配度 | 推荐理由 |
|:-:|:-------|:-----|:---|:-----|:-------|:-------|:---------|

## 库缺声明
- {维度}方向无匹配素材，已降级到{通用/跨赛道}
```

---

## 调用红线

| # | 红线 |
|:-:|:-----|
| 1 | 跳过层维度直接读目录 — 层维度决定读哪个模块，跳过=乱读 |
| 2 | 赛道不筛选就取素材 — 不同赛道的设定/套路不能直接套用 |
| 3 | 元爽点不匹配就定套路链 — 套路变体绑定元爽点，不匹配=爽点偏离 |
| 4 | 库缺时不声明 — 库缺是正常情况，不声明=下游不知道素材可靠性 |
