# 调用匹配 SOP

> 用途：writer skill 查询 pop-trope-library 时使用。
> 核心变化：剧情素材优先按“写作任务”检索，再按书名读取原始 L2/L3。

---

## 总流程

```text
1. 判定当前写作任务
2. 读 任务索引/00-索引.md
3. 进入匹配任务索引
4. 精读 3-5 个推荐案例
5. 按 案例消化卡 输出消化
6. 补读套路库/设定库/金手指库/文风库
7. 声明库缺、跨赛道化用、不可照搬项
```

不要跳过任务索引直接按书名翻剧情库。按书名翻卡只适合库存盘点，不适合写作检索。

---

## Part 0：任务判定

先把用户需求或当前 pipeline 环节翻译成写作任务。

| 当前需求 | 优先任务域 |
|:---------|:-----------|
| 开书、前 5 章、主角苏醒、系统第一次出现 | 开局 |
| 第一次打怪、升级、拿技能、拿钱 | 成长 |
| 被追杀、反杀、暗杀、BOSS 战、帮派危机 | 冲突 |
| 世界观震撼、制度黑暗、力量体系解释 | 设定 |
| 兄妹、战宠、同乡、追随者、羁绊 | 关系 |
| 建据点、招人、经营、声望扩散 | 势力 |
| 阶段胜利、新地图、债务回收 | 收束 |

输出查询条件：

```markdown
## Library 查询条件
- 写作任务：
- 赛道：
- 元爽点：
- 当前单元阶段：
- 需要补的模块：剧情/套路/设定/金手指/文风
```

---

## Part A：剧情素材匹配

### Step A1：查任务索引

1. 读 `任务索引/00-索引.md`。
2. 找到最接近的任务索引。
3. 若无匹配，声明：

```markdown
- 任务索引库缺：{缺的任务}
- 回退策略：从 剧情库/00-索引.md 按书名与 L2 标题筛选候选
```

### Step A2：精读案例

每个任务读取 3-5 个案例：

- P0 案例必须读。
- P1 案例按赛道/元爽点/任务相似度选择。
- L2 优先；L3 只在需要长线咬合、伏笔回收、跨单元节奏时读取。

精读时优先读：

1. 一句话单元
2. 结构分析表
3. 嵌套子线/子线咬合
4. 可迁移要点
5. 不可迁移部分
6. 单元边界/收益/新压力

### Step A3：输出案例消化

按 `references/案例消化卡-标准格式.md` 输出。最低限度包含：

- 可迁移结构
- 爽点证据
- 内容肌理
- 不可照搬项
- 本项目改写决策

---

## Part B：套路素材匹配

套路库回答“怎么写这个爽点/模式”。

1. 读 `套路库/00-总索引.md`。
2. 根据任务索引的配套素材或当前元爽点选择 1-3 张套路卡。
3. 精读“为什么爽”“公式”“红线”“案例索引”。

不要用套路库替代剧情库。套路给公式，剧情卡给完整行动弧和内容肌理。

---

## Part C：设定素材匹配

设定库回答“世界怎么运转”。

1. 读 `设定库/00-索引.md`。
2. 按赛道/卖点找到书级设定包。
3. 先读该书 `PRD.md`，再按 PRD 指向读取 L1 文件。
4. 把原书规则改写成本项目规则，不直接复制。

常见匹配：

| 需要 | 优先参考 |
|:-----|:---------|
| 硬规则西幻 / 封神 / 数值化法术 | 深渊主宰-拆解试水 |
| 修仙金融化 / 人体商品化 / 阶级压迫 | 没钱修什么仙 |
| 东方武道 / 名器分级 / 阶层压迫 | 让魔门再次伟大 |
| 现实都市暗面 / 系统错位 / 美式底层 | 我在美国搞内战-拆解试水 |

---

## Part D：金手指素材匹配

金手指库回答“设计什么外挂”，不是“怎么写外挂展开”。

1. 读 `金手指库/00-索引.md`。
2. 按类型、赛道、元爽点筛选设计卡。
3. 精读核心机制、代价设计、成长路径、赛道适配。
4. 需要叙事展开时，再读 `套路库/金手指体系与外挂进化.md`。

---

## Part E：文风素材匹配

文风库回答“正文怎么呈现”。

1. 读 `文风库/00-索引.md`。
2. 若指定书名，读 `文风库/{书名}.md`。
3. 按 scene 匹配场景卡。
4. 文风不得改写剧情骨架，只服务表达。

若文风档案缺失，必须声明，不要假装有风格锚。

---

## 标准输出

```markdown
## Library 查询与消化
- 写作任务：
- 读取任务索引：
- 精读案例：
- 补读素材：
- 案例消化摘要：
- 库缺声明：
- 跨赛道化用：
- 不可照搬项：
```

## 红线

| # | 红线 |
|:-:|:-----|
| 1 | 不走任务索引，直接按书名随便读剧情卡 |
| 2 | 只总结结构，不提爽点证据 |
| 3 | 只说“可迁移”，不说“不可照搬” |
| 4 | 用设定库替代剧情库，导致只有世界观没有行动弧 |
| 5 | 用套路库替代剧情库，导致只有公式没有内容肌理 |
| 6 | 用金手指 HOW 层套路替代 WHAT 层设计 |
| 7 | 使用 `_LEGACY/` 作为消费入口 |
| 8 | 库缺不声明，导致下游误以为素材可靠 |
