# 套路：系统漏洞利用（权限反杀）

## 元信息

- **套路名称**: 系统漏洞利用（权限反杀）
- **首次出现章节**: ch006
- **所属母套路**: 智取破局类套路
- **复用性**: 中高——需要「系统本身预设但使用者不知道的特性」作为发挥空间

## 触发条件

主角身处某个体系中（组织/功法/等级制度）且处于底层 → 该体系存在一个「为其他目的设计的模块」→ 主角发现这个模块在当下局面可被自己用来反制上层 → 利用上下级信息差完成权限反杀

## 套路公式

```
主角处于系统内底层/工具位 → 
了解系统某个「原本不是为了反制而设计」的功能 → 
在对方无法感知/无法阻止的情况下使用该功能 → 
反向控制原本的控制者
```

## 核心张力

这不是一般的知识碾压——主角使用的功能**本来就是系统设计的一部分**，但控制者（上层）不知道它存在，或者不知道它可被底层反向使用。关键在于：**这是系统默认给的权限，不是BUG**。

## 关键证据

- ch006: 力士体系的监控者身份——「力士需要知道的记忆之一，就是如何监控自己的持有者，防止他们修炼邪功。」
- ch006: 「仙门的「力士体系」之所以高级，除了辅助修炼之外，还有一个监控者的身份。」
- ch006: 真灵傀儡术的设定——「一种极为诡异歹毒的法术，可以让施法者以此掌控被施术者的身心，只需一念就可以让被施术者魂飞魄散。」
- ch006: 该法术的另一用途——「力士的记忆里记载着这门法术，就是为了让持有者不敢随意将力士拆解研究的底气。」
- ch006: 王平的认知反超——「王平之所以会知道这些，是因为力士需要知道的记忆之一，就是如何监控自己的持有者，防止他们修炼邪功。」

## 情绪收益

智取的快感（比对手聪明）→ 系统深层逻辑被揭开的认知爽感 → 底层逆袭的身份快感（工具反杀使用者）→ 「原来这系统还有这一层」的世界观拓展

## 变体设计

| 变体方向 | 示例 |
|---------|------|
| 奴隶的项圈有自毁程序→反向威胁主人 | 控制器的逆向运用 |
| 监视系统有「监视者也会被记录」的日志功能 | 监控的反监控 |
| 下级密码本中有上级的加密算法 | 信息系统的逆向工程 |

| **元爽点家族** | 绝境反杀 |
| **适用章位置** | 高潮 |

## 复用提示

- 此套路成功的核心在于：**读者在看第一遍时想不到这个方案，回头看又觉得理所应当**——这个功能是早就埋好的
- 「监控者」设定是极佳的反转伏笔——需要在前期提前预设（力士体系有监控功能），但不要过早透露具体用法
- 使用此套路时，要确保对方「无法阻止」的逻辑硬不可破——本第中尤道姑「没办法停下」是关键
