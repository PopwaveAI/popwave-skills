## 三、各模块主消费方式

| 模块 | 主查询键 | 消费方式 | 库缺时行为 |
|:-----|:---------|:---------|:----------|
| 立项库 | 书型 / 元爽点组合 / 题材焊接 | creative 读 00-索引 → 精读匹配条目 | 库空→走原有 creative 流程 |
| 设定库 | 书名 / 卖点 / 赛道 | 先读 00-索引 + PRD → 再读 L1-01~06 | 声明"本赛道库缺" |
| 文风库 | 书名 → scene | prose 全量加载 → 按 scene 匹配场景卡 | 声明"文风档案库缺" |
| 剧情库 | 标签 / track / meta_joy | plot 按内容标签目录查 → 读标签 00-索引.md 的 L3 速查表 → 需要深度参考时读 L3 独立文件；L2 卡直接读原始格式 | 声明库缺；不得用卷幕聚合页替代 |
| 套路库 | 模式名 / layer / track / meta_joy | creative 读总索引；chapter 精确读卡；qa 读红线段 | 无 frontmatter 时允许精确读取 |

## 四、兼容层说明

### 文风路径
```
文风库/{书名}.md              ← 唯一路径（新写入目标）
```

### 设定库历史路径
`设定库/框架/` `设定库/质感/` 中如有遗留文件，**标记为 legacy**，不删除但不在新规范中引用。框架/质感现在是书内维度，不是目录模型。

### 剧情库历史切片
`剧情库/_LEGACY/` 中的卷/幕/章级切片仅作为拆书证据保留，**不作为消费入口**。plot 必须读 L2 单元卡（`剧情库/{标签}/L2/*.md`）和 L3 剧情线（`剧情库/{标签}/L3/*.md`）。
