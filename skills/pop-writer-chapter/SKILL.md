# pop-writer-chapter · 章节设计包 v2.9.0

> **定位：消费 plot 的L2单元卡（含驱动决策列）+ 角色卡的剧情驱动接口，产出决策驱动的事件链设计包。每个事件不仅回答"发生了什么"，更回答"谁的决策导致了这件事"。**
> **核心约束：不碰文风。不写叙事者声音、不写句子节奏、不写修辞风格。**

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| 1 | **上游未就绪** — `剧情设计/幕/vol-XX/L2-{编号}.md` 或 `剧情设计/剧情线/` 缺失 → 终止 |
| 2 | **state.yaml 不存在也未初始化** — 终止（CH1 由 Step 1 初始化） |
| 3 | **出场角色不可追溯** — 事件引用角色不在剧情线登场人物中 → 退回 |
| 4 | **事件链无驱动角色** — 每个事件必须标注"驱动角色"和"角色决策"。事件不是自然发生的——是某个人的决策导致的。缺失 = 退回 |
| 5 | **本章关键决策未填写** — 模板已有此字段，必须从角色卡的决策转折点预判映射。留空 = 退回 |
| 6 | **无查询声明** — Step 2 必须查 trope-library 场景技法或 WebSearch。未查 = 退回 |
| 7 | **事件链驱动决策无溯源** — 每个驱动决策必须标注来自角色卡的哪个转折点 + L2 的哪个阶段。缺失 = 决策无源头。退回 |
| 8 | **本章关键决策无代价/不可逆后果** — 从角色卡/L2 映射代价和不可逆后果。缺失 = 决策无分量。退回 |

## 核心流程

### 步骤1：建立基线
**读什么：** `剧情设计/幕/vol-XX/L2-{编号}.md`（全文）、`状态/state-log.yaml`、`剧情设计/剧情线/{活跃线}.md`
**做什么：** 建立结构化基线——本章位置/Canvas约束/角色现状/活跃线上下文/篇幅预算。基线在内存，不落盘。
**❌ 门禁：** L2单元卡/state-log.yaml/剧情线 缺一终止。CH1 时初始化 state-log baseline #0（从角色卡补充角色状态到 baseline）。

详细指令见 `steps/step-1-read-canvas.md`。

### 步骤2：事件链骨架
**预加载：** 基线 + `正文/ch{上一章}.md` 最后 ~500 字 + `状态/角色/{登场角色}-角色卡.md`
**做什么：** 逐事件设计纯结构骨架——地点/参与角色/scene/POV/关键对白/感官锚点/事件内容/情绪目标/冲突层次/信息释放/估计字数。**不标注爽点，不部署套路。**
**❌ 门禁：** 事件数 < 章字数 ÷ 200 退回；缺 scene/POV/关键对白/感官锚点 退回；章首=高潮退回；连续 3 事件同情绪退回。

详细指令见 `steps/step-2-event-chain.md`。

### 步骤3：润色
**预加载：** 事件链骨架 + `references/payoff-guide.md`
**按需查：** `pop-trope-library/套路库/{套路名}.md`（部署套路时）+ `references/emotional-beats.md`（标注情绪弧时）
**做什么：** ①回扫事件链标注爽点（◆小爽点≥5 + ★中爽点≥1）→ ②查套路库部署套路 → ③补充设计层：情绪弧可视化/爽点机制表/章末钩子→预期回收/调味空间/契诃夫枪表/关键对白语气潜台词/信息释放清单
**❌ 门禁：** ◆小爽点 < 5 ⚠️ 警告；★中爽点 < 1 退回；钩子缺预期回收章退回；调味空间缺退回。

详细指令见 `steps/step-3-polish.md`。

### 步骤4：产出落盘
**读什么：** Step 3 润色后的设计包
**做什么：** ①质量自查（对照基线 Canvas 逐项检查 14 条——爽点/钩子/枪/情绪/密度/调味/对白/信息/决策溯源/决策代价/决策不可逆性/决策一致性）→ 不通过回退 Step 2/3 → ②通过后落盘：写 `章节设计包/chXXX-设计包.md` + 更新 `状态/state.yaml` + 存档 state-history/ch-{NNN}.yaml
**❌ 门禁：** 自查任一条不通过退回；缺落盘退回；state.yaml 未更新退回

详细指令见 `steps/step-4-output.md`。

## ❌ WRONG 示例

| ❌ 错误 | ✅ 正确 |
|:--------|:-------|
| 事件内容写「一场激烈的战斗爆发了」 | 事件内容写「目标转身拔剑——刀从侧面切入他右腕」 |
| 关键对白写「主角嘲讽对方」 | 关键对白写原文「废物。」 |
| 凭记忆写角色 before 状态 | 从 `状态/state-log.yaml` 读最后 baseline + event |
| 枪链更新写到独立 chekhov-tracker.md | 更新到 `状态/state.yaml` hooks 段 |
| 不查套路库直接设计事件 | 先查 `pop-trope-library/套路库/{套路名}.md` 节奏控制段 |
| 事件链阶段部署套路/标注爽点 | 事件链只写骨架，套路和爽点在 Step 3 润色时统一处理 |

## 本阶段边界

| 做什么 | 不做什么 |
|:-------|:---------|
| 事件链骨架设计（地点/角色/事件/情绪） | ❌ 不渲染正文 |
| 爽点标注 + 套路部署 + 润色层 | ❌ 不验证剧情逻辑 |
| 调味空间标注 | ❌ 不修改剧情线文档 |
| 设计包落盘 + 伏笔预判（不写 state-log） | ❌ 不替 prose 做文风决策 |

## 边界条件

| 场景 | 处理 |
|:-----|:-----|
| CH1 state-log.yaml 不存在 | Step 1 初始化：从角色卡补充角色状态到 baseline #0 |
| 登场角色卡缺失 | 终止，不凭记忆编造 |
| Canvas 无 payoff 供给 | Step 3 兜底制造中爽点（微释放/对话揭示/内部抉择） |
| 连续 ≥3 章无释放 | 通知 plot 建议增加 S 线 |
| 跨章弧线 | 标注 arc_span + 本章位置 + 情绪分工 |
