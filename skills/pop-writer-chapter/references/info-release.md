# 信息释放落地参考（Step 2 消费）

> 所属: pop-writer-chapter v2.2 · references/
> 说明: 此文件不独立执行。Agent 在 Step 2 逐事件设计时参考。

## 每设计一个事件时检查

### 1. 本章的信息释放清单

从幕纲 Canvas 的「设定线」段提取本章需交付的设定信息项。

### 2. 每个信息点三要素

| 问题 | 约束 |
|:-----|:-----|
| 在哪个事件释放？ | 标注在事件的信息释放字段 |
| 怎么释放？ | 实战展示 / 角色对话 / 叙事者说明 / 探索发现 |
| 在哪一句/哪个场景？ | 一句话描述释放场景 |

### 3. 信息密度检查

- 一个事件最多 2 个新信息点
- 章首事件不强制放信息
- 高复杂度设定必须分到不同事件逐步释放
- 读完所有事件后汇总为「信息释放落地清单」写入骨架

### 4. 按需读 L1 设定原文

按设定线指向的路径读取。读完即用，不预读全部。

## 红线

- ❌ P0 信息点未分配到任何事件 → 退回
- ❌ 一个事件 ≥ 3 个信息点 → 拆分
