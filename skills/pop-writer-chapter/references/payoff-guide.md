## 一、四级爽点承接总表

| 级别 | 谁设计的 | 你拿到什么 | 你的职责 |
|:-----|:--------|:----------|:--------|
| **小** | **你自己** | 什么都没有——你自己设计+承接 | **自设计 ≥5/章，逐事件标记 ◆小爽点** |
| **中** | **plot 定位置** | Canvas payoff_map 告诉你哪条线在"释放" | 按线号设计具体事件，标记 ★中爽点 |
| 大 | plot 定位置 | Canvas 中某线=大，指定章号 | 按章号设计全幕最高密度事件链 |
| 特大 | plot 定位置 | Canvas 中多线=大汇聚，指定章号 | 按章号设计全幕最大规模事件链 |

## 二、小爽点：自设计自承接

> 你是小爽点的唯一设计者和执行者。plot 不参与、不规划。

### 2.1 定义

> 小爽点 = 一次干脆利落、杀伐果决的斩杀/制服/打脸，或一次可感知的升级/变强/赚钱/地位跃迁，或一个即用即走的社会热点梗植入。主角不废话、不犹豫、不解释。读者读完 ≈ 3 秒，反应 = "嗯，爽。"

### 2.2 三种类型 + 案例

#### A. 斩杀型（一个动作结束）

| # | 书房 | 赛道 | 案例 |
|:-:|:-----|:-----|:----|
| 1 | 深渊主宰·ch4 | DND | 索伦暗巷背刺两恶棍: 一刀后心→捂住嘴→尸体擦干→钱袋摸走。没说一句废话 |
| 2 | 吞噬星空·ch30 | 科幻 | 铁毛野猪悍马冲撞→侧闪上撩→猪头抛飞→无头尸身冲出十几米 |
| 3 | 龙符·ch216 | 朝堂 | 玉权劝降话没说完→玉寒露抬手刀芒→刷→元帅头颅飞起 |
| 4 | 铸星者·v2ch5 | DND | 萧伯纳龙形落地→龙爪碾碎六臂蛇魔→全员通过后转身一脚踩死 |
| 5 | 海贼法典·ch351 | 同人 | 莫利亚不屑→威廉一爪刺穿心脏→五道气柱透体而出 |

#### B. 打脸/压制型（碾压性羞辱）

| # | 书房 | 赛道 | 案例 |
|:-:|:-----|:-----|:----|
| 6 | 圣王·ch43 | 仙侠 | 杨鸿烈"紫龙升天"→杨奇一耳光甩过去→所有紫气全破→"废物" |
| 7 | 吞噬星空·ch14 | 科幻 | 三保镖围攻→一脚踹飞+手刀劈骨折+竖劈跪坐→三招三人全废→锁喉悬空 |
| 8 | 深渊主宰·ch58 | DND | 年轻人露出怀疑→拔刀一挥→木头切成四块→闭嘴了。一个字没解释 |

#### C. 升级/觉醒型（可感知的质变）

| # | 书房 | 赛道 | 案例 |
|:-:|:-----|:-----|:----|
| 9 | 深渊主宰·ch51 | DND | 体力不支→当场升级→体质20超凡→伤口发痒体能回满→秒杀全场 |
| 10 | 吞噬星空·ch19 | 科幻 | 觉醒精神念力→钢刺贯穿假山上百次→假山爆碎→警报响起→缩脖子溜走 |

### 2.3 在事件链中设计

```
Step 2 逐事件设计时：

事件 N:
  内容    ← 「角色做了什么 → 什么结果」。一句话
  情绪目标
  冲突层次
  小爽点? ← 这个事件是否是一次干脆的斩杀/升级/打脸/梗植入？
            是 → 标记为 ◆小爽点
  信息释放
  字数

全部事件写完后自检：
- [ ] ◆小爽点 ≥ 5 个？
- 不足 → 追加：过渡场景中插入一个干净利落的斩杀或处境逆转
      或 → 把一个过渡事件升级为"即时有结果"的小爽点事件
```

## 三、中爽点：承接 plot 的 Canvas 布局

### 3.1 你的职责（不是设计位置，是执行事件）

> plot 在 Canvas 中决定了：哪条线、在哪一章释放（payoff_level=中）。
> 你拿到的是：act-XX.yaml 中的 `canvas.payoff_map[本章]`。
> **你的职责：把这条线设计为一个具体的、让读者认知变化的事件。**

### 3.2 承接流程

```
Step 1 读入 act-XX.yaml 的 canvas.payoff_map[本章]：

total_payoff ≥ 1 → Canvas 已供给中爽点位置
  lines[] 中标注了释放的线号和级别（如 "主线1:中", "主线3:小"）
  → 去 volume-XX.md §四 找到该线定义 → 为它设计一个"读者读完认知发生变化"的事件
  → 在事件链的对应事件上标记为 ★中爽点事件

total_payoff = 0 → 本章 plot 没安排任何线释放
  ⚠️ 这是 plot 层面的失误。但本章仍需 ≥ 1 中爽点。
  → 进入 §4 兜底流程
```

### 3.3 线号与事件设计对应

```
Canvas 标记 "主线1:中"：
  → 在 volume-XX.md §四 找到 主线1 线定义
  → 设计事件：让 主线1 线在本章有一次"信息揭示/关系转折/障碍突破/情绪落地"
  → 该事件标记 ★中爽点

Canvas 标记 "主线2:中"：
  → 主线2 是主角成长线
  → 设计事件：主角完成一次可感知的成长（非秒升级——至少有1章铺垫感）
  → 该事件标记 ★中爽点

示例：
  Canvas.entries[ch7]:
    主线1: "进入裂缝——看到了源头"
    主线1_payoff: "中"           ← plot 说 主线1 线在这一章释放

    你在 Step 2 事件链中：
    事件8: 进入裂缝→看到的不是洞口深处是一只眼睛→那眼睛在眨
           中爽点标记: ★中爽点(信息揭示)
```

## 四、Canvas 无供给时的兜底流程

> 有些章 plot 没安排任何线释放（total_payoff=0）。
> 但本章仍需 ≥ 1 中爽点。你必须自己造。

### 4.1 策略 B（首选）：已有主线/支线的微释放

```
逐线扫 Canvas.entries[本章]：
- 主线1 做了什么？→ "观察裂缝变化" → 升级为 "发现裂缝只对左侧有反应" → 信息揭示
- 主线3 做了什么？→ "和队友商量" → 升级为 "队友说漏了一个秘密" → 信息揭示
- 支线1 做了什么？→ "和妹妹吃饭" → 升级为 "妹妹问了一个不该她知道的问题" → 关系转折
```

### 4.2 策略 C（兜底）：独立中爽点事件

- 对话揭示——两个角色在过渡中"顺带说破"前面埋了多章的信息
- 内部抉择——主角在独处时做了一个决定（心理推进，非打斗）

### 4.3 策略 A（上游反馈）：连续 ≥3 章无释放 → 建议 plot 增加 S 线

> 策略 B 或 C 可单章解决；如果连续 ≥3 章都在兜底，说明 Canvas 设计有问题。
> 通知 plot："vol-XX act-YY 连续 3 章无线释放，建议增加一条 S 线或调整现有线节奏。"

## 五、大爽点 & 特大爽点：按位置执行

| 级别 | plot 交付 | 你执行 |
|:-----|:---------|:-------|
| 大爽点 | Canvas 某线=大，指定章号 | 该章按**全幕最高密度**设计事件链。以该线为中心，围绕它设计高潮事件+铺垫收束 |
| 特大爽点 | Canvas ≥2 线=大汇聚，指定章号 | 该章按**全幕最大规模**设计事件链。每条=大的线各分配至少一个高潮事件，多线交织 |

## 六、责任归属

| chapter-design 负责 | chapter-design 不负责 |
|:---------------------|:----------------------|
| ◆小爽点自设计自承接（≥5/章） | 中爽点位置规划（归 plot Canvas） |
| ★中爽点承接：按 Canvas 线号设计事件 | 大爽点位置规划（归 plot Canvas） |
| Canvas 无供给时兜底制造中爽点 | 特大爽点位置规划（归 plot Canvas） |
| 连续无释放 → 建议 plot 增加 S 线 | 文本脉冲（归 prose-render） |
