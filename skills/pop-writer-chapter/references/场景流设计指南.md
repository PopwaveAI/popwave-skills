# 场景流设计指南

> 所属: pop-writer-chapter v2.0.0 · references/
> 消费: Step 2 场景流设计时参考
> 说明: 此文件不独立执行。Agent 在 Step 2 逐场景设计时参考。

## 一、场景定义

- 场景 = 一个连续的时空单元
- 场景边界 = 时空转换 或 POV切换
- 每章 3-5 个场景
- 场景内事件不独立填表，而是自然流动描述

## 二、事件流（导演指令式，非逐事件填表）

v2 核心差异：场景内 3-5 事件以**自然流动描述**写成，不切割成独立条目。

**❌ WRONG（v1逐事件填表）：**
```
事件1: 目标拔剑
事件2: 侧切右腕
事件3: 尸体倒地
```

**✅ CORRECT（v2导演指令式）：**
```
目标转身拔剑——刀从侧面切入他右腕，尸体擦干后钱袋摸走，全程没说一句废话。
```

导演指令 = 一两句连贯话讲清这场戏的动作流和结果，prose 拿到后自行展开。

## 三、7字段/场景规范

| # | 字段 | 导演指令写法 |
|:-:|:-----|:------------|
| 1 | 场景指令 | 一句话指令："索伦暗巷猎杀两恶棍" |
| 2 | POV | 角色名或留空 |
| 3 | 参与角色 | 列角色名 |
| 4 | 事件流 | 连贯描述3-5事件，不切割 |
| 5 | 感官锚点 | "雨声启动" / "铁锈味启动" |
| 6 | 情绪走向 | "紧张→释放" |
| 7 | 估计字数 | 300-800字 |

## 四、场景间过渡

- 场景之间靠因果或时空跳转连接
- 不需要写过渡指令——prose 自行处理场景间衔接
- 但章末场景必须是钩子（悬念/信息/情绪）

## 五、篇幅控制

- 主设计包 ≤ 8K字
- 导演指令式天然控制篇幅——比逐事件填表省 40-60% 字数
- 超出时压缩事件流描述，保留指令核心

## 六、红线

- ❌ 场景数 < 3 或 > 5 → 退回
- ❌ 事件流写成逐事件填表格式 → 退回（违反导演指令式）
- ❌ 章首场景情绪 = 高潮级 → 退回
- ❌ 连续 2 场景同一情绪走向 → 退回
