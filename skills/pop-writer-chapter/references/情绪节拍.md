# 情绪节拍（v2）

> 所属: pop-writer-chapter v2.0.0 · references/
> 消费: Step 2 场景设计 + Step 3 情绪弧线标注时参考
> 适配: v1 emotional-beats，v2 以场景为情绪单位

## 一、情绪词汇表

每场景标注一个情绪目标。以下词汇表供参考：

| 情绪类别 | 可选词汇 |
|:---------|:---------|
| 正面 | 安心、释放、期待、温暖、激动、震撼、胜利 |
| 负面 | 不安、焦虑、紧张、恐惧、愤怒、悲伤、压抑 |
| 混合 | 好奇+不安、释放+期待、紧张+好奇、震撼+敬畏 |

## 二、场景级情绪走向

v2 以场景为情绪单位（非 v1 的事件级）：

- 每场景一个情绪走向：起点 → 终点
- 场景内事件的情绪在走向范围内流动
- 连续 2 个场景不能同一情绪走向

## 三、全章情绪弧线检查（Step 3 执行）

全部场景设计完成后检查：

### 1. 情绪起点
从上一章末尾或 state-log.yaml 取主角当前心理状态。起点 = 上一章的终点。

### 2. 定高潮场景
- 本章最高情绪强度在哪个场景？
- 是否为爽点（payoff）场景？

### 3. 定终点
- 章末情绪 = 钩子类型决定（悬念→好奇/不安；情绪→释放/温暖/震撼）
- 终点必须能衔接到下一章起点

### 4. 弧线检查清单
```
□ 情绪逐步递进（不是来回跳极）？
□ 高潮场景 = 最高强度？
□ 终点 = 下一章起点？
□ 无连续 3 个场景同一情绪？
```

## 四、红线

- ❌ 章首场景情绪 = 高潮级（紧张/震撼） → 没有铺垫，退回
- ❌ 连续 2 个场景同一情绪走向 → 退回
- ❌ 全章无高潮场景定位 → 退回

## 五、与 v1 的差异

| 维度 | v1（事件级） | v2（场景级） |
|:-----|:------------|:------------|
| 情绪标注单位 | 每事件 | 每场景 |
| 连续重复检查 | 连续3事件同情绪 | 连续2场景同情绪走向 |
| 弧线检查 | 事件序列 | 场景序列 |
