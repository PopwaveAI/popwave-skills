# Step 2：事件链设计

> 管线: pop-writer-chapter v2.2
> 模板: `templates/fact-skeleton.md`
> 参考: `references/character-scheduling.md` `references/location-orchestration.md` `references/emotional-beats.md` `references/info-release.md` `references/payoff-guide.md`

## 目的

将本章拆解为事件序列。每个事件设计时同步确定：在哪、谁、发生什么、读者什么感觉、要不要放设定。

## 一、什么是 Design 事件

> **一个事件 = 一个回合 = 一个独立的因果单位。**

```
❌ "他在据点里暗杀了二楼三个人"        ← 三个回合合并
❌ "推开门。门轴微响。他走进去。"       ← Render 微节拍
❌ "一场激烈的战斗爆发了"              ← 说了等于没说

✅ "目标转身拔剑——刀从侧面切入他右腕"   ← 一个回合：交锋+结果
```

| 场景 | 一个事件 = | 示例 |
|:-----|:----------|:-----|
| 战斗 | 一个回合/一次交锋 | 出击→结果 / 躲避→反击 |
| 对话 | 一次姿态变化或信息揭示 | 试探→对方沉默 / 摊牌→情绪崩了 |
| 探索 | 一次发现或一次判断 | 看到鳞片→判断非天然 |
| 过渡 | 一次情绪转变或新目标 | 看到妹妹的伤→决定带她去治伤 |

## 二、事件数量

```
硬性下限: Design 事件数 ≥ 章预期字数 ÷ 200
多线补充: 双线+4 / 三线+6 / 四线+8
```

| 篇幅 | 下限 | 常见 |
|:-----|:----:|:----:|
| 2000 字过渡章 | ≥10 | 10-14 |
| 2500 字常规/对话/探索章 | ≥12 | 12-18 |
| 2500 字战斗章（单线） | ≥12 | 16-24 |
| 3000 字战斗章（多线） | ≥15 | 20-30 |

## 三、执行流程

### 3.1 判断跨章弧线

检查 act-YY.md 相邻章的标题和 emotional_goal：
- 标题连续（"决战①""②""③"）→ 跨章
- 情绪递进（紧张→临界→释放）→ 跨章
- 字数预算 > 3500 且剧情 > 一章量 → 跨章

跨章则对整个弧线设计事件链 → 按章切分 → 每章找情绪落点。标注 `arc_span / 本章位置 / 情绪分工`。

### 3.2 读角色卡

对每个登场角色：
- 读 `状态/角色/{角色名}-角色卡.md` → core_desire、人格基线
- 读 `状态/entity-snapshot.yaml` → 当前等级/位置/情绪
- 角色卡不存在 → 退回

### 3.3 逐事件设计

每个事件同步确定所有字段：

```
事件N:
  地点      ← 从 Canvas 线摘要 + 卷纲地点池 → 写在事件里
  参与角色  ← 从 entity-snapshot 取 before 状态
  scene     ← 映射DNA场景卡的scene值（combat_stealth/dialogue_daily/...）
  POV       ← 感知锚点角色名。跨角色章必须显式标注
  关键对白  ← 不可替换的原文。多条用 | 分隔
  感官锚点  ← 具体感觉（"雨声启动""铁锈味启动"），不写抽象描述
  事件内容  ← 「角色做了什么 → 什么结果」。一个句子
  情绪目标  ← 一个词（参考 emotional-beats.md 词汇表）
  冲突层次  ← external / internal / interpersonal
  ◆小爽点   ← 干脆的斩杀/升级/打脸/梗植入？是 → ◆小爽点。整章 ≥ 5
  ★中爽点   ← 读者读完认知有变化？是 → ★中爽点。整章 ≥ 1
  plot_line ← 推进哪条线
  信息释放  ← 触发设定线交付时标注
  枪链引用  ← 对应剧情线枪链的节点号
  估计字数  ← 100-250 字
```

### 3.4 全局检查

| 维度 | 检查 | 不通过 |
|:-----|:-----|:------|
| 情绪弧线 | 逐步递进？章首非高潮？无连续3事件同一情绪？ | 退回 |
| 冲突层次 | 战斗/对话/危机章大多数标注了冲突层次？ | 警告 |
| 信息释放 | P0 全部分配？无事件≥3条？ | 退回 |
| 空间 | 连续事件间地理可达？ | 退回 |
| 因果 | 每个事件到下一个有"所以"或"然后"？ | 退回 |
| 钩子 | 章末事件是钩子？ | 退回 |
| 爽点 | ◆小爽点 ≥ 5，★中爽点 ≥ 1 | 退回 |

## 四、战斗章事件链示例（据点暗杀，~2500 字）

```
事件1:  暗巷观察据点 — 数清门口两人/二楼窗口一人 — 警觉 — 120
事件2:  绕到后巷 — 木箱堆叠，可攀上二楼 — 判断 — 100
事件3:  翻窗上二楼 — 窗台湿滑，左手打滑→抓稳 — 紧张 — 120
事件4:  落地无声 — 确认没惊动任何人 — 警觉 — 80
事件5:  听到鼾声 — 第一个目标在睡，门虚掩 — 判断 — 100
事件6:  反手握匕首 — 推门→门轴轻响→鼾声停了 — 紧张上升 — 120
事件7:  不等对方清醒 — 左手捂嘴→右手匕首刺入喉咙 — 冷酷 — 140
事件8:  确认死亡 — 血沫从嘴角溢出，没挣扎 — 冷酷+确认 — 80
事件9:  跨过尸体 — 走廊地板在脚下轻响 — 过渡 — 60
事件10: 第二个房间有光 — 一个醒着，趴在桌上写信 — 观察 — 100
事件11: 背对门口 — 油灯拉长影子 — 利用环境 — 80
事件12: 刀从侧面切入 — 油灯撞倒，房间黑暗 — 果断 — 120
事件13: 黑暗中不动五秒 — 确认楼下无动静 — 警觉 — 80
事件14: 第三个房间空 — 床铺没叠，窗户开着 — 疑问 — 100
事件15: 走廊尽头烟味 — 楼下大厅有人说话 — 新信息 — 100
事件16: 压低下楼 — 大厅火光，两人围炉喝酒 — 观察 — 120
事件17: 从背后解决第一个 — 刀从后颈切入 — 果断 — 120
事件18: 第二个正面交锋 — 三个回合→割断喉咙 — 激烈 — 160
事件19: 整栋楼静了 — 雨声和远处狗叫 — 喘息 — 100
```

19 事件 × ~110 字 = ~2100 字骨架。Render 在每个事件空间内自主拆 3-5 个微节拍。

## 红线

- ❌ 事件数 < 章字数 ÷ 200 → 退回
- ❌ 战斗事件写成模糊概括 → 退回
- ❌ Design 事件写成 Render 微节拍 → 退回
- ❌ 事件链因果断裂 → 退回
- ❌ 跨章弧线未标注 → 退回
