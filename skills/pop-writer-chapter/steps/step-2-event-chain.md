# Step 2：正式设计

> 管线: pop-writer-chapter v2.2
> 消费: Step 1 基线
> 产出: `章节设计包/chXXX-设计包.md`
> 模板: `templates/fact-skeleton.md`

## 目的

从基线出发，完成章节的全部设计工作：事件链 + 情绪弧 + 爽点机制 + 章末钩子 + 契诃夫枪 + 调味空间 + 角色 after 状态。不是只设计事件链——事件链是骨架，需要补充全部设计层。

## 执行顺序

### Phase A：预加载（一次性，必须读完再开始设计）

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `正文/ch{上一章}.md` 最后 ~500 字 | 叙事衔接——上一章的情绪落脚点、笔触、未闭合的张力 |
| 2 | `references/payoff-guide.md` | ◆小爽点自设计承接 + ★中爽点承接规则 |
| 3 | `状态/角色/{角色名}-角色卡.md` | 本章登场角色的 core_desire + 人格基线（仅 baseline §4 标注「登场」的角色） |

> 其余在设计中按需查：`pop-trope-library/套路库/{套路名}.md`（用到套路时）、`references/emotional-beats.md`（标注情绪时）、`references/character-scheduling.md`（设计角色事件时）、`references/location-orchestration.md`（设计地点时）、`references/info-release.md`（信息释放时）。

### Phase B：事件链设计

从基线 §6 的事件数范围出发，从本章开幕事件到章末钩子，逐个事件设计。

**每个事件必填字段：**

| 字段 | 要求 | 来源 |
|:-----|:-----|:-----|
| 事件序号 | N | — |
| 标题 | 简短概括 | 自行命名 |
| 地点 | 可追溯到剧情线文档的地点暗示 | 基线 + 卷纲地点描述 |
| 参与角色 | 必须在 baseline §4 本章登场角色中 | 基线 §4 |
| scene | DNA 场景卡映射值（combat_stealth / dialogue_daily 等） | 按事件类型推断 |
| POV | 感知锚点角色名。留空 = 默认主角 | 跨角色章必填 |
| 关键对白/数据 | 不可替换的原文。填空 = Render 自行发挥 | 剧情线文档 + 事件内容推断 |
| 感官锚点 | 启动事件的感官提示（"雨声启动"/"铁锈味启动"等） | 按场景推断 |
| 事件内容 | 「角色做了什么 → 什么结果」。一句话 | 从基线 Canvas 约束推导 |
| 情绪目标 | 一个词（警觉/紧张/冷酷/决断/...） | 参考 `references/emotional-beats.md` |
| 冲突层次 | external / internal / interpersonal（可选 1-3 层） | 按事件判断 |
| ◆小爽点? | 是否是一次干脆的斩杀/升级/打脸/梗植入？ | 按 `payoff-guide.md §二` 判断 |
| ★中爽点 | 读者读完认知是否有变化？注明类型（信息揭示/关系转折/障碍突破/情绪落地） | 按 `payoff-guide.md §三` 承接 |
| 所属线 | 该事件推进哪条剧情线 | 基线 §2 Canvas 活跃线 |
| 契诃夫枪引用 | 设伏/回收的枪点编号 | 基线 §5 枪链 |
| 信息释放 | 如触发设定交付，标注信息项 | 基线 Canvas 约束 |
| 估计字数 | 100-250 字 | 自行估算 |

**设计规则：**
- 事件链有因果——每个事件到下一个有「所以」或「然后」
- 章首事件情绪 ≠ 高潮级（紧张/震撼）——没有铺垫，退回
- 章末事件必须是钩子
- 连续 3 个事件不能同一情绪
- ◆小爽点 ≥ 5 个
- ★中爽点 ≥ 1 个

### Phase C：设计层补充

全部事件链设计完成后，补充以下设计层：

**C1. 情绪弧线可视化** — ASCII 图展示起点→推进→高潮→终点

**C2. 爽点机制表** — 每个 ◆小爽点 / ★中爽点标注触发事件、机制说明、强度

**C3. 章末钩子 → 预期回收** — 钩子类型+内容+强度+预期回收章

**C4. 调味空间** — 从本章情绪强度和套路使用红线综合判断：

| 维度 | 本章设定 | 依据 |
|:-----|:---------|:-----|
| 幽默程度 | 无/微量/适度/为主 | 本章情绪强度 + 套路节奏控制建议 |
| 叙事距离 | 贴近主角/旁观者/全知 | 套路节奏控制建议 |
| 情绪缓冲 | 是否需要/用什么方式 | 本章情绪弧线判断 |
| 调味禁忌 | 本章绝对不能出现的元素 | 套路库/{套路名}.md 使用红线 + 本章情绪类型 |

**C5. 关键对白语气/潜台词** — 不可替换对白标注说话人/语气/潜台词

**C6. 登场人物 before/after** — 从基线 §4 取 before，从事件链推导 after

**C7. 契诃夫枪表** — 本章设伏/回收的枪（供 Step 3 同步到 act-YY.md）

**C8. 信息释放落地清单** — 本章交付的设定信息项

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 事件数 < 章字数 ÷ 200 | ❌ 退回补事件 |
| ◆小爽点 < 5 | ⚠️ 警告（过渡章可放宽） |
| ★中爽点 < 1 | ❌ 退回补中爽点 |
| scene/POV/关键对白/感官锚点 任缺一 | ❌ 退回 |
| 情绪弧线缺/爽点表缺/钩子缺回收章/调味空间缺/枪表缺 | ❌ 退回 |
| 章末不是钩子 | ❌ 退回 |
| 事件链因果断裂 | ❌ 退回 |
| 章首情绪 = 高潮级 | ❌ 退回 |
| 连续 3 事件同一情绪 | ❌ 退回 |

## 产出

完整设计包内容（传给 Step 3 落盘）：
- 基线 §1-2 的基础信息
- Phase B 的完整事件链
- Phase C 的全部设计层补充
