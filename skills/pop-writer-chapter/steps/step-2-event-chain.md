## 场景流设计

以场景为组织单位，每场景包含3-5个有机流动的事件。场景边界 = 时空转换或POV切换。

### 场景定义
- 场景 = 一个连续的时空单元
- 每章 3-5 个场景
- 场景内事件不独立填表，而是自然流动描述

### 每场景填写

| 字段 | 要求 | 来源 |
|:-----|:-----|:-----|
| 场景标题 | 简短概括 | 自行命名 |
| 场景目的 | 这个场景为什么存在？推进什么？ | 从基线 Canvas 约束推导 |
| 参与角色 | 必须在基线 §4 本章登场角色中 | 基线 §4 |
| POV | 感知锚点角色名。留空 = 默认主角 | 跨角色章必填 |
| 关键对白 | 不可替换的原文。填空 = Render 自行发挥 | 剧情线 + 事件内容推断 |
| 感官锚点 | 启动场景的感官提示（"雨声启动"/"铁锈味启动"等） | 按场景推断 |
| 场景内事件流 | 3-5个事件的自然流动（不切割成独立条目） | 从基线 Canvas 约束推导 |
| 情绪走向 | 起点情绪 → 终点情绪 | 参考 emotional-beats 词汇表 |
| 估计字数 | 300-800 字 | 自行估算 |

> ★ 以下字段 **Step 2 不填**，留给 Step 3 润色时补充：
> - ◆小爽点 / ★中爽点
> - 契诃夫枪引用
> - 本章套路

**设计规则：**
- 场景内有因果——事件到事件有「所以」或「然后」
- 章首场景情绪 ≠ 高潮级（紧张/震撼）——没有铺垫，退回
- 章末场景必须是钩子（悬念/信息/情绪）
- 连续 2 个场景不能同一情绪走向
- **每章至少推进 3 条不同的 L2**（即至少 3 个场景的"场景目的"涉及不同 L2）
- **每个场景必须有"驱动角色"** — 场景内事件是某个人的决策导致的。在决策溯源附录中标注
- **"本章关键决策"从角色卡映射** — 登场人物表的"本章关键决策"字段从角色卡的决策转折点预判映射，不是凭空想

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 场景数 < 3 | ❌ 退回补场景 |
| **推进的 L2 不足 3 条不同线** | ❌ 退回——至少 3 个场景涉及不同 L2 |
| POV/关键对白/感官锚点 任缺一 | ❌ 退回 |
| 场景内因果断裂 | ❌ 退回 |
| 章首情绪 = 高潮级 | ❌ 退回 |
| 连续 2 场景同一情绪走向 | ❌ 退回 |
| 章末不是钩子 | ❌ 退回 |
| 本章关键决策未填 | ❌ 退回——从角色卡驱动接口映射 |
| Step 2 无查询声明 | ❌ 退回补查 |
