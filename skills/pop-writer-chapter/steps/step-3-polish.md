# Step 3：润色

> 管线: pop-writer-chapter v2.0.0
> 消费: Step 2 场景流骨架
> 产出: 润色后的完整设计包（爽点标注 + 情绪弧线 + 章末钩子）

## 目的

在场景流骨架上叠加 3 个子表：爽点标注、情绪弧线、章末钩子。**v2 只做 3 个子表，非 v1 的 8 个。**

## 预加载

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `references/爽点指南.md` | ◆小爽点/★中爽点设计规范 |
| 2 | `references/情绪节拍.md` | 情绪词汇表 + 弧线检查规则 |
| 3 | `pop-trope-library/套路库/{套路名}.md`（按需） | 场景技法节奏控制 |

## 做什么

### 子表 1：爽点标注

回扫场景流骨架，在事件流中标注爽点：

| 级别 | 要求 | 来源 |
|:-----|:-----|:-----|
| ◆小爽点 | ≥5/章，自设计自承接 | references/爽点指南.md §二 |
| ★中爽点 | ≥1/章，承接 plot Canvas 布局 | references/爽点指南.md §三 |

> Canvas 无中爽点供给时，走兜底流程（微释放/对话揭示/内部抉择），见 references/爽点指南.md §四。

### 子表 2：情绪弧线

按 references/情绪节拍.md 标注全章情绪走向：

1. **起点**：从上一章末尾或 state-log 取主角心理状态
2. **高潮**：定位本章最高情绪强度事件（= 爽点事件）
3. **终点**：章末情绪 = 钩子类型决定（悬念→好奇/不安；情绪→释放/震撼）
4. **弧线检查**：
   - [ ] 情绪逐步递进（不来回跳极）
   - [ ] 高潮事件 = 最高强度
   - [ ] 终点能衔接到下一章起点
   - [ ] 无连续 3 个场景同一情绪

### 子表 3：章末钩子

设计章末钩子，标注钩子类型和预期回收章：

| 钩子类型 | 适用场景 | 要求 |
|:---------|:---------|:-----|
| 悬念钩 | 信息揭示章 | 标注预期回收章号 |
| 情绪钩 | 情感高潮章 | 标注情绪余韵方向 |
| 行动钩 | 战斗/冲突章 | 标注下一章行动起点 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| ◆小爽点 < 5 | ⚠️ 警告，追加 |
| ★中爽点 < 1 | ❌ 退回，执行兜底流程 |
| 钩子缺预期回收章 | ❌ 退回 |
| 情绪弧线无高潮定位 | ❌ 退回 |
| 连续 3 场景同一情绪 | ❌ 退回 |

## 产出

润色后的完整设计包（场景流骨架 + 3 个子表），写入**内存**。Step 4 落盘并控制篇幅。
