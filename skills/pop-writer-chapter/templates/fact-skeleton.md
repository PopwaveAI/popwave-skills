# chXXX-设计包模板（事实骨架 + 登场人物卡 合并）

> 产出路径: `章节设计包/chXXX-设计包.md`
> 生产方式: pop-writer-chapter v2.2
> 下游消费者: pop-writer-prose

## 关键原则

- 事件只描述**内容**（谁、在哪、做什么），不描述**方式**（怎么写的、什么语气）
- 每个事件 100-250 字渲染量
- 章预期字数 ÷ 200 ≈ 基准事件数
- 角色 before 状态必须从 `状态/entity-snapshot.yaml` 取——不许凭记忆
- scene/POV/关键对白/感官锚点 四个字段是 prose 的强制输入
- 关键对白只填不可替换的原文
- 感官锚点只填具体感觉（"铁锈味启动"），不写抽象描述

## 填写自检

### 事件数量
- [ ] 事件数 ≥ 章预期字数 ÷ 200
- [ ] 每个事件 = 一个回合/一个因果单位

### 事件质量
- [ ] 每个事件标注了情绪目标
- [ ] 每个事件标注了 scene
- [ ] 每个事件标注了 POV
- [ ] 关键对白只填不可替换的原文
- [ ] 感官锚点是具体感觉描述
- [ ] 事件内容是「谁做了什么 → 什么结果」
- [ ] 事件内容不是 Render 微节拍
- [ ] 事件链因果不断裂
- [ ] 章末事件是钩子事件

### 角色状态
- [ ] 主角 before 状态与 entity-snapshot 一致
- [ ] 所有出场角色在剧情线登场人物中定义
- [ ] 每个配角有「本章台词风格」

### Canvas 对齐
- [ ] ◆小爽点 ≥ 5
- [ ] ★中爽点 ≥ 1
- [ ] 情绪节拍终点能衔接下一章起点
- [ ] 跨章弧线 arc_span/本章位置/情绪分工 三个字段必填
