# pop-writer-character · 角色系统设计 v2.0.1

> **定位：定义角色卡 Lv1~Lv4 分级标准、维度递增体系和字段规范。**
---

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **不跳过选级就填角色卡** — 必须先确认角色级别再加载对应模板 |
| ❌2 | **不跨级混用维度** — Lv2 不偷 Lv4 的「能力」维；Lv3 不漏 Lv2 的「标签」维 |
| ❌3 | **不对齐就输出** — 每 Lv 维度 = 上一 Lv 全部维度 + 本 Lv 新增维度。少一项即为不合格 |
| ❌4 | **设计目的和能力不可省略** — Lv1~Lv4 每级角色卡必须包含「设计目的」和「叙事能力」声明 |
| ❌5 | **产出只留摘要** — 写入角色卡后对话中不粘贴完整卡片。说"已写入 {路径}。摘要：{角色名} Lv{N}"。 |

## 核心流程

### 步骤 1：判断角色叙事权重 → 选 Lv
详见 `steps/step-1-select-level.md`

详见 `steps/step-2-write-card.md`

详见 `steps/step-3-verify.md`

## ❌ 错误示例

### WRONG 1：维度倒灌
```
Lv2 功能角色卡写了「能力」维度（战斗 S 级）
```
❌ 错误：「能力」是 Lv4 独占维度，Lv2 最多写到「驱动·功能」。✅ 正确：Lv2 角色卡只写到标签 + 驱动·功能。
### WRONG 2：选级膨胀
```
只出现半章的酒馆老板 → 选 Lv3 → 写弧线/叙事功能/心理驱力
```
❌ 错误：酒馆老板只做一件事（递消息），Lv1 足够。✅ 正确：先问"出现几幕？有没有独立弧线？" → Lv1 → 只写一句话摘要。
### WRONG 3：跨级混用字段
```
Lv1 路人角色卡里写了"驱力：想要/恐惧/背叛条件"
```
❌ 错误：「驱力」是 Lv3 才出现的升级维度，Lv1 只有一句话摘要。✅ 正确：Lv1 只写一句话摘要 + [可选] ★ 复用例。
### WRONG 4：Lv3 遗漏了 Lv2 的「标签」维
```
Lv3 配角卡：身份+驱力+叙事功能+弧线，无标签
```
❌ 错误：Lv3 = Lv2 全部维度 + 新增维度，标签是 Lv2 新增维，Lv3 必须继承。✅ 正确：逐项对照维度矩阵，确保 Lv3 包含全部 Lv2 维度 + 本级新增。
### WRONG 5：跳过选级直接开写
```
看到配角 → 直接打开 Lv4 模板 → 从心理到能力写满 16 维
```
❌ 错误：未做选级判断就加载模板。✅ 正确：先判断叙事权重 → 选 Lv → 加载对应 Lv 模板 → 逐维填写。

## 跨赛道复用说明

模板使用**通用写作术语**，不绑定特定题材：
- 如需题材特定字段（如网文的"境界/修为"、西幻的"职业/神性"、科幻的"植入体/改造"），消费方 skill 在对应 Lv 的扩展占位处按需附加
- **不改模板本身** — 模板只保留跨赛道通用字段

## 异常与边界条件

| # | 场景 | 触发条件 | 处理动作 |
|:-:|:-----|:---------|:---------|
| 1 | 级别选不定 | 角色介于两个 Lv 之间 | 先选低一级填写，末尾标注 `※ 待升级至 LvN` |
| 2 | 模板文件缺失 | `references/LvN-*.md` 不存在 | 按维度矩阵手写，**不编造字段** |
| 3 | 消费方跳过本 skill | pop-writer-creative/pop-writer-plot 未加载模板 | 不强制中断——本 skill 是推荐性标准，非阻塞 |
| 4 | 题材扩展冲突 | 消费方追加字段与已有维度重叠 | 追加字段放对应 Lv 末尾的 `【题材扩展】` 区块 |
| 5 | 角色设计中途合并 | 两个角色合并为一人 | 取较高 Lv 的模板填写，合并两者驱动 |
| 6 | 临时角色意外复出 | Lv1 角色在后续章节再次出现 | 从 Lv1 升级到 Lv2（补标签+驱动·功能），不重写摘要 |
| 7 | 批量创建角色 | 一次需要创建 5+ 角色卡 | 逐角色按叙事权重判断级别，分别加载对应模板 |
| 8 | 消费方不读取本 schema | 下游 skill 直接硬编码角色字段 | 本 skill 不阻塞管线，但产出标注"未使用 pop-writer-character" |

## 落盘检查点

| # | 检查项 | 预期产出 | 通过条件 |
|:-:|:-------|:---------|:---------|
| 1 | 角色卡文件 | `设计/角色层/{角色名}-{定位}.md` | 文件存在且格式与对应 Lv 模板一致 |
| 2 | 级别标注 | 角色卡顶部标注了 Lv 级别 | 与叙事权重匹配，维度数符合该级别标准 |
| 3 | 维度完整性 | 所有必填维度已填写 | 符合"上一级全部 + 本级新增"原则 |
| 4 | 设计目的 & 叙事能力 | 包含这两项声明 | 所有级别都必须有，且内容不为空 |
| 5 | 无跨级混用 | 未出现高一级的独占维度 | 逐项对照维度矩阵校验 |
