# pop-writer-character · 角色系统设计 v3.2.0

> **定位：从 PRD、世界设定、研究档案和 trope-library，产出带"剧情驱动接口"的分级角色卡。角色卡不只是档案——它是驱动整个故事的活人的设计文档。**
> **核心约束：先选级再写卡；不跨级混用维度；不改模板通用字段。**

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **跳过选级直接写卡** — 未先完成 Lv 判断就加载模板 → 终止，先执行 `steps/step-1-select-level.md` |
| ❌2 | **跨级混用维度** — 低级角色卡写入高级独占维度，或高级角色卡漏掉低级继承维度 → 退回重写 |
| ❌3 | **模板引用缺失时编造字段** — `references/LvN-*.md` 缺失却私自扩写字段 → 停止扩写，仅按现有维度矩阵手写 |
| ❌4 | **Lv4 角色卡未读创意溯源记录** — creative 的研究档案/创意溯源记录.md 包含素材原型和拟用转化手法，是主角/核心配角身份张力和说话风格的设计参考。不读 = 角色卡与创意源头脱节 |
| ❌5 | **角色卡含占位符** — 角色卡中不允许出现"（同前）"/"同上"/"内容同前"等占位符。检测到 = 退回补全。验证时执行 `Select-String -Path "状态\角色\*.md" -Pattern "（同前）|同上|内容同前"` |
| ❌6 | **无查询声明** — 未查 trope-library（设定库/角色与关系 + 剧情库 + 套路库）就写角色卡 = 在信息真空里造人。退回补查+产出查询声明 |
| ❌7 | **Lv4 角色卡无剧情驱动接口** — Lv4 角色卡必须包含"剧情驱动接口"段。缺失 = 角色卡是档案不是驱动力。退回补写 |
| ❌8 | **Lv4 角色卡无角色压力测试** — 必须把角色放进 PRD 世界宪法矛盾中推演决策。缺失 = 角色与剧情割裂。退回补测 |
| ❌9 | **决策转折点无代价或不可逆后果** — 每个转折点必须标注代价和不可逆后果。缺失 = 决策无分量。退回补写 |
| ❌10 | **三个转折点全为"最可能决策"** — 至少1个必须是"最戏剧决策"（出乎意料但合乎逻辑）。全为最可能 = 角色无惊喜。退回重新推演 |
| ❌11 | **Lv4主角卡无金手指结构化字段** — Lv4 主角卡必须包含"五b、金手指"段（9字段结构化表+压力场景）。缺失 = 金手指信息散落在自由文本中，下游 plot/chapter 无法消费 |

## 核心流程

### 步骤 1：判断角色叙事权重
**读什么：** 当前创作上下文 + `references/level-selection-methodology.md` + `pop-trope-library/设定库/角色与关系/00-索引.md` + `pop-trope-library/剧情库/00-索引.md`
**做什么：** 依据角色出场频次、叙事功能、是否有独立弧线，判断应使用的 Lv。同时查 trope-library 获取角色设计模式参考
**产出：** 级别判定结果：Lv1 / Lv2 / Lv3 / Lv4 + 查询声明
**❌ 门禁：** 角色权重仍摇摆不定 → 先按低一级处理，并标注 `※ 待升级至 LvN`；无查询声明→退回(❌6)

详细指令见 `steps/step-1-select-level.md`。

### 步骤 2：加载对应模板并填写角色卡
**读什么：** 对应 `references/LvN-*.md` 模板 + `references/level-selection-methodology.md` + `pop-trope-library/套路库/角色关系转折.md` + `pop-trope-library/套路库/新角色友好介绍.md`
> **读取确认**：Step 2 完成后，agent 必须在角色卡的"设计目的"中引用创意溯源记录的具体素材原型。未引用 = 未读取。
**做什么：** 按"上一级全部维度 + 本级新增维度"逐项填写角色卡，并补齐可直接驱动对白的说法风格。Lv4 角色额外完成角色压力测试 + 剧情驱动接口
**产出：** `设计/角色层/{角色名}-{定位}.md` + 查询声明 + 角色压力测试（Lv4 必产）+ 剧情驱动接口（Lv4 必产）
**❌ 门禁：** 缺必填维度、缺"设计目的"/"叙事能力"/"说法风格"任一项 → 退回补齐；无查询声明→退回(❌6)；Lv4 无剧情驱动接口→退回(❌7)；Lv4 无压力测试→退回(❌8)

详细指令见 `steps/step-2-write-card.md`。

### 步骤 3：校验并决定是否升降级
**读什么：** 已写角色卡 + `references/level-selection-methodology.md`
**做什么：** 逐项检查维度完整性、跨级污染、升级/降级必要性
**产出：** 校验通过的角色卡，或带升级标注的角色卡
**❌ 门禁：** 发现高级维度倒灌或低级维度缺失 → 退回步骤 2 重写

详细指令见 `steps/step-3-verify.md`。

## ❌ WRONG 示例

> ❌ 看到“配角”就直接打开 `references/Lv4-core.md` 把主角级整张卡写满
> ✅ 先判断这个角色出现几幕、是否有独立弧线；不满足则降到 Lv2 或 Lv3

> ❌ Lv2 功能角色卡写了“能力：战斗 S 级”
> ✅ Lv2 只写到“标签 + 驱动·功能”，能力维度属于 Lv4 独占

> ❌ Lv3 配角卡写了身份、驱力、弧线，却漏掉 Lv2 继承的“标签”
> ✅ 对照维度矩阵逐项核查，确保 Lv3 = Lv2 全部维度 + Lv3 新增维度

> ❌ 模板里没有“境界/神性/植入体”，就直接改 `references/LvN-*.md`
> ✅ 保持模板通用字段不动，把题材字段附加到角色卡 `【题材扩展】` 区块

## 边界条件

| 场景 | 触发条件 | 处理 |
|:-----|:---------|:-----|
| 前置信息不足 | 只有名字，没有定位/戏份/功能描述 | 先补最小角色定位，再执行选级 |
| 模板文件缺失 | `references/LvN-*.md` 不存在 | 按维度矩阵手写，不编造新字段 |
| 格式不匹配 | 角色卡结构与对应模板不一致 | 对齐模板标题与字段顺序后再落盘 |
| 循环依赖 | 角色级别依赖剧情线，剧情线又等待角色卡 | 先按当前已知信息定低一级，并标 `※ 待升级` |
| 用户中途改需求 | 用户要求把临时角色升为长期角色 | 升级到更高 Lv，补新增维度，不重写已成立信息 |
| 子 skill 不可用 | 上游消费方未读取本 skill | 本 skill 不阻塞，但产出注明“未使用角色分级标准” |
| 批量建卡 | 一次创建 5 张以上角色卡 | 逐角色独立选级，不允许整批套同一模板 |
| 角色合并 | 两名角色合并为一名 | 取较高 Lv 模板，合并驱动与弧线信息 |
