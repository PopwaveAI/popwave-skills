# Step 1：角色级别选择 + 生态位定位

> 管线: pop-writer-character v2.0.0
> 消费: pop-writer-world 产物（世界宪法 + 社会结构 + 生态图谱）
> 产出: 角色级别判定 + 生态位定位结果 + 查询声明

## 读什么

- `小说世界设定/世界宪法.md`（核心矛盾 + 力量体系 + 社会规则）
- `小说世界设定/社会结构.md`（阶层 + 势力 + 组织）
- `小说世界设定/生态图谱.md`（生态位置矩阵 + 势力关系网 + 资源流向）
- `全书立项PRD.md`（角色锚点 + 赛道定位）
- `references/角色卡规范.md`（v1 完整字段 + v2 生态位字段说明）

## 做什么

### 一、判断角色叙事权重（复用 v1 Lv1-Lv4 分级）

| 该角色 | 用级别 | 判断要点 |
|:-------|:------|:---------|
| 只出现 1 个场景，路人/背景板/一句话概括 | **Lv1** | 一句摘要 + 一句代表台词 |
| 为特定叙事功能服务，1-3 幕，无独立弧线 | **Lv2** | 稳定说 2-4 句对白，承担固定功能 |
| 贯穿 2+ 幕且有独立动机，有自己的弧线 | **Lv3** | 读者能从对白里认出这个人 |
| 故事主角 / 主要反派 / 贯穿全文的灵魂 | **Lv4** | 维护长期稳定的角色声线与独白风格 |

### 二、生态位定位（v2 新增）

角色不只是"存在于世界"，而是占据生态图谱中的特定位置。生态位置决定了角色可触达的资源、面临的结构性压力、与其他角色的天然关系。

1. **从生态图谱中选位**：每个角色必须锚定到生态图谱的一个位置节点
   - 该位置隶属哪个势力/阶层？
   - 该位置掌控什么资源？受制于什么规则？
   - 该位置与哪些其他位置有天然关系（合作/竞争/依附/对抗）？
2. **生态位置影响级别选择**：
   - 占据生态枢纽位置（多线交汇点）的角色 → 倾向升级处理
   - 占据边缘位置（单线触发）的角色 → 可降级处理
   - 生态图谱中标注"关键冲突点"的位置 → 至少 Lv3

### 三、主动查询（外部信息注入）

1. 读 `pop-trope-library/设定库/角色与关系/00-索引.md` → 角色身份张力、关系结构模式
2. 读 `pop-trope-library/剧情库/00-索引.md` → 同赛道 L3 卡"核心驱动力"段
3. 库缺时 WebSearch 补充（查询词：`"{赛道} {角色类型} 生态定位 角色设计"`）
4. 产出查询声明块，绑定红线

### 四、产出级别判定 + 生态位定位

对每个角色记录：

```
角色：{名称}
级别：Lv{N}（依据：{叙事权重判断}）
生态位置：{生态图谱节点名} — {隶属势力/阶层}
生态位依据：{为什么选这个位置}
※ 待升级至 LvN（如选不定，先选低一级）
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 生态图谱文件缺失 | 阻塞，退回 world skill 补产出 |
| 角色未锚定到生态图谱节点 | 退回补锚定（红线基础） |
| 无查询声明 | 退回补查（绑定红线） |
| 级别摇摆不定 | 先选低一级，标注 `※ 待升级` |

## 产出

- 角色级别判定清单（每角色：名称 + Lv + 生态位置 + 锚定依据）
- 查询声明块
