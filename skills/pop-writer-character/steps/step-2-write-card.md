# Step 2：角色卡撰写（v1 完整字段 + v2 生态位三字段）

> 管线: pop-writer-character v2.0.0
> 消费: Step 1 级别判定 + 生态图谱 + 世界宪法
> 产出: `设计/角色层/{角色名}-{定位}.md`（含生态位字段的完整角色卡）

## 读什么

- Step 1 产出的角色级别判定 + 生态位定位结果
- `小说世界设定/世界宪法.md`（核心矛盾，驱动角色压力测试）
- `小说世界设定/生态图谱.md`（生态关系网 + 冲突点来源）
- `references/角色卡规范.md`（v1 完整字段 + v2 生态位字段说明）
- `references/角色弧线指南.md`（决策转折点设计规范）
- `templates/角色卡-模板.md`（角色卡模板）

## 做什么

### 一、加载模板并按级别填写 v1 字段

严格遵循"上一级全部 + 本级新增"维度递增原则：

| 级别 | 维度范围 |
|:----|:---------|
| Lv1 | 摘要 + 代表性台词 |
| Lv2 | Lv1 + 标签 + 驱动·功能 + 说法风格（2-4 句） |
| Lv3 | Lv2 + 身份 + 驱力四层 + 弧线 + 关系网 |
| Lv4 | Lv3 + 核心缺陷 + 决策逻辑 + 剧情驱动接口 + 压力测试 |

所有级别必须包含：**设计目的** + **叙事能力** + **说法风格**。

### 二、填写 v2 新增生态位三字段（所有级别必填）

> v2 核心差异：角色不只是"存在于世界"，而是嵌入生态图谱。三字段将角色锚定到生态结构中。

#### 1. 生态位置

```
生态位置：{生态图谱节点名}
所属势力/阶层：{势力名 / 阶层名}
位置性质：{枢纽 / 中继 / 边缘 / 冲突前沿}
资源掌控：{该位置掌控什么资源 / 受制于什么规则}
```

#### 2. 生态关系

```
生态关系网：
| 关系对象 | 关系类型 | 关系性质 | 剧情接口 |
|:--------|:--------|:--------|:--------|
| {角色/势力名} | 合作/竞争/依附/对抗/共生 | {具体关系内容} | {驱动什么剧情线/副线} |
```

> 每个角色至少 2 条生态关系（红线 ❌3 基础）。

#### 3. 生态冲突点

```
生态冲突点：{角色在生态结构中面临的结构性矛盾}
冲突来源：{资源争夺 / 规则压制 / 阶层流动 / 势力博弈}
冲突升级路径：{这个冲突如何从结构性矛盾演变为剧情冲突}
```

> 生态冲突点是角色弧线和决策转折点的生态驱动源。Lv4 主角的生态冲突点必须能驱动主线。

### 三、Lv4 角色额外完成（复用 v1）

- **角色压力测试**：把角色放进世界宪法核心矛盾中推演决策
- **剧情驱动接口**：核心驱动力映射 + 驱动的剧情线清单 + 决策转折点预判（≥3）+ 角色化学反应

### 四、生态冲突驱动决策转折点（v2 强化）

Lv4 主角的决策转折点必须能追溯到生态冲突点：

| 转折点 | 生态冲突来源 | 角色决策 | 代价 | 不可逆后果 |
|:---|:---|:---|:---|:---|
| 第一个不可逆点（约1/4处） | {生态冲突} | {决策} | {代价} | {后果} |
| 核心认知颠覆（约1/2处） | {生态冲突} | {决策} | {代价} | {后果} |
| 最黑暗时刻（约3/4处） | {生态冲突} | {决策} | {代价} | {后果} |

> 3 个转折点中至少 1 个为"最戏剧决策"（出乎意料但合乎逻辑）。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 角色卡缺生态位字段（生态位置/生态关系/生态冲突点） | ❌ 退回补写（红线 ❌1） |
| Lv4 角色缺角色压力测试 | 退回补测 |
| Lv4 角色缺剧情驱动接口 | 退回补写 |
| 角色卡含占位符（"同上"/"待定"/"？"） | 退回补全 |
| 生态关系 < 2 条 | 退回补写 |
| Lv4 主角决策转折点 < 3 个 | ❌ 退回补写（红线 ❌2 基础） |
| 说法风格只写空标签（"毒舌/温柔"） | 退回补具体对白锚点 |

- `设计/角色层/{角色名}-{定位}.md` — 含 v1 全部字段 + v2 生态位三字段
- Lv4 角色额外产出：压力测试 + 剧情驱动接口
