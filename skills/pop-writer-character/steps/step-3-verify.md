# 第三步：校验 + 选填升级/降级

## 校验清单

对照 `references/level-selection-methodology.md` 的维度矩阵逐项检查，确保：

- [ ] 所有必填维度已填写
- [ ] 不跨级混用维度（Lv2 不偷 Lv4 的维度）
- [ ] 不对齐就输出（少一项即为不合格）
- [ ] 设计目的、叙事能力已填写（Lv1~Lv4 都必须有）
- [ ] 说法风格已填写，且至少含 1 句可直接入正文的示例台词
- [ ] 模板最小样例仅作参照，未被原样复制
- [ ] 角色级别与对白复杂度匹配（不是路人却写成播报器，也不是路人硬写成主角声线）
- [ ] **（★NEW）查询声明已产出** — 无查询声明 = 退回(❌6)
- [ ] **（★NEW，Lv4）剧情驱动接口已填写** — 核心驱动力映射+驱动剧情线清单+决策转折点预判+决策化学反应。缺失 = 退回(❌7)
- [ ] **（★NEW，Lv4）角色压力测试已产出** — 缺失 = 退回(❌8)
- [ ] **（★NEW）驱力和缺陷字段有具体内容** — 不允许"待定"/"待补充"
- [ ] **（★NEW，Lv4）决策转折点每个有代价和不可逆后果** — 代价必须是角色失去的具体事物；不可逆后果必须是回不去的具体状态。不允许"关系恶化"这种可逆描述。缺失 = 退回(❌9)
- [ ] **（★NEW，Lv4）至少1个转折点是"最戏剧决策"** — 3个全为"最可能"= 角色无惊喜。退回重新推演(❌10)
- [ ] **（★NEW，Lv4）角色特异性不为空** — 每个决策的"角色特异性"必须说明为什么只有这个角色会做。空 = 任何角色都会做这个决策 = 决策无灵魂。退回重新推演

## 选填升级/降级

| 场景 | 动作 |
|:-----|:------|
| 没想清楚 | 先按较低级别写，后续升级补全 |
| 写着写着角色变重要了 | 升级后补全缺失维度（回溯填写，不重写已有信息） |
| 降级 | 几乎不发生 — 已有多维信息的角色降级无意义 |

## 产出要求

产出只留摘要 — 写入角色卡后对话中不粘贴完整卡片。说"已写入 {路径}。摘要：{角色名} Lv{N}，{核心特征}。需展开告诉我。"
