# pop-writer-creative · 创意打磨 v4.3.0

> **定位**：从用户模糊想法到高质量立项 PRD 的**唯一入口**。PRD 产出后交付 world。
>
> v4.3.0：**管线对齐 + 模板瘦身**——(1) PRD 产出文件名统一为 `全书立项PRD.md`，不再放在 `创意种子/` 子目录；(2) reference 从 5 个精简为 2 个（`prd-模板-空白.md` + `prd-模板-魔门.md`），删除与 PRD 推导无关的 pitfalls/layer-architecture/缺口分析；(3) 红线 3+7 修正——PRD 可写核心创意命名，不可写 L1 施工细节；(4) 管线 PRD 同步更新（creative v4.2.0→v4.3.0，文件级接口表格式，四环节分组）。
>
> v4.1.0：**PRD 宪法结构最终定型**——本书剧情简述(~200字梗概) → 篇幅规划 → 爽点锚点(含L1-L4特征参数) → 核心卖点(≥5条，别人家vs本书vs为什么更对) → 美学基因推导链(含质感→力量→情感→材料→命名→视觉锚点场景) → 世界宪法 → 核心承诺+防崩阈值 → DNA+边界 → 加工矩阵(可选) → 跨域素材指引 → 用户画像。PRD 不做具体名/数值/分幕/转译规则——那些归 L1/plot。
>
> v4.0.0：PRD 移除加工矩阵/层架构/材料选择，改为宪法层结构。
>
> v3.6.0：**Phase 2 自问链 + 场景判定 R/M/O + 元模板**——消除「分类代替分析」。拿到元爽点标签后必须向下追问 L1-L4 特征参数；不匹配 7 类的走元模板生成自定义追问链；搜索策略不再基于类型模板，而是基于特征参数精准搜索。
>
> v3.5.0：**职责收窄**——只做 PRD 推导。素材采集/注入/样品试读/Phase Delta 全部移除。

## 路由

```
用户说"开书" / "我有一个想法"
  │
  ▼
Phase R · 路由诊断 → 判断输入粒度 → 决定搜索宽度
  详见 [steps/step-r.md](steps/step-r.md)
  │
  ▼
PRD 推导 (step-prd.md v4) — agent 研究提案模式
  Step 1: 最小化爽点确认（≤3 个问题）
    Phase 1: 用户问答 → 锁定 2-3 个元爽点类别
    Phase 2a: 判断输入场景（R单原作 / M多原作 / O无参考）
    Phase 2b: agent 自问链 → 7类常见+元模板，产出 L1-L4/L5 特征参数 ⭐
    Phase 3: 产出特征参数表 + 搜索目标列表
  Step 2: 联网搜索（关键词从特征参数而非类型模板提取）→ 生成 2-3 个故事概念选项
  Step 3: 用户选择 → 锁定概念 + 推导核心卖点(≥5条) + 用户画像
  Step 4: 从基因推导全书色彩 + 命名体系 + 视觉锚点场景
  Step 5: 推导世界宪法（4-6条不可违反规则，附违反后果）
  Step 6: 跨域素材指引（方向性，非详细方案）
  Step 7: 篇幅规划
  Step 8: 合成 PRD（宪法结构：剧情简述→篇幅→爽点→核心卖点→基因+命名+视觉锚点→宪法→承诺→DNA+边界→矩阵→素材→画像）
  Step 9: 验证 → 写入
  详见 [steps/step-prd.md](steps/step-prd.md)
  │
  ▼
全书立项PRD.md → 交付 world
```

## 红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase R 不可跳过** |
| ❌2 | **PRD 不可跳过** |
| ❌3 | **不预设 L1 级细节**——PRD 可以写核心创意命名（世界名/主角名/金手指名），但不可写属性数值、冷却时间、详细转译规则、分幕划分。写完每条问自己：「这是创意决策还是施工图？」创意决策留 PRD，施工图留 L1/plot。 |
| ❌4 | **不把拆书复盘当开书推演**——PRD 是开书第一天的文档，不是 37 章读完后的 catalog |
| ❌5 | **Phase 2 自问链不可跳过**——拿到元爽点标签不准停，必须向下追问 L1-L4 特征参数。无参数的爽点 = 没打磨的设计 |
| ❌6 | **不强行归类**——如果用户的元爽点不匹配 7 类常见模板，走元模板生成自定义追问链。把独特爽点塞进最接近的类别里 = 信息丢失 |
| ❌7 | **PRD 不承载 L1/plot 施工细节**——不写属性数值、冷却时间、分幕划分、详细转译规则。可以写核心创意命名，但写完每条问自己：「如果这条不写，下游设计师还有发挥空间吗？」如有，抽象层级对了。 |
