## 路由总图

### 模式 1：管线入口（开书）

```
用户说"开书"
  │
  ▼
Phase R · 路由诊断
  │
  ▼
★ Phase 爽点引擎（新）· 和用户共创元爽点
  │  ① 读 pop-trope-library/套路库/00-总索引.md → 获取已入库的元爽点-变体映射
  │  ② 读 pop-trope-library/references/元爽点-变体映射表.md → 确认 7 元爽点 × 变体家族
  │  ③ 通过 Q&A 挖掘本书的 2-3 个主元爽点
  │  产出：创意种子/爽点引擎.md
  │  每个元爽点 = 一套变体家族（套路库可查）
  │
  ├─ 用户有清晰核心想法 → 走 分支 A：PRD先行 → 定向采集 → 故事引擎
  │
  ├─ 用户有碎片化灵感 → 走 分支 B：广度采集 → PRD浮现 → 反审精炼 → 故事引擎
  │
  └─ 用户一片空白 → 走 分支 C：方向碰撞 → 选择深化 → 故事引擎
```

| 分支 | 适用场景 | 路径特征 | PRD 产生时机 |
|:----|:---------|:---------|:------------|
| A | "我要写一本XXX的书" | PRD → 筛选采集 | **素材采集前**（定加工坐标系） |
| B | "我喜欢A/B/C/D，怎么揉" | 采集 → PRD → 反审 | **素材采集后**（从碰撞中浮现） |
| C | "帮我想个故事" | 碰撞 → sketch → 深化 | **方向选定后**（从选择中提炼） |

### 模式 2：独立调起 — 新素材注入（创意转换）

详见 [pop-writer-reservoir v2.0.0](../pop-writer-reservoir/SKILL.md)（已吸收 pop-writer-fuse，后者已删除）。

Phase Delta 是 creative 的触发层，实际注入执行由 pop-writer-reservoir 完成。参见下方「Phase Delta」章节。creative 仅负责最终判定（D1/D2/D3）。

```
用户说"我想把这个XX加进书里"
  │
  ```md
  Phase Delta · creative 触发 → reservoir 执行（创意转换）
    │
    ├─ [creative 触发层]
    │     传递新素材 + 项目上下文（PRD/引擎/宪法/现有池）给 reservoir

    │     返回归档结果（含品类路由+安全等级+匹配度）
    │
    └─ [creative 判定层]
          ├─ ✅ D1: 直接注入 → 确认写入
          ├─ ⚠️ D2: 微调后注入 → 输出 PRD 微调方案（用户签字）

  ```

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase R 不可跳过** — 不诊断就开走 = 不知道自己在走哪条分支 |
| ❌2 | **W0 不可跳过** — 不查元素交叉库就开搜 = 浪费燃料 |
| ❌9 | **PRD 锁定后不退回方向生成** — PRD 是加工坐标系，锁定后下一步是 W0.5/W1 素材采集 → 故事引擎，不是重新生成方向提案。用户说"素材和创意"="我要 W1 采集"，不是"给我方向 A/B/C" |
| ❌3 | **PRD 不可跳过** — 无 PRD 则素材无加工坐标系（C 分支方向选定后补写） |
| ❌4 | **素材储备池不可跳过** — 素材不沉淀则 plot 阶段从零发明剧情 |
| ❌5 | **有对标书不触发 W2 退回** — 有锚点书但不查公共库 = 假对标 |
| ❌6 | **样品试读未通过直传 world** — Phase 0.5 不可跳过。通过后 creative 使命完成 |
| ❌7 | **主角设计四维不完整** — 身份/驱力/缺陷/唯一性任一缺失 → 退回 0.4 |
| ❌8 | **故事引擎读起来像配置表** — 不是故事提案 → 退回 Phase 0 |
| ❌9 | **产出只留摘要** — 写入文件后对话只说「已写入 {路径}。摘要：{核心}。」 |
| ❌10 | **同人二创误判为原创碰撞** — 锚定某部作品做同人二创 → 直接走分支 B 的锚点书子路径，不走方向 sketch 碰撞 |
| ❌11 | **融合后不贡献回按来源查启发库** — 如果融合了新的素材来源，必须写入按来源查启发库 |
| ❌12 | **素材储备池不分类直接堆** — 必须按主线索材/支线索材/背景素材/剩余焊接点四类归档，否则 plot 无法消费 |
| ❌D1 | **Phase Delta 不读项目产出就评估** — 必须找到 PRD.md（或从故事引擎提取）后才执行 |
| ❌D2 | **Phase Delta 不擅自修订宪法** — 即使建议微调，必须输出方案让用户签字 |
| ❌D3 | **Phase Delta 冲突时不沉默吞入** — 新元素明显与 PRD 冲突但未报告 |
| ❌D4 | **Phase Delta 注入不写储备池** — 评估完就结束，不写池 |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 用户说"我有三个喜欢的IP想揉" → Agent 直接出方向 sketch | 没先素材采集就碰撞，方向缺乏素材支撑 | 走分支 B → 先广度采集 → 从素材中浮现 PRD → 再精炼 |
| PRD 已锁定后用户说"素材和创意不够" → Agent 退回生成方向 A/B/C | PRD 锁定后不退回方向生成（红线❌9）。"素材和创意不够"= 需要W1深度采集，不是重新碰撞方向 | 继续走 W0.5 → W1 素材采集 → 素材储备池 → 故事引擎 |
| 用户说"我要写一本硬核历史修仙" → Agent 搜了一堆素材，没先定 PRD | 没 PRD 筛选器，素材平等全收，加工时无坐标系 | 走分支 A → PRD 先行 → 按 PRD 定向采集 |
| 素材采集完只写了故事引擎，没沉淀储备池 | 大量素材浪费，plot 从零编剧情 | 素材储备池四类归档 → plot 直接拉取素材当剧情种子 |
| 素材储备池只是一个巨长的素材清单，没有分类 | plot 无法判断"这个素材能当主线还是支线" | 按主线/支线/背景/剩余焊接点四类分类，标注优先级和潜在触发方式 |
| 用户只说"鹅腿事件" → Agent 直接解吸 | 用户没给素材内容，解吸出来的是臆测 | 先执行采集步骤（搜索/追问），补全素材内容后再解吸 |
| 项目有故事引擎但无 PRD.md → Agent 拒绝执行注入 | 项目 v2.0.0 之前启动，PRD 还没独立 | 从故事引擎提取六维度 PRD 后继续 |

## 收到任务后，按顺序执行

### Phase R：路由诊断（HARD GATE）

详见 [steps/step-r.md](steps/step-r.md)。

> **读什么**：用户初始输入
> **产出什么**：`_路由记录/分支选择.md`

从用户输入判断启动状态，走 A/B/C 中一条。决策依据：
- **分支 A**：用户有清晰的核心命题（"我要写一本 XXX 的书"）
- **分支 B**：用户有多个碎片化灵感/喜欢的 IP 想揉合
- **分支 C**：用户一片空白，"帮我想个故事"

---

### ★ Phase 爽点引擎：和用户共创元爽点（v3.0 NEW）

> **定位**：在 Phase R 之后、分文字幕之前，通过和用户共创确定本书的 2-3 个主元爽点。元爽点是全书爽的"基本粒子"——每种元爽点有自己的一套变体家族（套路库可查），为后续所有剧情选择和套路配套提供锚点。
>
> **为什么在 PRD 之前**：PRD 说"这本书是什么"，元爽点说"这本书的爽怎么生产"。没有元爽点，PRD 的"核心承诺"和"加工哲学"就是空中楼阁。
>
> **调用后续**：reservoir 拉种子时按元爽点筛选；plot 配套路时从对应变体家族选；chapter-design 部署具体变体；qa 检查元爽点多样性。

#### 元爽点库（初始版，随拆书持续扩展）

| 元爽点 | 本质 | 变体举例（套路库可查） |
|:-------|:-----|:---------------------|
| **信息差** | 我知道你不知道→我利用这个赢了 | 先知先觉/截胡/钓鱼博弈/身份反转/灵识透视/双面卧底/黑吃黑/老鼠传信/唇语接头 |
| **阶级革命** | 底层掀翻上位者→共情快感 | 阶级跃升·反向坠落/底层逻辑揭露/杀良冒功反噬/开源革命/过河拆桥报复/黔首非人论 |
| **绝境反杀** | 以为要死了→不但没死还赚了 | 因祸得福/玉石俱焚反转/死后觉醒/自毁禁术/以伤换伤/主角真死转生 |
| **身份跃迁** | 从没人看得起→被万众瞩目 | 身份晋升/名号反转/装备飞升/实力展示收徒/身份揭示/阶级跨越 |
| **世界观震撼** | 认知刷新→世界比想象的大 | 仙凡鸿沟/阶梯揭示/升维对比/黑暗真相/逛街见闻/二十四节炁揭秘 |
| **智斗博弈** | 多势力互相算计→谁棋高一着 | 双钩布局/螳螂捕蝉/心理战/套娃布局/双线叙事/反向诱惑 |
| **情感羁绊** | 关系从对立→信任/从疏远→亲密 | 牺牲守护/误会解除/战友之情/师徒传承/恩义两难/破碎感美人觉醒 |

#### 共创流程（Q&A 模式）

> **核心原则**：这不是表单填写——是和作者的开放式对话。先问 1-2 个问题，根据回答决定下一个问题，最后推导出元爽点星座，与用户确认。

**Step 1 — 开场（解释概念）**

> "写网文最怕什么？怕写了几章读者跑了。那怎么让读者一章一章地追？不是靠大纲写得规范——是靠每一章都给他一个'停下来的理由'。这个理由，我管它叫元爽点。"
>
> "比如信息差——读者跟着主角知道反派不知道的秘密，看着主角一步步布局，最后'原来他早就算好了'，那一瞬间的爽感就是信息差元爽点。比如阶级革命——底层掀翻压迫者，读者代入那种'出了口恶气'的感觉。"
>
> "我先问你几个问题，我们一起找到你这本书最核心的2-3个元爽点。有了这个锚点，后面的素材选择、套路搭配、章节设计都不用猜——直接从对应的变体家族里挑。"

**Step 2 — 核心问题库（每次问 1-2 个，根据回答决定下一个）**

| # | 问题 | 意图 | 倾向的元爽点信号 |
|:-:|:-----|:-----|:----------------|
| 1 | "你想让读者每次读完新章节后，最强烈的感受是什么？比如‘原来如此’的智力快感、‘太牛了’的力量震撼、‘呜呜呜’的情感共鸣？" | 探测元爽点的大方向 | 智力快感→信息差/智斗；力量震撼→绝境反杀/身份跃迁；情感共鸣→情感羁绊/阶级革命 |
| 2 | "你最上头的一本网文是哪本？那一章让你最舒服的是哪个瞬间？" | 通过具体作品定位爽点偏好 | 具体场景可反推元爽点 |
| 3 | "面对困境，你的主角第一反应是用脑子解、用拳头解、还是忍一忍等时机？" | 探测主角行为模式 | 用脑子→信息差/智斗；拳头→绝境反杀；忍耐→阶级革命/情感羁绊 |
| 4 | "读者看完你的书，最想跟朋友聊的是哪个情节——主角算了一大盘棋？还是干翻了大反派？还是那个让他破防的配角？" | 定位全书最高光时刻的元爽点类型 | 算棋→信息差/智斗；干翻→阶级革命/绝境反杀；破防→情感羁绊 |
| 5 | "你最不想写的爽点类型是什么？比如单纯的力量碾压、无脑装逼打脸？" | 排除不适配的元爽点 | 排除相应元爽点的所有变体 |

**Step 3 — 推导元爽点星座 → 用户确认**

从回答中推导 2-3 个主元爽点，说给用户听：

> "从你刚才说的，我提炼了3个元爽点：
> 1. **信息差** — 你说了喜欢主角布局，读者跟着线索一步步发现真相
> 2. **阶级革命** — 你说了想让读者代入底层掀翻制度，不是个人复仇
> 3. **绝境反杀** — 你说主角每次都是被逼到墙角再翻盘
>
> 这三个你觉得对吗？哪个是关键中的关键？要不要加或删？"

**Step 4 — 写入产出**

```markdown
# 爽点引擎 — 《书名暂定》

> 产出于 YYYY-MM-DD | 和用户通过 Q&A 共创

## 主元爽点（2-3 个）

### 元爽点 1：{名称}
- **为什么是这个**：{推导依据，引用用户原话}
- **变体家族**：{对应套路库中的变体列表}
- **全书占比**：{预估占全书爽点的大致比例，如 40%}

### 元爽点 2：{名称}
- **为什么是这个**：{推导依据}
- **变体家族**：{变体列表}
- **全书占比**：{预估比例}

## Phase Delta：项目运行中的新元素注入（★ 独立调起模式）

> **定位**：独立调起入口。当项目已过 creative/world/plot 阶段，作者想把某个新元素（热点事件/现实新闻/IP新增/新灵感）注入书里时，不重跑全流程——直接执行轻量级注入评估。
>
> **触发条件**：用户说"帮我把这个XX融进书里""最近有个XX热点能写进我的书吗""我想到一个新设定"——且当前项目已经有 PRD + 故事引擎 + 素材储备池（即已跑过管线入口流程）。
>
> **注意**：完整的素材储备注入功能已独立为 **[pop-writer-reservoir v2.1.0](../pop-writer-reservoir/SKILL.md)** skill（产出剧情储备卡格式）。Phase Delta 在此处作为 creative 内部的快捷注入入口，输出格式和评估逻辑与 pop-writer-reservoir 一致。重度注入场景（多素材/复杂适配）请直接调起 pop-writer-reservoir。

### 调用形态

```mermaid
flowchart LR
    A[用户说<br>"把这个XX加进书里"] --> B[creative 触发<br>Phase Delta]
    B --> C[调用 reservoir<br>执行9步注入]
    C --> D{creative 判定}
    D -->|D1 兼容| E[确认写入储备池]
    D -->|D2 微调| F[输出PRD微调方案<br>→ 用户确认]
    D -->|D3 冲突| G[输出冲突报告<br>→ 用户决策]
```

Phase Delta 执行前必须验证项目状态：

| 文件存在 | 动作 |
|:---------|:-----|
| `PRD.md` ✅ | 直接使用 |
| 无 PRD.md 但有 `故事引擎.md` | 从故事引擎提取六维度 PRD，存入 `创意种子/PRD.md` 后继续 |
| 无 PRD.md 也无故事引擎 | ❌ 这不是已运行项目 → 退回路管入口模式，走 Phase R 诊断 |

| 文件存在 | 动作 |
|:---------|:-----|
| `素材储备池.md` ✅ | 后续追加 |
| 无素材储备池 | 首版创建 |

**部分缺失标注**：缺失某些可选文件时降级执行——无法检测的维度标注"因文件缺失跳过的检查项"。

### 步骤顺序

详见 [steps/step-delta.md](steps/step-delta.md)。

```
Phase Delta · creative 触发 → reservoir 执行

  1. [creative] 前置检查：确认项目有 PRD.md（或从故事引擎提取）、素材储备池.md
  2. [creative] 调用 pop-writer-reservoir，传递：新素材 + PRD.md + 故事引擎.md + 世界宪法.md + 现有池子
  3. [reservoir 执行层] — 详见 pop-writer-reservoir v2.0.0
     ├─ Step 1~2: 素材采集（不足时搜索）+ 解吸+品类路由（→ 走 1A~1E）
     ├─ Step 3: ★ 受众立场与情绪评估（安全门禁，不可跳过）
     ├─ Step 4~6: PRD 契合度 → 宪法冲突 → 适配方式判定（≥2条路线）
     ├─ Step 7~8: 四类归档 → 写入储备池
     └─ 返回：写入结果 + 品类路由 + 安全等级 + 匹配度 + 适配路线
  4. [creative 判定层] — 收到 reservoir 结果后判定
     ├─ ✅ Phase D1: 直接注入 → 确认并告知用户

  5. [creative] 产出：素材储备池更新（含品类路由+安全等级标记）或 冲突报告

| # | 红线 |
|:-:|:-----|

## 素材储备池详解（★ 新增核心产出）

### 定位

素材储备池是 creative 给 **plot 的软资源池**。plot 在设计卷/幕剧情时，不再从零发明冲突，而是从储备池中拉取素材。

### 素材储备池 vs 故事引擎

| 维度 | 故事引擎（宪法） | 素材储备池（资源池） |
|:-----|:----------------|:-------------------|
| 性质 | **硬约束** | **软资源** |
| 使用方式 | world 必须遵守 | plot 可选拉取 |
| 修改频率 | 锁定后极少修改 | plot 卷间动态消耗/补充 |
| 产出时间 | creative 阶段 | creative 阶段首版，plot 阶段持续补充 |
| 消费者 | world（世界观建构） | plot（剧情设计） |

### 四类种子

| 类别 | 定义 | 示例 | plot 使用场景 |
|:-----|:-----|:-----|:-------------|
| **主线索材** | 可以直接触发主线事件的冲突/设定/机制 | "混沌四神教团作为主线反派势力" | plot Step 1 取用为主角头号对手 |
| **支线索材** | 可以独立成线的冲突/势力/机制/设定 | "深渊底层的克系外神" | plot Step 4 取用为第三卷支线展开 |

## 本阶段边界

### 本步做什么（管线入口模式）
- 路由诊断（Phase R）
- ★ 爽点引擎共创（Phase 爽点引擎）★ NEW
- PRD 撰写（A 先行 / B 浮现 / C 提炼）
- W0/W0.5/W1/W2 素材采集（按分支模式不同）
- 素材储备池沉淀
- 故事引擎设计 + 主角设计 + 样品试读

### 本步做什么（独立调起模式）
- Phase Delta 注入诊断
- 项目已有产物读取 + PRD 契合度评估 + 宪法冲突检测
- 素材储备池追加（如兼容）
- 冲突报告输出（如冲突）

### 本步不做什么（两模式通用）
- ❌ 不建世界观（那是 world 的活）
- ❌ 不设数值体系（那是 world Phase 4 的活）
- ❌ 不出角色卡（只出主角设计笔记，角色卡由 world 产出）
- ❌ 不设计剧情（素材储备池只提供种子，不设计剧情线——那是 plot 的活）
- ❌ 管线入口模式下不跳过样品试读（样品不通过，creative 使命未完成）
- ❌ 独立调起模式下不擅自修改宪法（即使有冲突，必须用户签字）

**越界检测**：
- 对话出现"这个世界的设定/力量等级/势力分布" → "这属于 world 的范围，到那一步处理。现在先让故事方向锁定。"
- 对话出现"这个素材能变成第X卷的冲突吗" → 标注进素材储备池，但不展开设计。"这是 plot 的活，素材我存好了，到那一步直接用。"

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 路由诊断不确定 A/B/C | 用 clarify 问一句："你目前脑子里有一个比较清晰的核心想法，还是手上有好几堆喜欢的东西想揉在一起，还是希望我帮你从零构思？" |
| 分支 A 的 PRD 在采集过程中被颠覆 | 标记 PRD v1 为"采集颠覆"，更新 PRD，重新标注素材筛选状态 |
| 分支 B 的素材堆积后浮现不出共同脉络 | 用 clarify 给 2-3 个候选脉络方向，让用户选 |
| 样品不通过 | 记录偏差点 → 退回 Phase 0 重新追问 |
| 样品不通过 x3 | 暂停管线。输出"创意分歧报告" |
| 方向 sketch 全部被拒 | 用 clarify(choices) 做结构化校准 |
| 无参考书 | 使用通用节奏模板，标注"无参考书" |
| deconstructor 产出缺失 | W2 改为查 pop-trope-library 公共库，不依赖 deconstructor 实时拆书 |
| WebSearch 不可用 | 降级为本地知识库，标注"未做在线搜索" |
| 用户中途换方向 | 保留当前产出为 v1，重新走 Phase R |
| 用户拒绝写样品 | 标注 `sample_approval: skipped_by_user`，告知 world |
| phase 文件缺失 | 从 SKILL.md 步骤描述推导执行 |
| Phase Delta：无 PRD.md 但有故事引擎.md | 从故事引擎提取七维度 PRD（一句话/类型锚点/核心承诺/加工哲学/DNA/边界/层架构），存入后继续 |
| Phase Delta：无 PRD.md 也无故事引擎 | 退回路管入口模式，走 Phase R 诊断 |
