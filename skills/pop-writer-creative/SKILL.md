# pop-writer-creative · 创意打磨 v2.0.1

> 你在开书管线中的位置：**pop-decon → pop-writer-creative → pop-writer-world → pop-writer-plot**
>
> 也可**独立调起**：项目运行中注入新元素（热点事件/IP/新灵感）时，直接触发 Phase Delta 流程。
>
> v2.0.1 变更：
> - 修复 Phase Delta 前置检查：无 PRD.md 但有故事引擎.md 时允许提取六维度 PRD 后继续，而非直接拒绝
> - 修复素材储备池引用：池模板托管在 creative，但持续注入由 pop-writer-reservoir 负责
> - 修复 Step 1.5 素材采集逻辑（在 step-inject.md 中）：用户输入不完整时采集而非臆测
>
> v2.0 核心变化：
> - **路由机制**：按用户启动状态走 A/B/C 三条分支 + 独立调起走 Phase Delta
> - **双产出模型**：产出物从单一「故事引擎」拆为「故事引擎.md（宪法约束）+ 素材储备池（剧情种子）」两件套
> - **PRD 独立**：PRD 从故事引擎中提前，视分支不同出现在素材前后
> - **Phase Delta**：项目中后期注入新元素，不重跑全流程（★ NEW）

## 路由总图

### 模式 1：管线入口（开书）

```
用户说"开书"
  │
  ▼
Phase R · 路由诊断
  │
  ├─ 用户有清晰核心想法 → 走 分支 A：PRD先行 → 定向采集 → 故事引擎
  │
  ├─ 用户有碎片化灵感 → 走 分支 B：广度采集 → PRD浮现 → 反审精炼 → 故事引擎
  │
  └─ 用户一片空白 → 走 分支 C：方向碰撞 → 选择深化 → 故事引擎
```

| 分支 | 适用场景 | 路径特征 | PRD 产生时机 |
|:----|:---------|:---------|:------------|
| A | "我要写一本XXX的书" | PRD → 筛选采集 | **素材采集前**（定加工坐标系） |
| B | "我喜欢A/B/C/D，怎么揉" | 采集 → PRD → 反审 | **素材采集后**（从碰撞中浮现） |
| C | "帮我想个故事" | 碰撞 → sketch → 深化 | **方向选定后**（从选择中提炼） |

### 模式 2：独立调起（项目中注入新元素）

```
用户说"我想把这个XX加进书里"
  │
  ▼
Phase Delta · 注入诊断（不重跑全流程）
  │
  ├─ ✅ 兼容 → 直接写入素材储备池，标注注入来源和触发场景
  ├─ ⚠️ 兼容但需微调 → 写池 + 建议 PRD/宪法微调（需用户确认）
  └─ ❌ 冲突 → 输出冲突报告，建议弃用或修宪
```

### 三条分支的产出物一致

无论哪条分支，creative 最终产出完全相同（只是产生顺序不同）：

```
creative 产出包
  │
  ├─ ① PRD.md          ← 基本法：这本书是什么/核心承诺/加工哲学/边界
  │
  ├─ ② 故事引擎.md      ← 宪法约束（PRD 的展开深化，给 world 的红线卡尺）
  │
  ├─ ③ 素材储备池.md    ← 剧情种子（给 plot 的软资源池）★ NEW
  │
  ├─ ④ 样品试读.md      ← 方向验证（用户签字后才交接）
  │
  └─ ⑤ 附加产出（视分支不同）
       ├─ _素材聚合/元素复用建议.md     ← W0 产出
       ├─ _素材聚合/跨域素材蒸馏.md     ← W1 产出
       ├─ _参考书分析/拆书融合摘要.md    ← W2 产出（有对标书时）
       ├─ _参考书分析/观察清单+差异化.md ← Phase 0.3 产出
       └─ _设计笔记/主角设计.md          ← Phase 0.4 产出
```

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase R 不可跳过** — 不诊断就开走 = 不知道自己在走哪条分支 |
| ❌2 | **W0 不可跳过** — 不查元素交叉库就开搜 = 浪费燃料 |
| ❌3 | **PRD 不可跳过** — 无 PRD 则素材无加工坐标系（C 分支方向选定后补写） |
| ❌4 | **素材储备池不可跳过** — 素材不沉淀则 plot 阶段从零发明剧情 |
| ❌5 | **有对标书不触发 W2 退回** — 有锚点书但不读 = 假对标 |
| ❌6 | **样品试读未通过直传 world** — Phase 0.5 不可跳过。通过后 creative 使命完成 |
| ❌7 | **主角设计四维不完整** — 身份/驱力/缺陷/唯一性任一缺失 → 退回 0.4 |
| ❌8 | **故事引擎读起来像配置表** — 不是故事提案 → 退回 Phase 0 |
| ❌9 | **产出只留摘要** — 写入文件后对话只说「已写入 {路径}。摘要：{核心}。」 |
| ❌10 | **同人二创误判为原创碰撞** — 锚定某部作品做同人二创 → 直接走分支 B 的锚点书子路径，不走方向 sketch 碰撞 |
| ❌11 | **融合后不贡献回元素交叉库** — 如果融合了新的素材来源，必须写入元素交叉库 |
| ❌12 | **素材储备池不分类直接堆** — 必须按主线索材/支线索材/背景素材/剩余焊接点四类归档，否则 plot 无法消费 |
| ❌D1 | **Phase Delta 不读项目产出就评估** — 必须找到 PRD.md（或从故事引擎提取）后才执行 |
| ❌D2 | **Phase Delta 不擅自修订宪法** — 即使建议微调，必须输出方案让用户签字 |
| ❌D3 | **Phase Delta 冲突时不沉默吞入** — 新元素明显与 PRD 冲突但未报告 |
| ❌D4 | **Phase Delta 注入不写储备池** — 评估完就结束，不写池 |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 用户说"我有三个喜欢的IP想揉" → Agent 直接出方向 sketch | 没先素材采集就碰撞，方向缺乏素材支撑 | 走分支 B → 先广度采集 → 从素材中浮现 PRD → 再精炼 |
| 用户说"我要写一本硬核历史修仙" → Agent 搜了一堆素材，没先定 PRD | 没 PRD 筛选器，素材平等全收，加工时无坐标系 | 走分支 A → PRD 先行 → 按 PRD 定向采集 |
| 素材采集完只写了故事引擎，没沉淀储备池 | 大量素材浪费，plot 从零编剧情 | 素材储备池四类归档 → plot 直接拉取素材当剧情种子 |
| 素材储备池只是一个巨长的素材清单，没有分类 | plot 无法判断"这个素材能当主线还是支线" | 按主线/支线/背景/剩余焊接点四类分类，标注优先级和潜在触发方式 |
| 用户只说"鹅腿事件" → Agent 直接解吸 | 用户没给素材内容，解吸出来的是臆测 | 先执行采集步骤（搜索/追问），补全素材内容后再解吸 |
| 项目有故事引擎但无 PRD.md → Agent 拒绝执行注入 | 项目 v2.0.0 之前启动，PRD 还没独立 | 从故事引擎提取六维度 PRD 后继续 |

## 收到任务后，按顺序执行

### Phase R：路由诊断（HARD GATE）

详见 [steps/step-r.md](steps/step-r.md)。

> **读什么**：用户初始输入
> **产出什么**：`_路由记录/分支选择.md`

从用户输入判断启动状态，走 A/B/C 中一条。决策依据：
- **分支 A**：用户有清晰的核心命题（"我要写一本 XXX 的书"）
- **分支 B**：用户有多个碎片化灵感/喜欢的 IP 想揉合
- **分支 C**：用户一片空白，"帮我想个故事"

---

### 分支 A：PRD 先行 → 定向采集

**适合**：用户脑子里已经有清晰的类型/题材/核心命题，只是需要加工坐标系来筛选素材。

#### 步骤顺序

```
Phase R → W0 → PRD（基本法）→ W0.5（带PRD定向提问）→ W1（按PRD筛选）→ W2（如有对标书）→ 素材储备池 → Phase 0 故事引擎 → 0.3/0.4/0.5
                            ↑                       ↑
                       加工坐标系已定          定向搜索，不浪费燃料
```

#### A-1. W0：查已归档的融合方案

详见 [steps/step-w0.md](steps/step-w0.md)。

> **分支 A 特殊处理**：带 PRD 筛选条件检索。例如 PRD 说"硬核历史修仙，拒绝降神" → 在元素交叉库中只检索与硬核历史/修仙/考据相关的归档方案。

产出同 W0 标准：`_素材聚合/元素复用建议.md`

#### A-2. PRD：撰写基本法

详见 [steps/step-prd.md](steps/step-prd.md)。

> **产出什么**：`00-原始设定/PRD.md`

从用户核心想法提炼：
- 一句话定位
- 类型锚点（主/辅）
- 核心承诺（读者读这本书得到什么）
- 加工哲学（这个类型如何加工外部元素）
- 不可妥协的 DNA（什么绝对不能丢）
- 边界（这本书不写什么）

**PRD 不是故事引擎**。PRD 是 3-5 条的轻量基本法，只回答"我这本书是什么"。不展开世界锚点、不设计主角、不画卷级结构——那些是故事引擎的事。

#### A-3. W0.5：问作者要元素

详见 [steps/step-w0.5.md](步骤未变更，参见 v1)。

> **产出什么**：`_创意元素/作者想法.md`

#### A-4. W1：跨域素材定向采集

详见 [steps/step-w1.md](steps/step-w1.md)。

> **分支 A 特殊处理**：按 PRD 的加工坐标系定向搜索。不是"有什么搜什么"，而是"PRD 说需要什么才搜什么"。

产出同 W1 标准：`_素材聚合/跨域素材蒸馏.md`

#### A-5. W2：拆书融合（有对标书时强制）

详见 [steps/step-w1.md](steps/step-w1.md) 的拆书融合部分。

> **分支 A 特殊处理**：按 PRD 决定拆什么书、重点看什么维度。

产出：`_参考书分析/拆书融合摘要.md`

#### A-6. 素材储备池

详见 [templates/materials-pool.tpl.md](templates/materials-pool.tpl.md)。

> **读什么**：W0/W0.5/W1/W2 全部产出
> **产出什么**：`00-原始设定/素材储备池.md`

将所有采集素材按四类归档。**特别说明**：分支 A 由于 PRD 先行，素材天然带有加工标签，归档时直接标注"已按 PRD 筛选"的分级。

#### A-7 → Phase 0 → 0.3 → 0.4 → 0.5

同标准流程（见下文"Phase 0 故事引擎"及之后）。

---

### 分支 B：广度采集 → PRD 浮现

**适合**：用户有多个碎片化灵感或喜欢的 IP，想揉合但还不知道具体的"这本书是什么"。

#### 步骤顺序

```
Phase R → W0（广度检索）→ W0.5（广度提问）→ W1（广度采集）→ W2（如有对标书）
→ 素材堆积 → PRD浮现（从素材中提炼共同脉络）→ 素材储备池（用PRD反审精炼）
→ Phase 0 故事引擎 → 0.3/0.4/0.5
```

#### B-1. W0：查已归档的融合方案（广度检索）

详见 [steps/step-w0.md](steps/step-w0.md)。

> **分支 B 特殊处理**：不设筛选条件，广度检索所有与用户碎片灵感相关的归档方案。目标是**收集尽可能多的焊接参考**。

#### B-2. W0.5：问作者要元素（广度提问）

详见 [steps/step-w0.5.md](步骤未变更)。

> **分支 B 特殊处理**：多问几轮。用户碎片灵感多时，往往有大量未表明的隐藏素材。建议追问 2-3 轮。

#### B-3. W1：跨域素材广度采集

详见 [steps/step-w1.md](steps/step-w1.md)。

> **分支 B 特殊处理**：广度搜索，按用户提到的所有碎片/灵感/IP 分别搜。目标不是深度，而是**覆盖面**——把所有可能的素材先拉进来。

#### B-4. W2：拆书融合（有对标书时强制）

同标准流程。

#### B-5. PRD 浮现

详见 [steps/step-prd.md](steps/step-prd.md) 的"浮现模式"。

> **产出什么**：`00-原始设定/PRD.md`
>
> **核心动作**：从堆积的素材中提炼共同脉络。
> 1. 把所有素材铺开
> 2. 找"这些素材之间有什么共同的冲突模式/审美取向/叙事基因"
> 3. 形成 2-3 个候选 PRD 方向
> 4. 用户选一个 → 锁定 PRD

这个 PRD 是**从素材长出来的**，不是预判的。

#### B-6. 素材储备池（PRD 反审精炼）

详见 [templates/materials-pool.tpl.md](templates/materials-pool.tpl.md)。

> **分支 B 特殊处理**：PRD 浮现后，用 PRD 反审所有已采集的素材：
> - 符合 PRD 的素材 → 标记可用优先级
> - 不符合 PRD 的素材 → 标注"PRD 反审后淘汰，可做支线候选"
> - 特别有趣的素材但不符 PRD → 放入剩余焊接点，标注"风格/基调不符，但剧情潜力高"

#### B-7 → Phase 0 → 0.3 → 0.4 → 0.5

同标准流程。

---

### 分支 C：空白型 → 方向碰撞 → 提炼 PRD

**适合**：用户一片空白，"帮我想个故事"。

#### 步骤顺序

```
Phase R → W0 → W0.5 → W1（最小广度）→ Phase 0（方向碰撞→用户选择→深化）
→ PRD提炼（从选定方向中提取）→ 素材储备池 → 0.3/0.4/0.5
```

其实分支 C 就是 v1.5 的默认流程（方向碰撞 → 深化），只是新增了 PRD 提炼和素材储备池两个产出步骤。

详细步骤参见 v1.5 的 Phase 0 流程和 [steps/step-0.md](steps/step-0.md)。在深化出故事引擎后，追加：
1. **PRD 提炼**：从选定的方向 sketch 中提炼 PRD.md（使用 [step-prd.md](steps/step-prd.md)）
2. **素材储备池**：碰撞过程中的淘汰方向、关联联想、未选元素沉淀为素材储备池

---

### Phase 0：故事引擎设计

> **读什么**：PRD.md + 素材储备池 + 全部素材产出
> **产出什么**：`00-原始设定/故事引擎.md`

#### 共同步骤（A/B/C 三分支统一）

1. 读取 PRD.md → 作为故事引擎的宪法基线
2. 读取素材储备池 → 选取主线索材进入故事引擎主线
3. 从 PRD 展开深化 → 填充故事引擎模板
4. 故事引擎锁定后，不可随意修改（修宪回路见异常处理）

#### 分支 A 特殊处理
PRD 先行所以自然展开。故事引擎 = PRD 的深化版本，第一条就是 PRD 内容。

#### 分支 B 特殊处理
PRD 从素材浮现后，故事引擎 = PRD + 从素材储备池中选取的高优先级主线索材。

#### 分支 C 特殊处理
故事引擎 = v1.5 标准流程（方向 sketch → 用户选择 → 深化），在选定方向后补写 PRD 作为第一章。

#### 共同产出
`00-原始设定/故事引擎.md`（模板见 [templates/story-engine.tpl.md](templates/story-engine.tpl.md)）

## Phase Delta：项目运行中的新元素注入（★ 独立调起模式）

> **定位**：独立调起入口。当项目已过 creative/world/plot 阶段，作者想把某个新元素（热点事件/现实新闻/IP新增/新灵感）注入书里时，不重跑全流程——直接执行轻量级注入评估。
>
> **触发条件**：用户说"帮我把这个XX融进书里""最近有个XX热点能写进我的书吗""我想到一个新设定"——且当前项目已经有 PRD + 故事引擎 + 素材储备池（即已跑过管线入口流程）。
>
> **注意**：完整的素材储备注入功能已独立为 **[pop-writer-reservoir](../pop-writer-reservoir/SKILL.md)** skill。Phase Delta 在此处作为 creative 内部的快捷注入入口，输出格式和评估逻辑与 pop-writer-reservoir 一致。重度注入场景（多素材/复杂适配）请直接调起 pop-writer-reservoir。

### 调用形态

```mermaid
flowchart LR
    A[用户说<br>"把这个XX加进书里"] --> B[Phase Delta<br>注入诊断]
    B --> C{PRD 契合度}
    C -->|兼容| D[写入素材储备池<br>+ 标注注入源]
    C -->|微调可兼容| E[输出微调建议<br>→ 用户确认后生效]
    C -->|冲突| F[输出冲突报告<br>→ 用户决策]
```

### 前置检查：定位项目已有产出

Phase Delta 执行前必须验证项目状态：

| 文件存在 | 动作 |
|:---------|:-----|
| `PRD.md` ✅ | 直接使用 |
| 无 PRD.md 但有 `故事引擎.md` | 从故事引擎提取六维度 PRD，存入 `00-原始设定/PRD.md` 后继续 |
| 无 PRD.md 也无故事引擎 | ❌ 这不是已运行项目 → 退回路管入口模式，走 Phase R 诊断 |

| 文件存在 | 动作 |
|:---------|:-----|
| `素材储备池.md` ✅ | 后续追加 |
| 无素材储备池 | 首版创建 |

**部分缺失标注**：缺失某些可选文件时降级执行——无法检测的维度标注"因文件缺失跳过的检查项"。

### 步骤顺序

详见 [steps/step-delta.md](steps/step-delta.md)。

```
Phase Delta 注入诊断
  1. 读项目已有产物（PRD / 故事引擎 / 世界宪法 / 素材储备池）
  2. PRD 契合度评估（新元素与 PRD 加工哲学是否兼容）
  3. 宪法冲突检测（新元素是否违反已锁定的宪法边界）
  4. 等级判定
     ├─ ✅ Phase D1: 直接注入
     ├─ ⚠️ Phase D2: 微调后注入（建议PRD/宪法微调方案）
     └─ ❌ Phase D3: 冲突报告 + 建议弃用
  5. 执行注入或输出报告
```

### Delta 流程的边界

| 动作 | 做 | 不做 |
|:-----|:---|:-----|
| 项目已有产物 | ✅ 读取 PRD.md + 故事引擎.md + 世界宪法.md + 素材储备池.md | ❌ 不回头重新采集全部素材 |
| 新元素处理 | ✅ 评估 PRD 契合度，写入素材储备池 | ❌ 不重新生成故事引擎 / 不修改 world 设定 |
| 宪法冲突 | ✅ 输出冲突报告，让用户决策 | ❌ 不擅自修改宪法 |
| 素材储备池 | ✅ 追加新元素，标注"注入"来源 | ❌ 不清空或重组现有池子 |
| 后续流程 | ✅ 告知用户注入结果 + 建议触发 plot 什么阶段 | ❌ 不自动进入 plot 流程（由用户决定什么时候用） |
| **受众立场**（★ 社会安全闸门） | ✅ 现实素材必须做三方立场分析；安全等级判定（✅/⚠️/🔴/☠️）；产出改编红线约束 | ❌ 绝不可跳过立场评估直接出融合方案 — 见 pop-writer-reservoir ❌R8 |

### 质量红线

| # | 红线 |
|:-:|:-----|
| ❌D1 | **不读项目已有产出就评估** — Phase Delta 必须在项目目录下找到 PRD.md（或从故事引擎提取后）执行。找不到 = 这不是已运行项目 = 走回管线入口 |
| ❌D2 | **不擅自修订宪法** — 即使建议微调，必须输出方案让用户签字，不可直接修改 |
| ❌D3 | **冲突时不沉默吞入** — 新元素明显与 PRD 冲突但未报告 = 污染宪法一致性 |
| ❌D4 | **注入不写入储备池** — 评估完就结束，不写池 = 信息无法被 plot 消费 |
| ❌D5 | **重复注入同一元素** |
| ❌D6 | **不评估受众立场就出融合方案** — 现实素材必须做三方立场分析+情绪检测+安全等级判定。跳过 = 你可能帮现实中的恶人洗白（真实教训：2026-06-17） |

### 典型使用场景

| 场景 | 处理 |
|:-----|:-----|
| 作者刷到一条社会新闻，觉得可以改写成书里一个事件 | 评估新闻核心冲突与 PRD 的匹配度 → 写入支线索材，标注"现实事件改编" |
| 作者新迷上一个 IP，想把这个 IP 的元素融进现有世界观 | 评估 IP 核心设定与 PRD/宪法的冲突 → 如果兼容写入储备池；如果有张力写入剩余焊接点 |
| 作者突然想到一个角色设定，但书已经写了一半 | 评估角色基底 vs PRD 主角边界 → 如果兼容写入背景素材，plot 卷间可用 |
| 作者发现一个历史冷知识，想嵌入书里当暗线 | 按 PRD 加工哲学判断这个冷知识适合当主线触发器还是背景色块 → 对应归档 |

## 素材储备池详解（★ 新增核心产出）

### 定位

素材储备池是 creative 给 **plot 的软资源池**。plot 在设计卷/幕剧情时，不再从零发明冲突，而是从储备池中拉取素材。

### 素材储备池 vs 故事引擎

| 维度 | 故事引擎（宪法） | 素材储备池（资源池） |
|:-----|:----------------|:-------------------|
| 性质 | **硬约束** | **软资源** |
| 使用方式 | world 必须遵守 | plot 可选拉取 |
| 修改频率 | 锁定后极少修改 | plot 卷间动态消耗/补充 |
| 产出时间 | creative 阶段 | creative 阶段首版，plot 阶段持续补充 |
| 消费者 | world（世界观建构） | plot（剧情设计） |

### 四类种子

| 类别 | 定义 | 示例 | plot 使用场景 |
|:-----|:-----|:-----|:-------------|
| **主线索材** | 可以直接触发主线事件的冲突/设定/机制 | "混沌四神教团作为主线反派势力" | plot Step 1 取用为主角头号对手 |
| **支线索材** | 可以独立成线的冲突/势力/机制/设定 | "深渊底层的克系外神" | plot Step 4 取用为第三卷支线展开 |
| **背景素材** | 世界观色块，不驱动剧情但丰富世界 | "灰骑士原体种子的训练流程" | plot Step 4.5 卷设计时嵌入背景设定 |
| **剩余焊接点** | 已构思了焊接方案但未在主线使用的元素 | "时间循环机制，原构思为金手指但被灰骑士种子替代" | plot Step 9 某幕缺乏冲突时取用 |

### 产出格式

详见 [templates/materials-pool.tpl.md](templates/materials-pool.tpl.md)。
素材储备池的持续注入由独立 skill **[pop-writer-reservoir](../pop-writer-reservoir/SKILL.md)** 负责。

## 本阶段边界

### 本步做什么（管线入口模式）
- 路由诊断（Phase R）
- PRD 撰写（A 先行 / B 浮现 / C 提炼）
- W0/W0.5/W1/W2 素材采集（按分支模式不同）
- 素材储备池沉淀
- 故事引擎设计 + 主角设计 + 样品试读

### 本步做什么（独立调起模式）
- Phase Delta 注入诊断
- 项目已有产物读取 + PRD 契合度评估 + 宪法冲突检测
- 素材储备池追加（如兼容）
- 冲突报告输出（如冲突）

### 本步不做什么（两模式通用）
- ❌ 不建世界观（那是 world 的活）
- ❌ 不设数值体系（那是 world Phase 4 的活）
- ❌ 不出角色卡（只出主角设计笔记，角色卡由 world 产出）
- ❌ 不设计剧情（素材储备池只提供种子，不设计剧情线——那是 plot 的活）
- ❌ 管线入口模式下不跳过样品试读（样品不通过，creative 使命未完成）
- ❌ 独立调起模式下不擅自修改宪法（即使有冲突，必须用户签字）

**越界检测**：
- 对话出现"这个世界的设定/力量等级/势力分布" → "这属于 world 的范围，到那一步处理。现在先让故事方向锁定。"
- 对话出现"这个素材能变成第X卷的冲突吗" → 标注进素材储备池，但不展开设计。"这是 plot 的活，素材我存好了，到那一步直接用。"

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 路由诊断不确定 A/B/C | 用 clarify 问一句："你目前脑子里有一个比较清晰的核心想法，还是手上有好几堆喜欢的东西想揉在一起，还是希望我帮你从零构思？" |
| 分支 A 的 PRD 在采集过程中被颠覆 | 标记 PRD v1 为"采集颠覆"，更新 PRD，重新标注素材筛选状态 |
| 分支 B 的素材堆积后浮现不出共同脉络 | 用 clarify 给 2-3 个候选脉络方向，让用户选 |
| 样品不通过 | 记录偏差点 → 退回 Phase 0 重新追问 |
| 样品不通过 x3 | 暂停管线。输出"创意分歧报告" |
| 方向 sketch 全部被拒 | 用 clarify(choices) 做结构化校准 |
| 无参考书 | 使用通用节奏模板，标注"无参考书" |
| deconstructor 产出缺失 | 跳过 W1，标注"无锚点书" |
| WebSearch 不可用 | 降级为本地知识库，标注"未做在线搜索" |
| 用户中途换方向 | 保留当前产出为 v1，重新走 Phase R |
| 用户拒绝写样品 | 标注 `sample_approval: skipped_by_user`，告知 world |
| phase 文件缺失 | 从 SKILL.md 步骤描述推导执行 |
| Phase Delta：无 PRD.md 但有故事引擎.md | 从故事引擎提取六维度 PRD（一句话/类型锚点/核心承诺/加工哲学/DNA/边界），存入后继续 |
| Phase Delta：无 PRD.md 也无故事引擎 | 退回路管入口模式，走 Phase R 诊断 |
