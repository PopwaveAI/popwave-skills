## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **跨域素材不得跳过（HARD GATE）** — Phase W0 不可跳过。燃料不足不进 Phase 0 |
| ❌2 | **故事引擎.md 已锁定** — 元素融合走完 + 方向 sketch 用户选择 + 深化 + 样品签字 |
| ❌3 | **样品试读未通过直传 world** — Phase 0.5 不可跳过。通过后 creative 使命完成 |
| ❌4 | **拆书成果未融合** — 有锚点书产出但不读取不融合 → 退回 W1 |
| ❌5 | **主角设计四维不完整** — 身份/驱力/缺陷/唯一性任一缺失 → 退回 0.4 |
| ❌6 | **故事引擎读起来像配置表** — 不是故事提案 → 退回 Phase 0 |
| ❌7 | **产出只留摘要** — 写入文件后对话只说「已写入 {路径}。摘要：{核心}。」|
| ❌8 | **同人二创误判为原创碰撞** — 用户提到锚定某部作品做同人二创/跨界融合 → 直接走 Phase 0 分支 B，不走方向sketch碰撞。误走分支 A 产出无用方向 → 退回重走 |
| ❌9 | **元素交叉库未查就开搜** — Phase W0 不可跳过。先查归档的融合方案，再走跨域搜索。有归档不读 = 浪费燃料 |
| ❌10 | **融合后不贡献回元素交叉库** — Phase 0 完成后，如果融合了新的素材来源（不限IP，热梗/事件/现象都行），必须写入元素交叉库。否则下一个人搜不到。 |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 用户说"写修仙" → Agent 追问"什么时代？什么地域？力量上限？" | 质询式追问 = 劝退，像审问不是合作 | 解吸"修仙" → 联想碰撞元素 → 生成 2-3 个轻量 sketch → 用户选一个深化 |
| 样品没通过 → Agent 说"先建世界观后面再改" | 不通过硬推进 = 带问题的 engine 建世界观，浪费后续工作 | 记录偏差 → 退回 Phase 0 重新碰撞 → 重写样品 |
| 用户说"开新书" → Agent 直接开始设计力量体系和世界地图 | 越界了。故事方向还没锁定就建世界，建出来的全是空的 | 先走完 Phase 0 故事引擎碰撞 → 方向锁定 → 样品签字 → 再交接 world |
| 用户说"基于《深渊主宰》写同人，主角替换+混沌四神+克系外神入侵" → Agent 按原创流程出三个方向sketch | 忽视锚点书已有方向，从零碰撞出的方向跟用户想要的无关 | 走 Phase 0 分支 B → 锚点书解析 → 替换点 + 元素叠层 → 直接产出 story-engine.md |
| 分析文档/融合方案用 YAML 格式输出 | **用户偏好冲突** — 用户明确要求用 Markdown 而非 YAML 作为文档格式规范，且不限于知名IP——热梗/热门事件也可作为跨域素材 | 所有分析文档、策略文档、融合方案一律用 Markdown。纯数据表可 YAML。不确定时优先 Markdown |

## 收到任务后，按顺序执行

### Phase W0：查已归档的融合方案（HARD GATE）

详见 [steps/step-w0.md](steps/step-w0.md)。

> **读什么**：`D:\popwave-skills\skills\pop-trope-library\元素交叉\`
> **产出什么**：`_素材聚合/元素复用建议.md`
> **门禁**：❌ 不可跳过。不查元素交叉库就开搜 = 浪费燃料

### Phase W0.5：问作者要元素（HARD GATE）

> **读什么**：用户本人（直接对话）
> **产出什么**：`_创意元素/作者想法.md`
> **门禁**：❌ 不可跳过。不问作者就开融合 = 替作者做决定

**为什么要这一步**：元素交叉库存的是"别人用过的方案"，但你的作者手里可能有自己的素材——某个画面、一段经历、一个现象、一个热梗。这些没入库，不问就没有。**这一步产出的文件放在项目目录下的 `_创意元素/`，不是 skill 库——这是作者自己的素材，不是系统资产。**

**操作步骤**：

1. 直接提问（语气自然，不像是审问）：
   > "除了我们刚刚聊的这些，你脑子里有没有什么画面/场景/人/故事片段，你特别想写进这本书里的？不限题材来源——可以是你看过的电影某个镜头、去过的某个地方、听到的一句对话、一个社会现象、一个网络热梗、甚至是你自己经历过的事。"

2. 如果用户给了，写入 `_创意元素/作者想法.md`，标注来源性质：
   - 来源（用户自述/亲身经历/灵感片段/其他）
   - 是否愿意公开到元素交叉库（供其他人复用）

3. 如果用户说"暂时没有"，标注 `author_elements: none`，跳过。**后续若用户随时提到新的想法，追加到 `_创意元素/作者想法.md`。**

4. **重要原则**：这一步不是"走流程"——用户给的元素往往是最个性化的、最有辨识度的。宁可在这一步花10分钟聊天，也不要在建完世界观后才发现缺失。

### Phase W1：跨域素材聚合（有对标书时强制）

详见 [steps/step-w1.md](steps/step-w1.md)。

> **读什么**：deconstructor Lv1 产出（原创借鉴模式）或 锚点书直接提取（同人二创模式）
> **产出什么**：`_参考书分析/拆书融合摘要.md`
> **门禁**：❌ 有对标书不触发退回

**路径 A — 原创借鉴（默认）**：用户想学习成功架构但不写同人。
1. 用户提到对标书 → 触发 deconstructor Lv1（下载+全文拆解）
2. 读取 deconstructor 产出 → 逐 T 提取参考值
3. 产出融合摘要 + M1/M2 转译（继承规律不继承事件）

**路径 B — 同人二创/熟书轻量提取**：用户锚定某本书做同人二创，且用户已熟知原作（不需要下载拆解）。
1. 直接从 agent 知识提取原作核心架构（时代/力量体系/位面/关键角色/势力）
2. 标注"强制保留"和"可替换"部分
3. 产出融合摘要（以替换点清单+叠层策略为主，而非 deconstructor 的逐层分析）
4. 标注 `extraction_method: knowledge_based`（而非 `deconstructor_lv1`）
### Phase 0：故事引擎设计（元素融合 SOP）

详见 [steps/step-0.md](steps/step-0.md)。

> **读什么**：W0 产出（元素复用建议）+ W1 产出（跨域素材）+ W2 产出（拆书融合摘要，如有）
> **产出什么**：`00-原始设定/故事引擎.md`

#### 分支 A：原创碰撞（默认）
用户从零构建原创世界观时使用。**开始前先把 W0 的复用建议读一遍**——如果已有归档方案，直接复用焊接方式，不用从头联想。

1. **碎片解吸 + 元素联想**：从用户碎片中提取"不可妥协元素"，联想 3-5 个候选碰撞元素。**如果 W0 有匹配方案，复用方案中的焊接方式**
2. **碰撞评级**：★★★ 强碰撞优先，★☆☆ 淘汰。取前 2-3 个
3. **落主角**：一个必须同时遵守两套规则、但两套规则互相否定的普通人
4. **轻量方向 sketches**：每个 ~150 字，含碰撞/钩子/第一画面/基调。不展开三层
5. **用户选择** → 只深化选中那一个 → 故事引擎.md
6. ✅ **贡献回元素交叉库**：如果这次融合了新的素材来源（不限IP，热梗/事件/现象都行），写入 `pop-trope-library/元素交叉/{来源类型}→{来源名}→{本书名}.md`，格式参照已有条目。

#### 分支 B：同人二创 / 跨界融合
当用户锚定某部已有作品做同人二创、跨界融合或角色替换时使用。**不要走方向sketch碰撞（方向已由锚点书锁定）——直接套修改后的架构产出 story-engine.md。开始前先把 W0 的复用建议读一遍。**

1. **锚点书解析**：提取原作骨架（时代/力量体系/位面/主要势力/关键人物），标注"强制保留"和"可替换"部分
2. **替换点确认**：列出角色替换（谁取代谁）、时间线调整、事件改写清单
3. **跨界元素叠层**：将新增IP元素焊接入锚点架构。问清每个添加元素——"它是替换了原作的什么，还是在原有间隙插入的?"
4. **直接产出故事引擎.md**（跳过方向sketch碰撞，方向已锁定）
5. 然后正常走 Phase 0.3/0.4/0.5
6. ✅ **贡献回元素交叉库**：如果叠层了新的跨界元素（不限IP，热梗/事件/现象都行），写入 `pop-trope-library/元素交叉/{来源类型}→{来源名}→{本书名}.md`

**❌ 典型错误**：用户说"基于《XXX》写同人，主角替换为YYY，再加ZZZ元素"→ Agent 按原创流程出方向sketch全被否定。正确做法：锚定原作 → 角色替换 + 元素叠层 → 直接生成 story-engine.md。

#### 共同步骤
故事引擎模板见 [templates/story-engine.tpl.md](templates/story-engine.tpl.md)。

### Phase 0.5：样品试读（核心闸门）

2. 写 500-2000 字

## 本阶段边界

### 本步做什么
- W0/W1 素材采集 → 方向碰撞 → 深化 → 样品 → 交接

### 本步不做什么
- ❌ 不建世界观（那是 world 的活）
- ❌ 不设数值体系（那是 world Phase 4 的活）
- ❌ 不出角色卡（只出主角设计笔记，角色卡由 world 产出）

**越界检测**：对话出现"这个世界的设定/力量等级/势力分布" → "这属于 world 的范围，到那一步处理。现在先让故事方向锁定。"

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 样品不通过 | 记录偏差点 → 退回 Phase 0 重新追问 |
| 样品不通过 x3 | 暂停管线。输出"创意分歧报告" |
| 方向sketch 全部被拒 | 用 clarify(choices) 做结构化校准：提供 2-4 个具体选项（基调/视角/元素配比/叙事重心），每个选项一句话。不退行盲目重新碰撞 |
| 方向sketch 被拒 x2 | 退行到问句模式：让用户用自然语言描述"我想要的感觉"。不从零出草图，而是追问已有方向"哪个部分不对" |
| 无参考书 | 使用通用节奏模板，标注"无参考书" |
| deconstructor 产出缺失 | 跳过 W1，标注"无锚点书" |
| WebSearch 不可用 | 降级为本地知识库，标注"未做在线搜索" |
| 用户中途换方向 | 保留当前 engine 为 v1，重新走 Phase 0 |
| 用户拒绝写样品 | 标注 `sample_approval: skipped_by_user`，告知 world |
| phase 文件缺失 | 从 SKILL.md 步骤描述推导执行 |
