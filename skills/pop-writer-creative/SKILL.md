# 创意打磨 v1.4.1

> 你在开书管线中的位置：**deconstructor → creative → world**。你是创意层唯一的守门员。**先采集燃料，再碰撞创意——没有足够的跨域素材和拆书数据，碰撞不出好方向。**

## 速查表

| 用户说 | 执行路径 | 前置条件 |
|--------|---------|---------|
| "开新书/方向碰撞" | W0(跨域素材) → W1(拆书融合) → Phase 0(故事引擎) → 0.3(参考书) → 0.4(主角) → 0.5(样品) | 无 |
| "我想写[题材]" | 同上，从用户碎片提取关键词 | — |
| "有参考书" | W1 强制执行拆书融合 | deconstructor 必须先产出 |
| "样品通过/继续推进" | 签字 → 交接 world | 样品必须通过 |

## ❌ 质量红线

| # | 标准 | 完工确认 |
|:-:|:-----|:--------:|
| ❌0 | **跨域素材不得跳过（HARD GATE）** — Phase W0 不可跳过。燃料不足不进 Phase 0 | [ ] |
| ❌1 | **故事引擎.md 已锁定** — 元素融合走完 + 方向 sketch 用户选择 + 深化 + 样品签字 | [ ] |
| ❌2 | **样品试读已通过** — Phase 0.5 不可跳过。通过后 creative 使命完成 | [ ] |
| ❌3 | **拆书成果已融合** — 有锚点书产出但不读取不融合 → 退回 | [ ] |
| ❌4 | **参考书策略已执行** — 如有对标书：观察清单已形成 + deconstructor Lv1 已触发 | [ ] |
| ❌5 | **主角设计四维完整** — 身份/驱力/缺陷/唯一性全部填充 | [ ] |
| ❌6 | **样品签字格式完整** — sample_approval 块写入故事引擎.md | [ ] |
| ❌7 | **故事引擎可被十分钟读完** — 读起来像故事提案不是配置表 | [ ] |
| ❌8 | **不准跳过样品试读直传 world** — world 收到没有样品签字的 engine → 退回 creative | [ ] |

## ❌ 产出纪律

| # | 规则 |
|:-:|:-----|
| ❌9 | **写入文件后对话中只留摘要（≤ 200 字），不粘贴完整产出。** 正确格式："已写入 {路径}。摘要：{核心}。需展开告诉我。" |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 用户说"写修仙" → Agent 追问"什么时代？什么地域？力量上限？" | 质询式追问 = 劝退，像审问不是合作 | 解吸"修仙" → 联想碰撞元素 → 生成 2-3 个轻量 sketch → 用户选一个深化 |
| 样品没通过 → Agent 说"先建世界观后面再改" | 不通过硬推进 = 带问题的 engine 建世界观，浪费后续工作 | 记录偏差 → 退回 Phase 0 重新碰撞 → 重写样品 |
| 用户说"开新书" → Agent 直接开始设计力量体系和世界地图 | 越界了。故事方向还没锁定就建世界，建出来的全是空的 | 先走完 Phase 0 故事引擎碰撞 → 方向锁定 → 样品签字 → 再交接 world |

## 收到任务后，按顺序执行

### Phase W0：跨域素材聚合（HARD GATE）

详见 [steps/step-w0.md](steps/step-w0.md)。

> **读什么**：用户碎片（初始输入）
> **产出什么**：`_素材聚合/跨域素材蒸馏.md`
> **门禁**：❌ W0 不可跳过。没有素材碰撞 = 贫瘠碰撞

1. 从用户碎片提取 3-5 个关键词
2. 跨域搜索：六大领域至少覆盖 4 个（地理/社会/生物/技术/文化/心理）
3. 素材蒸馏：每方向 ≥ 2 个种子，标注来源/关联度/可转化性
4. 碰撞联想：每个种子标注"能点燃什么方向"

### Phase W1：拆书融合（有对标书时强制）

详见 [steps/step-w1.md](steps/step-w1.md)。

> **读什么**：deconstructor Lv1 产出
> **产出什么**：`_参考书分析/拆书融合摘要.md`

1. 用户提到对标书 → 触发 deconstructor Lv1
2. 读取 deconstructor 产出 → 逐 T 提取参考值
3. 产出融合摘要 + M1/M2 转译（继承规律不继承事件）

### Phase 0：故事引擎设计（元素融合 SOP）

详见 [steps/step-0.md](steps/step-0.md)。

> **读什么**：W0 产出 + W1 产出（如有）
> **产出什么**：`00-原始设定/故事引擎.md`

1. **碎片解吸 + 元素联想**：从用户碎片中提取"不可妥协元素"，联想 3-5 个候选碰撞元素
2. **碰撞评级**：★★★ 强碰撞优先，★☆☆ 淘汰。取前 2-3 个
3. **落主角**：一个必须同时遵守两套规则、但两套规则互相否定的普通人
4. **轻量方向 sketches**：每个 ~150 字，含碰撞/钩子/第一画面/基调。不展开三层
5. **用户选择** → 只深化选中那一个 → 故事引擎.md

故事引擎模板见 [templates/story-engine.tpl.md](templates/story-engine.tpl.md)。

### Phase 0.3：参考书策略

详见 [steps/step-0.3.md](steps/step-0.3.md)。

> **读什么**：碰撞点（已确定的方向）
> **产出什么**：`_参考书分析/观察清单+差异化.md`

从碰撞点推导验证需求 → 形成观察清单 → 触发 deconstructor Lv1 → 差异化决策 → 反哺宪法

### Phase 0.4：主角设计

详见 [steps/step-0.4.md](steps/step-0.4.md)。

> **读什么**：碰撞点 + 方向 sketch
> **产出什么**：`_设计笔记/主角设计.md`

从碰撞点反推主角 → 四维展开（身份/驱力/缺陷/唯一性）→ 唯一性四级评估（必走）

### Phase 0.5：样品试读（核心闸门）

详见 [steps/step-0.5.md](steps/step-0.5.md)。

> **读什么**：故事引擎.md
> **产出什么**：`_样品试读/样品-v1.md`

1. 确定样品段落（关键场景，不写设定介绍）
2. 写 500-2000 字
3. 用户确认或退回。通过后签字锁定

## 本阶段边界

### 本步做什么
- W0/W1 素材采集 → 方向碰撞 → 深化 → 样品 → 交接

### 本步不做什么
- ❌ 不建世界观（那是 world 的活）
- ❌ 不设数值体系（那是 world Phase 4 的活）
- ❌ 不出角色卡（只出主角设计笔记，角色卡由 world 产出）

**越界检测**：对话出现"这个世界的设定/力量等级/势力分布" → "这属于 world 的范围，到那一步处理。现在先让故事方向锁定。"

## 落盘检查点

| 文件 | 路径 | 状态要求 |
|:-----|:-----|:---------|
| 跨域素材蒸馏 | `_素材聚合/跨域素材蒸馏.md` | 已产出，≥4 领域 |
| 拆书融合摘要 | `_参考书分析/拆书融合摘要.md` | 有对标书时必出，无标注"无锚点书" |
| 故事引擎 | `00-原始设定/故事引擎.md` | 已锁定，含方向选择记录 |
| 主角设计 | `_设计笔记/主角设计.md` | 四维完整 |
| 样品 | `_样品试读/样品-v{N}.md` | 用户签字 |

**落盘后动作**：交接给 pop-writer-world。

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 样品不通过 | 记录偏差点 → 退回 Phase 0 重新追问 |
| 样品不通过 x3 | 暂停管线。输出"创意分歧报告" |
| 无参考书 | 使用通用节奏模板，标注"无参考书" |
| deconstructor 产出缺失 | 跳过 W1，标注"无锚点书" |
| WebSearch 不可用 | 降级为本地知识库，标注"未做在线搜索" |
| 用户中途换方向 | 保留当前 engine 为 v1，重新走 Phase 0 |
| 用户拒绝写样品 | 标注 `sample_approval: skipped_by_user`，告知 world |
| phase 文件缺失 | 从 SKILL.md 步骤描述推导执行 |
