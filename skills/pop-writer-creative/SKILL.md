# pop-writer-creative · 创意打磨 v3.6.0

> **定位**：从用户模糊想法到高质量立项 PRD 的**唯一入口**。PRD 产出后交付 world。
>
> v4.1.0：**PRD 宪法结构最终定型**——核心故事(故事梗概) + 核心卖点(5分类:设定/世界观理论体系/剧情/主角金手指/全书级) + 美学基因推导链(含质感→力量→情感→命名→视觉锚点场景) + 用户画像 + 世界宪法 + 跨域素材指引 + 篇幅规划 + 核心承诺+防崩阈值 + DNA+边界。PRD 不做具体名/数值/分幕/转译规则——那些归 L1/plot。
>
> v4.0.0：PRD 移除加工矩阵/层架构/材料选择，改为宪法层结构。
>
> v3.6.0：**Phase 2 自问链 + 场景判定 R/M/O + 元模板**——消除「分类代替分析」。拿到元爽点标签后必须向下追问 L1-L4 特征参数；不匹配 7 类的走元模板生成自定义追问链；搜索策略不再基于类型模板，而是基于特征参数精准搜索。
>
> v3.5.0：**职责收窄**——只做 PRD 推导。素材采集/注入/样品试读/Phase Delta 全部移除。

## 路由

```
用户说"开书" / "我有一个想法"
  │
  ▼
Phase R · 路由诊断 → 判断输入粒度 → 决定搜索宽度
  详见 [steps/step-r.md](steps/step-r.md)
  │
  ▼
PRD 推导 (step-prd.md v4) — agent 研究提案模式
  Step 1: 最小化爽点确认（≤3 个问题）
    Phase 1: 用户问答 → 锁定 2-3 个元爽点类别
    Phase 2a: 判断输入场景（R单原作 / M多原作 / O无参考）
    Phase 2b: agent 自问链 → 7类常见+元模板，产出 L1-L4/L5 特征参数 ⭐
    Phase 3: 产出特征参数表 + 搜索目标列表
  Step 2: 联网搜索（关键词从特征参数而非类型模板提取）→ 生成 2-3 个故事概念选项
  Step 3: 用户选择 → 锁定概念 + 推导核心卖点(5分类≥6条) + 用户画像
  Step 4: 从基因推导全书色彩 + 命名体系 + 视觉锚点场景
  Step 5: 推导世界宪法（4-6条不可违反规则，附违反后果）
  Step 6: 跨域素材指引（方向性，非详细方案）
  Step 7: 篇幅规划
  Step 8: 合成 PRD（宪法结构：核心故事→爽点→核心卖点(5分类)→基因+命名+视觉锚点→画像→宪法→素材指引→篇幅→承诺→DNA+边界）
  Step 9: 验证 → 写入
  详见 [steps/step-prd.md](steps/step-prd.md)
  │
  ▼
PRD.md → 交付 world
```

## 红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **Phase R 不可跳过** |
| ❌2 | **PRD 不可跳过** |
| ❌3 | **不预设具体名字**——PRD 定义设计空间（源域+转译规则+方向），不写「太平经」「他化自在天」 |
| ❌4 | **不把拆书复盘当开书推演**——PRD 是开书第一天的文档，不是 37 章读完后的 catalog |
| ❌5 | **Phase 2 自问链不可跳过**——拿到元爽点标签不准停，必须向下追问 L1-L4 特征参数。无参数的爽点 = 没打磨的设计 |
| ❌6 | **不强行归类**——如果用户的元爽点不匹配 7 类常见模板，走元模板生成自定义追问链。把独特爽点塞进最接近的类别里 = 信息丢失 |
| ❌7 | **PRD 不承载 L1/plot 细节**——不写具体角色名/地名/属性数值/分幕划分/转译规则。写完每条问自己：「如果这条不写，下游设计师还有发挥空间吗？」如有，抽象层级对了 |
