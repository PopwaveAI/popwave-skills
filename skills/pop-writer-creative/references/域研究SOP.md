# 域研究SOP · 浅层域研究操作标准

> **用途**：creative Step 2.1 执行浅层域研究，产出域DNA摘要供碰撞引擎使用。
>
> **与采风SOP的关系**：reservoir 的采风SOP执行深层域研究（采风级别），本SOP是其浅层版本。creative用浅层版（1-2句/维），reservoir用深层版（2-3句/维+经典元素库+素材清单）。
>
> **替代关系**：v5.0起替代搜索SOP.md的三路搜索（同类作品/历史原型/金手指）。搜索SOP.md保留为deprecated参考。

---

## 一、核心理念

### 1.1 域研究 vs 三路搜索

| | 三路搜索（旧） | 域研究（新） |
|:---|:---|:---|
| 搜什么 | 同类作品/历史原型/金手指 | 每个域的结构DNA |
| 怎么搜 | 关键词搜索+参数匹配 | 四维DNA提取+trope-library查询 |
| 产出 | 搜索沉淀表（作品列表） | 域DNA摘要（结构理解） |
| 问题 | 不理解域的结构，只搜表面参数 | 深入理解域怎么讲故事/怎么运转 |

### 1.2 浅层 vs 深层

- **浅层（creative用）**：足够提取域DNA四维摘要，支撑碰撞合成。每维1-2句+经典元素≥3个
- **深层（reservoir用）**：采风级别，读原著/拆结构/化用功法。每维2-3句+经典元素库表+可提取素材清单

## 二、域识别

### 2.1 从用户输入识别域

| 场景 | 信号 | 域识别策略 | 示例 |
|:-----|:-----|:----------|:-----|
| R（单原作） | "XX书的东方版" | 主域=原作所属域，辅域=目标赛道域 | "深渊主宰的东方版"→主域=DND框架域，辅域=东方武侠域 |
| M（多原作） | "A的XX + B的XX + C的XX" | 每个原作对应一个域 | "武侠×DND×圣者浩劫"→3个域 |
| O（无参考） | "我想写一本黑暗修仙文" | 从类型+基调反推2-3个参考域 | "黑暗修仙"→仙侠修真域+克苏鲁域+网文升级流域 |

### 2.2 域识别清单格式

```
## 域识别清单
- 场景类型：{R/M/O}
- 域1：{域名}（{主域/辅域}）— 识别依据：{一句话}
- 域2：{域名}（{主域/辅域}）— 识别依据：{一句话}
- 域3（如有）：{域名}（{主域/辅域}）— 识别依据：{一句话}
```

## 三、域DNA浅层提取协议

### 3.1 四维提取方法

| 维度 | 提取方法 | 搜索关键词模板 | 最低产出 |
|:-----|:---------|:-------------|:---------|
| 叙事语法 | 搜"{域名} 叙事结构 节奏 爽点" + 读trope-library套路库 | "{域名} 叙事节奏 章末钩子 升级" | 1-2句叙事语法摘要 |
| 核心机制 | 搜"{域名} 力量体系 规则 设定" + 读trope-library设定库 | "{域名} 力量体系 规则 设定" | 1-2句核心机制摘要 |
| 美学基因 | 搜"{域名} 风格 美学 视觉" | "{域名} 风格 美学 视觉" | 1句美学基因摘要 |
| 经典元素库 | 搜"{域名} 代表作 经典桥段" | "{域名} 代表作 经典 元素" | 3-5个经典元素 |

### 3.2 搜索策略

- 每个域至少 **2次搜索**（1次搜结构/机制，1次搜经典元素）
- 搜索关键词从域的特征参数提取，不从类型模板硬编码
- trope-library查询优先（设定库/套路库/立项库），联网搜索补充

### 3.3 与trope-library的对接

| 模块 | 查什么 | 怎么查 |
|:-----|:-------|:-------|
| 立项库/ | 同类域焊接经验 | 读00-索引→匹配域组合 |
| 设定库/ | 域的书级设定包 | 读00-索引→匹配书目→读L1文件 |
| 套路库/ | 域的抽象模式卡 | 按调用匹配SOP三维查询 |
| 剧情库/ | 域的剧情线卡 | 按标签分目录查询 |

> 查询协议统一走 `references/调用匹配SOP.md` 三维查询（层×赛道×元爽点）。查询无匹配时标注"库缺"，降级到联网搜索补充，不静默跳过。

## 四、域DNA摘要格式

> 所有域提取完成后，统一沉淀为以下格式，供碰撞引擎 Step 2 消费。

```markdown
## 域DNA摘要

### 域1：{域名}
- 叙事语法：{1-2句——这个域怎么讲故事}
- 核心机制：{1-2句——这个域怎么运转}
- 美学基因：{1句——视觉/氛围调性}
- 经典元素库：{3-5个标志性元素，逗号分隔}
- trope-library匹配：{命中模块+文件名 / 标注"库缺"}

### 域2：{域名}
- 叙事语法：{...}
- 核心机制：{...}
- 美学基因：{...}
- 经典元素库：{...}
- trope-library匹配：{...}

### 域3（如有）：{...}
```

## 五、迭代策略

> 第一轮提取不达标时，按以下策略补充。

| 不达标情况 | 补充策略 |
|:-----------|:---------|
| 某维DNA只有废话（如"有升级体系"） | 换关键词重搜，加"具体机制/代价规则/节奏控制"等限定词 |
| 经典元素<3个 | 扩大搜索范围，从"代表作"扩展到"经典桥段/名场面/标志性设定" |
| trope-library无匹配 | 标注"库缺"，纯联网搜索补齐四维 |

**最大迭代轮次**：2轮。第3轮仍不达标 → 标注"域研究不足"，继续合成（不阻塞流程）。

## 六、质量自检

域DNA摘要完成后，交付碰撞引擎前，逐条自检：

| 检查项 | 标准 |
|:-------|:-----|
| 域识别完整 | 每个识别到的域都有四维DNA摘要？ |
| 叙事语法有结构 | 不是"有升级体系"这种废话，而是具体的节奏/爽点释放模式？ |
| 核心机制有规则 | 有明确的力量体系/代价结构/成长逻辑，不是泛泛而谈？ |
| 经典元素≥3个 | 每个域的经典元素库至少3个标志性元素？ |
| trope-library已查 | 每个域都查了trope-library，命中或标注"库缺"？ |
| 摘要格式规范 | 按第四节格式填写完整，供碰撞引擎直接消费？ |

---

## 七、与碰撞引擎的对接

域DNA摘要完成后，直接传递给碰撞引擎（`references/碰撞引擎.md`）的 Step 2（域DNA提取）：

| 对接点 | 域研究SOP产出 | 碰撞引擎消费 |
|:-------|:-------------|:------------|
| Step 2 域DNA提取 | 四维DNA摘要（每维1-2句） | 碰撞引擎直接引用，不再重复提取 |
| Step 3 焊接点发现 | 经典元素库（3-5个/域） | 从元素间共享语义发现焊接点 |
| Step 4 叙事语法判定 | 叙事语法维摘要 | 评估各域的叙事结构成熟度/连载适配度/爽点释放能力 |
| Step 5 跨域联动设计 | 核心机制+经典元素 | 设计元素借用/结构移植/意象蒸馏/功能反转 |

> 域研究SOP的产出是碰撞引擎的**唯一域知识输入**。域DNA摘要的质量直接决定碰撞合成的上限——垃圾进则垃圾出。
