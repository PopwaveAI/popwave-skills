# 管线 PRD 格式约定

> 适用于 `prd/01-管线架构/` 下的全链路依赖图文档。

## 文档结构

管线 PRD 只保留三块，不做闸门/路径速查/架构决策：

1. **管线顺序图**（ASCII art）
2. **一、管线文件接口**（四环节表格，含辅助 skill）
3. **附录 A：项目文件全貌**（目录树）

## 接口表格格式

四环节分表，每表列：`Skill | 入 | 出 | S/D | → | 用途`

```
### 创意环节

| Skill | 入 | 出 | S/D | → | 用途 |
|:------|:----|:----|:---:|:---|:-----|
| creative v4.2.0 | 用户 + library + WebSearch | `全书立项PRD.md` | S | reservoir, world, character, plot | 唯一立项宪法 |
```

- `入` = 该技能/batch 启动前必须存在的文件
- `出` = 落盘文件，用反引号包裹路径
- `S/D` = S 静态（一次产出）/ D 动态（持续维护）
- `→` = 下游消费者 skill 名
- `用途` = 一句话说明产出物用途

## 多批次技能

分阶段产出的技能（world、plot），拆为多行，每行独立入/出/→：

```
| world ① L1设定 | PRD + 素材储备池 + library | `L1-01~06.md` | S | character, plot | 世界蓝图 |
| world ② 数值+升级表 | PRD宪法 + L1力量 | `数值体系/*.md` `动态升级表.md` | S | character, plot | 战斗数值 |
| world ③ 起终点快照 | PRD + L1 + 数值体系 | `起点快照.md` `终点快照.md` | S | character, plot | 首末卷状态 |
```

禁止混批——不同批次的产出必须各自独立一行。

## 文件命名

- 产出文件名直接写，如 `全书立项PRD.md`，不带目录前缀
- 目录级产出用尾斜杠，如 `素材储备池/`、`数值体系/`
- `素材储备池/` 每素材独立 `.md`，不合并为单文件

## 死引用清理

- 已删除的产出（故事引擎.md、爽点引擎.md、世界宪法.md 等）必须移除
- 拆书 T1/T2/T3 → 统一改为 `library 注入`
- 创意种子/ 目录已废弃，全书立项PRD.md 落项目根目录
