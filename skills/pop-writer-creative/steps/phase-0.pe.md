# Phase 0：故事引擎设计 — 元素融合 SOP

> 该文件同时作为执行指令和参考指南，无需加载外部 ref 文件。

## 前置条件
用户已提供至少：一个故事想象（场景/概念/世界观/角色片段，不拘）。

**★ 已有对标书拆解产出？**
```
□ _参考书/{书名}/ 下是否存在 Lv1 拆解产出？
  → 是：先读取对标书的 story-engine ——提取其碰撞元素和碰撞方法
         禁止凭记忆写原著事实
  → 否：跳过，Agent 从零联想碰撞元素
```

## 核心原则

**这个 skill 的全部产出服务于人——和对齐人的需求。** 不是给机器看的配置文件，是给作者看的创意提案。

```
碎片 → 联想 → 碰撞评级 → 轻量方向 sketches → 用户选择 → 深化为故事引擎 → 锁定
```

- 方向 sketch 要轻——够区分就行，不需要三层都展开
- 选定后才深化——只深挖用户选的那一个方向
- 最终产出是故事引擎.md——给人读的叙事文档，不是 yaml

---

## 执行步骤

### Step 1 — 碎片解吸 + 元素联想

从用户碎片中解吸"不可妥协元素"。用户可能只给了 1 个元素，Agent 必须联想出 3-5 个候选碰撞元素。

```
用户："我想写修仙"
  → 不可妥协：力量体系（境界/突破/天劫）、飞升（修炼的终极目标）、逆天而行（对抗规则）

知识源（按优先级）：
  1. 拆书 Lv1 产出 → 对标书的碰撞元素和碰撞方法（可直接复用或逆向）
  2. 跨领域联想表 → 从不可妥协规则出发，联想会产生碰撞的领域
     修仙 → 克苏鲁（飞升终点≠天堂）/ 科技（灵气≠能量）/ 考古（过去的仙≠死去）/ 
            监狱（关着的≠囚犯）/ 大逃杀（成仙≠永生）/ 社畜（修仙≠自由）
  3. Phase 0.6 跨域素材 → 预筛选版本

生成 5-8 个候选组合，进入 Step 2
```

### Step 2 — 拆出不可妥协的底层规则

对每个被选中的元素，写出它最核心的、去掉就不再是它自己的那条规则。

```
克苏鲁：不可名状 / 认知即毁灭 / 人类在宇宙中微不足道
公务员：编制为王 / 考评决定命运 / 服从系统而非挑战系统
监狱：被关在最深处的东西，永远比监狱外面的东西更危险
```

### Step 3 — 找碰撞点 + 评级

两套（或多套）规则在哪一个具体情境下不可调和。

```
碰撞强度评级：

★★★ 强碰撞 — A 的存在否定 B 的存在基础 → 优先展开
      例：修真 × 克苏鲁 — 飞升不是去天堂，是变成旧日支配者
      例：修真 × 监狱 — "镇妖塔"压着的不是妖，是上一代飞升成功的人

★★☆ 中等   — A 和 B 互相掣肘但能找到平衡 → 展开一个方向
      例：修真 × 公务员 — 编制体系 vs 逆天之道的张力

★☆☆ 弱碰撞 — A 和 B 可以井水不犯河水 → 淘汰

取碰撞强度最高的 2-3 个组合进入 Step 4
```

### Step 4 — 落主角

谁最适合站在碰撞点上。

```
不是"一个强者"——是"一个必须同时遵守两套规则、但两套规则互相否定的普通人"。
```

### Step 5 — 生成轻量方向 sketches（~150字/个）★ 关键

> **不是完整 brief。是电梯演讲——够区分方向、不够写正文。用户选完才深化。**

每个 sketch 只含四件事：

```markdown
【方向 N · 标题】

碰撞：A × B — [碰撞点一句话]
钩子：[这个故事跟所有同类最大的不同，一句话]
第一画面：[读者翻开第一章看到的最关键的一个画面，2-3 句]
基调：像《XXX》的 [Y] 感觉
```

| 字段 | 粒度 | 为什么只要这么点 |
|:-----|:-----|:---------------|
| 碰撞 | 一句话 | 用户只需要判断"这个冲突我感兴趣吗" |
| 钩子 | 一句话 | 不需要展开卖点——选完再深挖 |
| 第一画面 | 2-3 句 | 够让脑中放电影就行——不是写正文 |
| 基调 | 一句话 + 对标 | 最快的感受锚定——比任何形容词都有效 |

**示例：**

```markdown
【方向 A · 飞升尽头】

碰撞：修真 × 克苏鲁 — 飞升的终点是变成旧日支配者的容器
钩子：修炼不是为了变强，是为了不要变强——每突破一个境界，离"不是自己"更近一步
第一画面：她站在飞升台上，看着师父的身体在金光中重组。重组后的东西穿着师父的脸，坐在云端对她笑。
基调：像《诡秘之主》的认知压迫感，但主角知道得比读者少——读者和主角一起发现恐怖

【方向 B · 镇妖塔】

碰撞：修真 × 监狱 — 镇妖塔下面压着的不是妖，是上一代飞升成功的人
钩子：他看守监狱二十年，直到在最底层看到了三十年前飞升的师父的名字
第一画面：金丹期的典狱长提着灯笼走下最后一层楼梯。干涸的血迹构成一个形状——他花了一整天才看懂那是什么。然后他吐了。
基调：像《我有一座恐怖屋》的日常×异常，但恐怖是慢性的——二十年才渗出来

【方向 C · 末位淘汰】

碰撞：修真 × 公务员 — 编制体系里，"逆天而行"是违纪行为
钩子：斗法满分、政论零分——他把考官打了，然后天庭下发了末位淘汰令：连续三次不过的飞升者，遣返人间重练
第一画面：成绩单砸在他脸上的时候，隔壁考场的术法部已经把整栋楼炸烂了。斗法第一名。面试的时候他把主考官从椅子上提起来。政论：零分。
基调：像《我师兄实在太稳健了》的修真社畜幽默 + 《琅琊榜》的体制绞杀感
```

### Step 6 — 用户选择

```
展示 2-3 个轻量 sketches → 用户选择
  ├─ "方向 A！就是它" → 进入 Step 7，只深化 A
  ├─ "B 的核心对，但想要 A 的克系元素" → 融合 A+B 的碰撞点 → 生成合并 sketch → 进入 Step 7
  ├─ "都不对，但我有另一个想法" → 回到 Step 1，用新元素重新联想
  └─ "A 和 C 都想看看" → 两个都深化（最多 2 个），对比后选
```

### Step 7 — 深化为故事引擎.md ★ 只做这一个方向

用户选定后，把 sketch 展开为完整的故事引擎文档。**这是 human-first 的叙事文档，不是 yaml 配置文件。**

产出：`创意种子/故事引擎.md`

## 故事引擎.md 模板

> 给人读的。叙事优先。给 world 用的结构化信息也放在里面，但格式是可读的。

```markdown
# 故事引擎 — 《书名暂定》

> 确认于 YYYY-MM-DD | 样品签字：v{N}

---

## 一、这本书是什么

### 一句话
[核心假说——一句能让人脑中放电影的话]

### 碰撞
**元素**：A × B × C
**碰撞点**：[两套（或多套）规则在哪里互斥]

### 世界锚点
- 时代：[什么纪元/时期]
- 地域：[什么位面/区域]
- 力量层级：凡人 → 超凡 → 破格 → 神话（故事主要落在 [X]）
- 跟现实/最常见设定最大的 1 个不同：[一句话]

### 主角
[一段话，含：他是谁、他站在碰撞点的哪一侧、他要什么、他怕什么]

---

## 二、读者为什么追

### 核心爽点
[不是"升级爽"——是具体的情感命题。一段话]

### 对标书差异
像《XXX》的 [Y] 部分，但 [Z] 完全不同。

### 读者画像
- 年龄段：[]
- 阅读场景：[睡前/通勤/摸鱼]
- 爽点偏好：[]
- 弃书阈值：[]

### 前三章承诺
[读者看完前三章，应该期待什么——不能含糊]

---

## 三、绝对不能做的事（宪法）

### 基调边界
- 可以做：[]
- 绝不能做：[]

### 主角边界
- 绝不会做：[]
- 一定会做：[]

### 世界规则底线
- []

### 叙事底线
- [视角/节奏/每 N 章至少一次什么]

---

## 四、第一页

### 开篇画面
[2-3 句，具体到画面。和方向 sketch 里的"第一画面"一致或深化]

### 签名画面
[如果这本书只让人记住一个画面，是什么]

---

## 五、结构预期

| 卷 | 开始 → 结束 | 核心命题 |
|:---|:-----------|:--------|
| 卷1 | [状态] → [状态] | [本卷追问] |
| 卷2 | [状态] → [状态] | [本卷追问] |
| 卷3 | [关键转折] | [本卷追问] |
| 卷4 | [终局选择] | [本卷追问] |
```

**填写原则：**

- 每个字段都应该是**一句话或一段话**——不是列表、不是关键词
- 宪法部分的"绝不能做"要具体到场景——"绝不能无理由杀戮"比"基调不能太黑暗"有效
- 结构预期是方向性的——不要求每卷精确，但要求每卷的起点、终点、核心追问能被人一句话复述
- **整个文档应该能被人从头读到尾，像读一份创意提案一样。十分钟能读完。**
