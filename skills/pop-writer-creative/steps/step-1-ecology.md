# Step 1：生态图谱调研

> 管线: pop-writer-creative v2.0.0
> 消费: 对标书拆解数据 / 用户素材 / 网络调研
> 产出: 生态图谱文件

## 读什么
- pop-decon 产出的对标书拆解数据（角色/势力/冲突）
- 用户提供的素材（人物设想/世界片段/灵感笔记）
- 如有网络调研需求：WebSearch 补充赛道生态参考

## 做什么

### 1. 角色群识别
从拆解数据和用户素材中提取≥5个角色群。角色群 = 共享生存方式 + 利益关联的群体集合。

| 步骤 | 动作 | 产出 |
|:-----|:-----|:-----|
| 1a | 扫描拆解角色表，按生存方式聚类 | 群组初分 |
| 1b | 补充用户素材中的群体 | 群组合并 |
| 1c | WebSearch 补充同赛道生态参考（如需要） | 外部参照 |

### 2. 每群5字段填写
对每个角色群填写以下5个字段：

| 字段 | 内容 | 说明 |
|:-----|:-----|:-----|
| 名称 | 群体代号 | 简洁可识别 |
| 生存方式 | 靠什么活 | 经济基础/资源获取 |
| 内部结构 | 怎么组织 | 层级/分工/权力分配 |
| 与其他群关系 | 谁是对手/盟友 | 至少标注与2个群的关系 |
| 可衍生冲突空间 | 能打什么仗 | ≥2个冲突方向 |

### 3. 生态与剧情交叉点标注
在生态图谱末尾标注≥3个跨群冲突交叉点，每个交叉点说明涉及哪些群、冲突性质、可衍生的剧情线方向。

## 门禁
| 检查项 | 失败动作 |
|:-------|:---------|
| 生态图谱≥3角色群（红线❌3） | 补充调研至≥3群再继续 |
| 每群5字段完整 | 缺字段补全后再进下一步 |
| ≥3个跨群冲突交叉点 | 补充冲突标注 |

## 产出
`生态图谱.md` — 含角色群清单 + 每群5字段 + 跨群冲突交叉点。使用 `templates/生态图谱-模板.md`。
