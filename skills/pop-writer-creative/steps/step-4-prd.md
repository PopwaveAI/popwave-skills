# Step 4：立项PRD产出

> 管线: pop-writer-creative v2.0.0
> 消费: 生态图谱 + 金手指行动引擎 + 文风DNA锁定声明
> 产出: 立项PRD.md

## 读什么
- Step 1 产出的 `生态图谱.md`
- Step 2 产出的 `金手指行动引擎.md`
- Step 3 产出的 `文风DNA锁定声明.md`

## 做什么

### 1. PRD模块填写
产出创意决策版PRD，包含以下模块：

| 模块 | 内容要求 | 决策导向 |
|:-----|:---------|:---------|
| 核心卖点 | 1-3句话说明本书最大卖点 | 写"卖什么"不写"怎么卖" |
| 目标读者 | 读者画像+阅读偏好 | 写"给谁看"不写"怎么吸引" |
| 对标分析 | 对标书+差异化点 | 写"像什么但不同在哪" |
| 生态摘要 | 生态图谱核心群+核心冲突 | 写"谁和谁打"不写"怎么打" |
| 金手指摘要 | 行动引擎核心机制+数值风格 | 写"驱动什么"不写"数值多少" |
| 文风DNA摘要 | 锁定的5维度关键参数 | 写"什么风格"不写"怎么写" |

### 2. 创意决策 vs 施工图边界
PRD每一条必须通过以下检查：

| 允许写（创意决策） | 禁止写（施工图） |
|:------------------|:----------------|
| 世界名/主角名/金手指名 | 属性数值/冷却时间/分幕划分 |
| 核心矛盾方向 | 章节级事件安排 |
| 卖点定位 | 节奏曲线/爽点分布 |
| 生态群格局 | 势力数值对比表 |

### 3. 交接声明
PRD末尾标注交接清单：
- 交付给 pop-writer-world：PRD + 生态图谱 + 金手指行动引擎
- 交付给 pop-writer-character：PRD + 生态图谱
- 交付给 pop-writer-plot：PRD + 金手指行动引擎

## 门禁
| 检查项 | 失败动作 |
|:-------|:---------|
| PRD写创意决策不写施工图（红线❌4） | 删除施工细节，改写为决策导向 |
| 6个模块全部填写 | 补全缺失模块 |
| 交接清单完整 | 补充交接声明 |

## 产出
`立项PRD.md` — 创意决策版PRD，含6模块 + 交接声明。使用 `templates/立项PRD-模板.md`。
