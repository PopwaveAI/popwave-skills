# Phase Delta：新元素注入诊断（独立调起模式）

> **定位**：项目运行中注入新元素的轻量级流程。不重跑全管线，只做 PRD 契合度评估 + 宪法冲突检测 + 素材储备池追加。
>
> **触发条件**：用户说"帮我把这个XX融进书里" / "最近有个XX热点能写进我的书吗" / "我想到一个新设定" —— 且项目目录下能找到已有产出。

---

## 前置检查：是否是已运行项目

Phase Delta 执行前必须验证项目状态。读取项目目录下的文件：

| 文件 | 必须？ | 说明 |
|:-----|:------:|:------|
| `创意种子/PRD.md` | ✅ 必须 | 没有 PRD = 没有加工坐标系 |
| `创意种子/故事引擎.md` | ✅ 必须 | 没有宪法 = 找不到冲突检测基线 |
| `储备剧情池/素材储备池.md` | ✅ 必须 | 必须知道现有池子里有什么 |
| `小说世界设定/世界宪法.md` | ✅ 必须 | 已锁定的世界边界（world 产出） |

> **规则**：以上四个文件**全部缺失**或项目目录未初始化 → 这不是已运行项目 → 退回路管入口模式（走 Phase R 诊断）。
>
> **部分缺失**：标注缺失项，降级执行——无法检测的维度（如无世界宪法则跳过宪法冲突检测）标注"因文件缺失跳过的检查项"。

---

## 步骤顺序

### Step 1：读取项目已有产出

读取以下文件的核心约束：

| 文件 | 读什么 | 
|:-----|:------|
| PRD.md | 一句话定位、加工哲学、边界 |
| 故事引擎.md | 宪法约束（绝对不能做的事、基调边界、主角边界、世界规则底线） |
| 世界宪法.md | 已锁定的世界参数约束（L1 维度约束） |
| 素材储备池.md | 现有池子分类和内容（避免重复注入） |

产出：`_注入诊断/已有产出摘要.md`（摘要形式，不超过 500 字）

### Step 2：PRD 契合度评估

评估新元素与 PRD 加工坐标系的匹配度。三个子问题：

| 问题 | 评估方式 |
|:-----|:---------|
| ① 新元素的**核心冲突**与 PRD 的核心承诺是否一致？ | 如果 PRD 说"高强度数值博弈"，新元素是"命运随机决定一切" → 不兼容 |
| ② 新元素的**加工难度**有多大？（需要改多少世界设定才能放进去） | 容易：直接设定替换 / 中等：需新增设定层 / 困难：需重构现有结构 |
| ③ 新元素如果放进去，**替换了什么**？（不挤占已有的结构就安全） | 参考同人二创工作流的"替换点确认"方法 |

**契合度等级**：

| 等级 | 定义 | 下一步 |
|:-----|:-----|:-------|
| ✅ **兼容** | 新元素与 PRD 一致，不冲突，且加工难度低 | 走 Phase D1 |
| ⚠️ **微调可兼容** | 新元素有价值但与 PRD 有张力，需小幅修订 PRD 或宪法 | 走 Phase D2 |
| ❌ **冲突** | 新元素与 PRD 的核心承诺背道而驰，或无法加工 | 走 Phase D3 |

### Step 3：宪法冲突检测

读取故事引擎.md 和世界宪法.md，逐条检查新元素是否违反任何已锁定的约束：

| 检查项 | 对比宪法条目 |
|:-------|:------------|
| 是否会改变主角的不可动摇特征？ | 故事引擎 → 主角边界 |
| 是否会改变故事的基调边界？ | 故事引擎 → 基调边界 |
| 是否会引入被宪法禁止的元素？ | 故事引擎 → 绝对不能做的事 |
| 是否会与已锁定的 L1 设定矛盾？ | 世界宪法 → 逐维约束 |
| 是否会打破数值体系的平衡？ | 世界宪法 → 数值约束 |

### Step 4：等级判定与执行

#### Phase D1：直接注入

新元素兼容 PRD，无宪法冲突。

1. 确定新元素在素材储备池中的归档类别（主线/支线/背景/剩余焊接点）
2. 判断推荐的触发场景（哪个卷/幕/剧情点可以使用）
3. 写入素材储备池末尾

**注入格式**：

```
---

## Phase Delta 注入 — {日期}

**注入来源**：[用户自述 / 热点事件 / IP新增 / 其他灵感]
**元素描述**：[1-2 句话]
**PRD 契合度**：✅ 兼容
**宪法冲突**：无
**归档类别**：[主线/支线/背景/剩余焊接点]
**建议触发场景**：[如果 plot 要使用这个素材，建议在XX阶段触发]
**加工预判**：[如果这个素材进入正文，加工方式应该是——按 PRD 加工哲学写 1-2 句话]
```

#### Phase D2：微调后注入

新元素有价值，但与 PRD 或宪法有张力。不擅自修订——输出微调方案让用户决策。

1. 写出具体的修订方案：
   - 需要修订 PRD 的哪一条
   - 需要修订宪法约束的哪一条
   - 修订后的版本是什么样子
   - 修订的影响范围（哪些下游产出需要同步更新）
2. 输出 `_注入诊断/微调方案.md`
3. 用户确认后执行：
   - ✅ 用户同意修订 → 更新 PRD.md/宪法 → 写入素材储备池
   - ❌ 用户拒绝修订 → 写入剩余焊接点，标注"PRD不符，用户选择不微调"

#### Phase D3：冲突报告

新元素与 PRD 或宪法冲突，无法兼容。

1. 输出冲突报告 `_注入诊断/冲突报告.md`，含：
   - 冲突的具体条目（引文 + 冲突原因）
   - 如果要强行加入需要改多少东西
   - 建议：弃用、或降低为背景素材、或等到下一本书再用
2. 如果用户坚持要加 → 通知用户需要走"修宪回路"（退回管线入口重跑 world）

### Step 5：交接

Phase Delta 完成后，确保：

- ✅ 素材储备池已更新（含新元素条目）
- ✅ 如涉及 PRD/宪法微调 → 文件已更新 + 通知 world 修正对应维度
- ✅ 输出注入摘要告知用户结果

## 典型注入场景与分类建议

| 场景 | 典型 PRD 契合度 | 建议归档 | 说明 |
|:-----|:--------------:|:---------|:-----|
| 社会热点事件（如某地大罢工→改写成书里一场政治冲突） | ⚠️ 微调（需适配力量体系设定） | 支线索材 | 核心冲突有价值，但需要魔幻化/奇幻化改写 |
| 热门 IP 元素（如看了一部新电影，想加类似的能力体系） | ✅ 兼容（如果与现有力量体系平行） | 背景素材或剩余焊接点 | 不冲撞主线，适合做支线或伏笔 |
| 历史冷知识（如某个朝代的税制改革） | ✅ 兼容（如果 PRD 说"历史真实感"） | 背景素材 | 适合嵌入世界设定层 |
| 完全相反的哲学设定（如命运论 vs 自由意志冲突） | ❌ 冲突（如果 PRD 已锁定一边） | 剩余焊接点 | 有大张力但可能不适合这本书，标注留给下一本 |
| 新角色设定（突然想到一个反派） | ⚠️ 微调（需检查角色 vs 现有势力格局） | 支线索材或背景素材 | 根据角色机能检查是否与已有角色池重叠 |

## 质量红线

| # | 红线 |
|:-:|:-----|
| ❌D1 | **不读项目已有产出就评估** — 必须找到 PRD/故事引擎/世界宪法/素材储备池 |
| ❌D2 | **不擅自修订宪法** — 微调方案必须用户签字 |
| ❌D3 | **冲突时不沉默吞入** — 检出冲突不报告 = 埋雷 |
| ❌D4 | **注入不写储备池** — 评估完就结束，信息丢失 |
| ❌D5 | **重复注入同一元素** — 检查素材储备池现有条目，避免同一元素注入两次 |
