# Step PRD v4：立项宪法推导 — agent 研究提案，用户选择

> **定位**：PRD = 这本书的**唯一立项文档**。爽点引擎的 Q&A、故事引擎的宪法约束，全部合入 PRD。
> **核心变化 v4（2026-06-20）**：
> - **agent 做研究，用户做选择。** 用户只有模糊想法——agent 通过联网搜索+公共数据库生成 2-3 个有具体对标支撑的故事概念选项，用户选。不再逐条追问「你的世界什么样」。
> - 推导链：**最小化爽点确认 → 联网研究生成故事概念选项 → 用户选择锁定 → 从选定概念推导全书色彩+命名体系 → 推导本书特性+用户画像+世界宪法+跨域素材指引 → 合成宪法**。
> **前置依赖**：无。所有推导从用户一句模糊想法 + 联网搜索开始。
> **产出**：`创意种子/PRD.md`（唯一的立项宪法）

---

## 推导流程总览

```
用户一句模糊想法（「我要写一本黑暗流东方仙侠，底层砸碎世界」）
    │
    ├─ Step 1: 最小化爽点确认（≤3 个问题）
    │      产出：2-3 主元爽点（含特征参数）+ 一句话核心冲动
    │
    ├─ Step 2: 联网搜索 + 公共数据库 → 故事概念生成  ★ 核心步骤
    │      产出：2-3 个故事概念选项（每项含世界观结构/核心冲突/金手指/对标参考）
    │
    ├─ Step 3: 用户选择 → 锁定概念 + 推导本书特性(≥5条) + 用户画像
    │      产出：锁定概念 + 5条独特卖点 + 目标读者画像
    │
    ├─ Step 4: 从基因推导全书色彩 + 命名体系
    │      产出：世界质感 + 力量美学 + 情感光谱 + 命名体系基调
    │
    ├─ Step 5: 推导世界宪法（4-6条不可违反的规则）
    │
    ├─ Step 6: 跨域素材指引（找什么方向参考，不是详细方案）
    │
    ├─ Step 7: 篇幅规划
    │
    ├─ Step 8: 合成 PRD + 核心承诺回引爽点
    │
    └─ Step 9: 验证 → 写入
```

## 三种撰写模式 + 一种微调模式

### 模式 A：用户有模糊方向（最常见）★

用户说「我想写一本 XXX 的书」——有方向但不清晰。

**流程**：Step 1-9 全走。agent 做研究提方案，用户选。

**产出特征**：PRD 语气是**断言式**——「基于你的方向和我们的研究，这本书是 X，不是 Y。决策已锁定。」

### 模式 B：用户有碎片素材（「我喜欢 A/B/C，怎么揉」）

用户有积累的素材但不知道怎么组合。

**流程**：
1. 先把素材铺开，搜索补充相关领域
2. 找共同脉络，形成 2-3 个候选 PRD 方向
3. 用户选一个 → Step 3-9 走完

**产出特征**：PRD 语气是**归纳式**——「从这些素材中提炼的共同方向是 X。」

### 模式 C：用户完全空白（「帮我想个故事」）

**流程**：
1. 搜索当前热门赛道 + 用户阅读偏好 → 提出几个方向选项
2. 用户选 → 碰撞深化 → Step 1-9 走完

**产出特征**：PRD 语气是**提炼式**——「从你选的方向，提炼出这本书的基本法。」

### 模式 D：PRD 微调（Phase Delta）

Phase Delta 判定新元素为「微调可兼容（⚠️）」→ 修订 PRD 的某一两条。

**流程**：确认冲突条目 → 输出修订方案 → 用户签字 → 更新 PRD.md

## 对话推导流程（Step 1-9）

### Step 1：最小化爽点确认（≤3 个问题）

> **不要长篇 Q&A。** 用户只有模糊想法，问 2-3 个关键问题锁定爽点方向即可。更多信息从搜索中补。

#### Phase 1 — 用户问答（从以下选 2-3 个，不超 3 个）

1. *「读者为什么凌晨三点还在点下一章？——是智力优越感（我知道主角知道但反派不知道）、阶级反抗的快感（掀翻压迫）、还是某种独特的情绪体验？」*
2. *「有没有一本书让你特别上瘾——不是因为文笔，是因为某个特定套路让你停不下来？」*
3. *「如果只用一个核心冲动描述这本书——『一个 X 的人，在 Y 的世界里，做 Z 的事』——X/Y/Z 各是什么？」*

#### Phase 2 — agent 自问链 ⭐（不打扰用户）

> **关键行为变更**：agent 拿到 ≤3 个元爽点类别后，不准停在这里。必须继续向下追问至少 3 层，产出该类别在当前项目中**具体怎么运作**的特征参数。这是消除「分类代替分析」的门禁——拿到标签只是起点，不是终点。
>
> 自问链基于已掌握的答案推导。如果某个参数无法从已有信息中确定，**不猜**——标记为 Step 2 搜索目标。

##### 2a — 判断输入场景

Phase 2 开始前，先判定当前输入属于哪种场景，这决定追问链的推导策略：

| 场景 | 信号 | 策略 |
|:-----|:-----|:-----|
| **R（单原作）** | 用户明确说「XX书的东方版」/「类似XX的小说」 | 拆原作的具体机制作为起点，翻译决策驱动 |
| **M（多原作）** | 用户说「我想要A的XX + B的XX + C的XX」 | 找共同基因 + 各自特色，融合方案驱动 |
| **O（无参考）** | 用户只说「我想写一本黑暗修仙文」/ 一个模糊概念 | 不从任何具体书出发，从类型+基调+爽点本身推导 |

> ⚠️ 场景 R 和 M 的「拆原作」只在特征参数层面拆（如「深渊主宰的背刺公式=信息差 L1=数值可视化 L2=实时 L3=免费但需要条件触发 L4=战斗+权谋」），**不在剧情层面拆**（不要抄具体情节、角色名、事件顺序）。PRD 红线 4 依然生效。

##### 2b — 自问链（7 类常见 + 元模板）

以下 7 条追问链覆盖最常见的元爽点。**但如果用户识别出的元爽点不在这 7 类之中，不要强行归类**——使用末尾的「元模板」生成自定义追问链。

对每个已识别的元爽点，agent 按对应模板自问。不匹配 7 类的，走元模板：

**① 信息差类追问链：**

```
L1 感官/媒介：信息差通过什么渠道传递？
   （看见面板数值 / 推演未来 / 感知破绽 / 听见心声 / 特殊联系 / 其他：____）
L2 时效性：信息差是永久能力 / 限时激活 / 一次性道具？
L3 代价：使用信息差要付出什么？
   （免费 / 消耗资源 / 有反噬风险 / 越级使用会暴露自己）
L4 叙事层：信息差在哪个层面发挥作用？
   （战斗层面→看穿破绽 / 权谋层面→预判博弈 / 世界观层面→知道隐藏真相）
```

**② 阶级革命类追问链：**

```
L1 压迫主体：压迫来自制度性的结构 / 个体上位者的恶意 / 两者混合？
L2 反抗方式：主角要推翻制度 / 取代上位者 / 逃离系统 / 还是从根本上颠覆游戏规则？
L3 压迫的具体体现：资源垄断 / 人身控制 / 知识封锁 / 身份歧视 / 暴力威慑？
   （至少写 2 个具体维度——这些就是中期冲突的原材料）
L4 革命路径阶段：个人反抗 → 组织底层 → 动摇制度 → 重新定义规则
   （当前定位在第几阶段？每阶段切换的标志是什么？）
```

**③ 绝境反杀类追问链：**

```
L1 绝境制造者：谁把主角逼到绝境？（环境 / 具体敌人 / 自身选择 / 系统规则）
L2 反杀代价：反杀需要付出什么？
   （无代价爽杀 / 有代价险胜 / 不可逆损失——影响读者预期）
L3 绝境频率：绝境的节奏是？（每章小险情 / 每幕大绝境 / 卷级生死关）
L4 读者情绪：绝境时读者的预期是什么？
   （紧张未知 / 悲壮 / 笃定主角会赢 / 希望渺茫）
```

**④ 身份跃迁类追问链：**

```
L1 跃迁方向：底层→上层 / 边缘→中心 / 隐匿→显赫 / 被否定→被认可？
L2 驱动力：跃迁靠能力展示 / 身份揭示 / 外部赋予 / 还是自我构建？
L3 节奏：匀速上升 / 突进+平台期 / 每步跃迁伴随什么失去？
L4 社会代价：跃迁后面对什么？（被嫉妒 / 被追杀 / 失去旧关系 / 被迫出卖原则）
```

**⑤ 世界观震撼类追问链：**

```
L1 震撼类型：力量阶梯刷新 / 历史真相揭露 / 宇宙规则揭秘 / 社会结构反转？
L2 节奏：每卷一次天花板刷新 / 持续碎片拼图 / 一次性大反转？
L3 主角应对：适应新规则 / 利用新规则 / 还是打破新规则？
L4 情感基调：敬畏 / 恐惧 / 兴奋 / 绝望后希望？
```

**⑥ 智斗博弈类追问链：**

```
L1 参与方：双边对抗 / 三角制衡 / 多方混战？（具体几个势力？）
L2 信息分布：完全信息 / 不完全信息 / 存在误导信息？
L3 主角优势：计算能力 / 信息渠道 / 人情筹码 / 预知能力？
L4 赌注级别：个人生死 / 势力存亡 / 世界命运？
```

**⑦ 情感羁绊类追问链：**

```
L1 关系类型：兄妹 / 师徒 / 战友 / 恋人 / 亦敌亦友 / 族群归属？
L2 羁绊功能：羁绊在叙事中扮演什么角色？
   （情感锚点「不杀的理由」/ 力量源泉「为TA变强」/ 软肋「被利用」/ 身份定义「我是谁」）
L3 叙事密度：全书贯穿 / 阶段性出场 / 只在关键时刻激活？
L4 张力来源：付出不对等 / 认知差 / 身份对立 / 牺牲风险？
```

**⑧ 元模板（不匹配 7 类时使用）：**

当 agent 识别出的元爽点不属于以上 7 类，**不要强行归类**。使用元模板生成自定义追问链，填入实际的爽点名：

```
自定义追问链 · {爽点名}

L1 感官入口：这个爽点通过哪种读者体验触达？
   （认知层→信息处理 / 情绪层→情感波动 / 生理层→体感反应 / 混合）
L2 触发条件：什么条件成立时这个爽点会爆发？
   （需要铺垫多久？需要哪些前置要素？）
L3 节奏频率：它的爆发节奏是怎样的？
   （脉冲型→来一下就走 / 累积型→慢慢涨然后释放 / 持续性→全程都在）
L4 升级路径：随着剧情推进，这个爽点的表现形态如何演化？
   （从简陋到复杂？从低频到高频？从单一到复合？）
L5 排他性：这个爽点和哪些其他爽点不兼容？
   （例：极致的智力信息差和极致的暴力爽感经常冲突，因为前者需要读者思考，后者需要读者停止思考）
```

> ⚠️ **元模板存在的意义**：7 类追问链是「常见情况的手册」，不是「爽点的全部可能」。当用户提出不在这 7 类中的爽点，agent 必须用元模板自行生成追问链，而非把用户的爽点硬塞进最接近的类别里——后者会导致信息丢失。

#### Phase 3 — 产出+缺口标记

```
爽点方向：
- 主元爽点1: {名称}（{一句话心理机制}）
  特征参数: L1={} L2={} L3={} L4={}
- 主元爽点2: {名称}
  特征参数: L1={} L2={} L3={} L4={}
- 主元爽点3（如有）:
  特征参数: L1={} L2={} L3={} L4={}
一句话核心冲动：{X的人在Y的世界做Z的事}

搜索目标（待 Step 2 补充）: {Phase 2 中无法确定的参数列表}
```

> ⚠️ **底层逻辑**：特征参数表直接指导 Step 2 的搜索方向——不再泛搜类型模板，而是基于参数精准搜。例如：
> - 「信息差 L1=感知破绽 L2=实时 L3=有反噬风险 L4=战斗+权谋」→ 搜「通过感知破绽实现实时信息差且有代价的功法/系统设定」
> - 「阶级革命 L1=制度性压迫 L2=颠覆规则 L3=知识封锁+资源垄断 L4=个人→组织」→ 搜「知识垄断制度如何被底层颠覆的案例/历史原型」
> - 找不到答案的参数就是 Step 2 的搜索主攻方向

### Step 2：联网搜索 + 公共数据库 → 故事概念生成  ★

> **这是最核心的步骤。agent 用研究数据生成选项，不是让用户自己设计世界。**

#### 2.1 搜索方向（三路并行）

> **搜索关键词的来源变更**：不再基于类型硬编码关键词。而是从 Step 1 Phase 3 的「搜索目标」和「特征参数」中提取精准关键词。每个搜索方向至少包含 2 个特征参数作为过滤条件。
>
> 示例：如果 Step 1 产出「信息差 L1=感知破绽 L2=实时 L3=有反噬风险 L4=战斗+权谋」→ 搜索 A 的关键词应该是「感知破绽+代价 实时信息差 小说 战斗 权谋」，而不是「系统流修仙小说 数据面板」。

**搜索 A：同类作品的世界观结构**
- 搜索关键词：`{类型} 小说 世界观 结构 设定 亮点`
- 目标：找到 3-5 本同类成功作品，提取他们的世界观结构（单世界/双世界？力量体系怎么设计的？核心冲突是什么？）
- 记录每个作品的结构特征和优缺点

**搜索 B：历史/神话/社会结构素材**
- 从用户的爽点方向出发，搜索相关的历史事件、社会结构、神话体系
- 例子：如果爽点是「阶级革命」→ 搜索黄巾起义/太平道/法国大革命/奴隶起义的组织结构
- 例子：如果爽点是「信息差碾压」→ 搜索间谍战/冷战情报网/三十六计/博弈论经典案例
- 目标：找到 2-3 个可用的历史/社会结构原型，作为世界观设计的「骨架」

**搜索 C：金手指原型**
- 搜索与用户爽点匹配的金手指类型
- 例子：如果用户说「死亡换身份」→ 搜索死后重生/身份切换/轮回类金手指的成功作品
- 目标：提取 2-3 种可用的金手指结构，分析其优缺点

#### 2.2 合成故事概念选项（2-3 个）

> **关键行为变更**：每个选项的「世界观结构」「核心冲突模式」「金手指方向」必须从 Step 1 Phase 3 的**特征参数表**出发推导，然后再用搜索结果填充细节。禁止直接套用搜索到的通用模版贴标签。
>
> ⚠️ **通用模版 vs 参数推导的差异**（这是之前你被用户批评「没有关联性」的根因）：
> - ❌ 错误做法：搜到「系统流修仙小说模板」→ 套用「杂役+望气术+宗门内斗」→ 标签为「深渊主宰东方版」
> - ✅ 正确做法：从特征参数出发 → 「信息差 L1=感知破绽 L2=实时 L3=有代价 L4=战斗+权谋」→ 推导「什么修仙能力能同时满足 L1-L4？→ 望气术？天机推演？杀意感知？→ 选哪个牺牲最小保留最多？」→ 搜索结果只是用来验证和填充细节
>
> ⚠️ **第三步合成策略取决于 Step 1 Phase 2 判定的场景**：
> - **场景 R（单原作）**：在特征参数表中标注哪些参数是从原作继承的、哪些是新创造的。对每个继承参数问：「这个机制翻译到东方语境下，保留什么？牺牲什么？获得什么？」用翻译决策表而非通用模板合成选项。
> - **场景 M（多原作）**：列出每本原作的贡献参数（A提供什么、B提供什么）。问：「这些参数能共存吗？」如冲突，标记为不可调和/需要妥协方案。用融合方案驱动选项合成。
> - **场景 O（无参考）**：不从任何具体作品出发。直接从特征参数推导：「这个 L1 参数 → 需要什么样的世界观来支撑？」「这个 L3 参数 → 需要什么样的代价规则？」用参数推导而非模板套用合成选项。
>
> 禁止生成「跟特征参数只有松散关联、实质上是通用模板」的概念。

**每个选项包含：**

```markdown
### 选项 {N}：{概念名称}

**一句话定位**：{让人脑中放电影的一句话}

**世界观结构**：
- {从搜索 A 中提取的结构——单世界还是双世界？位面关系？}
- {从搜索 B 中提取的历史/社会原型——哪个朝代的哪种压迫结构？}
- {世界的「吃人规则」具体怎么运作？}

**核心冲突模式**：
- {从搜索 B 中提取的冲突原型——信息差博弈？阶级对抗？多方渗透？}
- {主角的初始位置和上升路径是怎样的？}

**金手指方向**：
- {从搜索 C 中提取的结构——叫什么？核心机制？代价是什么？}
- {为什么这个金手指契合爽点方向？}

**对标参考**：
- {搜索中发现的 1-2 本结构相似的优秀作品}
- {与这些作品的核心差异化在哪里？}

**风险提示**：
- {这个方向的常见坑——从搜索 A 的作品缺点中提取}
```

#### 2.3 呈现选项给用户

> 一次性呈现 2-3 个选项。用 1-2 句话总结每个选项的「独特卖点」和「主要风险」。让用户选——不要追问细节。

### Step 3：用户选择 → 锁定概念 + 推导本书特性(≥5条) + 用户画像

**agent 要做的**：

1. 用户选一个选项（或提出修正——「我要 A 的世界观 + B 的金手指」）
2. 从锁定的故事概念中**直接提炼美学基因**——不需要重新问用户：

| 基因 | 从概念中的哪个部分推导 | 
|:-----|:---------------------|
| **基因1：类型+基调** | 一句话定位 + 对标参考 → 推导出类型锚点和基调 |
| **基因2：视角+世界规则** | 世界观结构 + 核心冲突模式 → 推导出主角视角和世界基本规则 |

3. **推导本书特性（≥5条）**——这本书和其他书最大的不同点是什么？每条必须回答：「别人家怎么做 → 本书怎么做 → 为什么这样更对」。低于5条不合格。

```
本书特性：
1. {特性1}：别人家{……}，本书{……}，因为{推导理由}
2. {特性2}：...
...
（≥5条）
```

4. **推导用户画像**——这本书写给谁看的？

```
用户画像：
- 核心读者：{谁}——{他们的核心诉求}
- 边缘读者：{谁}——{为什么也会看}
- 不服务：{谁}——{明确拒绝的读者类型}
```

**产出**：

```
锁定的故事概念：{选项 N + 用户修正}

美学基因：
| 基因1 | {类型+基调} | {排他性后果} |
| 基因2 | {视角+世界规则} | {排他性后果} |

本书特性（≥5条）：
1. {特性1}...
2. {特性2}...
...

用户画像：
- 核心读者：{...}
- 边缘读者：{...}
- 不服务：{...}
```

---

### Step 4：从基因推导全书色彩 + 命名体系

> **不再逐条问用户。** agent 基于锁定的故事概念直接写好质感/力量/情感/命名体系，一次性呈现，用户确认或微调。

**推导逻辑**：

| 色彩维度 | 从概念的哪部分推导 |
|:---------|:------------------|
| 世界质感 | 世界观结构 → 物理质感 + 空间感受 |
| 力量美学 | 核心冲突模式 + 金手指方向 → 力量外观 + 变强的代价 + 力量来源 |
| 情感光谱 | 基因1（基调）+ 爽点 → 禁用情感 + 情感节奏 + 情感的特殊质地 |
| 命名体系 | 两个基因共同决定 → 世界/力量/角色的命名用什么语汇体系？拒绝什么语汇体系？ |

**命名体系推导**（agent 自答，用户确认）：

> 命名不是起名字，是**调性锁定**。命名体系的选择决定了读者进入这个世界时的第一感受。agent 必须回答：
>
> - **世界命名语汇**：地理/势力/权力机构用什么语汇体系？（例：道教九品/佛教名相/民间直白/金融术语/西方奇幻）
> - **力量命名语汇**：功法/境界/金手指名称用什么语汇体系？（例：阴符经的盗天/丹道的炼化/金融的借贷/游戏的技能树）
> - **拒绝的语汇**：明确不用的命名风格，并说明为什么不匹配基因
> - **一条命名铁律**：全书命名不可违反的一条规则（例：「禁止用佛教概念做只是听起来酷但原意被掏空的名字」）

**agent 的输出格式**（一次性呈现）：

```markdown
### 世界质感
**基于你的世界观结构（{简述}），物理质感应该是：**
- {世界侧A}：{质感描述——因为 {推导理由}}
- {世界侧B（如有）}：{质感描述——因为 {推导理由}}

**空间感受：** {因为世界观的社会结构是 {X}，所以空间应该是 {Y}——这是 {Z} 的具象化}

### 力量美学
**基于你的冲突模式（{简述}），力量的外观应该是：**
- {力量侧A} = {形容词}——因为 {推导理由}
- {力量侧B（如有）} = {形容词}——因为 {推导理由}

**变强的代价：** {代价规则}——这条规则直接衍生出金手指的设计方向：{X}

**力量的来源不是天赋：** 因为基因2（{简述}），力量只能来自 {X/Y/Z}

### 情感光谱
**禁用情感：** ❌ {列表}——因为基调是 {基因1}

**情感节奏：** {名称}——因为爽点分布是 {简述}

**情感的特殊质地：** 因为视角是 {基因2}，所以 {特殊情感} 不是调味料——是自然产物
```

### Step 5：推导世界宪法（4-6条不可违反的规则）

> **世界宪法 = 书中世界必须遵循的根本法则。** 每条宪法从美学基因 + 全书色彩 + 本书特性中推导，回答的问题是：「为了让这本书的独特体验成立，世界必须遵守什么规则？」
>
> 宪法不是设定——它是设定之上的约束。例如不写「有九个境界」，而写「力量必须有可量化的代价，不可无代价突破」。

**推导逻辑**：

每条宪法从以下维度之一出发：
| 来源 | 推导出的宪法类型 | 示例 |
|:-----|:----------------|:-----|
| 美学基因1（类型/基调） | 世界的物理/超自然规则 | 「神力=意志力，只有坚定信念才能承载神性」|
| 美学基因2（视角/规则） | 社会的结构/权力规则 | 「阶级压迫不可逆，底层没有体制内上升通道」|
| 本书特性 | 叙事的驱动力规则 | 「信息差驱动冲突——所有核心冲突的本质是信息不对称」|
| 核心爽点 | 爽点的保障规则 | 「金钱即力量——力量不来自天赋或苦修，而来自钱」|

**agent 的输出格式**：

```
世界宪法：
## 第一条：{规则名称}
{规则描述（1-2句）}
来源：{从哪个维度推导} | 违反后果：{如果这条被打破，书的什么体验会崩塌}

## 第二条：{规则名称}
...

（4-6条，少于 4 条不合格——说明对世界的约束不够；多于 6 条需检查是否有些属于 L1 设定而非宪法）
```

> ⚠️ **宪法 vs 设定**：宪法回答「世界不可违反什么」，设定回答「世界具体有什么」。例如「阶级压迫不可逆」是宪法，「九品十八级官制」是设定。PRD 只写宪法，不写设定。

---

### Step 6：跨域素材指引

> **不要列详细素材方案。** PRD 的跨域素材指引是**方向性的**——告诉后续的 world/plot skill 应该去哪些领域找素材、找灵感、找拆书参考。具体的采纳/转译规则由 L1 设计阶段决定。

**推导逻辑**：从美学基因 + 世界宪法 + 本书特性中，推导出**最适合这本书的知识领域方向**。每个方向只写 1-2 句指引，不展开。

**agent 的输出格式**：

```
跨域素材指引：
- {方向1}：{1-2句指引}（例：「DND 3.5e 位面体系——读被遗忘的国度的位面结构和神祇系统，提取层级关系和神性转化逻辑」）
- {方向2}：...
- {方向3}：...

（3-5 个方向。每个方向只标注去哪个领域找什么，不写具体怎么用。）
```

> ⚠️ **指引 vs 方案**：「去搜 DND 3.5e 的位面体系和神祇设定」是指引。「把深渊主宰的背刺公式翻译成望气术」是方案。PRD 只写指引，方案留给 L1。

### Step 7：篇幅规划

**agent 直接提案**——基于类型 + 爽点密度 + 故事概念的复杂度：

*「基于你的类型（{X}）和故事概念的复杂度（{Y}），我建议目标字数 {Z} 万字，约 {N} 卷。这意味着每卷 {M} 章，每章至少 {P} 个爽点脉冲。每卷 {Q} 次质变，{R} 个新角色入场。如果同意，篇幅约束锁定。」*

### Step 8：合成 PRD + 核心承诺回引爽点

**合成顺序**（PRD.md 的最终宪法结构）：

```
# PRD · {书名}
爽点锚点（含特征参数）
→ 本书特性（≥5条）
→ 美学基因 + 命名体系
→ 用户画像
→ 世界宪法（4-6条）
→ 跨域素材指引
→ 篇幅规划
→ 核心承诺 + 防崩阈值
→ 不可妥协 DNA + 边界
```

**核心承诺**：Step 1 锁定的每个元爽点 → 一条对读者的承诺。每条 = 体验描述 + 元爽点 + 变体家族 + 可量化的防崩阈值。
