# Phase R：路由诊断（HARD GATE）

> **读什么**：用户初始输入（开书/创意打磨/方向碰撞 的第一次消息）或（项目中注入新元素的触发消息）
> **产出什么**：`_路由记录/分支选择.md`
> **门禁**：❌ 不可跳过。不诊断就开走 = 不知道自己在走哪条分支

---

## 调用模式识别（最先执行）

在进入三分支诊断之前，先判断当前调用是**管线入口模式**还是**独立调起模式**。

### 检测步骤

1. 检查当前项目目录是否存在以下文件：
   - `00-原始设定/PRD.md`
   - `00-原始设定/故事引擎.md`
   - `00-原始设定/素材储备池.md`
   - `00-原始设定/世界宪法.md`（可选，有则更完整）

2. 检查用户输入是否属于"注入"模式：

| 用户说的话 | 模式判定 |
|:----------|:---------|
| "帮我把这个XX融进书里" / "最近有个XX热点" / "我想到一个新设定" / "这个能写进我的书吗" | **注入信号** |
| "开新书" / "我想写个故事" / "创意打磨" / 无项目上下文 | **管线入口信号** |
| 项目文件存在 + 注入信号 | → **Phase Delta（独立调起）** |
| 项目文件不存在 + 任何输入 | → **Phase R（管线入口）** |
| 项目文件存在 + 管线入口信号（如"继续开书"） | → **Phase R（但标注已有产出）** |

### 产出

如果判定为 Phase Delta → 写入 `_路由记录/分支选择.md` 标注 `mode: delta`，然后详见 [steps/step-delta.md](steps/step-delta.md)，不继续三分支诊断。

如果判定为管线入口 → 继续三分支诊断。

---

## 三分支诊断逻辑（仅管线入口模式）

从用户的初始输入中提取 3 个信号维度，综合判断走哪条分支：

### 信号 1：用户的状态描述

| 用户说的话 | 信号 | 倾向分支 |
|:----------|:-----|:---------|
| "我要写一本XXX的书" / "核心是XXX" / "主题是XXX" | **有清晰核心命题** | A |
| "我想把A/B/C揉在一起" / "我喜欢XXX和XXX" / "怎么融合" | **多个碎片化灵感** | B |
| "帮我想个故事" / "有什么好idea" / "随便写个修仙" | **空白/让渡决策** | C |
| "基于《深渊主宰》写同人，主角替换，加XX元素" | **同人二创** | B 锚点书子路径 |

### 信号 2：用户提供的素材密集度

| 表现 | 倾向分支 |
|:-----|:---------|
| 用户提供了大量具体碎片（3 个以上设定/IP/元素） | B |
| 用户给了 1-2 个关键词或一句话 | A 或 C |
| 用户给了一个完整的核心假设 | A |

### 信号 3：用户对创作方向的控制欲

| 表现 | 倾向分支 |
|:-----|:---------|
| "必须如何如何" / "不接受什么" | A（有明确的自我约束） |
| "你觉得呢" / "你推荐什么" | C（让渡决策） |
| "这些能不能融合" / "冲突的地方怎么处理" | B（打磨阶段） |

### 综合决策矩阵

| 核心命题 | 素材密集 | 控制欲 | → 分支 |
|:---------|:---------|:-------|:-------|
| 清晰 | 低 | 强 | **A：PRD先行** |
| 清晰 | 高 | 中 | **A：但保留PRD后反审空间** |
| 模糊 | 高 | 弱 | **B：广度采集→浮现PRD** |
| 模糊 | 中 | 弱 | **B 或 C：多用几个问题确认** |
| 无 | 无 | 弱 | **C：方向碰撞** |
| 同人二创 | 中 | 中 | **B 锚点书子路径** |

---

## 操作步骤

### 1. 分析用户输入

提取上述 3 个信号，记入草稿。

### 2. 如信号不明确，追问一句

用 clarify 或自然语言问一句（语气自然，不像是走流程）：

> "我想确认一下你的状态——你脑子里现在有没有一个比较清晰的核心想法（比如'我想写一本XXX的书'），还是手上有好几堆喜欢的东西/灵感想揉在一起（比如'我喜欢A/B/C，想融合'），还是希望我从零帮你想个方向？"

### 3. 记录分支选择

写入 `_路由记录/分支选择.md`，格式如下：

```markdown
# 路由记录 · 分支选择

## 初始输入摘要

[用户的原始输入，简要概括]

## 信号分析

| 维度 | 分析 | 
|:-----|:-----|
| 核心命题 | [清晰 / 模糊 / 无] — 依据 |
| 素材密集度 | [高 / 中 / 低] — 依据 |
| 控制欲 | [强 / 中 / 弱] — 依据 |

## 分支选择

**分支：[A / B / C]**

理由：[一句话说明为什么走这条分支]

## 同人二创判定（仅分支 B 时检查）

- ❌ 否 / ✅ 是：用户锚定了[作品名]，做[同人/跨界融合]
- 如果是 → 标记为"锚点书子路径"，参考 `references/同人二创-跨界融合工作流.md`
```
