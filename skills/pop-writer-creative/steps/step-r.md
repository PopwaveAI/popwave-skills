# Step R：路由诊断（HARD GATE）

> **定位**：判断用户输入的模糊程度 + 调用模式 → 决定后续研究的**宽度**和**选项多样性**。
> **产出**：路由结论（不是文件）——直接传递给 step-research.md 的研究策略。
> **门禁**：❌ 不可跳过。
>
> 读取协议：❌ 禁止 Read 工具，用 `skill_view` 或 `Get-Content -Encoding UTF8 -Raw`。

---

## 调用模式判定

读取用户的第一条消息，先判定调用模式：

| 模式 | 信号 | 流程 |
|:-----|:-----|:-----|
| **1 新书** | "开书/新书/创意/我想写个故事" | Phase R → Phase 1（种子展开+深度研究）→ Phase 2（PRD推导）→ Phase 3（储备卡产出） |
| **2 素材注入** | "注入素材/内化/帮我把这个XX内化" | Phase R → Phase 1（mini，针对新素材）→ Phase 3（储备卡产出） |
| **3 主动丰富** | "升级池子/丰富储备池" | 缺口分析 → Phase 1（定向研究）→ Phase 3（储备卡产出） |

> Mode 2/3 跳过 Phase 2（PRD推导），直接从 Phase 1 进入 Phase 3。

---

## 输入分类（Mode 1 新书模式）

读取用户的第一条消息，判断属于哪种：

| 类型 | 信号 | 搜索策略 |
|---|---|---|
| **定向** | 「我要写一本 XXX 的书」——有类型+基调+大概方向 | **窄搜**——只搜同类成功作品的结构+特定历史/神话原型。生成 2 个选项 |
| **碎片** | 「我喜欢 A 的 XX + B 的 XX，想揉在一起」——有素材但不知道怎么组合 | **中搜**——搜每块素材的相关领域+找共同脉络。生成 2-3 个融合选项 |
| **空白** | 「帮我想个故事」——完全开放，无具体方向 | **宽搜**——搜当前热门赛道+用户阅读偏好。先提方向选项，用户选后再搜。生成 2-3 个方向选项 |

## 元素注入（Mode 1 新书模式）★NEW

> **定位**：用户给的元素是起点，不是全部。agent 主动补充外部元素，让用户判断是否纳入——扩大创意空间，避免在用户初始输入的有限框架里打转。
>
> **注入不是凭空脑暴**——先查 trope-library，从同赛道的套路和剧情线卡中提取元素变体，基于库做定向补充。

### 注入策略

| 类型 | 定义 | 数量 | 来源 |
|:-----|:-----|:-----|:-----|
| 🔵 **相似元素** | 同类型但不同执行方式的元素——拓宽"这个方向还能怎么玩" | 2-3 个 | trope-library 套路库同家族变体 + 剧情库同赛道卡 |
| 🔴 **相反元素** | 形成张力的对立面——制造冲突和反差 | 1-2 个 | trope-library 跨赛道卡 + 用户元素的反面推演 |
| ⚪ **盲区元素** | 该类型必备但用户没想到的——补齐类型常识 | 1-2 个 | trope-library 套路库总索引 + 元爽点-变体映射表 |

### 注入流程

1. 提取用户输入中的元素清单（类型+基调+方向+素材）
2. 查 `pop-trope-library/套路库/00-总索引.md` → 匹配同赛道套路家族
3. 查 `pop-trope-library/剧情库/00-索引.md` → 按标签筛选同赛道剧情线卡
4. 查 `pop-trope-library/references/元爽点-变体映射表.md` → 匹配元爽点变体
5. 从命中结果中提取 🔵相似/🔴相反/⚪盲区 元素
6. 库缺时 WebSearch 补充（查询词：`"{赛道} {元素类型} 变体 反例"`）

### 呈现格式

```
📋 元素清单

用户提供的元素：
- {元素1}（{类型/基调}）
- {元素2}（{素材}）
- ...

agent 补充的元素（请判断是否纳入）：
- 🔵 {相似元素} — 理由：{为什么建议} — 来源：{库/卡名}
- 🔵 {相似元素} — 理由：{...} — 来源：{...}
- 🔴 {相反元素} — 理由：{形成什么张力} — 来源：{...}
- ⚪ {盲区元素} — 理由：{该类型为什么需要这个} — 来源：{...}

请回复：纳入哪些？排除哪些？或补充你自己的想法。
```

### 关键约束

- ⛔ 注入元素必须标注来源（哪个库的哪张卡/哪次搜索），不允许"我觉得可以加这个"
- ⛔ 注入元素必须说明理由（为什么建议纳入、形成什么张力/补齐什么盲区）
- ⛔ 用户未确认纳入的元素不进入 step-research.md 的种子清单
