# Step W0.5：问作者要元素

> 分支 A/B/C 通用步骤
> 产出：`_创意元素/作者想法.md`

## 做什么

带着 PRD 的加工坐标系，向作者定向提问，收集作者脑子里的元素素材。

## 问什么

按 PRD 的类型锚点和核心承诺，定向提问：

1. **主角相关**：主角是什么样的人？什么背景？什么驱动力？
2. **世界相关**：这个世界有什么特殊设定？什么规则？什么禁忌？
3. **冲突相关**：主要的矛盾是什么？谁和谁的冲突？什么利益纠葛？
4. **元素相关**：有没有特别想写的场景/桥段/设定/角色？

## 用户说"我没有"时怎么处理

当用户说"我没有素材/你帮我提供/需要你来丰富"时：

1. **不退回方向生成** — PRD 已锁定，方向已定。下一步是素材采集，不是重新碰撞方向
2. **跳过 W0.5，直接进 W1** — 标注 `作者想法.md` 为"作者无个人素材，依赖W1采集"
3. **W1 提级** — 从"定向采集"升级为"深度采集"，因为作者没有提供方向性素材，W1 需要承担更多创意供给

## 产出格式

```markdown
# 作者想法

> 采集于 YYYY-MM-DD

## 主角
{作者提供的主角元素，或标注"作者无个人素材，依赖W1采集"}

## 世界
{同上}

## 冲突
{同上}

## 特别想写的
{同上}
```

## 门禁

- 文件必须存在（即使标注"作者无素材"也要落盘）
- 不可跳过（跳过 = W1 没有定向锚点）
