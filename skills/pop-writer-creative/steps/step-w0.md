# Step W0：查已归档的融合方案（HARD GATE）

> **读什么**：`D:\\popwave-skills\\skills\\pop-trope-library\\元素交叉\\`
> **产出什么**：`_素材聚合/元素复用建议.md`
> **门禁**：❌ 不可跳过。不查元素交叉库就开搜 = 浪费燃料
>
> **分支感知**：该步骤根据 Phase R 的分支诊断结果（A/B/C）调整检索策略。

---

## 操作步骤

### 1. 读取路由记录

首先读取 `_路由记录/分支选择.md`，确认当前分支。

### 2. 打开元素交叉库索引

`skill_view("pop-trope-library", "元素交叉/索引.md")`

按来源类型浏览已有的融合方案。如果索引为空（第一次用），跳过本步，记"首次使用，无归档"。

### 3. 按分支模式检索

#### 分支 A：PRD 筛选检索

PRD 已定，**带筛选条件**检索。不是搜所有匹配，而是只搜符合 PRD 加工坐标系的归档方案。

| 场景 | 动作 |
|:-----|:-----|
| PRD 说"高强度数值+社会学博弈" | 只检索"机制转化型"的归档（如 DND→某书的社会学改写），跳过纯粹设定交换型 |
| PRD 说"硬核历史" | 只检索考据向的融合方案，跳过仙魔向 |
| 未找到匹配 | 产出"无归档，需从头调研"，标注"按PRD筛选后空" |

**关键原则**：分支 A 的 W0 目标是"找到能直接复用 PRD 加工方式的方案"，不是搜参考。

#### 分支 B / C：广度检索

PRD 尚未定（B）或方向未定（C），**不设筛选条件**，广度检索所有与用户碎片灵感相关的归档方案。

| 场景 | 动作 |
|:-----|:-----|
| 用户提到多个碎片 | 按每个碎片分别检索 → 合并结果 |
| 找到匹配 | 读具体方案 → 提取焊接方式（留作 PRD 浮现时的参考库） |
| 未找到匹配 | 产出"无归档，需从头调研"，标注"广度检索后无匹配" |

**关键原则**：分支 B/C 的 W0 目标是"收集尽可能多的焊接参考"，为后续 PRD 浮现提供参照库。

### 4. 产出一份复用建议

如果找到匹配，写入 `_素材聚合/元素复用建议.md`，格式如下：

```
# 元素复用建议

## 检索模式

**分支：[A/B/C]**
**检索策略：[PRD筛选检索 / 广度检索]**
**PRD 筛选条件（分支A）：[实际筛选条件]**

## 已归档的参考源

匹配关键词：[用户输入的关键词]

### 参考源1：[来源名] → [应用作品]

- **提取元素**：[原始作品中哪些元素被提取了]
- **焊接方式**：[具体怎么转换的]
- **改动幅度**：[轻/中/重 —— 改了原作多少]
- **效果评估**：[读者反馈或客观效果]
- **PRD 匹配度（分支A）**：[高/中/低 —— 该方案是否符合本书PRD的加工哲学]

## 对本项目的建议

- 可直接复用的：[什么可以直接照搬]
- 需要调整的：[什么需要针对本项目修改]
- 需要回避的：[什么不适合本项目]
```

## 质量标准

| 检查 | 标准 |
|:-----|:-----|
| 已查索引 | 明确产出"有匹配"或"无归档" |
| 有匹配时已读具体方案 | 复用建议提到焊接方式表的具体条目 |
| 方案已蒸馏 | 复用建议不是照搬原文，而是针对本项目写了建议 |
| 分支感知（A） | 标注了每个方案的PRD匹配度 |

## 产出

`_素材聚合/元素复用建议.md`
