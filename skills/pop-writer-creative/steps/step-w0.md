# Step W0：查已归档的融合方案（HARD GATE）

> **读什么**：`D:\popwave-skills\skills\pop-trope-library\元素交叉\`
> **产出什么**：`_素材聚合/元素复用建议.md`
> **门禁**：❌ 不可跳过。不查元素交叉库就开搜 = 浪费燃料

---

## 操作步骤

### 1. 打开元素交叉库索引

`skill_view("pop-trope-library", "元素交叉/索引.md")`

按来源类型浏览已有的融合方案。如果索引为空（第一次用），跳过本步，记"首次使用，无归档"。

### 2. 按关键词匹配

从用户初始输入中提取 2-3 个核心关键词（如"冰雪""战争""王室""升级系统"），在索引中搜索匹配项。

| 场景 | 动作 |
|:-----|:-----|
| 找到匹配（如用户要冰雪线，索引有Frozen→深渊主宰） | 读具体融合方案文件 → 提取可复用的焊接方式 |
| 未找到匹配 | 产出"无归档，需从头调研" |

### 3. 产出一份复用建议

如果找到匹配，写入 `_素材聚合/元素复用建议.md`，格式如下：

```
# 元素复用建议

## 已归档的参考源

匹配关键词：[用户输入的关键词]

### 参考源1：[来源名] → [应用作品]

- **提取元素**：[原始作品中哪些元素被提取了]
- **焊接方式**：[具体怎么转换的]
- **改动幅度**：[轻/中/重 —— 改了原作多少]
- **效果评估**：[读者反馈或客观效果]

## 对本项目的建议

- 可直接复用的：[什么可以直接照搬]
- 需要调整的：[什么需要针对本项目修改]
- 需要回避的：[什么不适合本项目]
```

## 质量标准

| 检查 | 标准 |
|:-----|:-----|
| 已查索引 | 明确产出"有匹配"或"无归档" |
| 有匹配时已读具体方案 | 复用建议提到焊接方式表的具体条目 |
| 方案已蒸馏 | 复用建议不是照搬原文，而是针对本项目写了建议 |

## 产出

`_素材聚合/元素复用建议.md`
