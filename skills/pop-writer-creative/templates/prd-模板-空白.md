# PRD 宪法模板

> Step 8 合成 PRD 时的最终结构模板。推演语气（断言式），宪法层抽象，无具体名/数值/分幕/转译规则。

```markdown
# PRD · {书名}
> 一句话定调

## 〇、本书剧情简述
{一段150-200字的故事梗概——一个什么样的世界，发生什么样的危机，主角干了什么。
 不解释设定，只呈现冲突弧线。结尾锚定一个悬念或一个画面。}

---

## 一、篇幅规划

| 参数 | 数值 | 说明 |
|:-----|:-----|:-----|
| **目标总字数** | {N} 万字 | {卷数×章数×字数的推算。网文基准 2k-3k 字/章，非 4k-6k} |
| **预估卷数** | {N} 卷 | {每卷对应的代际跃迁/维度切换} |
| **单卷章数（均值）** | {N} 章 | {首卷和后续卷的节奏差异} |
| **幕/卷** | {N} 幕 | {每幕的核心冲突单元} |
| **升级阶梯密度** | 每卷 {N} 次质变 | {质变路径简述} |
| **剧情线数量** | 主线 {N} 条 + 支线 {N} 条 | {主线支线说明} |

### 篇幅对结构的约束
- **升级节奏**：...
- **爽点密度**：...
- **角色引入节奏**：...
- **世界揭示节奏**：...
- **{N} 万字的特殊约束**：...
- **篇幅联动验证**：目标总字数 ÷ 单章中位数(按2.5k) = 总章数 → 总章数 ÷ 卷数 = 单卷章数 → 单卷章数 ÷ 幕数 = 单幕章数。所有数字须自洽。

---

## 二、爽点锚点

> **重要：爽点名称必须写在元爽点抽象层（变强 / 建设 / 博弈 / 探索 / 合作），而不是具体的中后期事件名（果位晋升 / 洞天经营 / 权柄争夺）。**
> 元爽点贯穿全书不变，每卷的"落地形式"不同——例如「变强」在卷一=境界突破(开脉→洗髓)，在卷三=首次接触果位碎片。plot阶段会做逐卷特化。
> 错误示例：果位晋升30%、洞天经营20%、权柄争夺25%
> 正确示例：变强30%、建设20%、博弈25%、探索15%、合作10%

### 锚点①：{名称（用元爽点：变强/建设/博弈/探索/合作）}
{2-3句爽点描述——读者为什么爽？心理机制是什么？}

**变体家族**：
- {变体1}：...
- {变体2}：...

**兑现方式**：...

**该元爽点的不同阶段落地（留给plot填充，此处只给方向）**：
- {前期阶段}：{如"境界突破"或"装备升级"}
- {中期阶段}：{如"接触果位碎片"或"领地建设"}
- {后期阶段}：{如"神系博弈"或"法则掌控"}

**特征参数**：

| 维度 | 参数 |
|:-----|:-----|
| L1 {触发条件/感官媒介} | {...} |
| L2 {时效性/驱动力} | {...} |
| L3 {代价/频率} | {...} |
| L4 {叙事层/升级路径/社会代价} | {...} |

### 锚点②：{名称（用元爽点）}
...

### 锚点③：{名称（用元爽点）}
...
```

## 禁止出现在 PRD 中的内容

| 条目 | 原因 | 归到哪 |
|:-----|:-----|:-------|
| 属性数值 | PRD 定义「属性阈值突破触发质变」，不写力量12/敏捷19 | L1 数值设计 |
| 分幕/章节划分 | PRD 定义规模（5-6卷），不写具体幕数 | plot 卷纲 |
| 金手指具体机制 | PRD 定义设计哲学，不写冷却时间/消耗值 | L1 金手指设计 |
| 详细转译规则 | PRD 指引方向，方案留给 L1 | L1 素材设计 |
| 创意溯源交叉表 | PRD「去搜 X 领域」是指引，交叉表是方案 | L1 素材分析 |
| 逆向拆书语言 | 「基于前X章」「chXX已验证」「反向提炼」等复盘语气 | 不存在于创意 PRD |

> **可以写**：核心创意命名（世界名/主角名/金手指名/关键概念名）。红线 3 从「不预设具体名字」改为「不预设 L1 级施工细节」——创意决策留 PRD，施工图留 L1/plot。
