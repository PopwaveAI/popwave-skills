# pop-writer-game · 小说世界观AI文游化 v2.0.2

将写书过程中已沉淀的世界观、人设、剧情设定等资料，转化为**可游玩的 AI 文游（AI 驱动的文字游戏）**。产出是让读者能走进你的小说世界里、亲自体验剧情的**互动文字游戏**。

---

## 速查表

| 阶段 | 核心动作 | 产出 | 门禁 |
|:-----|:---------|:-----|:-----|
| **Phase A — 资料解析** | 读取原始资料 → 提取静态层+角色层+剧情层 | 结构化世界观数据 JSON | 原始资料缺失→中止 |

| **Phase C — 产出生成** | 生成指令文件 + HTML | 文游指令.md + AI文游.html | HTML 验证不通过→重做 |

## 执行流程

### Phase A — 资料解析

读取用户提供的世界设定、人设、剧情大纲等资料，提取三层数据（静态层/角色层/剧情层），产出结构化世界观数据 JSON。

→ 详见 [steps/01-资料解析.md](steps/01-资料解析.md)
→ 详细协议：[references/资料解析协议.md](references/资料解析协议.md)

### Phase B — 文游设计

基于 Phase A 产出的结构化 JSON，设计 6 个维度（玩家身份/核心玩法/分支节点/数值系统/结局设计/AI约束规则），产出 AI 文游指令文档。

→ 详见 [steps/02-文游设计.md](steps/02-文游设计.md)
→ 详细协议：[references/文游设计协议.md](references/文游设计协议.md)

### Phase C — 产出生成

生成两份核心产物：文游指令.md（AI 可直接读取）+ AI文游.html（交互式游玩界面），含零外部依赖、流式 API 驱动、localStorage 持久化。

→ 详见 [steps/03-产出生成.md](steps/03-产出生成.md)
→ 详细协议：[references/HTML产出规范.md](references/HTML产出规范.md)

## 文游指令设计规范

指令文件是丢给 AI 模型让它当游戏主持人的核心文件，必须包含七个章节：

```
一、你的角色 — AI 在游戏中扮演的角色（DM/旁白 + 所有NPC）
二、世界观设定 — 时代背景/地理势力/核心规则/关键名词
三、角色设定 — 可攻略角色/NPC 的详细信息 + 玩家可扮演的身份
四、游戏规则 — 开局流程/核心玩法/数值规则/分支与结局/AI行为约束/输出格式
五、AI 行为守则 — 人设一致性/世界观边界/叙事要求/数值管理
六、开局提示 — 给玩家的第一段引导文本 + 初始状态
```

完整模板见 [steps/03-产出生成.md](steps/03-产出生成.md)。

## 错误示例

### WRONG 1：硬编码 API key
```yaml
api:
  key: "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```
错误：一旦文件被提交到公开仓库，key 立即泄露。
正确：删除 frontmatter 中的 api 块，通过环境变量或交互式提示动态注入。

### WRONG 2：Phase A 不产出结构化 JSON
错误：Agent 说"我看懂了这些设定" → 直接进入 Phase B 设计游戏。
正确：先提取静态层/角色层/剧情层为 JSON，再进入 Phase B。

### WRONG 3：HTML 用了预设场景查表模式
错误：预设了 10 个场景分支，玩家只能在这 10 个里选。
正确：每次玩家操作必须调 AI API 实时生成，stream: true。

### WRONG 4：未转义 GAME_INSTRUCTION 中的反引号
错误：直接将文游指令.md 的 markdown 原文嵌入 HTML 的 JS 模板字面量。
正确：嵌入前用 `.replace(/`/g, '\\`')` 转义所有反引号。

### WRONG 5：输出目录覆盖问都不问
错误：检测到 `<小说名>-AI文游/` 已存在 → 直接覆盖。
正确：询问用户是否覆盖，还是创建带时间戳的备份目录（如 `<小说名>-AI文游_20260604/`）。

## 异常与边界条件

| # | 场景 | 处理方式 |
|:-:|:-----|:---------|
| 1 | 世界观/人设文件缺失或为空 | 中止 Phase A，明确提示缺少哪些文件，要求补充后再继续 |
| 2 | API 调用失败（网络错误 / 4xx / 5xx） | 指数退避重试最多 3 次（1s/2s/4s），3 次均失败显示手动重试按钮 |
| 3 | HTML 验证不通过（`</html>` 缺失等） | 重新执行 HTML 生成步骤，修复后再次验证，直至通过 |
| 4 | 用户未提供任何原始素材（空输入） | 要求至少提供小说简介、世界观概要或角色列表之一，否则无法启动 |
| 5 | 文游指令.md 的关键章节内容为空 | 回退到 Phase A 补充提取对应数据，确保指令完整后再进入 Phase C |
| 6 | 输出目录 `<小说名>-AI文游/` 已存在 | 询问用户是覆盖还是创建带时间戳的备份目录 |
| 7 | 用户要求移除所有 API 调用，使用纯预设内容 | 解释本 skill 的核心价值是 AI 实时驱动；如坚持，降级为静态分支游戏，需标注"非 AI 驱动版" |
| 8 | 嵌入 HTML 的文游指令包含未转义反引号 | 自动转义所有内部反引号为 `\``，防止 JS 模板字面量语法错误 |
