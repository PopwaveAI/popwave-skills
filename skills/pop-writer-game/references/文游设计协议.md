# 文游设计协议 · Phase B

## 目标

基于 Phase A 产出的结构化世界观数据，设计 AI 文游的玩法框架，输出完整的游戏主持指令。

## 设计流程

### Step 1：定位玩家身份

确定玩家在游戏中扮演什么角色：

| 模式 | 说明 | 适合场景 |
|:----|:----|:--------|
| **原著角色代入** | 玩家扮演原著中的某个角色，走该角色的故事线 | 角色厨/单推人向 |
| **穿越者模式** | 玩家作为外界者穿越进这个世界，可以自由互动 | 自由度最高，通用推荐 |
| **旁观者模式** | 玩家以记者/史官/系统等身份观察和影响故事 | 深度剧情探索向 |
| **原创角色** | 玩家在这个世界观中创建自己的角色 | 最强代入感 |

### Step 2：设计核心玩法

根据世界观类型推荐玩法：

| 世界观类型 | 推荐玩法 | 数值系统 |
|:----------|:--------|:--------|
| 仙侠/修真 | 修炼冒险 + 宗门经营 + 道侣养成 | 修为/好感度/声望 |
| 西幻 | 冒险探索 + 阵营选择 + 羁绊系统 | 战力/声望/黑化值 |
| 都市/现代 | 恋爱养成 + 事业经营 + 社交博弈 | 好感度/影响力/财富 |
| 科幻 | 生存探索 + 势力站队 + 技术研发 | 科技点/声望/生存值 |
| 古风/宫斗 | 权谋博弈 + 社交联盟 + 隐藏身份 | 权势/好感度/秘密值 |
| 无限流/快穿 | 副本攻略 + 角色收集 + 能力成长 | 积分/能力/羁绊 |

**核心原则**：
- 数值系统不超过 4 个维度
- 每个数值必须有明确的增减条件和可视化反馈
- 数值影响剧情走向，而非仅用于展示

### Step 3：设计分支节点

在故事中设计至少 3 个关键决策点：

```
分支节点设计模板：

节点1 ── 早期抉择（故事开端后不久）
  选项A → 路线α → 影响：数值+角色关系变化
  选项B → 路线β → 影响：数值+角色关系变化
  选项C（隐藏）→ 特殊路线 → 需特定条件触发

节点2 ── 中期转折（故事中段）
  ...

节点3 ── 终局抉择（影响最终结局）
  ...
```

### Step 4：设计结局体系

至少设计 2 种不同结局：

| 结局类型 | 说明 |
|:--------|:----|
| **主线结局** | 遵循核心剧情走向的结局（至少 1 个好结局 + 1 个坏结局） |
| **隐藏结局** | 需要特定条件/多周目才能触发的特殊结局 |
| **彩蛋结局** | 搞笑/meta/打破第四面墙的趣味结局（可选） |

### Step 5：设计 AI 约束规则

AI 作为游戏主持人，必须遵守以下约束：

```
【AI 行为守则】

1. 人设一致性
   - 所有角色必须严格遵循原始人设，不得 OOC
   - 角色说话风格、行为模式必须匹配设定

2. 世界观边界
   - 不得引入原始世界观中不存在的力量体系或设定
   - 角色能力不能超越原始设定中的上限

3. 叙事要求
   - 每次输出剧情文本 200~500 字（可调整）
   - 每轮对话结束时提供 3~4 个选项
   - 禁止替玩家做决定（除非特定剧情需要）
   - 禁止跳过关键剧情节点

4. 数值管理
   - 每次玩家的选择影响数值时必须明确告知
   - 数值达到阈值时必须触发对应事件
```

### Step 6：编写主持指令

将以上所有设计整合为一份完整的 AI 主持指令，结构如下：

```
# ════════════════════════════
# 《XXX》AI 文游：主持指令
# ════════════════════════════

## 一、核心设定
- 你的角色定位
- 世界观一句话概括
- 游戏基调（轻松/黑暗/幽默/史诗）

## 二、世界观百科
（从 Phase A 数据整理）

## 三、角色档案
（从 Phase A 数据整理）

## 四、玩家身份
- 可以扮演谁
- 开局创建角色的流程

## 五、游戏规则
- 玩法说明
- 数值系统及计算规则
- 分支节点定义
- 结局触发条件

## 六、AI 行为守则
- 人设约束
- 叙事格式
- 禁止事项

## 七、开局提示
- 给玩家的第一段引导文本
- 初始状态设置
```

## 纪律

- **所有玩法设计必须以原著设定为基础**，不要强行套用不匹配的玩法
- **指令文件必须用中文编写**，考虑国内 AI 模型的兼容性
- **指令文件最后要有"开玩提示"**——告诉玩家粘贴给哪个 AI 模型、第一句该说什么
- **AI 行为守则必须放在指令后半段**（前半段让 AI 先理解世界，再理解规则）
