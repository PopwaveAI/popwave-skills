# Step 01：资料解析（Phase A）

## 目标

读取用户提供的原始创作资料，提取三层结构化数据（静态层 + 角色层 + 剧情层），产出结构化世界观数据 JSON。

## 输入

用户提供的世界设定、人设、剧情大纲等资料（格式不限，支持 .md/.docx/.txt/YAML/JSON/图片等）

## 动作

| 层 | 提取内容 |
|:---|:---------|
| **静态层** | 地理/势力/规则体系/关键名词/专属设定 |
| **角色层** | 所有角色及关系网络、角色弧光 |
| **剧情层** | 核心冲突/关键事件/伏笔/分支可能性 |

## 产出

结构化世界观数据 JSON（包含 world / characters / plot 三部分）

## 门禁

原始资料缺失或为空 → 中止 Phase A，提示用户补充。要求至少提供小说简介、世界观概要或角色列表之一。
