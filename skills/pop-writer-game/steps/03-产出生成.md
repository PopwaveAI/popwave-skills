# Step 03：产出生成（Phase C）

## 目标

基于 Phase B 产出的文游指令草案 + 结构化 JSON，生成两份核心产物：文游指令.md（AI 可直接读取）和 AI文游.html（交互式游玩界面）。

## 输入

Phase B 产出的文游指令草案 + Phase A 产出的结构化 JSON

## 产出文件结构

```
<小说名>-AI文游/
├── 互动/
│   └── AI文游.html            ← 交互式HTML游玩界面（零外部依赖，双击即开）
├── 文游指令.md                ← 核心产物！丢给Claude/GPT/DeepSeek即可开玩
├── 世界观设定集.md             ← 从原始资料整理的结构化世界摘要（参考用）
└── 角色档案.md                ← 从原始资料整理的角色档案（参考用）
```

## 门禁

- 指令关键章节为空 → 回退 Phase A
- HTML 验证不通过 → 重新执行生成步骤，修复后再次验证

## 文游指令设计规范

指令文件是丢给 AI 模型让它当游戏主持人的核心文件，必须包含：

```
# ════════════════════════════
# 《XXX》AI 文游：主持指令
# ════════════════════════════

## 一、你的角色
描述 AI 在游戏中扮演的角色（游戏主持人/DM/旁白 + 所有NPC）

## 二、世界观设定
- 时代背景
- 地理/势力分布
- 核心规则（如力量体系、社会规则、禁忌）
- 关键名词解释

## 三、角色设定
- 主角/可攻略角色/重要NPC的详细信息（性格、背景、关系）
- 玩家可以扮演的角色身份

## 四、游戏规则
- 开局流程（如何创建角色、初始设定）
- 核心玩法（每日流程、交互方式、数值系统）
- 数值规则（好感度/修为等如何增减、触发条件）
- 分支与结局（关键节点、结局条件）
- AI行为约束（禁止做哪些事、必须保持人设）
- 输出格式要求（剧情描述长度、选项数量等）
```

## HTML 设计要求

| 要求 | 说明 |
|:----|:------|
| **零外部依赖** | 不加载CDN、不跨域、无外部字体/图标库。仅 API 调用需要网络，双击 .html 即可打开 |
| **自包含** | 所有数据（含文游指令）以文本嵌入 HTML 的 JS 常量中 |
| **AI 实时驱动** | 每次玩家操作实时调 API 生成剧情，而非预设分支 |
| **世界观主题** | 视觉风格适配故事世界的调性（仙侠/科幻/都市/西幻……） |
| **交互性** | 玩家可点击选择推进剧情，输入框支持自由输入 |
| **世界百科** | 内置世界观、角色、势力的查阅面板 |
| **进度持久化** | 使用 localStorage 保存游玩进度（含对话历史和状态） |
| **验证规范** | 交付前必须验证：`</html>`闭合 / 文游指令完整嵌入 / API 路径正确 / 双击可用 |

### API 配置

| 配置项 | 值 |
|:------|:---|
| Endpoint | `https://api.deepseek.com/v1/chat/completions` |
| Model | `deepseek-v4-flash` |
| Stream | `true` |
| Temperature | `0.85` |
| Max Tokens | `2048` |

API key 不得硬编码，通过 `$env:DEEPSEEK_API_KEY` 环境变量或交互式提示动态注入。

### HTML 功能模块参考

```
AI文游.html
├── [数据层]
│   ├── API_KEY — 由环境变量或用户输入注入
│   ├── GAME_INSTRUCTION — 完整文游指令（system prompt），内部反引号须转义
│   └── WORDS_DATA — 世界观参考 JSON（供侧边栏展示）
│
├── [引擎层]
│   ├── callAI() — 调 DeepSeek API，stream:true，send messages → 流式返回 JSON
│   │   ├── AbortController — 每次创建新实例，支持取消进行中的请求
│   │   ├── 超时兜底 — 30 秒无响应自动断开 + 提示重试
│   │   └── 指数退避重试 — 网络错误最多重试 3 次（1s/2s/4s）
│   ├── cancelAI() — 取消当前 API 请求，恢复 UI 交互状态
│   ├── parseStreamResponse() — 解析完整响应
│   ├── applyGameResult() — 将 AI 响应结果同步到 gameState + 触发渲染
│   └── gameState — { messages[], stats{}, inventory[], flags{}, history[] }
│
├── [UI层]
│   ├── 主界面（叙事面板）
│   ├── 角色状态栏
│   ├── Typing Indicator（AI 思考中动画）
│   ├── 侧边栏 / 抽屉面板（世界百科/角色档案/剧情回顾/存档管理）
│   ├── 底部工具栏（复制文游指令 / 重置游戏）
│   └── [兜底机制] cleanDisplayText / JSON fallback / showRetryButton / try-catch
```

### API 交互协议

**请求**（POST `https://api.deepseek.com/v1/chat/completions`）：
```json
{
  "model": "deepseek-v4-flash",
  "messages": [
    {"role": "system", "content": "（完整文游指令）"},
    {"role": "assistant", "content": "上一轮AI的叙事和选项..."},
    {"role": "user", "content": "玩家本次的选择或自由输入"}
  ],
  "temperature": 0.85,
  "max_tokens": 2048,
  "stream": true
}
```

**SSE 流式解析流程**：
```
fetch(stream: true) → response.body.getReader() → 逐块读取
for each chunk:
  ├─ 按 '\n' 分割行
  ├─ 取 'data: ' 前缀后的 JSON
  ├─ 检查 payload === '[DONE]' → 结束
  └─ choices[0].delta.content → 累积到 fullContent
       ├─ 若以 { 或 " 开头（旧 JSON 格式）→ 不显示，等流结束
       ├─ 若检测到 ---DATA--- → 停止追加
       └─ 否则 → 立即追加到叙事面板（requestAnimationFrame 节流）
stream 结束 → parseStreamResponse(fullContent) → { narrative, stats, choices, inventory, flags }
```

**响应格式**：
```
叙事文本（纯文字，300-500字，用 \n 表示换行）

---DATA---
{"stats":{"survival":60,"combat":8,"reputation":5,"shadow":3},"choices":["① 选项A","② 选项B","③ 选项C"],"inventory":["木棍","半块黑面包"],"flags":{"met_vivian":true}}
```

### API 响应解析（三格式兼容）

详见 [references/HTML产出规范.md](../references/HTML产出规范.md) 中的 `parseStreamResponse()` 实现：
1. 新格式：叙事文本 → `---DATA---` → JSON
2. 旧 JSON 格式：`{narrative: ..., stats: ..., choices: ...}`
3. 兜底：纯文本

## 实战陷阱

### 陷阱 1：GAME_INSTRUCTION 中的反引号
文游指令.md 里包含 markdown 代码块，嵌入 HTML 时使用 JS 模板字面量，内部反引号没有转义 → JS 语法错误。修复：嵌入前用 `.replace(/`/g, '\\`')` 转义所有反引号。

### 陷阱 2：流式格式兼容
AI 可能输出新格式（叙事文本 → `---DATA---` → JSON）或旧格式（纯 JSON）。流式阶段不检测格式会导致乱码。修复：流式循环中做 `isJsonFormat` 检测，JSON 格式等 stream 结束后一次性显示。

### 陷阱 3：重置游戏的 loading 状态锁
API 调用中（`isLoading = true`）点击重置，`callAI()` 被 `if (isLoading) return` 阻拦。修复：`resetGame()` 开头调用 `cancelAI()` 解除 loading 锁。

### 陷阱 4：超时与重试
API 可能因网络问题长时间无响应。修复：超时定时器（30 秒）+ 指数退避重试（1s/2s/4s，最多 3 次）+ 手动重试按钮。

### 陷阱 5：localStorage 恢复时的 JSON 解析失败
localStorage 数据可能因版本升级或手动篡改损坏，`JSON.parse` 直接抛异常导致页面白屏。修复：`try-catch` 包裹恢复逻辑，解析失败直接开始新游戏。

## 视觉风格

视觉设计必须适配世界观调性：

| 世界观类型 | 配色参考 | 字体风格 | 装饰元素 |
|:----------|:--------|:--------|:--------|
| **仙侠/古风** | 墨黑/朱红/青绿/米白 | 衬线风 | 水墨纹理、印章、卷轴 |
| **科幻/赛博** | 暗紫/霓虹蓝/荧光绿 | 无衬线、科技感 | 发光线条、网格 |
| **西幻/魔幻** | 深蓝/金币/羊皮纸色 | 中世纪风格 | 纹章、符文、羊皮纸 |
| **都市/现代** | 白/灰/品牌色 | 干净简约 | 卡片式、圆角 |
| **暗黑/恐怖** | 纯黑/暗红/灰白 | 粗重衬线 | 血迹、裂纹、模糊 |

## 纪律

1. **必须实时调 API** — 不准用预设场景查表模式
2. **必须验证** — 不验证 HTML 不准交付
3. **零外部依赖（除 API 外）**
4. **代码简洁** — 核心逻辑控制在 500 行以内
5. **Fallback 处理** — 指数退避重试 + 手动重试按钮
6. **不开新窗口** — 所有交互在单页面内完成
7. **API key 安全** — 禁止硬编码在文件中
8. **GAME_INSTRUCTION 必须完整** — 全文嵌入，不截断
9. **GAME_INSTRUCTION 反引号必须转义**
10. **同时支持旧 JSON 和新 ---DATA--- 格式**
11. **AbortController 必须实现**
12. **必须实现超时兜底**
13. **必须实现指数退避重试**
