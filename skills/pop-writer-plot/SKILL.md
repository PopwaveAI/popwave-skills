# pop-writer-plot · 剧情架构设计 v7.3.0

> **定位：从 PRD、世界设定、角色卡、素材储备池和 trope-library，产出卷纲、剧情线、幕纲。**
> **核心约束：文件收敛为 3 类：卷纲、剧情线、幕纲。Canvas 不废除，契诃夫枪链并入幕纲。**

## ❌ 质量红线

| 红线 | 说明 |
|:-----|:-----|
| ❌1 | **前置文件缺失仍开写** — `全书立项PRD.md`、L1 设定、角色卡、数值体系、素材储备池任一关键依赖缺失 → 终止并列缺口 |
| ❌2 | **未查库就定套路链** — 未查 `套路库/`、`剧情库/`、`references/元爽点-变体映射表.md` → 退回补查 |
| ❌3 | **文件泛滥** — 不再单独创建“剧情种子清单 / 分幕规划 / chekhov-tracker”，统一回写卷纲或幕纲 |
| ❌4 | **剧情线没有独立文档** — 主线/支线只写在卷纲或 Canvas 里 → 退回，补 `剧情设计/剧情线/*.md` |
| ❌5 | **Canvas 缺失或空转** — 幕纲没有 Canvas，或 Canvas 只列章号不标 payoff/活跃线 → 退回补齐 |
| ❌6 | **章锚点越权写成设计包** — 幕纲只给章级目标/钩子/Canvas，不写完整事件链和正文风格 |
| ❌7 | **枪链游离在外** — 枪链必须并入幕纲，同屏检查设伏、回收窗口和 Canvas payoff |

## 核心流程

### 步骤1：创建卷纲 · 战略层
**读什么：** `全书立项PRD.md`、`小说世界设定/L1-01~06.md`、`状态/角色/*.md`、`小说世界设定/数值体系/*.md`
**做什么：** 锁定本卷核心命题、读者感受、起点、终点、素材需求
**更新/产出：** `剧情设计/卷/卷{N}-卷纲.md`
**❌ 门禁：** 卷目标、起点、终点任一不清 → 不进入剧情卡筛选

详细指令见 `steps/step-1-volume-strategy.md`。

### 步骤2：更新卷纲 · 剧情卡筛选与卷级特化
**读什么：** 卷纲、`素材储备池/*.md`、`pop-trope-library/套路库/`、`pop-trope-library/剧情库/`
**做什么：** 根据卷战略筛选已初改剧情卡，写清本卷取舍、特化方向和候选主/支线雏形
**更新/产出：** `剧情设计/卷/卷{N}-卷纲.md`
**❌ 门禁：** 只列剧情卡名字、没有“为什么适合本卷 + 本卷怎么特化成候选线” → 退回补卷级特化

详细指令见 `steps/step-2-seed-pull.md`。

### 步骤3：主支线剧情线成文
**读什么：** 卷纲候选线雏形、角色池、数值体系、素材储备池、查库结果
**做什么：** 每条主线/支线独立成文，写清驱动力、门槛、时间轴、人物、套路链、枪链、起终点切片
**产出：** `剧情设计/剧情线/{主线,支线}-{编号}-{名称}.md`
**❌ 门禁：** 不允许只在 Canvas 或幕纲里定义剧情线

详细指令见 `steps/step-3-plotline-docs.md`。

### 步骤4：创建幕纲 · 分幕层
**读什么：** 全部剧情线文档 + 卷纲
**做什么：** 切分本卷幕结构，标注每幕戏剧功能、活跃主线/支线、核心冲突
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`
**❌ 门禁：** 每幕至少 1 主线 + 2 支线/副线活跃，否则退回补线或重切幕

详细指令见 `steps/step-4-act-plan.md`。

### 步骤5：更新幕纲 · 章锚点与 Canvas
**读什么：** 幕纲、剧情线文档（含每条线的契诃夫枪链）、`rank_schedule`
**做什么：** 写章锚点、Canvas 矩阵、节奏评估；从剧情线提取枪点为回收章预留 payoff 容量
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`
**❌ 门禁：** 每章至少 1 条线 payoff ≥ 中；任一线连续 3 章空铺垫后必须释放或说明缓冲原因；枪点回收章无 payoff 预留退回

详细指令见 `steps/step-5-chapter-anchors.md`。

### 步骤6：验证幕纲 · Canvas 节奏与爽点密度
**读什么：** 幕纲 Canvas、`references/payoff-design-guide.md`、剧情线枪链
**做什么：** 对照 payoff 设计规范逐项验核：中爽点密度、大爽点间隔、特大爽点汇聚、全书承诺兑现、线间平衡、枪链子项
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`（验核报告段）
**❌ 门禁：** 主线无枪链退回 Step3；其余密度/间隔/汇聚不达标标 ⚠️ 不阻塞

详细指令见 `steps/step-6-chekhov.md`。

## 文件职责边界

| 文件 | 管什么 | 不管什么 |
|:-----|:-------|:---------|
| 卷纲 | 卷战略、剧情卡筛选、卷级特化、候选线雏形 | 不写完整剧情线，不分章调度 |
| 剧情线文档 | 定义一条线写什么、为什么存在、怎么收束 | 不逐章调度密度 |
| 幕纲 | 分幕、章锚点、Canvas、枪链追踪 | 不替代剧情线，不写完整章节事件链 |
| 章节设计包 | 把锚点落成事件链 | 不重定卷战略和线结构 |

## Canvas 定义

> ❌ 因为每条剧情线已有独立文档，就删掉 Canvas
> ✅ 保留 Canvas，用它检查每章剧情密度、每条线连续空白、payoff 释放和多线汇聚
> ★ Canvas 设计时纳入剧情线枪链：枪点回收章预留 payoff 容量，同章回收不超过 2 个

> ❌ 枪链单独放 `chekhov-tracker.md`，导致和 Canvas 脱节
> ✅ 枪链并入幕纲，与章锚点和 Canvas 同屏维护。★ Step5 Canvas 设计时就消费枪链，Step6 只做验证。

## 边界情况

| 情况 | 判断 | 动作 |
|:-----|:-----|:-----|
| 数据不足 | 素材储备池没有可用剧情卡 | 在卷纲标注库缺，先用 PRD + 通用套路生成临时候选线 |
| 剧情库缺项 | 查不到同赛道剧情单元 | 明示库缺，不伪造来源 |
| 循环依赖 | 剧情线等角色，角色又等剧情线 | 先用现有角色卡定低保版本，标 `待角色升级` |
| 用户中断 | 用户改卷目标或主元爽点 | 回到步骤1，更新卷纲并重算候选线 |
| 子 skill 不可用 | trope-library 不可访问 | 明示降级，保留“未查库风险”段 |
| Canvas 过空 | 连续章节无中爽点 | 提前最近线释放，或加入功能支线 |
| 格式不匹配 | 旧项目仍有 `卷{N}-战略定位.md` / `剧情种子拉取清单.md` / `chekhov-tracker.md` | 不强删项目文件，迁移内容到卷纲或幕纲 |
