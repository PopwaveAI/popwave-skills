## 核心流程

### 步骤0：设计意图捕获

> **在写任何卷纲之前，先捕获用户对全书设计的顶层意图。**
> 这个步骤不做→后续所有产出的线结构都是"agent 默认值"→用户大概率不满意。

**读什么：** 全书立项PRD.md、角色卡、项目总控.md
**做什么：** 先问用户四个问题：
1. **主线结构意图**："你心中全书的顶层主线有几条？" — 例如：世界危机线 + 主角成长线 + 主角行动线。它们之间的关系是什么？每条线的卷一职责是什么？
2. **卷节奏意图**："你期望每卷的节奏感是什么？" — 快节奏升级 / 慢热铺垫 / 交替呼吸？
3. **爽点优先级意图**："PRD 列了多个元爽点，你心中卷一最优先兑现哪个？"
4. **角色弧线意图**："主角在卷一的弧线方向是什么？" — 崛起 / 挫折 / 蜕变 / 复仇？

**产出：** 在内存中建立 `用户设计意图模型`，在 Step 1 卷纲中逐条体现

**⛔ 门禁：** 用户给了明确意图前不进 Step 1。如果用户说"你定吧"——至少把你的默认结构给用户过目。

**典型案例（2026-06-22 东方版深渊主宰）：**
```
用户想要：
  主线1 = 世界危机（圣者浩劫真相+归墟扩张）
  主线2 = 主角成长线（力量升级弧线）
  主线3 = 主角行动线（基于过期剧本的决策链）
agent 建的：
  1条主线（江轩成长） + 6条支线/设定线/反派线
→ 结构错配直到 prose 阶段才暴露 → 全链断裂
→ 正确做法：Step 0 花 2 分钟确认上述四个问题
```

**适用场景：** 每个新书的 plot 阶段必须执行。如果是从 deconstructor/拆书数据生成 plot，也先问用户"保留原书线结构还是重新设计"。

### 步骤1：创建卷纲 · 战略层
**读什么：** `全书立项PRD.md`、`小说世界设定/L1-01~06.md`、`状态/角色/*.md`、`小说世界设定/数值体系/*.md`
**做什么：** 锁定本卷核心命题、读者感受、起点、终点、线数配置、篇幅预算、素材需求
**更新/产出：** `剧情设计/卷/卷{N}-卷纲.md`
**❌ 门禁：** 卷目标、起点、终点、线数配置、篇幅预算任一不清 → 不进入剧情卡筛选

详细指令见 `steps/step-1-volume-strategy.md`。

### 步骤2：更新卷纲 · 剧情卡筛选与卷级特化
**读什么：** 卷纲、`素材储备池/*.md`、`pop-trope-library/套路库/`、`pop-trope-library/剧情库/`
**做什么：** 根据卷战略筛选已初改剧情卡，写清本卷取舍、特化方向和候选主/支/设定线雏形
**更新/产出：** `剧情设计/卷/卷{N}-卷纲.md`
**❌ 门禁：** 只列剧情卡名字、没有“为什么适合本卷 + 本卷怎么特化成候选线” → 退回补卷级特化

详细指令见 `steps/step-2-seed-pull.md`。

### 步骤3：剧情线成文
**读什么：** 卷纲候选线雏形、角色池、数值体系、素材储备池、查库结果
**做什么：** 每条主线/支线/设定线独立成文，写清驱动力、门槛、时间轴、地点、人物、套路链、叙事债务、起终点切片
**产出：** `剧情设计/剧情线/{主线,支线,设定线}-{编号}-{名称}.md`
**❌ 门禁：** 不允许只在 Canvas 或幕纲里定义剧情线

详细指令见 `steps/step-3-plotline-docs.md`。

### 步骤4：创建幕纲 · 分幕层
**读什么：** 全部剧情线文档 + 卷纲
**做什么：** 切分本卷幕结构，标注每幕戏剧功能、活跃线、篇幅预算
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`
**❌ 门禁：** 每幕至少 1 主线 + 2 副线活跃；幕内活跃线超过 5 条需合并/错峰

详细指令见 `steps/step-4-act-plan.md`。

### 步骤5：更新幕纲 · 章锚点与 Canvas
**读什么：** 幕纲、剧情线文档（含每条线的叙事债务与回收链）、`rank_schedule`
**做什么：** 写章锚点、Canvas 矩阵、节奏评估；从剧情线提取叙事债务，为回收章预留 payoff 或设定验证容量
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`
**❌ 门禁：** 每章至少 1 条线 payoff ≥ 中；任一线连续 3 章空铺垫后必须释放或说明缓冲原因；叙事债务回收章无容量预留退回

详细指令见 `steps/step-5-chapter-anchors.md`。

### 步骤6：验证幕纲 · Canvas 节奏与爽点密度 + PRD 爽点一致性审计
**读什么：** 幕纲 Canvas、`references/payoff-design-guide.md`、剧情线叙事债务
**做什么：** 对照 payoff 设计规范逐项验核：中爽点密度、大爽点间隔、特大爽点汇聚、全书承诺兑现、线间平衡、叙事债务回收
**更新/产出：** `剧情设计/幕/vol-XX/act-YY.md`（验核报告段）
**❌ 门禁：** 主线无叙事债务链、设定/背景线无验证节点退回 Step3；其余密度/间隔/汇聚不达标标 ⚠️ 不阻塞

详细指令见 `steps/step-6-canvas-audit.md`。

## 文件职责边界

| 文件 | 管什么 | 不管什么 |
|:-----|:-------|:---------|
| 卷纲 | 卷战略、剧情卡筛选、卷级特化、候选线雏形 | 不写完整剧情线，不分章调度 |
| 剧情线文档 | 定义一条线写什么、为什么存在、怎么收束 | 不逐章调度密度 |
| 幕纲 | 分幕、章锚点、Canvas、叙事债务追踪 | 不替代剧情线，不写完整章节事件链 |
| 章节设计包 | 把锚点落成事件链 | 不重定卷战略和线结构 |

## Canvas 定义

> ❌ 因为每条剧情线已有独立文档，就删掉 Canvas
> ✅ 保留 Canvas，用它检查每章剧情密度、每条线连续空白、payoff 释放和多线汇聚
> ★ Canvas 设计时纳入剧情线叙事债务：债务回收章预留 payoff 或设定验证容量，同章重大债务回收不超过 2 个

> ❌ 把叙事债务简化成契诃夫枪，导致制度、信息差、设定递送、人物选择无法追踪
> ✅ 叙事债务并入幕纲，与章锚点和 Canvas 同屏维护。契诃夫枪只是“道具型债务”。

## 边界情况

| 情况 | 判断 | 动作 |
|:-----|:-----|:-----|
| 数据不足 | 素材储备池没有可用剧情卡 | 在卷纲标注库缺，先用 PRD + 通用套路生成临时候选线 |
| 剧情库缺项 | 查不到同赛道剧情单元 | 明示库缺，不伪造来源 |
| 循环依赖 | 剧情线等角色，角色又等剧情线 | 先用现有角色卡定低保版本，标 `待角色升级` |
| 用户中断 | 用户改卷目标或主元爽点 | 回到步骤1，更新卷纲并重算候选线 |
| 子 skill 不可用 | trope-library 不可访问 | 明示降级，保留“未查库风险”段 |
| Canvas 过空 | 连续章节无中爽点 | 提前最近线释放，或加入功能支线 |
| 格式不匹配 | 旧项目仍有 `卷{N}-战略定位.md` / `剧情种子拉取清单.md` / `chekhov-tracker.md` | 不强删项目文件，迁移内容到卷纲或幕纲 |
| **设定变更（★新增）** | world/character 阶段的上游设定发生了框架级变更（如加穿越者、改力量体系、换核心矛盾） | 先执行 expert-writer 管线回溯协议：
  ① 扫描全部剧情线文档，标注"穿越者/新设定对此线的影响"
  ② 重跑 Step 0（主线结构是否需要调整）
  ③ 更新卷纲战略层（起点/终点/三主线定位）
  ④ 逐线更新剧情线文档（注入新设定引擎）
  ⑤ 逐幕更新幕纲（章锚点/Canvas/叙事债务重新对齐）
  ⑥ 所有变更**按管线顺序自前向后**传播，不跳不过 |
