## Step 0：设计意图捕获

> **你不是在填表。你是这个故事的总导演。**
> 你需要做出的核心决策：全书爽点引擎是什么？每条 L3 线的冲突引擎是什么？每个关键节点是哪个角色的什么决策驱动的？
> 你需要的材料：PRD（宪法）、研究档案（创意根基）、角色卡（驱动力）、trope-library（经验库）。
> **⛔ 进入五个问题前，必须先完成两件事：(1) trope-library 查询声明 (2) 读取角色驱动接口。**

> **在写任何卷纲之前，先捕获用户对全书设计的顶层意图。** 跳过这步→后续所有线结构都是 agent 默认值→用户大概率不满意。

### 前置：识别全书类型

不同类型的书，主线大类结构不同。**不要默认就是三大类——3 个主线大类是通用小说的默认值，不是所有小说的正确值。**

| 书型 | 典型代表 | 主线大类结构差异 | 陷阱 |
|:-----|:---------|:------------|:-----|
| **诸天/副本穿越类** | 武林半侠传、无限恐怖 | 副本世界应单独作为一个主线大类（每个副本世界一条具体剧情线）。**副本世界本身就是全书最大卖点，不是主角行动的附庸** | ❌ 把副本世界降级为「主角行动」下属的一条具体剧情线「副本收割循环」，等于把内容支柱变成流程描述 |
| **传统修仙/玄幻类** | 凡人修仙传 | 3 个默认主线大类基本适用 | — |
| **都市/异能类** | — | 3 个默认主线大类基本适用 | — |
| **历史/权谋类** | — | 主角行动和世界大事件的边界模糊，优先靠 Step 0 确认 | — |

**陷阱复盘（2026-06-23 武林半侠传）：** agent 把 5 个副本世界合并成一条具体剧情线「副本世界收割循环线」归入主线大类主角行动。用户纠正：「每个副本世界本身就是全书最大卖点——它不应该是收割循环，它是内容本身。」正确做法：诸天流中副本世界是独立主线大类，每个世界一条具体剧情线。

### 五个问题

1. **主线大类意图** — 根据确定的全书类型，确认主线大类结构。如果非通用类型（如诸天流），主动提出额外的主线大类建议：
   - 通用默认：主线大类·世界大事件 / 主线大类·主角成长 / 主线大类·主角行动
   - 诸天/副本流额外主线大类：**主线大类·副本世界**（每个副本世界一条独立具体剧情线，含冲突引擎/读者追问/发展骨架）
   - 合不进主线大类的剧情线算独立支线（无主线大类，直接具体剧情线→剧情节点）。
2. **卷节奏意图** — 快节奏升级 / 慢热铺垫 / 交替呼吸？
3. **爽点优先级意图** — PRD 列了多个元爽点，卷一最优先兑现哪个？
4. **角色弧线意图** — 崛起 / 挫折 / 蜕变 / 复仇？
5. **全书大事件意图** — 从 PRD 识别全书级跨卷大事件（L4 全书事件）。
   - 有哪些跨卷的大事件？（如"圣者浩劫""演武令终极秘密"）
   - 每个大事件分几个演化阶段？分别跨越哪些卷？
   - 每个阶段由哪些 L3 剧情线承载？
   - 如果只有单卷或无跨卷大事件，标记为"无 L4 全书事件"（不是所有书都有跨卷大事件）

**⛔ 门禁：** 用户确认前不进 Step 1。用户说"你定吧"→先做书型判断再执行，但需告知用户判断结果并获得确认。

**典型案例（2026-06-22 东方版深渊主宰）：** agent 把主线大类当成具体剧情线，1主线+6支线 → 结构错配直到 prose 才暴露 → 全链断裂。正确做法：Step 0 花 2 分钟确认主线大类 + 具体剧情线初步清单。

## 文件职责边界

| 文件 | 管什么 | 不管什么 |
|:-----|:-------|:---------|
| L4 全书事件 | 跨卷大事件演化阶段、多线交汇、全局影响 | 不写单卷战略，不分章调度 |
| 卷纲 | 卷战略、剧情卡筛选、卷级特化、候选线雏形、引用 L4 parent_saga | 不写完整剧情线，不分章调度 |
| L3 剧情线文档 | 定义一条线怎么发展、怎么收束；L3-ID + parent_saga + 组成单元 | 不逐章调度密度 |
| L2 剧情单元卡 | 结构分析（7阶段+情绪弧线+主题）+ 执行调度（章锚点+Canvas+债务追踪） | 不替代剧情线，不写完整事件链 |
| 章节设计包 | 把锚点落成事件链 | 不重定卷战略和线结构 |

| 循环依赖（剧情线等角色） | 先用现有角色卡定低保版本，标 `待角色升级` |
| 用户中断/改卷目标 | 回到 Step 0，更新意图模型后重算 |
| trope-library 不可访问 | 明示降级，保留"未查库风险"段 |
| Canvas 过空（连续章无中爽点） | 提前最近线释放，或加入功能支线 |
| **上游框架级变更**（加穿越者/改力量体系/换核心矛盾） | 执行 expert-writer 回溯协议：扫描全部剧情线→重跑Step 0→更新卷纲→逐线更新→逐幕更新→按管线顺序传播 |
