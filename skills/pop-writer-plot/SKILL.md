# pop-writer-plot · 剧情架构设计 v7.1.0

> **定位：从 PRD、世界设定、角色卡、素材储备池和 trope-library，产出卷战略、剧情线、分幕、章锚点、Canvas 节奏与契诃夫枪链。**
> **核心约束：Canvas 不废除。剧情线独立文档管内容，Canvas 管章节密度、线间节奏和 payoff 释放。**

## 核心流程

### 步骤1：卷战略定位
**读什么：** `全书立项PRD.md`、`小说世界设定/L1-01~06.md`、`状态/角色/*.md`、`小说世界设定/数值体系/*.md`
**做什么：** 锁定本卷核心命题、读者感受、力量/关系/认知边界
**产出：** `剧情设计/卷/卷{N}-战略定位.md`
**❌ 门禁：** 卷目标、起点、终点任一不清 → 不进入拉种子

详细指令见 `steps/step-1-volume-strategy.md`。

### 步骤2：拉种子与查库
**读什么：** `素材储备池/*.md`、`pop-trope-library/套路库/`、`pop-trope-library/剧情库/`
**做什么：** 选择剧情储备卡，匹配套路链和剧情单元参考
**产出：** `剧情设计/卷/卷{N}-剧情种子拉取清单.md`
**❌ 门禁：** 查询无匹配素材时必须标注“本赛道/元爽点库缺”

详细指令见 `steps/step-2-seed-pull.md`。

### 步骤3：写剧情线文档
**读什么：** 种子清单、角色池、数值体系、素材储备池、查库结果
**做什么：** 每条主线/支线独立成文，写清驱动力、门槛、时间轴、人物、套路链、枪链、起终点切片
**产出：** `剧情设计/剧情线/{主线,支线}-{编号}-{名称}.md`
**❌ 门禁：** 不允许只在 Canvas 或分幕表里定义剧情线

详细指令见 `steps/step-3-plotline-docs.md`。

### 步骤4：分幕规划
**读什么：** 全部剧情线文档 + 卷战略定位
**做什么：** 切分本卷幕结构，标注每幕戏剧功能、活跃主线/支线、核心冲突
**产出：** `剧情设计/幕/vol-XX/分幕规划.md`
**❌ 门禁：** 每幕至少 1 主线 + 2 支线/副线活跃，否则退回补线或重切幕

详细指令见 `steps/step-4-act-plan.md`。

### 步骤5：章锚点与 Canvas
**读什么：** 分幕规划、剧情线文档、`rank_schedule`
**做什么：** 为每章设定情绪目标、爽点目标、钩子、活跃线，并用 Canvas 检查剧情密度和每条线的铺垫/释放节奏
**产出：** `剧情设计/幕/vol-XX/act-YY.md`
**❌ 门禁：** 每章至少 1 条线 payoff ≥ 中；任一线连续 3 章空铺垫后必须释放或说明缓冲原因

详细指令见 `steps/step-5-chapter-anchors.md`。

### 步骤6：契诃夫枪链
**读什么：** 剧情线文档中的枪链 + act 章锚点
**做什么：** 汇总设伏、回收窗口、状态和跨幕风险
**产出：** `剧情设计/幕/vol-XX/chekhov-tracker.md`
**❌ 门禁：** 主线没有枪链或枪链没有回收窗口 → 退回补齐

详细指令见 `steps/step-6-chekhov.md`。

## Canvas 定义

| 对象 | 职责 | 不负责 |
|:-----|:-----|:-------|
| 剧情线文档 | 定义一条线写什么、为什么存在、怎么收束 | 不逐章调度密度 |
| 分幕规划 | 定义一幕里哪些线活跃、冲突如何升级 | 不替代每章锚点 |
| Canvas | 逐章查看线的活跃、铺垫、释放、空白和多线汇聚 | 不承载完整剧情线内容 |
| 章节设计包 | 把锚点落成事件链 | 不重定卷战略和线结构 |

## ❌ WRONG 示例

> ❌ 因为每条剧情线已有独立文档，就删掉 Canvas
> ✅ 保留 Canvas，用它检查每章剧情密度、每条线连续空白、payoff 释放和多线汇聚

> ❌ 只写 `act-YY.md` 的 Canvas，不写 `剧情设计/剧情线/*.md`
> ✅ 先写剧情线独立文档，再让 Canvas 承接它们的按章节奏

> ❌ 查不到剧情库匹配项时静默自创
> ✅ 标注“本赛道/元爽点库缺”，再降级使用通用素材

## 边界条件

| 场景 | 触发条件 | 处理 |
|:-----|:---------|:-----|
| 前置文件缺失 | PRD/L1/角色卡/数值体系/素材池任一缺失 | 列缺口，停止进入剧情设计 |
| 数据不足 | 素材储备池没有可用剧情卡 | 标注库缺，先用 PRD + 通用套路生成临时线 |
| 格式不匹配 | 旧项目仍有 `volume-XX.md` 或 `act-YY.yaml` | 不强删，迁移为新路径下的战略/act.md |
| 循环依赖 | 剧情线等角色，角色又等剧情线 | 先用现有角色卡定低保版本，标 `待角色升级` |
| 用户中断 | 用户改卷目标或主元爽点 | 回到步骤1，重算种子和剧情线 |
| 子 skill 不可用 | trope-library 不可访问 | 明示降级，保留“未查库风险”段 |
| Canvas 过密 | 单章 ≥4 条线同时释放 | 拆分或转为两章释放，避免信息过载 |
| Canvas 过空 | 连续章节无中爽点 | 提前最近线释放，或加入功能支线 |
