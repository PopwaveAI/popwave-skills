## Step 0：设计意图捕获

> **在写任何卷纲之前，先捕获用户对全书设计的顶层意图。** 跳过这步→后续所有线结构都是 agent 默认值→用户大概率不满意。

先判断**全书类型**，再问四个问题：

### 前置：识别全书类型

不同类型的书，L1 大类结构不同。**不要默认就是三大类——3 个 L1 是通用小说的默认值，不是所有小说的正确值。**

| 书型 | 典型代表 | L1 结构差异 | 陷阱 |
|:-----|:---------|:------------|:-----|
| **诸天/副本穿越类** | 武林半侠传、无限恐怖 | 副本世界应单独作为一个 L1（每个副本世界一条 L2）。**副本世界本身就是全书最大卖点，不是主角行动的附庸** | ❌ 把副本世界降级为「主角行动」下属的一条 L2「副本收割循环」，等于把内容支柱变成流程描述 |
| **传统修仙/玄幻类** | 凡人修仙传 | 3 个默认 L1 基本适用 | — |
| **都市/异能类** | — | 3 个默认 L1 基本适用 | — |
| **历史/权谋类** | — | 主角行动和世界大事件的边界模糊，优先靠 Step 0 确认 | — |

**陷阱复盘（2026-06-23 武林半侠传）：** agent 把 5 个副本世界合并成一条 L2「副本世界收割循环线」归入 L1 主角行动。用户纠正：「每个副本世界本身就是全书最大卖点——它不应该是收割循环，它是内容本身。」正确做法：诸天流中副本世界是独立 L1，每个世界一条 L2 剧情线。

### 四个问题

1. **主线大类意图** — 根据确定的全书类型，确认 L1 大类结构。如果非通用类型（如诸天流），主动提出额外的 L1 建议：
   - 通用默认：L1 · 世界大事件 / L1 · 主角成长 / L1 · 主角行动
   - 诸天/副本流额外 L1：**L1 · 副本世界**（每个副本世界一条独立 L2，含冲突引擎/读者追问/发展骨架）
   - 合不进 L1 的剧情线算独立支线（无 L1，直接 L2→L3）。
2. **卷节奏意图** — 快节奏升级 / 慢热铺垫 / 交替呼吸？
3. **爽点优先级意图** — PRD 列了多个元爽点，卷一最优先兑现哪个？
4. **角色弧线意图** — 崛起 / 挫折 / 蜕变 / 复仇？

**⛔ 门禁：** 用户确认前不进 Step 1。用户说"你定吧"→先做书型判断再执行，但需告知用户判断结果并获得确认。

**典型案例（2026-06-22 东方版深渊主宰）：** agent 把 L1 大类当成 L2 具体剧情线，1主线+6支线 → 结构错配直到 prose 才暴露 → 全链断裂。正确做法：Step 0 花 2 分钟确认 L1 + L2 初步清单。

## 文件职责边界

| 文件 | 管什么 | 不管什么 |
|:-----|:-------|:---------|
| 卷纲 | 卷战略、剧情卡筛选、卷级特化、候选线雏形 | 不写完整剧情线，不分章调度 |
| 剧情线文档 | 定义一条线怎么发展、怎么收束 | 不逐章调度密度 |
| 幕纲 | 分幕、章锚点、Canvas、叙事债务追踪 | 不替代剧情线，不写完整事件链 |
| 章节设计包 | 把锚点落成事件链 | 不重定卷战略和线结构 |

## 边界情况

| 情况 | 动作 |
|:-----|:-----|
| 素材储备池无可用剧情卡 | 卷纲标注库缺，用 PRD + 通用套路生成临时候选线 |
| 剧情库缺项 | 明示库缺，不伪造来源 |
| 循环依赖（剧情线等角色） | 先用现有角色卡定低保版本，标 `待角色升级` |
| 用户中断/改卷目标 | 回到 Step 0，更新意图模型后重算 |
| trope-library 不可访问 | 明示降级，保留"未查库风险"段 |
| Canvas 过空（连续章无中爽点） | 提前最近线释放，或加入功能支线 |
| **上游框架级变更**（加穿越者/改力量体系/换核心矛盾） | 执行 expert-writer 回溯协议：扫描全部剧情线→重跑Step 0→更新卷纲→逐线更新→逐幕更新→按管线顺序传播 |
