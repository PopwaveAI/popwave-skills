# pop-writer-plot · 剧情架构设计 v6.5.0

> **卷 = 战略** — 目标、快照（起点→终点约束）、背景（时间/地理/角色/势力动机）、剧情线列表。  
> **幕 = 设计规格书** — 幕纲是 design specification，不是 data catalog。起点/终点快照是**设计包的输入契约**，不是冗余数据转储。  
> **章级切片归属**：章级逐章切片属于 design pack（pop-writer-chapter），不在幕纲中展开。
>
> **管线**：`卷纲 → 幕纲（设计规格）→ 设计包（按幕纲执行）→ 正文`
>
> 下游（pop-writer-chapter）读：`设计/卷/volume-XX.md`（卷级约束）+ `设计/幕/vol-XX/act-YY.md`（幕级设计规格）

## 0. 设计理念

卷（80-150章）= 战略，幕（20-35章）= 战术。一卷 = 3-6 幕。

```
卷 → `设计/卷/volume-XX.md`: 目标/背景(时间+地理+角色池+势力动机)/剧情线(主线1-3+分支线)/状态(开局→结束)
幕 → `设计/幕/vol-XX/act-YY.md`: Canvas矩阵(章节×剧情线·payoff释放图)/设计总纲(核心命题/副命题矩阵+设计意图)/设计约束(起点→终点快照)/剧情线阶段(含设计决策+指引)/情绪弧线与节奏/伏笔规划/渲染指引
```

### 素材储备池消费指南（creative v2.0 新增）

creative 产出的 `素材储备池.md` 包含四类剧情种子。plot 设计时应按以下方式拉取：

| 池分类 | 拉取时机 | 用法 |
|:-------|:---------|:-----|
| **主线索材** | Phase 0 全书架构 | 评估是否可以作为某卷的主线冲突。如素材已有完整叙事潜能 → 分配为主线3（主角行动）或支线 | 
| **支线索材** | Phase 1 卷设计 | 分配给具体卷的支线池。每卷选 1-2 个支线索材搭配使用 |
| **背景素材** | Phase 1 卷设计 | 嵌入 L1 维度对应的卷背景中（势力格局/资源特征等），不作为剧情驱动力 |
| **剩余焊接点** | Phase 2 幕纲编排 | 作为某幕的剧情转折或中期爆发点使用。焊接点天然带有跨界张力 → 适合做幕级高潮 |

**拉取纪律**：
- 素材储备池中的条目按优先级标注。优先拉取高优先级
- 拉取后在池中标注"已使用-卷X-幕Y"，避免重复
- 如果池中缺少 plot 需要的某种素材 → 自行发明，不退回 creative

爽点四级定义 + 方法论 → `references/payoff-design-guide.md`

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| 1 | **PRD + L1 设定已存在** — bootstrap 没过不进 plot |
| 2 | **卷设计已产出** — volume-XX.md 包含目标/背景/剧情线/快照 |
| 3 | **每卷至少 主线1/主线2/主线3 三条主线** — 世界危机 + 主角成长 + 主角行动 为必选，剧情线须经用户确认 |
| 4 | **每卷分支线（支线）≥ 3 条** — 支线必须与主线并行推进，任一幕内活跃支线不可少于 3 条。支线定义：与主要剧情无强绑定关系的次级冲突（如配角弧线、地方事件、世界观揭露线、次要势力博弈）。**连续 3 幕以上不足 3 条 → 退回补充** |
| 5 | **Canvas 矩阵已填充** — act-YY.md 中须有章节×剧情线的 payoff 释放图（行=剧情线，列=章号，单元格=payoff_level）。**每章至少 1 条线标为"中"或以上**，连续空白章≤2章。|
| 6 | **每章有 emotional_goal + payoff + end_hook** — act-*.md 章级切片齐备 |
| 7 | **爽点密度达标** — 小≥5/章 + 中≥1/章 / 大间隔≤5章 + 特大≥1/卷 |
| 8 | **幕内无连续 3 章同一情绪叠加组合** |
| 9 | **连续 2 章无信息释放 → 第 3 章必追加** |
| 10 | **首卷黄金窗口（vol-01/act-01）** — ch01 核心卖点亮相 + 钩子 / ch02 第一次正面战斗冲突（双方交手的战斗，非被动逃跑/致盲闪避/纯信息冲突。及格线：首杀混沌单位、首次主动施法对战、绝境反击。正面战斗最晚不超过ch03）/ 连续铺垫 ≤ 1章。任一违规 → P0 退回 |
| 11 | **素材储备池未读不进全书架构** — creative 产出的素材储备池（`素材储备池.md`）包含给 plot 的剧情种子。全书架构前必须先读储备池，拉取可用的主线索材/支线索材/剩余焊接点。不读 = 浪费燃料 |

| 阶段 | 步骤 | 前置读 | 产出 | 闸门 |

## ❌ 错误示例

| ❌ 错误 | 问题 | ✅ 正确 |
|:--------|:-----|:--------|
| 幕内连续放空（主线连续 4 章无进展） | 读者遗忘这条线 | 每条线每幕至少出现 1 次，连续空白 ≤ 3 章 |
| Canvas 矩阵留白（剧情线行全空） | 该线在本幕断掉 | 所有剧情线行至少有 1 个交点 |
| 幕纲格式错误（用 YAML 而非 Markdown） | 格式不兼容 | 使用 `设计/幕/vol-XX/act-YY.md` Markdown 格式 |
| 主线只设 2 条 | 剧情感单薄 | 必须产出 主线1（世界危机）+ 主线2（主角成长）+ 主线3（主角行动） |
| 地理用文本链而非 SVG 地图 | 无视觉定位 | 在 `三、关键背景` 中直接嵌入内联 SVG 标明相对位置 |
| Canvas矩阵缺少节奏评估 | 无法判断节奏健康度 | Canvas后紧跟节奏评估表（连续空白/大释放间隔/支线交替活跃度） |

## 异常与边界条件

| 场景 | 触发条件 | 处理方式 |
|:-----|:---------|:---------|
| 用户跳步（未做全书架构直接要卷设计） | Phase 0 产出不存在 | 拒绝，提示先完成 Phase 0 |
| 用户跳步（未做卷设计直接要幕纲） | volume-XX.md 不存在 | 拒绝，提示先完成卷定义 |
| act-*.md 格式错误（用 YAML 而非 Markdown） | 写入前检查，确保使用纯 Markdown |
| 幕内总章数超出卷章数预算 | act 覆盖章数 > volume 分配范围 | 提示预算超限，建议拆分幕或调整卷 |
| 剧情线确认闸门未通过 | 用户未确认三条主线 | 阻塞，等待确认 |
| 首卷黄金窗口违规 | ch01 无核心卖点/ch02 无正面战斗冲突/连续铺垫>1章/首章纯信息章 | P0 退回，重新设计前两章 |
| Canvas 维度不匹配 | 剧情线数量 ≠ canvas.entries 行数 | 自检报错，修复对齐 |
| 跨卷角色出场节奏断裂 | 角色在角色池但本卷无出场计划 | 警告，提示补充或移除 |
| Canvas矩阵连续空白章过多 | 同一章所有线都为空 | 把最近一条快释放的线提前到该章释放 |

## 本阶段边界

**做：** 全书架构 → 卷设计 → 幕纲编排 + 首卷黄金窗口检查
**不做：** ❌ 不设计章级细节（chapter-design） ❌ 不写正文（prose-render）

**越界检测：** "第X章怎么写/这章几个事件" → "这属于 chapter-design，先确认卷和幕结构。"
