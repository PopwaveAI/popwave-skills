# 主线大类分类指南：按书型判断主线大类结构

> 用途：Step 0 书型检测 + Step 3 剧情线成文时的主线大类分类参考。
> 核心原则：**不要默认就是三大类。3 个主线大类是通用小说的默认值，不是所有小说的正确值。**

## 书型↔主线大类 映射表

| 书型 | 适用主线大类 | 典型具体剧情线示例 | 说明 |
|:-----|:------------|:-------------|:-----|
| **通用（传统修仙/玄幻/都市）** | 世界大事件、主角成长、主角行动 | 血脉觉醒、帮会争霸、境界突破 | 3 个默认主线大类基本适用 |
| **诸天/副本穿越流** | 世界大事件、**副本世界**、主角成长、主角行动 | 碧血剑篇、侠客岛篇、笑傲江湖篇 | **副本世界是独立主线大类——每个副本世界是一条具体剧情线，不是主角行动的附庸** |
| **无限流** | 同上 | 生化危机副本、咒怨副本 | 同上，副本世界=内容本体 |
| **历史/权谋流** | 世界大事件、主角行动 | 朝堂博弈、边境战争 | 主角成长线可能弱化，主角行动和世界大事件边界模糊 |
| **日常/轻松流** | 主角行动、情感羁绊 | 开店经营、乡村日常 | 世界大事件可能缺失，以关系线和行动线为主 |

## 陷阱案例

### 案例1：诸天流把副本世界降级为「主角行动」的子类

**场景：** 武林半侠传（文抄公），5 个武侠副本世界（碧血剑/古龙/侠客岛/笑傲/多情）

**错误做法：** agent 把 5 个副本世界合并为一条具体剧情线「副本世界收割循环线」归入主线大类主角行动。表格写成：

```
主线大类主角行动 → 具体剧情线 副本世界收割循环线（13节点：穿越→修炼→回归→兑现）
```

**用户纠正：** 「他最精彩的副本世界，为什么你采集不到呢？它不应该是副本世界收割循环——副本世界本身就是全书最大卖点啊。」

**根因：** agent 把副本世界抽象成了机械流程（副本→修炼→回归），忽略了**每个副本世界本身就是独立的内容支柱**。读者追的就是「看主角怎么进这个世界搞事」，不是看收割流程。

**正确做法：**

```
主线大类·副本世界（独立主线大类）
  → 具体剧情线: 碧血剑篇（ch002-ch031）
  → 具体剧情线: 古龙综合篇（ch054-ch079）
  → 具体剧情线: 侠客岛篇（ch079-ch098）
  → 具体剧情线: 笑傲江湖篇（ch098-ch117）
  → 具体剧情线: 多情剑客无情剑篇（ch118-ch142）

主线大类·主角成长
  → 具体剧情线: 武道跃迁线（贯穿）
  → 具体剧情线: 金手指深化线（贯穿）

主线大类·世界大事件
  → 具体剧情线: 七魔门/七绝堂暗线（萌芽）

主线大类·主角行动
  → 具体剧情线: 双身份势力扩张线（贯穿）
  → 具体剧情线: 神刀教/大江盟格局线（贯穿）
```

**检测信号：** 如果卷纲中多次出现「穿越新世界」「进入XX副本」「回归主世界」等事件，且副本世界数量 ≥ 3 个，应考虑将副本世界提升为独立主线大类。

### 案例2：主线大类层级错配（东方版深渊主宰，2026-06-22）

**场景：** agent 把主线大类（世界大事件）和具体剧情线（剥皮者渗透）混淆，产出 1 条主线 + 6 条支线，直到 prose 阶段才发现结构错配。

**根因：** Step 0 跳过 → agent 按自己的默认值写结构 → 结构错配在整条 pipe 末端才暴露。

## 使用流程

1. Step 0 中先判断书型
2. 按上表选择主线大类结构
3. 告知用户选择的主线大类结构并确认
4. 用户确认后，Step 3 按此结构展开具体剧情线
