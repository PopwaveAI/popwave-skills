# 步骤2：更新卷纲 · 剧情卡筛选与卷级特化

> **更新：** `剧情设计/卷/卷{N}-卷纲.md`
> **模板：** `templates/volume-outline.md`
> **核心约束：** Step2 不新建“剧情种子清单”文件；在卷纲中追加剧情卡筛选、卷级特化和 Step3 候选线雏形。

## 读什么

- `剧情设计/卷/卷{N}-卷纲.md`
- `素材储备池/*.md`
- `pop-trope-library/套路库/00-总索引.md`
- `pop-trope-library/剧情库/`
- `pop-trope-library/references/元爽点-变体映射表.md`

## 做什么

1. 从卷纲战略层提取本卷命题、主元爽点、主冲突、角色成长压力、终点形态、线数配置。
2. 从素材储备池筛选状态可用、与卷战略匹配的剧情储备卡。
3. 按本卷主元爽点查 `套路库/`，记录可用套路链与不适配套路。
4. 按赛道、元爽点、冲突类型查 `剧情库/`，读取可改建剧情单元。
5. 对候选剧情卡做卷级特化：本卷取什么、不取什么、换成什么、承担什么功能。
6. 标注候选归属：主线 / 支线 / 设定线 / 背景线 / 关系线 / 成长线 / 反派线。
7. 写出 Step3 候选线雏形：线名建议、核心承诺/递送承诺、冲突引擎/递送机制、读者主追问、起点、终点、3 个推进节点、预期 payoff/理解收益、建议咬合线。
8. 对照卷纲线数预算，检查候选线是否不足或过载。

## 与 Step3 的边界

| 项目 | Step2 做 | Step3 做 |
|:-----|:---------|:---------|
| 剧情卡 | 筛选、查重、标注可用性 | 不再重新筛卡 |
| 卷级适配 | 写清本卷取舍与特化方向 | 沿用，不推翻 |
| 主/支/设定线 | 只给候选线雏形 | 写成独立剧情线文档 |
| 读者追问 | 只给主追问 | 扩展成主追问 + 阶段追问 |
| 发展骨架 | 只给 3 个推进节点 | 扩展成触发、推进、复杂化、反转、高潮、收束 |
| 咬合关系 | 只给建议咬合线 | 写清线间互相递送什么 |
| 叙事债务 | 标注可能的债务类型 | 在剧情线和幕纲中追踪回收 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 未查套路库 | 退回补查 |
| 未查剧情库 | 退回补查 |
| 查无匹配 | 标注“本赛道/元爽点库缺”，降级通用素材 |
| 只列剧情卡名字无卷级特化 | 退回补“为什么适合本卷 + 本卷怎么取舍” |
| 没有 Step3 可消费的候选线雏形 | 退回补线名、冲突引擎、读者追问、推进节点、payoff、建议咬合线 |
| 候选线少于卷纲预算 | 退回继续拉卡或降低卷规模 |
| 候选线多于卷纲预算 2 条以上 | 合并同功能线，避免 Canvas 过载 |

## 输出

更新 `剧情设计/卷/卷{N}-卷纲.md` 的“Step2 剧情卡筛选与卷级特化”。
