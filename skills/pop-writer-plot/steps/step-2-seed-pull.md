# 步骤2：更新卷纲 · 候选线产出

> **更新：** `剧情设计/卷/卷{N}-卷纲.md`
> **模板：** `templates/volume-outline.md`
> **核心目标：** 产出足够数量的候选线雏形供 Step 3 成文。储备池和库只是帮达到这个目标的手段，不是目的——池子不够就自己造。

## 读什么

> 全部读取，无例外。括号内是用途说明，不是优先级。

- `剧情设计/卷/卷{N}-卷纲.md`（Step1 战略层：命题/主元爽点/主冲突/线数预算）
- `素材储备池/*.md`（已有储备卡——优先复用，减少重复造轮子）
- `pop-trope-library/套路库/00-总索引.md`（套路链参考）
- `pop-trope-library/剧情库/`（可改建剧情单元参考）
- `pop-trope-library/references/元爽点-变体映射表.md`（爽点变体启发）

## 做什么

> **本步的唯一交付物是候选线雏形，数量必须 ≥ 卷纲线数预算。** 储备池筛选、库查询都是达成这个目标的手段。

1. 从卷纲战略层提取本卷命题、主元爽点、主冲突、角色成长压力、终点形态、线数配置。
2. **盘点已有储备**：从素材储备池筛选状态可用、与卷战略匹配的剧情储备卡。
3. **查库补充**：按本卷主元爽点查 `套路库/`，按赛道/元爽点/冲突类型查 `剧情库/`，读取可改建剧情单元。
4. **缺口生成**：储备池 + 库的候选线加起来仍不足卷纲预算时，基于 PRD 元爽点 + 卷战略 + 角色压力**直接生成新候选线雏形**。不标注"库缺"就停手。
5. 对全部候选剧情卡做卷级特化：本卷取什么、不取什么、换成什么、承担什么功能。
6. 标注候选归属：主线（所属主线大类）/ 支线（无主线大类）/ 设定线。
7. 写出 Step3 候选线雏形：线名建议、核心承诺/递送承诺、冲突引擎/递送机制、读者主追问、起点、终点、3 个推进节点、预期 payoff/理解收益、建议咬合线。
8. 对照卷纲线数预算，确认候选线数量达标。

## 与 Step3 的边界

| 项目 | Step2 做 | Step3 做 |
|:-----|:---------|:---------|
| 剧情卡 | 筛选、查重、标注可用性 | 不再重新筛卡 |
| 卷级适配 | 写清本卷取舍与特化方向 | 沿用，不推翻 |
| 主/支/设定线 | 只给候选线雏形 | 写成独立剧情线文档 |
| 读者追问 | 只给主追问 | 扩展成主追问 + 阶段追问 |
| 发展骨架 | 只给 3 个推进节点 | 扩展成触发、推进、复杂化、反转、高潮、收束 |
| 咬合关系 | 只给建议咬合线 | 写清线间互相递送什么 |
| 叙事债务 | 标注可能的债务类型 | 在剧情线和幕纲中追踪回收 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 未查套路库 | 退回补查 |
| 未查剧情库 | 退回补查 |
| 查无匹配 | 标注"本赛道/元爽点库缺"，**但不允许停手** — 基于PRD元爽点+卷战略直接生成候选线 |
| 只列剧情卡名字无卷级特化 | 退回补"为什么适合本卷 + 本卷怎么取舍" |
| 没有 Step3 可消费的候选线雏形 | 退回补线名、冲突引擎、读者追问、推进节点、payoff、建议咬合线 |
| **候选线总数 < 卷纲线数预算** | ❌ 退回 — 储备池+库不够就自己生成，必须补齐到预算数量 |
| 候选线多于卷纲预算 2 条以上 | 合并同功能线，避免 Canvas 过载 |

## 输出

更新 `剧情设计/卷/卷{N}-卷纲.md`：确认线数预算 + 在剧情线总览填入候选线名和主线大类归属（brief/节点数等留待Step3回填）。
