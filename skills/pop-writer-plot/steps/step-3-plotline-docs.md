# 步骤3：主支线剧情线成文

> **产出：** `剧情设计/剧情线/{主线,支线}-{编号}-{名称}.md`
> **模板：** `templates/plotline-doc.md`
> **核心约束：** Step3 承接卷纲中的候选线雏形，正式写成独立主线/支线文档；Canvas 不定义剧情线内容，只调度章节节奏。

## 读什么

- `剧情设计/卷/卷{N}-卷纲.md`（读取 Step2 候选线雏形）
- `状态/角色/*.md`
- `小说世界设定/数值体系/*.md`
- 已查到的套路库、剧情库条目

## 做什么

每条主线/支线单独写一个文档。Step3 不重新筛卡、不重定卷战略，只把候选线雏形扩写成六件套：

| 件套 | 要求 |
|:-----|:-----|
| 数值门槛 | 触发、推进、收束分别需要什么能力/等级/条件 |
| 植入时间轴 | 在卷、幕、章的大致位置与阶段 |
| 登场人物 | 谁推动、谁阻碍、谁见证 |
| 套路链 | 每个套路如何部署、预期情绪收益 |
| 契诃夫枪链 | 设伏位置、预期回收、回收方式 |
| 起终点切片 | 本线开局状态与本卷末状态 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 推翻卷纲候选线雏形 | 退回 Step2 重筛，不在 Step3 临场换线 |
| 只写主线无支线 | 检查是否能支撑每幕 1 主线 + 2 副线活跃，不够则补支线 |
| 缺数值门槛 | 回读数值体系补齐 |
| 缺角色承载 | 回读角色卡，或标注待 character 补卡 |
| 无枪链 | 主线必须补至少 1 条枪链 |

## 输出

每条线按 `templates/plotline-doc.md` 写入 `剧情设计/剧情线/`。
