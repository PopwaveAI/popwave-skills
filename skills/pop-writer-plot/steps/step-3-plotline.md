# Step 3：剧情线文档（L3/L2 结构 + 金手指任务驱动节点）

> 管线: pop-writer-plot v2.0.0
> 消费: Step 1 卷战略 + Step 2 危机交叉设计
> 产出: `剧情线/L3-{编号}-{名称}.md`（L3 剧情线）+ L2 剧情单元卡

## 读什么

- `卷/卷{N}-卷纲.md`（卷战略 + 危机交叉设计）
- 危机交叉设计文档（Step 2 产出）
- `状态/角色/*.md`（角色决策转折点预判 + 生态冲突点）
- `templates/剧情线-模板.md`（L3/L2 结构模板）

## 做什么

### 一、L3 剧情线文档（复用 v1 结构）

每条剧情线包含：

| 字段 | 说明 |
|:-----|:------|
| L3-ID | 剧情线编号 |
| 线名 | 剧情线名称 |
| 线型 | 主线 / 副线 / 设定线 / 背景线 |
| 核心驱动力 | 承压者 / 压力源 / 行动方向 / 失败代价 / 终局变化 |
| 骨架节点 | 剧情线的关键节点序列 |
| 叙事债务 | 铺垫与回收链 |

### 二、金手指任务驱动节点（v2 新增）

> v2 核心差异：剧情线中增加金手指任务驱动节点，金手指不只是"工具"而是"任务引擎"。

在 L3 剧情线的骨架节点中标注金手指任务驱动节点：

```
金手指任务驱动节点：
| 节点 | 金手指任务 | 任务来源 | 任务代价 | 完成条件 | 对剧情线的驱动 |
|:---|:---|:---|:---|:---|:---|
| {节点名} | {金手指需要完成什么任务} | {系统/世界/角色驱动} | {不完成的代价} | {完成标准} | {推动剧情线向哪个方向发展} |
```

设计要求：
- 每条主线至少 1 个金手指任务驱动节点
- 金手指任务必须与危机交叉设计中的"金手指系统"危机呼应
- 任务代价必须具体，驱动角色决策

### 三、L2 剧情单元卡（复用 v1 结构）

每幕产出 L2 剧情单元卡：
- 结构分析（7 阶段 + 情绪弧线 + 主题）
- 驱动决策列（溯源到角色卡决策转折点）
- 嵌套子线咬合

### 四、危机系统标注到剧情线节点

在 L3 骨架节点和 L2 阶段中标注危机来源系统：

```
| 骨架节点 | 危机系统 | 承接角色决策 |
|:---|:---|:---|
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 剧情线无独立文档 | 退回补写 |
| 主线缺金手指任务驱动节点 | 退回补写 |
| 驱动决策无溯源到角色卡转折点 | 退回补溯源 |
| L2 嵌套子线留空 | 退回补写 |
| 设计闭环缺失 | 退回补写 |

## 产出

- `剧情线/L3-{编号}-{名称}.md`（L3 剧情线 + 金手指任务驱动节点 + 危机系统标注）
- `幕/vol-XX/L2-{编号}-{单元名}.md`（L2 剧情单元卡）
