# 步骤4：创建幕纲 · 分幕层

> **产出/更新：** `剧情设计/幕/vol-XX/act-YY.md`
> **模板：** `templates/act-outline.md`
> **核心约束：** Step4 在剧情线成文后再分幕；幕纲先写幕功能和活跃线，不写逐章 Canvas 细节。

## 读什么

- `剧情设计/卷/卷{N}-卷纲.md`
- `剧情设计/剧情线/*.md`
- `小说世界设定/数值体系/动态升级表.md`

## 做什么

1. 根据剧情线数量、线型配置和卷纲终点，切分本卷幕结构。
2. 为每一幕定义戏剧功能：启动/升级/反转/压迫/爆发/收束。
3. 标注每幕活跃线：至少 1 条主线 + 2 条副线，其中副线可包含支线、设定/背景线、关系线、成长线、反派线。
4. 标注每幕力量门槛和角色压力。
5. 标注幕篇幅：本幕预估章数（按 ~2.5k 字/章）、占卷比例、各线章数分配。对齐卷纲的各幕分配。
6. 写入幕级待填区域：Canvas、章锚点、叙事债务追踪。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 没有剧情线文档就分幕 | 退回 Step3 |
| 每幕少于 3 条活跃线 | 退回补支线/设定线或重切幕 |
| 活跃剧情线表的状态/目标/变化列未填 | 退回补齐——空表 = 没做分幕规划 |
| 设定线在幕中无递送/验证/应用/代价状态 | 退回补设定线事件化节点 |
| 幕内活跃线超过 5 条 | 合并/错峰，避免 Canvas 过载 |
| 幕功能重复 | 退回重排升级曲线 |
| 幕终点不服务卷终点 | 退回卷纲检查 |

## 输出

按 `templates/act-outline.md` 创建或更新 `剧情设计/幕/vol-XX/act-YY.md` 的“Step4 分幕层”。
