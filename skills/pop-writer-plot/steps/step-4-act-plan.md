# 步骤4：创建 L2 剧情单元卡 · 分幕层

> **产出/更新：** `剧情设计/幕/vol-XX/L2-{编号}-{单元名}.md`（L2 剧情单元卡）
> **模板：** `templates/L2-剧情单元卡.tpl.md`
> **核心约束：** Step4 在 L3 剧情线成文后再分幕；单元卡先写结构分析层（7阶段+情绪弧线+主题+嵌套子线+边界信号）和活跃线，不写逐章 Canvas 细节。

> **你不是在切分 L3。你在设计一个让读者无法停下来的故事单元。**
> 每个 L2 单元必须回答：子线怎么咬合？节奏曲线是什么？哪个角色的决策是转折点？
> **⛔ 开填前必须先查 trope-library + 拆书 L2 参考，产出查询声明。**

- `小说世界设定/数值体系/动态升级表.md`

### 主动查询（外部信息注入 — ★NEW）
- **拆书 L2 参考**（如有）：学习子线咬合分析深度、节奏控制维度、7阶段结构
- **套路库具体卡**：查本单元元爽点对应的套路家族，学习部署方式+节奏控制
- **剧情库同赛道卡**：读"阶段推进"段，学习同赛道 L2 级别的结构设计
- 库缺时 WebSearch 补充（查询词：`"{赛道} {元爽点} L2 子线咬合 节奏控制"`）
- 产出查询声明块（见 SKILL.md 查询声明格式）

## 做什么

1. 根据剧情线数量、线型配置和卷纲终点，切分本卷幕结构。
2. 为每一幕定义戏剧功能：启动/升级/反转/压迫/爆发/收束。
3. 标注每幕活跃线（使用 L3-{编号} 引用）：至少 1 条主线 + 2 条副线活跃。三主线大类结构下，每幕每个主线大类至少 1 条具体剧情线活跃（共 ≥3 条主线），其中至少 1 条处于活跃释放状态。副线可包含支线（无主线大类）、设定/背景线、关系线、成长线、反派线。
4. 标注每幕力量门槛和角色压力。
5. 标注幕篇幅：本幕预估章数（按 ~2~3k 字/章）、占卷比例、各线章数分配。对齐卷纲的各幕分配。
6. 写入幕级待填区域：Canvas、章锚点、叙事债务追踪。

### v9.0.0 结构分析层填写（必填）

按 `templates/L2-剧情单元卡.tpl.md` 的结构分析层填写：

| 字段 | 要求 |
|:-----|:-----|
| L2-ID | 每个单元分配唯一 L2-{编号} |
| 所属线 | 标注 L3-{编号}（一条单元卡归属一条 L3 剧情线） |
| 单元类型 | 事件弧·成长型 / 对抗型 / 探索型 / 关系型 / 建置型 / 复合型 |
| 物理坐标 | 时间 / 地点 / 人物 / 力量 |
| 一句话单元 | 谁，在什么处境下，做了什么，结果把局面推向哪里 |
| 结构分析 | 7 阶段：铺垫 → 推进① → 触发 → 推进② → 转折 → 高潮 → 余波。每阶段标注功能、章节、事件 brief、情感设计、节奏 |
| 情绪弧线 | ASCII 图标注情绪强度变化和关键转折点 |
| 节奏蓝图（⛔必填） | 不允许只写"慢/中/快"。必须：(1) ASCII 节奏曲线图（每章节奏值 1-10）(2) 爆发点/减速带的位置+理由 (3) 用的节奏技法（数字递进/进度条/认知差/静动对比） |
| 主题表达 | 主题概括 + 主题与结构的对应 |
| 嵌套子线（⛔必填） | 至少 3 条子线。每条必须回答：(1) 为什么在这个单元活跃 (2) 和其他子线在哪里咬合、咬合逻辑是什么（因果/间歇/设局-收获/影子产出）(3) 去掉这条子线会失去什么。参考拆书 L2 的子线咬合分析深度 |
| 单元边界信号 | 起点信号 + 终点信号（事件类型突变/场景切换/角色状态/心态状态/叙事功能） |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 没有剧情线文档就分幕 | 退回 Step3 |
| 每幕少于 3 条活跃线 | 退回补支线/设定线或重切幕 |
| 活跃剧情线表的状态/目标/变化列未填 | 退回补齐——空表 = 没做分幕规划 |
| 设定线在幕中无递送/验证/应用/代价状态 | 退回补设定线事件化节点 |
| 幕内活跃线超过 5 条 | 合并/错峰，避免 Canvas 过载 |
| 幕功能重复 | 退回重排升级曲线 |
| 幕终点不服务卷终点 | 退回卷纲检查 |
| 结构分析 7 阶段未填 | 退回补齐 |
| 单元类型未标注 | 退回补标注 |
| 所属线（L3-ID）未填 | 退回补标注 |
| 嵌套子线留空或无论证 | 退回重新设计 |
| 节奏控制只写单字 | 退回重新设计 |
| Step 4 无查询声明 | 退回补查 |
| L2 驱动决策列无溯源 | ❌ 退回补写——每个阶段的驱动决策必须标注来自角色卡的哪个转折点 |
| L2 驱动决策无代价/不可逆后果 | ❌ 退回补写——从角色卡决策转折点预判映射 |
| L2 驱动决策与角色卡决策转折点不一致 | ❌ 退回重新映射——L2 的驱动决策必须与角色卡的决策转折点预判一致 |

## 输出

按 `templates/L2-剧情单元卡.tpl.md` 创建 `剧情设计/幕/vol-XX/L2-{编号}-{单元名}.md`，填：L2-ID + 所属线 + 单元类型 + 物理坐标 + 一句话单元 + 结构分析（7阶段）+ 情绪弧线 + 节奏控制分析 + 主题表达 + 嵌套子线 + 单元边界信号 + 活跃剧情线（L3-{编号}）+ 篇幅预算。

**回填 L3 剧情线的组成单元字段**：每个 L2 单元卡创建后，在对应 `剧情线/L3-{编号}-{名称}.md` 的头部"组成单元"字段和骨架节点的"承载单元"列回填 L2-{编号}。
