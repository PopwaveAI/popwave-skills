## 五、设定线

> **设定线不是一条故事弧——它不推进剧情，它控制"读者搞懂了什么"。**
> Canvas 中设定线占一列，逐章标记本章向读者交付了哪些世界观/等级/设定信息。
> 每行格式：`{信息项名}: {具体交付了什么}`。信息项名从下方设定披露承诺表取。
> 设线负载 = 0（无新设定）/ 1（适量）/ ≥2（过载警告）。
>
> 首章（ch01）设线负载 ≥ 1 —— 第 1 章至少让读者接触到本书最核心的 1 个设定概念。

### 设定线的交付原则

```
"不要在一个地方把所有设定倒完，也不要在一个地方一点不给。"
判定：读本章的读者如果跳过此章，会不会看不懂后面？
会 → 本章设线负载 ≥ 1。
```

### 设定披露承诺（本卷应传递的关键设定）

> 每一条承诺 = "这个信息最晚必须在第 X 章之前让读者接触到"。
> agent 在 Canvas 中逐章填设定线时，扫描承诺表确保未超 deadline。
> 自检时 rhythm_check.设线穿透.per_info_deadlines 从此表提取。

| 信息项 | 最晚章节 | 披露方式建议 | 对应载体 |

| {本卷最核心的设定概念} | ch{NN} | 实战展示 / 角色对话 / 叙事说明 | {主线/支线号} |
| {...} | | | |
