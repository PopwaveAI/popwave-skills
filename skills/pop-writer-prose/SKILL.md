# pop-writer-prose · 正文渲染 v3.6.0

> **定位：消费 chapter 的设计包 + 文风DNA，渲染为可读正文。**
> **核心约束：不碰剧情、不编造。设计包是唯一输入源——角色台词风格/行为基线全在设计包里，prose 不读角色卡。**

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| 1 | **设计包是唯一输入源** — `章节设计包/chXXX-设计包.md` 缺失 → 终止 |
| 2 | **不读角色卡** — 角色台词风格/行为基线已在设计包的登场人物表中 |
| 3 | **文风DNA已检查** — 缺失时输出 A/B 选项给用户，不自行降级 |
| 4 | **所有事件已渲染** — 每个骨架事件在正文中有对应段落 |
| 5 | **叙事者不解释** — 无「他意识到/他感到/他仿佛」等 AI 观感词 |
| 6 | **🔒 标记被遵守** — 设计包关键对白/数据原文照写 |
| 7 | **调味空间被遵守** — 设计包标「无幽默」但正文出梗 → 退回 |

## 核心流程

### 步骤1：读入 + 渲染
**读什么：** `章节设计包/chXXX-设计包.md` + 文风DNA + 上一章正文最后 ~800 字
**做什么：** ①读设计包（基础信息/登场人物台词风格/事件链/爽点/调味/钩子）→ ②加载 DNA 场景卡 → ③语感衔接 → ④首章开篇锻造（仅首章）→ ⑤逐事件渲染正文（scene→POV→感官锚点→事件内容→🔒对白嵌入）
**❌ 门禁：** 设计包/DNA 缺失终止；事件漏渲染退回

详细指令见 `steps/step-1-render.md`。

### 步骤2：验证 + 输出
**读什么：** 正文初稿 + DNA 原文 + 设计包
**做什么：** ①风格验证（偏差 ≥ 2 处退回）→ ②宪法检查（AI词/解说员句式/视角一致性）→ ③文本脉冲（500 字/微脉冲）→ ④字数检查（> 50% 退回）→ ⑤验证通过后写入 `正文/chXXX.md` + 章末状态块
**❌ 门禁：** 任一验证不通过退回 Step 1；落盘缺失退回

详细指令见 `steps/step-2-verify-output.md`。

## ❌ WRONG 示例

| ❌ 错误 | ✅ 正确 |
|:--------|:-------|
| 读角色卡来推断角色行为/台词 | 设计包的登场人物表已有台词风格和行为基线 |
| 从 act-YY.md 直接渲染绕过设计包 | 输入源必须是 chXXX-设计包.md |
| 设计包 🔒 标注「敏捷19（+1）」正文写成「敏捷20」 | 🔒 标记原文照写 |
| 文风DNA 空时自行降级 | 输出 A/B 选项给用户 |
| 设计包缺关键对白时自行编造面板数据 | 写感官反馈「一股暖流涌入身体」而非编造数值 |

## 边界条件
