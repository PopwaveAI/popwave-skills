# 外部DNA适配指南

> **定位**：当文风DNA来自另一部作品（如遮天DNA渲染深渊主宰设计包），DNA的场景卡不完全匹配设计包的事件类型时，如何系统性地适配。
>
> **适用场景**：跨体系风格迁移（西幻DND设计包→东方仙侠DNA）、同人二创、风格融合测试。
>
> **与合成模式的关系**：外部DNA适配 = 合成模式（无原文校验）+ 场景卡缺口填补。合成模式的全部风险（形似神不似/数据编造/对话偏移）在外部DNA适配中**加倍生效**。

---

## 1. 场景卡缺口分析

外部DNA通常不会精确覆盖设计包的所有事件类型。渲染前必须做缺口分析：

### 执行步骤

```markdown
1. 列出设计包的所有 scene 字段（逐事件提取）
2. 列出目标DNA的所有场景卡标题
3. 做交集 → 确认覆盖/缺口
```

### 示例（遮天DNA → 深渊主宰ch004设计包）

| 设计包事件类型 | 遮天场景卡 | 覆盖？ | 适配策略 |
|:--------------|:-----------|:------:|:---------|
| description_environment | ✅ 描写·环境引入 | ✅ | 直接映射 |
| dialogue_daily | ✅ 对话·日常互动 | ✅ | 直接映射 |
| narration_suspense | ✅ 叙述·孤独紧张 | ✅ | 直接映射 |
| narration_transition | ✅ 叙述·过渡/缓冲 | ✅ | 直接映射 |
| **combat_stealth** | ❌ 无对应卡 | ❌ | 见下方 |

遮天有 combat_early_skirmish（吼→金光→拳→崩碎）和 combat_mid_boss（大招名对抗），但没有任何潜行/暗杀场景卡。需要走降级适配。

## 2. 场景卡缺失时的降级策略（三级）

### 第一级：借用最接近的场景卡的叙事节奏

当DNA没有精确匹配的场景卡时，借用最接近卡的"段落呼吸"（句子长度/动词密度/标点节奏）：

```markdown
战斗类型：combat_stealth（潜行暗杀）
最接近的DNA场景卡：combat_early_skirmish（早期遭遇战）
→ 借用其"极短句链"节奏（遮天战斗每句一个动作/一个拟声词）
→ 但不借用其动作内容（不要写成"吼→金光→拳"）

实际写法：
"一道身影从阴影中凭空出现。寒光一闪，匕首精准地刺入他的后心。
与此同时，一只手从身后捂住了他的嘴。
'噗。'
匕首拔出。鲜血涌出。"
```

### 第二级：从通用维度提取替代规则

当没有可借用的场景卡时，退到DNA的通用维度层：

| 维度 | 提取什么 | 避坑 |
|:-----|:---------|:-----|
| 叙事者姿态 | 视角距离（贴/中/远） | 不提取"主角怎么想"的内容 |
| 对话引导与节奏 | 引导词风格（道/说/裸引语） | 不提取"什么场景用什么语气" |
| 环境描写策略 | 感官优先级、比喻方式 | 不提取"什么环境用什么比喻" |
| 战斗描写风格 | 段落呼吸、围观群众机制 | ❌ 不提取动作模板（吼→拳→崩碎） |
| 升级与成长描写 | 升级前的铺垫节奏 | ❌ 不提取升级形式（冲关/渡劫） |

### 第三级：只用 DNA 的全书稳定特征

最保守的降级——只使用DNA中标注为"全书稳定"的特征：

```
遮天稳定特征（§三）：
1. "道"字引导词的一致性 — 全书1821章保持稳定 → 对话全部用"道"
2. 围观群众作为战力注释系统 — 从CH028到CH709未中断
   → 在暗杀场景中：酒馆众人的议论充当这个功能
   → 在数据面板场景中：面板数值本身充当这个功能（数字=战力注释）
```

## 3. 围观群众机制的跨体系移植

遮天最具辨识度的叙事机制是"围观群众震惊"作为战力注释。外部DNA适配时，这个机制需要**移植而非复制**：

| 遮天场景 | 围观群众 | 功能 |
|:---------|:---------|:-----|
| 叶凡金乌大战 | 远处教主惊呼 | "这就是陆鸦啊！"→战力层级确认 |
| 老疯子独战群王 | 围观者心神欲裂 | "圣人之成败关乎亿万人族生死" |
| 荒古圣体初显 | 蒋逸晨逃跑 | 对手逃跑 = 实力碾压的铁证 |

### 移植到不同场景类型

| 设计包场景 | 围观群众的变体 | 移植示例 |
|:----------|:--------------|:---------|
| 酒馆社交 + 即将发生的暗杀 | 酒客的闲谈 | "索伦醒过来了，有人要倒霉了。" → 读者知道他说对了，恶徒不知道 |
| 数据面板弹出 | 面板数值本身 | 背刺*2造成12点伤害/16点伤害 → 数字即可，不加解说 |
| 世界观披露 | 主角内心的震惊对比 | "游戏中的琥珀城富丽堂皇……和眼前这片泡烂的贫民窟完全不同" |
| 日常温情 | 无围观群众 | 不强行移植——日常场景不需要战力注释 |

## 4. 遮天→深渊主宰适配备忘录（本会话实测）

### 4.1 环境描写适配

| 维度 | 遮天风格 | 适配到深渊主宰 | 示例 |
|:-----|:---------|:--------------|:-----|
| 开场结构 | 从大到小（宇宙→星球→地点→物体） | 从大到小（雨夜→贫民区→破屋→灶台→小女孩） | "雨水从黄昏就开始下了，整座琥珀城都笼罩在连绵不绝的水声中。贫民区深处一间快要塌了的木屋里……" |
| 比喻习惯 | "如/像/似" 贯穿 | 保持"如/像/似" 比喻 | "像风吹过水面泛起的波纹""像是一尊永恒的神炉" |
| 环境人格化 | 环境有意志（深渊吞噬心神） | 环境有情绪 | "夜雨像是不打算停了""雨把屋顶打得吱呀作响" |
| 感官比例 | 视觉占绝对主导，极少听觉/嗅觉/触觉 | 适配时引入听觉/嗅觉作为氛围底色 | 开篇从听觉切入（雨声），中间穿插嗅觉（霉味/恶臭），最后收束到视觉（阳光） |

### 4.2 对话引导词适配

| 遮天原始 | 适配到深渊主宰 | 示例 |
|:---------|:--------------|:------|
| 90%+ 用"道" | 保持"道" 主导 | "索伦道""薇薇安道" |
| 裸引语（无引导词） | 保持裸引语 | '"希斯。""哥哥。"' |
| 连续单句对话密集排列 | 保持密集排列 | 酒馆对话用"XXX道" 排布 |

### 4.3 战斗描写适配（跨类型关键）

当设计包战斗类型（暗杀/潜行）与DNA的战斗卡（碾压/大招）完全不同时：

```markdown
❌ 错误：强行套用DNA的战斗模板
"索伦一声大吼，震动整条小巷！金色的拳头不可阻挡！"
→ 这完全破坏了潜行暗杀的类型逻辑

✅ 正确：借用DNA的段落节奏，保留设计包的动作类型
"一道身影从阴影中凭空出现。寒光一闪。匕首刺入后心。
'噗。'
匕首拔出。鲜血涌出。尸体软软倒地。"
→ 短句链来自遮天（每句一个动作），动作类型来自设计包（潜行背刺）
```

**公式**：DNA的段落节奏 + 设计包的事件类型 = 跨类型战斗适配

### 4.4 数据面板适配

| 遮天的等级注释体系 | 深渊主宰的数据面板 | 适配方式 |
|:-------------------|:------------------|:---------|
| 围观群众惊呼"道宫五重天" | 背刺*2造成12点伤害 | 面板数据独立成段，不嵌入叙事句子 |
| "荒古圣体一吼震碎" | "获得杀戮经验30点" | 面板弹出间停顿动作节奏 |
| 法宝名+等级标注 | 战利品数量+品质 | 面板后自然衔接搜尸/摸钱 |

## 5. 先写中间章时的信息重建

当设计包从第N章开始（如从ch036开始做设计包）但用户想从ch01开始渲染时，缺少前N-1章的角色状态和关系基线。需要做：

### 5.1 从设计包反向推断的角色状态摘要

从ch036设计包的感官层+角色层提取可前推的信息：

```markdown
设计包ch036角色层：
- 索伦：警觉、谨慎、回忆闪回中带着不安 → 推断已有前世记忆
- 薇薇安：沉睡/困倦，被索伦抱入帐篷 → 推断关系已是"妹妹被保护"
- 神秘女主人：16级+女巫 → 推断索伦已融入北地商队体系

前推重建用于ch001的基线：
- 兄妹二人已在贫民区生存一段时间
- 索伦已觉醒前世记忆（游戏《诸神之战》数据融合）
- 索伦已建立保护薇薇安的执念
- 薇薇安对索伦有深度依赖和信任
- 外部威胁（索西亚人贩子）已经存在
```

### 5.2 缺失章节的"空白填充"红线

- ❌ 不得编造设计包证据不足的角色关系
- ❌ 不得设置设计包没有暗示的伏笔
- ✅ 使用感官和环境描写填充细节，不使用事实陈述

## 6. 适配质量检查清单

在完成标准合成模式检查清单（S1-S6）后，额外执行：

| # | 检查项 | 通过标准 |
|:-:|:-------|:---------|
| E1 | 场景卡缺口分析完成 | 缺口已识别并选择了适配策略 |
| E2 | 跨类型战斗不套模板 | 暗杀场景没有写成大吼+金光+拳 |
| E3 | 围观群众机制已移植 | 对应场景中能找到功能等效的"战力注释" |
| E4 | DNA稳定特征已使用 | "道"引导词一致性已保持 |
| E5 | 降级策略不编造DNA不存在的内容 | 所有从通用维度提取的规则都只涉及"怎么写"不涉及"写什么" |
