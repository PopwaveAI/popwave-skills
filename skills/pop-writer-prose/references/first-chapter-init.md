# 首章渲染初始化工作流

> **定位**：当 prose-render 首次进入一个新项目（`正文/` 目录为空），需要先创建 entity-snapshot 再进 Step 1。
> **管线阶段**：⚠️7 后触发，Step 1 前完成。

---

## 前置条件

- [ ] `写作资产/设计包v3/ch001-设计包.md` 存在（或 `设计包/ch001-设计包.md`）
- [ ] `设计/卷/` 下有卷纲文件（通常是 `卷1.md`），含起点/终点快照表
- [ ] `styles/` 或 `写作资产/文风DNA/` 下有文风DNA档案

---

## 执行步骤

### 1. 从卷纲提取初始状态

读 `设计/卷/卷(编号).md` 的**起点快照表**，提取：

| 数据 | 来源 |
|:-----|:------|
| 起始时间 | 卷纲「起点快照」时间行 |
| 起始位置 | 卷纲「位置」行 |
| 主角等级/职业 | 卷纲「等级」行 |
| 装备 | 卷纲「装备」行 |
| 核心配角 | 卷纲「关系」行 |
| 力量认知 | 卷纲「力量认知」行 |
| 冲突状态 | 卷纲「冲突状态」行 |

### 2. 从设计包角色层提取角色细节

读 `ch001-设计包.md` **§3 角色与人设（角色层）**，提取：
- 每登场角色的行为锚定（性格特质、内心冲突、行为表现）
- 角色关系基线
- 关键对白风格（角色各自的台词语气）

**注意**：设计包角色层记录的只是本章状态。对于"昏迷中"等特殊状态，如实记录，不推测"正常时的水平"。

### 3. 合并写入 entity-snapshot.yaml

格式：

```yaml
# 实体状态快照
# 最后更新章: 卷初（ch001 前）

## 主角团

### {主角名}
  status: alive
  class: {起始职业}
  level: Lv{起始等级}
  hp: {当前/最大，或状态描述}
  location: {起始地点}
  equipment: {无/列表}
  key_items: {无/列表}
  notes: {特殊状态（如昏迷/濒死/灵魂融合中）}

### {核心配角}
  status: alive
  relationship: {与主角的关系}
  location: {起始地点}
  notes: {关键特征}

### {宠物/伙伴}
  status: alive
  breed: {品种/种族}
  notes: {特殊状态}

## 外部环境

### 当前区域
  name: {区域名}
  threat_level: {低/中/高}
  controlled_by: {势力}
  notable_events: {近期大事件}

### 活跃威胁
  - {威胁1}: {描述}
  - {威胁2}: {描述}

## 全局状态
  timeline: {当前时间锚点}
  quest_log:
    - {短期目标1}
    - {短期目标2}
    - {长期目标}
```

### 4. 确认写入

验证：
- `entity-snapshot.yaml` 文件存在
- 主角团全员已覆盖（卷纲+设计包中出现的所有角色）
- 时间/地点与设计包一致
- 无编造数据——所有字段都有上游来源

---

## 常见陷阱

| 陷阱 | 表现 | 正确做法 |
|:-----|:-----|:---------|
| 编造未出场角色 | entity-snapshot 写入 ch001 后文才登场的角色 | 只写 ch001 出场角色 + 卷纲起点快照标注的角色 |
| 精确数值来源混淆 | hp/等级从设计包事件关键数据提取而非卷纲快照 | 卷纲起点快照 > 设计包角色层 > 设计包事件字段，优先级降序 |
| 遗漏环境状态 | entity-snapshot 只有角色没有区域/威胁 | 环境状态是 prose 渲染的感官基线，必须包含 |
