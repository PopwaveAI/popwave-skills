# 多风格对比测试 — 同一设计包×N种DNA渲染

> **定位**：取一个设计包，用N种不同的文风DNA分别渲染，产出N份正文存入风格命名的子文件夹，用于对比测试设计包精度和DNA迁移效果。
> **典型场景**：用户说"用设计包测试写正文""风格迁移测试""对比两个风格的渲染效果"。
> **前置条件**：设计包v3已就绪，每个目标风格对应一个styles/{风格名}.md DNA档案。

---

## 1. 与合成模式的关系

多风格测试 = 合成模式 × N。每个风格分支独立走合成模式流程（无原文校验），但输出结构不同：

| 维度 | 单章渲染 | 多风格测试 |
|:-----|:---------|:-----------|
| 输入源 | 1 设计包 + 1 DNA | 1 设计包 + N DNA |
| 输出路径 | `正文/chXXX.md` | `正文/{风格名}v3/chXXX.md` |
| 执行方式 | 单线程 | 并行委托（delegate_task，N≤3） |
| 比较维度 | — | 开篇对比/战斗风格/对话引导/环境引入 |
| 验收标准 | 字数≤20%，DNA合规 | 字数≤20%，每份DNA合规，且风格差异可辨别 |

---

## 2. 目录结构

```
正文/
├── 深渊主宰风格v3/
│   └── ch036-传奇之战（上）.md
├── 遮天风格v3/
│   └── ch036-传奇之战（上）.md
└── (单风格渲染直接放这里)
```

文件夹命名约定：`{风格名}v3`，保持与styles/{风格名}.md 的命名一致，方便追溯。

---

## 3. 执行流程

### Step A — 确认材料

| 检查项 | 通过标准 |
|:-------|:---------|
| 设计包 | `写作资产/设计包v3/chXXX-设计包.md` 可读，4层结构完整 |
| DNA 集 | styles/ 下每个目标风格都有对应.md文件，质量红标通过 |
| 输出路径 | `正文/{风格名}v3/` 文件夹已创建（不存在就建） |

### Step B — 分发渲染

用 `delegate_task` 并行调度，每个任务处理一个风格：

```
task 1: render(chXXX-design-pack, styles/深渊主宰.md) → 正文/深渊主宰风格v3/chXXX.md
task 2: render(chXXX-design-pack, styles/遮天.md)       → 正文/遮天风格v3/chXXX.md
task 3: render(chXXX-design-pack, styles/×××.md)        → 正文/×××风格v3/chXXX.md
```

每个任务的 context 中必须包含：
- 设计包完整摘要（事件链+🔒标记+关键对白+面板数据）
- 该风格的DNA要点（3-5个核心差异点，如"对话用'道'引导""战斗吼→拳四步""从大到小环境引入"）
- 输出路径

**:warning: 委托context的强制性约束（2026-06-16 新增，从实战bug提炼）：**

1. **剧情不变约束** — context中必须以醒目格式（单独成段加粗或分隔线区隔）包含：
   > **剧情约束：保留原文/设计包所有核心剧情事件不变，只改变叙事语言和风格。事件链是不可变更的骨架。不得自创原文不存在的战斗、不得改变角色生死状态、不得改变事件的发生顺序。**

2. **章节连续性摘要** — 多章渲染时（非单章测试），每章委托的context中必须包含上一章的结尾状态：
   ```markdown
   ### 章节连续性摘要
   - 上一章：ch{XX} {标题}
   - 上章末尾时间：{时间}
   - 上章末尾地点：{地点}
   - 主角状态：{关键状态变化}
   - 配角状态：{关键状态变化}
   - 最后一幕事件：{简述}
   - 本章起始点：{从哪一刻/哪一地点开始本章}
   ```

3. **单章独立文件** — 每章必须输出独立的 `ch{XX}-{标题}.md` 文件，不得合并输出。

**限制**：用户配置的 delegation.max_concurrent_children 决定并行数（当前=3）。超过限制时排队。

### Step C — 汇总对比

所有任务完成后，对比维度：

| 维度 | 对比方式 |
|:-----|:---------|
| 开篇句 | 复制各版本第1-3句并排 |
| 战斗节奏 | 选中第3-4个事件段落对比句式长度/动词密度 |
| 对话引导 | 搜索"道"/"说"/"道"/动作前置的分布 |
| 环境引入 | 对比出场方式和感官序列 |
| 🔒标记 | 所有版本均原文照写 → 一致 | 缺 → 补 |

### Step D — 后续动作

| 测试结论 | 建议动作 |
|:---------|:---------|
| 某风格字数和DNA合规，但开篇/战斗/对话和DNA风格描述不一致 | 退回该风格的Step 3修补 |
| 所有风格字数合规，但风格差异不明显 | DNA场景卡之间的区分度不够 → 退回pop-shared-dna步骤3重选场景卡 |
| 某版本🔒标记未原文照写 | 退回该版本Step 2修补 |
| **某版本剧情偏离原文/设计包**（子agent自创事件、改变角色生死、调换事件顺序） | **退回整批重做**。先检查父agent委托context是否缺少"剧情不变约束"语句（见 Step B 新增规范）。缺 → 父agent先修委托指令再重发。有但子agent仍偏离 → 升级为子agent训练数据问题 |
| 所有版本均达到质量标准 | 产出测试报告 → 可进入pop-writer-qa做正式质检 |

## 4. 风格迁移特化

当测试涉及**跨体系风格迁移**（如DND西幻设计包 → 东方仙侠DNA渲染），每个分支的 context 中需附加：

```
### 体系映射表（设计包→{目标风格}）
| 设计包概念 | {目标风格}概念 |
|-----------|---------------|
| 预言法术（九环） | 占卦推演/窥天机 |
| 流星爆（九环传奇法术） | 天降陨星/九转禁术 |
| 恐怖骑士（传奇召唤物） | 邪道尸煞武士/上古魔傀 |
| 梦魇（四阶恶魔） | 幽冥战兽/玄阴魔兽 |
| 女巫（16级施法者） | 隐世高人/金丹真人 |
| DND等级体系 | 修行境界体系 |
```

体系映射必须保持语义等价，不改变剧情功能和角色关系。

---

## 5. 输出规范

每个分支的最终输出文件必须包含：
1. Markdown正文（含章标题）
2. 章末 state_update 块

state_update 块的格式与单章渲染一致（见 pop-writer-prose SKILL.md 章末格式规范）。

---

## 6. 风险与红线

| # | 红线 |
|:-:|:-----|
| 🚫1 | **每个分支独立执行** — 不同风格的渲染不得互相读取对方的输出或 context。风格迁移映射表在 context 中一次性传递，不做跨任务数据交换。 |
| 🚫2 | **🔒标记在所有分支中一致** — 设计包的🔒标记对所有风格一视同仁。风格可以改变句法和用词，不可改变标记的内容（数字、面板数据、关键对白全文）。 |
| 🚫3 | **风格差异不可模糊** — 如果多风格测试产出的正文读起来"差不多"，说明DNA场景卡区分度不够，退回pop-shared-dna，不是继续优化prose渲染参数。 |
| 🚫4 | **体系迁移不可引入设计包没有的概念** — 即使"金丹真人"语义等价于"16级女巫"，如果设计包没有暗指"修仙"方向，迁移映射只做翻译不做扩展。 |
