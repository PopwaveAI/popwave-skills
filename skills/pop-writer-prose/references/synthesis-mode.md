# 合成模式 — 纯设计包+DNA渲染（无原文校验）

> **定位**：当 prose-render 无原文可参考，完全靠设计包事件链 + 文风DNA场景卡渲染正文时的操作规范。
> **适用场景**：新书写新章（非拆书管线）、同人二创（如深渊主宰）、跨文风迁移测试。
> **风险等级**：高于拆书模式。拆书有原文兜底，合成只有设计包精度锚点一个可靠性来源。

---

## 1. 拆书模式 vs 合成模式

| 维度 | 拆书模式（有原文） | 合成模式（无原文） |
|:-----|:------------------|:-------------------|
| 输入源 | 设计包 + DNA + 原文章节文件 | 设计包 + DNA **仅** |
| 事件核验 | 对比原文→事件是否忠实 | **无法核验** → 精度锚点是唯一约束 |
| 🔒标记 | 标记是对照原文的精确引用 | 标记是**强制性约束**——prose不可有任何偏离 |
| 信息缺漏 | 可回读原文补全 | **保留模糊**——宁可不写，不编造默认值 |
| 形似神不似 | 原文兜底，偏差可被实测发现 | **主要风险模型**——无原文则形似=神似假象 |
| 数据面板 | 原文有完整数值，锚点标注即可 | 必须在设计包 `关键对白/数据` 字段中完整明确写出 |
| 感官锚点 | 从原文直接提取 | 从DNA场景卡对照+设计包感官层推断 |

---

## 2. 合成模式风险模型

### R1: 形似神不似

设计包事件描述粒度不当时，prose 用通用模板填充细节：
- 设计包写"贫穷家庭背景" → prose写"死于瘟疫"（原文可能是"死于帮派火并"）
- 设计包写"希斯是条老狗" → prose写"普通的杂种土狗"（原文是"哥拉斯猎犬"）

**规避策略**：
1. 设计包精度锚点必须齐全（scene/POV/关键对白/感官锚点）
2. 任何未精确标注的细节保持模糊，用感官语言替代事实陈述
3. 写"希斯的毛发暗淡，步伐有些蹒跚" 而非 "它是一条普通的杂种狗"

### R2: 数据面板编造

设计包未在 `关键对白/数据` 字段标注面板数据时，prose 自行编造增益值和专长名。

**规避策略**：
1. 绝对遵守 ❌10 (prose)：设计包没标面板数据 → 写感官和行为反馈，不写系统面板
2. 写"一股暖流涌入四肢" 而非 "获得XX专长，力量+1"

### R3: 对话风格偏移

设计包只写了"说了些什么"但没标注语气→prose按自己默认风格写对话→偏离原文角色声线。

**规避策略**：
1. 对话必须从设计包角色层的关键对白标注中提取语气和潜台词
2. 没有语气标注时：**行动先于对话**——用身体动作传递情绪，对话只写必要信息
3. 薇薇安系角色：台词带省略号（"哥哥！……""是不是很没用？……"）

## 3. 合成模式质量检查清单

在完成标准 `pop-writer-prose` 落盘检查后，额外执行：

| # | 检查项 | 通过标准 |
|:-:|:-------|:---------|
| S1 | 设计包缺字段检查 | 每事件的4精度锚点（scene/POV/关键对白/感官锚点）全部非空 |
| S2 | 模糊区域标记 | 正文中所有设计包未精确标注的细节——都用了感官描写而非事实陈述 |
| S3 | 🔒标记全文通过 | 用 grep 搜索正文中所有 🔒 标记对应的内容，确认原文照写 |
| S4 | 无默认值注入 | 无"瘟疫/火灾/战争"等通用悲剧模板填充 |
| S5 | DNA场景卡已用 | 正文中每个事件的渲染手法可在DNA中找到对应场景卡 |
| S6 | 字数偏差 | 偏差≤20%（与拆书模式一致） |

## 4. 实操指引

### 当设计包精度不足时的降级策略

| 缺什么 | 怎么办 |
|:-------|:-------|
| 缺 scene 映射 | 根据事件类型（日常/交锋/披露）反推最接近的DNA场景卡 |
| 缺 POV | 默认设计包声明的全章POV角色 |
| 缺 关键对白/数据 | **写行为不写面板**。角色做了动作→身体反馈→结束，不弹数据面板 |
| 缺 感官锚点 | 从DNA场景卡的感官序列走默认规则（嗅觉→听觉→触觉→视觉） |

### 合成测试的价值

合成模式本身也是设计包质量的测试工具：
- 如果合成出来的正文"看起来合理但细节不对" → 设计包的精度锚点不足
- 如果合成出来的正文"细节全对但节奏不对" → DNA对接映射不够细
- 如果合成出来的正文"完全跑偏" → 设计包的事件描述本身就有问题

这也是本管线的核心设计哲学：
> **设计包精度决定prose下限，DNA决定prose上限。**
