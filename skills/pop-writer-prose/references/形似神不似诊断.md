# 形似神不似诊断参考

> **定位**：当 prose 产出的正文"看起来合理但细节不对"时的根因排查清单。
> **信条**：形似神不似不是 prose 的错，是设计包精度不够。

## 根因链

```
设计包事件描述模糊
  ↓
prose 在模糊处用自身语言模型填充 "合理默认值"
  ↓
默认值与原文具体设定不同（如"穷苦家庭→死于瘟疫"vs原文"神偷死在高塔"）
  ↓
读者/审查者感觉"不对"→形似神不似
```

## 四种典型误填充模式

| 模式 | 设计包写了 | prose 默认填充 | 正确做法 |
|:-----|:-----------|:---------------|:---------|
| 身世默认值 | 父母双亡（未写原因） | "死于瘟疫/饥荒" | 保持模糊：不写具体原因或写"那是很久以前的事了" |
| 战斗默认值 | 战斗胜利（未写具体收尾） | "捡起掉落物品，经验值+30" | 只写行为：走回洞穴入口，确认没有其他动静 |
| 数值默认值 | 属性提升（未写精确数值） | "获得【XX专长】力量+1" | 只写感受："一股暖流涌入手掌，四肢的疲劳消退了一些" |
| 对话默认值 | 角色质问（未写具体措辞） | "你为什么要这样做？"（通用模板） | 使用设计包标注的语气+潜台词重构，不自创措辞 |

## 排查步骤

当审查反馈"形似神不似"时：

```
1. 定位具体哪段/哪个事件感觉不对
2. 查设计包对应事件的描述精度：
   - 精确字段（有🔒标记）→ prose 是否严格遵循？
   - 模糊字段（无🔒标记）→ prose 是否填充了默认值？
3. 如果设计包模糊 → ❌ 问题在设计包，退回 design-pack 补精度
4. 如果设计包精确但 prose 未遵循 → ❌ 问题在 prose，退回重写
```

## 修复路径

| 发现 | 修复 | 归属 |
|:-----|:-----|:-----|
| 设计包事件描述模糊 | 在precision-anchor-format或v3格式中补充🔒字段 | pop-decon-design-pack |
| prose在模糊处填充了默认值 | 删除填充内容，保持原文模糊状态 | pop-writer-prose ❌12 |
| prose编造了设计包未写的事件 | 删除编造事件 | pop-writer-prose ❌1 |

## 预防

- 设计包v3 格式要求**每事件至少标注一个不可替换的精度锚点**（🔒关键对白/数据）
- 无🔒标记的事件，prose 渲染时在对应位置使用"感知模糊"写法——描述角色的感官和身体反应（"他听到门外有动静"）而非"知道"或"判断"（"他知道门外那个人的身份了"）
- 宁可少写三行，不多写一行编造
