# Step 1：读入 + 渲染

> 管线: pop-writer-prose v3.9

## 目的

读设计包 + 文风DNA → 直接渲染正文。不经过中间契约。

## 预加载

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `章节设计包/chXXX-设计包.md` | 唯一输入源 |
| 2 | 文风DNA | `pop-trope-library/文风库/{书名}.md` |

## 执行

### 1. 读设计包

从设计包提取：
- **基础信息**：本章核心目的、场景类型、预期字数
- **登场人物**：每个角色的 before/after + **台词风格**（简洁/克制/冷幽默/少年气/...）+ 关系动态
  - ★ prose 不读角色卡——台词风格和角色行为基线已在设计包的登场人物表中
- **场景流**：每场景的 POV / 🔒关键对白 / 感官锚点 / 场景内事件流 / 情绪走向 / 估计字数
- **爽点标注**：◆小爽点 / ★中爽点位置
- **创作方向建议**：幽默程度 / 叙事距离 / 情绪缓冲 / 创作方向
- **钩子**：章末钩子类型和方向

> ★ 不读 L2-{编号}.md、L1 设定、state-log、角色卡——全在设计包里。

### 2. 加载文风DNA

全量加载 → 建立两层感知：
- **层A** — 整体风格：句子长短？叙事者距离？战斗节奏？对话留白？
- **层B** — 本章场景卡定位：设计包 scene 字段 → DNA 场景卡 → 观察句+原文+时间演变

### 3. 语感衔接

读上一章正文最后 ~800 字 → 感受句子长度/节奏/叙事者距离。

### 4. 首章开篇锻造（仅首章）

生成 3 个开篇 50 字版本（含感官定位 + 氛围基调）→ 用户选一个。
用户说「后面不用问我了」→ 精简模式，后续不暂停。

### 5. 场景流渲染

#### 开幕段落（1-2 句）
- 空间/时间锚定 → 情绪传递
- 如场景1有感官锚点字段，第一句以该感官信号开头

#### 逐场景渲染
每个场景走相同流程：

**A. 理解场景意图** — 场景目的？情绪走向？推进哪条线？

**B. 选 POV** — 优先读场景 POV 字段 → 叙事者不进入非锚点角色内心

**C. 写场景段落**
| 原则 | 规则 |
|:-----|:-----|
| 限制内省 | 每章允许1-2处内心活动，用于情感高潮点；80%展示+20%内省 |
| 展示为主 | 情绪主要通过动作/环境/感官传递，关键决策点允许「他意识到/他感到」 |
| 场景内连续流动 | 事件在场景内有机融合，不逐事件独立成段 |
| 🔒对白 | 设计包「关键对白」字段的原文**必须**嵌入 |
| 风格对齐 | 直接应用 DNA 场景卡的模式 |

**D. 场景间过渡** — 场景之间1-2句自然过渡，不铺陈

#### 闭幕段落
- 章末 = 钩子位置。悬念→不闭合；情绪→安静收尾

### 6. 字数校准

每 2-3 场景检查一次累计字数，确保不偏离预期。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 章设计包不存在 | ❌ 终止，提示先完成 chapter |
| 设计包中场景流为空 | ❌ 终止，退回 chapter |
| 角色卡缺失（设计包中出现的角色） | ❌ 终止，退回 character |
| 风格锚点未加载 | ❌ 终止，退回加载 |

### 上游时效性检查（在存在性检查后执行）

| 检查项 | 失败动作 |
|:-------|:---------|
| 章设计包 `lastUpdatedAt` 早于项目总控的框架级变更时间 | ⚠️ 标注「设计包可能过期，建议先回 chapter 更新」 |
| 角色卡 `lastUpdatedAt` 早于项目总控的框架级变更时间 | ⚠️ 标注「角色卡可能过期」 |
| 卷纲 `lastUpdatedAt` 早于项目总控的框架级变更时间 | ⚠️ 标注「卷纲可能过期」 |
