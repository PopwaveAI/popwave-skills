# Step 2：逐事件渲染

> 管线: pop-writer-prose v2.0

---

## 目的

将事实骨架的每个事件，用文风DNA的规则直接渲染为可读正文。**不经过中间风格契约**——文风DNA文件已在 Step 1 读完，直接应用。

---

## 核心原则

```
文风DNA 已读进 context     ← Step 1 完成的
锚定章感知已建立                ← Step 1 完成的
人物卡状态已读取                ← Step 1 完成的

↑ 以上全部在 context 中，直接用于渲染，不需要再处理一次
```

---

## 执行

### 2.1 开幕段落

在进入第一个事件前，写 1-2 句开幕。功能：
- **空间锚定**：让读者知道「这章在哪」
- **时间锚定**：让读者知道「承接上一章的什么时刻」
- **情绪传递**：用环境/动作/感官传递起点情绪，不用叙事者说明
- **★ 感官锚点优先**：如果事件1有 `感官锚点` 字段（如"雨声启动""血腥味启动"），开幕段第一句以该感官信号开头；没有则自行发挥

### 2.2 逐事件渲染

每个骨架事件走相同的流程：

#### Step A：理解骨架意图
- 这个事件想传递什么情绪？ → 对照文风DNA的对应情绪的句式策略
- 这个事件在推进哪条情节线？
- 这个事件中角色在做什么？
- **★ 读 scene 字段**：如果事件有 `scene` 字段（如 combat_stealth），则以事件级为准映射DNA场景卡；没有则退而用章级场景类型匹配
- **★ 读感官锚点**：如果事件有 `感官锚点` 字段，当前事件的第一句以该感官信号开头
- **★ 读关键对白/数据**：如果事件有 `关键对白/数据` 字段，这些原文**必须**在当前事件的正文中出现——嵌入方式遵循DNA的数据嵌入规则

#### Step B：选 POV 距离
**★ 优先读设计包 POV**：如果事件有 `POV` 字段（如"薇薇安""索伦"），以该角色为本事件的感知锚点。叙事者不进入该角色的内心（除非该章约定内心视角），只跟随其视线、听觉、触觉、位置移动。

**没有 POV 字段时**，对照文风DNA中叙事者姿态维度选距离：
- 近距 → 第一句从角色的感官/动作切入
- 中距 → 从环境/空间切入
- 远距 → 从时间/宏观切入

**视角禁止行为**（选定时同步确定）：
- 第三人称限知 → 正文不写非视角锚点角色的内心活动
- 多视角限知 → 用段落分隔或章节分隔切换，不在一段内跳跃
- 全知 → 叙事者话外音需和正文自然融合

#### Step C：写段落
| 原则 | 规则 |
|:-----|:-----|
| **不解释** | 叙事者不跳出来解释。不写"他意识到/他感到/他仿佛/他心想" |
| **展示** | 情绪通过动作、环境、感官细节传递。不是"他紧张"而是"他的手按在刀柄上" |
| **对话** | 用人物卡的台词风格写。短对话 = 说完就停。长对话 = 有间隔和潜文本 |
| **信息释放** | 骨架标注了 info_release 的事件 → 按 release_method 嵌入设定 |
| **风格对齐** | 直接应用文风DNA中对应维度的模式——不用中间规则列表 |

#### Step D：事件间过渡
- 非同一地点的事件之间写 1-2 句过渡（空间移动/时间推进）
- 过渡不消耗情绪——快速过，不要铺陈

### 2.3 闭幕段落

- 章末段落 = 钩子的位置
- 如果钩子是悬念 → 最后一句不闭合，留缺口
- 如果钩子是情绪 → 最后一句是情绪落点，安静收尾
- 闭幕句子数：1-3 句，不超 50 字

---

## 宪法红线（渲染时对照）

- [ ] 无 AI 观感词：他感到、他仿佛、他意识到、他心想 → 改
- [ ] 无套话句式：首先、其次、总结来说、总而言之
- [ ] 无叙事者跳出来解释设定
- [ ] 战斗无"面板"数据出现（除非骨架标注了面板激活场景）
- [ ] 对话 = 人说话，不是角色念设定
- [ ] 骨架事件不可跳过——每个事件在正文中必须有对应段落
- [ ] **★ 事件关键对白/数据已落地**——对每个有 `关键对白/数据` 字段的事件，检查这些原文是否已在正文中出现。缺一条则退回修补
