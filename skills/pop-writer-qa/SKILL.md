# 爽点质检 / 双层 QA

> 版本：v1.0.1 | 2026-06-14

## 定位

**单一目的**：对产出物做双层质量审查——L1 硬门禁量化扫描（6 维门禁）+ L2 感觉型三层介入（纯感受反馈）。不评分、不给建议——L1 回答「有没有问题」，L2 回答「我读的时候怎么了」。

---

## 速查表

| 审什么 | 执行 L1？ | 执行 L2？ |
|--------|:---:|:---:|
| 单章正文完成后 | ✅ 完整 L1 + L2 | 穿帮/套路/AI味/伏笔超期/字数/钩子 → 全部过 |
| 设计包产出后（还没写正文） | ✅ L1 部分 | 仅检测事件数下限/中爽点≥1 |
| 全书完稿后 | ✅ L1 + 额外跨章统计 | 全章 L2 感觉型 + 节奏报告 |

---

## 前置条件

- [ ] 本章正文已产出（`正文/chXXX.md` 存在且可读）
- [ ] 本章设计包已产出（`设计包/chXXX-设计包.md` 存在且可读）
- [ ] entity-snapshot.yaml 存在（用于穿帮检测对标）
- [ ] 文风DNA 目录存在（用于 AI 味扫描参照）

---

## L1 硬门禁（量化扫描，不通过 = 退回）

| 门禁维度 | 检查内容 | 不通过 → |
|---------|---------|---------|
| 穿帮检测 | 死亡角色复活/地点矛盾/等级倒退/时间线错误/已毁地点出现/装备消失 | ❌ 标记 + 退回上游修复 |
| 套路重复检测 | 与前5章对比冲突类型/情绪落点/打脸结构相似度，>80%预警 | ⚠️ 预警 + 建议变体 |
| AI味硬扫描 | 参照 references/anti-ai-patterns.md（由 prose-render skill 维护）| ❌ 命中>3种 → 退回 |
| 伏笔超期预警 | >40章未回收=严重、>30章=中度、>20章=轻度 | ⚠️ 预警 |
| 字数合规 | 每章字数在目标区间内 | ❌ 超出±15% → 标注 |
| 钩子强度匹配 | 高潮章钩子力度≥L4、过渡章≥L1（参照 chapter-design 钩子体系）| ⚠️ 预警 |

**评级规则**：优秀✅(全过) / 良好⚠️(1-2预警) / 需调整⚠️(3-4预警) / 需重写❌(严重或5+)

---

## L2 感觉型三层介入

> 不评分、不给建议、不评价节奏、不评价画面、不比较。只回答「我读的时候怎么了」。

| 层级 | 检查视角 | 典型输出 |
|:-----|:---------|:---------|
| L2-1 逐段感觉 | 逐段标注"读到这里时我是什么感受" | "这段动作描写让我呼吸加速" / "这段说明让我想跳" |
| L2-2 穿帮感知 | 以普通读者视角标注"读到这里时产生的疑问/困惑" | "这里角色明明受伤了，下一页怎么在跑步？" |
| L2-3 全局感受 | 整章读完后的综合感受 | "这章读完我意犹未尽" / "这章读完我松了一口气" / "这章读完我想跳过下一章" |

## 执行流程

| 步骤 | 详细指令 | 门禁 |
|:-----|:---------|:----|
| Step 1 — 读入输入 | `steps/step-1-l1-audit.md` | 正文/设计包/entity-snapshot/anti-ai-patterns 必须全部可读 |
| Step 2 — L1 硬门禁扫描 | `steps/step-1-l1-audit.md` | 6维扫描 → 不通过退回上游，通过/预警进入L2 |
| Step 3 — L2 感觉型三层介入 | `steps/step-2-l2-review.md` | 逐段 → 穿帮感知 → 全局感受 |
| Step 4 — 输出 Gap Report | — | 合并 L1 + L2 + 量化评级 → 输出 |

## WRONG 示例

### WRONG 1：跳过 L1 直接做 L2
→ 正文有明显的穿帮（死亡角色又出现），但审查者跳过 L1 门禁扫描，直接进入 L2 感觉型审稿，最后给了"写得不错"的评价。
→ ✅ 正确：L1 是硬门禁，必须先过。L1 不通过 → 退回，不走 L2。

### WRONG 2：在 QA 阶段给出修改建议
→ "穿帮了，建议把这里改成角色B复活，或者把前面的死亡设定删掉。"
→ ✅ 正确：QA 只标记问题。具体怎么改，由上游 skill 决定。正确说法：「穿帮检测：ch{X} 中角色A已确认死亡（见 ch{X} 第{N}段），本章第{M}段又出现。标记为 P0。」

### WRONG 3：用 AI 味检测替代感觉型审稿
→ L2 审稿时只是在正文中搜索 "他感到/他仿佛/他意识到" 等关键词，没有真正进入"读者视角"逐段感受。
→ ✅ 正确：感觉型审稿的核心是"我读到这里时怎么了"。AI 味硬扫描是 L1 的门禁维度之一，不是 L2 的全部。

## 异常与边界条件

| 条件 | 处理方式 |
|:-----|:---------|
| 本章无设计包 | ❌ 无法执行完整 QA。仅能执行 L2 感觉型审稿（无对标基准）。告知用户缺少设计包。 |
| entity-snapshot 缺失 | ⚠️ L1 穿帮检测降级：仅在本章内做自洽性检查（时间/地点/状态一致性），不对标跨章快照。 |
| 本章为卷首章（无前5章） | ⚠️ 套路重复检测跳过；伏笔超期检测跳过；钩子强度检测按卷首章标准。 |
| 本章为终章 | ❌ 伏笔超期检测强制执行（全回收确认）；钩子检测标准切换为终章钩子。 |
| 设计包中事件数 < 3 | ❌ L1 事件数下限不通过。单章事件数至少 3。标记 P0。 |
| 正文文件存在但为空 | ❌ 无法执行 QA。告知用户正文文件为空。 |
| 本章为番外/幕间 | ⚠️ L1 门禁简化：仅执行 AI 味扫描 + 字数合规。其余维度跳过。 |
| 上游注明"本章为过渡章" | ⚠️ 钩子强度阈值降低至≥L1；情绪落点检测标准放宽。 |

## 本阶段边界

### 本步做什么
- L1 硬门禁扫描（6维）+ L2 感觉型三层介入
- 产出 Gap Report + 量化评分

### 本步不做什么
- ❌ 不直接修改正文（问题标记后由上游 skill 修复）
- ❌ 不在 QA 阶段重新设计剧情（那是 plot 的活）

**越界检测**：QA 中发现需要修改 → 说"这个问题标记为 [等级]，建议回到 [上游skill] 修复。"

---

## 落盘检查点

用户确认审查结果后：
- 写入：Gap Report（含 L1 硬门禁结果 + L2 感觉型报告 + 量化评分）
- 告知用户："✅ QA 完成。[评级]。如需修复，建议回到 [上游skill]。"
- ❌ 产出只留摘要 — 写入 Gap Report 后对话中不粘贴完整报告。说"QA 完成：[评级]。问题清单已写入报告。需逐项展开告诉我。"
