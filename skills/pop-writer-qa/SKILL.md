# 爽点质检 / 三层 QA

> 版本：v1.2.0 | 2026-06-17

## 定位

**单一目的**：对产出物做三层质量审查——L1 硬门禁量化扫描（6 维门禁）+ L2 感觉型三层介入（纯感受反馈）+ L3 原文对照（v3 测试管线专属，有原文锚定时启用）。
不评分、不给建议——L1 回答「有没有问题」，L2 回答「我读的时候怎么了」，L3 回答「与原文差在哪」。

---

## 速查表

| 审什么 | 执行 L1？ | 执行 L2？ | 执行 L3？ |
|--------|:---:|:---:|:---:|
| 单章正文完成后 | ✅ 完整 L1 + L2 | 穿帮/套路/AI味/伏笔超期/字数/钩子 → 全部过 | ⏭ 原文不可用时跳过 |
| 设计包产出后（还没写正文） | ✅ L1 部分 | 仅检测事件数下限/中爽点≥1 | ⏭ 跳过 |
| v3复现/续写测试（有原文锚定） | ✅ 完整 L1 | ✅ L2 | ✅ L3 逐段对照原文 |
| 全书完稿后 | ✅ L1 + 额外跨章统计 | ✅ 全章 L2 | ⏭ 跳过 |

---

## 前置条件

- [ ] 本章正文已产出（`正文/chXXX.md` 存在且可读）
- [ ] 本章设计包已产出（v2: `设计包/chXXX-设计包.md` **或** v3: `设计包v3/chXXX-设计包.md`，至少一个可读）
- [ ] entity-snapshot.yaml 存在（用于穿帮检测对标）
- [ ] 文风DNA 目录存在（用于 AI 味扫描参照）
- [ ] **L3 原文对照专用**：原文章节 `_temp/chapters/chXXX.txt` 存在（仅 v3 复现/续写测试时启用）

## L1 硬门禁（量化扫描，不通过 = 退回）

| 门禁维度 | 检查内容 | 不通过 → |
|---------|---------|---------|
| 穿帮检测 | 死亡角色复活/地点矛盾/等级倒退/时间线错误/已毁地点出现/装备消失 | ❌ 标记 + 退回上游修复 |
| 套路对齐检查（v3专用） | 设计包标注的套路类型（如"打脸装逼"）在正文中是否被正确执行。★读 pop-trope-library/套路库/{套路名}.md 的"使用红线"段（✅必须做/❌不要做）对照正文 | ⚠️ 套路未落实 → 预警；套路走形或违反使用红线 → ❌ 退回 |
| 原文污染检测（设计包专用） | 设计包中是否含有直接暴露的原文句子（>25字完整原文）——检查 `原文证据` 列 / `原文案例` 列 / `氛围渲染要点` 节 / `关键对白标注` 对白列 / `名句` 引文。参考 `references/design-pack-original-text-boundary.md` | ❌ 任一处完整原文句子未清理 → 退回 |
| 套路重复检测 | 与前5章对比冲突类型/情绪落点/打脸结构相似度，>80%预警 | ⚠️ 预警 + 建议变体 |
| **元爽点多样性检测 ★NEW** | **检查本章使用的元爽点是否与最近3章连续相同。查询爽点引擎.md确定本书的2-3个主元爽点，检查连续4章是否使用了同一元爽点及其全部变体** | **⚠️ 连续4章同一元爽点 → 预警 + 建议切换到另一个元爽点的变体家族。如：\"连续4章使用信息差元爽点，建议切换到阶级革命元爽点的变体\"** |
| AI味硬扫描 | 参照 references/anti-ai-patterns.md（由 prose-render skill 维护）| ❌ 命中>3种 → 退回 |
| 伏笔超期预警 | >40章未回收=严重、>30章=中度、>20章=轻度 | ⚠️ 预警 |
| 字数合规 | 每章字数在目标区间内 | ❌ 超出±15% → 标注 |
| 钩子强度匹配 | 高潮章钩子力度≥L4、过渡章≥L1（参照 chapter-design 钩子体系）| ⚠️ 预警 |

**评级规则**（L1+L2 通用模式）：优秀✅(全过) / 良好⚠️(1-2预警) / 需调整⚠️(3-4预警) / 需重写❌(严重或5+)

**评级规则**（v3 L3 模式启用时，覆盖通用模式）：

| 等级 | 标签 | L1六维 | L2感觉型 | L3原文对照 | 判定 |
|:----|:-----|:-------|:---------|:-----------|:-----|
| 4 | 🏆 优秀 | 全部✅ | 逐段感受match + 无穿帮感 | 90%+维度对齐 | 可交付 |
| 3 | ✅ 良好 | ≤1⚠️ | 全局感受一致，局部质感偏移 | 70-89%对齐 | 可接受 |
| 2 | ⚠️ 合格 | ≤1❌ | 核心情绪弧线未歪 | 50-69%对齐 | 需迭代 |
| 1 | ❌ 不合格 | ≥2❌ | 主情绪弧偏移 | <50%对齐 | 退回重构 |

## L2 感觉型三层介入

> 不评分、不给建议、不评价节奏、不评价画面、不比较。只回答「我读的时候怎么了」。

| 层级 | 检查视角 | 典型输出 |
|:-----|:---------|:---------|
| L2-1 逐段感觉 | 逐段标注"读到这里时我是什么感受" | "这段动作描写让我呼吸加速" / "这段说明让我想跳" |
| L2-2 穿帮感知 | 以普通读者视角标注"读到这里时产生的疑问/困惑" | "这里角色明明受伤了，下一页怎么在跑步？" |
| L2-3 全局感受 | 整章读完后的综合感受 | "这章读完我意犹未尽" / "这章读完我松了一口气" / "这章读完我想跳过下一章" |

## L3 原文对照（v3 测试专用）

> 仅在 v3 复现/续写测试模式启用。需要原文章节 `_temp/chapters/chXXX.txt` 存在。
> 回答「与原文差在哪」——逐事件对齐、标记差异等级。

| 层级 | 检查视角 | 典型输出 |
|:-----|:---------|:---------|
| L3-1 事件对齐 | 逐事件检查：正文vs原文的事件链一致性 | "事件5缺少狗头人对话中的具体咒骂内容" |
| L3-2 关键数据对齐 | 面板数值/对话/系统消息是否与原文一致 | "敏捷19（+1）被写成敏捷20，数值偏差" |
| L3-3 风格密度对齐 | 感官密度/段落呼吸/叙事距离 vs 原文对应段 | "原文此段有3个感官锚点，正文只有1个" |

### L3 评级标准

| 维度对齐率 | 评分 | 含义 |
|:----------|:----:|:-----|
| 事件链100% | ✅ | 无遗漏、无编造事件 |
| 关键数据100% | ✅ | 面板/对白/系统消息完全一致 |
| 感官密度≥80% | ⚠️ | 感官密度略低，可接受 |
| 叙事距离一致 | ✅ | POV全程守住，无越界 |
| 风格偏差<3处 | ⚠️ | 局部偏离原文笔触 |

### L3 典型问题标记

| 标记 | 含义 | 处理 |
|:----|:-----|:-----|
| 🅿️ 编造 | 正文写了设计包/原文没有的内容 | P0 → 退回prose |
| 🅿️ 遗漏 | 原文有但正文没写 | P0 → 退回prose |
| 🅿️ 偏移 | 事件链一致但细节不同 | P1 → 标注具体差异 |
| 🅿️ 模糊 | 原文精确但正文写模糊了 | P2 → 建议修正 |

## 执行流程

| 步骤 | 详细指令 | 门禁 |
|:-----|:---------|:----|
| Step 1 — 读入输入 | `steps/step-1-l1-audit.md` | 正文/设计包/entity-snapshot/anti-ai-patterns 必须全部可读 |

## WRONG 示例

### WRONG 1：跳过 L1 直接做 L2
→ 正文有明显的穿帮（死亡角色又出现），但审查者跳过 L1 门禁扫描，直接进入 L2 感觉型审稿，最后给了"写得不错"的评价。
→ ✅ 正确：L1 是硬门禁，必须先过。L1 不通过 → 退回，不走 L2。

### WRONG 2：在 QA 阶段给出修改建议
→ "穿帮了，建议把这里改成角色B复活，或者把前面的死亡设定删掉。"
→ ✅ 正确：QA 只标记问题。具体怎么改，由上游 skill 决定。正确说法：「穿帮检测：ch{X} 中角色A已确认死亡（见 ch{X} 第{N}段），本章第{M}段又出现。标记为 P0。」

### WRONG 3：用 AI 味检测替代感觉型审稿
→ L2 审稿时只是在正文中搜索 "他感到/他仿佛/他意识到" 等关键词，没有真正进入"读者视角"逐段感受。
→ ✅ 正确：感觉型审稿的核心是"我读到这里时怎么了"。AI 味硬扫描是 L1 的门禁维度之一，不是 L2 的全部。

## 异常与边界条件

| 条件 | 处理方式 |
|:-----|:---------|
| 本章无设计包 | ❌ 无法执行完整 QA。仅能执行 L2 感觉型审稿（无对标基准）。告知用户缺少设计包。 |
| entity-snapshot 缺失 | ⚠️ L1 穿帮检测降级：仅在本章内做自洽性检查（时间/地点/状态一致性），不对标跨章快照。 |
| 本章为卷首章（无前5章） | ⚠️ 套路重复检测跳过；伏笔超期检测跳过；钩子强度检测按卷首章标准。 |
| 本章为终章 | ❌ 伏笔超期检测强制执行（全回收确认）；钩子检测标准切换为终章钩子。 |
| 设计包中事件数 < 3 | ❌ L1 事件数下限不通过。单章事件数至少 3。标记 P0。 |
| 正文文件存在但为空 | ❌ 无法执行 QA。告知用户正文文件为空。 |
| 本章为番外/幕间 | ⚠️ L1 门禁简化：仅执行 AI 味扫描 + 字数合规。其余维度跳过。 |
| 上游注明"本章为过渡章" | ⚠️ 钩子强度阈值降低至≥L1；情绪落点检测标准放宽。 |

## 本阶段边界

### 本步做什么
- L1 硬门禁扫描（7维）+ L2 感觉型三层介入
- 产出 Gap Report + 量化评分

### 本步不做什么
- ❌ 不直接修改正文（问题标记后由上游 skill 修复）
- ❌ 不在 QA 阶段重新设计剧情（那是 plot 的活）

**越界检测**：QA 中发现需要修改 → 说"这个问题标记为 [等级]，建议回到 [上游skill] 修复。"

---

## 落盘检查点

用户确认审查结果后：
- 写入：Gap Report（含 L1 硬门禁结果 + L2 感觉型报告 + 量化评分）
- 告知用户："✅ QA 完成。[评级]。如需修复，建议回到 [上游skill]。"
- ❌ 产出只留摘要 — 写入 Gap Report 后对话中不粘贴完整报告。说"QA 完成：[评级]。问题清单已写入报告。需逐项展开告诉我。"
