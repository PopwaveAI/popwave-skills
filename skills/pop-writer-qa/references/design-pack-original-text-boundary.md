# 设计包原文污染检测与清理

> 用途：检测并清理设计包中直接暴露原文的部分。设计包应只含设计决策/结构拆解/感官锚点标注——不应包含完整原文句子。

## 原文污染的典型位置

设计包（v3 格式和旧 L1-L5 格式）中，原文直接暴露通常在以下位置：

| 位置 | 旧格式（L1/L2/L3/L4/L5） | v3 格式（1./2./3./4./5.） |
|------|--------------------------|----------------------------|
| **事件链表** | 无「原文证据」列 | `原文证据` 列 — 最后一列，每行含直接原文引用 |
| **套路表** | 无「原文案例」列 | `原文案例` 列 — 套路结构拆解后的原文引用 |
| **对白表** | `对白分析` 的「内容」列 | `关键对白标注` 的「对白」列 |
| **感官层** | `感官描写` 含完整感官句子 | `氛围渲染要点 (4.3)` — 逐句原文拆解（最严重）
| | `名句` / `台词集` 含完整对白 | `关键场景三段式 (4.2)`「感官锚点」列 |
| **🔒 标记** | 引号内的完整原文句子 | 引号内的完整原文句子 |
| **爽点机制** | 长引文（>25 字完整原文） | 长引文（>25 字完整原文） |

## 污染判定标准

| 等级 | 内容类型 | 示例 | 判定 |
|:----:|:---------|:-----|:----:|
| ❌ 污染 | 完整原文句子（>25 字） | `"索伦也被困在了落叶城，外面的大雪厚度太高，马车根本就走不动"` | 须移除 |
| ✅ 设计 | 短标注/关键词（<25 字） | `"索伦也被困在落叶城"` | 保留 |
| ✅ 设计 | 箭头式事件摘要 | `索伦被困→大雪→马车走不动` | 保留 |
| ✅ 设计 | 分析评述中的短引用证据 | `"苍白"、"流鼻涕"` | 保留 |
| ⚠️ 边界 | 角色名句（但可被上下文推断） | `"快找个地方躲起来！"` | 酌情：已嵌入分析则保留 |

## 批量清理方案（Python）

### 定位模式

| 目标 | Python 检测模式 |
|------|----------------|
| `原文证据` 列 | `'原文证据' in line and line.strip().startswith('|')` |
| `原文案例` 列 | `'原文案例' in line and line.strip().startswith('|')` |
| `对白` 列（关键对白标注） | 表头匹配 `### 3.2 关键对白标注` → 移除第4列（0-indexed: index=4） |
| `氛围渲染要点` 节 | 匹配 `^###\s+4\.3\s+氛围渲染要点` → 删除到下一个同级标题 |
| `名句` / `台词集` 引文 | 匹配 `"(.+)"` 在名句/台词集块中 → 保留标注前缀，删除引文正文 |
| 旧格式 `对白分析` 表 | 匹配 `^### 对白分析` → 移除「内容」列（索引3） |
| `关键场景三段式` 的感官锚点列 | 匹配 `^###\s+4\.2\s+关键场景三段式` → 移除第3列（索引3，感官锚点列） |
| 🔒 中的长引文 | 匹配 `> 🔒.*?「[^」]{25,}」` 或 `> 🔒.*?"[^"]{25,}"` → 替换为纯结构摘要 |

### 注意事项

1. **表头也要处理** — 移除数据列后，表头行（`| 字段名 | ... |`）和分隔行（`|:----|...|`）必须同步移除对应列
2. **空白行中断** — 节标题和表格之间可能有空白行，处理时先通过空白行（append 原样），再进表格处理循环
3. **双列名重复** — 部分文件的 `原文证据` 和 `原文案例` 列实际名称为空但结构占位存在，按表格列数判断
4. **旧格式兼容** — ch001-ch055 使用 L1/L2/L3/L4/L5 结构，无「原文证据」列但有「名句」和「对白分析」表
5. **文件备份** — 批量修改前先备份文件夹（187文件规模）
6. **多次通过** — 第一遍去除大节（4.3 氛围渲染要点），第二遍处理表列和嵌入式引文，避免正则冲突
