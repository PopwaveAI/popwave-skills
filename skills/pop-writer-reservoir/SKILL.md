# pop-writer-reservoir · 素材储备注入 v1.0.1

> **定位**：写作全链路的素材储备环节。被 pop-writer-creative 在初始阶段调用产出首版池子，也可**独立调起**——你在任何时候刷到好东西，想给当前项目储备剧情种子。

---

## 两种调用模式

### 模式 1：被 creative 调用（首版池子产出）

在 pop-writer-creative 的 Phase 0 或分支 A/B/C 流程结束时，自动触发本 skill：

```
creative Phase 0 产出故事引擎
  → 调用 pop-writer-reservoir
  → 读全部采集素材 + PRD
  → 按四类归档产出首版 素材储备池.md
  → 交接给 world + plot
```

### 模式 2：独立调起（项目中注入）

任何时候你看到好东西想储备：

```
你："帮我把这个XX内化到素材池"
  → 本 skill 独立启动
  → 读项目已有 PRD + 故事引擎 + 世界宪法 + 现有池子
  → PRD 契合度评估
  → 四类归档写入池子
  → 告知 plot 池子有更新
```

| 案例 | 重点展示 |
|:-----|:---------|

| `04-鹅腿事件-鬼婆化改编案例.md` | 复杂社会素材的黑暗化改造（鬼婆+伥鬼）+ 灰色结局 |
| `05-大同案-白骨契约案例.md` | 50/50撕裂事件的站边策略——选与目标读者一致的那边，不中立 |
| `05-大同案-剥离后结构案例.md` | （被废弃的对照版）站中立的教训——两边不讨好 |

每次执行注入时，可先快速浏览案例库，找到最接近当前素材类型/安全等级的那条，作为流程参考模板。

四类归档：
- **主线索材**：可直接触发主线事件
- **支线索材**：可独立成线的冲突/势力/机制
- **背景素材**：世界观色块，不驱动剧情但丰富世界
- **剩余焊接点**：已构思焊接方案但未在主线使用

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌R1 | **不读 PRD 就评估** — 必须找到 `PRD.md`（或从故事引擎提取后）执行。找不到 = 没有加工坐标系 = 通知用户先开书 |
| ❌R2 | **不分类直接堆** — 必须按主线/支线/背景/剩余焊接点四类归档，否则 plot 无法消费 |
| ❌R3 | **不写匹配度** — 每个素材必须标注 PRD 匹配度，否则 plot 不知道优先级 |
| ❌R4 | **重复注入** — 注入前检查储备池中是否已有同类素材，如有则合并而非堆叠 |
| ❌R5 | **独立调起时不修改宪法** — 即使发现冲突，也只写入储备池+标注冲突，不擅改 PRD.md/故事引擎.md |
| ❌R6 | **臆测素材内容** — 用户输入不完整时必须先采集（搜索/追问）再解吸。没采集到完整内容就凭空编造 = ❌ |
| ❌R7 | **搜索一次就放弃** — 搜索无结果时变更搜索词重试，最多 3 次。仍无结果才标记"素材不完整，降级" |
|| ❌R8 | **不评估受众立场就融合改编** — 现实素材必须做三方立场分析+情绪能量检测+安全等级判定。跳过此步直接出融合方案 = 你可能在帮现实中的恶人洗白，导致作者被网暴/社会性死亡。真实案例：2026-06-17 钟连宇水鬼事件测试中，第一次方案写成了她被胁迫值得同情——严重越线，被用户当场否决 |
|| ❌S1 | **信息采集深度不足** — 必须穿透L1表面层+L2结构层+L3系统层。只采了表面层就归档 = 不合格，退回重采 |
|| ❌S2 | **信息采集广度不足** — 每条素材至少覆盖事件本体 + 平行案例/社会分析/历史参照 中至少2个视角。只有一个视角 = 孤立事件，不能做融合判断 |
|| ❌S3 | **信源多样性不足** — 社会热点类素材必须≥3条独立信源（主流媒体+深度分析+不同立场）。单条信源 = 你的理解可能只是一个角度的真相 |
|| ❌S4 | **素材权重判定错误** — 必须按规模(文明/势力/社区/个体)×密度(高/中/低)矩阵判定。v1.1版误判：荷鲁斯(文明级高密度→给了1幕)、水鬼(社区级中密度→给了1卷)。错判 = 后续设计全部浪费 |
|| ❌S5 | **融合方案无具体角色** — 储备池必须包含至少1个具体角色(名字/动机/困境/弧线) + 1个具体选择(两条路径各有代价) + 1个具体场景(有画面/台词/情绪)。只有抽象概念没有可执行设计 = 没用 |
|| ❌S6 | **同情灭门者但不谴责他对无辜者的暴力** — 当融合方案中的受害者也对无辜第三方（尤其儿童）施加致命暴力时，必须通过主角明确表达谴责。沉默 = 洗白私刑。检验标准：把无辜者替换成两名儿童重读方案，如果读下来没有不适感 → 你在洗白 |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 素材是"钟连宇水鬼事件"（女孩假轻生害死民警）→ 融合方案写成"她也是被胁迫的可怜人" | 立场自杀！公众眼中女孩是恶人、英雄是烈士。写她值得同情 = 在帮害死英雄的人洗白。作者会被网暴、退圈、社会性死亡 | 做三方立场分析：施动方(恶人)→绝不同情/受动方(英雄)→绝对尊重。女孩必须是纯恶/主动投靠混沌。故事焦点在英雄的抉择不在恶人的苦衷 |
| 素材是"鹅腿阿姨卖鸭腿"→ 融合方案写成"老汤姆也是没办法" | 同样是洗白欺诈者。公众情绪是"被骗的愤怒"，写同情=踩痛处 | 老汤姆的悲剧不是"他可怜"，而是"他活得太久，谎言已经长进了骨头里，他自己都分不清真假了"——悲悯≠洗白 |
| 素材是历史事件→ 直接使用真实事件名称/朝代/人物 | 政治敏感/民族创伤/立场踩雷 | 剥离所有历史标识（去名/去朝代/去具体发生地），只保留冲突结构 |
| 融合方案里受害者也杀了无辜者（灭门案）→ 主角沉默/主角理解 | 主角沉默 = 认可私刑。男读者读完觉得"杀人是对的" → 作者被骂反社会 | 主角必须明确说「你错了」。可以同情遭遇，但不能认可他的选择。检验标准：把无辜者替换成两个孩子重读 |

## 收到任务后，按顺序执行

### 前置检查：确定项目状态

详见 [steps/step-inject.md](steps/step-inject.md)。

1. 检查项目目录是否有 `00-原始设定/PRD.md`
   - 有 ✅ → 已启动项目，走「注入流程」
   - 无 → 检查是否有 `00-原始设定/故事引擎.md`
     - 有 ✅ → 已启动项目（v2.0.0 之前），从故事引擎提取 PRD 后走注入流程
     - 无 ❌ → 未开书，输出"请先通过 pop-writer-creative 启动项目"
2. 检查是否有已有 `00-原始设定/素材储备池.md`
   - 有 ✅ → 后续追加
   - 无 ✅ → 首版创建（用模板）

### 注入流程

详见 [steps/step-inject.md](steps/step-inject.md)。

1. **读项目已有产出**：PRD.md / 故事引擎.md / 世界宪法.md / 现有素材储备池.md
2. **素材收集**（HARD GATE — 当用户输入不完整时）：用搜索/追问补全素材内容，拒绝臆测
3. **素材解吸**：从完整素材中提取 3 个核心（冲突核心/独特设定/情感基调）
4. **★ 受众立场与情绪评估**（社会性安全闸门）：分析现实素材在大众中的既定道德立场和情绪基调。三方立场分析（施动方/受动方/观察方）+ 情绪能量检测 + 安全等级判定（✅安全/⚠️敏感/🔴禁区/☠️自杀）。产出改编红线约束。**此步骤不可跳过 — 跳过意味着你可能在书里帮现实中的恶人洗白，导致作者被网暴/毁掉职业生涯**
5. **PRD 契合度评估**：按 PRD 加工哲学判断（契合/有张力/冲突）
6. **宪法冲突检测**：是否违反已锁定的宪法边界
7. **适配方式判定**：按素材类型 x 项目类型决定适配策略（原生改写/社会寓言/规律提取等）
8. **四类归档**：按主线/支线/背景/剩余焊接点分类
9. **写入储备池**：追加到 `00-原始设定/素材储备池.md`

### 产出后动作

```
写入完成后
  │
  ├─ 对话摘要："已内化 {素材名} → {归档类别}，PRD 匹配度 {高/中/低}"
  ├─ 如匹配度低："建议仅限支线/背景使用"
  ├─ 如冲突："标注冲突，池中已备注，使用前需确认是否修订宪法"
  └─ 通知下游："素材储备池有更新，plot 下次取用时可检阅新条目"
```

## 本阶段边界

### 本步做什么
- 素材注入、PRD 匹配度评估、宪法冲突检测
- 四类归档写入储备池
- 首版池子产出（被 creative 调用时）

### 本步不做什么
- ❌ 不修改 PRD/故事引擎/世界宪法（即使冲突也只标注）
- ❌ 不生成故事引擎（那是 creative 的活）
- ❌ 不设计剧情线（那是 plot 的活）
- ❌ 不采集跨域素材（那是 creative W1 的活——只处理用户主动提供的素材）

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 项目目录无 PRD.md 但有故事引擎.md | 从故事引擎提取 PRD（提取六维度：一句话/类型锚点/核心承诺/加工哲学/DNA/边界），存入 `00-原始设定/PRD.md` 后继续 |
| 项目目录无 PRD.md 也无故事引擎 | 输出"请先通过 pop-writer-creative 开书" |
| 首次调用（无现有池子） | 创建新的 `00-原始设定/素材储备池.md`，使用模板首次填充 |
| 已有池子 | 追加新条目，不覆盖/不清空现有内容 |
| 素材与现有池子条目高度重复 | 合并到现有条目（追加触发场景），不新增行 |
| 用户提供的素材过于模糊 | 执行采集步骤：按素材类型选择搜索/追问方式。搜索一次不够 → 换搜索词重试，最多 3 次 |
| 搜索三次仍无结果 | 标记"素材不完整，降级为简要笔记"，基于仅有的信息做最小归档 |
| 用户提供了多个完全不相关的素材 | 每个独立评估+独立归档，不做强行融合 |
| 素材是中文互联网热点，涉及真实地名/人名 | 搜索时标注信息来源，归档时去真实身份（只保留冲突结构，删除真实姓名/地名） |

##  实战沉淀（本会话产生的通用规律）

以下规律来自 2026-06-17 的5条素材测试，适用于所有后续注入。

### 规律一：立场不是可选——是必选

现实素材在大众中有既定道德立场。你的融合方案必须选择与目标读者一致的立场。不选=两边不讨好。

| 素材 | 公众对施动方的定位 | 正确的融合立场 |
|:-----|:------------------|:--------------|
| 水鬼事件（钟皆连） | 恶人/害死英雄 | 纯恶，任何时候不给同情。净化后那咋了 |
| 大同案 | 50/50撕裂，但男频读者站男方 | 站男生边，不讨好女读者 |
| 鹅腿事件 | 复杂（欺诈者但有真实付出） | 鬼婆化——悲悯但不洗白。她的悲剧不是她可怜，而是谎言已经长进了骨头里 |

### 规律二：安全等级决定改编策略

| 等级 | 改编策略 |
|:-----|:---------|
| 安全 | 自由加工 |
| 敏感 | 大幅剥离现实身份，只取结构 |
| 禁区 | 三层剥离：去名→去背景→去具体发生地 |
| 自杀 | 纯恶处理，焦点放在英雄的抉择不在恶人的苦衷 |
| 极度撕裂 | 站与目标读者一致的边，不中立 |

### 规律三：剧情预算由事件规模决定

| 影响范围 | 建议预算 |
|:---------|:---------|
| 文明级（战争/诸神/世界规则变更） | 2-4卷主线 |
| 势力级（宗教运动/王国战争） | 1-2卷主线 |
| 社区级（小镇/组织/局部阴谋） | 1幕支线 |
| 个体级（个人恩怨/秘密） | 1-3章短篇 |

**反例**：本会话中最初把荷鲁斯大叛乱（银河内战级）评为1幕支线，把水鬼事件（单点社会现象）评为1卷主线——完全错配。

### 规律四：每条素材一个独立文档

不要把所有素材塞进一个文件。每个素材独立一篇，包含从Step 1.5采集到融合方案的完整流程。池子只做索引。

### 规律五：同情受害者 ≠ 认可他伤害无辜（灭门者伦理）

当素材中的受害者后来也成了加害者——比如被彩礼骗局逼到灭门杀人的男人——融合方案必须守住以下底线：

| 正确 | 错误 |
|:-----|:-----|
| 他可以愤怒。读者也可以理解他为什么走到那一步。 | ❌ 不能默许他杀掉无辜者（尤其孩子）后还被主角沉默认可 |
| 主角说「我同情你，但你错了」 | ❌ 主角不能说「我理解你这么做」或帮他说好话 |
| 灭门者的悲剧在于他以为自己是在复仇，实际上他变成了和骗他的人一样烂的人 | ❌ 不能把灭门写成被逼无奈的反抗 |

**实战教训（2026-06-17 大同案）**：初版写江轩沉默，没法说灭门者错了。被用户指出这等于洗白私刑。修正后：江轩问「那两个孩子几岁？」灭门者答不出来。江轩说「你和她的区别在哪？她宰了好人灵魂换糖，你宰了两个和这件事无关的孩子。我可以同情你的遭遇，但我不会说你没错。」

**检验方法**：把灭门者的行为替换成杀死两个5岁孩子，重读方案——如果读下来没有不适感，说明你在洗白。
