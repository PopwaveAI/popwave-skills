## 三种调用模式

### 模式 1：被 creative 调用（首版池子产出）

在 pop-writer-creative 的 Phase 0 或分支 A/B/C 流程结束时，自动触发本 skill：

```
creative Phase 0 产出故事引擎
  → 调用 pop-writer-reservoir
  → 读全部采集素材 + PRD
  → 逐条素材产出剧情储备卡（带冲突公式/人物/套路/枪链）
  → 拼接为首版 素材储备池.md
  → 交接给 plot
```

### 模式 2：独立调起 — 用户提供素材（反应式）

任何时候你看到好东西想储备：

```
你："帮我把这个XX内化到素材池"
  → 本 skill 独立启动
  → 读项目已有 PRD + 故事引擎 + 世界宪法 + 现有池子
  → 安全门禁 → PRD评估 → 宪法检测
  → 产出剧情储备卡 → 追加到池子
  → 告知 plot 池子有更新
```

### 模式 3：主动丰富池子（主动式）★ v2.2.0 NEW

reservoir 自主搜索跨域素材来补充储备池，不依赖用户手动提供：

```
用户说"升级池子"或"丰富储备池"
  → 读 PRD.md + 爽点引擎.md + 故事引擎.md + 现有池子
  → 分析池子缺口：
    ├─ 现有储备卡覆盖了哪些元爽点？哪些元爽点缺素材？
    ├─ 现有储备卡覆盖了哪些品类？哪些品类是空白？
    └─ 现有储备卡数量和多样性是否足够支撑 plot 设计？
  → 按缺口方向搜索跨域素材：
    ├─ web_search 按 PRD 关键词定向搜
    ├─ web_search 按元爽点名变体搜
    └─ web_search 按空白品类搜
  → 逐条评估 → 安全门禁 → PRD契合度 → 宪法检测
  → 产出剧情储备卡 → 追加到池子
  → 告知 plot "池子新增了N条储备卡，覆盖了XX方向"
```

## 质量红线

| # | 红线 |
|:-:|:-----|
| R1 | 不读 PRD 就评估 — 必须找到 PRD.md（或从故事引擎提取后）执行 |
| R2 | 剧情储备卡缺核心冲突公式 — 没有公式 = plot 不知道这种子怎么用 |
| R3 | 剧情储备卡缺人物原型 — 没有人物 = plot 不能即时填充角色 |
| R4 | 剧情储备卡缺配套套路 — 没有套路参考 = 落地时 plot 要从零发明 |
| R5 | 剧情储备卡缺契诃夫枪链 — 没有预期伏笔规划 = 落地后容易忘记回收 |
| R6 | 独立调起时不修改宪法 — 即使发现冲突，也只写入储备卡+标注冲突 |
| R7 | 臆测素材内容 — 用户输入不完整时必须先采集 |
| R8 | 搜索一次就放弃 — 搜索无结果时换词重试最多3次 |
| R9 | 不评估受众立场就融合改编 — 跳过安全门禁可能导致洗白方案 |
| R10 | 未查设定库就评估素材 — pop-trope-library/设定库/ 有同赛道设定创意，不查=素材来源单一 |

## 注入流程

详见 [steps/step-inject.md](steps/step-inject.md)。

### 通用步骤（三种模式都走）

1. **前置检查：确定项目状态** — 检查 PRD.md / 故事引擎.md / 世界宪法.md / 现有素材储备池.md / pop-trope-library/设定库/（跨域设定素材源）
2. **素材采集** — 按模式不同（见下方）
3. **素材解吸 + 品类路由** → 走对应 1A~1E 子流程
4. **叙事母题统一 + 世界观映射**
5. **受众立场与情绪评估**（安全门禁，不可跳过）
6. **PRD 契合度评估**
7. **宪法冲突检测**
8. **适配方式判定 + 多路线评估**（>=2条路线）
9. **产出剧情储备卡**（按新模板：冲突公式/人物原型/配套套路/契诃夫枪链/情绪基调/创作笔记）
10. **写入储备池**：追加到 `储备剧情池/素材储备池.md`

### Step 2 按模式区分

#### 反应式（用户提供了具体素材）

> **当用户输入不完整时**：用搜索/追问补全素材内容，拒绝臆测。搜索一次不够→换词重试，最多3次。

#### 主动式（用户说"升级池子"或"丰富储备池"）

> **缺口分析** — 读 `爽点引擎.md` 看本书的元爽点星座+占比，读现有池子看已覆盖的和盲区：
>
> | 分析维度 | 问题 |
> |:---------|:-----|
> | 元爽点覆盖 | 现有储备卡覆盖了哪些元爽点的哪些变体？哪个元爽点缺素材？ |
> | 品类覆盖 | 已覆盖社会事件/IP/历史/神话中的哪些？哪个品类是空白？ |
> | 数量多样性 | 储备卡总数是否足够（一个元爽点至少 3-5 个变体）？ |
>
> **跨域搜索** — 按缺口方向 `web_search`：
> - 按 PRD 关键词定向搜（"黑暗流 双界穿越 阶级革命 灵感"）
> - 按元爽点变体搜（"信息差 布局 草蛇灰线 网文"）
> - 按空白品类搜（"神话 献祭 契约 黑暗改编"）
>
> **搜索纪律**：每次搜索都标注搜索词和来源 URL。每条素材至少 3 条独立信源才进入评估。

#### 混合式（用户提供了素材 + 要求丰富）

先走反应式处理用户提供的素材 → 再走主动式搜索补充。两轮产出合并为一次性写入。
