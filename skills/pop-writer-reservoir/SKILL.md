# pop-writer-reservoir · 创意转换引擎

> 外部素材 → 剧情储备卡 / 设定储备卡。剧情卡含元卖点、参演势力、剧情线发展链条；设定卡含设定内核、兼容性、可衍生冲突空间。

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **读取协议** — 禁止用 Read 工具读取 skill 文件，用 `skill_view` 或 `Get-Content -Encoding UTF8 -Raw` |
| ❌2 | **不读 PRD 就评估** — 必须先读 `全书立项PRD.md`。不读 PRD = 加工坐标系悬空 = 储备卡无法对齐 |
| ❌3 | **臆测素材内容** — 用户输入不完整时必须先采集（搜索无结果换词重试最多3次）。臆测 = 储备卡建立在假数据上 |
| ❌4 | **预设主角干预** — 储备卡假设主角不存在，主角怎么介入是 plot 阶段的事 |

## ⚠️ 步骤加载门禁

> **你必须完整加载对应步骤文件后，才能产出该步骤的输出文件。**
> 不能只读 SKILL.md 的加载指令清单摘要就凭印象产出。
> 加载指令清单告诉你"做什么"，step 文件告诉你"怎么做"。

| 你要写 | 必须先加载 | 跳过的后果 |
|:-------|:-----------|:-----------|
| 素材快照+安全评估+权重判定 | ⛔ `steps/step-inject-collect.md` + `references/采集SOP.md` | 采集深度不够，缺L2/L3层，安全评估跳过 |
| 剧情储备卡/设定储备卡 | ⛔ `steps/step-inject-archive.md` + `templates/剧情储备卡-模板.md` 或 `templates/设定储备卡-模板.md` | 储备卡结构残缺，缺元卖点/参演势力/剧情线发展链条 |

**门禁检查（每步开始时执行）：**
1. 上一步的产出文件是否存在且内容完整？
2. 当前步骤的 step 文件和 template 文件是否已加载到上下文？
3. 如果 2 不成立 → 立即加载再继续，不凭记忆写。

## 三种调用模式

| 模式 | 触发 | 流程 |
|:-----|:-----|:-----|
| 1 被 creative 调用 | creative 产出 PRD 后自动触发 | 读 PRD → 逐条素材产出储备卡 → 首版储备池 |
| 2 独立调起（反应式） | 用户说"帮我把这个XX内化" | 读 PRD+现有池子 → 安全门禁 → 产出储备卡 → 追加 |
| 3 主动丰富（主动式） | 用户说"升级池子/丰富储备池" | 分析缺口 → 跨域搜索 → 评估 → 产出储备卡 → 追加 |

## 储备池数量级约束

> **储备卡对应 L2 具体剧情线，不是 L1 主线大类。** plot 阶段的三主线大类（世界大事件/主角成长/主角行动）是 L1，全书固定；每卷每个 L1 大类下动态激活 2-3 条 L2。每条 L2 对应 ≥1 张剧情储备卡。支线无 L1，直接是 L2 级别。

### 首版储备池下限（被 creative 调用时）

> 首版储备池需要覆盖卷一可能激活的 L2 剧情线 + 支线 + 设定。不是全书所有 L2 的全集——后续卷的 L2 可以通过"主动丰富"模式逐步补充。

| 类型 | 采集级别 | 下限 | 对应关系 |
|:-----|:---------|:-----|:---------|
| 剧情储备卡（主线级） | 主线剧情级 | ≥6 张 | 卷一每个 L1 大类下 ≥2 条 L2，每条 L2 ≥1 张卡 |
| 剧情储备卡（支线级） | 支线剧情级 | ≥3 张 | 支线级剧情线（无L1），每条 ≥1 张卡 |
| 设定储备卡 | 背景素材级 | ≥3 张 | 世界核心设定的 3 个维度 |
| **总量下限** | | **≥12 张** | |

### 主动丰富模式检查清单

| 检查项 | 不满足时动作 |
|:-------|:------------|
| 卷一每个 L1 大类下 L2 剧情储备卡 ≥2 张 | 补采该大类下的素材 |
| 每个 PRD 元爽点至少有 1 张卡能落地 | 补采对应元爽点的素材 |
| 支线级储备卡 ≥3 张 | 补采支线级素材 |
| 设定储备卡 ≥3 张 | 补采设定级素材 |
| 剧情卡 vs 设定卡比例 ≥ 2:1 | 补采剧情级素材 |
| 新卷启动前 L2 储备卡覆盖不足 | 补采该卷计划激活的 L2 对应素材 |

### 单张卡质量下限

| 卡类型 | 必须包含 |
|:-------|:---------|
| 剧情储备卡（主线级） | 元卖点 + ≥3方参演势力 + 8~20节点剧情线发展链条 + 绝对之恶(靶子) |
| 剧情储备卡（支线级） | 元卖点 + ≥2方参演势力 + 5~15节点剧情线发展链条 |
| 设定储备卡 | 设定内核 + 兼容性分析 + ≥2个可衍生冲突空间 |

## 路由

```
用户说"注入素材/内化/升级池子"
  │
  ▼
Step Inject-A · 采集+评估 (step-inject-collect.md)
  Step 1: 读项目已有产出（PRD+现有池子+trope-library）
  Step 1.5: 素材采集（按 采集SOP.md 三层穿透+四视角覆盖）
  Step 2: 素材解吸（冲突核心+独特设定+情感基调）
  Step 2.5: 受众立场与情绪评估（按 安全门禁.md）
  Step 2.6: 素材权重与剧情预算判定（按 权重矩阵.md）
  Step 3: PRD 契合度评估 + 宪法冲突检测
  │
  ⏸ 评估完成，进入归档
  │
  ▼
Step Inject-B · 归档写入 (step-inject-archive.md)
  Step 4: 适配方式判定（事件性质→剧情卡 / 非事件性质→设定卡）
  Step 5: 产出储备卡（按对应模板）
  Step 6: 质量闸门检查 → 写入 `素材储备池/{素材名}.md`
```

## 产出

```
素材储备池/{素材名}.md — 每条素材独立文件（剧情储备卡 或 设定储备卡）
素材储备池/素材储备池.md — 索引页（速览表+权重说明，区分剧情/设定类型）
```

## 引擎登记（储备卡落盘后）

每张储备卡写入后，将卡内实体和设定规则登记到 pop-state-engine：

```bash
# 注册储备卡实体到知识图谱（properties 标注来源）
python {engine_scripts}/command_executor.py -p {项目路径} -a add-node -j '{
  "id": "{实体ID}",
  "type": "{character|location|faction|item|skill|concept}",
  "name": "{实体名}",
  "tags": "储备卡",
  "properties": "{\"source\": \"{储备卡名}\"}"
}'
```

```bash
# 写入储备卡设定规则到事实表（permanent 级）
python {engine_scripts}/command_executor.py -p {项目路径} -a set-fact -j '{
  "entity": "{实体名}",
  "attribute": "{属性名}",
  "value": "{值}",
  "chapter": 0,
  "importance": "permanent"
}'
```
