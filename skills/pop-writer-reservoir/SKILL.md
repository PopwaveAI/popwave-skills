# pop-writer-reservoir · 创意转换引擎

> 外部素材 → 剧情储备卡 / 设定储备卡。剧情卡含元卖点、参演势力、剧情线发展链条；设定卡含设定内核、兼容性、可衍生冲突空间。

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **读取协议** — 禁止用 Read 工具读取 skill 文件，用 `skill_view` 或 `Get-Content -Encoding UTF8 -Raw` |
| ❌2 | 不读 PRD 就评估 — 必须先读 `全书立项PRD.md` 获取加工坐标系、边界、不可妥协DNA |
| ❌3 | 储备卡缺元卖点 — 没有元卖点 = 不知道为什么选这个素材 |
| ❌4 | 剧情卡缺参演势力 — 没有≥3方势力 = plot 不知道冲突结构 |
| ❌5 | 剧情卡缺剧情线发展链条 — 没有8~20节点因果链 = plot 不知道事件怎么发展 |
| ❌6 | 设定卡缺设定内核 — 没有核心机制+关键规则+内在矛盾 = world 不知道怎么接入 |
| ❌7 | 设定卡缺兼容性分析 — 没有世界宪法/DNA兼容性 = 可能破坏本书核心体验 |
| ❌8 | 独立调起时不修改宪法 — 即使发现冲突，也只写入储备卡+标注冲突 |
| ❌9 | 臆测素材内容 — 用户输入不完整时必须先采集，搜索无结果换词重试最多3次 |
| ❌10 | 不评估受众立场就融合改编 — 跳过安全门禁可能导致洗白方案 |
| ❌11 | 未查 trope-library 就评估素材 — `pop-trope-library/设定库/` 有同赛道设定创意，不查=素材来源单一 |
| ❌12 | 预设主角干预 — 储备卡假设主角不存在，主角怎么介入是 plot 阶段的事 |

## 三种调用模式

| 模式 | 触发 | 流程 |
|:-----|:-----|:-----|
| 1 被 creative 调用 | creative 产出 PRD 后自动触发 | 读 PRD → 逐条素材产出储备卡 → 首版储备池 |
| 2 独立调起（反应式） | 用户说"帮我把这个XX内化" | 读 PRD+现有池子 → 安全门禁 → 产出储备卡 → 追加 |
| 3 主动丰富（主动式） | 用户说"升级池子/丰富储备池" | 分析缺口 → 跨域搜索 → 评估 → 产出储备卡 → 追加 |

## 路由

```
用户说"注入素材/内化/升级池子"
  │
  ▼
Step Inject-A · 采集+评估 (step-inject-collect.md)
  Step 1: 读项目已有产出（PRD+现有池子+trope-library）
  Step 1.5: 素材采集（按 采集SOP.md 三层穿透+四视角覆盖）
  Step 2: 素材解吸（冲突核心+独特设定+情感基调）
  Step 2.5: 受众立场与情绪评估（按 安全门禁.md）
  Step 2.6: 素材权重与剧情预算判定（按 权重矩阵.md）
  Step 3: PRD 契合度评估 + 宪法冲突检测
  │
  ⏸ 评估完成，进入归档
  │
  ▼
Step Inject-B · 归档写入 (step-inject-archive.md)
  Step 4: 适配方式判定（事件性质→剧情卡 / 非事件性质→设定卡）
  Step 5: 产出储备卡（按对应模板）
  Step 6: 质量闸门检查 → 写入 `素材储备池/{素材名}.md`
```

## 产出

```
素材储备池/{素材名}.md — 每条素材独立文件（剧情储备卡 或 设定储备卡）
素材储备池/素材储备池.md — 索引页（速览表+权重说明，区分剧情/设定类型）
```
