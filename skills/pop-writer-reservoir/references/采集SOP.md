# 素材采集 SOP

> Step 1.5 素材采集的操作标准。
> 核心原则：**采集不是漫无目的的信息收集，是带着 PRD 的需求定向采集。**
> 读取协议：❌ 禁止 Read 工具，用 `skill_view` 或 `Get-Content -Encoding UTF8 -Raw`。

---

## 一、采集目的性 — 为什么要采这个素材

> ❌ 错误做法：用户说"有个XX事件" → 直接搜事件详情 → 采集完再想"这能怎么用"
> ✅ 正确做法：先读 PRD → 确定采集目的（主线级/支线级/背景级）→ 带目的定向采集

### 1.1 采集前：从 PRD 确定采集方向

读 `全书立项PRD.md`，提取以下采集锚点：

| PRD 维度 | 采集锚点 | 采集时问什么 |
|:---------|:---------|:-------------|
| 元爽点（L1-L4参数） | 这个素材能不能触发本书的元爽点？ | 素材里有没有对应 L1 参数的机制？ |
| 世界宪法 | 这个素材的冲突结构能不能嵌入本书世界？ | 素材的权力结构/冲突模式和宪法是否兼容？ |
| 核心卖点 | 这个素材能不能强化某个卖点？ | 素材的独特性是否映射到本书的差异化？ |
| 跨域素材指引 | PRD 指引了哪些方向？这个素材属于其中吗？ | 素材是否落在 PRD 指引的 3-5 个方向内？ |

### 1.2 设定采集级别

根据 PRD 需求，给每条素材设定采集级别，决定采集深度和数量要求：

| 采集级别 | 信号 | 采集深度 | 最低信源数 | 产出要求 |
|:---------|:-----|:---------|:----------|:---------|
| **主线剧情级** | 素材可能支撑 ≥1卷主线 | L1+L2+L3 全穿透 + 四视角全覆盖 | ≥4 | 完整储备卡（冲突公式+人物+套路+枪链+brief） |
| **支线剧情级** | 素材可能支撑 1幕支线 | L1+L2+L3 + ≥2视角 | ≥3 | 完整储备卡 |
| **背景素材级** | 素材提供世界观质感/氛围 | L1+L2 | ≥2 | 简化储备卡（冲突公式+独特设定+情绪基调） |
| **灵感碎片级** | 素材只有 1 个亮点可用 | L1 | ≥1 | 灵感笔记（1段话，标注"待开发"） |

> ⚠️ 采集级别不是素材本身的属性，是**素材相对于本书 PRD 的价值定位**。同一个历史事件，对 A 书可能是主线级，对 B 书可能只是背景级。

### 1.3 采集级别判定流程

```
素材 + PRD 对照
  │
  ├─ 素材的冲突结构能支撑 ≥1卷的叙事？ → 主线剧情级
  ├─ 素材的冲突结构能支撑 1幕（10-20章）？ → 支线剧情级
  ├─ 素材有独特设定/画面但无大冲突？ → 背景素材级
  └─ 素材只有 1 个点子/意象？ → 灵感碎片级
```

## 二、判断用户输入是否完整

| 用户说了 | 完整度 | 动作 |
|:---------|:-------|:-----|
| "最近有个XX事件，大致是……（具体描述了冲突/人物/经过）" | **完整** | 跳过采集，直接进 Step 2 |
| "最近有个XX事件" / "我想到一个民俗叫XX" / "XX设定能不能用" | **不完整** | 需要采集 |
| "帮我把这个IP融进书里" / 仅提了一个名字 | **不完整** | 需要采集 |

## 三、执行采集

| 素材类型 | 采集方式 | 最小产出要求 |
|:---------|:---------|:-------------|
| 社会热点 | web_search（事件名+时间） | 能回答：发生了什么、谁和谁冲突、结果如何 |
| 民俗神话/民间传说 | web_search（故事名+完整故事） | 能回答：故事完整脉络、核心意象、原始文本 |
| 其他 IP | agent 知识 / web_search（设定关键词） | 能回答：IP 核心设定、叙事规律、关键机制 |
| 都市传说 | web_search（传说名+origin） | 能回答：传说内容、传播背景、核心恐怖点 |
| 用户自己想到的点子 | 用 clarify 追问 2-3 轮完整细节 | 用户口头补全到可执行的粒度 |

**拒绝臆测**：搜索后仍无法获取完整素材 → 标记"素材不完整，降级为简要笔记"。搜索无结果换词重试，最多3次。

## 四、深度要求（按采集级别分层）

### 主线/支线级 — 必须穿透三层

| 层 | 问题 | 最低产出 |
|:---|:-----|:---------|
| **L1 表面层** | 发生了什么？5W1H | ≥3条独立信源交叉验证 |
| **L2 结构层** | 内在逻辑？各方利益/动机/行为模式？ | 识别≥3个参与方，每个给出动机分析 |
| **L3 系统层** | 反映了什么更广泛的社会/文化/历史模式？ | 识别≥1条关联的更大链条 |

### 背景级 — L1+L2 即可
### 灵感碎片级 — L1 即可

---

## 五、广度要求（按采集级别分层）

### 主线级 — 四视角全覆盖

| 视角 | 核心问题 |
|:-----|:---------|
| **事件本体** | 事实是什么？ |
| **平行案例** | 有没有类似事件？是孤立还是模式？ |
| **社会分析** | 媒体/学者/公众怎么分析？社会根源？ |
| **历史参照** | 历史上有无类似模式？ |

### 支线级 — ≥2视角
### 背景级 — ≥1视角
### 灵感碎片级 — 无要求

---

## 六、信源多样性要求

| 素材类型 | 主线级 | 支线级 | 背景级 |
|:---------|:------:|:------:|:------:|
| 社会热点 | ≥4（≥1主流媒体+≥1深度分析+≥1不同立场+≥1补充） | ≥3 | ≥2 |
| 历史事件 | ≥3（≥1学术+≥1不同视角+≥1补充） | ≥2 | ≥2 |
| 民俗神话 | ≥3（≥1原始文本+≥1改编案例+≥1学术分析） | ≥2 | ≥2 |
| 其他 IP | ≥2（覆盖核心设定+关键事件+叙事规律） | ≥1 | ≥1 |

## 七、采集完成信号

| 采集级别 | 完成条件 |
|:---------|:---------|
| 主线级 | L1+L2+L3 全穿透 + 四视角全覆盖 + 信源数达标 + 开始识别模式 |
| 支线级 | L1+L2+L3 + ≥2视角 + 信源数达标 |
| 背景级 | L1+L2 + ≥1视角 + 信源数达标 |
| 灵感碎片级 | L1 + ≥1信源 |
