## Step 1.5：素材采集（HARD GATE — 当用户输入不完整时）

> **为什么需要这一步**：用户大概率只给一句话（"最近有个热点""我想到一个民俗故事"），素材内容本身是空白的。不先采集完整素材，Step 2 解吸出来的全是臆测。

### 1.5.1 判断用户输入是否完整

| 用户说了 | 完整度 | 动作 |
|:---------|:-------|:-----|
| "最近有个XX事件，大致是……（具体描述了冲突/人物/经过）" | **完整** | 跳过采集，直接进 Step 2 |
| "最近有个XX事件" / "我想到一个民俗叫XX" / "XX设定能不能用" | **不完整** | 需要采集 |
| "帮我把这个IP融进书里" / 仅提了一个名字 | **不完整** | 需要采集 |

### 1.5.2 执行采集

根据用户提示的关键词执行搜索/知识库查询，获取完整素材：

| 素材类型 | 采集方式 | 最小产出要求 |
|:---------|:---------|:-------------|
| 社会热点 | web_search（事件名+时间） | 能回答：发生了什么、谁和谁冲突、结果如何 |
| 民俗神话/民间传说 | web_search（故事名+完整故事） | 能回答：故事完整脉络、核心意象、原始文本 |
| 其他 IP | agent 知识 / web_search（设定关键词） | 能回答：IP 核心设定、叙事规律、关键机制 |
| 都市传说 | web_search（传说名+origin） | 能回答：传说内容、传播背景、核心恐怖点 |
| 用户自己想到的点子 | 用 clarify 追问 2-3 轮完整细节 | 用户口头补全到可执行的粒度 |

**拒绝臆测**：如果搜索后仍无法获取完整素材，标记"素材不完整，降级为简要笔记"，不能凭空编造素材内容。

### 1.5.3 信息采集深度/广度标准（★ SOP 硬规则）

以下规则不可违反。违反 = 素材采集不合格，退回重采。

#### 深度要求（每一条素材必须穿透以下三层）

| 层 | 问题 | 最低产出 |
|:---|:-----|:---------|
| **L1 表面层** | 发生了什么？谁、什么时间、什么地点、什么结果？ | 能回答 5W1H，不少于 3 条独立信源交叉验证 |
| **L2 结构层** | 这件事的内在逻辑是什么？各方的利益/动机/行为模式？ | 识别出至少 3 个参与方，每个参与方给出动机分析 |
| **L3 系统层** | 这件事反映了什么更广泛的社会/文化/历史模式？它不是一个孤立事件，它是什么趋势的缩影？ | 识别出至少 1 条与事件关联的更大链条（前后案例/相似模式/系统性原因） |

**深度质量门禁**：
- 只采集了 L1 表面层 → 不合格。必须意识到"我采集的只是一个孤例，它背后肯定有更深的模式"
- 采集了 L1+L2 但没到 L3 → 有条件合格。但在归档时备注"系统层分析待补充"
- 采集了 L1+L2+L3 三层 → 合格

#### 广度要求（每一条素材必须覆盖以下四个视角中的至少两个）

| 视角 | 核心问题 | 示例（水鬼事件） |
|:-----|:---------|:-----------------|
| **事件本体** | 这个事件本身的事实是什么？ | 钟皆连跳河→李燮牺牲→舆论发酵 |
| **平行案例** | 有没有类似的事件？它们是孤立发生还是一个模式？ | 小红书杀男教程→各地模仿案例→杭州救人反被举报 |
| **社会分析** | 媒体/学者/公众怎么分析这件事？它的社会根源是什么？ | 澎湃/虎嗅/知乎的深度分析→性别对立的结构性原因 |
| **历史参照** | 历史上有无类似模式？ | 古罗马角斗场时代的"被救者反噬救援者"故事 |

#### 信源多样性要求

| 素材类型 | 最少信源数 | 信源类型要求 |
|:---------|:----------|:-------------|
| 社会热点 | ≥3 | 至少 1 条主流媒体 + 1 条深度分析（虎嗅/澎湃/知乎长文） + 1 条不同立场的评论 |
| 历史事件 | ≥2 | 至少 1 条学术来源（知网/维基/社科文献） + 1 条不同视角（官方叙事 vs 民间叙事） |
| 民俗神话 | ≥2 | 至少 1 条原始文本（古籍/民间故事记录） + 1 条改编案例（已有同人/改编作） |
| 其他 IP | ≥1 | 但必须覆盖"核心设定"+"关键事件"+"叙事规律"三个维度 |

#### 采集完成信号

当你**同时**满足以下条件时，采集才算完成：
1. ✅ 最小产出要求已满足（5W1H 可回答）
2. ✅ 至少穿透到 L2 结构层
3. ✅ 至少覆盖 2 个广度视角
4. ✅ 信源数量满足多样性要求
5. ✅ 你不再觉得"对这件事只有一个印象级理解"——你开始在**识别模式**而不是**记住事实**

### 1.5.3 采集成果物

产出素材快照（写到对话中，不写文件——素材本身不持久化，只有归档后的条目进储备池）：

[采集到的完整素材描述，3-10 句话]

## 剧情改编四步法（★ 核心框架 — 所有素材必须按此框架走完四步）

> 素材储备池里一条合格素材的核心拷问，不是"这个改编对不对"——是"我的读者看完会不会爽"。
>
> **改编 = 原事实 + 各方立场挖掘 → 确定读者立场 → 释放读者情绪**

### 第一步：原事实本身

| 做什么 | 不做 |
|:-------|:-----|
| 采集事件的完整信息（5W1H + L1/L2/L3三层穿透） | 别急着下结论。事实没攒够之前，你的所有判断都是臆测 |

### 第二步：各方的立场、表现和问题挖掘

| 做什么 | 不做 |
|:-------|:-----|
| 找出所有相关方：谁主动/谁被动/谁在看/谁是系统 | 别只写"好人vs坏人"。真实世界很少有纯黑纯白——但**改编不一定需要还原复杂性** |
| 每个相关方问：他在这个事件里的利益是什么？他做了什么？他为什么这么做？ | |
| 最深的一层：这个事件暴露出什么系统性问题？ | |

### 第三步：读者的立场和看法（★ 核心）

**这是最重要的一步。跳过了这一步，你写出来的素材一定不痛不痒。**

| 问题 | 意义 |
|:-----|:------|
| 这本书的目标读者是谁？ | 男频 / 女频 / 悬疑 / 历史——不同读者对同一事件的天差地别 |
| 目标读者对这件事的既有立场是什么？ | 他们站在谁那边？他们的愤怒/同情/恐惧指向谁？ |
| 目标读者希望看到什么？ | 他们不想看"客观中立的分析"，他们想看**有人替他们出了这口气** |
| 如果写了一个"客观但不爽"的版本会怎样？ | 读者不会感激你的客观——他们会觉得你在给恶人洗白 |

**实战案例——大同案的迭代教训**：
- 第一次：我写了"中立剥离，谁都不得罪"的版本 → ❌ 被用户否决。因为男频读者不需要中立，他们需要一个和自己站在一起的叙事
- 第二次：我站了男生边，写了"系统保护女人迫害男人"的版本 → ✅ 用户认可立场，但高潮设计没对——我让江轩去和审判官辩论，这不是读者想看的
- 第三次：我让江轩一剑劈断亚空间灵能枢纽，整个产业链归零 → ✅ **读者想看的不是"讲道理"，是"有人把那个烂系统砸了"**

### 第四步：替读者释放情绪（爽）

**所有改编的最终目的是这个。前三步都是为这一步服务的。**

| 读者情绪 | 释放方式 | 实战案例 |
|:---------|:---------|:---------|
| **愤怒**（好人没好报） | 让加害者付出代价，让读者看到"报应" | 哭娃被净化后说了"那咋了"——读者愤怒拉满→江轩可以杀她/关她→情绪释放 |
| **憋屈**（系统不公） | 让主角把制造问题的源头砸了 | 江轩一剑劈断奸奇灵能枢纽，七座小镇的系统同时归零 |
| **无力**（一个人对抗不了大系统） | 让主角展示"一个人就够了" | 不需要军队不需要盟友，灰骑士的净化之火和一把剑就够 |
| **被背叛感**（信任被辜负） | 让背叛者面对真相，让读者看到"她再也骗不了人了" | 汤婆改配方后居民说"味道不对了"——她不是被杀的，是被抛弃的 |

### 四步法检查清单

写完每一条素材后，问自己四个问题：
1. ✅ 事实是否完整？（L1+L2+L3三层穿透）
2. ✅ 各方立场是否清晰？（谁、为什么、系统性问题是什么）
3. ✅ 我站的是目标读者的立场吗？（不是中立，不是客观——是站在读者的愤怒那边）
4. ✅ 读完这个改编，我的目标读者会**爽**吗？（如果答案不是明确的"会"，退回重做）

## Step 2：素材解吸

从用户提供的素材中提取 3 个核心维度：

### 2.1 冲突核心

> 这个素材的核心冲突是什么？不是一个设定/一段资讯，而是**谁和谁在争什么**。

```
示例——社会热点"某地官员腐败案"：
- 表层冲突：官员 vs 纪委/群众
- 深层冲突：系统腐败 vs 个人良知 / 规则 vs 人性
```

```
示例——民间传说"孟姜女哭长城"：
- 表层冲突：女子 vs 暴政
- 深层冲突：个体情感 vs 国家机器 / 传统孝道 vs 政治功利
```

### 2.2 独特设定

> 这个素材里最特别的 1-2 个设定/机制/画面，是什么让它值得被记住？

```
示例——都市传说"瘦长鬼影"：
- 独特设定：没有面孔的修长人形，诱拐儿童，受害者出现失忆症状
- 可用画面：树林中依稀可见的黑色轮廓，手指几乎触地
```

### 2.3 情感基调

> 这个素材传递什么情绪？适合什么叙事节奏？

```
示例——都市传说：
- 基调：悬疑/不安/缓慢侵蚀
- 不适合：热血/快节奏
```

## Step 2.5：受众立场与情绪评估（★ 核心新增 — 社会性安全闸门）

> **为什么需要这一步**：现实素材（社会热点/历史事件/争议IP）在大众中有**既定的道德立场和情绪基调**。你在书里怎么处理这个素材，读者会代入他们现实中的情绪。写对了是共鸣，写错了是网暴。
>
> **「一个英雄被一个恶人害死了——你写恶人可怜，读者会觉得你在给恶人洗白。这不是写得好不好的问题，是你还能不能在这个圈子活下去的问题。」**

### 2.5.1 三方立场分析

对于每个现实来源的素材，列出事件中的**三方立场**：施动方 / 受动方 / 观察方。

| 立场是谁 | 公众对他的定位 | 公众情绪 | 改编红线 |
|:---------|:--------------|:---------|:---------|
| **施动方**（主动行为者） | 正面/负面/争议 | 愤怒/同情/复杂 | ❌ 不能洗白公众认定的恶人 |
| **受动方**（承受行为者） | 正面/负面/争议 | 同情/愤怒/复杂 | ❌ 不能让英雄被侮辱 |
| **观察方**（第三方/系统） | 正面/负面/争议 | 信任/愤怒/麻木 | 可批判，但不能阴阳怪气 |

### 2.5.2 情绪能量检测

| 维度 | 问题 |
|:-----|:------|
| **主情绪** | 公众对这件事的第一反应是什么？（愤怒/悲伤/嘲讽/恐惧） |
| **副情绪** | 深入想之后，公众还感受到什么？（无力/绝望/希望/荒诞） |
| **情绪指向** | 情绪砸向谁？（施动方/受动方/系统/运气） |
| **情绪烈度** | 这件事对公众的冲击有多大？（日常讨论 vs 全民震怒 vs 现象级撕裂） |

### 2.5.3 适配安全评估

| 等级 | 含义 | 改编策略 |
|:-----|:-----|:---------|
| ✅ **安全** | 素材在大众中没有强烈情绪绑定 | 自由加工 |
| ⚠️ **敏感** | 素材涉及争议，改编需要保持距离 | 必须大幅剥离现实身份，只取结构 |
| 🔴 **禁区** | 素材涉及公众深层创伤或政治敏感 | 不建议直接改编；如必须用，需做三层剥离：去名→去背景→去具体发生地 |
| ☠️ **自杀** | 改编会导致受众认为你在为恶人辩护/侮辱英雄 | **绝对不要正面使用**；只能在完全相反的立场下侧面引用 |

### 2.5.4 产出

写入解吸记录中，作为后续所有融合决策的约束条件。格式：

```
# 受众立场评估

## 三方立场
| 立场 | 公众定位 | 公众情绪 | 红线 |
|------|---------|---------|------|
| {施动方} | {定位} | {情绪} | {红线} |
| {受动方} | {定位} | {情绪} | {红线} |
| {观察方} | {定位} | {情绪} | {红线} |

## 情绪能量
- 主情绪：{情绪}
- 副情绪：{情绪}
- 情绪指向：{对象}
- 烈度：{等级}

## 安全等级
{✅ 安全 / ⚠️ 敏感 / 🔴 禁区 / ☠️ 自杀}

## 改编约束
1. {具体改编中不能触碰的红线}
2. {具体改编中不能触碰的红线}
3. {具体改编中不能触碰的红线}
```

## Step 2.6：素材权重与剧情预算判定（★ 核心新增 — 防预算错配）

> **为什么需要这一步**：这是 v1.1 版本中犯的最严重的错误——把荷鲁斯大叛乱（银河内战级）当成了支线素材，把水鬼事件（单点社会现象）当成了主线素材。权重判定错了，后续所有设计都白做。

### 2.6.1 事件规模评估

从素材的**影响范围**和**持续时间**判断它在叙事中的基础权重：

| 等级 | 影响范围 | 示例 | 建议预算 |
|:-----|:---------|:-----|:---------|
| 🌍 **文明级** | 影响整个世界观（国家/大陆/宇宙）的格局，改变力量平衡 | 文明战争、诸神陨落、世界规则变更 | **2-4卷** 主线 |
| 🏛️ **势力级** | 影响一个或多个主要势力/国家/种族 | 宗教运动、王国战争、大瘟疫 | **1-2卷** 主线 |
| 🏠 **社区级** | 影响一个城镇/组织/小团体 | 小镇信仰崩塌、地方性腐败、局部阴谋 | **1幕（10-20章）** 支线 |
| 👤 **个体级** | 主要影响单个角色或几个人的关系 | 一个人的堕落、一段恩怨、一个秘密 | **1-3章** 短篇/闪回 |

### 2.6.2 叙事密度评估

同等规模的事件，叙事密度不同适配预算也不同：

| 密度 | 特征 | 预算调整 |
|:-----|:-----|:---------|
| **高密度** | 事件内在有多条子线、多个参与方、多次转折 | 预算上限 |
| **中密度** | 事件有一条主线 + 1-2条支线 | 预算中位 |
| **低密度** | 事件是一条单线，一个高潮 | 预算下限 |

### 2.6.3 权重判定矩阵

| 规模 | 高密度 | 中密度 | 低密度 |
|:-----|:-------|:-------|:-------|
| 🌍 文明级 | 4卷主线 | 3卷主线 | 2卷主线 |
| 🏛️ 势力级 | 2卷主线 | 1卷主线 | 1幕支线 |
| 🏠 社区级 | 2幕支线 | 1幕支线 | 1-3章短篇 |
| 👤 个体级 | 1幕支线 | 1-3章短篇 | 1-2章闪回 |

### 2.6.4 案例对照（来自 v2.0 实战）

| 素材 | 规模 | 密度 | 判定 | 我的 v1.1 误判 | 差距原因 |
|:-----|:-----|:-----|:-----|:--------------|:---------|
| 荷鲁斯大叛乱 | 🌍 文明级 | 高 | 4卷主线 | 1幕支线 | 只看了"叛乱"两个字，没看到"银河内战"的规模 |
| 义和团运动 | 🌍 文明级 | 高 | 3卷主线 | 1幕支线 | 只看了"民间信仰"四个字，没看到"半殖民地崩溃"的规模 |
| 水鬼事件 | 🏠 社区级 | 中 | 1幕支线 | 1卷主线 | 被情绪冲击误导，误判了事件的实际规模 |
| 鹅腿事件 | 🏠 社区级 | 中 | 1幕支线 | 1卷主线 | 被社会热度误导，忘了热度不等于叙事密度 |

### 2.6.5 产出

写入解吸记录中。如果判定为主线级（≥1卷），在归档时标记"high_budget"。

## Step 3：PRD 契合度评估

用项目 PRD 的加工哲学对照新素材：

### 3.1 加工匹配度

| PRD 说 | 素材符合 | 匹配度 |
|:-------|:---------|:-------|
| "一切元素都要经过机制→现实后果的拷问" | 素材天然有机制化的冲突链条 | **高** |
| "写实化，不降神" | 素材是都市传说，需去超自然元素 | **中**（需改写） |
| "硬核历史考据" | 素材是现代段子 | **低**（不推荐） |

### 3.2 边界检查

| PRD 边界 | 素材是否触碰 | 处理 |
|:---------|:------------|:-----|
| "不写主角降智" | 素材需要主角做蠢事来推进 | ❌ 冲突 → 备注\"需改写\" |
| "不写后宫" | 素材涉及多角关系 | ✅ 无冲突 |

### 3.3 宪法冲突检测

对照故事引擎.md 的"绝对不能做的事"章节，逐条检查。

## Step 4：适配方式判定

根据素材类型和项目类型，决定适配策略：

| 素材来源 | 项目类型 | 适配方式 | 示例 |
|:---------|:---------|:---------|:-----|
| 社会热点/现实事件 | 都市/现实 | **原生改写**：保留核心冲突，换人物名字 | 腐败案→书里的帮派清洗 |
| 社会热点/现实事件 | 玄幻/修仙 | **社会寓言**：保留权力结构，换包装层 | 腐败案→宗门长老会腐败 |
| 民俗神话 | 玄幻/修仙 | **直接嵌入**：保留原神话骨架，注入本世界观 | 孟姜女→某个修真者妻子的悲愿 |
| 民俗神话 | 都市 | **解构重塑**：取核心意象，去超自然 | 孟姜女→一个普通女人对抗拆迁办 |
| 其他 IP | 不限 | **规律提取**：取 IP 的叙事规律，不取设定 | 不是把哈利波特搬进来，是提取"凡人进入异世界重新确立身份"的规律 |
| 都市传说 | 悬疑/恐怖 | **原浆保留**：保持诡异的不可解感 | 瘦长鬼影→直接当都市怪谈线索 |
| 都市传说 | 玄幻 | **魔法化**：给超自然一个本世界解释 | 瘦长鬼影→某个灵体修士的残影 |

## Step 5：归档与写入

### 5.1 四类选择规则

| 评估结果 | 归档类别 |
|:---------|:---------|
| 冲突核心与本书主线天然契合 | **主线索材** |
| 冲突核心契合但影响范围有限 | **支线索材** |
| 只有独特设定/画面好，但无核心冲突 | **背景素材** |
| 好点子但 PRD 匹配度低或宪法冲突 | **剩余焊接点**（标注冲突原因） |

### 5.2 写入格式

打开 `00-原始设定/素材储备池.md`，在对应分类表格底部追加新行。

独立调起追加的条目标注`injected`来源：

```
| 4 | {素材名} | injected:{来源类型} | {简介} | {触发方式} | {优先级} | {PRD匹配度} | {适配类型} | {适配方式} |
```

### 5.3 防重复检查

在追加前遍历现有池子的「素材名」列，如有相似素材：
- 高度重复：追加到现有条目的「触发方式」列末尾（分号分隔）
- 部分相似：新增行，在「简介」中注明"与【第X条】相似，但……"

## 质量闸门（最终提交前逐一检查）

> 以下六道门全部通过后，才允许写入文件。任何一道未通过 → 退回对应步骤修正。

### 闸门 A：信息完整性

| 检查项 | 通过标准 | 验证方式 |
|:-------|:---------|:---------|
| 5W1H | L1表面层全部可回答 | 自己写一遍事件摘要，如果写不顺就是信息不全 |
| 三层穿透 | L1+L2+L3 已采集 | 检查采集记录是否有系统层分析 |
| 广度覆盖 | ≥2 个视角 | 检查采集记录是否有平行案例或历史参照 |

### 闸门 B：立场与情绪评估

| 检查项 | 通过标准 |
|:-------|:---------|
| 三方立场已识别 | 施动方/受动方/观察方都有明确的公众定位 |
| 安全等级已判定 | ✅/⚠️/🔴/☠️ 四选一，不能模糊 |
| 改编红线已列出 | ≥3 条具体红线，可执行 |

### 闸门 C：权重判定合理性

| 检查项 | 通过标准 |
|:-------|:---------|
| 规模已评估 | 文明级/势力级/社区级/个体级 四选一 |
| 密度已评估 | 高/中/低 三选一 |
| 预算与判定一致 | "文明级高密度 → 4卷主线"这种不会出现错配 |

### 闸门 D：权重复核——反向检查

**问自己一个问题**：
> 如果我用相反的权重去设计这个素材，会怎样？

| 场景 | 如果缩得太小 | 如果放得太大 | 正确位置 |
|:-----|:------------|:------------|:---------|
| 荷鲁斯大叛乱 | 1幕就写完 → 浪费史诗材料，读者会骂"这就完了？" | — | 多卷主线 ✓ |
| 水鬼事件 | — | 写成1卷主线 → 读者觉得注水，一两个案子撑不起一卷 | 1幕支线 ✓ |
| 义和团运动 | 1幕就写完 → 丢失历史纵深，变成"又一个邪教"的故事 | — | 多卷主线 ✓ |
| 鹅腿事件 | — | 写成1卷主线 → 读者觉得你在水字数 | 1幕支线 ✓ |

### 闸门 E：融合方案质量

| 检查项 | 通过标准 |
|:-------|:---------|
| 至少设计了 1 个具体角色 | 不是"某人"——有名字/动机/困境/弧线 |
| 至少设计了 1 个具体选择 | 不是"主角会面对这个"——两条路径都有代价 |
| 至少设计了 1 个具体场景 | 不是"会发生一场战斗"——有画面/台词/情绪 |
| 融合方案有核心拷问 | 有一条能用一句话说清的、与本书主题照应的终极问题 |

### 闸门 F：回溯——如果我在写这本书，这个素材储备对我有用吗？

> 最后问自己：**如果我是一个正在写这本书的网文作者，打开素材储备池，看到这条素材，我能在 5 分钟内直接拿来用吗？**

- ✅ 能——5分钟内我知道"这个素材适合哪条线、放在哪卷、核心角色是谁、核心选择是什么"
- ❌ 不能——退回补充。储备池里的每一条都应该是**可执行的设计方案**，不是灵感碎片
