# 生态角色群调研SOP

> 适用: pop-writer-v2-creative Step 1
> 定位: 生态图谱调研的标准操作流程，替代 v1 的域DNA研究

## 一、为什么用生态图谱替代域DNA

v1 用域DNA（赛道美学/经典元素/叙事节奏）做立项调研，问题是域DNA偏向"氛围参考"，与剧情驱动脱节。v2 改用生态图谱，直接从角色群生存方式推导冲突空间，让调研产出天然对接剧情。

## 二、角色群定义

角色群 ≠ 势力组织。角色群 = 共享生存方式 + 利益关联的群体集合。

判定标准：
- 同一群内成员的生存方式高度相似
- 同一群内成员有共同利益（至少面对共同威胁）
- 不同群之间有可识别的边界（地理/阶层/职业/种族）

## 三、调研流程

### 阶段1：群体识别（从拆解数据）
1. 扫描 pop-decon 角色表，提取每个角色的生存方式标签
2. 按生存方式聚类，初步划分群组
3. 目标：≥5个群（最低3个，红线❌3）

### 阶段2：群体深挖（每群5字段）
对每个群填写：
- **名称**：简洁代号（如"矿城苦力""宗门内门弟子"）
- **生存方式**：经济基础 + 资源获取途径
- **内部结构**：层级 / 分工 / 权力分配
- **与其他群关系**：至少标注与2个群的关系（对立/依附/共生/竞争）
- **可衍生冲突空间**：≥2个冲突方向

### 阶段3：跨群交叉点
在生态图谱末尾标注≥3个跨群冲突交叉点：
- 涉及哪些群
- 冲突性质（资源争夺/权力博弈/生存威胁/理念对立）
- 可衍生的剧情线方向

## 四、调研质量检查清单

| 检查项 | 合格标准 |
|:-------|:---------|
| 群体数量 | ≥5个（最低3个） |
| 字段完整度 | 每群5字段无遗漏 |
| 关系网密度 | 每群至少与2个群有关系标注 |
| 冲突交叉点 | ≥3个跨群交叉点 |
| 冲突多样性 | 冲突性质不重复（至少2种类型） |

## 五、与v1域DNA的映射关系

| v1 域DNA | v2 生态图谱 |
|:---------|:-----------|
| 赛道美学 → | 生态群内部结构的质感 |
| 经典元素 → | 角色群的生存方式 |
| 叙事节奏 → | 跨群冲突交叉点的节奏 |
| 域DNA摘要 → | 生态图谱文件 |

生态图谱不是域DNA的子集，是替代。域DNA的"氛围参考"功能由文风DNA（Step 3）承接。
