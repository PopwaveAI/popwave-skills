# 金手指设计指南

> 适用: pop-writer-v2-creative Step 2
> 定位: 5维度行动引擎设计规范 + 数值风格声明

## 一、核心原则：行动驱动

v2 金手指设计的核心是"行动驱动"——金手指不能只是被动观察/标注工具，必须能驱动主角行动。每个维度必须回答："这个维度让主角必须做什么？"

如果金手指只有观察/标注功能（如"能看到别人看不到的信息"），则标注风险并提示用户补充行动机制。

## 二、5维度详解

### 维度1：任务发布
- 定义：金手指如何向主角派发任务
- 行动驱动方式：任务触发事件 → 主角被迫行动
- 设计要点：任务来源（系统/NPC/金手指自主判断）+ 任务类型 + 拒绝代价
- 剧情锚点：每个任务 = 一个剧情弧线的触发器

### 维度2：技能获取
- 定义：主角如何通过金手指获得新能力
- 行动驱动方式：获得条件 → 解锁新行动选项
- 设计要点：获取路径（学习/吞噬/解锁/融合）+ 条件门槛 + 能力边界
- 剧情锚点：每次技能获取 = 一次行动能力跃迁

### 维度3：经验值
- 定义：行动如何积累并转化为成长
- 行动驱动方式：积累节奏 → 成长里程碑
- 设计要点：积累方式（战斗/任务/探索/特殊行为）+ 转化规则 + 里程碑节点
- 剧情锚点：经验积累 = 主角持续行动的内在动力

### 维度4：血量系统
- 定义：生存压力机制
- 行动驱动方式：HP阈值 → 危机触发 → 决策压力
- 设计要点：HP上限 + 恢复方式 + 阈值触发事件（如HP≤4触发危机）
- 剧情锚点：血量压力 = 紧迫感和决策权重

### 维度5：战宠系统
- 定义：伴随主角成长的外部力量
- 行动驱动方式：战宠进化 → 战力跃迁 → 情感线
- 设计要点：战宠获取 + 进化条件 + 战斗定位 + 情感纽带
- 剧情锚点：战宠 = 战斗线 + 情感线的双轨驱动

## 三、数值风格声明

金手指在 creative 阶段只声明数值风格倾向，具体数值留 world 阶段。

| 风格 | 特征 | 典型表现 | 适用场景 |
|:-----|:-----|:---------|:---------|
| 戏剧优先 | 数值服务于剧情节点 | HP=4触发危机，不是HP≤20%触发 | 情绪驱动型 |
| 模糊优先 | 不精确追踪数值 | 经验+10但不显示进度条 | 沉浸优先型 |
| 精确优先 | 明确数值面板 | 完整属性面板+数值对比 | 系统流 |

混合风格允许（如"戏剧优先+模糊优先"），但必须声明主次。

## 四、与v1金手指设计的差异

| v1 | v2 |
|:---|:---|
| 查金手指库做WHAT层参考 | 生态图谱驱动WHAT层 |
| 机制/代价/成长路径三要素 | 5维度行动引擎 |
| 数值留world | 数值风格声明在creative |
| 可只有观察功能 | 必须含≥1个行动驱动机制 |
