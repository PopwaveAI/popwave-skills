# 金手指数值化指南

> 适用: pop-writer-v2-world Step 3
> 定位: 5维度行动引擎的数值化规范，数值服务于戏剧性

## 一、核心原则：数值是剧情工具

v2 金手指数值化的核心原则：数值不是数学题，是剧情工具。每个数值都必须回答"这个数值在剧情中起什么作用"。

数值化分三种风格（从 creative 继承）：
- **戏剧优先**：数值阈值绑定剧情节点（HP=4触发危机）
- **模糊优先**：数值粗略不精确追踪（经验+10但不显示进度条）
- **精确优先**：数值面板完整可追踪（完整属性面板+数值对比表）

## 二、5维度数值化规范

### 维度1：任务发布
| 数值化项 | 规范 |
|:---------|:-----|
| 任务难度分级 | 与力量体系等级对齐（如F-SSS或1-10级） |
| 奖励数值 | 经验值/技能点/物品，与任务难度成正比 |
| 戏剧性服务点 | 高难任务=剧情高潮触发器 |

### 维度2：技能获取
| 数值化项 | 规范 |
|:---------|:-----|
| 获取条件量化 | 所需经验/材料/触发事件，与力量体系阶段对齐 |
| 技能效果数值 | 效果范围/持续时间/冷却，服务于战斗张力 |
| 戏剧性服务点 | 解锁条件=剧情里程碑 |

### 维度3：经验值
| 数值化项 | 规范 |
|:---------|:-----|
| 每行动经验值 | 战斗/任务/探索/特殊行为各给多少 |
| 升级阈值 | 每级所需经验，递增曲线还是阶梯式 |
| 戏剧性服务点 | 升级节奏=爽点节奏 |

### 维度4：血量系统
| 数值化项 | 规范 |
|:---------|:-----|
| HP上限 | 与力量体系等级挂钩 |
| 恢复速率 | 休息/物品/技能恢复 |
| 阈值触发 | 戏剧优先时用整数阈值（HP=4触发危机），不用百分比 |
| 戏剧性服务点 | HP阈值=紧迫感和决策压力 |

### 维度5：战宠系统
| 数值化项 | 规范 |
|:---------|:-----|
| 战宠等级 | 与主角力量体系等级平行或挂钩 |
| 属性加成 | 战宠为主角提供多少战力加成 |
| 进化阈值 | 进化所需条件（经验/材料/事件） |
| 戏剧性服务点 | 进化=战力跃迁剧情节点 |

## 三、行动驱动校验

数值化后必须校验金手指仍保持行动驱动（从 creative 继承，红线❌1）：

| 维度 | 校验问题 | 通过标准 |
|:-----|:---------|:---------|
| 任务发布 | 数值化后任务是否仍驱动行动？ | 任务数值→主角必须行动 |
| 技能获取 | 数值化后获取是否仍需行动？ | 条件达成→必须主动行动 |
| 经验值 | 数值化后积累是否仍驱动行动？ | 经验积累→持续行动动力 |
| 血量系统 | 数值化后HP是否仍驱动行动？ | HP阈值→必须决策行动 |
| 战宠系统 | 数值化后进化是否仍驱动行动？ | 进化条件→必须行动 |

## 四、与v1数值体系的差异

| v1 | v2 |
|:---|:---|
| 从L1力量体系推导战力值 | 从金手指行动引擎推导数值 |
| 数值独立于剧情 | 数值服务于戏剧性 |
| 精确数值面板为主 | 三种风格可选，戏剧优先推荐 |
| 不校验行动驱动 | 从creative继承行动驱动校验 |
