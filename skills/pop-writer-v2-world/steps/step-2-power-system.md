# Step 2：力量体系

> 管线: pop-writer-v2-world v1.0.0
> 消费: 世界宪法 + 金手指行动引擎
> 产出: 力量体系文件

## 读什么
- Step 1 产出的 `世界宪法.md`
- creative 的 `金手指行动引擎.md`

## 做什么

### 1. 从金手指行动引擎推导力量体系
金手指行动引擎的5维度决定了力量体系的结构。推导关系：

| 金手指维度 | 力量体系映射 |
|:-----------|:-------------|
| 任务发布 | 力量考核节点（任务难度=力量等级门槛） |
| 技能获取 | 力量分支路径（技能树=力量体系分支） |
| 经验值 | 力量等级阶梯（经验积累=升级节奏） |
| 血量系统 | 力量对抗标尺（HP差异=等级压制幅度） |
| 战宠系统 | 力量辅助维度（战宠等级=力量加成） |

### 2. 力量等级设计
从经验值维度推导等级体系：

| 项目 | 内容 |
|:-----|:-----|
| 等级总数 | {从经验值积累节奏推导} |
| 阶段划分 | {每个阶段含几个等级} |
| 突破条件 | {从技能获取维度推导} |
| 能力边界 | {每级能做什么/不能做什么} |
| 跨级约束 | {越级战斗规则} |

### 3. 与世界宪法的对齐
检查力量体系是否与宪法中的魔法法则一致：
- 力量来源是否与宪法一致
- 力量限制是否与宪法一致
- 力量代价是否与宪法一致

## 门禁
| 检查项 | 失败动作 |
|:-------|:---------|
| 力量体系从金手指行动引擎推导 | 重新推导，标注来源维度 |
| 与世界宪法无矛盾 | 修正矛盾条款 |
| 等级/阶段/边界全部定义 | 补全缺失项 |

## 产出
`力量体系.md` — 含等级表 + 阶段划分 + 能力边界 + 金手指维度映射。
