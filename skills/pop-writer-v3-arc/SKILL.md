# pop-writer-v3-arc · 事后复盘与账本校准

arc 的职责是把已经写完的正文重新拆书，校准下一轮 plot 的起点。

核心原则：**剧情线是事后命名的，设定是使用后冻结的。**

## 触发

- 每个 L2/幕级单元最后一章落盘后触发。
- 用户主动要求复盘、查线、查设定漂移、准备下一单元时触发。

## 输入

| 输入 | 用途 |
|:-----|:-----|
| 本单元正文 | 事后拆解的事实来源 |
| plot 状态契约/运行日志/单元卡 | 对照设计轨迹 |
| 设定账本 | 检查新增设定、漂移和冻结状态 |
| 活记忆 | 更新下一单元起点 |
| 上一轮 L3 卡 | 延伸长线 |

## 做什么

1. **反拆实际单元卡**：从正文中提取实际物理坐标、事件链、情绪波形、边界信号。
2. **命名真实剧情线**：只命名正文里真正推进过的线，不保留空线。
3. **更新设定账本**：把实际使用过的新设定冻结；把未使用设定缺口继续保留；标记矛盾。
4. **校准 plot 偏差**：记录设计运行日志与正文实际差异，但不默认回退。
5. **更新活记忆**：用自然语言压缩当前状态，作为下一单元状态快照。
6. **更新 L3 长线**：把跨单元线的阶段变化追加进去。

## 红线

| 编号 | 红线 |
|:-----|:-----|
| R1 | 不从 plot 设计稿命名剧情线，必须从正文事实中提取。 |
| R2 | 不保留没有实际推进的空子线。 |
| R3 | 设定账本以正文实际使用为准；未使用的设定只能留作缺口或废弃。 |
| R4 | 发现重大偏差时先报告影响范围，不自动大回退。 |
| R5 | 活记忆只保留下一单元必需状态，不写复盘长文。 |

## 步骤加载

| 步骤 | 文件 | 说明 |
|:-----|:-----|:-----|
| step-1 | `steps/step-1-macro-check.md` | 实际正文 vs 运行日志/单元契约偏差检查 |
| step-2 | `steps/step-2-setting-prune.md` | 设定账本冻结、缺口、废弃、矛盾处理 |
| step-3 | `steps/step-3-幕纲-update.md` | 把单元卡从设计稿更新为实际记录 |
| step-4 | `steps/step-4-l3-card-distill.md` | 从实际正文提炼/更新 L3 长线 |
| step-5 | `steps/step-5-memory-compress.md` | 压缩活记忆，形成下一单元起点 |

## 输出

- `弧线校准报告/arc-XX.md`
- 更新后的 `卷纲/幕NNN-{名称}.md` 实际记录
- 更新后的 `卷纲/运行/幕NNN-{名称}-设定账本.md`
- `卷纲/L3-NNN.md` 长线卡
- 更新后的 `活记忆/活记忆.yaml`
