# 弧线校准

> 管线: 涌现式写作v3.5 · 弧线校准 skill v1.4.0
> 定位: 宏观检查 + 设定修剪 + 幕纲更新 + L3卡提炼 + 活记忆压缩

每个L2单元结束时触发完整arc，执行5步全跑。不省token——每个单元都跑完整流程，确保管线反馈闭环无延迟。

## 校准触发条件

| 触发类型 | 条件 |
|:---------|:-----|
| L2单元结束触发 | L2单元最后一章的Step5（落盘）完成后自动触发arc（完整5步全跑） |
| 用户触发 | 用户主动要求宏观检查 |

触发示例：

```
L2-001（ch001-005）写完 → arc完整执行（5步全跑）
L2-002（ch006-010）写完 → arc完整执行（5步全跑）
L2-003（ch011-015）写完 → arc完整执行（5步全跑）
```

## ❌ 质量红线

| 编号 | 红线 | 检查方式 |
|:-----|:-----|:---------|
| ❌1 | 每个L2单元结束时必须触发arc完整5步（不省token） | 检查每个L2单元结束后是否有对应的arc报告 |
| ❌2 | 幕纲更新必须把"设计方案"更新为"实际记录"（物理坐标+子线状态+咬合关系全部回填） | 检查幕纲物理坐标段是否有实际回填值，不能只有设计占位 |
| ❌3 | L3卡不事前设计，只从实际写作中事后提炼 | 检查L3卡创建时间在L2单元完成之后，不在plot设计阶段创建 |
| ❌4 | 回退重写必须用户确认范围后执行 | 严重回退前展示影响范围，等待用户确认 |

| 子流程 | 触发条件 | 参考文件 |

## 路由

upstream `expert-writer`（每L2单元最后一章Step5落盘完成后）→ **本skill** → downstream `expert-writer`（校准后返回涌现写作循环，进入下一个L2单元设计）

## 产出

- `弧线校准报告/arc-XX.md`（落盘，含5步完整记录）
- 更新后的幕纲（`卷纲/幕NNN-{名称}.md`，设计方案→实际记录，供下一个L2设计时优先级1参考）
- L3卡（`卷纲/L3-NNN.md`，新创建或追加阶段更新，供后续L2设计时优先级4参考）
- 活记忆压缩（自然语言合并，baseline+events→新baseline）
- 设定修剪记录（`写作参考/已废弃/`，不删除历史，只标记状态）
- 回退指令（如需，用户确认后执行）
