# 回退执行流程

> 适用: pop-writer-v3-arc Step 1（条件子流程）
> 定位: 三级回退的详细操作流程
> v3.5更新: 去除种子文档引用，回退对象改为活记忆+正文+幕纲

## 一、回退哲学

弧线校准发现问题后，根据严重度选择回退级别。**回退重写必须用户确认范围后执行（红线❌4）。** 回退不是惩罚，是纠偏——让故事回到正确轨道。

回退机制是Step 1宏观检查的产出之一，不作为独立步骤编号。当Step 1判定为严重回退时，暂停主流程执行回退，完成后继续Step 2。

## 二、严重度判定矩阵

| 严重度 | 判定标准 | 处理方式 | 是否需用户确认 |
|:-------|:---------|:---------|:-------------|
| 轻微 | 单章质量问题，不影响后续 | 打补丁：重写该章 | 否 |
| 中等 | 2-3章累积偏差，活记忆可吸收 | 微调方向提示 | 否 |
| 严重 | 弧线级偏移（主线进度❌ / 设定一致❌ / 幕纲偏离度>40%） | 回退重写 | 是（红线❌4） |
| 无需 | 六项检查全部✅且完成度检查无预警 | 无动作 | 否 |

### 判定决策树

```
六项检查有❌吗？
├─ 是 → ❌涉及多章吗？
│       ├─ 是 → 严重（候选回退重写）
│       └─ 否 → 评估❌性质
│               ├─ 弧线级（主线进度/设定一致/幕纲偏离度>40%）→ 严重
│               └─ 单章级 → 轻微（重写该章）
└─ 否 → 幕纲完成度检查有预警吗？
        ├─ 是，预警>=3项 → 中等（微调方向）
        ├─ 是，预警<3项 → 轻微（记录待关注）
        └─ 否 → 无需回退
```

## 三、轻微回退：打补丁（重写该章）

### 适用场景
- 单章导演意图验证未通过
- 单章事件密度过低
- 单章钩子不合格

### 执行流程

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 定位问题章 | 从Step 1检查结果中定位具体章号 |
| 2 | 重新进入涌现写作循环 | 加载幕纲+活记忆（该章前的状态）+设定 |
| 3 | 重写该章正文 | 覆盖 `正文/chXXX.md` |
| 4 | 更新活记忆 | 删除该章event，追加重写后的event |
| 5 | 不影响其他章节 | 正文其他章节和活记忆其他event不动 |

## 四、中等回退：微调方向提示

### 适用场景
- 2-3章累积偏差（如事件密度持续偏低）
- 幕纲完成度检查预警>=3项
- 子线推进覆盖率不足

### 执行流程

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 识别偏差项 | 从Step 1检查结果中提取⚠️项 |
| 2 | 生成纠偏方向提示 | 针对每个偏差项生成1-2条方向提示 |
| 3 | 写入活记忆 | 方向提示追加到最近event的备注 |
| 4 | 不删任何内容 | 正文和活记忆均保留 |
| 5 | 标记待验证 | 在校准报告中标记"待验证"，下次arc重点检查 |

### 方向提示格式示例

```
方向提示（arc纠偏）L2-00N+1:
- 事件密度需提升至>=8个/章（本单元平均5个）
- 子线推进覆盖率需提升（本单元仅推进50%设计子线）
- 幕纲偏离度40%+，下一L2设计时需更贴合实际
```

## 五、严重回退：回退重写

### 适用场景
- 主线进度❌（设计事件大面积未兑现）
- 设定一致❌（行为准则矛盾/金手指限制被突破）
- 幕纲偏离度>40%（实际剧情与设计严重偏差）
- 多项❌且涉及多章

### 执行流程

**红线❌4：必须用户确认范围后执行。**

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 确定回退点 | 确定回退到第N章（偏差开始前的最后一章） |
| 2 | 计算影响范围 | 列出将删除的正文章号 + 活记忆entries + 幕纲更新记录 |
| 3 | 输出回退建议 | 在校准报告中写明回退建议 |
| 4 | 展示给用户 | 向用户展示影响范围，请求确认 |
| 5 | 等待用户确认 | 用户确认后才执行；用户拒绝=不执行，降级为中等回退 |
| 6 | 备份 | 备份活记忆.yaml和幕纲 |
| 7 | 删活记忆entries | 删除entries中chapter > N的所有条目 |
| 8 | 删正文 | 删除 `正文/chXXX.md` 中所有ch > N的文件 |
| 9 | 幕纲回退 | 如回退范围内有幕纲更新（Step 3产出），恢复幕纲到设计阶段版本 |
| 10 | L3卡回退 | 如回退范围内有L3卡更新（Step 4产出），恢复L3卡到上一阶段版本 |
| 11 | 更新项目总控 | 章号回退到N，arc计数+1 |
| 12 | 从N+1章重写 | 重新进入涌现写作循环，从第N+1章开始 |

### 影响范围展示格式（给用户确认）

```
回退建议：回退到第 ch{N} 章

将删除：
- 正文：ch{N+1} ~ ch{M}（共{M-N}章）
- 活记忆event：chapter {N+1} ~ {M}（共{M-N}条）
- 幕纲更新记录：{如回退范围内包含arc Step 3更新，需回退到设计阶段版本}
- L3卡更新记录：{如回退范围内包含arc Step 4更新，需回退到上一阶段版本}

将保留：
- 正文：ch1 ~ ch{N}
- 活记忆baseline + event: chapter 0 ~ {N}
- 幕纲：设计阶段版本（回退arc更新）
- L3卡：上一阶段版本（回退arc更新）

请确认是否执行回退？
```

### 注意事项
- 严重回退代价高（重写整个L2单元），必须用户确认
- 回退后从N+1章重写时，需带上校准发现的问题作为方向提示
- 回退不删除`写作参考/已废弃/`的修剪记录（历史不删）
- 回退后活记忆可能需要重新压缩（如event数量变化）

## 六、回退后的衔接

| 回退级别 | 衔接动作 |
|:---------|:---------|
| 轻微 | 重写该章后，继续正常arc流程（Step 2起） |
| 中等 | 方向提示写入后，继续正常arc流程（Step 2起） |
| 严重 | 回退后从N+1章重新进入涌现写作循环，带纠偏方向提示 |

所有级别的回退处理完成后，继续arc的Step 2（设定修剪）→Step 3（幕纲更新）→Step 4（L3卡提炼）→Step 5（活记忆压缩），然后输出校准报告落盘。
