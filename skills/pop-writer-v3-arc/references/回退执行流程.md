# 回退执行流程

> 适用: pop-writer-v3-arc Step 3
> 定位: 三级回退的详细操作流程

## 一、回退哲学

弧线校准发现问题后，根据严重度选择回退级别。**回退重写必须用户确认范围后执行（红线❌1）。** 回退不是惩罚，是纠偏——让故事回到正确轨道。

## 二、严重度判定矩阵

| 严重度 | 判定标准 | 处理方式 | 是否需用户确认 |
|:-------|:---------|:---------|:-------------|
| 轻微 | 单章质量问题，不影响后续 | 打补丁：重写该章 | 否 |
| 中等 | 2-3章累积偏差，活记忆可吸收 | 微调方向提示 | 否 |
| 严重 | 弧线级偏移 | 回退重写：删entries+重写10-20章 | 是（红线❌1） |
| 无需 | 六项检查全部✅ | 无动作 | 否 |

### 判定决策树

```
六项检查有❌吗？
├─ 是 → ❌涉及多章吗？
│       ├─ 是 → 严重（候选回退重写）
│       └─ 否 → 评估❌性质
│               ├─ 弧线级（角色弧线/战力/目的地）→ 严重
│               └─ 单章级 → 轻微（重写该章）
└─ 否 → ⚠️累积>=3项吗？
        ├─ 是 → 中等（微调方向）
        └─ 否 → 无需回退
```

## 三、轻微回退：打补丁（重写该章）

### 适用场景
- 单章反思未通过但未在涌现写作时纠正
- 单章事件密度过低
- 单章钩子不合格

### 执行流程

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 定位问题章 | 从Step 1检查结果中定位具体章号 |
| 2 | 重新进入涌现写作环 | 加载种子+活记忆（该章前的状态）+文风DNA |
| 3 | 重写该章正文 | 覆盖 `正文/chXXX.md` |
| 4 | 更新活记忆 | 删除该章event，追加重写后的event |
| 5 | 不影响其他章节 | 正文其他章节和活记忆其他event不动 |

### 注意事项
- 重写时需参考方向提示，确保纠偏
- 重写后需重新通过五问反思
- 活记忆中该章的event替换，不是追加

## 四、中等回退：微调方向提示

### 适用场景
- 2-3章累积偏差（如事件密度持续偏低）
- 角色弧线轻微漂移
- 节奏波形单调

### 执行流程

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 识别偏差项 | 从Step 1检查结果中提取⚠️项 |
| 2 | 生成纠偏方向提示 | 针对每个偏差项生成1-2条方向提示 |
| 3 | 写入活记忆 | 方向提示追加到最近event的direction_hint字段 |
| 4 | 不删任何内容 | 正文和活记忆均保留 |
| 5 | 标记待验证 | 在校准报告中标记"待验证"，下次校准重点检查 |

### 方向提示格式示例

```
方向提示（校准纠偏）ch{N+1}:
- 事件密度需提升至>=8个/章（最近3章平均5个）
- 角色决策风格回归"风险规避型"（最近2章偏激进）
- 下2章需触及MS-13线索（已8章未出现）
```

### 注意事项
- 方向提示不指定具体事件，只说方向
- 纠偏靠后续章节自然完成，不强制重写
- 下次校准时必须验证偏差项是否改善

## 五、严重回退：回退重写

### 适用场景
- 角色弧线❌（人设漂移/驱动力断层）
- 战力曲线❌（连续跳级/严重膨胀）
- 目的地连续20章无进展❌
- 多项❌且涉及多章

### 执行流程

**红线❌1：必须用户确认范围后执行。**

| 步骤 | 动作 | 说明 |
|:-----|:-----|:-----|
| 1 | 确定回退点 | 确定回退到第N章（偏差开始前的最后一章） |
| 2 | 计算影响范围 | 列出将删除的正文章号 + 活记忆entries + 种子生长记录 |
| 3 | 输出回退建议 | 在校准报告中写明回退建议 |
| 4 | 展示给用户 | 向用户展示影响范围，请求确认 |
| 5 | 等待用户确认 | 用户确认后才执行；用户拒绝=不执行，降级为中等回退 |
| 6 | 备份 | 备份活记忆.yaml和种子/文件夹 |
| 7 | 删活记忆entries | 删除entries中chapter > N的所有条目 |
| 8 | 删正文 | 删除 `正文/chXXX.md` 中所有ch > N的文件 |
| 9 | 种子版本回退 | 如回退范围内有种子生长，`种子/_log.md`变更日志标记"回退"，`种子/_index.yaml`版本号回退 |
| 10 | 更新项目总控 | 章号回退到N，弧线计数+1 |
| 11 | 从N+1章重写 | 重新进入涌现写作环，从第N+1章开始 |

### 影响范围展示格式（给用户确认）

```
回退建议：回退到第 ch{N} 章

将删除：
- 正文：ch{N+1} ~ ch{M}（共{M-N}章）
- 活记忆event：chapter {N+1} ~ {M}（共{M-N}条）
- 种子生长记录：{N_grow}条（版本 {X.X.X} → {Y.Y.Y}）

将保留：
- 正文：ch1 ~ ch{N}
- 活记忆baseline + event: chapter 0 ~ {N}
- 种子文件夹：版本 {Y.Y.Y}（回退到生长前）

请确认是否执行回退？
```

### 种子版本回退规则

| 情况 | 处理 |
|:-----|:-----|
| 回退范围内无种子生长 | 种子版本不变 |
| 回退范围内有种子生长 | `种子/_log.md`变更日志追加"回退"记录，`种子/_index.yaml`版本号回退到最后一次生长前 |
| 回退范围内有种子修剪 | 修剪记录保留在`种子/_log.md`已关闭区（不撤销修剪），版本号相应调整 |

### 注意事项
- 严重回退代价高（重写10-20章），必须用户确认
- 回退后从N+1章重写时，需带上校准发现的问题作为方向提示
- 回退不删除`种子/_log.md`已关闭区的修剪记录（历史不删）
- 回退后活记忆可能需要重新压缩（如event数量变化）

## 六、回退后的衔接

| 回退级别 | 衔接动作 |
|:---------|:---------|
| 轻微 | 重写该章后，继续正常涌现写作环 |
| 中等 | 方向提示写入后，继续正常涌现写作环 |
| 严重 | 回退后从N+1章重新进入涌现写作环，带纠偏方向提示 |

所有级别的回退完成后，继续Step 4（活记忆压缩），然后输出校准报告落盘。
