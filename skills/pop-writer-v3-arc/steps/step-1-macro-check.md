# Step 1：六项宏观检查

> 管线: pop-writer-v3-arc v1.2.0
> 消费: 活记忆全量 + 种子文档 + 正文（最近10-20章）
> 产出: 六项宏观检查结果表

## 读什么

- `活记忆/活记忆.yaml`（全量：最后baseline + 全部event）
- `种子文档.md`（当前版本，含六要素+已关闭区+变更日志）
- `正文/chXXX.md`（最近10-20章正文，用于交叉验证）
- `项目总控.md`（章号+弧线计数+种子版本）

## 做什么

逐项执行六项宏观检查。每项检查必须引用活记忆具体数据作为证据，不可凭感觉判定。详细标准见 `references/宏观检查清单.md`。

### 检查1：种子追踪

扫描活记忆 `seeds` 组件中所有伏笔条目，逐条核对状态。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐条检查每条伏笔的埋设章号 | 距当前章 >20章且状态仍为"活跃" → ⚠️超期预警 |
| 检查是否有已回收伏笔未标记 | 正文已兑现但状态未更新 → ❌状态错误 |
| 统计活跃/已回收/超期数量 | 输出分布表 |

### 检查2：战力曲线

读取活记忆 `power_curve` 组件，绘制主角能力值时间序列。

| 检查动作 | 判定 |
|:---------|:-----|
| 检查相邻章节能力值变化幅度 | 连续跳级（单章增幅>2个台阶）→ ❌ |
| 对比种子§成长路径的阶段设计 | 实际成长偏离设计路径 → ⚠️ |
| 检查是否有长期停滞（>10章无变化） | 停滞且无剧情理由 → ⚠️ |

### 检查3：角色弧线

读取活记忆 `characters` 组件，追踪主角驱动力三层演化。

| 检查动作 | 判定 |
|:---------|:-----|
| 检查驱动力三层（表层/中层/深层）激活进度 | 深层驱动力应在ch15后浮现，未浮现→⚠️ |
| 检查角色决策风格是否一致 | 决策风格与种子§主角引擎定义漂移 → ❌ |
| 检查是否有长期停滞（>10章角色状态无变化） | 停滞且无剧情理由 → ⚠️ |

### 检查4：宏观节奏

读取活记忆 `rhythm_log` 组件，分析事件密度波形。

| 检查动作 | 判定 |
|:---------|:-----|
| 统计最近章节的事件密度趋势 | >=3章连续低密度（<6个事件/章）→ ⚠️ |
| 检查爽点兑现频率 | 连续3章无爽点兑现 → ❌ |
| 检查钩子类型分布 | 连续3章非即时钩子 → ⚠️ |

### 检查5：目的地进度

读取活记忆 `destinations` 组件，核对各目的地到达状态。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐个检查5-10个目的地的当前状态 | 连续20章无任何目的地进展 → ❌ |
| 检查是否有目的地偏离 | 涌现方向与目的地矛盾 → ⚠️ |
| 检查已到达目的地是否标记 | 已到达但状态未更新 → ❌ |

### 检查6：冲突轴

读取活记忆 `seeds` 组件中活跃线索部分 + 种子§冲突轴。

| 检查动作 | 判定 |
|:---------|:-----|
| 逐条检查活跃线索的最近触及章号 | >=10章未触及的线索 → ⚠️休眠预警 |
| 检查是否需要新线索 | 活跃线索<3条 → ⚠️需补充 |
| 检查四层生态是否都有冲突覆盖 | 某层连续10章无冲突 → ⚠️ |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 六项检查全部完成 | 缺项补查，不可跳过 |
| 每项检查有具体活记忆数据引用 | 无数据引用=重做该项 |
| 检查结果状态明确（✅/⚠️/❌） | 状态模糊=重判该项 |
