# Step 1：实际单元反拆与偏差检查

> arc 从正文事实出发，不从设计稿出发。

## 消费

- 本单元正文
- plot 状态契约
- plot 运行日志
- plot 单元卡
- 设定账本
- 活记忆

## 1. 反拆实际单元卡

从正文提取：

- 实际时间线。
- 实际地点轨迹。
- 实际人物状态变化。
- 实际力量/资源变化。
- 实际事件链。
- 实际章末推力。

## 2. 对照单元终点契约

逐项判断：

| 契约项 | 实际是否达成 | 偏差 | 处理 |
|:-------|:-------------|:-----|:-----|
| 人物状态 | | | |
| 力量/资源 | | | |
| 关系/声望 | | | |
| 冲突闭合 | | | |
| 新压力/线索 | | | |

偏差不自动回退。只在影响下一单元因果时标记为严重。

## 3. 命名真实剧情线

只保留正文里实际推进过的线。

每条线必须有：

- 起点事件。
- 推进节点。
- 不可逆变化。
- 闭合或延伸状态。

没有推进的空线删除。

## 4. 检查设定漂移

标记：

- 正文新增但未入账的设定。
- 正文与硬设定冲突。
- plot 设定缺口已被正文解决但未冻结。
- plot 设定缺口仍未解决。

交给 step-2 处理。

## 5. 输出

写入 arc 报告：

```markdown
## 实际单元反拆

## 终点契约达成情况

## 真实剧情线

## 设定漂移/缺口

## 下一单元输入建议
```
