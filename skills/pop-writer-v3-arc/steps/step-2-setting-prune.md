# Step 2：设定账本维护

> 设定以正文实际使用为准。不是删设定，而是冻结、保留缺口、标记废弃或矛盾。

## 做什么

### 1. 冻结正文实际使用的新设定

格式：

```markdown
- {设定}：首次出现 chXXX；服务事件 {事件}；后续约束 {规则}
```

### 2. 保留待解释缺口

未解决但仍有叙事价值的缺口继续保留，给下一轮 plot 使用。

### 3. 标记废弃设定

如果 plot 预想设定没有进入正文，且后续无价值，移入废弃区或标注未采用。

### 4. 标记矛盾

如果正文与硬设定冲突，不自动修改正文。记录影响范围，交给用户或主流程决定是否回退。

## 输出

更新 `卷纲/运行/幕NNN-{名称}-设定账本.md`：

```markdown
## 已冻结设定
## 本单元新增设定
## 待解释缺口
## 废弃/未采用
## 矛盾待处理
## 禁止漂移
```
