# Step 3：回退机制

> 管线: pop-writer-v3-arc v1.2.0
> 消费: Step 1宏观检查结果 + Step 2种子修剪结果
> 产出: 回退建议（写入校准报告）

## 读什么

- Step 1 六项宏观检查结果（状态标记 + 问题清单）
- Step 2 种子修剪记录
- `references/回退执行流程.md`（三级回退详细操作）
- `活记忆/活记忆.yaml`（评估回退影响范围）

## 做什么

根据宏观检查结果判定问题严重度，输出回退建议。**回退重写必须用户确认范围后执行（红线❌1）。**

### 1. 严重度判定

综合六项检查结果，判定问题严重度：

| 严重度 | 判定标准 | 处理方式 |
|:-------|:---------|:---------|
| 轻微 | 单章质量问题（反思未通过），不影响后续 | 打补丁：重写该章 |
| 中等 | 2-3章累积偏差，活记忆可吸收 | 微调方向提示 + 后续章节自然纠正 |
| 严重 | 弧线级偏移（角色弧线❌ / 战力曲线❌ / 目的地连续20章无进展❌） | 回退重写：删entries + 重写10-20章 |

判定逻辑：
- 六项检查中有任意❌且涉及多章 → 评估为严重
- 六项检查中有⚠️累积>=3项 → 评估为中等
- 六项检查中仅有单章⚠️ → 评估为轻微
- 六项检查全部✅ → 无需回退

### 2. 轻微回退（打补丁）

| 动作 | 说明 |
|:-----|:-----|
| 定位问题章 | 从检查结果中定位具体章号 |
| 重写该章 | 重新进入涌现写作环，仅重写指定章节 |
| 更新活记忆 | 删除该章event，追加重写后的event |
| 不影响其他章节 | 正文其他章节不动 |

### 3. 中等回退（微调方向）

| 动作 | 说明 |
|:-----|:-----|
| 生成纠偏方向提示 | 针对偏差项生成2-3条方向提示 |
| 写入活记忆 | 方向提示追加到最近event的direction_hint |
| 不删任何章节 | 正文和活记忆均保留，靠后续章节自然纠正 |
| 下次校准验证 | 标记"待验证"，下次校准时重点检查偏差项 |

### 4. 严重回退（回退重写）

**红线❌1：必须用户确认范围后执行。**

| 动作 | 说明 |
|:-----|:-----|
| 输出回退建议 | 建议回退到第N章，列出影响范围 |
| 展示影响范围 | 将删除的正文章号 + 活记忆entries + 种子生长记录列出 |
| 等待用户确认 | 用户确认后才执行，不确认=不执行 |
| 执行回退 | 删活记忆entries中chapter>N的条目 → 删正文ch>N的文件 → 从第N+1章重写 |
| 种子版本回退 | 如回退范围内有种子生长，变更日志标记"回退"，版本号回退到生长前 |

### 5. 无需回退

如果六项检查全部✅：
- 记录"本次校准无需回退"
- 直接进入Step 4

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 严重回退必须用户确认范围（红线❌1） | 未确认=不执行，记录待确认状态 |
| 回退建议有明确严重度判定 | 无判定依据=重新评估 |
| 严重回退影响范围完整列出 | 范围不完整=补全后再提交用户 |

## 产出

回退建议表（填入校准报告草稿）：

| 项目 | 内容 |
|:-----|:-----|
| 严重度 | 轻微/中等/严重/无需回退 |
| 判定依据 | 引用Step 1检查结果 |
| 回退建议 | 具体动作描述 |
| 影响范围 | 涉及章号 + 活记忆entries + 种子版本 |
| 用户确认状态 | 待确认 / 已确认 / 无需确认 |
