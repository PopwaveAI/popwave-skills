# Step 4：L3卡提炼/更新

> 管线: pop-writer-v3-arc v1.4.0
> 消费: 更新后的L2卡（Step 3产出） + 活记忆events + 已有L3卡（如存在） + 正文
> 产出: L3卡（新创建或追加阶段更新，供后续L2设计时优先级4参考）

## 读什么

- `卷纲/L2-NNN.md`（Step 3更新后的L2卡——实际记录版本，重点读嵌套子线+子线咬合+物理坐标）
- `活记忆/活记忆.yaml`（本单元全部events——自然语言状态变化日志）
- `卷纲/L3-NNN.md`（已有的L3剧情线卡，如存在——读取全部已有L3卡）
- `正文/chXXX.md`（本单元全部章节正文，用于提取剧情线细节）
- `templates/L3卡-模板.md`（首次创建L3卡时加载模板）
- `项目总控.md`（确认已有L3卡编号，确定新L3卡编号）

## 做什么

**红线❌3：L3卡不事前设计，只从实际写作中事后提炼。** L3卡在L2单元完成后才创建/更新，不在plot设计阶段创建。

L3卡追踪的是**剧情线**（不是叙事单元）。一张L3卡覆盖一条剧情线贯穿多个L2单元的全过程。第一个L2单元完成后，arc提炼L3卡初始版本；后续每个L2单元完成后，arc更新L3卡。

### 1. 识别本单元推进的剧情线

从更新后的L2卡嵌套子线提取本单元推进了哪些剧情线：

```
提取流程：
1. 读取L2卡嵌套子线段，逐条子线分析
2. 每条子线对应一条剧情线（可能多条子线属于同一剧情线的不同阶段）
3. 检查是否有设计中未预见但正文实际涌现的新剧情线
4. 输出本单元推进的剧情线列表
```

剧情线识别示例：
```
L2-001嵌套子线：
  子线A：系统成长（Lv1→Lv2+技能解锁）→ 剧情线：系统觉醒与职业成长
  子线B：铁钳帮对抗（反杀+据点清理）→ 剧情线：黑帮对抗与庇护者
  子线C：灵能觉醒（耳鸣+感知）→ 剧情线：灵能觉醒与超自然接触（新涌现）

本单元推进的剧情线：
  ① 系统觉醒与职业成长（已有L3卡 / 新建L3卡）
  ② 黑帮对抗与庇护者（已有L3卡 / 新建L3卡）
  ③ 灵能觉醒与超自然接触（新涌现，需新建L3卡）
```

### 2. 对已有L3卡：追加阶段更新

对每条已有L3卡，追加新阶段并更新三个维度：

#### 2a. 阶段推进表追加

在L3卡阶段推进表追加新阶段（S+1）：

| 阶段编号 | 阶段名称 | 章节范围 | 承载L2单元 | 功能 | 核心成就 |
|:--------:|---------|:--------:|:----------:|------|---------|
| S{已有} | {已有阶段名} | chXXX-chXXX | L2-00{N-1} | {已有} | {已有} |
| **S{+1}** | **{新阶段名}** | **chXXX-chXXX** | **L2-00{N}** | **{本单元叙事功能}** | **{本单元核心成就}** |

阶段名称提炼原则：用一个短语概括本阶段剧情线的核心进展（如"系统觉醒与生存""声望积累与初始成长"）。

#### 2b. 力量体系快照更新

更新L3卡物理坐标段中的力量体系快照表，追加本单元结束时的实际数值：

| 维度 | 起点（ch001） | 上阶段终点 | **本阶段终点** | 变化幅度 |
|------|-------------|-----------|-------------|---------|
| 等级/职业 | LV0/无职业 | LV1 | **LV2** | +1级 |
| 核心技能 | 战术匕首专精 | +吟游表演 | **+街头感知** | 新技能 |
| ... | ... | ... | **...** | ... |

#### 2c. 叙事债务表更新

在L3卡叙事债务与回收段更新：

| 操作 | 说明 |
|:-----|:-----|
| 标记已回收债务 | 本单元中回收的伏笔，更新回收位置+回收方式 |
| 追加新债务 | 本单元新埋设的伏笔，填写债务编号+内容+埋设位置+预期回收点 |
| 更新超期债务 | 检查是否有债务超期（>20章未回收），标注超期预警 |

债务编号延续已有L3卡的编号（如L3-001已有D1-D5，本单元新埋设从D6开始）。

### 3. 对新剧情线：创建新L3卡

对新涌现的剧情线（已有L3卡中没有覆盖的），创建新L3卡。

加载 `templates/L3卡-模板.md`，填写以下内容：

| L3卡段 | 首次创建填写内容 |
|:-------|:---------------|
| 元数据 | 剧情线编号(L3-NNN)+名称+章节范围+线型+承载L2单元+叙事功能 |
| 物理坐标 | 时间轴（本单元）+地点轨迹+力量体系快照（起点→本阶段终点） |
| 一句话剧情线 | 用一段话概括这条剧情线从起点到当前的核心发展 |
| 核心驱动力 | 这条剧情线的驱动力分层（生存压力/系统压力/身份觉醒/社会压迫等） |
| 阶段推进表 | 填入初始阶段S1（本单元对应阶段） |
| 起点终点快照 | 起点快照（本单元开始时的状态）+ 当前终点快照（本单元结束时的状态） |
| 叙事债务与回收 | 本单元埋设的新债务（D1, D2...） |
| 与其他L3线咬合 | 本剧情线与其他已有L3线的关系 |
| 可迁移部分 | 从本单元提炼的可迁移设计经验 |
| 不可迁移部分 | 本单元专属的不可复制要素 |

### 4. 更新L3卡间咬合关系

检查本单元推进的各剧情线之间的咬合关系，更新每张L3卡的"与其他L3线咬合"段：

| 其他L3线 | 咬合方式 | 关键咬合点 |
|---------|---------|-----------|
| L3-00X | 本单元中两条剧情线的交汇方式 | 具体章节中的咬合事件 |

咬合关系更新示例：
```
L3-001（系统成长线）与L3-002（黑帮对抗线）咬合：
  咬合方式：系统成长提供能力支撑（升级后战力提升），黑帮对抗提供经验来源
  关键咬合点：ch003铁钳帮反杀→系统升级（击杀回报），两条线在此交汇
```

### 5. 产出L3卡

将更新后/新创建的L3卡写入 `卷纲/L3-NNN.md`。

```
输出文件：
  - 已有L3卡：追加阶段更新（覆写卷纲/L3-NNN.md）
  - 新L3卡：创建新文件（卷纲/L3-NNN.md，NNN为递增编号）
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| L3卡不事前设计，只从实际写作提炼（红线❌3） | 发现在plot阶段创建L3卡=删除并在此步骤重新提炼 |
| 已有L3卡已追加新阶段（S+1） | 未追加=补追加 |
| 新剧情线已创建L3卡 | 遗漏=补创建 |
| 力量体系快照已更新 | 未更新=补更新 |
| 叙事债务表已更新（新债务+回收标记） | 未更新=补更新 |
| L3卡间咬合关系已更新 | 未更新=补更新 |
