# Pop Writer V3 Create

把已经设计好的章级导演包/章节运行日志渲染成正文。create 的核心不是“再编剧情”，而是把 plot 已推演出的事件、压力、爽点、钩子写成可读、好看、衔接顺的网文正文，并把案例消化中的爽点证据真正落到场面里。文风质感服务爽点，不允许质感压过爽点。

章级导演包优先于运行日志。运行日志回答“发生了什么”，导演包回答“这一章怎么施工”：POV、地点、人物、事件链、关键对白方向、感官锚点、爽点外显、钩子、禁改项。缺导演包时只能降级为草案，除非用户明确要求直接试写。

## 执行模式

- `formal`：本 `SKILL.md`、`steps/step-1-create.md`、`templates/创作-模板.md` 已读取，且八项输入齐全。只在此模式下生成正式正文。
- `draft`：本 skill 已读取，但输入包有缺口。只能生成降级草案或缺口报告。
- `trial`：本 skill 未由 expert 正式调度，或用户明确要求快速试写。只能试写，不得声称完整按管线生成。

输出必须在创作记录写入 `execution` 凭证，如实列出已加载文件和缺失输入。

## 输入

- 当前章级导演包：本章功能、POV、地点、人物、事件链、关键对白方向、感官锚点、爽点外显、章末钩子、禁改项。
- 当前章节运行日志：本章发生什么、按什么顺序发生、每段压力如何升级。
- 案例消化摘要：本章学习了哪些案例的结构、爽点证据和内容肌理。
- 爽文审计结果：本章必须兑现的主角主动、可数收益、信息差、敌人误判、章末追读。
- 状态快照：开章时人物、地点、资源、伤势、关系、敌我态势。
- 设定账本：本章涉及的规则、组织、地理、能力、物价、限制。
- 文风DNA：句式、节奏、视角、爽感表达、信息密度。
- 上章末尾：用于无缝衔接。

## 执行

读 `steps/step-1-create.md`，按 `templates/创作-模板.md` 输出。

## 网文优先级

当文风DNA与网文爽感冲突时，按以下顺序裁决：

1. 主角主动操作清楚。
2. 压力和目标清楚。
3. 收益/损失外显清楚。
4. 敌人误判或旁人反应清楚。
5. 章末追读清楚。
6. 最后才是氛围、隐喻、克制、文学质感。

## 红线

- 不改章节运行日志的核心事件因果。
- 不新增会破坏设定账本的能力、物品、组织或世界规则。
- 不把爽点写成解释性总结，必须落在行动、反应、代价和局势变化上。
- 不允许只有氛围和悬念而没有爽文动作闭环。
- 不允许把“让读者自己品”当作关键爽点写法。关键爽点必须外显：面板、数字、敌人反应、局势翻转、资源到账至少出现一种。
- 不允许连续 800 字以上只有观察、解释、回忆、氛围，没有主角操作或局势反馈。
- 没有章级导演包时，不得声称正文已按完整管线生成，只能标注“降级草案”。
- 没有 `execution.mode: formal` 时，不得把产物称为正式 create 正文。
- 不默认调用 revise；修订由用户或 expert-writer 明确触发。
