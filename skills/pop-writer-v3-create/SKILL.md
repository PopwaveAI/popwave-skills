# pop-writer-v3-create · 事件轨迹正文化

create 的职责是把 plot 已经运行出的“章节事件仿真日志”写成正文。

核心原则：**create 写正文，不重跑 plot。**

## 输入优先级

| 优先级 | 输入 | 用途 |
|:------|:-----|:-----|
| P0 | 当前章运行日志段落 | 本章发生什么、因果、状态变化 |
| P0 | 单元卡当前章导演指令 | 情绪目标、章末推力、子线推进 |
| P0 | 设定账本 + 引用硬设定 | 保证世界观和软设定不漂移 |
| P0 | 上章末尾 | 语气和悬念衔接 |
| P1 | 活记忆/状态快照 | 当前资源、关系、未了事项 |
| P1 | 文风 DNA 原文 | 句式、节奏、叙事距离、段落气味 |

## 允许与禁止

| 允许 | 禁止 |
|:-----|:-----|
| 扩写场景细节、动作、对白、心理、感官 | 改变运行日志中的事件顺序和结算 |
| 补足过渡和读者情绪体验 | 新增会改变单元终点的大事件 |
| 根据设定账本使用已冻结设定 | 临时发明未入账重大设定 |
| 让爽点在正文中显性化 | 把爽点写成解释性旁白 |
| 微调章节内局部动作以更顺 | 推翻 plot 的子线咬合 |

## 写作方法

1. 先读当前章运行日志，复述本章“事件链”和“结算”。
2. 读取单元卡当前章导演指令，确认情绪目标和章末推力。
3. 读取设定账本，确认本章可用设定和禁止漂移点。
4. 读取上章末尾，确定开篇承接。
5. 读取文风 DNA 原文，进入文风语感。
6. 写正文：用场景流承载事件，不逐条翻译日志。
7. 章末停在 plot 指定的推力点，不写过头。

## 正文质量标准

- 第一屏有冲突、悬念、反差或行动，不用长铺垫开场。
- 主角每章至少做一个改变局面的主动行动。
- 行动有即时反馈：数值、资源、名声、关系、情报、敌人状态至少一项变化。
- 设定通过事件露出，不写百科段。
- 对话服务冲突、信息释放或关系变化，不注水。
- 章末必须把下一章推出来。

## 红线

| 编号 | 红线 |
|:-----|:-----|
| R1 | 当前章运行日志缺失时终止，退回 plot，不自行编大纲。 |
| R2 | 不新增未入设定账本的重大世界观、势力、能力规则。 |
| R3 | 不改变本章结算项：人物、资源、关系、威胁、情报状态。 |
| R4 | 不把运行日志逐条写成说明文，必须写成场景流正文。 |
| R5 | 章末不能越过运行日志指定的推力点。 |

| 步骤 | 文件 | 说明 |
|:-----|:-----|:-----|
| step-1 | `steps/step-1-create.md` | 读取运行日志和设定账本，正文化当前章 |

## 输出

- `正文/chXXX.md`
- 创作决策记录：记录是否有局部展开、是否新增轻量细节、是否触碰设定账本。
- context receipt：确认读取了运行日志、单元卡、设定账本、上章末尾、文风 DNA。
