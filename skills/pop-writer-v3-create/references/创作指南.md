# 创作指南

> 所属: pop-writer-v3-create v1.5.0 · references/
> 消费: step-1 创作执行时参考
> 定位: 创作端技法知识——网文风格基准+世界观卖点传递+场景流渲染+压力源逼近+章末钩子+事件密度+主角行为一致性

## 零、网文风格基准（v1.4新增，最高优先级）

> 这条在所有技法之上。先满足网文风格基准，再谈场景流/钩子/密度。

### 核心原则：你在写网文，不是写文学

读者是手机滑动阅读的普通人。他们要的是"发生了什么+然后呢"，不是"这段氛围描写多精致"。

| 维度 | 网文要求 | 禁止的传统文学化 |
|:-----|:---------|:---------------|
| **事件推进** | 每段都在推事件，氛围渲染≤3段必须有事件转折 | 连续5段以上纯氛围/情绪渲染不推事件 |
| **信息清晰度** | 读者读完一段就知道发生了什么 | 含蓄暗示需要反复品味才理解 |
| **世界观传递** | 通过角色对话/回忆/事件直接传递，读者一眼能get | 通过环境隐喻暗示，读者要自己猜 |
| **章末收束** | 情绪到位就收，钩子干净利落 | 情绪高峰后再铺3段总结性描述 |
| **语言难度** | 小学生能直接看懂 | 长句嵌套从句、生僻词汇、文学化修辞 |

### 第一性原理：爽文 vs 严肃文学（v1.5新增）

> 红线和自检告诉你"不要做什么"。第一性原理告诉你"为什么"——理解了为什么，你才知道每一段该怎么写。

#### 读者模型

你写的每一段，读者是一个**12岁小学生**，用手机滑动阅读。
- 他每30秒滑一次屏幕（约15-20行）
- 他每滑一次要看到新东西——新信息/新冲突/新转折/新爽点
- 如果连续3次滑动（45-60行）都是氛围/情绪描写，他会退出
- 他看不懂隐喻和象征，需要直接给

#### 节奏物理量

把"网文感"从抽象感觉变成可测量的指标：

| 物理量 | 量化标准 | 违反=节奏过慢 |
|:-------|:---------|:-------------|
| 事件间隔 | 每≤20行必须推进一个事件转折 | 连续20行无事件转折 |
| 感官描写 | 连续≤3行 | 连续4行以上纯感官描写 |
| 情绪描写 | 连续≤2行 | 连续3行以上纯情绪渲染 |
| 事件密度 | 每章8-12个事件转折 | 整章<8个转折 |
| 爽感释放 | 每章≥1个爽感爆发点 | 整章无爽感释放 |

#### 爽感公式

"爽"不是抽象概念——是有明确结构的体验循环。每章必须触发≥1个公式：

| 公式 | 结构 | 释放点 |
|:-----|:-----|:-------|
| 压力→突破 | 困境→解决→状态提升 | 突围成功的那一刻 |
| 弱→强 | 当前弱小→获得能力/升级 | HP恢复/技能解锁的瞬间 |
| 未知→揭示 | 谜团→揭晓→认知刷新 | 真相揭开的冲击 |
| 被欺→反击 | 被压迫→主动反击→成功 | 反击得手的那一击 |

**爽感释放必须有"爆发"感，不是安静的过渡：**

| WRONG（安静的伪爽感） | CORRECT（有冲击的真爽感） |
|:---------------------|:------------------------|
| "全身像被温水从头浇透" | 灵能觉醒→HP从5跳到60→身体瞬间恢复→他能看见了 |
| "他感觉变强了一点" | 面板弹出：HP 5→15→30→60，虚弱感被一只手从身体里抽走 |
| "他默默记住了这一切" | 他记住了车牌NYC-4827——下次见面，他不会是猎物 |

#### 默认快节奏

不是"不要慢"——是**默认快**。每段默认推进事件，只有以下理由允许慢下来：

| 允许慢的理由 | 上限 | 超出=废话 |
|:------------|:-----|:---------|
| 爽感前的蓄力（制造期待） | ≤2段 | 第3段开始删 |
| 关键信息传递（世界观卖点） | ≤3段 | 第4段开始删 |
| 情绪锚定（兄妹情/关键转折） | ≤2段 | 第3段开始删 |

#### 主角主动性

主角必须主动驱动事件，不是被动反应：

| ✅ 主动模式 | ❌ 被动模式 |
|:-----------|:-----------|
| 看到→评估→决定→行动 | 看到→恶心→记住→走 |
| 遇到→判断→反击/逃脱 | 遇到→害怕→等待→被救 |
| 信息→分析→制定计划 | 信息→震惊→消化→接受 |

每章主角必须至少：
- 1个**主动决策**（基于信息做出选择）
- 1个**主动行动**（执行决策，改变处境）

### 世界观卖点传递（核心机制）

**如果本书的核心卖点是魔改世界观（如"邪神渗透美国大选""超凡复苏""万灵浩劫"），必须在前三章让读者直接感知到这个卖点。**

传递方式（从导演意图.worldview_delivery获取）：
- **角色回忆闪回**：主角穿越后的记忆融合，自然带出世界观差异
- **系统面板注释**：面板上的特殊字段（如"灵魂：？？？？（不可读）"）暗示世界规则
- **角色对话**：配角一句话传递关键信息（"城东那片的疫病，年年底层区死人最多"）
- **事件冲击**：直接展示世界观特征的事件（邪神信徒仪式/超凡生物出现）

**禁止的传递方式：**
- 等后面再慢慢揭示（读者前三章没get到卖点就弃书了）
- 全程环境隐喻暗示（读者"全程懵逼"）
- 只传递表层设定不传递核心卖点（只写"领主/城堡"不写"邪神渗透"）

### 自检：每章写完问自己

1. 读者读完这章，知道发生了什么吗？（不知道=失败）
2. 读者读完这章，对这个世界有什么了解了？（不了解=世界观传递不足）
3. 读者读完这章，想看下一章吗？（不想=钩子/爽感不足）
4. 这章有没有连续3段以上不推事件的氛围渲染？（有=精简）

## 一、职责边界

创作子skill专注故事结构层：
- **做：** 网文风格基准+世界观卖点传递+场景流渲染、压力源逼近、章末钩子、子线推进、金手指使用、角色决策、主角行为一致性检查
- **不做：** 文风DNA对齐（→修订层）、8020展示/内省比例（→修订层）、AI观感词清理（→修订层）

创作端初稿不需要文风完美，只需故事结构扎实——**网文风格基准已满足**+场景有机流动、压力在逼近、钩子有力度、主角行为一致。文风打磨是修订层的事。

> v3.5：create在导演意图框架内自由涌现。导演意图说"这章要达到什么效果"，create决定"怎么写"。导演意图的世界观传递约束（worldview_delivery）是硬约束——必须通过正文传递。

## 二、chapter+prose合并

### 合并理由

v2管线中chapter和prose分离——chapter产出章纲，prose消费章纲渲染正文。每一层传递都是信息损失：设定层→大纲摘要→章纲摘要的摘要→prose拿到第三手信息渲染。

v3合并chapter+prose：agent拿着导演意图+状态快照+设定文件直接涌现，做实时决策。设计决策和渲染执行在人类作者脑中是同时发生的，分离导致prose拿到的是"设计摘要"而非"创作冲动"。

### 合并后的agent自由度

agent在导演意图框架内自主决定：
- 本章推进哪条子线（从导演意图.sublines_advance中选取）
- 哪个压力源逼近（从状态快照.pressures中选取must_advance=true的压力源）
- 金手指怎么使用（受限于设定文件中金手指的能力+限制）
- 角色做什么决策（受限于设定文件中主角引擎的驱动力三层+缺陷+决策风格+**行为准则**）
- 章末钩子怎么设置（落实导演意图.chapter_hook）
- 场景如何组织（无预设场景流设计包，agent自主构建场景）

> 导演意图给方向不给路径——不预设具体场景和台词，create在框架内自由涌现。

## 三、场景流渲染

### 场景内连续流动

按场景流渲染，事件在场景内有机融合，**不逐事件独立成段**。

**WRONG（逐事件渲染）：**
```
索伦拔出匕首。他绕到恶棍身后。匕首刺入后心。他捂住对方的嘴。
```

**CORRECT（场景流渲染）：**
```
索伦绕到恶棍身后，匕首无声刺入后心——一只手捂住嘴，另一只手摸走钱袋，全程没发出一点声响。
```

### 场景渲染流程

每个场景走5步：

| 步骤 | 内容 |
|:-----|:-----|
| A. 理解意图 | 场景要推进什么？情绪走向？推进哪条子线？（对照导演意图） |
| B. 选POV | 主角POV（单主角假设），不进入非锚点角色内心 |
| C. 写场景段落 | 场景内连续流动 |
| D. 控制节奏 | 按事件密度8-12个/章分配场景内转折（对照导演意图.pacing） |
| E. 场景间过渡 | 1-2句自然过渡 |

### 开幕段落

- 空间/时间锚定 → 情绪传递（1-2句）
- 如有感官锚点，第一句以该感官信号开头
- 例：「雨声先到。然后是铁锈味。」

### 场景间过渡

- 1-2句自然过渡，不铺陈
- 过渡方式：时空跳转（"三天后"）/ 感官切换（"声音消失了"）/ 情绪转折
- 不写"与此同时"式的平行剪辑——保持单线POV

### 闭幕段落

章末 = 钩子位置：
- 悬念钩 → 不闭合，留疑问
- 情绪钩 → 安静收尾，情绪余韵
- 行动钩 → 蓄势待发，停在行动起点

**章末收束纪律（v1.4新增）：**
- 情绪到位就收，**禁止在情绪高峰后再铺总结性描述**
- 钩子放在最后1-2段，干净利落
- 禁止"两个人在同一个破帐篷里...今夜，这就够了"式的拖拉收尾——情绪已经在前面到位了，再来一段总结是在消耗读者耐心
- 自检：把最后3段删掉，如果故事不缺什么，说明那3段是废话

## 四、压力源逼近

本章正文必须至少推进一个压力源的显性表现。

从状态快照的压力仪表盘（pressures）中选取本章活跃的压力源（must_advance=true优先），在正文中体现其逼近：
- 倒计时型压力：倒计时数值变化/主角感知到时间紧迫
- 威胁型压力：威胁逼近的具体表现（巡逻搜人/催债上门/病发征兆）
- 困境型压力：道德困境的具体呈现

多个压力源交叉时产生道德困境——这是压力矩阵设计的核心目的（压力矩阵全量在写作参考/设定/中，由settings_ref注入）。

## 五、章末钩子设计

导演意图.chapter_hook指定了钩子类型和内容，创作子skill负责落实到正文最后1-3段。

### 钩子类型

| 钩子类型 | 写法 | 示例 |
|:---------|:-----|:-----|
| 悬念钩 | 不闭合，留疑问 | "门被推开，站在那里的人让他血液凝固。" |
| 情绪钩 | 安静收尾，情绪余韵 | "他看着手中的照片，很久很久没有说话。" |
| 行动钩 | 蓄势待发，停在行动起点 | "他握紧匕首，推开了那扇门。" |

### 钩子强度标准

- 即时危机型钩子（下章开头需解决的危机）优于远期悬念型（"7天后决战"）
- 钩子必须让读者有"必须看下一章"的冲动
- 禁止无钩子收尾（"他躺在床上睡着了"式结尾）

## 六、事件密度控制

每章8-12个事件转折。目标小说是11-13个。不允许整章只写一件事的铺垫。

创作时注意：
- 每个场景内至少2-3个转折点
- 场景之间的过渡本身也可以是转折
- 禁止整章只写一件事的铺垫
- 禁止整章走路+思考+回忆（1个事件）

## 七、主角行为一致性检查

创作子skill在涌现写作时，必须加载设定文件中主角引擎的"行为准则"字段（由settings_ref注入），逐场景检查主角行为一致性。

### 检查流程

| 步骤 | 检查内容 | 不一致时处理 |
|:-----|:---------|:------------|
| 1. 加载行为准则 | 背景映射/战斗风格/道德底线/行为禁忌（设定文件主角引擎段） | 准则缺失=退回seed补全 |
| 2. 逐场景行为预测 | 基于行为准则预测"主角在这个场景会怎么行动" | — |
| 3. 行为一致性验证 | 正文中的主角行为是否符合预测 | 不符合=修正行为描写 |
| 4. 行为禁忌检查 | 正文是否触碰行为禁忌 | 触碰=修正（除非有显性破例剧情） |

### 出戏检查红线

| 出戏类型 | 表现 | 处理 |
|:---------|:-----|:-----|
| 背景不符 | {主角背景}不符的行为表现（如职业训练缺失、技能水平异常） | 退回重写该场景 |
| 道德底线越界 | 无显性破例剧情却触碰底线 | 退回重写 |
| 行为禁忌违反 | 无合理原因却做禁忌之事 | 退回重写 |

### 行为准则四维说明

| 子项 | 内容 | 示例 |
|:-----|:-----|:-----|
| 背景映射 | 主角背景→行为规则映射 | {主角背景}→对应行为规则（如职业训练决定的反应模式） |
| 战斗风格 | 一句话定义战斗方式 | {主角战斗风格}（如近身压制/远程牵制/智取型） |
| 道德底线 | 不可逾越的红线 | 不伤无辜/不背叛盟友 |
| 行为禁忌 | 绝不会做的事 | 不退缩/不暴露身份/不留后患 |

### 显性破例剧情

行为准则不是绝对铁律——当故事需要主角突破行为准则时（如"从不留活口"的主角遇到必须留活口的情境），必须有**显性破例剧情**：
- 正文中明确展示主角内心的挣扎和决策过程
- 破例必须有合理的故事原因（非随意突破）
- 破例后需在后续章节有呼应（行为准则演化→L2卡更新）

无显性破例剧情却触碰行为禁忌 = 出戏，必须退回重写。
