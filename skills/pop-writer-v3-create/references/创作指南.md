# 创作指南

> 所属: pop-writer-v3-create v1.0.0 · references/
> 消费: step-1 创作执行时参考
> 定位: 创作端只管故事涌现——场景流渲染+压力源逼近+章末钩子+主角行为一致性
> v3.2改动: 从emerge迁移到create子skill；新增"八、主角行为一致性检查"章节

## 一、创作子skill的职责边界

创作子skill专注故事结构层：
- **做：** 场景流渲染、压力源逼近、章末钩子、线索推进、金手指使用、角色决策、主角行为一致性检查
- **不做：** 文风DNA对齐（→修订层）、8020展示/内省比例（→修订层）、AI观感词清理（→修订层）

创作端写出来的初稿不需要文风完美，只需要故事结构扎实——场景有机流动、压力在逼近、钩子有力度、主角行为一致。文风打磨是修订层的事。

## 二、chapter+prose合并

### 合并理由

v2管线中chapter和prose分离——chapter产出章纲（场景流设计包），prose消费章纲渲染正文。每一层传递都是信息损失：设定层→大纲摘要→章纲摘要的摘要→prose拿到第三手信息渲染。

v3合并chapter+prose：agent拿着种子+活记忆+chapter_plan直接涌现，做实时决策。设计决策和渲染执行在人类作者脑中是同时发生的，分离导致prose拿到的是"设计摘要"而非"创作冲动"。

### 合并后的agent自由度

agent在chapter_plan框架内自主决定：
- 本章推进哪条活跃线索（从chapter_plan.plotlines_advanced中选取）
- 压力矩阵中哪个压力源逼近（从chapter_plan.crisis中选取）
- 金手指怎么使用（受限于种子的能力+限制）
- 角色做什么决策（受限于主角引擎的驱动力三层+缺陷+决策风格+**行为准则**）
- 章末钩子怎么设置（从chapter_plan.hook中落实）
- 场景如何组织（无预设场景流设计包，agent自主构建场景）

## 三、场景流渲染

### 场景内连续流动

按场景流渲染，事件在场景内有机融合，**不逐事件独立成段**。

**WRONG（逐事件渲染）：**
```
索伦拔出匕首。他绕到恶棍身后。匕首刺入后心。他捂住对方的嘴。
```

**CORRECT（场景流渲染）：**
```
索伦绕到恶棍身后，匕首无声刺入后心——一只手捂住嘴，另一只手摸走钱袋，全程没发出一点声响。
```

### 场景渲染流程

每个场景走5步：

| 步骤 | 内容 |
|:-----|:-----|
| A. 理解意图 | 场景要推进什么？情绪走向？推进哪条线？ |
| B. 选POV | 主角POV（单主角假设），不进入非锚点角色内心 |
| C. 写场景段落 | 场景内连续流动 |
| D. 控制节奏 | 按事件密度8-12个/章分配场景内转折 |
| E. 场景间过渡 | 1-2句自然过渡 |

### 开幕段落

- 空间/时间锚定 → 情绪传递（1-2句）
- 如有感官锚点，第一句以该感官信号开头
- 例：「雨声先到。然后是铁锈味。」

### 场景间过渡

- 1-2句自然过渡，不铺陈
- 过渡方式：时空跳转（"三天后"）/ 感官切换（"声音消失了"）/ 情绪转折
- 不写"与此同时"式的平行剪辑——保持单线POV

### 闭幕段落

章末 = 钩子位置：
- 悬念钩 → 不闭合，留疑问
- 情绪钩 → 安静收尾，情绪余韵
- 行动钩 → 蓄势待发，停在行动起点

## 四、压力源逼近（种子消费）

本章正文必须至少推进一个压力源的显性表现。

从种子压力矩阵中选取本章活跃的压力源，在正文中体现其逼近：
- 倒计时型压力：倒计时数值变化/主角感知到时间紧迫
- 威胁型压力：威胁逼近的具体表现（巡逻搜人/催债上门/病发征兆）
- 困境型压力：道德困境的具体呈现

多个压力源交叉时产生道德困境——这是压力矩阵设计的核心目的。

## 五、章末钩子设计

chapter_plan.hook指定了钩子类型和内容，创作子skill负责落实到正文最后1-3段。

### 钩子类型

| 钩子类型 | 写法 | 示例 |
|:---------|:-----|:-----|
| 悬念钩 | 不闭合，留疑问 | "门被推开，站在那里的人让他血液凝固。" |
| 情绪钩 | 安静收尾，情绪余韵 | "他看着手中的照片，很久很久没有说话。" |
| 行动钩 | 蓄势待发，停在行动起点 | "他握紧匕首，推开了那扇门。" |

### 钩子强度标准

- 即时危机型钩子（下章开头需解决的危机）优于远期悬念型（"7天后决战"）
- 钩子必须让读者有"必须看下一章"的冲动
- 禁止无钩子收尾（"他躺在床上睡着了"式结尾）

## 六、事件密度控制

每章8-12个事件转折。目标小说是11-13个。不允许整章只写一件事的铺垫。

创作时注意：
- 每个场景内至少2-3个转折点
- 场景之间的过渡本身也可以是转折
- 禁止整章只写一件事的铺垫
- 禁止整章走路+思考+回忆（1个事件）

## 八、主角行为一致性检查（v3.2新增）

创作子skill在涌现写作时，必须加载种子主角引擎的"行为准则"字段，逐场景检查主角行为一致性。

### 检查流程

| 步骤 | 检查内容 | 不一致时处理 |
|:-----|:---------|:------------|
| 1. 加载行为准则 | 背景映射/战斗风格/道德底线/行为禁忌 | 准则缺失=退回seed补全 |
| 2. 逐场景行为预测 | 基于行为准则预测"主角在这个场景会怎么行动" | — |
| 3. 行为一致性验证 | 正文中的主角行为是否符合预测 | 不符合=修正行为描写 |
| 4. 行为禁忌检查 | 正文是否触碰行为禁忌 | 触碰=修正（除非有显性破例剧情） |

### 出戏检查红线

| 出戏类型 | 表现 | 处理 |
|:---------|:-----|:-----|
| 背景不符 | CIA精锐不灭口、战术动作简单 | 退回重写该场景 |
| 道德底线越界 | 无显性破例剧情却触碰底线 | 退回重写 |
| 行为禁忌违反 | 无合理原因却做禁忌之事 | 退回重写 |

### 行为准则四维说明

| 子项 | 内容 | 示例 |
|:-----|:-----|:-----|
| 背景映射 | 主角背景→行为规则映射 | CIA黑色行动精锐→不留活口/先手压制/战术优先 |
| 战斗风格 | 一句话定义战斗方式 | CQB近身压制+无声击杀+利用环境 |
| 道德底线 | 不可逾越的红线 | 不杀平民/不背叛盟友 |
| 行为禁忌 | 绝不会做的事 | 不谈判/不留活口/不暴露身份 |

### 显性破例剧情

行为准则不是绝对铁律——当故事需要主角突破行为准则时（如"从不留活口"的主角遇到必须留活口的情境），必须有**显性破例剧情**：
- 正文中明确展示主角内心的挣扎和决策过程
- 破例必须有合理的故事原因（非随意突破）
- 破例后需在后续章节有呼应（行为准则演化→种子生长）

无显性破例剧情却触碰行为禁忌 = 出戏，必须退回重写。
