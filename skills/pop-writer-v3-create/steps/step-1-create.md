# Step 1：涌现写作创作

> 管线: pop-writer-v3-create v1.3.0
> 类型: **子skill**（由expert-writer Step 3调度，context隔离）
> 消费: 导演意图 + 状态快照 + 上章末尾 + 设定文件 + info_acquired + L2卡(可选) + 创作模板
> 产出: 正文初稿 + 创作决策记录 + context receipt（预留）
> 红线: ❌1 必须在导演意图框架内创作（叙事功能/事件链/情绪曲线/章末钩子必须覆盖）；❌2 不加载文风DNA；❌3 仅可更新L2卡本章聚焦相关内容；❌4 正文≥2500字

## 子skill调度说明

### 输入context（精简，不传会话历史）

> v3.5 取消种子文档。create 的输入从"种子文档全文"改为导演意图+状态快照+设定文件。种子文档原功能被 L2卡+写作参考吸收。

| 输入项 | 来源 | 内容 |
|:-------|:-----|:-----|
| 导演意图（director_intent） | L2卡结构分析表本章行（主会话Step0提取） | 叙事功能/事件链/情感曲线/节奏/子线推进/章末钩子/三问/设定引用 |
| 状态快照（state_snapshot） | 活记忆+L2卡物理坐标投影（主会话Step1产出，≤400字） | 主角当前状态（能力/心理/处境/资源/目标）+压力仪表盘+未了事项（伏笔/承诺/揭示） |
| 上章末尾（prev_chapter_tail） | `正文/ch{上一章}.md` 末尾1200字 | 语感衔接+悬念承接 |
| 设定文件（settings_ref） | L2卡设定引用指针指向的文件（主会话Step2强制读取后注入） | 主角引擎（含行为准则）/金手指/压力矩阵等设定内容 |
| info_acquired | 主会话Step2按需获取（写作参考/知识沉淀/） | 增量信息（制度细节/场景技法/伏笔回收线索） |
| L2卡（l2_card） | `卷纲/L2-NNN.md`（可选注入） | 整体参考（结构分析表/嵌套子线/物理坐标） |
| 创作模板 | `templates/创作-模板.md` | 正文涌现结构+创作决策记录格式+context receipt预留格式 |

**不传入的内容：** 主会话历史、Step0-2的执行过程、文风DNA（红线❌2）、修订指南、质检模板。创作子skill专注故事涌现，不被文风/AI观感词/8020分散注意力。

### 输出

| 输出项 | 格式 | 传递给 |
|:-------|:-----|:-------|
| 正文初稿 | markdown | expert-writer主会话→调度revise |
| 创作决策记录 | YAML | expert-writer主会话→Step4 receipt检查 |
| context receipt | YAML | expert-writer主会话→Step4 receipt检查（批次1预留格式，批次2完整实现） |
| L2卡本章聚焦微调记录（如有） | YAML（并入创作决策记录） | expert-writer主会话→Step5；同步回写L2卡 |

> **L2卡写入权限（红线❌3）：** create在涌现写作中仅可更新L2卡结构分析表本章行（导演意图来源），如事件brief与实际发生有偏差可微调。不得修改结构分析表其他行、嵌套子线、物理坐标、子线咬合。若写作中微调了本章行，必须在创作决策记录中记录调整内容+原因。

## 做什么

### Step 1: 上下文确认

按 `templates/创作-模板.md` §一 逐项确认输入context已加载：
- 导演意图已加载（含叙事功能/事件链/情感曲线/章末钩子/三问/设定引用）
- 状态快照已读取
- 设定文件已读取（settings_ref指针指向的全读，含主角引擎.行为准则）
- 上章末尾已读取（首章除外）
- **门禁：** 导演意图缺失 = 退回主会话Step0重做。行为准则缺失（设定文件中主角引擎段无行为准则字段）= 退回seed补全。

### Step 2: 导演意图框架确认

逐项确认导演意图的7个核心维度已理解，确定创作方向：
- **叙事功能（narrative_function）：** 本章在全书中完成什么叙事功能
- **事件链（event_chain）：** 本章核心事件的动作链方向
- **情感曲线（emotion_curve）：** 情绪走向
- **节奏（pacing）：** 快慢节奏
- **子线推进（sublines_advance）：** 本章推进哪些子线
- **章末钩子（chapter_hook）：** 章末留什么悬念
- **三问（three_questions）：** info（读者必须知道什么）/pressure（什么压力必须推进）/hook（留什么钩子）
- **门禁：** 导演意图7项中任一项缺失 = 退回主会话Step0重做（红线❌1）

> 导演意图说"这章要达到什么效果"，create决定"怎么写"——不预设场景台词，在导演意图框架内自由涌现。

### Step 3: 涌现写作

在导演意图框架内自主涌现正文。导演意图定义'达成什么'，create自主决定场景设计/情节编排/角色决策（涌现空间）。不再有预设场景填空。做实时决策（推进子线/压力源选择/金手指使用/角色决策/钩子设置/场景组织）。
- 参考 `references/创作指南.md` §二（chapter+prose合并原理）§三（场景流渲染技法）。
- 涌现时实时对照导演意图：叙事功能是否在推进？事件链核心事件是否覆盖？情感曲线是否对齐？
- **门禁：** 逐事件渲染（非场景流） = 退回重写。

### Step 4: 压力源逼近

本章正文必须至少推进一个压力源的显性表现。从状态快照的压力仪表盘（pressures）中选取本章活跃的压力源，在正文中体现其逼近。
- 参考 `references/创作指南.md` §四（压力源逼近技法）。
- **门禁：** 本章无压力源逼近 = 退回重写。

### Step 5: 章末钩子

将导演意图的 chapter_hook 落实到正文最后1-3段。
- 参考 `references/创作指南.md` §五（章末钩子设计）。
- **门禁：** 章末无钩子 = 退回重写（红线❌1 章末钩子必须覆盖）。

### Step 6: 事件密度检查

确认本章事件转折数达标。
- 参考 `references/创作指南.md` §六（事件密度控制）。
- **门禁：** 事件密度不足（整章只写一件事） = 退回重写。

### Step 7: 字数下限门禁（红线❌4）

统计正文纯文字字数（不含markdown标记）。
- 门禁：正文 < 2500字 = 退回重写（信息密度不足）
- 目标区间2500-3500字，事件密度8-12个/章
- 字数不足通常意味着场景展开不够/事件密度过低，需补充展开而非注水

### Step 8: 行为一致性检查

逐场景检查主角行为是否符合设定文件中主角引擎的"行为准则"字段（由settings_ref注入）。
- 参考 `references/创作指南.md` §七（主角行为一致性检查）。
- **门禁：** 行为不符合准则（无显性破例剧情） = 退回重写该场景。

### Step 9: 导演意图覆盖核对（红线❌1）

逐项核对正文是否覆盖导演意图的核心维度：
- [ ] 叙事功能（narrative_function）已体现
- [ ] 事件链核心事件已覆盖
- [ ] 情感曲线（emotion_curve）已对齐
- [ ] 子线推进（sublines_advance）已落实
- [ ] 章末钩子（chapter_hook）已落地
- **门禁：** 任一项未覆盖 = 退回重写（红线❌1）。

### Step 10: 创作决策记录

按 `templates/创作-模板.md` §三 填写创作决策记录YAML，记录关键决策及其依据（导演意图/设定文件/状态快照）。
- **L2卡本章行微调记录（如有）：** 若写作中微调了L2卡结构分析表本章行（如事件brief与实际偏差），在创作决策记录中记录：调整内容（调整前→调整后）+调整原因。同时将调整回写到L2卡本章行（红线❌3：仅限本章行，不得修改其他段）。

### Step 11: context receipt 产出（批次1预留）

按 `templates/创作-模板.md` §五 填写 context receipt YAML，确认实际接收了哪些注入项。
- 批次1：预留格式填写，确认 received 各项 status + excluded 中的 style_dna。
- 批次2：完整实现receipt产出与主会话一致性检查。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| **导演意图框架未覆盖（叙事功能/事件链/情绪曲线/章末钩子）（红线❌1）** | **退回重写** |
| **加载了文风DNA（红线❌2）** | **移除文风DNA注入** |
| **修改了L2卡结构分析表其他行/嵌套子线/物理坐标（红线❌3）** | **退回，仅保留本章行改动** |
| **正文<2500字（字数下限门禁）（红线❌4）** | **退回重写** |
| 逐事件渲染（非场景流） | 退回重写 |
| 本章无压力源逼近 | 退回重写 |
| 章末无钩子 | 退回重写 |
| 事件密度>15个/章（信息过载） | 退回精简 |
| 主角行为不符合行为准则（设定文件主角引擎段） | 退回重写该场景 |
| 导演意图缺失 | 退回主会话Step0重做 |
| 设定文件未读取（settings_ref指针未全读） | 退回主会话Step2补读 |
| 行为准则缺失（设定文件主角引擎段无行为准则字段） | 退回seed补全 |
| 导演意图/状态快照不存在 | 终止，提示先完成上游 |
| 上下文确认表未填 | 补填后再继续 |

> 创作子skill**不做**文风DNA对齐/8020比例/AI观感词清理检查（交给revise子skill）。
