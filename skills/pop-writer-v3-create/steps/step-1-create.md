# Step 1 - 正文创作

## 先核对

写前先判定执行模式，再确认八项材料：

| 模式 | 条件 | 输出限制 |
| --- | --- | --- |
| formal | 已读取 create SKILL、step、template；八项材料齐全 | 可输出正式正文 |
| draft | create 文件已读取，但八项材料有缺口 | 只能输出降级草案或缺口报告 |
| trial | 未经 create 完整加载，或用户只要快速试写 | 只能试写，不称正式正文 |

八项材料：

1. 上章末尾是否能接住本章开场。
2. 章级导演包是否给出本章功能、POV、事件链、关键对白方向、感官锚点、爽点外显、钩子和禁改项。
3. 状态快照是否明确主角处境。
4. 运行日志是否给出本章事件链。
5. 设定账本是否覆盖本章关键规则。
6. 案例消化摘要是否说明本章借鉴的爽点证据。
7. 爽文审计结果是否明确本章动作闭环。
8. 文风DNA是否足够指导语感。

缺失材料能从项目文件补齐就补齐；不能补齐时，在正文中保守处理，不擅自发明硬设定。缺章级导演包时，只能标记为“降级草案”，不得声称完整按管线生成。缺 create skill/step/template 加载凭证时，只能标记为“试写”。

## 文风DNA 切片读取

文风DNA 文件含 8+ 张场景卡（每张 ≥500 字原文摘录），全文 15KB+。**禁止全文 Get-Content + Substring 截断**——截断会丢失场景卡原文摘录（v4 模板核心价值），导致 agent 拿到"目录"而非"正文"。

### 切片协议

1. 从导演包提取 `scene` 字段值列表（如 `[narration_suspense, dialogue_daily, description_environment]`）
2. 按文风DNA.md 的 `###` 标题切分，匹配 scene 字段对应的场景卡
3. 追加"通用维度"整段 + "全书稳定特征"整段（这些对每章都适用）
4. 拼接为 `$DNA_SLICE`，写入创作记录 `execution.dna_slices`

### PowerShell 切片脚本（复制执行，替换两个参数）

```powershell
# ===== 参数（替换为你项目的实际值）=====
$dnaPath = "{填入文风DNA.md 完整路径}"
$scenes  = @("scene1", "scene2")  # 从导演包 scene 字段提取
# =====================================

$c = Get-Content $dnaPath -Raw -Encoding UTF8
$ms = [regex]::Matches($c, '(?m)^### ')
$dict = @{}
for ($i=0; $i -lt $ms.Count; $i++) {
  $s = $ms[$i].Index
  $e = if ($i -lt $ms.Count-1) {$ms[$i+1].Index} else {$c.Length}
  $b = $c.Substring($s, $e-$s)
  if ($b -match 'scene:\s*`?([^`\r\n]+)`?') { $dict[$Matches[1].Trim()] = $b }
}
$out = ""
foreach ($sv in $scenes) { if ($dict.ContainsKey($sv)) { $out += $dict[$sv] + "`n`n" } }
# 追加通用维度 + 全书稳定特征（H2 段）
$h2s = [regex]::Matches($c, '(?m)^## ')
for ($i=0; $i -lt $h2s.Count; $i++) {
  $s = $h2s[$i].Index
  $e = if ($i -lt $h2s.Count-1) {$h2s[$i+1].Index} else {$c.Length}
  $title = ($c.Substring($s, 200) -split "`n")[0]
  if ($title -match '通用维度|全书稳定特征') { $out += $c.Substring($s, $e-$s).TrimEnd() + "`n`n" }
}
Write-Output $out
```

> 脚本逻辑：按 `###` 切分 → 提取每块 `scene:` 值 → 按 $scenes 匹配 → 追加 H2 标题含"通用维度"/"全书稳定特征"的整段。输出约 3-5KB（vs 全文 15KB+），注意力聚焦在相关场景。

### 校验

- [ ] 切片输出 > 1000 字符（过小说明匹配失败）
- [ ] 切片包含目标场景卡的原文摘录（非空引用块）
- [ ] 切片包含通用维度段
- 未通过 → 检查 scene 字段值拼写 / 文风DNA文件路径

## 写法

- 开场承接上章最后一个动作、危险或疑问。
- 每一场都让主角做选择，选择带来代价或收益。
- 设定信息通过冲突、交易、观察、失败来显影，不用百科说明。
- 爽点必须有压迫前置、行动突破、旁人反应、局势收益。
- 关键爽点要外显，不要只藏在氛围里。优先使用：面板弹窗、数值变化、敌人反应、旁人态度变化、资源到账、身份变化、追杀等级变化。
- 每章至少完成一次动作闭环：压力逼近 -> 主角判断 -> 主动行动 -> 可数收益/明确损失 -> 新压力。
- 章末留下主动钩子：主角做出下一步选择，或威胁逼到眼前。

## 导演包消费

正文必须逐项消费导演包：

| 导演包字段 | 正文落地方式 |
| --- | --- |
| 本章叙事功能 | 决定开场和章末，不擅自改剧情目标 |
| POV | 不乱切视角 |
| 地点与空间动线 | 让动作有物理坐标 |
| 事件链 | 保持因果顺序 |
| 关键对白方向 | 写成可读对白，不只概述 |
| 感官锚点 | 用短句落地氛围，不能吞掉动作 |
| 爽点机制 | 写出压迫、操作、反馈 |
| 爽点外显方式 | 面板/数字/反应/到账/身份变化至少一种 |
| 禁改项 | 不改核心事件和硬设定 |

## 网文感硬规则

- 每 600-800 字至少出现一次“推进读者快感的反馈”：新信息、新收益、新威胁、敌人误判、主角操作成功/失败。
- 每章至少 2 次让读者明确感到“主角在操作”，不是只在感受世界。
- 每章至少 1 次外部反馈：敌人、路人、系统、环境或资源状态对主角行动作出反应。
- 面板流正文不要太克制。涉及升级、经验、技能、任务、威胁评估时，允许清楚弹出并给读者读到。
- 讽刺点不要全靠隐喻。关键讽刺可以用短句直给，让读者当场获得情绪释放。
- 氛围段必须短，写完立刻回到目标、行动、风险或收益。

## 严肃文学偏移自检

写完后检查：

| 症状 | 修正 |
|:-----|:-----|
| 连续大段感官/心理，没有操作 | 插入主角选择、动作、试探或资源盘点 |
| 设定很高级，但读者不知道爽在哪里 | 加面板反馈、收益到账、敌人反应 |
| 主角像旁观者 | 改成主角主动测试规则、利用规则、钻规则漏洞 |
| 讽刺太含蓄 | 加一个独句段或角色反应，让情绪落地 |
| 章末只是悬疑 | 改成“主角下一步要做什么”或“威胁已经逼到哪” |

## 输出

按 `templates/创作-模板.md` 输出正文和简短创作记录。创作记录只写真实偏离点、设定新增风险和下一章衔接点，并必须包含 `execution` 凭证。`execution.mode` 不是 `formal` 时，不得把产物称为正式正文。
