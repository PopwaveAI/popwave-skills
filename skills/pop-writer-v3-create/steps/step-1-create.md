# Step 1 - 正文创作

## 先核对

写前先判定执行模式，再确认输入材料：

| 模式 | 条件 | 输出限制 |
| --- | --- | --- |
| formal | 已读取 create SKILL、step、template；输入材料齐全；章卡边界和微beat施工表通过 | 可输出正式正文 |
| draft | create 文件已读取，但输入包、章卡边界或微beat施工表有缺口 | 只能输出降级草案或缺口报告 |
| trial | 未经 create 完整加载，或用户只要快速试写 | 只能试写，不称正式正文 |

输入材料：

1. 上章末尾是否能接住本章开场。
2. plot 章级施工卡是否给出章卡边界、POV、scene、宏观剧情骨架、微beat施工表、主爽点、主角主动动作、可数收益/明确损失、爽点外显、章末钩子和禁改项。
3. 状态快照是否明确主角处境。
4. 运行日志是否给出本章事件链。
5. 设定账本是否覆盖本章关键规则。
6. 本章角色锚点是否明确：只包含本章登场角色的反应模式、说话方式、关系状态和本章功能，不读取整份角色储备池。
7. 幕级外部燃料台/案例消化摘要是否说明本幕借鉴的爽点证据。
8. 爽文审计结果或施工卡字段是否明确本章动作闭环。
9. 文风DNA是否足够指导语感。
10. 读者表达偏好是否明确直白度、旁白/心理许可、环境描写长度和爽点外显强度。
11. 若存在 review 下一章修正规则，是否已读取并写入本章约束。

缺失材料能从项目文件补齐就补齐；不能补齐时，在正文中保守处理，不擅自发明硬设定。缺 plot 章级施工卡时，只能标记为“降级草案”，不得声称完整按管线生成。缺 create skill/step/template 加载凭证时，只能标记为“试写”。

## 章卡边界校验

写前从施工卡提取：

- 当前章编号和标题。
- 本章允许写到哪里结束。
- 下一章保留什么事件、坐标、战斗、揭示、结算或情绪兑现。
- 本章禁止提前吞并什么。

执行规则：

- 正文只能覆盖当前章施工卡，不得把下一章的核心事件顺手写掉。
- 如果正文自然滑到下一章内容，在当前章主动钩子处停下，记录“下一章保留项”。
- 如果施工卡没有章卡边界，`execution.mode` 只能是 `draft`。
- 如果写完发现正文已吞并下一章，必须拆分或标记为失败，不能在创作记录里写 `formal`。

## 微beat施工表校验

formal 正文必须有微beat施工表：

- 2500-3500字章节至少10-14个微beat；1800-2200字短章至少8-10个微beat。
- 每个微beat必须能写成具体场面动作，不能只是“推理/转场/战斗/解释设定”。
- 写作时按微beat顺序推进；可以合并相邻小动作，但不能跳过主角动作、阻力和反馈。
- 如果微beat不足，先回 plot 补施工卡，不要靠正文阶段加环境描写补字。

## 文风DNA 全文读取协议

文风DNA 文件含 8+ 张场景卡（每张 ≥500 字原文摘录），全文 15KB+。**Read 工具对大文件有随机截断（实测保留率 9%-80%），会丢失场景卡原文摘录（v4 模板核心价值），导致 agent 拿到"目录"而非"正文"。**

### 强制 exec 全文读取

写正文前必须用 PowerShell `Get-Content -Raw` 读取文风DNA全文，禁止使用 Read 工具读取：

```powershell
# ===== 参数（替换为你项目的实际值）=====
$dnaPath = "{填入文风DNA.md 完整路径}"
# =====================================
$dna = Get-Content $dnaPath -Raw -Encoding UTF8
Write-Output "文风DNA全文长度: $($dna.Length) chars"
Write-Output $dna
```

执行规则：
- **每次写新章节前必须执行**。不允许"我上一轮已经读过了"作为跳过理由——对话历史里的摘要不等于全文。
- 禁止用 Read 工具读取文风DNA文件——Read 工具会截断，返回内容只有原始文件的 9%-80%。
- 禁止只读取文风DNA的目录、摘要或概述——必须读取包含所有场景卡原文摘录的完整文件。
- 如果文风DNA文件不存在，先回 world-foundation 部署文风DNA，不得在没有文风DNA的情况下写正式正文。

### 场景卡定位（可选，用于注意力聚焦）

全文读取后，可从 plot 章级施工卡提取 `scene` 字段值列表（如 `[narration_suspense, dialogue_daily, description_environment]`），在全文中定位到匹配的场景卡重点参照——但其他场景卡也必须保留在上下文中，不得丢弃。

### 笔触 vs 内容红线

文风DNA传递的是**笔触**（句式怎么断、感官怎么排、叙事距离怎么站、段落怎么呼吸、对话怎么引导），不是**内容**（打谁、怎么打、打完想什么、战术怎么设计、角色怎么复盘）。

| ✅ 从文风DNA学笔触 | ❌ 不从文风DNA学内容 |
|:-------------------|:---------------------|
| 战斗用短句链+拟声词推进节奏 | 战斗中要写战术思维/量化复盘 |
| 情绪用身体动作外化不用直述 | 角色击杀后要有角色化余韵 |
| 独句段用于情绪收束/时间跳跃 | boss战必须濒死再反击 |
| 嗅觉→听觉→视觉的感官启动顺序 | 暗杀要按潜行→背刺→搜刮流程 |
| 叙事者锁定角色肩膀后侧 | 主角要在脑中计算胜算百分比 |

文风DNA的原文摘录是让 agent **感受作者怎么写**，不是让 agent **抄作者写了什么**。原文摘录里索伦的战术思维、战斗复盘、击杀余韵是深渊主宰的**剧情内容**，不是文风——agent 不该把这些提取为创作要求。

### 校验

- [ ] exec 返回的文风DNA长度 > 5000 字符（过小说明文件可能损坏或路径错误）
- [ ] 返回内容包含 `###` 场景卡标题和原文摘录引用块
- [ ] 返回内容包含"通用维度"和"全书稳定特征"段落
- 未通过 → 检查文件路径 / 文风DNA文件完整性

## 写法

- 开场承接上章最后一个动作、危险或疑问。
- 开场 120 字内出现目标、异常、危险、系统反馈或主动动作之一。
- 每一场都让主角做选择，选择带来代价或收益。
- 设定信息通过冲突、交易、观察、失败来显影，不用百科说明。
- 爽点必须有压迫前置、行动突破、旁人反应、局势收益。
- 关键爽点要外显，不要只藏在氛围里。优先使用：面板弹窗、数值变化、敌人反应、旁人态度变化、资源到账、身份变化、追杀等级变化。
- 每章至少完成一次动作闭环：压力逼近 -> 主角判断 -> 主动行动 -> 可数收益/明确损失 -> 新压力。
- 章末留下主动钩子：主角做出下一步选择，或威胁逼到眼前。
- 当读者表达偏好与文风DNA冲突时，以读者表达偏好裁决：小白直给、爽点外显、理解成本低优先。

## plot 施工卡消费

正文必须逐项消费 plot 章级施工卡：

| 施工卡字段 | 正文落地方式 |
| --- | --- |
| 章卡边界 | 只写当前章，保留下一章事件 |
| POV | 不乱切视角 |
| scene | 用于文风DNA切片 |
| 宏观剧情骨架 | 保持因果顺序，不擅自改剧情目标 |
| 微beat施工表 | 作为正文段落推进骨架，防止字数不足和靠氛围补字 |
| 主爽点 | 写出压迫、操作、反馈 |
| 主角主动动作 | 主角必须在正文中做出选择和行动 |
| 可数收益/明确损失 | 用资源、状态、关系、名声、经验等外显 |
| 爽点外显 | 面板/数字/反应/到账/身份变化至少一种 |
| 系统/战斗节点 | 数值和能力表现不能写错 |
| 章末钩子 | 改成主动钩子或威胁逼近 |
| 禁改项 | 不改核心事件和硬设定 |

## 角色锚点消费

- 只读本章登场角色锚点，不读整份 `角色储备池.md`。
- 反应模式、说话方式从幕纲/施工卡继承，不在正文阶段重设。
- 若本章需要未给出的主要角色，只能标记缺口或写成临时功能角色，不得擅自新增长期角色。
- C级路人只承担反馈功能：恐惧、羡慕、传谣、误判、围观等，不建立人物小传。

关键对白方向、场景锚点可作为可选增强；缺失时不得因此降级。

## 网文感硬规则

- 读者能直接看懂本章目标、压力、收益和下一步，不靠“品”。
- 每 600-800 字至少出现一次“推进读者快感的反馈”：新信息、新收益、新威胁、敌人误判、主角操作成功/失败。
- 每章至少 2 次让读者明确感到“主角在操作”，不是只在感受世界。
- 每章至少 1 次外部反馈：敌人、路人、系统、环境或资源状态对主角行动作出反应。
- 面板流正文不要太克制。涉及升级、经验、技能、任务、威胁评估时，允许清楚弹出并给读者读到。
- 讽刺点不要全靠隐喻。关键讽刺可以用短句直给，让读者当场获得情绪释放。
- 氛围段必须短，写完立刻回到目标、行动、风险或收益。

## 严肃文学偏移自检

写完后检查：

| 症状 | 修正 |
|:-----|:-----|
| 连续大段感官/心理，没有操作 | 插入主角选择、动作、试探或资源盘点 |
| 设定很高级，但读者不知道爽在哪里 | 加面板反馈、收益到账、敌人反应 |
| 主角像旁观者 | 改成主角主动测试规则、利用规则、钻规则漏洞 |
| 讽刺太含蓄 | 加一个独句段或角色反应，让情绪落地 |
| 章末只是悬疑 | 改成“主角下一步要做什么”或“威胁已经逼到哪” |
| 为了凑字写雾、气味、触感 | 回到微beat表，补动作、阻力、反馈 |
| 正文顺手写到下一章 | 在本章主动钩子处停下，拆出下一章 |

## 输出

按 `templates/创作-模板.md` 输出正文和简短创作记录。创作记录只写真实偏离点、设定新增风险和下一章衔接点，并必须包含 `execution` 凭证。`execution.mode` 不是 `formal` 时，不得把产物称为正式正文。
