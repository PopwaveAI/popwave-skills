# Step 1：事件轨迹正文化

> create 把 plot 已经运行出的当前章事件轨迹写成正文。不要重新设计剧情。

## 消费

- 当前章运行日志段落：`卷纲/运行/幕NNN-{名称}-运行日志.md`
- 单元卡当前章导演指令：`卷纲/幕NNN-{名称}.md`
- 设定账本：`卷纲/运行/幕NNN-{名称}-设定账本.md`
- 引用硬设定：`写作参考/设定/*.md`
- 上章末尾：`正文/ch{上一章}.md` 末尾
- 活记忆/状态快照
- 文风 DNA：`写作资产/文风库/{书名}.md`
- 创作模板：`templates/创作-模板.md`

## 1. 上下文确认

必须确认：

- 当前章运行日志已读取。
- 当前章导演指令已读取。
- 设定账本已读取。
- 引用硬设定已读取。
- 上章末尾已读取（首章除外）。
- 文风 DNA 已读取。

缺少运行日志或设定账本时，终止并退回 plot。

## 2. 复述本章不可改写项

写作前先列出：

```markdown
## 本章不可改写项
- 事件链：
- 本章结算：
- 章末推力：
- 禁止漂移设定：
```

这些内容在正文中可以扩写，但不能改变。

## 3. 场景流规划

把运行日志转换成 3-6 个正文场景。

每个场景必须对应一个作用：

- 承接上章。
- 主角主动行动。
- 冲突或压力显性化。
- 爽点释放/即时反馈。
- 设定通过事件露出。
- 章末推力。

不要逐条翻译运行日志。要把它写成连续场景。

## 4. 正文写作

写作要求：

- 第一屏直接进入冲突、悬念、反差或行动。
- 设定只通过动作、对话、面板、异常事件或他人反应露出。
- 主角行动必须改变局面。
- 行动后立刻给反馈：数值、资源、名声、关系、敌人状态、情报至少一项。
- 对话要推动关系或释放信息。
- 章末停在运行日志指定的推力点。

## 5. 自检

- [ ] 事件链未被改变。
- [ ] 本章结算未被改变。
- [ ] 未新增未入设定账本的重大设定。
- [ ] 设定没有百科式灌输。
- [ ] 主角有主动行动。
- [ ] 章末推力落地且没有写过头。
- [ ] 文风 DNA 已用于句式、节奏和叙事距离。

## 6. 输出

```markdown

{正文}
```
