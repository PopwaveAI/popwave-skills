# Step 1：涌现写作创作

> 管线: pop-writer-v3-create v1.1.0
> 类型: **子skill**（由expert-writer Step 2调度，context隔离）
> 消费: 种子六要素 + 活记忆 + 上章末尾 + chapter_plan + info_acquired + 创作模板
> 产出: 正文初稿 + 创作决策记录
> 红线: ❌1 context隔离——传入精简context，不传会话历史；❌2 主角行为必须符合种子行为准则

## 子skill调度说明

### 输入context（精简，不传会话历史）

| 输入项 | 来源 | 内容 |
|:-------|:-----|:-----|
| 种子六要素 | `种子/_index.yaml`+六要素文件（压力矩阵/主角引擎/金手指/冲突轴/成长路径/目的地） | 压力矩阵/主角引擎/金手指/冲突轴+活跃线索/成长路径/目的地 |
| 活记忆七组件 | `活记忆/活记忆.yaml`（最后baseline+event） | 种子追踪/压力状态/角色状态/世界规则/节奏日志/战力曲线/目的地进度 |

## 做什么

### Step 1: 上下文确认

按 `templates/创作-模板.md` §一 逐项确认输入context已加载，包括行为准则（种子主角引擎.行为准则字段）。
- **门禁：** 行为准则缺失 = 退回seed补全。

### Step 2: 涌现写作

在chapter_plan任务list框架内自由创作正文——任务list定义'达成什么'，create自主决定场景设计/情节编排/角色决策（涌现空间）。不再有预设场景填空。做实时决策（推进线索/压力源选择/金手指使用/角色决策/钩子设置/场景组织）。
- 参考 `references/创作指南.md` §二（chapter+prose合并原理）§三（场景流渲染技法）。
- **门禁：** 逐事件渲染（非场景流） = 退回重写。

### Step 3: 压力源逼近

本章正文必须至少推进一个压力源的显性表现。
- 参考 `references/创作指南.md` §四（压力源逼近技法）。
- **门禁：** 本章无压力源逼近 = 退回重写。

### Step 4: 章末钩子

将chapter_plan.hook落实到正文最后1-3段。
- 参考 `references/创作指南.md` §五（章末钩子设计）。
- **门禁：** 章末无钩子 = 退回重写。

### Step 5: 事件密度检查

确认本章事件转折数达标。
- 参考 `references/创作指南.md` §六（事件密度控制）。
- **门禁：** 事件密度不足（整章只写一件事） = 退回重写。

### Step 5.5: 字数下限门禁（v3.4新增）

统计正文纯文字字数（不含markdown标记）。
- 门禁：正文 < 2500字 = 退回重写（信息密度不足）
- 目标区间2500-3500字，事件密度8-12个/章
- 字数不足通常意味着场景展开不够/事件密度过低，需补充展开而非注水

### Step 6: 行为一致性检查（红线❌2）

逐场景检查主角行为是否符合种子行为准则。
- 参考 `references/创作指南.md` §七（主角行为一致性检查）。
- **门禁：** 行为不符合准则（无显性破例剧情） = 退回重写该场景。

### Step 7: 创作决策记录

按 `templates/创作-模板.md` §三 填写创作决策记录YAML，记录关键决策及其种子依据。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 逐事件渲染（非场景流） | 退回重写 |
| 本章无压力源逼近 | 退回重写 |
| 章末无钩子 | 退回重写 |
| 正文<2500字（字数下限门禁） | 退回重写 |
| 事件密度>15个/章（信息过载） | 退回精简 |
| **主角行为不符合行为准则（红线❌2）** | **退回重写该场景** |
| 行为准则缺失（种子/主角引擎.md 无行为准则字段） | 退回seed补全 |
| 种子/活记忆不存在 | 终止，提示先完成上游 |
| 上下文确认表未填 | 补填后再继续 |

> 创作子skill**不做**文风DNA对齐/8020比例/AI观感词清理检查（交给revise子skill）。
