# Step 1：涌现写作创作

> 管线: pop-writer-v3-create v1.0.0
> 类型: **子skill**（由pop-writer-v3-emerge Step 2调度，context隔离）
> 消费: 种子六要素 + 活记忆 + 上章末尾 + chapter_plan + info_acquired + 创作模板
> 产出: 正文初稿 + 创作决策记录
> 红线: ❌1 context隔离——传入精简context，不传会话历史；❌2 主角行为必须符合种子行为准则

## 子skill调度说明

### 输入context（精简，不传会话历史）

调度此子skill时，emerge调度器只传入以下精简context：

| 输入项 | 来源 | 内容 |
|:-------|:-----|:-----|
| 种子六要素 | `种子文档.md`（当前版本） | 压力矩阵/主角引擎/金手指/冲突轴+活跃线索/成长路径/目的地 |
| 活记忆七组件 | `活记忆/活记忆.yaml`（最后baseline+event） | 种子追踪/压力状态/角色状态/世界规则/节奏日志/战力曲线/目的地进度 |
| 上章末尾 | `正文/ch{上一章}.md` 最后~800字 | 语感衔接+悬念承接 |
| chapter_plan | Step 0产出 | 场景设计/线索推进/爽点设计/危机设计/钩子设计 |
| info_acquired | Step 1产出 | 增量信息（制度细节/场景技法/伏笔回收线索） |
| 创作模板 | `templates/创作-模板.md` | 正文涌现结构+创作决策记录格式 |

**不传入的内容：** 主会话历史、Step 0/1的执行过程、文风DNA、修订指南、质检模板。创作子skill专注故事涌现，不被文风/AI观感词/8020分散注意力。

### 输出

| 输出项 | 格式 | 传递给 |
|:-------|:-----|:-------|
| 正文初稿 | markdown | emerge调度器→Step 3（调度revise） |
| 创作决策记录 | YAML | emerge调度器→Step 4（调度qa） |

## 目的

创作子skill专注故事涌现——场景流渲染+压力源逼近+章末钩子+主角行为一致性检查。不做文风对齐（修订层负责）、不做8020比例检查（修订层负责）、不做AI观感词清理（修订层负责）。创作端只管故事结构层：事件如何组织、压力如何逼近、钩子如何设置、行为是否一致。

**v3.2核心改动：** 从emerge内嵌step拆为独立skill，context隔离从纪律约束变为架构边界。新增主角行为一致性检查（红线❌2），解决主角出戏问题。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `种子文档.md`（当前版本） | 六要素全量驻留：压力矩阵/主角引擎（含行为准则）/金手指/冲突轴+活跃线索/成长路径/目的地 |
| 2 | `活记忆/活记忆.yaml`（最后baseline+event） | 七组件当前状态 |
| 3 | `正文/ch{上一章}.md` 最后~800字 | 语感衔接+悬念承接 |

| 5 | Step 1产出的info_acquired | 增量信息（如有） |
| 6 | `references/创作指南.md` | 场景流渲染+压力源逼近+章末钩子技法+主角行为一致性检查流程 |
| 7 | `templates/创作-模板.md` | 正文涌现结构+创作决策记录格式+行为准则确认项 |

> 注意：创作子skill**不加载文风DNA**。文风DNA在revise子skill加载。

## 做什么

### 1. 上下文确认

按 `templates/创作-模板.md` 的上下文确认表，逐项确认输入context已加载：

| 确认项 | 状态 |
|:-------|:-----|
| 种子六要素已加载 | ☐ |
| 活记忆七组件已读取 | ☐ |
| 上章末尾已读取 | ☐ |
| chapter_plan已加载 | ☐ |
| info_acquired已加载（如有） | ☐ |
| 行为准则已加载 | ☐ |
| 创作模板已加载 | ☐ |

> 行为准则缺失（种子文档主角引擎无行为准则字段）= 退回seed补全。

### 2. 涌现写作（chapter+prose合并，无章纲约束）

agent在chapter_plan的框架内自由创作，做实时决策：

- 本章推进哪条活跃线索（从chapter_plan.plotlines_advanced中选取）
- 压力矩阵中哪个压力源逼近（从chapter_plan.crisis中选取）
- 金手指怎么使用（受限于种子的能力+限制）
- 角色做什么决策（受限于主角引擎的驱动力三层+缺陷+决策风格+**行为准则**）
- 章末钩子怎么设置（从chapter_plan.hook中落实）

**自由度边界：** chapter_plan给的是方向（场景/线索/爽点/危机/钩子），具体怎么写由创作子skill涌现。不是写章纲，是直接写正文。

### 3. 场景流渲染

按 `references/创作指南.md`，按场景流渲染（非逐事件）：

| 原则 | 规则 |
|:-----|:-----|
| 场景流渲染 | 事件在场景内有机融合，不逐事件独立成段 |
| 开幕段落 | 空间/时间锚定→情绪传递（1-2句） |
| 场景间过渡 | 1-2句自然过渡，不铺陈 |
| 闭幕段落 | 章末=钩子位置（悬念/情绪/行动） |

**WRONG（逐事件渲染）：**
```
索伦拔出匕首。他绕到恶棍身后。匕首刺入后心。他捂住对方的嘴。
```

**CORRECT（场景流渲染）：**
```
索伦绕到恶棍身后，匕首无声刺入后心——一只手捂住嘴，另一只手摸走钱袋，全程没发出一点声响。
```

### 4. 压力源逼近（种子消费）

本章正文必须至少推进一个压力源的显性表现。从种子压力矩阵中选取本章活跃的压力源，在正文中体现其逼近：

- 倒计时型压力：倒计时数值变化/主角感知到时间紧迫
- 威胁型压力：威胁逼近的具体表现（巡逻搜人/催债上门/病发征兆）
- 困境型压力：道德困境的具体呈现

多个压力源交叉时产生道德困境——这是压力矩阵设计的核心目的。

### 5. 章末钩子落实

chapter_plan.hook指定了钩子类型和内容，创作子skill负责将其落实到正文最后1-3段：

| 钩子类型 | 写法 |
|:---------|:-----|
| 悬念钩 | 不闭合，留疑问 |
| 情绪钩 | 安静收尾，情绪余韵 |
| 行动钩 | 蓄势待发，停在行动起点 |

### 6. 创作决策记录

在写作过程中，记录关键创作决策（按 `templates/创作-模板.md` 的创作决策记录YAML格式）：

```yaml
creation_log:
  chapter: {章号}
  decisions:
    - point: "推进线索T001"
      choice: "选择触面而非深化，因为上章已深化过"
      seed_basis: "种子冲突轴帮派层"
    - point: "压力源选择"
      choice: "催债人逼近而非帮派搜人"
      seed_basis: "种子压力矩阵-催债倒计时"
    - point: "金手指使用"
      choice: "L1危险感知技能触发"
      seed_basis: "种子金手指L1能力"
  word_count: {字数}
  scene_count: {场景数}
  event_count: {事件数}
```

### 7. 主角行为一致性检查（v3.2新增，红线❌2）

按 `references/创作指南.md` 的"八、主角行为一致性检查"章节，逐场景检查主角行为一致性：

| 步骤 | 检查内容 | 不一致时处理 |
|:-----|:---------|:------------|
| 加载行为准则 | 背景映射/战斗风格/道德底线/行为禁忌 | 准则缺失=退回seed补全 |
| 逐场景行为预测 | 基于行为准则预测"主角在这个场景会怎么行动" | — |
| 行为一致性验证 | 正文中的主角行为是否符合预测 | 不符合=修正行为描写 |
| 行为禁忌检查 | 正文是否触碰行为禁忌 | 触碰=修正（除非有显性破例剧情） |

**出戏检查红线：**

| 出戏类型 | 表现 | 处理 |
|:---------|:-----|:-----|
| 背景不符 | CIA精锐不灭口、战术动作简单 | 退回重写该场景 |
| 道德底线越界 | 无显性破例剧情却触碰底线 | 退回重写 |
| 行为禁忌违反 | 无合理原因却做禁忌之事 | 退回重写 |

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 逐事件渲染（非场景流） | 退回重写 |
| 本章无压力源逼近 | 退回重写 |
| 章末无钩子 | 退回重写 |
| **主角行为不符合行为准则（红线❌2）** | **退回重写该场景** |
| 行为准则缺失（种子文档无行为准则字段） | 退回seed补全 |
| 种子/活记忆不存在 | 终止，提示先完成上游 |
| 上下文确认表未填 | 补填后再继续 |

> 注意：创作子skill**不做**以下检查（交给修订层）：
> - ❌ 文风DNA对齐 → revise子skill
> - ❌ 8020展示/内省比例 → revise子skill
> - ❌ AI观感词清理 → revise子skill
