## 三、创作决策记录YAML

```yaml
creation_log:
  chapter: {章号}
  decisions:
    - point: "推进线索"
      choice: "{推进哪条线索，推进方式（触面/深化/回收）}"
      seed_basis: "{种子冲突轴/活跃线索}"

    - point: "压力源选择"
      choice: "{哪个压力源逼近}"
      seed_basis: "{种子压力矩阵}"

    - point: "金手指使用"
      choice: "{金手指怎么使用}"
      seed_basis: "{种子金手指L?能力+限制}"

    - point: "角色决策"
      choice: "{主角做了什么决策}"
      seed_basis: "{主角引擎驱动力?层+缺陷+决策风格+行为准则}"

    - point: "章末钩子"
      choice: "{钩子类型和内容}"
      seed_basis: "{chapter_plan.hook}"

    - point: "行为一致性"
      choice: "{主角行为是否符合行为准则？如有破例，说明剧情原因}"
      seed_basis: "{主角引擎.行为准则}"

  word_count: {正文字数}
  scene_count: {场景数}
  event_count: {事件转折总数}
  pressure_sources_advanced: [{本章逼近的压力源列表}]
  plotlines_advanced: [{本章推进的线索列表}]
  rewards_given: [{本章给读者的爽点列表}]
  behavior_consistency_passed: true  # 红线❌2
```

## 四、门禁表

| 门禁项 | 检查结果 | 失败动作 |
|:-------|:---------|:---------|
| 场景流渲染（非逐事件） | ☐ 通过 / ☐ 退回 | 退回重写 |
| 本章有压力源逼近 | ☐ 通过 / ☐ 退回 | 退回重写 |
| 章末有钩子 | ☐ 通过 / ☐ 退回 | 退回重写 |
| **主角行为符合行为准则（红线❌2）** | **☐ 通过 / ☐ 退回** | **退回重写该场景** |
| 上下文确认表已填 | ☐ 通过 / ☐ 补填 | 补填后再继续 |
| 种子/活记忆存在 | ☐ 通过 / ☐ 终止 | 终止，提示先完成上游 |

> 创作子skill**不做**文风DNA对齐/8020比例/AI观感词清理检查（交给revise子skill）。
