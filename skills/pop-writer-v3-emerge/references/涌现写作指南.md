# 涌现写作指南

> 所属: pop-writer-v3-emerge v1.0.0 · references/
> 消费: Step 2 涌现写作时参考
> 红线: ❌1 文风DNA缺失=硬阻塞终止
> 继承: v2 prose的文风DNA硬阻塞+场景流渲染+80/20内省

## 一、chapter+prose合并

### 合并理由

v2管线中chapter和prose分离——chapter产出章纲（场景流设计包），prose消费章纲渲染正文。每一层传递都是信息损失：设定层→大纲摘要→章纲摘要的摘要→prose拿到第三手信息渲染。

v3合并chapter+prose：agent拿着种子+活记忆直接涌现，做实时决策，没有章纲约束。设计决策和渲染执行在人类作者脑中是同时发生的，分离导致prose拿到的是"设计摘要"而非"创作冲动"。

### 合并后的agent自由度

agent在写作过程中自主决定：
- 本章推进哪条活跃线索（从种子冲突轴+活跃线索中选取）
- 压力矩阵中哪个压力源逼近（从种子压力矩阵中选取）
- 金手指怎么使用（受限于种子的能力+限制）
- 角色做什么决策（受限于主角引擎的驱动力三层+缺陷+决策风格）
- 章末钩子怎么设置
- 场景如何组织（无预设场景流设计包，agent自主构建场景）

### 上下文构成

| 上下文要素 | 来源 | 驻留方式 |
|:-----------|:-----|:---------|
| 种子七要素 | 种子文档（当前版本） | 全量驻留 |
| 活记忆七组件 | 活记忆.yaml（最后baseline+event） | 全量驻留 |
| 上一章正文末尾 | 正文/ch{上一章}.md 最后~800字 | 语感衔接 |
| 文风DNA | 写作资产/文风库/{书名}.md | 硬阻塞+全量驻留 |
| 方向提示 | 活记忆event.direction_hint | 方向感 |
| 增量信息 | Step 1 info_needs | 融入上下文 |

## 二、文风DNA硬阻塞（红线1）

### 硬阻塞规则

| 情况 | 处理 |
|:-----|:-----|
| DNA缺失 | 硬阻塞终止（不降级、不A/B、不空写） |
| DNA部分缺失 | 硬阻塞终止 |
| DNA路径错误 | 终止，提示修复路径 |

> DNA空 = 终止。不降级、不A/B、不空写。

### DNA路径解析

1. 从种子文档解析文风DNA引用路径 `写作资产/文风库/{书名}.md`
2. 加载文风DNA文件
3. 路径解析失败 → 终止，提示退回pop-writer-v3-seed修复

### DNA档案加载

全量加载DNA，建立两层感知：

**层A — 整体风格：**
- 句子长短（短句为主/长短交替/长句铺陈）
- 叙事者距离（紧贴感官/中等距离/上帝视角）
- 战斗节奏（快切/慢镜/混用）
- 对话留白（无引导词/有引导词/心理穿插）

**层B — 场景卡定位：**
- 观察句（DNA原文范例）
- 原文锚定（该场景类型的原文片段）
- 时间演变（该场景类型在全书中的演变趋势）

### 句式/叙事距离/感官密度锚定

加载后建立三个锚定基线：

| 锚定维度 | 基线来源 | 渲染时检查 |
|:---------|:---------|:----------|
| 句式 | DNA层A句式规则 | 正文句式是否匹配 |
| 叙事距离 | DNA层A叙事者距离 | 正文是否越界 |
| 感官密度 | DNA场景卡观察句 | 正文感官锚点数量 |

## 三、场景流渲染（继承v2 prose）

### 场景内连续流动

按场景流渲染，事件在场景内有机融合，**不逐事件独立成段**。

**WRONG（逐事件渲染）：**
```
索伦拔出匕首。他绕到恶棍身后。匕首刺入后心。他捂住对方的嘴。
```

**CORRECT（场景流渲染）：**
```
索伦绕到恶棍身后，匕首无声刺入后心——一只手捂住嘴，另一只手摸走钱袋，全程没发出一点声响。
```

### 场景渲染流程

每个场景走5步：

| 步骤 | 内容 |
|:-----|:-----|
| A. 理解意图 | 场景要推进什么？情绪走向？推进哪条线？ |
| B. 选POV | 主角POV（单主角假设），不进入非锚点角色内心 |
| C. 写场景段落 | 场景内连续流动 + DNA风格对齐 |
| D. 控制比例 | 按场景类型控制展示/内省比例 |
| E. 场景间过渡 | 1-2句自然过渡 |

### 开幕段落

- 空间/时间锚定 → 情绪传递（1-2句）
- 如有感官锚点，第一句以该感官信号开头
- 例：「雨声先到。然后是铁锈味。」

### 场景间过渡

- 1-2句自然过渡，不铺陈
- 过渡方式：时空跳转（"三天后"）/ 感官切换（"声音消失了"）/ 情绪转折
- 不写"与此同时"式的平行剪辑——保持单线POV

### 闭幕段落

章末 = 钩子位置：
- 悬念钩 → 不闭合，留疑问
- 情绪钩 → 安静收尾，情绪余韵
- 行动钩 → 蓄势待发，停在行动起点

## 四、80%展示+20%内省（继承v2 prose）

### 总则

80%展示+20%内省。展示 = 通过动作/环境/感官/对白传递情绪。内省 = 直接写角色内心活动。内省仅用于情感高潮点和关键决策点。

### 4场景类型展示/内省比例

| 场景类型 | 展示 | 内省 | 内省使用场景 |
|:---------|:----:|:----:|:------------|
| 行动场景 | 90% | 10% | 仅生死瞬间一闪念 |
| 情感场景 | 70% | 30% | 关系转折/情感爆发点 |
| 对话场景 | 85% | 15% | 对话间隙的心理反应 |
| 发现场景 | 80% | 20% | 信息揭示后的认知冲击 |

### 内省使用场景

允许内省的位置：
1. 关键决策点——角色做重大选择前的心理活动（1-2句）
2. 情感高潮点——情绪达到顶点时的内心感受（1-2句）
3. 信息揭示后——认知被颠覆时的冲击反应（1-2句）

禁止内省的位置：
- 普通动作描写中插入心理
- 连续2段以上纯内省
- 用内省替代展示（"他很害怕" 而非展示害怕的动作）

### 质量标准

| 检查项 | 标准 |
|:-------|:-----|
| 每章内省总次数 | 1-2处（情感场景可到3处） |
| 单次内省长度 | ≤ 2句 |
| 内省位置 | 仅关键决策点/情感高潮/信息揭示后 |
| 展示手段 | 动作/环境/感官/对白 ≥ 4种 |

## 五、AI观感词规避

逐段自检，命中即修正：

| 检查项 | 命中标准 |
|:-------|:---------|
| 心理动词 | "他感到/他意识到/他仿佛" |
| 对比句式 | "不是…而是…" ≥ 2次 |
| 解说员句式 | 叙事者解释设定（应通过角色对话/行动展示） |
| 视角越界 | 限知视角写非锚点角色内心 |

命中 > 3种 → 退回重写。

## 六、压力源逼近（种子消费）

本章正文必须至少推进一个压力源的显性表现。

从种子压力矩阵中选取本章活跃的压力源，在正文中体现其逼近：
- 倒计时型压力：倒计时数值变化/主角感知到时间紧迫
- 威胁型压力：威胁逼近的具体表现（巡逻搜人/催债上门/病发征兆）
- 困境型压力：道德困境的具体呈现

多个压力源交叉时产生道德困境——这是压力矩阵设计的核心目的。

## 七、与v2 prose的差异

| 维度 | v2 prose | v3 emerge |
|:-----|:---------|:----------|
| 输入 | 场景流设计包（chapter产出） | 种子+活记忆（无章纲） |
| 决策权 | chapter预设场景，prose渲染 | agent自主决策场景+渲染 |
| 信息损失 | 3次（设定→大纲→章纲→正文） | 1次（种子→正文） |
| 文风DNA | 硬阻塞（保留） | 硬阻塞（保留） |
| 场景流渲染 | 按设计包场景流（保留） | agent自主构建场景流（继承技法） |
| 80/20 | 按场景类型比例（保留） | 按场景类型比例（保留） |
| 压力源 | 设计包预设 | agent从种子压力矩阵自主选取 |
