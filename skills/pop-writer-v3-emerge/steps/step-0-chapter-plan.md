# Step 0：本章规划

> 管线: pop-writer-v3-emerge v1.2.0
> 消费: 种子文档(六要素) + 活记忆 + 上一章正文 + 方向提示 + 网文爽感机制
> 产出: 本章写作规划（写入活记忆event的`chapter_plan`字段）

## 目的

写之前先想明白。看种子+上一章正文，判断这章要怎么写——写什么场景、推进什么线索、给什么爽点、制造什么危机、留什么钩子。规划做完，再进Step 1强制化信息获取。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | `种子文档.md`（当前版本） | 六要素：压力矩阵/主角引擎/金手指/冲突轴+活跃线索/成长路径/目的地——这章可用的故事引擎 |
| 2 | `活记忆/活记忆.yaml`（最后baseline+event） | 七组件当前状态——压力倒计时到哪了、哪些线索超期、节奏趋势 |
| 3 | `正文/ch{上一章}.md`（末尾500字） | 衔接——上章留了什么悬念，本章从哪里接 |
| 4 | 上一章方向提示（活记忆event.direction_hint） | 上章质检标记的待处理项（必须给奖励/必须加压/必须触面某线索） |
| 5 | `references/网文爽感机制.md` | **写作法则**——本章规划必须对照检查 |

## 做什么

### 1. 故事状态感知

从活记忆七组件提取当前状态：

| 组件 | 提取什么 | 用于判断 |
|:-----|:---------|:---------|
| 压力状态 | 哪些倒计时在走？剩多少？ | 本章必须推进哪个压力源 |
| 种子追踪 | 哪些线索活跃？哪些超期预警？ | 本章该触面哪条线索 |
| 节奏日志 | 最近3章的事件密度趋势 | 本章该加速还是减速 |
| 战力曲线 | 主角当前能力水平 | 本章能力使用边界 |
| 目的地进度 | 最近的目的地进展 | 本章是否推进 |
| 角色状态 | 主角心理/资源/关系 | 驱动力哪层在激活 |

### 2. 本章规划（5个决策点）

基于状态感知+种子+方向提示，做5个规划决策：

**① 场景设计**：这章写什么场景？（1-3个场景，每个场景有明确目的）
- 场景来源：种子冲突轴的哪层生态？哪个压力源在逼近？哪条活跃线索推进？

**② 线索推进**：这章推进哪3条以上不同的活跃线索？
- 从活记忆种子追踪选≥3条线索，标注推进方式（触面/深化/回收）

**③ 爽点设计**：这章给读者什么实质奖励？（至少1个）
- 对照`references/网文爽感机制.md`的奖励类型（技能/装备/金钱/信息/情感/地位）
- 爽点必须即时兑现，不能延迟到下章

**④ 危机设计**：这章的压力怎么走？
- 至少1个压力源在本章逼近或爆发
- 章末必须留即时危机钩子（下章开头需解决的危机，不是远期悬念）

**⑤ 钩子设计**：章末留什么？
- 即时危机型钩子（如"催债人敲门"），不是远期悬念型（如"7天后决战"）

### 3. 网文爽感机制对照检查

将本章规划对照`references/网文爽感机制.md`的10条法则：

| 法则 | 本章规划是否满足 | 如不满足的调整 |
|:-----|:----------------|:-------------|
| 压力机制 | ☐ | 补充压力源逼近 |
| 即时奖励循环 | ☐ | 补充爽点 |
| 不间断危机链 | ☐ | 补充章末危机钩子 |
| 高事件密度(8-12) | ☐ | 补充场景内转折 |
| 主角主动决策 | ☐ | 改被动为主动 |
| 养成感可见 | ☐ | 补充能力/资源变化 |
| 信息披露节奏 | ☐ | 补充世界信息 |
| 80%展示20%内省 | ☐ | 标注内省点 |
| 多线交叉(≥3线) | ☐ | 补充线索推进 |
| 情绪闭环 | ☐ | 检查情绪弧线 |

不满足的法则，回到步骤2调整规划。


## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 种子文档不存在 | 终止，提示先完成pop-writer-v3-seed |
| 活记忆不存在 | 终止，提示先初始化活记忆 |
| 网文爽感机制10条法则全部对照（红线❌5） | 未对照的法则回到步骤2调整 |
| ≥3条线索推进 | 补充线索 |
| ≥1个即时爽点 | 补充爽点 |
| 章末即时危机钩子 | 补充钩子 |
