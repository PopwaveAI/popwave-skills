# Step 3：修订子agent（context隔离）

> 管线: pop-writer-v3-emerge v1.2.0
> 类型: **子agent**（由expert-writer Execute层调度，context隔离）
> 消费: Step 2正文初稿 + 文风DNA + 种子六要素 + 活记忆 + 修订checklist
> 产出: 修订稿 + 修订记录
> 红线: ❌1 文风DNA缺失=修订层硬阻塞终止；❌7 context隔离

## 子agent调度说明

### 输入context（精简，不传会话历史）

调度此子agent时，只传入以下精简context：

| 输入项 | 来源 | 内容 |
|:-------|:-----|:-----|
| 正文初稿 | Step 2创作子agent产出 | 待修订的正文全文 |
| 文风DNA | `写作资产/文风库/{书名}.md` | 句式/叙事距离/感官密度/对话留白锚定（硬阻塞，红线❌1） |
| 种子六要素 | `种子文档.md`（当前版本） | 主角引擎/金手指/冲突轴+活跃线索（用于人设丰富+bug修复） |
| 活记忆七组件 | `活记忆/活记忆.yaml` | 角色状态/世界规则/种子追踪（用于bug修复+人设丰富） |
| 修订checklist | `templates/修订checklist-模板.md` | 5项修订任务checklist |

**不传入的内容：** 主会话历史、Step 0/1/2的执行过程、创作决策记录（修订层不评判创作决策，只看正文质量）、质检模板。

### 输出

| 输出项 | 格式 | 传递给 |
|:-------|:-----|:-------|
| 修订稿 | markdown | Step 4质检子agent |
| 修订记录 | YAML | Step 4质检子agent（用于验证修订质量） |

## 目的

修订子agent对创作初稿做5项修订任务：文风对齐、人设丰富、爽点验证、bug修复、AI观感词清理。文风DNA在此层加载并硬阻塞（红线❌1从创作端移到修订层）。修订层不改变故事结构（不增删场景/不改线索走向），只做渲染层面的打磨。

**v3.1核心改动：** 文风DNA从创作端移到修订层。创作端专注故事涌现不受文风干扰，修订层专门做文风对齐。这样创作端不会因为同时管文风+故事导致注意力分散。

| 3 | `种子文档.md`（当前版本） | 主角引擎/金手指/冲突轴+活跃线索（人设丰富+bug修复的依据） |

## 做什么

### 1. 文风DNA硬阻塞检查（红线❌1）

- 加载文风DNA文件 `写作资产/文风库/{书名}.md`
- DNA缺失/路径错误/文件空 → **硬阻塞终止**（不降级、不A/B、不空写）
- DNA加载成功 → 建立两层感知：
  - 层A整体风格：句子长短/叙事者距离/战斗节奏/对话留白
  - 层B场景卡：观察句+原文锚定+时间演变

> 文风DNA是项目资产（由pop-writer-v3-seed的step-3蒸馏产出），不进种子六要素。修订层直接从项目资产加载。

### 2. 五项修订任务

按 `references/修订指南.md` 和 `templates/修订checklist-模板.md`，逐项执行：

#### 任务1：文风对齐

| 子项 | 检查内容 | 标准 |
|:-----|:---------|:-----|
| DNA加载 | 文风DNA已加载 | 硬阻塞通过 |
| 层A整体风格 | 句式/叙事距离/战斗节奏/对话留白 | 正文风格匹配DNA层A |
| 层B场景卡 | 场景类型对应的观察句/原文锚定 | 正文感官锚点数量达标 |
| 8020比例 | 80%展示+20%内省 | 按场景类型比例（行动90/10、情感70/30、对话85/15、发现80/20） |

**门禁：8020超标=退回。** 连续2段以上纯内省=退回。

#### 任务2：人设丰富

| 子项 | 检查内容 | 标准 |
|:-----|:---------|:-----|
| 主角引擎对齐 | 行为是否符合驱动力三层+缺陷+决策风格 | 不符合=修正 |
| 心理层次 | 决策前的心理活动是否有层次（不是扁平反应） | 关键决策点有1-2句心理 |

#### 任务3：爽点验证

| 子项 | 检查内容 | 标准 |
|:-----|:---------|:-----|
| 即时奖励 | 主角行动后是否有即时反馈 | 至少1次实质奖励 |
| 养成感 | 能力/资源/地位变化是否可见 | 读者能感知变化 |
| 情绪闭环 | 情绪弧线是否完整（紧张→释放→新紧张） | 起点情绪→转折→终点情绪 |

#### 任务4：bug修复

| 子项 | 检查内容 | 标准 |
|:-----|:---------|:-----|
| 世界规则 | 正文是否违反种子确立的世界规则 | 违反=修正 |
| 伏笔一致性 | 正文是否与活记忆种子追踪的已埋伏笔矛盾 | 矛盾=修正 |
| 金手指限制 | 金手指使用是否超出种子设定的能力+限制 | 超出=修正 |
| 时间线 | 正文时间线是否与活记忆节奏日志矛盾 | 矛盾=修正 |

#### 任务5：AI观感词清理

逐段扫描，命中即修正：

| 检查项 | 命中标准 | 修正方式 |
|:-------|:---------|:---------|
| 心理动词 | "他感到/他意识到/他仿佛" | 改为展示（动作/感官/对白） |
| 对比句式 | "不是…而是…" ≥ 2次 | 减少到≤1次或重写 |
| 解说员句式 | 叙事者解释设定 | 改为角色对话/行动展示 |
| 视角越界 | 限知视角写非锚点角色内心 | 删除越界段落 |

**门禁：AI观感词命中>3种=退回重写。**

### 3. 修订记录

按 `templates/修订checklist-模板.md` 的修订记录YAML格式，记录修订内容：

  chapter: {章号}
  style_alignment:
    dna_loaded: true
    layer_a_check: "句式匹配，叙事距离紧贴感官"
    layer_b_check: "感官锚点3处/场景"
    ratio_8020: "通过（行动场景90/10）"

    psychology_depth: "决策点有2句心理活动"

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| **文风DNA缺失（红线❌1）** | **硬阻塞终止** |
| **AI观感词命中>3种** | **退回Step 2重写** |
| **8020比例超标** | **退回重写** |
| 世界规则违反 | 修正后再继续 |
| 金手指超出限制 | 修正后再继续 |
| 伏笔矛盾 | 修正后再继续 |
| 时间线矛盾 | 修正后再继续 |
