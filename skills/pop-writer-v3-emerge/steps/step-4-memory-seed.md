# Step 4：活记忆更新+种子生长

> 管线: pop-writer-v3-emerge v1.0.0
> 消费: Step 2正文 + Step 3反思报告 + 活记忆 + 种子文档
> 产出: 活记忆event追加 + 种子文档（生长时更新版本）

## 目的

反思通过后，更新活记忆七组件。同时agent自主判断是否触发种子生长——发现新压力源/新线索/新规则时，自主决定是否追加到种子。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | Step 2正文（已通过反思） | 提取本章状态变化 |
| 2 | Step 3反思报告 | 不达标项的处理标记 |
| 3 | `活记忆/活记忆.yaml`（最后baseline+event） | 追加event的基准 |
| 4 | `种子文档.md`（当前版本） | 种子生长的基础 |
| 5 | `references/活种子生长触发规则.md` | 生长场景+判断原则+版本管理 |

## 做什么

### 1. 活记忆七组件更新（红线3：唯一写入者）

向 `活记忆/活记忆.yaml` 追加本章event，更新七组件：

| 组件 | 更新内容 | 数据形态 |
|:-----|:---------|:---------|
| 种子追踪 | 本章新埋伏笔（章号+内容+预期回收点）；已回收伏笔标记状态变更 | 列表追加 |
| 压力状态 | 倒计时当前值变化；新压力源激活 | 键值对更新 |
| 角色状态 | 主角的数值/心理/资源/关系变化增量 | 章末追加 |
| 世界规则 | 本章新确立的规则（只增不改） | 规则表追加 |
| 节奏日志 | 本章事件密度/爽点数/钩子类型 | 章号+三项指标 |
| 战力曲线 | 主角能力值变化（章号+能力值） | 折线数据追加 |
| 目的地进度 | 目的地到达状态变化 | 状态更新 |

event条目格式：
```yaml
- type: event
  chapter: {章号}
  timestamp: ...
  seeds_added: []           # 新埋伏笔
  pressure_changes: {}      # 压力变化
  character_changes: {}     # 角色变化
  world_rules_added: []     # 新世界规则
  rhythm_entry: {events: N, rewards: N, hook_type: ...}
  power_entry: {ch: N, ...}
  destination_updates: []
  seed_growths: []          # 种子生长记录（如有）
  direction_hint: ""        # Step 5填写
  info_needs: []            # Step 1记录
```

### 2. 种子生长判断（agent自主判断，权限全给agent）

按 `references/活种子生长触发规则.md`，agent判断本章是否发现需要追加到种子的新要素：

**判断原则：**
- 如果新发现的要素能生成未来剧情 → 追加到种子
- 如果只是当前章节的一次性事件 → 不追加，只记入活记忆
- 如果不确定 → 追加（宁多勿少，弧线校准时可修剪）

**生长场景（示例，非穷举）：**

| 触发场景 | 生长要素 | 示例 |
|:---|:---|:---|
| 发现新压力源 | 压力矩阵 | ch5发现强化剂依赖风险→追加 |
| 发现新线索 | 活跃线索 | ch8发现巫毒教与裂隙关联→追加 |
| 确立新世界规则 | 世界规则 | ch10发现系统等级与精神力关系→追加 |
| 成长路径细化 | 成长路径 | ch12选择巫医职业→阶段4细化 |
| 发现新目标 | 目的地 | ch15发现裂隙本质线索→追加 |

### 3. 种子生长执行（红线4：版本号+变更日志）

如果agent判断触发种子生长：

1. 更新种子文档对应要素
2. 版本号 minor+1（如 1.0.0→1.1.0）
3. 变更日志追加记录
4. 同步更新活记忆对应组件
5. 在event的`seed_growths`字段记录生长内容

变更日志格式：
```markdown
| 版本 | 章号 | 类型 | 变更内容 | 触发原因 |
|:-----|:-----|:-----|:---------|:---------|
| 1.1.0 | 5 | grow | 压力矩阵追加"强化剂" | ch5发现新压力源 |
```

版本号规则（语义化）：
- **major**（X.0.0）：种子要素结构变化（新增/删除要素类型）
- **minor**（1.X.0）：生长（追加新压力源/新线索/新规则/新目的地）
- **patch**（1.0.X）：修剪/调整（弧线校准时执行，非本步骤）

### 4. 不触发种子生长的情况

如果agent判断本章无新要素需要追加到种子：
- 不更新种子文档
- 不更新版本号
- 在event的`seed_growths`字段填空数组`[]`
- 活记忆照常更新七组件

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 活记忆event未追加（红线3） | 补追加后再继续 |
| 七组件未完整更新 | 补全后再继续 |
| 种子生长但未更新版本号（红线4） | 补更新版本号+变更日志 |
| 种子生长但未更新变更日志（红线4） | 补追加变更日志 |
| 种子生长但活记忆未同步 | 补同步后再继续 |

## 产出

1. `活记忆/活记忆.yaml` — 追加本章event（七组件更新）
2. `种子文档.md` — 如触发生长，更新要素+版本号+变更日志
3. event的`seed_growths`字段 — 记录生长内容（或空数组）
