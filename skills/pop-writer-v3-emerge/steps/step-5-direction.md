## 做什么

### 1. 方向提示生成

方向提示是1-2句话的方向感，**不是章纲**。不指定具体事件——只说"需要给奖励、需要加压、需要触面某线索、需要即时钩子"。具体怎么实现，由下一章写作agent涌现。

方向提示来源：
- Step 3反思报告中不达标项的处理标记
- 活记忆种子追踪组件的超期预警伏笔
- 活记忆压力状态的倒计时停滞情况
- 种子活跃线索的长期未触及情况

方向提示格式：
```
方向提示 ch{下一章号}:
- {方向项1}
- {方向项2}
```

方向项类型（非穷举）：
- **需要给奖励**：上章只有信息无实质获得
- **需要加压**：某倒计时已N章未推进
- **需要触面某线索**：某活跃线索已N章未出现
- **需要即时钩子**：上章钩子不够即时
- **需要加速/减速**：事件密度偏离趋势

示例：
```
方向提示 ch004:
- 必须给奖励（ch3只有信息，无实质获得）
- 压力需加码（血量倒计时已3章未推进）
- 回收短线：MS-13追杀线已3章未出现，需触面
- 钩子必须是即时危机
```

### 2. 回写方向提示到活记忆

将方向提示写入本章event的`direction_hint`字段。

### 3. 正文落盘

将Step 2正文（已通过反思）写入 `正文/chXXX.md`：

```markdown
# chXXX · {本章标题}

{正文全文}
```

### 4. 项目总控更新

更新 `项目总控.md`：
- 当前章号+1
- 种子版本（如本章触发了种子生长，更新为新版本号）
- 弧线校准计数（距下次校准的剩余章数-1）
- 阶段状态（涌现写作中）

### 5. 弧线校准触发检查

检查是否满足弧线校准触发条件：
- 定期触发：距上次校准已满10-20章（建议值15章）
- 事件触发：活记忆任意组件触发预警（伏笔超期>20章/战力连续跳级/连续5章低密度/目的地20章无进展/线索10章未触及）

如触发 → 提示用户进入 `pop-writer-v3-arc` 弧线校准。
如未触发 → 继续下一章，回到Step 1。

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 方向提示未生成 | 补生成后再继续 |
| 方向提示写了具体事件（非方向感） | 退回，改为方向感 |
| 正文未落盘 | 补落盘后再继续 |
| 项目总控未更新 | 补更新后再继续 |
| 活记忆direction_hint未回写 | 补回写后再继续 |
