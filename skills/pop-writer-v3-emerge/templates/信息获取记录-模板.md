# 信息获取记录模板

> 所属: pop-writer-v3-emerge v1.2.0 · templates/
> 消费: Step 1 信息获取强制化产出
> 写入位置: 活记忆event条目的 info_acquired 字段
> 红线: ❌6 信息获取必须读索引——索引.md未读取=退回补读

## 使用说明

每章Step 1完成后，将信息获取结果填写为YAML格式，写入活记忆event的`info_acquired`字段。

**强制要求：** `index_read`必须为true（红线❌6）。4类检查均无明确需求时，`needs`填空数组`[]`，这是合法的。

## 模板

```yaml
info_acquired:
  chapter: {章号}
  index_read: true  # 红线❌6：索引.md已读取（必须为true）
  needs:
    # —— 类型1：制度/流程细节 ——
    # 判断问题：这章涉及什么我不熟悉的现实制度/流程？
    # 信息源：WebSearch（→写入知识库）
    - type: 制度/流程
      question: {具体问题，如"美国催债公司的追债流程和手段"}
      source: WebSearch→知识库/制度流程/美国催债法流程.md
      result: {搜索提取的关键信息}
      kb_new: true  # 是否新建了知识库文件
      merged: true  # 是否已融入写作上下文

    # —— 类型2：场景技法 ——
    # 判断问题：这章的场景类型需要参考技法吗？
    # 信息源：library套路库
    - type: 场景技法
      question: {具体问题，如"催债人上门的冲突场景技法"}
      source: library套路库/{套路名}.md
      result: {提取的场景技法}
      kb_new: false
      merged: true

    # —— 类型3：已埋伏笔/前文细节 ——
    # 判断问题：这章需要呼应哪些已埋伏笔？
    # 信息源：项目本身（活记忆种子追踪+前章正文）
    - type: 已埋伏笔
      question: {具体问题，如"前身'布鲁斯·韦恩'的背景细节"}
      source: 活记忆种子追踪 + ch{章号}正文
      result: {提取的伏笔细节}
      kb_new: false
      merged: true

    # —— 类型4：创意参考 ——
    # 判断问题：这章有什么场景需要灵感参考？
    # 信息源：WebSearch（→写入知识库）/ library
    - type: 创意参考
      question: {具体问题，如"绝境反杀的节奏控制"}
      source: WebSearch→知识库/创意参考/绝境反杀节奏.md
      result: {提取的创意参考}
      kb_new: true
      merged: true

  # 本章新建的知识库文件列表
  kb_new_files:
    - 写作资产/知识库/制度流程/美国催债法流程.md
    - 写作资产/知识库/创意参考/绝境反杀节奏.md
  
  # 索引是否已更新
  index_updated: true
  
  # 所有信息已融入写作上下文
  all_merged: true

# —— 无需求示例 ——
# info_acquired:
#   chapter: {下一章号}
#   index_read: true  # 索引已读取（红线❌6通过）
#   needs: []  # 常规章节，4类检查均无明确需求，无需查询
#   kb_new_files: []
#   index_updated: false
#   all_merged: false
```

## 填写规范

| 字段 | 说明 |
|:-----|:-----|
| chapter | 章号 |
| index_read | **必须为true**（红线❌6）。索引.md已读取 |
| needs | 信息需求清单，空数组=无需求（合法） |
| needs[].type | 4类之一：制度/流程、场景技法、已埋伏笔、创意参考 |
| needs[].question | 具体问题（一句话） |
| needs[].source | 信息源：WebSearch→知识库/{分类}/{文件名}.md / library套路库/{名}.md / 活记忆种子追踪+前章正文 |
| needs[].result | 获取到的关键信息摘要 |
| needs[].kb_new | 是否新建了知识库文件（true/false） |
| needs[].merged | 是否已融入写作上下文（true/false） |
| kb_new_files | 本章新建的知识库文件路径列表 |
| index_updated | 索引是否已更新（有新文件时必须为true） |
| all_merged | 所有需求是否已融入（true/false） |

## 知识库目录结构说明

知识库位于 `写作资产/知识库/`，按分类组织：

```
写作资产/知识库/
├── 索引.md                    # 知识库总索引（强制读取）
├── 制度流程/                  # 现实制度/流程调研
├── 场景技法/                  # 场景写法技法
├── 创意参考/                  # 创意灵感参考
└── 伏笔追踪/                  # 伏笔细节
```

## 索引.md格式模板

首次执行时创建 `写作资产/知识库/索引.md`：

```markdown
# 知识库索引

> 最后更新: ch{章号} · {timestamp}

## 制度流程/
| 文件 | 创建章号 | 关键词 | 简述 |
|:-----|:---------|:-------|:-----|
| {文件名}.md | ch{N} | {关键词} | {简述} |

## 场景技法/
| 文件 | 创建章号 | 关键词 | 简述 |
|:-----|:---------|:-------|:-----|

## 创意参考/
| 文件 | 创建章号 | 关键词 | 简述 |
|:-----|:---------|:-------|:-----|

## 伏笔追踪/
| 文件 | 创建章号 | 关键词 | 简述 |
|:-----|:---------|:-------|:-----|
```

## 不需查的示例

常规章节（日常生存、已建立机制的运作）不需要查。但**索引必须已读取**：

```yaml
info_acquired:
  chapter: 4
  index_read: true  # 索引已读取（红线❌6通过）
  needs: []  # ch4是主角日常生存+已建立的捡垃圾机制运作，4类检查均无明确需求
  kb_new_files: []
  index_updated: false
  all_merged: false
```
