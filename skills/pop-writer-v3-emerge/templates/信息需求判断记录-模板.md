# 信息需求判断记录模板

> 所属: pop-writer-v3-emerge v1.0.0 · templates/
> 消费: Step 1 信息需求判断产出
> 写入位置: 活记忆event条目的 info_needs 字段

## 使用说明

每章Step 1完成后，将信息需求判断结果填写为YAML格式，写入活记忆event的`info_needs`字段。

4类检查均无明确需求时，`needs`填空数组`[]`，`merged`填`false`。

## 模板

```yaml
info_needs:
  - chapter: {章号}
    needs:
      # —— 类型1：制度/流程细节 ——
      # 判断问题：这章涉及什么我不熟悉的现实制度/流程？
      # 信息源：WebSearch
      - type: 制度/流程
        question: {具体问题，如"美国催债公司的追债流程和手段"}
        source: WebSearch
        result: {搜索提取的关键信息}
        merged: true  # 是否已融入写作上下文

      # —— 类型2：同类场景写法 ——
      # 判断问题：这章的场景类型需要参考技法吗？
      # 信息源：library套路库
      - type: 同类场景写法
        question: {具体问题，如"催债人上门的冲突场景技法"}
        source: library套路库/{套路名}.md
        result: {提取的场景技法}
        merged: true

      # —— 类型3：文风渲染参考 ——
      # 判断问题：这章的文风渲染需要参考吗？
      # 信息源：library文风库
      - type: 文风渲染参考
        question: {具体问题，如"冷硬对话节奏、暴力场景感官描写"}
        source: library文风库/{书名}.md
        result: {提取的渲染参考}
        merged: true

      # —— 类型4：已埋伏笔/前文细节 ——
      # 判断问题：这章需要呼应哪些已埋伏笔？
      # 信息源：项目本身（活记忆种子追踪+前章正文）
      - type: 已埋伏笔
        question: {具体问题，如"前身'布鲁斯·韦恩'的背景细节"}
        source: 活记忆种子追踪 + ch{章号}正文
        result: {提取的伏笔细节}
        merged: true

    # 所有信息已融入写作上下文
    all_merged: true

  # —— 无需求示例 ——
  - chapter: {下一章号}
    needs: []  # 常规章节，4类检查均无明确需求，无需查询
    all_merged: false
```

## 填写规范

| 字段 | 说明 |
|:-----|:-----|
| chapter | 章号 |
| needs | 信息需求清单，空数组=无需求 |
| needs[].type | 4类之一：制度/流程、同类场景写法、文风渲染参考、已埋伏笔 |
| needs[].question | 具体问题（一句话） |
| needs[].source | 信息源：WebSearch / library套路库/{名}.md / library文风库/{名}.md / 活记忆种子追踪+前章正文 |
| needs[].result | 获取到的关键信息摘要 |
| needs[].merged | 是否已融入写作上下文（true/false） |
| all_merged | 所有需求是否已融入（true/false） |

## 不需查的示例

常规章节（日常生存、已建立机制的运作）不需要查。示例：

```yaml
info_needs:
  - chapter: 4
    needs: []  # ch4是主角日常生存+已建立的捡垃圾机制运作，4类检查均无明确需求
    all_merged: false
```
