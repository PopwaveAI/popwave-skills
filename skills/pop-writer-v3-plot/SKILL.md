# Pop Writer V3 Plot

plot 负责解决“如何正向写出逆向拆书级别的剧情单元卡”。不要先填表，要先让一个单元在世界里真实运行。

## 输入

- seed 产出的世界观底盘、主角引擎、金手指规则、文风DNA、初始设定账本。
- 上一单元 arc 复盘或当前活记忆。
- 用户指定的目标：卷方向、单元目标、章节数、爽点偏好、禁忌。

## 核心流程

1. 建立状态契约：人物位置、资源、能力限制、敌我压力、未兑现承诺、潜在种子。
2. 做种子碰撞：让人物目标、世界规则、组织利益、金手指限制、环境压力互相撞击。
3. 写章节运行日志：像已经写过正文一样，逐章记录事件、选择、代价、收益、设定显影、钩子。
4. 维护设定账本：运行中产生的新规则、新组织、新物品，只冻结必要部分。
5. 固化幕纲：把运行日志整理成可供 create 直接写正文的单元剧情卡。

## 执行

- 读 `steps/step-1-volume-anchor.md` 确定卷级方向。
- 读 `steps/step-2-幕纲-design.md` 生成单元幕纲。
- 按 `templates/幕纲-模板.md` 输出。

## 红线

- 不用旧版填表式写作环。
- 不做空泛爽点标签，爽点必须来自局势变化。
- 不一次性设计全世界设定，只冻结本单元运行必需的设定。
- 每章必须能被 create 直接写成正文。
