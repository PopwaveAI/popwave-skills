# pop-writer-v3-plot · 剧情单元运行器

plot 的目标不是“写一张漂亮幕纲表”，而是**正向运行一个可被拆书的故事单元**。

核心原则：**不要正向填单元卡；先跑出章节事件轨迹，再把轨迹编译成单元卡。**

## 产物分层

| 产物 | 路径 | 用途 |
|:-----|:-----|:-----|
| 单元状态契约 | `卷纲/运行/幕NNN-{名称}-状态契约.md` | 起点、终点、种子池、设定缺口、不可逆变化 |
| 章节事件仿真日志 | `卷纲/运行/幕NNN-{名称}-运行日志.md` | 每章 300-600 字，只写事件因果和状态变化 |
| 设定账本 | `卷纲/运行/幕NNN-{名称}-设定账本.md` | 新孵化设定、已冻结设定、待解释缺口 |
| 单元卡/幕纲 | `卷纲/幕NNN-{名称}.md` | 从运行日志编译的可执行幕纲，供 create 使用 |

## 核心流程

### 1. 建立状态快照

读取上一单元 arc 更新后的实际记录、活记忆、seed 产物和卷级方向，形成当前棋盘：

- 主角当前能力、资源、心理、社会身份。
- 身边人状态和关系变化。
- 外部压力和未解决危险。
- 已冻结设定和待解释缺口。
- 上一章/上一单元留下的钩子。

### 2. 写单元终点契约

先定本单元结束时必须发生的不可逆变化，再倒推章节承担。

合格写法：

```markdown
本单元终点：
- 主角从无名者变成“割喉者”，名声覆盖贫民区。
- 盗贼从 1 级升到 3 级，获得第一个超凡专长。
- 科尔帮冲突闭合，萨维势力转为敬畏。
- 薇薇安从被欺负者转为被敬畏者妹妹。
- 世界暗线以剥皮事件露头，指向下一单元藏宝洞。
```

### 3. 列设定缺口

不要补全百科。只写本单元运行需要哪些设定空位：

```markdown
设定缺口：
- 主角需要一种低风险但专业感强的练级方式。
- 贫民区需要一个能触碰主角底线的地头势力。
- 世界灾变需要一个早期异常事件，但不能直接揭开主线。
```

### 4. 运行章节事件仿真日志

每章写 300-600 字。不是正文，不追求文采；追求因果密度、动作具体、状态可结算。

每章必须包含：

- 开局状态：本章从上一章什么状态接上。
- 种子碰撞：哪些压力/底线/资源/异常发生碰撞。
- 主角主动行动：主角具体做什么。
- 事件链：2-4 个具体事件转折。
- 爽点定位：行动如何立刻改变局面。
- 设定孵化：本章有没有按需生成新设定。
- 结算：人物、资源、关系、威胁、情报的不可逆变化。
- 章末推力：下一章为什么必然发生。

### 5. 冻结设定账本

只有被事件使用过的设定才允许出生。出生后写入账本。

```markdown
## 已冻结设定
- 盗贼可通过撬锁/拆陷阱获得少量职业经验。
- 贫民区帮派默认用窥探、勒索、拐卖试探弱者。

## 本单元新增设定
- “割喉者”名号通过尸体和传言传播，而非官方记录。

## 待解释缺口
- 晨曦神殿为什么无力介入贫民区？
```

### 6. 编译单元卡

从运行日志中提取结构，而不是重新设计：

- 元数据、物理坐标、一句话单元。
- 结构分析/章节导演指令。
- 嵌套子线和咬合关系。
- 单元边界信号。
- 可迁移要点/不可迁移部分。
- create 需要的逐章事件 brief、设定账本指针、章末钩子。

## 红线

| 编号 | 红线 |
|:-----|:-----|
| R1 | 禁止直接填单元卡。必须先有状态契约和章节运行日志。 |
| R2 | 运行日志不是正文；每章 300-600 字，写事件仿真，不写完整文采。 |
| R3 | 每章必须有一个主角主动行动、一个不可逆状态变化、一个章末推力。 |
| R4 | 设定只有在事件使用时才允许出生；出生后必须写入设定账本。 |
| R5 | 前置只放种子，不提前命名完整剧情线；剧情线从运行日志中复盘提取。 |
| R6 | 单元卡所有子线、咬合、边界信号必须能回指到运行日志。 |

## 步骤加载

| step | 文件 | 产出 |
|:-----|:-----|:-----|
| step-1 | `steps/step-1-volume-anchor.md` | 卷级方向锚和单元切分 |
| step-2 | `steps/step-2-幕纲-design.md` | 状态契约、运行日志、设定账本、单元卡 |

## 交接给 create

create 优先读取：

1. 当前章运行日志段落。
2. 单元卡中当前章导演指令。
3. 设定账本和被引用的硬设定。
4. 上章末尾和活记忆状态。

create 不需要重新推理本单元结构，只负责把已经运行过的事件轨迹写成正文。
