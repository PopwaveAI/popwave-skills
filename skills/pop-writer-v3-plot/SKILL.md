# pop-writer-v3-plot · L2剧情单元卡设计 v1.3.0

从 v1.0 的"任务链设计"重构为 v3.5 的"L2剧情单元卡设计"。核心变化：**取消种子文档，L2卡成为唯一运行时活文档**；plot 不再写入种子文档，而是直接产出 L2 卡。

核心隐喻：L2卡 = 跨章叙事单元的活文档（4-10章/卡），定义"这组章节要达到什么效果"（导演意图层），不预设具体场景和台词。每章写作前 emerge 从 L2 卡结构分析表提取导演意图；每个单元结束时 arc 把 L2 卡从"设计方案"更新为"实际记录"。

## 设计理念（v3.5）

- **约束驱动涌现**：不预设路径，但预设约束；给方向不给路径
- **信息注入优先**：plot 设计 L2 卡时从 library 匹配同类 L2 卡作为结构标杆——v2 雪花法失败的根因是信息注入不足，agent 没见过好的结构就设计出空洞的东西
- **判断前移**：信息需求判断在 plot 设计时一次性完成，写作时 emerge 只执行不判断
- **卡在导演意图粒度**：不下探到场景设计层，保留创作空间同时提供质量框架

## ❌ 质量红线

| 编号 | 红线 | 检查方式 |
|:-----|:-----|:---------|
| ❌1 | L2卡结构分析表必须含"设定引用"列（每章标注涉及的设定文件路径） | 逐章核对指针列非空且路径存在 |
| ❌2 | 事件brief必须用动作链（具体动作+因果），禁止纯功能描述 | 抽查brief含"→"动作链与具体动作 |
| ❌3 | plot设计L2卡时必须从library匹配同类L2卡（≥2张）作为参考 | 确认library参考卡路径已记录在产出中 |
| ❌4 | L2卡活跃嵌套子线≤5条（防膨胀，超出需说明必要性） | 计数嵌套子线条目 |
| ❌5 | L2卡设计前必须完成篇幅确认（v1.3新增） | step-1 Step 0 篇幅确认环节未执行=硬阻塞 |

## 路由

upstream `pop-writer-v3-seed`（产出写作参考/设定/）+ `pop-trope-library`（同类L2卡参考）→ **本skill** → downstream `expert-writer`

> plot 设计 L2 卡如遇信息缺口，委托 `pop-research`（独立调研skill）补缺。

## 产出

- `卷纲/卷N-方向锚.md`（≤300字）：卷定位+起点终点四维度+主冲突
- `卷纲/L2-NNN.md`（L2剧情单元卡）：唯一运行时活文档，含设定引用指针

## 多层信息依赖（设计L2卡时）

plot 设计 L2 卡时的信息依赖优先级（详见 step-2）：

| 优先级 | 信息源 | 用途 | 必须/可选 |
|:-------|:-------|:-----|:---------|
| 1 | 前一个L2卡实际内容（arc更新后） | 衔接：主角当前状态+子线延伸状态+未回收伏笔 | 必须（L2-001除外，首张无前序） |
| 2 | 写作参考/设定/ | 全局设定 ground truth | 必须 |
| 3 | library匹配的同类L2卡（2-3张） | 结构标杆——同类单元的设计维度参考 | 必须 |
| 4 | L3剧情线卡（如有） | 宏观参考：剧情线已推进到哪、哪些债务待回收 | 可选（L2-001除外） |

关键设计决策：**plot 不事前设计 L3**。L3 由 arc 从实际写作中事后提炼。plot 设计 L2 卡时，如 L3 已存在则作为宏观参考，不存在则不依赖。
