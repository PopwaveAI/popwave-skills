# Pop Writer V3 Plot

plot 负责解决"如何正向写出逆向拆书级别的剧情单元卡"。它不是一次性出稿器，而是可被用户中途干预的剧情规划工作台。

## 执行模式

- `formal`：本 `SKILL.md` 和必要 step 文件已读取，上游宪法门通过，过程文件、候选竞技、审计和最终幕纲质量门全部完成。
- `draft`：本 skill 已读取，但上游底盘、案例消化、候选竞技、审计或章级导演包有缺口。
- `trial`：未完整加载本 skill，或用户只要快速剧情方向。

只有 `formal` 可以称为正式幕纲，并标记"可直接给 create"。`draft/trial` 不得写入 `library/单元剧情卡/`，也不得在回报中说"幕纲完成"。

## 上游宪法门

进入 Step 0 前必须确认 `library/设定账本/world-产物索引.md` 存在。若缺失，只能输出缺口报告，不得写正式卷纲或幕纲。

## 输入

- seed/world-foundation 产出的全书 PRD、设定涌现目录、L1 世界包、起点快照、世界宪法、动态升级表、主角引擎、金手指规则、文风DNA、初始设定账本。
- 上一单元 arc 复盘或当前活记忆。
- 用户指定的目标：卷方向、单元目标、章节数、爽点偏好、禁忌、外部参考母版。
- seed 输出的 `library/设定账本/跨媒介母版雷达.md`；若缺失，plot 必须临时生成本幕母版扫描。
- 公共资料库：优先读取 `pop-trope-library` 的 `任务索引/`，再按索引指向精读剧情库 L2/L3、套路库、设定库、金手指库。

## 公共知识库

路径：`$env:APPDATA\popwave\remote-skills\pop-trope-library`

| 子库 | 用途 |
|:-----|:-----|
| `任务索引/` | 写作任务入口，先回答"当前任务该读哪几张案例、读哪些段、学什么" |
| `剧情库/` | 按书保存的 L2 单元卡和 L3 剧情线原始高保真资产，由任务索引指向后精读 |
| `套路库/` | 场景技法（战斗/扮猪吃虎/绝境反杀等），部署套路时查节奏控制段 |

查库方式：先读 `$env:APPDATA\popwave\remote-skills\pop-trope-library\任务索引\00-索引.md`，进入匹配任务索引，再读取索引推荐的 L2/L3。

## 核心流程

0. 全书/卷级尺度契约与卷级设计意图：确认全书/当前卷体量、商业承诺、主角变化、主压力、主收益、禁区和不可牺牲项；禁止写幕细节。
1. 卷级战略锚、卷内幕序列与当前幕切片：输出卷级战略锚和 5-7 个幕的功能位，锁定当前幕编号、章节预算和功能位；仍禁止写逐章剧情。
2. 任务定位与案例消化：按任务读案例，先声明资料覆盖率；如有复刻/嫁接意图，先读取 `04-复刻路由表.md`，plot 只处理"复刻位置=plot"的条目；同时主动做外部母版扫描。
3. 事件源发散：从 library 调研、外部类型母版、逻辑强制发散、角色视角事件池多路生成事件原料。
4. 事件碰撞与筛选：按多元素整合范式标注相似、相反、盲区、互斥、可融合、强因果事件，形成候选事件矿池。
5. 故事引擎铸造：从事件矿池中组装"压力引擎、收益引擎、敌人误判引擎、主角操作引擎、情绪/爽点曲线、长线债务"。
6. 互斥候选方案：至少生成 3 套不同剧情引擎驱动的方案。每套必须有不同主压力、不同第一主动动作、不同收益路径、不同敌人误判链。
7. 竞技场 PK：逐章对比三套方案，按追读、爽点、主角操作、压力升级、设定行动化、正文可写性打分，必须淘汰至少一套。
8. 胜者合成：只允许吸收败者的局部强点，不允许把三套方案堆叠成杂烩。
9. 反证式质量门：缺单章导演意图、状态快照、设定引用、可数收益、爽点外显、事件拍点、create交接，均不可输出最终稿。
10. 设定账本增量：运行中产生的新规则、新组织、新物品，只冻结必要部分。
11. 幕纲压缩：把胜出仿真整理成可供 create 直接写正文的唯一可用稿。

> Step 3-7 的详细执行规则（事件源四路、外部母版移植、母版本土化门、角色视角事件池、事件碰撞分类、故事引擎六问、候选竞技规则）在 `steps/step-3-simulation.md` 里。

## 产物分层

| 层级 | 路径 | 用途 | 可否给 create |
|:-----|:-----|:-----|:---:|
| 卷级战略稿 | `卷纲/卷N-战略锚.md`、`卷纲/卷N-幕序列.md` | 给用户确认卷方向、商业承诺、体量、禁区和幕功能位 | 否 |
| 过程文件 | `卷纲/运行/幕XXX-*` | 复盘、审计、中途干预、定位失败原因 | 否 |
| 唯一可用幕纲 | `卷纲/幕纲-幕XXX-{标题}.md` | create 的唯一直接输入 | 是，且必须通过质量门 |
| 章级导演包 | 包含在最终幕纲内，或另存 `卷纲/章节设计包/幕XXX-chYYY-设计包.md` | 每章正文施工卡 | 是 |

每次 plot 完成后必须写 `卷纲/运行/幕XXX-产物索引.md`。`library/单元剧情卡/` 不是 plot 的默认最终输出位置。

## 最终幕纲质量门

缺以下任一项，只能标记为"草案"：

- 每章有导演意图：叙事功能、事件动作链、情绪/爽点曲线、章末钩子。
- 每章有开章状态快照：人物位置、资源、伤势、压力、未兑现承诺。
- 每章有设定引用：本章需要读取哪些设定账本/文风DNA/金手指文件。
- 每章有主角主动动作：不是醒来、发现、被告知，而是主动选择推动局势。
- 每章有可数收益/明确损失：经验、钱、装备、线索、声望、关系、伤势、代价至少一种。
- 每章有爽点外显方式：面板、数字、敌人误判、旁人反应、资源到账、身份变化至少一种。
- 每章有章级导演包：地点、在场人物、POV、事件链、关键对白方向、感官锚点、爽点外显、章末钩子、状态更新。

## 红线

- 禁止黑箱合并 Step 0-4。过程文件必须落盘到 `卷纲/运行/`。
- 禁止把卷纲写成弱版幕纲。卷纲不得包含逐章剧情、事件矿池、候选竞技。
- 禁止越级设计。Step 0 只做全书/卷级，Step 1 才切当前幕，Step 2 之后才允许进入案例和剧情事件。
- 禁止假多方案。至少三套互斥方案（不同主压力/第一主动动作/收益路径/敌人误判链），且必须淘汰至少一套。
- 禁止未通过质量门的幕纲写入 `library/单元剧情卡/`。
- 禁止上游 PRD/L1 世界包/动态升级表缺失时硬写正式幕纲。
- 没有 `execution.mode: formal` 时，不得称正式幕纲完成。
