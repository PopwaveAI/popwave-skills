# Step 0 - 尺度契约与用户对齐

剧情设计先对齐体量，再进入幕纲。不要默认短篇化。

## 1. 默认网文尺度

若用户没有另行指定，按以下尺度理解：

| 层级 | 常规目标 | Agent 可执行压缩 |
|:-----|:---------|:----------------|
| 全文 | 500章起步 | 先规划 3-5 卷骨架，不一次性细化全文 |
| 单卷 | 60-80章 | 可先细化当前卷 30-50章，但不得默认 10章完卷 |
| 单幕/L2 | 10-15章 | 可先详细展开 8-12章；试跑时才允许 3-6章 |
| 单章 | 2k-3k字 | 正文 create 按此执行 |

如果用户说“试跑/小样/验证流程”，可以压缩到 3-6 章幕；否则不要把开局幕压成 3-4 章。

## 2. 必须落盘的对齐文件

先写：

```text
卷纲/运行/幕XXX-00-尺度契约.md
```

内容包括：

- 全文预估章数。
- 当前卷预估章数。
- 当前幕预估章数。
- 允许 agent 详细展开到哪里。
- 哪些部分只做骨架，不做细纲。
- 需要用户确认的问题。

## 3. 用户确认点

除非用户明确说“无需中途确认/全自动跑完”，完成尺度契约和卷幕方向草案后必须停下，向用户确认：

1. 当前卷是不是太短/太长。
2. 当前幕是不是太短/太长。
3. 第一幕主方向是否成立。
4. 是否继续进入案例消化和深推演。

## 4. 禁止

- 禁止把“完整执行”理解为“跳过用户确认”。
- 禁止用 10 章当默认卷长。
- 禁止用 3-4 章当默认幕长。
- 禁止只在最终幕纲里展示所有过程，必须保留阶段文件。
