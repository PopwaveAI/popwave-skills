# Step 0 - 全书/卷级尺度契约与卷级设计意图

剧情设计先定“全书/当前卷”，再切到“当前幕”。不要在 Step 0 讨论当前幕具体起点、终点、事件链。

Step 0 只回答三件事：

1. 全书大概是什么体量。
2. 当前卷承诺给读者什么。
3. 当前卷有哪些边界和不可牺牲项。

当前幕只允许一句话带过：本轮后续将从当前卷中切出哪个幕做深推演。幕的具体功能位、起点终点、候选方向留到 Step 1。

## 1. 默认网文尺度

若用户没有另行指定，按以下尺度理解：

| 层级 | 常规目标 | Agent 可执行压缩 |
|:-----|:---------|:----------------|
| 全文 | 500章起步 | 先规划 3-5 卷骨架，不一次性细化全文 |
| 单卷 | 60-80章 | 可先细化当前卷 50-80章，不得默认 10章完卷 |
| 单幕/L2 | 10-15章 | 可先详细展开 8-12章；试跑时才允许 3-6章 |
| 单章 | 2k-3k字 | 正文 create 按此执行 |

如果用户说“试跑/小样/验证流程”，可以压缩到 3-6 章幕；否则不要把开局幕压成 3-4 章。

## 2. 输出路径

```text
卷纲/运行/卷N-00-全书卷级尺度契约.md
卷纲/运行/卷N-00b-卷级设计意图契约.md
```

禁止在 Step 0 输出 `幕XXX-*` 文件。文件名出现 `幕XXX` 会诱导 agent 跳级到幕细节。

## 3. 全书/卷级尺度契约

必须包含：

- 全文预估卷数与章数。
- 当前卷序号。
- 当前卷预估章数。
- 当前卷预估幕数。
- 每幕默认章数范围。
- 本轮允许详细展开到哪里。
- 哪些部分只做骨架，不做细纲。
- 是否允许压缩，以及压缩风险。

## 4. 卷级设计意图契约

必须包含：

- 当前卷商业承诺：读者追什么。
- 当前卷主爽点：生存、反杀、升级、经营、探索、权谋、救援等，最多选 1-2 个主味型。
- 当前卷主角变化：主角从什么状态变成什么状态。
- 当前卷主压力：谁/什么系统在压迫主角。
- 当前卷主收益：卷末能兑现什么身份、资源、能力、地盘、认知或关系。
- 当前卷禁区：不能提前揭露、不能跳级获得、不能破坏的设定。
- 当前卷不可牺牲项：长期压力、长期敌人、主角成长、金手指反馈、关键关系、设定禁区。
- 外部母版使用边界：本卷可借鉴哪些类型机制，不能让外部母版夺走什么。
- 当前待切片幕：只写“后续将从卷N切出幕XXX”，不得写幕起点终点和具体事件。

## 5. 用户确认点

除非用户明确说“无需中途确认/全自动跑完”，完成 Step 0 后必须停下，向用户确认：

```markdown
## Step 0 待确认：卷级
- 全文/当前卷体量：
- 当前卷商业承诺：
- 当前卷主角变化：
- 当前卷主压力：
- 当前卷主收益：
- 当前卷禁区：
- 当前卷不可牺牲项：
- 是否继续进入 Step 1：卷级战略锚、幕序列、当前幕切片？
```

## 6. 禁止

- 禁止在 Step 0 设计当前幕具体剧情。
- 禁止在 Step 0 写当前幕起点、终点、逐章事件、候选方案。
- 禁止把“完整执行”理解为“跳过用户确认”。
- 禁止用 10 章当默认卷长。
- 禁止用 3-4 章当默认幕长。
- 禁止只在最终幕纲里展示所有过程，必须保留阶段文件。
- 禁止没有卷级设计意图契约就进入幕纲深推演。
- 禁止把外部母版当成默认唯一骨架，除非用户明确确认本卷/本幕目的就是该类型。
