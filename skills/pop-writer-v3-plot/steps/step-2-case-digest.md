# Step 2 - 幕纲

把卷纲幕序列中的一幕展开为分章施工卡。逐幕循环执行，每幕产出一个幕纲。

按 `templates/幕纲-模板.md` 输出到 `卷纲/幕纲-幕NNN-{标题}.md`。

## 2a. 定位确认

从卷纲幕序列复制当前幕的：

- 幕编号 + 章节范围
- 功能位（一句话）
- 主角弧线：起点→终点
- 主压力
- 主收益
- 上幕衔接 / 下幕衔接

不需要重新设计，只复制确认。

## 2b. 选种子

从卷纲事件种子池中摆出与当前幕功能位相关的种子，供用户挑选：

1. 列出相关主线种子（3-5条）
2. 列出相关支线/暗线种子（2-3条）
3. 列出相关世界观/设定种子（1-2条）
4. 询问用户是否要补充种子（"我想加个电影里的剧情""这段我看过类似的处理"）

用户确认选定的种子列表后，进入故事推演。

如果用户提供了参考剧情（电影/小说/游戏片段），做简要案例消化：
- 可迁移的结构/机制是什么（不是剧情摘要）
- 爽点证据是什么
- 本幕怎么落地
- 不要编号管理，不要证据绑定清单，brief即可

## 2c. 故事推演

基于选定的种子，展开为分章beat链。

### 登场人物

从卷纲舞台表中选取本幕登场人物，填幕初/幕末状态。

### 地点 + 时间

本幕覆盖的地点和时间跨度。

### 分章剧情展开

每章一个块：

```
### ch001 · {章名}
- 字数基准：XXXX字
- 情绪目标：一句话
- 剧情骨架：beat1→beat2→beat3（每个beat一句话）
- 系统/战斗节点：
- 章末钩子：
```

节奏要求：
- 每章3-5个beat
- 事件间隔≤20行
- 每章至少1个爽点（压力→突破/弱→强/未知→揭示/被欺→反击）
- 每章主角至少1个主动决策+1个行动
- 章末钩子必须是主角主动决策型

### 子线追踪

本幕内推进的子线（从卷纲种子池中选取的相关项）：

| 子线 | 幕初状态 | 各章推进 | 幕末状态 |
|------|----------|----------|----------|

## 2d. Create 交接

幕纲末尾输出给 create 的施工指令：

- 文风注意：
- 不可改动事件：
- 可自由发挥空间：
- 爽点外显要求：

## 2e. 用户确认门

输出幕纲到 `卷纲/幕纲-幕NNN-{标题}.md`，暂停。

向用户确认：
- 分章beat链是否合理
- 节奏是否需要调整
- 是否交 create 写正文

用户确认后交 create。写完正文后回来做下一幕幕纲（回到 2a）。
