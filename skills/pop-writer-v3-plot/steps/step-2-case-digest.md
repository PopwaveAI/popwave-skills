# Step 2 - 幕纲

把卷纲幕序列中的一幕展开为分章施工卡。逐幕循环执行，每幕产出一个幕纲。

按 `templates/幕纲-模板.md` 输出到 `卷纲/幕纲-幕NNN-{标题}.md`。

## 2a. 定位确认

从卷纲幕序列生成/继承当前幕定位，必须使用模板六项：

- 幕定位（一句话：这幕在卷中干什么）
- 主爽点（1-2个）
- 读者期待（读者读完这幕应该得到什么）
- 主角弧线：起点→终点
- 上幕衔接：怎么进来的
- 下幕衔接：最后怎么勾住下一幕

输出当前幕定位后暂停，请用户确认或修改。用户确认前，不进入选种子和幕级外部燃料台。

## 2b. 选种子

从卷纲事件种子池中摆出与当前幕功能位相关的种子，供用户挑选：

1. 列出相关主线种子（3-5条）
2. 列出相关支线/暗线种子（2-3条）
3. 列出相关世界观/设定种子（1-2条）
4. 询问用户是否要补充种子（"我想加个电影里的剧情""这段我看过类似的处理"）

用户确认选定的种子列表后，进入故事推演。

## 2c. 幕级外部燃料台

幕纲阶段窄用外部信息：只精读当前幕对应的主母版单元；辅助桥段最多1-2个，并说明补哪个缺口。

### 资料覆盖声明

- 主母版当前单元：已读范围（完整L2/章节设计包/片段）
- 辅助桥段：已读范围
- 未覆盖但可能影响本幕：

最低要求：主母版当前单元必须读完整；若只有片段，幕纲只能标记为草案。

### 主母版当前单元

允许短复述原剧情，保留具体事件形状：

- 原剧情短复述：谁想要什么、发生了哪几个动作、爽点在哪里兑现。
- 本幕要保留的功能：练级/清算/地牢/撤离/大战等。
- 本项目变形方式：用本书设定和主角解法替换原解法。

### 辅助桥段

只在当前幕缺少新鲜感、反转、战斗调度、地下规则、追踪逃亡等具体燃料时使用：

| 来源 | 补的缺口 | 原桥段短复述 | 本项目变形 | 采用章 |
|------|----------|--------------|------------|--------|

不要输出大型证据绑定清单。每个辅助桥段1-3句即可。

## 2d. 故事推演

基于选定的种子，展开为分章beat链。

### 登场人物

从卷纲舞台表中选取本幕登场人物，填幕初/幕末状态。

### 地点 + 时间

本幕覆盖的地点和时间跨度。

### 分章剧情展开

每章一个块：

```
### ch001 · {章名}
- 字数基准：XXXX字
- POV：主角限制视角/配角短视角/妹妹视角等
- scene：1-3个文风DNA场景标签
- 情绪目标：一句话
- 主爽点：一句话说明读者爽在哪里
- 剧情骨架：beat1→beat2→beat3（每个beat一句话）
- 主角主动动作：
- 可数收益/明确损失：
- 爽点外显：面板/数字/敌人反应/旁人态度/资源到账/身份变化等
- 系统/战斗节点：
- 章末钩子：
- 禁改项：
```

节奏要求：
- 每章3-5个beat
- 事件间隔≤20行
- 每章必须有主爽点（压力→突破/弱→强/未知→揭示/被欺→反击），不要只写情绪目标
- 每章必须有 POV 和 scene；scene 用于 create 读取文风DNA
- 每章主角至少1个主动决策+1个行动
- 每章必须有可数收益或明确损失：钱/经验/装备/名声/情报/地盘/关系/代价
- 每章必须有爽点外显方式，不能只写"爽"
- 章末钩子必须是主角主动决策型

### 子线追踪

本幕内推进的子线（从卷纲种子池中选取的相关项）：

| 子线 | 幕初状态 | 各章推进 | 幕末状态 |
|------|----------|----------|----------|

## 2e. Create 交接

幕纲末尾输出给 create 的施工指令：

- 文风注意：
- 不可改动事件：
- 可自由发挥空间：
- 爽点外显要求：

## 2f. 用户确认门

输出幕纲到 `卷纲/幕纲-幕NNN-{标题}.md`，暂停。

向用户确认：
- 分章beat链是否合理
- 节奏是否需要调整
- 幕级外部燃料是否需要追加或替换
- 是否交 create 写正文

用户确认后交 create。写完正文后回来做下一幕幕纲（回到 2a）。
