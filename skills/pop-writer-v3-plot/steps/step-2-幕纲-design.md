# Step 2：剧情单元运行与幕纲编译

> 定位：plot 核心步骤。不要直接填幕纲表；先运行故事，再编译单元卡。

## 消费

- `卷纲/卷N-方向锚.md`
- `卷纲/推演/卷N-故事大纲.md`（如有）
- 上一单元 arc 后的实际单元卡（L2-001 可跳过）
- `活记忆/活记忆.yaml`
- `写作参考/设定/故事DNA.md`
- `写作参考/设定/世界宪法.md`
- `写作参考/设定/力量体系.md`
- `写作参考/设定/主角引擎.md`
- `写作参考/设定/金手指.md`

- `写作参考/设定/初始种子池.md`

## 产出

- `卷纲/运行/幕NNN-{名称}-状态契约.md`
- `卷纲/运行/幕NNN-{名称}-运行日志.md`
- `卷纲/运行/幕NNN-{名称}-设定账本.md`
- `卷纲/幕NNN-{名称}.md`

## 1. 读取并建立当前棋盘

用自然语言写当前状态，不要复制长设定。

必须包含：

- 主角：能力、资源、心理、社会身份、短期目标。
- 身边人：关系、风险、可被触发的情感底线。
- 外部压力：未解决敌人、制度压力、世界异常、经济压力。
- 已冻结设定：本单元必须遵守的硬规则。
- 待解释缺口：本单元可能需要孵化的软设定。
- 上一章/上一单元钩子：读者还在等什么。

## 2. 写单元终点契约

先定义单元结束时的不可逆变化，再倒推章节职责。

格式：

```markdown
# 幕NNN 状态契约

## 起点状态
{当前棋盘}

## 终点契约
- 人物状态变化：
- 力量/资源变化：
- 关系/声望变化：
- 冲突闭合：
- 新压力/新线索：

## 章节职责倒推
| 章节 | 必须完成的不可逆变化 | 为什么必须在这里发生 |
|:-----|:---------------------|:---------------------|
| chXXX | | |
```

## 3. 列本单元设定缺口

只写“缺口”，不要急着补设定。

```markdown
## 设定缺口
- 练级/赚钱/行动需要什么机制？
- 冲突对象需要什么社会位置？
- 世界异常需要通过什么事件露头？
- 情感线需要什么外部压力触发？
```

## 4. 运行章节事件仿真日志

每章 300-600 字。它不是正文，不写完整文采；它是事件仿真和状态结算。

每章格式：

```markdown
### chXXX · {本章运行目标}

**开局状态**：{从上一章什么状态接上}
**种子碰撞**：{压力/底线/资源/异常/冲突对象如何碰撞}
**事件仿真**：{300-600字。写清主角具体行动、2-4个事件转折、对手反应、即时反馈、结尾推力。}
**爽点定位**：{主角行动如何立刻改变局面}
**设定孵化**：{新生设定；无则写“无”}
**本章结算**：
- 人物：
- 资源/能力：
- 关系/声望：
- 威胁/情报：
**章末推力**：{下一章为什么必然发生}
```

### 运行规则

- 每章必须有主角主动行动。
- 每章必须有不可逆状态变化。
- 每章至少让两个种子发生碰撞。
- 每章只能孵化本章事件需要的设定。
- 章末推力必须把下一章推出来。
- 如果一章只是“铺垫/过渡/介绍设定”，重跑。

## 5. 维护设定账本

从运行日志中提取，不从脑内补全。

```markdown
# 幕NNN 设定账本

## 已冻结设定
- {本单元前已经存在且本单元继续使用的设定}

## 本单元新增设定
- {本单元因事件需要而出生的设定，写明首次出现章节和作用}

## 待解释缺口
- {本单元仍未解释，但后续需要处理的设定缺口}

## 禁止漂移
- {后续 create/revise 不得改动的规则}
```

## 6. 从运行日志编译单元卡

复制 `templates/幕纲-模板.md`，填入：

- 元数据与物理坐标。
- 一句话单元。
- 状态契约摘要。
- 章节运行日志摘要。
- 章节导演指令表。
- 设定账本指针。
- 嵌套子线与咬合。
- 单元边界信号。
- 可迁移要点/不可迁移部分。

## 7. 提取剧情线

剧情线只从运行日志中提取，不能前置硬编。

合格提取：

```markdown
### 子线A：帮派清算线（闭合）
起点：ch006 加里斯接触。
触发：ch007 窥探妹妹，触碰底线。
爆发：ch008 一夜清算。
余波：ch009 名声扩散。
```

不合格：

```markdown
本章推进主线/成长线/情感线。
```

## 8. 自检

- [ ] 已产出状态契约。
- [ ] 已产出每章 300-600 字运行日志。
- [ ] 每章都有主角主动行动。
- [ ] 每章都有不可逆结算。
- [ ] 每章章末推力能推出下一章。
- [ ] 新设定都来自事件需要，并写入设定账本。
- [ ] 单元卡内容可回指运行日志。
- [ ] 子线为事后提取，不是前置填表。

## 交接

把当前章运行日志段落、单元卡当前章导演指令、设定账本和引用硬设定交给 create。
