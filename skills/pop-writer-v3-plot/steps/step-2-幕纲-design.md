# Step 2 - 幕纲设计

## 1. 状态契约

写清单元开局的硬状态：

- 主角在哪里，手里有什么，缺什么。
- 敌人或环境正在怎样逼近。
- 金手指能做什么，不能做什么，代价是什么。
- 上一单元留下了哪些承诺、伏笔、仇恨、误会、债务。
- 本单元必须显影哪些设定，哪些设定只埋不说。

## 2. 潜在种子碰撞

不要把种子当“灵感列表”。让它们互相撞：

- 人物目标撞世界规则。
- 金手指限制撞生存压力。
- 组织利益撞主角短期选择。
- 旧承诺撞新机会。
- 环境资源撞敌人行动。

碰撞后只保留能产生事件链的种子。

## 3. 章节运行日志

逐章写“已经发生过什么”，而不是写“本章要表达什么”。

每章至少包含：

- 开章状态。
- 触发事件。
- 主角选择。
- 阻力升级。
- 设定如何通过事件显影。
- 爽点落点：谁吃亏、谁震惊、主角获得什么局势收益。
- 章末新状态和下一章钩子。

## 4. 设定账本增量

运行日志中出现的新设定分三类：

- 冻结：后文必须遵守的硬规则。
- 待定：只在本单元临时使用，arc 后确认。
- 禁止扩写：气氛词、比喻、一次性细节，不进入设定。

## 5. 幕纲固化

把运行日志整理成单元剧情卡。要求 create 看到每章日志后可以直接写正文，不需要再替你设计主事件。
