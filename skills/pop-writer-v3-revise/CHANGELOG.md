## v1.2.0 — 2026-06-28 — v3.5 涌现式写作管线重构

**定位升级：** 从 v3.4 渲染层修订升级为 v3.5 渲染层+导演意图验证。新增 v3.5 核心质量机制——导演意图验证。

### 新增

- **导演意图验证（红线❌3，v3.5核心）**——重写后对照主会话注入的导演意图验证清单检查5项：①叙事功能是否匹配 ②事件链是否覆盖（核心事件是否都出现）③情绪曲线是否对齐 ④子线是否推进 ⑤章末钩子是否有。不通过则标注偏差项写入修订记录，不自动退回——退回由主会话 Step4 决定
- **导演意图验证清单输入**——主会话从导演意图提取5项验证标准作为检查清单注入（revise 不接收导演意图完整内容，创作决策不回传）
- **上章末尾输入**——新增 `正文/chXXX.md`末尾输入用于衔接检查（重写稿开头与上章末尾自洽）
- **context receipt 预留接口**——产出时附带 context receipt 确认实际接收的注入项（draft/style_dna/l2_slice/director_intent_checklist/prev_chapter_tail）与确认未接收项（director_intent/info_acquired/l2_card_full/state_snapshot）。本批次预留格式说明，完整一致性检查在批次2由主会话Step4执行

### 变更

- **输入来源调整**——种子文档已取消（v3.5取消种子文档，功能被L2卡+写作参考吸收），要素切片来源改为：主角引擎→`写作参考/设定/主角引擎.md`、金手指→`写作参考/设定/金手指.md`、活跃线索→从L2卡嵌套子线提取（由主会话注入）
- **红线重构为4条**——
  - ❌1 文风DNA必须 `Get-Content -Raw` 完整加载，禁止 read 工具 limit 截断（合并原❌1 DNA缺失硬阻塞 + 原❌4 limit截断退回重载）
  - ❌2 绝不改写事实信息（人名/动作/事件顺序/对话内容/场景/数值），只改渲染层（原❌3）
  - ❌3 导演意图验证5项必须全部检查（新增，不通过标注偏差不自动退回——退回由主会话决定）
  - ❌4 字数终检 2500-3500字（原字数终检门禁升格为红线）
- **context隔离不再单列为红线**——改由 context manifest/receipt 机制白盒化
- **不接收项明确**——导演意图完整内容/状态快照/info_acquired/L2卡完整内容均不接收（创作决策不回传）

### 保持不变

- 完全重写模式（基于正文骨架+文风DNA重写渲染层）
- 文风DNA终验（7项核心特征逐条对照附正文证据）
- 事实信息一致性检查（6项逐项比对零改写）
- AI观感词清理（≤3种命中）
- 字数终检（2500-3500字）
- 硬约束绝不改写事实信息只改渲染层
