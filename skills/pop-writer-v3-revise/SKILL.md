# pop-writer-v3-revise · 用户指定再创作

revise 不再是每章必经工序。它是**按需重写/渲染工具**。

核心原则：**不要把 revise 变成第二个审核员；它只在用户要求或明确偏差时介入。**

## 触发条件

- 用户明确要求重写、改文风、加强爽点、加快节奏、压缩、扩写。
- create 正文明显偏离运行日志结算。
- 正文使用了未入账重大设定，需要回退修正。
- 文风 DNA 未体现，用户要求更贴近标杆原文。

## 输入

| 输入 | 用途 |
|:-----|:-----|
| 原正文 | 重写对象 |
| 用户修改要求 | 本次 revise 的唯一优先目标 |
| 当前章运行日志 | 确保事实和结算不变 |
| 单元卡当前章导演指令 | 确保情绪目标和章末推力不变 |
| 设定账本 | 防止世界观漂移 |
| 文风 DNA 原文 | 渲染依据 |

## 修订模式

| 模式 | 使用场景 | 允许改动 |
|:-----|:---------|:---------|
| 轻修 | 小范围文风/节奏问题 | 不改事件，只改句段、对白、节奏 |
| 重写 | 用户要求整章重写或正文结构失败 | 保持运行日志结算，重组场景流 |
| 压缩 | 正文拖沓 | 删除渲染冗余，保留事件链 |
| 扩写 | 情绪/动作不足 | 增加场景细节，不新增重大事件 |

## 红线

| 编号 | 红线 |
|:-----|:-----|
| R1 | 不改变运行日志中的本章结算。 |
| R2 | 不新增未入设定账本的重大设定。 |
| R3 | 不把用户没有要求的问题扩展成全面质检清单。 |
| R4 | 文风 DNA 必须完整读取；不能凭抽象风格词重写。 |
| R5 | 如果用户要求会破坏单元终点契约，先提示风险并给替代方案。 |

## 步骤加载

| 步骤 | 文件 | 说明 |
|:-----|:-----|:-----|
| step-1 | `steps/step-1-revise.md` | 按用户目标重写/轻修/压缩/扩写 |

## 输出

- 修订后正文。
- 简短修订记录：采用模式、响应的用户目标、未改变的结算项、是否触碰设定账本。
