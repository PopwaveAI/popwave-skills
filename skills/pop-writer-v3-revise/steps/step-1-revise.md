# Step 1：修订

> 管线: pop-writer-v3-revise v1.2.0
> 类型: **子skill**（由expert-writer主会话涌现写作循环Step 3调度，context隔离）
> 消费: create子skill正文初稿 + 文风DNA + L2卡要素切片（主角引擎+金手指+活跃线索）+ 导演意图验证清单 + 上章末尾 + 修订checklist
> 产出: 重写稿 + 修订记录 + context receipt（预留）
> 红线: ❌1 文风DNA必须Get-Content -Raw完整加载禁止limit截断；❌2 绝不改写事实信息只改渲染层；❌3 导演意图验证5项必须全部检查（不通过标注偏差不自动退回）；❌4 字数终检2500-3500字

## 子skill调度说明

### 输入context（精简，不传会话历史）

|:-------|:-----|:-----|

| L2卡要素切片·金手指 | `写作参考/设定/金手指.md` | 能力+限制+行动驱动（事实一致性比对依据） |

### 导演意图验证清单格式（主会话注入）

主会话从L2卡结构分析表本章行提取导演意图后，将其转换为5项验证标准注入revise（不含导演意图完整内容）：

```yaml
director_intent_checklist:
  # 由主会话从导演意图提取的5项验证标准（不含导演意图本体——创作决策不回传）
  1_narrative_function:
    expected: "首次猎杀+战力跃升"        # 验证标准（导演意图的叙事功能）
    check: "本章是否完成了该叙事功能？"
  2_event_chain_coverage:
    expected_events: ["搜索圈逼近","决定反杀","废弃厂房双杀","系统升级","灵能警告"]
    check: "核心事件是否都出现了？"
  3_emotion_curve_alignment:
    expected_curve: "紧迫→冷酷→满足→警觉"
    check: "是否有该情绪节奏变化？"
  4_sublines_advance:
    expected_sublines: ["①系统成长","③教团渗透","④灵能觉醒","⑤铁钳帮"]
    check: "标注的子线是否都有进展？"
  5_chapter_hook:
    expected_hook: "灵能警告+灰色注释（超自然悬念）"
    check: "是否留下了该章末钩子？"
```

## 目的

修订子skill基于正文初稿+文风DNA完全重写渲染层，并对照导演意图验证清单检查产出。核心逻辑：

- **输入：** 正文初稿（故事事实全部保留不改）+ 文风DNA（渲染引擎完全重写）+ L2卡要素切片（事实一致性比对）+ 导演意图验证清单（5项验证对照）+ 上章末尾（衔接检查）
- **执行：** 完整加载文风DNA（`Get-Content -Raw`禁止limit截断，红线❌1），基于初稿直接重写
- **硬约束❌2：** 绝不改写事实信息（人名/动作/事件顺序/对话内容/场景/数值），只改渲染层（句式/叙事距离/感官密度/对话留白/8020比例）
- **质检职责：** 完全重写+事实信息一致性检查+文风DNA终验+导演意图验证（v3.5新增）+AI观感词清理+字数终检

**v3.5核心改动：** 新增导演意图验证（红线❌3）——重写后对照验证清单检查5项，不通过标注偏差项不自动退回（退回由主会话Step4决定）。种子文档取消，要素切片来源改为写作参考/设定/。

## 做什么

### 1. 文风DNA硬阻塞检查（红线❌1）+ 完整加载

- 加载文风DNA文件 `写作资产/文风库/{书名}.md`
- **加载方式（红线❌1）：** 必须使用 `Get-Content -Encoding UTF8 -Raw` 完整加载，**禁止read工具limit截断**。文风DNA必须全量读入，任何limit截断=硬阻塞终止
- DNA缺失/路径错误/文件空 → **硬阻塞终止**（不降级、不A/B、不空写）
- DNA加载成功 → 建立两层感知：
  - 层A整体风格：句子长短/叙事者距离/战斗节奏/对话留白
  - 层B场景卡：观察句+原文锚定+时间演变

> 文风DNA是项目资产（由pop-writer-v3-seed的step-3蒸馏产出），位于 `写作资产/文风库/`，不进L2卡要素切片。修订层直接从项目资产加载，必须完整加载不可截断。

### 2. 完全重写（基于正文骨架+文风DNA）

基于正文初稿的事实骨架（人名/动作/事件顺序/对话内容/场景/数值），用文风DNA的渲染引擎完全重写渲染层：

| 重写维度 | DNA来源 | 重写标准 |
|:---------|:---------|:----------|
| 句式 | DNA层A句式规则 | 正文句式匹配DNA层A |
| 叙事距离 | DNA层A叙事者距离 | 正文叙事距离匹配DNA（紧贴感官/中等距离/上帝视角） |
| 感官密度 | DNA场景卡观察句 | 正文感官锚点数量达标 |
| 对话留白 | DNA层A对话留白 | 正文对话引导词/心理穿插匹配DNA |
| 8020比例 | DNA层A+场景类型 | 按场景类型比例（行动90/10、情感70/30、对话85/15、发现80/20） |

**硬约束❌2：** 重写时绝不改写事实信息——人名/动作/事件顺序/对话内容/场景/数值必须与初稿一致。只改渲染层，不改故事骨架。

**衔接检查：** 重写稿开头须与上章末尾自洽（情绪/场景/时间线衔接），不自洽则调整重写稿开头渲染层（不改事实信息）。

**门禁：8020超标=退回重写。** 连续2段以上纯内省=退回重写。

### 3. 事实信息一致性检查（6项逐项比对）

完全重写后，将重写稿与正文初稿逐项比对，确保事实信息零改写：

| # | 比对项 | 检查内容 | 标准 |
|:-:|:-------|:---------|:-----|
| 1 | 人名 | 角色名称/称呼是否一致 | 零改写 |
| 2 | 动作 | 角色动作/行为是否一致 | 零改写（渲染可变，动作本身不可变） |
| 3 | 事件顺序 | 事件发生先后顺序是否一致 | 零改写 |
| 4 | 对话内容 | 对话传递的信息/意图是否一致 | 零改写（引导词渲染可变，内容不可变） |
| 5 | 场景 | 场景设置/地点/环境是否一致 | 零改写 |
| 6 | 数值 | 数值/距离/数量/时间等量化信息是否一致 | 零改写 |

**L2卡要素切片一致性比对（补充）：** 重写稿中主角行为须符合 `写作参考/设定/主角引擎.md` 的驱动力+行为准则；金手指使用须在 `写作参考/设定/金手指.md` 的能力+限制内；活跃线索推进须与主会话注入的L2卡活跃线索一致。

**门禁：任一项事实信息被改写=退回重写（红线❌2）。** 6项全部通过才可继续。

### 4. 文风DNA终验（7项核心特征逐条对照附正文证据）

完全重写后，对重写稿做文风DNA终验，7项核心特征逐条对照，每项必须附正文证据：

| # | 核心特征 | 终验内容 | 证据要求 |
|:-:|:---------|:---------|:---------|
| 1 | 场景卡公式 | 正文是否符合DNA场景卡公式（观察句+原文锚定+时间演变） | 引用正文片段 |
| 2 | 数据嵌入格式 | 数值/数据嵌入是否符合DNA格式（戏剧/模糊/精确） | 引用正文片段 |
| 3 | 感官序列 | 感官描写顺序是否符合DNA感官序列锚定 | 引用正文片段 |
| 4 | 段落呼吸 | 段落长短/节奏是否符合DNA段落呼吸规律 | 引用正文片段 |
| 5 | 叙事者距离 | 叙事者距离是否符合DNA层A设定 | 引用正文片段 |
| 6 | 对话引导词 | 对话引导词/留白是否符合DNA层A对话留白 | 引用正文片段 |
| 7 | 8020比例 | 展示/内省比例是否符合场景类型要求 | 引用正文片段 |

**门禁：终验任一项无正文证据=退回重写。** 7项全部附证据才可通过。

### 5. 导演意图验证（5项，v3.5新增，红线❌3）

完全重写后，对照主会话注入的导演意图验证清单，逐项检查重写稿是否完成导演意图定义的5项质量维度。这是 v3.5 的核心质量机制。

> **关键：** revise 只接收验证清单（5项验证标准），不接收导演意图完整内容（创作决策不回传）。验证清单由主会话从导演意图提取。

| # | 验证项 | 检查内容 | 通过标准 | 不通过处理 |
|:-:|:-------|:---------|:---------|:-----------|
| 1 | 叙事功能是否匹配 | 这章是否完成了导演意图定义的叙事功能？ | 重写稿正文确实达成验证清单中的叙事功能 | 标注偏差项 |
| 2 | 事件链是否覆盖 | 导演意图中的核心事件是否都出现了？ | 验证清单中列出的核心事件在正文中全部出现 | 标注缺失事件 |
| 3 | 情绪曲线是否对齐 | 是否有导演意图定义的情绪节奏变化？ | 重写稿情绪节奏与验证清单中的情绪曲线对齐 | 标注偏差段落 |
| 4 | 子线是否推进 | 导演意图中标注的子线是否都有进展？ | 验证清单中标注的子线在正文中均有进展 | 标注未推进子线 |
| 5 | 章末钩子是否有 | 是否留下了导演意图定义的章末钩子？ | 重写稿章末留下验证清单中定义的章末钩子 | 标注钩子缺失 |

**门禁处理（红线❌3，特殊）：**
- 5项全部通过 → 验证通过，继续下一步
- 任一项不通过 → **标注偏差项写入修订记录**（deviation_items），**不自动退回**——退回决策由主会话 Step4 决定
- 全部不通过项汇总为 `intent_verification_result`，供主会话 Step4 对照检查

> 与其他门禁不同：导演意图验证不触发自动退回。revise 完成验证并标注偏差后正常产出重写稿+修订记录，由主会话 Step4 收到 receipt 后决定是否 `intent_mismatch` 退回重写。这是 v3.5 的设计——渲染层只负责验证标注，退回决策权在主会话。

### 6. AI观感词清理

逐段扫描，命中即修正：

| 检查项 | 命中标准 | 修正方式 |
|:-------|:---------|:---------|
| 心理动词 | "他感到/他意识到/他仿佛" | 改为展示（动作/感官/对白） |
| 对比句式 | "不是…而是…" ≥ 2次 | 减少到≤1次或重写 |
| 解说员句式 | 叙事者解释设定 | 改为角色对话/行动展示 |
| 视角越界 | 限知视角写非锚点角色内心 | 删除越界段落 |

**门禁：AI观感词命中>3种=退回重写。**

### 7. 字数终检（红线❌4）

完全重写后对重写稿做字数终检：

| 检查项 | 标准 | 失败动作 |
|:-------|:-----|:---------|
| 字数区间 | 2500-3500字 | <2500字=退回create子skill（事实信息不足，故事层问题）；>3500字=强制精简渲染层（保留事实信息） |
| 事件密度 | ≤15个事件/章 | 事件密度>15=标记过载，提示后续章节放缓 |

**字数终检通过区间：2500-3500字（红线❌4）。**

### 8. context receipt 产出（预留接口）

按 `SKILL.md` 中预留的 context receipt 格式，确认实际接收的注入项（draft/style_dna/l2_slice/director_intent_checklist/prev_chapter_tail）与确认未接收项（director_intent/info_acquired/l2_card_full/state_snapshot）。本批次先产出 receipt 格式，完整一致性检查在批次2由主会话Step4执行。

```yaml
  context_receipt:                          # 预留接口，批次2完整实现
    received: [draft, style_dna, l2_slice, director_intent_checklist, prev_chapter_tail]
    not_received: [director_intent, info_acquired, l2_card_full, state_snapshot]
    anomalies: []
  degraded_master_execution: false
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| **文风DNA未完整加载/limit截断（红线❌1）** | **硬阻塞终止** |
| **重写时改写事实信息（红线❌2）** | **退回重写** |
| **导演意图验证5项（红线❌3，特殊）** | **标注偏差项写入修订记录，不自动退回——退回由主会话Step4决定** |
| **字数终检不在2500-3500字区间（红线❌4）** | **<2500退回create；>3500强制精简渲染层** |
| **AI观感词命中>3种** | **退回重写** |
| **8020比例超标** | **退回重写** |
| **事实信息一致性6项任一不通过** | **退回重写** |
| **文风DNA终验7项任一无证据** | **退回重写** |

> 注意：导演意图验证（红线❌3）是唯一不触发自动退回的红线——revise 完成验证并标注偏差后正常产出，退回决策权在主会话Step4。
