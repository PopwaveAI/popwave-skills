# Step 1：按需修订/重写

> revise 只响应用户目标或明确偏差，不做默认全量质检。

## 消费

- 原正文
- 用户修改要求
- 当前章运行日志
- 当前章导演指令
- 设定账本
- 文风 DNA

## 1. 判定修订模式

| 模式 | 条件 |
|:-----|:-----|
| 轻修 | 用户只要求语言、节奏、局部爽点 |
| 重写 | 用户要求重写整章，或正文结构明显失败 |
| 压缩 | 用户认为拖沓、解释过多 |
| 扩写 | 用户认为场景、动作、情绪不足 |

## 2. 锁定不可改写项

从运行日志和设定账本列出：

- 事件链。
- 本章结算。
- 章末推力。
- 禁止漂移设定。

如果用户要求改变这些内容，先提示会影响 plot/arc，建议退回 plot 重跑。

## 3. 执行修订

### 轻修

只改句段、对白节奏、描写顺序，不改事件。

### 重写

重组场景流，但保持运行日志结算不变。

### 压缩

删除解释、重复心理、无效对白，保留动作链和反馈。

### 扩写

补足动作、场景、情绪和对白，但不新增重大事件或设定。

## 4. 自检

- [ ] 响应了用户明确要求。
- [ ] 未改变运行日志结算。
- [ ] 未新增未入账重大设定。
- [ ] 未越过章末推力。
- [ ] 文风 DNA 已完整读取并体现。

## 5. 输出

- 修订后正文。
- 简短记录：

```yaml
revise_record:
  mode: light|rewrite|compress|expand
  user_goal: ""
  unchanged_settlement:
    - ""
  setting_ledger_touched:
    - ""
  risk: none
```
