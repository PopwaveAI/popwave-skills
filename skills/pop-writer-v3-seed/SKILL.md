# Pop Writer V3 Seed

seed 负责搭建可生长的写作底盘。它不写幕纲，不预设所有剧情线，只产出后续 plot 能碰撞出事件的硬规则、角色驱动和初始种子。

seed 的第一责任不是“设定丰富”，而是证明这本书能按网文方式启动：第一章有压力、主角有操作、金手指有反馈、读者有可数收益、章末有追读。任何设定如果不能服务这个启动闭环，只能进入“待孵化”，不得冻结为硬设定。

## 第一性原则

- 网文先于百科：先回答读者为什么追，再回答世界为什么成立。
- 行动先于解释：设定必须改变主角行动路线，不能只提供背景说明。
- 收益先于氛围：核心爽点必须外显为面板、数字、资源、身份、敌人反应、旁人反应或局势翻转。
- 约束先于能力：金手指的限制和代价必须制造下一步行动，而不是把问题抹平。
- 证据先于灵感：用户提到参考书、library 存在相关资料时，必须读取并输出证据绑定；没有证据只能标为“模型推断”。
- 开局先于全书：seed 只冻结第一卷/第一幕必须遵守的硬规则，不设计全世界百科。

## 核心产物

- 网文立项契约：赛道、读者承诺、主味型、第一章追读承诺、禁区。
- 故事DNA：这本书的读者承诺、主味型、主角爽感。
- 世界宪法：不可随意改动的底层规则和禁区。
- 主角引擎：欲望、底线、恐惧、行动偏好、典型选择。
- 金手指规则：触发、反馈、限制、代价、可视化收益。
- 第一章可爽性验证：压力、目标、主角第一操作、第一次反馈、第一笔收益/损失、章末钩子。
- 设定生长语法：后续新设定何时允许出生，何时必须冻结。
- 初始种子池：压力、资源、敌人、缺口、异常信号。
- 文风DNA：语感、节奏、信息密度、视角和爽感表达。
- Library 查询方向：本书后续应优先查哪些任务标签、设定库书目、剧情库案例、金手指库类型。
- 跨媒介母版雷达：本书后续应主动参考哪些影视、美剧、漫画、游戏、电影的类型机制，用于 plot 的外部事件源。

## 默认增量调研

seed 不能只等用户给参考。设计底盘时必须主动建立“参考雷达”，但不能把参考雷达写成剧情大纲。

用户明确提到参考书、作品、类型母版时，seed 必须先澄清它是“参考作品”还是“本书设定名”。若项目 library 有相关资料，必须先读资料再设计。不要把模型对同名实体的常识当成参考书内容。

### Library 调研

路径：`$env:APPDATA\popwave\remote-skills\pop-trope-library`

| 子库 | 用途 |
|:-----|:-----|
| `设定库/` | 同赛道书的 L1 六件套设定（世界蓝图/力量体系/历史/物种/势力/资源），设计世界观时先查再建 |
| `金手指库/` | 按类型分类的金手指设计卡，设计金手指时先查再设计 |
| `立项库/` | 同类域焊接经验，立项时参考 |

查库方式：先列出 `$env:APPDATA\popwave\remote-skills\pop-trope-library` 下的 `设定库/`、`金手指库/`、`立项库/` 可用条目，再按用户关键词和赛道选择 2-5 份精读。输出时必须给出“证据绑定清单”：哪些结论来自哪份资料，哪些是模型推断，哪些仍需用户确认。

### 跨媒介母版调研

每次 seed 至少主动提出 5-12 个跨媒介类型母版候选，覆盖影视、美剧、漫画、游戏、电影中的强类型机制。若当前运行环境有搜索工具，可围绕“当前赛道 + 类型机制”做简短检索；若不能联网或无搜索工具，则用模型已知的经典母版，并声明“未联网验证”。

不要照搬外部作品内容，只抽取机制：

- 倒计时救援。
- 复仇清单。
- 追踪链。
- 密室/封闭空间压迫。
- 逃杀/生存赛制。
- 潜入/抢劫准备链。
- 怪物逐级显影。
- 队伍职业互补。
- 规则漏洞破解。
- 据点经营与防守升级。

输出到 `library/设定账本/跨媒介母版雷达.md`，供 plot 每幕主动筛选，不需要用户提醒。

## 执行步骤

1. `steps/step-0-creative-co-creation.md`：和用户共创核心卖点。
2. `steps/step-1-ecology-to-pressure.md`：把世界生态转成压力系统。
3. `steps/step-2-golden-finger.md`：设计金手指行动引擎。
4. `steps/step-3-dna-lock.md`：锁定文风DNA。
5. `steps/step-4-setting-output.md`：输出设定底盘。
6. `steps/step-5-maintenance.md`：维护设定账本和索引。

需要金手指细化时读 `references/金手指设计指南.md`。

## 红线

- 不把设定补成百科全书。
- 不提前写完整剧情线。
- 不让金手指无代价解决所有问题。
- 不让软设定绕开硬规则。
- 不允许把 seed 产物写到 `卷纲/运行/` 当成卷纲过程文件。seed 的正式产物必须进入 `library/设定账本/`、`library/文风DNA/`，过程文件进入 `library/设定账本/运行/`。
- 不允许只输出“压力矩阵/金手指设计”就宣称底盘完成。必须有 `故事DNA.md`、`世界宪法.md`、`主角引擎.md`、`金手指.md`、`初始种子池.md`、`Library查询方向.md`。
- 不允许金手指只像设定说明书。必须通过第一章可爽性验证：触发场景、主角第一主动操作、反馈形式、第一笔收益/损失、下章新压力。
- 不允许用“轻松爽文”当口号。必须写明幽默/吐槽/打脸/资源到账/面板反馈分别在哪里出现。
- 不允许在没有 library 证据绑定时说“参考了某书”。只能说“按用户描述推断”。
- 每个产物都要能服务 plot 的状态契约和种子碰撞。
- seed 不替 plot 消化案例，但必须给出可供 plot 检索公共库的任务标签、赛道判断和跨媒介母版雷达。
