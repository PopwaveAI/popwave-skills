# 种子搜索法 SOP

> 适用: pop-writer-v3-seed Step 1
> 定位: 制度性压迫维度调研标准流程，从 creative 生态调研扩展而来

## 一、为什么需要种子搜索法

涌现式管线的核心是"压力矩阵驱动剧情"。压力矩阵不能是大模型空想——必须是真实的制度性压迫维度，来自目标社会/世界的真实制度分析。种子搜索法提供系统化的调研流程。

## 二、调研目标

不是调研"世界观设定"，是调研"人吃人的制度"。每个制度分析"不参与=什么后果"，转化为压力源。

## 三、调研流程

### 阶段1：制度识别
扫描目标社会/世界的核心制度维度：

| 维度 | 调研问题 | 信息源 |
|:-----|:---------|:-------|
| 医疗 | 无医保会怎样？药价多少？ | WebSearch + library设定库 |
| 法律 | 破产/离婚/债务的法律后果？ | WebSearch |
| 经济 | 高利贷/信用卡/催债的运作方式？ | WebSearch |
| 暴力 | 帮派/黑帮如何控制街区？ | WebSearch + library设定库 |
| 宗教 | 邪教/地下组织如何运作？ | WebSearch + library设定库 |
| 身份 | 无身份/流浪汉如何被系统追踪？ | WebSearch |
| 职业 | 非法职业（走私/收尸/地下医疗）的生存逻辑？ | WebSearch |
| 超自然 | 世界中的超自然威胁如何运作？ | 用户设定 + library |

### 阶段2：压力源转化
每个制度转化为压力源：

| 转化步骤 | 问题 |
|:---------|:-----|
| 机制 | 这个制度怎么运作？ |
| 杀人方式 | 不参与/被盯上=什么后果？ |
| 显性表现 | 读者看到什么具体场景？ |
| 活跃阶段 | 这个压力源在什么章节显性？ |
| 来源标注 | 这个压力源来自哪个调研结果？ |

### 阶段3：交叉困境识别
多个压力源交叉时产生道德困境：

| 交叉组合 | 产生的困境 |
|:---------|:-----------|
| 医疗缺失 × 帮派控制 | 受伤不能去医院，帮派在搜人 |
| 保护别人 × 身份暴露 | 保护身边人导致身份暴露 |
| 催债 × 金手指搜刮 | 需要钱还债但搜刮暴露能力 |

## 四、调研质量检查

| 检查项 | 合格标准 |
|:-------|:---------|
| 压力源数量 | ≥3个（红线❌3） |
| 来源标注 | 每个压力源有来源（WebSearch/library/用户） |
| 交叉困境 | ≥3个交叉困境 |
| 显性表现 | 每个压力源有具体的读者可见场景 |

## 五、与 creative 生态调研的关系

种子搜索法从 creative 的"角色群生态调研"扩展而来：

| creative 生态调研 | v3-seed 种子搜索法 |
|:-----------------|:-------------------|
| 调研角色群生存方式 | 调研制度性压迫维度 |
| 产出生态图谱 | 产出压力矩阵+冲突轴 |
| 角色群5字段 | 压力源6字段（含来源标注） |
| 跨群冲突交叉点 | 压力源交叉困境 |

生态图谱不单独存盘，转化为压力矩阵+冲突轴压缩进种子。
