# Step 0：创意共创

> 管线: pop-writer-v3-seed v1.4.0
> 消费: 用户初始创意元素 / library立项库 / WebSearch赛道参考
> 产出: 创意共识单（核心卖点+元素清单+创作方向）→ 引导step-1~4的六要素构建
> 性质: 对话密集型——用户不一定有完整idea，需要agent引导共创

## 目的

用户开始写书时，往往只有一个模糊的想法（"我想写一个CIA特工在美国底层生存的故事"），还没有完整的创作体系。直接跳到生态调研是本末倒置——先要想清楚"这本书卖什么"，再做调研。

本步骤通过元素融合+核心卖点提炼+用户QA，在调研之前建立创作共识。六要素不是agent自己产出的，是和用户逐步共创出来的。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | 用户的初始创意描述 | 用户可能只有一句话，也可能有详细设想 |
| 2 | library 立项库/00-索引 | 同类书型参考，判断赛道 |
| 3 | library 立项库/元爽点-变体映射表 | 爽点参考，帮助识别核心卖点 |
| 4 | WebSearch（如需要） | 赛道热度/同类作品/读者偏好参考 |

## 做什么

### 1. 用户初始元素收集

询问用户的初始创意元素。用户可能提供的元素类型：

| 元素类型 | 示例 | 用户可能说 |
|:---------|:-----|:-----------|
| 主角设想 | 背景/性格/能力 | "主角是退役CIA特工" |
| 世界观片段 | 时代/地点/规则 | "现代美国底层" |
| 金手指构想 | 系统/能力/外挂 | "有个系统面板" |
| 情绪诉求 | 爽感/情绪/体验 | "想看底层逆袭打脸" |
| 对标作品 | 参考书/风格 | "像《我在北美搞内战》" |

**用户说得越多越好，但哪怕只有一句话也要开始共创。** 不要求用户提供完整设定。

### 2. 元素融合（agent主动补充）

在用户提供元素基础上，agent**主动补充三类元素供用户判断**：

| 补充类型 | 做什么 | 为什么 | 示例 |
|:---------|:-----|:-------|:-----|
| **相似元素** | 提供与用户元素相近的变体 | 拓宽选择面，用户可能没意识到更多可能 | 用户说"CIA特工"→补充"退役特工/叛逃特工/卧底特工/被陷害的特工" |
| **相反元素** | 提供与用户元素对立的选项 | 制造张力，对立面往往是最好的冲突来源 | 用户说"底层生存"→补充"上层堕落/中产破产/富豪隐居" |
| **盲区元素** | 提供用户完全没提到但相关的元素 | 补充用户视野盲区，避免同质化 | 用户说"系统面板"→补充"系统是敌对的/系统是碎片化的/系统需要献祭" |

每类补充2-3个选项，用表格呈现给用户：

```markdown
## 元素融合表

### 你提供的元素
| 元素类型 | 你的设想 |
|:---------|:---------|
| 主角 | 退役CIA特工 |
| 世界观 | 现代美国底层 |
| 金手指 | 系统面板 |
| 情绪诉求 | 底层逆袭打脸 |

### 相似元素（拓展你的选择）
| 你的元素 | 相似变体 | 选择？ |
|:---------|:---------|:------|
| 退役CIA特工 | 叛逃特工（被追杀，更紧张） | ☐ |
| 退役CIA特工 | 卧底暴露特工（有身份危机） | ☐ |
| 系统面板 | 碎片化系统（需收集，有成长线） | ☐ |

### 相反元素（制造张力）
| 你的元素 | 对立选项 | 张力来源 | 选择？ |
|:---------|:---------|:---------|:------|
| 底层生存 | 上层堕落（富二代破产） | 从高到低的落差感 | ☐ |
| 系统面板 | 系统是敌对的（系统要杀他） | 金手指本身就是威胁 | ☐ |

### 盲区元素（你可能没想到）
| 元素 | 为什么值得考虑 | 选择？ |
|:-----|:---------------|:------|
| 主角有道德瑕疵（曾做过脏活） | 增加行为准则的复杂性和道德张力 | ☐ |
| 系统与CIA有关联（CIA的实验产物） | 连接主角背景和金手指，避免两张皮 | ☐ |
```

**用户逐项选择后，形成融合元素清单。** 用户也可以全部拒绝或自行补充。

### 3. 核心卖点提炼

从融合元素清单中提炼本书核心卖点。核心卖点回答"读者为什么要看这本书"。

| 提炼维度 | 问题 | 示例 |
|:---------|:-----|:-----|
| 核心爽点 | 读者最期待什么体验？ | CIA精锐用战术碾压底层混混的反差爽感 |
| 差异化 | 和同类作品比有什么不同？ | 不是重生/穿越，是真实技能+系统面板的混合 |
| 情绪承诺 | 读者读完会 feeling 什么？ | 从绝望到掌控的逆袭快感 |

核心卖点用一句话+一段话表述：

```
核心卖点（一句话）：CIA黑色行动精锐沦落美国底层，用战术碾压+系统面板在制度性压迫中杀出血路。
核心卖点（展开）：主角不是穿越者开挂，是真实能力的降维打击+系统的成长线，在"不行动就死"的压力矩阵中，读者获得"绝境反杀"的即时爽感。
```

**核心卖点必须经用户确认。** 如果用户不满意，回到步骤2调整元素融合。

### 4. 创作方向共识

基于核心卖点，与用户达成创作方向共识——这直接引导后续六要素的构建：

| 方向维度 | 共识内容 | 引导哪个要素 |
|:---------|:---------|:-------------|
| 压力来源方向 | 什么样的制度在压迫主角？ | → step-1 压力矩阵 |
| 主角行为基调 | 主角是什么样的人？怎么行动？ | → step-4 主角引擎+行为准则 |
| 金手指定位 | 系统在故事中扮演什么角色？ | → step-2 金手指 |
| 冲突主轴 | 核心矛盾是什么？ | → step-1 冲突轴 |
| 文风方向 | 什么文风适合这个故事？ | → step-3 文风DNA |
| 成长终点 | 主角最终要去哪？ | → step-4 目的地 |

创作方向共识**不定义具体内容**（那是后续步骤的事），只定义方向。例如："压力来源方向：美国底层的制度性压迫（医疗/法律/经济/暴力）"——具体压力矩阵在step-1调研后构建。

### 5. 用户QA确认

将创意共识单呈现给用户做最终确认：

```markdown
## 创意共识单

### 核心卖点
{一句话+展开}

### 融合元素清单
| 元素类型 | 最终选择 | 来源 |
|:---------|:---------|:-----|
| 主角 | {选择} | 用户/相似/相反/盲区 |
| 世界观 | {选择} | ... |
| 金手指 | {选择} | ... |
| 情绪诉求 | {选择} | ... |

### 创作方向
| 方向维度 | 共识 |
|:---------|:-----|
| 压力来源 | {方向} |
| 主角行为 | {方向} |
| 金手指定位 | {方向} |
| 冲突主轴 | {方向} |
| 文风方向 | {方向} |
| 成长终点 | {方向} |

---
✅ 用户确认以上创意共识，进入step-1生态调研→压力矩阵构建
❌ 用户要求调整 → 回到步骤2/3修改
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 用户初始元素已收集（至少1个元素） | 继续引导用户提供更多 |
| 三类补充元素各≥2个选项 | 补充至≥2个 |
| 核心卖点已提炼（一句话+展开） | 退回重新提炼 |
| 创作方向6维度全部填写 | 补全缺失维度 |
| **用户已确认创意共识单** | **等待用户确认后再进step-1** |

## 产出

创意共识单（不独立存盘，作为step-1~4的输入上下文），包含：
- 核心卖点（一句话+展开）
- 融合元素清单（用户选择+来源标注）
- 创作方向共识（6维度）
