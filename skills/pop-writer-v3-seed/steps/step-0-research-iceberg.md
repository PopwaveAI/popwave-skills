# Step 0 - Research Iceberg

目的：先建立“水下八分之七”，再允许立项。此步不写 PRD，不设计金手指，不写幕纲。

## 输出路径

```text
library/设定账本/研究档案/00-研究索引.md
library/设定账本/研究档案/01-种子展开图.md
library/设定账本/研究档案/02-外部母版卡.md
library/设定账本/研究档案/03-多元素整合记录.md
library/设定账本/研究档案/04-复刻路由表.md
library/设定账本/运行/00a-输入消歧与证据绑定.md
```

## 1. 输入消歧

用户提到书名、角色、体系、桥段时，先判定它属于哪类：

| 提法 | 类型 | 处理 |
| --- | --- | --- |
| 参考作品 | 查 library/外部资料，先判断是元素来源还是复刻对象；若有复刻/嫁接意图，先进入 `04-复刻路由表.md` |
| 本书设定名 | 作为项目内命名候选，不外推原作设定 |
| 类型母版 | 抽取倒计时、追踪链、赛制、逃杀、复仇等机制 |
| 用户原创设定 | 进入种子池，等待碰撞 |
| 歧义项 | 明确列出两种以上解释，不自作主张 |

若用户说“参考某书角色”，但 library 没有该书资料，不得假装熟悉。写明“证据不足，只能按用户描述和模型常识推断”。

用户明确给出参考书、多个题材元素或多个母版时，必须调用公共 `expert-writer/references/多元素整合范式.md`。参考书只是多元素输入的一种，禁止只写“DND体系/阶级冲突/系统流”这类标签后直接进入概念 PK。

用户表达“像/照着/复刻/高保真/一比一/拿某体系/嫁接某世界观”等意图时，必须先调用 `expert-writer/references/复刻协议.md`，把复刻深度和复刻位置拆清楚，再进入多元素整合。

## 2. 最小用户确认

最多问 3 个问题，且只问会改变书的骨架的问题：

- 这本书的读者预期更偏哪类：起点男频、番茄快爽、轻小说、女频、严肃向？
- 全书大致体量：长篇 500 章+、中长篇 200-400 章、短篇实验？
- 用户最不想丢的核心种子是什么？

用户已给足信息时不要问，直接执行。

## 4. 复刻路由表（有复刻意图时）

当用户要求参考、复刻、嫁接或高保真保留某作品表层骨架时，先输出 `04-复刻路由表.md`。复刻路由表只回答“复刻到什么程度、交给哪个阶段”，不做元素碰撞。

| 参考对象 | 复刻深度 | 复刻位置 | 保留什么 | 改造什么 | 交给哪个阶段 | 当前 seed 只做什么 |
| --- | --- | --- | --- | --- | --- | --- |
|  | L0/L1/L2/L3/L4 | seed/PRD/world/plot/create-revise |  |  |  |  |

seed 阶段只能锁定立项意图、PRD 约束和下游路由。禁止在这里直接产出 L1 世界包、金手指细节、完整章节链或正文语感方案。

## 5. 多元素整合记录（深度思考硬门）

当输入包含多个种子、参考书、题材、母版、金手指方向或现实机制时，必须按 `expert-writer/references/多元素整合范式.md` 输出整合记录。

若用户给出 1 本以上参考书，参考书案例消化是 Step 1“元素抽取”和 Step 3“机制拆解”的证据来源，必须额外读取与参考书对应的剧情库、任务索引、金手指库、设定库或文风库材料，至少覆盖：

- 每本参考书至少 1 张 L2 或 L3 剧情卡；若无剧情卡，读取最接近的任务索引或拆书 PRD 并标注缺口。
- 开局/首次收益/第一次身份跃迁/核心金手指验证等与本书种子最相关的案例。
- 参考书的“叙事语法”，不是只摘设定名词。

输出 `03-多元素整合记录.md`。如本轮输入包含参考书，可在同一文件内附“参考书机制拆解”小节，但不得在本文件里判定 L1-L4 复刻深度；复刻深度与位置只写入 `04-复刻路由表.md`。最小格式：

1. 元素抽取。
2. 相似/相反/盲区发散。
3. 机制转译。
4. 交叉碰撞。
5. 候选竞技与反证。

## 6. 外部母版扫描

不等用户提醒，主动扫描至少 3 个外部母版，来源可以是影视、美剧、漫画、游戏、电影、真实行业机制。

| 母版ID | 母版 | 类型承诺 | 可迁移机制 | 改造项 | 适合度 |
| --- | --- | --- | --- | --- | --- |
| M-01 |  | 倒计时/追踪/赛制/经营/复仇/潜入 |  |  | 高/中/低 |

默认只抽机制，不搬角色、台词、专有设定；但用户明确要求表层嫁接、高保真剧情复刻或同人/二创时，按 `复刻协议` 写入 `04-复刻路由表.md`，不得用“不能搬表层”否决用户意图。

## 7. 种子展开

从用户输入和证据中抽 5 类种子：

| 种子类型 | 例子 | 展开方式 |
| --- | --- | --- |
| 主角种子 | 身份、弱势、欲望 | 起点/终点/行动偏好 |
| 世界种子 | 核心异常、社会结构 | 规则机器/资源/势力 |
| 爽点种子 | 升级、打脸、经营、救赎 | 可见反馈/重复兑现 |
| 类型种子 | D&D、末世、都市邪窟、诸天 | 类型规则/读者预期 |
| 反差种子 | 弱者掌权、邪力守家 | 冲突张力/审美新鲜感 |

每个种子至少生成：

- 相似展开：同类网文怎么做。
- 相反展开：反过来怎么做会新鲜。
- 盲区展开：常规作品容易忽略什么。
- 商业展开：如何让读者每 3-5 章获得反馈。

## 质量门

通过后才能进入 Step 1：

- 有证据绑定表，或明确库缺/模型推断。
- 多元素输入时，有 `03-多元素整合记录.md`，且至少包含“元素抽取/相似相反盲区/机制转译/交叉碰撞/候选反证”。用户给参考书时，记录内必须包含参考书案例消化证据。
- 复刻/嫁接输入时，有 `04-复刻路由表.md`，且明确复刻深度、复刻位置和阶段分工。
- 有至少 3 个外部母版候选。
- 有种子展开图，不只是用户原话复述。
- 没有提前锁定唯一故事。
