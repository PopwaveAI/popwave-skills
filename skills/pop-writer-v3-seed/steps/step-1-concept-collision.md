# Step 1 - Concept Collision

目的：把研究档案变成 2-3 个可选的新书概念，而不是直接锁定 agent 第一灵感。

## 输出路径

```text
library/设定账本/运行/00-故事概念候选PK.md
```

## 1. 碰撞素材

读取 Step 0 的研究档案，至少提取：

- 3-5 个核心种子。
- 3 个 library 证据。
- 用户给参考书时，必须提取 `03-参考书案例消化摘要.md` 中至少 3 条“叙事语法/爽点兑现链/原创焊点”。
- 1-2 个外部母版机制。
- 1 个用户不可牺牲项。
- 1 个禁区或风险。

## 2. 碰撞模式

至少使用 3 种模式生成候选：

| 模式 | 用法 |
| --- | --- |
| 颠覆重构 | 把常规类型的强者视角改成弱者/底层/反派/旁门视角 |
| 类型嫁接 | 用外部母版的节奏机制嫁接网文爽点 |
| 三域融合 | 主角种子 + 世界种子 + 金手指机制三者强耦合 |
| 功能反转 | 把危险/诅咒/债务变成主角收益来源 |
| 叙事语法移植 | 把电影追踪链、游戏升级树、漫画组织战移成连载结构 |

合格碰撞不是拼贴。移除任一核心域后，故事卖点应明显塌陷；否则只是元素堆叠。

参考书参与碰撞时，必须先写“机制转译句”：

```markdown
参考书机制：{案例ID} 的有效结构是……
本项目转译：改成……
原创焊点：与原书根本不同的是……
```

如果只能写出“借鉴某某体系/某某氛围”，说明案例消化失败，退回 Step 0。

## 3. 候选概念

生成 2-3 个互斥候选，每个按同等颗粒度输出：

| 字段 | 要求 |
| --- | --- |
| 一句话卖点 | 读者为什么点进来 |
| 主角起点 | 第一章具体弱势 |
| 第一卷终点 | 卷末可感知变化 |
| 全书终点 | 长篇终局期待 |
| 世界机器 | 世界如何持续制造压力/资源/身份 |
| 元爽点 | 1-3 个，可见反馈方式 |
| 金手指方向 | 只写方向，不锁机制细节 |
| 第一章承诺 | 开局能看到什么爽感苗头 |
| 风险 | 最可能写崩在哪里 |
| 证据 | S/G/P/M 证据 ID |
| 参考书机制转译 | 案例ID + 机制转译句 + 原创焊点 |

## 4. PK 维度

| 候选 | 网文爽点密度 | 长篇生长 | 主角主动性 | 世界新鲜感 | 金手指空间 | 正文可写性 | 风险 |
| --- | --- | --- | --- | --- | --- | --- | --- |

必须允许任一候选失败。不得先选 A 再让 B/C 陪跑。

## 5. 用户断点

除非用户明确要求“自动继续”，Step 1 后必须停下：

```markdown
## 待用户确认

推荐进入 PRD 的候选：
推荐理由：
可混入的败者强点：
不可混入项：

请确认：选择 A/B/C，或指定混合方式。
```

用户说“你定”时，选择最高分方案，但必须把败者强点吸收/淘汰理由写清楚。

## 质量门

- 候选之间至少在主角起点、世界机器、元爽点、金手指方向、第一卷终点中 3 项不同。
- 每个候选都有证据 ID，不是空想。
- 用户给参考书时，每个候选必须至少绑定 1 条案例机制转译，且不能只复用“DND/系统/阶级”标签。
- 推荐方案不是“元素最多”，而是最能长期连载、最能持续爽。
