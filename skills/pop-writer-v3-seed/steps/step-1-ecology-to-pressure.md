# Step 1：委托pop-research调研→压力矩阵+冲突轴

> 管线: pop-writer-v3-seed v1.6.0
> 消费: Step 0创意共识单（压力来源方向+冲突主轴方向+核心卖点） / library设定库+套路库+剧情库 / pop-research调研产出
> 产出: `写作参考/设定/压力矩阵全量.md`（pop-research产出） + `写作参考/设定/冲突轴全量.md`（seed设计） + `写作参考/知识沉淀/`研究档案（pop-research产出）

## v3.5 变更说明

v3.5将调研工作委托给独立的pop-research skill。seed不再内嵌种子搜索法SOP（三界压力框架已提取为pop-research的内部SOP）。step-1的职责变为：
1. 调用pop-research传入书型+核心卖点，获取压力矩阵全量+研究档案
2. 基于调研结果设计冲突轴全量（四层生态+活跃线索）
3. 用户QA确认

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | Step 0产出的创意共识单（上下文） | 压力来源方向+冲突主轴方向+核心卖点——传给pop-research作为调研输入 |
| 2 | library 设定库/{对标书}/ | 对标书世界观设计、压力源参考 |

## 做什么

### 1. 委托pop-research调研（v3.5核心变更）

调用pop-research skill，传入调研需求：

```
调研委托参数：
  - 书型：{从Step 0创意共识单获取}
  - 核心卖点：{从Step 0创意共识单获取}
  - 压力来源方向：{从Step 0创意共识单获取}
  - 期望深度：种子级调研（三界框架+分界调研+交叉困境）
```

pop-research执行流程（seed不内嵌，仅调用）：
1. 三界框架定界（社会界/自然界/超自然界）
2. 分界调研（社会界6维度/自然界6维度/超自然界5维度）
3. 压力源转化（6字段结构化）
4. 交叉困境识别（6种交叉类型）
5. 质量检查（≥3个压力源、≥3个交叉困境、含具体案例）
6. 存盘到 `写作参考/知识沉淀/`（种子展开图+交叉困境分析）
7. 更新 `写作参考/索引.md`
8. 返回路径指针给seed

**pop-research产出：**
- `写作参考/设定/压力矩阵全量.md`（三界压力源+交叉困境，6字段结构化）
- `写作参考/知识沉淀/种子展开图-{书名}.md`（三界压力调研详情）
- `写作参考/知识沉淀/交叉困境分析-{书名}.md`（交叉困境详细分析，含同界/跨界标注）

> 注：pop-research是独立skill（尚未创建）。seed在step-1中预留调用接口，传入书型+核心卖点。pop-research创建后将按上述流程执行调研。

### 2. 压力矩阵确认

读取pop-research产出的 `写作参考/设定/压力矩阵全量.md`，确认质量达标：

| 检查项 | 合格标准 |
|:-------|:---------|
| 压力源数量 | ≥3个（红线相关） |
| 界属标注 | 每个压力源标注所属界 |
| 来源标注 | 每个压力源有来源（调研/library/用户） |
| 交叉困境 | ≥3个交叉困境（至少1个跨界） |
| 显性表现 | 每个压力源有具体的读者可见场景 |

如不达标，退回pop-research补充调研。

### 3. 冲突轴+活跃线索设计（seed职责）

基于pop-research调研结果，设计冲突轴全量。冲突轴是创意设计工作，不属于调研范畴：

#### 3a. 四层生态结构

从生态图谱转化为冲突轴：

| 生态层 | 构成 | 产生的危机类型 |
|:-------|:-----|:-------------|
| 帮派层 | 暴力组织 | 暴力威胁、地盘争端 |
| 流浪者层 | 底层生存者 | 信息/资源/灰色经济 |
| 系统层 | 警察/催债/社工 | 规则压制、身份追踪 |
| 地下层 | 超自然/邪教 | 隐藏威胁、系统真相 |

每层独立产生危机，多层交叉产生困境。

> 注：四层生态是通用框架。根据书型调整——如玄幻小说可能改为"宗门层/散修层/天道层/远古层"。

#### 3b. 活跃线索初设

3-5条当前活跃的张力线：

```
- 线索ID: T001
  描述: {一句话张力线}
  状态: 酝酿
  涉及生态层: {层}×{层}
```

### 4. 冲突轴全量落盘

将冲突轴全量写入 `写作参考/设定/冲突轴全量.md`：

- 四层生态结构
- 活跃线索清单（3-5条）
- 与压力矩阵的交叉关系标注

### 5. 用户QA确认

将压力矩阵+冲突轴全量呈现给用户确认：

```markdown
## 压力矩阵+冲突轴确认

### 压力矩阵（pop-research产出）
{压力源表格}

### 交叉困境（pop-research产出）
{交叉困境列表}

### 冲突轴（seed设计）
{四层生态+活跃线索}

---
✅ 用户确认，进入step-2金手指设计
❌ 用户要求调整 → 退回pop-research补充/调整冲突轴
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| pop-research已调用并返回产出 | 重新调用 |
| 压力矩阵全量.md存在且≥3压力源 | 退回pop-research补充 |
| 每个压力源有来源标注 | 补充来源 |
| 冲突轴≥3层生态 | 补充生态层 |
| ≥3条活跃线索 | 补充线索 |
| **用户已确认压力矩阵+冲突轴** | **等待用户确认后再进step-2** |

4. `写作参考/设定/冲突轴全量.md`（四层生态+活跃线索）
