# Step 2 - PRD Lock

目的：把胜出概念锁成全书立项宪法。PRD 是下游 world/plot/create 的裁决依据，不是宣传稿。

## 输出路径

```text
library/设定账本/新书立项PRD.md
library/设定账本/运行/01-PRD锁定记录.md
```

## 必读模板

写 PRD 前读取 `templates/新书立项PRD-模板.md`。

## PRD 必填块

1. 故事摘要：150-250 字，说明主角、世界异常、核心爽点、长篇方向。
2. 尺度规划：全书章数、卷数、第一卷章数、每卷幕数、每幕章数、单章字数。
3. 赛道与读者：男频/女频/起点/番茄/轻小说等，读者追读理由。
4. 主角起点/第一卷终点/全书终点。
5. 元爽点：1-3 个，每个必须有可见反馈和重复兑现方式。
6. 世界机器：世界如何持续制造资源、压力、身份、势力、升级。
7. 核心卖点：至少 5 条“别人怎么做 -> 本书怎么做 -> 为什么更对”。
8. 世界宪法草案：4-6 条硬规则，写清违反后果。
9. 不可妥协 DNA：必须保留的设定、审美、爽点、关系。
10. 禁区：不能提前兑现、不能解释、不能短篇化解决的内容。
11. 参考证据：library/外部母版分别学什么、不学什么。
12. 下游交接：world 必须建哪些 L1，plot 第一卷必须避免什么。

## 篇幅默认

如果用户没有指定，默认按长篇网文理解：

- 全书 500 章以上。
- 第一卷 60-80 章。
- 每卷 5-7 幕。
- 每幕 10-15 章。
- 单章 2000-3000 字。

可以按 agent 能力压缩展示，但不能把书的结构尺度压成 10 章卷、3 章幕。

## 用户确认

如果用户未明确要求自动继续，PRD 输出后停下确认以下项：

- 一句话商业卖点。
- 篇幅体量。
- 主角起点/第一卷终点/全书终点。
- 元爽点。
- 世界机器。
- 禁区。

## 质量门

不通过则不能进入 Step 3：

- PRD 有具体体量，不是“若干章”。
- 元爽点不是压力描述。
- 世界机器能解释长期剧情来源。
- 主角终点能反推金手指和升级表。
- 禁区能防止 plot 短篇化。
