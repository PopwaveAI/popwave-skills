# Step 3：文风DNA蒸馏→文风库文件

> 管线: pop-writer-v3-seed v1.6.0
> 消费: Step 0创意共识单（文风方向+核心卖点） / 对标书原文 / 用户指定文风参考 / pop-shared-dna / library文风库
> 产出: 写作资产/文风库/{书名}.md（文风DNA档案，项目资产）

## v3.5 说明

文风DNA是项目资产，存放在 `写作资产/文风库/{书名}.md`。v3.5中文风DNA不变，仍由seed step-3蒸馏产出。revise阶段完整加载文风DNA作为硬阻塞。

## 读什么

| # | 文件 | 为什么 |
|:-:|:-----|:-------|
| 1 | Step 0产出的创意共识单（上下文） | 文风方向+核心卖点——引导文风DNA蒸馏的目标方向 |
| 2 | 对标书原文（pop-decon 产出，如可用） | 蒸馏源文本 |
| 3 | 用户指定的文风参考文本（如无对标书） | 蒸馏源文本 |
| 4 | pop-shared-dna 文风拆解技能（如已安装） | 蒸馏方法 |
| 5 | library 文风库/{对标书}.md | 对标书文风DNA档案 |

## 做什么

### 1. 文风源确认

按优先级确认文风来源：

| 优先级 | 来源 | 条件 |
|:-------|:-----|:-----|
| 1 | 对标书原文 | pop-decon 提供了原文片段 |
| 2 | 用户指定参考文本 | 用户明确指定文风参考 |
| 3 | library 文风库 | 已有文风DNA档案 |

### 2. 文风DNA蒸馏

用 pop-shared-dna（或等效方法）蒸馏文风DNA，产出 `写作资产/文风库/{书名}.md`。

5维度拆解：
- 句式（长短句节奏/复合句密度/排比/倒装/省略）
- 叙事距离（近距离贴脸/中距离旁观/远距离俯瞰）
- 感官密度（五感描写频率/哪种感官主导）
- 对话风格（对话占比/口语化程度/信息密度/潜台词）
- 数据嵌入方式（数值/系统信息如何融入叙事）

硬阻塞声明：文风DNA缺失 = 项目初始化终止（红线❌1）。

### 3. 用户QA确认

将文风DNA档案呈现给用户确认：

```markdown
## 文风DNA确认

### 文风源
{对标书/用户指定/library}

### 5维度拆解
1. 句式：{描述}
2. 叙事距离：{描述}
3. 感官密度：{描述}
4. 对话风格：{描述}
5. 数据嵌入：{描述}

---
✅ 用户确认，进入step-4设定产出
❌ 用户要求调整 → 回到蒸馏步骤调整
```

## 门禁

| 检查项 | 失败动作 |
|:-------|:---------|
| 文风DNA已蒸馏（红线❌1） | 扫描写作资产/文风库/，空=终止整个skill |
| 文风库文件已落盘 | 补充蒸馏流程 |
| **用户已确认文风DNA** | **等待用户确认后再进step-4** |
