## 做什么

### 1. 主角引擎设计

驱动力三层设计（解决长线不足的关键）：

| 层级 | 内容 | 激活时机 | 与压力矩阵的关系 |
|:-----|:-----|:---------|:----------------|
| 表层 | 活下去，活好 | ch1即激活 | 直接应对压力矩阵 |
| 中层 | 保护身边的人 | 遇到需要保护的人时 | 天然冲突——消耗资源、暴露位置 |
| 深层 | 推翻/改变制度 | 能力成长后浮现 | 从"求生"到"开战" |

- 缺陷：身体/社会身份/资源的三重约束
- 决策风格：一句话定义（如"实用主义+杀伐果断"）

**行为准则设计：**

主角的背景决定其行为模式。每本书的主角风格不同，行为准则是"这个背景的人在这本书里怎么行动"的映射。

| 子项 | 内容 | 示例 |
|:-----|:-----|:-----|
| 背景映射 | 主角背景→行为规则映射 | {主角背景}→{行为规则1}/{行为规则2} |
| 战斗风格 | 一句话定义战斗方式 | {近身压制/远程牵制/智取型}+{特征} |
| 道德底线 | 不可逾越的红线 | {不做什么} |
| 行为禁忌 | 绝不会做的事 | {绝不做什么} |

设计原则：
- 行为准则必须与驱动力三层一致（驱动力决定"为什么做"，行为准则决定"怎么做"）
- 行为准则必须与缺陷有张力（如"不留活口"的准则 × "资源匮乏"的缺陷 = 不得不冒险处理尸体）
- 行为准则不是设定堆砌，是行为预测器——写作时能据此判断"主角在这个场景会怎么行动"

### 2. 成长路径设计

| 阶段 | 能力里程碑 | 心态变化 | 资源积累 | 对应目的地 | last_updated_ch |
|:-----|:---------|:---------|:---------|:-----------|:----------------|
| 1 | {里程碑} | {变化} | {积累} | D001 | 0 |
| 2 | {里程碑} | {变化} | {积累} | D002 | 0 |
| ... | | | | | 0 |

每一步对应一次"质变时刻"。成长路径在 ch4-ch10 的表层驱动力真空期提供中期驱动力。

### 3. 目的地设计

5-10个模糊路标。格式：序号+描述+预估范围+状态。不指定章节，允许调整。

### 4. 种子文件夹组装

参照 `templates/种子文件夹结构-模板.md`，将核心卖点（Step 0产出）+ 本步骤产出的主角引擎/成长路径/目的地，与 Step 1-2 已写入的 压力矩阵.md/冲突轴.md/金手指.md 组装为完整种子文件夹：

**创建以下文件：**

| 文件 | 来源 | 内容 |
|:-----|:-----|:-----|
| `种子/核心卖点.md` | Step 0 创意共识单 | 核心卖点一句话+展开+爽点+差异化+情绪承诺 |
| `种子/主角引擎.md` | 本步骤§1设计 | 驱动力三层+缺陷+决策风格+行为准则（含 last_updated_ch） |
| `种子/成长路径.md` | 本步骤§2设计 | 成长阶段表格（含 last_updated_ch） |

| `种子/_index.yaml` | 本步骤组装 | 种子引擎索引（版本+要素文件清单+last_updated_ch） |
| `种子/_log.md` | 本步骤组装 | 变更日志（init记录） |

## 门禁
| 检查项 | 失败动作 |
|:-------|:---------|
| 六要素全部填写 | 补全缺失要素 |
| 种子验证通过（能生成前3章走向） | 退回补强种子 |
| 种子写驱动力不写设定清单（红线❌4） | 删除设定堆砌，改写为驱动力导向 |

| `种子/_log.md` 已创建 | 补创建日志 |
| 活记忆baseline #0已初始化 | 补充初始化 |
