# Step 4：种子组装+验证

> 管线: pop-writer-v3-seed v1.1.0
> 消费: Step 1-2 产出（压力矩阵+冲突轴+金手指）+ Step 3 产出（文风库文件，不写入种子）+ 用户输入
> 产出: 种子文档.md（活种子v1.0.0）+ 活记忆baseline #0

## 读什么
- Step 1 产出的种子§压力矩阵+冲突轴
- Step 2 产出的种子§金手指
- Step 3 产出的文风库文件（`写作资产/文风库/{书名}.md`，项目资产，不写入种子）
- 用户提供的主角设想/世界观片段

## 做什么

### 1. 主角引擎设计（creative不产出的新增要素）

驱动力三层设计（解决长线不足的关键）：

| 层级 | 内容 | 激活时机 | 与压力矩阵的关系 |
|:-----|:-----|:---------|:----------------|
| 表层 | 活下去，活好 | ch1即激活 | 直接应对压力矩阵 |
| 中层 | 保护身边的人 | 遇到需要保护的人时 | 天然冲突——消耗资源、暴露位置 |
| 深层 | 推翻/改变制度 | 能力成长后浮现 | 从"求生"到"开战" |

- 缺陷：身体/社会身份/资源的三重约束
- 决策风格：一句话定义（如"实用主义+杀伐果断"）

### 2. 成长路径设计（creative不产出的新增要素）

| 阶段 | 能力里程碑 | 心态变化 | 资源积累 | 对应目的地 | last_updated_ch |
|:-----|:---------|:---------|:---------|:-----------|:----------------|
| 1 | {里程碑} | {变化} | {积累} | D001 | 0 |
| 2 | {里程碑} | {变化} | {积累} | D002 | 0 |
| ... | | | | | 0 |

每一步对应一次"质变时刻"。成长路径在 ch4-ch10 的表层驱动力真空期提供中期驱动力。

### 3. 目的地设计（creative不产出的新增要素）

5-10个模糊路标。格式：序号+描述+预估范围+状态。不指定章节，允许调整。

### 5. 种子验证

验证六要素能否生成前 3 章的全部剧情走向：

| 章 | 种子要素驱动 | 涌现的剧情走向（不指定具体事件） |
|:---|:---|:---|
| ch001 | {哪些要素} | {大致走向} |
| ch002 | {哪些要素} | {大致走向} |
| ch003 | {哪些要素} | {大致走向} |

### 6. 活记忆baseline #0初始化

种子验证通过后，初始化活记忆：

```yaml
metadata:
  last_compacted_at: 0
  total_chapters: 0
  seed_version: "1.0.0"

entries:
  - type: baseline
    chapter: 0
    timestamp: ...
    seeds: []
    pressure: {初始倒计时值}
    characters: {主角初始状态}
    world_rules: []
    rhythm_log: []
    power_curve: []
    destinations: [{初始状态}]
```

## 门禁
| 检查项 | 失败动作 |
|:-------|:---------|
| 六要素全部填写 | 补全缺失要素 |
| 种子验证通过（能生成前3章走向） | 退回补强种子 |
| 种子写驱动力不写设定清单（红线❌4） | 删除设定堆砌，改写为驱动力导向 |
| 活记忆baseline #0已初始化 | 补充初始化 |
