# Step 4 - Seed To World Handoff

目的：把 PRD 和研究档案整理成 world-foundation 可消费的交接包。此步不写 L1、不设计金手指、不做文风DNA。

## 输出路径

```text
library/设定账本/seed-to-world交接包.md
library/设定账本/seed-产物索引.md
```

## 交接包必填

| 模块 | 要求 |
| --- | --- |
| PRD路径 | 指向 `library/设定账本/新书立项PRD.md` |
| 长篇承诺书路径 | 指向 `library/设定账本/长篇承诺书.md` |
| 用户确认 | 写明用户选了哪个候选，或自动选择理由 |
| 不可牺牲项 | PRD 中不可被 world 改写的商业承诺、元爽点、主角路径、禁区 |
| Foundation硬问题 | world-foundation 必须回答的世界规模、势力层级、资源经济、主角穿透路径问题 |
| 卷级质变草案 | 每卷主质变、需要的世界层级、禁止提前兑现项 |
| 可涌现项 | 允许 world-foundation 发散的世界规则、势力、资源、力量体系、金手指方向 |
| 证据索引 | S/G/P/M/V 证据ID与路径 |
| 外部母版 | 可继续使用的外部母版与水土不服风险 |
| 风险清单 | 最可能导致设定平庸、严肃文学偏移、短篇化的点 |
| 用户待确认 | 进入 world 前是否需要确认力量体系结构、命名风格、主角终点 |

## 可涌现与不可涌现边界

必须明确分开：

```markdown
## 不可涌现
- 已锁商业卖点：
- 已锁主角起点/终点：
- 已锁元爽点：
- 已锁商业爽点金字塔：
- 已锁卷级质变草案：
- 已锁禁区：

## 允许涌现
- 世界规模落地方式：
- 势力/战力/资源金字塔：
- 世界核心规则候选：
- 力量体系结构候选：
- 势力/资源候选：
- 金手指方向与待回答问题：
- 文风表达候选：
```

## Seed 产物索引

读取 `templates/Seed产物索引-模板.md`，状态只能是：

- 可用：通过质量门。
- 草案：可讨论但不可交给下游。
- 缺失：必须补齐。
- 阻断：缺它不能进入 world-foundation。

## 交接包模板

读取 `templates/seed-to-world交接包-模板.md`。`execution.mode` 只能在研究、候选PK、长篇承诺书、PRD、交接包、索引全部通过质量门后写 `formal`。

## 可进入 world-foundation 的最低门

- 研究索引存在。
- 种子展开图存在。
- 外部母版卡存在。
- 故事概念候选PK存在。
- 长篇承诺书存在。
- 新书立项PRD存在。
- seed-to-world交接包存在。
- PRD 的篇幅、元爽点、主角起终点、世界机器草案、卷级质变草案、禁区明确。
