# Step 4：故事种子与世界生长语法产出

> 目标：产出 plot 可运行的初始条件，而不是完整百科设定。

## 消费

- Step 0 创意共识
- Step 1 压力矩阵/冲突轴/调研档案
- Step 2 金手指设计
- Step 3 文风 DNA
- 用户补充方向

## 1. 写故事 DNA

用叙事性自然语言写，不要写设定清单。

结构：

```markdown
# 故事 DNA

## 这是一个什么故事
{主角、世界、核心冲突、读者期待}

## 主角
{他想要什么，怕什么，底线是什么，为什么读者想看他赢}

## 爽感方式
{主角如何通过行动改变局面，读者每章期待什么反馈}

## 写给后续 agent 的话
{读者是谁、节奏要求、爽感要求，用自然语言，不写检查清单}
```

## 2. 冻结硬设定

写 `世界宪法.md`：

- 世界基本类型。
- 底层秩序和不可变规则。
- 核心禁忌。
- 主线大压力。
- 不允许 plot/create 随意改动的事实。

写 `力量体系.md`：

- 能力如何获得。
- 能力如何升级。
- 能力如何反馈给读者。
- 能力限制和代价。
- 可见进度条或成长信号。

## 3. 写主角引擎

写 `主角引擎.md`：

- 表层欲望。
- 中层底线。
- 深层驱动力。
- 决策风格。
- 行动爽点。
- 行为禁忌。
- 典型主动行动。

## 4. 写核心卖点

写 `核心卖点.md`：

- 一句话卖点。
- 核心情绪承诺。
- 爽点类型比例。
- 读者为什么继续看。
- 和同类作品的差异。

## 5. 写设定生长语法

写 `设定生长语法.md`，明确 plot 如何按需孵化软设定：

```markdown
# 设定生长语法

## 允许出生的条件
- 设定必须被当前事件使用。
- 设定必须解决一个明确剧情缺口。
- 设定必须产生行动、限制、反馈或新压力。

## 出生后冻结规则
- 首次出现章节必须写入设定账本。
- 后续不得无解释改写。
- 与硬设定冲突时以硬设定为准。

## 禁止事项
- 禁止提前补全百科。
- 禁止只为氛围新增设定。
- 禁止用设定解释替代事件。
```

## 7. 初始化活记忆

```yaml
  ch0初始：{≤200字，自然语言概括主角当前处境、能力、资源、关系、压力和短期目标。}
events: []
```

## 8. 更新索引

`写作参考/索引.md` 必须列出所有设定文件和文风资产。

## 门禁

- [ ] 故事 DNA 是叙事文，不是设定表。
- [ ] 硬设定足以约束后续因果。
- [ ] 软设定没有被提前补完。
- [ ] 设定生长语法已写明“出生条件”和“冻结规则”。
- [ ] 初始种子池只包含压力/底线/资源/冲突对象/异常/缺口。
- [ ] 活记忆 baseline#0 已初始化。

## 交接

seed 完成后，交给 `pop-writer-v3-plot`。plot 将从初始种子池和设定缺口开始运行第一个剧情单元。
