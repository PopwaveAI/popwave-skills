# Step 4 - 设定底盘输出

把前面共创结果写成 plot 可消费的文件。

## 输出文件

- `library/设定账本/故事DNA.md`
- `library/设定账本/世界宪法.md`
- `library/设定账本/力量体系.md`
- `library/设定账本/主角引擎.md`
- `library/设定账本/金手指.md`
- `library/设定账本/核心卖点.md`
- `library/设定账本/设定生长语法.md`
- `library/设定账本/初始种子池.md`
- `library/设定账本/Library查询方向.md`
- `library/设定账本/跨媒介母版雷达.md`
- `library/文风DNA/{书名}.md`
- `活记忆/活记忆.yaml`
- `library/设定账本/seed-产物索引.md`

## 要求

- 硬设定写清约束。
- 软设定只写缺口和生长条件。
- 初始种子池必须能转化为 plot 的事件碰撞。
- Library 查询方向必须包含：赛道、主任务标签、主爽点标签、候选参考书目、金手指类型、需要声明的库缺口。
- 跨媒介母版雷达必须包含：候选母版、媒介类型、可迁移机制、适合的写作阶段、不能照搬项、plot 优先级。
- 每个设定文件开头必须写“来源”：用户确认 / library 证据 / 模型推断 / 待确认。不能把推断伪装成已确认硬设定。
- `seed-产物索引.md` 必须告诉用户哪些文件是正式底盘，哪些只是过程文件，哪些仍是草案。

## 底盘完成门

缺以下任一项，不得说“seed 完成”：

- `故事DNA.md` 明确读者承诺、主味型、第一章追读承诺。
- `世界宪法.md` 明确 5-10 条硬规则和禁区。
- `主角引擎.md` 明确欲望、底线、行动偏好、第一卷变化。
- `金手指.md` 通过首章可用性测试。
- `初始种子池.md` 至少 12 个可供 plot 碰撞的压力/资源/敌人/异常种子。
- `Library查询方向.md` 有任务标签和资料库证据绑定。
- `跨媒介母版雷达.md` 至少 5 个候选母版。
- `活记忆.yaml` 只保留当前写作必须知道的短状态。
