## 设定/设定生长语法.md

```markdown
# 设定生长语法

## 允许出生的条件
- 设定必须被当前事件使用。
- 设定必须解决明确剧情缺口。
- 设定必须产生行动、限制、反馈或新压力。

## 出生后冻结规则
- 首次出现章节必须写入设定账本。
- 后续不得无解释改写。
- 与硬设定冲突时以硬设定为准。

## 禁止事项
- 禁止提前补全百科。
- 禁止只为氛围新增设定。
- 禁止用设定解释替代事件。
```

## 使用原则

1. 硬设定约束全书因果，低频修改。
2. 软设定只写缺口，由 plot 在事件运行中按需孵化。
3. 新设定出生后进入设定账本，由 arc 在单元结束后校准。
4. 索引必须同步更新，避免下游找不到设定来源。
