# Step 5 - Protagonist And Goldfinger

目的：在 Foundation 结构工程和 L1 世界包成立后，设计主角引擎、角色储备池和金手指。角色储备池只保存弱剧情耦合锚点；金手指必须从世界机器和主角穿透模型中长出来。

## 角色储备池

读取 `templates/角色储备池-模板.md`。角色不是人物小传，而是给 plot 使用的可切片角色资源。

### 生成 SOP

角色不得凭空产出，但不要单独写独立前置清单。把来源依据和角色原型参考直接写进 S/A 角色表格，让每一行自己说明“为什么需要这个角色、参考什么原型、只迁移什么、不照搬什么”。

来源只能来自：

- `主角引擎.md`：主角欲望、恐惧、底线、行动偏好需要什么镜像/阻碍/锚点。
- `Foundation-势力战力资源总表.md`：每个第一卷可触达层级需要哪些首脑/干部/执行人/接口人。
- `Foundation-卷级质变阶梯.md`：第一卷质变需要哪些见证者、压迫者、交易者、误判者。
- `L1-势力格局.md`：能主动制造事件的势力需要哪些代表性角色或槽位。
- `L1-资源物品.md`：资源获取、流通、定价、黑市、官方管制需要哪些接口角色。
- `开局契约.md`：第一章弱势、触发条件、第一反馈需要哪些最小人物。

没有来源的角色不得进入储备池；只能写入“待 plot 临场补”。

S/A 角色必须有 2-3 个角色原型参考。原型可以来自历史人物、现实真人类型、影视/动漫/游戏/IP 角色、职业纪录片、新闻人物类型、经典小说角色等。原型不是换皮对象，只能迁移：

- 气质和压迫感。
- 反应模式和决策习惯。
- 处境张力和社会位置。
- 关系动力和冲突方式。
- 能制造爽点的反馈方式。

禁止迁移身份、专有设定、剧情链、台词、标志性场面和可识别组合。S/A 如果找不到足够原型参考，宁可降级为 B 槽位，不要硬写成重要角色。

### 数量约束

按第一卷可用资源预算生成，不为全书百科铺人：

| 等级 | 数量 | 具名规则 |
| --- | ---: | --- |
| S 核心角色 | 1-3 人，必须包含主角 | 可具名；超过 3 人必须降级 |
| A 卷级主角色候选 | 3-6 人 | 可具名；承担主压力/盟友/交易/调查/核心误判 |
| B 功能角色槽位 | 6-12 个槽位 | 默认不具名；plot 选用后再具名或合并 |
| C 路人反馈模板 | 4-8 类 | 不具名，不进长期人物账 |

总具名角色建议不超过 8 个（S+A）。若世界结构确实需要更多，必须写明超额理由，并把超额角色标为 draft。

### 生成顺序

1. 从主角引擎生成 S 级主角和必要长期镜像/锚点。
2. 从势力战力资源总表生成 A/B 角色或槽位：每个第一卷开放层级最多 1 个 A 级代表，1-2 个 B 级槽位。
3. 从 L1 势力格局补事件制造者：只保留能制造事件、误判、交易、追捕或反馈的角色。
4. 从资源物品补接口角色：医生、商人、线人、鉴定者、执法员等优先做 B 槽位。
5. 从开局契约补第一章最小人物：只补不可缺的压迫者/见证者/反馈者。
6. 生成 C 级反馈模板：底层居民、围观者、打手、店员、基层员工等，不单独具名。

### 职责边界

world-foundation 只写弱剧情耦合信息：

- 姓名/称号/代号。
- 所属层级/势力。
- 职务/阶层位置。
- 一句话人设。
- 欲望类型、恐惧/底线。
- 反应模式、说话方式。
- 可制造事件类型。
- 能提供的爽点反馈。
- 与主角的初始关系类型。
- 长期/卷级/功能/路人层级。

plot 才补强剧情耦合信息：

- 本卷是否选用。
- 登场幕、退场幕。
- 本卷功能位。
- 是否合并/拆分/具名。
- 关系变化、死亡、背叛、晋升、揭露、交易。
- 具体冲突、给出的信息/资源/误判/打脸反馈。

### 角色资源预算

| 等级 | 信息量 | 产出要求 |
| --- | ---: | --- |
| S 核心角色 | 120-200字/人 | 主角、长期搭档/宿敌/核心反派候选；只写稳定锚点 |
| A 卷级主角色候选 | 80-120字/人 | 本卷主反派、关键盟友、中间商、调查者等候选 |
| B 功能角色槽位 | 40-80字/槽位 | 小头目、干部、黑市商人、医生、线人、执法员等 |
| C 路人反馈模板 | 20-40字/类 | 围观者、底层居民、打手、店员等反馈类型 |

质量要求：

- 不平均展开所有角色；S/A 少而准，B/C 用槽位和模板。
- 每个 S/A/B 必须能映射到世界尺度金字塔或势力战力资源总表。
- 每个 S/A/B 必须说明可制造事件类型和主角可利用点。
- 每个 S/A 必须填写来源文件/需求依据、2-3 个角色原型参考、迁移点和不照搬边界；缺任一项则删除或降级为 B 槽位。
- 每个 B 必须填写槽位来源；没有槽位来源的 B 删除或降级为 C 级反馈模板。
- S+A 具名角色超过 8 个时，必须优先降级 A 为 B 槽位。
- 不写具体章节、不写剧情弧线、不写未来死亡/背叛/晋升。
- C级不单独建人物卡，只给反馈模板。

## 质量门

- 金手指不是系统换皮。
- 金手指不是观察器，必须驱动行动。
- 金手指离开本世界会失效或降级。
- 金手指必须解释主角如何穿透至少前三卷的世界层级。
- 角色储备池必须在 S/A 行内写明来源依据和多个角色原型参考；必须分层，且没有超出数量/字数预算的人物小传。
- 第一章反馈不抹平弱势。
- 代价能制造短期剧情后果。
