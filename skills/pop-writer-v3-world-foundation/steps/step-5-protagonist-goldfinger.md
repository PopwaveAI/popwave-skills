# Step 5 - Protagonist And Goldfinger

目的：在 Foundation 结构工程和 L1 世界包成立后，设计主角引擎和金手指。金手指必须从世界机器和主角穿透模型中长出来。

## 输出路径

```text
library/设定账本/主角引擎.md
library/设定账本/金手指.md
library/设定账本/开局契约.md
library/设定账本/设定涌现/10-金手指候选竞技.md
```

## 主角引擎

| 模块 | 必须回答 |
| --- | --- |
| 表层欲望 | 第一卷想得到什么 |
| 深层需要 | 长篇真正缺什么 |
| 恐惧 | 最怕失去什么 |
| 底线 | 什么事绝不做，或做了会付出代价 |
| 行动偏好 | 暴力/交易/欺诈/规则漏洞/经营/救援/信息差 |
| 第一章主动动作 | 弱势下能主动做的第一件事 |
| 决策缺陷 | 什么选择会惹出更大麻烦 |
| 第一卷变化 | 从什么人变成什么人 |
| 长篇穿透方式 | 主角凭什么逐层突破世界金字塔 |

## 金手指候选竞技

读取 `templates/金手指候选竞技-模板.md`。至少 3 个互斥候选，必须在反馈形式、行动驱动、代价类型、世界耦合、长线终点中至少 4 项不同。

## 质量门

- 金手指不是系统换皮。
- 金手指不是观察器，必须驱动行动。
- 金手指离开本世界会失效或降级。
- 金手指必须解释主角如何穿透至少前三卷的世界层级。
- 第一章反馈不抹平弱势。
- 代价能制造短期剧情后果。
