## 文风DNA

文风DNA不是原创总结，不得凭感觉写“冷峻/黑色/史诗/克制”等抽象标签。

### 1. 选择来源

先读取公共文风库或用户项目 library 中的候选文风文件：

```text
$env:APPDATA/popwave/remote-skills/pop-trope-library/文风库/
项目/library/文风DNA/
用户显式 @ 引用的文风文件
```

若路径不存在、候选不足或无法确认适配对象，输出“文风DNA缺口报告”，不得生成正式 `library/文风DNA/文风DNA.md`。

### 2. 复制原件

从候选中选择最适配本书的 1 个主文风原件；必要时可选择 1 个辅文风原件。正式 `library/文风DNA/文风DNA.md` 必须以复制原件为主体，保留原件的样例、句法、对白、节奏和禁用项，不得改写成概括性表格。

允许追加但不允许替代原件的内容：

```text
## 本项目适配说明
- 主文风来源：
- 辅文风来源：
- 适用场景：
- 不适用场景：
- 本书必须强化的网文反馈：
- create 阶段必须保留的原文样例：
```

### 3. 适配原则

| 项目 | 要求 |
| --- | --- |
| 来源可追溯 | 写清原件路径、复制范围、选择理由 |
| 原件优先 | 原件内容占主体，项目说明只做附录 |
| 样例保留 | 保留可模仿片段；没有样例的文件不得作为主文风 |
| 爽点服务 | 若原件偏氛围，必须在适配说明中标明如何外显面板、数字、旁人反应、敌人误判、资源到账 |
| 禁止改写 | 禁止把原件压缩成“某某风格 × 某某气质”的一句话定义 |
| 禁止补完 | 子代理失败或未读文风库时，只能标记缺口，不能手写补完 |

### 4. 质量门

满足以下条件才可标记文风DNA正式可用：

- 已读取至少 2 个候选文风文件，或用户明确指定 1 个文风原件。
- 正式文件包含“来源路径”和“复制范围”。
- 正式文件保留原件中的可执行样例。
- 项目适配说明不超过原件主体长度的 30%。
- 没有用原创标签替代原文样例。

## 读者表达偏好

读取 `templates/读者表达偏好-模板.md`，输出到 `library/写作配置/读者表达偏好.md`。

这个文件是 create 的表达裁决器，不是文风DNA的重复总结。它必须明确：

- 目标读者要小白直给、硬核克制、轻松吐槽、群像史诗，还是混合。
- 情绪能否用旁白/心理/独白直说；哪些场景必须直说。
- 推理过程是完整展示、结论先出，还是只保留关键判断。
- 环境描写单次最多几句，何时必须立刻回到动作。
- 爽点是否必须外显为面板、数字、敌人反应、资源到账、身份变化。
- 严肃文学偏移的高危写法和替代写法。

默认网文商业向项目采用“小白直给优先”：关键情绪、收益、压力、敌人误判和读者期待不得只靠细节暗示。

## 项目资料总目录

读取 `templates/项目资料总目录-模板.md`，输出到 `library/项目资料总目录.md`。

总目录是资料路由，不是文件清单。必须按任务场景写清：

- seed/world/plot/create/review 各自必读哪些文件。
- 哪些文件只按需读取。
- 哪些过程文件不得直接当正式设定。
- create 只能读取哪些切片，不能全量吞设定。
- review 诊断问题时应追加哪些资料。

如果总目录缺失，后续 agent 容易只读最近文件或读错过程文件，world 不得标记为完整交接。

## World 产物索引

读取 `templates/World产物索引-模板.md`。全部满足才标记可进入 plot：

- 设定涌现目录完整。
- Foundation 商业爽点金字塔、世界尺度金字塔、势力战力资源总表、卷级质变阶梯、主角穿透模型完整。
- L1 世界包完整。
- 起点快照、世界宪法、动态升级表可用。
- 主角引擎、金手指、开局契约可用。
- 文风DNA有原件来源、复制范围、项目化适配说明；读者表达偏好、Library 查询方向、跨媒介母版雷达、项目资料总目录可用。
