# 读者表达偏好

> 本文件裁决“怎么把内容说给目标读者听”。文风DNA负责像谁，读者表达偏好负责直白到什么程度、爽点显到什么程度。

## 基本定位

| 项 | 内容 |
| --- | --- |
| 目标读者 | 小白直给 / 硬核规则流 / 轻松吐槽 / 群像史诗 / 混合 |
| 首要体验 | 快速看懂 / 快速爽 / 沉浸感 / 智斗感 / 情绪共鸣 |
| 表达裁决 | 当文风DNA与本文件冲突时，以本文件为准；爽点和理解优先于质感 |

## 表达参数

| 参数 | 本书取值 | 写法要求 | 禁止 |
| --- | --- | --- | --- |
| 情绪直白度 | 低/中/高 | 关键情绪允许旁白、心理或短独白直给 | 只靠手指、眼神、沉默让读者猜 |
| 推理透明度 | 完整/结论先出/只保留关键判断 | 网文向优先“结论先出，再补1句理由” | 连续三段解释推理链 |
| 环境描写长度 | 0-1句/1-2句/3句以上 | 环境必须服务目标、风险或收益 | 纯气氛开场超过120字 |
| 爽点外显强度 | 低/中/高 | 面板、数字、敌人反应、资源到账、身份变化至少一种落地 | 爽点藏在氛围或隐喻里 |
| 心理活动许可 | 禁止/短句/允许小段 | 关键选择前可用1-2句心理直给 | 长篇自我剖析 |
| 旁白许可 | 禁止/短旁白/可用 | 用短旁白降低理解成本 | 总结腔替代场面 |
| 说明腔容忍度 | 低/中/高 | 设定通过行动、交易、失败显影；必要说明压成短句 | 设定集插播 |

## 小白文修正表

| 严肃文学偏移 | 小白文替代 |
| --- | --- |
| 手指停了一秒、目光沉下去 | 他死死攥住刀，指节发白 |
| 环境嗅觉/触觉铺三段 | 一句定场，第二句进目标 |
| 让读者自己品情绪 | 直接写他怕、怒、急、忍不住 |
| 完整解释因果链 | 先给判断，再给一句依据 |
| 章末只留悬念 | 章末给主角下一步主动选择 |

## Create 硬要求

- 每章开场 120 字内必须出现目标、异常、危险、系统反馈或主动动作之一。
- 每 600-800 字必须有一次新信息、新收益、新威胁、敌人误判、主角操作成败或外部反应。
- 关键爽点不得只靠细节暗示，必须让读者当场知道爽在哪里。
- 情绪可以直说，但直说后必须跟动作，不许停在抒情。
