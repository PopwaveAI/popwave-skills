# 小说世界构筑 v1.2.1

> **不在"发明"世界——在兑现宪法。** 每个设定必须追溯到故事引擎中的一条。凭空添加视为不合格。

## 速查表

| 我要 | 操作 | 前置条件 |
|------|------|---------|
| L1 设定层 | 六件套骨架 → 深度展开 → 交叉关联 | 故事引擎已锁定 |
| 稳定性检验 | 运行 7 项 checklist | L1 六篇全部产出 |
| 角色卡 | 选级(Lv1-Lv4) → 加载模板 → 填写 | 稳定性通过 |
| 数值体系 | 战力/等级排期/怪物映射/碰撞曲线 | Phase 1 力量体系完成 |
| 起点快照 | 主角状态+世界状态+配角+伏笔 | Phase 1-4 全部完成 |
| 世界宪法 | 逐维提取约束条件 | 起点快照已确认 |

## ❌ 质量红线

| # | 红线 |
|:-:|:-----|
| ❌1 | **宪法未锁定不进 world** — 故事引擎.md 未锁定时不得进入任何 Phase |
| ❌2 | **不凭空发明** — 每个设定必须追溯到宪法中的一条 |
| ❌3 | **L1 六篇不可缺一** |
| ❌4 | **稳定性不通不进角色卡** |

| ❌6 | **数值体系不与 L1 矛盾** |
| ❌7 | **起点快照未经用户确认不进宪法** |
| ❌8 | **世界宪法必须覆盖所有 L1 维度** |

## ❌ 产出纪律（防上下文溢出）

| # | 规则 |
|:-:|:-----|
| ❌9 | **写入文件后，对话中只留摘要（≤ 200 字），不粘贴完整产出。** 文件内容已通过工具调用完整写入磁盘，重复粘贴到对话只会让上下文膨胀。每轮回复末尾的说"已写入 {路径}，摘要：{一句话核心}，需展开任何一段告诉我。" |

## ❌ WRONG 示例

| ❌ 错误做法 | 问题 | ✅ 正确做法 |
|:-----------|:-----|:------------|
| 用户说"建个修仙世界" → Agent 直接开写"灵气分九等、境界分炼气到化神" | 凭空发明，设定没有从故事引擎.md 追溯 | 先读故事引擎.md → 提取宪法约束 → 每个设定标注"来自引擎第X条" |
| L1 六篇只写了 5 篇 → Agent 说"够了先做稳定性检验" | L1 六篇不可缺一，缺一篇稳定性检查会漏掉维度 | 六篇全部产出后再进 Phase 2 稳定性。缺一篇就退回补 |
| 数值体系境界从参考书照搬 → "斗者斗师斗王"直接拿来用 | 数值不与 L1 矛盾规则，照搬参考书导致体系不匹配 | 从本故事 L1-02 力量体系推导战力值，参考书只借鉴密度/节奏 |
| 力量体系凭假设直接写 → 用户说"建DND 3.5e体系" → Agent 自定"凡人期 Lv1-9→传奇期 Lv10-14"被用户否定 | 未确认等级命名和范围就动笔，用户心智模型与 agent 假设不匹配 | 写 L1-02 力量体系前先确认用户对等级名的预期：列出命名提案让用户选/改（如"超凡者 1-20级？还是凡人期→传奇期？"），确认后再填充 |

## 收到任务后，按顺序执行

### Phase 1：L1 设定层推演

详见 [steps/step-1.md](steps/step-1.md) ~ [steps/step-6.md](steps/step-6.md)。

> **读什么**：故事引擎.md + 跨域素材蒸馏.md + 拆书融合摘要.md（如有）+ deconstructor T1/T2/T4（如有）
> **产出什么**：`00-原始设定/L1-元设定层/01-06.md`
> **门禁**：❌ 每个设定必须能在引擎中找到根
> **写 L1-02 力量体系前必做**：向用户确认等级命名/分段范围。列出命名提案（如"超凡者 1-20级 vs 凡人→传奇→半神"），用户确认后再填充。不可凭假设动笔。

## 本阶段边界

### 本步做什么
- L1 设定 → 稳定性 → 角色 → 数值 → 起点 → 宪法

### 本步不做什么
- ❌ 不重新发明故事引擎（那是 creative 的活）
- ❌ 不设计剧情或章节（那是 plot 的活）
- ❌ 不设计具体章级事件（那是 chapter-design 的活）

**越界检测**：对话出现"第X章的剧情/这个情节怎么走" → "这属于 plot 的范围，到那一步处理。现在先把世界参数锁定。"

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 宪法更新 | 标记受影响维度 → 重推 → 连锁重建下游 |
| L1 单篇修订 | 修改 → 重跑交叉关联 → 重跑稳定性 → 检查下游 |
| deconstructor 缺失 | 数值用平台默认节奏 |
| 角色卡模板缺失 | 按维度矩阵手写，不编造字段 |
| 宪法与 L1 不一致 | 先修 L1 → 重跑稳定性 → 再提取宪法 |
| 起点快照与宪法冲突 | 优先修正起点快照；若宪法需修订 → 退回 creative |
