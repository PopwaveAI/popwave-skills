## 错误的做法

| 错误 | 问题 | 正确 |
|:-----|:-----|:------|
| 用户说建个修仙世界 → Agent 直接开写灵气分九等、境界分炼气到化神 | 凭空发明，设定没有从故事引擎.md 追溯 | 先读故事引擎.md → 提取宪法约束 → 每个设定标注来自引擎第X条 |
| L1 六篇只写了 5 篇 → Agent 说够了先做稳定性检验 | 缺一篇稳定性检查会漏掉维度 | 六篇全部产出后再进 Phase 2 稳定性 |
| 数值体系境界从参考书照搬 → 斗者斗师斗王直接拿来用 | 数值不与 L1 矛盾规则 | 从本故事 L1-02 力量体系推导战力值，参考书只借鉴密度/节奏 |
| 力量体系凭假设直接写 → Agent 自定凡人期 Lv1-9 被用户否定 | 未确认等级命名就动笔 | 写 L1-02 前先确认用户对等级名的预期，列出提案让用户选 |
| 用户要暗黑风/原创命名 → Agent 只给一个方案用户否了 | 单一方案把用户锁在一条轨道上，不给选择面 | 提供 2-3 个不同隐喻方向的方案（如畸变系/焚烧系/工具系），让用户混合搭配。详见 `references/power-system-naming.md` |
| 用户说"A概念+B概念" → Agent 只选一个丢掉另一个 | 用户给了两个概念意味着两个都想要，放弃一个=丢失信息 | 按 dual-concept-merging 法融合：主概念做方向，副概念做方法，每层同时体现两个概念。详见 `references/power-system-naming.md §三` |
| 数值体系反拆时基于单卷数据归纳成全书铁律 | **论据范围越界** | 每份反拆文档标注范围声明，注明如果后卷推翻此模式的预测条件 |
| 收到跨域融合决策后直接建 L1 六件套，没有先做融合适配 | **融合决策悬空**— creative 说了"灰骑士种子替代金手指"，但 L1-02 力量体系完全没体现这条 | 先走 Phase 0 融合适配，逐维标注"本维度的受融合影响点"，再建世界 |
| 数值体系只填战斗数字（攻击力/HP/等级），不做社会学建模 | **缺失防崩维度**— 没有人口密度/社会地位/破坏力对世界的冲击，写作时容易"XX级满地走" | Phase 4 必须包含静态金字塔（每层的人口 x 地位 x 破坏力 x 战斗张力） |
| 直接进 plot 设计事件，没有先规划主角全书能级跃迁路径 | **主角路径悬空**— plot 不知道主角在每卷该碰什么级别的对手 | 宪法锁定后先走 Phase 7 动态升级表，再交接 plot |
| 分析文档/策略文档用 YAML 格式 | **用户偏好冲突** — 用户要求用 Markdown 而非 YAML 作为文档格式规范 | 所有分析文档、模板说明、策略文档一律用 Markdown。YAML 仅限外部接口数据交换。不确定时优先 Markdown |
| 从 creative 收到 story-engine.md 后直接建 L1，没有先读 PRD.md 和素材储备池.md | **跳过加工坐标系** — PRD.md 是 creative v2.0 的核心新增产出，包含加工哲学和边界。不读 PRD = 建世界时没有「这本书的元素加工规则」参照 | Phase 0 前先读 PRD.md（确认加工坐标系）→ 读素材储备池.md（供世界观色块参考）→ 再读故事引擎.md（宪法约束）→ 然后进入融合适配 |

## 收到任务后，按顺序执行

### Phase 0：设计意图捕获 + 融合适配（creative→world 传递协议）

**Phase 0a：用户心智模型捕获**

> 在开始任何 L1 设定之前，先捕获用户对世界的顶层设计意图。
> 这个步骤不做→后续所有设定都是"agent 默认值"→用户大概率不满意。

先问用户三个问题：
1. **"你心中这个世界的核心矛盾是什么？"** — 确认世界宪法方向
2. **"力量体系的总体框架是几层几阶？"** — 确认数值结构（不是命名，是结构）
3. **"命名风格偏好什么方向？"** — 确认质感层方向（传统/原创/混合）

**⛔ 门禁：** 用户给了明确框架前不进 Phase 0b。如果用户说"你定吧"——至少把你的默认框架给用户过目。

**典型案例（2026-06-22 东方版深渊主宰）：**
```
agent 先给了 3 套花哨命名方案 → 用户说"太土了要传统" → 搜了文抄公 → 用户又说"凶灾荒劫适合高武" → 最终结构从 11 阶改了 6 阶
→ 7 轮对话的根因：没有先问结构再问命名
→ 正确做法：先问"几层几阶"（1轮）→ 再问"命名偏好"（1轮）→ 给 1-2 个对齐结构+偏好的方案（1轮）→ 定稿
```

**Phase 0b：融合适配**

**触发条件**：本作是跨界融合/同人二创（creative 有跨域素材蒸馏.md + 元素叠层表时强制执行）

**读什么**：
- creative 的 `PRD.md`（本书基本法，加工坐标系）
- creative 的 `故事引擎.md`（含融合决策）
- creative 的 `素材储备池.md`（背景素材可供世界观色块参考）
- creative 的 `跨域素材蒸馏.md`（含焊接点定义）
- `pop-trope-library/设定库/`（★NEW 设定创意池，先看PRD卖点匹配书目→再读L1文件）
- 如为二创：`references/derivative-gap-analysis.md`（在 expert-writer 下）

**做什么**：逐维回答"融合决策对本 L1 维度的影响是什么"

| L1 维度 | 层归属 | 受融合影响点 | 处理方式 |
|:--------|:-------|:------------|:---------|
| L1-01 世界蓝图 | 框架层 | 新元素是否改变了地理/位面结构？ | 标注新增/替换/删除 |
| L1-02 力量体系·机制段 | 框架层 | 新元素是否增加了并行成长线？ | 标注叠加/替换/冲突 |
| L1-02 力量体系·命名段 | 质感层 | 新元素是否带来了新的命名体系？ | 标注命名方向 |
| L1-03 历史与驱动力 | 机制层 | 新元素是否改变了核心矛盾？ | 标注新增势力冲突 |
| L1-04 物种与天赋 | 质感层 | 新元素是否带来了新种族？ | 标注新增/替换物种 |
| L1-05 势力格局 | 机制层 | 新元素是否插入了新势力？ | 标注新增势力及其位置 |
| L1-06 资源与物品 | 框架层 | 新元素是否引入了新资源/装备？ | 标注新物品体系 |
| L1-07 术语与文明底色 ★NEW | 质感层 | 新元素是否带来了新的术语体系？ | 标注新增术语及文明底色 |
| 静态金字塔 | 框架层 | 融合后金字塔层级是否改变？顶端是否展宽？ | 标注层级变化 |
| 动态升级表 | 框架层 | 融合后主角爬梯路径是否不同？ | 标注路径偏移 |

#### Phase 1b：质感组推演（质感层）

> ⚠️ **质感层不是打勾流程，是创作环节。**
> 质感层决定读者"感觉"这个世界是什么样的——每个术语、每个命名、每段文明底色都是读者直接体验的东西。
> 框架层是骨架，质感层是血肉。读者看不见骨架，只摸得到血肉。

**质感层产出质量门禁（不通过退回重做）：**

| # | 门禁 | 不合格表现 | 合格标准 |
|:-:|:-----|:---------|:---------|
| T1 | **命名原创性** | 等级名/功法名/术语直接照搬类型标配（如修真用"炼气筑基金丹元婴"、西幻用"魔法师战士牧师"） | 命名有本书独有的隐喻系统——即使借用了类型词，也必须有重新定义的内涵。至少3个核心术语是本书原创 |
| T2 | **命名哲学** | 只列名称+一句话含义，没有命名体系的设计逻辑 | 必须说明：这个世界的命名哲学是什么？（如"所有术语都来自冶金锻造"或"所有功法名都是古代刑名"）——命名共享同一个隐喻母题 |
| T3 | **文明底色深度** | "语言风格：冷峻简练"一句话 | 至少覆盖：语言风格（含2-3个语感示例句）/ 命名禁忌（不出现什么+为什么）/ 文化符号（3+个反复出现的意象）/ 社会价值观（这个世界的人相信什么） |
| T4 | **术语词典信息量** | 术语只有"术语名+含义"两列 | 每个术语至少含：含义 + 对应框架层规则 + 来源域 + 在叙事中的功能（这个术语在故事里干什么用） |
| T5 | **待命名清单清零** | L1-02机制段的"待命名机制清单"有未命名项 | Phase 1b 完成时，待命名清单必须全部清零——每个机制都有质感层的命名 |

### Phase 7：动态升级表（新增——world→plot 的桥梁）

**前置条件**：Phase 1-6 全部完成（世界参数已锁定）

**核心问题回答：**
- 主角在全书中从哪个力量层爬到哪个层？
- 中间经过几个跃迁台阶？
- 每次跃迁的触发条件是什么（外部事件/自我驱动/被卷入）？
- 每次跃迁后主角的社会地位如何重置？
- 每次跃迁后博弈层级怎么变化？
- 每次跃迁要付什么代价防止金字塔被冲垮？
- 每个阶段内的对手匹配规律是什么？

**产出格式：** `小说世界设定/动态升级表.md`

**建议结构：**

```
一、全书跃迁路径（起点层 → 跃迁1 → 跃迁2 → ... → 终点层）
二、每次跃迁详表
   | 跃迁节点 | 主角当前能级 | 触发事件 | 社会地位重置 | 新地图天花板 | 博弈层级变化 | 代价 |
三、各阶段对手匹配规则
   | 阶段 | 常规对手级 | 精英对手级 | BOSS级 | 上位NPC级 |
四、防崩机制（什么情况下允许临时越级？代价是什么？）
```

**门禁**：动态表未经用户确认不进 plot。

### 卷级节奏约束（★新增 — world→plot 的防崩协议）

**问题背景：** 2026-06-22 东方版深渊主宰案例中，phase 7 动态升级表定义了「全书跃迁路径：开脉未成 → 开脉圆满 → 洗髓 → 明劲 → ...」，但没有约束「每卷的能级跨度」。结果卷一 plot 设计了 50 章，主角从开脉未成→开脉圆满（仅半阶跨度），PRD 承诺「武侠层 ch1-ch30」完全落空。

**核心规则：**

```yaml
# Phase 7 必填产出
per-volume-span:
  vol-1:
    start: 开脉未成
    end: 洗髓以上       # 门禁：单卷≥2阶，不得原地踏步
    max_chapters: 55   # 门禁：同一大境界不超过 55章
  vol-2:
    start: 洗髓
    end: 暗劲以上
    max_chapters: 55
  vol-3:
    start: 暗劲
    end: 宗师/大宗师
    max_chapters: 55
  # ...后续卷类推
```

**产出格式：** 在 `小说世界设定/动态升级表.md` 末尾增加「六、卷级节奏约束表」

**门禁：**
1. 单卷主角能级跨度 < 1 完整小阶（如开脉未成→开脉圆满=半阶违规）→ 退回做节奏协商
2. plot 阶段接收时先验证：本卷设计章数是否超过动态升级表约定的 max_chapters → 超过退回
3. 如果 PRD 有明确的阶段承诺（如「武侠层 ch1-ch30」），以 PRD 为更高优先级

**WRONG 示例：**

| ❌ 错误 | ✅ 正确 |
|:--------|:--------|
| 动态升级表只写全书路径，不写每卷约束 → plot 50 章主角原地踏步 | 每卷写明起点终点和 max_chapters → plot 设计时自然收敛到合理节奏 |
| 动态升级表写得细但 plot 不验证 → 文档是文档，执行是执行 | plot Step 1 加载动态升级表后先校验本卷跨度是否达标，不达标退回 |

## 本阶段边界

### 本步做什么
- Phase 0: 融合适配 → Phase 1: L1 设定 → Phase 2: 稳定性 → Phase 3: 角色 → Phase 4: 数值（含静态金字塔）→ Phase 5: 起点 → Phase 6: 宪法 → Phase 7: 动态升级表
- 可参考 creative 的 `素材储备池.md` 作为世界观色块和背景设定的素材源头

### 本步不做什么
- 不重新发明故事引擎（那是 creative 的活）
- 不设计剧情或章节（那是 plot 的活）
- 不设计具体章级事件（那是 chapter-design 的活）
- 不重做 creative 的融合决策（Phase 0 只摊开，不推翻）

**越界检测**：对话出现"第X章的剧情/这个情节怎么走" → "这属于 plot 的范围，到那一步处理。现在先把世界参数锁定。"

## 异常与边界条件

| 场景 | 处理 |
|:-----|:-----|
| 宪法更新 | 标记受影响维度 → 重推 → 连锁重建下游 |
| L1 单篇修订 | 修改 → 重跑交叉关联 → 重跑稳定性 → 检查下游 |
| deconstructor 缺失 | 数值用平台默认节奏 |
| 角色卡模板缺失 | 按维度矩阵手写，不编造字段 |
| 宪法与 L1 不一致 | 先修 L1 → 重跑稳定性 → 再提取宪法 |
| 起点快照与宪法冲突 | 优先修正起点快照；若宪法需修订 → 退回 creative |
| 非跨界/非二创项目 | Phase 0 跳过，直接进 Phase 1 |
| 用户说数值直接从故事宪法推导 | 这是正确的设计方向——数值体系的节奏设计是为了服务叙事弧线，不是独立存在的数字表格。在 Phase 4 Part A（战斗数字）排期时应回查：每卷需要主角达到什么能级来配合那卷的冲突等级 |
