# 数值体系反拆方法论

> 正向构建（建世界观时造数值体系） → 见 pop-writer-world Phase 4 和 Phase 7
> 反向破译（拆已有小说时提取数值体系） → 见本文
> 两表框架在正反两个方向共享同一个底层逻辑，只是方向相反。

---

## 核心框架：两表法

数值体系反拆 = 两张表 + 一套范围声明。

### 表1：静态金字塔

**定义**：世界的力量阶层分布图——防止写崩的核心约束。

| 反拆追问 | 对应维度 |
|:---------|:---------|
| 这个层级有多少人？ | 数量/密度（万人级/千人级/个位数） |
| 这个层级在社会中什么位置？ | 社会地位（底层/中层/统治层/国家级） |
| 一个普通人遇到这个层级的人会怎样？ | 生产力/破坏力（能拆楼/能灭城/不可力敌） |
| 这个层级的战斗读起来什么感觉？ | 战斗张力（势均力敌/碾压/逃命/旁观） |

模板见：`templates/socioeconomic-pyramid.tpl.md`

### 表2：动态升级表

**定义**：主角能级提升时，作者让他接触层级/博弈层级/冲突规模同步升级但不破坏静态金字塔的机制。

| 反拆追问 | 对应维度 |
|:---------|:---------|
| 主角当前能级 vs 本层天花板差多少？ | 成长空间 |
| 主角在新地图的初始社会地位是多少？ | 社会重置 |
| 对手是否遵循对手匹配规律？ | 对手匹配 |
| 主角换地图是因为自己要走还是外部事件？ | 地图切换触发 |

模板见：`templates/dynamic-escalation-table.tpl.md`

## 方法论红线

### 红线1：范围声明（全书级 vs 单卷级）

拆了几卷就说几卷的话。

- 每份反拆文档开头/结尾标注范围声明
- 明确标注如果后续卷推翻此模式的预测条件

### 红线2：微观 vs 宏观混淆

- 把 combat stats 当成数值体系的全部是不够的
- 静态金字塔 = 社会政治经济学的数值建模
- 动态升级表 = 主角爬梯时的人脉/博弈/冲突升级机制

### 红线3：不把原文现象当成作者意图

- 现象 → 推测 → 标注推测强度（弱/中/强）

### 红线4：数值曲线从故事宪法推导，不是凭空设定

- 数值体系的节奏设计是为了服务叙事弧线
- 每卷需要主角达到什么能级来配合那卷的冲突等级
- 升级速度受叙事节奏控制，不只是数学曲线

## 反拆步骤

### Step 1：划定证据范围

明确基于哪些章节的数据在说话。

### Step 2：提取静态金字塔

1. 列出所有在证据范围内实际出场的实体，标注层级
2. 统计每个层级的出场频次
3. 推断人口密度估计
4. 标注社会地位和破坏力

### Step 3：提取动态升级表

1. 标记主角每跨一次地图时的能级
2. 新地图中主角的初始社会地位 vs 旧地图中的地位
3. 主角换地图的原因（自查/被卷入/外力破坏）
4. 主角跨越层级的方式（硬实力/外挂/智取/借势）
5. 每次跃迁的代价

### Step 4：标注范围声明

每份反拆输出都必须有范围声明。
