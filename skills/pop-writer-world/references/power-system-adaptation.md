# 力量体系适配指南

> 用途：将 combat_capability.tpl.yaml / L1-02.tpl.md 适配到不同题材
> 关键原则：**写 L1-02 前先确认用户对等级名的预期**，不凭假设动笔。

---

## 一、等级命名确认清单

写 `L1-02 力量体系.md` 之前，务必向用户确认：

```
□ 你认为这个世界的等级怎么分？（给用户自由描述空间）
□ 参考的是什么体系？（DND 3.5e / DND 5e / 修仙 / 自创）
□ 每个大段的名称是什么？
□ 每个大段的等级范围是多少？
□ 20级在你的体系中是"凡人巅峰"还是"超凡者起步"？
```

**为什么重要**：不同用户对"1级意味着什么"的理解不同。DND 3.5e 的 1 级=有职业的冒险者，修仙的炼气期=刚刚入道起点。猜错了方向就会像本轮会话一样被纠正两次。

## 三、combat_capability.yaml 模板适配

模板中的修仙示例（炼气→筑基）只是占位示例。适配时：

1. 用上表替换 `ranks[].name`
2. 调整 `min_attack` / `max_attack` 的数值范围——数值不重要，**相对比例**才重要
3. `breakthrough` 字段从本故事 L1-02 力量体系提取
4. `iconic_scene` 字段写本世界特有的标志性画面（不是套话）

### 检查清单

```
□ 段位名已经用户确认
□ min_attack 和 max_attack 的相对差距随等级递增（低阶跨度小，高阶跨度大）
□ breakthrough 条件从本故事的 L1-02 提取，不是套模板
□ iconic_scene 是独特的、有画面的（"一拳打碎青石" vs 通用描述）
```
