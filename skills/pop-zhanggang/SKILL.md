# pop-zhanggang · 章纲流

> 输入章纲 → 拆解冲突 → 填充细节 → 推动节奏 → 输出正文。每章3100-3500字。v1.1.0

## 做什么

| 输入 | 输出 | 下游 |
|:-----|:-----|:-----|
| 后续两章章纲 + 前文 | 第一章正文（3100-3500字）+ 章末钩子（为第二章铺垫） | 无（正文产出） |

## 怎么操作

> execution.mode: 两步串行 | 强保障：本 SKILL.md 由 host 层每次 run 强制注入 | 弱保障：steps/ + templates/ 需 agent 主动 readFile

| 步骤 | 做什么 | 产出 | step 文件 |
|:-----|:-----|:-----|:----------|
| Step 1 | 前文分析（风格/矛盾/主线/情绪/关键信息/金手指状态） | 前文分析卡 | `steps/step1.md` |
| Step 2 | 拆解框架（4段式骨架）+ 填充细节 + 节奏与收尾 → 写正文 | 本章正文+章末钩子 | `steps/step2.md` |

### 4段式骨架

| 段 | 字数 | 内容 |
|----|------|------|
| 开篇 | 650-750字 | 渲染场景氛围，交代人物行动动机 |
| 发展 | 950-1050字 | 探索/行动过程，1-2个小阻碍 |
| 高潮 | 850-950字 | 达成核心目标，同步引发新冲突 |
| 结尾 | 650-750字 | 即时反应，留悬念钩子 |

## 红线

1. **读取协议**：读取 skill 文件用 `Get-Content -Encoding UTF8 -Raw`，禁用 Read 工具
2. 每章结尾必须是期待感对话钩子——禁止事件完结或情绪落地
3. 正文开头完美衔接前文结尾——禁止与前文重复剧情
4. 描写每章控制在3-8个，每个≤30字——拒绝过多形容词/比喻/拟人
5. 禁止AI叙述词和总结性/展望性内容——输出精炼，保持阅读感
6. 创作前必须先做前文分析——不分析直接写正文视为违规
