# Step 2 · 拆解框架 + 填充细节 + 节奏收尾 → 写正文

> 3步逻辑一气呵成，输出本章正文 + 章末钩子。

---

## 2a. 第一步：拆解框架（占30%）

**做什么**：从章纲提炼3要素，按4段式分配篇幅。

**怎么做**：
1. 从用户提供的章纲中提炼3要素：
   - **核心冲突/事件**：本章要解决什么问题？
   - **人物目标**：主角在本章要达成什么？
   - **场景限定**：本章发生在哪里？什么时间？
2. 按4段式骨架分配篇幅（参考 `templates/4段式骨架.tpl.md`）：

| 段 | 字数 | 内容 |
|----|------|------|
| 开篇 | 650-750字 | 渲染场景氛围，交代人物行动动机（为何出现在该场景） |
| 发展 | 950-1050字 | 描写探索/行动过程，加入1-2个小阻碍（寻找时的意外、环境干扰），逐步接近核心目标 |
| 高潮 | 850-950字 | 达成核心目标（找到关键物品/触发关键事件），同步引发新冲突（突发危险、意外人物出现） |
| 结尾 | 650-750字 | 刻画人物即时反应，留下悬念钩子（未解决的危险、线索升级、人物反转） |

**总字数目标**：3100-3500字

**产出**：4段式骨架（内部参考，不输出）

---

## 2b. 第二步：填充细节（占60%）

**做什么**：在4段式骨架内填充场景、人物、情节细节。

**怎么做**：

### 场景描写
- 结合场景限定，从**听觉、视觉、触觉/嗅觉**多维度刻画
- 用环境烘托情绪（如"雨夜旧仓库"需写雨声、光线、霉味等）
- **描写总量控制**：每章3-8个描写，每个≤30字

### 人物刻画（三件套）
| 手法 | 说明 |
|------|------|
| 动作链 | 拆解连续动作，避免概括性描述（如"他走过去拿起来"→ 拆成3-4个连续动作） |
| 心理活动 | 结合事件加入回忆、疑惑、情绪波动 |
| 简短对话/独白 | 塑造人物形象，推动剧情 |

### 情节打磨
- 所有细节围绕**核心冲突**和**人物目标**展开
- 拒绝无关凑数内容
- 加入适当剧情对话（在合适的剧情里）

**产出**：正文草稿（内部参考）

---

## 2c. 第三步：节奏与收尾（占10%）

**做什么**：控制节奏，收尾留悬念。

**怎么做**：

### 节奏控制
- **慢描写**（场景、心理）与**快行动**（冲突、探索）交替
- 避免平铺直叙
- 情绪值强烈，有吸引力

### 结尾要求（红线）
1. **必须留下"未完成"悬念**：
   - 突发危险
   - 线索升级
   - 人物反转
   - 期待感对话钩子
2. 禁止事件完结或情绪落地
3. 强制引导读者阅读下一章

### 章末钩子写法
- **对话钩子优先**：用一句有期待感的对话结尾
- 钩子必须与第二章的章纲呼应——为第二章留好铺垫

**产出**：正文定稿

---

## 2d. 章节名生成

**做什么**：创作一句话章节名。

**怎么做**：
1. 一句话，**10字以上**
2. 展现本章的：
   - 噱头
   - 或爽点
   - 或反差隐喻（可玩梗）
3. 放在正文开头

**示例**：
> 「这小子居然把刀架在市长脖子上」
> 「他随手一摸，摸出个活人」
> 「本以为是青铜，结果是王者」

---

## 2e. 输出格式

**只输出以下内容**（前文分析/创作思路/下一章思路不输出）：

```
【第N章正文 | 为第N+1章铺垫：XXX（一句话说明铺垫了什么）】

章节名

（正文内容，3100-3500字，衔接前文，章末对话钩子收尾）
```

**必须输出**：
- 本章是第几章
- 为第几章做好了什么铺垫（一句话提醒用户）

**禁止输出**：
- 前文分析
- 本章创作思路
- 下一章简要思路
- 总结性/展望性内容
- AI叙述词

---

## 2f. 自检清单（写完后内部过一遍，不输出）

- [ ] 开头是否完美衔接前文结尾？
- [ ] 是否有与前文重复的剧情？
- [ ] 章末是否是期待感对话钩子？
- [ ] 描写是否控制在3-8个，每个≤30字？
- [ ] 是否减少了形容词/比喻/拟人？
- [ ] 情绪值是否够强烈？
- [ ] 是否检查了剧情漏洞/道具错误/系统奖励错误？
- [ ] 章节名是否≥10字且有噱头/反差？
- [ ] 是否输出了"为第N+1章铺垫：XXX"？
- [ ] 文风是否与前文一致？
- [ ] 是否符合年代背景和现实社会规则？
