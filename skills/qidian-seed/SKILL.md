# seed

> 故事基底构建器。从零构建一个番茄小说项目——种子（世界观/角色/金手指）+ 世界展开（主角引擎/角色储备池/世界圣经）。一个skill完成从创意到可操作设定全链路。

---

## 做什么

输入：用户的创意想法/题材方向/目标读者群
输出：4个文件

| 产出文件 | 阶段 | consumer | 核心内容 |
|---------|------|----------|---------|
| `seed-种子文档.md` | 阶段一 | plot-design | 核心卖点/角色地基/世界观/金手指/流派/三阶段走向/故事弧 |
| `主角引擎.md` | 阶段二 | plot-design | 主角9字段行为模式表 |
| `角色储备池.md` | 阶段二 | plot-design | S/A/B/C四级角色 |
| `世界圣经.md` | 阶段二 | plot-design（按需） | 世界底座——舞台/历史/神系/力量体系/社会结构/势力格局 |

与下游的对接：plot-design加载种子文档+主角引擎+角色储备池+世界圣经，设计卷纲→幕纲→施工卡；prose-render直接加载流派skill。

---

## 核心理念

**种子建立有张力的世界，世界展开把张力转化为可操作的设计锚点。**

种子说"阵营：守序邪恶"——世界展开回答"守序邪恶在缺钱时偷、在获取情报时威胁、在利用完人后抛弃"。
种子说"半精灵在贫民区被歧视"——世界展开回答"哪些NPC会歧视、怎么歧视、主角能怎么利用"。

不做的事：
- 不编排剧情（plot-design的活）
- 不写笔触风格（流派skill的活）
- 不预设危机线（plot-design的活）

## 怎么操作（SOP骨架）

> **execution.mode**: formal（必读输入齐全落盘）/ draft（有缺口可补全继续）/ trial（快速试做不落盘）。检测 current-state.md 是否存在，不存在→formal；存在但缺字段→draft；用户说"试做"→trial。

### 阶段一：种子构建（Step 0-7）

### Step 0：脑洞碰撞

> 种子的质量上限由脑洞决定。线性提炼用户输入=不够精彩。必须碰撞出多个候选，PK后取最优。

**0a. 加载赛道文档**
根据用户题材，加载 `references/{赛道}剧情模式.md`（来自plot-design）。了解赛道的冲突模式/危机偏好/燃料类型。

**0b. 生成3个候选概念**
从用户创意出发，向3个不同方向推演。每个候选必须有差异化的核心钩子——不是同一思路的微调，是不同的"这本书好看在哪"。

| 候选 | 一句话卖点 | 核心钩子 | 主角主动方式 | 世界压迫 | 最容易写崩的点 |
|-----|---------|---------|------------|---------|-------------|

差异化维度（3个候选至少在3个维度上各自不同）：
- 核心爽感不同（打脸/碾压/经营/解谜/逆袭）
- 主角驱动不同（求生/复仇/野心/责任/好奇）
- 世界压迫不同（阶级/天赋/势力/规则/资源）

**0c. 候选PK + 推荐**
逐项对比：兴奋度 / 可持续性(250章+) / 独特性 / 执行难度。推荐一个 + 吸收败者强点。

**质量门禁**：如果没有一个候选让你"想立刻读到正文"，回到0b重新生成。不要用"还行"的脑洞开局——一本书250章，脑洞不精彩后面全白搭。

### Step 1：核心卖点定位

从PK胜出的候选概念中提炼：
- **一句话卖点**：用一句话说清"这个故事讲什么+为什么好看"
- **元爽点**：读者追读的核心驱动力（不是"主角变强"，是具体的爽感机制）
- **追读理由**：读者翻到下一章的具体理由
- **爽点外显方式**：爽感在正文中怎么呈现（面板/参照物梯子/打脸/信息差碾压）
- **独特性声明**：同赛道100本书里，读者为什么选这本？一句话说清差异化
- **核心悬念**：贯穿全书250章的一个问题（不是"主角能不能变强"，是让读者持续好奇的具体谜题）

### Step 2：角色地基

不只建主角——还要建对手和关系动力学。一本书250章，主角需要一个能持续施压的对手体系。

| 项 | 要求 |
|----|------|
| **主角身份** | 一句话：社会角色+核心特质 |
| **主角欲望** | 一句话：具体到可操作目标 |
| **主角伤口** | 一句话：内心脆弱点 |
| **核心对手身份** | 一句话：对手是谁+为什么与主角形成结构性冲突 |
| **核心对手欲望** | 一句话：对手要什么（必须与主角欲望形成对立，不是"坏人"） |
| **关系动力学** | 主角与对手/盟友之间的张力来自什么？（利益冲突/理念对立/信息差/信任博弈） |

### Step 3：世界观构建

世界观是250章的弹药库，不是几张设定卡。每条世界规则必须能回答："这条规则能反复产生什么类型的冲突？"

| 设定项 | 内容 | 冲突潜力 | 主角可利用的缝隙 |
|--------|------|---------|---------------|
| 背景设定 | 世界怎么变成现在这样的 | 这条背景会产生什么结构性矛盾？ | |
| 力量体系 | 主角能做什么/不能做什么 | 力量体系本身能产生什么决策困境？ | |
| 社会结构 | 谁在上谁在下，为什么 | 阶级/势力之间的矛盾点在哪？ | |
| 地理分区 | 故事发生的舞台层次 | 不同区域之间的规则差异能产生什么冲突？ | |
| 金手指 | 主角的独特优势是什么 | 金手指本身就是冲突发动机（有代价/有博弈/有风险） | |

**必须额外覆盖以下系统**（每个标注主角可利用的缝隙）：
- **经济系统**：钱从哪来、到哪去、主角怎么获取（不是"有金币"，是具体的财路和分配机制）
- **政治系统**：谁定规则、规则怎么压迫主角、主角怎么利用规则缝隙
- **文化规范**：什么行为在这个世界是正常的但读者会觉得震惊（制造代入落差）
- **信息不对称**：什么信息是分层流动的（主角能利用的信息差在哪）

**红线**：世界设定只有"地理+势力+力量体系"=不合格。缺经济/政治/文化=废设定。

### Step 4：金手指巧思设计

| 设计项 | 内容 |
|--------|------|
| 金手指是什么 | 一句话 |
| 巧思维度 | 6种至少选1：决策困境/使用博弈/风险代价/信息差武器/自生长能力/成长节奏控制 |
| 反模式检查 | 4种必须避免：纯翻译器/纯正向循环/无使用场景限制/成长速度过慢 |
| 与世界/角色的焊接 | 金手指在世界观的哪条冲突潜力上产生决策困境？和角色的哪个欲望/伤口形成张力？ |

### Step 5：流派选择 + 笔触DNA声明 + 三阶段走向

**5a. 流派选择**

| 项 | 内容 |
|----|------|
| 流派目录 | 如 `prose-render-dndlike/`。如无匹配流派，留空使用通用prose-render |
| 笔触DNA模板 | 从流派目录的 `dna/` 中选择 |
| 风格关键词 | 3-5个词概括本书风格 |
| 数值表达方式 | 面板弹出/参照物梯子/感官化——选哪些 |
| 例外条款 | 本项目中哪些笔触DNA红线让位于种子需求 |

**5b. 三阶段走向声明**

| 项 | 内容 |
|----|------|
| 开局阶段 | 凡人期/超凡期（一般凡人期开局） |
| 起点等级 | 主角开局等级 |
| 终点阶段 | 凡人期结束/超凡期结束/神明级 |
| 终点等级 | 主角全书最终等级段 |
| 神明级定位 | 主要舞台/背景威胁/不涉及 |

三阶段走向决定Step 10世界圣经的产出深度。

### 种子压力测试（阶段一收尾）

> 种子是250章的弹药库。如果弹药不够，后面全在空转。必须在进入阶段二前通过以下检查：

- [ ] **脑洞兴奋度**：这个概念让你"想立刻读到正文"吗？如果不上不下，回到Step 0重碰撞。
- [ ] **冲突密度**：从种子能直接推导出50章以上的不同冲突吗？（不是"主角变强"这种泛泛的，是具体的"谁和谁因为什么冲突"）
- [ ] **世界纵深**：世界有≥3个可反复利用的独立冲突源吗？（经济/政治/势力/文化各算一个维度）
- [ ] **角色生态**：主角+对手+至少2个关键关系都有地基吗？
- [ ] **长线续航**：有≥5条跨卷伏笔吗？每卷方向和卷末质变都清晰吗？
- [ ] **赛道辨识度**：同赛道100本书里，这个种子的差异化在哪？

任一不满足：针对性回到对应Step补强，不进阶段二。

### 阶段二：世界展开（Step 8-10）

> 阶段一完成后连续执行，无需用户确认。从种子文档展开为3个设定文件。

### Step 8：主角引擎

从种子的角色地基出发，展开为9字段行为模式表。

| 模块 | 必须回答 |
|------|---------|
| 表层欲望 | 第一卷想得到什么（具体到可操作） |
| 深层需要 | 长篇真正缺什么（内心层面） |
| 恐惧 | 最怕失去什么 |
| 底线 | 什么事绝不做，或做了会付出代价 |
| **行动偏好** | 暴力/交易/欺诈/规则漏洞/经营/救援/信息差——面对问题时偏好哪种方式 |
| **第一章主动动作** | 弱势下能主动做的第一件事（主角主动发起，不是被动触发） |
| 决策缺陷 | 什么选择会惹出更大麻烦 |
| 第一卷变化 | 从什么人变成什么人 |
| 长篇穿透方式 | 主角凭什么逐层突破世界层级 |

**红线**：
- "行动偏好"必须具体——不是"用智慧"，是"偏好用信息差操控/威胁/偷窃"
- "第一章主动动作"必须是主角主动发起的——不是"面板激活"（被动触发），是"用面板检定追踪"（主动使用）
- 阵营标签必须转化为操作性定义

输出到 `主角引擎.md`。

### Step 9：角色储备池

从主角引擎+种子世界观出发，生成S/A/B/C四级角色。

| 等级 | 数量 | 信息量 | 是否逐人建卡 |
|------|------|--------|------------|
| S 核心角色 | 1-3人（含主角） | 120-200字/人 | 是 |
| A 卷级主角色候选 | 3-6人 | 80-120字/人 | 是 |
| B 功能角色槽位 | 6-12个槽位 | 40-80字/槽位 | 槽位即可 |
| C 路人反馈模板 | 4-8类 | 20-40字/类 | 否 |

**S/A角色必填字段**：来源依据/角色原型参考（2-3个）/稳定锚点/欲望与压力/反应模式/说话方式/可制造事件类型/主角可利用点

**红线**：
- S/A角色必须有2-3个原型参考——不是凭空捏造
- 原型只迁移气质/反应/处境张力/冲突方式，禁止照搬身份/专有设定/剧情链/台词
- "反应模式"必须是操作性定义
- B级是槽位不是人物

输出到 `角色储备池.md`。

### Step 10：世界圣经

从种子世界观+三阶段走向出发，展开为完整世界底座。参考 `templates/世界圣经-模板.md`。

| 模块 | 内容 |
|------|------|
| 3a 世界舞台 | 舞台规模/地理分区/文明水平 |
| 3b 历史纵深 | 纪元划分/关键历史事件/当前时代特征 |
| 3c 神系 | 神明/关系/干预方式/对凡人影响（如三阶段走向涉及神明级） |
| 3d 力量体系（核心） | 等级框架（1-25+，含阶名/定性/社会角色/稀缺度）/属性标尺（具象锚点）/各等级段战斗表现/成长路径 |
| 3e 三阶社会结构 | 凡人期社会/超凡期社会/神明级社会——建到三阶段走向声明的终点阶段 |
| 3f 势力格局 | 跨三阶主要势力（含所属阶段/首脑水平/能制造的事件） |
| 3g 资源物品 | 稀缺原因/获取方式/可数反馈/可驱动行动 |
| 3h 核心矛盾 | 世界级矛盾 |

**红线**：
- 力量体系必须标注各等级段战斗表现
- 三阶社会结构必须建到三阶段走向声明的终点阶段
- 属性标尺每个关键节点必须有具象锚点
- 势力格局必须标注"能制造的事件"
- 产出深度必须与三阶段走向声明匹配

输出到 `世界圣经.md`。

## 红线

1. **脑洞碰撞红线**：种子必须经过≥3个候选PK，不能从用户输入直接提炼。线性提炼=不合格。
2. **冲突潜力红线**：每条世界设定必须标注"能产生什么冲突"。不能产生冲突的设定=废设定。
3. **金手指巧思红线**：金手指必须产生决策困境或剧情发动机。纯翻译器/纯正向循环=不合格。
4. **角色地基红线**：主角+核心对手都必须有具体到可操作的欲望。只有主角没有对手=不合格。
5. **世界纵深红线**：世界设定必须覆盖经济/政治/文化系统，只有地理+势力+力量体系=不合格。
6. **不预设危机红线**：种子不编排具体危机线。危机设计是plot-design的职责。
7. **主角引擎红线**：阵营标签必须转化为操作性定义。
8. **角色储备池红线**：S/A角色必须有原型参考和反应模式操作性定义。
9. **世界圣经红线**：力量体系必须标注各等级段战斗表现；三阶社会结构必须建到终点阶段；属性标尺每个关键节点必须有具象锚点。
10. **三阶段匹配红线**：世界圣经产出深度必须与三阶段走向声明匹配。
11. **职责边界红线**：只写弱剧情耦合的稳定锚点。不写出场章节/不写剧情弧线/不写具体冲突。
12. **种子深度红线**：种子必须通过种子压力测试6项检查。不通过=不进阶段二。
13. **长线伏笔红线**：种子必须有≥5条跨卷伏笔。只有3句话走向=不合格。
