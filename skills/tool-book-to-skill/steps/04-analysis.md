# Step 3-4: 分析结构 + 询问用途

## 第四步：分析结构 + 询问用途

读取 full_text.txt 前 8000 字符，识别：书名/作者、章节结构、核心主题。

询问用途：

> "这个 Skill 主要帮你做什么？"
> 1. 工作时应用作者的框架
> 2. 用作者的心智模型思考
> 3. 查阅具体章节和概念
> 4. 以上全部

- 只选 3 → `DEPTH=reference`（精简快查）
- 含 1/2/4 → `DEPTH=study`（深度，含实例和推理）

## 模式 2：仅分析

> 用户说"先分析"、"只提取"、"我先看看再生成"时走此模式。

**流程：** 执行 Step 0-3，产出结构化提取报告后停止。
**输出：** 分析报告（框架、原则、技法清单）

## 模式 3：从分析结果生成

> 用户已有分析笔记或之前运行过仅分析。

**流程：** 跳过 Step 0-3，用已有分析作为输入，执行 Step 4-9。
**输出：** 从已有分析生成 Skill 文件。
