## 执行流程

按顺序执行 Phase 1 -> Phase 2 -> Phase 3。每一步的详细 SOP 见对应 `steps/` 文件。

```
Phase 1 (6问确认) -> Phase 2 (规划+二次确认) -> Phase 3 (逐章创作)
                                                        |
                                               每5章 -> 节奏报告
```

---

## 三大黄金法则

1. **展示而非讲述** -- 用动作和对话表现，不要直接陈述
2. **冲突驱动剧情 + 情绪过山车** -- 每章必须有冲突，每3章完成一次"压-小扬-压-爆"循环
3. **悬念承上启下** -- 每章结尾必须留下钩子，且钩子类型要轮换

## 红线

| # | 红线 | 说明 |
|:-:|:-----|:-----|
| 1 | 6问未完成不进 Phase 2 | 任一问题用户未选择 -- 必须追问至有答案。不可跳过或代填 |
| 2 | Phase 2 产出不全不进 Phase 3 | 5项产出（大纲/人物档案/情绪曲线/黄金开篇三选一/用户确认）缺一不可 |
| 3 | 每章必须 2200-2800 字 | 低于2200需扩充，高于2800需精简。不达标不交付 |
| 4 | 每章结尾必须有钩子 | 钩子类型必须轮换，禁止连续3章使用同一类型 |
| 5 | 每5章必须生成节奏报告 | 报告不可跳过。内容包含情绪曲线/爆点密度/争论点预测 |
| 6 | 质量自查分数 < 60 不交付 | 使用 quality-checklist.md 逐项检查，未达标退回重写 |

## Drop Check

> 每阶段完成后逐项勾检，确保无遗漏。

### Phase 1 Drop Check

```
□ 情绪标签已记录
□ 题材已记录
□ 主角设定（含职业/身份）已记录
□ 核心反差点已记录
□ 核心冲突已记录
□ 章节数量已确认
□ 全部6问用户已选择（无一跳过或代填）
```

### Phase 2 Drop Check

```
□ 项目文件夹已创建
□ 00-大纲.md 已生成（含 TODO list）
□ 01-人物档案.md 已生成
□ 02-情绪曲线.md 已生成（含钩子类型预分配）
□ 03-黄金开篇.md 已生成（三个版本）
□ 规划摘要已向用户展示
□ 用户已确认全部规划内容
```

### Phase 3 Drop Check（单章）

```
□ 00-大纲.md 已读取，本章标记为"进行中"
□ 上一章摘要和结尾钩子已读取
□ 02-情绪曲线.md 已读取，本章情绪位置已确认
□ 开头钩子已设计（前20%有即时冲突）
□ 3-5个场景已规划
□ 正文已撰写，字数在 2200-2800 区间
□ 结尾钩子已设置，类型未与前三章重复
□ 连贯性已检查
□ AI味已去除，番茄味已增强
□ 质量自查分数 > 60
□ 字数脚本已运行确认
□ 章节摘要已写入 00-大纲.md
□ 00-大纲.md TODO list 已标记"完成"
□ 钩子类型已录入 02-情绪曲线.md
```

### Phase 3 Drop Check（每5章）

```
□ 节奏报告已生成（情绪曲线/爆点密度/争论点预测）
□ 已询问用户数据反馈
□ 已根据反馈调整后续大纲
```

## ❌ WRONG 示例

| # | 错误 | 正确 |
|:-:|:-----|:-----|
| W1 | 6问未全部完成就跳过 Phase 1 进入 Phase 2 规划 | 任一问题用户未选择 — 必须追问至有答案。不可跳过或代填 |
| W2 | 章节字数不足 2200 字直接交付，不检查字数红线 | 每章必须 2200-2800 字。低于 2200 需扩充，高于 2800 需精简 |
| W3 | 连续 3 章使用相同类型的悬疑钩子，读者产生疲劳 | 钩子类型必须轮换，禁止连续 3 章使用同一类型 |

## 异常与边界条件

| 场景 | 触发条件 | 处理动作 |
|:-----|:---------|:---------|
| 用户中途改变题材/情绪标签 | Phase 2/3 用户说"换一个方向" | 退回 Phase 1 重新6问 + 重做规划。不打补丁，不局部修改 |
| 用户自定义章节数 | Phase 1 Step 1.6 用户输入"自定义" | 询问具体章节数，确认后记录。需告知字数预估 |
| 大纲与情绪曲线冲突 | Phase 3 写前分析发现情绪曲线位置与大纲剧情不吻合 | 暂停创作，先修正 02-情绪曲线.md，再继续 |
| 连续钩子类型重复 | Phase 3 发现前三章使用同一钩子类型 | 更换钩子类型，修正 02-情绪曲线.md 中的预分配 |
| 字数脚本执行失败 | Phase 3 Step 3.4 运行 check_chapter_wordcount_tomato.py 报错 | 手动统计字数，确保在区间内。标注"脚本不可用，手动统计" |
| 质量自查分数持续 < 60 | Phase 3 连续2次自查不达标 | 暂停本章，回溯上一章是否埋了坑或节奏问题。修正后再重写本章 |
| 用户数据反馈与预设冲突 | Phase 3 每5章反馈显示高人气配角非计划重点 | 建议为用户偏好的配角加戏，调整后续大纲，但保留主线结构 |
| references 文件缺失 | Phase 2/3 引用 references/ 下的文件找不到 | 使用内置默认逻辑继续（如 chapter-template 缺失则用通用章节结构）。产出标注"无 reference 文件" |
| 用户跳过规划要求直接写 | Phase 1 之后用户说"直接写，不需要规划" | 强制执行 Phase 2 最小集：至少产出大纲和人物档案。告知"无规划创作容易崩" |

**原则**：异常先告知用户，再按规则处理。绝不静默跳过或编造数据。

## 字数检查

使用 `scripts/check_chapter_wordcount_tomato.py`：

```bash
# 检查单个章节
python scripts/check_chapter_wordcount_tomato.py novels/小说名/第01章.md

# 检查所有章节
python scripts/check_chapter_wordcount_tomato.py --all novels/小说名/
```

低于2200字需扩充，高于2800字需精简。
