# Step 6：Draft v1 — SOP

## 第一步：撰写正文

**写作规则：**

| 规则 | 说明 |
|:-----|:------|
| 先回答问题，再展开细节 | 前2段内回答用户核心问题 |
| 关键词自然出现 | 不用固定密度硬规则 |
| 不确定信息加限制语 | 或直接删除 |
| 时间敏感性标注 | 产品/价格/年份/版本标明时效性 |
| 禁止占位符 | 不输出 TODO/待补充/这里可加案例 |
| 英文自然 | 避免中式英语/过度模板腔 |
| 段落可网页阅读 | 短段+强信息密度+清晰小标题 |
| 结尾收束 | 给出自然下一步动作或总结 |

**格式禁用命令：** 「—」破折号禁止使用 → 用逗号/冒号/括号替代。

## 第二步：质量基线

- 标题具备博客可读性（不是关键词列表）
- 首段可独立作为摘要阅读
- H2/H3层级清晰，适合直接贴入CMS
- FAQ答案简洁直接，不重复正文
- Meta description 像真实站点的文案
- 全文字章风格统一
