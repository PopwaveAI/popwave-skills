# Step 02 -- 执行联网操作

> 根据 Step 01 的类型判断，走对应的执行路径。

---

## 路径 A：简单搜索

1. WebSearch 执行查询（单次，不重复变体搜索）
2. 如有具体 URL -- WebFetch 获取全文
3. WebFetch 失败 -- 尝试 curl/wget 下载
4. 输出内容必须附来源 URL

---

## 路径 B：真实浏览器

1. 加载 tool-web-access skill
2. 启动 CDP Chrome
3. 根据用户指令执行访问/登录/抓取
4. 操作结束后关闭浏览器，清理临时数据

---

## 路径 C：文件下载

1. 确认文件 URL 可公开访问
2. curl/wget 下载到指定路径
3. 校验文件完整性

---

## 门禁

| 门禁 |
|:------|
| 路径 A：搜索变体重复 1 次以上 -- 停止，告知用户换关键词 |
| 路径 B：CDP Chrome 启动失败 -- 降级为 WebFetch，标注"浏览器不可用" |
| 路径 C：下载文件体积异常（过小）-- 重新下载一次，仍失败则告知 |
