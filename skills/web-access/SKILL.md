# Web Access — 联网操作统一入口

> 所有需要联网的操作都必须经过此 skill。搜索、爬取、登录、社交媒体、动态页面——不走此 skill 的联网操作一律禁止。

---

## 速查表：用户意图 → 执行方式

| 用户说什么 | 执行方式 | 使用工具 | 备注 |
|:-----------|:---------|:---------|:-----|
| "搜一下X" / "查资料" / "搜索" | **搜索引擎查询** | WebSearch | 简单信息查找，无需渲染 |
| "打开XX网页" / "看看这个页面" | **网页抓取** | WebFetch | 静态 HTML/Markdown 页面 |
| "抓取小红书/微博/推特的内容" | **真实浏览器 + 登录态** | web-access (CDP Chrome) | 必须登录，走真实浏览器 |
| "帮我登录XX网站" / "表单操作" | **真实浏览器交互** | web-access (CDP Chrome) | 需要模拟人为操作 |
| "这个网站内容加载不出来" / "动态渲染" | **真实浏览器渲染** | web-access (CDP Chrome) | SPA / JS 动态渲染页面 |
| "下载这个PDF/文件" | **HTTP 直连下载** | curl / wget | 仅限可公开访问的文件 |

## ❌ 质量红线

| # | 红线 | 说明 |
|:-:|:-----|:-----|
| ❌1 | **不走 web-access 的联网操作** | 所有联网必须走此 skill。直接调用 WebSearch/WebFetch 但未以该 skill 为入口 → 退回 |
| ❌2 | **登录态信息明文存储/泄露** | 账号密码、cookies、token 不得写入日志、输出到终端或在对话中展示原文 |
| ❌3 | **对无法访问的页面反复重试** | 首次失败后至多重试 1 次。连续失败 → 告知用户并提供替代方案，不陷入死循环 |
| ❌4 | **爬取频率过高触发反爬** | 批次请求间隔 ≥2 秒。反爬触发 → 立即停止 + 切换策略（加延时 / 换UA / 手动引导） |
| ❌5 | **用 curl/wget 绕过有登录/验证的页面** | 需要登录或验证的页面必须使用真实浏览器。curl/wget 只用于公开静态资源 |
| ❌6 | **获取的内容不标注来源** | 每次输出搜索结果/网页内容必须附 URL 来源 |

## 核心流程（3 步 SOP）

### 第一步：判断联网类型

用户说出联网需求后，先判断属于哪种类型：

| 类型 | 工具 | 条件 |
|:-----|:-----|:-----|
| **简单搜索**（查新闻、找资料、单页信息） | WebSearch / WebFetch | 无需登录，页面可公开访问 |
| **真实浏览器**（社交媒体、登录态、动态渲染、表单交互） | web-access CDP Chrome | 需要登录 / JS 动态渲染 / 反爬严格 |
| **文件下载**（PDF、ZIP等公开文件） | curl / wget | 可公开访问，URL 已知 |

❌ 门禁：判断为"真实浏览器"类型但尝试用 WebFetch/curl 绕过 → 退回，必须启动真实浏览器。

### 第二步：执行联网操作

**简单搜索流程：**
1. WebSearch 执行查询（单次，不重复变体搜索）
2. 如有具体 URL → WebFetch 获取全文
3. WebFetch 失败 → 尝试 curl/wget 下载
4. 输出内容必须附来源 URL

**真实浏览器流程：**
1. 加载 web-access skill
2. 启动 CDP Chrome
3. 根据用户指令执行访问/登录/抓取
4. 操作结束后关闭浏览器，清理临时数据

**文件下载流程：**
1. 确认文件 URL 可公开访问
2. curl/wget 下载到指定路径
3. 校验文件完整性

### 第三步：输出与标注

- 输出内容必须标注来源 URL
- 搜索不到 → 明确告知"未找到"，不编造
- 页面不完整 → 标注"页面加载不完整"

## ❌ 错误示例

### WRONG 1：动态渲染页面用 WebFetch 硬拉

```
用户：帮我看看这个SPA页面里加载的数据
Agent：WebFetch 拉取 → 拿到一堆空壳 HTML，JS 渲染的内容全没有
❌ 错误：SPA 页面的内容是 JS 动态渲染的，WebFetch 只能拿到空骨架
✅ 正确：判断为动态渲染类型，启动真实浏览器打开页面，等 JS 渲染完成后再提取内容
```

### WRONG 2：需要登录的页面用 curl 下载

```
用户：帮我下载我订阅的那个付费报告
Agent：curl 直接 GET → 服务器返回 302 到登录页 / 返回"请先登录"
❌ 错误：付费/登录内容不能用 curl/wget 绕过鉴权
✅ 正确：提示用户需要登录，使用真实浏览器模拟登录操作，或要求用户提供 cookies/token
```

### WRONG 3：搜索失败后不告知用户，静默编造

```
用户：帮我查一下XX产品的市场份额
Agent：搜索没找到结果 → 自己编了一个数字"XX产品市场份额约12%"
❌ 错误：找不到数据就编造，用户拿到的是虚假信息
✅ 正确：告知用户"未搜索到该产品的市场份额数据"，询问是否要换关键词或换个数据源
```

### WRONG 4：反爬触发后继续猛爬

```
用户：爬取这个列表的所有 100 页数据
Agent：连续快速请求 → 第 5 页时返回 429 / 验证码
Agent：继续重试 → 被封 IP
❌ 错误：反爬触发后不切换策略，导致 IP 被封
✅ 正确：检测到反爬 → 立即停止当前批次，告知用户，切换策略（加延时 / 换UA / 换代理 / 人工介入）
```

## 异常与边界条件

| 场景 | 触发条件 | 处理动作 |
|:-----|:---------|:---------|
| **WebFetch 返回空/乱码** | 页面被屏蔽、需登录、或反爬拦截 | 先尝试 curl/wget 降级。若仍失败，提示用户"该页面无法直接访问，可能需要登录或验证"。不反复重试 |
| **CDP Chrome 无法启动** | 真实浏览器启动失败（端口占用/Chrome 未安装/沙箱冲突） | 输出错误信息，降级为 WebFetch（标注"浏览器不可用，使用 HTTP 直接抓取"） |
| **搜索结果不相关/无结果** | 用户搜索意图不明确，搜索引擎返回 0 相关结果 | 告知用户"未搜索到相关结果"，建议换关键词。不编造，不使用近似内容冒充 |
| **目标网站有反爬/频率限制** | 请求返回 429/503 或出现验证码 | 立即停止当前请求。若可降级 → 使用 WebFetch + 延时 ≥5s；若需登录 → 转真实浏览器。告知用户反爬状态 |
| **下载的文件损坏/不完整** | 文件下载完成后体积异常（过小）或格式无法打开 | 重新下载一次。仍失败 → 告知用户文件可能已损坏，提供来源 URL 供用户手动下载 |
| **用户提供的 URL 无法访问**（404/连接超时） | WebFetch/curl 返回 HTTP 4xx/5xx 或超时 | 告知用户 URL 不可访问和原因（404/超时/拒绝连接），建议检查链接是否正确 |
| **社交媒体内容需要登录**（小红书/微博/推特） | 访问社交页面时返回登录重定向或被截断 | 提示用户需要登录。引导用户通过 web-access 的真实浏览器操作，或提供 cookies 手动注入 |
| **多语言页面编码异常** | 页面返回非 UTF-8 编码，内容显示为乱码 | 先检查 HTTP 头 Content-Type 的 charset，手动指定编码重试。仍乱码 → 告知用户编码不兼容 |
| **页面内容被 JS 大量截断/懒加载** | WebFetch 只拿到首屏少量数据，JS 加载的后续内容缺失 | 若需全量内容 → 切到真实浏览器模拟滚动加载。告知用户"页面使用懒加载，需要浏览器环境获取完整内容" |
| **长时间无响应/超时（>30s）** | WebFetch/浏览器打开页面后长时间不返回 | 设置超时上限 30s。超时后强制中断，告知用户"页面加载超时"，建议检查网络或换时间段重试 |

**原则**：异常先告知用户，再按规则处理。绝不静默跳过或静默编造数据。
