# Step 1 外部检索与研究工作流

## 目标

把“搜索”变成可复用的研究过程：知道搜什么、为什么信、哪些能用、哪里仍不确定。最终产物不是链接堆，而是一份能直接进入内容策划的研究包。

## 一、先判断搜索任务

| 类型 | 要解决的问题 | 默认深度 |
|---|---|---|
| 选题探索 | 读者在搜什么，哪些角度已经同质化 | 标准 |
| 素材检索 | 有哪些设定、例子、分类和写法 | 标准 |
| 事实核验 | 某个数据、规则、职业细节是否准确 | 快速；高风险则深度 |
| SEO研究 | 核心词、长尾词、搜索式问法 | 快速 |
| 竞品结构 | 高质量帖子如何起标题、分页和举例 | 标准 |
| 系统选题库 | 一个主题未来能拆成哪些系列 | 深度 |

## 二、建立搜索矩阵

不要只搜一个宽泛词。为当前主题生成4至8组查询，覆盖下列维度中至少三个：

### 1. 核心概念

`主题 + 是什么 / 类型 / 分类 / 设定 / 素材`

### 2. 写作问题

`主题 + 怎么写 / 写法 / 误区 / 避坑 / 例子 / 名场面`

### 3. 用户痛点

`主题 + 卡文 / 不会写 / 太悬浮 / 没有张力 / 同质化`

### 4. 平台表达

`主题 + 小红书 / 知乎 / 公众号 / 网文作者`

需要观察某个平台时可使用站点限定查询，但搜索摘要不等于页面原文，也不能据此判断互动量。

### 5. 事实核验

`具体说法 + 官方 / 规定 / 报告 / 论文 / 原始来源`

### 6. 反面与争议

`主题 + 争议 / 反例 / 不推荐 / 为什么不好用`

反面搜索能防止只找到互相转载的同一种说法。

## 三、分层搜索

### Layer 0：用户材料

用户提供的DOCX、截图、原文和明确要求优先级最高。先提取其核心内容、来源声明和明显风险。

### Layer 1：内部文库

搜索知夏已有资料，主要用于发现重复主题、历史结论和用户已确认偏好，不强行把旧素材拼进新稿。

### Layer 2：外部高质量来源

- 官方机构、平台规则、原始报告、论文和专业文档。
- 有编辑责任的专业媒体、出版社、行业机构。
- 原作者公开说明或作品页面。

用于事实、规则、数据和职业常识。

### Layer 3：平台用户内容

小红书、知乎、微博、论坛和创作者内容可用于观察：

- 读者如何描述卡点。
- 标题与搜索用词。
- 内容结构与案例密度。
- 评论区提出的真实问题。

不能仅凭用户内容确认专业事实，也不能复制其独特原句、完整清单和账号风格标识。

### Layer 4：补充与反证

主动搜索相反意见、失败案例和边界条件，检查当前结论是否过度绝对。

## 四、来源分级

| 等级 | 来源 | 可以支持什么 |
|---|---|---|
| A | 官方、原始文件、论文、作者一手材料 | 事实、规则、精确数据 |
| B | 专业机构、出版社、可靠编辑媒体 | 背景、方法、行业解释 |
| C | 小红书、知乎、论坛、个人创作者 | 语言、痛点、选题和案例线索 |
| D | 无来源聚合、洗稿站、营销软文 | 只作搜索线索，不进入成稿 |

涉及可变化信息、平台规则、数据、法律、医疗、金融或职业专业常识时，必须优先找A/B级来源。

## 五、交叉验证

- 重要事实尽量由两个相互独立的来源支持。
- 多个网页复制同一段文字不算独立来源。
- 数字要核对统计口径、时间和样本，不能只看搜索摘要。
- “全网热门、爆款、互动率高、平台推荐”没有页面证据时一律不写成事实。
- 对小红书帖子，只在能直接看到页面时记录标题、发布时间和互动数据；截图和搜索摘要不足时标记待核。
- 无法确认的内容可以作为创作灵感，但要从事实陈述改成“可尝试的写法”。

## 六、研究记录

每个有效来源记录：

```text
来源标题：
来源类型/等级：
链接或文件：
发布时间或版本：
支持的具体结论：
可用于本篇的部分：
限制/待核：
```

不要大段复制原文。保存能够支持结论的最小必要摘录，并遵守引用和版权限制。

## 七、搜索停止条件

满足以下条件即可停止：

1. 已找到足够支撑本篇内容的分类、例子或方法。
2. 重要事实已经交叉确认，或明确标记待核。
3. 连续两轮查询没有出现新的有效角度。
4. SEO核心词和1至2个长尾词已经确定。
5. 继续搜索只会增加重复链接，不能改变策划判断。

## 八、研究包格式

Step 1结束时输出给Step 2：

```text
研究主题：
搜索深度：快速 / 标准 / 深度
核心搜索词：

已确认信息：
- 结论｜来源等级｜来源

平台语言与读者痛点：
- 用户会怎样搜索或描述问题

可做选题：
- 选题角度｜交付物｜差异点

可复用素材：
- 分类、公式、案例类型

风险与待核：
- 哪些说法不能当事实

SEO建议：
- 核心词｜长尾词
```

研究包可以内部使用；确认闸门A只向用户展示压缩后的策划卡，不把搜索过程整段倾倒给用户。

## 九、小红书检索边界

- 小红书存在登录、个性化推荐和搜索结果不完全公开等限制，网页搜索无法等价复现用户账号里的完整结果。
- 需要精确参考某篇帖子时，优先使用用户提供的链接、截图或当前已打开页面。
- 无法直接打开帖子时，只把搜索结果当线索，不声称已阅读全文。
- 实时热度与互动数据会变化；没有当前页面证据时不报精确数字。

这个边界不会阻止Agent完成闭环：Agent仍可完成主题研究、跨来源核验、SEO查询、内容策划和成稿；只有“某篇小红书当前精确数据”可能需要用户提供页面或登录态。
