# 内容任务路由

本文件只负责选择内容模式。具体文字槽位从 `templates/copy/` 复制填充。

## 路由表

| 用户信号 | 内容模式 | 文案模板 | HTML模板 |
|---|---|---|---|
| 50个、100个、每页更多、直接拿走 | 数量清单 | `templates/copy/list-post.md` | `templates/html/list-9page.html` |
| 举例丰富、讲透、完整名场面、高张力CP | 单组深写 | `templates/copy/deep-case-post.md` | `templates/html/deep-case-9page.html` |
| 怎么写、步骤、避坑、检查表 | 方法教程 | `templates/copy/tutorial-post.md` | 按内容选择深写母版 |
| 酸涩感、破碎感、高智感等单概念 | 概念讲透 | `templates/copy/concept-post.md` | 按内容选择深写母版 |
| 只要标题、正文、tag、评论 | 发布配文 | `templates/copy/publish-caption.md` | 不生成HTML |

## 判断顺序

1. 先看用户是否明确点名参考版本；参考版本的信息层级优先。
2. 再判断“丰富”指条目更多还是单组更深。
3. 最后根据素材长度、页数上限和手机可读性决定数量。

## 选择标准

### 数量清单

- 核心价值是省时间、可收藏、快速查找。
- 单条信息短，通常由“素材 + 适用场景”构成。
- 每页10至15条，不能靠重复换词填满。

### 单组深写

- 核心价值是理解关系或方法如何落地。
- 每页只讲一组，固定“人设/核心 → 写法 → 完整案例”。
- 完整案例必须出现目标、选择、代价和变化。

### 方法教程

- 每一步必须是可执行动作。
- 第二页先给结论与适用范围。
- 每一步带示例或反例，末页给操作检查表。

### 概念讲透

- 用定义、机制、相近概念、误区和正反例推进。
- 概念必须落到人物目标、障碍、动作或台词。

## 边界

- 不能因为原始资料有38条就必须发布38条；先去重和校正。
- 不能把“字更丰富”自动理解为“条目更多”。
- 不能为了9页形式牺牲内容完整性；页数不足时优先减少条目。
