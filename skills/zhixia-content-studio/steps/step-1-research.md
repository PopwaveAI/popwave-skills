# Step 1：检索与研究

## 前置门禁检查

- 已从 `SKILL.md` 输入路由确认当前任务需要本Step。
- 只改HTML时禁止进入本Step。
- 完整文案默认快速核验，不准重新搜索整篇素材。

## I/O

**读什么：** 用户材料、`references/research-sop.md`、相关 `references/knowledge/*.md`。

**做什么：** 提取资料、生成搜索矩阵、执行外部与内部检索、分级来源、交叉核验。

**产出：** 一份研究包，供Step 2策划或Step 3文案核验使用。

**门禁：** 未标记事实、观点和待核项，不得进入内容策划。

## 执行

1. 判断检索档位：
   - 快速：5至8个有效来源；完整文案、SEO补充、单点核验。
   - 标准：10至15个有效来源；只有选题或新内容生产。
   - 深度：20个以上；系统选题研究或高风险事实。
2. 提取用户DOCX、截图、参考帖或文案，形成临时摘要。
3. 运行：

```bash
python3 scripts/build_search_matrix.py "主题" --platform 小红书
python3 scripts/search_knowledge.py "关键词"
```

4. 外部检索至少覆盖三个维度：用户搜索表达、写作方法/案例、事实核验；需要时增加争议与反例。
5. 来源分级：
   - A：官方、原始文件、论文、一手材料。
   - B：专业机构、出版社、可靠编辑媒体。
   - C：小红书、知乎、论坛、创作者内容。
   - D：无来源聚合与洗稿站，只作线索。
6. 重要事实尽量由两个独立来源支持；复制同一段的多个网页不算独立来源。
7. 平台用户内容只用于观察语言、痛点和结构，不凭搜索摘要宣称互动量。
8. 连续两轮搜索没有新角度，且当前内容已有足够证据时停止。

## 研究包模板

```text
研究主题：

已确认信息：结论｜来源等级｜来源
平台语言与读者痛点：
可做选题：角度｜交付物｜差异点
可复用素材：
风险与待核：
SEO候选：核心词｜长尾词
```

## ❌ WRONG / ✅ CORRECT

❌ 只搜“校园CP”，看到几个标题就宣布“全网热门”。

✅ 建立类型、写法、痛点、平台表达和反例查询；区分可用灵感与可核验事实。
