# Step 3：文案生成与SEO

## 前置门禁检查

- 已有确认后的策划卡，或任务为“完整文案质检/直出”。
- 已完整读取 `references/soul.md`、`references/seo-sop.md`。
- 已按任务选择一个 `templates/copy/*.md`。

## I/O

**读什么：** 策划卡、用户文案、SOUL、SEO SOP、对应文案模板、主题知识。

**做什么：** 生成封面、分页文案、发布标题、正文、置顶评论、标签与SEO配置。

**产出：** 文案包和第2页样稿。

**门禁：** 默认等待用户确认B；未确认不生成完整HTML。

## 执行

1. 先按模板写封面与分页，不套统一文艺腔。
2. 清单版每条保留“素材 + 适用场景”；深写版每页固定“人设/核心 → 写法 → 完整案例”。
3. SEO配置：
   - 1个核心词。
   - 1至2个长尾词。
   - 核心词进入封面/标题和正文前三句。
   - 另一长尾词自然进入置顶评论。
4. 标题、正文和评论制造真实省力感：已筛选、已分类、有数量、可直接改写。
5. 数量必须真实；“完整案例”必须含目标、动作、代价和关系变化。
6. 调用 `anti-ai-taste` 做质检，删除空话、课程腔、夸张效果和机械关键词。

## 确认闸门B：文案样稿

```text
封面：主标题 + 副标题
第2页：完整文案
其余分页：标题目录
发布标题：
SEO：核心词 + 长尾词
```

互动模式下交用户确认标题、语气和内容密度。直出模式下内部检查后继续。

## ❌ WRONG / ✅ CORRECT

❌ 先写完9页HTML，再问用户标题和内容是否合适。

✅ 先确认封面与第2页，用户认可内容密度后再套HTML母版。
