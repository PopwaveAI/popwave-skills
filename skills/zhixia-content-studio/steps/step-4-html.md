# Step 4：HTML视觉化

## 前置门禁检查

- 已有确认后的封面、分页文案和SEO文案；只改HTML任务除外。
- 已完整读取 `references/html-template-guide.md` 和 `references/visual-qa.md`。
- 已选择 `templates/html/list-9page.html` 或 `deep-case-9page.html`。

## I/O

**读什么：** 文案包、HTML模板、视觉验收规则、当前业务目录中的历史版本。

**做什么：** 复制模板、填充槽位、新建版本、校验结构和视觉。

**产出：** 新版本HTML；保留旧版本。

**门禁：** 页数、数量、背景路径和页面溢出检查未通过不得交付。

## 执行

1. 按任务选择模板：
   - 清单素材 → `templates/html/list-9page.html`。
   - 单组深写 → `templates/html/deep-case-9page.html`。
2. 在业务目录 `versions/` 新建 `vN_描述.html`，不覆盖现有文件。
3. 替换所有 `{{...}}` 槽位，清理未使用占位符和历史人物名。
4. 固定视觉：
   - 1080×1920竖版。
   - 主标题中心位于画布高度42%至50%。
   - 第2页直接进入核心内容。
   - 知夏奶油纸、蓝色铅笔背景与固定页脚。
5. 清单版每页尽量填满但不缩成小字；深写版一页一组。
6. 校验：
   - 页面数与页码一致。
   - 标题数量与实际条目一致。
   - 无重复项、无断裂背景路径、无残留模板槽位。
   - 正文未溢出，底部空白符合内容模式。
   - 手机缩略图能读出数量和核心素材词。
7. 同步保存发布文案Markdown，不把发布配文塞进图片页面。

## ❌ WRONG / ✅ CORRECT

❌ 每次从空白HTML重写CSS，导致标题位置和字号反复漂移。

✅ 从用户认可的母版复制，只调整槽位内容和必要密度参数。
