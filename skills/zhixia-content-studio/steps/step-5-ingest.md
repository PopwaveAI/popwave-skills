# Step 5：可选资产沉淀

## 前置门禁检查

- HTML或文案已经稳定，用户要求入库，或本次产生了明确可复用的新规则。
- 已完整读取 `references/knowledge/index.md`。
- 未经核验的原始材料禁止直接进入知识库。

## I/O

**读什么：** 用户原始文件、研究包、最终文案、最终HTML、对应主题知识。

**做什么：** 归档来源，提炼可复用规则，更新知识索引或模板。

**产出：** 来源归档、主题知识增量、索引或模板版本。

**门禁：** 不得把临时观点、无依据热度或整篇他人内容当作知识沉淀。

## 执行

1. 原始文件归档到工作区 `资料库源文件/对应主题/`。
2. 判断资产类型：
   - 事实与题材知识 → `references/knowledge/对应主题.md`。
   - 搜索、SEO和质量方法 → 对应 `references/*-sop.md`。
   - 账号口吻与价值观 → `references/soul.md`，必须谨慎修改。
   - 用户认可的新布局 → 提炼槽位后更新 `templates/`。
   - 单次业务成稿 → 留在业务目录，不进入Skill正文。
3. 每条知识记录来源、适用范围、风险与待核项。
4. 更新 `references/knowledge/index.md` 的入口和检索词。
5. 若修改Skill结构或模板，更新版本号和 `CHANGELOG.md`。
6. 交付时说明本次沉淀了什么、放在哪里。

## ❌ WRONG / ✅ CORRECT

❌ 每收到一份DOCX就把全文塞进Skill，造成重复、冲突和加载膨胀。

✅ 原始文件归档；Skill只沉淀分类、公式、边界、用户认可模板和可检索入口。

---

## ⛔ 完成门禁

确认来源可追溯、索引已更新、模板不含具体业务人物与文案后，本次流程结束。
