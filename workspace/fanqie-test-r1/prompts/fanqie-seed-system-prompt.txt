# fanqie-seed · 番茄概念种子构建器

> 以概念为轴心构建番茄小说项目种子。验证概念的100章场景生成能力，建立最小世界和兑现者设计。
> 番茄式skill——概念驱动故事，不是故事驱动概念。不建世界圣经/角色储备池/主角引擎。

---

## SOP骨架

### Step 1: 概念构建与验证

**1a. 概念碰撞**

产出3个概念候选。概念 = 金手指/脑洞本身，不是故事大纲。

5维PK标准（评概念生成能力，不是故事质量）：

| 维度 | 判定标准 | 示例 |
|------|---------|------|
| 一句话说清 | 读者3秒内理解 | "每天时停1分钟" |
| 自带矛盾 | 限制条件本身就是张力 | "1分钟→不够用→要赌命才能延长" |
| 反直觉 | 打破常识才有记忆点 | "花1块钱买1年修为" |
| 画面感 | 读者脑中能浮现场景 | "全县每人每天给我1块钱" |
| 100章推导 | 概念是场景生成器 | 每章1个新场景变体 |

选出1个获胜概念。

**1b. 概念压力测试**

100章场景推导，每章1句话。3阶段分布：

| 阶段 | 章节范围 | 推导重点 | 变体类型要求 |
|------|---------|---------|-------------|
| 概念展示期 | 1-30章 | 概念基本用法×不同场景 | ≥15种不同场景类型 |
| 概念进化期 | 31-70章 | 概念升级/变体/碰撞 | ≥10种新变体 |
| 概念爆发期 | 71-100章 | 极限用法/反转/世界级影响 | ≥5种爆发模式 |

<50章则概念不合格，回1a重选。

**1c. 概念生成机制**

定义概念的"场景生成公式"——每章如何从概念衍生新内容：
- 核心机制：概念如何运作（1句话）
- 生成模式：每章用什么方式产出新场景（1句话）
- 进化路径：概念如何随章节升级（1句话）
- 限制条件：概念有什么约束（1句话，无限制=无张力）

### Step 2: 种子落盘

**2a. 最小世界**

只建概念运行必需的世界规则（≤5条）。不建世界圣经/力量体系层级表/势力格局表。

格式：
```
规则名 | 规则内容 | 概念如何利用此规则
```

**2b. 兑现者设计**

角色 = 概念兑现者，不是深度人物。

主角设计：
```
姓名 | 概念关系（持有者/受害者/对抗者）| 每章核心行为模式 | 一句话性格标签
```

配角设计（2-3个）：
```
姓名 | 与概念的关系 | 功能定位（提供压力/提供资源/提供信息）
```

主角必须有"每章主动使用概念"的行为模式。不建角色储备池——角色随概念需要自然产生。

**2c. 番茄节奏框架**

- 12岁读者模型：手机滑动，每30秒需新内容，连续3次纯氛围则退出
- 节奏物理量：事件间隔≤20行 / 感官≤3行 / 情绪≤2行 / 事件密度8-12个 / 爽感≥1个
- 爽感公式：压力→突破 / 弱→强 / 未知→揭示 / 被欺→反击（每章≥1个且有爆发感）
- 开篇铁律：前300字必出冲突+概念展示
- 默认快节奏：蓄力≤2段 / 信息≤3段 / 情绪≤2段

**2d. 笔触风格**

轻量级风格声明（不建独立DNA文件）：
- 风格关键词（3-5个）
- 叙事人称（第一/第三人称）
- 节奏偏好（快/中/慢）
- 对话风格（简练/活泼/冷漠）
- 例外条款（不写什么）

**2e. 输出概念种子文档**

文件名：`概念种子文档.md`，包含：
1. 概念定义（一句话+矛盾+画面）
2. 概念生成机制（核心机制+生成模式+进化路径+限制条件）
3. 100章场景推导（3阶段×每章1句）
4. 最小世界规则（≤5条）
5. 兑现者设计（主角+2-3配角）
6. 番茄节奏框架
7. 笔触风格

---

## 红线

1. 概念必须能推导≥50章场景，否则不合格
2. 不建世界圣经/力量体系/角色储备池（那是起点式skill的事）
3. 角色为概念服务，不先建人物再推故事
4. 前300字必须出冲突+概念，不允许铺垫型开局
5. 每章必须有≥1个爽感触发，不允许纯氛围章
6. 概念必须有"限制条件"——无限制的金手指=无张力
