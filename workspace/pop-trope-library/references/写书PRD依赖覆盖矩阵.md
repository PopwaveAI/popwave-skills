# 写书 PRD 依赖覆盖矩阵

> 用途：从写书管线反推 library 应覆盖哪些素材，避免库只会“收藏”，不会“被写作调用”。

---

## 写作环节 → library 依赖

| 写作环节 | 核心问题 | 首选入口 | 补充入口 |
|:---------|:---------|:---------|:---------|
| seed/creative | 这本书立什么卖点、什么赛道、什么金手指 | `立项库/` + `金手指库/` | `设定库/00-索引.md` |
| world | 世界怎么运转 | `设定库/{书名}/PRD.md` | L1 文件、世界宪法 |
| plot | 当前剧情任务怎么推进 | `任务索引/` | `剧情库/{书名}/L2/`、`L3/`、`套路库/` |
| create | 正文如何兑现动作闭环和爽点 | plot 输出的案例消化 | `文风库/`、相关 L2 |
| revise/qa | 是否偏离爽点、设定、文风 | 案例消化 + 套路红线 | 设定库/文风库 |
| arc | 单元后复盘与回流 | 本单元运行日志 | 任务索引补充建议 |

---

## 覆盖缺口判定

| 症状 | 优先补哪里 |
|:-----|:-----------|
| plot 读了案例仍写不出高质量幕纲 | `任务索引/` + `案例消化` |
| 正文不爽，只有事件推进 | `套路库/` 红线 + L2 爽点证据 |
| 设定漂浮，剧情用不上 | `设定库/{书名}/PRD.md` 匹配逻辑 |
| 金手指平庸 | `金手指库/` WHAT 层设计卡 |
| 文风不像参考书 | `文风库/` 场景卡 |
| agent 找不到 library | 模块索引 + 任务索引路径 |

---

## 当前重点

1. 任务索引要优先覆盖高频写作任务：开局、第一次收益、绝境反杀、世界观揭示、关系羁绊、势力初建。
2. 存量高质量 L2 不要重写正文，优先补消费摘要和任务索引。
3. 每次 arc 复盘发现有效案例，都应反向建议更新任务索引。
