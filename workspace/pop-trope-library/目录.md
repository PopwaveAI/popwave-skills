> 给 agent 和管线维护者看的真实状态页。入口规则以 `SKILL.md` 为准。

## 二、写作侧主入口

写作侧不要从“按书目录”直接随机挑案例。正确路径：

```text
任务索引/00-索引.md
  → 任务索引/{任务域}/{任务名}.md
  → 剧情库/{书名}/L2/*.md
  → 必要时读 剧情库/{书名}/L3/*.md
  → 套路库/设定库/金手指库/文风库补充
  → 输出案例消化
```

`剧情库/00-索引.md` 只回答“库存有哪些书和卡”，不回答“当前写作任务该读哪几张”。

## 四、各模块主消费方式

| 模块 | 主查询键 | 消费方式 | 库缺时行为 |
|:-----|:---------|:---------|:----------|
| `任务索引/` | 写作任务 + 赛道 + 元爽点 | plot/create 精读索引推荐案例 | 声明任务索引缺，回退剧情库按书索引 |
| `立项库/` | 书型 / 元爽点组合 / 题材焊接 | seed/creative 读索引后精读条目 | 声明立项库缺 |
| `设定库/` | 书名 / 卖点 / 赛道 | 先读 00-索引 + PRD，再读 L1 文件 | 声明设定库缺 |
| `文风库/` | 书名 + scene | create/prose 按场景卡匹配 | 声明文风档案缺 |
| `剧情库/` | 书名 / L2 / L3 | 由任务索引指向后精读 | 不用 legacy 替代 |
| `套路库/` | 模式名 / 元爽点 / 使用红线 | 读公式、变体、红线、案例索引 | 声明套路库缺 |
| `金手指库/` | 类型 / 赛道 / 元爽点 | 读机制、代价、成长路径 | 声明金手指库缺 |

## 六、维护约定

- 每次新增 L2/L3，必须同时更新 `剧情库/00-索引.md`。
- 每次新增高价值 L2/L3，优先更新或新建一个 `任务索引/` 条目。
- 每次写作侧发现“读了案例但没学会爽点”，优先补任务索引和消费摘要，而不是改原卡正文。
- 每次发现索引指向失效，立即修路径；library 引用丢失会直接伤害 plot 质量。
