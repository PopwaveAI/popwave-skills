# fanqie-plot · 番茄场景推导引擎

> 从概念种子推导每章场景卡。极轻量——不建幕纲、不建施工卡，只产出1页场景卡驱动正文写作。
> 核心逻辑：概念变体池 → 选取变体 → 场景卡。每章场景卡≤500字（施工卡的1/4）。

---

## SOP骨架

### Step 1: 概念变体池维护

从概念种子的100章推导中选取当前阶段的变体，并补充新变体。

变体池格式：
```
| 变体编号 | 变体描述 | 生成场景类型 | 爽点类型 | 预计使用章节 |
|---------|---------|-------------|---------|-------------|
| V001 | 时停偷看考卷 | 校园/日常 | 未知→揭示 | ch1 |
| V002 | 时停赌命延长 | 赌博/生死 | 弱→强 | ch3 |
```

变体来源：
1. 概念种子100章推导（按当前章节阶段选取）
2. 已写章节的涌现信息（正文产生的新设定/新角色→新变体）
3. 外部搜索补强（子agent搜索相似作品的场景模式）

### Step 2: 场景卡批量生成

每章1张场景卡，≤500字。每次生成5-10章批次。

场景卡格式：
```
## chNNN · 章名

概念变体：V0XX - 变体描述（本章用哪种方式兑现概念）
核心爽点：压力→突破 / 弱→强 / 未知→揭示 / 被欺→反击（选1）
爽点触发位：前1/3 / 中段 / 章末
钩子：章末留什么悬念（1句话）
节奏约束：事件密度X / 概念交互≥X次 / 前300字出冲突
边界：写到哪停 / 不能碰什么（各1句话）
前章衔接：上一章结尾的最后一件事（1句话）
```

**场景卡自检**（每张卡生成后）：
- 概念变体是否明确？无变体=废章
- 爽点是否标注？无爽点=废章
- 钩子是否有追读动力？无钩子=废章
- 与前章衔接是否自然？

## 番茄节奏红线（7条）

| 红线 | 要求 |
|------|------|
| 前3章完成概念展示+首爽 | ch1概念激活，ch3首爽释放 |
| 每3章≥1高潮 | combat_reversal或等效爆发 |
| 前5章有概念升级 | ch5前概念必须进化1次 |
| 铺垫≤1章 | 番茄不允许2章铺垫 |
| 连续2章无概念交互=不合格 | 每章必须有概念使用 |
| 不得连续2章同章型 | 6种章型轮换 |
| 每章前1/3有钩子或mini爽点 | 不允许慢热开局 |

章型速查（详见 fanqie-write/references/章型定义.md）：
opening_shift / exploration_gain / confrontation_pressure / combat_reversal / high_tier_display / reveal_hook

## 产出

`场景卡-batchNN.md`，包含5-10章场景卡。

---

## 红线

1. 场景卡≤500字/章（不预写正文）
2. 每章必须标注概念变体
3. 番茄节奏红线7条全量执行
4. 连续2章不得使用相同概念变体
5. 连续2章不得使用相同爽点类型
