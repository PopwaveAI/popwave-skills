## 定义
网格化剧情细纲：
网文情节本质是主角与配角、不同圈子之间的互动。在设计主角与不同配角/圈子的故事线时，需要考虑以下几点：
1. 双方关系随着每一次互动而变化，最开始通常陌生而彼此防备，忌交浅言深或不合常理的信任
2. 不同配角/圈子之间存在信息壁垒，仅可知自身所处关系/故事线的信息
3. 主角与不同配角/圈子建联/结怨的方式应当多元化，避免套路

因此，可采用网格化的方式来规划主角与不同配角/圈子的故事线，规范如下：
1. 以配角/圈子为列标题，表示主角与这些配角/圈子的故事线
2. 每行用于设计主角与不同配角/圈子之间发生的一段完整情节/事件（非场景片段），记录在对应的单元格中，其他单元格留空
 
## 示例

|张三|李四|石头村|
|---|---|---|
|<设计主角与张三不打不相识的事件，主角获得XXX，张三对主角的印象为XXX，主角发现XXX，计划前往石头村>| | |
| | |<设计主角在石头村附近遭遇XXX，发现XXX，并在石头村XXX，救下石头村村长的孙子XX……>|
| |<主角帮助石头村的XXX干XXX，前往后山秘境遇到李四，XXXX>| |
[依此类推]

## 要求
1. 行兼具编排时序与主角行动线的双重作用，确保情节紧凑、逻辑清晰，一个事件推导出下一事件
2. 以列分离关系/故事线，便于观察各条线的节奏与信息壁垒是否完善
3. 根据剧情需要，及时扩展新的列，注重维护好行与列的剧情设计
4. 注意情节发生的场景、环境，防止主角利用现场不具备的条件产生剧情bug，避免机械将神
