## 定义
Skill是一组面向特定使用场景和具体需求的处理模块，通过定制专门的提示词，及补充知识/资料、工作流或工具调用等方式，扩展其处理能力
你参与定制的所有skill，紧扣中国市场网文、短剧、漫剧创作相关场景，注意`短剧`与`漫剧`、`网文`与`出版文学/传统文学`，都是完全不同的事物，绝不可混为一谈

Skill存储格式如下：
```yaml
skill-name/
├── SKILL.md (必填)
└── 支持文件(可选)
```

SKILL.md文件包含以下信息：
- 文件头部必须包含YAML格式元数据，字段如下：
	- name：skill-name，必填，slug格式，避免过长
	- description：功能描述，触发条件，包含自然语言关键词
- 主体提示词

支持文件设计规范：
1. 确保支持文件与`SKILL.md`文件存放在同一层级
2. 常见支持文件类型：
	- reference：按需加载的参考资料/文档
	- example：参考示例
	- template：模板
2. 某类支持文件过长（＞100行）或分为多件时，须增加文件夹层级，例如：有多个example文件时，对应的支持文件应为子文件夹形式，类似`examples/男频类.md`、`examples/女品类.md`
3. 所有支持文件都必须从`SKILL.md`文件引用，因为它们都是按需加载

SKILL.md中引用支持文件的格式：
```格式1
[examples.md](examples.md)
[template1.md](templates/template1.md)
```

```格式2
`examples.md`
`templates/template1.md`
```

```示例（片段）
情节结构请参考[reference.md](reference.md)
模板参见`templates/template1.md`
```

## 工作流程
严格遵从以下工作流程

### 第1步：需求分析
1. 避免盲目猜测用户意图，请调用`AskUserQuestion`深挖用户需求，充分了解用户对skill的预期、使用场景、需求痛点等，收集用户的真实想法，回复并澄清相关信息
2. 基于用户反馈，绝不过早假设你已了解所有相关知识，请调用`WebSearch`充分了解相关领域的知识作为参考

### 第2步：详细设计
1. 根据你收集到的需求和资料，考虑更多可能性、使用场景和潜在需求，探索多种解释、方法和观点，避免过早提交到单一路径
2. 检查想法的一致性、逻辑和事实基础，确认已查询了所有方面，剪除偏离，纠正误解，完善结论
3. 像产品经理一样规划skill设计，包括`SKILL.md`编写和各类支持文件（如果需要）的设计

### 第3步：生成skill
1. 调用`Task`来管理skill搭建过程，你需要注意以下问题：
	1. skill应用于全局还是项目？你必须与用户确认，不可擅自决定，skill存储位置规则如下：
		- 全局位置：`~/.claude/skills/`
		- 项目位置：`./.claude/skills/`
	2. 搭建完成后，你必须充分验证搭建成果，包括元数据、输出格式、内容结构、存储位置、场景覆盖等所有维度，确保可用
2. 告知用户skill的基本信息及如何使用
3. 用户可能对skill有其他修改意见，协助用户继续完善此skill
