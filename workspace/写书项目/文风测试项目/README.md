# 文风测试项目：高张力战斗场景描写

## 项目目标

明确编写高张力战斗场景的具体步骤，通过AB test + skill SOP方式筛选最优方法论。

参考标杆：海贼王1122卡普救科比、一拳超人S1最终话的视觉冲击力——拳拳到肉的打击感、巨大场景破坏、绚丽光效。

## 核心问题

> "文字如何表达光污染和打击感？"

答案不是"写得更用力"，而是**用文字的节奏模拟镜头的语言**。

### 文风DNA系统的缺口

| 缺口 | 说明 | 影响 |
|:-----|:-----|:-----|
| **分镜感未形式化** | 遮天提到"短句推进长句渲染"但没有建立"段落=镜头"的映射体系 | agent无法系统性运用分镜技法 |
| **打击感无技法库** | 现有DNA描述"做了什么"但不教"如何让文字有冲击力" | agent写出的是动作说明书而非拳头 |
| **光效/色彩无转写方法** | 遮天有颜色词分析但无"色温→文字"的转换指南 | agent只能写"耀眼/绚丽"等空泛词 |
| **描写性 vs 指令性** | 文风DNA是描述性的（作者做了什么），不是指令性的（agent该怎么做） | DNA是原材料，缺加工指令 |
| **无视觉媒体桥接** | 完全基于文本采样，没有从动漫/漫画提取视觉信息的通道 | 无法利用动画的现成分镜语言 |

## 「看图识图→转写文字」方案评估

### 方案描述

让agent观看动漫画面（截图/分镜）→ 识别画面元素 → 转写为文字。

### 评估结论：部分靠谱，需三层结构

#### 图像能提供的（靠谱的部分）

1. **分镜构图**：特写/全景/仰角/俯角 → 直接映射到段落结构（短段=特写，长段=全景）
2. **视觉元素清单**：光效类型、破坏形态、色彩搭配 → 强制具体化，消灭"绚丽/耀眼"等空泛词
3. **连续帧的节奏**：漫画分镜的"格子大小+留白" → 映射到段落的"长短+空行"
4. **"巅峰长什么样"的参照**：给agent一个具体的"好"的标准

#### 图像不能提供的（不靠谱的部分）

1. **时间节奏**：文字的阅读速度由句子长度控制，图像无法传递这个
2. **重量/触觉**：打击感的核心是"拳头的重量穿过纸面"，纯视觉信息无法传递
3. **声音设计**：打击感的50%来自声音（闷响/碎裂/寂静），图像完全缺失
4. **情感锚点**：卡普那一拳的震撼来自20年角色积累，不是画面本身
5. **留白技法**：图像展示一切，文字必须选择"不写什么"——这是图像教不了的
6. **因果链**：图像展示"结果"，但文字需要展示"原因→过程→结果→余波"的完整链条

### 核心判断

> **瓶颈不在"agent不知道巅峰战斗长什么样"，而在"agent不知道如何让文字感觉像巅峰战斗"。**

我自己的测试已经证明了这一点：第二版我能描述所有视觉元素（引力针、光子弹、蘑菇云），但写出来远不如第一版——因为第一版有情感锚点（快死的小孩、发抖的手、"能走吗"），第二版只有能力展示。

图像可以作为**视觉词汇注入器**和**分镜结构参考**，但不能单独驱动优质战斗文字。

## 推荐方案：三层渲染架构

### Layer 1: 视觉参考层（图像输入）

**功能**：提供"巅峰战斗长什么样"的具体参照

**输入**：动漫截图/漫画分镜（OPM S1最终话、海贼王1122等）

**处理**：
- 分析每张图的构图类型（特写/中景/全景/鸟瞰）
- 提取视觉元素清单（光效类型、破坏形态、色彩组合、动态线）
- 识别分镜序列（连续图片的节奏：快切→慢镜头→定格）

**输出**：`视觉元素清单` + `分镜结构表`

### Layer 2: 文风DNA层（文本节奏）

**功能**：提供"文字如何呼吸"的节奏蓝图

**输入**：现有文风DNA场景卡（优先使用遮天combat_epic + 深渊主宰combat_mid_boss）

**处理**：
- 提取段落呼吸模式（独句段/短句链/多句段的分布）
- 提取感官序列（嗅觉→听觉→触觉→视觉的顺序）
- 提取叙事者距离（贴身/旁观/远景）
- 提取"压→爆→收"三拍情绪节奏

**输出**：`节奏蓝图` + `感官序列` + `情绪节拍`

### Layer 3: 渲染指令层（转写技法）

**功能**：将Layer 1的视觉元素填入Layer 2的节奏骨架，用具体技法保证打击感

**核心技法**：

#### 技法1：压缩-膨胀定律
- 撞击瞬间：≤1句，独立成段（压缩）
- 撞击后果：≥8段，多感官级联展开（膨胀）
- 禁止在撞击瞬间用形容词修饰力量

#### 技法2：段落=镜头
- 短段（1-2句）= 快切/特写
- 长段（4句+）= 慢镜头/全景
- 空行 = 硬切
- 禁止在同一段内切换视角

#### 技法3：感官叠加
- 每次重大撞击至少覆盖3种感官
- 视觉+听觉+触觉/嗅觉/本体觉
- 写"消失"而非"出现"（冲击波抽空空气→写窒息感）

#### 技法4：级联破坏
- 破坏按"点→线→面→体"扩散
- 用参照物锚定规模（方圆X米/云层消失/从太空可见）
- 给破坏一个可视的"地质指纹"（玻璃化/液化/晶化）

#### 技法5：反差蒙太奇
- 最强的一击，写最轻的动作
- 最大破坏，写最小细节（血珠在玻璃地面砸出裂纹）
- 战斗中至少1个"安静时刻"（人物简短对话/微小动作）

#### 技法6：色温替代形容词
- 禁止用"绚丽/璀璨/耀眼"
- 用物理概念：色温(K值)、光谱、折射、透镜效应、曝光过度
- 光的后果＞光本身（温度/阴影消失/颜色失真/视网膜灼伤）

#### 技法7：8拍节奏引擎
```
慢(绝望铺垫 3-4段)
  → 快(硬切 1句独立成段)
    → 爆(撞击 1字/1句)
      → 胀(后果展开 8-15段，多感官级联)
        → 静(人物特写 2-3段)
          → 升(二次威胁+光效 3-5段)
            → 终(终极爆发，感官过载)
              → 收(安静收尾，情感落地)
```

#### 技法8：力量为谁而用
- 每场战斗必须回答：这股力量为谁而用？代价是什么？
- 力量本身不产生张力，张力来自情感前提
- 必须有一个"在乎的人"在战斗中或战斗外

## AB Test 方案

### 测试目标

验证哪种方法能稳定产出高张力战斗文字。

### 测试变量

| 方法 | 输入 | 核心差异 |
|:-----|:-----|:--------|
| **A法** | 文风DNA + 渲染指令（纯文本） | 现有方案强化版 |
| **B法** | 图像参考 + 文风DNA + 渲染指令（三层架构） | 完整方案 |
| **C法** | 图像参考 + 渲染指令（无DNA） | 验证DNA是否必要 |

### 测试场景

同一个战斗骨架（人物、动机、能力设定相同），三种方法各生成一章正文。

### 评估维度（每项1-5分）

| 维度 | 评估标准 | 权重 |
|:-----|:--------|:-----|
| 打击感 | 读到拳头落地时身体有没有反应 | 25% |
| 分镜感 | 段落节奏是否有"镜头切换"的感觉 | 20% |
| 光效转写 | 光效描写是否具体可感（非空泛形容词） | 15% |
| 场景破坏 | 破坏是否有级联感，规模是否有参照物 | 15% |
| 情感锚点 | 是否在乎战斗结果（有无情感前提） | 15% |
| 节奏呼吸 | 快慢交替是否自然，有无"喘不过气"段落 | 10% |

### 测试流程

1. 准备阶段：收集3-5张参考动漫截图（OPM/海贼王），编写战斗骨架
2. 执行阶段：三种方法各生成一章3000-5000字正文
3. 评估阶段：按6维度打分 + 对比分析
4. 迭代阶段：胜出方法进入下一轮优化
