# 套路：家族会议决策

## 定义
家族核心成员聚在一起讨论重大事务，通过不同角色的观点碰撞最终形成决策。

## 叙功能
- 展示家族内部权力结构与分工
- 通过不同角色的态度/立场塑造人物
- 将决策过程合理化（不产生"主角光环式"的降智或开挂）
- 让读者理解家族战略

## 本卷实例（ch044-ch045）

### 参与角色与角色定位

| 角色 | 在决策中的定位 | 观点/倾向 |
|------|---------------|-----------|
| 李木田 | 族长/最终拍板人 | 保守、远见、战略型：选择"拖"，利用汲家 |
| 李通崖 | 次子/执行者 | 被交易打动但有分寸，如实汇报困境 |
| 李项平 | 少族长/未来继承人 | 务实、理性：指出"没符箓可给"，推动问题拆解 |
| 陆江仙 | (背后观察者/实际决定者) | 对阵法心动，但无法参与对话 |

### 决策流程

```
信息收集（李通崖接见万天仓）
       ↓
报告困境（李通崖："为难"）
       ↓
问题识别（李项平："我家可没有什么符箓可给他"）
       ↓
可行性分析（法鉴只能打妖不能打人）
       ↓
方案提出（李木田：暂缓、利用汲家）
       ↓
时间表确立（2-3年、待玉京轮）
       ↓
定策："拖"
```

### 与其他套路的关系
- 家族会议通常承接**外部压力与制衡**套路（会上分析外敌）
- 会议的输出往往是**战略拖延**或**战争/外交决策**
- 会议中暴露的问题会推动修炼成长线

### 写作要点
| 要素 | 说明 |
|------|------|
| 角色分工 | 不同角色在会议上扮演不同角色（信息提供者、质疑者、拍板人） |
| 信息完整 | 决策前必须让读者知道所有关键信息 |
| 逻辑链条 | 从问题到结论的推导必须清晰可循 |
| 会后执行 | 决策后要有执行计划或下一阶段预告 |
