## 二、使用说明

写作时按需加载对应参考域文件，**不要一次性读完所有文件**，避免信息过载。推荐流程：

1. **定位当前写作内容所属层级**（例如写宗门扩张 → 参考「设定基建层」相关文件）
2. **打开对应的参考域文件，快速浏览「核心发现」与「具体映射」部分**
3. **对照映射关系反查原文**，确认理解后再落笔
4. **写完可用参考域检验**：是否充分利用了已有创意来源？

## 四、更新维护规范

1. **增补不增文件**：新发现的创意来源直接追加到对应的参考域文件中，**不要另建新文件**。
2. **每条记录格式**：`章节归属 | 参考来源（具体出处） | 原文触发点 → 转化后的呈现 | 备注`
3. **密度调整**：若某参考域在后续章节出现频率显著增加或减少，更新文件头部的密度评级（★）。
4. **覆盖范围更新**：每次新增章节后，在文件头部覆盖范围字段追加章节号。
5. **版本标记**：每次批量更新后在文件末尾追加一条变更记录（`> *最后更新：YYYY-MM-DD，新增chXxx-chYyy分析*`）。
6. **废弃处理**：若某参考域被后续章节证伪或推翻，保留原记录但添加 `⚠️ [已修正]` 标记，并附上修正说明，切勿直接删除历史数据。
