# 敌人阶梯假说与起点skill管线优化PRD

> 创建日期：2026-07-15  
> 状态：待评审

---

## 1. 背景与问题

现有起点小说创作skill群采用 seed → world → plot → write 的管线。plot层以丹哈蒙故事圈和功能位作为幕序列的设计起点，先定"这一幕的功能是什么"（建置/上升/冲突/转折），再往里填事件，事件中自然产生敌人。

这个流程有一个结构性缺陷：**只看一卷就动手设计plot**。没有全书卷序规划，无法提前知道后续卷的敌人是谁，导致力量体系成长节奏是盲的，敌人出现带有随机性，跨卷伏笔和敌人成长线无法系统性规划。

与此同时，在拆解深渊主宰全书1107章和验证起点Top20热门小说的过程中，我们发现了一个贯穿所有成功作品的叙事结构规律——**幕与幕之间的切换节点是敌人/危机**。这个发现为plot层的设计方法提供了新的理论基础。

---

## 2. 核心发现：敌人阶梯假说

### 2.1 假说内容

一本小说的本质结构是：主角从起点到终点的旅程，路上按照世界观和力量体系，在中间节点不断放置强度曲线上升的敌人，给主角制造危机并促使其变强，然后消灭（全部或部分）的循环。

用公式表达：

```
幕切换 = 新敌人/新危机出现
  → 主角遭遇 → 变强（升级/新能力/新势力）→ 解决（全部 or 部分）
  → 残留问题 + 新威胁 = 下一幕的"新敌人"
```

### 2.2 分形嵌套

这个循环是分形的，在卷级、幕级、章级三个尺度上重复同一逻辑：

| 层级 | 循环周期 | 敌人特征 | 示例（深渊主宰卷1） |
|------|----------|----------|---------------------|
| 卷级 | 80-120章/圈 | 量级跃迁的大BOSS | 贫民区混混 → 深渊蛇魔 → 吸血鬼神子 |
| 幕级 | 8-15章/圈 | 同级轮换的中BOSS | 混混 → 科尔帮 → 蛇魔 → 浩劫阴云 |
| 章级 | 1章/圈 | 微型危机单章闭环 | 遇到狗头人 → 猎杀 → 获得经验 |

大循环嵌入小循环，逻辑完全一致，只是时间尺度和敌人量级不同。

### 2.3 验证证据

#### 深渊主宰（9卷1107章，已有全链路拆解数据）

每一卷的切换点 = 新一级敌人的出现，敌人量级严格递增：

| 卷 | 卷级敌人/危机 | 量级 | 主角状态变化 |
|----|---------------|------|-------------|
| 卷1·琥珀城初醒 | 贫民区混混 → 深渊蛇魔 | 凡人级 | 1级→4级，割喉者成名 |
| 卷2·盐与血之路 | 蜥蜴人 → 食人魔督军 | 凡人级天花板 | 兼职巫师，获得传说度 |
| 卷3·王者之证 | 海盗头目 → 大衮之子 | 超凡入门 | 割喉者→幽灵船主 |
| 卷4·海外群岛 | 熊灵精魂 → 传奇刺客 | 超凡级 | 建城→领主 |
| 卷5·屠神之战 | 吸血鬼神子（半神级动态敌人） | 半神级 | 海盗王→半神→弑神者 |

关键规律：每次换卷 = 换一个量级更大的敌人。卷1杀小混混，卷5杀半神，卷8杀诸神。敌人量级与主角力量体系的三段式（凡人/超凡/神圣）严格对应。

#### 更多起点热门作品验证

### 2.4 关键细化：敌人量级跃迁 vs 同级轮换

"敌人作为切换节点"严格来说分为两种：

- **同卷内幕切换** = 同级敌人轮换（卷1内：混混→科尔帮→蛇魔，都是"贫民区级"威胁递进）
- **卷切换** = 敌人量级跃迁（贫民区混混→蜥蜴人→海盗→半神→真神）

卷级切换是质变，幕级切换是量变。两者都遵循"主角→敌人→遭遇变强→解决"的循环，但卷级切换还伴随着主角力量体系的跨段（如凡人→超凡）。

### 2.5 补丁：约15-20%的非敌人驱动切换

不是所有幕切换都靠"新敌人"驱动。约15-20%的幕切换靠的是：

- **内部危机**（薇薇安失控、歌莉娅怀孕暴露）
- **环境灾变**（圣者浩劫爆发、魔网崩溃）
- **主角主动选择**（索伦决定建城、前往深渊）

但即使是这些，本质上仍然是抽象的"危机"。**敌人是具象的危机，危机是抽象的敌人**，两者是同一事物的两面。假说中的"敌人/危机"已涵盖这种情况。

## 3. 敌人设计的三维度框架

### 3.1 三维度定义

| 维度 | 含义 | 判断标准 |
|------|------|----------|
| 功能性 | 敌人推动难度曲线上升的作用 | 打完这个敌人，主角获得了什么、暴露了什么？如果什么都没变，就是废节点 |
| 有趣性 | 敌人逼主角换解题方式的能力 | 能否用和上一个敌人完全相同的方式打败？如果能，就不有趣 |
| 立体度 | 敌人作为独立角色的完整度 | 有自己的动机、欲望、伤口、成长弧线 |

### 3.2 权重判断

**功能性是地基，有趣是上层建筑，立体是奢侈品。**

对于爽文/网文，敌人有自己的完整动机和情感弧线是加分项，但不是必需品。深渊主宰里的混混没有立体度，但功能完美——逼主角第一次杀人、第一次获得经验、第一次展现果断。先保证功能和有趣，有余力再补立体。

### 3.3 按角色级别分配设计深度

不仅敌人需要设计，盟友和重要NPC同样需要。设计深度按角色在故事中的分量分级：

| 角色级别 | 设计深度 | 产出物 | 人物原型 |
|----------|----------|--------|----------|
| 章级杂兵/路人 | 功能层 | 一句话：逼主角获得什么 / 提供什么信息 | 不要求 |
| 幕级BOSS/盟友 | 功能 + 有趣 | 战斗机制卡或关系卡 + 人物原型 | 必须：出处 + 参考对象 + 体感描述 |
| 卷级BOSS/核心盟友 | 功能 + 有趣 + 立体 | 完整人物小传 + 人物原型 | 必须：出处 + 参考对象 + 体感描述 |
| 跨卷动态敌人/长期伙伴 | 全维度 + 成长线 | 人物小传 + 独立成长曲线 + 人物原型 | 必须：出处 + 参考对象 + 体感描述 |

### 3.4 人物原型要求

幕级及以上的敌人、盟友、重要NPC必须标注人物原型。人物原型不是简单的"性格标签"，而是一个可感知的具象参考，让设计者和写作者在脑海中立刻调出这个角色的行为方式、说话语气、决策风格。

人物原型需包含三个要素：

| 要素 | 含义 | 示例 |
|------|------|------|
| 出处 | 原型来自哪部作品的哪个角色 | 《深渊主宰》索伦 |
| 参考对象 | 具体参考了这个角色的哪个方面 | 参考索伦的"数据流冷静+杀伐果断"决策风格 |
| 体感描述 | 用一两句话描述这个角色给人的直观感受，让写作者能立刻"进入"这个角色 | 冰冷、精准、像手术刀一样切入要害，杀完人后平静地擦拭刀刃 |

人物原型可以来自任何来源：小说、影视、动漫、游戏、历史人物、现实中认识的人。一个角色可以混合多个原型（如"索伦的冷静 + 汉尼拔的优雅 + 织田信长的野心"），但必须标注清楚每个原型贡献了什么。

原型标注的价值：当多个sub-agent并行写作时，人物原型是保证角色一致性的最轻量锚点。写作者不需要读完整的人物小传，只需要看到原型标注就能快速校准角色的语气和行为方式。

### 3.5 动态敌人机制

动态敌人（敌人也会成长）是精彩度和难度的上阶设计：

- 主角有成长曲线，动态敌人也有
- 两条曲线在某些节点交叉（正面冲突），某些节点平行（各自发展）
- 动态敌人的成长速度可以略快于主角，制造"他比我先一步"的压力感
- 最终在某个卷级BOSS战中，两条曲线汇聚

深渊主宰的吸血鬼神子莱茵哈特就是典型：他从和主角同级的凡人，一路封神成伪神，和主角形成"双线赛跑"结构。这种结构的精彩度远超"Boss蹲在副本等主角来打"。

## 4. 管线重构方案

### 4.1 重构后的完整管线

```
seed → world → 卷序规划 → plot → 白描 → 写作
```

核心变化：在world和plot之间新增"卷序规划"层，并将plot层的设计起点从"功能位"改为"敌人阶梯"。

### 4.2 各阶段变更说明

#### seed（不变）

故事一句话核心、起点终点声明。新增要求：起点终点声明需明确主角在力量体系三段式中的起止位置。

#### world（需修改）

不能泛泛设计，必须产出三样东西：

1. **力量体系三段式骨架**——凡人/超凡/神圣（或同等的三段划分），每段之间的跃迁条件是什么。不是"修炼到多少级"，而是"什么经历/代价/认知突破才能跨段"。跃迁条件本身就是最大的敌人/危机。

2. **敌人生态图**——世界结构里每个力量层级天然存在哪些类型的威胁。不是具体某个人，是"这个层级的世界会自然产生什么样的压力源"。比如凡人段：帮派/野兽/生存环境；超凡段：半神/邪教/其他神子；神圣段：诸神/深渊领主/法则本身。

3. **世界矛盾轴**——这个世界有什么根本性冲突是贯穿全书的。这个矛盾轴是终极敌人的来源。

#### 卷序规划（新增步骤）

基于力量体系三段骨架 + 起点终点，排出全书卷序。产出物为**全书敌人阶梯表**：

```
凡人段（卷1-2）：凡人级敌人阶梯
  卷1：生存压力 → 帮派 → 低阶超凡生物
  卷2：旅途威胁 → 蜥蜴人 → 食人魔督军（凡人段天花板）
─── 跨段危机：圣者浩劫预兆 ───
超凡段（卷3-6）：超凡级敌人阶梯
  卷3：海盗 → 深海巨兽（获取神性起点）
  卷4：邪神 → 传奇刺客（建城扎根）
  卷5：吸血鬼神子（半神级动态敌人）
  卷6：猎杀之神马拉（真神级，跨段BOSS）
─── 跨段危机：封神/恐惧魔神复活 ───
神圣段（卷7-9）：神圣级敌人阶梯
  卷7-8：恶魔领主 → 诸神陨落 → 恐惧魔神
  卷9：最终敌（死亡法则/命运/世界秩序本身）
```

每个大段交界处设计一个**跨段级危机**——世界级事件，逼主角完成质变。

全书敌人阶梯表是后续plot层的设计依据。设计卷3时已经知道卷4-5的敌人是谁，可以提前埋线。

#### plot（需修改）

进入单卷设计，此时手里已有全书卷序。plot层做四件事：

1. **核心角色设计**——主角引擎（身份/欲望/伤口）+ 主要盟友
2. **敌人设计**——从卷序规划取出本卷卷级敌人，按级别分配设计深度（见§3.3）
3. **幕级敌人阶梯**——把本卷卷级敌人拆解成幕级敌人序列，填入敌人-成长耦合表
4. **剧情白描**——以"主角如何遭遇敌人 → 为什么打不过 → 如何变强 → 如何解决"为叙事骨架

#### 白描 → 写作（不变）

## 5. 敌人-成长耦合机制

### 5.1 耦合原则

每个敌人必须和主角的一种成长方式绑定：

| 敌人类型 | 主角获得 | 示例 |
|----------|----------|------|
| 数值型敌人 | 经验值/等级提升/属性突破 | 打混混→获得杀戮经验→升级盗贼 |
| 机制型敌人 | 新技能/新专长/战术思维 | 打蛇魔→学会利用地形陷阱 |
| 信息型敌人 | 情报/世界观认知/伏笔线索 | 打邪教徒→获知圣者浩劫逼近 |
| 关系型敌人 | 盟友/信仰/情感羁绊 | 保护安娜公主→获得王国盟约 |
| 系统型敌人 | 势力/领地/社会地位跃迁 | 击败海盗王→成为新海盗王 |

### 5.2 因果链要求

形成因果链：杀A获得的B能力 → 恰好是打C的必要条件。不是随机变强，是被敌人逼着定向变强。

深渊主宰卷1的因果链示例：

```
打混混 → 获得杀戮经验 → 升级盗贼
  → 获得超凡敏捷 → 才能潜入藏宝洞
    → 获得万象无常牌 → 牌召唤蛇魔
      → 蛇魔太强 → 临阵突破4级 → 获得蛇魔长剑
        → 长剑卖钱 → 获得旅行资本 → 踏上北行之路
```

每一步都是上一个敌人的解决 = 下一个敌人的触发条件。

### 5.3 残留设计（全部 vs 部分解决）

不是每个敌人都要完全解决：

- **完全解决**（约80%的幕）：敌人彻底消灭，该量级清零，进入下一量级
- **部分解决**（约20%的幕）：打退但没打死 / 打死了但引出更大麻烦

部分解决是推动跨幕/跨卷的关键手法。深渊主宰卷1结尾：吸血鬼神子没被主角打死，反而封神成功，直接成为卷2-5的长期威胁。

### 6.3 敌人阶梯与丹哈蒙圈的关系

两者正交，叠加使用：

| 层级 | 丹哈蒙圈回答 | 敌人阶梯回答 |
|------|-------------|-------------|
| 卷纲 | 这一卷的情绪节奏怎么走 | 这一卷打谁、为什么打、打完变什么 |
| 幕纲 | 这一幕的情绪功能是什么 | 这一幕的敌人是谁、逼主角获得什么 |

设计流程：先排敌人阶梯（定打谁），再套丹哈蒙圈（定情绪节奏）。功能位是敌人阶梯的自然结果，不是设计起点。

## 7. 全书因果链贯穿原则

整个管线里有一条因果链必须贯穿：

```
力量体系定义了"主角能变到什么程度"
  → 卷序规划定义了"主角在每卷变到什么程度"
    → 敌人阶梯定义了"谁逼主角变到这个程度"
      → 敌人-成长耦合定义了"打完这个敌人主角获得了什么"
        → 获得的东西恰好是打下一个敌人的必要条件
```

这条链不能断。任何一个敌人如果和这条链无关，它就是废节点。

---

## 8. 风险与注意事项

### 8.1 过度机械化的风险

敌人阶梯法可能导致plot设计变成"排boss表"的机械操作。需在skill中强调：敌人阶梯是骨架，不是全部。每个幕仍然需要叙事白描来赋予血肉——角色的情感、世界的质感、伏笔的铺设，这些不能被敌人阶梯替代。

### 8.2 卷序规划的信息量风险

全书卷序规划要求在world阶段就对全书有宏观把握。对于涌现式写作（边写边发展），这可能与"活种子机制"冲突。折中方案：卷序规划只定到三段式的大段级别 + 每段大致卷数 + 跨段危机，不要求精确定位每一卷的每一个敌人。后续卷的敌人可以在进入该卷plot设计时再细化。

### 8.3 动态敌人的复杂度风险

动态敌人追踪表增加了系统复杂度。建议限制全书动态敌人数量不超过2-3个，否则追踪成本过高且容易失控。

---

## 9. 验收标准

1. 重构后的管线能产出全书敌人阶梯表，且每卷的卷级敌人量级严格递增
2. 每个幕级敌人都能填入敌人-成长耦合表，且形成因果链（打A获得的B = 打C的必要条件）
3. 卷级BOSS有完整人物小传，跨卷动态敌人有独立成长曲线追踪表
4. 幕级及以上的敌人、盟友、重要NPC均标注人物原型（出处+参考对象+体感描述）
5. 现有的丹哈蒙圈、三段式战斗、施工卡等执行层模块不受影响
6. 在深渊主宰拆解数据上回溯验证：重构后的设计流程能覆盖原作的所有敌人节点
