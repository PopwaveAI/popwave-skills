# pop-fanqie-seed · 番茄梗搜索引擎

> 梗×机制×限制=金手指。好梗自带画面、自带反差、自带传播力、自带行为引擎。
> seed做三件事：搜热门赛道，碰撞行为框架出梗，四眼法验证能不能跑长线。不写种子文档——梗本身就是种子。

---

## SOP骨架

### Step 1: 搜梗

梗 = 一个具体的、有反差的、能让人脑补画面的点子。不是抽象标签，不是概念描述，是那个让人一看就"卧槽这个有意思"的东西。

**1a. 先问用户要需求**

第一步永远是问用户想要什么。不是等用户主动给——是主动问：
- "你想写什么类型的？有没有什么想表达的、最近看到的、觉得有意思的？"
- "有没有一个画面、一句话、一个场景在你脑子里？"
- "题材有偏好吗？都市/玄幻/科幻/古代/末世？"

根据用户回答分三条路径：

**路径A：用户直接给梗**
用户给了一个具体的梗/画面/一句话。
- "走夜路被黄鼠狼拦住问话，用公文套话回答"
- "给怪谈写员工手册——禁止食用同事"
- "合乎周礼——用古代周礼句式怼现代场景"

→ 跳到Step 2合成。

**路径B：用户给方向/偏好**
用户给了题材方向、偏好、或一个模糊的想法。
- "我想写都市+搞笑的"
- "最近觉得职场题材有意思"
- "想写个不那么严肃的"

→ agent根据用户方向搜索找梗（见1b）。

**路径C：用户没需求/完全开放**
用户说"你看着来""随便""你有什么好的"。
→ agent走扫榜找热点逻辑（见1c）。

**1b. 按方向搜梗**

用户给了方向时，执行3轮搜索：

| 轮次 | 搜索词公式 | 找什么 |
|------|---------|------|
| 第1轮 | 用户方向关键词 + "热梗/meme/流行/出圈" | 找正在传播的具体梗 |
| 第2轮 | "网文/小说" + 用户方向关键词 + "金手指/能力设定" + "有意思/离谱/反差" | 找有意思的力量机制 |
| 第3轮 | 用户方向关键词 + "反差/反转/搞笑/离谱" + "真实事件/案例" | 找现实反差素材 |

**1c. 扫榜找热点（用户没需求时）**

用户完全开放时，不猜方向——去扫番茄/起点的实时热门榜单，找正在火的题材和概念：

| 轮次 | 搜索词公式 | 找什么 |
|------|---------|------|
| 第1轮 | "番茄小说/起点中文网" + "热门/排行榜/畅销/2024/2025" | 找当前什么在火 |
| 第2轮 | "B站/抖音/知乎" + "小说推荐/安利/出圈" + "2024/2025" | 找正在出圈的概念 |
| 第3轮 | "网文" + "新题材/新套路/新趋势" + "2024/2025" | 找正在冒头的新方向 |

从扫榜结果中提炼3个不同方向的梗候选，让用户选。不是给3个同赛道的——是给3个差异化的方向让用户挑。

**1d. 行为框架碰撞**

搜索只找得到热门赛道，找不到好梗。好梗是把一个 mundane 行为框架撞到热门赛道上碰出来的。

**碰撞公式**：热门赛道（读者池）× mundane行为框架（故事引擎）× 具体场景画面（传播力）

行为框架来源——与赛道无关的日常领域，自带流程、甲乙方关系、可博弈规则：

| 素材池 | 示例行为 |
|-------|---------|
| 职场行为 | 签合同/写手册/定KPI/走流程 |
| 社会服务 | 接投诉/送快递/做客服 |
| 法律框架 | 维权/执法/合规审查 |
| 民间习俗 | 讨封问话/红白事流程 |
| 日常生活 | 做饭/装修/带娃/相亲 |

**碰撞方法**：把行为框架的流程步骤逐条映射到赛道场景。例——"HR行为"碰撞"规则怪谈"：招聘→怎么招募怪物入职？签合同→怪物要求什么条款？绩效考核→怪物KPI怎么定？每条映射就是一个章节，碰撞自然产出50章+空间。

**碰撞产出**：提炼2-3个最有画面的行为反差点，作为梗候选核心素材。

**搜梗标准——好梗必须满足4条：**
- **有画面**：读完脑子里立刻浮现场景，不用想
- **有反差**：两个不该在一起的东西放在一起了（古代×现代、严肃×日常、高贵×卑微）
- **有传播力**：网上已经有人在讨论/转发/二创
- **有行为引擎**：主角每章在做一件具体的事（谈判/处理投诉/写手册），不是在操作一个系统（抽卡/升级/兑换）

**搜梗产出——至少给用户3个梗候选：**

```
## 梗候选

### 梗1：[梗名/一句话]
来源：[哪里搜到的 + 行为框架碰撞]
反差点：[什么行为 × 什么赛道]
画面：[一句话描述脑补的场景——必须包含正在发生的具体博弈]
行为引擎：[主角每章在做什么具体行为]
可扩展方向：[不同对象/场景变体...]（标注能跑多远）

### 梗2：...
### 梗3：...
```

用户选1个，或要求重新搜。

### Step 2: 合成金手指

选定梗之后，搜匹配的机制和限制，三者碰撞合成金手指。

**2a. 金手指合成**

```
金手指 = 梗（提供形式——怎么表现）
       × 机制（提供动力——为什么有效果）
       × 限制（提供张力——为什么不能随便用）
```

**合成示例库（14个showcase）：**

*用户原创型（4个）：*

| 梗 | 机制 | 限制 | 合成金手指 |
|----|------|------|-----------|
| 大周礼时代（模仿古人说话） | 言出法随（说话有效果） | 每人一生仅生效一次 | 用周礼体说话能引人向善，但每个人只生效一次——要选对时机 |
| 黄鼠狼讨封（民间讨封梗） | 话术成真（说出口就当真） | 只能用公文套话格式 | 被妖怪拦住问话时用红头文件格式回答，对方必须照办——但格式错了也照办 |
| 全县每人每天给我一块钱 | 价值交换（给什么得什么） | 只有328元本金 | 花1块钱买1年修为，但穷到只有328元——每笔钱要精算 |
| 反女拳复仇 | 因果倒转（恶行反弹） | 只反弹女拳行为不反弹其他 | 被女拳攻击时伤害反弹——但必须先承受一次攻击才能反弹 |

*番茄top20逆推型（6个）：*

| 梗 | 机制 | 限制 | 合成金手指 | 来源 |
|----|------|------|-----------|------|
| 死亡游戏大逃杀 | 通关获得规则能力 | 每关规则不同必须变换解法 | 在死亡游戏中通关获得规则能力，但每关规则不同必须变换解法 | 十日终焉 |
| 工地实习挖出怪物 | 击杀吸收（杀A获得A能力） | 能力链依赖（必须用A能力杀B） | 杀怪物获得该能力，但必须用当前能力杀下一个 | 大一749 |
| 每天一分钟时间静止 | 命运轮盘赌命收容禁忌物 | 只有一分钟+赌命 | 每天时停一分钟收容禁忌物，时间极短且赌命 | 时停起手 |
| 重生抢在内测前 | 盗神天赋偷NPC东西带入现实 | 内测只有7小时不可重复 | 重生回内测前抢S级天赋，7小时内偷的东西带入现实 | 游戏入侵 |
| 诸神降临世界变信仰游戏 | 试炼通关获得资源 | 7天强制试炼不参加就死 | 通关试炼获得资源，7天强制参加不打就死 | 诸神愚戏 |
| 五百年前风水师醒来 | 风水术法有效 | 现代社会规则不同+对手也有术法 | 五百年前风水师在现代社会用风水术，现代规则不同且对手也有术法 | 风水之王 |

*新创作型（4个）：*

| 梗 | 机制 | 限制 | 合成金手指 |
|----|------|------|-----------|
| 扶不扶（老人摔倒社会梗） | 善行值兑换 | 只有扶了被讹的人才有值 | 扶被讹的老人获得善行值兑换能力，越被骂越强 |
| 差评返现（消费者维权梗） | 差评即真实 | 写好评反噬自己 | 写的差评会变成现实，但写好评会反噬自己 |
| 打工人魂穿老板 | 魂穿控制 | 只能穿24小时且对方必须是压榨过自己的人 | 魂穿压榨过自己的老板24小时，只限仇人 |
| 全网黑转粉 | 黑粉值转化 | 必须真的被误解不能自导自演 | 被全网黑时获得能力加成，自导自演无效 |

**反面教材（不叫化学反应，叫贴标签）：**
- ❌ "周礼×都市×搞笑" → 没有机制没有限制，不是金手指
- ❌ "体制内×规则怪谈" → 两个赛道标签拼在一起，没有故事引擎
- ❌ "重生+都市+复仇" → 三个赛道标签，缺机制缺限制
- ❌ "每天随机天赋+三合一合成" → 纯系统机制，无行为引擎，主角在操作系统而非做事

**2b. 四眼法验证**

验证对象是合成后的金手指：

**2b-1. 有没有画面？**
- ✅ "用周礼体说话引人向善，每人一次" → 画面立刻浮现
- ❌ "获得成长型系统" → 没画面

**2b-2. 限制条件是不是天然存在？**
- ✅ "每人一生一次" → 限制天然存在
- ❌ "无敌系统" → 没限制

**2b-3. 能不能想象出十几种不同场景？**
- ✅ "周礼体引人向善" → 校园怼霸凌者、商战怼奸商、直播怼喷子……
- ❌ "获得读心术" → 第3个场景就腻

**2b-4. 有没有行为引擎？**
- ✅ "给怪谈写员工手册" → 每个新怪物=新合同谈判=新冲突，行为驱动
- ❌ "每天随机天赋掉落" → 每天抽卡=换数值，机制驱动
- 判断标准：主角每章在做一件具体的事（谈判/投诉/写手册），还是在操作一个系统（抽卡/升级）？

**验证结果**：
- 四项全过 → 放行，进Step 3
- 任何一项不过 → 打回，换机制或调整限制，重新合成

### Step 3: 落盘

**只产出一个文件**：`梗.md`

```
# 梗

## 金手指
- 梗：（一句话）
- 梗来源：（用户给的/搜索找到的 + 行为框架碰撞）
- 行为引擎：（主角每章在做什么具体行为）
- 机制：（一句话）
- 限制：（一句话）
- 合成金手指：（一句话——梗×机制×限制的产物）

## 番茄简介
【关键词1】+【关键词2】+【关键词3】+【关键词4】

简介（150-300字，必须包含：梗展示+限制暗示+第一个爽点的期待感）
```

不写种子文档。不写100章推导。不写可扩展方向。不写世界规则。不写节奏框架。不写笔触风格。
**梗就是种子。金手指就是引擎。拿到梗直接去写。**

## 红线

1. 第一步必须先问用户需求——不等用户主动给，主动问
2. 搜梗必须调用搜索工具——禁止凭记忆编造
3. 必须给用户≥3个梗候选——不得只给1个
4. 梗必须经过行为框架碰撞——禁止只有赛道标签无行为引擎
5. 金手指三要素齐全——梗×机制×限制缺一不可
6. 四眼法四项全过才放行
7. 只产出`梗.md`——不写种子文档/世界规则/节奏框架/笔触风格
8. 梗不得与番茄Top20核心机制雷同
