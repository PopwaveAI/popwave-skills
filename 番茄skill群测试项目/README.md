## 核心发现沉淀

### 种子质量是天花板（R2→R4→R5验证）

- 好种子下限就高，差种子怎么写都不行
- 种子变体（无论哪种）+ 任何skill配置 > 原种子 + 完整skill
- S3冲突预埋是最有效的种子增强方式——给agent"戏剧骨架"而非设定堆砌

### skill的"先想后写"流程是最大杀手（R4验证）

- 章意图4问（"先想后写"流程）是质量最大杀手，砍掉后质量立即回升
- 验收检查表是第二杀手
- 去AI味6条作为参考时无害，作为检查表时有伤害
- 正向情感引导 > 负向规则约束

### 行为框架碰撞提升梗质量（R6验证）

- v4.1.0新增的"行为框架碰撞"使skill组3/3产出行为引擎型梗
- 非skill组仅1/3有行为引擎（2/3是系统机制型）
- 行为引擎型梗：主角每章在做具体的事（鉴定/稽查/拟协议）
- 系统机制型梗：主角在操作系统（抽卡/升级/兑换）

### 心法擅长情感，燃料擅长结构（R5验证）

- T2心法在S1/S2（情感向种子）上赢——金句更亮、beat更心酸
- T1燃料在S3（结构向种子）上赢——冲突链执行更完整
- 种子变体和skill配置之间存在交互效应，不存在单一"最优组合"

## 测试方法论

### sub-agent隔离执行

所有测试通过 TRAE sub-agent 执行，核心原则：
- 每组sub-agent只看到自己的选题和配置
- 不透露其他选题、其他组配置、或预期结论
- skill组注入SKILL.md全文，非skill组给自由创意指令

### 三种测试模式

1. **全链路测试**：seed→plot→write→review 顺序执行，验证端到端链路
2. **控制变量测试**：固定一个变量（种子或skill配置），变换另一个，隔离影响
3. **AB对照测试**：同一选题跑skill vs 非skill，对比创意质量

详细协议见 `pipeline_v3.py`。
