# fanqie-plot · 番茄场景推导引擎

> 从概念种子推导每章场景卡。极轻量——不建幕纲、不建施工卡，只产出1页场景卡驱动正文写作。
> 核心逻辑：概念变体池 → 选取变体 → 场景卡。每章场景卡≤500字（施工卡的1/4）。

---

## SOP骨架

### Step 1: 概念变体池维护

从概念种子的100章推导中选取当前阶段的变体，并补充新变体。

变体池格式：
```
| 变体编号 | 变体描述 | 生成场景类型 | 爽点类型 | 预计使用章节 |
|---------|---------|-------------|---------|-------------|
| V001 | 时停偷看考卷 | 校园/日常 | 未知→揭示 | ch1 |
| V002 | 时停赌命延长 | 赌博/生死 | 弱→强 | ch3 |
```

变体来源：
1. 概念种子100章推导（按当前章节阶段选取）
2. 已写章节的涌现信息（正文产生的新设定/新角色→新变体）
3. 外部搜索补强（子agent搜索相似作品的场景模式）

### Step 2: 场景卡批量生成

每章1张场景卡，≤500字。每次生成5-10章批次。

场景卡格式：
```
## chNNN · 章名

概念变体：V0XX - 变体描述（本章用哪种方式兑现概念）
核心爽点：压力→突破 / 弱→强 / 未知→揭示 / 被欺→反击（选1）
爽点触发位：前1/3 / 中段 / 章末
钩子：章末留什么悬念（1句话）
节奏约束：事件密度X / 概念交互≥X次 / 前300字出冲突
边界：写到哪停 / 不能碰什么（各1句话）
前章衔接：上一章结尾的最后一件事（1句话）
```

**场景卡自检**（每张卡生成后）：
- 概念变体是否明确？无变体=废章
- 爽点是否标注？无爽点=废章
- 钩子是否有追读动力？无钩子=废章
- 与前章衔接是否自然？

---

## 番茄节奏红线（7条）

| 红线 | 要求 |
|------|------|
| 前3章完成概念展示+首爽 | ch1概念激活，ch3首爽释放 |
| 每3章≥1高潮 | combat_reversal或等效爆发 |
| 前5章有概念升级 | ch5前概念必须进化1次 |
| 铺垫≤1章 | 番茄不允许2章铺垫 |
| 连续2章无概念交互=不合格 | 每章必须有概念使用 |
| 不得连续2章同章型 | 6种章型轮换 |
| 每章前1/3有钩子或mini爽点 | 不允许慢热开局 |

章型速查（详见 fanqie-write/references/章型定义.md）：
opening_shift / exploration_gain / confrontation_pressure / combat_reversal / high_tier_display / reveal_hook

# 附录：章型定义

# 章型定义（番茄版）

> fanqie-plot 和 fanqie-write 共用的章型体系。
> 每种章型定义7节拍骨架+概念兑现点+番茄密度约束。

---

## 6种章型速查

| 章型 | 使用条件 | 读者期待 | 概念兑现点 |
|:----|:--------|:--------|:----------|
| opening_shift | 第一章/概念首次展示 | 先信现实→被概念打开 | 节拍2-3：概念首次激活 |
| exploration_gain | 进入新场景/发现资源 | 跟着主角探索未知 | 节拍4-6：概念在新场景中的应用 |
| confrontation_pressure | 强弱对话/势力试探 | 看打脸反转 | 节拍5-6：概念提供信息差优势 |
| combat_reversal | 被压制/追杀/首杀 | 看绝境翻盘 | 节拍4-6：概念是翻盘关键 |
| high_tier_display | 远超主角层级的存在出现 | 看格局碾压+立志 | 节拍3-4：概念展示层级差距 |
| reveal_hook | 规则揭露/新敌人/升级 | 看真相炸裂 | 节拍5-6：概念揭示隐藏信息 |

---

## 番茄密度约束

| 章型 | 事件密度 | 爽感 | 概念交互 | 面板/金手指 |
|:----|:--------|:------|:--------|:-----------|
| opening_shift | 8-10 | ≥1（概念激活） | ≥1次（首次展示） | 概念首次弹出 |
| exploration_gain | 8-12 | ≥1（资源获取） | ≥1次（辅助探索） | 概念辅助检定 |
| confrontation_pressure | 8-12 | ≥1（打脸/反转） | ≥1次（信息差） | 概念分析数据 |
| combat_reversal | 10-12 | ≥1（翻盘爆发） | ≥1次（战斗关键） | 概念结算反馈 |
| high_tier_display | 8-10 | ≥1（格局震撼） | 1次（对比差距） | 概念显示差距 |
| reveal_hook | 8-10 | ≥1（真相揭示） | ≥1次（揭示信息） | 概念弹出伏笔 |

密度分配原则：前5章不允许连续2章无战斗类章型（combat_reversal或有战斗潜力的confrontation_pressure）。前5章至少1章是combat_reversal。

---

## 7节拍骨架

### opening_shift（开篇/概念激活章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 现实常识/日常/主角处境（前300字必出冲突） |
| 2 第一信息变化 | 概念激活——不合常理的数据/能力出现 |
| 3 压力/诱因 | 概念的限制条件暴露（无限制=无张力） |
| 4 主角动作或判断 | 主动使用概念 |
| 5 阻碍升级 | 概念使用引发更大问题（爽感爆发点） |
| 6 可见反馈 | 概念反馈——数据/物品/信息到手 |
| 7 章末钩子 | 更大的威胁或可能性出现 |

### exploration_gain（探索/资源露头章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 环境反常或新场景入场（感官序列：嗅→听→触→视） |
| 2 资源/线索露头 | 概念辅助发现目标 |
| 3 危险阻挡 | 风险评估——概念限制条件施压 |
| 4 判断收益，冒险 | 主动选择深入 |
| 5 危险扩大或深处异动 | 爽感爆发点 |
| 6 资源到手/能力变化 | 概念结算反馈 |
| 7 更深处还有更大变化 | 章末钩子 |

### confrontation_pressure（对峙/压力章）

| 节拍 | 功能 |
|:----|:----|
| 1 敌方/高位者给压力 | 出场即轻视 |
| 2 对方暴露目的/误判 | 筹码出现 |
| 3 语言羞辱/身份压制 | 被欺蓄力 |
| 4 沉默/反问/亮筹码 | 概念提供信息差 |
| 5 对方加码/旁人围观 | 爽感爆发点（被欺→反击） |
| 6 对方失态/局势微反转 | 打脸反馈 |
| 7 矛盾升级到下一场行动 | 章末钩子 |

### combat_reversal（战斗/反击章）

| 节拍 | 功能 |
|:----|:----|
| 1 明确对方优势/主角劣势 | 出场即轻视 |
| 2 主角发现缝隙/变量 | 概念提供关键信息 |
| 3 对方攻击/羞辱/封路 | 压制蓄力 |
| 4 不解释，直接执行 | 概念驱动战术 |
| 5 对方后手/位阶压制超预期 | 爽感爆发点（压力→突破） |
| 6 身体损伤/敌人失态 | 代价可见+概念结算反馈 |
| 7 反击引来更高后果 | 章末钩子 |

### high_tier_display（高位展示章）

| 节拍 | 功能 |
|:----|:----|
| 1 高位存在出场/气息压场 | 威压即事件 |
| 2 普通判断失效 | 独句段 |
| 3 旁观者/环境承受不住 | 概念显示差距——侧面定级 |
| 4 观察/避让/借势/立志 | 主角动作 |
| 5 高位行为造成大范围后果 | 爽感爆发点（未知→揭示） |
| 6 天地/组织/群众影响变化 | 反馈 |
| 7 主角未来必须抵达此层级 | 章末钩子 |

### reveal_hook（揭秘/章末钩子章）

| 节拍 | 功能 |
|:----|:----|
| 1 上一场结果/异常余波 | 余波开场 |
| 2 新解释出现 | 概念揭示隐藏信息 |
| 3 解释带来更大危险 | 短句施压 |
| 4 选择追查/隐瞒/反击 | 主角动作 |
| 5 新信息牵出更高层 | 爽感爆发点（未知→揭示） |
| 6 关系/组织判断改变 | 反馈 |
| 7 下一章必须进入新场面 | 章末钩子 |

## 技法推荐组合

每种章型推荐的微观技法组合（不超过3类）：

| 章型 | 推荐组合 |
|------|---------|
| opening_shift | 信息差博弈 + 感官锚点 + 预期违背 |
| exploration_gain | 世界观渐进揭示 + 感官锚点 + 后果链设计 |
| confrontation_pressure | 信息差博弈 + 人设微操 + 节奏微操 |
| combat_reversal | 节奏微操 + 感官锚点 + 后果链设计 |
| high_tier_display | 高位视角插入 + 恐怖奇观美学化 + 感官锚点 |
| reveal_hook | 预期违背 + 后果链设计 + 信息差博弈 |
