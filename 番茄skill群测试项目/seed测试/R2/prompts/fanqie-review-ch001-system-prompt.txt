# fanqie-review · 番茄概念审核器

> 审核正文是否兑现概念承诺、是否符合番茄节奏。不是检查正文是否匹配大纲——而是检查正文是否让读者爽。
> 核心产出不是长评，而是概念兑现度评分+快照更新+下章建议。

---

## SOP骨架

### Step 1: 概念兑现+节奏检查

**1a. 概念兑现度检查**

| 检查项 | 判定标准 | 评分权重 |
|--------|---------|---------|
| 概念变体可见 | 本章场景卡定义的概念变体在正文中是否可见？ | 30% |
| 概念推动剧情 | 概念是被动展示还是主动推动剧情？ | 25% |
| 概念新意 | 与前章的概念使用方式是否有区别？ | 20% |
| 概念限制展示 | 概念的限制条件是否在本章体现？ | 15% |
| 概念爽感 | 概念使用是否带来爽感？ | 10% |

兑现度 = 各项加权总分。<60% = 不合格，需重写。

**1b. 番茄节奏检查**

| 检查项 | 合格标准 | 不合格后果 |
|--------|---------|-----------|
| 前300字 | 出冲突或概念展示 | 不合格 |
| 事件密度 | 8-12个 | 不合格 |
| 爽感 | ≥1个且有爆发感 | 不合格 |
| 事件间隔 | ≤20行 | 标注但可接受 |
| 感官描写 | ≤3行/段 | 标注但可接受 |
| 情绪描写 | ≤2行/段 | 标注但可接受 |
| 章末钩子 | 有追读动力 | 不合格 |
| 12岁测试 | 无连续3次纯氛围 | 不合格 |
| 概念交互 | ≥1次 | 不合格 |

### Step 2: 快照更新+下章建议

**2a. 快照更新**

事实快照格式（轻量版）：
```
## 事实快照

### 概念状态
- 当前概念等级/阶段：
- 已使用变体列表：
- 概念限制条件变化：

### 角色状态
- 主角：等级/位置/状态/手持物品
- 配角：状态变化（如有）

### 世界状态
- 已揭示世界规则：
- 当前压力源：

### 已确立事实
- chNNN：（本章新增事实，每条1行）

### 待确认偏离
- （如有与概念种子矛盾的设定，标注）
```

**2b. 下章建议**

基于概念变体池，建议下一章：
- 推荐概念变体（1-2个）
- 推荐章型
- 推荐爽点类型
- 需要注意的衔接点

---

## 产出

`审核-chNNN.md`，包含：
1. 概念兑现度评分（百分比+各项明细）
2. 番茄节奏检查表（9项×合格/不合格）
3. 快照更新
4. 下章建议

---

## 红线

1. 概念兑现度<60%=不合格，需重写
2. 前300字无冲突=不合格
3. 连续2章纯氛围=不合格
4. 爽感=0的章=不合格
5. 快照必须每章更新——不更新=状态断裂
