## 2a. 剧情结构设计（基于骨架设定）

> 在写白描前，先设计剧情结构——幕序列/高潮场景/悬念分层。从骨架.md的势力/危机/设定范围推导，不能凭空设计。

### 2a-1. 幕序列（3-4幕）

从骨架.md的势力结构（4层梯度）+危机倒计时+第一卷设定范围推导幕序列。每幕=一个"对赌难度升级"的弧段。

```markdown
### 第一幕：「幕名」
- **功能位**：建置（建立世界+第一次小对赌+埋大对赌的伏笔）
- **对赌难度**：LV1-2（从骨架力量层级表取）
- **弧线**：（一句话——主角从XX到XX，付出了什么代价）
- **碰撞势力**：（从骨架势力结构表取——第1层势力）
- **场景地标**：（从骨架城市结构表取）
- **章节预算**：ch001-025（ch001已由seed产出，实际写ch002-025）
- **幕末钩子**：第一幕结束时读者在想什么？

### 第二幕～第四幕同上格式
```

每幕的设计原则：**对赌难度必须逐幕升级**。

### 2a-2. 核心高潮点（2-3个）

从骨架.md的危机倒计时叠加节点推导高潮。

```markdown
### 高潮1：「名称」
- **低谷**：（主角面对什么困境——从骨架危机倒计时表取）
- **高潮**：（主角如何爆发——从骨架力量层级+金手指机制推导）

- **对应危机叠加**：（哪几条倒计时同时到期）

### 高潮2/3同上格式
```

## 2b. 叙事流剧情白描

### 2b-1. 叙事流写法（禁止提纲式）

| 禁止 | 要求 |
|:--|:--|
| "ch010，卡森动手了。不是直接打上门——是切断了公会的补给线。" | "公会断粮的第三天。大厅里的委托板空了，墙角的漏雨滴在空椅子上。亚力克斯坐在柜台后面，把最后一壶茶倒进杯子里——没客人了，用不着泡两壶。'被掐住了，'他说。" |
| "与此同时，灰蛇在赌场里摔了杯子。" | 必须交代灰蛇为什么摔杯子、他知道了什么、他下一步要干什么——用场景展示，不写时间戳 |
| "歌洛莉出现。她问了一句。" | "马车停在公会门口的时候，雨刚好停了。车厢里走下来一个穿深蓝色长袍的女人。亚伦的手按在匕首柄上。不是因为怕——是因为她的影子在路灯下分成了两道。" |

**核心规则**：
- 每一段白描必须有至少1个**具体画面**（读者能在脑子里看到的）
- 每一段白描必须有至少1个**情绪锚点**（主角/配角此时此刻的感受——不是"他感到愤怒"，是"他把茶杯捏碎了，然后一条一条地捡起碎片"）
- 不使用"ch0XX"做段落开头——段落的边界是氛围切换，不是章节号

### 2b-2. 信息差/伏笔/钩子内联标注

三种标注内联在白描中，不断白描的叙事流：

```
（信息差：读者知道亚伦手里只剩3张牌了，但灰蛇不知道）
（伏笔：下水道的尸体穿着神殿区的衣物——ch015揭示这些是祭坛的供品）
（钩子：门缝里塞进来一封信，落款是北地女巫议会的冰霜玫瑰印章）
```

**标注原则**：
- 每段白描至少1个标注（信息差/伏笔/钩子，任选）
- 标注不是分析——是总结这段给读者的"额外信息"
- 伏笔必须标注预计揭示章

## 2b-3. 推演流程

基于2a剧情结构设计（幕序列+高潮点+悬念分层）+骨架.md设定，按幕分段写白描。

### 每段白描的结构

```markdown
## 第一幕：「幕名」

### [场景标题——气氛关键词]
[叙事流正文，3-5段，不写章节号，以氛围切换分界]
（信息差：XXX）
（伏笔：XXX，chXX揭示）
（钩子：XXX）

### [下一个场景]
...
```

### 推演步骤

1. 从2a-1幕序列取第一幕的弧线
2. 查骨架.md城市结构表→确定场景发生的地标
3. 查骨架.md力量层级表→确定主角当前对赌难度
4. 查骨架.md势力结构表→确定碰撞的势力
5. 查骨架.md危机倒计时表→确定推进的倒计时
6. 写成叙事流（氛围+情绪+画面+事件推进）
7. 标注信息差/伏笔/钩子

---

### 2b-4. 白描质量自检

每段白描写完后，逐条自检：

| 自检项 | 标准 | 不通过处理 |
|:--|:--|:--|
| 有画面吗？ | 读完后脑子里能浮现场景——不能只是"谁做了什么" | 补画面 |
| 有情绪吗？ | 能感受到主角/配角的情绪——不能只是事件陈述 | 补情绪锚点 |
| 有温度吗？ | 能感受到这一段是紧张/温馨/压抑/轻松——不能中性平淡 | 强化氛围关键词 |
| 有标注吗？ | 每段至少1个标注（信息差/伏笔/钩子） | 补标注 |
| 不写章节号吗？ | 段落不以"ch0XX"开头 | 改写 |

# 创意.md

## 一句话简介
灵根废柴的陈渡被踢到濒临倒闭的杂物科当科长，他没法修仙，但他用KPI、绩效考核和资源整合，对腐朽的宗门体制进行一场自上而下的"降维打击改革"。

## 创意
一个被宗门淘汰的废灵根，用现代企业管理学改革修仙宗门——当别人在修炼时，他在写《杂物科第四季度业绩考核表》；当别人突破金丹时，他在跟丹峰谈"百草订单"的返点。修仙靠灵根，管理靠脑子——而他脑子够用。

## 创意来源
用户选定方向：王道1——宗门公务员：修仙界底层改革。情绪基调：荒诞轻喜剧+权谋博弈。

## 金手指
- **创意**（形式）：宗门《管理条例》——陈渡发现修仙界的"制度文件"对灵根废柴也有效，只要文件格式规范、流程合规、领导签字盖章，就能产生类似法术的效果。
  - 具体表现：他写的公文贴在门上能当阵法用，签的调令能让人瞬间移动，盖了章的采购单能凭空变出灵石——前提是格式绝对规范、流程一个不落、签字盖章齐全。
- **机制**（动力）：制度即力量——在宗门体系内，文件的效力不依赖灵力，而依赖流程的完整性和权力层级的背书。陈渡不需要灵气，他只需要知道"这个文件该找谁签字、用什么格式、走什么流程"。
  - 核心逻辑：一个章=一个小法术，两个章连签=一个中阶法术，宗门大印+掌门签字=高阶禁咒级。制度越完善，文件威力越大。
- **限制**（张力）：
  1. **格式反噬**：公文格式错一个字、少一个章、走错一个流程，法术就会失控——申请宿舍调令写错名字，申请者会被传送到粪坑；采购单盖错章，采购物会变成攻击性物品攻击陈渡本人。每次用金手指都有"格式焦虑"。
  2. **职权范围限制**：陈渡签发的文件只对"杂物科编制内"或"签字盖章流程中涉及的对象"有效。他想用文件攻击宗主？不行——宗主不在他的管辖范围内，文件没效力。他必须先把对方纳入自己的"职权射程"。
  3. **制度反噬代价**：每成功签发一份"越级文件"（如绕过峰主直接跟宗主汇报的调令），陈渡的"制度反噬值"上升。反噬值到临界点，宗门天道会自动生成针对他的"制度漏洞"——比如安排一个职权完全重叠的副科长来制衡他，或突然空降一个"制度监察使"。每一份文件的背后，都在加固或松动他的权力根基。
  4. **签字背书透支**：陈渡可以替别人签字背书（帮下属请假、替外门弟子担保物资），每背一次书，他的个人"制度信誉度"就消耗一分。信誉度归零时，他签发的任何文件都会自动失效——包括他手上所有已生效的管理制度。所以背书的对象和数量必须精准控制。

## 行为引擎
