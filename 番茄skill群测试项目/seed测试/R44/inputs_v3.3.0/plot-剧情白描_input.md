## 定位

plot 消费 world 产出的骨架.md（全书设定），专注**叙事创作**——先从全书设定中圈出本卷涉及的设定并充实（本卷设定快照），再列出≥6条分支剧情线，然后分幕串起（卷40~60章/幕8~12章/4~5幕），最后把结构化的世界设定变成"给朋友讲故事"的叙事流白描。

**与 world 的分工（v3.3.0明确边界）**：
- world 专注**设定设计**（理性推导）——世界是什么样的，产出骨架.md（全书设定3-8卷200万字级）
- plot 专注**叙事创作**（感性创作）——故事怎么讲，从骨架.md全书设定中切片消费本卷设定，产出剧情白描.md
- **plot职责（v3.3.0四层）**：2a本卷设定快照→2b分支剧情线≥6条→2c分幕设计→2d叙事流白描
- **plot不做**：设定设计（力量体系/地图/势力/危机的全书构筑是world的事）

## 幕/卷统一格式定义（world+plot共用）

| 单元 | 定义 | 章数 |
|:--|:--|:--|
| **幕** | 完整起承转合的剧情单元 | 8~12章 |
| **卷** | 多个幕，主角/世界完成一次大的推进 | 40~60章（即4~5幕） |

**换算**：一卷=4~5幕（8~12章×4~5=32~60章，落在40~60区间）。
**plot视角**：每卷分4~5幕写白描，每幕8~12章的起承转合。

1. **必须加载骨架.md**——禁止凭空编造世界。骨架缺失标注缺失，不阻塞但质量下降

7. **Step文件链式加载**——Step1内化在本文件直接执行→读取steps/step2.md→读取steps/step3.md

# Step 2: 本卷设定快照+分支剧情线+分幕设计+叙事流剧情白描

> **v3.3.0：统一幕/卷格式+重构四层结构。** 幕=8~12章起承转合单元，卷=40~60章4~5幕。plot先从骨架全书设定圈出本卷涉及的设定并充实（2a本卷设定快照），再列出≥6条分支剧情线（2b），然后基于分支线分幕（2c），最后按幕写叙事流白描（2d）。

## 2a. 本卷设定快照（从骨架全书设定圈出+充实）

> **v3.3.0：从"切片"升级为"快照"。** 不只是从骨架圈范围，还要往地图里填充势力和彼此矛盾——不是空地图，是可直接写白描的实地图。

### 2a-1. 确定本卷卷号

从用户输入或项目状态确定当前要写第几卷。第一卷从ch001（seed黄金首章）之后开始；后续卷从上一卷卷末钩子延续。

### 2a-2. 从骨架.md全书设定圈出本卷范围

查骨架.md的"各卷设定切片预览"（world Step 3c产出），取本卷对应的范围：
- 力量体系范围（从全书5-7层中取本卷涉及的2-3层）
- 涉及地图区域（从全书所有区域中取本卷涉及的2-3个）
- 势力范围（从全书4层中取本卷涉及的层级）
- 危机范围（从全书终极+阶段性危机中取本卷爆发的）

### 2a-3. 充实为本卷设定快照

圈出范围后，**充实**成可直接写白描的实地图。不发明新设定，但要把骨架的设定充实成本卷可写的密度：

```markdown
### 本卷（第X卷）设定快照

#### 力量体系范围
- **主角起点快照**：阶位N（开卷时主角的力量状态——具体到能做什么/不能做什么）
- **主角终点快照**：阶位M（卷末主角的力量状态——本卷成长目标）
- **卷级敌人的力量范围**：阶位X-阶位Y（本卷主要敌人的力量分布，和主角的差距在哪）

#### 登场势力/角色
- **本卷碰撞势力**：势力A（第N层）+势力B（第M层）——每个势力标注：控制什么资源/弱点/和上层关系
- **本卷登场角色**：主角+配角A+配角B+反派A+反派B——每个角色标注：核心欲望/势力归属/和主角的关系张力

#### 涉及地图（填充势力和矛盾，不是空地图）
- **区域A**：[具体描述——这个区域长什么样/谁盘踞在这里/这里有什么资源]
  - 盘踞势力：势力A（控制XXX资源）
  - 内部矛盾：势力A内部XXX派vsXXX派的冲突
  - 和区域B的矛盾：势力A和势力B在XXX上的利益冲突
- **区域B**：[具体描述]
  - 盘踞势力：势力B
  - 内部矛盾：...
  - 和区域A的矛盾：...
- **区域间矛盾网络**：区域A的势力A ↔ 区域B的势力B（矛盾核心：XXX）——这是本卷白描的冲突地基

#### 整卷主要矛盾/主要剧情走向
- **主要矛盾**：（一句话——本卷主角要解决的核心问题是什么）
- **主要剧情走向**：（3-5句话——本卷从开头到结尾的大致走向，不是分幕，是方向）
```

### 2a-4. 快照自检

| 自检项 | 标准 | 不通过处理 |
|:--|:--|:--|
| 力量范围从骨架取？ | 不能自己发明力量层级 | 回骨架取 |
| 地图是空的吗？ | 每个区域必须填充盘踞势力+内部矛盾+区域间矛盾 | 补充实 |
| 势力矛盾网络有吗？ | 区域间矛盾必须标注矛盾核心 | 补矛盾 |
| 主角起点/终点快照有吗？ | 必须具体到能做什么/不能做什么 | 补具体 |
| 卷级敌人力量范围标注了吗？ | 必须和主角力量有差距 | 补差距 |
| 整卷主要矛盾一句话能说清吗？ | 不能模糊 | 重新提炼 |

**快照通过后，后续2b分支剧情线、2c分幕、2d白描都基于本卷设定快照，不再直接引用全书设定。**

## 2b. 分支剧情线（≥6条）

> **v3.3.0新增。** 先有线网，再组织成幕。每幕内部多线交织，不是单线跑。从2a本卷设定快照推导分支线。

### 2b-1. 列出≥6条分支剧情线

比例分布：主线1~2条 + 支线3~4条 + 暗线1~2条。

```markdown
### 分支剧情线

#### 主线（1~2条）
1. **[主线名]**（主角成长线）
   - 类型：主线
   - 起：[本卷开始时主角的状态/目标]
   - 止：[本卷结束时主角的状态/目标]
   - 涉及势力：势力A / 势力B
   - 涉及地图：区域A / 区域B
   - 和主要矛盾的关系：[这条线如何推进整卷主要矛盾]

2. **[主线名]**（卷级危机线，如有第二条主线）
   - ...

#### 支线（3~4条）
3. **[支线名]**（配角线/势力博弈线/悬念揭示线）
   - 类型：支线
   - 起：...
   - 止：...
   - 涉及势力：...
   - 涉及地图：...
   - 和主线的关系：[这条支线如何影响主线]

4. **[支线名]**
   - ...

5. **[支线名]**
   - ...

6. **[支线名]**
   - ...

#### 暗线（1~2条）
7. **[暗线名]**（为后续卷埋的伏笔）
   - 类型：暗线
   - 起：[本卷埋下的种子]
   - 止：[本卷末暗示，但不揭示——留给后续卷]
   - 涉及势力：...
   - 涉及地图：...
   - 预计揭示卷：第X卷

8. **[暗线名]**（如有第二条暗线）
   - ...
```

### 2b-2. 分支线自检

| 自检项 | 标准 | 不通过处理 |
|:--|:--|:--|
| 总数≥6条？ | 主线1~2+支线3~4+暗线1~2 | 补线 |
| 每条有起止吗？ | 起止必须具体到状态/目标 | 补具体 |
| 每条标注涉及势力/地图？ | 必须从2a快照取 | 回快照取 |
| 支线影响主线吗？ | 支线不能独立于主线——必须标注和主线的关系 | 补关系 |
| 暗线不揭示吗？ | 暗线本卷只埋种子不揭示，标注预计揭示卷 | 确认不揭示 |

## 2c. 分幕设计（卷40~60章/幕8~12章/4~5幕）

> **v3.3.0：基于分支线分幕。** 不再单线分幕——先把分支线分配到幕，每幕内部多线交织。每幕是完整起承转合单元，8~12章。

### 2c-1. 确定本卷章数和幕数

本卷总章数=40~60章（通常第一卷25章可放宽，但目标40~60章）。幕数=4~5幕，每幕8~12章。

```markdown
### 本卷结构
- 总章数：ch002-ch0XX（共XX章，ch001已由seed产出）
- 幕数：N幕
- 幕1：ch002-ch0XX（XX章）
- 幕2：ch0XX-ch0XX（XX章）
- ...
```

### 2c-2. 分配分支线到幕

把2b的分支线分配到每幕——每幕至少推进2~3条分支线（不是只推主线）。

```markdown
### 幕1：「幕名」（ch002-ch0XX，XX章）
- **起承转合**：起（XXX）→承（XXX）→转（XXX）→合（XXX）
- **本幕推进的分支线**：
  - 主线1：[本幕推进到哪]
  - 支线3：[本幕推进到哪]
  - 暗线7：[本幕埋下什么种子]
- **对赌难度**：阶位N（从2a快照取）
- **碰撞势力**：势力A（从2a快照取）
- **场景地标**：区域A（从2a快照取）
- **幕末钩子**：本幕结束时读者在想什么？

### 幕2：「幕名」（ch0XX-ch0XX，XX章）
- **起承转合**：...
- **本幕推进的分支线**：
  - 主线1：[本幕推进到哪]
  - 主线2：[本幕推进到哪]
  - 支线4：[本幕推进到哪]
- ...

### 幕3~幕N同上格式
```

每幕的设计原则：
- **对赌难度必须逐幕升级**
- **每幕必须有完整起承转合**（不是只有"弧线"）
- **每幕至少推进2~3条分支线**（不是只推主线）

### 2c-3. 分幕自检

| 自检项 | 标准 | 不通过处理 |
|:--|:--|:--|
| 每幕8~12章？ | 不能太短（<8）或太长（>12） | 调整章数 |
| 每幕有起承转合？ | 四段缺一不可 | 补完整 |
| 每幕推进≥2条分支线？ | 不能只推主线 | 分配更多线 |
| 对赌难度逐幕升级？ | 不能平级或倒退 | 调整难度曲线 |
| 总章数40~60？ | 卷的格式要求 | 调整幕数或幕章数 |

## 2d. 叙事流剧情白描

### 2d-1. 叙事流写法（禁止提纲式）

| 禁止 | 要求 |
|:--|:--|
| "ch010，卡森动手了。不是直接打上门——是切断了公会的补给线。" | "公会断粮的第三天。大厅里的委托板空了，墙角的漏雨滴在空椅子上。亚力克斯坐在柜台后面，把最后一壶茶倒进杯子里——没客人了，用不着泡两壶。'被掐住了，'他说。" |

**核心规则**：
- 每一段白描必须有至少1个**具体画面**（读者能在脑子里看到的）
- 每一段白描必须有至少1个**情绪锚点**（主角/配角此时此刻的感受——不是"他感到愤怒"，是"他把茶杯捏碎了，然后一条一条地捡起碎片"）
- 不使用"ch0XX"做段落开头——段落的边界是氛围切换，不是章节号

### 2d-2. 信息差/伏笔/钩子内联标注

三种标注内联在白描中，不断白描的叙事流：

```
（信息差：读者知道亚伦手里只剩3张牌了，但灰蛇不知道）
（伏笔：下水道的尸体穿着神殿区的衣物——ch015揭示这些是祭坛的供品）
（钩子：门缝里塞进来一封信，落款是北地女巫议会的冰霜玫瑰印章）
```

**标注原则**：
- 每段白描至少1个标注（信息差/伏笔/钩子，任选）
- 标注不是分析——是总结这段给读者的"额外信息"
- 伏笔必须标注预计揭示章

# 创意.md

## 一句话简介
灵根废柴的陈渡被踢到濒临倒闭的杂物科当科长，他没法修仙，但他用KPI、绩效考核和资源整合，对腐朽的宗门体制进行一场自上而下的"降维打击改革"。

## 创意来源

## 金手指
- **创意**（形式）：宗门《管理条例》——陈渡发现修仙界的"制度文件"对灵根废柴也有效，只要文件格式规范、流程合规、领导签字盖章，就能产生类似法术的效果。
  - 具体表现：他写的公文贴在门上能当阵法用，签的调令能让人瞬间移动，盖了章的采购单能凭空变出灵石——前提是格式绝对规范、流程一个不落、签字盖章齐全。
- **机制**（动力）：制度即力量——在宗门体系内，文件的效力不依赖灵力，而依赖流程的完整性和权力层级的背书。陈渡不需要灵气，他只需要知道"这个文件该找谁签字、用什么格式、走什么流程"。
  - 核心逻辑：一个章=一个小法术，两个章连签=一个中阶法术，宗门大印+掌门签字=高阶禁咒级。制度越完善，文件威力越大。
- **限制**（张力）：
  1. **格式反噬**：公文格式错一个字、少一个章、走错一个流程，法术就会失控——申请宿舍调令写错名字，申请者会被传送到粪坑；采购单盖错章，采购物会变成攻击性物品攻击陈渡本人。每次用金手指都有"格式焦虑"。
  2. **职权范围限制**：陈渡签发的文件只对"杂物科编制内"或"签字盖章流程中涉及的对象"有效。他想用文件攻击宗主？不行——宗主不在他的管辖范围内，文件没效力。他必须先把对方纳入自己的"职权射程"。
  3. **制度反噬代价**：每成功签发一份"越级文件"（如绕过峰主直接跟宗主汇报的调令），陈渡的"制度反噬值"上升。反噬值到临界点，宗门天道会自动生成针对他的"制度漏洞"——比如安排一个职权完全重叠的副科长来制衡他，或突然空降一个"制度监察使"。每一份文件的背后，都在加固或松动他的权力根基。
  4. **签字背书透支**：陈渡可以替别人签字背书（帮下属请假、替外门弟子担保物资），每背一次书，他的个人"制度信誉度"就消耗一分。信誉度归零时，他签发的任何文件都会自动失效——包括他手上所有已生效的管理制度。所以背书的对象和数量必须精准控制。

### 危机压力（从限制条款推导）

**推导链**：
```

限制2(职权范围限制) → 必须先扩大职权才能处理更高层问题 → 阶段性危机：职权限界
限制3(制度反噬代价) → 越级文件触发反噬 → 阶段性危机：监察使降临
限制4(签字背书透支) → 信誉度耗尽 → 阶段性危机：制度信誉清零
终极危机(制度完全崩坏的可能性) → 当反噬值累积到临界点，宗门天道自动重置制度——所有文件失效、所有职权归零、陈渡被开除
```
