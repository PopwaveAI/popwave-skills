1. **每章必须全文加载剧情白描.md+骨架.md+current-state.md**——禁止用current-state.md替代立项包

4. **篇幅硬限制**：2000-2500字，**不准超过2500字**。超2500字=废章，必须裁剪
5. **章型必选**：每章必须从4种章型中选1个，按7节拍骨架组织结构
6. **爽感≥1/章**：每章至少1个爽感触发，用爽点引擎设计
7. **微观技法选择**：Step 2必须选3-4类微观技法并落地（含v7.2.0新增动作链+多感官优先级，必选）

## 1b. 设定库精选注入（禁止全文注入）

**红线：设定库不全文注入，只精选与本章相关的设定**

### 精选规则

根据本章白描卡/场景卡的章型+事件，只注入本章会用到的设定维度：

| 本章特征 | 注入的设定维度 |
|---------|--------------|
| 战斗章 | 力量体系（属性/技能/专长）+ 物种与天赋（种族特征） |
| 探索章 | 世界蓝图（地理）+ 势力格局（组织） |
| 交易章 | 资源与物品（价格/物品） |
| 对话章 | 术语与文明底色（专有名词）+ 历史与驱动力（背景） |
| 成长章 | 力量体系（升级/职业）+ 物种与天赋（天赋觉醒） |

### 注入方式

- 每个维度只提取与本章事件直接相关的段落（≤500字/维度）
- 最多注入2个维度（≤1000字总设定）
- 禁止注入"与本章无关的设定历史/势力详情/物品清单"

**原则：设定库是write层的"查字典"，不是"读全书"。write需要什么查什么，不查的就不注入。**

**如设定库不存在**：跳过本步，依赖剧情白描（剧情白描应该已经消化了设定库的关键信息）。

### 动作链落地规范（v7.2.0核心迭代）

**核心原则**：禁止用1个动词概括连续动作，必须拆成2-3个细动作构成链条。

**对比示例**：

| 概括式（禁止） | 动作链（要求） |
|--------------|--------------|
| 他杀死了光头。 | 他踏前一步，左手扣住光头的下颌，右手将匕首送进喉咙，刀尖从颈后穿出。 |
| 他甩了甩匕首上的血。 | 他甩动手腕，血珠顺着刀刃滑落，滴在泥地里，发出轻微的嗒嗒声。 |
| 她喂哥哥喝粥。 | 她舀起一勺粥，吹了几口气，用另一只手掰开他的嘴，小心翼翼把粥灌进去。 |
| 他站起来，走到门口。 | 他撑着床板坐起来，脚踩在地上试了试力气，扶着墙一步步挪到门口。 |

**适用场景**：
- 战斗动作（刺/砍/挡/闪）必须动作链
- 日常动作（吃/走/拿/放）关键节点必须动作链
- 情绪动作（哭/笑/怒/怕）必须动作链

**数量约束**：每章至少3处动作链，战斗章至少5处。

### 多感官优先级落地规范（v7.2.0核心迭代）

**核心原则**：每个场景至少覆盖3种感官，不堆叠（每种感官≤3行）。

**感官优先级表**：

| 场景类型 | 主感官 | 次感官 | 辅感官 |
|---------|-------|-------|-------|
| 室内日常 | 视觉 | 听觉 | 嗅觉 |
| 战斗场景 | 触觉 | 视觉 | 听觉 |
| 户外场景 | 视觉 | 听觉 | 嗅觉/触觉 |
| 情绪场景 | 触觉 | 听觉 | 视觉 |
| 神秘场景 | 听觉 | 嗅觉 | 视觉 |

**对比示例**：

| 单感官（不足） | 多感官（要求） |
|--------------|--------------|
| 雨下了一整夜。 | 雨砸在破木板上，噼啪响（听觉）。屋顶的漏洞滴着水，在地上积成小洼（视觉）。屋里弥漫着霉味和湿气（嗅觉）。 |
| 他杀死了光头。 | 匕首刺穿喉咙的瞬间，他感觉到刀刃卡在软骨里的阻力（触觉）。血溅在手上，温热黏腻（触觉）。光头的喉咙发出咯咯声（听觉）。 |

**数量约束**：每章至少2处完整多感官描写（3种感官以上），战斗章至少3处。

## 五层指导

### 第1层·笔触层

**如启用笔触DNA**：叙事距离/句式/物象/对话/情绪外化从DNA中来。

**如未启用**：用通用笔触约束：
- 叙事距离：近为主，关键场景切近
- 句式：短句为主，长句用于铺垫
- 对话引导词只用"道"
- 系统面板用【】内联格式，禁止markdown代码块
- 情绪外化：通过动作和物象，禁止直接写"他感到..."
- 角色思考碎片化，禁止完整分析句

### 第2层·节奏层

遵守节奏物理量（番茄12岁小学生读者模型）：

| 指标 | 约束 |
|:----|:----|
| 事件间隔 | ≤20行 |
| 感官连续 | ≤3行 |
| 情绪连续 | ≤2行 |
| 事件密度 | 8-12个/章 |
| 蓄力上限 | ≤2段 |
| 信息释放 | ≤3段 |
| 微钩子 | 每3-5行 |

**按章型7节拍骨架组织结构**。每节拍有明确功能，不要混乱。

### 第3层·格局层

- 本章在长线中的位置（推进了哪条L2剧情线？）
- 本章的势力格局变化（主角的位置变化？）
- 本章的世界观信息释放（释放了什么设定？）

### 第4层·爽感层

- **主推爽感必须触发**（章型绑定的主推爽感）
- **爽感闭环**：触发→爆发→后果
- **爆发后三段式**：爆发≤3行 → 反馈≤2段 → 余韵≥1段
- **小爽沿途**：每3-5事件1个小爽
- **即时奖励**：每章≥1次行动→可见回报

### 第5层·微观层（v7.2.0增强）

落地Step 2c选的3-4类微观技法：

**基础6类技法**：
- **信息差博弈**：围绕"谁知道什么"设计至少1次博弈
- **感官锚点**：每场景至少1个具体生理感官细节
- **预期违背**：建立1个预期，然后打破
- **节奏微操**：一章内快慢交替至少2次
- **人设微操**：主角+1配角各有1个小动作/小习惯展示
- **后果链设计**：章内事件产生至少1个可见后果涟漪

**v7.2.0新增技法**：
- **动作链** ⭐：战斗/日常/情绪动作必须拆成2-3个细动作，禁止概括式。每章至少3处，战斗章至少5处
- **多感官优先级** ⭐：每场景至少覆盖3种感官（视觉+听觉+触觉/嗅觉），不堆叠（每种≤3行）。每章至少2处完整多感官描写，战斗章至少3处

**动作链落地检查**（写作时自检）：
- [ ] 是否有"他做了XX"这种概括式动作？
- [ ] 战斗动作是否拆成2-3个细动作？
- [ ] 情绪动作是否通过身体细节外化？

**多感官落地检查**（写作时自检）：
- [ ] 开篇场景是否覆盖3种感官？
- [ ] 战斗场景是否覆盖触觉+视觉+听觉？
- [ ] 是否有连续超过3行的同一种感官？

- 超过2500字
- **用1个动词概括连续动作**（v7.2.0新增）

## 4a. 字数检查

**统计正文字数**（不含交付面板，不含markdown标记）：

| 字数区间 | 判断 | 处理 |
|---------|------|------|
| 2000-2500字 | ✅ 合格 | 进入4b落盘 |
| 1800-2000字 | ⚠️ 偏短 | 检查信息密度，补充爽感反馈或微观技法落地 |
| 2500-3000字 | ❌ 超标 | **必须裁剪**：删除描述性文字、压缩铺垫段、保留剧情推进和对话 |
| 超过3000字 | ❌❌ 废章 | **重写**：保留核心剧情推进和对话，删除所有非必要描写 |

**裁剪优先级**（从先删到后删）：
1. 环境描写（不是物象的环境描写）
2. 人物外貌描写
3. 内心独白
4. 铺垫段
5. 对话（最后删，对话是番茄读者最爱看的）

**禁止裁剪**：
- 剧情推进点
- 爽感爆发段
- 系统面板
- 章末钩子

**自动化字数自检**：模型自报字数不可信（R29测试10章全部虚报，平均虚报1637字）。**必须调用字数自检脚本验证**：

```bash
python skills/pop-fanqie-write/scripts/wordcount_check.py <章节文件.md> --auto
```

脚本会：
1. 统计实际中文字符数（不含标点/空格/markdown标记）
2. 对比自报字数，标记虚报
3. 如超标(>2500字)，调用API裁剪到2000-2500字
4. 如一次裁剪未达标，自动二次裁剪
5. 更新交付面板的字数字段

**红线：落盘前必须执行字数自检脚本，禁止直接用模型自报字数落盘。**

# 章型定义

> 番茄适配的4种章型，每种7节拍骨架。plot选章型时锁定密度约束，write按7节拍骨架渲染。

## 4种章型速查

| 章型 | 使用条件 | 读者期待 | 主动行为类型 |
|:----|:--------|:--------|:------------|
| opening_shift | 第一章/世界规则松动/金手指激活 | 先信现实→被打开更大世界 | 收获/质变级反馈 |
| confrontation_pressure | 对峙/打脸/信息差博弈/势力试探 | 看打脸反转 | 宏观层面解决问题 |
| combat_reversal | 战斗/绝境翻盘/首杀/暗杀 | 看绝境翻盘 | 战斗 |
| reveal_hook | 揭秘/新敌人/升级/章末钩子 | 看真相炸裂 | 获得反馈/质变级反馈 |

---

## 密度约束

每种章型自带密度约束。plot选类型时自动锁定，write必须遵守。

| 章型 | 战斗密度 | 系统交互密度 | 冲突密度 |
|:----|:--------|:--------|:--------|
| opening_shift | 无要求 | ≥1次（金手指激活） | ≥1次外部压力 |
| confrontation_pressure | 无要求 | ≥1次（主角用系统分析） | ≥1次信息差博弈 |
| combat_reversal | 必须有战斗（占比≥30%） | 战斗后必须弹系统反馈 | ≥1次正面交锋 |
| reveal_hook | 无要求 | 可弹升级/伏笔/异常面板 | ≥1次信息揭露一半 |

**密度分配原则**：前5章不允许连续2章无战斗类章型。前5章至少1章是combat_reversal。

## 7节拍骨架

每种章型7个节拍。节拍功能定义是通用的，笔触绑定在笔触DNA模板中。

### opening_shift（开篇/世界松动章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 现实常识/日常/机构秩序 |
| 2 第一信息变化 | 不合常理的数据/痕迹/目击 |
| 3 压力/诱因 | 权威或群体无法解释 |
| 4 主角动作或判断 | 接近异常 |
| 5 阻碍升级 | 异常是更大物象的一角（爽感爆发点） |
| 6 可见反馈 | 群体失语/机构失效/常识崩裂 |
| 7 章末钩子 | 异常继续逼近或呼叫下一层真相 |

### confrontation_pressure（对峙/压力章）

| 节拍 | 功能 |
|:----|:----|
| 1 敌方/高位者给压力 | 出场即轻视 |
| 2 对方暴露目的/误判 | 筹码出现 |
| 3 语言羞辱/身份压制 | 被欺蓄力 |
| 4 沉默/反问/亮筹码 | 暗蓄 |
| 5 对方加码/旁人围观 | 爽感爆发点（被欺→反击） |
| 6 对方失态/局势微反转 | 打脸反馈 |
| 7 矛盾升级到下一场行动 | 章末钩子 |

### combat_reversal（战斗/反击章）

| 节拍 | 功能 |
|:----|:----|
| 1 明确对方优势/主角劣势 | 出场即轻视 |
| 2 主角发现缝隙/变量 | 物象出现 |
| 3 对方攻击/羞辱/封路 | 压制蓄力 |
| 4 不解释，直接执行 | 暗蓄 |
| 5 对方后手/位阶压制超预期 | 爽感爆发点（压力→突破）。内部=压制段≤3段+爆发段≤3行 |
| 6 身体损伤/法器破碎/敌人失态 | 代价必须可见且不可逆 |
| 7 反击引来更高后果 | 章末钩子 |

### reveal_hook（揭秘/章末钩子章）

| 节拍 | 功能 |
|:----|:----|
| 1 上一场结果/异常余波 | 余波开场 |
| 2 新解释出现 | 信息句清楚 |
| 3 解释带来更大危险 | 短句施压 |
| 4 选择追查/隐瞒/反击 | 主角动作 |
| 5 新信息牵出更高层 | 爽感爆发点（未知→揭示） |
| 6 关系/组织判断改变 | 反馈 |
| 7 下一章必须进入新场面 | 章末钩子 |

# 爽点引擎

> 5种爽感引擎，每章至少选1种触发。爽感必须闭环：触发→爆发→后果。

---

## 5种爽感引擎

### 1. 打脸循环

**触发**：对方轻视主角（≥2层）
**爆发**：翻盘≤3行，对方有可见失态
**后果**：对方失态（摔杯/变色/退步）
**约束**：轻视≥2层才能翻盘，翻盘≤3行

**示例**：
- 触发：管事嘲笑陈锋"中国来的穷学生"
- 爆发：陈锋掏出因果棋盘标注的管事贪腐证据
- 后果：管事脸色发白，旁边的侍者后退半步

### 2. 压力突破

**触发**：主角陷入绝境/资源耗尽/生命危险
**爆发**：突破≤3行，方式不可预测
**后果**：主角获得新能力/新资源/新认知

**示例**：
- 触发：斩杀线血量4/12，伤口失血
- 爆发：闭眼掐阎摩印，系统激活，标注周围NPC弱点
- 后果：获得"因果观测"能力

### 3. 未知揭示

**触发**：主角或读者发现不合常理的线索
**爆发**：揭示≤3行，揭示范围扩大
**后果**：主角认知框架改变，引发新行动

**示例**：
- 触发：搜尸发现死者身上的因果线
- 爆发：因果重演，看到死者生前被器官贩子追杀
- 后果：主角意识到印度地下器官产业链的存在

### 4. 小爽沿途

**触发**：每3-5事件必须有1个小爽
**爆发**：小爽≤2行
**后果**：主角获得可见回报（金钱/物品/信息/技能）

**示例**：
- 触发：战斗后搜尸
- 爆发：获得【战术匕首专精】技能
- 后果：主角战斗力提升

### 5. 即时奖励

**触发**：主角主动行动
**爆发**：行动→可见回报闭环
**后果**：主角获得数值/技能/物质回报

**示例**：
- 触发：主角选择救流浪汉
- 爆发：获得流浪汉的友谊+经验值
- 后果：主角在街头有了第一个盟友

## 爆发后三段式

所有爽感爆发后必须有三段式：
1. **爆发**：爽感瞬间（≤3行）
2. **反馈**：对方/环境/系统的可见反应（≤2段）
3. **余韵**：主角的收获或新压力（≥1段）

**反例**：爆发后直接跳到下一事件，没有反馈和余韵=废爽。

---

## 爽感公式

参考番茄读者模型（12岁小学生滑动阅读）：
- **压力→突破**：压力越大，突破越爽
- **弱→强**：反差越大，爽感越强
- **未知→揭示**：揭示越深，爽感越强
- **被欺→反击**：被欺越狠，反击越爽

**核心原则**：爽感必须有铺垫。无铺垫的爽感=空爽，读者无感。

每种章型有主推爽感引擎：

| 章型 | 主推爽感 | 次推爽感 |
|------|---------|---------|
| opening_shift | 压力突破 | 未知揭示 |
| confrontation_pressure | 打脸循环 | 即时奖励 |
| combat_reversal | 压力突破 | 小爽沿途 |
| reveal_hook | 未知揭示 | 小爽沿途 |

主推爽感必触发，次推爽感可选。

# 雨夜中，她将最后的米粥喂进哥哥嘴里

雨砸在破木板上，噼啪响。

薇薇安蹲在火堆前，眯着眼吹气，火星溅起来，烫得她往后缩。炉子里的湿木柴冒着浓烟，呛得她咳了两声。她用手背擦了擦眼睛，又凑近去，小心翼翼地往陶罐里添了最后一把米。

米粒在沸水里翻滚。

屋外的雨声很大，大得像整个世界都在往下塌。薇薇安回头看了一眼床上躺着的人——哥哥索伦，脸色白得像死人，嘴唇干裂，呼吸微弱得几乎感觉不到。

她数了数碗里的米粒。

只有小半碗。

罐里的粥煮得很稀，稀得能照见火光。薇薇安舀了一勺，吹了吹，凑到哥哥嘴边。他的嘴唇没张开，粥顺着嘴角流下来，滴在破枕头上。

“哥哥，你喝一点。”薇薇安小声说。

没有人回答。

她掰开哥哥的嘴唇，撬开他的牙关，小心翼翼把粥灌进去。这一次，他的喉结动了动，咽下去了。

“乖，再喝一口。”

她又舀了一勺。

老狗希斯趴在床脚，耳朵竖着，喉咙里发出低沉的呜呜声。它警惕地盯着门的方向，尾巴僵硬不动。这是一条老迈的猎犬，已经十四岁了，牙齿松动，左眼白翳，走路时后腿会发抖。但它还是每天早上爬起来，跟在她身后。

门外突然传来脚步声。

希斯站起来，发出低吼。薇薇安的手一抖，粥洒了几滴在手上。

脚步声走近了，在她的门前停下。有人在敲门，三声，很用力。

“小姑娘。”一个沙哑的声音说，“你哥哥死了没有？”

薇薇安没说话。她把陶罐放下，伸手抱住希斯的脖子。

“开门。”门外的男人说，“我听到你煮东西了。你还有米，是不是？”

“没有。”

“你撒谎。”

“真的没有了。”薇薇安的声音很小，但她尽量不让它发抖，“就……就剩最后一点，我哥哥要喝。”

门外沉默了一会儿。

“行。”那个声音说，“那你让狗别叫，我不进来。但你听好了——你哥哥要是死了，你一个人活不过三天。到时候我来找你，你把门留着。”

脚步声远了。

薇薇安的眼泪掉下来，但她很快用手背擦掉。老狗希斯回过头，用湿漉漉的鼻子蹭了蹭她的脸。

“我没事。”她小声说，“我不怕。”

她重新拿起勺子。

雨还在下。屋顶有好几处漏水，她拿了好几个破碗接着，地上已经积了几摊水。屋里弥漫着霉味和湿气（听觉+视觉+嗅觉——多感官①）。火堆噼啪响，偶尔有雨滴从烟囱里落下来，溅起一小片灰。

她想起三天前。

三天前的傍晚，哥哥从外面回来，浑身是血。他把她推进屋里，用木板抵住门，把最后一块面包塞进她手里——然后倒在地上，再也没有起来。

她不知道发生了什么。只知道那些血不是哥哥的。

薇薇安今年八岁。她瘦得像一根柴火棍，胳膊细得像树枝，手腕上全是青紫的指印。她知道自己长得很瘦，所以她每天只吃一顿饭，把省下来的食物给哥哥。

她想过一件事。

如果哥哥醒不过来，她就把自己卖掉。街上有人收孩子，尤其是女孩，瘦一点也不要紧。给的钱够给她哥哥买口棺材，请牧师来做一场简单的祈祷——这是她唯一能想到的办法。

她没跟任何人说过。

又舀了一勺粥，灌进哥哥嘴里。这一次，他的眼皮动了一下。

薇薇安没注意到。

希斯突然站起来，尾巴摇了摇。它走到床边，用鼻子拱了拱哥哥的手。

“希斯，别闹。”

老狗没有听。它用舌头舔了舔哥哥的手指，又回头看她，喉咙里发出轻轻的呜呜声。

薇薇安怔了一下。

她把勺子放下，伸手摸了摸哥哥的脸。

热的。

他的眼皮又动了一下，这一次动得很明显。

“哥哥？”

没有回答。但她看到他的手指蜷曲了一下，抓住了破褥子。

“哥哥！”

她扑过去，抓住他的手。他的手很凉，但手指在用力，捏了捏她的手心。

那一瞬间，薇薇安的眼泪决堤了。

“哥哥——哥哥——”

啜泣从她嘴里冒出来，压都压不住，越哭越大声，越哭越委屈，哭到说不出话。老狗希斯凑过来，用舌头舔她的脸，舔她的眼泪，又去舔哥哥的手。

床上的男人睁开眼睛。

那双眼睛很黑，黑得像深渊。不是人类的瞳色——薇薇安没注意到这个细节。她扑上去，抱住哥哥的脖子，把脸埋进他的胸口，放声大哭。

“哥哥……你终于醒了……你终于醒了……”

“我做了什么梦……”索伦的声音很哑，像沙砾摩擦，“梦到自己在打游戏……全是0和1……”

“你在说什么？”薇薇安抬起泪眼。

索伦眨了眨眼。

【系统初始化完成】

【欢迎回来，玩家：索伦】

【当前状态：濒死虚弱】

他看着半空中浮现的淡蓝色光屏。

光符像小虫一样钻进他的眼睛里，一行一行地滑过去——属性面板、技能树、装备栏、背包栏。他能看到自己的生命值正在缓慢恢复：4/12。他能看到自己的敏捷数值是19，智力是18。他能看到自己有两个职业：平民LV1，盗贼LV1。

这不是游戏的第一天。这是他玩了十年的那个游戏——他记得每一张地图，每一个Boss，每一次技能更新。

他只记得最后一次关服维护。登录的时候，世界一片漆黑。再睁开眼，就是这里了。

“哥哥？”薇薇安又喊了一声，声音里带着害怕，“你怎么了？”

索伦的瞳孔收缩了一下。他看着眼前这个小女孩——很瘦，很瘦，瘦得像一根柴火棍。脸颊凹陷，眼窝深陷，嘴唇干裂。

他认出了她。

这是他的妹妹。不是游戏里的NPC。是他在这个世界里，唯一的亲人。

“薇薇安。”他开口，声音很轻，“你多久没吃东西了？”

女孩愣了一下，低下头。

“我吃了。”她小声说。

“吃什么了？”

“……粥。”

“还有呢？”

她不说话了。

索伦深吸一口气，撑着床板坐起来。身体像生锈一样，每一块肌肉都在疼。他用手按住额头，发现额头滚烫，温度高得吓人。

【状态：中度饥饿（体力消耗+30%）】

【状态：轻度发烧（专注-5）】

面板上弹出一行红色警告。他看到了，但他没有关掉。他盯着那些数字，眉头皱了起来。

这个面板，不是游戏里的默认界面。

这是他自己写的MOD。

他花了整整一年半，写了七千多行代码。

索伦把目光从面板上移开，转向薇薇安。她还在哭，眼泪一颗一颗地掉。他伸手摸了摸她的头，动作很轻——她没有躲。

“哥哥没事。”

“可是你流了好多血……”

“那不是我的血。”他说，“别人的。”

这只狗，十四岁了。在贫民窟，能活十四年的狗很少见。大多数狗活不过五年——要么被人抓去吃掉，要么饿死，要么被帮派打死。

它只吃薇薇安给的。

索伦看着希斯的眼睛——浑浊，白翳，几乎看不见瞳孔。他想起了一些碎片：昏昏迷迷的时候，总有一个毛茸茸的东西贴着他，给他暖着腿。

女孩低下头，不说话了。

索伦的手紧了一下。他用力按了按床板，发出咯吱的声响。火堆里的木柴噼啪爆开，火星溅到地上，很快熄灭了。屋顶漏下来的水滴，滴答滴答地落进破碗里。（听觉+视觉+触觉——多感官②，火堆余温）

“我知道。”

索伦站起来。他撑着墙壁，一步一步走向门口。走到门口，他掀开破布帘子，往外看了一眼。

“那些欺负你的人。”他说，“他们叫什么名字？”

薇薇安愣住了。

“什么欺负？”

“你的手腕。”索伦说，“谁弄的？”

女孩下意识地把手背到身后，咬着嘴唇，不说话。

索伦没有再问。他转过身，推开门，走进了雨幕当中。破木板门在他身后关上，发出苍老的吱嘎声。

薇薇安站在原地。

一只粗糙的老狗爪子，轻轻搭在了她的脚背上。
