# R35复盘：四张地图叠加推演法

> 测试目标：验证"四张地图叠加推演法"能否从创意+世界观出发，产出高质量剧情白描
> 核心问题：plot skill能否借鉴此方法，从seed的创意产出剧情白描？

---

## 1. 测试方法

### 1.1 输入

| 输入项 | 来源 | 内容 |
|--------|------|------|
| 四张地图 | BG3游戏反向拆解 | 地理+力量+势力+危机 |
| 主角设定 | 深渊主宰原设定 | 索伦+薇薇安+游戏数据流金手指 |
| L2单元卡 | 深渊主宰原Phase2 | 3个单元卡（苏醒/藏宝洞/商队） |
| 白描格式 | pop-decon-volume step-0 | 叙事流，禁止拆结构标签 |

### 1.2 推演过程

对每个L2单元卡执行：
1. 取一句话单元描述
2. 查四张地图找坐标
3. 用因果链串联
4. 标注信息差
5. 串成叙事流

### 1.3 产出

- 四张地图文件（4个）
- 索伦注入设定（1个）
- 推演过程文件（1个）
- 卷1剧情白描（1个）

| 因果链 | ✅ 完整闭环 | ✅ 完整闭环 | 一致 |
| 信息差标注 | ✅ 自然点出 | ✅ 自然点出 | 一致 |
| 力量节点 | ✅ 明确标注 | ✅ 明确标注 | 一致 |
| 势力碰撞 | ✅ 有起因后果 | ✅ 有起因后果 | 一致 |
| 危机推进 | ✅ 多层倒计时 | ✅ 多层倒计时 | 一致 |
| 地理移动 | ✅ 清晰 | ✅ 清晰 | 一致 |

**结论**：R35推演白描在格式和密度上与原版Phase1.5接近。

### 2.2 R35白描的独特优势

**1. 跨世界观融合自然**
- 索伦的金手指（游戏数据流）与BG3的金手指（夺心魔幼体）巧妙融合
- 薇薇安的魔神神子身份与主脑觉醒形成双倒计时
- "每使用一次数据流，主脑分身就能感知到他的位置"——原版没有这种限制

**2. 信息差层次更丰富**
- 原版只有"读者知道角色不知道"
- R35增加了"角色知道角色不知道"（索伦知道主脑3个月后觉醒，但不知道戈塔什的存在）
- 三方信息差：读者 > 索伦 > 薇薇安（薇薇安连自己是魔神神子都不知道）

**3. 危机倒计时更紧迫**
- 原版：圣者浩劫15年倒计时（缓慢）
- R35：感染14天+主脑3个月+魔神同步复活（三重紧迫）

## 3. 四张地图推演法的核心价值

### 3.1 解决了"从创意到白描"的gap

之前plot skill的问题：骨架.md是结构化战略，但write层需要叙事流。中间缺了"翻译"环节。

**四张地图推演法就是翻译环节**：
```
骨架.md（结构） × 四张地图（坐标） × 因果链 = 剧情白描（叙事）
```

### 3.2 每张地图的功能明确

| 地图 | 功能 | 缺失会怎样 |
|------|------|----------|
| 地理地图 | 锚定摄像头位置 | 白描会"飘"，没有空间感 |
| 力量地图 | 标注主角成长曲线 | 白描会"平"，没有升级爽感 |
| 势力地图 | 设计碰撞和演变 | 白描会"散"，没有格局感 |
| 危机地图 | 提供时间压力源 | 白描会"慢"，没有紧迫感 |

**四张地图缺一不可**。

### 3.3 推演过程可复用

R35证明了推演过程的可复用性：
- 同样的四张地图，换一个主角（比如换成阿斯代伦），能推演出完全不同的白描
- 同样的主角，换一套四张地图（比如换成DND原版费伦），能推演出完全不同的白描

**推演法 = 主角设定 × 四张地图**，变量可控。

## 4. 对plot skill的启示

### 4.1 plot需要增加"剧情白描"产出环节

当前plot只产出骨架.md，缺失白描。建议增加：

```
plot skill新流程：
  Step 1: 加载立项包（含四张地图雏形）
  Step 2: 第一卷详规（骨架.md）
  Step 3: 【新增】四张地图叠加推演 → 剧情白描.md
  Step 4: 落盘（骨架.md + 剧情白描.md）
```

### 4.2 四张地图的产出责任划分

| 地图 | 应该由谁产出 | 理由 |
|------|----------|------|
| 地理地图 | seed阶段 | 世界观的一部分，立项时确定 |
| 力量地图 | seed阶段 | 金手指+职业体系，立项时确定 |
| 势力地图 | seed阶段 | 核心矛盾+主要势力，立项时确定 |
| 危机地图 | seed阶段 | 世界危机+时间倒计时，立项时确定 |
| 四张地图叠加 | plot阶段 | 卷纲设计时叠加，推演白描 |

**结论**：四张地图的"原料"由seed产出，plot负责叠加推演。

### 4.3 剧情白描的SOP设计建议

基于R35推演过程，plot的剧情白描SOP应该是：

```
对每个L2单元卡：
  1. 查地理地图 → 确定摄像头位置
  2. 查力量地图 → 确定主角等级变化
  3. 查势力地图 → 确定碰撞和演变
  4. 查危机地图 → 确定倒计时推进
  5. 因果链串联 → 生成叙事段
  6. 标注信息差 → 增加"读者知道但角色不知道"

把所有L2单元卡的叙事段串成卷级白描，加卷末钩子。
```

## 6. 核心结论

### 6.1 四张地图推演法可行

R35验证了"四张地图叠加推演法"能从创意+世界观出发，产出质量接近原版Phase1.5的剧情白描。

### 6.2 推演法的本质

剧情白描不是"把骨架扩写得详细"，而是"把L2单元卡的每个事件节点在四张地图上找到对应坐标，用因果链串起来"。

**公式**：`剧情白描 = L2单元卡 × 四张地图叠加 × 因果链推理`

### 6.3 对plot skill的改造方向

plot需要增加"剧情白描"产出环节，核心SOP是：
- 四张地图由seed阶段产出（作为立项包的一部分）
- plot阶段叠加四张地图推演白描
- 每个L2单元卡执行6步推演（查地图→找坐标→因果链→信息差→叙事段→串联）

### 6.4 老板的博德之门3类比验证

老板用BG3类比长篇网文的设计思路是正确的：
- 至上真神主线 = 危机地图
- 6个起源角色线 = 势力地图的多线碰撞
- 反派强度gap = 力量地图的成长曲线
- BG3地图塞支线 = 地理地图的摄像头移动
- 摄像头锁定主角 = 叙事视角锁定

**长篇网文的plot层设计与3A游戏的设计思路是同构的**，只是节奏更快、密度更高。
