# pop-qidian-write

> **兜底模板**。将剧情锚点渲染为番茄小说平台可消费的章节正文。
> 当用户未声明流派或库中无流派专属skill时，使用本文件。
> 当存在流派专属skill（如 `pop-qidian-write-dndlike/`）时，直接加载该流派skill，**不加载本文件**。
>
> **v3.0.0 核心变化**：
> - 新增 **DNA三态协议**（启用态/缺失态/trial模式）——笔触DNA按 `素材/文风锚定.md` 是否存在三态切换，缺失态不内置默认风格
> - 新增 **设定库精选注入**（≤3维度，每维度≤500字，总计≤1500字）——禁止全文注入设定文件，角色库维度必选
> - 新增 **角色库消费**——`设计/角色库/角色库.md` 是角色唯一源，write 不得自行发明角色
> - **爽感层锚定坐标系跃迁**——每章爽感爆发必须锚定坐标系位置跃迁，体现天赋加速×代价约束的张力

## 做什么

输入：项目上下文（seed/current-state/世界圣经/设计·主角设计.md/设计·角色库·角色库.md）+ 本章锚点 + **设定库精选（≤3维度≤1500字）** + **笔触DNA（三态协议）**
输出：2000-2500字的章节正文，不准超过3000字，附执行报告

文件架构：
- **SKILL.md（本文件）**：6种章型骨架 + SOP（含设定库精选注入+DNA三态协议+微观技法事前选择） + 红线 + 速查表
- **references/**：通用参考层——爽点/节奏/格局/位阶 + 微观技法（通用技法6 + 情境技法6 + 流派专属5，共17个独立文件含原文案例），按需加载
- **dna/**：流派笔触DNA文件——叙事距离/句式/物象/对话/情绪外化/章型节拍绑定，按需加载（与 `素材/文风锚定.md` 的DNA三态协议并列：dna/ 提供流派章型节拍绑定，文风锚定.md 提供全书笔触风格锚定）
- **素材/文风锚定.md**（项目空间，非skill自带）：笔触DNA标准插槽源，DNA三态协议的判定依据

> **流派专属skill**：`pop-qidian-write-dndlike/`（D&D数据面板流）等——完整独立的写作skill，包含自己的SOP+章型骨架+红线+速查表+流派技法+dna。写正文时只需加载流派skill，无需加载本文件。

## execution.mode

引用 PRD §4.5。本 skill 的 formal 必读输入：current-state.md（含下一章硬推进+人物状态+燃料队列+伏笔债务）+ dna/ 目录下的笔触DNA文件 + 用户本轮要求 齐全且达标。缺 current-state 不得标 formal。三档切换条件见 PRD §4.5，不在此重复。

| mode | 全局条件 | 落盘 |
| --- | --- | --- |
| `formal` | 本 skill 必读输入齐全且达标 | 正式落盘产出 |
| `draft` | 必读输入有缺口但可补全继续 | 产出标记 draft，落盘 |
| `trial` | 快速试做，不称完成 | 不落盘，仅对话内 |

## 正文落盘规则

正文完成后，落盘到 `涌现/正文/{书名}-第{N}章-{标题}.txt`，对话中只回摘要+钩子+创作记录。trial 模式不落盘，仅对话内。

## 怎么操作（SOP骨架）

### Step 1：加载项目上下文

读取项目的核心文件：
- `seed-种子文档.md` — 故事种子（主角/卖点/不可牺牲项/世界观/金手指/风格关键词/禁区）
- `幕纲-幕NNN-{标题}.md` — 本幕章节流概览 + prose-render交接 + 主角收益 + 主动行为表
- `current-state.md` — **下一章硬推进 + 人物状态 + 燃料队列 + 伏笔债务**（current-state 是涌现式管道给 write 的精确交接物，Step 3.5的章意图思考基于 current-state 的硬推进深化）
- `设计/主角设计.md` — 主角定位+金手指设计+**爽感矛盾公式（坐标系门槛×天赋加速×代价约束）**，Step 5爽感层锚定坐标系跃迁的依据
- `设计/角色库/角色库.md` — **角色唯一源**，本章出场角色的性格/口头禅/背景/行为模式均从此取
- `dna/` 目录下的笔触DNA文件 — 流派笔触风格（叙事距离/句式/物象/对话/情绪外化/章型节拍绑定）
- `世界圣经.md` — 法则层（力量体系/地理/历史/神系）+ 活跃层（本幕NPC/地点/组织/风俗/资源）。三层世界架构：法则层(seed建) + 活跃层(plot按幕建) + 快照层(review每章更)

### Step 2：设定库精选注入（禁止全文注入）

**红线：设定库不全文注入，只精选与本章相关的设定。每章最多注入3个维度，每维度≤500字，总计≤1500字。角色库维度必选。**

#### 注入维度选择（按章型/章号）

| 本章特征 | 注入的设定维度 | 说明 |
|:--|:--|:--|
| 战斗章（combat_reversal/high_tier_display） | 力量体系（坐标系阶位/各等级段战斗表现）+ 敌人生态（物种/天赋/弱点）+ **角色库·对战双方角色卡** | 双方性格弱点+冲突动机 |
| 对峙章（confrontation_pressure） | 势力格局（组织/位阶）+ 力量体系（参照物梯子）+ **角色库·本章出场角色卡** | 对手性格+口头禅+筹码 |
| 揭秘章（reveal_hook） | 历史与驱动力（背景）+ 力量体系（规则层）+ **角色库·本章出场角色卡** | 性格描述+背景故事 |
| 开篇章（opening_shift） | 力量体系（坐标系展示）+ 世界蓝图（地理）+ **角色库·主角角色卡** | 主角起点位置+核心欲望 |
| 探索章（exploration_gain） | 世界蓝图（地理/资源）+ 势力格局（组织）+ **角色库·本章出场角色卡** | 探索动机+遭遇角色 |
| 日常/过渡章 | 势力格局（组织）+ 地图（地理）+ **角色库·本章出场角色卡** | 风俗/资源/人物关系 |

- 每个维度只提取与本章事件直接相关的段落（≤500字/维度）
- 最多注入3个维度（≤1500字总设定）
- **角色库维度必选**——每章必须注入本章出场角色的角色卡
- 禁止注入"与本章无关的设定历史/势力详情/物品清单"

#### 角色库消费规则

- 角色库源文件：`设计/角色库/角色库.md`（角色唯一源）
- 注入方式：从角色库中提取本章出场角色（对照分幕设计的出场角色清单）的角色详细卡
- write 写对手戏时必须参考角色库中双方的性格描述和口头禅
- write **不得自行发明角色**——如剧情需要角色库中不存在的新角色，必须先回写 `设计/角色库/角色库.md`（追加到增量更新日志），再写正文

> **注意**：本文件作为兜底模板，不提供流派专属技法文件。如需流派专属技法（战斗模式/面板叙事/场景卡等），应使用流派专属skill（如 `pop-qidian-write-dndlike/`）。

### Step 2.5：DNA三态协议（素材/文风锚定.md）

**DNA源文件**：`项目空间/素材/文风锚定.md`（笔触DNA标准插槽，第一优先级）。与 `dna/` 目录下的流派笔触DNA文件并列——`dna/` 提供流派章型节拍绑定，`素材/文风锚定.md` 提供全书笔触风格锚定。

#### 三态协议

**启用态**（`素材/文风锚定.md` 存在）：
- **按需加载，禁止全文注入**：
  1. 先读DNA文档的结构索引
  2. **必须全读**：通用维度（≥4个，全部读取）+ 全书稳定特征（全部读取）
  3. **定向读取**：根据本章场景类型，只读对应的场景卡（1-3张），不读无关场景卡
  4. **禁止使用limit参数截断读取**——每个区块必须完整读取到底
- 笔触层包括：叙事距离/句式节奏/物象/对话引导词/情绪外化/感官顺序/段落呼吸/系统面板格式（如有）

**场景卡加载对照表**：

| 本章特征 | 加载的场景卡 |
|:--|:--|
| 日常/过渡/对话密集 | 对话·日常 + 描写·环境引入 + 叙述·过渡缓冲 |
| 战斗/Boss战 | 战斗·中坚boss战（+ 战斗·群战 如有） |
| 揭秘/真相/心理戏 | 叙述·揭秘心理戏 + 对话·信息交换 |
| 情感高潮/告别/牺牲 | 对话·情感高潮 + 叙述·孤独紧张 |
| 暗杀/偷袭/潜行 | 战斗·暗杀背刺 + 描写·环境引入 |
| 开场/新场景 | 描写·环境引入 + 描写·角色外貌 |
| 多场景混合 | 最多加载3张场景卡，取最核心的3种 |

**缺失态**（`素材/文风锚定.md` 不存在）：
- Step 2.5 必须提示用户："未检测到笔触DNA（素材/文风锚定.md），建议先用 pop-dna-style 提取并部署"
- **write 不内置任何默认风格**——不得用记忆中的风格代替DNA
- 用户拒绝部署 → 进入trial模式

**trial模式**（用户声明基础风格）：
- 用户须显式声明基础风格倾向（如"冷硬短句""网文白话流"等），或接受API自由发挥
- write skill 不内置任何笔触约束，按用户声明风格写作，不加载DNA文件
- 执行报告必须标注"trial模式·无DNA驱动"

**消费规则**：只学笔触（句式节奏/感官顺序/叙事距离/段落呼吸/对话引导词习惯/系统面板格式），禁止从DNA原文摘录中提取剧情设计/战斗表现/角色思维。

### Step 3：选择章型

根据 `current-state.md` 的本章硬锚点，选择最合适的章型。章型的完整定义（使用条件/密度约束/7节拍骨架/技法推荐）在 `references/章型定义.md`，必读。

6种章型速查：

| 章型 | 使用条件 | 读者期待 |
|:----|:--------|:--------|
| opening_shift | 第一章/世界规则松动/面板激活 | 先信现实→被打开更大世界 |
| exploration_gain | 进入新地点/发现资源线索 | 跟着主角探索未知 |
| confrontation_pressure | 强弱位阶对话/势力试探/布局算计 | 看打脸反转 |
| combat_reversal | 被压制/追杀/需打穿误判/首杀/暗杀 | 看绝境翻盘 |
| high_tier_display | 远超主角层级的战斗/人物出现 | 看格局碾压+立志 |
| reveal_hook | 规则揭露/旧伏笔变义/新敌人/升级揭秘 | 看真相炸裂 |

### Step 4：加载references

根据章型和场景需要，加载对应的references：

| reference | 何时加载 |
|:---------|:--------|
| `爽文章节类型池.md`（位于plot-design/references/） | Step 3.5必加载 |
| `爽点引擎.md` | 每章必加载（爽感是番茄核心） |
| `格局手法.md` | 灵能视角/超自然场景/战斗章必加载 |
| `位阶表达工具箱.md` | 需要表达力量差距时加载 |
| `通用技法/`（5个文件） | Step 4.5选定后加载对应文件——信息差博弈/后果链设计/人设微操/节奏微操/预期违背 |
| `情境技法/`（6个文件） | Step 4.5选定后加载对应文件——社会梗融入/感官锚点/情感锚点/世界观渐进/恐怖奇观/高位视角 |
| `流派专属/`（5个文件） | Step 4.5选定后加载对应文件——规则博弈/数值面板/非人视角/仪式感/知识降维 |

### Step 3.5：章意图思考

**在选完章型之后、加载技法之前，先想清楚这章到底要给读者什么体验。**

加载 `爽文章节类型池.md`（来自plot-design/references/），然后回答以下问题：

```
## 章意图思考
- 本章定位：（从10类章节类型中选1个主类型，可选1个副类型）
- 体验设计：为了让读者获得____体验，我需要重点写好____场景
- 信息植入：本章需要传递____信息，用____方式植入（回忆/对话/事件/场景，禁止旁白）
- 角色连接：本章需要加深读者对____角色的感情，通过____手段
- 爽感设计：本章的爽感点在____，压力在前____，释放在后____
- 主动行为核验：本章主角的主动行为类型是____（战斗/宏观解决问题/主动出击/收获/获得反馈/质变级反馈和成长）。如果选了"战斗"，战斗段落是否有沉浸感？如果选了"获得反馈"或"质变级反馈和成长"，面板是否弹出？
- 本章必须避免的：（不要走偏到什么方向，1~2条）
```

**红线**：
1. 意图思考必须在写作之前完成，不可跳过
2. 意图思考的结果指导后续技法选择和写作——如果选了"情感锤击型"，则Step 4.5必须包含"情感锚点设计"技法
3. 如果施工卡中已有"本章意图声明"（来自plot-design），则核验并深化，不重复填写
4. **禁止走偏到传统文学思路**——意图思考服务于爽文读者体验，不是作者表达欲

**为什么需要这一步**：
AI按固定SOP填空容易产出"节拍都在但没有灵魂"的章节。意图思考环节强制agent停下来想："这一章读者读完应该获得什么体验？应该爱上谁？应该对什么产生期待？"然后再按章型+技法执行。

### Step 4.5：微观技法事前选择

从技法文件（通用技法6 + 情境技法6 + 流派专属5）中选择**2-3类**应用于本章。不是全用，是选对、用到位。选定后**必须加载对应的技法文件**，参考其中的子技法、原文案例和"怎么用"指导来写作。

### Step 5：写作

按章型的7节拍推进剧情。每个节拍执行五层指导：

| 层 | 来源 | 决定 |
|:--|:-----|:----|
| 笔触层 | **DNA三态协议（启用态从 `素材/文风锚定.md` 取 / trial模式用户声明）** + seed风格关键词 + 世界圣经力量体系 | 怎么写（叙事距离/句式/对话气口/战斗表现/参照物梯子/POV规则） |
| 节奏层 | references/节奏约束.md | 多快写（事件密度/感官上限/爽感触发/微钩子） |
| 格局层 | references/格局表达.md | 写多大（空间跃迁/时间纵深/侧面定级/规则对抗/史诗口吻） |
| 爽感层 | references/爽点设计.md + **设计/主角设计.md的爽感矛盾公式** | 怎么爽（打脸循环/小爽链/爆发后三段式/即时奖励） |
| 微观层 | Step 4.5选定的技法文件 | 单章怎么精彩（17类技法中选定的2-3类，参考文件中的子技法+原文案例） |

**五层冲突时的优先级**：爽感层 > 节奏层 > 微观层 > 格局层 > 笔触层。因为番茄读者不会给你时间。

**微观层执行要求**：选定的2-3类技法必须在正文中可见落地。写完后在执行报告中标注每类技法落在了哪个段落。

#### 爽感层·坐标系跃迁锚定（v3.0.0新增硬约束）

爽感爆发必须锚定**坐标系位置跃迁**，不能是脱离坐标系的纯情感爽感。

- **爽感矛盾公式**（来自 `设计/主角设计.md`）：坐标系门槛 × 天赋加速 × 代价约束 = 爽感强度（三因子缺一不可）
  - 坐标系门槛：力量体系（坐标系）中跨越阶位的难度——门槛越高，突破时爽感越强
  - 天赋加速：金手指如何加速跨越——加速越明显，爽感越强
  - 代价约束：金手指的代价/限制——代价越大，突破时的张力越强
- **每章爽感爆发必须体现**：天赋加速 vs 代价约束的张力——爆发时刻是"主角借金手指跨越坐标系门槛"的瞬间，不是单纯的情绪宣泄
- **坐标系位置跃迁**：每章爽感爆发点必须能让读者明确感知主角在坐标系中的位置变化（阶位提升/势力位阶变化/信息层级跨越），纯打脸无坐标系位移=不合格

**v3.0.0新增验收项**：

- **DNA三态执行检查**：是否按三态协议执行？trial模式是否在执行报告中标注？
- **爽感坐标系跃迁检查**：本章爽感爆发点是否锚定了坐标系位置跃迁？是否体现天赋加速×代价约束的张力？

### Step 7：移交review

- 如果review决策为"正文需修改"：按review的修改项清单重写，然后再次移交review
- 如果review决策为"人工裁决"：暂停等待人工判断

## 红线

1. **隔离红线**：不搬运原作专有元素（专有人名/地名/招式名/口癖/专有桥段）。违反=不合格。
2. **锚点红线**：current-state.md的硬锚点必须按序踩中。不可触发项不得触发。违反=不合格。
3. **节奏红线**：事件密度≥8/章。爽感≥1/章。微钩子每3-5行一个。违反=不合格。
4. **Warp红线**：soul.md定义的特色失控段（如灵能视角段）是唯一允许打破节奏约束的区域，但感官连续≤5行需切口打断。
5. **篇幅红线**：2000-2500中文字，不准超过3000字。
6. **微观技法红线**：Step 4.5必须完成。选择卡必须填写。选定的2-3类技法必须在正文中可见落地。**必须加载对应技法文件**（非凭记忆写作）。未填选择卡=不合格。
7. **三层世界红线**：写正文时遇到世界圣经未覆盖的世界细节，不能现编——必须先在活跃层落盘（通知plot补建），再写正文。现编不落盘=后续漏洞。
8. **设定库精选注入红线**（v3.0.0）：禁止全文注入设定文件。每章最多注入3个维度，每维度≤500字，总计≤1500字。角色库维度必选。违反=不合格。
9. **DNA三态协议红线**（v3.0.0）：DNA三态协议必须执行。缺失态必须提示用户部署，**不内置任何默认风格**。trial模式必须在执行报告中标注。违反=不合格。
10. **爽感坐标系跃迁红线**（v3.0.0）：每章爽感爆发必须锚定坐标系位置跃迁，体现天赋加速×代价约束的张力。脱离坐标系的纯情感爽感=不合格。
11. **角色库唯一源红线**（v3.0.0）：`设计/角色库/角色库.md` 是角色唯一源，write 不得自行发明角色。如需新角色必须先回写角色库再写正文。违反=不合格。

## 2b 金手指设计

### 金手指类型+名称
- **类型**：穿越知识（结合重生信息差——不完全是重生，是获得了未来某条时间线的记忆碎片）
- **名称**：**九世杀生忆**

### 加速机制
- **加速点**：骨架.md组成2的“反馈”环节——主角在杀戮→吸收浊气→晋升的循环中，金手指不增加浊气量，而是让主角知道“杀谁收益最大、怎么杀风险最低”，同时提供“前世记忆中的关键节点”作为情报信息差。
- **加速方式**：金手指让主角在每次遭遇强大敌人/机缘时，触发“记忆碎片”——可能是一段未来某条时间线的影像（某个已被杀死的合道修士的击杀录像）、一个隐藏的弱点（这条龙的心口有一块逆鳞）、一个危险地点（这个秘境里面有陷阱）。本质上是一条只有他能看的“攻略”。
- **与众生攀登方式关系**：主角的攀登方式（散修的野路子）不变——他依然要靠自己找猎杀目标、自己生存、自己承担风险。金手指只是放大了“选目标”这个环节的效率，让他少踩坑、多踩点关键收益。这就如同一把好刀让猎人效率更高，但猎人还是得自己进山。

### 限制（红线级）
- **使用限制**：记忆碎片并非随叫随到——只有在主角遇到“前世时间线中的关键节点”（如某个曾经存在过的强大修士的陨落地、某个秘境的开启前兆）时，金手指才被动触发回忆片段。平时无法主动查询。
- **成长限制**：金手指记忆不全——主角只能看到“某条时间线中某个人的视角”，视角有限（不是全知全能的“系统提示”），且存在视角盲区（被杀的修士看不到自己被杀的真相）。因此他的“攻略”总有残缺，要靠自己补全。
- **坐标系意义保留声明**：**金手指不取消坐标系本身的意义——主角仍然需要一步步淬血开脉→业火炼神→煞灵化形……每层都需要亲手积累足够的杀戮浊气，金手指只提供“打哪儿更快”的情报，不提供“打完后直接晋级”的捷径。**

### 代价
- **单次代价**：动用记忆碎片后，主角自身会暂时失去对应的一段记忆（以“遗忘”为代价——可能是忘了某个不重要的人的脸、某个学过的低阶神通口诀、某个去过的地方的路）。这让他每次动用金手指都“丢失一部分自己”。
- **累积代价**：记忆碎片用得越多，丢失的记忆越多。到后期，他可能连自己为什么要变强（保护重要的人）都忘了，变成一个只知道杀戮任务的机器。他必须在变得更强和保持人性之间做抉择——每次使用金手指，都可能是在“抹除自己的一部分”。
- **代价来源对齐**：与骨架.md组成4（代价结构）对齐——通用代价是“心魔侵蚀”（杀戮导致人性流失），金手指的代价是“记忆丢失”（放大心魔侵蚀的具象化表现——你不是单纯变成业兽，你是慢慢忘了自己是谁）。代价是对通用代价的放大和戏剧化。

### 与引擎关系
- **金手指在引擎中的定位**：组成2（运转机制）的加速器——是“反馈”环节的增强（主角能从死亡修士的记忆中看到攻略，避免走弯路），不是引擎本身。
- **引擎闭环是否改变**：不改变——主角仍然依照“杀戮→吸收浊气→晋升”的通用闭环修炼。金手指不改变这个闭环，只让他在闭环内走得更准更快。

从此他踏入修真界，但发现自己一无所有：没功法、没资源、没门派背景。他只能靠野路子——潜伏凡人城暗杀逃亡修士、偷渡进宗门狩猎区蹭妖兽、赌命进秘境做亡命徒。金手指“九世杀生忆”在第一次杀人时被激活，让他看到了“前世死在这散修手中的修士视角”，从此他有了一个残缺的攻略系统：知道哪些地方有宝藏、哪些妖兽有弱点、哪些修士背后有人碰不得。

### 主线1：沈劫的杀戮升级线（没资源型·散修攀登方式）
- **类型**：主线
- **起止状态**：起点——淬血开脉·第一轮，体内灵池未激活，仅靠猎刀求生 → 终点——业火炼神·第三轮，业火初燃，可运转低阶神通
- **涉及势力/地图**：业火城→妖兽山脉外围→妖兽山脉中圈
- **与主要矛盾的关系**：主线推进矛盾——主角每次升级都需直面更高阶敌人，从“躲避战斗”到“主动猎杀”的转变正体现矛盾升级
- **对应的众生攀登方式**：没资源型散修的“野路子”——靠信息差找高收益低风险目标，用金手指验证攻略，先杀落单目标再挑战群居目标

### 主线2：金手指“九世杀生忆”的秘密线（金手指情报驱动）
- **类型**：主线
- **起止状态**：起点——金手指被动触发，主角只能接受记忆碎片，不知道为什么 → 终点——主角开始意识到“这些记忆碎片是未来时间线的残影”，但尚不能主动运用
- **涉及势力/地图**：妖兽山脉（初次触发）→旧战场遗迹（记忆碎片中的关键地点）
- **与主要矛盾的关系**：金手指让主角越级挑战成为可能，但每次动用都要付出记忆丢失的代价，形成“变强”与“失去自我”的张力
- **对应的众生攀登方式**：没资源型攀登方式的加速器——放大“选择目标”的环节效率，但不改变攀登方式本身

### 支线3：宋伯的过往引路线（掉下来型·宋伯的攀登方式）
- **类型**：支线
- **起止状态**：起点——宋伯隐居民间不愿提过往 → 终点——宋伯将“血鸦令”交给沈劫作为后路，点明“煞谷有当年人情可用”
- **涉及势力/地图**：业火城（宋伯的酒馆）
- **与主要矛盾的关系**：宋伯是本卷唯一的“经验包”，他的经验帮主角规避了许多新手陷阱（如妖兽中圈的潮汐陷阱、业火城散修的潜规则）
- **对应的众生攀登方式**：掉下来型——宋伯曾是“爬到顶型”（煞谷猎杀团骨干），但一次任务失败导致修为跌落，退隐底层，靠经验活着的“失败者”

### 支线4：阿七的伙伴建立线（有资源型·阿七的攀登方式）
- **类型**：支线
- **起止状态**：起点——阿七是独行散修，与主角初次相遇即交易关系 → 终点——阿七愿意与主角组队深入妖兽山脉中圈，建立信任基础
- **涉及势力/地图**：业火城→妖兽山脉外围
- **与主要矛盾的关系**：阿七是主角第一次的“同伴”，他的信任困境（幼年目睹村民互相出卖）在这条线上得到第一次验证——他必须在利益交换和情感信任之间找到平衡
- **对应的众生攀登方式**：有资源型——阿七有家族留下的残缺功法，虽然功法不全但让他起步比散修更稳，他属于“有点家底但也不够用”的中等散修

### 支线5：妖兽潮危机预警线（暗线·灵脉波动）
- **类型**：暗线（本卷仅埋种子，不揭示）
- **起止状态**：起点——金手指记忆碎片闪过“业火城将被妖兽潮攻破”的片段 → 终点——主角记住了这个警告但无法验证，决定先升级再说

- **与主要矛盾的关系**：妖兽潮是本卷末至第二卷的转折点，但对本卷前五章：主角看到片段后决定“趁妖兽潮来临前全力升级以在灾难中有自保之力”
- **对应的众生攀登方式**：没资源型的“信息差优势”——只有主角知道未来会发生什么，其他散修毫无准备，这给了他先机
- **预计揭示卷**：第二卷

### 支线6：虎爷势力的压制线（爬到顶层型·虎爷的攀登方式）
- **类型**：支线

- **涉及势力/地图**：业火城（铁血帮地盘）
- **与主要矛盾的关系**：虎爷势力代表业火城散修阶层的“规则”——他们从底层爬上来后，反过来压制新散修，控制资源。主角必须打破这个规则才能活下来
- **对应的众生攀登方式**：爬到顶层型——虎爷当初也是没资源型散修，爬上第六轮后开始拿底层散修当养料，是典型的“屠龙者变恶龙”

- **预计揭示卷**：第一卷末至第二卷初

### 第一章：觉醒业火

#### 一章白描（约1000字）

残阳如血。

沈劫跪在妖兽山脉边缘的碎石堆上，双手发抖，猎刀插在身前那具尸体的后颈。

尸体是个散修——长发束髻，身穿灰袍，腰间别着一枚劣质玉佩。半个时辰前，这人还笑眯眯地拍着他的肩说：“小兄弟，山脉深处有座古墓，我一个人进不去，你帮我望风，得了宝贝分你三成。”

沈劫知道这是陷阱。他从七岁起就在落星城的街头摸爬滚打，见过十个散修有八个靠骗底层凡人做“诱饵”活命——把人引进妖兽的伏击圈，等人被咬得半死时再出手猎妖兽，顺便洗劫死者的遗物。

他猜到了开头，但没猜到结局。

那个散修的“陷阱妖兽”——一只灵级初期的赤纹豹——确实被引了出来。但沈劫没按散修预想的那样尖叫逃跑，而是在草丛翻滚后直接冲向散修的背后，把猎刀横在对方脖子上。

“别动。”

散修先是一愣，随即笑了：“小子，你知道我是谁吗？”

“淬血开脉·第二轮，业力刚满两年——你昨晚在酒馆吹牛时说的。”沈劫的刀没动，“你在落星城骗过十七个凡人进山，十五个死了，两个疯了。”

散修的笑容僵住了。

“你怎么知道？”

“有人告诉我。”

——金手指“九世杀生忆”在昨晚第一次触发：沈劫在落星城酒馆的角落睡着时，脑海突然闪过一段碎片——

（一个男人跪在赤纹豹面前，胸口中了一爪，血从指缝渗出，用尽最后力气说：“兄弟……别相信散修……他们……”画面戛然而止。）

那是前世死在同一个散修陷阱中的另一个凡人修士的记忆。

沈劫不知道这记
