## 三层骨架理论（本skill的设计基底·消费视角）

男频小说的世界由三层骨架构成，缺一不可：

| 层 | 概念 | 定义 | 作用 | 设计方 |
|:--|:--|:--|:--|:--|
| 1 舞台 | **力量体系**(广义坐标系) | 任何形式的阶层金字塔 | 划出舞台，定义强弱刻度和位置空间 | **seed Phase 1·1d** → 骨架.md |
| 2 动力 | **动力引擎**(众生攀登系统) | 众生为什么爬+怎么爬+付出什么的系统 | 让坐标系活起来的世界级动力系统 | **seed Phase 1·1e** → 骨架.md |
| 3 助力 | **金手指**(助力定位) | 帮助主角合理化跨越坐标系的工具 | 降低跨越难度，但不能取消坐标系本身的意义 | **seed Phase 2·2b** → 主角设计.md |

> **v2.0.0核心变化**：三层骨架的设计已前移到seed（Phase 1骨架层 + Phase 2主角层）。world不再自行设计力量体系/动力引擎/金手指，改为消费骨架.md（力量体系+动力引擎）+ 主角设计.md（金手指）生长血肉（地图/势力/危机）。world保留对骨架的消费验证能力——发现问题报错回seed修改，不在world层自行补全或修改骨架。

> 剧情(表现) = 众生攀登过程的碰撞，交给 plot 层。world 只负责血肉(地图/势力/危机)。

**关键原则**（贯穿全skill）：
- **world不发明骨架，只消费骨架**——力量体系/动力引擎/金手指以骨架.md+主角设计.md为准
- 力量体系广义化——所有男频小说都有，历史赛道的门阀社会地位金字塔也是力量体系
- 动力引擎设计的是**众生如何爬**，不是主角如何爬——主角是众生之一
- 引擎是世界级的，不是主角专属
- 众生攀登方式的差异本身就是势力/人物的来源——不要脱离引擎凭空设计势力
- 金手指是助力，不能喧宾夺主

## 幕/卷统一格式定义（world+plot共用）

| 单元 | 定义 | 章数 |
|:--|:--|:--|
| **幕** | 完整起承转合的剧情单元 | 8~12章 |
| **卷** | 多个幕，主角/世界完成一次大的推进 | 40~60章（即4~5幕） |

**换算**：一卷=4~5幕（8~12章×4~5=32~60章，落在40~60区间）。

**world视角**：全书3-8卷×40~60章=120~480章（200万字级）。
**plot视角**：每卷分4~5幕写白描，每幕8~12章的起承转合。

## 2c. 地图（从坐标系空间分布生长）

> 地图从力量体系(坐标系)的空间分布生长，不是凭空设计。坐标系在空间上怎么分布，地图就怎么画。
>
> **v2.0.0强制约束：地图的空间分布必须映射坐标系（骨架.md的力量体系），空间法则（力量禁区/阶位壁垒）由力量体系决定。**

推导逻辑（MMO式）：
```
骨架.md主养成线阶位 → 每个阶位的"区域主题" → 城市/世界地图
  阶位1："新手区" → 贫民区+赌场
  阶位2："城镇区" → 码头+商业街+公会
  阶位3："高级区" → 贵族区+神殿区+地下城
```

每个区域回答：
- 对应哪个力量阶位？（坐标系空间化——引用骨架.md主养成线阶位）
- 这个区域提供什么**攀登资源**？（从骨架.md动力引擎运转机制推导——众生在不同区域攀登方式不同）
- 有什么**空间法则**？（力量禁区/阶位壁垒/不可逾越区——**由骨架.md力量体系决定**）
- 主角的**核心行为**是什么？（从骨架.md动力引擎推导——主角是众生之一，遵循众生攀登逻辑）
- 有什么**信息差**可以利用？

输出：城市结构表（区域/对应阶位/攀登资源/空间法则/主角行为/信息差来源/势力归属/出现卷——标注哪卷进入此区域）

## 2d. 势力（从众生攀登方式差异生长）

> 势力从众生攀登方式的差异中生长，不是从力量体系+地图凭空设计。势力 = 众生中**同一种攀登方式的群体**——骨架.md动力引擎组成3"众生攀登方式分层"直接生长出势力。
>
> **v2.0.0强制约束：势力从骨架.md动力引擎组成3（众生攀登方式分层）生长，每个势力标注对应的攀登方式类型。**

推导逻辑：
```
骨架.md动力引擎组成3·众生攀登方式分层：
  爬到顶层的人怎么维持 → 顶层势力（垄断资源/封锁门槛/打压后来者）
  有资源的人怎么爬     → 中层势力（资源积累/联姻/投靠）
  没资源的人怎么爬     → 底层势力（命换/投机/依附）
  掉下来的人怎么办     → 散人或反派（背景/反例/同路人）
```

每层势力必须回答（设定视角，不回答剧情怎么赢）：
1. **这个势力是哪种攀登方式的群体？**（回溯骨架.md动力引擎组成3——不能脱离引擎凭空设计）
2. **标注对应的攀登方式类型**（v2.0.0强制——有资源型/没资源型/爬到顶层型/掉下来型）
3. **这个势力控制什么资源？**（资源=进入下一阶位的钥匙）

> **关键原则：融合不是机械拼接，必须自洽。** 取A书的某组成+B书的某组成，必须检查兼容性。

| 检查项 | 检查内容 | 不通过的后果 |
|:--|:--|:--|

| 金手指不喧宾夺主 | 融合后的金手指是否取消坐标系意义？ | 喧宾夺主=需加强限制 |
| 众生攀登方式自洽 | 不同群体的攀登方式是否与坐标系匹配？ | 不匹配=势力来源断裂 |

## 动力引擎（众生攀登系统）

### 组成1：驱动逻辑

| 字段 | 内容 |
|------|------|
| **驱动力类型** | **生存压力 + 零和资源争夺**（世界级弱肉强食，无人能幸免） |
| **驱动力描述** | “深渊法则”强制所有生灵：**不杀戮就永远无法获得经验，不升级就会被更强者吞噬**。世界不存在世外桃源——没有敌人的地方，就意味着没有经验来源，自身等级停滞，最终被更高等级的流浪猎杀者盯上并灭杀。所有生灵都必须在竞争中不断向上爬，否则就是别人的养分。这是底层法则，没有任何个体能超脱。 |
| **主角欲望对齐**（Phase 2回填） | 主角的复仇欲望是这套驱动力的个人化表达：仇人是高阶主宰者（高等级+强属性），主角必须遵循世界规则杀戮升级才能获得挑战他的资格。 |

### 组成2：运转机制（全员通用闭环）

| 字段 | 内容 |
|------|------|
| **输入** | 1. 战斗/杀戮（主动攻击敌人，造成致命伤）<br>2. 魂魄收割（被杀目标的灵魂会被灵魂石板强制摄取） |
| **转化** | 被杀目标的“灵魂能量”被灵魂石板解析成：灵魂碎片（经验值）+ 属性源质（小概率）+ 技能之书碎片（小概率）+ 专长传承石（稀有）。转化效率取决于目标的等级差（同级转化率100%，低级目标转化率大幅衰减）。 |
| **输出** | 1. 主等级提升（灵魂碎片累积到阈值）<br>2. 子养成线强化（消耗碎片资源）<br>3. 临时增益（战斗中的肾上腺素爆发，面板数据短暂提升） |
| **反馈** | 升级后能挑战更强的敌人 → 获得更高品质的灵魂碎片（转化率更高） → 更快升级 → 挑战更强敌人。形成**正向飞轮**。但同时等级越高，越会吸引高阶猎手（他们需要杀手锏经验）。 |
| **闭环图** | **“猎杀→汲取灵魂→强化面板（升级/加属性/学技能）→猎杀更强者”** |

### 组成3：众生攀登方式分层

| 群体 | 攀登方式 | 对应势力层 |
|------|----------|------------|
| **有资源的人**（贵族、商会、神殿） | 花钱购买低阶奴隶/魔物进行“圈养猎杀”，通过控制经验来源稳步升级；或雇佣高等级护卫带自己刷高级怪（经验分成）。 | 各领主、大商会、神殿教会。他们控制着训练营、斗兽场、地牢等可控经验场。 |
| **没资源的人**（平民、流浪者、逃兵） | 必须亲自冒险进入野区（黑森林、深渊裂隙、废城）猎杀野怪或落单的同行。风险极高（死亡概率大），但若成功，收益全部归自己，可能一战冲级。 | 自由佣兵团、探险队、街道黑帮。成员多为底层杀戮者，互相防备。 |
| **爬到顶层的人**（主宰者） | 垄断“高等级禁区”（例如深渊的中层区域），禁止低等级者进入。同时通过制定规则（例如对领地的经验税），从下位者的杀戮行为中抽取一定比例的灵魂碎片（世界规则允许领主级设下“领域契约”）。 | 深渊领主、主宰者同盟。他们维持等级霸权，防止新人威胁自己的地位。 |
| **爬不动/掉下来的人** | 成为“耗材”——被高阶者有计划地猎杀用来升级。或者躲在最边缘的区域放弃升级（但无法真正退出，因为野兽也会攻击）。部分人组成“拒绝杀戮”的异教派（但常被当作移动经验包狩猎）。 | 废弃区、避难所（非常稀少且风险高）。他们是世界规则的牺牲品。 |

### 组成4：代价结构

| 字段 | 内容 |
|------|------|
| **通用代价** | 1. **受伤与死亡风险**：每次战斗都可能被反杀。<br>2. **灵魂磨损**：持续杀戮会导致精神污染和灵魂石板出现“裂纹”（影响属性上限），需要消耗珍贵灵魂净水修复。<br>3. **仇恨累积**：杀死敌人后，其亲友或所属势力会标记你为“仇敌”。 |
| **群体差异代价** | 1. **有资源者**：主要付出金钱资源（购买奴隶、护卫、修复材料），身体风险较低，但容易依赖外部力量，自身实战胜率偏低。<br>2. **没资源者**：付出极高生命风险，每一场战斗都是赌博，但一旦成功，真实战斗经验丰富，升级后战力更强。<br>3. **爬不动/掉下来者**：付出尊严与自由，成为他人养料，灵魂被剥离时极度痛苦。 |
| **代价累积效应** | 等级越高，升级所需灵魂碎片量指数级增长。同时，高阶者更容易被其他高阶者盯上（因为击杀高阶者获得的海量经验是快速突破的捷径）。大部分人在15级后会主动隐身低调，避免成为众矢之的。 |

## 2b 金手指设计

### 金手指类型+名称
- **类型**：系统（D&D数据面板的增强变体——数据之眼）。
- **名称**：**“深渊数据之眼”**。

### 加速机制
- **加速点**：骨架.md组成2（运转机制）中——**输入**（战斗/杀戮环节）和**转化**（灵魂碎片转化率）的双重加速。
- **加速方式**：
  1. **战斗输入阶段**：数据之眼能瞬间看到敌人的具体属性、技能冷却、弱点暴露窗口（比如“3秒后敌人左肩会暴露真身”）。主角可以打最高效的收割战，而不是盲目乱砍。
  2. **转化阶段**：数据之眼能显示当前击杀目标掉落的“灵魂碎片品质”和“掉落概率”，让主角优先选择高收益目标（而不是随机乱杀浪费精力）。
- **与众生攀登方式关系**：主角的攀登方式（没资源者的单枪匹马野区猎杀）完全不变。金手指只是提供了“野区地图上的精确采矿路线”和“BOSS的弱点提示”，让他在每次战斗中都打出高于面板的收益。

### 限制（红线级）
- **使用限制**：
  1. 每次激活数据之眼消耗精神力——连续使用3场高烈度战斗会出现“眼压过载”（双眼剧痛、视野模糊）。
  2. 对于等级高于主角5级以上的敌人，数据之眼只能看到模糊信息（如“力量？？？”），无法直接看到弱点窗口。
- **成长限制**：
  1. 数据之眼本身不升级——它始终基于主角当前的灵魂石板解析能力。主角等级越高，能获取的信息越精确（但上限是“能看到同阶及低一阶敌人的完整面板”）。
  2. 无法通过金手指跳级——主角必须自己从1级打到20级。
- **坐标系意义保留声明**：**金手指不取消坐标系本身的意义——主角仍然必须通过杀戮获取经验升级，无法跳阶，无法无视等级差；数据之眼只是让他打得更准、效率更高，但没法让他“不杀怪就升级”。**

### 代价
- **单次代价**：每次激活数据之眼（≥5秒）消耗5%~10%精神力。如果精神力见底，主角会呕吐、视线重影、思维迟缓。
- **累积代价**：过度使用→精神力上限永久下降（主角的“灵魂石板”会出现裂纹，影响技能学习速度）。为了恢复上限，需要消耗珍贵的“灵魂净水”，而这种东西只有领主级以上势力才能稳定产出。
- **代价来源对齐**：与骨架.md组成4（代价结构）的通用代价“灵魂磨损”一致——金手指的代价只是放大和加速了这个磨损过程。

### 与引擎关系
- **金手指在引擎中的定位**：组成2（运转机制）的加速器。
- **引擎闭环是否改变**：不改变——金手指只加速主角在“输入”和“转化”环节的效率；底层闭环“猎杀→汲取灵魂→强化面板→猎杀更强者”没有变。
- **主角驱动逻辑是否改变**：不改变——主角的核心欲望（复仇）仍由骨架.md组成1（世界级驱动力：生存压力+零和资源争夺）驱动，数据之眼只是让他爬得快，不是给他创造新欲望。

### 梗×机制×限制公式
| 维度 | 内容 |
|:--|:--|
| **梗** | 一个在“数据化看得见一切”的深渊世界里，主角的挂是“能比别人多看一点”——看见技能冷却倒计时、看见弱点暴露窗口、看见敌人下一秒的属性压制会消失。他是战场上的“破绽猎手”。 |
| **机制** | 每场战斗，主角可主动激活数据之眼（消耗精神力），在短暂时间内看到敌人面板数据、技能冷却和弱点暴露窗口。弱点窗口只有1-2秒，必须精准出击。 |
| **限制** | 1.精神力有限，最多连续看3场。2.高等级敌人信息模糊。3.过度使用会导致精神力上限永久下降（需灵魂净水修复）。 |

### 反模式检查
- [x] **纯翻译器（不命中）**：数据之眼不是把“打怪升级”翻译成“系统任务”，而是作为战斗辅助工具。
- [x] **纯正向循环（不命中）**：有精神力消耗和永久上限下降的代价，不是纯正反馈。
- [x] **无场景限制（不命中）**：使用有限制（精神力消耗+高等级敌人看不全），不是宇宙通用挂。
- [x] **成长过慢（不命中）**：前3章主角就能依靠金手指实现1次越级击杀（用弱点窗口+精准操作带走4级怪），兑现速度快。

### 喧宾夺主检查（红线级）
- [x] **金手指不让坐标系失去意义**：主角仍需要升级、仍需要杀怪、仍需要面对等级差的压制。数据之眼只是“帮你打得更准”，不是“帮你直接升5级”。
- [x] **金手指不取消众生攀登的意义**：无资源者仍然只有主角一个人在靠数据之眼获益；其他无资源者仍然在拼运气和蛮干。主角的攀登方式（亲自野区猎杀）没有变。
- [x] **金手指不取代引擎**：金手指只是优化“输入”和“转化”环节；底层引擎——杀戮→经验→升级的飞轮——完全没变。

## 2c 爽感矛盾设计

### 爽感矛盾公式
| 三因子 | 具体内容 |
|:--|:--|
| **坐标系门槛** | D&D等级制完全透明——主角（3级碎骨者）杀5级噬魂者，概率掉落、转化率极低、极可能被反杀。越级挑战是所有无资源者的最大门槛。 |
| **天赋加速** | 数据之眼让主角能看到敌人的“技能冷却倒计时”和“弱点暴露窗口”（1-2秒）。他可以通过精准卡窗口、极限走位、打属性弱点来弥补属性差，越1-2级击杀不再是“天方夜谭”。 |
| **代价约束** | 每场战斗激活数据之眼消耗精神力；若战斗中精神力枯竭，极可能在下一波技能时失手被反杀。过度使用永久降低精神力上限。 |
| **公式应用** | **高门槛（等级差+属性差）+ 金手指加速（弱点窗口精准越级）+ 代价约束（精神力消耗→精神力上限下降）** = 爽感强度：每一次越级击杀都像刀尖上跳舞，读者会紧张“他精神力够不够”“会不会耗尽之后暴毙”。当主角成功卡窗口打出致命一击时，压力释放感极强。 |

### 爽感闭环模板
| 阶段 | 内容 |
|:--|:--|
| **触发** | 主角被一只4级魔物（或落单同行）逼到绝境：对方等级更高、属性更优、技能更全。 |
| **爆发** | 主角激活数据之眼，看到对方一个技能进入冷却倒计时（持续2秒）——他必须在这2秒内完成近身、要害攻击、拉开距离。他完美执行：极限闪避→卡窗口爆杀→对方倒地。 |
| **后果** | 1. **位置跃迁**：经验够升一级（从3级涨到4级）。<br>2. **代价兑现**：精神力几乎耗尽，主角出现重影和呕吐。必须找一个安全角落恢复。<br>3. **新挑战**：击杀引来一队5级赏金猎人的注意——他们看到主角的“越级击杀”判定他是个潜能巨大的猎物（经验包）。 |

### 爽感类型清单
**主要爽感类型（2种）**：

| 爽感类型 | 典型场景 |
|:--|:--|
| **压力→突破** | 主角被高等级敌人压制到极限（血线见底、精神力告罄），卡数据之眼的最后一波弱点窗口完成反杀。 |
| **弱→强** | 主角以3级碎骨者身份，靠数据之眼+极限操作越级击杀4级/5级敌人——读者看到等级差时的“不可能”最终变成“看到了吗？他真的做到了”。 |

**次要爽感类型**：
| 爽感类型 | 典型场景 |
|:--|:--|
| **未知→揭示** | 数据之眼第一次揭示敌人的“属性面板”和“技能冷却”——读者跟着主角第一次看到这世界的“底层代码”，有强烈的探索爽感。 |
| **被欺→反击** | 主角被其他流浪者/势力踩脸（因为低等级），沉默不语；等对方放松警惕时，用数据之眼找到对方的战斗习惯破绽，一场快速斩杀完成复仇。 |

### 爽感节奏建议
| 节奏层级 | 频率 | 内容&安排 |
|:--|:--|:--|
| **章爽点** | 每章≥1个 | 每章至少包含1次小型冲突：如一次击中要害的精准打击（数据之眼触发）、一次成功规避伤害（看到技能冷却后闪避）、或一次猎杀并升级的面板弹出。 |

## 执行步骤

### Step 2c: 地图设计
设计MMO式地图，空间分布必须映射坐标系（骨架.md的D&D力量体系），空间法则由力量体系决定。

### Step 2d: 势力设计
设计势力，必须从众生攀登方式差异生长（骨架.md动力引擎组成3），每个势力标注对应的攀登方式类型。

### Step 2e: 危机设计
设计危机，必须与动力引擎驱动逻辑对齐（骨架.md），危机是引擎运转的阻力表现。

### Step 3: 全书设定（精简版——仅第一卷）

- 第一卷坐标系位置变化区间
- 第一卷引擎状态+演化节点
- 第一卷势力格局
- 第一卷主要区域地图
