## 做什么

输入：骨架.md + 主角设计.md + 设计/全书设定/（势力.md+全书配角.md）+ 创意.md + ch001.txt
输出：`设计/角色库/角色库.md`（角色总表+角色详细卡+增量更新日志）

**定位**：character不前置建库——world完成后根据全书设定规划分幕，按分幕设计出场角色清单按需批量塑造。character是world→plot之间的必经加工站，把骨架.md+主角设计.md+全书设定中的毛坯角色塑造成有灵魂的角色卡。

**与world的分工**：world做设定（地图/势力/危机/全书设定），产出设计/全书设定/；character做角色（性格/小传/成长数据），产出设计/角色库/角色库.md。

**与plot的分工**：character先跑，产出角色库；plot消费角色库写剧情白描。character不消费plot的产出（character在plot之前），而是根据全书设定预判分幕需求来设计出场角色清单。

## 核心约束

1. **每个具名角色必须标注攀登方式类型**——与骨架.md动力引擎组成3（众生攀登方式分层）对齐：有资源型/没资源型/爬到顶层型/掉下来型。角色的行为逻辑由攀登方式决定。
2. **角色的等级坐标必须与力量体系对齐**——角色的等级坐标引用骨架.md力量体系坐标系（主养成线阶位），不自行发明等级。
3. **角色库是write的唯一角色源**——write写正文时所有角色信息以角色库为准，不得在正文中创造与角色库矛盾的角色设定。
4. **不前置建库**——按分幕设计出场角色清单按需批量塑造，不预设配角/反派数量。

## 2a. 规划分幕+提取角色清单

### 规划第一卷分幕

从 `设计/全书设定/` 的各卷设定切片中，提取第一卷的设定范围：
- 第一卷覆盖的力量阶位范围（骨架.md主养成线）
- 第一卷碰撞的势力层级（势力.md势力4层梯度——第一卷通常碰撞前2层）
- 第一卷的主要危机（危机体系——第一卷涉及的阶段性危机）

根据以上信息，规划第一卷的分幕结构（4-5幕，每幕8-12章），标注每幕的：
- 主要场景区域
- 碰撞的势力层级
- 出场角色需求

### 提取角色清单

从以下来源提取角色清单：

从 `设计/骨架.md` 取：
- 动力引擎组成3·众生攀登方式分层——每种攀登方式的代表角色候选

从 `设计/全书设定/势力.md` 取：
- 敌人4层梯度（反派层级参照）
- 势力格局（势力归属校验）

从 `设计/全书设定/全书配角.md` 取：
- 全书配角设定（已规划的配角名单）

从 `设计/主角设计.md` 取：
- 主角定位（名字/身份/核心动机/金手指/爽感矛盾）

从 `设计/创意.md` 取：
- 主角轮廓（记忆点/行为引擎）
- 三阶段走向（第一卷处于哪个阶段）

从 `正文/ch001.txt` 取：
- 已出场角色（不可冲突）
- 角色初始状态

将以上来源去重合并，按分幕归属组织角色清单。每个角色的初始信息：
- 角色名
- 定位（主角/配角/反派·{层级}）
- 攀登方式类型（有资源型/没资源型/爬到顶层型/掉下来型——从骨架.md动力引擎组成3推导）
- 等级坐标（当前阶位——从骨架.md主养成线推导）
- 势力归属
- 首次出场幕

## 2b. 主角深度卡（1个）

从主角设计.md展开，产出完整角色卡。

**攀登方式类型**：标注主角属于哪种攀登方式（通常主角是"没资源型"或"掉下来型"——从底层爬起，金手指加速）。

**等级坐标**：标注主角的起点阶位（骨架.md主养成线阶位），与力量体系坐标系对齐。

**性格描述要求**：必须是具体行为倾向，不是"善良""勇敢"。
- ✅ "嘴硬心软，从不承认自己关心别人，但总在暗地里帮忙"
- ❌ "善良、勇敢、聪明"

**小传要求**：≥200字。角色的过去决定了他现在为什么这样做。

**成长数据**：起点阶位→终点阶位（全书终卷）+关键转折事件。与骨架.md主养成线阶位对齐。

**口头禅**：标志性用语，写对话时直接调用。

## 2c. 配角池（按分幕设计实际出场角色数）

从角色清单中提取所有标注为"配角"的出场角色，逐个塑造。

**塑造要求**：
- **攀登方式类型**：每个配角必须标注攀登方式类型（与骨架.md动力引擎组成3对齐）。配角的攀登方式决定他的行为逻辑——有资源型的配角靠资源行动，没资源型的配角靠命换/投机行动
- **等级坐标**：每个配角的等级坐标与骨架.md主养成线阶位对齐
- 参考势力.md势力格局，为每个配角锚定势力归属
- 配角必须有**独立动机**——不是"为主角服务"的工具人。每个配角有自己的欲望和目标，和主角的目的一部分重合一部分冲突
- 性格描述强制产出"超出预期的性格细节"——如冷面杀手怕猫、反派大佬按时给母亲打电话、贪财的商人偷偷资助孤儿院
- 每个配角≥200字背景故事
- 每个配角有口头禅

**配角类型参考**（不是固定模板，由分幕设计决定需要什么类型的配角）：
- 情感锚点型（驱动主角行为的出发点）
- 幽默担当型（调节节奏）
- 引路人型（带主角进入更大的世界）
- 战斗伙伴型（配合主角战斗，有视觉冲击力）
- 信息源型（提供情报，推动剧情）
- 对手型（非反派但和主角有竞争关系）

## 2d. 反派卡（按分幕设计实际出场角色数）

从角色清单中提取所有标注为"反派"的出场角色，参考势力.md敌人4层梯度逐个塑造。

**塑造要求**：
- **攀登方式类型**：每个反派必须标注攀登方式类型。反派通常是"爬到顶层型"（垄断者——维护既得利益）或"有资源型"（资源型——靠资源压制底层）
- **等级坐标**：每个反派的等级坐标与骨架.md主养成线阶位对齐，反派阶位通常高于主角当前阶位
- 每个反派必须有：**性格弱点**（主角可利用）+ **与主角的冲突必然性**（不是"碰巧挡路"——攀登方式差异导致冲突必然性）
- 标注所属敌人层级（街头层/地方势力层/执法系统层/幕后金主层）
- 每个反派≥200字背景故事——反派也要有"为什么变成这样"的过去
- 每个反派有口头禅

**反派梯度设计**（与势力.md势力4层梯度对齐）：
- 街头层反派：第一卷前半段的主要对手，实力和主角接近或略强
- 地方势力层反派：第一卷后半段的对手，实力明显高于主角，主角需要越级对抗
- 执法系统层反派：暗示存在，第二卷碰撞
- 幕后金主层反派：全书终极对手，第一卷只在暗线中出现

# 任务

按起点character v1.0.0 SOP，基于骨架+主角设计+world产出，建角色库。

## 输入

### 骨架.md（动力引擎组成3——众生攀登方式分层）
# 骨架

## 力量体系（坐标系）

### 广义化声明
本项目的坐标系采用 **D&D数据面板体系**：所有众生天生被赋予一块“灵魂石板”（数据面板），其上记录着职业等级（主养成线）、六围属性、技能等级与专长（子养成线）。世界的强弱刻度由这些面板数据严格界定——**没有隐藏实力，一切皆可量化**。这是一种**绝对透明的阶层金字塔**：等级决定权限，属性决定能力上限，技能和专长决定战斗风格。数据是唯一的真理，杀戮是唯一的数据来源。

### 第一层：世界源力量

| 字段 | 内容 |
|------|------|
| **源力量名称** | **‘深渊法则’·灵魂石板** |
| **源力量来源** | 自“诸神黄昏”后，旧神陨落，世界法则崩坏，但残留的“原始规则”并未消失，而是以数据化的形式烙印在所有生灵的灵魂中。这块灵魂石板记录了生灵的一切数据，并遵循“**杀戮→经验→进化**”的底层法则。世界变成了一个巨大的角斗场，所有生灵都被迫遵守这套规则。暗黑基调：这不是恩赐，而是诅咒——不杀戮就无法变强，不努力变强就会被他人杀戮。 |
| **源力量→主养成线转化链** | 灵魂石板（源） → 通过**杀戮/战斗**获取**灵魂碎片（经验值）** → 碎片积累满时触发**等级晋升** → 主养成线（职业等级）提升。 |
| **源力量→敌分级标准** | 敌人的强度直接由**等级+属性面板**判定。同一等级内，属性与专长的差异决定了实际战力。高阶敌人常拥有**暗影器官**（特殊天赋），击杀后能掉落更强的灵魂碎片。 |

### 第二层：主养成线（位阶）

职业等级共**20级**，分为五个“深渊阶位”。每个阶位名源于“深渊主宰”体系中的古老称谓，带有黑暗史诗气质。

## 2b 金手指设计

### 金手指类型+名称
- **类型**：系统（D&D数据面板的增强变体——数据之眼）。
- **名称**：**“深渊数据之眼”**。

### 加速机制
- **加速点**：骨架.md组成2（运转机制）中——**输入**（战斗/杀戮环节）和**转化**（灵魂碎片转化率）的双重加速。
- **加速方式**：
  1. **战斗输入阶段**：数据之眼能瞬间看到敌人的具体属性、技能冷却、弱点暴露窗口（比如“3秒后敌人左肩会暴露真身”）。主角可以打最高效的收割战，而不是盲目乱砍。
  2. **转化阶段**：数据之眼能显示当前击杀目标掉落的“灵魂碎片品质”和“掉落概率”，让主角优先选择高收益目标（而不是随机乱杀浪费精力）。
- **与众生攀登方式关系**：主角的攀登方式（没资源者的单枪匹马野区猎杀）完全不变。金手指只是提供了“野区地图上的精确采矿路线”和“BOSS的弱点提示”，让他在每次战斗中都打出高于面板的收益。

### 限制（红线级）
- **使用限制**：
  1. 每次激活数据之眼消耗精神力——连续使用3场高烈度战斗会出现“眼压过载”（双眼剧痛、视野模糊）。
  2. 对于等级高于主角5级以上的敌人，数据之眼只能看到模糊信息（如“力量？？？”），无法直接看到弱点窗口。
- **成长限制**：
  1. 数据之眼本身不升级——它始终基于主角当前的灵魂石板解析能力。主角等级越高，能获取的信息越精确（但上限是“能看到同阶及低一阶敌人的完整面板”）。
  2. 无法通过金手指跳级——主角必须自己从1级打到20级。
- **坐标系意义保留声明**：**金手指不取消坐标系本身的意义——主角仍然必须通过杀戮获取经验升级，无法跳阶，无法无视等级差；数据之眼只是让他打得更准、效率更高，但没法让他“不杀怪就升级”。**

### 代价
- **单次代价**：每次激活数据之眼（≥5秒）消耗5%~10%精神力。如果精神力见底，主角会呕吐、视线重影、思维迟缓。
- **累积代价**：过度使用→精神力上限永久下降（主角的“灵魂石板”会出现裂纹，影响技能学习速度）。为了恢复上限，需要消耗珍贵的“灵魂净水”，而这种东西只有领主级以上势力才能稳定产出。
- **代价来源对齐**：与骨架.md组成4（代价结构）的通用代价“灵魂磨损”一致——金手指的代价只是放大和加速了这个磨损过程。

### 与引擎关系
- **金手指在引擎中的定位**：组成2（运转机制）的加速器。
- **引擎闭环是否改变**：不改变——金手指只加速主角在“输入”和“转化”环节的效率；底层闭环“猎杀→汲取灵魂→强化面板→猎杀更强者”没有变。
- **主角驱动逻辑是否改变**：不改变——主角的核心欲望（复仇）仍由骨架.md组成1（世界级驱动力：生存压力+零和资源争夺）驱动，数据之眼只是让他爬得快，不是给他创造新欲望。

### 梗×机制×限制公式
| 维度 | 内容 |
|:--|:--|
| **梗** | 一个在“数据化看得见一切”的深渊世界里，主角的挂是“能比别人多看一点”——看见技能冷却倒计时、看见弱点暴露窗口、看见敌人下一秒的属性压制会消失。他是战场上的“破绽猎手”。 |
| **机制** | 每场战斗，主角可主动激活数据之眼（消耗精神力），在短暂时间内看到敌人面板数据、技能冷却和弱点暴露窗口。弱点窗口只有1-2秒，必须精准出击。 |
