# pop-qidian-write-dndlike

> D&D数据面板流写作引擎（深渊主宰版本）。将剧情锚点渲染为番茄小说平台可消费的章节正文。
> **本文件是完整独立的写作skill——写正文时只需加载本文件，无需加载 `pop-qidian-write/SKILL.md`。**
> `pop-qidian-write/` 作为兜底模板，仅在无流派专属skill时使用。
> **版本：v1.1.0** — v1.1.0：对齐三层骨架重构——加载主角设计.md+角色库.md，爽感层锚定坐标系跃迁

## 正文落盘规则

正文完成后，落盘到 `涌现/正文/{书名}-第{N}章-{标题}.txt`，对话中只回摘要+钩子+创作记录。trial 模式不落盘，仅对话内。

## 做什么

输入：项目上下文（soul/seed/current-state/mechanics/人物库）+ 笔触DNA + 本章锚点
输出：2000-2500字的章节正文，不准超过3000字，附执行报告

文件架构：
- **SKILL.md（本文件）**：完整SOP（8步） + 6章型骨架 + 红线 + 速查表
- **dna/深渊主宰-笔触DNA.md**：笔触风格（叙事距离/句式段落/物象/对话/情绪外化/6章型节拍绑定/章节过渡/感官序列）
- **流派技法/**：D&D专属技法（战斗模式×3等级段/面板叙事/数据化分析/场景卡/特殊条件战/多线叙事），按需加载
- **../pop-qidian-write/references/**：通用参考层（爽点/节奏/格局/位阶 + 通用技法6 + 情境技法6 + 流派专属5），按需加载

## 流派定义

D&D数据面板流的核心特征：
1. **数据面板意识流**：角色脑内浮现数据化UI，伤害/成长/发现用数据格式呈现
2. **属性驱动战斗**：战斗结果由属性数值决定，面板弹出验证结果
3. **等级制度爽感**：升级是核心爽点，面板弹出等级提升=多巴胺注射
4. **怪物鉴定**：遭遇新敌人时弹出数据块，读者和主角同时获取信息
5. **参照物锚定**：属性数值必须有具象锚点（20敏捷=超越绝大多数人类）

## 怎么操作（SOP）

### Step 1：加载项目上下文

读取项目的核心文件：
- `seed-种子文档.md` — 故事种子（主角/卖点/不可牺牲项/世界观/金手指/三阶段走向）
- `幕纲-幕NNN-{标题}.md` — 本幕章节流概览 + prose-render交接 + 主角收益 + 主动行为表
- `current-state.md` — **下一章硬推进 + 人物状态 + 燃料队列 + 伏笔债务**（current-state 是涌现式管道给 write 的精确交接物，Step 2的章意图思考基于 current-state 的硬推进深化）
- `设计/主角设计.md` — 爽感矛盾公式：坐标系门槛×天赋加速×代价约束（爽感层锚定坐标系跃迁的依据）
- `设计/角色库/角色库.md` — 角色唯一源（角色名/性格/关系以此为准，禁止自行发明角色）

### Step 2：章意图思考 + 面板弹出判断

**读完施工卡后、选择章型之前，先想清楚这章到底要给读者什么体验。**

加载 `爽文章节类型池.md`（来自plot-design/references/），然后回答以下问题：

```
## 章意图思考
- 本章定位：（从10类章节类型中选1个主类型，可选1个副类型）
- 体验设计：为了让读者获得____体验，我需要重点写好____场景
- 信息植入：本章需要传递____信息，用____方式植入（回忆/对话/事件/场景，禁止旁白）
- 角色连接：本章需要加深读者对____角色的感情，通过____手段
- 爽感设计：本章的爽感点在____，压力在前____，释放在后____
- 主动行为核验：本章主角的主动行为类型是____（战斗/宏观解决问题/主动出击/收获/获得反馈/质变级反馈和成长）。如果选了"战斗"，战斗段落是否有沉浸感？如果选了"获得反馈"或"质变级反馈和成长"，面板是否弹出？
- 本章必须避免的：（不要走偏到什么方向，1~2条）
```

**面板弹出判断**（D&D数据面板流专属）：

| 施工卡事件 | 是否弹面板 | 弹什么面板 |
|-----------|----------|-----------|
| 击杀敌人 | 必弹 | 伤害结算面板（格式查当前等级段） |
| 升级 | 必弹 | 职业提升面板（格式查当前等级段） |
| 遭遇新怪物 | 必弹 | 怪物鉴定面板 |
| 技能检定 | 关键检定弹 | 技能名+成功/失败+数值 |
| 翻尸体/战利品 | 弹 | 物品名+属性+价值 |
| confrontation_pressure章 | 可弹 | 内心属性评估面板 |
| reveal_hook章 | 可弹 | 世界事件面板/传说度面板/状态总览 |
| 无战斗章 | 不弹战斗面板 | — |

**红线**：
1. 意图思考必须在写作之前完成，不可跳过
2. 意图思考的结果指导后续章型选择和技法选择——如果选了"情感锤击型"，则Step 5必须包含"情感锚点设计"技法
3. 如果施工卡中已有"本章意图声明"（来自plot-design），则核验并深化，不重复填写
4. **禁止走偏到传统文学思路**——意图思考服务于爽文读者体验，不是作者表达欲

**为什么需要这一步**：
AI按固定SOP填空容易产出"节拍都在但没有灵魂"的章节。意图思考环节强制agent停下来想："这一章读者读完应该获得什么体验？应该爱上谁？应该对什么产生期待？"然后再按章型+技法执行。

### Step 3：选择章型

根据 Step 2 的意图思考 + `current-state.md` 的本章硬锚点，选择最合适的章型。章型的完整定义（使用条件/密度约束/7节拍骨架/技法推荐）在 `../pop-qidian-write/references/章型定义.md`，必读。

6种章型速查：

| 章型 | 使用条件 | 读者期待 |
|:----|:--------|:--------|
| opening_shift | 第一章/世界规则松动/面板激活 | 先信现实→被打开更大世界 |
| exploration_gain | 进入新地点/发现资源线索 | 跟着主角探索未知 |
| confrontation_pressure | 强弱位阶对话/势力试探/布局算计 | 看打脸反转 |
| combat_reversal | 被压制/追杀/需打穿误判/首杀/暗杀 | 看绝境翻盘 |
| high_tier_display | 远超主角层级的战斗/人物出现 | 看格局碾压+立志 |
| reveal_hook | 规则揭露/旧伏笔变义/新敌人/升级揭秘 | 看真相炸裂 |

### Step 4：加载DNA + references

**4a. 笔触DNA**：从 `dna/深渊主宰-笔触DNA.md` 加载。提供：
- 笔触风格（叙事距离/句式段落呼吸/物象写法/对话写法/情绪外化）
- 6种章型的节拍笔触绑定
- 章节过渡3种模式 + 感官序列

**4b. 流派技法按需加载**：根据主角当前等级（从 `current-state.md` 读取）+ 施工卡标注，加载对应技法文件：

| 条件 | 加载文件 |
|------------|---------|
| 当前等级1-10级 | `流派技法/战斗模式-凡人期.md` |
| 当前等级11-25级 | `流派技法/战斗模式-超凡期.md` |
| 当前等级25+级 | `流派技法/战斗模式-神明级.md` |
| 全等级段通用 | `流派技法/面板叙事.md` + `流派技法/数据化分析.md` |
| 施工卡标注特殊条件 | `流派技法/特殊条件战.md`（盲战/追逃/围城/水下/多武器/陷阱） |
| 施工卡标注多线叙事 | `流派技法/多线叙事.md`（多地点快切/世界级事件/敌方POV/旁观者/支线/神明感知） |

**4c. 场景卡全量加载**：`流派技法/场景卡.md` 全量加载（20+种场景公式，每条1行，不构成上下文负担）

**4d. references按需加载**（位于 `../pop-qidian-write/references/`）：

| reference | 何时加载 |
|:---------|:--------|
| `爽点引擎.md` | 每章必加载（爽感是番茄核心） |

| `通用技法/`（6个文件） | Step 5选定后加载对应文件 |
| `情境技法/`（6个文件） | Step 5选定后加载对应文件 |
| `流派专属/`（5个文件） | Step 5选定后加载对应文件 |

### Step 6：写作

按章型的7节拍推进剧情。每个节拍执行五层指导：

| 层 | 来源 | 决定 |
|:--|:-----|:----|
| 笔触层 | 笔触DNA | 怎么写（叙事距离/句式/对话气口/战斗模式分层/参照物梯子/POV规则） |
| 节奏层 | `../pop-qidian-write/references/节奏约束.md` | 多快写（事件密度/感官上限/爽感触发/微钩子） |
| 格局层 | `../pop-qidian-write/references/格局表达.md` | 写多大（空间跃迁/时间纵深/侧面定级/规则对抗/史诗口吻） |
| 爽感层 | `../pop-qidian-write/references/爽点设计.md` | 怎么爽（打脸循环/小爽链/爆发后三段式/即时奖励） |
| 微观层 | Step 5选定的技法文件 | 单章怎么精彩（17类技法中选定的2-3类，参考文件中的子技法+原文案例） |

**爽感层锚定约束（坐标系跃迁）**：
- 爽感锚定"跨越坐标系的瞬间"——爆发时刻必须体现天赋加速 vs 代价约束的张力（依据 `设计/主角设计.md` 的爽感矛盾公式：坐标系门槛×天赋加速×代价约束）
- 每章爽感爆发必须锚定坐标系位置跃迁（从A坐标系跨入B坐标系，或同一坐标系内的位阶跃迁），不能是脱离坐标系的纯情感爽感
- 爆发点 = 坐标系门槛被天赋加速突破的瞬间；代价约束必须在爆发后可见（身体/资源/关系层面的不可逆消耗）

**五层冲突时的优先级**：爽感层 > 节奏层 > 微观层 > 格局层 > 笔触层。因为番茄读者不会给你时间。

**微观层执行要求**：选定的2-3类技法必须在正文中可见落地。写完后在执行报告中标注每类技法落在了哪个段落。

### Step 7：验收

写完后逐项检查验收表（见速查表）。在正文末尾追加执行报告（含微观技法选择卡+技法落地标注）。

### Step 8：移交review

正文渲染完成后，移交review skill进行编辑审核。review会检查正文与施工卡的gap、判断谁更好、输出摘要和状态快照。

- 如果review决策为"正文采纳"：进入下一章渲染（使用review产出的状态快照作为前文衔接）
- 如果review决策为"正文需修改"：按review的修改项清单重写，然后再次移交review
- 如果review决策为"人工裁决"：暂停等待人工判断

## 6种章型骨架

每种章型7个节拍。节拍功能定义是通用的，笔触绑定在笔触DNA中。

### opening_shift（开篇/世界松动章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 现实常识/日常/机构秩序 |
| 2 第一信息变化 | 不合常理的数据/痕迹/目击 |
| 3 压力/诱因 | 权威或群体无法解释 |
| 4 主角动作或判断 | 接近异常 |
| 5 阻碍升级 | 异常是更大物象的一角（爽感爆发点） |
| 6 可见反馈 | 群体失语/机构失效/常识崩裂 |
| 7 章末钩子 | 异常继续逼近或呼叫下一层真相 |

### exploration_gain（探索/资源露头章）

| 节拍 | 功能 |
|:----|:----|
| 1 起笔 | 环境反常或生物异常 |
| 2 资源/线索露头 | 发现目标物象 |
| 3 危险阻挡 | 或别人不敢拿 |
| 4 判断收益，冒险 | 选择深入 |
| 5 危险扩大或深处异动 | 爽感爆发点 |
| 6 资源到手/能力变化 | 获得回报 |
| 7 更深处还有更大变化 | 章末钩子 |

### confrontation_pressure（对峙/压力章）

| 节拍 | 功能 |
|:----|:----|
| 1 敌方/高位者给压力 | 出场即轻视 |
| 2 对方暴露目的/误判 | 筹码出现 |
| 3 语言羞辱/身份压制 | 被欺蓄力 |
| 4 沉默/反问/亮筹码 | 暗蓄 |
| 5 对方加码/旁人围观 | 爽感爆发点（被欺→反击） |
| 6 对方失态/局势微反转 | 打脸反馈 |
| 7 矛盾升级到下一场行动 | 章末钩子 |

### combat_reversal（战斗/反击章）

| 节拍 | 功能 |
|:----|:----|
| 1 明确对方优势/主角劣势 | 出场即轻视 |
| 2 主角发现缝隙/变量 | 物象出现 |

## 全局笔触DNA

### 数据面板（核心笔触·游戏看板）

数据面板是第一辨识度特征。面板本质是"游戏看板"——主角脑内浮现的数据化UI，把世界中发生的事情（伤害/成长/发现）用数据化格式呈现。

系统消息不以弹窗/光屏形式出现，而是作为"意识流中的数据流"直接嵌入叙述。格式固定：用冒号分隔数值，竖列排列。

**面板弹出时机和格式**：由 `流派技法/面板叙事.md` 规定（各等级段递进）。

**原文锚点**：

> 姓名：索伦。种族：半精灵。属性：力量12、敏捷19（+1）、体质15、智力18（+1）、感知15、魅力16。阵营：守序邪恶。职业：5级平民/1级盗贼【一阶】。生命值：12/12。经验值：150（未分配）。

> "阴影躲藏成功！""背刺发动！""成功发动背刺*2，对目标造成12点伤害！""目标死亡！""抽取目标灵魂能量，获得杀戮经验30点。"

> "巫师职业提升到5级。""获得25点技能点，生命值提高9点。""领悟职业技能【法术反制】。"

### 章节过渡

不是每章开头都用感官序列。感官序列只在**新场景首次出场**时使用。章节之间的过渡有3种模式：

**模式A：时间跳跃+日常复归**——大战后/事件后的清晨或次日，用日常行为暗示时间流逝。

> 第二天。当薇薇安睁开眼睛，有些迷糊地揉了揉眼角，准备爬起来做早餐时。眼前已经盛好了一碗香气扑鼻的燕麦粥。

**模式B：战场收尾+世界构建**——战斗结束后不立刻切场景，而是展开战场清理/政治后果/社会反应。

> 食人魔已经彻底溃散了。骑兵在尽可能的追击，很多疲倦的战士都一屁股坐在了地上，到处都是伤者的哀号声，牧师在用为数不多的法术救治……

**模式C：若无其事+暗杀后日常**——杀完人后第二天该干什么干什么，强调主角的冷。

> 天亮之后。索伦若无其事地朝着商队驻地走去，昨晚上干的一票他没有留下来任何痕迹……

**使用指引**：
- 上一章以战斗/紧张结尾 → 下一章用模式A或C（日常复归）
- 上一章以日常/信息结尾 → 下一章用感官序列（新场景引入）
- 连续两章不得使用相同的过渡模式

### 感官序列

**新场景首次出场**时使用：嗅觉→听觉→触觉→视觉。不是每章开头都用——如果章节过渡用的是模式A/B/C，则不需要感官序列开场。

**原文锚点**：

> 空气中永远存在着一股恶臭味。（嗅觉）密密麻麻的低矮屋棚里面，可以看到许多表情麻木的人正在忙碌。（视觉——从近到远）

> 一股劲风伴着恶臭味，食人魔的味道比豺狼人还要难闻。（嗅觉）呼啸声从头顶袭过。（听觉）索伦不敢有丝毫的停留，翻滚躲避。（触觉/动感）

### 叙事距离

叙事者始终固定在**主角肩膀后侧**，不超过一臂。但允许短暂切换到其他角色视角（配角/敌人/旁观者），切换需有明确标志且快速切回。

**POV切换规则**：由 `流派技法/多线叙事.md` 规定（各社会阶段不同）。

允许的镜头切换：
- 配角视角开场（章节前1/3用配角视角建立情感锚点，然后切回主角）
- 敌人视角插入（战斗/暗杀时切到敌人视角写"来不及反应/不可置信"，3-5行后切回）
- 旁观者视角插入（主角行为后切到旁观者视角侧面定级，2-3行后切回）

### 句式呼吸

独句段用于三种场景：情感爆发、动作转折、时间跳跃。战斗正文采用短句链——3-5行段落密集排列，每段一个动作节奏。

### 对话力场

"道"是唯一使用的引导动词。动作前置（"索伦将煎蛋放进碗里，无奈道"）。战斗对话不用引导词——技能名以破折号引号标识。

### 情绪收束——递减式

杀完人后用一句冷评收束，然后零过渡翻篇。但**首次杀人例外**——第一次杀人时主角可以有短暂的心理波动（手指发抖/盯着自己的手看两秒/意识到"杀了个人"），这比面无表情更有冲击力。随着杀戮次数增加，心理波动递减，逐步过渡到"一句冷评收束"的常态。

递减节奏参考：
- 第1次杀人：短暂心理波动（3-5行），波动后立刻翻尸体
- 第2-3次杀人：一句冷评+一个微动作（擦匕首/看一眼尸体）
- 第3次以后：一句冷评收束，零过渡翻篇

### 主角定位

- 面对威胁时先算后动
- 面无表情（情绪出口是瞳孔/呼吸/手指）
- 动则致命不留活口
- 杀完人面无表情翻尸体找财物
- 面对妹妹时语气变轻动作变轻
- 情绪不通过面部

## 使用说明

本文件为场景公式速查表。每条场景卡1行公式，从原作采样中提取。写作时根据施工卡章意图+战斗类型+等级段，从下表选取对应场景公式，按公式序列组织叙事节奏。

选取规则：
- 每章选取1种场景公式作为主骨架
- 可在同章内混用2种（如战斗场景+翻尸场景），但主骨架只能有1种
- 连续3章不得使用同一种场景公式（见红线）

## 红线

1. **场景公式去重红线**：连续3章不得使用同一种场景公式。第4章必须切换。违反=不合格。
2. **等级段匹配红线**：超凡期+专属场景卡（6-9、19）不得用于凡人期。神明级专属场景卡（10-11、20-22）不得用于超凡期及以下。违反=不合格。
3. **主骨架唯一红线**：每章主骨架场景公式只能有1种。可在同章内混用1种辅助场景卡（如战斗+翻尸），但主骨架不得混用。违反=不合格。
4. **公式序列红线**：选定场景卡后，必须按公式序列组织叙事节奏，不得跳过关键环节。如"暗杀/偷袭"的"面板"环节不可省略。违反=不合格。

## 一、面板本质

面板是"游戏看板"——角色脑内浮现的数据化UI，把世界中发生的事情用数据化格式呈现。

核心原则：
- 面板不是弹窗、不是光屏、不是全息投影、不是任何物理可见的界面
- 面板是角色"意识流中的数据流"——系统消息直接嵌入叙述，作为主角内心感知的一部分
- 面板本身就是叙事，不是叙事的中断
- 读者通过面板获取数值信息，主角通过面板"感知"战斗结果和世界变化

面板的本质是"主角脑内的数据化感知"。当主角砍出一刀，他的意识中自动浮现伤害数值；当主角升级，他的意识中自动浮现属性变化。这种感知是即时的、本能的，不需要主动操作。

## 二、面板弹出时机表

| 场景 | 弹什么 | 频率 | 格式来源 |
|------|--------|------|---------|
| 主角首次觉醒 | 完整属性面板 | 全书1次 | 等级段面板文件 |
| 每次击杀 | 伤害结算面板 | 每次击杀必弹 | 等级段面板文件 |
| 升级 | 职业提升面板 | 每次升级必弹 | 等级段面板文件 |
| 遭遇新怪物 | 怪物鉴定面板 | 首次遭遇弹 | 数据化分析文件 |
| 技能检定 | 技能名+成功/失败+数值 | 关键检定弹 | — |
| 战利品 | 物品名+属性+价值 | 翻尸体时弹 | — |

**频率说明**：
- "必弹"=无条件弹出，漏弹即不合格
- "首次遭遇弹"=同类怪物仅首次弹出，变种/精英/首领视为新类型
- "关键检定弹"=非每次检定都弹，仅在结果影响剧情走向时弹出
- "可弹"=根据章意图和叙事节奏决定是否弹出，弹与不弹都合格

## 三、面板意识流写法

### 3.1 基本格式

面板用引号包裹的数据流形式嵌入叙述，用省略号"……"隔开叙事文字与数据流。

**示例**：

> 索伦的匕首没入了对方的后背，刀锋精准地切入了脊椎之间的缝隙。
>
> "背刺发动！"
> "成功发动背刺*2，对目标造成27点伤害！"
> "目标死亡！"
> "抽取目标灵魂能量，获得杀戮经验45点。"
>
> 噗通。
>
> 尸体倒在了地上。

### 3.2 省略号分隔规则

- 面板内部行与行之间可省略号连接，形成连续数据流："跳斩成功命中！……额外附加8（18－10）点力量伤害加成！……对目标造成23点穿刺伤害！……"
- 叙事文字与面板之间用空行分隔，不用省略号
- 面板弹出时不中断叙事节奏——面板本身是叙事的一部分，不是插入的"系统提示"

### 3.3 面板不是弹窗

| 错误写法 | 正确写法 |
|---------|---------|
| "眼前弹出一道光屏，上面显示着……" | 面板直接以数据流形式出现在叙述中 |
| "系统弹窗浮现：……" | "背刺发动！" 直接作为意识流出现 |
| "半透明的面板在视野中展开……" | 面板是主角脑内的感知，不需要视觉化展开 |

### 3.4 战斗中面板插入位置

| 插入时机 | 说明 |
|---------|------|
| 击杀瞬间 | 敌人死亡的刹那弹出伤害结算面板 |
| 战斗中决策 | 主角做出战术判断时可弹内心评估面板 |
| 战斗结束 | 可弹战后总结面板（训练/高强度战后） |
| 遭遇鉴定 | 首次遭遇新怪物时弹怪物鉴定面板 |
| 剧情节点 | 非战斗章节在关键剧情转折处可弹状态面板 |

## 四、各等级段面板复杂度递进

面板格式随主角等级递进，复杂度逐级提升。当前等级段的面板格式从对应等级段文件加载。

| 等级段 | 面板风格 | 核心特征 | 行数参考 |
|--------|---------|---------|---------|
| 1-5级 | 简洁 | 5行：技能链→伤害→死亡→经验 | 3-5行 |
| 6-10级 | 技能链 | 多技能连携结算，技能名破折号引出 | 4-6行 |
| 11-15级 | 多目标结算+环境 | 多目标分别结算，环境因素计入面板 | 6-10行 |
| 16-20级 | 规则结算+领域效果 | 法则/规则级结算，领域展开效果写入面板 | 8-15行 |
| 21-25级 | 概念级结算 | 概念对抗、法则改写，面板含抽象描述 | 10-20行 |
| 25+级 | 叙事性描述+开放性提示 | 面板趋于叙事化，含开放性提示而非固定数值 | 不限 |

**递进逻辑**：
- 低等级面板是"纯数据"——数字+技能名，读者一眼看懂
- 中等级面板加入"环境/多目标"——战场复杂性上升
- 高等级面板加入"规则/概念"——战斗从数值比拼升级为法则对抗
- 顶级面板趋于"叙事化"——数字退居其次，面板本身成为世界观叙事

## 五、内心属性评估面板（confrontation_pressure特有）

这是主角"在算"的可视化，不是真实面板弹出。用于confrontation_pressure章，展示主角面对强敌/困境时的内心评估过程。

**格式**：
```
目标评估：XX。等级：X级。威胁等级：XX。
直接交战胜率：X%。
行动方案：XX。
```

**示例**：

> 索伦的目光扫过对方的装备和站姿，脑中飞速运转。
>
> 目标评估：高阶牧师。等级：12级。威胁等级：高。
> 直接交战胜率：35%。
> 行动方案：利用地形分割战场，优先击杀护从，再以暗影系技能克制神圣护盾。

**写法要点**：
- 这是主角的"内心算账"，不是系统弹出的面板——不需要引号包裹
- "胜率X%"是主角的主观估算，不是系统精确计算，可以有误差
- "行动方案"必须与后续剧情一致——主角说了怎么打，后面就怎么打
- 仅在confrontation_pressure章使用，其他章节不用此格式

## 六、世界事件面板（reveal_hook特有，首次出现于幕末）

用于reveal_hook章，展示世界级事件对主角的影响。首次出现于幕末（第一幕结尾），此后在卷末/重大剧情节点可重复使用。

**格式**：
```
【世界事件】XX
地点：XX
影响：XX
```

**示例**：

> 远处的天际线被一道血红色的光芒撕裂，大地传来沉闷的震颤。索伦停下脚步，心中涌起一股不安。
>
> 【世界事件】恶魔之门开启
> 地点：迷雾山脉北部裂谷
> 影响：恶魔涌出，周边城镇面临毁灭威胁。冒险者公会发布S级讨伐令。

**写法要点**：
- 世界事件面板用方括号【】包裹事件名，与怪物鉴定面板的【种族类型】格式区分
- "地点"和"影响"各1行，信息精炼
- 世界事件面板弹出后，该事件必须在后续5章内对主角产生直接影响
- 幕末首次弹出世界事件面板时，标志着剧情从"个人成长"向"世界级冲突"升级

## 七、传说度面板（reveal_hook特有）

用于reveal_hook章，展示主角的传闻传播情况。当主角的行为（击杀强敌/完成壮举）开始在外界流传时弹出。

**格式**：
```
传说度变化：无→民间传闻。
"XX"——XX。
知晓范围：XX。
可信度：X成传言、X成猜测、X成事实。
```

**示例**：

> 酒馆里，几个冒险者压低了声音交谈着什么。索伦端着酒杯，隐约听到了自己的名字。
>
> 传说度变化：无→民间传闻。
> "暗影行者"——据说是一个能融入阴影的杀手，从不留下活口。
> 知晓范围：港口城市及周边村落。
> 可信度：3成传言、4成猜测、3成事实。

**写法要点**：
- 传说度变化用"旧→新"格式，体现传播进度
- 名号用引号包裹，后接破折号+传闻内容
- "可信度"三段式：传言/猜测/事实，比例之和为10成
- 传说度面板通常在主角"听闻自己传闻"的场景中弹出，与叙事场景绑定
- 传说度等级递进：无→民间传闻→冒险者圈子传闻→区域传闻→大陆传闻→传奇→神话

## 八、完整状态总览（幕末/卷末特有）

用于幕末或卷末，完整展示主角当前状态。相当于"存档点"，让读者在幕/卷结束时清晰了解主角的成长进度。

**格式**：
```
姓名：XX。种族：XX。
```

## 一、怪物鉴定面板

首次遭遇新怪物类型时弹出。面板以意识流数据流形式嵌入叙述。同类怪物后续不再重复弹（变种/精英/首领视为新类型）。

### 1.1 简版（凡人期）

适用于1-10级（凡人期），面板简洁，5行以内。

**格式**：
```
"[名称]【[种族类型]】（[阶位]）。"
"挑战等级X，[个人/生物]等级X，[生命类型]。"
"最高属性X点，最低属性X点，总属性值X－X点。"
"特殊专长：[专长1]、[专长2]……"
"挑战难度：[字母]+/-。"
```

**示例**：

> "半兽人战士【人型生物】（一阶）。"
> "挑战等级2，生物等级3，人型生命。"
> "最高属性力量17点，最低属性魅力6点，总属性值12－17点。"
> "特殊专长：狂暴、野蛮打击。"
> "挑战难度：D+。"

### 1.2 完整版（超凡期+）

适用于11级以上（超凡期及更高等级段），面板含专属资源条、技能、能力、专长、类法术能力等完整信息。

**格式**：
```
生命：XXX/XXX。
恶魔之力：0/XXX。（或类似专属资源条）
技能：潜行X、学识X、威吓X……
能力：火焰之躯、火焰之鞭、恶魔利爪（+1天生魔法武器）、恶魔皮肤（伤害减免+10）。
专长：多重攻击、武器掌握：长剑。
类法术能力：惊恐术（每天一次）、火球术（每天一次）、召唤恶魔。
```

**示例**：

> "炎魔将军【恶魔】（五阶）。"
> 生命：280/280。
> 恶魔之力：0/150。
> 技能：潜行12、学识18、威吓20。
> 能力：火焰之躯、火焰之鞭、恶魔利爪（+3天生魔法武器）、恶魔皮肤（伤害减免+15/善良）。
> 专长：多重攻击、武器掌握：长剑、精通重击。
> 类法术能力：惊恐术（每天一次）、火球术（每天一次）、召唤恶魔。

### 1.3 简版与完整版对比

| 维度 | 简版（凡人期） | 完整版（超凡期+） |
|------|--------------|-----------------|
| 行数 | 5行 | 7-10行 |
| 生命值 | 无（用生命类型代替） | 有（XXX/XXX格式） |
| 专属资源条 | 无 | 有（恶魔之力/神力/魔力等） |
| 技能 | 无 | 有（含数值） |
| 能力 | 用"特殊专长"概括 | 逐项列举（含具体数值） |
| 类法术能力 | 无 | 有（含使用次数） |
| 适用等级 | 1-10级 | 11级+ |

### 1.4 怪物鉴定写法要点

- 种族类型用方括号【】：【人型生物】【亡灵】【野兽】【异怪】【恶魔】【龙类】
- 阶位用圆括号（）：（一阶）（二阶）……（五阶）……凡人期敌人通常为一至三阶
- 挑战等级为阿拉伯数字，挑战难度为字母+/-：D- < D < D+ < C- < C < C+ < B- < B < B+ < A- < A < A+
- 总属性值格式"X－X"表示最低属性到最高属性的范围
- 完整版的专属资源条根据怪物类型变化：恶魔用"恶魔之力"，亡灵用"负能量"，龙类用"龙息蓄能"等
- 怪物鉴定面板弹出后，后续同类怪物不再重复弹（变种/精英/首领除外）

## 二、属性体感反推

不直接看敌人面板时，通过身体感受反推敌人属性。这是凡人期最常用的"属性感知"写法——主角通过战斗中的体感，将身体反应转化为数值判断。

### 2.1 力量反推

| 体感描写 | 反推属性 | 说明 |
|---------|---------|------|
| "手臂发麻" | 力量16点以上 | 格挡后手臂酸麻，对方力量明显高于常人 |
| "身体沉了一下" | 力量碾压 | 格挡时身体被巨力下压，力量差距极大 |
| "几乎握不住武器" | 力量18点以上 | 武器差点脱手，对方力量达到怪物级 |
| "被震退两步" | 力量20+ | 格挡后身体被迫后退，力量完全不在一个层级 |

**示例**：

> 铛！
>
> 铁剑重重地砸在匕首上，索伦的手臂猛地一沉，一股巨力顺着刀身传来。仅仅是这一击，他的手臂就开始发麻。
>
> 力量16点以上。索伦在心中迅速做出了判断。正面对抗毫无胜算，他必须闪避。

### 2.2 敏捷反推

| 体感描写 | 反推属性 | 说明 |
|---------|---------|------|
| "完全攻击不到他" | 敏捷差距过大 | 对方速度远超主角，攻击全部落空 |
| "只看到一道残影" | 敏捷18+ | 对方移动速度超出视觉追踪能力 |
| "他的动作快得不可思议" | 敏捷16-18 | 对方速度明显高于主角，但勉强能跟上 |

**示例**：

> 索伦的匕首刺出，但对方只是微微侧头，刀锋擦着耳朵飞过。第二轮攻击，对方依然只是侧身——索伦完全攻击不到他。
>
> 敏捷差距太大了。这个人至少有18点以上的敏捷。

### 2.3 体质反推

| 体感描写 | 反推属性 | 说明 |
|---------|---------|------|
| "刀砍上去像砍石头" | 体质16+ | 物理攻击几乎无效，对方皮甲/皮肤极硬 |
| "受了重伤依然行动如常" | 体质18+ | 重伤不减战斗力，体质惊人 |
| "一刀下去没有溅出多少血" | 亡灵/构装体 | 非生命类型，体质反推无效 |

### 2.4 体感反推写法要点

- 反推后必须紧跟战术决策（闪避/游击/撤退），不能只判断不行动
- 反推是主角的"战斗直觉"，不需要面板弹出——是主角的经验判断，可以有误差
- 体感反推用于"来不及弹怪物鉴定面板"的场景（遭遇战中来不及鉴定）
- 同一场战斗中，体感反推最多使用2次（力量/敏捷/体质各算1次），第3次起应弹正式面板或用旁白概括

## 三、伤害类型分离（超凡期+）

超凡期（11级+）开始，伤害不再是一个总数，而是分类型结算。不同伤害类型对应不同的免疫/减伤规则。

### 3.1 基本格式

伤害分类型弹出，每种伤害类型单独结算：

**示例**：

> "你对目标造成了12点的劈斩伤害！……你对目标造成了3点的暗影伤害！"

### 3.2 含免疫/减伤信息

当敌人对某种伤害类型有免疫或减伤时，面板必须反映：

**示例**：

> "半神巫妖可以免疫95%以上的物理伤害！……阴影袭杀可以附加18点的额外伤害，可是现在居然才打出来了3点的暗影伤害。"

**写法要点**：
- 免疫/减伤信息用感叹号+省略号引出，语气带惊讶感（"可是现在居然才打出来了"）
- 免疫百分比用"95%以上"等模糊表述，不用精确到小数
- 减伤后的实际伤害值必须低于理论值，差额即减伤效果
- 伤害类型分离是超凡期核心面板特征，凡人期不用（凡人期伤害是一个总数）

### 3.3 常见伤害类型

| 伤害类型大类 | 具体类型 | 典型免疫者 |
|------------|---------|-----------|
| 物理伤害 | 劈斩、穿刺、钝击 | 巫妖（免疫95%+）、构装体（减伤） |
| 能量伤害 | 火焰、寒冰、闪电、酸蚀、暗影、神圣 | 火元素（免疫火焰）、亡灵（免疫寒冰） |
| 特殊伤害 | 力场、毒素、心灵 | 构装体（免疫心灵）、亡灵（免疫毒素） |

### 3.4 伤害类型分离写法要点

- 物理伤害细分劈斩/穿刺/钝击，与武器类型对应（剑=劈斩，匕首=穿刺，锤=钝击）
- 能量伤害与技能属性对应（暗影系技能=暗影伤害，神圣系技能=神圣伤害）
- 每次攻击的面板中，伤害类型不超过3种（物理+技能附加+特殊效果）
- 免疫/减伤信息只在首次对该敌人造成该类型伤害时弹出，后续同类攻击用旁白概括

## 四、属性标尺锚定

### 4.1 锚定规则

## 五、挑战等级评估

用"冒险者队伍vs BOSS"的视角反推自身或敌人实力。挑战等级不是单纯的数值对比，而是"需要什么样的队伍才能打赢"的实战评估。

### 5.1 评估格式

**示例**：

> 如果按照前世冒险者们的评估，索伦现在的挑战等级应该是25左右。
> 换句话来说。只有凑齐了一个全部都是高阶传奇的冒险者队伍，才有可能不到50%的几率干掉他。

### 5.2 评估写法要点

- 挑战等级评估用"前世冒险者们的评估"作为信息来源——主角穿越者的视角，用游戏化经验评估世界
- 评估结果用"队伍配置+胜率"的格式表述："全部都是高阶传奇的冒险者队伍"+"不到50%的几率"
- 挑战等级评估用于关键节点：主角实力质变时、遭遇超规格强敌时、幕末/卷末实力盘点时
- 评估可以用于自身（"索伦现在的挑战等级"）也可以用于敌人（"这个BOSS的挑战等级"）

### 5.3 挑战等级与挑战难度的区别

| 维度 | 挑战等级 | 挑战难度 |
|------|---------|---------|
| 表述方式 | 阿拉伯数字（1、2、25……） | 字母+/-（D-、C+、A……） |
| 出现位置 | 怪物鉴定面板第2行 + 独立评估段 | 怪物鉴定面板末行 |
| 含义 | 综合实力数值化 | 对主角个体的难度评级 |
| 使用频率 | 凡人期面板内 + 超凡期独立评估段 | 仅凡人期面板内 |

## 七、红线

1. **怪物鉴定红线**：首次遭遇新怪物类型必弹怪物鉴定面板。同类怪物后续不重复弹（变种/精英/首领除外）。漏弹=不合格。
2. **属性锚定红线**：面板弹出属性数值时，必须在同章内用参照物锚定。同一参照物跨章间隔 ≥ 3章。纯数字无锚定=不合格。
3. **伤害类型分离红线**：伤害类型分离只在超凡期+使用。凡人期伤害为一个总数，不分类型。超凡期+不分离伤害类型=不合格。
4. **体感反推红线**：体感反推后必须紧跟战术决策，不能只判断不行动。同一场战斗中体感反推最多2次。
5. **面板格式匹配红线**：简版怪物鉴定面板仅用于凡人期，完整版用于超凡期+。格式错位=不合格。
6. **挑战等级评估红线**：挑战等级评估仅在关键节点使用（实力质变/遭遇超规格强敌/幕末卷末盘点），不滥用。

## 二、战斗模式总览

凡人期共5大战斗模式，覆盖15种子类型。每种模式有独立的节奏特征、面板格式和拟声词偏好。

| # | 战斗模式 | 子类型数 | 核心特征 | 是否弹面板 | 典型拟声词 |
|---|---------|---------|---------|-----------|-----------|
| 1 | 暗杀/偷袭 | 4 | 极短句推进→面板→翻尸 | 必弹（协作型除外） | 噗通/嗦 |
| 2 | 正面遭遇战 | 5 | 拟声词起头→格挡→属性判断→反击 | 部分弹 | 铛铛铛/铛 |
| 3 | 游击消耗战 | 1 | 翻滚躲避→蹭刀→拉开→反复 | 补刀时弹 | 无固定 |
| 4 | 终结技爆发 | 1 | 压力铺垫→技能名→动作链→视觉冲击 | 必弹 | 破折号引出 |
| 5 | 训练对战 | 1 | 试探→对抗→总结，非生死 | 弹成长面板 | 铛铛铛 |

## 三、战斗模式详解

### 模式1：暗杀/偷袭

#### 节奏特征
> 极短句（3-5字）快速推进 → 面板弹出 → 解释性长句补充数值逻辑 → 翻尸/善后

暗杀是凡人期最高频的战斗模式。核心爽点在于"信息不对称"——主角拥有阴影躲藏+背刺的技能链，敌人毫无防备。节奏必须极快，短句像刀锋一样切割叙事，面板在"目标死亡"的瞬间弹出，制造多巴胺注射效果。

---

#### 1.1 伏击秒杀型

**触发条件**：主角预设埋伏点，敌人路径可预判，属性差距允许一击必杀。

**叙事序列**：
1. 预设埋伏（环境描写，1-2句）
2. 阴影躲藏检定（1句过渡）
3. 背刺发动（极短句）
4. 面板弹出（5行标准格式）
5. 尸体倒地（拟声词"噗通"）
6. 翻尸/善后（1-2句收束）

**原文锚点**：

> 索伦已经潜伏在了阴影当中，身上的气息完全收敛，整个人仿佛化作了一块冰冷的岩石。随着脚步声逐渐接近，他的瞳孔微微收缩，手中的匕首无声地翻转。
>
> "阴影躲藏成功！"
> "背刺发动！"
> "成功发动背刺*2，对目标造成27点伤害！"
> "目标死亡！"
> "抽取目标灵魂能量，获得杀戮经验45点。"
>
> 噗通。
>
> 尸体倒在了地上，索伦迅速上前翻找了一下尸体。

**面板格式（5行标准暗杀击杀面板）**：
```
"阴影躲藏成功！"
"背刺发动！"
"成功发动背刺*2，对目标造成X点伤害！"
"目标死亡！"
"抽取目标灵魂能量，获得杀戮经验X点。"
```

**写法要点**：
- 面板前铺垫不超过3句，用环境感知+身体微动作（瞳孔收缩/匕首翻转/气息收敛）
- 面板后"噗通"独占一行，仅2字，制造停顿感
- 翻尸用"迅速上前翻找"一笔带过，不展开翻找过程（留给战利品面板）
- 伤害值控制在个位至低两位数（凡人期数值规模）

---

#### 1.2 未秒杀转型

**触发条件**：背刺伤害不足以秒杀（目标血量高/有减伤/背刺伤害偏低），转入正面交锋。

**叙事序列**：
1. 阴影躲藏+背刺（同伏击秒杀型）
2. 面板弹出——但第4行变为"目标严重受伤"而非"目标死亡"
3. 伤害值带括号附加，如"18（+1）"
4. 目标反应（惊醒/怒吼/反击）
5. 转入正面遭遇战叙事

**原文锚点**：

> 匕首刺入了对方的后背，但并没有刺中要害。对方的身体猛地一震，随即发出了一声怒吼。
>
> "阴影躲藏成功！"
> "背刺发动！"
> "成功发动背刺*2，对目标造成18（+1）点伤害！"
> "目标严重受伤！"
>
> 对方已经转过身来，眼中满是愤怒和杀意。

**面板格式（暗杀未秒杀面板）**：
```
"阴影躲藏成功！"
"背刺发动！"
"成功发动背刺*2，对目标造成X（+X）点伤害！"
"目标严重受伤！"
```

**写法要点**：
- 伤害值后的括号附加表示额外加成来源（专长/道具/环境），必须在本章内交代来源
- "目标严重受伤"是转型的信号弹，读者立刻知道要转入正面战
- 转型过渡用敌人的反应句衔接（怒吼/转身/拔武器），不超过2句
- 不弹第5行"杀戮经验"——因为没杀死

**伏击秒杀型 vs 未秒杀转型对比**：

| 维度 | 伏击秒杀型 | 未秒杀转型 |
|------|-----------|-----------|
| 面板第4行 | "目标死亡！" | "目标严重受伤！" |
| 面板第5行 | 有（杀戮经验） | 无 |
| 伤害值格式 | 纯数字"27" | 带括号"18（+1）" |
| 面板后拟声词 | 噗通（尸体倒地） | 无（敌人站着） |
| 面板后叙事 | 翻尸 | 敌人反应+转入正面战 |
| 总行数 | 5行 | 4行 |

---

#### 1.3 协作伏击型

**触发条件**：主角与队友配合伏击，近战+远程交替。

**叙事序列**：
1. 队友远程射击（拟声词"嗦"）
2. 主角近战背刺（技能名破折号引出）
3. 交替节奏（A行动→B行动→A行动）
4. 无面板，纯叙事推进

**原文锚点**：

> 一支箭矢破空而来——嗦！
>
> 箭矢准确地命中了目标的后背，对方身形一顿。就在这一瞬间，索伦从阴影中暴起，匕首划出一道弧线——背刺！
>
> 匕首没入了对方的腰侧，对方发出一声闷哼。索伦迅速后撤，另一支箭矢紧随其后。

**写法要点**：
- 无面板是本类型的标志——协作伏击的爽点在配合默契而非数值结算
- 拟声词"嗦"用于远程攻击，与近战的"噗嗤"区分角色
- 技能名用破折号引出："——背刺！"，不加引号，嵌入叙事流
- 交替节奏严格A→B→A，每次切换不超过2句
- 若协作伏击最终击杀，则击杀瞬间弹面板（回到伏击秒杀型格式）

---

#### 1.4 信息逼供型

**触发条件**：暗杀目标有情报价值，先逼供再补刀。

**叙事序列**：
1. 暗杀接近（阴影躲藏）
2. 制服但不击杀（控制动作）
3. 逼供对话（2-4轮）
4. 补刀击杀
5. 补刀面板（格式同伏击秒杀型）

**原文锚点**：

> 索伦的匕首抵在了对方的喉咙上，冰冷的刀锋让对方浑身一僵。
>
> "说。他们在哪里？"索伦的声音平静得没有一丝波澜。
>
> 对方颤抖着嘴唇，断断续续地说出了几个字。索伦微微点头，匕首轻轻一送——
>
> "背刺发动！"
> "成功发动背刺*2，对目标造成22点伤害！"
> "目标死亡！"
> "抽取目标灵魂能量，获得杀戮经验38点。"

**写法要点**：
- 逼供对话不超过4轮，凡人期信息密度不宜过高
- 补刀面板可省略"阴影躲藏成功"行（因为已经暴露，不再是偷袭状态）
- 补刀动作用"匕首轻轻一送"或类似短句，配合极短句节奏
- 逼供获得的信息必须在后续3章内回收使用

---

### 模式2：正面遭遇战

#### 节奏特征
> 拟声词起头 → 动作句（中长） → 属性判断/战术旁白 → 反击动作

正面遭遇战是凡人期的"硬碰硬"模式。核心张力在于属性差距——主角是盗贼系，力量/体质不如战士系敌人，必须用技能+战术弥补。每种子类型的差异在于武器搭配和战术逻辑。

---

#### 2.1 匕首格挡长剑

**触发条件**：主角持匕首，敌人持长剑，属性差距不大但武器劣势。

**叙事序列**：
1. 拟声词"铛铛铛"（三连格挡）
2. 格挡体感描写
3. 战术旁白（皮甲克制逻辑/武器优劣分析）
4. 侧步反击

**原文锚点**：

> 铛铛铛！
>
> 长剑连续劈下，索伦用匕首硬生生接了三击。手臂传来阵阵酸麻，但皮甲吸收了大部分冲击力。对方的力量不算太强，长剑的优势在于攻击距离，但在近身距离内反而施展不开。
>
> 索伦侧身一步，匕首沿着长剑的剑身滑下，直刺对方的手腕。

**写法要点**：
- "铛铛铛"三连格挡是标志性拟声词，必须独占一行或紧跟动作
- 皮甲克制逻辑：皮甲=轻甲=不影响敏捷+减伤，是盗贼系的装备优势
- 战术旁白不超过3句，必须包含"武器优劣"或"装备克制"的逻辑判断
- 反击用"侧步/侧身"起手，体现盗贼的敏捷优势

---

#### 2.2 匕首架住铁剑

**触发条件**：敌人力量远超主角，格挡时通过体感反推敌人属性。

**叙事序列**：
1. 格挡（拟声词"铛"）
2. 体感反推属性（"手臂发麻"="力量16点以上"）
3. 闪避决策
4. 闪避动作

**原文锚点**：

> 铛！
>
> 铁剑重重地砸在匕首上，索伦的手臂猛地一沉，一股巨力顺着刀身传来。仅仅是这一击，他的手臂就开始发麻。
>
> 力量16点以上。索伦在心中迅速做出了判断。正面对抗毫无胜算，他必须闪避。

**写法要点**：
- 体感反推是凡人期特有的"属性感知"写法——身体感受直接转化为数值判断
- 反推公式：手臂发麻=力量16+；几乎握不住武器=力量18+；被震退=力量20+
- 反推后必须紧跟战术决策（闪避/游击/撤退），不能只判断不行动
- 与2.1的区别：2.1是"能打但劣势"，2.2是"打不了必须闪"

**匕首格挡长剑 vs 匕首架住铁剑对比**：

| 维度 | 匕首格挡长剑（2.1） | 匕首架住铁剑（2.2） |
|------|-------------------|-------------------|
| 拟声词 | 铛铛铛（三连） | 铛（单次） |
| 力量判断 | "力量不算太强" | "力量16点以上" |
| 格挡后行动 | 侧步反击 | 闪避撤退 |
| 战术逻辑 | 武器优劣+皮甲克制 | 属性差距→放弃正面对抗 |
| 叙事重心 | 战术旁白 | 属性反推 |
| 适用场景 | 势均力敌或略劣 | 敌人力量碾压 |

---

#### 2.3 长剑游斗

**触发条件**：主角持长剑，面对多个弱敌或单个速敌，以速度优势绕杀。

**叙事序列**：
1. 拟声词起头
2. 极简概括动作
3. 属性差距说明（一句话）
4. 一笔带过结果

**原文锚点**：

> 铛铛铛——索伦的长剑连续挥出，每一剑都精准地切中要害。对方的反应速度比他慢了不止一截，敏捷属性的差距在这种级别的战斗中几乎是不可弥补的。
>
> 三息之后，战斗结束。

**写法要点**：
- 游斗型是凡人期最"快"的正面战写法——不展开过程，只给结果
- "一笔带过"是刻意的：用"三息之后""数息之间"等时间词跳过过程
- 属性差距说明必须在一句话内完成，不展开论证
- 适用于碾压局（主角属性远超敌人）或速杀局（敌人数量多但弱）

---

#### 2.4 弯刀对重武器

**触发条件**：主角持弯刀，敌人持重武器（战斧/重锤），利用速度+技能叠加伤害。

**叙事序列**：
1. 单声"铛"（格挡）
2. 格挡+能量穿透描写（阴影袭杀叠加暗影伤害）
3. 连续攻击动作链
4. 面板（若有技能伤害叠加）

**原文锚点**：

> 铛。
>
> 战斧砸在弯刀上，但索伦的阴影能量同时涌出，沿着刀身渗透进对方的武器。暗影伤害叠加在格挡的反震中，对方的手臂不由自主地一颤。
>
> 索伦没有给对方喘息的机会，弯刀连续斩出——第一刀横切，第二刀竖劈，第三刀直刺。

**写法要点**：
- 弯刀的核心特征是"穿透+斩落"，与匕首的"刺入+格挡崩裂"区分
- 能量穿透写法：物理格挡的同时，技能能量沿武器传导，造成额外伤害
- 连续攻击用"第一刀/第二刀/第三刀"编号推进，节奏递进
- 暗影伤害是凡人期的技能附加伤害来源，数值通常为个位数

---

#### 2.5 潜行vs潜行

**触发条件**：双方都有潜行能力，战斗从"发现"开始而非"接触"开始。

**叙事序列**：
1. 视觉触发（"光线扭曲"等潜行破绽描写）
2. 预判斩击
3. 扑杀
4. 无格挡音效，无面板

**原文锚点**：

> 索伦的瞳孔骤然收缩——前方的光线出现了一丝微不可察的扭曲。那是潜行者在移动时的破绽，普通人根本无法察觉，但对于同样掌握潜行的人来说，这等同于暴露。
>
> 他没有犹豫，匕首直接刺向那片扭曲的空气。
>
> 一声闷哼。鲜血飞溅。

**写法要点**：
- 无格挡音效（"铛"系列）——因为被偷袭方来不及格挡
- 无面板——潜行vs潜行的爽点在于"谁先发现谁"，不是数值结算
- 视觉触发描写是核心："光线扭曲""空气波动""阴影错位"等
- 结果用极短句收束："一声闷哼。鲜血飞溅。"——与暗杀型的极短句节奏呼应

**五种正面遭遇战子类型对比**：

| 子类型 | 武器组合 | 拟声词 | 面板 | 核心战术 | 节奏 |
|--------|---------|--------|------|---------|------|
| 2.1 匕首格挡长剑 | 匕首vs长剑 | 铛铛铛 | 可弹 | 皮甲克制+侧步反击 | 中速 |
| 2.2 匕首架住铁剑 | 匕首vs铁剑 | 铛 | 不弹 | 体感反推→闪避 | 中速 |
| 2.3 长剑游斗 | 长剑vs弱敌 | 铛铛铛 | 可弹 | 速度碾压→一笔带过 | 极快 |
| 2.4 弯刀对重武器 | 弯刀vs重武器 | 铛 | 可弹 | 能量穿透+连续攻击 | 中快 |
| 2.5 潜行vs潜行 | 匕首vs潜行者 | 无 | 不弹 | 视觉触发→预判扑杀 | 快 |

---

### 模式3：游击消耗战

#### 节奏特征
> 翻滚躲避 → 蹭刀 → 拉开距离 → 反复消耗 → 补刀

游击消耗战适用于属性/体型远超主角的怪物。主角无法正面对抗，必须通过反复的"蹭刀+拉开"逐步消耗敌人血量。主角会受伤——这是凡人期少有的"主角吃亏"战斗模式，用于制造紧张感和成长感。

**叙事序列**：
1. 怪物鉴定面板（首次遭遇必弹）
2. 正面试探→发现打不过（1-2句）
3. 游击循环（3-5轮）：
   - 翻滚躲避敌人攻击
   - 蹭一刀（小额伤害）
   - 拉开距离
   - 喘息/评估
4. 主角受伤描写（骨折/撕裂伤）
5. 补刀击杀
6. 击杀面板

**原文锚点**：

> 那怪物的手臂粗得像树干，一拳砸下来带着呼啸的风声。索伦堪堪翻滚避开，地面被砸出了一个坑。
>
> 他趁机在怪物的腿上划了一刀——鲜血流出，但怪物似乎毫不在意。
>
> 索伦迅速后退，拉开距离。这怪物的力量至少在18点以上，体质更是惊人，正面硬抗必死无疑。
>
> 第二轮。翻滚，蹭刀，拉开。
> 第三轮。翻滚，蹭刀，拉开。
>
> 第四轮时，怪物的爪子擦过了索伦的肩膀，撕裂了皮甲，留下一道深深的伤口。剧痛传来，但索伦咬紧牙关。
>
> 又过了三轮，怪物的动作终于开始迟缓。索伦抓住机会，一个翻滚到了怪物的背后——背刺！

**写法要点**：
- 游击循环用"第X轮"编号或"翻滚，蹭刀，拉开"三词重复推进
- 主角受伤是必须的——骨折/撕裂伤/淤伤，至少1处可见伤
- 受伤后不立即恢复，伤势持续到战斗结束
- 补刀可回到暗杀面板格式（如果最终是背刺补刀）
- 怪物血量不显示——读者通过"动作迟缓""流血增多"感知消耗进度
- 游击消耗战的章字数应比暗杀型多30%-50%，因为过程长

### 模式4：终结技爆发

#### 节奏特征
> 压力铺垫（资源耗尽/敌人包围） → 破折号引出技能名 → 身体内部描写 → 连续动作链 → 视觉冲击结果

终结技爆发是凡人期的"高潮战斗"模式。在持续压力达到临界点后，主角释放剑势技能，一击翻盘。节奏从"极慢"（压力铺垫）骤变为"极快"（动作链），制造爆发感。

**叙事序列**：
1. 压力铺垫（2-4句）：资源耗尽/敌人包围/血量告急/退路断绝
2. 破折号引出技能名："——剑势【重劈】！"
3. 身体内部描写（1-2句）：能量涌动/肌肉绷紧/血脉沸腾
4. 连续动作链（3-4个动作，编号或递进）
5. 视觉冲击结果（1-2句）：血肉飞溅/头颅滚落/身躯断裂
6. 技能击杀面板

**原文锚点 - 剑势【重劈】**：

> 索伦的体力已经快要见底，而面前的敌人依然步步紧逼。退无可退。
>
> 他深吸一口气，体内的力量涌向双臂——剑势【重劈】！
>
> 肌肉绷紧，力量在瞬间凝聚到了极限。索伦猛地跃起，长剑高举过顶，带着千钧之势劈下——斜斩！从左肩到右肋，剑刃切开了甲胄和血肉。紧接着横斩！头颅飞起。
>
> "跳斩成功命中！……"
> "额外附加8（18－10）点力量伤害加成！……对目标造成23点穿刺伤害！……"
> "目标死亡。"
> "成功击杀半兽人战士！……抽取目标灵魂能量！……"
> "获得45点杀戮经验。"

**原文锚点 - 剑势【斩首】成功**：

> ——剑势【斩首】！
>
> 长剑划过一道弧线，精准地切向对方的脖颈。剑刃入肉，骨头碎裂，头颅脱离了身躯，在空中旋转着飞出。
>
> 鲜血喷涌如泉。

**原文锚点 - 剑势【斩首】失败**：

> ——剑势【斩首】！
>
> 长剑斩向对方的脖颈，但角度偏了一分，剑刃卡在了颈骨上。
>
> "发动剑势【斩首】！……"
> "斩首失败！……你对目标造成重击效果！……目标受到18点的劈斩伤害！……目标进入流血状态！……"

**面板格式 - 技能击杀面板（跨阶击杀）**：
```
"跳斩成功命中！……"
"额外附加X（XX－10）点力量伤害加成！……对目标造成X点穿刺伤害！……"
"目标死亡。"
"成功击杀XX！……抽取目标灵魂能量！……"
"获得X点杀戮经验。"
```

**面板格式 - 技能失败面板**：
```
"发动剑势【斩首】！……"
"斩首失败！……你对目标造成重击效果！……目标受到X点的劈斩伤害！……目标进入流血状态！……"
```

**写法要点**：
- 压力铺垫必须让读者感到"主角要输了"——否则爆发没有释放感
- 技能名用破折号引出："——剑势【XX】！"，"剑势"二字+方括号技能名是凡人期终结技的命名格式
- 身体内部描写用"能量涌动/肌肉绷紧/血脉沸腾"等体感词，不超过2句
- 动作链用3-4个动作递进：蓄力→跳跃→斜斩→横斩，每个动作用短句
- 视觉冲击结果要"画面感"：血肉飞溅/头颅滚落/身躯断裂，给读者一个定格画面
- 斩首可成功可失败——失败时弹失败面板，敌人进入"流血状态"debuff，主角需要补刀
- 技能击杀面板的"（XX－10）"格式表示属性加成公式，XX为力量值，10为基准值

**剑势【重劈】vs 剑势【斩首】对比**：

| 维度 | 剑势【重劈】 | 剑势【斩首】 |
|------|-------------|-------------|
| 动作链 | 蓄力→跳跃斩击→斜斩→横斩头颅 | 弧线斩向脖颈 |
| 结果 | 必定成功（伤害足够时） | 可成功可失败 |
| 失败后果 | 无失败逻辑 | 卡颈骨→重击效果+流血状态 |
| 面板格式 | 技能击杀面板 | 成功=技能击杀面板；失败=技能失败面板 |
| 视觉冲击 | 斜斩+横斩=身躯断裂+头颅飞起 | 头颅脱离/剑刃卡颈骨 |
| 适用场景 | 群战终结/碾压局收尾 | 单挑决胜/赌命一击 |

### 模式5：训练对战

#### 节奏特征
> 试探 → 对抗 → 总结。非生死战斗，节奏舒缓，侧重成长描写。

训练对战是非生死战斗模式，用于展示主角技能成长和属性提升。不弹击杀面板，但可弹成长面板和技能阈值质变描写。

**叙事序列**：
1. 试探阶段："铛铛铛"格挡→被动防御
2. 对抗阶段：逐渐适应→找到节奏→反击
3. 总结阶段：战后总结面板（技能成长明细）

**原文锚点**：

> 铛铛铛。
>
> 索伦被动地格挡着对方的攻击，一开始还有些手忙脚乱，但渐渐地，他开始适应了对方的节奏。对方的攻击虽然凶猛，但套路并不复杂——三连击后必有一个短暂的停顿。
>
> 在第七次格挡后，索伦抓住了那个停顿，匕首猛地刺出，停在了对方的喉咙前。
>
> "战斗结束。"
> "经过一场艰难的战斗，你的技艺获得提升！……你成功对训练假人发动嘲讽！……你成功对训练假人发动威吓！……"
> "威吓+10，嘲讽+5，闪避（翻滚）+2，闪避（腾挪）+1，招架+1，野外生存+2。"
> "你掌握了战斗技能——剑势【重劈】。"

**面板格式 - 战后总结面板**：
```
"战斗结束。"
"经过一场艰难的战斗，你的技艺获得提升！……你成功对XX发动嘲讽！……你成功对XX发动威吓！……"
"威吓+10，嘲讽+5，闪避（翻滚）+2，闪避（腾挪）+1，招架+1，野外生存+2。"
"你掌握了战斗技能——剑势【重劈】。"
```

**技能阈值质变描写**：

当某项技能达到关键阈值时（如潜行100点），触发质变描写：

> 随着潜行技能的数值突破了100点，索伦感觉到了一种微妙的变化。他的身体仿佛与周围的阴影融为一体，不再是"躲藏"在阴影中，而是"成为"了阴影的一部分。
>
> "潜行技能突破100点阈值！"
> "领悟潜行进阶技能——朦胧术。"
> "朦胧术：在阴影环境中，你的存在感降低50%，被动触发，不消耗法力。"

**写法要点**：
- 训练对战的格挡音效与正面战相同（铛铛铛），但节奏更慢，有"试错"过程
- "逐渐适应"是关键过渡——从被动防御到找到节奏，体现成长
- 战后总结面板列出所有成长的技能，数值增幅为个位数
- 技能阈值质变（如潜行100→朦胧术）是训练对战的高光时刻，可单独成段
- 训练对战不弹击杀面板（因为没有击杀）
- 可包含"新技能习得"行："你掌握了战斗技能——剑势【XX】。"

## 五、怪物鉴定面板格式

首次遭遇新怪物类型时必弹。面板以意识流数据流形式嵌入叙述。

```
"[名称]【[种族类型]】（[阶位]）。"
"挑战等级X，[个人/生物]等级X，[生命类型]。"
"最高属性X点，最低属性X点，总属性值X－X点。"
"特殊专长：[专长1]、[专长2]……"
"挑战难度：[字母]+/-。"
```

**示例**：

> "半兽人战士【人型生物】（一阶）。"
> "挑战等级2，生物等级3，人型生命。"
> "最高属性力量17点，最低属性魅力6点，总属性值12－17点。"
> "特殊专长：狂暴、野蛮打击。"
> "挑战难度：D+。"

**写法要点**：
- 种族类型用方括号【】：【人型生物】【亡灵】【野兽】【异怪】
- 阶位用圆括号（）：（一阶）（二阶）（三阶）……凡人期敌人通常为一至三阶
- 挑战等级为阿拉伯数字，挑战难度为字母+/-：D- < D < D+ < C- < C < C+ < B- < B < B+ < A- < A < A+
- 总属性值格式"X－X"表示最低属性到最高属性的范围
- 特殊专长列举2-4个，用顿号分隔
- 怪物鉴定面板弹出后，后续同类怪物不再重复弹（除非出现变种/精英/首领）

## 六、拟声词体系

| 拟声词 | 用途 | 出现位置 | 独占行 |
|--------|------|---------|--------|
| 噗通 | 尸体倒地（暗杀） | 暗杀击杀面板后 | 是 |
| 扑通 | 尸体倒地（非暗杀） | 正面战/游击战击杀后 | 是 |
| 铛铛铛 | 连续格挡（三连） | 正面战格挡起头 | 是 |
| 铛 | 单次格挡 | 正面战单次格挡/弯刀格挡 | 是 |
| 铿锵 | 金属碰撞/武器出鞘 | 战斗准备/武器交锋 | 否 |
| 嗦 | 箭矢飞行 | 协作伏击远程攻击 | 是 |
| 噗嗤 | 利器刺入肉体 | 背刺/匕首刺入 | 否 |
| 砰 / 砰砰砰 | 撞击声（围攻） | 多敌人围攻/重武器砸地 | 否 |

**使用规则**：
- 拟声词独占行时，后面不加标点（"噗通"而非"噗通。"）
- 同一段战斗中，同一种拟声词最多出现2次，第3次起用动作描写替代
- "噗通"专用于暗杀后的尸体倒地，"扑通"用于非暗杀的尸体倒地，不可混用
- "铛铛铛"为三连格挡专用，单次格挡用"铛"，不可用"铛铛"（双连无此用法）

## 七、句式节奏特征

### 7.1 暗杀型节奏
```
极短句（3-5字）×2-3 → 面板（5行）→ 拟声词独占行 → 解释性长句（1-2句）
```
> 示例：气息收敛。匕首翻转。→ 面板 → 噗通 → 尸体倒在地上，索伦迅速上前翻找。

### 7.2 正面交锋型节奏
```
拟声词独占行 → 动作句（中长，15-25字）→ 属性判断/战术旁白（1-2句）→ 反击动作（短句）
```
> 示例：铛铛铛！→ 长剑连续劈下，索伦用匕首硬生生接了三击。→ 力量不算太强，长剑的优势在于攻击距离。→ 侧身一步，匕首沿剑身滑下。

### 7.3 技能爆发型节奏
```
压力铺垫（中长句×2-4）→ 破折号引出技能名（极短句）→ 身体内部描写（中句×1-2）→ 连续动作链（短句×3-4）→ 视觉冲击结果（中句×1-2）→ 面板
```
> 示例：体力见底，退无可退。→ ——剑势【重劈】！→ 肌肉绷紧，力量凝聚到极限。→ 跃起，高举，劈下，斜斩，横斩。→ 头颅飞起，鲜血喷涌。→ 面板

### 7.4 游斗型节奏
```
拟声词 → 极简概括（1句）→ 属性差距说明（1句）→ 一笔带过结果（1句）
```
> 示例：铛铛铛——长剑连续挥出，每一剑都切中要害。→ 敏捷差距不可弥补。→ 三息之后，战斗结束。

### 7.5 协作型节奏
```
A行动（2句）→ B行动（2句）→ A行动（2句）→ ……循环
```
> 示例：箭矢破空——嗦！→ 命中后背，对方身形一顿。→ 索伦暴起，匕首划出弧线——背刺！→ 匕首没入腰侧，对方闷哼。→ 索伦后撤，另一支箭矢紧随其后。

**五种节奏型对比表**：

| 节奏型 | 句长 | 面板位置 | 拟声词 | 核心节奏感 |
|--------|------|---------|--------|-----------|
| 暗杀型 | 极短→长 | 中间 | 噗通（面板后） | 快-停-快 |
| 正面交锋型 | 中长→短 | 可选 | 铛铛铛（起头） | 稳-稳-快 |
| 技能爆发型 | 长→极短→短→长 | 末尾 | 破折号引出 | 慢-爆-快-收 |
| 游斗型 | 中→短 | 可选 | 铛铛铛（起头） | 快-快-止 |
| 协作型 | 中→中→中 | 无 | 嗦（区分角色） | 交替-交替-交替 |

## 八、武器差异写法

| 武器 | 战斗侧重 | 标志拟声词 | 标志动作 | 适用模式 |
|------|---------|-----------|---------|---------|
| 匕首 | 刺入+格挡崩裂 | 噗通/铛铛铛 | 背刺/侧步刺 | 暗杀/正面格挡/游击 |
| 弯刀 | 穿透+斩落 | 嗦/铛 | 能量穿透/连续斩 | 正面交锋/终结技 |
| 长剑 | 游斗+架住 | 铛铛铛 | 架住/绕杀/一笔带过 | 游斗/正面格挡/终结技 |
| 弓弩 | 远程穿透 | 嗦 | 破空/命中 | 协作伏击/游击消耗 |

**武器切换写法**：
- 同一场战斗中主角可切换武器，切换时用1句过渡："索伦收起匕首，拔出了长剑。铿锵。"
- 武器切换标志着战斗模式切换（如暗杀→正面战 = 匕首→长剑）
- 武器出鞘声"铿锵"仅用于切换时，不用于战斗开始时的初始武器

## 十、红线

1. **等级段红线**：凡人期战斗模式只适用1-10级。11级+请加载 `流派技法/战斗模式-超凡期.md`。
2. **模式去重红线**：连续3章不得使用同一种战斗模式（含子类型）。第4章必须切换。
3. **击杀面板红线**：每次击杀必弹面板。无面板击杀=不合格。例外：协作伏击型（1.3）和潜行vs潜行（2.5）无面板。
4. **参照物锚定红线**：属性数值必须有参照物锚定（同章内）。同一参照物跨章间隔 ≥ 3章。
5. **面板意识流红线**：面板以意识流数据流形式嵌入叙述，不以弹窗/光屏/全息投影形式出现。
6. **数值规模红线**：凡人期伤害值控制在个位至两位数（最大不超过99）。杀戮经验不超过200。不得出现三位数以上的伤害值。
7. **拟声词独占行红线**：噗通/扑通/铛铛铛/铛/嗦独占行时不加标点。不可混用噗通（暗杀）与扑通（非暗杀）。
8. **怪物鉴定红线**：首次遭遇新怪物类型必弹怪物鉴定面板。同类怪物后续不重复弹（变种/精英/首领除外）。
9. **技能命名红线**：凡人期终结技命名格式为"剑势【XX】"，用破折号引出。不得使用超凡期的"奥义""领域""神技"等命名。
10. **受伤持续性红线**：游击消耗战中主角的伤势必须持续到战斗结束，不可中途自愈。战后需要描写伤势处理。

# 爽点引擎

> 不是配额表，是选型工具。5种引擎各自的原理+原文锚点+适用场景。你选引擎，不是凑数量。

---

## 引擎1：打脸循环

### 原理

番茄第一爽感。不是"主角赢了"，是"看不起主角的人输了"。读者代入的是被轻视的主角，爽的是翻盘后对方的表情。

### 结构

```
轻视（对方嚣张/误判/羞辱）
→ 压制（对方用实际行动碾压）
→ 暗蓄（主角不辩解，默默准备）
→ 翻盘（1-3行，"咔"一下到位）
→ 失态（对方崩溃/哑口/承认）
```

### 原文锚点

> "你这个死老太婆，杀不了你的本体，我还杀不了你的精神印记吗？！"
> "死老太婆你不是喜欢追杀吗？"叶凡冲了上去，喝道："今天我就追杀给你看！"

> "太让我吃惊了……这颠覆了常理！"姬惠依然处在惊憾中，金书一冲而过，将她割裂。

### 关键

- 轻视要厚——至少让对方嚣张2句+碾压1次，读者越愤怒翻盘越爽
- 翻盘要快——≤3行，越绝境配越短
- 翻盘要有智力感——不是蛮力也不是运气，是信息/判断/能力的组合
- 失态要可见——不是"脸色微变"，是具体的崩溃行为+台词

### 适用场景

对峙章/战斗章。对方是智能生物（能嚣张/能失态）。

---

## 引擎2：压力突破

### 原理

主角被逼到绝境，读者跟主角一起憋着一口气。突破的瞬间，这口气释放出来就是爽。

### 结构

```
绝境（枪无效/能力被封/体力耗尽）
→ 发现（缝隙/变量/关键信息，1句闪回）
→ 突破（1-2行，一招翻盘）
→ 回味（主角站起来/喘气/环视，停1拍）
```

### 原文锚点

> 叶凡抱鼎而上，跳脱出了九位大帝的合围之势，逆冲向九颗神秘的大星。
> 他一拳轰去，这一击的力量发自全身……一拳既出，星域失色。

### 关键

- 绝境要真——读者要相信主角真的可能输
- 突破的钥匙要在前文埋过（笔记里的方法/之前观察到的弱点），不能凭空
- 突破后停一拍——让读者"爽完"再重启压力

### 适用场景

战斗章/探索章。对方可以是非智能体（怪物/机关/环境）。

---

## 引擎3：未知揭示

### 原理

读者跟着主角累积疑问，到某个瞬间真相炸裂——原来如此！这个"原来如此"就是爽。

### 结构

```
疑问累积（3-5个微钩子制造悬念）
→ 线索收束（2-3条线索在某一刻串联）
→ 真相炸裂（1-2句揭示，颠覆之前的认知）
→ 后果展开（揭示改变了一切）
```

### 原文锚点

> 九具百米长的庞大龙尸，大半截躯体静静地横在山巅上，少半截躯体垂挂在山崖下，犹如黑色的钢铁长城，充满了震撼性的力感，极其冲击人的视觉。

读者从"这是什么"到"龙尸"到"九具"到"百米长"——每一句都在增加信息量，到"钢铁长城"时炸裂。

### 关键

- 疑问要真——不是故弄玄虚，是读者真的想知道
- 揭示要颠覆——不是"答案出来了"，是"原来是这样！"
- 揭示后改变行动——揭示不是终点，是新行动的起点

### 适用场景

开篇章/探索章/揭秘章。

---

## 引擎4：小爽沿途

### 原理

不是只有大爽才爽。沿途撒小爽让读者"一路爽到大爆"。小爽的本质是"主角牛逼"的微瞬间。

### 5种小爽

| 类型 | 本质 | 示例 |
|:----|:----|:----|
| 信息优势 | 主角知道别人不知道的 | "他看了便签三秒。他知道了。" |
| 判断正确 | 主角判断被验证 | "他选了左边。右边传来碎裂声。" |
| 技能展示 | 主角展示专业能力 | "一眼看出鞋印是警用勤务靴。" |
| 对手失误 | 对手犯错被主角抓住 | "它笑了。它不该笑。" |
| 主动选择 | 主角在压力下主动选择 | "半年没碰的枪。今天他没犹豫。" |

### 关键

- 小爽用1-3句，不展开
- 大爽前撒2-3个小爽做铺垫（让读者"预感"主角要赢）
- 大爽后不再撒小爽（保持纯净回味）
- 小爽之间用压力间隔，不连爽

### 适用场景

所有章型。小爽是调味料不是主菜。

---

## 引擎5：即时奖励

### 原理

番茄核心机制：行动→可见回报。主角每做一件有价值的事，必须有可见的即时回报。读者需要看到主角在变强。

### 原文锚点

> 叶凡大喝一声，轮海沸腾，五脏通明，四肢璀璨，脊椎骨如大龙冲天。

每次战斗后叶凡的轮海都在变化——这就是即时奖励。读者看到"他在变强"。

### 关键

- 每章至少1次行动→回报闭环
- 回报必须可见、具体——不是"他变强了"，是物理标志（掌心烙印/新能力/新情报）
- 回报有代价但不抵消爽感（失去工具但获得能力）

### 适用场景

所有章型。即时奖励是追读率的跨章驱动力。

# 信息差博弈

> 通用技法。围绕"谁知道什么"设计博弈，读者、主角、其他角色三方信息不对等产生快感。
> 来源：19本番茄头部小说前20章拆解

---

## 核心原理

信息差是单章内最可靠的快感来源。不是主角打脸了谁，是**读者比角色聪明**——"我看穿了，你不知道"。好的信息差有**风险感**——主角可能被识破。如果主角演戏毫无风险，信息差就变成了自嗨。

---

## 子技法与案例

### 1. 主角演戏，配角被骗，读者看穿全程

**操作**：主角假装某态度/行为，对方信以为真，读者全程看穿。

**案例**：乱世荒年Ch7
> 江尘突然加价到60两，全家懵了。江有林"你疯了？"。江尘拍柴刀"我就是要气死她！"。孙金梅吓得跑了。
> 然后江尘对父亲说："当然是装的。"
> 读者全程知道他在演戏，但孙金梅真跑了——爽点不是"骗到了"，是"读者比孙金梅聪明"。

**怎么用**：设计一个主角的"假动作"，让配角上当。关键是让读者在第3行就知道是假的，但配角到第N行才知道（或永远不知道）。

### 2. 配角各怀心思，读者看到全部算盘

**操作**：同一场景中多个角色各有算盘，读者通过视角切换看到所有人的内心。

**案例**：乱世荒年Ch9
> 江有林以为儿子来要田契→主动给出（父爱驱动）
> 陈巧翠怀疑骗田契→催丈夫去查看→听到只是学陷阱→松口气→又算计"那田不就归我们种了"（贪婪驱动）
> 读者看到两个人的内心，但两人互相不知道对方的真实想法。

**怎么用**：在同一个场景中，让2-3个角色各有目的。通过内心独白或小动作暗示每个人的算盘，但角色之间互相不知。

### 3. 延迟揭晓——先制造疑惑，后章揭晓

**操作**：主角做出不合常理的行为，读者疑惑"为什么"，到后文才揭晓原因。

**案例**：乱世荒年Ch6→Ch7
> Ch6结尾：江尘主动喊"60两！"——全家和读者都懵了。为什么要60两？
> Ch7开头揭晓：是演戏，为了让孙金梅知难而退。喊60两的瞬间制造了悬念，延迟揭晓让读者追到下一章。

**怎么用**：在章末让主角做一个反常行为，不解释原因。下一章开头揭晓。延迟不能超过1章——太久读者会忘记。

### 4. 多层套娃式——3层以上信息不对等叠加

**操作**：在一章内叠加多层信息不对等，每层揭开都产生新快感。

**案例**：十日终焉Ch1-3（5层叠加）
> 第1层：读者不知道山羊头是谁——恐惧来自未知
> 第2层：山羊头说"九位"但实际十人——有人隐藏了身份
> 第3层：齐夏发现一人不是参与者——读者跟主角同步推理
> 第4层：齐夏抽到说谎者牌——读者知道角色不知道（他会暴露吗？）
> 第5层：齐夏决定用假名"李明"——读者知道他在赌，其他角色不知道

**怎么用**：设计3-5层信息差。每章至少揭开1层，同时埋下新1层。层层叠加让读者持续参与推理。

### 5. 主角对读者隐瞒——主角知道但读者不知道

**操作**：主角拥有读者不知道的信息，制造悬念。读者会不断猜测"他到底知道什么"。

**案例**：诡舍Ch1
> 宁秋水看到胖子跳车后红绿灯上飘着人皮，所有人的反应都很大——"有人干呕""女人腿软差点没站稳"。
> 但宁秋水"反应远没有众人大"。
> 为什么？他知道什么？他经历过？读者不知道，但这个差异制造了强烈悬念。

**怎么用**：在关键时刻让主角的反应与常人不同——过于冷静或过于恐惧。不解释原因，让读者自己猜。

### 6. 身份套层——主角身份本身是多层的信息差

**操作**：主角有多重身份，每重身份对不同角色呈现不同面。信息差从"谁知道什么"升级为"谁是谁知道什么"。

**案例**：我不是戏神Ch1
> 陈伶 = 现代编导陈伶（穿越者意识）+ 原主陈伶（被父母杀死的少年）+ 灾厄载体（融合者）
> 三重身份的信息差：父母不知道"儿子"已经换了一个灵魂；执法者不知道陈伶是灾厄；陈伶自己也不确定自己是融合者还是灾厄
> "破碎的记忆彼此厮杀"——身份本身是冲突源

**怎么用**：给主角设计2-3重身份。不同角色认识主角的不同面。读者知道全部，但角色只知一面。

### 7. 骗子主角信息操控——主角习惯性说谎，读者判断真假

**操作**：主角是惯骗/情报操控者，对自己人也撒谎。读者需要在每句话中判断真假。

**案例**：诸神愚戏Ch1-20
> 程实是惯骗，他告诉队友"我知道【诞育】新生儿的原因"——"绝对保真"——但读者通过"谎言已经成了他的'习惯'和'约束'"知道他在编。
> Ch20揭示他偷了曹三岁的信仰和时间——读者此前不知道，被主角骗了。

**怎么用**：让主角有"说谎"的习惯。部分谎言读者能看穿（制造优越感），部分谎言读者被骗（制造惊喜）。关键是读者永远在猜"这句是真是假"。

### 8. 被动信息差——主角受限于条件无法告知

**操作**：主角知道真相但受限于形态/条件/规则无法告知其他人，信息差来自"如果他们知道会怎样"。

**案例**：玄鉴仙族Ch1-8
> 陆江仙是一面镜子，没有手脚不能说话。他知道这是修仙世界，但李家人不知道。他只能通过镜面发光指引方向。
> 信息差的爽点不是"我看穿了"，而是"如果李家人知道真相会怎样"——读者替主角着急。

**案例**：末日生存方案供应商Ch39
> 苏烬无法甄别两个人谁是感染者——信息不足驱动恐惧。不是"谁骗谁"，是"信息本身不完整"。

---

## 何时变套路

- 主角演戏毫无风险，变成无脑碾压
- 信息差永远只朝一个方向（主角永远比配角聪明）
- 延迟揭晓超过2章，读者忘记悬念
- 多层套娃变成"为了复杂而复杂"，读者跟不上

好的信息差有**双向性**——主角可能被骗（我不是戏神Ch1读者被程实骗了），配角可能识破（诸神愚戏韩蒙起疑）。

# 感官锚点

> 情境技法。用具体的生理感官细节建立代入感，不写"他很难过"，写身体反应。
> 来源：19本番茄头部小说前20章拆解

---

## 核心原理

读者不能"理解"饥饿，但能"尝到"甜润香。人的大脑在阅读时，感官描写会激活与实际体验相同的脑区——你写"冷"，读者皮肤会收缩；你写"甜"，读者口腔会分泌唾液。这就是感官锚点的底层机制：用具体的生理反应绕过理性认知，直接把读者"拽进"场景里。

一个精准的细节胜过面面俱到的环境描写。乱世荒年写饥饿，不需要描写餐桌、粮仓、米缸、田地——三个字"甜！润！香！"就够了，因为这三个字是饥饿者吃到蛋白质时的真实味觉爆发。面面俱到的环境描写是"上帝视角"，而一个精准的感官细节是"角色视角"，后者天然带有代入感。

感官锚点在老白文中是核心武器——老白读者已经厌倦了"他很愤怒""她很悲伤"这种标签式情绪，他们要的是"胃酸翻涌""两腿打颤"这种身体级别的真实。小白文中感官锚点是可选加分项，但一旦用对，能瞬间提升文本质感。

---

## 子技法与案例

### 1. 味觉锚点

**操作**：在角色最匮乏或最渴望的时刻，用极短的味觉描写制造冲击。不写"他饿极了"，写食物入口那一瞬间的味觉爆发。越饥饿，味觉描写越简短越有力。

**案例**：乱世荒年Ch8
> 甜！润！香！

乱世荒年的主角长期蛋白质匮乏，吃到肉的那一刻，作者没有写心理活动、没有写感激涕零，只用三个感叹号 + 三个单字味觉。这种极简的味觉爆发，恰恰模拟了饥饿者吃到高蛋白食物时的真实体验——大脑被味觉信号淹没，来不及组织语言。捞尸人则用了另一种味觉锚点："红薯粥，吃起来带着甜味，但不扛饱""胃里开始冒酸水儿"——甜味是希望，"不扛饱"和"冒酸水"是绝望，味觉的甜蜜和身体的饥饿形成反差，比直接写"他还饿"有力十倍。

**怎么用**：找到角色最匮乏的那个感官维度（饥饿的人写味觉、寒冷的人写触觉），在满足的瞬间用最短的句子爆发。味觉锚点的关键是"克制"——越饥饿越不能写长，三个字比三句话更扎心。

---

### 2. 触觉锚点

**操作**：用皮肤、肌肉、骨骼层面的触觉反应替代情绪标签。冷不是"感到寒冷"，是"两腿打颤"；粗糙不是"皮肤不好"，是"皴裂的皮肤下青筋毕露"。触觉描写的颗粒度要到组织级别。

**案例**：乱世荒年Ch15
> 寒风太烈，两腿打颤。

不写"天气很冷""他冻得不行"，直接写身体反应"两腿打颤"——这是肌肉不受意志控制的生理信号，读者读到这句话时，自己的腿部肌肉会产生微弱的共振感。捞尸人写身体质感则用"粗糙皴裂的皮肤下青筋毕露"——不写"他干的是体力活"，用皮肤纹理和血管凸起这两个触觉视觉化的细节，让读者"摸到"了那双手的粗粝。

**怎么用**：把"他感到X"替换成"他的身体做了Y"。情绪是主观的，触觉是客观的——读者无法验证"他很冷"，但能验证"两腿打颤"是不是冷的反应。选择那些不受意志控制的生理反应（颤抖、起鸡皮疙瘩、肌肉痉挛），因为它们最能证明感受的真实性。

---

### 3. 嗅觉锚点

**操作**：用气味建立场景的"空气质感"。嗅觉是最容易触发记忆和情绪的感官，一个气味能瞬间定义一个空间。写嗅觉要写"混合气味"——单一气味是实验室，混合气味才是真实世界。

**案例**：十日终焉
> 山羊身上独有的膻腥味，更有一股隐隐的腐烂气息。

不写"环境很恐怖"，用山羊的膻腥味打底（日常气味），再叠加"隐隐的腐烂气息"（异常气味）——日常与异常的嗅觉叠加，比单纯写"有腐烂的味道"更让人不安，因为读者会下意识分辨"哪个味道不对"。乱世荒年Ch12则用"鸡骚味熏得睁不开眼"——气味浓烈到生理性闭眼，把嗅觉直接转化成触觉/视觉反应。诡舍用"让人闻之欲呕的血腥味"——"欲呕"是嗅觉引发的生理反应，不是描写气味本身，而是描写身体对气味的拒绝。

**怎么用**：先写一个日常气味建立基底（烟火气、土腥味、汗味），再叠加一个异常气味制造不安。嗅觉锚点的关键是"身体反应"——不要写"味道很难闻"，写"熏得睁不开眼""闻之欲呕"，用身体的排斥反应来证明气味的强度。

---

### 4. 痛觉锚点

**操作**：用具体的疼痛类型和位置替代"他受伤了"。痛觉要写到组织级别——不是"很疼"，是"肌肉撕裂般的刺痛"；不是"不舒服"，是"胃酸翻涌"。疼痛的位置和类型本身就是叙事信息。

**案例**：乱世荒年Ch16
> 肩膀传来肌肉撕裂般的刺痛。

"肩膀"是位置，"肌肉撕裂"是疼痛类型，"刺痛"是疼痛性质——三个信息压缩在一句话里，读者不仅知道他受伤了，还知道伤在哪里、伤到了什么组织、疼的性质是什么。大奉打更人用"胃酸翻涌"——不写"他恶心"，写"胃酸翻涌"是器官级别的反应，读者自己的胃会跟着紧缩一下。痛觉锚点的妙处在于：疼痛是最难假装的情绪，角色在痛，读者就信他真的在经历。

**怎么用**：把"他感到疼痛/不适"拆解为三个要素：位置（哪里痛）+ 类型（什么组织在痛）+ 性质（刺痛/钝痛/灼烧/撕裂）。三个要素齐备，读者就能"定位"疼痛，产生体感共振。避免使用"剧痛""隐痛"这类模糊词，要用比喻性的组织描写（如"撕裂""灼烧""翻涌"）。

---

### 5. 量词锚点

**操作**：用具体的数字和换算关系建立价值感知。不写"很贵"，写"三十两——普通家庭一年花销不超二十两"。量词锚点的核心是"参照系"——给读者一个可比较的锚点，让抽象数字变成可感知的重量。

**案例**：乱世荒年Ch6
> 三十两——普通家庭一年花销不超二十两。

三十两是数字，"普通家庭一年花销不超二十两"是参照系——读者瞬间理解三十两意味着"超出普通家庭一年开销"，这个钱的分量就实体化了。Ch14又用"一只山鸡120文，抵十几斤粟米"——120文是数字，"十几斤粟米"是参照系，读者通过换算理解了山鸡在荒年的稀缺价值。量词锚点不只用于金钱，也适用于时间、距离、重量——任何需要让读者"感受"到数量的场景。

**怎么用**：每出现一个关键数字，立刻跟一个"参照系换算"。参照系必须是读者能理解的日常事物（一年开销、十几斤粮食、三天的路程）。不要假设读者知道"一两银子值多少"——主动换算，让数字落地。但注意：不是每个数字都需要换算，只换算那些对情节和角色命运有实质影响的数字。

---

### 6. 方言/语言质感

**操作**：用方言词汇、俚语、脏话建立角色的地域和社会阶层质感。语言质感不等于"加几句方言"，而是让角色的表达方式本身就是人物画像。一句话的说话方式能暴露角色的出身、教育、性格。

**案例**：捞尸人
> "细那康子""骚蹄子""尻蹶子"

捞尸人的角色语言带着浓烈的江湖气和地域感，这些词汇不是标准普通话能替代的——"尻蹶子"比"屁股"更有画面感，"骚蹄子"比"贱人"更有阶层感。十日终焉的花臂男说粤语"冚家铲""粉肠"——粤语脏话一出，角色的地域身份、社会阶层、性格暴躁程度全部建立，不需要任何额外的外貌描写。方言/语言质感的本质是：角色怎么说话，比角色长什么样更重要。

**怎么用**：给每个重要角色设计2-3个标志性的方言词或口头禅，在对话中自然使用。方言词不需要读者完全懂——"冚家铲"很多北方读者不完全明白，但能从语境猜到是脏话，这种"半懂不懂"反而增加了真实感。注意控制频率，方言词太多会影响阅读流畅度，每章出现3-5次为宜。

---

### 7. 美学化暴力描写

**操作**：用美学意象替代直白的暴力描写。不写"脑浆四溅"，写"粉白色的东西如倾洒的颜料"。美学化暴力不是美化暴力，而是用美的意象承载暴力的冲击——美的形式和暴力的内容之间的张力，比直接描写血腥更令人不安。

**案例**：十日终焉
> 粉白色的东西如倾洒的颜料，霎时间在桌面上横向铺开。

不写"脑浆四溅"，写"粉白色的东西如倾洒的颜料"——把暴力结果转化成视觉美学意象。"粉白色"是色彩，"倾洒的颜料"是动态，"在桌面上横向铺开"是空间延展。读者的第一反应是"画面感"，第二反应才是"这是脑浆"——这种延迟认知制造了更强的冲击。美学化暴力的核心是"间离效果"——先用美吸引你，再用暴力的真相击碎你。

**怎么用**：当需要描写暴力场景时，先找一个美学意象（颜料、花朵、丝绸、水墨），把暴力结果"翻译"成这个意象的形态。关键是"延迟认知"——先让读者欣赏画面，再揭示这是暴力。注意：此技法适用于需要制造心理冲击的关键场景，不要滥用，否则会变成"每个暴力场景都在炫技"。

---

### 8. 旁观者差异化反应

**操作**：同一恐怖事件，不同角色有不同级别的反应，按反应强度排列形成"恐怖梯度"。最恐怖的不是所有人都在尖叫，而是有人尖叫、有人发抖、有人沉默——沉默的那个最让人不安。

**案例**：诡舍
> 胖子看到人皮→女人腿软差点没站稳→有人干呕→宁秋水"反应远没有众人大"

四个角色四种反应，从生理性（干呕）到姿态性（腿软）到心理性（反应较小），形成递减的恐怖梯度。宁秋水的"反应远没有众人大"反而是最让人不安的——为什么她不怕？她见过什么？一个角色的"不反应"比所有人的"大反应"更能制造悬念。十日终焉同理：头颅撞碎→女生放声尖叫→齐夏"发起抖来"→花臂男噤声。花臂男是江湖人，他的"噤声"比女生的"尖叫"更能说明这个场景的恐怖程度——连混混都吓到说不出话。

**怎么用**：写群体恐怖场景时，至少设计3-4个角色的差异化反应，按反应强度从强到弱排列。最弱的那个反应（沉默、冷静、微笑）是制造悬念的关键——他为什么不害怕？这个疑问会驱动读者追读。差异化反应还能同时完成多角色刻画，一举两得。

---

### 9. 错位感官（恐惧型）

**操作**：让物理规律、日常常识、感官体验发生"错位"，用"不对劲"制造恐惧。恐惧的本质不是"看到了可怕的东西"，而是"世界突然不按规律运行了"。错位越微妙，恐惧越深。

**案例**：诡异入侵
> 相框贴墙→跃过四十公分宽的香案直接落地。

相框贴着墙，按物理规律应该垂直落下，但它"跃过四十公分宽的香案"才落地——物体违反了物理规律。这种错位之所以恐怖，是因为它攻击的是读者对现实的基本信任。诡异入侵还使用了多种错位变体：人迹缺失恐怖"清明节村子不可能没人"——该有人的时候没有人；声音骤停恐怖"镇上所有狗同时被掐住了脖子"——该有声音的时候突然寂静；熟人变形恐怖"梦中母亲蓬头垢面，四五十了——失踪时才三十出头"——该不变的东西变了；易燃性错位"往年下雨这纸怎么点都不容易烧起来，今儿个这雨跟油一样"——物质的性质反常了。

**怎么用**：找到场景中一个"应该如此"的常识（火是热的、物体会落地、活人不会飞），让它"不应该如此"。错位要微妙——不是鬼怪直接出现，而是一个小小的物理异常，让读者自己发现"不对劲"。错位感官最适合用在恐怖/悬疑类型，但注意：每章最多用一次，用多了就不再恐怖，变成"又一个灵异事件"。

---

### 10. 知识锚点

**操作**：用真实的专业知识细节建立可信度。读者不需要懂这个知识，但能感受到"这个作者做过功课"。知识锚点的力量在于：一个真实细节能辐射到整个场景，让虚构的世界也显得真实。

**案例**：末日生存
> 从锁骨两侧下一厘米下刀，切到胸骨中线。

这是真实的解剖学Y字形切口路径——从锁骨两侧下方起始，汇合到胸骨中线。读者可能不懂解剖学，但"锁骨两侧""下一厘米""胸骨中线"这些精确的解剖术语和数字，会传递出"这是真的"的信号。知识锚点的辐射效应是：当读者在一个细节上感受到专业度，他们会下意识信任整个叙事——如果作者连解剖切口都写得这么准，那他写的丧尸设定应该也靠谱。

**怎么用**：在关键场景中嵌入1-2个真实的专业细节（医学术语、军事流程、法律条文、工程参数），用精确的名词和数字让细节"硬"起来。不需要解释这个知识——解释反而破坏质感，读者只需要感受到"专业感"。注意：知识锚点是调味品不是主菜，一个场景最多一个知识点，多了就变成科普文了。

---

## 何时变套路

- 感官描写变成"每段必有环境描写"——每个场景开头都来一段天气+光线+温度+气味的全景描写，读者会直接跳过
- 错位感官变成"每章必有物理规律错位"——第一次相框飞过香案是恐怖，第十次就是"又来了"，恐惧感归零
- 美学化暴力变成"每次打斗都用颜料比喻"——美学意象重复使用就不再有间离效果，变成矫情
- 方言词泛滥到影响阅读——每句话都有方言，读者光猜词义就累死了，没有余力感受质感
- 好的做法：一个精准的细节胜过面面俱到。乱世荒年只写"甜！润！香！"三个字就够了，不需要描写肉的色泽、纹理、烹饪方式。感官锚点的核心是"少即是多"——在关键时刻用一个精准的感官细节击穿读者，其余时候留白

# 预期违背

> 通用技法。故意建立读者对类型/套路的预期，然后打破它。
> 来源：19本番茄头部小说前20章拆解

---

## 核心原理

预期违背的核心是**惊喜感**——读者以为知道接下来会发生什么，结果发生了意料之外但情理之中的事。不是"不套路"，是"先建立套路预期再打破"。好的预期违背是**稀缺的**——一本书只用几次。如果每个场景都反转，读者会预判你的预判，反过来变成新套路。

---

## 子技法与案例

### 1. 金手指拒绝——读者期待标配外挂，作者偏不给

**操作**：在穿越/觉醒/重生等标配金手指的类型中，故意不给读者预期的外挂，让读者好奇"那他怎么活"。

**案例**：大奉打更人Ch1
> 许七安穿越古代。读者预期：系统/金手指/老爷爷。
> "系统？"——"系统不搭理他"——"没有系统，竟然没有系统！"
> 整个穿越文类型暗示该有系统，作者偏不给。读者的好奇心瞬间被勾起——"没有系统怎么破局？"

**怎么用**：找到你的类型中"标配"的东西（系统/老爷爷/重生记忆/特殊血脉），然后故意不给。但不给的同时要给一个**替代方案**——大奉给了警校知识+推理能力。纯粹什么都不给读者会觉得作者在虐主。

### 2. 英雄动机解构——读者期待英雄行为，揭示真实动机

**操作**：角色做出英雄行为（救人/牺牲），但真实动机出乎意料（为了钱/为了虚荣/为了赎罪）。

**案例**：别叫我恶魔Ch1
> 任杰冲进火海救人——读者预期英雄动机
> "生当作人杰，死亦为鬼雄"——悲壮
> 回头问"救出一人奖励二百吧？"——英雄是为了钱
> "这英雄我当定了！"——动机解构后，英雄行为变成了打工赚钱

**怎么用**：让角色做出看似高尚的行为，然后用一句话揭示真实动机。动机要"意外但合理"——任杰确实在救人，但他确实也是为了钱，两者不矛盾。解构不是否定英雄行为，是给英雄行为一个不完美的理由。

### 3. 悲壮消解——在悲壮/庄严场面中植入荒诞

**操作**：先建立悲壮/庄严氛围，然后用一个荒诞细节瞬间打破。打破后氛围不回来——读者在笑中接受了悲壮。

**案例**：别叫我恶魔Ch1
> 任杰被废墟压住——濒死——悲壮
> 看到旁边少女——注意力在安全裤印花上——"狗头印花不错，下次别穿了"
> 悲壮被荒诞消解，但消解后读者反而更心疼任杰——因为他快死了还在看印花。

**怎么用**：在悲壮段落的第3-4行插入一个荒诞细节。细节要与悲壮氛围反差极大（生死→印花、战斗→肚子叫）。关键是**只插一句**——不要把悲壮段落变成搞笑段落，只用一句话打破然后回到正轨。

### 4. 元套路自觉——角色自己知道套路存在并主动反套路

**操作**：角色有类型/影视/小说的阅片量，知道自己身处套路中，主动打破套路。这不是作者的预期违背，是角色层面的反套路。

**案例**：精神病院Ch20
> "在电影里立这种flag的人一般都没好下场"
> "他很干脆的把这个flag拔了出来，撕吧撕吧丢进了太平洋"
> 角色知道"宣布好消息=出事"的套路，所以提前说了——既是反套路，也是角色魅力的展现。

**案例**：时停起手
> "前世他吃的饼够多了，老板天天画，他早就吃腻了"
> 角色知道"被画饼=被利用"的套路，所以不吃。

**怎么用**：让角色在关键时刻"意识到"自己正在经历某个套路。用内心独白或一句台词表达"我知道接下来会发生什么"——然后做与套路相反的选择。关键是角色要有"元意识"——知道套路的角色比不知道套路的角色更让读者惊喜。

### 5. 反英雄爽文化——主角主动拥抱反英雄身份

**操作**：读者预期主角有道德底线，主角主动声明自己没有底线——但用自嘲或幽默的方式让读者接受。

**案例**：游戏入侵Ch13
> "她重生而来不是为了当道德标兵的"
> 虞寻歌偷窃、杀人、割喉——但作者通过自嘲让读者接受：
> "虞寻歌拍了拍自己：你虽然偷东西抢东西还喜欢趴床底，但你可真是个好姑娘"
> 反英雄的自我认同让读者觉得"爽"而非"恶"。

**怎么用**：让主角在做出道德灰度行为后，用一句自嘲式独白"承认"自己的缺陷。自嘲消解了读者的道德不适感——"她知道自己不完美，但不装完美"比"假装完美"更让读者喜欢。

### 6. 预期内反转——读者预期反转，反转的方式出乎意料

**操作**：在读者已经预期到"会有反转"的情况下，反转的**方式**出乎意料。

**案例**：十日终焉Ch3
> 齐夏抽到说谎者牌——读者预期他会紧张/慌乱
> 他"不动声色的将卡牌扣住"——比读者更冷静
> 然后开始冷静制定策略——不是"慌乱中被逼出策略"，是"主动选择策略"
> 反转的方式不是"意外获救"，是"主角比读者更冷静"——出乎意料但符合人设。

**怎么用**：在读者预期到反转时，不要给"预期中的反转"（如突然获救/突然觉醒）。给一个"主角层面的反转"——主角的反应比读者预期更冷静/更极端/更反常。

---

## 何时变套路

- 每个场景都要反转——读者会预判你的预判，反过来变成新套路
- 金手指拒绝后不给替代方案——纯粹虐主读者会弃书
- 元套路自觉用太多——角色每章都"我知道套路"会变成新的烦人套路
- 反英雄爽文化变成"主角无底线"——必须有道德锚点让读者不讨厌他

好的预期违背是**稀缺的**——大奉只在第一章用了"没有系统"，此后不再重复。一本书用3-5次预期违背，每次都在关键时刻，效果最好。

## 2b 金手指设计

### 金手指类型+名称
- **类型**：系统（D&D数据面板的增强变体——数据之眼）。
- **名称**：**“深渊数据之眼”**。

### 加速机制
- **加速点**：骨架.md组成2（运转机制）中——**输入**（战斗/杀戮环节）和**转化**（灵魂碎片转化率）的双重加速。
- **加速方式**：
  1. **战斗输入阶段**：数据之眼能瞬间看到敌人的具体属性、技能冷却、弱点暴露窗口（比如“3秒后敌人左肩会暴露真身”）。主角可以打最高效的收割战，而不是盲目乱砍。
  2. **转化阶段**：数据之眼能显示当前击杀目标掉落的“灵魂碎片品质”和“掉落概率”，让主角优先选择高收益目标（而不是随机乱杀浪费精力）。
- **与众生攀登方式关系**：主角的攀登方式（没资源者的单枪匹马野区猎杀）完全不变。金手指只是提供了“野区地图上的精确采矿路线”和“BOSS的弱点提示”，让他在每次战斗中都打出高于面板的收益。

### 限制（红线级）
- **使用限制**：
  1. 每次激活数据之眼消耗精神力——连续使用3场高烈度战斗会出现“眼压过载”（双眼剧痛、视野模糊）。
  2. 对于等级高于主角5级以上的敌人，数据之眼只能看到模糊信息（如“力量？？？”），无法直接看到弱点窗口。
- **成长限制**：
  1. 数据之眼本身不升级——它始终基于主角当前的灵魂石板解析能力。主角等级越高，能获取的信息越精确（但上限是“能看到同阶及低一阶敌人的完整面板”）。
  2. 无法通过金手指跳级——主角必须自己从1级打到20级。
- **坐标系意义保留声明**：**金手指不取消坐标系本身的意义——主角仍然必须通过杀戮获取经验升级，无法跳阶，无法无视等级差；数据之眼只是让他打得更准、效率更高，但没法让他“不杀怪就升级”。**

### 代价
- **单次代价**：每次激活数据之眼（≥5秒）消耗5%~10%精神力。如果精神力见底，主角会呕吐、视线重影、思维迟缓。
- **累积代价**：过度使用→精神力上限永久下降（主角的“灵魂石板”会出现裂纹，影响技能学习速度）。为了恢复上限，需要消耗珍贵的“灵魂净水”，而这种东西只有领主级以上势力才能稳定产出。
- **代价来源对齐**：与骨架.md组成4（代价结构）的通用代价“灵魂磨损”一致——金手指的代价只是放大和加速了这个磨损过程。

### 与引擎关系
- **金手指在引擎中的定位**：组成2（运转机制）的加速器。
- **引擎闭环是否改变**：不改变——金手指只加速主角在“输入”和“转化”环节的效率；底层闭环“猎杀→汲取灵魂→强化面板→猎杀更强者”没有变。
- **主角驱动逻辑是否改变**：不改变——主角的核心欲望（复仇）仍由骨架.md组成1（世界级驱动力：生存压力+零和资源争夺）驱动，

  - 铁骨（反派·铁骨帮帮主·衰老期）——核心欲望：稳定帮会，维护自身统治。本卷张力：无力控制内斗，最终被血斧吞并。
  - 血斧（反派·血斧佣兵团团长·正式登场ch005）——核心欲望：吞并铁骨帮，扩大猎杀路线控制权。本卷张力：作为幕后势力压力，推动铁骨帮内斗加速。

#### 涉及地图
- 边缘废墟（起点区域）：碎骨者1-3级。没有势力归属，但有铁骨帮控制低级魔物刷新点。废墟内无特殊法则。
- 黑森林外围（核心猎场）：碎骨者3-5级。有幽暗法则（夜间视野降低50%，数据之眼消耗翻倍）。孤狼猎团占据高收益路线，铁骨帮控制部分入口。
- 废弃矿洞（隐藏副本·ch004）：碎骨者4-6级。有法则裂缝（数据之眼随机中断）。内有5级蜘蛛母皇+旧神壁画（仇人线索）。
- 格杀镇（安全区·ch005后半）：碎骨者2-7级。有领主法则（禁止械斗）。血斧佣兵团和黑手套商会据点在此。本卷末主角首次到达。

#### 整卷主要矛盾/主要剧情走向
- **主要矛盾**：主角必须在“不被任何势力吞并”的前提下，利用金手指加速升级到足以离开边缘废墟的等级（≥5级），同时避免被铁骨帮内斗牵连导致提前暴露越级猎杀能力。
- **主要剧情走向**：主角在边缘废墟和黑森林外围猎杀升级（3→4级）→被迫介入铁骨帮副帮主内斗→击杀黑牙（6级）完成越级突破→发现仇人线索→升级至5级→血斧吞并铁骨帮→主角离开边缘废墟进入格杀镇。全程是“被迫卷入上层纷争的底层逆袭”。

## 2b 分支剧情线（≥6条）

### 主线1：越级猎杀升级线
**类型**：主线 | **涉及势力/地图**：边缘废墟→黑森林外围→废弃矿洞 | **众生攀登方式**：没资源型（野区独狼模式）
- **起点状态**：主角3级，只能打1-2级魔物，升级速度极慢
- **终点状态（第5章末）**：主角5级，完成3次越级猎杀（第一次4级暗影狼→第二次5级蜘蛛母皇→第三次6级黑牙），获得仇人线索
- **与主要矛盾的关系**：这条线是主角生存的基础——不升级就活不过下一章。越级猎杀暴露能力→被铁骨帮盯上→被迫卷入内斗。
- **众生攀登方式标注**：体现没资源者在野区的生存策略——不抱团、不依附、全靠个人战斗力和信息差。金手指（数据之眼）是“信息差”的具象化。

### 主线2：铁骨帮内斗卷入线
**类型**：主线 | **涉及势力/地图**：铁骨帮（边缘废墟+铁骨街） | **众生攀登方式**：没资源型（内卷式帮会模式——争抢有限猎杀路线资源）
- **起点状态**：铁骨帮内斗潜伏期，黑牙和疤脸表面和平，主角尚未进入帮会视野
- **终点状态（第5章末）**：黑牙被击杀，铁骨帮内斗终结→血斧吞并铁骨帮→疤脸成为附庸→主角获得自由身
- **与主要矛盾的关系**：这条线是“主角无法独善其身”的体现——他是碎骨者3级，在三个6级噬魂者面前毫无话语权，必须用计谋而不是硬拼。
- **众生攀登方式标注**：体现没资源者内部资源争夺的残酷——帮会成员靠上缴碎片换取猎杀路线使用权，帮会生存依赖上贡保护费给更上层势力。资源链的每一环都在吸血。

### 支线1：疤脸的忠诚裂变
**类型**：支线 | **涉及势力/地图**：铁骨帮 | **众生攀登方式**：没资源型（保守派——试图在不改变底层结构的前提下维持现状）
- **起点状态**：疤脸对帮主铁骨保持忠诚，反对黑牙激进夺权
- **终点状态（第5章末）**：疤脸选择与主角合作除掉黑牙→背叛铁骨→成为血斧附庸。发现“忠诚”在底层帮会中无法独善其身。
- **与主要矛盾的关系**：疤脸是主角在铁骨帮内的盟友和牵制者。疤脸的决策直接影响主角的生存空间。
- **众生攀登方式标注**：体现没资源者中“保守派”的困境——不想改变规则、不想依附上层，但内卷环境逼着每个人选择阵营。

### 支线2：黑牙的野心自焚
**类型**：支线 | **涉及势力/地图**：铁骨帮+血斧佣兵团（幕后接触） | **众生攀登方式**：没资源型→激进攀附型（试图通过投靠上层势力跳过中期积累阶段）
- **起点状态**：黑牙暗中与血斧接触，承诺如果自己上位就把铁骨帮30%猎杀路线交给血斧
- **终点状态（第4章末）**：黑牙被疤脸+主角联手击杀，野心落空
- **与主要矛盾的关系**：黑牙是主角在本卷的主要直接敌人。他最先发现主角越级猎杀能力，想除掉主角以巩固自身地位。
- **众生攀登方式标注**：体现没资源者中“投机型”的失败路径——想跳过阶段直接攀附上层，结果成为上层势力的棋子（血斧只是利用他来加速铁骨帮内耗，并无意真正扶持他）。

### 支线3：血斧佣兵团的暗影
**类型**：支线（暗线铺垫） | **涉及势力/地图**：血斧佣兵团（格杀镇） | **众生攀登方式**：有资源型（圈养魔物、控制猎杀路线的组织化攀登）
- **起点状态**：血斧团长的视角——黑牙主动联系他谈条件，他先答应但不承诺任何实质支持，让铁骨帮内耗加速
- **终点状态（第5章末）**：铁骨帮内斗结束→血斧以“帮助稳定局势”为名吞并铁骨帮，控制边缘废墟的猎杀路线
- **与主要矛盾的关系**：血斧是本卷真正的“上层势力压迫者”——他不需要亲自出手，只靠一个承诺就让铁骨帮自相残杀，然后不费一兵一卒吞并。
- **众生攀登方式标注**：体现“有资源的人”如何利用资源差碾压没资源的人——血斧不需要亲自战斗，只需要控制情报和资源流通，就能让底层势力为他内耗。

### 支线4：废弃矿洞的旧神线索
**类型**：支线（伏笔） | **涉及势力/地图**：黑森林·废弃矿洞 | **众生攀登方式**：无（纯探索）
- **起点状态**：主角不知道矿洞存在，纯粹是猎杀4级魔物时偶然发现洞口
- **终点状态（第4章末）**：主角击杀蜘蛛母皇→在矿洞深处看到旧神壁画→发现仇人的名字出现在壁画边缘→获得第一条复仇线索
- **与主要矛盾的关系**：这条线是“复仇”核心欲望的第一次具象化触发。在本卷末尾成为主角离开边缘废墟的底层驱动力。
- **众生攀登方式标注**：无需要标注的攀登方式。

### 暗线1：复仇背后的信息积累
**类型**：暗线（本卷只埋种子） | **涉及势力/地图**：全境 | **众生攀登方式**：无
- **起止状态**：起点→主角只知道仇人是15级以上的深渊领主，其他信息为零。终点→知道仇人的名字出现在废弃矿洞的旧神壁画边缘（“裂隙深渊领主·玛格索斯”），但这个名字对他来说仍然是一个空洞的代号。
- **与主要矛盾的关系**：这是主角所有行动的底层驱动力，但不直接参与本卷冲突。只在第4章末短暂浮出水面，第5章末又沉回水面之下。
- **众生攀登方式标注**：无。
- **预计揭示卷**：第三卷（主角升至深渊领主阶位后，开始系统性调查玛格索斯的背景）。

### 暗线2：世界的裂痕
**类型**：暗线（本卷只埋暗示） | **涉及势力/地图**：废弃矿洞 | **众生攀登方式**：无
- **起止状态**：起点→无异常。终点→主角在矿洞深处感觉到一种“法则裂缝”带来的不协调感——像是这个世界的数据面板背后有什么东西在漏出来。
- **与主要矛盾的关系**：与主线无直接关系。但这是为后期“世界观真相揭露”埋的伏笔——暗示深渊法则并非完美无缺，旧神的陨落可能留下了系统性漏洞。
- **众生攀登方式标注**：无。
- **预计揭示卷**：第五卷（主角升至主宰者阶位后，发现深渊法则的系统性漏洞）。
