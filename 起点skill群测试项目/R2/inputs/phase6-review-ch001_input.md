## 做什么

输入：本章施工卡（plot-design 产出） + 本章正文（prose-render 产出） + DNA 笔触要求 + 骨架.md（seed 产出） + 主角设计.md（seed 产出）
输出：审核报告（四维审核结论 + gap 分析 + 决策） + 本章摘要 + 状态快照 + current-state 更新

核心原则：**不是强制正文服从大纲，也不是放任正文偏离大纲，而是作为编辑判断"谁更好"。**
审核优先级：**符合性 > 笔触 > 好看度 > 沉淀。前面不通过，后面不审。**

## Step 1: 符合性检查（正文 vs 施工卡 + 骨架 + 设定）

> 第一道闸门：正文是否兑现了施工卡承诺的故事，是否遵守骨架（坐标系 + 动力引擎）的约束。
> 符合性是第一闸门——偏离核心事件（主线丢失）=废章，骨架一致性不通过=废章，天赋约束取消坐标系意义=废章。

### Step 1.0：加载审核上下文

读取：
- 本章施工卡（来自 plot-design 的幕纲）
- 本章正文（来自 prose-render 的渲染产出）
- 骨架.md（来自 seed——力量体系坐标系 + 动力引擎众生攀登系统）【v3.0.0 新增】
- 主角设计.md（来自 seed——金手指设计 + 爽感矛盾公式 + 攀登方式）【v3.0.0 新增】
- 种子文档（风格关键词/不可牺牲项/禁区）
- 流派 skill 的笔触 DNA（如 `prose-render-dndlike/dna/深渊主宰-笔触DNA.md`——叙事距离/面板格式/章节过渡/战斗模式分层/POV 规则等）
- 种子文档的不可牺牲项与禁区（用于红线检查）
- 前一章状态快照（如有——用于衔接检查）
- 世界圣经活跃层（本幕的 NPC/地点/组织/风俗/资源——检查正文是否与活跃层一致）

### 1a. 核心事件对照（gap 逐 beat 对比）——保留 v2.0.1

将施工卡的 7 节拍硬锚点和微 beat 施工表，与正文实际内容逐项对比。

对比方法：
1. 施工卡的每个节拍/beat，在正文中找到对应段落
2. 标注匹配状态：✅匹配 / ⚠️偏移 / ❌缺失 / ➕新增

输出 gap 清单：
```
| beat# | 施工卡内容 | 正文实际 | 状态 | gap类型 |
|-------|-----------|---------|------|---------|
| 1 | 催收员隔门喊话 | 催收员隔门喊话 | ✅ | — |
| 7 | 决定今晚动手→磨刀 | 暗杀执行→翻尸体→面板结算 | ⚠️ | 提速（吞并ch04） |
| + | — | 索伦去十字巷获取情报 | ➕ | 新增（大纲无） |
```

#### gap 分类（保留 v2.0.1 的 5 种类型）

| 类型 | 含义 | 典型场景 |
|------|------|---------|
| **提速** | 正文把后续章节的内容提前写了 | ch03 写了 ch04 的暗杀执行 |
| **延迟** | 正文在一个 beat 上花了过多篇幅，没写完本章该写的 | ch02 花 3000 字写面板激活，没写到门上催收刻痕 |
| **偏离** | 正文走了和施工卡不同的方向 | 施工卡写"暗杀"，正文写了"谈判" |
| **遗漏** | 正文跳过了施工卡中的某个 beat | 施工卡有"翻尸体发现纸条"，正文没写 |
| **新增** | 正文写了施工卡中没有的内容 | 正文新增了一条"格里芬势力线" |

#### 质量判断（保留 v2.0.1）

对每个 gap，回答一个问题：**正文版本更好还是大纲版本更好？**

判断维度：
1. **叙事自然度**：正文版本是否比大纲分章更自然？
2. **读者体验**：正文版本是否给读者更好的阅读体验？
3. **爽感完整性**：正文版本是否保持了爽感的完整闭环？
4. **后续影响**：如果采纳正文版本，后续施工卡需要怎么调整？调整成本是否可接受？
5. **字数合理**：正文是否在 2000-2500 字范围内且不超过 3000 字？

判断结论（每个 gap 一个）：
- **正文更好**：采纳正文，调整后续大纲
- **大纲更好**：标记正文需要修改，附修改建议
- **各有优劣**：标注优劣，建议人工裁决

**核心事件判定**：
- 全部兑现 → 进 1b
- 有偏离但主线未丢 → 标注偏离，进 1b（下游需评估偏离是否可接受）
- 主线丢失（核心事件未写/被替换成别的） → **废章，打回重写**

### 1b. 骨架一致性检查（新增）

打开骨架.md，对照本章正文检查骨架约束是否被遵守。骨架（力量体系 + 动力引擎）是硬约束，不通过不得放行。

#### 1b-1. 力量体系一致性
- 坐标系是否被遵守：主角/配角的能力表现是否在骨架.md 力量体系四层结构（世界源力量/主养成线/子养成线/交叉规则）定义的阶位范围内？
- 越级检查：有没有越级但不付代价？（越级必须通过金手指加速 + 代价兑现，不能凭空跨越坐标系阶位）
- 属性数值检查：面板数值/等级是否与坐标系阶位刻度一致？有没有属性偷跑？

#### 1b-2. 动力引擎一致性
- 众生攀登逻辑有没有断裂：本章的攀登行为是否符合骨架.md 动力引擎组成 3（众生攀登方式分层）？主角的攀登方式是否与众生通用闭环一致（输入→转化→输出→反馈，只是更快）？
- 驱动逻辑连续性：本章行为动机是否与骨架.md 组成 1（驱动逻辑——世界级驱动力）对齐？
- 代价结构兑现：本章的攀登是否兑现了骨架.md 组成 4（代价结构——通用代价 + 群体差异代价 + 累积效应）？

**判定**：
- 通过 → 进 1c
- 不通过 → **废章，打回重写**（骨架一致性是硬约束，越级不付代价/攀登逻辑断裂/驱动逻辑脱钩=废章）

### 1c. 天赋约束检查（新增）

打开主角设计.md 的金手指设计（2b），对照本章正文检查：

| 检查项 | 判定标准 | 不合格处理 |
|--------|---------|-----------|
| 金手指不喧宾夺主 | 金手指是否让坐标系失去意义？是否取消了众生攀登的意义？ | **废章**（金手指取消坐标系意义=废章） |
| 限制兑现 | 金手指限制（主角设计.md 2b 声明的"不能取消坐标系本身的意义"等限制）是否被遵守？有没有突破声明的限制？ | 标注越界点，打回修改 |
| 代价兑现 | 金手指使用是否付出了代价（爽感矛盾公式的"代价约束"因子）？有没有无代价白嫖？ | 标注未付代价点 |

**判定**：
- 通过 → 进 1d
- 金手指取消坐标系意义 → **废章**
- 限制/代价未兑现 → 标注越界点，打回修改

### 1d. 剧情映射检查（新增）

检查正文中的血肉（地图/势力/人物/剧情）是否映射骨架坐标系，有没有脱离坐标系的无效填充：

| 检查项 | 判定标准 | 不合格处理 |
|--------|---------|-----------|
| 地图映射 | 本章出现的地点是否在坐标系中有位置意义？（不是随便一个地名，而是坐标系中的某个阶位/势力范围） | 标注脱离坐标系的地点 |
| 势力映射 | 本章涉及的势力是否在坐标系中有层级定位？势力关系是否与坐标系一致？ | 标注脱钩势力 |
| 人物映射 | 本章出场人物是否在坐标系中有位置？（不是路人甲，而是坐标系某阶位/某攀登方式的代表） | 标注无效人物 |
| 剧情映射 | 本章事件是否推动坐标系内的攀登/博弈？有没有与坐标系无关的无效剧情填充？ | 标注无效剧情 |

**判定**：
- 通过 → 进 1e
- 不通过 → 标注脱离坐标系的血肉清单，建议删改

### 1e. 角色一致性检查

对照角色库 + 主角设计.md 的攀登方式，检查：

| 检查项 | 判定标准 | 不合格处理 |
|--------|---------|-----------|
| 角色性格一致性 | 角色行为是否符合角色库中的性格描述 | ✓/✗（如偏离，写明哪句不符合） |
| 攀登方式一致性 | 角色行为是否符合其攀登方式类型（对应骨架.md 组成 3 众生攀登方式分层中的某一种） | ✓/✗（如偏离，标注哪句不符合） |
| 角色出场连续性 | 分幕设计标注该出场的角色是否全部出场 | ✓/✗（如缺失，写明谁没出场） |
| 角色状态连续性 | 角色力量层级/关系状态是否前后衔接 | ✓/✗（如断裂，写明哪里不衔接） |
| 面板使用检查 | 战斗/升级场景是否使用了 DNA 面板格式 | ✓/✗（如缺失，标注哪个场景缺面板） |

**角色不一致 → 废章标记，打回修正**。

### 1f. 爽感闭环检查

#### 三段式检查
- 本章主推爽感类型：______
- 触发段：有/无（位置）
- 爆发段：有/无（位置）
- 后果段：有/无（位置）
- 三段缺一 → 废章标记，打回补全

#### 爽感因子检查（新增）
爽感矛盾公式（坐标系门槛 × 天赋加速 × 代价约束）三因子，本章是否触发了至少两个？
- 坐标系门槛因子：是否体现了跨越坐标系阶位的难度？
- 天赋加速因子：是否体现了金手指加速跨越？
- 代价约束因子：是否体现了金手指代价/限制带来的张力？
- 触发因子数：__ / 3（<2 → 爽感强度不足，标注建议补足哪个因子）

### 红线检查（保留 v2.0.1）

检查正文是否违反硬性红线（不管 gap 判断结果，红线违反必须修改）：

| 红线 | 检查方法 |
|------|---------|
| 隔离红线 | 正文中是否出现原作专有元素（非同人测试模式下） |
| 篇幅红线 | 正文字数是否在 DNA 要求的范围内 |
| 主动行为红线 | 本章是否有 ≥1 个主角主动行为 |
| 面板格式红线 | 面板是否用引号包裹系统消息嵌入正文（非代码块） |
| 设定强调纪律 | 同一设定约束是否在一章中提及 >1 次 |
| 战斗多样性红线 | 是否连续 3 章使用同一种战斗模式 |
| 三层世界红线 | 正文中出现的地名/NPC/组织/风俗是否在活跃层有记录？正文是否现编了未落盘的世界细节？ |

## Step 2: 笔触检查

### 2a. AI 味 7 项检查（从番茄 review 移植）

定性判断，不打分。逐条检查：

| 检查项 | 判定标准 | 不合格处理 |
|--------|---------|-----------|
| 句式重复 | 同一句式连续使用 2 次以上？ | 标注位置 + 建议替换 |
| 情绪直给 | "他感到 XX""心里一 XX""不由得"？ | 标注位置 + 建议用动作展示 |
| 结尾收束 | 章末停在总结性旁白？ | 标注 + 建议停在动作或对话 |
| 描写堆叠 | 连续 3 行以上纯感官描写无动作推进？ | 标注 + 建议插入动作 |
| 信息重复 | 同一信息一章内出现 2 次以上？ | 标注 + 建议删除 |
| 对话死板 | 连续 6 行以上纯对话无动作？ | 标注 + 建议穿插动作 |
| 高疲劳词 | 冷笑/蝼蚁/轰然炸裂/倒吸凉气/瞳孔骤缩/满场死寂等单章 ≥2 次？ | 标注 + 建议替换 |

### 2b. 笔触 DNA 一致性检查（保留 v2.0.1 的 8 维度）

对照 DNA 笔触要求，检查正文的笔触符合度：

| 维度 | 检查内容 |
|------|---------|
| 叙事距离 | 是否以主角视角为主体（≥60%）？镜头切换是否有标志且快速切回？ |
| 对话引导词 | 是否使用 DNA 指定的引导动词？（如深渊主宰 DNA 用"道"不用"说"） |
| 面板弹出方式 | 是否作为意识流数据流嵌入正文？是否不停顿动作节奏？ |
| 章节过渡 | 是否使用了正确的过渡模式？是否连续两章使用相同过渡？ |
| 感官序列 | 新场景首次出场是否使用了感官序列？非新场景是否避免了感官序列？ |
| 情绪收束 | 杀完人后是否有冷评收束（或首次杀人的递减式心理波动）？ |
| 参照物梯子 | 面板数值弹出时是否用参照物建立认知标尺？ |
| 主角定位 | 面对威胁时是否先算后动？情绪是否通过瞳孔/呼吸/手指表达而非面部？ |

## Step 3: 好看度检查（定性 4 问，新增）

> 这一步最主观也最重要。不要被技法 checkboxes 蒙蔽——技法全 ✓ 不代表好看。

逐条回答，必须给出具体证据，不能泛泛说"还行"：

1. **有没有劲？**（读完这章有没有兴奋感）
   - 有 → 引用最抓人的 1-2 句正文
   - 没有 → 问题在哪？（节奏太慢/信息太少/主角太怂/废话太多/场景太常见）

2. **记忆点？**（读完能记住什么具体画面/台词/反转）
   - 找得到 → 引用原文
   - 找不到 → 本章缺乏记忆点，标注"无记忆点"

3. **哪里无聊？**（哪段想跳过）
   - 标注具体段落位置（如"第 3-5 段屋顶夜景描写偏长"）
   - 建议：删/压缩/插入动作/换场景

4. **代入感？**（有没有跟主角共情的瞬间）
   - 读者能想象出画面吗？（哪段画面感强/弱）
   - 主角的选择和读者会重叠吗？（主角是否主动决策）
   - 配角像活人吗？（有没有独立动机/小动作）

## 红线

1. **gap 必须逐 beat 检查**：不能只看总体感觉，必须逐个 beat 对比。跳过 beat 对比=不合格。
2. **质量判断必须有理由**：每个"正文更好"或"大纲更好"的判断必须附理由（哪个维度更好、为什么）。无理由的判断=不合格。
3. **红线违反不可妥协**：即使正文在叙事上更好，如果违反了硬性红线（如面板用代码块而非引号），仍需修改。
4. **骨架一致性检查不通过=废章**（新增）：骨架（力量体系 + 动力引擎）是硬约束，正文越级不付代价/动力引擎攀登逻辑断裂/驱动逻辑脱钩=废章，必须打回重写。
5. **天赋约束检查发现金手指取消坐标系意义=废章**（新增）：金手指喧宾夺主，让坐标系失去意义/取消众生攀登意义=废章。金手指是引擎加速器，不是引擎本身。
6. **摘要必须客观**：摘要只记录"发生了什么"，不评价"写得好不好"。
7. **状态快照必须准确**：状态快照的数据必须和正文中的面板数据一致。如果正文有属性偷跑，快照标注"正文数据异常"。
8. **主角变化必须每章沉淀**（新增）：位置/能力/资产/心态/关系五项缺一不可，不沉淀=状态断裂。
9. **钩子追踪必须列待回收列表**（新增）：超出 3 章未回收的钩子必须标红预警。
10. **不打分**（新增）：定性判断"通过/需修改/废章"，不给百分比或星数。

# 第一章：碎骨者的黎明

黎明前的边缘废墟笼罩在一层灰白色的雾气中。

空气中弥漫着腐烂和铁锈的气味——那是废墟特有的味道，混合了朽木、霉变的布料、以及某种说不清道不明的血腥味。陆渊蹲在一面坍塌了大半的石墙后面，目光穿过雾气的缝隙，锁定三十步外的一只猎物。

那是一只暗影狼。

黑色的皮毛在灰雾中几乎无法辨认，只有在它微微侧头时，眼睛反射出一点幽绿色的荧光——那是最低级的灵魂能量溢出造成的视觉特征。它正低头啃食一具半腐烂的鼠类尸体，利齿撕开皮肉的细微声音穿透雾气传来。

陆渊没有立刻动手。

他闭上眼睛，感受着脑海深处的那股力量涌动——像是一根冰冷的针扎进意识深处，然后在眼球后方的某个位置扩散开来。那股力量触及眼眶的瞬间，他的视野中浮现出一组半透明的数据流。

“深渊数据之眼，激活。”

只有他自己能听到这个声音——不是耳朵听到的，是意识深处直接灌注的信息流。面板以数据块的形式浮现在视野右侧，像是有人在他脑子里打开了一页笔记：

“暗影狼【野兽】（二阶）。”
“挑战等级3，生物等级4，野兽生命。”
“最高属性敏捷17点，最低属性智力3点，总属性值10－17点。”
“特殊专长：夜间潜行、撕咬锁定。”
“挑战难度：D-。”

陆渊的瞳孔微微收缩。

4级。比他高1级。

按照这个世界最朴素的实力判定规则，碎骨者3级打3级魔物需要拼命，打4级魔物——理论上是“完全打不过”。但数据之眼给了他一个关键的窗口：这只暗影狼的最高属性是敏捷17点，最低是智力3点。速度快、攻击高，但战术判断能力几乎为零。只要不犯错，它就是个高攻低防的木桩。

关键是——不要让它进入战斗状态。

暗影狼的“夜间潜行”专长让它在这片灰雾中几乎不可见，但它的“撕咬锁定”需要先锁定目标位置。如果能在锁定完成之前完成击杀……陆渊的大脑飞速运转，数据的流动让他的眼神变得更加冷静。他缓缓调整呼吸，将身体的重心压低到几乎贴地的角度——他的动作经过无数次的打磨，肌肉在记忆中的轨迹下无声地移动。

那块残破的灵魂石板碎片在他的胸口微微发热。他从不把碎片拿给别人看——那是他在一座废弃神庙的废墟下面发现的，是他觉醒灵魂石板的关键，也是数据之眼的源头。那碎片的存在，是他在这个世界中唯一比其他人多出的筹码。

暗影狼还在埋头啃食。它没有察觉危险。

陆渊的匕首从腰间的皮鞘中无声抽出。那柄匕首的刀刃已经有了几道细微的缺口——是他在这两个月里和3级魔物搏杀时留下的痕迹。他没有钱换更好的武器，只能将就着用。但这并不妨碍他精准地将刀锋对准目标的后颈——喉咙与脊椎之间的缝隙。

他深吸一口气……

然后动了。

身体像绷紧的弓弦突然松开，他整个人从石墙后弹射而出。雾气被他的身体切开，他跨越三十步的距离只用了不到四息——最后两步时他已经完全压低了重心，匕首向前刺出——

刀锋没入。

精准地切入了暗影狼的喉咙，刀刃从颈椎的缝隙中穿出，沾着温热鲜血的刀尖从另一侧的皮毛中透出。暗影狼甚至来不及发出一声呜咽——它的大脑在刀锋切断脊髓的瞬间就停止了向身体发送指令。

“背刺发动！”
“成功发动背刺*2，对目标造成24点伤害！”
“目标死亡！”
“抽取目标灵魂能量，获得杀戮经验45点。”

数据流在陆渊的视野中一闪而过。

他抽出匕首，迅速后退一步。

噗通。

暗影狼的尸体侧倒在腐烂的鼠尸旁边，鲜血从喉部的伤口中汩汩涌出，浸湿了泥土和碎石。陆渊低头看着自己的战利品——这是他第一次越级击杀4级魔物。前两个月他一直在猎杀1-2级的魔物，那些只有10-15点经验值，积攒了一个多月才从2级升到3级。而这一次——45点经验——他的经验条跳了一小截。

他蹲下身，熟练地开始处理尸体。暗影狼的皮毛能卖几个铜币——边缘废墟的旧货商收这种货，用来做低级软甲的内衬。狼牙也是好东西，可以磨成粗糙的护身符，卖给那些连灵魂石板都没觉醒的普通人。

“发了一笔。”他在心里默默地想。

但就在这时，他的腹部猛地一沉——一阵剧痛从胃部传来，像是有什么东西在他体内用力拧了一把。他的身体晃了一下，连忙单手撑住地面稳住身体。额头开始冒出冷汗，视线也变得模糊起来。

“数据之眼连续激活47秒……精神力消耗达到阈值……建议暂停使用。”

他深吸一口气，用力闭了一下眼睛，再睁开时视野恢复了清晰。但那股恶心的眩晕感还残留在身体里——像是刚喝下了半瓶劣质酒，头脑发胀、胃里翻涌。他掏出腰间的水袋灌了一口冷水，强迫自己压下呕吐的冲动。

“5%精神力消耗……果然不是开玩笑的。”

他收起匕首，蹲在暗影狼的尸体旁边，忽然感觉到胸口的灵魂石板碎片发出了一阵温暖的震动。他猛地低头——那块碎片正在微微发光，表面浮现出一道细微的裂纹。

不……不是裂纹，是裂开的部分正在向周围延伸——像是有新的纹路在石板上生长出来。他的手指触碰到碎片，一股冰凉的能量顺着指尖流入掌心，然后沿着手臂逆流而上，直冲脑海。

“灵魂碎片吸收完成。经验值达到升级阈值。”

陆渊的动作停住了。

他闭上眼睛，看到自己的意识中浮现出一组新的数据流——比暗影狼的鉴定面板更复杂，行数更多：

“碎骨者职业提升到4级。”
“获得5点技能点，生命值提高8点。”
“获得1点自由属性。”

他的眼睛睁开时，瞳孔中闪过一丝前所未有的光亮。他抬起右手，握了握拳头——他能感觉到力量在肌肉中流淌，像是身体里原本干涸的河道突然注入了水流。所有感官都变得更加敏锐：远处的一片落叶落地的声音、雾气中飘来的泥土味道、连自己心脏跳动的节奏都清晰可闻。

他低头看了一眼自己的手臂——原本因为营养不良而略显单薄的肌肉，此刻隐约能看出轮廓的变化。那是一种写在身体上的变化——面板的数值变成了实实在在的体感。

碎骨者4级。

“一个月前我还被一只2级的铁皮鼠追着咬，”他自言自语道，声音低沉，“三天前才能稳定猎杀3级魔物……现在我已经能越级杀4级了。”

他沉默了几秒钟，然后低头继续处理暗影狼的尸体——从皮毛开始剥离，每一刀都精准、迅速、不留残余。那柄缺了口的匕首在他的手里，比刚才更稳了。

他一边干活一边想着刚才的数据——5点技能点，1点自由属性。技能点可以分配到潜行、背刺、野外生存这些基础技能上。自由属性……他毫不犹豫地加在了敏捷上。

“敏捷是第一优先级。速度决定了你能不能在敌人反应之前完成击杀，决定了你能不能在被围攻时全身而退。力量可以靠技巧弥补，敏捷不行——敏捷不够就是不够，你的刀永远比敌人慢半拍。”

他的敏捷从17点变成了18点——那一点差距在数据面板上只是数字的跳动，但在他身体里，他感觉自己的动作轻快了几分，像是在那一瞬间卸掉了一层看不见的束缚。

他处理完暗影狼的皮毛时，手指划过刀锋上的缺口——他在心里记了一笔：这柄匕首最多再用一个星期，缺口太深了，再继续用下去可能在战斗中崩刃——那会要了他的命。他需要一柄新的匕首，但一柄好的精钢匕首在格杀镇的武器铺里至少要8个银币——而他全部的家当加起来不到2个银币。

他撕下一块布条，将剥离下来的暗影狼皮卷起来绑好，然后站起身。天色已经开始泛白——雾气在晨光中变成了一种暗淡的橙黄色，像是火焰熄灭后的余烬。

他抬头望向雾气的深处。那是黑森林的方向——雾中央更大、更浓的区域，仿佛藏着什么他不知道的东西。黑森林的边界隐隐可见，那一片幽暗的树影在雾气中微微扭曲，像是某种活物的呼吸。

“下一个猎场——黑森林外围。”

他的声音很平静，像是在陈述一件已经决定好的事。

黑森林外围的猎杀路线是被铁骨帮控制的——他们每天向进入猎场的猎人收取“场地使用费”，要么交钱，要么上缴一半猎物的灵魂碎片作为保护费。铁骨帮的老大是一个7级噬魂者，在边缘废墟这地方已经是顶层战力。

陆渊当然不打算交费。

他又不是固定猎场的常客——他本来就是个没有势力归属的单干户，在一个地方猎杀几天就换个区域。只要他不在铁骨帮的核心地盘上待太久，被抓住的概率并不高。而且——他现在是碎骨者4级了，虽然仍然打不过铁骨帮的高级成员，但逃命的概率总归高了一些。

他看了一眼手中的狼皮，嘴角浮起一丝冷意。

“那就试试看吧。”

边缘废墟的黎明安静得不像话。远处传来几声早起的鸟叫——那种没有被污染的、正常的鸟叫。在边缘废墟待久了，连听到正常鸟叫都让人觉得奢侈。

陆渊踩过灰白色的碎石，朝着黑森林的方向走去。他的身影在灰雾中很快变得模糊，然后消失不见。

他走后约莫三刻钟，一道瘦长的黑影无声地出现在暗影狼尸体旁边。

那是一个穿着破烂皮甲的男人，脸上有一道从额头斜切到下巴的刀疤，让他的半张脸看起来像是被人用刀劈开过。他蹲下身子，用手指蘸了蘸尚未完全凝固的狼血，放在鼻子前闻了闻。

然后他看到了地面上那一道清晰的刀刃痕迹——刀锋切入脊椎的精确角度，以及地面上几乎没有挣扎痕迹的现场。

他的眼神变得凝重。

“一刀毙命……切入位置是脊椎和咽喉之间的缝隙……精确得像是用尺子量过的。”他低声喃喃道，“这水平……不是一般人干的。至少得是个4级以上的暗杀者。”

他站起身，目光望向陆渊消失的方向——黑森林外围的入口。

“这小子……啧。”

他转身走向废墟的另一个方向，步子比来时要快一些。

## 2b 金手指设计

### 金手指类型+名称
- **类型**：系统（D&D数据面板的增强变体——数据之眼）。
- **名称**：**“深渊数据之眼”**。

### 加速机制
- **加速点**：骨架.md组成2（运转机制）中——**输入**（战斗/杀戮环节）和**转化**（灵魂碎片转化率）的双重加速。
- **加速方式**：
  1. **战斗输入阶段**：数据之眼能瞬间看到敌人的具体属性、技能冷却、弱点暴露窗口（比如“3秒后敌人左肩会暴露真身”）。主角可以打最高效的收割战，而不是盲目乱砍。
  2. **转化阶段**：数据之眼能显示当前击杀目标掉落的“灵魂碎片品质”和“掉落概率”，让主角优先选择高收益目标（而不是随机乱杀浪费精力）。
- **与众生攀登方式关系**：主角的攀登方式（没资源者的单枪匹马野区猎杀）完全不变。金手指只是提供了“野区地图上的精确采矿路线”和“BOSS的弱点提示”，让他在每次战斗中都打出高于面板的收益。

### 限制（红线级）
- **使用限制**：
  1. 每次激活数据之眼消耗精神力——连续使用3场高烈度战斗会出现“眼压过载”（双眼剧痛、视野模糊）。
  2. 对于等级高于主角5级以上的敌人，数据之眼只能看到模糊信息（如“力量？？？”），无法直接看到弱点窗口。
- **成长限制**：
  1. 数据之眼本身不升级——它始终基于主角当前的灵魂石板解析能力。主角等级越高，能获取的信息越精确（但上限是“能看到同阶及低一阶敌人的完整面板”）。
  2. 无法通过金手指跳级——主角必须自己从1级打到20级。
- **坐标系意义保留声明**：**金手指不取消坐标系本身的意义——主角仍然必须通过杀戮获取经验升级，无法跳阶，无法无视等级差；数据之眼只是让他打得更准、效率更高，但没法让他“不杀怪就升级”。**

### 代价
- **单次代价**：每次激活数据之眼（≥5秒）消耗5%~10%精神力。如果精神力见底，主角会呕吐、视线重影、思维迟缓。
- **累积代价**：过度使用→精神力上限永久下降（主角的“灵魂石板”会出现裂纹，影响技能学习速度）。为了恢复上限，需要消耗珍贵的“灵魂净水”，而这种东西只有领主级以上势力才能稳定产出。
- **代价来源对齐**：与骨架.md组成4（代价结构）的通用代价“灵魂磨损”一致——金手指的代价只是放大和加速了这个磨损过程。

### 与引擎关系
- **金手指在引擎中的定位**：组成2（运转机制）的加速器。
- **引擎闭环是否改变**：不改变——金手指只加速主角在“输入”和“转化”环节的效率；底层闭环“猎杀→汲取灵魂→强化面板→猎杀更强者”没有变。
